This sample offline HTML is for displaying quick Help after pausing a game.

Overview

This sample is for offline HTML to display quick Help after pausing a game. This sample demonstrates the following features.

  • Using an application screen capture for the background
  • Passing the application state to the offline HTML using a URL parameter
  • Disabling the zoom feature
  • Independently controlling spatial navigation

File Structure

This sample offline HTML is located in Samples/Sources/Applications/WebDemo/html_document/sample/quick_help.htdocs.

Required Environment

This sample is only supported in the NX development hardware environment.

How to Use

Application operations are as follows.

Input Operation
B Button Returns to the menu.
+/- Button Starts the offline web applet, assuming the game is paused, and displays the offline HTML.

Offline HTML operations are as follows.

Input Operation
L Stick or the directional buttons Moves the spatial navigation.
Touch Selects the Help display target.
B Button, +/- Button Exits the offline web applet and returns to the application.

Cautions

Nothing specific.

Execution Procedure

Build the WebDemo sample program and then run it. From the menu, select this sample.

Description

This sample has the following flow.

  • Transitions to the application after selecting this sample from the menu.
  • Displays an image during gameplay in the application.
  • Press the +/- Button to start the offline web applet, assuming the game is paused, and display the offline HTML.
  • After offline HTML display ends, the application once again displays an image during gameplay.

Using an Application Screen Capture for the Background

Specifies OfflineBackgroundKind_ApplicationCapture for the nn::web::ShowOfflineHtmlPageArg::SetBackgroundKind() function.

Passing the Application State to the Offline HTML Using a URL Parameter

When starting the offline web applet, data is passed during the game using a URL parameter. It passes the categories, types, positions, and sizes of targets that exist on the screen when the game is paused. The offline HTML uses the passes parameters to create a page.

Disabling the Zoom Feature

Uses the viewport attribute from the meta element to disable the zoom feature.

Independently Controlling Spatial Navigation

Uses a keydown event to detect when the controller's directional button was pressed. Call preventDefault() to continually disable the default spatial navigation controls, to search for elements in the pressed direction that can be focus targets, and to move spatial navigation to the next element to take focus.
