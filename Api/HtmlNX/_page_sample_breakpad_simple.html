Short guide to the sample program that uses the breakpad library to get debugging information from an NSS file.

Overview

A sample program for the breakpad library, which gets the function names associated with specified memory addresses.

File Structure

This sample program is located in Samples/Sources/Applications/BreakpadSimple/ .

Required Environment

You must link your code against libbreakpad.a to use this feature. The libbreakpad.a static library is bundled with the Breakpad package and does not need to be obtained separately through the Nintendo Dev Interface.

Using the Demo

Nothing specific.

Important Information

Because the NSS file must be included in the ROM, you can use this library only for development. Do not use it in the retail product.

Execution Procedure

Copy the BreakpadSimple.nss file that was generated when building the sample program under the dataSrc folder, rebuild it, and run it.

Description

This sample program has the following flow.

  • The program's NSS file located below the dataSrc directory is loaded.
  • Display the information in debug_info in the NSS file.
  • Use debug_info to look up the function names associated with the backtrace addresses returned by nn::diag::GetBacktrace.
