This documentation describes the simple graphics sample program.

Overview

This sample shows how to work with shader and texture graphics using the gfx library.

File Structure

This sample program is located in Samples/Sources/Applications/GfxSimple/ .

Required Environment

You must be able to use the screen display.

How to Use

Nothing specific.

Important Information

Nothing specific.

Execution Procedure

Build the sample program and then run it.

Description

This sample program loads shader resources and texture resources and then renders a simple polygon while animating it using shader variations. The resources that will be used are converted from a pre-build event before the build. A shader converter and a texture converter are used to convert the resource.

This sample program has the following flow.

  • Get the display and initialize the layers.
  • Initialize the device.
  • Initialize a memory pool.
  • Initialize the swap chain.
  • Initialize the queue.
  • Initialize the command buffer.
  • Initialize the viewport.
  • Initialize the rasterizer state.
  • Initialize the blend state.
  • Initialize the depth stencil state.
  • Initialize the vertex state.
  • Initialize the vertex buffer.
  • Initialize the index buffer.
  • Initialize the constant buffer.
  • Initialize the sampler.
  • Initialize the descriptor pool.
  • Initialize the texture resources.
  • Initialize the shader resources.
  • Begin the loop.
  • Select a shader variation.
  • Create the command list.
  • Execute the command list.
  • Display the results on the screen.
  • Return to the start of the loop.
  • Free all objects used.
