◆ FlushMappedRange()

void nvn::Buffer::FlushMappedRange ( ptrdiff_t  offset,
size_t  size 
) const

Flush previous CPU writes to a range of a buffer to be visible to the GPU.

Note
On NX, calling nn::os::FlushDataCache on an equivalent memory range will have the same effect as calling this function. The flush is performed with 64-byte granularity, matching the size of a CPU cache line. The GPU cache line size is 128 bytes, so there may be interactions with nearby memory ranges that share a cache line with the flushed range. If two non-coherent memory ranges are separately managed by the CPU and the GPU, it is recommended that they be allocated from distinct 128-byte aligned ranges. See 3.2. CPU Access and Coherency in the NVN Programming Guide for details and recommendations.
Parameters
offsetOffset (in bytes) of the first byte of the buffer range to flush.
sizeSize (in bytes) of the buffer range to flush.
