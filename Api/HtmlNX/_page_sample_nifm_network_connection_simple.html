A sample program that demonstrates network connection requests and socket communications.

Overview

First, the NIFM API is used to make a request to the system to use the network connection. If the result of the request is that the network connection is available, the program conducts socket communications.

File Structure

This sample program is located in Samples/Sources/Applications/NifmNetworkConnectionSimple/ .

Required Environment

Requires an environment where network connections are possible. In addition, the appropriate network connection settings must be imported ahead of time, using the settings manager. Sample network connection settings files (networkSettings_*.txt) are located in Samples/Sources/Applications/NifmNetworkConnectionSimple/ . For more information, see the nifm Library Registering Network Connection Settings documentation.

How to Use

Nothing specific.

Important Information

The steps to take if the connection to the network or communication fails have not been determined.

Execution Procedure

Build the sample program and then run it.

135  // The NIFM library must be initialized before the NIFM API can be used.
136  nn::nifm::Initialize();

138  while (NN_STATIC_CONDITION(1))
139  {
140  // Submit a request to use network connection before communication processing,
141  // and confirm availability of the network connection.
142 #if 1
143  // If blocking APIs are being used:
144  nn::nifm::SubmitNetworkRequestAndWait();
145 #else
146  // If non-blocking APIs are being used and if checking for completion by polling:
147  nn::nifm::SubmitNetworkRequest();

152  }
153 #endif

155  if (nn::nifm::IsNetworkAvailable())
156  {
157  Printf("Network is available.\n");
158  break;
159  }
160 
161  // Handle use requests that result in failure.
162  if (nn::nifm::HandleNetworkRequestErrorResult())
163  {
164  // Resubmitting the use request may result in it being accepted.
165  Printf("Network is not available. SubmitNetworkRequest again.\n");
166  continue;
167  }
168 
169  Printf("Network is not available.\n");
170  break;
171  }
172 
173  // Perform the communications process if a network connection is available.
174  if (nn::nifm::IsNetworkAvailable())
175  {
176  // Communication processing.
177  bool isSuccessful = ExecuteTcpIpCommunication();
178 
179  if (!isSuccessful)
180  {
181  // If the process fails because the network connection failed, display a message about that failure.
182  if (nn::nifm::IsNetworkAvailable())
183  {
184  Printf("Communication failed.\n");
185  }
186  else
187  {
188  Printf("Lost network.\n");
189 
190  // This function may be called again to display the reason for disconnection (not required).
191  nn::nifm::HandleNetworkRequestErrorResult();
192  }
193  }
194  }
195 
196 
197  // Withdraw a request for a network connection if it is no longer needed.
198  // If the network connection is lost for unexpected reasons,
199  // you do not need to explicitly withdraw the request,
200  // but there is no problem if you do.
201  nn::nifm::CancelNetworkRequest();
202  NN_ASSERT(!nn::nifm::IsNetworkAvailable());
203 }

Sample Program Description

This sample program has the following flow.

  • Initialize NIFM.
  • Make a request to the system to use the network connection and determine whether it is available before conducting communication.
  • Perform HTTP communication with example.com and output the obtained content and results to a log.

The following file shows the execution results of this sample program. The Release version of the sample does not display the Curl output (the received content). The Curl output (received content) will also not be included in development hardware output.
