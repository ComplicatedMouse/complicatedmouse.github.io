Detailed Description

Library to analyze nxdmp files.

Details
Core Dump Report (CDR) is a utility for reading nxdmp files. It can be used to extract basic information from a nxdmp file including: the exception code and description, the loaded modules, the threads list, and the call stacks for each thread.

◆ AddSymbolStorePath()

CDR_DLL_DECL Error nn::cdr::AddSymbolStorePath ( const char *  pPath)

Adds the specified Symbol Store path to the list of known symbol paths.

Parameters
[in]pPathPath to symbol store.

Details
NX Symbol Store is a tool for archiving debug symbols. AddSymbolStorePath() can be used to link archived debug symbols to Core Dump Report. AddSymbolStorePath() searches through the specified path and symbol store cache and adds any symbols to the list of known symbols. AddSymbolStorePath can be called before or after Open. Cached symbols can be used for multiple nxdmp files. Use ClearSymbolPaths to clear this cache. See NXSymStore document for details about NX Symbol Store.

Returns
nn::cdr::Error_Ok if successful, else an error code.

◆ GetScreenshotData()

CDR_DLL_DECL Error nn::cdr::GetScreenshotData ( void *  pOutBuffer,
size_t  bufferSize 
)

Get screenshot image data from nxdmp.

Parameters
[out]pOutBufferThe buffer to get image data.
[in]bufferSizeThe buffer size in byte.

Details
Get the pixel array information of screenshot in nxdmp. Pixel arrangement information is copied to pOutBuffer in the following sequence when pixel information is expressed by color (RGB) information (1 byte) + (x, y) coordinates.
[ B(0,0), G(0,0), R(0,0), B(1,0), G(1,0), R(1,0) B(x,y), G(x,y), R(x,y) ]
The required buffer size can be get with (width * height * 3) obtained by nn::tm::GetScreenshotSize. If bufferSize does not match the screen size, an error will be returned.

Returns
nn::cdr::Error_Ok if successful, else an error code.
Examples
CdrSimple/CdrSimple.cpp.
