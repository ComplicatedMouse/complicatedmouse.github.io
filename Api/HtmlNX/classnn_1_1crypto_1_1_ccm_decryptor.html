Detailed Description

template<typename BlockCipher>
class nn::crypto::CcmDecryptor< BlockCipher >

Class for running authenticated decryption in CBC mode.

Template Parameters
BlockCipherThe implementation class for the block cipher.

Details
CCM uses the cipher mode with verification stipulated by NIST SP 800-38C. Implementations by this class use the methods described in Appendix A of that documentation for the input value format and for the functions for generating counter values.

In CCM mode, a MAC (message authentication code) is computed at the same time as the encryption or decryption operation. Data alteration can be detected by checking whether the same MAC was output by the encryption and decryption processes.

This class is a template class that takes the block cipher implementation class as a template parameter. CCM mode requires a block cipher with a block length of 128 bits. Use Aes128CcmDecryptor or a similar class in your actual code.

The implementation of the block cipher class specified in the template must satisfy the following conditions. For more information including about the parameters of each function, see AesEncryptor.

  1. Has a BlockSize constant that defines the size of the block to process. In CCM mode, you can only use a value of 16 here.
  2. Has a EncryptBlock function that encrypts a single block.

Note that CCM mode decryption does not use decryption of the block cipher, but encryption.

This class has the following states. The state immediately after instantiation is uninitialized.

  • Uninitialized state.
  • Ready to process additional authentication data.
  • Ready for decryption.
  • Data entry complete.
  • Complete.

Initializes the process for CCM mode authenticated decryption.

Parameters
[in]pBlockCipherInstance of the block cipher implementation class.
[in]pNoncePointer to the nonce data.
[in]nonceSizeSize in bytes of the nonce data.
[in]aadSizeSize in bytes of the additional authentication data.
[in]dataSizeSize in bytes of the data to decrypt.
[in]macSizeMessage authentication code size in bytes.
Precondition
  • The EncryptBlock member function of the instance specified in pBlockCipher can be called.
  • nonceSize >= 7 && nonceSize <= 13.
  • macSize >= 4 && macSize <= 16 && macSize % 2 == 0.
  • aadSize >= 0.
  • dataSize >= 0.
  • (nonceSize == 7) ? dataSize <= INT64_MAX : dataSize < (1LL << ((15 - nonceSize) * 8)).
Postcondition
  • If aadSize is 0, the instance is ready for decryption.
  • If aadSize is not 0, the instance is ready to process additional authentication data.

Details
Specify an instance of the class that implements the block cipher to use for encryption in pBlockCipher. The instance must be initialized with the required key, and other settings, so that EncryptBlock is ready to be called.

Use the same nonce, additional authentication data, ciphertext data, and MAC size that were used in the encryption process.

The instance specified in pBlockCipher is not copied. Do not perform any operations on pBlockCipher until the processing sequence is complete.

An internal copy of the nonce data is created.

Inputs additional authentication data.

Parameters
[in]pAadPointer to the buffer storing the additional authentication data.
[in]aadSizeSize in bytes of the additional authentication data to input.
Precondition
  • The instance is ready to process additional authentication data.
  • The total size of the data input by UpdateAad does not exceed the specified aadSize.
Postcondition
  • If a nonzero dataSize was specified in Initialize(), the instance becomes ready for decryption when the size of the additional authentication data reaches aadSize.
  • If a dataSize of 0 was specified in Initialize(), the instance enters a data entry complete state when the size of the additional authentication data reaches aadSize.

Details
The additional authentication data is only part of a MAC and not subject to encryption. The specified amount of data is read from pSrc to calculate the MAC.

This function can be split among multiple calls. Note, however, that the entered data is treated as a single contiguous set of data, so the order of input is significant.

All additional authentication data must be added before starting the Update() process.

Runs authenticated decryption in CCM mode.

Parameters
[out]pDstPointer to the buffer that receives decrypted data.
[in]dstSizeSize of the buffer pointed to by pDst.
[in]pSrcPointer to the buffer storing the data to decrypt.
[in]srcSizeSize in bytes of the data to decrypt.
Returns
Returns the actual number of bytes that were decrypted.
Precondition
  • The instance is ready for decryption.
  • The total size of the data input by Update does not exceed the specified dataSize.
  • dstSize >= srcSize
  • The region specified by pDst,dstSize does not overlap with the region specified by pSrc,srcSize.
Postcondition
  • The instance enters the data entry complete state when the size of the additional data for decryption reaches the dataSize specified in Initialize().
  • The decrypted result is written to pDst.

Details
Because decryption takes place byte-by-byte in CCM mode, the dstSize required for each call must be the same as srcSize.

The buffer required for the total dstSize for all calls is also the same as the total srcSize.

Gets the final MAC.

Parameters
[out]pMacPointer to the buffer storing the MAC.
[in]macSizeSize in bytes of the buffer specified in pMac.
Precondition
  • The instance has a state of data entry complete or complete.
  • The macSize is at least as large as the macSize specified in Initialize().
Postcondition
  • The MAC is written to pMac.
  • The instance is in a completed state.

Details
This function gets a MAC computed from the additional authentication data and ciphertext data entered up to the point when the function is called.

The calculated MAC is sent along with the ciphertext and additional authentication data for the encryption process. In the decryption process, you can detect whether the ciphertext, or additional authentication data, has been altered by checking if the calculated MAC matches the received MAC.
