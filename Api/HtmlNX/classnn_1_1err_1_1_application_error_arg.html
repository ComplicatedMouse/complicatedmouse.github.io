Detailed Description

Class that builds the information for displaying application-specific errors.

Details
After configuring the required information, you can also display errors by passing the information to ShowApplicationError. The data size of this class is large at ApplicationErrorArg::DataSize bytes, so take note of the stack size of the thread when placing it on the stack.

NX Platform Only:
The information set in this class is displayed by the error viewer applet via ShowApplicationError. When you set the information, refer to System Features Guide > Library Applets > Error Viewer Applet > Feature for Displaying Application-Specific Errors and also note the following points.
  • Messages are encoded in UTF-8 without BOM.
  • LF is used as the newline code in the messages.
  • The maximum length for messages in dialog boxes is 1024 characters, regardless of single-byte or double-byte.
    • Messages are not displayed if they exceed 1024 characters.

nn::err::ApplicationErrorArg::ApplicationErrorArg ( ApplicationErrorCodeNumber  number,
const char *  dialogViewMessage,
const char *  fullScreenViewMessage,
const nn::settings::LanguageCode &  languageCode 
)

Configures the error information and creates the class.

Parameters
[in]numberThe error code number.
[in]dialogViewMessageThe message for display in the dialog box (UTF-8). The maximum length is MessageLengthMax including the NULL terminator.
[in]fullScreenViewMessageThe message for full-screen display (UTF-8). The maximum length is MessageLengthMax including the NULL terminator.
[in]languageCodeThe language to use for buttons and other UI components.
Precondition
  • number <= 9999.
  • dialogViewMessage != nullptr.
  • fullScreenViewMessage != nullptr.
  • Included in the language codes (obtainable using settings::GetAvailableLanguageCodes) that languageCode can use for the language setting.

Details
When the error is not displayed in a dialog box, it sets an empty string. Similarly, when the error is not displayed on the full screen, it sets an empty string. The message is copied to the data region of this class, the caller of the buffer used in input does not need to retain it.

void nn::err::ApplicationErrorArg::SetLanguageCode ( const nn::settings::LanguageCode &  languageCode)

The language to use for the message that is displayed in the UI parts, such as the buttons during error display.

Parameters
[in]languageCodeThe language to use for the message that is displayed in the UI parts, such as the buttons during error display.
Precondition
  • Included in the language codes (obtainable using settings::GetAvailableLanguageCodes) that languageCode can use for the language setting.

Details
The specified language is used in the button to close the error display, and the button to switch from error display in a dialog box to full-screen display (when display in a dialog box and full-screen display are both supported), and in the displayed message.
