◆ enableAdditionalEffectBus

bool nn::atk::SoundSystem::SoundSystemParam::enableAdditionalEffectBus

The use of internal features is prohibited.

Feature for evaluating the availability of the additional effect bus after adding a submix.

Details
In addition to the existing 6 channel x 4 bus configuration, a 2 channel x 3 bus configuration for submixes was added. The default is false. When adding effects to the additional effects bus using nn::atk::SoundSystem::AppendEffect, use no more than two channels, and use only nn::atk::ChannelIndex_FrontLeft and nn::atk::ChannelIndex_FrontRight for the effect channel settings.

Note that the user submix setting that can be configured using enableCustomSubMix are not applied when you use this setting.

◆ enableAdditionalSubMix

bool nn::atk::SoundSystem::SoundSystemParam::enableAdditionalSubMix

The use of internal features is prohibited.

A feature for determining the availability of the additional submix.

Details
Mixes the regular submix, the additional submix, and the final mix, in that order. You can add nn::atk::EffectAux to the additional submix. Use the same operations you would when adding an effect to a regular submix. Add using nn::atk::SoundSystem::AppendEffectToAdditionalSubMix, delete using nn::atk::SoundSystem::ClearEffectFromAdditionalSubMix or nn::atk::SoundSystem::RemoveEffectFromAdditionalSubMix, and confirm the deletion using nn::atk::SoundSystem::IsClearEffectFromAdditionalSubMixFinished.

Note that the user submix setting that can be configured using enableCustomSubMix are not applied when you use this setting.

◆ enableTaskThread

bool nn::atk::SoundSystem::SoundSystemParam::enableTaskThread

The use of internal features is prohibited.

Flag indicating whether to use the task thread. The default value is true.

Details
When making this value false, make sure that the playback methods needed for the stream sound and player heap task threads are not in use.

When this value is set to false, task threads can no longer be created, and a value of 0 can be set for taskThreadStackSize, and taskThreadCommandBufferSize. Attempts at playback that use stream sounds or player heaps will cause an assertion failure.

◆ enableEffect

bool nn::atk::SoundSystem::SoundSystemParam::enableEffect

The use of internal features is prohibited.

Flag indicating whether to use effects.

Details
The default value is true. When setting this value to false, make sure that effects will not be used.

When this value is false, the buffer required for using effects is no longer needed. The size of the buffer needed for sound system initialization can be reduced. Attempting to add or remove effects to or from the sound system causes an assertion failure.

◆ enableSubMix

bool nn::atk::SoundSystem::SoundSystemParam::enableSubMix

The use of internal features is prohibited.

Flag indicating whether to use SubMix.

Details
The default value is true. When setting this value to false, make sure that effects are not added to SubMix.

If this value is false, the voice output will not go through SubMix, and it is possible to reduce the load on the audio renderer. An assertion failure occurs when attempting to add or remove effects from SubMix.

◆ enableCompatibleDownMixSetting

bool nn::atk::SoundSystem::SoundSystemParam::enableCompatibleDownMixSetting

This flag specifies whether to enable downmixing that is compatible with the NW4F nw::snd library.

The default is false.

Details
If this value is set to true, the downmixing that is done when the atk library's number of output channels differs from the number of output channels of the actual audio device will be changed to a downmixing process that is compatible with the NW4F nw::snd library. Note that enabling NW4F nw::snd compatible processing puts a processing load on the application because the downmixing is performed by the audio renderer managed inside the atk library instead of by the system's default downmixing process.

◆ enableCompatibleBusVolume

bool nn::atk::SoundSystem::SoundSystemParam::enableCompatibleBusVolume

The use of internal features is prohibited.

This flag specifies whether to enable NW4F nw::snd-compatible effect bus volume calculations. The default is false.

Details
Specify true to use a process compatible with NW4F nw::snd when setting the effect send volume for buses not being used for the effect.

Normally, if an effect send volume is specified for buses that are not being used for effects, that value is mixed with the main send and applied to the final output. But when this parameter is set to true, the volume is muted on buses that are not being used for effects, and even if an effect send volume is specified, it is no longer applied to the final output.

◆ enableCompatiblePanCurve

bool nn::atk::SoundSystem::SoundSystemParam::enableCompatiblePanCurve

Flag that indicates whether to enable NW4F nw::snd compatible pan curves.

The default is false.

Details
If you set the value to true, the pan curve is similar to the pan curve for NW4F nw::snd. This setting has the following two effects.

  • When stream sounds are played, the pan curve is fixed to nn::atk::PanCurve_Sqrt, and the pan mode is fixed to nn::atk::PanMode_Dual.
  • If the mode has been set to nn::atk::OutputMode_Surround by the nn::atk::SoundSystem::SetOutputMode function, the sound volume decreases when the pan value is -0.5f or lower and 0.5f and higher.
    • Note that even if you have set the pan mode to dual pan and are playing stereo sound, the volume changes in this same way because the pan value meets these conditions due to internal processing. (A pan of -1.0f and 1.0f is specified for the left and right.) For more information about pan curves, see Pan Operations.

◆ subMixTotalChannelCount

int nn::atk::SoundSystem::SoundSystemParam::subMixTotalChannelCount

Sets the maximum number of channels that can be used simultaneously when using a user-defined submix structure.

Details
The total number of channels used by a single submix is represented by (the number of submix buses) x (the number of submix channels). The value that is set is the sum total of the channels for all submixes in use. When a user-defined submix structure is being used, the value is set to 0 by default.
