Detailed Description

The class for profanity filtering.

Details
Use this class to reference the profanity list installed in the system and check whether the strings entered by the user contain profanity. For more information about conducting profanity filtering, see the Ngc Library documentation.

Profane words for each language have been registered for testing purposes. Use these test strings to confirm that the NGC library has been incorporated correctly in your application. For more information, see nn::ngc::ProfanityFilterPatternList.

This class is not thread-safe. Operations are not ensured if the instance of this class is accessed from multiple threads.

Examples
NgcSimple/NgcSimple.cpp.

◆ CheckProfanityWords() [1/3]

virtual nn::Result nn::ngc::ProfanityFilter::CheckProfanityWords ( Bit32 *  pCheckResults,
Bit32  patterns,
const char **  ppWords,
size_t  wordCount 
)
virtual

Uses the pattern list created automatically for the specified language setting and the specified pattern list to check phrases strung together from a number of specified words to determine whether there would be a problem if they are shown on the screen.

Parameters
[out]pCheckResultsThe buffer for storing the result of the check. It must be an array with the same number of entries specified by wordCount.
[in]patternsA pattern list that adds to the list created automatically for the specified language setting. Specify the values for each variation for nn::ngc::ProfanityFilterPatternList. To set multiple values, specify a logical OR of the values. This pattern specification is ignored if you have specified to check using the pattern list created automatically for the specified language setting. Specify 0 if you are not adding a pattern list.
[in]ppWordsThe array of null-terminated strings to check. Up to nn::ngc::WordLengthMax characters, including the null character, can be checked. If a string exceeds this maximum, the portion exceeding the maximum is not checked. Store them using UTF-8 character encoding.
[in]wordCountThe number of strings to check. The maximum is nn::ngc::WordCountMax.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The checking succeeded.
  • nn::ngc::ResultNotInitialized Not initialized.
  • nn::ngc::ResultInvalidPointer An invalid pointer was specified.
  • nn::ngc::ResultInvalidSize An invalid number of strings was specified.
  • nn::ngc::ResultInvalidCharacter The specified string is invalid as a UTF-8 string.

Details
This function is designed for checking screen names and other single words. To process text that has been entered by the user that consists of two or more words, we recommend using the ProfanityFilter::MaskProfanityWordsInText function.

If you set checkDesiredLanguage to true when initializing the ProfanityFilter class, the pattern list for the highest priority language that the application supports, returned by nn::oe::GetDesiredLanguage, is used in addition to the pattern list used automatically based on the system language settings. For applications that do not use nn::oe::GetDesiredLanguage, we recommend adding an independent pattern list corresponding to the application's supported languages.

The processing by this function can be time-consuming (it blocks). The processing time can vary depending on the system version. The frame rate can decline if this function is called from the main thread, so we recommend running it from another thread. This function can simultaneously check as many as nn::ngc::WordCountMax individual strings. You can complete the task faster by checking multiple strings at the same time, rather than one at a time.

The results of the check on each string are stored as an array in pCheckResults. Perform a logical AND with the values of the patterns in nn::ngc::ProfanityFilterPatternList to check the strings. The value becomes 0 if no problem is determined with the list specified by patterns.

Strings that include the at sign (@), which could indicate the string is an email address, are always identified as problem phrases. On the other hand, strings that contain many numbers, which could indicate a phone number, are not identified as problem phrases. To check these kinds of strings, use the nn::ngc::CountNumbers function.

The pattern list dynamically references objects saved outside of the ngc library. As a result, the checked content can vary depending on the version of the SDK or on development hardware that was initialized with that version. Do not write code that assumes that results for specific strings will remain constant, from the perspective of compatibility.

Checks the specified words using the pattern list created automatically for the specified language setting and the specified pattern list to determine whether there would be a problem if they are shown as text on the screen.

Parameters
[out]pProfanityWordCountThe pointer for getting how many times illegal strings showed up in the text. If this information is not necessary, specify nullptr.
[in,out]pTextThe pointer to the text to check. Any illegal strings in the text are replaced with asterisks. For this reason, the application must specify a writable buffer. The function can check a maximum of nn::ngc::TextLengthMax characters, including the null character. If a string longer than this maximum is passed to the function, it will not check anything beyond the maximum length. Store them using UTF-8 character encoding.
[in]patternsA pattern list that adds to the list created automatically for the specified language setting. Specify the values for each variation for nn::ngc::ProfanityFilterPatternList. To set multiple values, specify a logical OR of the values. This pattern specification is ignored if you have specified to check using the pattern list created automatically for the specified language setting. Specify 0 if you are not adding a pattern list.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The checking succeeded.
  • nn::ngc::ResultNotInitialized Not initialized.
  • nn::ngc::ResultInvalidPointer An invalid pointer was specified.
  • nn::ngc::ResultInvalidSize An invalid number of strings was specified.
  • nn::ngc::ResultInvalidCharacter The specified string is invalid as a UTF-8 string.

Details
This function is designed for checking text that includes two or more words. To check screen names and other short words, we recommend using the ProfanityFilter::CheckProfanityWords function.

This function checks whether there are any illegal strings among the specified strings and masks those illegal strings with asterisks.

This function treats the following characters as word separators.

  • Carriage return, line feed, next line, line break, paragraph break, tab, vertical tab, form feed, single-byte space and single-byte symbols .,[!" @#@$@%@&'()*+/:;@<=@>? @@@\^_`{|}~-] double-byte space and double-byte symbols ""'

If word separators are included in illegal strings, they are still judged to be illegal strings.

If you set checkDesiredLanguage to true when initializing the ProfanityFilter class, the pattern list for the highest priority language that the application supports, returned by nn::oe::GetDesiredLanguage, is used in addition to the pattern list used automatically based on the system language settings. For applications that do not use nn::oe::GetDesiredLanguage, we recommend adding an independent pattern list corresponding to the application's supported languages.

The processing by this function can be time-consuming (it blocks). The processing time can vary depending on the system version and the length of the strings passed to the function. The frame rate can decline if this function is called from the main thread, so we recommend running it from another thread.

The pattern list dynamically references objects saved outside of the ngc library. As a result, the checked content can vary depending on the version of the SDK or on development hardware that was initialized with that version. Do not write code that assumes that results for specific strings will remain constant, from the perspective of compatibility.

◆ SetMaskMode()

void nn::ngc::ProfanityFilter::SetMaskMode ( MaskMode  mode)

Checks text and sets what action to take when problem phrases are discovered.

Parameters
[in]modeSpecify MaskMode_OverWrite to replace all characters in the discovered problem phrase with asterisks. (This is the default.) Specify MaskMode_ReplaceByOneCharacter to replace the discovered problem phrase with a single asterisk.

Details
In the default mode, where all characters in the discovered problem phrase are replaced with asterisks, there is a chance that the processed text with the asterisks will spill outside of the frame displayed on the screen. For example, if you are using a proportional font, the i and * characters have different widths, so there is a chance that the overwritten text will extend beyond the screen.

If you specify false for this function, the discovered problem phrase is replaced with a single asterisk and you do not need to worry about the user interface.

This setting is enabled only when text is checked by the ProfanityFilter::MaskProfanityWordsInText function. It has no effect on behavior when individual words are checked by the ProfanityFilter::CheckProfanityWords function.

The specified setting is reset to the default (MaskMode_OverWrite) when the ProfanityFilter::Initialize function is called.
