Detailed Description

Class that expresses the state of previously called asynchronous processes.

Details
The following are performed for previously called asynchronous processes.

  • Getting events that take the notification of asynchronous process completion.
  • Getting the state of asynchronous process completion, including the incomplete state.
  • Getting results when asynchronous processes are complete.
  • Requesting a cancellation when an asynchronous process is not complete.

AsyncContext objects have the following states.

  • Disabled.
    • The object is not associated with any asynchronous processes.
    • The only call possible in this state is a destructor.
  • Enabled.
    • This object is associated with a specific asynchronous process.
    • The previously mentioned actions are possible for associated asynchronous processes when in this state.

An enabled object is available by setting the AsyncContent object as the parameters for the respective functions. Note that those functions will destroy any previous content for AsyncContent objects set as parameters. As a result, those functions fail if the set AsyncContent objects are enabled and the asynchronous processes are running.

Gets the event signaled by the asynchronous processing completion notification. This event can be used as long as *this is enabled. Consequently, you must not access this event after *this is destroyed.

◆ Cancel()

Result nn::account::AsyncContext::Cancel ( )

Requests cancellation for asynchronous processes.

Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The process succeeded.
Precondition
  • *this is enabled.

Details
Requests cancellation for asynchronous processes. This function returns without waiting for cancellation completion.

After requesting cancellation, the asynchronous process completes as soon as it can. Completion notification is signaled. This function does nothing when the asynchronous process is already complete.

◆ GetResult()

Result nn::account::AsyncContext::GetResult ( )

Gets the result of asynchronous processing.

Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The process succeeded.
  • nn::account::ResultCancelled Indicates that the process was canceled by calling the Cancel() function. Do not handle this result as an error.
  • nn::account::ResultNetworkServiceAccountUnavailable (This result is only returned in cases delineated in the function reference for the function that was used to get *this.) Determined that a network service account could not be used during asynchronous processing. Make the network service account available by running EnsureNetworkServiceAccountAvailable(), which allows you to resolve this error.
  • nn::account::ResultNetworkCommunicationError (This result is only returned in cases where Internet communication is possible, as delineated in the function reference for the function that was used to get *this.) A problem occurred for network communication during asynchronous processing. Make every effort not to ignore this error when it occurs. Use the nn::err::ShowError() function to notify the user.
  • nn::ssl::ResultSslService (This result is only returned in cases where Internet communication is possible, as delineated in the function reference for the function that was used to get *this.) Indicates that a problem with network communication occurred during the process. Make every effort not to ignore this error when it occurs. Use the nn::err::ShowError() function to notify the user.
Precondition
  • *this is enabled.
  • The asynchronous process is completed.

Details
Gets the result of asynchronous processing. Returned results are defined for each asynchronous process, respectively. For more information, see the API Reference for the function used to get *this.
