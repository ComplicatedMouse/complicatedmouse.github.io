Describes the sample program for the audio CPU renderer.

Overview

This sample uses the audio CPU renderer and performs audio playback.

File Structure

This sample program is located in Samples/Sources/Applications/AudioCpuRenderer/ .

Required Environment

You must be able to use audio output.

How to Use

When the sample program is executed, the sine wave will be generated automatically. Use input from the keyboard or DebugPad to manipulate the sound effects and sine wave during playback.

Input Operation
A Button Plays SampleSe0 once.
[B] Plays SampleSe1 once.
[X] Plays SampleSe2 once.
[Y] Plays SampleSe3 once.
Left Stick Raise and lower the volume of the sine wave.
Right Stick Raises and lowers the pitch of the sine wave.
START or SPACEBAR Exits the program.

Important Information

This sample can only run in an environment with an audio output device that supports a sample rate of 32,000 Hz or 48,000 Hz.

Execution Procedure

Build the sample program and then run it.

Description

This sample program performs audio rendering on the CPU that runs the animation. The audio rendered results are extracted using CircularBufferSink and passed to and played back by the audio output.

Aside from performing audio rendering on the CPU that runs the animation, this sample also uses features described in the AudioRenderer sample, AudioSink sample, and AudioOut sample. For a basic description of the audio renderer, see those samples.

This sample program has the following flow.

  • Initialize the file system.
  • Initialize and prepare the audio renderer and audio output.
  • Initialize and prepare the audio output.
  • Initialize audio rendering.
  • Change parameters and implement state acquisition.
  • Execute audio rendering.
  • Transfer the rendering results to the audio output.
  • End rendering.
  • Close the audio renderer and audio output.
  • Free the input waveform.

First, initialize the file system. Use a resource file that is converted during build. Appropriately change the location of the resource directory.

Initialize the audio renderer. In this sample, the CPU (nn::audio::AudioRendererRenderingDevice_Cpu) is specified as the device that performs rendering and nn::audio::AudioRendererExecutionMode_ManualExecution is specified for the manual execution mode of the rendering mode. The audio renderer that is initialized by specifying these two options can call the nn::audio::ExecuteAudioRendererRendering() function to perform audio rendering on the CPU that runs the animation. The rendering processing is executed internally by the function that is called. There is a restriction where DeviceSink cannot be used for audio renderers that enable the preceding features. To play the rendering results, you must use CircularBufferSink to get the rendering results and transfer them to the audio output or a different audio renderer. This sample uses the audio output to play the rendering results.

Initialize the audio output. The audio output was prepared in the audio renderer to omit some processing (such as down-mix processing). We recommend initializing it with the same number of channels as CircularBufferSink.

The main loop performs update processing for the audio renderer and audio rendering processing, and writes to the audio output. After calling nn::audio::ExecuteAudioRendererRendering() and performing the audio rendering processing, convert the results from the CircularBufferSink output block interleaved results to interleaved format, and then write them to the audio output.

When a request to exit the program is made, this repeated process is exited and playback stops, the audio renderer and audio output are closed, and the memory is freed.
