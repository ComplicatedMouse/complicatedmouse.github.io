nn::Result nn::nfp::StartNicknameAndOwnerSettings ( DeviceHandle *  pOutDeviceHandle,
bool *  pOutIsRegistered,
RegisterInfo *  pOutRegisterInfo,

const TagInfo &  tagInfo,
const RegisterInfo &  registerInfo 
)

Starts the amiibo Settings for the nickname and owner settings.

Parameters
[out]pOutDeviceHandleSpecifies a pointer to a buffer for storing the NFC device handle that updated the tag.
[out]pOutIsRegisteredSpecifies a pointer to the buffer that stores the existence of registration data after updating.
[out]pOutRegisterInfoSpecifies a pointer to the buffer that stores the existence of registration data after updating. Only reference when there is registration data in pOutIsRegistered.
[in]startParamThe common parameters required when starting amiibo Settings.
[in]tagInfoData that indicates the tag to update.
[in]registerInfoRegistration data when this API was called. Specify the value acquired using GetRegisterInfo.
Precondition
  • GetState() == State_Init
  • pOutDeviceHandle must point to an accessible address.
  • pOutIsRegistered must point to an accessible address.
  • pOutRegisterInfo must point to an accessible address.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess Tag update succeeded.
  • ResultNotUpdated Tag was not updated.

Details
Starts the amiibo Settings for the nickname and owner settings. Control does not return from this API until amiibo Settings complete. There is no valid data for pOutDeviceHandle, pOutIsRegistered, or pOutRegisterInfo when ResultNotUpdated is returned. In such cases, do not reference those pointers. registerInfo is used to reference the nickname and owner data from before the settings. Tags are not reread after amiibo Settings begin. Always specify this when registration data has already been added. Specify RegisterInfo as the parameter instead of this API when registration data has not yet been added. Use StartNicknameAndOwnerSettings(DeviceHandle* pOutDeviceHandle, bool* pOutIsRegistered, RegisterInfo* pOutRegisterInfo, const AmiiboSettingsStartParam& startParam, const TagInfo& tagInfo).

◆ AttachActivateEvent()

nn::Result nn::nfp::AttachActivateEvent ( nn::os::SystemEventType *  pEvent,
const DeviceHandle &  deviceHandle 
)

Sets an event to report the detection of a tag.

Parameters
[in,out]pEventThe event to use to report the detection of a tag.
[in]deviceHandleNFC device handle.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess The event was set successfully.
  • ResultNfcDeviceNotFound Could not find an NFC device in the correct state. You need to call ListDevices() and reacquire the NFC device handles.
  • ResultNfcDisabled NFC functionality is disabled. You will need to retry after enabling the NFC functionality.
Precondition
  • pEvent must point to an accessible address.
  • GetState() == State_Init.
Postcondition
pEvent will be signaled for the specified NFC device when a tag is detected.

Details
Call this function after the NFP library is initialized and before the StartDetection() function starts tag detection for each NFC device. The event set with this function is signaled for the specified NFC device when a tag is detected. This event is converted to a signal even when non-NFP tags are detected.

The event is initialized within the API, so there is no need for the user to initialize it. Events are initialized to auto-clear mode.

Make sure that you call nn::os::DestroySystemEvent() to destroy the event after it is no longer required.

◆ AttachAvailabilityChangeEvent()

void nn::nfp::AttachAvailabilityChangeEvent ( nn::os::SystemEventType *  pEvent)

Set an event to report the change in availability of the NFC device.

Parameters
[in,out]pEventSpecify the event to use in the notification.
Precondition
  • pEvent must point to an accessible address.
  • GetState() == State_Init.
Postcondition
When the availability of the NFC device changes, pEvent will be signaled.

Details
Execute after initializing the NFP library. The set event is signaled by this function at the following times.

  • The NFC device is disconnected.
  • The NFC device is connected.
  • The application in focus has changed.
  • The system woke from sleep mode.
  • The NFC feature was disabled.
  • The NFC feature was enabled.

An efficient way of checking the state of the NFC device is by calling ListDevices() and GetDeviceState() when the event is signaled. The event is initialized within the API, so there is no need for the user to initialize it. Events are initialized to auto-clear mode.

Make sure that you call nn::os::DestroySystemEvent() to destroy the event after it is no longer required.

◆ StartDetection()

nn::Result nn::nfp::StartDetection ( const DeviceHandle &  deviceHandle)

Start the process of tag detection.

Parameters
[in]deviceHandleThe NFC device handle.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess Detection has started.
  • ResultNfcDeviceNotFound Could not find an NFC device in the correct state. You must call ListDevices() and get the NFC device handles again.
  • ResultNfcDisabled NFC functionality is disabled. You will need to retry after enabling the NFC functionality.
  • ResultMaxNfcDeviceActivated You cannot use NFC functionality for the specified NFC device. The maximum number of NFC devices using that NFC functionality has been reached. You must try again after reducing the number of NFC devices using the NFC functionality.
  • ResultConflictFunction You cannot use NFC functionality because a competing feature is in use. You must retry after the use of the competing feature ends.
Precondition
  • GetState() == State_Init.
  • GetDeviceState() == DeviceState_Init || GetDeviceState() == DeviceState_Deactive.
Postcondition
  • GetDeviceState() == DeviceState_Search.

Details
Starts detecting tags for the specified NFC device. Must occur either immediately after initialization of the library or immediately after a detected tag has been lost (removed). (Returns ResultNfcDeviceNotFound when the state is incorrect.) If the event is set to AttachActivateEvent or AttachDeactivateEvent, it is cleared to a nonsignaled state as soon as a signaled state is detected.
