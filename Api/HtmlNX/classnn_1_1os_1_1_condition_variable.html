Detailed Description

Class for handling condition variables.

Details
Condition variable classes are used in conjunction with mutex classes, and make it possible to wait for thread execution until certain conditions have been fulfilled.

The nn::os::ConditionVariable class is a utility class that wraps calls to synchronization control functions related to nn::os::ConditionVariableType objects. The member functions provided by this class are implemented by issuing functions like nn::os::WaitConditionVariable() that take the nn::os::ConditionVariableType type as arguments.

Examples

void nn::os::ConditionVariable::Wait ( nn::os::MutexType &  mutex)

Waits for a condition variable.

Parameters
[in]mutexReference to a MutexType object that is currently locked.

Details
Unlocks the specified mutex and waits for a condition variable atomically. If the condition variable is signaled by Signal() or Broadcast(), the mutex is locked again and returned to the caller.

It is also possible to specify a reference to a nn::os::Mutex class instance directly. In this case, the implicit conversion operator for the Mutex class is applied.

For more information, see nn::os::WaitConditionVariable().

[in]timeoutThe time until timeout.
Returns
Returns the reason why waiting for the condition variable has stopped.

Details
Unlocks the specified mutex and waits for a condition variable atomically. If the condition variable is signaled by Signal() or Broadcast(), the mutex is locked again and returned to the caller.

It is also possible to specify a reference to a nn::os::Mutex class instance directly. In this case, the implicit conversion operator for the Mutex class is applied.

If the amount of time specified by timeout elapses after the current thread starts waiting on the condition variable, a time-out occurs and the thread automatically stops waiting. The mutex is locked again and returned to the caller.

For more information, see nn::os::TimedWaitConditionVariable().
