Detailed Description

Manages buttons in a list.

Use this class to execute state updates and other processes for all buttons at the same time.

Details
With the button design, each button does not manage whether it is being hit. Instead, the ButtonGroup class manages which button is currently being hit. Buttons belonging to the same ButtonGroup simultaneously enter a state where just one button has been hit.

◆ Update()

virtual void nn::ui2d::ButtonGroup::Update ( const nn::util::Float2 *  pPos,
bool  isDown,
bool  isRelease = false 
)
virtual

Updates the state.

Details
Call this member function one time to advance gameplay. The following processes are performed.

  • If IsUpdateHitBox() returns true, this function recalculates the button hit detection area.
  • The function then checks the button currently being hit based on the cursor coordinates retrieved. The button hit is not obtained if IsExcludeDown() is set to true and another button is currently being pressed that also has IsExcludeDown() set to true.
  • Issues an On, Off, or Down action on the button according to the button hit just accepted and the value specified in the isDown parameter.
  • Calls the button Update member function.
Parameters
[in]pPosThe position currently being pointed to by the pointer. Make sure that you pass data in the layout coordinate system. (The center is 0,0, and the upper right represents the positive quadrant.)
[in]isDownWhether the button is pressed. Usually, if enabled, the A Button is pressed on a Wii Remote or the mouse has been clicked, but you can also assign other operations.
[in]isReleaseWhether the button is released. Normally, this is either when the A Button is released on the Wii Remote or when the left button is released on the mouse. The default value is false. If you do not use the drag action performed with a mouse, there is no problem if this value is always false.

◆ SetUpdateHitBox()

void nn::ui2d::ButtonGroup::SetUpdateHitBox ( bool  b)

Determines whether to update button hit detection.

Details
By default, the determination of a button hit is updated within the Update() member function. Specify false with this member function to prevent updating.

If you do so, button hit detection acts strange when the button is animated to grow larger or move position, but you can reduce processing by not performing hit detection.

Make sure that you use this function when the size and position of the button does not change inside the screen.

Also note when setting false that you must call UpdateHitBox for the button at least once at an appropriate location in the code.

Parameters
[in]btrue enables updating for button hit detection.

◆ SetExcludeDown()

void nn::ui2d::ButtonGroup::SetExcludeDown ( bool  b)

Sets whether to execute a process that prevents other buttons from being pressed after one button is pressed.

Details
Although this setting can be made for each individual button, if it is specified as Off for the ButtonGroup, settings at the button level are ignored, and this process is not executed. True is specified by default.

Specify false with this member function to reduce processing when exclusive processing is not needed when a button is pressed.

Parameters
[in]bSpecify true to execute a process that prevents other buttons from being pressed after one button is pressed.

◆ SetHitTestEnabled()

void nn::ui2d::ButtonGroup::SetHitTestEnabled ( bool  b)

Determines whether to execute a hit detection.

Details
Specify false to not perform a hit detection. True is specified by default.

One use case would be when a button hit is not wanted during an animation or when a dialog box appears.

Even if you use this method, the button state is not necessarily forcibly changed. For example, if this method is called when the mouse pointer touches a button, the pointer misses and the OFF animation plays for the button.

Parameters
[in]bSpecify true to execute button hit detection.

◆ UpdateHitBoxAll()

void nn::ui2d::ButtonGroup::UpdateHitBoxAll ( )

Calls UpdateHitBox for all buttons included in the group.

Details
You do not need to use this member function when hit detection is configured to update for each frame.

Call as-needed when updating is not configured. Note that when you call this method, you must call nn::ui2d::Layout::Calculate() at least once because UpdateHitBox references the global matrix of the pane.
