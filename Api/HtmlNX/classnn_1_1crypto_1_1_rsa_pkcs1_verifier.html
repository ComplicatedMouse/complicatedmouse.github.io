Detailed Description

template<size_t KeySize, typename HashFunction>
class nn::crypto::RsaPkcs1Verifier< KeySize, HashFunction >

Class that authenticates RSA-PKCS1 signatures.

Template Parameters
KeySizeSize of the key to use in bytes.
HashFunctionImplementation class for the hash function to use.

Details
RSA-PKCS1 signatures are a signature method that uses the RSA signature algorithm stipulated with PKCS #1.

This class is a template class that takes the key size and hash function implementation class as template parameters. Use Rsa2048Pkcs1Sha256Verifier or a similar class in your actual code.

The key size can be specified in byte (8-bit) units up to a maximum of 512 bytes (4096 bits).

The implementation of the hash function class specified in the template must satisfy the following conditions. For more information about parameters of each function, see Sha1Generator or similar classes.

  1. Has a BlockSize constant that defines the size of the block to process.
  2. Has a HashSize constant that defines the output size of the hash value.
  3. -# The Asn1ObjectIdentifier object ID has a DER-encrypted binary array.
  4. -# The Asn1ObjectIdentifierSize object ID has a constant defined by the size of the DER-encrypted binary array.
  5. Has an Initialize function that begins processing.
  6. Has an Update function that updates the hash value on input.
  7. Has a GetHash function that gets the final hash value.

This class has the following states. The state immediately after instantiation is uninitialized.

  • Uninitialized state.
  • Ready.
  • Complete.

Invalid parameters are returned when initialization fails.
Precondition
  • modulusSize <= KeySize.
  • publicExponentSize <= 3.
Postcondition
  • The instance is in a ready state.

Details
Enters parameters and initializes the signature authentication process. An internal copy of the parameters is created.

The modulus and exponent are always treated as unsigned numbers in big-endian format.

A value up to three bytes in size, such as 65537, can be specified for the RSA public exponent.

Postcondition

Details
It can get the hash value for the data to authenticate after signature authentication.

Note that if you reduce the frequency of signature verification using the obtained hash value, security is reduced. For this reason, be careful when handling the obtained hash value.
