Detailed Description

template<typename BlockCipher>
class nn::crypto::GcmEncryptor< BlockCipher >

Class for running authenticated encryption in GCM mode.

Template Parameters
BlockCipherThe implementation class for the block cipher to use.

Details
GCM is an authenticated encryption-use mode, prescribed with NIST SP 800-38D.

In GCM mode, a message authentication code (MAC) is computed at the same time as the encryption or decryption operation. Data alteration can be detected by checking whether the same MAC was output by the encryption and decryption processes.

This class is a template class that takes the block cipher implementation class as a template parameter. GCM mode requires a block cipher with a block length of 128 bits. Use Aes128GcmEncryptor or something similar for actual use.

The implementation of the block cipher class specified in the template must satisfy the following conditions. For more information including about the parameters of each function, see the reference for the AesEncryptor class.

  1. Has a BlockSize constant that defines the size of the block to process. In GCM mode, you can only use a block cipher value of 16 here.
  2. Has a EncryptBlock function that encrypts a single block.

This class has the following states. The state immediately after instantiation is uninitialized.

  • Uninitialized state.
  • Ready to process additional authentication data.
  • Ready for encryption.
  • Complete.

Parameters

[in]pIvPointer to the initial data for the counter.
[in]ivSizeSize of the initial counter data in bytes.
Precondition
  • The EncryptBlock member function of the instance specified in pBlockCipher can be called.
Postcondition
  • The instance is ready to process additional authentication data.

Details
Specify an instance of the class that implements the block cipher to use for encryption in pBlockCipher. The instance must be initialized with the required key, and other settings, so that EncryptBlock is ready to be called.

A different counter default must be used for each encryption. We recommend a size of 12 bytes.

The instance specified in pBlockCipher is not copied. Do not perform any operations on pBlockCipher until the processing sequence is complete.

Default counter data is copied internally.

Inputs additional authentication data.

Parameters
[in]pAadPointer to the buffer storing the additional authentication data.
[in]aadSizeSize in bytes of the additional authentication data to input.
Precondition
  • The instance is ready to process additional authentication data.

Details
Additional authentication data is only affected by the MAC and not by encryption. The specified amount of data is read from pSrc to calculate the MAC.

This function can be split among multiple calls. Note, however, that the entered data is treated as a single contiguous set of data, so the order of input is significant.

All additional authentication data must be added before starting the Update() process.

Returns
Returns the actual number of bytes that were encrypted.
Precondition
  • The instance is either ready to process additional authentication data or ready to encrypt.
  • dstSize >= srcSize
  • The region specified by pDst,dstSize does not overlap with the region specified by pSrc,srcSize.
Postcondition
  • The instance is ready for encryption.

Details
Because encryption takes place byte-by-byte in GCM mode, the dstSize required for each call must be the same as srcSize.

The buffer required for the total dstSize for all calls is also the same as the total srcSize.

size_t  macSize

Gets the final MAC.

Parameters
[out]pMacPointer to the buffer storing the MAC.
[in]macSizeSize in bytes of the buffer specified in pMac.
Precondition
  • The instance is ready to process additional authentication data, ready to encrypt, or complete.
  • macSize >= MacSize
Postcondition
  • The MAC is written to pMac.
  • The instance is in a completed state.

Details
This function gets the MAC computed from the additional authentication data and plaintext data entered up to the point when the function is called.

Send the calculated MAC along with the ciphertext and additional authentication data for the encryption process. In the decryption process, make sure the calculated MAC matches the received MAC. This can be used to detect any alterations to the ciphertext or additional authentication data.
