Detailed Description

Class for using CTR mode AES encryption with a key size of 256 bits.

Details
This class has the following states. The state immediately after instantiation is uninitialized.

  • Uninitialized state
  • Ready.

Initializes the process for partial AES-CTR mode encryption with a key size of 256 bits.

Parameters
[in]pKeyPointer to the key data.
[in]keySizeSize in bytes of the key data.
[in]pIvPointer to the initial data for the counter.
[in]ivSizeSize of the initial counter data in bytes.
[in]srcOffsetOffset to start processing.
Precondition
  • keySize == 32.
  • ivSize == IvSize
  • srcOffset >= 0.
Postcondition
  • The instance is in a ready state.

Details
This process takes the key data and initial counter value as input and initializes it so that encryption can start from the srcOffset position specified in the data array. Next, call Update() to perform the process from the data at that position and partially encrypt the data array.

Internal copies of the key data and initial counter data are created.

Parameters
[in]pIvPointer to the initial data for the counter.

Precondition
  • The instance is in a ready state.
  • ivSize == IvSize

Details
Switches the encryption target from the currently processing data string to a data string to be encrypted with the same key. Use the Initialize() function to switch to a data string to be encrypted with a different key.

An internal copy of the initial counter value is created.

Decrypts data using AES-CTR mode encryption with a key size of 256 bits.

Parameters
[out]pDstPointer to the buffer that receives encrypted data.
[in]dstSizeSize of the buffer pointed to by pDst.
[in]pSrcPointer to the buffer storing the data to encrypt.
[in]srcSizeSize in bytes of the data to encrypt.
Returns
Returns the actual number of bytes that were encrypted.
Precondition
  • The instance is in a ready state.
  • dstSize >= srcSize
  • The region specified by pDst,dstSize does not overlap with the region specified by pSrc,srcSize.
Postcondition
  • The encrypted result is written to pDst.

Details
Because encryption takes place byte-by-byte in CTR mode, the dstSize required for each call must be the same as srcSize.

The buffer required for the total dstSize for all calls is also the same as the total srcSize.
