This sample program describes how to use audio renderer sinks.

Overview

This sample program describes the sinks that serve as the destinations where the rendering results of the audio renderer are output. In particular, it demonstrates how to use CircularBufferSinkType objects.

File Structure

This sample program is located in Samples/Sources/Applications/AudioSink/ .

Required Environment

You must be able to use audio output.

How to Use

When you run the sample program, the following methods of operation are output and all sound effects are played through once.

Input Operation
A Button Plays SampleSe0 once from the left channel.
[B] Plays SampleSe1 once from the right channel.
[X] Plays SampleSe2 once from the left channel.
[Y] Plays SampleSe3 once from the right channel.
START or SPACEBAR Exits the program.

Press the A, B, X, and Y Buttons to play the corresponding sound effects from the left or right channels. When a button is pressed, the output level for each channel is output to a log under the method of operation.

Execution Procedure

Build the sample program and then run it.

Description

The audio renderer processes the waveforms and the rendered sample data is ultimately output to the FinalMixType mix buffers. Each sink receives the sample data that was in those FinalMixType mix buffers and processes that data using the feature intrinsic to that sink type.

This sample program is designed primarily to describe CircularBufferSinkType. CircularBufferSinkType is a sink type that provides a feature for writing the rendering results of the audio renderer to memory. You can use this feature for audio recording and rendering and otherwise using sample data for debugging and tool development. This sample program demonstrates how to process data obtained from CircularBufferSinkType and how to display the sound pressure levels output from the audio renderer.

The main tasks demonstrated in this sample program are as follows.

  • Specifying the required number of sinks for nn::audio::AudioRendererParameter.sinkCount when initializing the audio renderer
  • Getting the amount of memory required for CircularBufferSinkType
  • Adding CircularBufferSinkType to FinalMixType
  • Reading the sample data from CircularBufferSinkType
