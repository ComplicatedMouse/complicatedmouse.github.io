Detailed Description

template<typename HashFunction>
class nn::crypto::HmacGenerator< HashFunction >

Class for calculating an HMAC.

Template Parameters
HashFunctionThe implementation class for the hash function to use.

Details
An HMAC is a message authentication code (MAC) generated using a hash function.

This class is a template class that takes the hash function implementation class as a template parameter. Use HmacSha1Generator, HmacSha256Generator, or a similar class in your actual code.

The implementation of the hash function class specified in the template must satisfy the following conditions. For more information including about the parameters of each function, see the reference for Sha256Generator or other relevant class.

  1. Has a BlockSize constant that defines the size of the block to process.
  2. Has a HashSize constant that defines the output size of the hash value.
  3. Has an Initialize function that begins processing.
  4. Has an Update function that updates the hash value on input.
  5. Has a GetHash function that gets the final hash value.

This class has the following states. The state immediately after instantiation is uninitialized.

  • Uninitialized state.
  • Ready.
  • Complete.

Initializes the MAC calculation.

Parameters
[in]pKeyPointer to the key data.
[in]keySizeSize in bytes of the key data.
Postcondition
  • The instance is in a ready state.

Details
This function takes key data as input and initializes the MAC calculation. An internal copy of the key data is created.

HMAC supports a key of any size, but from the standpoint of encryption strength, we recommend using a key size of at least HashSize. A hash value is calculated by the specified hash function for a key that is at least as large as BlockSize, and used as the HashSize key.
