Overview

Describes the sample program for nn::time::CalculateSpanBetween.

It is possible to measure the actual time elapsed for shutting down the application.

For more information, see the nn::time::CalculateSpanBetween "CalculateSpanBetween function reference."

File Structure

This sample program is located in Samples/Sources/Applications/TimeCalculateSpanBetween/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Initialize the time library.
  • Run the sample that measures the elapsed time.
  • Finalize the time library.

This feature can measure the passage of time spanning events such as shutting down the application and turning off the system by permanently retaining the value of nn::time::ClockSnapshot, which is the starting point in time, and storing it a location such as in save data.

Under some conditions, it is not possible to use nn::time::CalculateSpanBetween() to calculate the elapsed time. For more information, see the API Reference.

If conditions preclude straddling application ending and starting, you can calculate the elapsed time at higher resolutions using the system tick value that you can get with nn::os::GetSystemTick.

For functions that return nn::Result, NN_ABORT_UNLESS_RESULT_SUCCESS is used to handle cases where success always occurs because the pre-conditions are met.
