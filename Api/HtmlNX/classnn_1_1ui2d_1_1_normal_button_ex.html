Detailed Description

This button can return to ON status after the Down animation has played.

Details
The behavior is the same as NormalButton, but it operates by using LayoutEx AnimatorEx. This version is the test implementation, so do not use it.

◆ UpdateHitBox()

virtual void nn::ui2d::AnimButton::UpdateHitBox ( )
virtualinherited

Updates the hit-detection area.

Details
Because this process is not performed by calc, an explicit call is required. However, because it is usually called by calc in the ButtonGroup class, it does not need to be called individually. Software is designed so you can set whether to perform updates using a ButtonGroup option.

If the button does not change too much in size, you can conserve processing by not updating the hit area.

Call after running nn::ui2d::Layout::Calculate() at least once because this process uses the GlobalMatrix for the pane that represents button hits.

This class does not support hit-detection in rotating panes. You must override this member function and the IsHit member function to support rotating pane hit detection.

◆ PlayDisableAnim()

virtual void nn::ui2d::AnimButton::PlayDisableAnim ( bool  isDisable)
virtualinherited

Plays back a disabled animation.

Details
Plays back a disabled animation. Note that the playback of the Disable animation is independent of the active/inactive state of the button. Take note of this. If you want to make the button inactive and play the Disable animation at the same time, call both SetActive(false) and PlayDisableAnim(true).

Note that nothing changes, even after calling PlayDisableAnim(true) after animations have already been disabled, because animation playback uses PlayFromCurrent.

Parameters
[in]isDisableIf true, the Disable animation is played forward and takes the disabled animation state. If false, the Disable animation is played in reverse and takes the enabled animation state.

Disables all animations being maintained by the button.

Details
This function gets all animations using GetXXXAnimator, and performs the same operation as SetEnabled(false) for each of them. Usually, this function is not needed; however, it is useful when you want to stop the normal operation of the button and play a different animation. ◆ SetActive()

virtual void nn::ui2d::ButtonBase::SetActive ( bool  active)
virtualinherited

Determines whether the button is enabled.

Details
On and Down actions are not accepted when a button is inactive. However, the Off action is accepted even when a button is inactive. The reason is that if a button becomes inactive when the cursor hits it, the button then changes to the Off state as a matter of course.

Use the SetAcceptActionAll() function to set a button not to accept any actions.

Also note that forcible changes can be made even to inactive buttons by using functions such as ForceOn().

Parameters
[in]activeWhether to enable.

◆ IsAcceptAction()

bool nn::ui2d::ButtonBase::IsAcceptAction ( Action  action) const
inherited

Determines whether the button accepts the action currently specified.

Details
Note that this setting is independent from the status handled by IsEnabled/SetEnabled. In other words, SetEnabled() && IsAcceptAction(ACTION_ON) is required to accept an On action.

All actions are accepted by default.

However, forcible changes made using functions like ForceOn() are always accepted regardless of this setting.

Parameters
