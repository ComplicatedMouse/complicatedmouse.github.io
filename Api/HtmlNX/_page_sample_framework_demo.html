This sample program uses the GraphicsFramework class to describe various frameworks.

Overview

This documentation describes the FrameworkDemo sample program.

File Structure

This sample program is located in Samples/Sources/Applications/FrameworkDemo .

Required Environment

You must be able to use the screen display.

How to Use

Use input from the DebugPad, or the keyboard and mouse when using a PC, to perform the following operations.

Common Operations

Input Operation
Up and down on the +Control Pad, or the up and down arrow keys. Switches the selected item for editing.

FrameworkMode:

Input Operation
Left and right on the +Control Pad, or the left and right arrow keys. Changes the framework mode.

PresentInterval:

Input Operation
Left and right on the +Control Pad, or the left and right arrow keys. Changes the screen V-sync count.

CalculateSleepCpu:

Input Operation
Left and right on the +Control Pad, or the left and right arrow keys. Increases or decreases the CPU load for processing calculations.

MakeCommandSleepCpu:

Input Operation
Left and right on the +Control Pad, or the left and right arrow keys. Increases or decreases the CPU load for generating commands.

DrawCount:

Input Operation
Left and right on the +Control Pad, or the left and right arrow keys. Increases or decreases the GPU load.

Cautions

Nothing specific.

Execution Procedure

Build the sample program and then run it.

Description

This demo uses the GraphicsFramework class to describe various frameworks. The results of switching between frameworks or increasing and decreasing processing load can be verified from the processing load meter.
