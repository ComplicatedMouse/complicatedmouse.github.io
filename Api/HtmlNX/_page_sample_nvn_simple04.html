This sample renders a scene into an application-specific texture, which is then used to draw a secondary scene that is then displayed to screen.

This sample introduces how to render to a texture and multi-pass rendering.

Overview

This sample implements the rendering of a scene that utilizes two passes to introduces how to render to a texture and multi-pass rendering. One pass to render several shapes into an application-defined texture (i.e. a texture other than the display target texture used by GraphicsSystem), and then another pass to use that texture to render the final scene destined for display.

Description

The first pass is implemented with the help of the SimplePrimitiveRenderer class that is in SharedLib.

Illustrated in this demo is how to create and set color and depth textures to be renderered into, and how to use the color texture in a subsequent pass to render a textured primitive (the method of which is identical to TexturedCube). Two separate command buffers are used and submitted separately, one for each pass. This seems to be a sufficient mechanism to guarantee that the render texture is ready to be consumed by the second pass after it is produced by the first pass.
