Detailed Description

The class used to retrieve bundled CA certificates in DER format.

Attention

Before using features in nn::ssl::FixedBuiltInManager, it is HIGHLY recommended to consider if your requirement can be achieved with your own implementation without this class.

This class may be used ONLY if below limitations are acceptable and requirements are satisfied.

LIMITATIONS

  • Certificates provided by nn::ssl::FixedBuiltInManager would NOT be updated by the system update
    • The certificates are built-in to the library and not retrieved from the system OS, so certificates that can be obtained by this class is fixed at the build time
  • If the time comes where you are required to deal with extra certificates that are not included in nn::ssl::FixedBuiltInManager in the future, it is required to handle it by yourself (e.g. preparing a certificate and supply it to your code)
    • If it is necessary to obtain certificates via GetBuiltInCertificates(), you can supply them to nn::ssl::FixedBuiltInManager by AddTemporaryBuiltInCertificate() before calling GetBuiltInCertificates() Supplied certificates can be obtained by CaCertificateId returned from AddTemporaryBuiltInCertificate() If CaCertificateId_All is used, that also returns supplied certificates in addition to other certificates in nn::ssl::FixedBuiltInManager
    • When something is modified to deal with the situation, it is required to guarantee the operation of the application by yourself

REQUIREMENTS

For reasons explained above, it is required to get an approval by Nintendo to use features in this class in the product. Please contact Nintendo for further details.

Details
This class may be used by applications to retrieve CA certificates which are bundled with the platform.

The certificates are built-in to the library and not retrieved from the system OS. This class will not get newer certificates added from an OS system update. The bundled CA certificates are each assigned IDs by Nintendo, which can be found in the header ssl_CaCertificateId.h.

Note that this feature is not included in the default SDK library. It is required to link libnn_sslFixedBuiltInManager library explicitly.

Set a custom memory allocator that will be used by nn::ssl::FixedBuiltInManager.

Details
nn::ssl::FixedBuiltInManager::SetAllocator() may only be called once and must be called before any other member function of nn::ssl::FixedBuiltInManager or a failure will occur. Calling this function is optional. If it is not called, a default global heap allocator will be used.

Parameters
[in]allocatorReference to a custom allocator that will be used for all dynamic allocations from nn::ssl::FixedBuiltInManager.
Returns
The result of the operation is returned


Return Values:

ResultSuccess Success

ResultAllocatorAlreadySet The allocator has already been set. It may not be set more than once. If any other member function of nn::ssl::FixedBuiltInManager has been called, the default allocator will be set and this error will occur.

ResultInsufficientMemory The system failed to allocate memory, try again later.

Get the built-in certificates from the nn::ssl layer.

Details
This can retreave all built-in certificates included in the static library. Any certificates added with nn::ssl::FixedBuiltInManager::AddTemporaryBuiltInCertificate() may also be retrieved with either CaCertificateId_All or by using the generated certificate Id.

Parameters
[out]pOutCertInfoPointer to a pointer which will receive the start of an array of BuiltInCertificateInfo structures within pOutBuffer. This array may be accessed to retrieve the details of each requested certificate. If CaCertificateId_All is requested, the last element of this array will have its id field set to CaCertificateId_All. Using this information, the caller can get all built-in certificates without having to know the IDs ahead of time. Any certificates added with nn::ssl::FixedBuiltInManager::AddTemporaryBuiltInCertificate will also be included.
[out]pOutBufferPointer to a buffer to place the certificates and meta data. After this call, the contents of this buffer should not be directly interpreted. Use pOutCertInfo to get the certificate details.
[in]bufSizeThe total size of the buffer at pOutBuffer.
[in]pInIdsPointer to buffer containing the certificate IDs which are being retrieved. If CaCertificateId_All is specified in any element then all built-in CA certificates will be retrieved (assuming the provided output buffer is large enough.)
[in]idCountThe total number of IDs in the buffer.
Returns
The result of the get operation is returned


Return Values:

ResultSuccess Success

ResultInvalidPointer The provided buffer pointer is invalid.

ResultBufferTooShort The provided buffer is not large enough.

ResultInvalidIdCount A value of 0 was provided for the ID count.

ResultInsufficientMemory The system failed to allocate memory, try again later.

ResultErrorLower An internal error occurred while trying to retrieve the certificates.

Details
The life of the certificate within the store will match the life of the application process.
