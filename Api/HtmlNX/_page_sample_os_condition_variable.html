A sample program for the condition variable feature.

Overview

This sample shows the condition variable feature.

For more information about using the condition variable feature, see the Condition Variable Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsConditionVariable/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Two threads are created with nnMain().
  • The two threads are ThreadFunction1() and ThreadFunction2().
  • Each thread performs mutually different condition variable notification and condition variable standby processes.
  • nnMain() waits until the two threads complete.
  • The two threads are destroyed by nnMain().

The two threads have a char type g_CharCode variable as a shared resource, and the character code is transmitted from ThreadFunction1() to ThreadFunction2(). The character code that can be transmitted at one time is only one char-type character.

ThreadFunction1() transmits one character with SendCharCode(). ThreadFunction2() receives one character with ReceiveCharCode() and adds the received character to a log.

SendCharCode() waits until g_CharCode is 0. At this time, the g_ConditionEmpty condition variable waits with WaitConditionVariable().

Meanwhile, ReceiveCharCode() waits until g_CharCode != 0. At this time, the g_ConditionNotEmpty condition variable waits with WaitConditionVariable().

The other thread that changes to a state to answer g_CharCode issues the SignalConditionVariable() to each condition variable. In other words, the transmitting thread notifies NotEmpty to the receive wait, and the receiving thread notifies Empty to the transmit wait.
