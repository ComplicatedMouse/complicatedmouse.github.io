◆ AttachActivateEvent()

nn::Result nn::nfc::AttachActivateEvent ( nn::os::SystemEventType *  pEvent,
const DeviceHandle &  deviceHandle 
)

Sets an event to report the detection of a tag.

Parameters
[in,out]pEventThe event to use to report the detection of a tag.
[in]deviceHandleNFC device handle.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess The event was set successfully.
  • ResultNfcDeviceNotFound Could not find an NFC device in the correct state. You need to call ListDevices() and get the NFC device handles again.
  • ResultNfcDisabled NFC functionality is disabled. You will need to retry after enabling the NFC functionality.
Precondition
  • pEvent must point to an accessible address.
  • GetState() == State_Init.
Postcondition
pEvent will be signaled for the specified NFC device when a tag is detected.

Details
Call this function for each NFC device after the nfc library has been initialized and before the StartDetection() function starts tag detection. The event set with this function is signaled for the specified NFC device when a tag is detected.

The event is initialized within the API, so there is no need for the user to initialize it. Events are initialized to auto-clear mode.

Make sure that you call nn::os::DestroySystemEvent() to destroy the event after it is no longer required.

◆ AttachAvailabilityChangeEvent()

void nn::nfc::AttachAvailabilityChangeEvent ( nn::os::SystemEventType *  pEvent)

Set an event to report the change in availability of the NFC device.

Parameters
[in,out]pEventSpecify the event to use in the notification.
Precondition
  • pEvent must point to an accessible address.
  • GetState() == State_Init.
Postcondition
When the availability of the NFC device changes, pEvent will be signaled.

Details
Run after the initialization of the nfc library. The set event is signaled by this function at the following times.

  • The NFC device is disconnected.
  • The NFC device is connected.
  • The application in focus has changed.
  • The system woke from sleep mode.
  • The NFC feature was disabled.
  • The NFC feature was enabled.

An efficient way of checking the state of the NFC device is by calling ListDevices() and GetDeviceState() when the event is signaled. The event is initialized within the API, so there is no need for the user to initialize it. Events are initialized to auto-clear mode.

Make sure that you call nn::os::DestroySystemEvent() to destroy the event after it is no longer required.

Sends a command to a tag via pass-through communication.

Parameters
[out]pOutBufferThe buffer to get the response data from.
[out]pOutSizeThe size of the data actually obtained.
[in]deviceHandleNFC device handle.
[in]pDataData to send.
[in]dataSizeSize of the data to send.
[in]bufferSizeThe size of the response data buffer.
[in]timeoutThe maximum amount of time to wait for a response.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess Indicates that sending was successful.
  • ResultNfcDeviceNotFound Could not find an NFC device in the correct state. You need to call ListDevices() and reacquire the NFC device handles.
  • ResultNfcDisabled NFC functionality is disabled. You must enable NFC functionality and then try again, starting with StartDetection() for tag detection.
  • ResultAccessError Indicates failure to send. You must confirm that the method you are using to access the tag is correct (such as the content of the command and the maximum time to wait for a response) and then start the process again from the point of tag detection using the StartDetection() function.
  • ResultNeedRestart Indicates failure to send. The tag you are trying to send the command to may have been lost. You must try again, starting with StartDetection() for tag detection.
  • ResultNotSupported Indicates that the tag is not a supported NFC tag. You must check the tag in use.
Precondition
  • pOutBuffer must point to an accessible address.
  • pOutSize must point to an accessible address.
  • 0 < bufferSize.
  • pData must point to an accessible address.
  • 0 < dataSize.
  • GetState() == State_Init.

Details
Sends a command to a tag via pass-through communication. The tag must have been already detected and not lost. (Returns ResultNfcDeviceNotFound when the state is incorrect.) There is no guarantee for the content in the response data buffer if the size of the actually obtained response data exceeds the size of the buffer. Do not reference the buffer.

◆ KeepPassThroughSession()

nn::Result nn::nfc::KeepPassThroughSession ( const DeviceHandle &  deviceHandle)

Keeps the session that was used for pass-through communication.

Parameters
[in]deviceHandleNFC device handle.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess Indicates that the session has been successfully kept.
  • ResultNfcDeviceNotFound Could not find an NFC device in the correct state. You must call ListDevices() and get the NFC device handles again.
  • ResultNfcDisabled NFC functionality is disabled. You must enable NFC functionality and then try again, starting with StartDetection() for tag detection.
  • ResultNeedRestart Indicates that keeping the session has failed. The tag you are trying to get information for may have been removed (lost). You must try again, starting with StartDetection() for tag detection.
Precondition
  • GetState() == State_Init.

Details
The tag must have been detected and not lost, and the session not being kept already. (Returns ResultNfcDeviceNotFound when the state is incorrect.) If you call SendCommandByPassThrough() multiple times without calling this function, there is no guarantee that the same session will be used for sending the various commands. If you need to guarantee that it is the same, call this function before you call SendCommandByPassThrough(). For example, if you are going to send a command that requires prior password authentication, first detect the tag, and then call this function, and then send a command for password authentication, and lastly send the command that requires password authentication. When the session no longer needs to be kept, call ReleasePassThroughSession() to release the session and revert to the state where the session is not being kept. More power is consumed when a session is being kept, and notification events for lost tags are not generated, so only keep sessions when it is necessary.

◆ StartDetection() [1/2]

nn::Result nn::nfc::StartDetection ( const DeviceHandle &  deviceHandle)

Starts the process of tag detection.

Parameters
[in]deviceHandleNFC device handle.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess Detection has started.
  • ResultNfcDeviceNotFound Could not find an NFC device in the correct state. You need to call ListDevices() and reacquire the NFC device handles.
  • ResultNfcDisabled NFC functionality is disabled. You will need to retry after enabling the NFC functionality.
  • ResultMaxNfcDeviceActivated You cannot use NFC functionality for the specified NFC device. The maximum number of NFC devices using that NFC functionality has been reached. You will need to retry after reducing the number of NFC devices using the NFC functionality.
  • ResultConflictFunction You cannot use NFC functionality because a competing feature is in use. You must retry after the use of the competing feature ends.
Precondition
  • GetState() == State_Init.
  • GetDeviceState() == DeviceState_Init || GetDeviceState() == DeviceState_Deactive.
Postcondition
  • GetDeviceState() == DeviceState_Search.

Details
Starts detecting tags for the specified NFC device. Must occur either immediately after initialization of the library or immediately after a detected tag has been lost (removed). (Returns ResultNfcDeviceNotFound when the state is incorrect.) If the event is set to AttachActivateEvent or AttachDeactivateEvent, it is cleared to a nonsignaled state as soon as a signaled state is detected. This function works the same as StartDetection(const DeviceHandle& deviceHandle, NfcProtocol protocolFilter) if NfcProtocol_All has been specified for the target protocol to detect.

nn::Result nn::nfc::GetTagInfo ( TagInfo *  pOutTagInfo,
const DeviceHandle &  deviceHandle 
)

Gets the tag information.

Parameters
[out]pOutTagInfoPointer to the location of the TagInfo.
[in]deviceHandleNFC device handle.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess Obtained successfully.
  • ResultNfcDeviceNotFound Could not find an NFC device in the correct state. You must call ListDevices() and get the NFC device handles again.
  • ResultNfcDisabled NFC functionality is disabled. You must enable NFC functionality and then try again, starting with StartDetection() for tag detection.
  • ResultNeedRestart Failed to get. The tag you are trying to get information for may have been removed (lost). You must try again, starting with StartDetection() for tag detection.
Precondition
  • pOutTagInfo must point to an accessible address.
  • GetState() == State_Init

Details
Run this function on tags that have been detected. (Returns ResultNfcDeviceNotFound when the state is incorrect.)
