Detailed Description

Structure for designating initialization parameters.

Details
Because this structure is so big, we recommend creating it on the heap and not on the stack.

◆ noPlotWorkMemorySize

size_t nn::font::TextureCache::InitializeArg::noPlotWorkMemorySize

The size of the work memory for a scalable font engine instance created separately from the one used for plotting so that processing can occur concurrently with plotting.

In most situations the default size is fine, but depending on the font, this size may be insufficient. Increase the size as needed. If the size is insufficient, an assertion failure occurs and the message "error:201" is displayed. The default size is 50 KB.

◆ isUsingNoPlotWorkMemorySizeAsTotalSize

bool nn::font::TextureCache::InitializeArg::isUsingNoPlotWorkMemorySizeAsTotalSize

Indicates whether to always use a fixed noPlotWorkMemorySize instead of multiplying by the number of font faces.

If true is set, memory equivalent to noPlotWorkMemorySize is always allocated regardless of the number of font faces. However, even if it is set to true, memory is allocated for just the number of cores via multiplication. The default is false.

◆ isDrawMultiCore

Determines whether to use this texture cache for drawing on multiple cores.

More memory is used if drawing is performed on multiple cores because, for parallel processing, there must be as many instances of the scalable font engine as there are cores. The default is false.

◆ coreCount

int nn::font::TextureCache::InitializeArg::coreCount

If drawing is to be performed on multiple cores, this specifies the number of cores in the running platform.

Set so pGetCoreNumberFunction() < coreCount. This feature is predominantly for multi-core processing on production devices. When using it on a PC, you can either process with fewer cores than are on production devices or you can turn multi-core functionality off.

◆ isAutoHint

bool nn::font::TextureCache::InitializeArg::isAutoHint

Indicates whether to force hinting.

Normally, set this parameter to false. If drawing breaks down or the program crashes as a result of using particular fonts, setting this parameter to true fixes the problem. However, setting it to true slightly changes the drawing result by a few pixels. Hint Data With Older Fonts It has been confirmed that some fonts will crash because of a rounding error when font rendering calculations are underway.

These crashes can be avoided by setting this parameter to true, which causes the hint information calculations to be made at run time. The default is false.

◆ isExactWidthReturnedAfterPlot

bool nn::font::TextureCache::InitializeArg::isExactWidthReturnedAfterPlot

The character widths obtained using nn::font::ScalableFont::GetCharWidth or nn::font::ScalableFont::GetCharWidths can have a measurement error of roughly one pixel, compared to actual rendering widths because the calculations are a mockup prior to plot completion.

Setting this parameter to true returns accurate widths for post-plot rendering results. Setting this parameter to false returns the same width values, both before and after plotting. Use false when the same widths for both before and after plotting are desired, as when using a repetition process based on character widths. The default is true.
