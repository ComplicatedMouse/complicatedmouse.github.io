Detailed Description

Class for handling system ticks.

Details
The system tick value is a system-dependent representation of time. It is a 64-bit integer that counts the number of ticks since an arbitrary date. The speed at which the system tick value increases depends on the target environment and operating conditions. A separate function is provided for obtaining the frequency (number of ticks per second).

System ticks can be converted to and from nn::TimeSpan which is a natural representation of time, using conversion constructors and operators. However, due to differences in the speed at which system ticks and other time formats advance, these types cannot be used together. Even if the values can be converted to the same nn::TimeSpan units, simultaneous measurements of time will not provide the same values.

For example, this time advances at a different speed than the time that is handled inside a library (such as a sound or video playback library). To synchronize an application with sound playback, study the specifications for each library and use a time format with matching speed.
