Overview

The sample program that shows how to use web applet and offline web applet features.

File Structure

This sample program is located in Samples/Sources/Applications/WebDemo/ .

Required Environment

This sample program is only supported in the NX development hardware environment.

How to Use

For more information about operations, see the instructions in each sample.

Important Information

Nothing specific.

Execution Procedure

Build the sample program and then run it.

Description

This sample is comprised of the application, multiple sample webpages, and the menu for selecting sample webpages. The sample webpages and menu are displayed in the offline web applet. They are available as offline HTML.

This sample has the following basic flow.

  • The application draws a gray screen.
  • The application starts the offline web applet and displays the menu.
  • If a webpage is selected from the menu, the application is notified of the type of the selected webpage.
  • The application starts the offline web applet and displays a webpage according to the type of selected webpage.
  • Returns to the application from the offline web applet after finishing display of the webpage.
  • The application starts the offline web applet and displays the menu again.

The processing of each sample webpage includes the application part and the webpage part. For more information about processing, see the instructions in each sample.

When the web applet or Offline Web Applet is started from the application and a webpage is displayed, and if the user presses the HOME Button and starts the HOME Menu from the system applet, the system closes the applet started by the application.

The application uses nn::web::OfflineHtmlPageReturnValue::GetOfflineExitReason() to get the reason that the offline web applet was terminated. If the applet was closed by the system, it starts the offline web applet again and restores the webpage.
