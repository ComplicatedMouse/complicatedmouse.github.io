Detailed Description

The profanity filter base class.

Details
In the future, an updated profanity filter that represents a trade-off between memory consumption and speed may be released. This class will serve as the base for that filter.

Bit32  patterns,

pure virtual

Uses the specified pattern list to check phrases strung together from a number of specified words to determine whether there would be a problem if they are shown as text on the screen.

Parameters
[out]pCheckResultsThe buffer for storing the result of the check. It must be an array with the same number of entries specified by wordCount.
[in]patternsThe pattern list to check the words against.
[in]ppWordsThe array of null-terminated strings to check. Store them using UTF-8 character encoding.
[in]wordCountThe number of strings to check. The maximum is nn::ngc::WordCountMax.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The checking succeeded.
  • nn::ngc::ResultNotInitialized Not initialized.
  • nn::ngc::ResultInvalidPointer An invalid pointer was specified.
  • nn::ngc::ResultInvalidSize An invalid number of strings was specified.

Details
This function checks screen names and other single words. It cannot be used for processing text entered by the user that contains two or more words. To check that kind of text, use the ProfanityFilterBase::MaskProfanityWordsInText function instead.

The pattern list for the highest priority language that the application supports, returned by nn::oe::GetDesiredLanguage, will be used in addition to the pattern list used automatically based on the system language settings. For applications that do not use nn::oe::GetDesiredLanguage, we recommend adding an independent pattern list corresponding to the application's supported languages.

The processing by this function can be time-consuming (it blocks). The processing time can vary depending on the system version. The frame rate can decline if this function is called from the main thread, so we recommend running it from another thread. This function can simultaneously check as many as nn::ngc::WordCountMax individual strings. You can complete the task faster by checking multiple strings at the same time, rather than one at a time.

The results of the check on each string are stored as an array in pCheckResults. Perform a logical AND with the values of the patterns in nn::ngc::ProfanityFilterPatternList to check the strings. The value becomes 0 if no problem is determined with the list specified by patterns.

For more information about determining which pattern lists must be used for the checks, see the latest version of the NGC library manual.

Strings that include the at sign (@), which could indicate the string is an email address, are always identified as problem phrases. On the other hand, strings that contain many numbers, which could indicate a phone number, are not identified as problem phrases. To check these kinds of strings, use the nn::ngc::CountNumbers function.

Uses the specified pattern list to check whether the specified text would pose a problem if shown as strings on the screen.

Parameters
[out]pProfanityWordCountThe pointer for getting how many times illegal strings showed up in the text. If this information is not necessary, specify nullptr.
[in,out]pTextThe pointer to the text to check. Any illegal strings in the text are replaced with asterisks. For this reason, the application must specify a writable buffer. The function can check a maximum of nn::ngc::TextLengthMax characters, including the terminating null character. If a string longer than this maximum is passed to the function, it will not check anything beyond the maximum length. Store them using UTF-8 character encoding.
[in]patternsA profanity pattern list.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The checking succeeded.
  • nn::ngc::ResultNotInitialized Not initialized.
  • nn::ngc::ResultInvalidPointer An invalid pointer was specified.
  • nn::ngc::ResultInvalidSize An invalid number of strings was specified.

Details
This function processes text that includes two or more words. It cannot be used for screen names and other short words. For those kinds of strings, use the ProfanityFilterBase::CheckProfanityWords function instead.

This function checks whether there are any illegal strings among the specified strings and masks those illegal strings with asterisks.

This function treats the following characters as word separators.

  • Carriage return, line feed, next line, line break, paragraph break, tab, vertical tab, form feed, single-byte space and single-byte symbols ｡､.,[!" #$%&'()*+/:;<=>? @\^_`{|}~-] double-byte space and double-byte symbols。、．，［！ ""＃＄％＆‘'（）＊＋／：；＜＝＞？＠￥＾＿｀｛｜｝～－］

If word separators are included in illegal strings, they are still judged to be illegal strings.

The pattern list for the highest priority language that the application supports, returned by nn::oe::GetDesiredLanguage, will be used in addition to the pattern list used automatically based on the system language settings. For applications that do not use nn::oe::GetDesiredLanguage, we recommend adding an independent pattern list corresponding to the application's supported languages.

The processing by this function can be time-consuming (it blocks). The processing time can vary depending on the system version and the length of the strings passed to the function. The frame rate can decline if this function is called from the main thread, so we recommend running it from another thread.

For more information about determining which pattern lists must be used for the checks, see the latest version of the NGC library manual.
