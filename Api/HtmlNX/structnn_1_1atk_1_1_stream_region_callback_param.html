Detailed Description

Defines the parameters to use as input and output for the callback used with the stream jump feature.

Details
These parameters are passed as the arguments of the callback. They are referenced and get overwritten by the next region's information.

◆ regionNo

int nn::atk::StreamRegionCallbackParam::regionNo

Region number.

Details
Stores the region number specified by the previous callback. If no value has ever been specified, it is set to 0. Use the callback function to write the next region number to play. If a region number and region name are used at the same time, the region name takes precedence.

If a negative number or a value greater than regionCount is specified for the region number, an assertion failure occurs.

◆ regionName

char nn::atk::StreamRegionCallbackParam::regionName[RegionNameLengthMax+1]

Region name.

Details
Stores the region name specified by the previous callback. If no value has ever been specified, or if no name was specified by the previous callback, it is set to an empty string.

Specify the name of the next region to play to search for that region and jump to there if it is found. If that region cannot be found, or if false was specified for isRegionNameEnabled, the value of regionNo will be used. If a region number and region name are used at the same time, the region name takes precedence.

◆ isRegionNameEnabled

bool nn::atk::StreamRegionCallbackParam::isRegionNameEnabled

Flag indicating whether to enable the use of region names to specify regions.

Details
When set to true, regions can be specified using region names. This value stores the setting that was specified by the previous callback. If no value has ever been specified, it is set to false.

int nn::atk::StreamRegionCallbackParam::regionCount

The number of regions.

Details
If a number greater than regionCount is specified for regionNo, an assertion failure occurs.
