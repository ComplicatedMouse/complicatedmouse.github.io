◆ InitializeFromFencedGLSync()

NVNboolean nvn::Sync::InitializeFromFencedGLSync ( Device *  device,
const void *  glSync,
SyncInteropStatus *  status 
)

Use a fenced and flushed GLsync object to initialize a NVN Sync object.

Use this function instead of Initialize for NVN Sync objects created from GLsync objects. This function initializes a Sync object that signals after a corresponding GLsync object. This enables OpenGL to NVN synchronization (fencing is done on the OpenGL object and waiting on its NVN counterpart). The value returned indicates the status of the Sync object. See nvn::Sync::CreateGLSync.

Parameters
deviceDevice owning the Sync object.
glSyncThe name of the OpenGL or OpenGL ES sync to use as a source.
  • The GLsync object must be a valid OpenGL or OpenGL ES GLsync object.
  • The GLsync object must have already been flushed in the GL command stream.
  • It needs to be valid for the lifetime of the Sync object.
[out]statusDetailed error status.

SyncInteropStatus nvn::Sync::CreateGLSync ( uint64_t *  glSync) const

Use a fenced and flushed NVN Sync object to create a GLsync.

This function initializes a GLsync object that signals after a corresponding NVN sync object. This enables NVN to OpenGL synchronization (fencing is done on the NVN Sync object and waiting on its GLsync counterpart). The value returned indicates the status of the Sync object. See nvn::Sync::InitializeFromFencedGLSync.
