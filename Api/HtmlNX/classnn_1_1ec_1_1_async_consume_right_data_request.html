Detailed Description

Asynchronous request class that consumes the rights information stored on the server.

Details
Use SetupAsyncRequest in ConsumableServiceItemManager to initialize this object. Initialize the object appropriately before running the asynchronous request.

Examples
EcConsumableServiceItemManager/EcConsumableServiceItemManager.cpp.

Result nn::ec::AsyncConsumeRightDataRequest::Begin ( )

Starts consumption of unconsumed license information.


Return Values:

nn::ResultSuccess The process succeeded.

nn::ec::ResultShopServiceAccessOverRequest Indicates that service resources, necessary for the asynchronous request, could not be allocated. If there is an asynchronous request that has not been destroyed, destroy the instance and then run the request again. Otherwise, reduce the simultaneous use count in ShopServiceAccessor.

nn::ec::ResultShopServiceAccessInsufficientWorkMemory Insufficient work memory for ShopServiceAccessor.

nn::ec::ResultInsufficientWorkMemory Insufficient work memory for ConsumableServiceItemManager. During initialization, pass work memory of nn::ec::RequiredWorkMemorySizeForConsumableServiceItem bytes or greater.

nn::ec::ResultInternetRequestNotAccepted Indicates that an Internet connection request has not been submitted and accepted.

Precondition
  • Initialize() was already executed. (Executed by SetupAsyncRequest in ConsumableServiceItemManager.)

Details
The manager consumes the stored license information from the server. After the license information has been consumed, it can no longer be obtained with AsyncGetConsumableRightDataRequest.

This function may return an nn::ec::ShopServiceAccessor::AsyncResponse error. For more information about this error handling, see the ShopServiceAccessor::AsyncResponse reference.

This function returns an error if nn::nifm::IsNetworkAvailable() == true is not satisfied.

This function sends a request to the server. Function calls that involve sending requests to the server have restrictions on how frequently they may be called. While this function still returns immediately if this limit is exceeded, the start of the request processing will be delayed.

Result nn::ec::AsyncConsumeRightDataRequest::End ( )

Waits for license information consumption to complete.


Return Values:

nn::ResultSuccess The process succeeded.

nn::ec::ResultShopServiceAccessCanceled Indicates that the process was canceled on request from the caller.

nn::ec::ResultShopServiceAccessInsufficientWorkMemory Insufficient work memory for ShopServiceAccessor.

nn::ec::ResultInsufficientWorkMemory Insufficient work memory for ConsumableServiceItemManager. During initialization, pass work memory of nn::ec::RequiredWorkMemorySizeForConsumableServiceItem bytes or greater.

nn::ec::ResultShopServiceAccessRequestTimeout Indicates that the request to communicate with the server timed out.

nn::account::ResultNetworkServiceAccountUnavailable Indicates that the network service account for the specified user account is unavailable. See the section on the handling of errors in the documentation for the account library and make the network service account available.

nn::ec::ResultShowErrorCodeRequired Indicates that a response error code, that requires the display of the error viewer, has been received from the connected shop service server.

Precondition
  • Begin() was already executed.

Details
If the asynchronous processing is incomplete, the system waits for completion in the function.

This function may return an nn::ec::ShopServiceAccessor::AsyncResponse error. For more information about this error handling, see the ShopServiceAccessor::AsyncResponse reference.

If this function succeeds, the item ID for which the license was consumed can be obtained with GetProvidableItemIds(). If this function fails, the manager is in a state where it must determine whether it is possible to provide the license information (IsRecoveryRequired() == true). Run AsyncRecoverRightDataRequest to restore the license information that can be provided.
