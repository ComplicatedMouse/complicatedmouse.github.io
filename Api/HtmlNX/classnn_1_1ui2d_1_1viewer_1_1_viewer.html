◆ DrawFirstFrameCaptureTextureAndUpdateLayout()

void nn::ui2d::viewer::Viewer::DrawFirstFrameCaptureTextureAndUpdateLayout ( nn::gfx::Device *  pDevice,
nn::gfx::CommandBuffer &  commandBuffer,
int  constantBufferIndex 
)

Updates the layout data, including the capture texture processing for the first frame.

Details
Calling this method executes processing equivalent to capture processing for the first frame capture texture and UpdateLayout in the current frame. In the viewer library, UpdateLayout maps and unmaps the constant buffer, so it is not possible to perform UpdateLayout & DrawCaptureTexture for the capture texture at the 0th frame and then UpdateLayout in the actual frames. For this reason, a specialized method for the first frame capture texture is available to include all of this processing in the mapping/unmapping process for the constant buffer.

Parameters
[in]pDeviceDevice.
[in]commandBufferCommand buffer to add commands.
[in]constantBufferIndexIndex of the constant buffer to update.

◆ UpdateInputs()

void nn::ui2d::viewer::Viewer::UpdateInputs ( nn::gfx::Device *  pDevice,
const InputDeviceState &  inputDeviceState 
)

Runs the viewer input process.

Parameters
[in]pDeviceDevice.
[in]inputDeviceStateInput device status.

Details
Internally, this function acts to switch viewer features, switch animation playback processes, and update button groups (UpdateButtonGroup()) in association with device input. The switching of viewer features and the switching of animation playback processes (play, stop, control frames, and switch animations) are processes just like those of the SDK's standard viewer. You do not need to use the processes of the standard viewer when creating a custom viewer, so call UpdateButtonGroup() on its own.
