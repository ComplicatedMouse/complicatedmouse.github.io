A sample program for the mutex feature.

Overview

This sample shows the mutex feature.

For more information about using the mutex feature, see the Mutex Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsMutex/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Two threads are created with nnMain().
  • The two threads both use ThreadSample().
  • With ThreadSample(), in the one million loops, g_SharedCounter, which is the conflict resource, is incremented.
  • Execution waits for the two threads to complete in nnMain().
  • Last, the g_SharedCounter value is displayed.
  • The two threads are destroyed by nnMain().

With the two threads, when g_SharedCounter is incremented, because LockMutex() and UnlockMutex() are being performed during the lock interval, they are excluded from other threads.

For this reason, the final g_SharedCounter value is displayed as follows.

g_SharedCounter = 2000000
Ok.

Meanwhile, if the sample program is run with LockMutex() (A) and UnlockMutex() (B) deleted (or commented out), the final g_SharedCounter value is not shown, but the following message is displayed and an assertion failure occurs. Note: The displayed counter value will differ according to the environment being executed.

g_SharedCounter = 1368729
*** Invalid counter value!

In this way, a locked interval is created by using the mutex feature and resources that are normally exclusive can be safely used by operating with shared resources in the locked interval.
