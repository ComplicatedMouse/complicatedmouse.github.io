Detailed Description

The audio renderer parameters.

Details
You can get the nn::audio audio renderer by using nn::audio::OpenAudioRenderer(). When getting it, you must either specify this structure as the parameter argument or else specify the work buffer used by the audio renderer in the workBuffer parameter. The buffer size required for this workBuffer changes depending on the various values specified in parameter.

You can get the workBuffer size by initializing and configuring this structure using the following procedure.

  • Initialize this structure with nn::audio::InitializeAudioRendererParameter().
  • Set the required values for each member variable based on the application design.
  • Specify the structure for which the member variables have been specified using the nn::audio::GetAudioRendererConfigWorkBufferSize() and get the size.

The meanings of each member variable are as follows.

  • sampleRate is the sample rate of the sample data output by the audio rendered. The audio renderer output from FinalMixType will be the sample rate specified here. If the sample rate supported by the platform differs from this value, the audio renderer will convert it to an appropriate sample rate.
  • sampleCount is the number of samples maintained by a single mix buffer. For more information about the mix buffers, see Mix Buffers in the Audio Renderer documentation.
  • mixBufferCount is the maximum number of mix buffers that are used by the audio renderer. The total number of mix buffers that can be used by each FinalMixType and SubMixType is determined by this member variable.
  • subMixCount is the maximum number of SubMixType instances used by the audio renderer. Set the maximum number that can be used based on the application design.
  • voiceCount is the maximum number of voice units that the audio renderer uses. The voice unit is the unit for the internal resources used for VoiceType processing. When you create a VoiceType object, the number of voice units used by the audio renderer corresponds to the VoiceType setting. For more information about how the number of voice units used corresponds to the VoiceType setting, see the description of the nn::audio::AcquireVoiceSlot() function.

The maximum number of simultaneous sounds that are played by the audio rendered is determined by this member variable. However, in some cases the number of sounds that are played may be restricted at a number lower than the specified value due to the processing load for each platform and the audio playback restriction feature based on the VoiceType priority.

  • sinkCount specifies the number of connection destinations for audio renderer output. For more information, see SinkType and FinalMixType.
  • effectCount is the maximum number of effects that can be used by the audio renderer. Depending on the effect, a separate work buffer may be required other than the buffer specified as workBuffer in nn::audio::OpenAudioRenderer(). For more information, see the descriptions of each effect.
  • performanceFrameCount specifies the maximum number of audio frames for the performance information accumulated by the audio renderer. For more information, see nn::audio::GetRequiredBufferSizeForPerformanceFrames().

There are effective value combinations for each member variable depending on the support state of each platform. You can verify using nn::audio::IsValidAudioRendererParameter() whether each of the specified member variable combinations can be used.
