Detailed Description

An automatically adjustable network clock that can be updated to the system time at any timing.

Details
This clock indicates the current time. You can get a value that can be converted into a time expression that a human can understand. If the clock has already been adjusted, the value can be obtained without connecting the device to the network. The clock time is not affected by automatic system adjustments unless the Adjust function is called, which allows rollbacks and large increases in clock time to be managed appropriately and makes this clock easy to use in applications.

This clock is not affected by modifications to the time made by the user but can be updated to the system time (nn::time::StandardNetworkSystemClock) by calling the Adjust function, at which point the indicated time may be rolled back or forward by more than the actual elapsed time. In addition, although the clock continues to be incremented when the device is turned off, the clock is invalidated and becomes unable to return time values if the power supply is lost as a result of a dead battery. You recover from this state by having the device get the time from the network again. Calculations for elapsed time involving clock adjustments and invalidations may not be strictly accurate, but because there are regular adjustments to the correct time, you can trust those calculations to a certain extent.

This class's member functions are thread-safe. This class satisfies the standard C++ TrivialClock requirements and provides features equivalent to std::chrono::system_clock. However, you cannot detect clock manipulations and invalidations using only the standard C++ interface. In addition, any time obtained during the clock's invalidation remains a fixed value.

Clock Accuracy

This clock is not affected by the automatic time updates executed regularly by the system and is only updated when the Adjust function is called. As a result, time values provided by this clock are typically less accurate than those provided by nn::time::StandardNetworkSystemClock. For more information, see the Network Clock Automatic Adjustment documentation. Use the nn::time::StandardNetworkSystemClock::GetCurrentTime function when a more accurate current time value is needed.

static Result nn::time::AdjustableNetworkSystemClock::GetCurrentTime ( PosixTime *  pOutPosixTime)
static

Gets the current time indicated by the clock.

Parameters
[out]pOutPosixTimeA pointer to a buffer storing the time.
Returns
The result of the operation. If the preconditions are not met, the function fails without returning one of the following enumerated result values.
Return Values:

ResultSuccess Succeeded.

ResultClockInvalid The clock is invalid.

Precondition
  • The time library is initialized.
  • pOutPosixTime != nullptr.

Details

Windows Platform Only:
This function always succeeds in the Windows environment and returns a time value based on the PC's system clock time.

The obtained time value may be different than the latest system time.

Call the Adjust function to update the obtained time to the latest system time.

As long as the power for the device remains on, the network clock will never transition from the state in which this function can return the current time to a state in which this function cannot return the current time. However, the clock may automatically transition from a state in which this function cannot return a value to a state in which this function can return a value.

◆ Adjust()

static void nn::time::AdjustableNetworkSystemClock::Adjust ( )
static

Adjusts the clock time to the latest system time.

Precondition
  • The time library is initialized.

Details
Adjusts the time that can be obtained with the GetCurrentTime function to match the latest system time. Calling this function does not ensure that the GetCurrentTime function will succeed.

Calling this function may cause the time returned by the GetCurrentTime function to be rolled back or rolled forward by a large amount.
