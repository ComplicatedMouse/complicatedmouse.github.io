Overview

Describes the sample application that provides an example of calling the controller support applet.

File Structure

This sample program is located in Samples/Sources/Applications/HidControllerSequence/ .

Required Environment

Pair the controller to the development hardware ahead of time.

How to Use

Run the sample application.

The controller support applet is called immediately after starting when no controller has been connected. The start screen appears when the controller is connected. You move to the title screen with button input from any of the connected controllers.

You can move among the options using either the up and down directional buttons or the control stick. Press the A Button to start the game with the number of players displayed where the cursor is pointing. When there are not enough connected controllers for the number of players, the controller support applet is called.

You can manipulate players using either the right and left directional buttons or the control stick on the game screen. The interrupt dialog box appears when you press the + or - Buttons using a player's controller. You return to the title screen when you select Return to Title in the interrupt dialog box.

Important Information

Make sure that the controller is sufficiently charged before using it.

Execution Procedure

Build the application and run it.

Description

The sample application includes the following scenes.

  • Start screen
  • Title screen, where you select the number of players
  • Game screen
  • Screens displayed by dialog boxes

If the application is not using the controller support applet, we recommend displaying a message in the application that informs the user to prepare additional controllers if there are not enough controllers when the application starts, when control is returned to the application, or while the application is running.
