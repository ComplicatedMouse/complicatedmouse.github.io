void nn::init::InitializeAllocator ( void *  address,
size_t  size,
bool  isCacheEnable 
)

Initializes a memory allocator.

Parameters
[in]addressStarting address of the memory region to be used by the memory allocator.
[in]sizeSize of the region to be used by the memory allocator.
[in]isCacheEnableWhether to enable the memory cache feature of each thread.
Precondition
  • First time the API is called in the program.
  • size > 0.

Details
Initializes a memory allocator.

By default in the SDK, new and malloc use the allocator initialized by this function for actual memory allocation. As a result, the size of memory that can be allocated using new and malloc is determined by the value specified for size in this function.

The memory allocator uses nn::mem::StandardAllocator in its internal implementation. As a result, specify address, size, and isCacheEnable which are specified by the arguments of this API, by referring to the following API.

  • void nn::mem::StandardAllocator::Initialize(void* addr, size_t size, bool isCacheEnable);

When the default nninitStartup() function is used, the entire allocatable memory heap is obtained, and this API is used to initialize the allocator. You can select whether to enable caching (specify true for isCacheEnable) for each thread in the default nninitStartup(). If you want to change the buffer size used in the memory allocator, see the nninitStartup Sample sample.

If you will be conducting your own memory management, do the following.

  • Define your own nninitStartup() function.
  • Define your own malloc(), free(), calloc(), realloc(), aligned_alloc(), and malloc_usable_size().
  • Do not call the InitializeAllocator() function.

If any of these steps or conditions are not met, a link error occurs at build time. For more information, see About malloc() and Other Functions That Get Memory in the NintendoSDK Documents.

NX Platform Only:
Also see the nninitStartupWithMalloc sample.
Windows Platform Only:
  • This API does nothing and returns in a Windows environment.
  • The standard Windows API functions (malloc(), free(), calloc(), and realloc()) are called in a Windows environment.
  • Depending on your version of VisualStudio, you might not be able to use aligned_alloc().

◆ GetAllocator()

nn::mem::StandardAllocator* nn::init::GetAllocator ( )

Gets the internal implementation class object for the memory allocator.

Precondition
  • That the memory allocator has been initialized using InitializeAllocator().
Returns
Returns the internal implementation class object for the memory allocator.

Details
The memory allocator uses nn::mem::StandardAllocator in its internal implementation. This API returns the nn::mem::StandardAllocator object that the memory allocator is using.

Windows Platform Only:
  • This API always returns NULL in a Windows environment.
