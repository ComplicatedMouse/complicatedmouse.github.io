void nn::g3d::ResModel::Setup ( nn::gfx::Device *  pDevice,
nn::gfx::MemoryPool *  pMemoryPool,
ptrdiff_t  memoryPoolOffset 
)

The use of internal features is prohibited. Sets up the model.

Parameters
[in]pDevicePointer to the device.
[in]pMemoryPoolPointer to the memory pool where ResFile is located.
[in]memoryPoolOffsetOffset from the start of the memory pool where the ResFile index buffers and vertex buffers are located.
Precondition
  • pDevice is initialized.
  • ResFile is loaded to pMemoryPool.

Details
The user loads ResFile to the locally prepared memory pool, which is then used by this function to set up ResFile. This function cannot be used to create memory internally. If ResFile is located in the memory pool, use the alignment of the file. Because ResFile involves CPU access, we recommend setting the memory pool property to CpuCached. Performance will be lower if you set CpuUncached. You cannot use CpuInvisible.

◆ BindTexture()

BindResult nn::g3d::ResModel::BindTexture ( TextureBindCallback  pCallback,
void *  pUserData 
)

Resolves references for the texture.

Parameters
[in]pCallbackPointer to the callback function invoked when resolving texture references.
[in]pUserDataPointer to the user data to be passed as a parameter when the callback is invoked.
Returns
Returns BindResult.

Details
Pass the texture name as an argument and call pCallback. For pCallback, return a pointer to an appropriate nn::g3d::TextureRef value. When an invalid nn::g3d::TextureRef value is returned as a callback value, the texture reference resolution is treated as if it failed. Inappropriate access may occur if the render process is performed without resolving texture references. Check the return value to confirm whether texture reference resolution succeeded.

Callbacks only occur for textures for which there is no reference resolution yet. You can determine whether a reference has been resolved by using the nn::g3d::ResMaterial::IsTextureBound() function.
