A description of the sample program of the virtual address memory features.

Overview

This section describes a sample program that demonstrates how to use virtual address memory features.

For more information about using the virtual address memory features, see the manual for the virtual address memory features and the API Reference for the nn::os namespace.

File Structure

This sample program is located in Samples/Sources/Applications/OsVirtualAddressMemory/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • The nnMain() function calls the VirtualAddressMemorySample() function.
  • VirtualAddressMemorySample() allocates virtual address space.
  • Physical memory is assigned for the allocated virtual address space.
  • Partial freeing of assigned physical memory.
  • Physical memory is assigned to all of the allocated virtual address space, including the region to which physical memory is already assigned.
  • The amount of memory being used by the system for virtual address memory is displayed.
  • The address space is freed.

The VirtualAddressMemorySample() function first allocates 2 MiB for virtual address space. Physical memory is assigned and freed in this allocated virtual address space in the following order.

  • Physical memory is assigned to the middle half of the space (a size of 1 MiB).
  • A portion of the assigned physical memory (the middle 8 KiB) is freed.
  • Physical memory is assigned to the entire space (a size of 2 MiB).

As long as the physical memory is assigned inside the allocated virtual address space, it can be assigned anywhere and not just at the front. In addition, you can free just a portion of the assigned physical memory.

Call the GetVirtualAddressMemoryResourceUsage() function to get the size of memory being used by the system for virtual address memory. For more information, see Determining the Value for SystemResourceSize in the Os Library documentation.

As the last step, the virtual address space is freed. At this time, any physical memory that has been assigned in this space is also freed.
