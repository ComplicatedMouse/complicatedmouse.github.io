◆ Flag

enum nn::g3d::ResShape::Flag

The flag for shapes.

Enumerator
Flag_OwnVertex 

Has ownership of ResVertex.

Flag_SubMeshBoundaryConsistent 

Submesh boundaries between the meshes are maintained.

Flag_NotOwnMesh 

No ownership for the ResMesh array.

Will be changed to OwnMesh when the binary major version is updated.

Flag_UseBoneBoundingSphere 

The use of internal features is prohibited. Keep a bounding sphere for every bone that is affected.

Arrange the content of pRadiusArray as: radius * number of meshes + (center, radius) * number of bones.

The use of internal features is prohibited. Sets up the shape.

Parameters
[in]pDevicePointer to the device.
[in]pMemoryPoolPointer to the memory pool where ResFile is located.
[in]memoryPoolOffsetOffset from the start of the memory pool where the ResFile index buffers and vertex buffers are located.
Precondition
  • pDevice is initialized.
  • ResFile is loaded to pMemoryPool.

Details
The user loads ResFile to the locally prepared memory pool, which is then used by this function to set up ResFile. This function cannot be used to create memory internally. If ResFile is located in the memory pool, use the alignment of the file. Because ResFile involves CPU access, we recommend setting the memory pool property to CpuCached. Performance will be lower if you set CpuUncached. You cannot use CpuInvisible.

◆ IsSubMeshBoundaryConsistent()

bool nn::g3d::ResShape::IsSubMeshBoundaryConsistent ( ) const

Determines whether submesh boundaries between the meshes are retained.

Returns
Returns true when submesh boundaries are maintained, or returns false otherwise.

Details
When submesh boundaries are maintained, the number of submeshes within all meshes is the same and the shapes of the bounding boxes for submeshes indicated by the same submesh index are nearly identical.

Submesh boundaries are retained for LOD models created using the intermediate file optimizer's polygon reduction feature. Submesh boundaries are not retained for LOD models that were created using some external tool and then merged using the intermediate file optimizer.
