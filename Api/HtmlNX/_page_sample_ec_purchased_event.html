Sample That Checks For Purchase Reports

  • Description

Describes the sample program that checks for reports of purchases.

Overview

This sample program can check whether a purchase report arrives when an item has been purchased.

File Structure

This sample program is located in Samples/Sources/Applications/EcPurchasedEvent/ .

Required Environment

There must be at least one user account on the development hardware and it must be associated with a Nintendo Account. This sample program can only be built for and run on development hardware. Note that its use is not supported within a Windows environment.

How to Use

Build and run this program as you would an ordinary sample program. The execution of this program is displayed on the screen as text. Use the following procedure.

A user selection screens appears right after you start the program. Select a user that is linked to the Nintendo Account. Select Cancel to end the process.

After a user has been selected, press the Up Button and Down Button to move the cursor, and then press the A Button to accept and jump to the shop screen where the item can be purchased. If an item is purchased, a report of the purchase arrives and is displayed. Press the B Button to clear all purchase report event information.

Important Information

Nothing specific.

Execution Procedure

Build the sample program and then run it.

Description

Processes occur in the following order.

  • Initialize the feature for detecting the purchase of items on the shop (nn::ec::InitializePurchasedEvent "nn::ec::InitializePurchasedEvent").
  • Get the event that is signaled when an item is purchased on the shop (nn::ec::GetPurchasedEvent "nn::ec::GetPurchasedEvent").
  • Extract the information about the item that was purchased on the shop (nn::ec::PopPurchasedItemInfo "nn::ec::PopPurchasedItemInfo").

For more information about these functions, see the API Reference.

For more information, see the supplemental information found throughout the following source code.

127 // Helper class for managing the expected values of the blink conditions.
128 class BlinkConditionHelper
129 {
130 public:
131  // The blink conditions being managed.
132  enum Condition
133  {
134  Condition_Visible,
135  Condition_Invisible,
136  };
137 
138  // The default blink interval.
139  static const nn::TimeSpan DefaultInterval;
140 
141  // Constructor.
142  explicit BlinkConditionHelper(const nn::TimeSpan& blinkInterval = DefaultInterval) NN_NOEXCEPT
143  : m_Interval(blinkInterval)
144  , m_Origin(nn::os::GetSystemTick())
145  , m_Condition(Condition_Visible)
146  {
147  }
148 
149  // Reset the managed blink condition to the default.
150  void Reset(Condition condition = Condition_Visible) NN_NOEXCEPT
151  {
152  m_Condition = condition;
153  m_Origin = nn::os::GetSystemTick();
154  }
155 
156  // Get the currently expected blink condition.
157  Condition GetCondition() const NN_NOEXCEPT
158  {
159  return m_Condition;
160  }
161 
162  // Update the elapsed time of the blink condition.
163  // Return the currently expected blink condition in the return value.
164  Condition Update() NN_NOEXCEPT
165  {
166  const auto now = nn::os::GetSystemTick();
167  if ((now - m_Origin).ToTimeSpan() >= m_Interval)
168  {
169  m_Origin = now;
170  m_Condition = (Condition_Visible == m_Condition) ? Condition_Invisible : Condition_Visible;
171  }
172  return GetCondition();
173  }
174 
175 private:
176  const nn::TimeSpan m_Interval;
177  nn::os::Tick m_Origin;
178  Condition m_Condition;
179 };
