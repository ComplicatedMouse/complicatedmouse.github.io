Detailed Description

Class for handling readers-writer locks.

Details
A readers-writer lock is a kind of synchronization object that is used for mutual exclusion among threads. This feature is used for mutual exclusion of a shared resource for which many readers threads have read access and a single writer thread has write access.

The nn::os::ReaderWriterLock class is a utility class that wraps calls to synchronization control functions that handle objects of type nn::os::ReaderWriterLockType. The member functions provided by this class are implemented as calls to functions like nn::os::AcquireReadLock() that take nn::os::ReaderWriterLockType-type values as their parameters.

This class fulfills the BasicLockable and Lockable requirements in the C++ standard, so it can be used as a type argument for class templates such as std::lock_guard and std::unique_lock. As locks for these, you get a Writer lock.

In addition, this class has the member functions lock_shared() and unlock_shared(), so it also can serve as a type argument for the std::shared_lock class template. As locks for these, you get a Readers lock.

As a result, multiple member functions can perform the same process, such as ReaderWriterLock::AcquireWriteLock and lock(), and ReaderWriterLock::AcquireReadLock and lock_shared().
