Detailed Description

This class communicates cancellation requests to the thread that executes asynchronous processing.

Details
CancellationTokenSource is a class communicates cancellation requests to the thread that executes asynchronous processing. To be specific, this class provides the following functionality.

  • Gets the CancellationToken object.
  • Sends a cancellation request to the CancellationToken object that was obtained.

By passing the CancellationToken object obtained by this class from on thread (A) to another thread (B) you can send a cancellation request from A to B. Thread B, which receives the cancellation request, can then cancel cooperative asynchronous processing between the two threads.

Note that CancellationTokenSource does not contain the functionality to forcibly terminate asynchronous processing that is running in other threads. The behavior of the thread that receives the cancellation request depends on the implementation of the asynchronous processing that the thread is currently running. For more information about what kind of state in which the thread receiving the cancellation request terminates the process early, or the execution results that are returned, see the various asynchronous API references.

This class requires dynamic memory allocation to share internal states with the CancellationToken object. The allocated memory is released when all CancellationToken objects that reference the internal state and CancellationTokenSource destructors have been called.

◆ RequestCancel()

void nn::async::CancellationTokenSource::RequestCancel ( )

Sends a cancellation request to the CancellationToken object.

Details
Sends a cancellation request to all CancellationToken objected obtained by the GetToken() function.

If RequestCancel() has already been called, nothing happens. In addition, cancellation requests that have already been made cannot be canceled.
