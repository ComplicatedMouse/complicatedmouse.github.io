Detailed Description

Represents scalable fonts.

Details
The fonts can be used in the same way as fonts like ResFont that inherit nn::font::Font but with these differences: characters cannot be displayed after the class has been created, and you must explicitly specify characters needed for display and then plot them in the texture cache.

◆ RegisterGlyph() [1/2]

bool nn::font::ScalableFont::RegisterGlyph ( uint32_t  code,
int  lockGroup 
) const

Registers the specified glyph as a character to draw.

Details
Characters registered as those to be drawn must be plotted anew and registered as such if they are not in the texture cache. These characters become available after UpdateTextureCache and CompleteTextureCache have finished.

Characters registered as those to be drawn can be used as is without any plotting when they are already in the texture cache. These characters are registered as required, so they will not be deleted.

There are a total of 32 groups, numbered 0 through 31, for locked characters. Characters are locked at the group level. As a result, for example, when characters already registered in group 0 are registered again in group 1, both would be locked and both would need to become unlocked before they could be cleared.

Parameters
[in]codeThe glyph to register as a character to draw.
[in]lockGroupThe group when locking glyphs. Specify a numeric value in the range from 0 to 31. If 1 is specified, glyphs are not locked.
Returns
Returns true if the registered glyph needs to be plotted. In other words, that glyph can be drawn as is without executing UpdateTextureCache when false is returned.

◆ GetCharWidth()

virtual int nn::font::ScalableFont::GetCharWidth ( uint32_t  c) const
virtual

Gets the character width of a character.

Details
If the character has already been plotted, this function returns the width of that character. If the character has not been plotted yet, the operation of the function differs depending on what was configured for InitializeArg at the time of initialization.

If both isAlternateCharSpaceWithOriginalWidth and isAlternateCharSpaceWithOriginalWidthForNotReadyChar are false, the process is set to Always use the alternate character in substitution for a character that is not plotted. In this case, the GetCharWidth function returns the width of the alternate character as the width of the unplotted character. In this way, the width of the character that gets drawn is the same as the GetCharWidth result.

On the other hand, the process might be set to either Always handle unplotted character as blank space with the same width as the character that otherwise would be displayed or Use alternate character if the unplotted character is not in the font and if in the font then handle as a blank space with the same width as the character that otherwise would be displayed. In either of these cases, the GetCharWidth function gets the width of the unplotted character directly from the scalable font engine and returns this width, rather than the width of the alternate character.

If the function is getting the width from the scalable font engine, it is slower than if the character were in the texture cache. Note: On the Wii U console this takes about five microseconds per character.

This function causes an assertion error when the scalable font engine runs out of memory. Increase the InitializeArg::noPlotWorkMemorySize for TextureCache in such cases.

Parameters
[in]cCharacter code of the character whose width is to be found.
Returns
Returns the character width, in pixels.

Implements nn::font::Font.
