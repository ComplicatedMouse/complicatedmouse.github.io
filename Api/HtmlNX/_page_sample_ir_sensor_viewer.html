Getting the State of the IR Motion Camera Sample

This sample program uses the nn::irsensor library to get the value of the Joy-Con IR Motion Camera in each mode and displays the results.

Overview

This sample program presents a visualization of the data obtained from the IR Motion Camera.

File Structure

This sample program is located in Samples/Sources/Applications/IrSensorViewer/ .

Required Environment

When running in a Windows environment, connect the Bluetooth dongle to the computer ahead of time, and pair the controller to the computer. When running in an SDEV or EDEV environment, pair the controller to the SDEV or EDEV ahead of time.

Using the Demo

A window opens when you run the sample program. The status of the IR Motion Camera is displayed above the window, When you operate the controller, you can see how the state changes.

To end the sample program, press either the + Button or the - Button.

Important Information

Make sure that the controller is sufficiently charged before using it. The only supported controller style is JoyRight. We recommend making function calls from a thread other than the main loop because the IR Motion Camera API has functions that block the thread for extended periods. For more information, see the API Reference.

Execution Procedure

Build the sample program and then run it.

Description

  • Get the Npad.
  • Get the IR Motion Camera handle from the Npad.
  • Get the default settings for moment mode, clustering mode, and image transfer mode.
  • Start the IR Motion Camera in moment mode.
  • Begin the loop.
  • Get input from Npad.
  • Update the Config settings according to the Npad input.
  • If the R Button on the Npad is pressed, adjust the IR Motion Camera to reflect the switched mode and the Config settings.
  • Gets the data for the mode that is currently enabled by the IR Motion Camera.
  • Make the acquired data visible.
  • Return to the start of the loop.
