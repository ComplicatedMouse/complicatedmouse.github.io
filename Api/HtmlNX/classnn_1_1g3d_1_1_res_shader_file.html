◆ Unrelocate()

void nn::g3d::ResShaderFile::Unrelocate ( )

Puts ResShaderFile in a relocatable state.

Details
After Unrelocate() has been called, resource files can be moved to a different memory address. Before moving, you must call Cleanup() for ResShaderArchive. To make it available again, you must call Relocate() or ResCast().

◆ Setup() [1/4]

void nn::g3d::ResShaderFile::Setup ( nn::gfx::Device *  pDevice)

Sets up the shader archive.

Parameters
[in]pDevicePointer to the device.
Precondition
  • pDevice is initialized.

Details
Calls Setup() for the placed shader archives.

Shader initialization is not performed. Shader initialization will be delayed until the nn::g3d::ResShaderProgram::Update() function has been called. nn::g3d::ResShaderProgram::Update() is called from the nn::g3d::ShaderSelector::UpdateVariation() function.

Use this if the same shading model is not being handled by multiple threads, or if the same shading model is being handled by multiple threads but mutual exclusion is implemented manually by the user.

Examples
G3dDemo/Edit.cpp, G3dDemo/NnsG3dSimple.cpp, G3dDemo/Outline.cpp, G3dDemo/Parallel.cpp, G3dDemo/ShapeAnimation.cpp, G3dDemo/Simple.cpp, G3dDemo/SkeletalAnimation.cpp, G3dDemo/Tessellation.cpp, and G3dDemo/Town.cpp.

Sets up the shader archive.

Parameters
[in]pDevicePointer to the device.
[in]pMemoryPoolPointer to the memory pool where ResShaderFile is located.
[in]memoryPoolOffsetOffset in the memory pool where ResShaderFile is located.
[in]memoryPoolSizeThe size of the memory pool where ResShaderFile is located.
Precondition
  • pDevice is initialized.
  • Resources are loaded to pMemoryPool.

Details
Calls Setup() for the placed shader archives.

The user loads ResShaderFile to the locally prepared memory pool, which is then used by this function to set up ResShaderFile. This function cannot be used to create memory internally. If you are placing ResShaderFile in a memory pool, place it so the start of the shader file matches the file alignment. To get the alignment of the file, get the header using GetFileHeader() and then call nn::util::BinaryFileHeader::GetAlignment(). CPU access takes place in ResShaderFile, so we recommend setting the memory pool properties to CpuCached | GpuCached | ShaderCode. Performance will be lower if you set CpuUncached. You cannot use CpuInvisible.

Shader initialization is not performed. Shader initialization will be delayed until the nn::g3d::ResShaderProgram::Update() function has been called. nn::g3d::ResShaderProgram::Update() is called from the nn::g3d::ShaderSelector::UpdateVariation() function.

Use this if the same shading model is not being handled by multiple threads, or if the same shading model is being handled by multiple threads but mutual exclusion is implemented manually by the user.
