Describes the sample program for audio effects that use Aux.

Overview

This sample program uses Aux and the audio effects implemented as CPU processing to apply effect processing to playback samples.

File Structure

This sample program is located in Samples/Sources/Applications/AudioEffectAuxUtility/ .

Required Environment

You must be able to use audio output.

How to Use

Sound effects can be played back using input from the keyboard or DebugPad.

Input Operation
A Button Plays SampleSe0 once.
[B] Plays SampleSe1 once.
[R] Updates the AuxI3dl2Reverb effect.
[X] Updates the AuxReverb effect settings.
[Y] Updates the AuxDelay effect settings.
Up/Down Buttons Adjusts the sound effect volume for the effect target.
Left/Right Buttons Adjusts the pan to the left and right for the effect target.
[L] Switches the sound effect for the effect target.
START or SPACEBAR Quits the program.

Important Information

Nothing specific.

Execution Procedure

Build the sample program and then run it.

Description

This sample program performs audio effect processing using Aux.

It shows how to use Aux effects.

Aux Effects

The aux effect feature allows you to get audio sample data from the audio renderer and to write that data back. This feature enables the developer to insert proprietary signal processing.

The following flow is used.

  • Allocate the buffer to use for aux effects.
  • Add the aux effect to SubMix and FinalMix.
  • Get the sample data from the audio renderer.
  • Perform signal processing. (For the purposes of this sample, this processing consists of simple volume and pan adjustments.)
  • Write the sample data back to the audio renderer.

First allocate the buffer to use for the aux effect. Developers will process the sample data that is written to this buffer. Increasing the size of the buffer increases the maximum delay amount. Yet if the buffer is too small, it results in the sound getting cut off, because the developer has too little time for proprietary processing.

Next, add the aux effect to SubMix and FinalMix. Also set the channels to send the Aux effects to. Data taken from the audio renderer is block-interleaved. You can include just channel blocks for the specified channels.

Finally, signal processing occurs within the main loop. If the processing takes too much time, the sound cuts out. The developer must also write back to the audio renderer the same number of sample data units that were obtained from the audio renderer.

AuxI3dl2Reverb Effects

The I3dl2Reverb effect is a reverberation effect that partially complies with the Interactive 3D Audio Level 2 specification.

For more information about usage, see nn::audio::InitializeAuxI3dl2Reverb() and nn::audio::ProcessAuxI3dl2Reverb().
