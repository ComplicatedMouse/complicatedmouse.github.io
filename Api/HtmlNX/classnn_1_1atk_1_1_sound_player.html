Detailed Description

The player class for playing sounds.

Details
To get an instance of the SoundPlayer class, call SoundArchivePlayer::GetSoundPlayer.

A single SoundPlayer instance can play multiple sounds. You can set the number of sounds that can be played simultaneously. If this number is exceeded, the player priorities assigned to each sound are compared, and the sound with the lowest priority is stopped.

SoundPlayer can have the following effects on sounds played on the player. 

   - Volume: @ref SetVolume, @ref StopAllSound, @ref PauseAllSound, @ref SetOutputVolume
   - LPF: <tt>@ref SetLowPassFilterFrequency</tt>.
   - Biquad filter: <tt>@ref SetBiquadFilter</tt>.
   - Output Lines: SetDefaultOutputLine

In addition to these effects, these parameters can be affected by the following.

  • Effects of data specified by SoundMaker
  • Effects of SoundHandle or SoundActor
  • Effects of 3D sound calculations (Sound3DEngine))

◆ StopAllSound()

void nn::atk::SoundPlayer::StopAllSound ( int  fadeFrames)

Stops all sounds.

Details
Stops all sounds being played by the player. Performs processing identical to that when stopping a sound by using the handle class.

You can fade out over the number of frames specified by fadeFrames. Fade-out is not performed when 0 is specified. However, when a sequence sound is being played, the envelope release is played and all attenuation is completed before the sound is stopped.

The volume control for fade-out is also used for fade-in. Number of elapsed frames during a fade out. The number of fade-out frames correlates to the speed of change from maximum volume to silence. If a fade out is specified during a fade in, or in other similar situations, it may complete sooner than the specified frame count.

Parameters
[in]fadeFramesNumber of elapsed frames during a fade-out. The number of frames is based on the number of calls to SoundArchivePlayer::Update. Specify a value of 0 or greater for the fadeFrames parameter. If you specify a negative value, the function works the same as if you had specified 0.

◆ SetVolume()

void nn::atk::SoundPlayer::SetVolume ( float  volume)

Changes the player volume.

Details
The player's audio volume affects all sounds currently being played by the player.

The value specified by this function operates independently from any other volume parameter set for the sound and they are all layered.

The volume parameter takes a ratio of 0.0 or greater. Volume does not change if 1.0 is specified. If 0.0 is specified, there is no sound. The default value is 1.0.

Note that after this value is combined with the other volume parameters, the final volume is clamped to the range from nn::audio::VoiceType::GetVolumeMin() to nn::audio::VoiceType::GetVolumeMax(). For more information about combining volume parameters, see Sound Parameter Control Operations.

Parameters
[in]volumeThe player volume ratio (0.0 or greater).

◆ SetBiquadFilter()

void nn::atk::SoundPlayer::SetBiquadFilter ( int  type,
float  value 
)

Changes the biquad filter setting of the player.

Details
The player's biquad filter affects all sounds currently being played by the player.

The biquad filter parameter can be configured in multiple locations, but the settings are not layered. Rather, the biquad filter takes the value that is set in the location with the highest priority, as follows. If the biquad filter parameter is set in a higher-priority location, the settings in locations of lower priority are overwritten. 

   -# The setting in the sound handle
   -# The setting in the sound player
   -# The setting in the ambient parameter structure
   -# The setting in the sequence data

type is the filter type, and it takes an enumerator of the BiquadFilterType enumerated type. In addition to the presets, you can also select a user-registered filter.

value is the extent to which the filter is applied, specified in the range from 0.0f to 1.0f. The meaning of this value varies depending on the type of coefficient used by the filter.

When used together with the low-pass filter from SetLowPassFilterFrequency, the effects are overlaid.

Parameters
[in]typeThe filter type (BiquadFilterType_Min to BiquadFilterType_Max).
[in]valueHow to apply the filter (0.0f to 1.0f).

◆ SetDefaultOutputLine()

void nn::atk::SoundPlayer::SetDefaultOutputLine ( uint32_t  lineFlag)

Sets the output line for sounds played by the player.

Details
When sounds are played, they are sent to the output line set by this function. You can also set a different output line for each sound by calling SoundHandle::SetOutputLine.

This function cannot change the output line of sounds that are currently being played with the player. The new setting for the output line is applied to sounds that the player plays subsequent to the change made by this function. To change the output line while sounds are playing in the player, call SoundHandle::SetOutputLine for each sound.

Output lines are specified as the logical OR of the OutputLine bit flags. The specified value overwrites the value set previously. In the default state, only nn::atk::OutputLine_Main is specified.

◆ SetMainSend()

void nn::atk::SoundPlayer::SetMainSend ( float  send)

Changes the main send volume of the player.

Details
The player's send volume affects all sounds currently being played by the player. The value specified by this function operates independently from any other volume parameter set for the sound and they are all layered.

The main send is a parameter that adjusts the volume of sounds sent to the main output. This parameter is mostly used to adjust the balance of the dry and wet components of an effect.

The send parameter specifies a relative change in the send volume, and the ultimate send volume is the sum of the values specified by the various functions and SoundMaker, clamped in the range from 0.0 to 1.0. The values configurable in SoundMaker are interpreted in the runtime as values in the range from 0.0 to 1.0.

In other words, a specification of 0.0 for the send volume represents a setting that does not change the send volume specified by the various functions and SoundMaker. A specification of 1.0 for the send volume represents a setting that adds to the send volume so the maximum volume is sent to the bus, regardless of the value set by SoundMaker. A specification of â€“1.0 for the send volume represents a setting that subtracts from the send volume so that no send volume is sent to the bus, regardless of the value set by SoundMaker.

The default value is 0.0.

Parameters
[in]sendRelative change in the send volume, taking 0.0 as a reference point.

◆ SetPlayableSoundCount()

void nn::atk::SoundPlayer::SetPlayableSoundCount ( int  count)

Sets the number of sounds that can be played at the same time.

Details
If the number of sounds for playback exceed the configured sound count, the player priorities for each sound are compared, and the sounds with the lowest priority are stopped.

If you use the SoundArchivePlayer class, the number of simultaneously playable sounds specified in the sound archive is automatically set at initialization time.

When using the player heap, the upper limit on the number of sounds that can be played back simultaneously is limited to the number of sounds specified for simultaneous playback during setup. If a value greater than the upper limit is specified, it is reduced to the upper-limit value.

Parameters
[in]countThe number of sounds that can be played simultaneously.
