A sample program that uses the decompression library.

Overview

This sample program uses the decompression library to batch decompress compressed data.

For more information about using the decompression library, see the utility library API reference for the nn::util namespace.

File Structure

This sample program is located in Samples/Sources/Applications/UtilDecompression/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

In the sample program, batch decompression occurs for SampleData, which was originally a string and is now compressed data in GZIP format. The decompression library supports two other formats: ZLIB and RAW DEFLATE.

This sample program has the following flow.

  • Write the size of SampleData, which serves as the source for the decompression data, to the log.
    • The compression source data for SampleData is a string that was compressed using Compression Sample.
  • Get the size of the data after decompression using the decompression library's nn::util::GetGzipDecompressedSize() function.
    • It is possible to get the decompressed data size from the compressed data before performing decompression.
    • This feature is only available with the GZIP format.
  • Allocate the pDecompressedBuffer memory region as a buffer for the decompressed data.
    • Get the required size from the nn::util::GetGzipDecompressedSize() function.
  • Allocate a memory region for the work buffer, pWorkBuffer, required for batch decompression.
    • Get the required size from the nn::util::DecompressGzipWorkBufferSize() function.
  • Call the decompression library's nn::util::DecompressGzip() function and batch decompress the data.
    • Store the decompressed data in pOutDecompressedBuffer.
  • Write the string that results from the decompression to the log.

The following file shows the execution results of this sample program. The string that is written to the log matches the SampleData string from Compression Sample.
