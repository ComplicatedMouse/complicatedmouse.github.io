Detailed Description

Namespace for the program diagnostics library.

Details
This library is for running diagnostics on program behavior. You can reference the macro that defines the program-diagnostic library from List of Macros.

Windows Platform Only:
This library operates in a Windows environment by referencing PDB files. Note that this library does not work correctly if PDB files or the files that are referenced by PDB files are unavailable.

Typedef Documentation

◆ AbortObserver

typedef void(* nn::diag::AbortObserver) (const AbortInfo &abortInfo)

Defines the type of the abort observer.

Parameters
[in]abortInfoThe data for aborts.

Details
The abort observer is a callback function called during aborts.

The callback function must be re-entrant because it might be called simultaneously from multiple threads.

It may be called by a function that is not exception-safe. This call must not generate an exception.

Functions of this type are assumed to run logging and similar processes.

The behavior of the abort observer, when it does not return, is undefined.

Attention
The abort observer does not become disabled, even in the production environment.

◆ AssertionFailureHandler

typedef AssertionFailureOperation(* nn::diag::AssertionFailureHandler) (const AssertionInfo &assertionInfo)

The type of the callback function that is called when the conditions passed to an NN_ASSERT macro are not satisfied.

Parameters
[in]assertionInfoThe data for the assertion.
Returns
Returns the process specified after returning from the function using nn::diag::AssertionFailureOperation.

Details
The return value can be used to specify the processing that takes place after returning from the function. In general, select Abort. If you select Continue, extra care is required because execution will continue even if the conditions passed to the NN_ASSERT macros are not satisfied.

The callback function must be re-entrant because it might be called simultaneously from multiple threads.

It may be called by a function that is not exception-safe. This call must not generate an exception.

Functions of this type are assumed to run logging and similar processes.

◆ GetBacktrace() [1/3]

int nn::diag::GetBacktrace ( uintptr_t *  pOutArray,
int  arrayCountMax 
)

Gets the back trace of the called thread.

Parameters
[out]pOutArrayThe array that stores the addresses.
[in]arrayCountMaxThe maximum number that can be stored in pOutArray.
Returns
Returns the number of addresses stored in pOutArray.
Precondition
  • pOutArray != nullptr.
  • arrayCountMax > 0.

Details
pOutArray stores return addresses for functions in the backtrace, in the reverse order they were called in, with the most recent first.

Addresses for older stack frames might have been discarded when the return value is equal to arrayCountMax.

Pass an array that is sufficiently large enough to get a complete backtrace.

Note that pOutArray also includes return addresses for functions implemented internally by the SDK.

NX Platform Only:
This function does not work correctly if you specify the -fomit-frame-pointer option for the build.
Examples
BreakpadSimple/BreakpadSimple.cpp, CapstoneSimple/CapstoneSimple.cpp, and OsUserExceptionHandlerSample.cpp.

int nn::diag::GetBacktrace ( uintptr_t *  pOutArray,
int  arrayCountMax,
uintptr_t  fp,
uintptr_t  sp,
uintptr_t  pc 
)

Gets the backtrace from the register information.

Parameters
[out]pOutArrayThe array that stores the addresses.
[in]arrayCountMaxThe maximum number that can be stored in pOutArray.
[in]fpThe frame pointer of the thread getting the backtrace.
[in]spThe stack pointer of the thread getting the backtrace.
[in]pcThe program counter of the thread getting the backtrace.
Returns
Returns the number of addresses stored in pOutArray.
Precondition
  • pOutArray != nullptr.
  • arrayCountMax > 0.

Details
The assumed use is for getting a backtrace for threads where exceptions occurred, from the user exception handler.

Call this function, passing the register information for the thread where an exception occurred, based on the exception data in the user exception handler.

pOutArray[0] stores pc, the program counter value.

From pOutArray[1] onward, the return addresses for functions in the backtrace are stored, in the reverse order they were called in, with the most recent first.

Addresses for older stack frames might have been discarded when the return value is equal to arrayCountMax.

Pass an array that is sufficiently large enough to get a complete backtrace.

Note that pOutArray also includes return addresses for functions implemented internally by the SDK.

NX Platform Only:
This function does not work correctly if you specify the -fomit-frame-pointer option for the build.
Windows Platform Only:
This API cannot be used in a Windows environment.

uintptr_t nn::diag::GetModulePath ( char *  outPathBuffer,
size_t  pathBufferSize,
uintptr_t  address 
)

Gets the path of the NSS file used to create the module that includes the address specified in the parameter.

Parameters
[out]outPathBufferThe buffer storing the path.
[in]pathBufferSizeThe size, in bytes, of the buffer for storing the path.
[in]addressThe address to use for the module search.
Returns
Indicates the address of the module. If no module can be found, the function returns 0.
Precondition
  • outPathBuffer != nullptr
  • pathBufferSize > 0
  • address > 0.

Details
outPathBuffer stores the obtained module path. outPathBuffer is null-terminated. If the path is longer than pathBufferSize, up to (pathBufferSize - 1) bytes of the path are stored. The character code of the path is UTF-8. If the module is a system module, the module name, not the path, will be obtained.

Examples
ImagePngEncoding/ImagePngEncoding.cpp.
