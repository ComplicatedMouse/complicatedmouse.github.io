Detailed Description

The vibration node that provides functionality as a source for vibration values by playing back vibration files.

Details
VibrationPlayer is the vibration node providing the functionality to generate vibrations by referencing the binary data of the vibration file that has been expanded in the memory of the application.
The application must expand the binary data of the vibration file to play ahead of time in its memory, and make sure that this memory is not overwritten until playback is finished.
Vibration files are parsed using the nn::hid::ParseVibrationFile function, and then vibration values are read out in order from the vibration file using the nn::hid::RetrieveVibrationValue function.

The vibration values read out from the vibration file are transmitted to the vibration node of the connection target every time nn::hid::VibrationNode::Update is called.

◆ Play()

void nn::hid::VibrationPlayer::Play ( )

Begins playback of the vibration file that was referenced by VibrationPlayer.

Precondition
  • IsPlayable() == true.
  • IsPlaying() == false.
  • 0 <= GetCurrentPosition() < GetFileInfo().sampleLength.
  • If IsLoop() == true, then 0 <= GetLoopStartPosition() < GetLoopEndPosition() <= the sample length of the vibration file being referenced.
Postcondition
  • IsPlaying() == true.

Details
Decodes the vibration file referenced by VibrationPlayer and generates vibration values, starting in order from CurrentPosition.
When VibrationPlayer is connected to another vibration node, the played vibration values are sent to that vibration node every time nn::hid::VibrationNode::Update is called.

When IsLoop() is false, playback stops automatically when it completes at the end of the vibration sample within the vibration file, at which point the values are as follows: IsPlaying() is false and GetCurrentPosition() is 0.
When IsLoop() is true, the vibration file continues to loop playback until the Stop() function is called explicitly.

Parameters
[in]speedThe speed ratio when playing a vibration file.
Precondition
  • speed >= 0.0f.
Postcondition
  • GetPlaySpeed() = speed.

Details
If you specify 1.0f, the file is played at its original speed. If you specify 2.0f, the play time is reduced to half without changing the frequency. If you specify 0.5f, the play time is doubled without changing the frequency.

◆ Update()

static void nn::hid::VibrationNode::Update ( )
staticinherited

Updates the entire directed graph for the vibration value flow, and transmits vibration values from the separate vibration nodes to the nodes they are connected to.

Details
Calling this function updates the internal states of all vibration nodes. When this occurs, vibration values are transmitted between adjacent vibration nodes, from connection source to connection target. Connection source nodes are updated before connection target node.

You can call this function at any time, but we recommend you use a timer and call it at an interval of DefaultVibrationSampleInterval.

This function is not thread-safe. Use mutual exclusion to prevent the constructor and destructor of this class and derived classes from being called simultaneously.
