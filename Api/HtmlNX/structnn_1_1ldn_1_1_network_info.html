Detailed Description

The structure that stores scan results.

Details
This structure stores network information and can be used to store the Scan function results or as an argument for the Connect function. ldn library users do not need to change the contents of this structure.

While valid values will always be stored in common, networkId and ldn are not parameters for general access points, and are only enabled when common.networkType == NetworkType_Ldn is satisfied.

This type of variable requires a large memory area, so we do not recommend allocating it in the stack. If it is allocated in the stack, be careful to prevent the stack from overflowing during execution.

Examples
