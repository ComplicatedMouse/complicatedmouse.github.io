A sample program dealing with nn::time::StandardUserSystemClock.

Overview

This sample deals with nn::time::StandardUserSystemClock. This clock indicates the current time held by the device. You can get a value that can be converted into a time expression that a human can understand.

For more information, see StandardUserSystemClock.

File Structure

This sample program is located in Samples/Sources/Applications/TimeStandardUserSystemClock/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Initialize the time library.
  • Run the sample that gets the Unix time (POSIX) using nn::time::PosixTime, and convert that number into a date and time that humans can understand using nn::time::CalendarTime.
  • Run the sample that detects whether the time has been manipulated.
  • Finalize the time library.

By permanently storing the nn::time::SystemClockContext value somewhere like save data, the feature for detecting whether the time has been manipulated can even detect manipulations of the time spanning over events like shutting down the application and powering down the system.

For functions that return nn::Result, NN_ABORT_UNLESS_RESULT_SUCCESS is used to handle cases where success always occurs because the pre-conditions are met.
