Detailed Description

API declarations for light message queues.

Details
A light message queue is a feature for establishing synchronization among threads by sending and receiving data.

Light message queues differ in the following manner from the message queue feature that supports nn::os::MessageQueueType.

  • When the message queue is in a NotFull state when sending data, the local thread does not enter a waiting state due to other light message queue processing functions.
  • When the message queue is in a NotEmpty state when receiving data, the local thread does not enter a waiting state due to other light message queue processing functions.
  • You cannot specify this as the target for multiple wait.
