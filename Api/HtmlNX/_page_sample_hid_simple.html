This sample program shows how to use the nn::hid library to get the states of human interface devices.

Overview

This sample program shows how to display the states of a gamepad, a debug controller, and a touch screen.

File Structure

This sample program is located in Samples/Sources/Applications/HidSimple/ .

Required Environment

In the Windows environment, use an XInput-compatible gamepad for gamepad operations, use the keyboard for debug controller operations, and use the mouse for touch screen operations. Up to four gamepads can be used. The fifth is recognized as a debug controller. In environments where Windows Touch messages are enabled, that controller is also treated as touch screen input.

How to Use

A window opens when you run the sample program. The window displays the states of the different HID devices, and when you operate the gamepad or debug controller, you can see how the state changes. For the touch screen, this window displays touch information when a touch is detected.

Important Information

Nothing is displayed in the console window. The initial values assigned to keys assume that the gamepad is an XInput-compatible gamepad, and that the debug controller uses the ELECOM JC-W01UWH.

Execution Procedure

Build the sample program and then run it.

Description

The process flow is described in the comments of the nns::hid sample library source code.
