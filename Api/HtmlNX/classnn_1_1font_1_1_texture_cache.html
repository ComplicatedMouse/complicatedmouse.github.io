Detailed Description

The class representing the texture cache for scalable fonts.

Details
Multiple scalable font classes can use a single texture cache. In that case, multiple font sizes are mixed in a single texture cache.

◆ RegisterGlyph() [1/2]

bool nn::font::TextureCache::RegisterGlyph ( uint32_t  code,
uint32_t  fontSize,
uint16_t  fontFace,
int  lockGroup 
)

Registers the specified glyph as a character to draw.

Details
Characters registered as those to be drawn must be plotted anew and registered as such if they are not in the texture cache. These characters become available after UpdateTextureCache and CompleteTextureCache have finished.

Characters registered as those to be drawn can be used as is without any plotting when they are already in the texture cache. These characters are registered as required, so they will not be deleted.

There are a total of 32 groups, numbered 0 through 31, for locked characters. Characters are locked at the group level, so if, for example, a character already registered in group 0 is registered again in group 1, it would become locked in both groups, and both locks would need to cleared before the character could be become a target for removal.

Parameters
[in]codeThe glyph to register as a character to draw.
[in]fontSizeThe font size.
[in]fontFaceNumeric value specifying the font face.
[in]lockGroupThe group when locking glyphs. Specify a numeric value in the range from 0 to 31. If -1 is specified, glyphs are not locked.
Returns
Returns true if the registered glyph needs to be plotted. In other words, that glyph can be drawn as is without executing UpdateTextureCache when false is returned.

◆ UpdateTextureCache()

void nn::font::TextureCache::UpdateTextureCache ( )

Updates the texture cache and makes a new plot if a plot is needed.

Details
This function can take an extremely long time. When the screen is not blacked out and the game is running, take adequate measures such as calling the function in a subthread so that processing does not slow down.

The function can run in parallel with UpdateTextureCache and GetGlyph, in addition to drawing. However, do not call RegisterGlyph, RegisterGlyphWithLength, or CompleteTextureCache while UpdateTextureCache is running.

This function consumes as much as 20 KB of the stack. If you are running it on a thread, be careful about the stack size.

◆ CompleteTextureCache()

void nn::font::TextureCache::CompleteTextureCache ( )

Completes the texture cache.

Details
Call this function after UpdateTextureCache has completed. It cannot be run at the same time as drawing.

If you call this function, glyphs that do not meet any of the following conditions cannot be used.

  • The glyph has been drawn one or more times (that is, GetGlyph has been called) since the last call to CompleteTextureCache.
  • The glyph was registered as a character to draw (such as with RegisterGlyph) after the last call to CompleteTextureCache.
  • The glyph is locked in a group.

◆ ResetTextureCache()

void nn::font::TextureCache::ResetTextureCache ( )

Resets the contents of the texture cache and returns it to a state similar to just after initialization.

Details
Calling this function also deletes the alternate character that was registered at initialization. Before calling functions to draw fonts, call RegisterAlternateCharGlyph for each font that will be using this cache.

After that, continue as you would with Initialize, and run both UpdateTextureCache and CompleteTextureCache once before you draw with fonts using GetGlyph.

◆ CreateNewLine()

bool nn::font::TextureCache::CreateNewLine ( LineHeight  lineHeight)

Creates a new line in the texture cache.

Details
Normally you do not need to use this method, because new lines are created automatically when there are not enough lines in the texture cache.

You would need to use this method in situations where 32 lines are almost always entered but on rare occasions 64 lines are required. (For example, when tall characters, such as Latin characters with cedillas, appear.) In such cases, if texture cache is already filled with 32 lines, a new set of 64 lines cannot be created and characters that require 64 lines cannot be drawn.

To avoid this problem, use this function immediately after initializing TextureCache to create one texture cache with 64 lines.

For more information, see "I can't expand characters even though there are plenty of characters in texture cache that can be deleted" in the Overview FAQ.

Parameters
[in]lineHeightLine height.
Returns
Returns true if a line is created. If space is insufficient and a line cannot be created, returns false.

◆ DumpGlyphTreeMap()

void nn::font::TextureCache::DumpGlyphTreeMap ( ) const

Dumps the contents of the balanced binary tree in texture cache.

Details
Running this member function sends the following type of string to the console using the number of characters included in the cache and the same number of rows.

0 0x32 40 rp lock:0x1800

This line represents one glyph. The first value represents the font type in a range from 0 to 31. The second value represents the encoding, and the third value represents the size. The string that follows is an internal flag value. The value after lock: is a 32-bit value representing the locked state. In this example, the glyph is locked in groups 11 and 12.

Indenting is used to represent the location in the balanced binary tree where the glyph is located.

Finally, we have:

dump 102 nodes.

This is what is displayed. It indicates the number of glyphs being managed in texture cache.

Because you cannot register glyphs anew if this number exceeds the value specified for glyphNodeCountMax in InitializeArg, make sure that you provide a sufficient number.

This function does nothing in the Release build.

◆ GetNoSpaceError()

NoSpaceError nn::font::TextureCache::GetNoSpaceError ( ) const

Determines whether an error occurred because of a lack of space in texture cache.

Details
After an error is generated, this function is not cleared until ClearNoSpaceError is called. Error handling is performed in the following way.

  • NoSpaceError_NoError: No error occurred.
  • NoSpaceError_LackOfGlyphNodeSpace: There is no space in the glyph node. Increase the value of InitializeArg::glyphNodeCountMax.
  • NoSpaceError_TextureCacheLineIsOverMax: The number of texture cache lines has exceeded the limit. Either increase the value of InitializeArg::textureCacheWidth or decrease the kinds and sizes of fonts being managed by texture cache.
  • NoSpaceError_LackOfTextureCacheSpace: There is no free space in texture cache. Increase the value of InitializeArg::textureCacheWidth or InitializeArg::textureCacheHeight.
Returns
If no error has been generated, the function returns NoSpaceError_NoError. Otherwise, it returns a value corresponding to the error.
