Detailed Description

The class that handles character code assembly.

Details
Character codes are represented as UTF-16 codes.

◆ Enqueue()

int nn::kpr::KeyCodeComposer::Enqueue ( const uint16_t  buffer[],
int  count 
)
noexcept

Adds a character code to the end of the character code sequence currently being composed.

Parameters
[in]bufferCharacter code sequence.
[in]countLength of buffer.
Returns
The number of additional character codes.
Precondition
  • buffer != nullptr.
  • 0 <= count.
Postcondition
  • For the return value n, 0 <= n && n <= count.

Details
Character codes are added one at a time from the beginning of the sequence, and the status of the character code sequence that is being composed is updated each time. When a confirmed sequence occurs in the character code sequence, no more character codes will be added from that point. If there is already a confirmed sequence, no character codes will be added. To add key codes again, you can call Dequeue() and extract the confirmed sequence until GetDequeueableCount() is 0.

The length of the character code sequence that was extracted.
Precondition

  • 0 <= count.

Details
Extracts a maximum of count elements from the end of the unconfirmed sequence of the character code sequence that is currently being composed.

If the number of character codes extracted is less than the length of outBuffer, no changes are applied to the remaining area. Character codes are stored in FIFO order.
