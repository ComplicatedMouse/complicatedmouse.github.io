This sample program shows how to use the multi-audio renderer.

Overview

This sample uses multiple audio memory renderers to play audio.

File Structure

This sample program is located in Samples/Sources/Applications/AudioMultipleRenderer/ .

Required Environment

You must be able to use audio output.

How to Use

When you execute the sample program, two audio renderers will begin audio playback.

Input Operation
A Button Start or stop AudioRenderer 1.
[B] Start or stop AudioRenderer 2.
START or SPACEBAR Exits the program.

Important Information

Nothing specific.

Execution Procedure

Build the sample program and then run it.

Description

This sample program performs audio rendering using multiple audio renderers. A maximum of nn::audio::AudioRendererCountMax audio renderers can be used simultaneously. For more information about using the audio renderer, see the Samples/Sources/Applications/AudioRenderer/ sample.

The audio renderers individually manage the resources they are linked to. For this reason, note that resources linked to specific AudioRenderer instances such as VoiceType, MemoryPoolType, and AudioRendererConfig cannot be shared between AudioRenderer instances.

This sample program has the following flow.

  • Create two threads for playback using the audio renderers. The following processing is executed for the threads that were created.
  • Initialize the audio renderer.
  • Prepare FinalMix.
  • Set the output device.
  • Prepare the input waveform, allocate Voice, and set parameters.
  • Initialize audio rendering.
  • Verify the finalization request for the rendering processing.
  • End rendering.
  • Close the audio renderer.
  • Free the input waveform.
