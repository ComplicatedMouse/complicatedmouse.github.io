Detailed Description

Declarations for the logging API.

Details

The meta information for the log.

The log contains metadata.

Optional values can be set for the following elements of metadata.

  • Module name.
  • Severity.
  • Verbosity.

The following elements of metadata are set automatically.

  • The filename.
  • Line number
  • Function Name

Module name.

Specify a string. For SDK modules, prefix the module name with $.

Severity.

Specify nn::diag::LogSeverity.

Verbosity.

Specify an integer of 0 or larger. A larger numeric value indicates more verbosity.

The verbosity element is designed to be used to adjust the quantity and details of the logs that are generated. The following examples show the logs when file writing fails.

Severity. Verbosity. Log Content Description
nn::diag::LogSeverity_Error 0 Failed to write the file. Minimum information needed to notify of an error.
nn::diag::LogSeverity_Error 1 The file path: xxxx. Information that might be of interest even to people who did not develop this module.
nn::diag::LogSeverity_Error 2 Write position: yyyy. Information that would only be of interest to the developer of this module.
: : : :
NX Platform Only:
You can use Target Manager to check the log metadata. For more information, see Displaying Metadata in NintendoSDK Documents. A feature will be added to Target Manager for filtering how logs are displayed using the log metadata. For example, you can display logs for specific modules, or hide logs where the verbosity exceeds a certain value.

Enabling and Disabling Logs

Logs are enabled if NN_ENABLE_LOG is defined. Logs are disabled and the macro is expanded into an empty operation if NN_DISABLE_LOG is defined. You cannot specify NN_ENABLE_LOG and NN_DISABLE_LOG at the same time. If neither macro is defined, the enabled determination depends on the NintendoSDK build type.

  • Debug or Develop: Enabled.
  • Release: Disabled.

For SDK versions earlier than 8.0.0, and when neither macro is defined, the determination is enabled regardless of the NintendoSDK build type.

Log Encoding

Logs are encoded in UTF-8.

If you are using the locale's default method of character encoding of necessity in the Windows environment, define the NN_LOG_USE_DEFAULT_LOCALE_CHARSET macro before including the NintendoSDK header files.

For more information, see Notes About Log and Assert Strings in NintendoSDK Documents.
