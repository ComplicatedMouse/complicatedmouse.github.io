Detailed Description

Word wrapping class.

  • WordWrapping::CalculateWordWrapping() Inserts a line feed code at the appropriate location within the string.

Searches for a good place to break the line.

UTF-16 version.

Parameters
[in]pBeginThe start character of the line.
[in]pEndThe end position of the string.
[in]funcThe callback function that gets the amount of text to display per line.
[in]configWord wrap behavior.
Returns
Returns the position of the character just before a line break.

Details
Use this function when you want to find a good line break position without using the full-blown word wrapping feature.

Searches for a good place to break the line within the range returned by the func callback function.

If there is not a good place to break the line within the range returned by the func callback function, this function returns the position of the last character in the line.

Wraps text.

UTF-16 version.

Parameters
[out]pWriteSizeThe length of the text stored in pOutTextBuffer.
[out]pOutTextBufferThe text with line breaks inserted at the appropriate positions.
[in]outTextBufferSizeThe number of characters that may be stored in pOutTextBuffer. Includes the terminating null character.
[in]pInputTextThe text to wrap. (It must be null-terminated.)
[in]funcThe callback function that gets the amount of text to display per line.
[in]configWord wrap behavior.
Returns
Returns true if the text is wrapped successfully. Returns false if the function fails for any reason.

Details
To wrap text properly, the LF newline code must be used to indicate line breaks. CR and CRLF are not supported.

If a suitable location to wrap a line is not found, the line is force-wrapped at the position indicated by func.

The function terminates in the following cases. pOutTextuBuffer also terminates with NULL in these cases.

  • pOutTextuBuffer is not of sufficient size.
  • The number of lines after wrapping exceeds the number of lines specified by config.

If you specify an address for pWriteSize, the length of the text stored in pOutTextBuffer is written to that address.
