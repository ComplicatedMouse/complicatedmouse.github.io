Using User Accounts in an Application

This sample demonstrates how to use of the features of the account library in an application.

Overview

This sample program presents ways of using the features of the account library in an application. The sample program demonstrates the use of the following features of the account library.

  • Selecting, opening, and closing user accounts
  • Getting the attributes of user accounts
  • Using the user's network service account
  • Getting the authorization for the user's Nintendo Account

File Structure

This sample program is located in Samples/Sources/Applications/AccountApplication .

Required Environment

This sample program can only be built for and run on development hardware. Note that its use is not supported within a Windows environment.

How to Use

Build and run this program as you would an ordinary sample program. The output during the course of this program is mainly for drawing text to the screen.

Description

This sample program is operated using DebugPad. However, if the development hardware is operating in handheld mode and Npad is enabled, those operations take precedence.

The sample program organizes the scenes as follows.

  1. Root sequence (AccountApplication_RootSequence.h)
    • This sequence is for switching between SingleUserMode and MultiUserMode. The scene transitions depending on which mode is selected.
  2. SingleUserMode sequence (AccountApplication_SingleUserModeSequence.h)
    • In this sequence, (1) a user is selected and opened, and then (2) operations are performed on the opened user.
    • This sequence is comprised of scenes that assume the application is a single-player application.
    • The following operations can be performed in step 2 on the opened user.
      • Update the user's save data thumbnail.
      • Authenticate the user's network service account.
      • Get the authorization for the user's Nintendo Account.
      • Reset the sequence (return to step (1) of the SingleUserMode sequence).
      • Exit the sequence (return to the Root sequence).
  3. MultiUserMode sequence (AccountApplication_MultiUserModeSequence.h)
    • In this sequence, (1) multiple users are selected and opened, and then (2) operations are performed on the opened users.
    • This sequence is comprised of scenes that assume the application is a multi-player application.
    • The following operations can be performed in step 2 on the opened user.
      • Reset the sequence (return to step 1 of the MultiUserMode sequence).
      • Exit the sequence (return to the Root sequence).

The sample program demonstrates the use of the following features of the account library.

  • Selecting, opening, and closing user accounts
    • For implementation examples of managing the user account Open and Closed states in the application, see UserStateManager.
    • An example sequence where the application handles one user at a time is implemented as SingleUserModeSequence.
    • An example sequence where the application handles multiple users simultaneously is implemented as MultiUserModeSequence.
    • The sample also supports cases where the user is selected before the application starts.
      • For an implementation example, see the RootSequence constructor and the UserStateManager::TryOpenPreselected() function that it calls.
      • To check operations in those cases, enable the feature to restrict the opening of user accounts as described in the development support features of DevMenuCommand.
  • Getting the attributes of user accounts
    • This sample get the user's nickname.
      • See the location that uses nn::account::GetNickname().
  • Using the user's network service account
    • For the specific implementation, see the implementation of SceneOperation::SetTaskForNetworkServiceAccountAuthentication() in SingleUserModeSequence and the AcquireNetworkServiceAccountIdToken() function that it calls.
    • This sample program does not support communication with the application server.
  • Getting the authorization for the user's Nintendo Account
    • For the specific implementation, see the implementation of SceneOperation::SetTaskForNintendoAccountAuthorization() in SingleUserModeSequence and the AcquireNintendoAccountAuthorization() function that it calls.
    • This sample program does not support communication with the application server.

For more information, see the comments in the sample program headers and source files.
