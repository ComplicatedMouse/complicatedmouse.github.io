◆ maxEditableItems

int nn::atk::viewer::SoundArchiveEditor::Configs::maxEditableItems

Specifies the maximum number of editable items.

< If not specified, the default value (DefaultEditableItemCountMax) is used. < Change the setting if you want to target more items, or if you know in advance that there are relatively few target items and you want to reduce the amount of memory used, even by a small amount.

◆ maxEditableFiles

int nn::atk::viewer::SoundArchiveEditor::Configs::maxEditableFiles

Specifies the maximum number of editable files.

< If not specified, the default value (DefaultEditableFileCountMax) is used. < Change the setting if you want to target more files, or if you know in advance that there are relatively few target files and you want to reduce the amount of memory used, even by a small amount. < If isEditableFilesCountMaxExpandable is true, this value is used as the initial value.

◆ maxItemName

int nn::atk::viewer::SoundArchiveEditor::Configs::maxItemName

Specifies the maximum value for item name length.

< If not specified, the default value (DefaultItemNameLengthMax) is used. < Edit if you want to target extremely long item names, or you know in advance that the used names are short and you want to reduce the amount of memory used, even by a small amount.

◆ isEditableItemsCountMaxExpandable

bool nn::atk::viewer::SoundArchiveEditor::Configs::isEditableItemsCountMaxExpandable

Specifies whether the number of editable items expands dynamically.

< Specify true to dynamically expand memory when the number of editable items reaches the upper limit. < When that happens, memory is reallocated and can cause fragmentation. < Specify false to operate within the range specified by maxEditableItems.

◆ isEditableFilesCountMaxExpandable

bool nn::atk::viewer::SoundArchiveEditor::Configs::isEditableFilesCountMaxExpandable

Specifies whether the number of editable files expands dynamically.

< Specify true to dynamically expand memory when the number of editable files reaches the upper limit. < When that happens, memory is reallocated and can cause fragmentation. < Specify false to operate within the range specified by maxEditableFiles.
