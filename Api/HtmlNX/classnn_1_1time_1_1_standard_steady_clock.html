Detailed Description

The monotonic increasing clock with a power supply external to SoC.

Details
Although the values indicated by this clock cannot be converted to a time expression that humans could understand, it can be used to calculate elapsed time without any impact from time modifications, because it is incremented over a fixed interval regardless of how the user or system manipulates time. What's more, the clock continues to be incremented even after the application ends or the device is turned off, enabling you to calculate the elapsed time between ending the application and starting it again.

If the power supply dies because of a loss of battery power, the value indicated by the clock is reset.

If conditions preclude straddling application termination and starting, you can calculate the elapsed time at higher resolutions using the system tick value that you can get with nn::os::GetSystemTick.

Note that the source for this clock is not tough with regard to hardware spoofing attacks.

This class meets the requirements of the C++ standard TrivialClock, and provides features in line with std::steady_clock. It is only assured when the value is not reset. Note that this is not the case when involving conditions where it is possible that power to the clock could be lost, such as when ending an application. You cannot detect a value reset when you are only using the C++ standard interface.

Accumulated Inaccuracy in the Monotonic Increasing Clock

The monotonic increasing clock can accumulate errors, when incrementing, on the order of several minutes per month, compared to the pace of actual time. This error range varies with the degree of device use and with differences in the device itself.

nn::time::StandardUserSystemClock, and nn::time::StandardNetworkSystemClock use increments to the monotonic increasing clock to advance the time. The accumulated error affects the accuracy of these clocks.

◆ now()

static time_point nn::time::StandardSteadyClock::now ( )
static

Gets the current time indicated by this clock.

Returns
Returns the current time indicated by this clock as a nn::time::StandardSteadyClock::time_point type.
Precondition
  • The time library is initialized.

Details
You cannot detect whether the clock was reset with the return value of this function. Note that when the clock was reset, the value could be rolled back.

That said, a clock reset will not occur while the application is running.
