Sample program showing how to use the Target Manager (tm) library.

Overview

Describes how to create host tools using the Target Manager (tm) library.

For more information about creating projects, see the tm Library API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/TmSimple/ .

Required Environment

Target Manager must be installed and executed ahead of time.

Execution Procedure

Build the sample program and then run it.

282  _ASSERT(hHtcsUpdateEvent != NULL);

286  nn::tm::ErrorText errorText;

289  return errorUpdateEvent;
290  }

292  nn::tm::HtcsAddress addresses[64];
293  int32_t size;
294  while (true)
295  {
296  WaitForSingleObject(hHtcsUpdateEvent, INFINITE);
297  nn::tm::Error errorGetHtcsPoints = nn::tm::GetHtcsEntryEndPoints(addresses, &size, sizeof(addresses) / sizeof(addresses[0]), nullptr, nullptr);
298  if (errorGetHtcsPoints != nn::tm::Error_Ok)
299  {
300  nn::tm::ErrorText errorText;
301  nn::tm::GetErrorTextEn(&errorText, errorGetHtcsPoints);

303  continue;
304  }
305  std::string out;
306  out += "---- htcs ports updated ----\n";
307  for (int i = 0; i < size; i++)
308  {
309  out += std::to_string(i);
310  out += ", ";
311  out += addresses[i].ipAndPortAddress;
312  out += "\n";

314  out += "---- htcs port list end ----\n";

714  int32_t targetCount;
715  RETURN_IF_FAILURE(nn::tm::GetTargetCount(&targetCount));

719  return;
720  }

722  std::unique_ptr<nn::tm::TargetHandle[]> pTargetHandles(new nn::tm::TargetHandle[targetCount]);
723  RETURN_IF_FAILURE(nn::tm::GetTargets(pTargetHandles.get(), &targetCount, targetCount));
724 
725  printf("Please input the option number of the target to remove:\n");
726  ShowRegisteredTargetsWithoutDetails(pTargetHandles.get(), targetCount);
727  printf("%d) ", targetCount);
728  printf("\tRemove all targets\n");
729  printf("other) ");
730  printf("\tReturn to menu\n");
731  int selectedTarget = GetInputNumber();
732  if (0 <= selectedTarget && selectedTarget < targetCount)
733  {
734  nn::tm::TargetHandle target = pTargetHandles[selectedTarget];
735  RETURN_IF_FAILURE(nn::tm::RemoveTarget(target));
736  printf("Removed.\n");

Description

This sample program has the following flow.

At startup.

  • List the targets that can be added to Target Manager on this PC.
  • List the targets that are already added to this instance of Target Manager.
  • Wait for input from the user and branch the process according to the input.

0: AddTarget()

  • Add to Target Manager the target that was selected from the list of addable targets.
  • If the selected target is already added, display an indication of that fact and return to the main menu.

1: RemoveTarget()

  • Remove from Target Manager the target that was selected from the list of added targets.
  • If Remove all targets has been selected, remove all targets.

2: RunSampleProgram()

  • Run the sample program on the target that was selected from the list of targets added to Target Manager.
  • Try connecting the target with Target Manager and skip all subsequent processes if they cannot be connected.
  • After startup, send the information about the started application to the console.
  • Wait for input from the user and conduct the process corresponding to that input.
    • 0: Save a screenshot. The saved screenshot can be checked on Target Manager.
    • 1: Save a dump file. The user selects a quick dump or file dump at run time.
    • 2: Exit the sample program and return to the main menu.
  • If the application closes due to an error or other reason, the system returns to the main menu at the point when it receives user input.

3: RunAnyProgram()

  • Run any program on the target that was selected from the list of targets added to Target Manager.
  • Try connecting the target with Target Manager and skip all subsequent processes if they cannot be connected.
  • After startup, send the information about the started application to the console.
  • Wait for input from the user and conduct the process corresponding to that input.
    • 0: Save a screenshot. The saved screenshot can be checked on Target Manager.
    • 1: Save a dump file. The user selects a quick dump or file dump at run time.
    • 2: Exit the sample program and return to the main menu.
  • If the application closes due to an error or other reason, the system returns to the main menu at the point when it receives user input.

4: TargetSetting()

  • Change the various target settings.
  • First, select the target from the list of added targets.
  • Send detailed information about the selected target to the console.
  • Perform processing according to the input.
    • 0: Connect the specified target to Target Manager.
    • 1: Disconnect the specified target from Target Manager.
    • 2: Set the selected target as the default target.
    • 3: Rename the target to the string entered by the user.
    • 4: Set the working directory in the specified target. When you perform this process, the working directory becomes C:\.
    • 5: Turn off the power. This action is applicable only when connected to the target.
    • 6: Soft reset the target. This action can be selected only when connected to the target.
    • 7: Hard reset the target.
    • 8: Perform operations for the development hardware log.
      • 0: Send the log to the console.
      • 1: Start saving to a text file for the log.
      • 2: Stop sending to the text file for the log.
      • 3: Start saving to a JSON file for the log.
      • 4: Stop saving to the JSON file for the log.
    • 9: Turn on the power. Applicable only with the SDEV.
    • 10: Switch the docking state based on the user input. Applicable only with the SDEV.
      • 0: Handheld mode
      • 1: TV mode
    • 11: Simulate the press of the POWER Button. Applicable only with the SDEV.
      • 0: Emulate 1 second.
      • 1: Emulate 4 seconds.
      • 2: Emulate 8 seconds.

5: IncompleteDiscoveryScan().

  • This process is for checking how to change DiscoveryState according to DiscoveryScan.
  • After the selection is made, DiscoveryScan is stopped after it has been started, DiscoveryState is returned, and the process returns to the main menu automatically.

6: Rescan and View Target()

  • List the targets that can be added to Target Manager on this PC.
  • List the targets that are already added to this instance of Target Manager.

Other: EXIT

  • Exits the program.
