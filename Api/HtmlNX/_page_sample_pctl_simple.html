This sample program verifies the restrictions based on the parental controls settings.

Overview

This simple sample starts and ends features for communicating freely with others by using nn::pctl::TryBeginFreeCommunication and nn::pctl::EndFreeCommunication.

File Structure

This sample program is located in Samples/Sources/Applications/PctlSimple/ .

Required Environment

You must be able to use the screen display. This program also operates in a Windows environment, but to check the restrictions set in the parental controls, you must run it in a development hardware environment.

How to Use

Entering A at the initial screen (main scene) transitions it to a sub-scene. At this point, if the "freely communicate with others" item is restricted by parental controls, it will not transition to an error display.

Entering B at the sub-scene returns to the main scene, and entering B at the main scene ends the application.

Important Information

To check whether operations are actually restricted, you must use the HOME Menu or some other means to configure the parental controls settings before the application is started. In addition, although this sample program supports it, to apply restrictions you must both call the PCTL function and add the parental controls settings to the program NMETA file. For reference, you can review the settings that are included in the PctlSimple.*.nmeta file located in the same directory as this sample program.

Execution Procedure

Build the sample program and then run it. However, restriction checks will not be performed correctly if the NMETA file settings are not loaded, so you must execute this process in an environment where the management data has been loaded, such as by creating an NSP file and installing it.

Description

This sample program demonstrates starting and ending features for communicating freely with others by using nn::pctl::TryBeginFreeCommunication and nn::pctl::EndFreeCommunication.

This sample program contains two scenes: a main scene and a sub-scene. Of these scenes, the sub-scene assumes use of the "communicate freely with others" feature, and performs restriction verification before scene transition. The settings are based on the parental controls settings that were configured ahead of time, and if the restrictions are active it will not transition to the sub-scene.

Specifically, the following processes are performed.

  • nnMain: Initialization/finalization and starting the main loop.
  • MainLoop: Main loop.
    • The screen changes based on user input.
  • ChangeSceneToMain: Transition to the main screen.
  • ChangeSceneToSub: Transition to the sub-screen.
    • Checks for restrictions on the feature before transitioning to the sub-screen.
  • PrintMain: Draw the main screen.
  • PrintSub: Draw the sub-screen.
