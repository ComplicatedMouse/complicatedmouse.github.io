This is a description of the sample programs for the NFP library.

Overview

The nfp library uses near-field communication (NFC) to read and write NFP tags, which are NFC tags that are formatted for NFP. You can associate your applications with amiibo when using the nfp library.

This sample program supports the following amiibo operations.

  • Reading and writing of the amiibo application area.
  • Restoring amiibo with corrupted data.
  • Deleting the amiibo application area from the amiibo settings.
  • Restoring amiibo with corrupted data from the amiibo settings.
  • Adding the Mii characters for the amiibo owner and the nickname from the amiibo settings.

Writes a four-byte counter to the application area using the sample program. The counter is set to zero when the application area is first created. The counter is then incremented by one every time it is subsequently written to.

File Structure

This sample program is located in Samples/Sources/Applications/NfpSimple/ .

How to Use

The state of the nfp library appears when you run the sample program. You can change the state of the nfp library by using a gamepad.

Important Information

Using the sample program, you cannot restore corrupted amiibo data on development hardware when that amiibo has never been used. This is because no backup data will have been saved. You can either initialize the amiibo from the amiibo settings in advance, or use development hardware that has backup data to restore the amiibo data.

Execution Procedure

Build the sample program and then run it.

Description

Different operations will be run for NfpSimple, depending on the state of the amiibo. When you want to run specific operations, you will need to have an amiibo that already meets the following conditions.

  • Create the application area. Use an amiibo for which the NoftWriter tool was just used to write the amiibo data list.
  • Delete the application area. Use an amiibo that was written using an application whose access ID differs from NfpSimple.
  • Restoring amiibo with corrupted data. Use an amiibo with corrupted data.
  • Adding the Mii characters for the amiibo owner and the nickname. Use an amiibo for which the NoftWriter tool was just used to write the amiibo data list.
