This sample introduces how to use nvn::Window::SetCrop to adjust render resolution.

The render resolution is adjusted according to the gpu workload to maintain 60fps.

Overview

This sample implements a dynamic resolution rendering to demonstrate the usage of nvn::Window::SetCrop which adjusts resolution based on the gpu workload to maintain that application can run at 60fps. Some cubes and lights are displayed to screen to create variable workload. When gpu draw time increased, render resolution will be lowered to maintain target fps. And when gpu draw time decreased, render resolution will be raised.

Description

This sample implements a basic dynamic resolution rendering to demonstrate the usage of nvn::Window::SetCrop to adjust render resolution. We render 80 cubes and up to 80 lights with forward rendering to create enough gpu workload. The gpu draw time will be increased when more lights are activied. To maintain target frame rate, we adjust the percentage of resolution to lower the gpu workload and which will increased to get better rendering quality when getting low workload. The percentage of resolution is between 100%-50%, and target frame rate is 60fps. Note that when using nvn::Window::SetCrop, the window resolution will be changed so UI/text contents will also be rendered with lower resolution which may make text looks blurry.

nvn::Window::SetCrop is also been used to change render resolution when changing operation mode between handheld mode and console mode. It serves the same purpose as nn::vi::SetLayerCrop which can be used with other Graphics APIs like OpenGl and Vulkan.
