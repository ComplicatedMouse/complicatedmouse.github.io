Detailed Description

Decodes JPEG data.

Details
Decodes JPEG data and generates pixel data. By specifying a downscaling parameter you can also decode the data into a small buffer. There is also a feature for identifying the location of the binary storing the Exif information from the provided JPEG data.

JPEG Data That Can Be Decoded

The library supports the decoding of the following formats of JPEG data. There is no assurance that all existing JPEG data can be decoded.

  • Baseline and progressive are supported as the scan method.
  • Lossless JPEG is not supported.
  • As a rule, all pixel sampling ratios are supported, including 4:0:0.

Estimating the Size of Working Memory

The size of the working buffer used when decoding the JPEG data is calculated based on the size of the image (the width and height) and the decoding parameters. For most regular JPEG data, the largest working buffer is the one needed to decode JPEG data that was encoded with the following parameters. You can use that amount of working buffer to estimate the buffer for the application to reserve. When making this estimate, use pixel data for the most pixels you expect the application to handle.

Encode Parameter Baseline JPEG Setting Progressive JPEG Setting
Color space YCbCr YCbCr
Sampling ratio 4:2:0 4:4:4
Number of Huffman tables 4 8
Number of quantization tables 4 4

You need more working memory to decode progressive JPEG data than baseline JPEG data. If the application will be dealing with only baseline JPEG, you do not need to worry about the memory size needed for progressive JPEG. On the other hand, if the application will be dealing with progressive JPEG, be sure to reserve working memory for that.

When unspecified JPEG data is decoded, the working buffer derived using this method may prove to be insufficient, depending on the combination of parameters set when that JPEG data was encoded. If unspecified JPEG data will be included among the decoding targets, consider this possibility when designing your application.

Object States

The JpegDecoder object can take the following three states.

  • Unregistered JPEG Data
    • The state where no JPEG data is set in the JpegDecoder object.
  • Registered JPEG Data
    • The state where JPEG data is set in the object, but the data and the parameters have not been analyzed.
  • Analyzed JPEG Data
    • The state of the JpegDecoder object after the last of the set JPEG data and parameters have been analyzed.

Parallel Processing

You can create multiple JpegDecoder class objects to decode multiple sets of JPEG data in parallel.

◆ SetResolutionDenominator()

void nn::image::JpegDecoder::SetResolutionDenominator ( const int  denom)

Sets the scaling to use when decoding.

Parameters
[in]denomA scaling denominator. You can specify 1, 2, 4, 8, or 16.
Precondition
For denom, you must specify either 1, 2, 4, 8, or 16.
Postcondition
If the state of the JpegDecoder object is Analyzed JPEG Data, it is changed to Registered JPEG Data. There is no such change from any other state.

Details
If the image scaling denominator is set to N, the pixel data generated by the Decode() function is scaled both vertically and horizontally to 1/N of its original size. However, the effective value of this scaling value depends on the block size of the JPEG data set by the SetImageData() function. If 16 is specified as the scaling value for JPEG data that has a block size of 8, the decoded image will be scaled by 1/8 vertically and horizontally.

This setting is not necessary if you are using the default value of 1, or if your objective is to get the Exif information.

◆ SetPixelFormat()

void nn::image::JpegDecoder::SetPixelFormat ( PixelFormat  pxFormat)

Sets the pixel format to use when decoding.

Parameters
[in]pxFormatThe pixel format of the decoded pixel data, specified as an enumerator of nn::image::PixelFormat.
Precondition
pxFormat must be specified as either nn::image::PixelFormat_Rgb24 or nn::image::PixelFormat_Rgba32.
Postcondition
If the state of the JpegDecoder object is Analyzed JPEG Data, it is changed to Registered JPEG Data. There is no such change from any other state.

Details
This function specifies the pixel format for the pixel data of the Decode() function. If you do not call this function, the pixel format becomes nn::image::PixelFormat_Rgba32 by default.

Relatively less work memory is required if nn::image::PixelFormat_Rgb24 is specified as the pixel format as opposed to nn::image::PixelFormat_Rgba32.

◆ Decode()

JpegStatus nn::image::JpegDecoder::Decode ( void *  outPixelBuf,
const size_t  pixelBufSize,
const int  lineAlignment 
)

Decodes the JPEG data and writes the pixel data to the specified output buffer.

Parameters
[out]outPixelBufThe location used to save the output pixel data. One pixel is output in the pixel format specified by SetPixelFormat(). If SetPixelFormat() has not been called, the resulting data is in 32-bit RGBA format.
[in]pixelBufSizeSize of outPixelBuf in bytes.
[in]lineAlignmentLine alignment for outPixelBuf. For example, you would specify 2 to generate an image that is 129 pixels wide to a pixel buffer that is 130 pixels wide.
Returns
Returns the execution result.
Return Values:

nn::image::JpegStatus_Ok Indicates that the process ended normally.

nn::image::JpegStatus_WrongFormat Indicates that data in an invalid format was set for the JPEG data. This data cannot be decoded.

nn::image::JpegStatus_UnsupportedFormat Indicates that data is set in a format not supported by the JpegDecoder class. This JPEG data is not supported so it cannot be decoded.

nn::image::JpegStatus_OutOfMemory Indicates that the process ran out of working memory during decoding. Either there is not enough memory available to the memory allocator specified by the constructor, or the input included unexpected JPEG data.

Precondition
All of the following conditions must be satisfied.
  • The size of outPixelBuf must be at least equal to ((the image width line alignment lineAlignment) × (the image height) × (the number of bytes per pixel)).
  • The state is Analyzed JPEG Data.
Postcondition
When this function generates pixel data in a pixel format that includes an alpha channel, the value of this alpha channel in outPixelBuf is fixed to 255. Nothing is written to the padding region for alignment.

Details
Decodes JPEG data, using the set JPEG data and the scaling setting. The function returns something other than JpegStatus_Ok if the set JPEG data is invalid or in an unsupported format.

Sometimes the Decode() function returns an error on JPEG data for which the Analyze() function returned nn::image::JpegStatus_Ok. This happens because the Decode() function uses information not referenced by the Analyze() function. Accordingly, you must check for errors from both functions.

Decodes JPEG data using the parameters already prepared inside the specified JpegDecoder object.

Parameters
[out]outPixelBufThe location used to save the output pixel data. One pixel is output in the pixel format specified by the SetPixelFormat() function of the object specified by decoder. If SetPixelFormat() is not called by the object specified by decoder, the resulting data is in 32-bit RGBA format.
[in]pixelBufSizeSize of outPixelBuf in bytes.
[in]lineAlignmentLine alignment for outPixelBuf. For example, you would specify 2 to generate an image that is 129 pixels wide to a pixel buffer that is 130 pixels wide.
[in]jpegDataJPEG data to decode.
[in]jpegSizeSize of jpegData in bytes.
[in]decoderThe nn::image::JpegDecoder object whose JPEG data has been analyzed.
Returns
Returns the execution result.
Return Values:

nn::image::JpegStatus_Ok Indicates that the process ended normally.

nn::image::JpegStatus_WrongFormat Indicates that data in an invalid format was set for the JPEG data. This data cannot be decoded.

nn::image::JpegStatus_UnsupportedFormat Indicates that the specified data is in a format not supported by the JpegDecoder class. This JPEG data is not supported so it cannot be decoded.

nn::image::JpegStatus_OutOfMemory Indicates that the process ran out of working memory during decoding. Either there is not enough memory available to the memory allocator specified by the constructor, or the input included unexpected JPEG data.

Precondition
All of the following conditions must be satisfied.
  • The size of outPixelBuf must be at least equal to ((the image width line alignment lineAlignment) × (the image height) × (the number of bytes per pixel)).
  • jpegData is the same size (width and height) as the JPEG data specified by the SetImageData() function of the object specified by decoder.
  • The state of decoder is Analyzed JPEG Data.
Postcondition
When this function generates pixel data in a pixel format that includes an alpha channel, the value of this alpha channel in outPixelBuf is fixed to 255. Nothing is written to the padding region for alignment.

Details
Decodes JPEG data according to the parameters and decoding settings held by the specified decoder. Multiple images can be decoded more efficiently this way because it omits the process of analyzing the JPEG data and the decoding settings necessary for decoding.

This function assumes that the input JPEG data is the same size (width and height) and has the same encoding parameters as the JPEG data specified by the SetImageData() function of the object specified by decoder. If some other kind of JPEG data is specified, this function may experience insufficient working memory and return nn::image::JpegStatus_OutOfMemory.

Sometimes this function returns an error on JPEG data for which the Analyze() function of the object specified by decoder returned nn::image::JpegStatus_Ok. This happens because this function uses information not referenced by that Analyze() function. Accordingly, you need to run an error check on the value returned by this function.

Details
Analyzes the header of the provided JPEG data and gets the region (address and size) where the Exif information is stored. This function only conducts the analysis required to get the region where the Exif information is stored.

This function is faster than creating an instance of JpegDecoder and using the Analyze() function. For this reason, we recommend using this function if all you want to do is get the region with Exif information from the JPEG data.
