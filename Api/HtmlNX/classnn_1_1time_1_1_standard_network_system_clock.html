Detailed Description

The clock representing the clock automatically adjusted from the network.

Details
This clock indicates the current time. You can get a value that can be converted into a time expression that a human can understand. If the clock has already been adjusted, the value can be obtained without connecting the device to the network.

This clock is not affected by modifications to time made by the user; it is, however, affected by automatic system adjustments and the indicated time can be ahead of or behind the actual elapsed time. In addition, although the clock continues to be incremented when the device is turned off, the clock is invalidated and becomes unable to return time values if the power supply is lost as a result of a dead battery. You recover from this state by having the device get the time from the network again. Calculations for elapsed time involving clock adjustments and invalidations may not be strictly accurate, but because there are regular adjustments to the correct time, you can trust those calculations to a certain extent. In addition, while the device is turned on, this clock will not change from a state where the value can be returned to a state where it cannot be returned. Conversely, there are times when it may automatically transition from a state where it cannot return a value, to a state where it can return a value, even when an application is running.

This class satisfies the standard C++ TrivialClock requirements and provides features equivalent to std::chrono::system_clock. However, you cannot detect clock manipulations and invalidations using only the standard C++ interface. In addition, any time obtained during the clock's invalidation remains a fixed value.

Network Clock Automatic Adjustment

The automatic adjustment performed by the system, is automatically performed periodically when the device can connect to the network. Retries occur automatically when issues such as communication errors and server problems happen. Note that this clock might not return values, even when the device can use the network, as a result of the continued occurrence of communication errors or server problems.

Network Clock Time Accuracy

When incrementing, this clock accumulates errors on the order of several minutes per month because it uses nn::time::StandardSteadyClock. This accumulation error was cleared every time the time was corrected by the server. The delays in communication that occurred when correcting the time, however, introduced gaps on the order of dozens of seconds from the point of correction. See also Accumulation Errors for the Monotonic Increasing Clock .

◆ IsAccuracySufficient()

static bool nn::time::StandardNetworkSystemClock::IsAccuracySufficient ( )
static

Determines whether the clock is sufficiently accurate.

Returns
Returns a value indicating whether the clock is sufficiently accurate.
Return Values:

true Indicates that you can rely on the accuracy of the clock.

false Indicates that you cannot rely on the accuracy of the clock.

Precondition
  • The time library is initialized.

Details
Use this function if the accuracy of the network clock is of particular concern to your application during prolonged periods when not connected to the Internet.

If over 30 days have passed since the last adjustment of the time, the function will return false on the assumption that the cumulative margin of error has exceeded a certain amount. The return value of this function will change to true after the time has been adjusted. For more information, see the Network Clock Automatic Adjustment documentation.

Sometimes the value returned by this function might switch from true to false and vice versa even while the application is running. For more information about clock accuracy, see the Network Clock Time Accuracy documentation.
