Overview

Describes how to call the controller support applet from the sample application game screen.

File Structure

This sample program is located in Samples/Sources/Applications/HidControllerSequence/ .

Required Environment

Pair the controller to the development hardware ahead of time.

How to Use

Players move when you use the right and left directional buttons or when you pull the control stick down. Players jump when you press the A Button. Players run faster when you press the B Button. The interrupt dialog box opens when you press the + or - Buttons. Only the player who opens the interrupt dialog box has rights to select its controls when two or more players are involved.

Important Information

Make sure that the controller is sufficiently charged before using it.

Execution Procedure

Build the application and run it.

Description

Game screen processing has the following flow.

  • Confirm that controllers are connected.
  • Display the connection confirmation message.
  • Confirm player controller input.
  • Display the interrupt dialog box.
  • Transition to the title screen.

During single-player play.

The controller support applet is called under the following circumstances during single-player gameplay.

  • When the master player controller is lost
  • When detecting that a new wireless controller has been connected
  • When detecting input from a different controller

During multiplayer gameplay.

The controller support applet is called under the following circumstances during multiplayer gameplay.

  • When the operating player's controller is lost

During multiplayer gameplay, the controller support applet is called immediately without setting a margin if the system detects that the player's controller has disconnected.
