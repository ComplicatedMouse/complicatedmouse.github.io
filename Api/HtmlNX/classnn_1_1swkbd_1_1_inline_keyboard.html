Detailed Description

Class for using the software keyboard in inline mode.

Member Enumeration Documentation

◆ WindowMode

enum nn::swkbd::InlineKeyboard::WindowMode

Specifies whether to separate the keytops and the footer or to use a single window.

Enumerator
WindowMode_Single 

In single window mode, the footer and the keytops are scaled and moved together.

This setting is the default.

WindowMode_Floating 

In floating mode, the footer and the keytops are separated.

WindowMode_AdjacentFloating 

In adjacent floating mode, the footer and the keytops are displayed floating adjacent to one another.

No frame is displayed below the keytops.

◆ Initialize()

bool nn::swkbd::InlineKeyboard::Initialize ( void *  pWorkBuf)

Initializes the software keyboard in inline mode.

Precondition
  • pWorkBuf != nullptr.
Parameters
[in]pWorkBufPointer to the starting address of the buffer. Specify the size (in bytes) obtained by the nn::swkbd::InlineKeyboard::GetRequiredWorkBufferSize() function.
Returns
Returns the result of the software keyboard initialization process. Returns true if the process was successful. Returns false if the software keyboard has already been initialized.

Details
Note that if this function is called while the error viewer or another applet is running, the function waits and is released from waiting when that running applet ends. Conversely, if a function that starts another applet is called after Initialize() has been called, that other applet does not start until Finalize() is called. In addition, external factors such as user operation may automatically produce the same state that occurs when Finalize is called. When that happens, the callback function set by SetFinishedKeyboardCallback() is called, so handle the situation appropriately, for example by calling Initialize() again.

void nn::swkbd::InlineKeyboard::SetVolume ( float  volume)

Sets the overall software keyboard volume.

If initialized in inline mode, the software keyboard will not follow any settings configured with nn::oe::SetExpectedVolumeBalance. Parameters

◆ SetUserWordInfo()

bool nn::swkbd::InlineKeyboard::SetUserWordInfo ( const UserWordInfo &  info)

Passes the user dictionary to the software keyboard.

Parameters
[in]infoUser dictionary parameters.
Returns
true is returned if there are no problems, and false is returned if the settings could not be configured.
Precondition
  • The software keyboard is initialized.
  • info.pWorkBuf != nullptr.

Details
- The software keyboard initialization process continues until the FinishedInitializeCallback callback function is called, so display requests sent during initialization may sometimes be unsuccessful. Make sure that the FinishedInitializeCallback callback function has been called before calling this function.

  • The user data specified to this function is copied to the software keyboard memory. After the data is copied, the callback function specified in SetReleasedUserWordInfoCallback() is called. If you need to re-register the user dictionary, first make sure that the callback function specified in SetReleasedUserWordInfoCallback() has been called.
  • You cannot configure settings while the keyboard is displayed.

◆ SetWindowMode()

bool nn::swkbd::InlineKeyboard::SetWindowMode ( WindowMode  mode)

Specifies whether to separate the keytops and the footer in the displayed window.

Parameters
[in]modeWindow mode. Determines how the keytops and the footer are displayed.
Returns
Returns whether the window mode could be set.

Details
Specifies whether to display the footer and the keytops separately from one another. The default setting is WindowMode_Single. If WindowMode_Floating or WindowMode_AdjacentFloating is specified, the footer is displayed at a fixed size at the bottom of screen. If WindowMode_Single is specified, the footer and the keytops are integrated with one another and moved and scaled together. Although only the keytops are scaled, they are displayed adjacent to the top footer, so if you do not need the frame at the bottom of the keytops, specify WindowMode_AdjacentFloating to hide the frame at the bottom. This setting cannot be configured while the keyboard is being displayed.

◆ GetTouchRectangles()

int nn::swkbd::InlineKeyboard::GetTouchRectangles ( Rect *  primary,
Rect *  secondary 
) const

Gets rectangles used by the software keyboard for touch triggers.

Parameters
[out]primaryThe output address for the first rectangle.
[out]secondaryThe output address for the second rectangle. (Only for floating mode.)
Returns
Returns the number of rectangles.

Details
Gets information about the rectangles that the software keyboard uses to cover the screen. This will likely be used for masks displayed on the touch panel, so the coordinates are returned in a pixel-based coordinate system extending from (0, 0) to (1280, 720), with the origin at the upper-left corner. secondary only stores a value if a setting other than WindowMode_Single is specified in the SetWindowMode function. This function returns the rectangles regardless of the setting specified in SetTouchEnabled.

◆ Calc()

State nn::swkbd::InlineKeyboard::Calc ( )

The main software keyboard process.

Call this function in every game frame.

Returns
Returns the current display state of the software keyboard. When the software keyboard display state is State_Appear or State_SelectKeyTop, input from the controller is processed by the software keyboard. Do not use buttons other than those specified in AppearArg::invalidButtonFlag in your application.

Details
The decision to execute callback functions like SetFinishedInitializeCallback is based on the call to this function, so make sure that you call this function in every game frame until the software keyboard is closed.

◆ GetMaxHeight()

int nn::swkbd::InlineKeyboard::GetMaxHeight ( )

Gets the maximum height (in pixels) of the screen region that is currently usable by the software keyboard based on the currently specified parameters.

Returns
Returns the maximum height of the screen region that is currently usable by the software keyboard (in pixels from the bottom of the screen).

Details
The maximum height of the screen region that is usable by the software keyboard changes according to the keyboard layout, whether predictive text input is enabled, whether a user dictionary is currently in use, and whether a custom dictionary is currently in use. When any of these settings are changed, call this function as appropriate to check whether the maximum height of the available screen region has changed. The height also changes if only the footer or only the footer and the area showing conversion candidates is displayed (the time while Calc() is returning State_Miniaturized).

◆ SetFinishedKeyboardCallback()

void nn::swkbd::InlineKeyboard::SetFinishedKeyboardCallback ( FinishedKeyboardCallback  pCallback)

Sets the callback function to call when the software keyboard is closed.

Parameters
[in]pCallbackPointer to the callback function to call when the software keyboard is closed.

Details
The configured callback function is called inside the Calc() and GetImage() functions. The software keyboard is sometimes closed in a way that is unexpected by the application (that is, other than when the user intentionally closes it). In this case, the callback function specified in this function is called. Specifically, a callback occurs if the application calls Calc() or GetImage() after the HOME Button is pressed and system settings is started from the HOME Menu.

◆ GetImage()

bool nn::swkbd::InlineKeyboard::GetImage ( void *  pBuffer,
size_t  bufferSize 
)

Gets the rendered image of the software keyboard.

Parameters
[in]pBufferPointer to the buffer for getting the image.
[in]bufferSizeSize of the buffer for getting the image (in bytes).
Precondition
  • pBuffer != nullptr.
  • pBuffer satisfies the buffer size and alignment requirements returned by the GetImageMemoryRequirement function.
  • bufferSize is equal to the pOutRequiredSize value returned by the GetImageMemoryRequirement function.
Returns
Returns whether the software keyboard rendering result image was obtained.

Details
This function blocks for several milliseconds, so call the function on a thread for which blocking is not a problem. The image data is received in the following format.

  • Linear tiling.
  • Origin is in the upper left.
  • RGBA image in the sRGB color space with 8 bits per channel
