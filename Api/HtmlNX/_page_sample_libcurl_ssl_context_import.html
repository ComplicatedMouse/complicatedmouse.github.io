HTTP(S) download using easy interface with a SSL context shared between handles.

HTTP(S) using easy interface with a SSL context shared between handles that are used one at a time.

Overview

First the nifm library is used to request a network connection from the system. Once a network connection is acquired the socket and curl libraries are initialized. For each provided url an easy handle is created, options set, the shared nn::ssl::Context is imported, and resource fetched.

File Structure

The sample program is located at Samples/Sources/Applications/Libcurl/SslContextImport .

Environmental Requirements

You must use the settings manager in advance to import network connection settings. For details, see Registering Network Connection Settings for more information.

The sample fetches one or more resources based on the number of URLs given as arguments to the program. Below, we use http://www.openssl.org but any HTTP(S) URL that is available to your network will suffice.

Execution Procedure

You must provide one or more HTTP(S) URLs as arguments at execution time.
  Example

  • Run with default parameters.
    LibcurlSpeedTest http://www.openssl.org
  • Run with multiple parameters.
    LibcurlSpeedTest https://www.nintendo.com https://www.nintendo.co.jp

Description

The sample program has the following flow.

  • Initialize the nifm library with nn::nifm::Initialize(), submit a network request with nn::nifm::SubmitNetworkRequest(), and wait for nn::nifm::IsNetworkAvailable() to return true.
  • Initialize the socket library with nn::socket::Initialize().
  • Initialize the curl library with curl_global_init().
  • Initialize the HttpsHelper.
  • For each URL specified in the program arguments: – Create the easy handle with curl_easy_init(). – If enabled in the source file, enable all built-in compression accept encodings. – Set the easy handle options with curl_easy_setopt(). – If enabled in the source file, set up proxy settings. – If the URL contains "https" then the sample program uses CURLOPT_SSL_CONTEXT with the SSL communication. – Send an HTTP or HTTPS request with curl_easy_perform(). – On success the content of the HTTP(S) response is output to the log. On error the error is printed.
  • The libraries initialized on start-up are finalized in reverse order and the program exits.
