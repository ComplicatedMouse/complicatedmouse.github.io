Detailed Description

Analyzes the Exif information.

Details
This class analyzes Exif format binary to get the JPEG metadata. It supports Exif file format version 2.3.

Validity of the Obtained Information

The ExifExtractor class does not ensure the validity of the content of the strings and binary that it gets. It only checks that the strings are terminated with '\0' and that the range is the range defined for the element. As a result, the application must check for data corruption or threats when the JPEG data is altered externally or internally. For more information about which checks the ExifExtractor class performs on the various values, see the descriptions for the various member functions.

Object States

The ExifExtractor object can take the following three states.

  • Exif information not registered.
    • The state where Exif information binary has not been set in the ExifExtractor object.
  • Exif information registered.
    • The state where Exif information binary has been set in the object, but the information has not been analyzed.
  • Exif information analyzed.
    • The state of the ExifExtractor object after the last of the set Exif information binary has been analyzed.

◆ Analyze()

JpegStatus nn::image::ExifExtractor::Analyze ( )

Analyzes the specified Exif information binary and determines the composition of the Exif information.

Returns
Returns the execution result.
Return Values:

nn::image::JpegStatus_Ok Indicates that the process ended normally.

nn::image::JpegStatus_WrongFormat Indicates that the set Exif information binary has an invalid format. The metadata in this binary cannot be obtained.

Precondition
The state of the object must be either "Exif information registered" or "Exif information analyzed."
Postcondition
If the function returns JpegStatus_Ok, the state is "Exif information analyzed."

Details
Analyzes the specified Exif information binary and gets the composition of the Exif information, which is needed to get the various pieces of metadata. If invalid Exif information binary has been passed, the function returns JpegStatus_WrongFormat.

This function only inspects the composition of the Exif information and does not inspect the content. Even if this function returns JpegStatus_Ok, the ExtractMaker() and the other Extract-series functions might not necessarily get valid information.

Examples

const char* nn::image::ExifExtractor::ExtractMaker ( size_t *  pOutSize) const

Gets the "maker name" of the capture device for the JPEG data.

Parameters
[out]pOutSizeStores the size of the maker name, including the terminating '\0', in bytes.
Returns
Returns the pointer to the maker name. If the function cannot get this, it returns nullptr.
Precondition
The state must be "Exif information analyzed."

Details
Gets the maker name (0x010F) TIFF tag of the 0th IFD. Obtainable strings are inspected in the following terms.

  • All characters are in the range from 0x00 to 0x7F.
  • The string is terminated with '\0'.

If a string does not meet these terms, it is assumed not to contain the value.

◆ ExtractOrientation()

bool nn::image::ExifExtractor::ExtractOrientation ( ExifOrientation *  pOutValue) const

Gets the orientation of the image-capture device at the time of image capture.

Parameters
[out]pOutValueStores the image orientation.
Returns
Returns true if the value exists, or false otherwise.
Precondition
The state must be "Exif information analyzed."

Details
Gets the image orientation (0x0112) TIFF tag of the 0th IFD. If an appropriate orientation has been set for the image, this value can be used to determine the upright image. For example, if the device was rotated 90 degrees clockwise when the image was captured, the value is normally set to ExifOrientation_Rotate270. By rotating the image by 270 degrees clockwise you get the upright image.

If an undefined value has been set for the image orientation (0x0112), it is treated as if no value exists.

Precondition

Details
Gets the effective image width (0xA002) and height (0xA003)) Exif tags of the 0th IDF. If neither tag or only one or the other of these tags is registered, it is treated as if this size information does not exist.

When dealing with regular JPEG data, the effective dimensions are sometimes different from the values obtained by the nn::image::JpegDecoder::GetAnalyzedDimension() function.
