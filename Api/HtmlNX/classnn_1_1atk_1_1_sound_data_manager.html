Initializes the sound data manager.

Details
Sound data managers must be initialized before they are used. Initialization associates the sound data manager with a sound archive and allows it to be registered with the sound archive player.

Use GetRequiredMemSize to get the memory size required by the sound data manager.

Parameters
[in]pArchiveThe sound archive used by the player to which this sound data manager is registered.
[in]bufferPointer to the buffer. It must be aligned to a BufferAlignSize byte boundary.
[in]sizeBuffer size.
Precondition
  • pArchive != nullptr.
  • buffer != nullptr.
  • buffer is aligned to BufferAlignSize bytes.
  • size >= GetRequiredMemSize(pArchive).
Returns
Returns true if initialization succeeded, or false otherwise.

◆ SetFileAddressInGroupFile()

bool nn::atk::SoundDataManager::SetFileAddressInGroupFile ( const void *  groupFile,
size_t  size 
)

Registers a file contained in an independently loaded group file in the file management table.

Details
To output the group file (BFGRP) separately from the sound archive file (BFSAR), in SoundMaker, in Group List, set Output Method to User Managed.

If another address is already registered, it is overwritten.

Passing an invalid group file returns false.

For more information, see the Demo/snd/loadGroupManually/ sample and documentation.

Comparisons to LoadData

Compared to loading data by using SoundDataManager::LoadData, this function has the following advantages. 

   - You can register sound data for <strong>Goomba</strong> in one group,

and set the output mode for that group to User managed. (If the group name is GROUP_GOOMBA, GROUP_GOOMBA.bfgrp files are output.) 

   - The <strong>Goomba</strong> graphics data (model, texture, and so on)

and GROUP_GOOMBA.bfgrp files are archived. (SoundMaker does not provide a general-purpose archive. The user must prepare one.) 

   - This archive loads on the stage where <strong>Goomba</strong> appears.

This function ensures that the sound data for Goomba is already loaded when that character appears in a stage, helping you avoid situations where there is no sound because you forgot to load the data.

When using LoadData you must use some method to determine which stage Goomba appears on, and then load the Goomba sound data.

Put simply, this function is powerful when the placement of objects changes in various stages.

Note, however, that you must call this function or ClearFileAddressInGroupFile, which differs from the use of LoadData.

Also note that sounds cannot be played using the player heap if they only belong to the User managed group. (To enable playing, load the group file and register it in SoundDataManager using this function. The player heap is not used.)

To play sounds using the player heap, adjust the group configuration to include one of the following sets of relevant sound data. 

   - Not included in the <strong>User managed</strong> group.
   - Included in the <strong>User managed</strong> group, but also

included in a non-user managed group (link or embedded).

Returns
Returns true if registration succeeds, or false if an invalid group is passed.
Parameters
[in]groupFileAddress of an already loaded group file.
[in]sizeSize of that file.
See also
ClearFileAddressInGroupFile

size_t  size 
)

Unregisters the file specified by SetFileAddressInGroupFile.

Details
Note that the sound thread will be blocked.

This function must be called before the group file specified by SetFileAddressInGroupFile is freed from memory.

If multiple group files have been loaded into a contiguous memory space and you want to remove them from memory, you can call this function once to remove them all at the same time. To do so, pass the first address to groupFile, and pass the difference between the first and last address to size.

Parameters
[in]groupFileThe address of the group file to remove from registration.
[in]sizeSize of that file.

size_t  size

Destroys data in the file management table that was loaded independently.

Details
Note that the sound thread will be blocked.

This function must be called before releasing the independently loaded data from memory.

If multiple sets of independently loaded data have been loaded into a contiguous memory space and you want to free that data from memory, you can call this function once to remove them all. To do so, pass the starting address to this contiguous region, and for the size parameter, pass the difference between the starting address and the ending address.

Parameters
[in]addressThe starting address of the data to destroy.
[in]sizeSize of the data.

Loads the data in a sound archive.

Details
The memory required to load data is allocated by pAllocator. Returns false if sufficient memory cannot be allocated.

This function synchronously loads sound data. When loading sound data asynchronously, call this function from a thread used for loading. This function is thread-safe provided that no operations are performed during the loading process on the associated sound archive or on a sound heap passed in an argument.

Parameters
[in]idItem ID of the data to load.
[in]pAllocatorAllocates memory for loading data.
[in]loadFlagFlag that loads related data.
[in]loadBlockSizeThe block size (in bytes) when data is loaded in blocks.
Returns
Returns true if loading succeeded, or false otherwise.

◆ CancelLoading()

bool nn::atk::detail::SoundArchiveLoader::CancelLoading ( )
inherited

Cancels the executing instance of LoadData.

Details
If a nonzero value is specified for loadBlockSize, LoadData loads data in blocks of that size. This function cancels the block loading and returns control from LoadData.

Even if loading is canceled, the memory required for the original load remains secured. You must call a deallocation process such as SoundHeap::LoadState to enable the heap.

Returns
Returns true if data is currently loaded, or false otherwise.
