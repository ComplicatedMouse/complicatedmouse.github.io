Detailed Description

The connection.

Details
This class represents a connection implemented using the socket library and the ssl library.

Examples
WebSocketChat/WebSocketChat.cpp, and WebSocketEcho/WebSocketEcho.cpp.

Sets a function to use to cancel the connection process.

Parameters
[in]pFunctionThe function.
[in]pParamUser parameters for the function.

Details
The cancellation function set using this function is called periodically starting when the Connect() function is called until the Connect() function is complete. If the cancellation function returns true, the connection process is canceled.

◆ SetSocketOptionFunction()

void nn::websocket::Connection::SetSocketOptionFunction ( SocketOptionFunction  pFunction,
void *  pParam 
)

Sets the socket option function.

Parameters
[in]pFunctionThe function.
[in]pParamUser parameters for the function.

Details
Pass the TCP/IP socket, created when starting connection processing, to the socket option function. You can use the following functions in the socket option function to perform tuning on the socket parameters.

  • nn::socket::GetSockOpt()
  • nn::socket::SetSockOpt()

For more information about socket options, see NintendoSDK Documents > Supported Socket Options.

Attention
Changing socket options may result in unstable behavior. Make sure that you make a limited number of socket option changes.

◆ SetSslContextCertificateImportFunction()

void *  pParam 
)

Sets the SSL context certificate import function.

Parameters
[in]pFunctionThe function.
[in]pParamUser parameters for the function.

Details
This class creates an SSL context for when using SSL communication. If you want to import the application's own certificate into the SSL context, use this function to specify the import function.

The created SSL context gets passed to the specified import function. Use the following functions to actually import the certificate within the import function.

  • nn::ssl::Context::ImportClientPki()
  • nn::ssl::Context::ImportServerPki()

◆ Connect()

virtual nn::Result nn::websocket::Connection::Connect ( const char *  pHostName,
uint16_t  portNumber,
bool  isSecure 
)
virtual

Connects to the server.

Parameters
[in]pHostNameHost name.
[in]portNumberPort number.
[in]isSecureWhether to use SSL communication.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The process succeeded.
  • nn::websocket::ResultOperationCanceled The connection has been canceled. (The cancel function returned true.)
  • nn::websocket::ResultCommunicationError, Communication error. This result will also be picked up by nn::websocket::ResultOperationCanceled, so be careful about the order of error handling. must be handled.
  • nn::websocket::ResultOutOfMemory Memory allocation failed.
Precondition
  • pHostName != nullptr.

Details
This class supports proxy connections. It gets any proxy settings used for the current network connection and then uses the HTTP CONNECT method to start tunneling if those settings are valid.

When using SSL communications, an SSL handshake is performed after the connection to the socket is established.

Implements nn::websocket::ConnectionBase.
