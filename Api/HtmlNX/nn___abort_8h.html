Detailed Description

Declarations for API resources to abort execution when conditions are not satisfied.

◆ NN_ABORT

#define NN_ABORT (   ...)     NN_DETAIL_ABORT_UNLESS_IMPL(false, "NN_ABORT", ##__VA_ARGS__)

Stops execution.

Details
The abort observer is called before stopping execution.

If NN_ENABLE_ABORT_MESSAGE is defined, the specified message will be passed to the abort observer. If NN_DISABLE_ABORT_MESSAGE is defined, an empty message will be passed. If neither macro is defined, the message that is passed depends on the NintendoSDK build type:

  • Debug or Develop: Specified message
  • Release: Empty message

This macro is always enabled.

This macro is always enabled.

Parameters
[in]...Format string and arguments for the message to add to the log.

◆ NN_ABORT_UNLESS

#define NN_ABORT_UNLESS (   condition,
  ... 
)     NN_DETAIL_ABORT_UNLESS_IMPL((condition), NN_MACRO_STRINGIZE(condition), ##__VA_ARGS__)

Tests whether a condition has been satisfied and stops execution if it has not.

Details
Does nothing if the conditional expression specified in condition evaluates to true.

Stops execution if condition evaluates to false.

The abort observer is called before stopping execution.

If NN_ENABLE_ABORT_MESSAGE is defined, the specified message will be passed to the abort observer. If NN_DISABLE_ABORT_MESSAGE is defined, an empty message will be passed. If neither macro is defined, the message that is passed depends on the NintendoSDK build type:

  • Debug or Develop: Specified message
  • Release: Empty message

This macro is always enabled.

This macro is always enabled.

Parameters
[in]conditionThe condition to test.
[in]...Format string and arguments for the message to add to the log if the condition has not been satisfied.
