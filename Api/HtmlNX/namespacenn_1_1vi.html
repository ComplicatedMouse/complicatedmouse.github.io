◆ FrameStatus

enum nn::vi::FrameStatus : std::uint64_t

States for frames to be presented on the display.

Details
Once the application is finished programming rendering commands and invokes a presentation API, then the frame is enqueued for use on the display. When rendering is complete, then it will be presented to the display.

Enumerator
FrameStatus_Unknown 

The current status of the frame is unknown.

FrameStatus_Enqueued 

The frame will be presented to the display after rendering is complete.

FrameStatus_Presented 

The frame has been shown on the display.

◆ ListFrameHistory()

int nn::vi::ListFrameHistory ( FrameInfo *  pOutFrames,
int  frameCountMax,
const Layer *  pLayer 
)

Query the history of frames associated with a Layer.

Parameters
[out]pOutFramesBuffer to receive frame status information.
[in]frameCountMaxThe maximum number of FrameInfo objects that exist in the buffer.
[in]pLayerLayer handle.
Returns
The number of valid FrameInfo objects written to pOutFrames.
Precondition
  • VI library is initialized.
  • frameCountMax >= 0
  • pLayer != nullptr and pLayer is a valid handle.
Postcondition
The recent history of frames is written to pOutFrames and the number of entries written is returned.

If frameCountMax == 0 or pOutFrames == nullptr, no data is written and the maximum length of the history is returned. This is the maximum length that will be tracked at a time and may be different from the current length.

Details
The information is sorted by frame number in descending order (i.e. most recent frames first).

This function may not be supported on all platforms. In this case, preconditions will be verified, but the function will always return 0. Please check the Display documentation for platform-specific details.

Examples
ViTimestamp/ViTimestamp.cpp.

◆ CreateLayer() [1/3]

Result nn::vi::CreateLayer ( Layer **  pOutLayer,
Display *  pDisplay 
)

Creates a new layer on a particular display.

Parameters
[out]pOutLayerLayer handle.
[in]pDisplayDisplay handle.
Returns
Whether the layer was created.
Return Values:

ResultSuccess Layer created successfully.

ResultOperationFailed An error occurred.

Precondition
VI library is initialized.
pOutLayer != nullptr
pDisplay != nullptr and pDisplay is a valid handle.
Postcondition
A valid handle is written to pOutLayer.

Details
The layer is visible by default. Until a frame is submitted by the process, the layer will be blank.

The textures associated with this layer have restrictions due to a limitation with downscaling:
width <= 1920
height <= 1080
This restriction may be lifted in the future.

A particular display may only support a limited number of layers system-wide. If a process exceeds its quota then this function will return ResultOperationFailed. At minimum, a process can use a single layer to display content. To query how many layers can be created by the process, use ListDisplays().

If this API is used with a graphics API that does not have an explicit swap chain, like EGL, then the rendering dimensions will use default values of 1280x720.

Defines the rectangle inside the layer's textures that will be displayed.

Parameters
[in]pLayerLayer handle.
[in]xX-coordinate of the rectangle in texture coordinates.
[in]yY-coordinate of the rectangle in texture coordinates.
[in]widthWidth of the rectangle in texture coordinates.
[in]heightHeight of the rectangle in texture coordinates.
Precondition
  • VI library is initialized.
  • pLayer != nullptr and pLayer is a valid handle.
  • x >= 0
  • y >= 0
  • width >= 0
  • height >= 0
Postcondition
Only the rectangle specified by (x, y), (x + width, y), (x, y + height), and (x + width, y + height) will be displayed from the textures.

Details
Passing a width or height of 0 will reset the crop rectangle to match the dimensions of the texture.

This function serves the same purpose as nvnWindowSetCrop.
