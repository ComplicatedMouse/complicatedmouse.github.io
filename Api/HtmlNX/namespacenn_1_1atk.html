◆ SequenceUserProcCallback

typedef void(* nn::atk::SequenceUserProcCallback) (uint16_t procId, SequenceUserProcCallbackParam *param, void *arg)

The callback of a user procedure that can be used by sequence sounds.

Details
This callback is invoked by the userproc command in the sequence data. It is invoked from the sound thread during the frame in which the userproc command was processed.

The procId parameter is passed the value that was specified as a parameter to the userproc command of the sequence data.

The param parameter can be used within the callback. The value that is set as data can be accessed. You can overwrite this value to apply it to sound data.

For more information about the userproc command, see the Sequence Data Manual.

Parameters
[in]procIdThe procedure ID specified by the sequence data.
[in]paramThe parameter that can be used in the callback.
[in]argA user argument.
See also
SoundArchivePlayer::SetSequenceUserProcCallback
SequenceUserProcCallbackParam

◆ StreamRegionCallback

typedef StreamRegionCallbackResult( * nn::atk::StreamRegionCallback) (StreamRegionCallbackParam *param, void *arg)

The type definition for the callback used with the stream jump feature.

Details
This callback is called from the task thread when the header information for the stream sound is finished being loaded into memory, and when the block including the end of the region assigned to the waveform is finished being loaded into memory. Note that the callback is not called when playback has ended, but rather when the data is loaded into memory.

For more information about the callback and the task thread, see Stream Sounds and Threads in the programming guide.

Parameters
[in,out]paramThe parameters that store the previous region information and that will store the next region information.
[in]argThe user argument for the callback function.
Returns
Returns whether to end the sound or jump to the next region. Make sure that your implementation returns StreamRegionCallbackResult_Continue to jump to the next region, or StreamRegionCallbackResult_Finish to end with the current region.
See also
StreamRegionCallbackParam
StreamRegionCallbackResult
SoundStartable::StartInfo

◆ SequenceTickRoundingMode

enum nn::atk::SequenceTickRoundingMode

The mode representing the rounding direction for sequence sound ticks.

Details
The mode represents whether to process the sequence sound tick in the former audio frame or the latter audio frame when the timing of the tick falls between audio frames.

See also
SoundArchive::SoundArchivePlayerInfo
Enumerator
SequenceTickRoundingMode_RoundUp 

Rounds up the sequence sound tick, in terms of the timing.

Details
In some cases, sequence commands will be processed with a timing that is slower than the specified tempo.

SequenceTickRoundingMode_RoundDown 

Rounds down the sequence sound tick, in terms of the timing.

Details
This playback method is compatible with NintendoSDK 13.0.0 and earlier. Sequence commands are sometimes processed with a timing that is faster than the specified tempo.

◆ BiquadFilterType

enum nn::atk::BiquadFilterType

Enumerated type that represents the kind of biquad filter.

Details
With Biquad filters, by changing the filter coefficients You can apply various types of filters.

Specify any of the constants in the range from BiquadFilterType_LowPassFilter to BiquadFilterType_BandPassFilter2048 to use the preset coefficients provided by the library.

The values for these preset filter coefficients are those that result when setting the audio renderer's rendering sample rate to 32 kHz. Note that you might need to change the filter coefficients, depending on the audio renderer's sampling rate.

Use a user-defined filter coefficient instead of the preset one when using a rendering sample rate that is not 32 kHz. For more information about using a user-defined BiquadFilter, see SoundSystem::SetBiquadFilterCallback. For more information about changing the rendering sample rate, see SoundSystem::SoundSystemParam::rendererSampleRate.

Specify BiquadFilterType_None if you are not going to use a biquad filter.

To register a user-defined filter coefficient designed by the user, specify a value in the range from BiquadFilterType_UserMin to BiquadFilterType_UserMax for the type number.

The biquad filter can be used from the SoundHandle::SetBiquadFilter function.

◆ PauseMode

enum nn::atk::PauseMode

The use of internal features is prohibited.

Represents the pause settings for SoundHandle and the like.

Enumerator
PauseMode_Nw4fSndCompatible 

Represents the pause setting that retains compatibility with nw::snd in NW4F.

Details
When pausing, the fade-out volume drops to zero on fadeFrames number of calls to SoundArchivePlayer::Update, and the pause completes on (fadeFrames + 1) number of calls. When resuming, the fade in completes in fadeFrames frames. If fadeFrames is 0, the behavior is the same as if -1 were specified.

Note that the fade-out volume reaches zero and the pause completes at different times, so even if 0 or 1 is specified for fadeFrames the pause will not complete until SoundArchivePlayer::Update has been called twice, and the playback position will advance in the muted state.

PauseMode_PauseImmediately 

Represents the setting where the pause completes at the same time that the fade out completes.

Details
When pausing, the pause completes when SoundArchivePlayer::Update has been called fadeFrames times. When resuming, the fade in completes in fadeFrames frames. If fadeFrames is 0, the behavior is the same as if -1 were specified.

When a pause is performed using the PauseMode_Nw4fSndCompatible setting, at the very least the playback position will advance by the amount of the SoundArchivePlayer::Update update interval. For this reason, use this setting instead if you are going to cue a short audio source using Pause.

PauseMode_Default 

This is the default pause setting.
