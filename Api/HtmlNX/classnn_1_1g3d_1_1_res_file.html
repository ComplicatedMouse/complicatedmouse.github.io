◆ Unrelocate()

void nn::g3d::ResFile::Unrelocate ( )

Puts ResFile in a relocatable state.

Details
After Unrelocate has been called, resource files can be moved to a different memory address. Before moving, you must call Cleanup() for ResFile. To make it available again, you must call Relocate() or ResCast().

void nn::g3d::ResFile::Setup ( nn::gfx::Device *  pDevice,
nn::gfx::MemoryPool *  pMemoryPool,
ptrdiff_t  memoryPoolOffset,
size_t  memoryPoolSize 
)

Sets up the resource.

Parameters
[in]pDevicePointer to the device.
[in]pMemoryPoolPointer to the memory pool where ResFile is located.
[in]memoryPoolOffsetOffset in the memory pool where ResFile is located.
[in]memoryPoolSizeThe size of the memory pool where ResFile is located.
Precondition
  • pDevice is initialized.
  • ResFile is loaded to pMemoryPool.

Details
Calls Setup for subordinate models, textures, and animations. Call before constructing the instance.

The user loads ResFile to the locally prepared memory pool, which is then used by this function to set up ResFile. This function cannot be used to create memory internally. If ResFile is located in the memory pool, use the alignment of the file. To get the alignment of the file, get the header using GetFileHeader() and then call nn::util::BinaryFileHeader::GetAlignment(). Because ResFile involves CPU access, we recommend setting the memory pool property to CpuCached. Performance will be lower if you set CpuUncached. You cannot use CpuInvisible.

◆ BindTexture()

BindResult nn::g3d::ResFile::BindTexture ( TextureBindCallback  pCallback,
void *  pUserData 
)

Resolves references for the texture.

Parameters
[in]pCallbackPointer to the callback function invoked when resolving texture references.
[in]pUserDataPointer to the user data to be passed as a parameter when the callback is invoked.
Returns
Returns BindResult.
Precondition
  • pCallback != NULL.

Details
Pass the texture name as an argument and call pCallback. For pCallback, return a pointer to an appropriate nn::g3d::TextureRef value. When an invalid nn::g3d::TextureRef value is returned as a callback value, the texture reference resolution is treated as if it failed. Inappropriate access may occur if the render process is performed without resolving texture references. Check the return value to confirm whether texture reference resolution succeeded.

Callbacks only occur for textures for which there is no reference resolution yet. You can determine whether a reference has been resolved by using either the nn::g3d::ResMaterial::IsTextureBound() or nn::g3d::ResMaterialAnim::IsTextureBound() function.

Examples
G3dDemo/Simple.cpp, MiiG3dHeadwear/MiiG3dHeadwear.cpp, and MiiG3dSimple/MiiG3dSimple.cpp.
