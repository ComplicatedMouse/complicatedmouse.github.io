Describes how to reset the count for continuous idle time using application-specific definitions.

Overview

This sample program describes how to reset the count for continuous idle time with application-specific definitions using the application operating environment library. It is designed for applications that have their own unique styles of operation.

File Structure

This sample program is located in Samples/Sources/Applications/OeIdlePolicyApiForApplicationSpecificControl/ .

Required Environment

This sample program can only be built and run on the NX platform.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it. To check the effects of the features called in this sample program, you must set the time to auto-sleep in handheld mode to the shortest time (one minute) in the system settings ahead of time, and start the program in handheld mode.

Sample Program Description

This sample program acts as if there was some user operation after 50 seconds and resets the count for continuous idle time.

This sample program has the following flow.

  • The following loop is repeated indefinitely.
    • Sleep for 50 seconds.
    • Call nn::oe::ReportUserIsActive() and explicitly declare that there was a user operation.
      • By calling this function, auto-sleep will not be activated while the sample program is running even if there is no controller input or other activity.
      • If screen burn-in reduction was active until this time, the call to this function will cancel it.
