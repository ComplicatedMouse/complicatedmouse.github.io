A sample program for the thread feature.

Overview

This sample shows the thread feature.

For more information about using the thread feature, see the Thread Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsThread/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Two threads are created with nnMain().
  • The two threads are ThreadFunction1() and ThreadFunction2().
  • ThreadFunction1() generates its own thread information and ends.
  • ThreadFunction2() waits for ThreadFunction1() to end.
  • ThreadFunction2() generates its own thread information.
  • ThreadFunction2() changes its own thread name.
  • ThreadFunction2() changes its own priority.
  • ThreadFunction2() again generates its own thread information and ends.
  • nnMain() waits until the two threads complete.
  • The two threads are destroyed by nnMain().

This sample program performs changing and getting the thread name and changing and getting the priority. nnMain() controls from the creation of the main thread to the termination of the main thread.
