Detailed Description

Callback class for checking the time.

Details
This class is used in nn::time::ClaimPeriodicBenefitWithUserSystemClock().

Classes that inherit this class must implement the processes for checking the time (IsSameTiming() and IsMatched()). Such class objects can then be used by passing them to nn::time::ClaimPeriodicBenefitWithUserSystemClock().

The IsSameTiming() and IsMatched() functions get called as appropriate via nn::time::ClaimPeriodicBenefitWithUserSystemClock(), but never implement processes that assume these functions will always be called successfully.

Parameters

Returns
The determination result.
Return Values:

true The objects represent the same timing.

false The objects do not represent the same timing.

Details
This function prevents benefits from being repeatedly claimed at the same time.
Implement this function such that when two time objects are compared, the function returns true if the time objects represent the same benefit receipt timing and returns false otherwise.
If you do not want to prevent benefits from being repeatedly claimed, implement this function to always return false.

The following implementations are examples.

  • To prevent benefits from being claimed again on the same day, return true if lhs and rhs represent the same date.
  • To prevent benefits from being claimed again during the same week, return true if lhs and rhs belong to the same week.

Determines whether the specified time represents a valid time when benefits can be claimed.

Parameters
[in]calendarTimeThe time at which to perform the determination.
Returns
The determination result.
Return Values:

true The specified time is a valid time when benefits can be claimed.

false The specified time is not a valid time when benefits can be claimed.

Details
This function determines whether the specified time represents a valid time when benefits can be claimed (without comparing the specified time to the previous time when benefits were claimed).
Implement the function to return true if the time object specified for calendarTime represents a time when benefits can be claimed, and to return false otherwise.

The following implementations are examples.

  • If you want it to be possible to claim benefits at any time on a particular day after the date changes or the week changes, for example, do nothing and simply return true.
  • If you want to limit benefit claims to Sunday of every week, return true if the day of week component of calendarTime is Sunday. (Use nn::time::GetDayOfWeek() to get the day of week.)
  • If you want it to be possible to claim benefits every day from 09:00 to 12:00, return true if the time component of calendarTime is within this time range.
