A sample program for the fiber feature.

Overview

This sample shows how to use the fiber feature.

For more information about using the fiber feature, see the Fiber Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsFiber/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Two fibers are initialized in nnMain().
  • The two fibers use FiberFunction1() and FiberFunction2().
  • The SwitchToFiber() function is used to switch from nnMain() to FiberFunction1().
  • The SwitchToFiber() function is used to switch from FiberFunction1() to FiberFunction2().
  • The SwitchToFiber() function is used to switch from FiberFunction2() to FiberFunction1().
  • The SwitchToFiber() function is used to switch from FiberFunction1() to FiberFunction2().
  • The SwitchToFiber() function is used to switch from FiberFunction2() to nnMain().
  • The two fibers are destroyed in nnMain().

The SwitchToFiber() function is called in nnMain() and the fiber functions to switch to another fiber. Before and after each call, the order of execution is logged and the identity of the current fiber is checked.

As shown in this sample program, when you use the fiber feature, you can call the SwitchToFiber() function to temporarily suspend processing and move execution to another fiber, and when control returns to a fiber, it resumes execution from the code where SwitchToFiber() was called.
