This section describes a sample program for an independent build system.

Overview

This sample program builds an independent build system that does not use the VSI provided by NintendoSDK. This sample program can be built using GNU Make in MSYS2.

File Structure

This sample program is located in Samples/Sources/Applications/IndependentBuildSystem .

Required Environment

MSYS2 (64-bit) Download the installer from the MSYS2 homepage and install it.
GNU Make Run the following command in MSYS2 and install it.
pacman -S make

When running Make, the NINTENDO_SDK_ROOT environment variables must have already been configured. If you installed NintendoSDK from NDI, the NINTENDO_SDK_ROOT is configured automatically.

Operation has been verified in the following environment.

  • msys2-x86_64-20190524.exe
  • GNU Make 4.2.1 (Built for x86_64-pc-msys)

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are saved to a log.

Execution Procedure

In MSYS2, set Samples/Sources/Applications/IndependentBuildSystem as the current directory and execute the make command. The application is built and generated under the Samples/Sources/Applications/IndependentBuildSystem/Binaries directory.

You can pass the following options to make.

Options Available Values
BUILD_TARGET NX-NXFP2-a32 / NX-NXFP2-a64
BUILD_TYPE Debug / Develop / Release
APPLICATION_FORMAT nspd / nsp

For example, you can use the following command to export a 32-bit binary file for the NX development hardware environment as an NSP file with the Release build type.

<tt>{.sh}</tt>
<tt>make BUILD_TARGET=NX-NXFP2-a32 BUILD_TYPE=Release APPLICATION_FORMAT=nsp</tt>

You can use Target Manager or RunOnTarget to run the built application. For more information, see Features > Controlling the Development Hardware > Target Manager or Features > Controlling the Development Hardware > RunOnTarget. For more information about debugging applications, see Features > Building > VSI > Debugging a Built Program.

This sample program can only be built for and run on development hardware. Note that builds for the Windows environment are not supported.

Description

GNU Make references the following settings files.

Files. Content
NintendoSdkSettings.mk This file summarizes the settings required for applications that use NintendoSDK.
SampleSettings.mk The file that summarizes the settings used in this sample.
SampleStaticLibrary.mk A sample Makefile that creates a static library.
SampleDynamicModule.mk A sample Makefile that creates a dynamic library.
SampleApplication.mk A sample Makefile that creates an application program.
Makefile Specifies SampleApplication.mk and executes make.

NintendoSDK assumes that some compiler options and linker options are specified or not specified. For more information about the options that must be specified, see NintendoSdkSettings.mk. For more information about each option, see NintendoSDK Documents > Features > Building > Build System > Guide to Setting Up the Build System Environment > Creating an Independent Build System (NX Development Hardware Environment).
