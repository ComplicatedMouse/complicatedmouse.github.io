Detailed Description

Namespace for host-target socket communication.

Feature Overview

Host target communication socket is abbreviated as HTCS. The HTCS library uses a socket interface to provide a means of communication between the host and the target. You cannot ordinarily use HTCS library features for a release build type. Define NN_ENABLE_HTC to use those features within a release build type.

How to Use

On the target side, this library is essentially used the same way as a socket. On the host side, use a socket interface such as Winsock. To establish a connection with the host, you must first run TargetManager.exe. For more information about the connection procedure, see the connection sequence section of the manual.

◆ Select()

int nn::htcs::Select ( int  numberOfDescriptors,
FdSet *  pReadDescriptors,
FdSet *  pWriteDescriptors,
FdSet *  pExceptDescriptors,
TimeVal *  pTimeout 
)

Monitors multiple synchronous input and output.

Parameters
[in]numberOfDescriptorsThe value of the maximum value of the descriptor ID, plus one.
[in,out]pReadDescriptorsFdSet for the read descriptor.
[in,out]pWriteDescriptorsFdSet for the write descriptor.
[in,out]pExceptDescriptorsFdSet for the exception descriptor.
[in]pTimeoutThe timeout time.
Returns
The result of the operation. If the process does not succeed, the descriptor set does not change.
Return Values:

1 Of the descriptors belonging to the descriptor set, returns the number for which preparations are complete.

0 The process timed out.

-1 The process failed. Get the error details using the GetLastError() function.

Details
The descriptor set is stored as a bit field in an integer array. Select() monitors the input and output descriptor sets that have addresses passed by pReadDescriptors, pWriteDescriptors, and pExceptDescriptors, and among these descriptors, checks whether there are descriptors for which read preparations are completed in pReadDescriptors, whether there are descriptors for which write preparations are completed in pWriteDescriptors, and whether there are pending descriptors in the exception state in pExceptDescriptors. The only exception state that can be detected is data outside the bandwidth received by the socket. Descriptors equal to the initial numberOfDescriptors number are checked in each set. In other words, descriptors are checked for an ID belonging to a descriptor set from 0 to numberOfDescriptors - 1. When Select() returns from the process, the specified descriptor set is replaced with a subset consisting of descriptors for which preparations for the requested operation are completed. At the same time, the total number of descriptors for which preparations are completed among all descriptor sets is returned as the return value.

If pTimeout is not a null pointer, this argument specifies the maximum time to wait for Select() to complete. The specified time may be extended by system activity.

If pTimeout is a null pointer, Select() is infinitely blocked.

If Select() is used for polling purposes, the pTimeout parameter must not be a null pointer, and a TimeVal structure set to a value of 0 must be specified.

If there are no target descriptors for processing, any descriptor set can be specified as a null pointer.

Examples
Htcs/HtcsTargetApp/HtcsTargetApp.cpp.
