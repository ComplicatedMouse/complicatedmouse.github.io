Detailed Description

(Beta version.) Provides features for editing sound archives.

Details
When you start editing, the content of the sound project on the connected SoundMaker is replaced by the content of the specified SoundArchive.

Initializes the memory used by SoundArchiveEditor, sets an upper limit for the number of editable sounds, and sets the sound archive that will be the target of editing.

Details
Always call this function before using SoundArchiveEditor.

When you start editing a sound archive, access to the specified SoundArchive is hooked by this SoundArchiveEditor, and the content of the sound project on SoundMaker has priority.

SoundEditSession controls starting and ending sound archive editing.

About the Buffer

This buffer is used to cache various types of data, including parameters and waveform files.

Make sure that there is enough memory in advance because the amount of memory used changes depending on the actual number of sounds and waveforms.

Make sure that you specify a buffer of adequate size for the data that will be the target of editing.

If you run out of memory while using SoundArchiveEditor, you will not be able to add any more items for editing.

In addition, if items are targeted and untargeted for in game editing, fragmentation of the memory inside the buffer will progress.

We recommend allocating a buffer that is on the large side, even if there is still free space, because sometimes contiguous memory cannot be allocated, causing a failure.

If memory becomes fragmented, you can resolve the problem by using SoundMaker to resend the entire cache.

[in]bufferThe buffer that SoundArchiveEditor will use.
[in]bufferLengthThe length of buffer.

[in]soundArchivePlayerThe sound archive player to edit.

◆ UpdateCaches()

Result nn::atk::viewer::SoundArchiveEditor::UpdateCaches ( ICanceller *  canceller = NULL,
void *  cancellerParam = NULL 
)

Updates the cache.

Details
Caches all uncached items.

This function must be called to enable (to cache) the items specified as edit targets by SoundMaker.

We recommend either closing all caches or running this function on a worker thread because it does not return control until an error occurs.

Use the UpdateCachesHelper helper class to create a worker thread and call this function from that thread.

If you use UpdateCachesHelper, you do not need to make a separate call to this function.

Canceling the Process

Specify canceller to cancel the cache updating process.

If the ICanceller::IsCanceled() function returns true while cache is being updated, the UpdateCaches() function waits for the currently processing item cache process and then returns control.

The standard implementations of ICanceller are Canceller and TimeoutCanceller.

Inherit ICanceller to implement your own conditions for canceling a process.

[in]cancellerThe cache-update cancel object.
[in]cancellerParamThe parameter passed to the cancel object.
Returns
Returns the execution result.

◆ GetItemCacheState()

CacheState nn::atk::viewer::SoundArchiveEditor::GetItemCacheState ( const char *  name) const

Gets the cache state for the specified item.

Details
If this function returns CacheState_Cached, calling the StartSound() function with the target SoundArchivePlayer replaces parameters and data.

If the function returns CACHE_STATE_CACHING, either the system is in the process of caching, or caching might have stopped because of insufficient memory or another error.

Use the IsOutOfMemory function to check for insufficient memory, and use the GetLastError function to check for other errors.

Parameters
[in]nameThe name for a sound or bank.
Returns
Returns the cache state for the specified item.
