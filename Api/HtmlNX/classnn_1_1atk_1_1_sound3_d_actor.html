Detailed Description

The 3D sound actor class representing sound sources in 3D space.

Details
A single actor can manage multiple sounds at the same time.

While the actor instance is in existence, the nn::atk::Sound3DManager class calculates and sets the sound parameters based on the position information set for the actor. The sounds for that actor instance continue to play even after the instance ceases to exist, but the 3D parameters are no longer updated.

Because this type has members that are of the nn::util::Vector3fType type, it must be aligned when allocating in memory. Align to a nn::atk::Sound3DActor::BufferAlignSize byte boundary.

Initializes a 3D sound actor.

Details
3D sound actors must be initialized before they are used. If the constructor with arguments is called, there is no need to call this function. (However, calling this function anyway is not a problem.)

The SoundActor::Initialize function is called internally by this function.

To reuse the same Sound3DActor instance, first call the Finalize function and then call this function again.

The SoundArchivePlayer::Update function might throw an exception if you finalize the Sound3DManager instance passed to this function before the sound played by Sound3DActor has finished.

The exception is thrown because the sound played by the Sound3DActor object references the Sound3DManager object as a callback. If you are going to finalize the Sound3DManager instance, make sure that all sounds that are referencing the object have finished playing first.

◆ SetPosition()

void nn::atk::Sound3DActor::SetPosition ( const nn::util::Vector3fType &  position)

Sets information about the actor position.

Details
It sets the position information for 3D sound actors. The coordinates are used to calculate 3D sound parameters.

When new position information is set, the difference between that information and the information for the previous position is calculated, and velocity is set automatically for the 3D sound actor. The velocity of the actor causes pitch changes because of the Doppler effect. The velocity can also be set explicitly by using nn::atk::Sound3DActor::SetVelocity.

When the coordinates jump to a different position, the velocity is calculated based on the transverse distance. This can produce abrupt changes in pitch if the movement is extremely fast. Note that in such cases, you must explicitly set the velocity by calling nn::atk::Sound3DActor::SetVelocity after calling this function.

Parameters
[in]positionThe position coordinates of the actor.

◆ StopAllSound()

void nn::atk::SoundActor::StopAllSound ( int  fadeFrames)
inherited

Stops all sounds being played by the actor.

Details
Performs processing identical to that when stopping a sound by using the handle class.

You can fade out over the number of frames specified by fadeFrames. Fade-out is not performed when 0 is specified. However, for sounds playing as sequence sounds, the sounds stop after all attenuation has completed in the release stage of the envelope.

The volume control for fade-out is also used for fade-in. Number of elapsed frames during a fade out. The number of fade-out frames correlates to the speed of change from maximum volume to silence. If a fade out is specified during a fade in, or in other similar situations, it may complete sooner than the specified frame count.

Parameters
[in]fadeFramesNumber of elapsed frames during a fade-out. The number of frames is based on the number of calls to SoundArchivePlayer::Update. Specify a value of 0 or greater for the fadeFrames parameter. If you specify a negative value, the function works the same as if you had specified 0.

◆ SetPitch()

void nn::atk::SoundActor::SetPitch ( float  pitch)
inherited

Changes the pitch of the actor.

Details
The actor pitch affects all sounds that the actor is playing.

The value specified by this function operates independently of other pitch parameters, and they are all layered.

pitch specifies the frequency ratio. In other words, pitch is not affected if 1.0 is specified. Specifying 2.0 doubles the playback frequency, increasing the pitch by one octave. If 0.5 is specified, the pitch is one octave lower. The default value is 1.0.

Note that after this value is combined with the other pitch parameters, the final pitch is clamped to the range from nn::audio::VoiceType::GetPitchMin() to nn::audio::VoiceType::GetPitchMax(). For more information about combining pitch parameters, see the atk Library Sound Parameter Control Operations documentation.

Parameters
[in]pitchThe frequency ratio of the pitch change.

void nn::atk::SoundActor::SetBiquadFilter ( int  type,
float  value 
)
inherited

Changes the type and coefficient of the actor's biquad filter.

Details
The actor's biquad filter effect is applied to all sounds currently being played by the actor.

When the biquad filter is configured in multiple locations, the filter effect is not layered, but rather set to the configuration in the location with the highest priority.

type is the filter type, and it takes an enumerator of the BiquadFilterType enumerated type. In addition to the presets, you can also select a user-registered filter.

For type, specify a value between BiquadFilterType_Min and BiquadFilterType_Max. For more information about BiquadFilterType_Min and BiquadFilterType_Max, see BiquadFilterType.

value is the extent to which the filter is applied, specified in the range from 0.0f to 1.0f. The meaning of this value varies depending on the type of coefficient used by the filter.

When used together with the low-pass filter from SetLowPassFilterFrequency, the effects are overlaid.

Parameters
[in]typeThe filter type (BiquadFilterType_Min to BiquadFilterType_Max).
[in]valueHow to apply the filter (0.0f to 1.0f).

Changes the actor's pan (left/right localization) for the specified output destination (GamePad or main).

Details
The actor pan affects all sounds that the actor is playing.

The value specified by this function operates independently of other pan parameters, and they are all layered.

pan specifies a value that represents the relative change in the location of the sound. If 0.0 is specified, the pan value that is set within data is unchanged. If 1.0 is specified, the sound positioned in the center becomes the sound positioned on the right end. If 1.0 is specified, the sound positioned in the center becomes the sound positioned on the left end. The default value is 0.0.

Parameters
[in]deviceThe output device.
[in]panRelative change in the pan, with 0.0 as the base value.

◆ SetPlayableSoundCount()

void nn::atk::SoundActor::SetPlayableSoundCount ( int  actorPlayerId,
int  count 
)
inherited

Sets the number of sounds that can be played at the same time.

Details
There are four actor players in a sound actor. You can set a limit on the number of sounds that can be simultaneously played by each actor player. This setting is independent of the simultaneous sound count that is set in the sound player.

If the number of sounds for playback exceed the configured sound count, the player priorities for each sound are compared, and the sounds with the lowest priority are stopped.

The initial value is unlimited for actor player 0, and 1 for all other actor players.

Parameters
[in]actorPlayerIdThe actor player number.
[in]countThe number of sounds that can be played simultaneously.
