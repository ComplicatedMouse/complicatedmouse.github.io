A sample program for the message queue feature.

Overview

This sample shows using the message queue feature.

For more information about using the message queue feature, see the Message Queue Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsMessageQueue/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Two threads are created and one message queue is initialized with nnMain().
  • ThreadFunction1() sends a string pointer to the message queue.
  • ThreadFunction2() sends the string received from the message queue to the log.
  • This process is repeated for the number of lines.
  • nnMain() waits until the two threads complete.
  • The two threads are destroyed by nnMain().

The two threads send a pointer to a string from ThreadFunction1() to ThreadFunction2() via the message queue. Three pieces of data can be sent at one time, and if the buffer is full, ThreadFunction1() is blocked. Also, when the buffer has space, the wait is released, and the next send is performed.

ThreadFunction2() is blocked while the buffer is empty, and if even a single piece of data is sent to the buffer, that data is received and added to the log. If the end mark (0x00) is sent last, the threads terminate.
