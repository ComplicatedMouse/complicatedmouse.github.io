The description of the sample program that uses multiple features within NintendoSDK.

Overview

A composite sample comprising the following features.

  • Primitive rendering using the nns::gfx::PrimitiveRenderer library
  • Status acquisition for human-interface devices using the nn::hid library
  • Audio playback using the audio renderer
  • File handling using the nn::fs library
  • Writing user save data using the nn::fs library
  • Displaying current times and detecting time operations using the nn::time::StandardUserSystemClock library
  • Status acquisition for SixAxisSensor using Npad

File Structure

This sample program is located in Samples/Sources/Applications/Demo1/ .

Required Environment

You must be able to use the screen display.

How to Use

Switch between screens using the + Button or - Button. Terminate the Demo1 sample program by pressing the + and - Buttons at the same time. The other operations work in the same way they worked in the various samples from which they were adopted.

Important Information

Create an account before running the sample. The user save data process will create a file after the sample runs.

Execution Procedure

Build the sample program and then run it.

Description

This sample program has the following flow.

  • Initialize the library to manipulate the application operating environment.
  • Select the Normal Mode performance.
  • Select the Boost Mode performance.
  • Initialize memory resources for the graphics system.
  • Set the memory allocator for the graphics developer tools and the debug layer.
  • Start the various demo programs.
  • Free all objects used.
