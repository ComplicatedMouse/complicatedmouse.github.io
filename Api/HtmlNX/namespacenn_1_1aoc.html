Detailed Description

Namespace for the downloadable content library.

Feature Overview

The downloadable content library provides functionality for accessing downloadable content.

Linked With the fs Library

The downloadable content library does not directly provide functionality for loading downloadable content. The fs library must be used to load downloadable content.

To load downloadable content, you must call the functions in the following order.

  • Call nn::aoc::ListAddOnContent to get the index for the mount target.
  • Call nn::fs::MountAddOnContent to mount the downloadable content. (Data obtained using nn::aoc::ListAddOnContent is used at this point.)
  • Use the fs library to access mounted downloadable content.
  • Call nn::fs::Unmount to unmount the downloadable content.

For more information, see the sample.

Lists the downloadable content indices for installed and licensed downloadable content instances.

Parameters
[out]outIndicesThe buffer for storing the listed indices.
[in]offsetThe offset of the downloadable content indices to be listed.
[in]countThe maximum number to list.
Returns
Returns the number of listed downloadable content indices.
Precondition
  • (The maximum number of elements for outIndices) >= count.
  • offset >= 0.

Details
Lists the downloadable content indices for installed and licensed downloadable content instances. Stores the number of downloadable content indices specified with count, beginning from the offset index for the listed downloadable content indices, to outIndices. Operations are undefined when count exceeds the number of elements in the buffer specified with outIndices. The listed content indices are sorted in ascending order.

Examples
AocSimple/AocSimple.cpp, and EcCatalog/Aoc/AocItemScene.cpp.

◆ GetAddOnContentListChangedEvent()

void nn::aoc::GetAddOnContentListChangedEvent ( os::SystemEvent *  outEvent)

Gets the notification event that is used when usable downloadable content has changed.

Parameters
[out]outEventEvent object.

Details
Notification only occurs when there was an update after getting the event. List available downloadable content after using this function to get the event. This will allow you to accurately detect available downloadable content.

This event is an AutoClear event. Reuse this event to detect changes for the next available downloadable content after receiving the initial notification.

The available downloadable content are updated in the following situations.

  • When downloadable content has been installed by either purchasing or re-downloading the content
  • When the available downloadable content changes along with the opening and closing of user accounts.

You must separately specify in the NMETA file if you want to enable the installation of downloadable content when running this function. The available downloadable content changes along with the opening and closing of user accounts only if this has been specifically allowed in the NMETA file.

◆ ShowAddOnContentLostError()

int  count

Displays the downloadable content lost error.

Parameters
[in]indicesArray of the lost AddOnContentIndex.
[in]countNumber of indices.

Details
Displays a message that communicates to the user that the downloadable content was lost. Use this feature in cases where the save data was created with specific downloadable content present but that downloadable content is currently unavailable.

This function only informs the user that the content was lost and prompts them to resolve the issue, and does not resolve the situation itself. In addition, because there is no assurance that you can get the content again, we recommend designing your application to avoid excessive penalties to the user even if the downloadable content does not exist. For example, enable save data to continue to be used in locations unrelated to the downloadable content.

This API has one error display function, and the caller function is blocked while the error is being displayed.
