Detailed Description

◆ NN_ASSERT

#define NN_ASSERT (   condition,
  ... 
)     NN_DETAIL_ASSERT_IMPL((condition), NN_MACRO_STRINGIZE(condition), ##__VA_ARGS__);

Tests whether a condition is satisfied.

Details
Does nothing if the conditional expression specified in condition evaluates to true.

If condition evaluates to false, an assertion failure handler is called.

If NN_ENABLE_ABORT_MESSAGE is defined, the specified message will be passed to the assertion failure handler. If NN_DISABLE_ABORT_MESSAGE is defined, an empty message will be passed. You cannot define NN_ENABLE_ABORT_MESSAGE and NN_DISABLE_ABORT_MESSAGE at the same time. If neither macro is defined, the message that is passed depends on the NintendoSDK build type:

  • Debug or Develop: Specified message
  • Release: Empty message

This macro is enabled if NN_ENABLE_ASSERT is defined. Logs are disabled and the macro is expanded into an empty operation when NN_DISABLE_ASSERT is defined. You cannot define NN_ENABLE_ASSERT and NN_DISABLE_ASSERT at the same time. If neither macro is defined, the enabled determination depends on the NintendoSDK build type.

  • Debug or Develop: Enabled.
  • Release: Disabled.

For SDK versions earlier than 8.0.0, and when neither macro is defined, the determination is enabled regardless of the NintendoSDK build type.

Parameters
[in]conditionThe condition to test.
[in]...Format string and arguments for the message to add to the log if the condition has not been satisfied.
