A sample program showing how to use the barrier feature.

Overview

This sample uses the barrier feature.

For more information about using the barrier feature, see the Barrier Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsBarrier/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Three threads are created with nnMain().
  • The three threads are ThreadFunction1(), ThreadFunction2(), and ThreadFunction3().
  • Each ThreadFunctionN() is joined with the barrier feature after waiting an appropriate amount of time.
  • After synchronizing with the barrier, each simultaneously sends a log and ends.
  • The three threads are destroyed by nnMain().

Although it is unknown which of the three threads first arrives at AwaitBarrier(), at the time when all three threads are joined, the three threads restart their operations. For this reason, the final log display is as follows.

Thread2: Start
Thread3: Start
Thread1: Start
Thread2: Wait 140 msec
Thread3: Wait 60 msec
Thread1: Wait 150 msec
Thread3: Wait 150 msec
Thread2: Wait 110 msec
Thread1: Wait 130 msec
Thread3: Wait 30 msec
Thread3: Wait barrier synchronization
Thread2: Wait 70 msec
Thread1: Wait 150 msec
Thread2: Wait barrier synchronization
Thread1: Wait barrier synchronization
Thread1: Resume after barrier wait
Thread2: Resume after barrier wait
Thread3: Resume after barrier wait
Ok.

The display order is random until each thread enters "barrier synchronization wait." The last "restart operations" is displayed in order after all three threads enter "barrier synchronization wait." (However, the display order differs according to the execution environment.)

In this way, multiple threads can be joined and restarting execution can be controlled by using the barrier feature.
