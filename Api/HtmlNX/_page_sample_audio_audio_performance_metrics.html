This sample program shows techniques for gathering performance information about audio renderers.

Overview

The sample gathers performance information about audio rendering processes.

File Structure

This sample program is located in Samples/Sources/Applications/AudioPerformanceMetrics/ .

Required Environment

You must be able to use audio output.

How to Use

Background music and sound effects are played for a fixed time and then automatically stopped. During that time, the processing load of the audio renderer rendering processes is periodically output to a log.

Input Operation
A Button Plays SampleSe0 once.
[B] Plays SampleSe1 once.
[X] Plays SampleSe2 once.
[Y] Plays SampleSe3 once.
[L] Enables or disables a delay.
START or SPACEBAR Exits the program.

Important Information

The processing load information gathered using the feature described in this sample program is only valid for this release. Processing loads may be improved in future releases.

Execution Procedure

Build the sample program and then run it.

Description

Audio renderers apply processes, such as reshaping waveforms, mixing together several waveforms, and adding effects, and then output the results to an audio device. These processes are known as rendering processes. In addition, audio renderers repeat the rendering processes at a time interval for which the number of samples specified for nn::audio::AudioRendererParameter.sampleCount is equal to one period. This time interval is known as an audio frame. To ensure that audio data is played smoothly and without interruptions, the rendering processes must be completed within the audio frame.

This sample program illustrates techniques for determining how much of each audio frame is used by audio renderers; that is, the audio renderer processing load. In this documentation, this processing load is called performance information.

Implementing a special initialization process makes it possible for audio renderers to automatically and continuously record their performance information. The developer can then periodically retrieve that performance data to assess the performance of the audio renderers.

The overall procedure for getting the performance information is as follows.

  • When initializing an audio renderer, specify a time period for which to store performance information.
  • Specify a buffer to which to write the performance information, and update the audio renderer.
  • Get the buffer to which the performance information was written, and parse the contents of that buffer.
