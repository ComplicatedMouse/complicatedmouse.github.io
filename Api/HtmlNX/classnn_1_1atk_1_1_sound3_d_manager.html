Detailed Description

The class for calculating and managing 3D sound parameters.

Details
Sound parameter calculations are performed using the information in the nn::atk::Sound3DListener and nn::atk::Sound3DActor classes.

The SoundArchivePlayer::Update() function might throw an exception if you finalize the Sound3DManager instance passed by the argument of the Sound3DActor::Initialize() function before the sound played by Sound3DActor has finished.

The exception is thrown because the sound played by the Sound3DActor object references the Sound3DManager object as a callback. If you are going to finalize the Sound3DManager instance, make sure that all sounds that are referencing the object have finished playing first.

void nn::atk::Sound3DManager::AddListener ( Sound3DListener *  listener)

Registers a 3D sound listener.

Details
To play sounds, a 3D sound listener must be registered in the 3D Sound Manager.

Multiple 3D sound listeners can be registered. For more information about registering multiple listeners, see Programming Guide > 3D Sound > Multi-Listeners.

However, a single 3D sound listener cannot be registered in multiple 3D sound managers. A single 3D sound listener also cannot be registered multiple times in a single 3D sound manager.

◆ SetSonicVelocity()

void nn::atk::Sound3DManager::SetSonicVelocity ( float  sonicVelocity)

Sets the sonic velocity that is configured for 3D sound.

Details
The velocity is used in pitch change calculations, in accordance with the Doppler effect.

The units of the value are the sonic velocity for a single frame. Because the speed of sound is approximately 340 meters per second, if 1.0f in the unit system for 3D space coordinates is equivalent to one meter and a sound is moved for 60 frames, the sonic velocity value must be set to 340.0f/60.

If the sonic velocity is set to 0.0f, the Doppler effect does not occur. The default value is 0.0f.

Parameters
[in]sonicVelocityThe sonic velocity.

◆ SetBiquadFilterType()

void nn::atk::Sound3DManager::SetBiquadFilterType ( int  type)

Sets the type of biquad filter that is configured for 3D sound.

Details
The biquad filter parameter can be configured in multiple locations, but the settings are not layered. Rather, the biquad filter takes the value that is set in the location with the highest priority, as follows. If the biquad filter parameter is set in a higher-priority location, the settings in locations of lower priority are overwritten. 

   -# The setting in the sound handle
   -# The setting in the sound player
   -# The setting in the ambient parameter structure
   -# The setting in the sequence data

type is the filter type, and it takes an enumerator of the BiquadFilterType enumerated type. In addition to the presets, you can also select a user-registered filter.

For type, specify a value between BiquadFilterType_Min and BiquadFilterType_Max. Stops on an assertion failure if the specified value is outside of this range. (Only for the Debug/Development versions. This setting is ignored for Release versions. Although a value is set, normal operations are not assured.)

Parameters
[in]typeThe biquad filter type.
