Detailed Description

Represents the generic result of an operation.

Details
This class is used as a generic type for the return value of functions that could potentially fail. An instance of this class can store all types of Result values.

The instance generated by the default constructor for this class has not been initialized, so it may not contain a valid value. Valid Result values are generated by converting from an instance of the error Result class defined by each library, or an instance of the ResultSuccess class.

Result contains the following two integers.

  • Module
  • Description

Module is the value that represents the category for the module that Result belongs to. Description is the value that indicates the meaning of the Result within the module.

ResultSuccess, which is the Result value that indicates success, has a Module and Description value of 0.

Each library defines its own error Result for the independent errors of that library. The error Result contains the following values.

  • Module (nonzero)
  • Description range (minimum and maximum)

In addition, it contains the following static member function.

  • bool Includes(Result result)

This function returns true if both of the following conditions are satisfied.

  • The result Module is equal to the error Result Module.
  • The result Description is within the Description range of the error Result.

Use the Includes() function for this handling when returning Result values for various functions. For example, the nn::fs::CreateFile() function returns the following three types of error Result values.

  • nn::fs::ResultPathNotFound
  • nn::fs::ResultPathAlreadyExists

  • nn::fs::ResultUsableSpaceNotEnough

    You must include code handling in a way similar to the following.

nn::Result result = nn::fs::CreateFile(path, size);
if (!result.IsSuccess())
{
// When the process is not successful.
if (nn::fs::ResultPathNotFound::Includes(result))
{
// Handling when the directory in the path does not exist.
}
else if (nn::fs::ResultPathAlreadyExists::Includes(result))
{
// Handling when the target file already exists.
}
else if (nn::fs::ResultUsableSpaceNotEnough::Includes(result))
{
// Handling when there is insufficient free space.
}
else
{
// Cancel the process because of an unexpected error.
NN_ABORT_UNLESS_RESULT_SUCCESS(result)
}
}
// The process when successful.

Note the following when handling Result values.

  • Use the IsSuccess() and Includes() functions to perform handling.
  • Make sure to avoid using the GetModule() or GetDescription() values. These values may only be used for debugging.

The return value of the GetInnerValueForDebug() function can be used to display the Result value for debugging. In addition, this return value can be passed to Result Value Search to view detailed information.
