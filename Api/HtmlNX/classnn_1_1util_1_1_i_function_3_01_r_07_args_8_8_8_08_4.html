Detailed Description

template<typename R, typename... Args>
class nn::util::IFunction< R(Args...)>

Specialized version of IFunction that takes the function type.

Template Parameters
RThe type of the function's return values.
ArgsThe type of the function's arguments.

Details
Class template for the abstraction of functions that take Args-type arguments and return R-type return values. The instance classes of this class template are abstract classes and cannot be instantiated.

Gets an object whose base type is the IFunction<R(Args...)> of the specified callable object.

Template Parameters
FThe callable type for the arguments.
Parameters
fThe callable object.
Returns
An object whose base type is the IFunction<R(Args...)> of the specified callable object.

Details
In most cases, instead of using this function you would use the MakeIFunction() function template, which automatically infers the types specified by R and Args.

F must be move constructible, and it must be callable for the Args argument group and the R return values. The following types that meet these conditions can be specified.

  • Function pointer type.
  • Function object type.
  • Member function pointer type.
  • Data member pointer type.
  • std::reference_wrapper<function object type>

This function returns the following kind of object.

  • An instance of the final class for which IFunction<R(Args...)> is the base class.
  • Has an F-type member field in which f is copied.
  • An implementation of IFunction<R(Args...)>::operator()(Args...) where operator() is called on the copied f.

For this reason, the result passed to this function by a lambda expression can be passed without modification to places where the argument is something like const IFunction<R(Args...)>&.

In the following kinds of situations, use std::ref() as needed to pass the value wrapped in std::reference_wrapper.

  • Situations where you need to consider the cost of copying f.
  • Situations where you want to pass a reference to f.
  • Situations where F::operator()(Args...) is a non-const member function and does not compile.
