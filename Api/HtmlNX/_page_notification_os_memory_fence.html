Memory Fence

Some processors employ out-of-order execution of commands to optimize program performance.

Out-of-order execution also changes the order of memory access. In multiprocessor environments, this can result in incorrect recording of the order in which memory is accessed by each processor core.

The memory fence feature is for enforcing constraints on this kind of reordering of memory access operations. When out-of-order execution is constrained, the order of memory access is recorded in the same way from all processor cores.

The OS library provides the following nine memory fence functions.

  • nn::os::FenceMemoryStoreStore()
  • nn::os::FenceMemoryStoreLoad()
  • nn::os::FenceMemoryStoreAny()
  • nn::os::FenceMemoryLoadStore()
  • nn::os::FenceMemoryLoadLoad()
  • nn::os::FenceMemoryLoadAny()
  • nn::os::FenceMemoryAnyStore()
  • nn::os::FenceMemoryAnyLoad()
  • nn::os::FenceMemoryAnyAny()

The name of each function follows the nn::os::FenceMemoryXxxxYyyy() pattern, where Xxxx and Yyyy indicate the type of memory access: Store, Load, or Any. The function name can be any permutation of these access types.

Xxxx represents the type of memory access placed in front of the memory fence function, and Yyyy represents the type of memory access placed behind the function. The function restricts out-of-order memory access of both types and enforces an ordering constraint.

For instance, the nn::os::FenceMemoryLoadStore() function enforces a contraint that prevents a memory load operation issued before the function is called from being executed after a memory store operation issued after the function is called. In other words, memory load operations issued prior to the fence complete execution before memory store operations issued after the fence.

These nine functions are implemented using optimized machine code for each target processor. If an exact match for the function is not available as a machine instruction, a more restrictive machine instruction is used to ensure the correct order of memory access indicated by the function.
