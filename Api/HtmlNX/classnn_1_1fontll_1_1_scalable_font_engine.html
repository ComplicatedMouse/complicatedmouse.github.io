◆ ScalableFontEngine()

nn::fontll::ScalableFontEngine::ScalableFontEngine ( )

Constructor.

Details
Actual initialization is handled by the Initialize function. This constructor only initializes member variables.

This class does not internally use a locking mechanism to account for use from multiple threads.
If you need to use this class simultaneously inside multiple threads, create an instance for each thread or implement a locking mechanism within the user code.

◆ Initialize()

int nn::fontll::ScalableFontEngine::Initialize ( void *  pHeap,
uint32_t  heapSize 
)

Initializes.


Parameters
[in]pHeapA pointer to the working memory.
[in]heapSizeThe size of the working memory.

Details
Working memory is used for the following purposes.

  • Listing loaded fonts
  • Storing information about loaded fonts
  • Caching scaled font information
  • Caching glyph (character) bitmaps

    If memory runs out, you can automatically release the internal cache and handle memory as appropriate.
    When the memory is totally depleted, the following error code is returned: 201(nn::fontll::ErrorMemoryAllocationFail). The appropriate size to use for working memory depends on how it is being used (the type of font, the number of characters being used simultaneously, and the font size).
    The engine will run using a working buffer of under 100 KB if the font size is roughly 64 pixels.
    We recommend finding the right size through trial and error while tracking performance.
Returns
Returns a result code.
If successful, returns Success.
Examples
FontllSaveToJpeg/FontllSaveToJpeg.cpp.

Load fonts.

Parameters
[in]pNameBufferPointer to the buffer that stores the output null-terminated font name string for memory management purposes in the library.
[in]pMemPointer to the font file data.
[in]indexThe font number. When using a TTC file, specify the index of the font to use. For all other formats, always specify 0.
[in]nameBufferLenLength of the buffer (pNameBuffer) to store the output font name string. Specify a value of at least 2 but no more than FontNameLengthMax.

Details
The following types of scalable fonts are supported.

  • System fonts (BFTTF)
  • TrueType (TTF)
  • TrueType Collection (TTC)
  • OpenType (OTF)


To load a BFTTF file, first decode it using the ScalableFontEngineHelper::Decode function, and then pass it to the font engine.

Returns
Returns a result code.
Returns Success or one of the following error codes.
ErrorFontNotFound
ErrorFontNameInUse

◆ SetAutoHint()

void nn::fontll::ScalableFontEngine::SetAutoHint ( bool  isAutoHint)

Sets whether to force hinting.

Normally set this parameter to false. If drawing breaks down or the program crashes as a result of using particular fonts, setting this parameter to true immediately before the AcquireGlyphmap member function fixes the problem. However, setting it to true slightly changes the drawing result by a few pixels. Initially, calculations for drawing fonts are based on the hint information of the font itself, but there have been confirmed cases of particular fonts causing programs to crash due to rounding errors in those calculations. These crashes can be avoided by passing true to this member function, which causes the hint information calculations to be made at run time.

Parameters
[in]isAutoHintSpecifies whether hinting is forced.
