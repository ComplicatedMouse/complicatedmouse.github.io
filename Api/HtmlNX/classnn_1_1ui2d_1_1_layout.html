nn::ui2d::Layout::Layout ( nn::gfx::Device *  pDevice,
const Layout &  src,

explicit

Copy constructor.

Details
Copies the whole pane tree. If that pane tree has a parts layout, that parts layout is also copied.

When copying the parts layout, the root pane is not suspended from the pane tree hierarchy. For this reason, you can attach AppendChild (for example) from any position.

Note that any animation created in the source layout is not copied. Groups, however, are copied.

Note that if panes are being inherited and expanded independently in the application, you must implement the copy constructor again in the derived class objects of the Layout class, including the ClonePaneTreeWithPartsLayout method used inside this method.

Parameters
[in]pDeviceDevice.
[in]srcThe layout to copy.

◆ CalculateImpl()

virtual void nn::ui2d::Layout::CalculateImpl ( DrawInfo &  drawInfo,
bool  forceGlbMtxDirty 
)
protectedvirtual

Calculates for rendering.

Details
The DrawInfo object passed by the drawInfo parameter is updated inside the function. Note that this cannot be shared by multiple threads. To run the process on multiple threads, create and run a copy of DrawInfo on each thread.

Set the forceGlbMtxDirty parameter to true to force recalculation of the global matrix of the pane. Panes are designed such that the global matrix is only recalculated if it is the first time the matrix is being calculated or if the SRT and size of a pane and its parent pane have been changed. However, you must also recalculate if the view matrix set in DrawInfo has been changed.

Pass true to forceGlbMtxDirty if the view matrix set in DrawInfo has been changed.

There is no need to specify this argument when you render an orthogonal projection. To move the camera during perspective projection, you must specify this argument.
