◆ GetItemFileId()

FileId nn::atk::SoundArchive::GetItemFileId ( ItemId  id) const
inherited

Gets the file ID that corresponds to an item ID.

Details
The types of item IDs that can be converted are sounds (STRM, WSD, and SEQ files), wave sound sets, sequence sound sets, banks, waveform archives, and groups.

The file ID that the function returns when you specify the ID for a wave sound set is the same file ID that it returns when you specify the ID for any wave sound in that set.

If the ID of a sequence sound set is specified, the same file ID returns if the most recent sequence sound ID of the sequence sounds includedin the particular sequence sound set has been specified. Returns the same file ID.

If a player ID is specified, the function returns SoundArchive::InvalidId.

The file ID obtained by this function can be used by the SoundDataManager::SetFileAddress function.

Parameters
[in]idItem ID in the sound archive.
Returns
Returns the file ID that corresponds to the specified item ID.

◆ ReadSoundUserParam()

bool nn::atk::SoundArchive::ReadSoundUserParam ( uint32_t *  pOutValue,
ItemId  soundId,
int  index 
) const
inherited

Gets the user parameter of the sound that has the specified ID.

Details
For index, specify any value between 0 and UserParamIndexMax. If the number passed for index is different from the user parameter numbers enabled in the User Parameters tab of the Project Settings in SoundMaker, the function fails to get the parameter and returns false.

"User parameter (unmarked)" in SoundMaker is equivalent to index = 0. "User parameters 1 to 3" in SoundMaker is equivalent to index = 1 to 3.

The function also fails to get the parameter and returns false if an invalid sound ID is passed for soundId.

When the function succeeds, the obtained parameter is stored in value and true is returned.

The value for the user parameter when 0 is passed in index is the same as the value obtained with GetSoundUserParam. (GetSoundUserParam is retained to maintain compatibility with earlier versions.)

Parameters
[out]pOutValueThe user parameter storage location.
[in]soundIdSound ID in the sound archive.
[in]indexThe user parameter number.
Returns
Returns true if the sound's user parameter is obtained successfully, or false otherwise.

bool nn::atk::SoundArchive::ReadStreamSoundFilePath ( char *  outFilePathBuffer,
size_t  filePathBufferSize,
ItemId  soundId 
) const
inherited

Gets the file path of the stream sound that the specified ID references.

Details
When using nn::atk::FsSoundArchive, it can be used after calling nn::atk::FsSoundArchive::Open.

When using nn::atk::MemorySoundArchive, or nn::atk::AddonSoundArchive, it can be used after setup by nn::atk::SoundArchive::SetExternalFileRoot has been completed.

Parameters
[out]outFilePathBufferThe file path of the stream sound referenced by the specified ID, including any null characters, if the operation completed successfully.
[in]filePathBufferSizeThe size of outFilePathBuffer. Specify a size that can store the file path, including the null characters.
[in]soundIdThe ID of the stream sound for which to get the path.
Returns
Returns the value that indicates whether the file path was successfully loaded. Returns true if successful. Returns false when processing cannot continue correctly, such as when an ID other than stream sound, or an ID that does not exist is specified for soundId.
See also
nn::atk::FsSoundArchive::Open
nn::atk::SoundArchive::SetExternalFileRoot
