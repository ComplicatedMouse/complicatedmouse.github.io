Detailed Description

template<typename BlockCipher>
class nn::crypto::CtrDecryptor< BlockCipher >

Class for decrypting in CTR mode.

Template Parameters
BlockCipherThe implementation class for the block cipher to use.

Details
The cipher mode for CTR is as stipulated in NIST SP 800-38A. Implementations of this class use a simple incrementer as the function to increment the counter value.

This class is a template class that takes the block cipher implementation class as a template parameter. Use Aes128CtrDecryptor or a similar class in your actual code.

The implementation of the block cipher class specified in the template must satisfy the following conditions. For more information including about the parameters of each function, see AesEncryptor.

  1. Has a BlockSize constant that defines the size of the block to process.
  2. Has a EncryptBlock function that encrypts a single block.

Note that CTR mode decryption does not use decryption of the block cipher, but encryption.

This class has the following states. The state immediately after instantiation is uninitialized.

  • Uninitialized state
  • Ready.

Parameters
[in]pBlockCipherInstance of the block cipher implementation class.
[in]pIvPointer to the initial data for the counter.
[in]ivSizeSize of the initial counter data in bytes.
Precondition
  • The EncryptBlock member function of the instance specified in pBlockCipher can be called.
  • ivSize == IvSize
Postcondition
  • The instance is in a ready state.

Details
Specify an instance of the class that implements the block cipher to use for encryption in pBlockCipher. The instance must be initialized with the required key, and other settings, so that EncryptBlock is ready to be called.

The instance specified in pBlockCipher is not copied. Do not perform any operations on pBlockCipher until the processing sequence is complete.

In CTR mode, the IV is used as the initial counter value. An internal copy of the initial counter value is created.

Initializes the process for partial decryption in CTR mode.

Parameters

Precondition

  • ivSize == IvSize
  • srcOffset >= 0.
Postcondition

Details
Initializes CTR mode processing so that decryption processing can start from the srcOffset position specified in the data array. Next, call Update() to perform the processing from the data at that position and partially decrypt the data array.

Specify an instance of the class that implements the block cipher to use for encryption in pBlockCipher. The instance must be initialized with the required key, and other settings, so that EncryptBlock is ready to be called.

The instance specified in pBlockCipher is not copied. Do not perform any operations on pBlockCipher until the processing sequence is complete.

In CTR mode, the IV is used as the initial counter value. An internal copy of the initial counter value is created.

Runs decryption in CTR mode.

Parameters
[out]pDstPointer to the buffer that receives decrypted data.
[in]dstSizeSize of the buffer pointed to by pDst.
[in]pSrcPointer to the buffer storing the data to decrypt.
[in]srcSizeSize in bytes of the data to decrypt.
Returns
Returns the actual number of bytes that were decrypted.
Precondition
  • The instance is in a ready state.
  • dstSize >= srcSize
  • The region specified by pDst,dstSize does not overlap with the region specified by pSrc,srcSize.
Postcondition
  • The decrypted result is written to pDst.

Details
Because decryption takes place byte-by-byte in CTR mode, the dstSize required for each call must be the same as srcSize.

The buffer required for the total dstSize for all calls is also the same as the total srcSize.
