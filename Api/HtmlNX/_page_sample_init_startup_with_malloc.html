Describes a second sample program that uses the nninitStartup() function.

Overview

This sample program shows how to define an application-specific nninitStartup() function, and how to implement custom versions of malloc(), free(), calloc(), realloc(), aligned_alloc(), and malloc_usable_size().

For more information, see nn::init::Startup().

File Structure

This sample program is found in Samples/Sources/Applications/InitStartupWithMalloc/ . This sample cannot be used in a Windows environment. Visual Studio files are only provided for development device environments.

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Initialize memory allocator for malloc() using nninitStartup().
  • Define proprietary malloc(), free(), calloc(), realloc(), aligned_alloc(), and malloc_usable_size() functions.
  • Output the memory address obtained by this malloc() in nnMain() to the log, and then end.

Unlike nninitStartup Sample, this sample does not use the default SDK malloc(). Instead, it defines and uses a proprietary malloc().

First, nninitStartup() is called, initializing the statically allocated g_MallocBuffer[] array as memory used by the memory allocator. nn::mem::StandardAllocator() is used for the allocator.

The static object constructor is called after nninitStartup() in most cases. The constructor is run inside nninitStartup and should never run after that, so the implementation is as follows.

  • Use nn::util::TypedStorage to only allocate memory space for the static object g_SampleAllocator, and suppress the constructor from automatically starting.
  • Use g_SampleAllocator, using placement new to explicitly start the constructor in nninitStartup().

The allocator is usable after this. You will use this allocator in the implementation of malloc() and free().

nnMain() starts last. Call malloc() in nnMain(), output the obtained address to the log, and then end.

The following log is output.

step1: A nninitStartup() function is invoked.
step1: MallocBuffer is 0x0020d000 - 0x0060d000

step2: nnMain() function is invoked.
step2: malloc() address = 0x00210000
step2: nnMain() function is finished.

If you intend to use the default malloc() provided by the SDK, see nninitStartup Sample.

This sample cannot be used in a Windows environment. Defining a proprietary malloc() or free() in a Windows environment causes a symbol collision and a link error.
