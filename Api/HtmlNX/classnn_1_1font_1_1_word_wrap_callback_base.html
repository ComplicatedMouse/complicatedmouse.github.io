Gets the amount of input text that fits on a single line.

Details
Returns the end of the range of the string defined by the two parameters (pBegin and pEnd) that can display on a single line. The character referred to by this return value is not included in the string that displays on the single line.

This function is called by the word-wrapping functions (WordWrapping::CalcWordWrapping() and WordWrapping::FindLineBreak()). It determines the maximum amount of text that can fit on a single line, so that the word-wrapping functions can wrap the text at an appropriate position within that range.

The pBegin and pEnd parameters indicate positions within the text passed to the word-wrapping function. The text is not necessarily null-terminated.

Use the WideTextWriter::FindPosOfWidthLimit() function to get how much text can be displayed on a single line.

Parameters
[in]pBeginThe first character in the current line.
[in]pEndThe end of the text that the user passed into the word-wrapping function.
Returns
Returns the position of the first character that does not fit on the first line.

Gets the byte size of the tag.

Details
Tags are normally exempt from word wrapping, so using this function to return the number of bytes comprising the tag skips the word wrap processing for the tag for a specified size.

This function is called each time when character codes shorter than 0x20 appear.

Parameters
[out]pNonBreakingLengthProhibits line feeds, for the specified number of characters, for the string following the tag. The units are in characters rather than bytes.
[in]pStrPointer to the start of the tag.
[in]pEndThe end of the text that the user passed into the word-wrapping function.
Returns
The number of bytes to skip. Returns 0 if not a tag.
