◆ AcceptPolicy

enum nn::ldn::AcceptPolicy

Policy for access point connection acceptance.

Details
Policy for access point connection acceptance.

AcceptPolicy_AlwaysAccept is used by default. To restrict station connection at the start of the access point, first start access point mode using the nn::ldn::OpenAccessPoint() function, and then use the nn::ldn::SetStationAcceptPolicy() function to restrict access before using the nn::ldn::CreateNetwork() function to create the network.

You can also use the nn::ldn::SetStationAcceptPolicy() function to change the station accept policy after using the nn::ldn::CreateNetwork() function to create the network.

◆ DisconnectReason

enum nn::ldn::DisconnectReason

The reason for station disconnection.

Details
The reason for station disconnection. Use the GetDisconnectReason() function to get the reason.

DisconnectReason_None indicates that the station was never disconnected since initializing the ldn library, or is connected to the network.

DisconnectReason_DisconnectedByUser is set when the station side application calls the Disconnect() function and disconnects, and DisconnectReason_DisconnectedBySystem is set when the connection is disconnected by a system request such as starting system communication or entering sleep mode.

DisconnectReason_DestroyedByUser is set when the access point application calls the DestroyNetwork() function and destroys the network, and DisconnectReason_DestroyedBySystem is set when the network is destroyed by a system request such as starting system communication or entering sleep mode.

DisconnectReason_Rejected is set when Reject() is used to disconnect the station, and DisconnectReason_SignalLost is set when the station leaves the network due to network instability.

However, there is no guarantee of being able to get the correct reason for disconnects for the following three factors when disconnected from the access point.

  • DisconnectReason_DestroyedByUser
  • DisconnectReason_DestroyedBySystem
  • DisconnectReason_Rejected

DisconnectReason_Unknown is returned instead if the function fails to get the reason for disconnection. Failure to get the disconnect reason occurs particularly often when the communication environment is unstable. The results will not change even if GetDisconnectReason() is executed repeatedly.

Enumerator
DisconnectReason_Unknown 

Unknown disconnection reason.

DisconnectReason_None 

Not disconnected.

DisconnectReason_DisconnectedByUser 

Disconnected from the network after a request from the application.

DisconnectReason_DisconnectedBySystem 

Disconnected from the network after a request from the system.

DisconnectReason_DestroyedByUser 

The access point destroyed the network after a request from the application.

DisconnectReason_DestroyedBySystem 

The access point destroyed the network after a request from the system.

DisconnectReason_Rejected

DisconnectReason_SignalLost 

Disconnected because of an unstable communication environment.

◆ SecurityMode

enum nn::ldn::SecurityMode

The security mode for local communication.

Details
Production devices ignore the SecurityMode settings in local communication and work under the assumption that the SecurityMode_Product is always specified. The system encrypts all data for sending and receiving.

On the other hand, it is possible for the application to specify a policy for protecting communications only when running the local communication library on development hardware. This feature is intended for debugging the application communication features, and you can disable encryption of data communications by specifying SecurityMode_Debug in the securityMode of SecurityConfig that is passed to the CreateNetwork function. However, encryption will only be disabled for the data sent and received by the application, and will not be disabled for the information exchanged for system management.

Normally, the station only connects to access points that match the SecurityMode, but if SecurityMode_Any is specified, the station can connect to either SecurityMode_Debug, or SecurityMode_Product access points. This setting only works for development hardware, and SecurityMode_Any and SecurityMode_Debug settings are ignored for production devices.

Enumerator
SecurityMode_Any 

No security mode is specified.

SecurityMode_Product 

The product-based security mode.

SecurityMode_Debug 

The debugging security mode.

◆ Initialize()

Result nn::ldn::Initialize ( )

Initializes the local communication library.

Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess must be handled.
  • ResultDeviceNotAvailable must be handled.
Precondition
  • GetState() == State_None.
Postcondition
  • GetState() == State_Initialized.

Details
Initializes the local communication library.

If the local communication library is successfully initialized, the communication device will be appropriated for local communication, Finalize and cannot be used for other communication features until the function is called. Always call Finalize to release the communication device after using the local communication library. If a nifm library network use request has been received when this function is called, that network use request is denied and the Internet connection is disconnected as soon as the function succeeds.

If local communication cannot start, for example because the system is using the communication features or the system is in airplane mode, the function returns ResultDeviceNotAvailable. When that error occurs, you can pass a Result instance to the nn::err::ShowError() function to tell the user what measures must be taken to begin local communication.

Because the initialization of the local communication library waits for other communication features to end, this function might not return processing for as long as several seconds.

Examples
LdnBasic/LdnAccessPoint/LdnAccessPoint.cpp, and LdnBasic/LdnStation/LdnStation.cpp.

◆ Finalize()

void nn::ldn::Finalize ( )

Releases the local communication library.

Precondition
  • GetState() != State_None.
Postcondition
  • GetState() == State_None.

Details
Releases the local communication library. The occupied communication device is released by executing this function, and other communication features become usable.

If there is a network created by the CreateNetwork function when this function is called, that network is destroyed as though DestroyNetwork were executed, and then the local communication library is released. Similarly, if a network connection established by the Connect function is active when this function is called, that network connection is disconnected as though the Disconnect function was called, and then the local communication library is released. Note that in this kind of situation, it takes time to destroy or disconnect from the network, so this function may not return processing for a maximum of 500 milliseconds.

Examples
LdnBasic/LdnAccessPoint/LdnAccessPoint.cpp, and LdnBasic/LdnStation/LdnStation.cpp.

◆ AttachStateChangeEvent()

void nn::ldn::AttachStateChangeEvent ( nn::os::SystemEventType *  pOutSystemEvent)

Gets the event indicating a change in connection status.

Parameters
[out]pOutSystemEventThe event's output destination.
Precondition
  • GetState() != State_None.
  • pOutSystemEvent != nullptr.

Details
Gets the event indicating a change in connection status.

The event will be in a signaled state if the number of nodes belonging to the same network changes. Specifically, the following four conditions indicate that the event is in the signaled state.

  • An access point builds a network.
  • An access point destroys a network.
  • Any station, including the local station, connects to the network.
  • Any station, including the local station, leaves the network.

This event is set as an auto-reset event, so it will reset automatically after the application receives the signal.

Examples
LdnBasic/LdnAccessPoint/LdnAccessPoint.cpp, and LdnBasic/LdnStation/LdnStation.cpp.

◆ Scan()

Result nn::ldn::Scan ( NetworkInfo *  outBuffer,
int *  pOutCount,
int  bufferCount,
const ScanFilter &  filter,
int  channel 
)

Scans for nearby networks.

Parameters
[out]outBufferThe scan results' output destination.
[out]pOutCountThe number of scan results sent to outBuffer.
[in]bufferCountThe number of scan results that can be stored in outBuffer.
[in]filterSpecifies the scan condition.
[in]channelSpecifies the channel to scan.
Returns
Returns a nn::Result that represents the process result.
  • ResultSuccess must be handled.
  • ResultInvalidState must be handled.
  • ResultDeviceNotAvailable must be handled.
Precondition
  • GetState() != State_None.
  • outBuffer != nullptr.
  • pOutCount != nullptr.
  • 0 < bufferCount.
  • channel == 1 || channel == 6 || channel == 11 || channel == 36 || channel == 40 || channel == 44 || channel == 48 || channel == AutoChannel.

Details
Searches for the networks surrounding the wireless channel specified by channel and stores the networks that satisfy the conditions specified in filter to outBuffer and pOutCount. If multiple networks satisfy the conditions, networks will be stored in the outBuffer in the order of connection quality, but only a maximum of ScanResultCountMax networks can be discovered by this function at the same time.

For more information about specifying the network to scan, see ScanFilter.

The wireless channel can only be specified in the development environment, so the behavior in the production environment is always the same as if AutoChannel was specified for channel. For AutoChannel, it searches all of the 1, 6, and 11 channels in the 2.4-GHz band. Note that the 36, 40, 44, and 48 channels in the 5-GHz band are not included. Approximately 110 milliseconds are required when searching for each wireless channel, and this function will not return processing until searching for all wireless channels has completed.

Execute the Scan function in one of the following states: State_Station, State_StationConnected, or State_AccessPointCreated. ResultInvalidState is returned if you run this function in any other state. Because there is a lot of packet loss and delay while searching the surrounding networks, be careful when executing the Scan function in the State_StationConnected state or State_AccessPointCreated state.

Examples
LdnBasic/LdnStation/LdnStation.cpp.

Precondition
  • GetState() != State_None.
  • pOutNetwork != nullptr.

Details
Gets information about the currently joined network.

Access points can run this function after you create a network using the CreateNetwork function and connect stations to the network using the Connect function. ResultInvalidState is returned if you run this function in a state other than State_StationConnected or State_AccessPointCreated.

For more information about the information stored in pOutNetworkInfo, see NetworkInfo.

In pOutUpdates, the point at which this function previously obtained network information is treated as the starting point, and the changes in the node connection state until the current time are output as a NodeLatestUpdate structure. This array index corresponds to the node numbers (nodeId) for NodeInfo. However, immediately after the network was built or immediately after connecting to the network, the GetNetworkInfo function will calculate the difference using the state in which there are no nodes, including the access point, as the starting point. ◆ NodeCountMax

nn::ldn::NodeCountMax = 8

Maximum number of nodes that can connect to the local network.

Details
Maximum number of nodes that can connect to the local network. This number includes the access point, so the maximum number of stations in the local network is NodeCountMax - 1. The maximum number of stations is defined as StationCountMax.

When required, the access point can further restrict the maximum number of stations that can connect via the local network. To do this, specify a value smaller than NodeCountMax for the nodeCountMax member of the NetworkConfig structure that is passed to the CreateNetwork function.

Examples
LdnBasic/LdnAccessPoint/LdnAccessPoint.cpp, and LdnBasic/LdnStation/LdnStation.cpp.

◆ StationCountMax

nn::ldn::StationCountMax = NodeCountMax - 1

Maximum number of stations that can connect to the local network.

Details
Maximum number of stations that can connect to the local network. This number does not include the access point, so the maximum number of nodes in the local network is StationCountMax + 1. The maximum number of nodes, including the access point, is defined as NodeCountMax.

When required, the access point can further restrict the maximum number of stations that can connect via the local network. To do this, specify a value smaller than NodeCountMax for the nodeCountMax member of the NetworkConfig structure that is passed to the CreateNetwork function.
