This sample program shows how to use the Socket API for socket communication.

Overview

Uses a TCP socket to transmit messages between server and client.

File Structure

This sample program is located in Samples/Sources/Applications/SocketBasic/ .

Required Environment

Network connection settings must be imported in advance using SettingsManager. For more information about importing, see the nifm Library documentation.

How to Use

Run the server first, and then run the client. Specify the following options to run the server or the client.

  • -server
    • Starts the program as the server.
  • -client server=<ip_addr>:<port> [message]
    • Starts the program as the client.
    • For <ip_addr>:<port>, specify the server IP address and port number. You can determine the server's IP address and port number by checking the server's execution log.
    • For [message], specify the message to send to the server. If it is omitted, "Test message" is sent as the message.

Important Information

Handling for network connection failures is still TBD.

Execution Procedure

Build the sample program and then run it.

Description

This sample program has the following flow.

  • Using the nifm library, request a network connection from the system.
  • Initialize the socket library.
  • Based on the option that has specified, start the server or the client.
  • Connect the client to the server, and send a message to the server.
  • The server returns the message it received from the client back to the client.
  • The client disconnects from the server and ends processing after receiving the message from the server.
  • The server also ends processing.
