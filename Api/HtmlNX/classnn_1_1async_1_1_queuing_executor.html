Detailed Description

Implementation of the Executor class that uses task queues.

Details
The QueuingExecutor class uses an internal task queue to manage the objects (tasks) representing asynchronous processes and it can be executed from any thread. The tasks added to QueuingExecutor are scheduled in FIFO order.

QueuingExecutor does not have any thread resources for executing asynchronous processes. The tasks in the task queue are executed on the thread that called the following member functions.

  • ExecuteAll()
  • LoopExecution()
  • TryExecuteOne()

We recommend having your application create a thread to execute the asynchronous processes and then call these functions from that thread.

◆ ~QueuingExecutor()

nn::async::QueuingExecutor::~QueuingExecutor ( )

Destroys a QueuingExecutor object.

Precondition
  • The task queue is empty.

Details
To empty the task queue, call the ExecuteAll() function.

Be careful not to call the destructor while tasks are executing. If any threads are still making calls to LoopExecution(), first use RequestLoopExecutionExit() to confirm that all threads have returned from the function call and then call the destructor.

Member Function Documentation

◆ TryExecuteOne()

bool nn::async::QueuingExecutor::TryExecuteOne ( )

If there are any executable tasks in the task queue, this function executes one of them.

Returns
Returns true if a task was executed.

Details
If there are no executable tasks in the task queue, the function immediately returns false. Note, however, that this does not always mean that the task queue is empty. To empty the task queue, call the ExecuteAll() function.

This function can be called simultaneously from multiple threads.

◆ LoopExecution()

void nn::async::QueuingExecutor::LoopExecution ( )

Uses the calling thread as a dedicated thread for the execution of tasks.

Details
Continuously executes tasks added to the task queue.

If the task queue becomes empty, this function waits for new tasks to be added without returning control to the caller. To end the execution of this function, use the RequestLoopExecutionExit() function.

When RequestLoopExecutionExit() has already been called and when IsRequestedLoopExecutionExit() == true is already the case, this function immediately returns to where it was called from.

This function can be called simultaneously from multiple threads.

◆ RequestLoopExecutionExit()

void nn::async::QueuingExecutor::RequestLoopExecutionExit ( )

Requests the end of execution of the LoopExecution() function.

Postcondition
  • this->IsRequestedLoopExecutionExit() == true.

Details
Use this function to return control of the thread in which LoopExecution() is being called back to the caller. If the thread is executing a task inside LoopExecution(), it returns to the caller after the task is completed.

After this function has been called you cannot make a new call to LoopExecution() to execute tasks.

◆ IsRequestedLoopExecutionExit()

bool nn::async::QueuingExecutor::IsRequestedLoopExecutionExit ( )

Checks whether the run completion of LoopExecution() has already been requested.

Returns
true if RequestLoopExecutionExit() has been called at least once.

◆ ExecuteAll()

void nn::async::QueuingExecutor::ExecuteAll ( )

Executes all tasks in the task queue.

Details
If you halt the addition of new tasks to the QueuingExecutor object before you call the function, the task queue is ensured to be empty after the call to this function.

This function can be called simultaneously from multiple threads.
