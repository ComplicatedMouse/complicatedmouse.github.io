◆ SetBuffer()

void nn::prepo::PlayReport::SetBuffer ( void *  buffer,
size_t  size 
)

Sets a buffer.

Parameters
[in]bufferThe buffer.
[in]sizeThe buffer size.
Precondition
  • buffer != nullptr.
  • size >= ReportBufferSizeMin.

Details
Report data is saved to the buffer set using this function. This buffer must not be freed until this object is destroyed.

Use the following formula to calculate the optimal size of the buffer to set with this function.

  • ReportBufferSizeMin + (the sum of the sizes of all key values)

Calculate the size of each key value using the following methods.

  • String
    • KeySizeMax + StringBufferSizeMin + (the size of the string)
  • Binary
    • KeySizeMax + BinaryBufferSizeMin + (the size of the binary)
  • Array
    • KeySizeMax + ArrayBufferSizeMin + (the size of the array)
  • Structure
    • KeySizeMax + StructBufferSizeMin + (the size of the structure)
  • Other
    • PrimitiveKeyValueSizeMax

For more information about calculating the sizes of arrays and structures, see Array::SetBuffer() and Struct::SetBuffer().

◆ Save() [1/2]

nn::Result nn::prepo::PlayReport::Save ( )

Saves report data.

(The user account is not specified.)

Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess Succeeded.
  • nn::prepo::ResultInvalidEventId Invalid event ID.
  • nn::prepo::ResultInvalidReportData Invalid report data.
Precondition
  • A valid event ID has been set.

Details
If you use this function, user accounts are not associated with the report data, meaning you cannot aggregate data for distinct users.

This function saves report data that is unrelated to the status of the user accounts.

Report data will be sent if any one of the Nintendo Accounts linked to user accounts on that console has opted in to receiving that data.

The report data recorded in the buffer is cleared after this function has been called.

Attention
To avoid excess server load, this function is limited to 900 calls in 180 minutes. This function returns nn:ResultSuccess when the limit is exceeded. However, report data is not saved and a warning log is generated.

◆ Save() [2/2]

nn::Result nn::prepo::PlayReport::Save ( const nn::account::Uid &  uid)

Saves report data.

Parameters
[in]uidThe user account.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess Succeeded.
  • nn::prepo::ResultInvalidArgument Invalid argument.
  • nn::prepo::ResultInvalidEventId Invalid event ID.
  • nn::prepo::ResultInvalidReportData Invalid report data.
Precondition
  • A valid event ID has been set.

Details
Report data will be saved if a Nintendo Account is linked to the specified user account (even without opting in to send information). If no account is linked to a Nintendo Account, the report data will not be saved.

The report data will be sent only if the Nintendo Account linked to the user account has opted in to receive information.

The report data recorded in the buffer is cleared after this function has been called.

Attention
To avoid excess server load, this function is limited to 900 calls in 180 minutes. This function returns nn:ResultSuccess when the limit is exceeded. However, report data is not saved and a warning log is generated.
