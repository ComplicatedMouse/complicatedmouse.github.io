This sample program demonstrates how to use the cdr library.

Overview

This section describes how to create host tools using the cdr library.

For more information about creating projects, see the Cdr Library API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/CdrSimple/ .

Required Environment

An NXDMP file must be created ahead of time.

Using the Demo

  1. The program waits for the input of an NXDMP file for analysis, so specify the path to the NXDMP file.
  2. The program waits for the input of the directory where the NSS file corresponding to the NXDMP file is located, so specify the path to that directory.

Execution Procedure

Build the sample program and then run it.

99 int ShowThreadInformation(const nn::cdr::Thread &thread)
100 {
101  printf("ThreadId: %lld, ThreadName: \"%s\", Core: %hd, Priority: %d, Affinity Mask: 0x%08X, ", thread.id, thread.name, thread.core, thread.priority, thread.affinityMask);
102  switch (thread.state)
103  {
104  case nn::cdr::ThreadState_NotStarted:
105  printf("ThreadState: NotStarted\n");
106  break;
107  case nn::cdr::ThreadState_Running:
108  printf("ThreadState: Running\n");
109  break;
110  case nn::cdr::ThreadState_Waiting:
111  printf("ThreadState: Waiting\n");
112  break;
113  case nn::cdr::ThreadState_Suspended:
114  printf("ThreadState: Suspended\n");
115  break;
116  case nn::cdr::ThreadState_Terminated:
117  printf("ThreadState: Terminated\n");
118  break;
119  default:
120  printf("ThreadState: Unknown\n");
121  break;
122  }
123 
124  int nCallStackEntries = 0;
125  RETURN_IF_FAILURE(nn::cdr::GetCallStackEntryCount(&nCallStackEntries, thread.id));
126 
127  std::unique_ptr<nn::cdr::CallStackEntry[]> pEntries(new nn::cdr::CallStackEntry[nCallStackEntries]);
128  RETURN_IF_FAILURE(nn::cdr::GetCallStackEntries(pEntries.get(), &nCallStackEntries, nCallStackEntries, thread.id));
129 
130  for (int iEntry = 0; iEntry < nCallStackEntries; ++iEntry)
131  {
132  if (!pEntries[iEntry].isSymbolsLoaded)
133  {
134  printf("# Warning: nss file for Module %d is not loaded. Some information would be missing or incorrect.\n", pEntries[iEntry].moduleIndex);

136  printf(" Call Stack[%d]:\n", iEntry);

139  printf(" Line Number: %d\n", pEntries[iEntry].lineNumber);

142 
143  nn::cdr::FilePath filePath;
144  int lineNumber = 0;
145  RETURN_IF_FAILURE(nn::cdr::GetSource(&filePath, &lineNumber, pEntries[iEntry].instructionAddress));
146  printf(" GetSource() for the address 0x%llx: \"%s\" Line %d\n\n", pEntries[iEntry].instructionAddress, filePath.filePath, lineNumber);
147  }
148  return 0;
149 }

Sample Program Description

This sample program has the following flow.

  • Wait for input of the NXPDM file and then load that file.
  • Wait for input of the directory where the NSS file is located and then specify it for the NSS search target directory.
  • Get and display the thread information.
  • Get and display the module information.
  • Close the NXPDM file.
