Detailed Description

WebSocket object.

Details
Performs a WebSocket communication process.

This class supports the permessage-deflate extension defined in RFC 7692.

Examples
WebSocketChat/WebSocketChat.cpp, and WebSocketEcho/WebSocketEcho.cpp.

◆ HandshakeCompletionEventHandler

using nn::websocket::WebSocket::HandshakeCompletionEventHandler = bool (*)(const HeaderList& headers, const char* pSubProtocol, void* pParam)

Handshake completion event handler.

Parameters
[in]headersList of response headers.
[in]pSubProtocolThe selected subprotocol.
[in]pParamUser parameter.
Returns
Whether the connection was accepted.

Details
You can use this to check the server response to the handshake that is performed when connecting to the server.

If the server has selected one of the subprotocols in the list presented by the application, a valid value is set in pSubProtocol.
If the server has not selected any of them, nullptr is set in pSubProtocol.

The application can set the event handler that corresponds to the selected subprotocol.
The following functions can be safely called inside this event handler.

  • SetOpenEventHandler()
  • SetMessageEventHandler()
  • SetCloseEventHandler()
  • SetErrorEventHandler()

If there is a problem with the server response, you can make the Open() function fail by returning false to deny the connection.
When that happens, the Open() function returns nn::websocket::ResultHandshakeCouldntAccept().

This event handler is called one time during the processing of Open().

Attention
The string in headers is only valid while the event handler is being called.
If you want to reference this string after exiting the event handler, you must copy it to another buffer.

◆ SetReceiveBufferSize()

void nn::websocket::WebSocket::SetReceiveBufferSize ( size_t  size)

Sets the receive buffer size.

Parameters
[in]sizeThe size of the receive buffer.
Precondition
  • size >= ReceiveBufferSizeMin.
  • Already initialized.
  • Not connected.

Details
The receive buffer indicates the buffer specified for ConnectionBase::Receive(), which is called within Perform().

Increasing the size of the receive buffer increases the amount of data loaded each time, so it may increase performance for applications that receive a large amount of data.

Note
The default value for the receive buffer size is DefaultReceiveBufferSize.

◆ Open() [1/3]

nn::Result nn::websocket::WebSocket::Open ( const char *  pUri)

Connects to the server.

Parameters
[in]pUriThe URI.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The process succeeded.
  • nn::websocket::ResultOutOfMemory Memory allocation failed.
  • nn::websocket::ResultNotSupported This feature is not supported.
  • nn::websocket::ResultOperationCanceled The connection has been canceled. (The cancel function returned true.)
  • nn::websocket::ResultHandshakeCouldntAccept Could not get handshake accepted. (The handshake completion event handler returned false.)
  • nn::websocket::ResultCommunicationError, Communication error.
    This result also captures the nn::websocket::ResultOperationCanceled and nn::websocket::ResultHandshakeCouldntAccept results, so be careful about the order of error handling. must be handled.
Precondition
  • pUri != nullptr.
  • Already initialized.
  • The connection has already been set.

Details
Uses the connection set using SetConnection() to connect to the server, and initiates a WebSocket handshake process.

This function can be called multiple times.
When this function is called while a connection is already active, the current connection is forcibly terminated, and the connection process is restarted.

During the reconnection process, the following behaviors can occur.

  • When forcibly terminated, the close event handler for StatusCode::AbnormalClosure is called.
  • When forcibly terminated, all unprocessed send requests are deleted.
  • During the reconnection process, IsAvailable() returns false.

If the connection succeeds, Perform() is called periodically to perform the post-connection communication processes.

Examples
WebSocketEcho/WebSocketEcho.cpp.

  • nn::websocket::ResultOutOfMemory Memory allocation failed.
  • nn::websocket::ResultNotSupported This feature is not supported.
  • nn::websocket::ResultOperationCanceled The connection has been canceled. (The cancel function returned true.)
  • nn::websocket::ResultHandshakeCouldntAccept Could not get the handshake accepted. (The handshake completion event handler returned false.)
  • nn::websocket::ResultCommunicationError, Communication error.
    This result also captures the nn::websocket::ResultOperationCanceled and nn::websocket::ResultHandshakeCouldntAccept results, so be careful about the order of error handling. must be handled.
Precondition
  • pUri != nullptr.
  • count >= 0.
  • When count > 0, ppSubProtocols != nullptr.
  • Already initialized.

Details
Uses the connection set using SetConnection() to connect to the server, and initiates a WebSocket handshake process.

The server selects one of the subprotocols from the list that is presented, but none is selected if there is no agreement with any of them.
You can use the handshake completion event handler to check the server's selection result.

This function can be called multiple times.
When this function is called while a connection is already active, the current connection is forcibly terminated, and the connection process is restarted.

During the reconnection process, the following behaviors can occur.

  • When forcibly terminated, the close event handler for StatusCode::AbnormalClosure is called.
  • When forcibly terminated, all unprocessed send requests are deleted.
  • During the reconnection process, IsAvailable() returns false.

If the connection succeeds, Perform() is called periodically to perform the post-connection communication processes.

◆ Perform()

void nn::websocket::WebSocket::Perform ( )

Starts the communication process.

Precondition
  • Already initialized.

Details
This function processes reading of receive data and sends the data that has accumulated in the send queue.

A single call to this function spawns a process that continues until the following conditions are satisfied.

  • Reads all of the receive data. (nn::socket::Errno::EAgain occurs.)
  • The send buffer is empty, or the send buffer overflows during the send process.

This function must be called periodically during the period of time when IsAvailable() returns true.

Examples
WebSocketEcho/WebSocketEcho.cpp.
