Detailed Description

Declarations of API resources for busy mutex objects.

Details
The busy mutex is one kind of synchronization object for implementing mutually exclusive control among threads. Compared to a regular mutex, the operations of other threads:

  • While the lock is set, the operations of other threads on the same core are blocked.
  • While the lock is set, the use of the SDK API is prohibited, with the exception of some operating system functions.
  • There is smaller overhead compared to a regular mutex when contention occurs between CPU cores.

A busy mutex has no advantage over a regular mutex if contention only occurs within the same CPU core. It offers a performance advantage when there is a high proportion of contention between CPU cores.

On the other hand, the operations of other threads are blocked while the lock is set, so if the locked state is prolonged, processes that are intended to have high priority are delayed, which can cause an overall decline in performance.

Consider this possibility and be very careful when using the busy mutex functionality.

Macro Definition Documentation

◆ NN_OS_BUSY_MUTEX_INITIALIZER

#define NN_OS_BUSY_MUTEX_INITIALIZER ( )

Statically initializes nn::os::BusyMutexType objects.

Details
The macro in which to expand the initialization list for the static initialization of nn::os::BusyMutexType objects.

For more information about busy mutex functionality, see nn::os::InitializeBusyMutex().

This macro can be used only to perform static initialization of nn::os::BusyMutexType objects as shown in the following example. Objects initialized this way do not need to be explicitly destroyed by nn::os::FinalizeBusyMutex().

Example:

nn::os::BusyMutexType g_StaticBusyMutex = NN_OS_BUSY_MUTEX_INITIALIZER();
