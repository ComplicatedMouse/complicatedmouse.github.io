Overview

This sample implements the drawing of some triangles in screen space coordinates to introduces the basics of shader and vertex data setup. Two triangles are rendered in screen space coordinates in a very simple fashion where no shader uniforms are used.

Description

This application simply draws two triangles in screen space coordinates with no shader uniforms used. In this application we create the following objects:

  • A GraphicsShader object to load shader binary code and bind vertex buffer data to attributes.
  • A GraphicsMemoryPool object into which we load vertex data for the triangles via nvn::Buffers.
  • nvn::VertexAttribState and nvn::VertexStreamState objects to describe the format of our vertex data.
  • A GraphicsRenderState object to generate commands for setting GPU render state.
  • A GraphicsCommandBuffer object to gather all of our GPU commands for rendering.

The NvnSimpleShaderCompiler tool, whose source code is included in this solution, is used to convert the glsl shader code into a binary format that is then loaded and used to generate a GraphicsShader object. The conversion command is integrated into the build through a Visual Studio Pre-Build Event.
