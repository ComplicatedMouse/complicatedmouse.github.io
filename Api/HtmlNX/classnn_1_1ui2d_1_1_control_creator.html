Detailed Description

The base class for creating controls.

Details
This class is passed as an augment of the Layout::Build() member function to build controls when that layout and the parts layouts have controls.

Because this class is a purely virtual class, you must inherit it and implement the CreateControl member function to actually use it.

The default control is the name for the implementation of control prepared by the layout library in the ctrl namespace. With the default control, you create a control using the DefaultControlCreator that inherits ControlCreator.

You can independently implement control without using the default control.

Creates a control corresponding to the specified control source.

Parameters
[in]pDeviceDevice.
[in]pLayoutThe layout that creates the control.
[in]controlSrcInformation about the control that is to be created. The index of this object is created in a temporary region, so do not keep the pointer after you exit this function. If you need to keep the information, copy it to store it.

Details
Called from inside the Layout::Build() function when there is a control in the layout resource.

The call takes place after the pane tree has been constructed. In other words, the layout passed by the argument is in the built state, excluding the control.

The created control is not managed by Layout, so you must implement a measure such as having ControlCreator store it internally.

There are several ways of using this feature.

  • Like with the DefaultControlCreator class, enter the created control in a member list and search for it by the parts pane name when you want to use it.
  • Keep a pointer to the layout inside a class that inherits from ControlCreator and delegate to the appropriate member function for the layout after CreateControl has been called. Controls are created for the layout based on this information and maintained in internal member variables.

Implemented in nn::ui2d::DefaultControlCreator.
