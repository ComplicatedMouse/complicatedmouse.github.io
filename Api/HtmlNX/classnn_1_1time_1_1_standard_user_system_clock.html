Detailed Description

The clock representing the user clock, which is the device clock that the user can manipulate.

Details
This clock indicates the current time held by the device. You can get a value that can be converted into a time expression that a human can understand. This clock returns the same time as nn::time::StandardNetworkSystemClock and it is affected not only by manipulations of time by the user, but also by automatic corrections to time by the system when any of the following applies.

  • Use Internet to Adjust Time is enabled in System Settings.
  • Automatic Correction is checked in the DevMenu time settings.
  • DevMenuCommand time get-user-clock-auto-correction shows enabled.

This clock is affected by modifications to time made by the user or system; the indicated time can be ahead of or behind the actual elapsed time. In addition, the clock is incremented when the device is powered down, and the value is reset when the power supply is lost as a result of a dead battery. When this happens, you can detect clock manipulations and resets, but you cannot rely on the calculation results for the time elapsed before and after those events.

This class satisfies the standard C++ TrivialClock requirements and provides features equivalent to std::chrono::system_clock. You cannot, however, detect clock manipulations or a value reset when you are only using the C++ standard interface.

This clock uses nn::time::StandardSteadyClock to increment, and therefore is affected by the nn::time::StandardSteadyClock accumulation errors. See also Accumulation Errors for the Monotonic Increase Clock .

Details
Comparing the context allows you to check whether the clock was manipulated.

Note that it is not only user-triggered time operations that can result in context mismatches but also system-triggered automatic time corrections. You can check whether system-based automatic corrections of this clock are enabled with any of the following methods.

  • Use Internet to Adjust Time is enabled in System Settings.
  • Automatic Correction is checked in the DevMenu time settings.
  • DevMenuCommand time get-user-clock-auto-correction shows enabled.
