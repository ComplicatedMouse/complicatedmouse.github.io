This sample program executes video playback using the movie API.

Overview

This sample loads an MP4 file into the NX system and plays it.

File Structure

This sample program is located in Samples/Sources/Applications/MovieDecoderPlayer/ .

Required Environment

No additions are necessary.

Using the Demo

If you execute the sample program, the specified video will be played back automatically. The program exits automatically after playback is complete.

Cautions

Nothing specific.

Execution Procedure

<MovieDecoderPlayer executable> <media path> -me <media extension> <optional parameters>

<media extension> : mp4, m4a, webm, mkv, ogg, ivf, h264

<optional parameters>

  • -l 0: Do not perform looped playback.
  • -l 1: Perform looped playback.
  • -aging 0: Aging test disabled. (The aging test is a test that destroys the Decoder, the Extractor, and other resources, each time and repeats playback.)
  • -aging 1: Aging test enabled.
  • -core 1: The CPU core mask is 0x1=0001. That is, it uses Core 0. (The CPU core mask can be checked using the CPU profiler log.)
  • -core 2: The CPU core mask is 0x2=0010. That is, it uses Core 1.
  • -core 4: The CPU core mask is 0x4=0100. That is, it uses Core 2.
  • -t 0: Disables memory allocation tracking.
  • -t 1: Enables memory allocation tracking. (Note: Enabling this may cause performance during playback to deteriorate.)
  • -dm 1: Enables movie::DecoderMode_Cpu mode.
  • -dm 2: Enables movie::DecoderMode_NativeTexture mode. The application passes NVN_FORMAT_RGBA8-formatted NVN textures to the video decoder. (Default)

Example (Getting video data from the host PC) MovieDecoderPlayer.nsp c:\H264_AAC_1280x720_30sec_01.mp4 -me mp4

Description

Plays the specified video file on the NX system.

About Attached Video Files

  • H264_AAC_1280x720_30sec_01.mp4 is an H.264 video / AAC audio MP4 file.
  • H264_DRC_720p_1080p_720p.h264 is an H.264 video file that dynamically changes resolution.
