◆ PresenceStatus

enum nn::friends::PresenceStatus

Indicates the presence state.

Details
Presence states change as follows.

  • When the user account is open: The selected user account's presence state changes to online.
  • When the user account is closed: The selected user account's presence state changes to offline.
  • When declaring the start of online play: The selected user account's presence state changes to in online play.
  • When declaring the end of online play: The selected user account's presence state changes to online.
  • When connected to a network: An open user account's presence state changes to online if it was offline.
  • When disconnected from a network: The presence states of all user accounts change to offline. Declarations of the start of online play are automatically obtained. (That is, declare the end.)
Enumerator
PresenceStatus_Offline 

Offline.

Friends will not see you as playing a game.

PresenceStatus_Online 

Online.

Friends will see you as playing a game.

PresenceStatus_OnlinePlay 

In online play.

Friends will see that you are in online play, whether that is competitive or cooperative play.

◆ NotificationType

enum nn::friends::NotificationType

Types of notifications.

Details
We recommend the following handling for each notification.

  • NotificationType_FriendListUpdated Updates the entire friend list using GetFriendList.
  • NotificationType_FriendPresenceUpdated Updates friend information individually using Friend::Update and UpdateFriendInfo.

Returns
The process result.
Precondition

  • outAccountIds != nullptr.
  • offset >= 0.
  • count > 0.
  • The specified user account is in an open state.

Details
Gets the network service account IDs for the specified number of friends, starting from the specified offset. Setting a filter enables you to get just those friends that match the filter conditions.

This function gets the friend list from the cache in console storage. It enables you to get the friend list even when not connected to the network yet.

Note
The cache in the console storage is updated at an appropriate time, such as after waking from sleep mode or when updated data is detected on the server.

Details

)

Adds users who have played together.

Parameters
[in]uidThe user account.
[in]keyThe registration key for the other party.
[in]inAppScreenNameThe display name within the application for the other party.
[in]myInAppScreenNameYour display name within the application.
Returns
The process result.
Precondition
  • The specified user account is in an open state.

Details
Applications add users who have played together with the following procedure.

  1. Get the registration key: Get the registration key for the local user using GetPlayHistoryRegistrationKey.
  2. Create a session.
  3. Distribute the registration key and the display name within the application: Distribute the first obtained registration key and display name within the application to all users within the session.
  4. Add the registration key and the display name within the application: Use this function to add the registration keys and display names within the application for all users within the session.

In the context of online play, a session refers to a P2P session. (Not a matchmaking session.) This is because users who can actually play together are those who created the P2P session. We do not know if that can occur at the level of matchmaking session creation.

Examples
FriendsPlayHistory/FriendsPlayHistory.cpp.

◆ GetFriendInvitationNotificationEvent()

os::SystemEventType* nn::friends::GetFriendInvitationNotificationEvent ( )

Gets a SystemEventType object representing the notification of a friend invitation.

Returns
Returns a pointer to a SystemEventType object representing the notification of a friend invitation.

Details
This function gets an nn::os::SystemEventType object representing the notification of a friend invitation. The obtained SystemEventType object is signaled when the application itself is notified of a friend invitation.

Normally, this system event is cleared automatically by the system, so a program cannot explicitly clear it. Normally, this event is cleared automatically by the call to nn::friends::TryPopFriendInvitationNotificationInfo().

In some cases, this event will have been signaled already when the program starts. In other cases, it will be signaled while the program is running. In either case, get the application parameter using the nn::friends::TryPopFriendInvitationNotificationInfo() function.

The obtained SystemEventType object will not be destroyed. To destroy it, call the nn::os::DestroySystemEvent() function.

Examples
FriendsFriendInvitation/FriendsFriendInvitation.cpp.

◆ TryPopFriendInvitationNotificationInfo()

bool nn::friends::TryPopFriendInvitationNotificationInfo ( account::Uid *  pOutUid,
size_t *  pOutSize,
void *  pOutBuffer,
size_t  bufferSize 
)

Gets the information associated with a friend invitation notification.

Parameters
[out]pOutUidThe UID representing the user who is the target of the friend invitation.
[out]pOutSizeThe storage location for the actual size of the application parameter.
[out]pOutBufferThe storage location for the application parameter.
[in]bufferSizeThe size of the storage location for the application parameter.
Returns
Returns a bool value indicating whether there is a friend invitation notification.

Details
If the notification of a friend invitation has occurred, this function gets the UID of the target user and the application parameter associated with that friend invitation.

If this function returns true, the notification of a friend invitation has occurred, and the UID of the target user and the application parameter associated with that friend invitation have been saved in the locations specified by the arguments. The event obtained by the nn::friends::GetFriendInvitationNotificationEvent() function will be cleared automatically at the same time. If the application parameter was not configured, the value 0 is stored in pOutSize.

If the next friend invitation notification occurs while this function is still getting the information about the previous friend invitation notification, that old information is destroyed and only the information about the latest friend invitation is obtained.

If this function returns false, the notification of a friend invitation has not occurred, and no change is made to the content in pOutUid, pOutSize, pOutBuffer.

Examples
FriendsFriendInvitation/FriendsFriendInvitation.cpp.
