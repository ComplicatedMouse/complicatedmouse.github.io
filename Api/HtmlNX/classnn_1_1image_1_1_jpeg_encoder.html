Detailed Description

Encodes JPEG data.

Details
Encodes the provided pixel data based on the specified parameters to output JPEG data. There is also a feature for creating Exif information from the provided nn::image::ExifBuilder object and appending it to the JPEG data.

Specifications of Output JPEG Data

The pixel data is output as baseline JPEG encoded in YCbCr color space. For the pixel data sampling ratio, you can select 4:2:0 (the default), 4:2:2, or 4:4:4. You can also specify an encoding quality to adjust the image quality and the file size.

Size of Working Memory and Output

The size of working memory needed for JPEG encoding and the size of the output JPEG data both vary depending on the pixel data that is input and the encoding parameters that are configured.

The required size of working memory is uniquely based on the amount of pixel data and the sampling ratio, and has nothing to do with the content of that pixel data. If the application will be encoding data with an unspecified sampling ratio, you can determine an upper limit to the reserved working memory by acting ahead of time to encode the data with the following parameters.

| Pixel Size | Sampling ratio | |– :| |The maximum size the application will handle. | 4:2:0|

On the other hand, it is hard to calculate ahead of time how much JPEG data will be output, because it varies depending on the content of the pixel data (the colors and patterns). For reference purposes, the following table shows the output buffer size for different amounts of pixel data and encoding parameters. Object States

The JpegEncoder object can take the following three states.

  • Unregistered Pixel Data
    • The state where no pixel data is set in the JpegEncoder object.
  • Registered Pixel Data
    • The state where pixel data is set in the object, but the data and the parameters have not been analyzed.
  • Analyzed Pixel Data
    • The state of the JpegEncoder object after the last of the set pixel data and parameters have been analyzed.

Parallel Processing

You can create multiple JpegEncoder class objects to encode multiple sets of pixel data in parallel.

Examples

Specifies a pixel format for the pixel data to JPEG encode and registers the data.

Parameters
[in]pixelDataThe pixel data to encode. The pixels in this pixel data must be in the pixel format specified by pxFormat. If the pixel data includes an alpha channel, that value in the alpha channel is ignored. The padding region for alignment is also ignored.
[in]pxFormatPixel format of the pixel data to encode.
[in]dimensionSize (width and height) of pixelData.
[in]lineAlignmentLine alignment of the pixel data. For example, you would specify 2 for a 129 pixel-wide image stored in a 130 pixel-wide pixel buffer.
Precondition
All of the following conditions must be satisfied.
  • pixelData must be the size specified by dimension and line-aligned to the value specified by lineAlignment.
  • pxFormat must be specified as either nn::image::PixelFormat_Rgb24 or nn::image::PixelFormat_Rgba32.
Postcondition
The state of the object becomes Registered Pixel Data and the Analyze() function can be called.

Details
Specifies a pixel format for the pixel data to encode and registers the data. Relatively less work memory is required if nn::image::PixelFormat_Rgb24 is specified as the pixel format as opposed to nn::image::PixelFormat_Rgba32.

After you have called this function, do not change the content registered here for pixelData until the instance is destroyed or this function is called again.

JpegStatus nn::image::JpegEncoder::Encode ( size_t *  pOutActualCodedSize,
void *  outJpegBuf,
const size_t  jpegBufSize,
ExifBuilder *  pExifBuilder 
)

Encodes pixel data as JPEG data.

Parameters
[out]pOutActualCodedSizeThe location used to store the actual byte size of the output JPEG data.
[out]outJpegBufThe location used to save the output JPEG data. The size of the region where the output JPEG data will be stored is undefined and varies depending on the pixel data being encoded and the settings for image quality and sampling ratio.
[in]jpegBufSizeSize of outJpegBuf in bytes.
[in]pExifBuildernn::image::ExifBuilder object for creating the Exif information to append to the JPEG data. To omit the output of Exif information, specify nullptr.
Returns
Returns the execution result.
Return Values:

nn::image::JpegStatus_Ok Indicates that the process ended normally.

nn::image::JpegStatus_ShortOutput Indicates that the output buffer ran short during encoding. Either specify a larger buffer for the output buffer, or adjust the image quality and sampling ratio. Setting smaller values for the image quality and sampling ratio reduces the required buffer size, but image quality suffers.

nn::image::JpegStatus_OutOfMemory There was not enough work memory during encoding. Not enough memory is available to the memory allocator specified by the constructor.

Precondition
All of the following conditions must be satisfied.
  • The nn::image::ExifBuilder::IsBuildable() function returns true when a value other than nullptr is set for pExifBuilder.
  • The state is Analyzed Pixel Data.
Postcondition
If a value other than nullptr is set for pExifBuilder, Exif information is appended to the JPEG data output to outJpegBuf. At that time, changes are made in the working memory of the ExifBuilder object.

Details
Encodes pixel data, using the configured pixel data, the image quality setting, and the sampling ratio. Exif information is appended to the output JPEG data if a valid value has been specified for pExifBuilder.

Sometimes the Encode() function returns an error on pixel data for which the Analyze() function returned JpegStatus_Ok. This happens because the Encode() function uses information not referenced by the Analyze() function. Accordingly, you must check for errors from both functions.

You cannot calculate the size of the output buffer ahead of time, so make sure that you allocate enough space for your purposes. The description for the JpegEncoder class provides a rough guide with examples of the size of the output buffer for representative pixel sizes and encoding settings.

Examples
ImageJpegEncoding/ImageJpegEncoding.cpp.
