Detailed Description

The vibration node that functions as the recipient of vibration values.

Details
VibrationTarget is the vibration node that functions to receive the vibration values sent from other vibration nodes and send them to an actual vibration motor.
It uses the nn::hid::SendVibrationValue function to send vibration values.
Because it inherits the nn::hid::VibrationMixer class, you can use vibration values sent from multiple vibration nodes before sending to a vibration motor.

The mixed vibration values are sent to the target vibration node every time nn::hid::VibrationNode::Update is called.

◆ Update()

static void nn::hid::VibrationNode::Update ( )
staticinherited

Updates the entire directed graph for the vibration value flow, and transmits vibration values from the separate vibration nodes to the nodes they are connected to.

Details
Calling this function updates the internal states of all vibration nodes. When this occurs, vibration values are transmitted between adjacent vibration nodes, from connection source to connection target. Connection source nodes are updated before connection target node.

You can call this function at any time, but we recommend you use a timer and call it at an interval of DefaultVibrationSampleInterval.

This function is not thread-safe. Use mutual exclusion to prevent the constructor and destructor of this class and derived classes from being called simultaneously.
