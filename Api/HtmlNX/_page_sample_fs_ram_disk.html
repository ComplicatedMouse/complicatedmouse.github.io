Description of the sample program for the RamDisk feature.

Overview

This page describes the sample program for the RamDisk feature.

For more information about file and directory operations, see the File System Library API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/FsRamDisk/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

80 {
81  nn::Result result;

83  // Allocates 32 MB of memory for RamDisk.
84  // Consider the amount of memory to allocate based on the needs of your application.
85  const size_t SpaceSize = 32 * 1024 * 1024;
86 
87  // Calculate the size of the buffer required for RamDisk and allocate it.
88  size_t bufferSize;
89  bufferSize = nn::fs::GetRamDiskBufferSize(SpaceSize);
90 
91  void* ramDiskBuffer = std::malloc(bufferSize);
92  NN_ABORT_UNLESS_NOT_NULL(ramDiskBuffer);
93 
94  // Mount RamDisk with the name 'ram.'
95  NN_LOG("Mount RamDisk as \"ram\"\n");
96  NN_LOG("SpaceSize : %zu\n", SpaceSize);
97  NN_LOG("BufferSize : %zu\n", bufferSize);
98  result = nn::fs::MountRamDisk("ram", ramDiskBuffer, bufferSize);
99  // Always abort when a failure occurs.
100  NN_ABORT_UNLESS_RESULT_SUCCESS(result);
101 
102  // Ensure that the file system is always empty immediately after calling nn::fs::MountRamDisk().
103 
104  const size_t FileSize = 1024;
105  const size_t WriteBufferSize = 1024;
106  const size_t ReadBufferSize = 1024;
107  static char writeBuffer[WriteBufferSize];
108  static char readBuffer[ReadBufferSize];
109 
110  // Create files and write data.
111  {
112  result = nn::fs::CreateFile("ram:/file", FileSize);
113  if( nn::fs::ResultPathNotFound::Includes(result) )
114  {
115  // There are no directories within the path.
116  // This error handling is not necessary when parent directories exist without fail.
117  }
118  else if( nn::fs::ResultPathAlreadyExists::Includes(result)
119  || nn::fs::ResultTargetLocked::Includes(result) )
120  {
121  // The target file already exists.
122  // This error handling is not needed when the file already exists and that is not an issue.
123  // Note that there is no assurance that the file size is equal to FileSize when you are not using error handling.
124  // Delete the target file and re-create it when necessary.
125  }
126  else if( nn::fs::ResultUsableSpaceNotEnough::Includes(result) )
127  {
128  // Insufficient data save area.
129  // Increase the data save region.
130  NN_ABORT("Usable space not enough.\n");
131  return;
132  }
133  // No further error handling is necessary because
134  // the process is aborted in the library if it fails for any other reason.
135 
136  // Write initial data.
137  NN_LOG("Create ram:/file\n");
138  nn::fs::FileHandle fileHandle;
139  result = nn::fs::OpenFile(&fileHandle, "ram:/file", nn::fs::OpenMode_Write);
140  if( nn::fs::ResultPathNotFound::Includes(result) )
141  {
142  // The target file cannot be found.
143  // This error handling is not necessary when you are only opening existing files.
144  }
145  else if( nn::fs::ResultTargetLocked::Includes(result) )
146  {
147  // The target file is already open.
148  // This error handling is not necessary when there is no possibility of the file being open already.
149  }
150  // No further error handling is necessary because
151  // the process is aborted in the library if it fails for any other reason.
152  for (int i = 0; i < WriteBufferSize; i++)
153  {
154  writeBuffer[i] = static_cast<char>(i);
155  }
156  NN_LOG("Write ram:/file\n");
157  result = nn::fs::WriteFile(fileHandle, 0, &writeBuffer, sizeof(writeBuffer), nn::fs::WriteOption::MakeValue(nn::fs::WriteOptionFlag_Flush));
158  if( nn::fs::ResultUsableSpaceNotEnough::Includes(result) )
159  {
160  // Insufficient data save area.
161  // This error handling is not necessary when you would not specify OpenMode_AllowAppend when opening files.
162  }
163  // No further error handling is necessary because
164  // the process is aborted in the library if it fails for any other reason.
165 
166  // There is no need to call nn::fs::Commit() because RamDisk does not support journaling features.
167  nn::fs::CloseFile(fileHandle);
168  }
169 
170  // Read data from the created files.
171  {
172  nn::fs::FileHandle fileHandle;
173  result = nn::fs::OpenFile(&fileHandle, "ram:/file", nn::fs::OpenMode_Read);
174  if( nn::fs::ResultPathNotFound::Includes(result) )
175  {
176  // The target file cannot be found.
177  // This error handling is not necessary when you are only opening existing files.
178  }
179  else if( nn::fs::ResultTargetLocked::Includes(result) )
180  {
181  // The target file is already open.
182  // This error handling is not necessary when there is no possibility of the file being open already.
183  }
184  // No further error handling is necessary because
185  // the process is aborted in the library if it fails for any other reason.
186 
187  // Read data.
188  size_t readSize = 0;
189  NN_LOG("Read ram:/file\n");
190 
191  // No error handling is needed because an abort occurs within the library when loading fails.
192  (void)nn::fs::ReadFile(&readSize, fileHandle, 0, &readBuffer, sizeof(readBuffer));
193 
194  NN_ASSERT_EQUAL(readSize, sizeof(readBuffer));
195  NN_ASSERT(std::memcmp(writeBuffer, readBuffer, ReadBufferSize) == 0);
196 
197  nn::fs::CloseFile(fileHandle);
198  }
199 
200  // Unmount.
201  NN_LOG("Unmount \"ram\"\n\n");
202  nn::fs::Unmount("ram");
203 
204  return;
205 }

Sample Program Description

This sample program has the following flow.

  • Use the GetRamDiskBufferSize function to get the buffer size required for RamDisk.
  • Use the MountRamDisk function to mount RamDisk.
  • Create and write files.
  • Load the created files.
