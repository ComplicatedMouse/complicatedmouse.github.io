Detailed Description

The base class for the vibration node.

Details
The vibration node is a utility class that provides features for propagating vibration values when used in combination with the VibrationNodeConnection class. It comprises instances of the vibration node and information about connections between those instances, all as a single directed graph (a directed graph of the vibration value flow). The instances of the vibration node serve as nodes on this directed graph, while the instances of VibrationNodeConnection serve as arcs (directed edges).

The base class VibrationNode provides features for connecting vibration nodes and for setting modulation factors when transmitting vibration values. The derived classes VibrationPlayer, VibrationWriter, VibrationMixer, and VibrationTarget provide the features for creating, mixing, and receiving vibration values. By inheriting VibrationNode, you can create vibration nodes with features that differ from the parent node.

To update the internal state of all vibration nodes, call the Update() function. When this occurs, vibration values are transmitted between adjacent vibration nodes, from connection source to connection target.

You can send a different vibration value to every target from a single vibration node by using the SetModulationTo() function to set a different modulation factor for each vibration node the vibration value is being sent to.

The constructor, destructor, and the Update() function for this class and its derived classes are not thread-safe. Use mutual exclusion to prevent the constructor and destructor from being called simultaneously with Update().

◆ Update()

static void nn::hid::VibrationNode::Update ( )
static

Updates the entire directed graph for the vibration value flow, and transmits vibration values from the separate vibration nodes to the nodes they are connected to.

Details
Calling this function updates the internal states of all vibration nodes. When this occurs, vibration values are transmitted between adjacent vibration nodes, from connection source to connection target. Connection source nodes are updated before connection target node.

You can call this function at any time, but we recommend you use a timer and call it at an interval of DefaultVibrationSampleInterval.

This function is not thread-safe. Use mutual exclusion to prevent the constructor and destructor of this class and derived classes from being called simultaneously.
