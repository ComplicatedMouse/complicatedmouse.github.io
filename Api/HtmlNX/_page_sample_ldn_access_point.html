Describes the LDN library access point sample.

Overview

This sample demonstrates an LDN library access point. LdnStation is a sample of an LDN library station.

LdnAccessPoint and LdnStation are planned for deletion. For their replacements, see the added LdnSimple .

File Structure

This sample program is located in Samples/Sources/Applications/LdnBasic/LdnAccessPoint/ .

How to Use

When you start the sample program, a log message describes how to use it. Operate the sample with a Debug Pad according to those instructions.

Important Information

This sample program conducts wireless communications in the 2.4-GHz band. Avoid using it in environments where there many other wireless communication devices use the same frequency band. When the wireless signal strength is poor, the following types of issues frequently occur.

  • The target LDN network cannot be found.
  • Fail to connect to the LDN network.
  • UDP packet loss rates increase.

If you are limited in what you can perform to improve the signal environment, you can modify the sample program to make use of the 5-GHz band. To do this, change the channel member of the nn::ldn::NetworkConfig structure (which you pass as an argument to nn::ldn::CreateNetwork()) from the nn::ldn::AutoChannel value to a value of 36, 40, 44, or 48. You must also change the LdnStation sample accordingly. For a description, see the LdnStation sample.

Execution Procedure

Build the sample program and then run it.

Description

The LDN Library provides functionality to connect multiple targets without going through a router or the Internet, and for the applications running on each target to communicate with each other.

Up to eight target units can connect and communicate with each other by using the LDN library. The first target becomes the access point, which constructs the LDN network. The other targets then search for the LDN network and connect to it as stations.

This sample program constructs an LDN network as an access point. After the network is set up, use the LdnStation sample to connect a station to the LDN network. After the LDN network has been set up, communication will take place via UDP using the socket library. All targets have their own counter, and those counter values will be shared among all targets. Specifically, stations send their own counter to the access point via unicast, and the access point broadcasts everyone's counter (including its own) to all connected stations.
