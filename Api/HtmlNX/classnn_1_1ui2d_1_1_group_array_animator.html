Detailed Description

The Animator class is used for animating all groups associated with the animation resource.

Examples
Ui2dViewerSimple/main.cpp.

Member Enumeration Documentation

◆ PlayType

enum nn::ui2d::Animator::PlayType
inherited

Indicates how to play the animation.

Enumerator
PlayType_OneShot 

Indicates that the animation plays to the last frame (or the first frame with reverse playback), and then stops.

PlayType_Loop 

Indicates that the animation plays to the last frame (or the first frame with reverse playback), and then starts again from the first frame (or the last frame with reverse playback).

PlayType_RoundTrip 

Indicates that the animation plays to the last frame (or first frame with reverse playback), and then plays in the other direction.

◆ FlagMask

Enumerator

FlagMask_PlayStatusTurnedMin

FlagMask_PlayStatusEndNext

◆ Play()

virtual void nn::ui2d::Animator::Play ( PlayType  type,
float  speed 
)
virtualinherited

Plays the animation from the beginning.

Details
If a negative value is assigned to speed, plays in reverse from the end, rather than forward from the start.

Regardless of the value of the current frame, 0 is set for forward playback, and the maximum frame value is set for reverse playback.

The only playback type that can be specified when the animation is a Wait animation is PlayType_OneShot. Regardless of the value set for speed, playback stops after one frame and the frame position becomes 0.

Parameters
[in]typePlayback type.
[in]speedPlayback speed. This unit defines how many frames to advance per call to UpdateFrame. Specify 1.f to play at the same speed as set by the designer. Specify a negative number to play to play in reverse. The absolute value of the speed cannot exceed the number set by GetFrameMax(). An assert is made in such cases. Playback is not available for production versions.

◆ PlayAuto()

virtual void nn::ui2d::Animator::PlayAuto ( float  speed)
virtualinherited

Plays the animation from the beginning using the playback type specified in LayoutEditor.

Details
PlayType_Loop is the result when IsLoopData() returns true when a loop was selected in the Layout Editor. In all other cases it is PlayType_OneShot.

If a negative value is assigned to speed, plays in reverse from the end, rather than forward from the start.

Regardless of the value of the current frame, 0 is set for forward playback, and the maximum frame value is set for reverse playback.

When the animation is a Wait animation, playback ends after a single frame and the frame position becomes 0, regardless of the value of speed.

Parameters
[in]speedPlayback speed. This unit defines how many frames to advance per call to UpdateFrame. Specify 1.f to play at the same speed as set by the designer. Specify a negative number to play to play in reverse. The absolute value of the speed cannot exceed the number set by GetFrameMax(). An assert is made in such cases. Playback is not available for production versions.
Examples
Ui2dViewerSimple/main.cpp.

◆ IsEndTrigger()

bool nn::ui2d::Animator::IsEndTrigger ( ) const
inherited

After UpdateFrame, indicates that playback has ended when PlayType is PlayType_OneShot.

Details
The function differs from IsMaxFrame and IsMinFrame in that you can detect the end of playback regardless of whether a positive or negative value has been assigned to speed.

Note that this flag is not set if a member function like Stop is called explicitly to stop the animation. If you want to find out whether an animation is playing, use the IsPlaying member function.

Note that this member function returns false when UpdateFrame is next called.

Returns
Returns whether playback has ended.

Forces the binding of an animation of another pane.

Details
Binds the animation of the pane specified by pSrcPane to the pane specified by pDstPane.

Binds animations in panes that were originally separate, so it could cause buggy results if, for example, the panes are of different types. Use this feature with care.
