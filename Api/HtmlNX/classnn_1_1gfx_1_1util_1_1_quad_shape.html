Detailed Description

Calculates the vertex information for the quadrilateral.

[in]primitiveTopologyPrimitive topology.
Precondition
  • ( PrimitiveShapeFormat_Pos <= vertexFormat ) && ( vertexFormat <= PrimitiveShapeFormat_Default )
  • If displaying with surface modeling, primitiveTopology == nn::gfx::PrimitiveTopology_TriangleList.

Details
Creates a square centered at the origin in the xy-plane, with a width of 2.0 and a height of 2.0.

When using normals, you can get the vectors in the positive z-axis direction as normals.

Writes the vertex and index information to the buffer that is passed.

Parameters
[out]pVertexMemoryPointer to the memory for writing vertex information.
[in]vertexSizeSize of allocated memory (in bytes).
[out]pIndexMemoryPointer to the memory for writing index information.
[in]indexSizeSize of allocated memory (in bytes).
Precondition
  • pVertexMemory != NULL.
  • pIndexMemory != NULL.
  • vertexSize >= GetVertexBufferSize().
  • indexSize >= GetIndexBufferSize().

Details
Vertex data includes three components: coordinates, normals, and UV. Using an interleaved format for just a component specified using vertexFormat of the constructor, the order for writing vertex buffers is coordinates (nn::gfx::AttributeFormat_32_32_32_Float), normals (nn::gfx::AttributeFormat_32_32_32_Float), and UV (nn::gfx::AttributeFormat_32_32_Float). For example, if nn::gfx::util::PrimitiveShapeFormat_Pos or nn::gfx::util::PrimitiveShapeFormat_Uv is specified for vertexFormat, | the vertexes are written sequentially in the {x,y,z,u,v} format.

The index information is written in the order of each vertex in a counterclockwise direction (nn::gfx::FrontFace_Ccw) when the primitive is viewed from the front. You can use GetIndexBufferFormat() to get the format of the written index information.
