This documentation describes the Edit sample program. This sample program shows how to use the viewer library for real-time editing with 3DEditor.

Overview

This sample uses the viewer library for real-time editing with 3DEditor.

File Structure

This sample program is located in Samples/Sources/Applications/G3dDemo/ .

Required Environment

Nothing specific.

How to Use

The following operations are possible using input from the Joy-Con controllers, the DebugPad, or a keyboard when using a PC.

Input Operation
START Button, SPACEBAR, Joy-Con + Button: Returns to the menu.
Directional Buttons: Switches the menu selection state.
A Button, A key, or left-mouse click: Engages the selected menu option.

Important Information

To conduct real-time editing, you must start 3DEditor and Target Manager and connect them to the application.

Execution Procedure

Build the sample program, select Edit from the menu, and run.

Description

Connecting this sample program to 3DEditor enables it to display the model that has been loaded into 3DEditor.

This sample program has the following flow.

  • Initialize the viewport.
  • Load and initialize the shader archive.
  • Load and initialize the model resource, and initialize the texture.
  • Bind the shader, texture, and model.
  • Initialize the viewer library.
  • Start the thread for communication with 3DEditor.
  • Begin the loop.
  • Process button input (such as registering a command to the viewer library).
  • Run the commands registered in the viewer library.
  • Update models.
  • Perform calculations for each matrix.
  • Generate the command list.
  • Execute the command list.
  • Swap the scan buffer.
  • Return to the start of the loop.
