Detailed Description

Allocator that manages the memory in the memory pool that is temporarily used.

Details
Dynamically allocates the memory pool memory that is temporarily used during rendering. Also provides a feature to deallocate all memory at the same time after rendering completes.

Specifies the memory pool and the range for the offset to be managed, and then initializes.

Parameters
[in]pMemoryPoolThe memory pool.
[in]baseOffsetThe start of the offset for the memory to manage.
[in]sizeThe length, in bytes, of the memory to manage.
[in]allocatableAlignmentMaxThe maximum alignment required for the memory allocated from this allocator.
[in]historyLengthMaxThe maximum number of history entries to record.
Precondition
IsInitialized() == false.
pMemoryPool must be a pointer to a valid memory pool.
baseOffset >= 0.
baseOffset % allocatableAlignmentMax == 0.
size >= 1 && size <= SizeMax.
allocatableAlignmentMax <= AlignmentMax.
allocatableAlignmentMax must be a power of 2.
historyLengthMax >= 1.
There must be a memory block configured to have the required alignment size, set with SetMemory().
Postcondition
IsInitialized() == true.
IsRecording() == false.
GetHistoryLength() == 0.

Details
This object targets the memory region with the range specified in the memory pool for management. A contiguous valid range must be specified for the memory region.

baseOffset must be an integer multiple of allocatableAlignmentMax. You can specify an alignment with a value that is at most equal to that specified in allocatableAlignmentMax, using Allocate().

Calling this instance for the same instance is not thread-safe.

Releases the oldest recorded memory from among those recorded by this object.

Precondition
IsInitialized() == true.
GetHistoryLength() > 0.
Postcondition
Releases the oldest memory recorded in the history.
The value returned by GetHistoryLength() is decremented by 1.

Details
The memory allocated between the calls to Begin() and End() is treated as a single unit and released. The memory allocated individually using Allocate() cannot be released on an individual basis. Any released memory becomes immediately available for allocation. Multiply calling this instance for the same instance is not thread-safe. Calling both this function for the same instance and the function that allocates memory at the same time is thread-safe. End()

◆ FillHistory()

void nn::gfx::util::TransientMemoryPoolAllocator::FillHistory ( )

Enters the history for the 0th allocation.

Precondition
IsInitialized() == true.
IsRecording() == false.
Postcondition
GetHistoryLength() == GetHistoryLengthMax().

Details
Adds to the history for the 0th allocation until the history is full. Equivalent to the following code.

TransientMemoryPoolAllocator transient;
while(transient.GetHistoryLength() < transient.GetHistoryLengthMax())
{
transient.Begin();
transient.End();
}
