This sample implements the displaying of English, Japanese, and Korean strings to introduces how to render font with nn::fontll scalable font engine.

Overview

This sample implements the displaying of English, Japanese, and Korean strings to illustrate how to utilize the nn::fontll library in the NintendoSDK together with NVN to rasterize and render text to screen using the internal scalable fonts that the NX includes in its system software (and which is also supported on Windows).

Description

Two font-related classes in the SharedLib are used for this demo, namely Font, and SimpleTextRenderer.

See the Font class implementation on how to create an nn::fontll::ScalableFontEngine object, how to load the NX system font through nn::pl, and how nn::fontll::GlyphMap objects are acquired and their properties are used to rasterize characters into a two-dimensional linear buffer.

See the SimpleTextRenderer class for an example implementation of taking rasterized text in linear buffers and rendering them through NVN, which involves copying the buffers over to textures that are then drawn while blending with the frame buffer contents.

Note that in NintendoSDK, UTF-8 is used as standard character encoding. The SimpleTextRenderer recives UTF-8 strings and converts them to UTF-32 for supporting multi bytes character rendering. So add the u8 prefix to string literals to ensuring the input strings use UTF-8 character encoding.
