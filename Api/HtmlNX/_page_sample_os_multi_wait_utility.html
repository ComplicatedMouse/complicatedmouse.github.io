A sample program for the multi-wait feature utility.

Overview

This sample shows the multi-wait feature utility.

For more information about using this multi-wait feature, see the Multi-Wait Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsMultiWaitUtility/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Two threads are created and one message queue and one semaphore event are initialized with nnMain().
  • MultiWaitThreadFunction() performs a multi-wait for semaphore release, event signaling, sending a message to the message queue, and finalizing SignalThread.
    • A log corresponding to objects on standby is exported and the signals for those objects are cleared.
    • Finalizes when SignalThread finalizes.
  • SignalThreadFunction() performs the following in order: semaphore release, event signaling, sending a message to the message queue, and finalizing the thread.
  • nnMain() waits until the two threads complete.
  • The two threads are destroyed by nnMain().

The following file shows the execution results of this sample program.

BEGIN
Semaphore is releasing.
Event is signaling.
Message Queue is not empty.
SignalThread was finished.
END

SignalThreadFunction() issues the various signals, which are picked up by MultiWaitThreadFunction() as it performs its multi-wait. The corresponding functions are called, and a log is exported.

This sample is a rewrite of the OsMultiWait code using the multi-wait feature utility.
