◆ isPredictionEnabled

bool nn::swkbd::AppearArg::isPredictionEnabled

Specifies whether to use predictive text input.

Specify true to use the feature.

Details
The value specified with isPredictionEnabled is valid when all of the following criteria are met.

  • KeyboardMode is KeyboardMode_LanguageSet1, KeyboardMode_LanguageSet1Latin, or KeyboardMode_Alphabet.
  • invalidCharFlag is not set.

◆ isCancelButtonDisabled

bool nn::swkbd::AppearArg::isCancelButtonDisabled

You can disable the input cancel button.

Specify true to prevent the software keyboard from being terminated with the B Button or X Button. While it is more difficult for the user to cancel via input, cancellation may occur if the following procedure is performed.

  1. Press the HOME Button while the keyboard is displayed and start system settings or other software from the HOME Menu.
  2. If you then resume the application, the nn::swkbd::ShowKeyboard() function returns nn::swkbd::ResultCanceled.

If you want to make it more difficult to accept empty input, set a value of 1 or more for textMinLength.

◆ miniaturizationMode

MiniaturizationMode nn::swkbd::AppearArg::miniaturizationMode

You can specify the behavior of the hardware keyboard when it is in use.

If text input is often performed in tandem with other operations in your application, specify a mode that prioritizes input from the hardware keyboard.

Details
The modes that prioritize input from the hardware keyboard are MiniaturizationMode_Auto and MiniaturizationMode_Forced. If either of these modes is specified, the layout of the software keyboard disappears and only the footer and the area showing conversion candidates is displayed when the hardware keyboard is being used. While the keyboard layout is hidden (State_Miniaturized), a wider area of the screen is available to the application, and all input from the controller can be used by the application. That also creates the need for the application to render to the screen region that was previously hidden by the keyboard layout, and to implement appropriate means for processing the controller input.

In addition, when the mode is MiniaturizationMode_Auto or MiniaturizationMode_Forced, the WindowMode is forced to WindowMode_Single.

◆ startSamplingNumber

int64_t nn::swkbd::AppearArg::startSamplingNumber

Sets the starting position for input from the USB keyboard that will be processed right after the software keyboard has started.

Normally, input from the USB keyboard is accepted after the software keyboard has finished starting, but if a value has been set here, input from the USB keyboard prior to the start of the software keyboard is processed retroactively. If you will be starting the software keyboard when there is key input from the USB keyboard, specify nn::hid::KeyboardState::samplingNumber for that key input.
