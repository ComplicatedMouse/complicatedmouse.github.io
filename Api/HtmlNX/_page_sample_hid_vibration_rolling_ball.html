Documentation for an interactive technical demo using the vibration feature.

Overview

An interactive demo that allows you to experience the effects of the vibration feature. The vibration feature allows you to feel the weight and position of a ball that you are manipulating on the screen.

File Structure

This sample program is located in Samples/Sources/Applications/HidVibrationRollingBall .

Required Environment

This does not run on Windows. When running in an SDEV or EDEV environment, pair the controller to the SDEV or EDEV ahead of time. You must connect one controller.

Using the Demo

Start the sample program and pick up the controller. We recommend that you experience this demo with the Joy-Con attached to the EDEV.

As you tilt the controller, a ball moves left and right on the screen. The controller begins to vibrate as you move the ball inside the green frame.

You can press the L Button or R Button to switch to a different type of ball. Each of the three different types of ball produces a different type of vibration. The white stone ball produces the heaviest vibration and the colorful beach ball the lightest.

To feel the difference that these vibrations make, you can switch them on and off by pressing the X Button.

Each button allows you to change one of the demo's settings.

  • L Button: Switch to a different type of ball.
  • R Button: Switch to a different type of ball.
  • B Button: Initialize the position of the ball.
  • X Button: Enable or disable vibration feedback.
  • A Button: Mute or unmute audio.
  • Y Button: Show or hide graphics.

The actual vibration values are shown in the upper-right corner of the screen.

To end the sample program, press the + Button and the - Button at the same time.

Cautions

Make sure that the controller is sufficiently charged before using it.

Execution Procedure

Build the sample program and then run it.

Description

This sample program has the following flow.

  • Decide which controller to use (the one whose buttons were last pressed or whose sensors detected a change).
  • Get the controller's acceleration.
  • Update the position of the ball based on the acceleration data.
  • Generate a rolling sensation while the ball is rolling.
  • Generate a percussive sensation when the ball runs into a wall.
  • The vibration feedback varies with the type of the ball. To accomplish this effect, vibrations are generated using the same vibration file but with different parameters (the gain and pitch from nn::hid::VibrationModulation).
