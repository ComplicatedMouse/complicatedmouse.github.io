Detailed Description

Class for handling user account presence.

Details
Presence is functionality that conveys your information, including your online status and the application you are playing, to friends in real time.

The information configured in this class can be obtained on a friend's console via FriendPresence.

Users can set the range of presence disclosure. They can also elect to send no information at all to friends. As a result, there is no assurance that data will reach friends.

The possible ranges of presence disclosure are as follows.

  • Nonpublic
  • Just to favorite friends
  • To all friends

The application does not need to account for the range of presence disclosure for the system to perform presence filtering.

For changes in the status of the user's own presence, see PresenceStatus.

If you want to declare the beginning and end of online play and simultaneously update the friend presence, use the following functions.

  • UserPresence::DeclareOpenOnlinePlaySession
  • UserPresence::DeclareCloseOnlinePlaySession

Declarations for the start and end of online play will maintain the current state when neither of those functions is called.

Precondition

Details
The game mode description is the string that appears at the top of My Page. You can use an application to let friends know what you are doing, even if they do not have the same application open.

There is no character count restriction on the game mode description. (The game mode description scrolls in My Page.)

This function registers a game mode description using the sys_description key in the application-specific region. For more information about the specific restrictions, see UserPresence::SetAppValue.

Returns the result of processing.
Precondition
  • key != nullptr.
  • value != nullptr.

Details
We do not recommend using this function. It can only be used for the purpose of retaining compatibility, for example to rebuild a released application with the newest SDK.

Setting the application-specific information enables you to convey any data to friends.

This size of the region for recording application-specific information is PresenceAppFieldSize. Each key value consumes a size equal to the key's character count plus the value's byte count, plus two bytes for a terminating null character.

Key values that can be set by the application have the following restrictions.

  • Key Usable characters include the ASCII alphanumeric characters and the underscore [ _ ]. You cannot use keys that begin with sys_. They are reserved by the system.
  • Values You can only configure UTF-8 strings.
