Detailed Description

The class representing connections between vibration nodes.

Details
VibrationNodeConnection is a utility class that provides features for propagating vibration values when used in combination with vibration nodes (VibrationNode objects).
The instance of theVibrationNodeConnection class can indicate connections between vibration nodes.

The VibrationNodeConnection class instance can be in a state that indicates or does not indicate connections between vibration nodes.
Use the IsConnected() function to get its current state.

An instance with a state indicating rumbling holds information about the connection source and connection destination, and indicates a connection from one vibration node to another.
This vibration node serves as an arc (a directed edge) in the directed graph node structure.

On the other hand, an instance whose state does not indicate connection has no functionality in the directed graph.
To change its state to indicate a connection, call the Connect() function.

  • IsConnected() == true.

  • GetDestination() == pDestination.
  • pSource->GetConnectionTo(pDestination) == this.
  • GetModulation() == default value.

Details
Connects the specified nodes and initializes the instance to indicate the connection.

If the preconditions are not satisfied, the instance is initialized in the state that does not indicate a connection.

  • GetDestination() == pDestination.

Details
Connects the specified nodes and puts the instance in the state indicating the connection between nodes.
Note that when the destructor is called for this instance, the nodes will be automatically disconnected.
Also be careful not to generate circular constructs in the directed graph of the vibration value flow when making your connections.

void nn::hid::VibrationNodeConnection::Disconnect ( )

Disconnects the connections between vibration nodes.

Precondition
  • IsConnected() == true.
Postcondition
  • IsConnected() == true.
  • GetSource() == nullptr.
  • GetDestination() == nullptr.

Details
If you call this function on an instance when it is in the state indicating a connection between vibration nodes, the connection between the relevant nodes is disconnected and the instance is put in the state that does not indicate a connection.
