This sample for the tm library API gets development hardware images and videos.

Overview

This program is a sample that introduces how to use the RemoteVideo API in the tm library.

For more information about creating projects, see the tm Library API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/TmRemoteVideo .

Required Environment

Target Manager must be installed ahead of time.

How to Use

Nothing specific.

Important Information

Nothing specific.

Execution Procedure

Build the sample program and then run it.

266 bool UseVideoStream(nn::tm::TargetHandle target, const char* tempDirectoryPath)
267 {
268  // Because nn::tm::SaveRecentVideo() sometimes fails at nn::tm::Error_InternalError,
269  // CloseTargetVideo() is called if SaveRecentVideo() fails, and the process is retried from OpenTargetVideo().
270  bool saveSuccess = false;
271  for (int i = 0; i < RetryCount; i++)
272  {
273  if (!OpenTargetVideo(target))
274  {
275  return false;
276  }
277  Sleep(5000);
278  if (SaveRecentVideo(target, tempDirectoryPath))
279  {
280  saveSuccess = true;
281  break;
282  }
283  if (!CloseTargetVideo(target))
284  {
285  return false;
286  }
287  }
288 
289  if (!saveSuccess)
290  {
291  return false;
292  }
293 
294  if (!GetScreenImageSizeAndData(target))
295  {
296  return false;
297  }
298  if (!CloseTargetVideo(target))
299  {
300  return false;
301  }
302  return true;
303 }

Sample Program Description

This sample program has the following flow.

  • Starts an application on the target side.
  • Registers VideoUpdateEvent.
  • Starts recording.
  • Ends recording.
  • Opens the video stream.
  • The recorded content is exported to TEMP%/Nintendo/.
  • Gets the screen image data and its size.
  • Closes the video stream.
  • Clears the VideoUpdateEvent registration.
  • Closes the application that is running on the target.
