Detailed Description

Provided more information about playback parameters that are passed when a sound is played.

Details
They are passed to functions like SoundStartable::StartSound. Even if this structure is not used, sounds can be played back.

You can start playback from partway through a sound by setting values for startOffsetType and startOffset. You must set both of these parameters at the same time. To enable the configured parameter, set the enableFlag bit flag to EnableFlagBit_StartOffset.

Even for the same offset value, the processing time required to begin playback from part way through a sound may vary, depending on the type of data. Note that in some cases this processing will take a long time.

Sequence Sound

When you idle until part way through a sequence sound, the larger the size of the start offset, the longer the processing takes.

Stream Sound

If the data is PCM, there is almost no processing time involved in playback from midstream, regardless of the size of the start offset. But if the data is ADPCM, there are as many as 14,336 samples per channel, and it takes time for the CPU to decode the data. With ADPCM, the starting position for playback is changed to the 14-sample boundary immediately preceding the specified position.

Wave Sound

If the data is PCM, there is almost no processing time involved in playback from midstream, regardless of the size of the start offset. If the data is ADPCM, the amount of processing time corresponds to the size of the starting offset because all samples from the start of the data through the playback starting position must be decoded by the CPU. With ADPCM, the starting position for playback is changed to the 14-sample boundary immediately preceding the specified position.

Sounds are normally played using the player with the ID set in the sound archive. If you want to specify a different player for playback, specify it in playerId. Sets the playerId. To enable the configured parameter, set the enableFlag bit flag to EnableFlagBit_PlayerId.

Sounds are normally played using the player priority set in the sound archive. If you want to specify a different priority for playback, set it in playerPriority. Sets the playerPriority. To enable the configured parameter, set the enableFlag bit flag to EnableFlagBit_PlayerPriority.

actorPlayerId is the number of the actor player to use when sound is played using the SoundActor class. It takes a value in the range from 0 to 3. To enable the configured parameter, set the enableFlag bit flag to EnableFlagBit_ActorPlayerId.

sequenceSoundInfo contains the arguments relating to sequence sounds. It is only enabled when sequence sounds are playing. You can overwrite the sequence sound information that is set within a sound archive. For more information, see SequenceSoundInfo. To enable the configured parameter, set the enableFlag bit flag to EnableFlagBit_SequenceSoundInfo.

◆ delayCount

Although this works, we recommend using delayTime for playback. (Note that this is disabled when EnableFlagBit_DelayTime is specified.) Because the amount of delay depends on the audio frame update interval and frequency during run time, calculation must occur in the application.
