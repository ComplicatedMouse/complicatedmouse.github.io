Overview

This sample program shows how to define a proprietary nninitStartup() function. For more information, see nn::init::Startup().

File Structure

This sample program is located in Samples/Sources/Applications/InitStartup/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

This sample program has the following flow.

  • Allocate memory heap in nninitStartup(), and implement memory settings for malloc().
  • Place a StaticObjectClass class as a static object, and issue malloc() using the constructor.
  • Output the memory address obtained by this malloc() in nnMain() to the log, and then end.

The important point is the order of starting nninitStartup(), the static object constructor, and nnMain().

nninitStartup() is called first, initializing the memory heap. Next, nn::init::InitializeAllocator() is issued. It sets the memory space used by the default SDK malloc() and new. For the features that can be used in nninitStartup(), see List of available APIs from nninitStartup().

Next, the static object constructor starts. A StaticObjectClass instance is globally placed in this sample. This indicates that malloc() can be called in the constructor.

nnMain() starts last. Output the address obtained from malloc() in the constructor to the log, and then end.

The following log is output.

step1: A nninitStartup() function is invoked.

step2: A constructor of StaticObjectClass is invoked.
step2: Because nn::init::InitializeAllocator() has been already called,
step2: we can use malloc() here.

step3: nnMain() function is invoked.
step3: A malloc() address in StaticObjectClass: address = 0x80007000
step3: nnMain() function is finished.
NX Platform Only:
This sample uses the SDK default malloc(). If you intend to use a custom malloc() definition, see nninitStartupWithMalloc.

The SDK's default nninitStartup() function is called if nninitStartup() is not defined. In the default nninitStartup() function, the allocator is initialized using nn::init::InitializeAllocator() to allow the use of malloc and other functions earlier in the development process. For more information, see the API Reference.
