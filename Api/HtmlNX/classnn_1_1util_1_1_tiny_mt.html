Detailed Description

The class for generating pseudo-random numbers using TinyMT.

Details
TinyMT is a small variant of the Mersenne Twister algorithm for generating pseudo-random numbers. It generates pseudo-random numbers with lower quality than MT, but with good quality nevertheless. Mersenne Twister needs an internal state of 2.5 KB, but TinyMT only needs an internal state of 16 bytes.

In this class, the parameters (mat1, mat2, and tmat) are hard-coded to 0x8f7011ee, 0xfc78ff1f, and 0x3793fdff respectively.

The period of the random number generator is 2^127 – 1. Ordinarily you will not cycle through the sequence of random numbers.

void nn::util::TinyMt::SaveState ( TinyMt::State *  pStateBuffer)

Saves the internal state.

Parameters
[in]pStateBufferThe buffer where the internal state will be saved.

Details
Saves the internal state to the specified buffer. The saved internal state can be restored by using RestoreState.

By saving the values obtained with this function, you can restart random number generation later. In addition, this can be used to reproduce the same random sequence.

◆ GenerateRandomN()

uint32_t nn::util::TinyMt::GenerateRandomN ( uint16_t  num)

Generates a random number that is less than the specified number.

Parameters
[in]numThe type of value to return.
Returns
Returns a random integer in the specified range.

Details
Generates a random number in the range of integers from 0 to less than num. The return value type is the type of num.

Note that the probability of returning each value is not completely equal. For example, when specifying 6 for num, the probability that 0, 1, 3, or 4 is returned is 0x2AAAAAAB / 0x100000000, but the probability that 2 or 5 is returned is 0x2AAAAAAA / 0x100000000; the probability that 2 or 5 appears is 0x2AAAAAAA / 0x2AAAAAAB times the probability that 0, 1, 3, or 4 appears.

The bias in the probability increases with a larger num; when specifying the maximum value 0xFFFF for num, the probability that 0 appears is 65538 / 65537 times (1.0000153 times) the probability that a nonzero number appears.

This bias does not occur when num is a power of 2.
