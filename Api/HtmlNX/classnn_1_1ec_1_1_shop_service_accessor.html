Detailed Description

The shop service accessor class.

Details
This class explicitly specifies and accesses the connected shop service server. It implements the following procedure to access a shop service server.

  • The Initialize(const ShopService&) is used to specify the shop service server to connect to.
  • The Request() function is used to request communication with the target shop service server.

The Request() function accepts the requests asynchronously. These asynchronous requests are implemented in the order in which they are added to the queue. The AsyncResponse class is used to check for completion of the accepted asynchronous requests and as a handle to control interruptions.

The ceiling limit on the issuance of asynchronous requests depends on the internally shared resources (service resources). nn::ec::ResultShopServiceAccessOverRequest is returned if a request fails because of the ceiling limit.

Below are some basic guidelines for the use of service resources.

  • If only one instance of the ShopServiceAccessor class is being used, 16 requests can be registered.
  • If two or more instances of the ShopServiceAccessor class are being used, fewer than 16 requests can be registered, depending on the state of the service resources.
Reference
The following code presents an example of the procedure for using this class to process requests to the shop service server API (in this case, a HTTP GET method request to "/v1/applications/{applicationId}/consumables").
NN_ALIGNAS(nn::os::MemoryPageSize) Bit8 g_WorkMemory[nn::os::MemoryPageSize * 16];
void main()
{
const account::Uid user = GetUser();
nn::ec::InitializeForShopServiceAccessors(g_WorkMemory, sizeof(g_WorkMemory));
{
nn::ec::ShopServiceAccessor accessor;
accessor.Initialize(nn::ec::ShopService(nn::ec::ShopService::Type_Catalog);
nn::ec::ShopServiceAccessor::AsyncResponse response;
accessor.Request(&response, user, nn::ec::ShopService::Method_Get, "/v1/applications/{applicationId}/consumables? country={country}&lang=ja");
size_t responseSize;
response.GetSize(&responseSize);
std::unique_ptr<char[]> responseBuffer(new char[responseSize]);
response.Get(responseBuffer.get(), responseSize);
}
nn::ec::FinalizeForShopServiceAccessors();
}

Requests the implementation of a communication request (a timeout-specifiable interface, with post data).

Parameters
[out]pOutHandleA pointer to the interface for receiving the asynchronous request handle (AsyncResponse).
[in]userIdThe identifier for the user that wants shop service access.
[in]methodThe HTTP method to request.
[in]pPathThe URL path string. (A null-terminated string.)
[in]postDataThe data container for the post data field. Stream encoding is not performed automatically on the specified data string. Set to conform with the post data field encoding, requested of the connected shop service server.
[in]timeoutThe communication timeout time, in seconds. If a value of less than 1 second is specified, the default value is used (DefaultTimeout). The maximum number of seconds that can be specified is INT_MAX. If a larger value is specified, it is rounded down to INT_MAX.
Returns
Returns the execution result. Report Result values that cannot be handled by the application to the error viewer.
Return Values:

nn::ResultSuccess The process succeeded.

nn::ec::ResultShopServiceAccessInvalidCharacter The request parameters include an invalid character.

nn::ec::ResultShopServiceAccessOverRequest Service resources necessary for the asynchronous request could not be allocated. If there are any issued asynchronous request handles (AsyncResponse) that have not been destroyed, destroy them and make the request again. Alternately, reduce the number of ShopServiceAccessor instances being used at the same time.

nn::ec::ResultShopServiceAccessInsufficientWorkMemory The work memory (the work buffer) for the asynchronous request is not large enough.

nn::ec::ResultInternetRequestNotAccepted An Internet connection request has not been submitted and accepted.

Precondition
  • Initialization by Initialize(const ShopService&) was successful.
  • The request parameters expected for variableParameters are prepared.
  • pOutHandle != nullptr.
  • userId == true.
  • nullptr != pPath.
  • '/' == pPath[0].

Details
You can get the request response by using AsyncResponse::Get().
