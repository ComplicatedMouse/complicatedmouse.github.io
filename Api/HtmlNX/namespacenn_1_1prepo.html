Detailed Description

Namespace for the play report library.

Details
The play report feature allows any type of report data to be saved on the system in the form of event logs (key-value pairs). For example, to create a heat map for an action game, the following report could be created.

// Record that an enemy has been killed.
nn::prepo::PlayReport report("killed_event");
report.SetBuffer(buffer, sizeof (buffer));
report.Add("map", map.id);
report.Add("position_x", me.position.x);
report.Add("position_y", me.position.y);
report.Add("position_z", me.position.z);
report.Add("enemy_id", enemy.id);
report.Add("enemy_position_x", enemy.position.x);
report.Add("enemy_position_y", enemy.position.y);
report.Add("enemy_position_z", enemy.position.z);
report.Save();

// Record that the character has fallen.
nn::prepo::PlayReport report("fallen_event");
report.SetBuffer(buffer, sizeof (buffer));
report.Add("map", map.id);
report.Add("position_x", me.position.x);
report.Add("position_y", me.position.y);
report.Add("position_z", me.position.z);
report.Save();

The keys and values added with Add() are copied to the buffer set by SetBuffer(). For this reason, you do not need to keep the keys and values that have already been added in memory.

Report data is typically sent automatically when Internet communication is possible. However, report data is not automatically sent during peer-to-peer communications, during streaming video playback, or during any other processes that have prescribed bandwidth requirements.

Use RequestImmediateTransmission to send report data at a specific time.

Report data is deleted from system storage after being successfully sent.

If report data continues to accumulate due to the system being in an environment in which Internet communications are not possible, the system storage may eventually run out of free space. When this happens, the system begins deleting old report data to make space for new report data. As a result, saved report data is not necessarily sent to the server.

Up to KeyValueCountMax key-value pairs can be saved per event. At least one key-value pair must be saved for every event.

The maximum total size of key values for one event is ReportBufferSizeMax bytes.

Event IDs, keys, and values specified by the application are subject to the following restrictions.

  • Event IDs
    • Characters allowed: [0-9a-z_]
    • No more than EventIdLengthMax bytes long
  • Keys
    • Characters allowed: [0-9a-zA-Z_]
    • No more than KeyLengthMax bytes long
    • The first character cannot be a number.
  • Values
    • Signed 64-bit integer
    • Any 64-bit ID
    • Double-precision floating-point number
      • A value other than NaN, +INF, or -INF
    • UTF-8 string
    • Boolean value
    • Binary
    • Array
    • Structure
Windows Platform Only:
In the Windows environment, only emulation for creating and saving report data processing is supported. It does not send data to the play report compilation server.

◆ RequestImmediateTransmission()

nn::Result nn::prepo::RequestImmediateTransmission ( )

Requests immediate transmission of any report data saved on the system.

Returns
The process result.

Details
Requests immediate transmission of any report data saved on the system at the time this function is called. Report data recorded after this function is called will not be included in the data that is immediately transmitted.

Attention
To avoid excess server load, this function is limited to 10 calls in 10 minutes. This function returns nn:ResultSuccess when the limit is exceeded. However, no request for immediate transmission is made, and a warning log is generated.

Parameters

  • nn::prepo::ResultInvalidKey Invalid key string.
  • nn::prepo::ResultDuplicatedKey The specified key is already registered.
  • nn::prepo::ResultKeyValueCountLimitReached The number of key-value pairs has reached the limit.
  • nn::prepo::ResultOutOfResource Insufficient free space in the buffer.
  • nn::prepo::ResultTooBigReportData The report data is too large.
Precondition
  • outReport != nullptr.
  • key != nullptr.

Details
The value being added is type Any64BitId.

By including this ID in the report, you can distinguish the interruptions when analyzing the application. The ID bit width is not large enough to avoid duplications across all devices, so make sure that you keep your summaries within the same device_id or nsa_id.

As an example, say you are saving the stage_clear event every time the user clears a stage, as shown in the following code. The key for the ID would be session_id.

nn::prepo::PlayReport report("stage_clear");
nn::Bit8 buffer[nn::prepo::PlayReport::BufferSizeMin + 4 * nn::prepo::PrimitiveKeyValueSizeMax];
report.SetBuffer(buffer, sizeof (buffer));
NN_ABORT_UNLESS_RESULT_SUCCESS(report.Add("stage_id", stageId));
NN_ABORT_UNLESS_RESULT_SUCCESS(report.Add("time", time));
NN_ABORT_UNLESS_RESULT_SUCCESS(report.Add("score", score));
NN_ABORT_UNLESS_RESULT_SUCCESS(nn::prepo::AddSessionId(&report, "session_id"));
report.Save(user);

FROM
(SELECT

◆ AddInternetConnectionStatus()

nn::Result nn::prepo::AddInternetConnectionStatus ( nn::prepo::PlayReport *  outReport,
const char *  networkInterfaceTypeKey,
const char *  linkLevelKey,
const char *  frequencyBandKey 
)

Adds key-value pairs to the report with information about the current Internet connection.

Parameters
[out]outReportThe report to which to add the key-value pairs with information about the current Internet connection.
[in]networkInterfaceTypeKeyThe key for the values for the network interface type.
[in]linkLevelKeyThe key for the values for connection quality with the access point.
[in]frequencyBandKeyThe key for the values for the frequency band used for the wireless connection.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess Succeeded.
  • nn::prepo::ResultInvalidKey Invalid key string.
  • nn::prepo::ResultDuplicatedKey The specified key is already registered.
  • nn::prepo::ResultKeyValueCountLimitReached The number of key-value pairs has reached the limit.
  • nn::prepo::ResultOutOfResource Insufficient free space in the buffer.
  • nn::prepo::ResultTooBigReportData The report data is too large.
Precondition
  • outReport != nullptr.
  • networkInterfaceTypeKey != nullptr || linkLevelKey != nullptr || frequencyBandKey != nullptr.

Details
If nullptr is specified for a key, that key-value pair will not be added to the report.

The values store the information at the instant that nn::prepo::PlayReport::Save() is called.

If the console is connected to the Internet, the status is considered to be connected even if the application has not requested use of the network.

Network Interface Type

Values are string type values.

Values Description
"None" No network interface is being used.
"IEEE 802.11" IEEE 802.11 wireless connection.
"Ethernet" Ethernet wired connection.

Connection Quality With Access Point

Values are integer type values.

Values Description
0 Represents a state of extremely poor connection quality, or no connection. This value also will be set for the key if the console is not connected to the Internet or if it is connected with a wired connection.
1 Represents a state where connectivity is bad.
2 Represents a state where connectivity is not good.
3 Represents a state where connectivity is good.

Frequency Band Used for Wireless Connection

Values are string type values.

Values Description
"None" Not connected to the Internet, or connected with a wired connection.
"2.4 GHz" Wireless connection using the 2.4 GHz band.
"5 GHz" Wireless connection using the 5 GHz band.
