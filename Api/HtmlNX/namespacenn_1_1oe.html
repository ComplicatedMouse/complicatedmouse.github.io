Detailed Description

The namespace for the library for operations in the application's operating environment.

Details

Feature Overview

Operating environment is abbreviated as oe. The oe library provides features related to the program's operating environment, such as controlling changes in program state and modifying performance configuration.

◆ FocusState

enum nn::oe::FocusState

Enumerated type that indicates the application focus state type.

Details
Enumerated type that indicates the application focus state type.

The following are the application focus state types. To get the current focus state, use nn::oe::GetCurrentFocusState().

Focus State Type Enum Representing State
In-focus state. nn::oe::FocusState_InFocus
Out-of-focus state. nn::oe::FocusState_OutOfFocus
Background-focus state. nn::oe::FocusState_Background

Where there will be an actual notification will vary, based on the nn::oe::SetFocusHandlingMode() settings.

The meaning of each state is as follows.

In-focus state.

This state indicates that the program is a target for direct user operations from the HOME Menu, directly after starting the program, or in other cases.

In this state, processes such as imaging, rendering, audio playback, and controller playback are needed.

Out-of-focus state,

In this state, the library applet was started from the in-focus state.

Programs can perform processing that is similar to that of being in an in-focus state. In that state, however, the library applet will appear on screen and controller input will not be accepted. Volume settings could also be restricted for audio playback, depending on the library applet.

These restrictions will be automatically lifted when you return to the in-focus state.

Program execution and audio playback will automatically be temporarily interrupted when out of focus, depending on the nn::oe::SetFocusHandlingMode() settings.

Background-focus state.

In this state, the HOME Menu is in the foreground.

Programs can perform processing that is nearly identical to that of being in an in-focus state. In that state, however, the HOME menu will appear on screen and controller input will not be accepted. Volume settings will be at zero (0) for audio playback.

These restrictions will be automatically lifted when you return to the in-focus state.

The running of the program and audio playback, will automatically be temporarily interrupted, depending on the nn::oe::SetFocusHandlingMode() settings.

By default, the application logo is automatically removed after control has moved to the application program. ◆ SetCpuBoostMode()

void nn::oe::SetCpuBoostMode ( CpuBoostMode  mode)

Sets the CPU boost mode.

Parameters
[in]modeCPU boost mode types.

Details
Changes the CPU boost mode to the boost mode specified by mode.

If mode is set to any value other than CpuBoostMode_Normal, the CPU boost feature is enabled and a system request is made to change the processing power of the CPU and GPU. Make sure that you restore the mode to CpuBoostMode_Normal after the boosted segment is over.

The effect of this function is controlled by the system according to the state of focus of the program, so the program does not need to call the function according to changes in its focus state. CPU processing power does not necessarily always remain boosted when the CPU boost feature is enabled. If the console overheats or safety measures otherwise need to be taken, the system might automatically switch the CPU back to its normal processing power.

The operations of the various CPU boost modes are as follows. For more information, see CPU Boost Functionality.

oe::CpuBoostMode_Normal Is Applied for the Mode

  • This state is the same as the default state of the program immediately after it starts.
  • The CPU, GPU, and memory clock rates for the performance specified by oe::SetPerformanceConfiguration() are applied.

oe::CpuBoostMode_FastLoad Is Applied for the Mode

  • The highest CPU processing power possible based on the internal state of the system is assigned automatically, regardless of the value set by oe::SetPerformanceConfiguration().
  • The GPU processing power is around 20% of the power when oe::PerformanceConfiguration_Cpu1020MhzGpu307MhzEmc1331Mhz is set.
  • The memory (EMC) clock rate is as follows.
    • When the performance mode is normal mode: 1331 MHz.
    • When the performance mode is boost mode: 1600 MHz.

Note

The specification for CPU boost functionality might change in the future. Check the revision history when the SDK version is updated.

Examples
G3dDemo/Town.cpp.

This function is intended to be used in a scene where the user is expected to go into an idle state for a while, such as during the ending credits.

If extension is enabled by this function, the sleep and burn-in reduction features will be determined based on the standard system settings or operating mode. The start time policies for auto-sleep and screen burn-in reduction will be extended according to the following descriptions.

  • When the starting condition for screen burn-in reduction is less than 15 minutes of idle state time, extend the starting condition to 15 minutes.

System Settings Name Settings
Auto-sleep when playing on the system screen 1 minute.
Auto-sleep when playing on the TV 1 hour.
Turn off auto-sleep when playing back content On

The operating mode. When to start screen burn-in reduction When to start auto-sleep
While video is displayed on the console 45 seconds 1 minute.
While video is displayed on the TV 5 minutes. 1 hour.

The operating mode. When to start screen burn-in reduction When to start auto-sleep
While video is displayed on the console 15 minutes 30 minutes.
While video is displayed on the TV 15 minutes 1 hour.
