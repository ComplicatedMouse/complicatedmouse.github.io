Detailed Description

A handle class that controls playback sounds.

Details

Overview

To associate a sound handle with a sound, call SoundArchivePlayer::PrepareSound or SoundArchivePlayer::StartSound.

If the handle is already associated with another sound, this association is removed, and it is newly associated with the sound to play back.

The association between the handle and the sound is removed, and that handle can no longer control that sound in the following situations. 

   - The sound has reached its end and has stopped.
   - <tt>@ref Stop</tt> is called with fade frame 0.
   - <tt>@ref Stop</tt> is called with <tt>1</tt> or greater specified for the fade frame, and the

time for that number of frames has passed and the sound has stopped.

  • SoundHandle::DetachSound has been explicitly called.
  • Sound is stopped with the player priority decision.

As a result, you can check whether a sound has stopped after being played to the end by monitoring IsAttachedSound for each frame. However, you must consider the effect of sounds that were stopped by calls to Stop, explicit calls to DetachSound, or determinations made about player priorities.

For more information about making determinations about player priorities, see Priority Operations in the Sound System documentation.

Types of Sound Handles

There are three types of sounds: sequence sounds, stream sounds, and wave sounds. The SoundHandle class can be used to handle operations that are common to all these sounds.

The SequenceSoundHandle, StreamSoundHandle, and WaveSoundHandle classes provide operations for specific sound types that cannot be handled by the SoundHandle class.

Parameters by Output Line

You can set volume, pitch, pan, and other parameters in the sound handle, but they are also affected by the following functions.

Overall

  • Volume: SetVolume, FadeIn, Pause, Stop
  • Pan: SetPan.
  • Surround pan: SetSurroundPan.
  • Main send: SetMainSend.
  • Effect send: SetEffectSend.
  • Pitch: SetPitch.
  • LPF: SetLowPassFilterFrequency.
  • Biquad filter: SetBiquadFilter.
  • Output Lines: SetOutputLine, ResetOutputLine

TV output

  • Volume: SetOutputVolume, in addition to overall volume.
  • Pan: SetOutputPan, in addition to overall pan.
  • Surround pan: SetOutputSurroundPan, in addition to overall pan.
  • Main send: SetOutputMainSend, in addition to overall main send.
  • Effect send: SetOutputEffectSend, in addition to the overall effect send.

In addition to these effects, these parameters can be affected by the following.

  • Effects of data specified by SoundMaker
  • Effects of SoundPlayer or SoundActor
  • Effects of 3D sound calculations (Sound3DEngine))

◆ Stop()

void nn::atk::SoundHandle::Stop ( int  fadeFrames)

Stops a sound.

Details
Stops the sound associated with the handle. If the handle is invalid, the function does nothing.

You can fade out over the number of frames specified by fadeFrames. Fade-out is not performed when 0 is specified. However, when a sequence sound is being played, the envelope release is played and all attenuation is completed before the sound is stopped.

The volume control for fade-out is also used for fade-in. Number of elapsed frames during a fade out. The number of fade-out frames correlates to the speed of change from maximum volume to silence. If a fade out is specified during a fade in, or in other similar situations, it may complete sooner than the specified frame count.

Parameters
[in]fadeFramesNumber of elapsed frames during a fade-out. The number of frames is based on the number of calls to SoundArchivePlayer::Update. Specify a value of 0 or greater for the fadeFrames parameter. If you specify a negative value, the function works the same as if you had specified 0.

◆ Pause() [1/2]

void nn::atk::SoundHandle::Pause ( bool  flag,
int  fadeFrames 
)

Pauses or restarts a sound.

Details
Pauses or restarts the sound associated with the handle. If the handle is invalid, the function does nothing.

You can pause the sound while fading out, or restart it while fading in, for the number of frames specified by fadeFrames.

Fading when pausing and resuming sounds operates independently from the fade-in when resuming a sound and the fade-out when pausing a sound.

If 0 is specified for fadeFrames, and the sound is paused, the volume is set to zero without a fade-out after the first SoundArchivePlayer::Update, and after the second SoundArchivePlayer::Update, the sound is paused. Note that the playback time advances in the silent state during the interval between the first and second calls to SoundArchivePlayer::Update. However, you do not need to track this issue because playback resumes after the first SoundArchivePlayer::Update without fade-in.

In addition, the number of frames that elapse during fading correlates to the speed of change from the maximum volume to silence, or from silence to the maximum volume. A fade operation can complete in less than the specified number of frames, for example, if a fade-out is started in the middle of a fade-in.

For more information about pause states, see GetPauseState and PauseState.

Parameters
[in]flagPlayback pauses if this parameter is true, and restarts if it is false.
[in]fadeFramesNumber of frames used to fade in and out. The number of frames is based on the number of calls to SoundArchivePlayer::Update. If you specify a negative value, the function works the same as if you had specified 0.

Mutes or unmutes the sound.

Details
Mutes or unmutes the sound associated with the handle. If the handle is invalid, the function does nothing.

You can mute the sound while fading out, or unmute it while fading in, for the number of frames specified by fadeFrames. Specify 0 to mute or unmute the sound immediately.

The fade in and fade out operations for muting and unmuting are independent from the fade-in and fade-out operations for playback start, playback stop, pausing, and resuming audio. The number of frames that elapse during fading correlates to the speed of change from the maximum volume to silence, and vice versa. A fade operation may complete in less than the specified number of frames if, for example, it is started in the middle of another fade.

For more information about mute states, see GetMuteState and MuteState.

◆ SetVolume()

void nn::atk::SoundHandle::SetVolume ( float  volume,
int  frames = 0 
)

Changes the volume of the sound.

Details
Changes the volume of the sound associated with the handle. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other volume parameters, and they are all layered.

The volume parameter takes a ratio of 0.0 or greater. Volume does not change if 1.0 is specified. If 0.0 is specified, there is no sound. The default value is 1.0.

Note that after this value is combined with the other volume parameters, the final volume is clamped to the range from nn::audio::VoiceType::GetVolumeMin() to nn::audio::VoiceType::GetVolumeMax(). For more information about combining volume parameters, see Sound Parameter Control Operations.

The volume changes over the number of frames specified by frames. If this function is called during a volume change, the volume changes over the number of newly specified frames, using the volume value at that point as the base point.

If you want to set each output line separately, use the SetOutputVolume function.

Parameters
[in]volumeThe volume change ratio (0.0 or greater).
[in]framesThe number of frames that elapse when the volume changes. The number of frames is based on the number of calls to SoundArchivePlayer::Update. Specify a value of 0 or greater for the frames parameter. If you specify a negative value, the function works the same as if you had specified 0.
Precondition
volume >= 0.0f.

◆ SetPan()

void nn::atk::SoundHandle::SetPan ( float  pan)

Changes the pan (right/left location) of the sound.

Details
Changes the pan of the sound associated with the handle. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other pan parameters, and they are all layered.

pan specifies a value that represents the relative change in the location of the sound. If the waveform that is input is a monaural waveform, pan works as follows.

  • If 0.0 is specified, the pan is not changed from the value set in the data.
  • If 1.0 is specified, the sound positioned in the center becomes the sound positioned on the right end.
  • If 1.0 is specified, the sound positioned in the center becomes the sound positioned on the left end.

If the waveform that is input is a stereo waveform and the pan mode is set to Dual Pan, pan works as follows.

  • If 0.0 is specified, the left channel waveform is positioned at the left end, and the right channel waveform is positioned at the right end.
  • If 2.0 is specified, the waveforms of both channels are positioned at the right end.
  • If 2.0 is specified, the waveforms of both channels are positioned at the left end.
  • If 1.0 or 1.0 is specified, one or the other of the waveforms is positioned in the center.

If the pan mode is set to Balance Pan, pan works as follows.

  • If 1.0 is specified, you can only hear the right-channel waveform.
  • If 1.0 is specified, you can only hear the left-channel waveform.

The default value is 0.0.

If MixMode_MixVolume has been specified by SetMixMode, this value is ignored even if it has been specified.

If you want to set each output line separately, use the SetOutputPan function.

Parameters
[in]panRelative change in the pan, with 0.0 as the base value.

◆ SetSurroundPan()

void nn::atk::SoundHandle::SetSurroundPan ( float  surroundPan)

Changes the surround pan (front-to-back location) of the sound.

Details
Changes the surround pan of the sound associated with the handle. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other surround parameters, and they are all layered.

surroundPan specifies a value that represents the relative change in the location of the sound. If 0.0 is specified, the pan value that is set within data is unchanged. If 1.0 is specified, the sound positioned at the front becomes the sound positioned in the center. If 2.0 is specified, the sound positioned at the front becomes the sound positioned at the back. Specify a negative number if you want to move the location toward the front.

The default value is 0.0.

If MixMode_MixVolume has been specified by SetMixMode, this value is ignored even if it has been specified.

The values set for surround pan are only applied when the output mode is surround. For more information, see nn::atk::SoundSystem::SetOutputMode.

Parameters
[in]surroundPanRelative change in the surround pan, taking 0.0 as a reference point.

◆ SetMixMode()

void nn::atk::SoundHandle::SetMixMode ( MixMode  mixMode)

Changes how to set how the volume for the sound is distributed among the speakers.

Details
Changes how to set how the volume is distributed for the sound associated with the handle. If the handle is invalid, the function does nothing.

For mixMode, specify MixMode_Pan or MixMode_MixVolume. If MixMode_Pan is specified, the volume is distributed by setting values for the pan or the surround pan. When this is done, any volume value that has been set by SetMixVolume will be ignored.

If MixMode_MixVolume has been specified, you must set a volume for each speaker for 5.1 channel audio. When this is done, any values set for pan or surround pan will be ignored.

◆ SetMainSend()

void nn::atk::SoundHandle::SetMainSend ( float  send)

Changes the main send volume of the sound.

Details
Changes the main send volume for the sound associated with the handle. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other main send parameters, and they are all layered.

The main send is a parameter that is used to adjust the volume of sound to send to output after an effect is sent. This parameter is mostly used to adjust the balance of the dry and wet components of an effect.

The send parameter specifies a relative change in the send volume, and the ultimate send volume is the sum of the values specified by the various functions and SoundMaker, clamped in the range from 0.0 to 1.0. The values configurable in SoundMaker are interpreted in the runtime as values in the range from 0.0 to 1.0.

In other words, a specification of 0.0 for the send volume represents a setting that does not change the send volume specified by the various functions and SoundMaker. A specification of 1.0 for the send volume represents a setting that adds to the send volume so the maximum volume is sent to the bus, regardless of the value set by SoundMaker. A specification of 1.0 for the send volume represents a setting that subtracts from the send volume so that no send volume is sent to the bus, regardless of the value set by SoundMaker.

The default value is 0.0.

If you want to set each output line separately, use the SetOutputMainSend function.

Parameters
[in]sendRelative change in the send volume, taking 0.0 as a reference point.
See also
SetEffectSend
SetOutputMainSend

Details
Changes the pitch of the sound associated with the handle. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other pitch parameters, and they are all layered.

Details

The biquad filter parameter can be configured in multiple locations, but the settings are not layered. Rather, the biquad filter takes the value that is set in the location with the highest priority, as follows. If the biquad filter parameter is set in a higher-priority location, the settings in locations of lower priority are overwritten. 

   -# The setting in the sound handle

   -# The setting in the ambient parameter structure
   -# The setting in the sequence data

If an out-of-range value is set, processing stops on an assertion failure in the Debug and Development versions. It is ignored in the Release version.

value is the extent to which the filter is applied, specified in the range from 0.0f to 1.0f. The meaning of this value varies depending on the type of coefficient used by the filter.

When used together with the low-pass filter from SetLowPassFilterFrequency, the effects are overlaid.
