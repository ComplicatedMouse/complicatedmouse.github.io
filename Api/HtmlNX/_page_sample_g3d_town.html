This documentation describes the Town sample program. This sample program is close in content to a real game.

Overview

This sample program is close in content to a real game, showing views in the distance, terrain, and a number of characters and other models.

File Structure

This sample program is located in Samples/Sources/Applications/G3dDemo/ .

Required Environment

Nothing specific.

How to Use

Use input from the Joy-Con or DebugPad, or the keyboard and mouse when using a PC, to perform the following operations.

Common Operations for All Modes

Input Operation
START Button, SPACEBAR, Joy-Con + Button: Returns to the menu.
Press the Right Stick or click the left mouse button: Camera rotation. When using the mouse, the camera rotates based on the direction and distance of the click from the center of the screen.
Press the Left Stick or click the right mouse button: Camera movement. When using the mouse, the camera moves based on the direction and distance of the click from the center of the screen.
ZR Button or F key: Moves the camera to the selected model.
Directional Buttons: Switches the menu selection state.
A Button, A key, or left-mouse click: Engages the selected menu option.
Y ButtonSelects the window bar of the menu window.
Long press of the Y Button and ZL Button or ZR Button:Switches the selection menu window.

Edit Mode

Item Operation
Model Settings: Configures the model settings.
Focus: Sets the focus on the selected model.
Attach: Attaches the model if 3DEditor is connected. No action is taken if 3DEditor is not connected.
Shape: Changes the selected shape in the selected model.
SubMesh: Changes the submesh index of the selection target containing the selected shape.
Anim Speed Changes the animation speed of the model being edited.
BoundingBox Draws the bounding box maintained for each submesh.
BoundingSphere Draws the bounding sphere of the model.
Draw Single Shape Draws only the selected shape.
Draw Single SubMesh Draws only the selected submesh.
Shader Settings Configures settings for attaching and detaching shaders.
tex_average If 3DEditor is connected, attaches the shader for calculating the average brightness of the screen. No action is taken if 3DEditor is not connected.
Model Settings: Configures the camera settings.
Position: Changes the camera position.
Target: Changes the camera look-at point.
Draw ViewVolume: Draws the camera view frustum.
View: Sets the camera type.
Freeze View Volume: Stops the camera view frustum from updating.
Open FrameBuffer Windows: Opens the framebuffer display window.

Debug Mode 1

Item Operation
Global Settings: Configures settings for the entire scene.
Model Animation Enables or disables model animation.
Light Animation Enables or disables light animation.
Wireframe Shows or hides the wireframe.
Ice Enables or disables the snow shader.
Visualize LOD: Enables or disables the visible shader for the LOD level of the rendered model.
Visualize Mip: Enables or disables the visible shader for the mipmap level of the rendered model textures.
Shadow Enables or disables shadow processing.
Water Enables or disables water surface processing.
Z-Prepass Enables and disables z pre-pass.

Overdraw Enables or disables overdraw measurement for the final render fragment shader.

Important Information

Nothing specific.

Execution Procedure

Build the sample program. From the menu, select Town, and then run it.

Description

This sample program has the following flow.

  • Initialize the render targets.
  • Load and initialize the shader archive.
  • Load and initialize the model resources, and bind the textures.
  • Bind shaders and dynamic textures to models.
  • Bind the model and animation.
  • Begin the loop.
  • Begin execution of the command list generated in the previous frame.
  • Perform calculations for each matrix.
  • Update models and shaders.
  • Generate commands for creating shadow maps.
  • Generate commands for rendering the water surface.
  • Generate commands for z pre-pass.
  • Generate commands for creating the geometry buffer (G-buffer).
  • Generate commands for light pre-pass.
  • Generate commands for the ultimate result of rendering (opaque).
  • Generate commands for the ultimate result of rendering (translucent).
  • Generate commands for a compute shader for calculating screen brightness.
  • Generate commands for displaying the overdraw results.
  • Wait for execution of the command list to complete.
  • Perform calculations that require waiting on the GPU.
  • Swap the scan buffer.
  • Return to the start of the loop.
