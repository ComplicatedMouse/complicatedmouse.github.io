Describes the Tessellation sample program. The sample program uses tessellation to change the number of mesh divisions corresponding to the distance from the camera.

Overview

The sample uses tessellation to change the number of mesh divisions corresponding to the distance from the camera.

File Structure

This sample program is located in Samples/Sources/Applications/G3dDemo/ .

Required Environment

Nothing specific.

How to Use

Use input from the Joy-Con or DebugPad, or the keyboard and mouse when using a PC, to perform the following operations.

Input Operation
START Button, SPACEBAR, Joy-Con + Button: Returns to the menu.
Press the Right Stick or click the left mouse button: Camera rotation. When using the mouse, the camera rotates based on the direction and distance of the click from the center of the screen.
Press the Left Stick or click the right mouse button: Camera movement. When using the mouse, the camera moves based on the direction and distance of the click from the center of the screen.
Directional Buttons: Switches the menu selection state.
A Button, A key, or left-mouse click: Engages the selected menu option.
Y ButtonSelects the window bar of the menu window.

Select each item to do the following operations.

Item Operation
Up and down on the +Control Pad, or the up and down arrow keys Switches the selected item for editing.
Left and right on the +Control Pad, or the left and right arrow keys Changes the value of the item selected for editing.
Wireframe Enables or disables wireframe drawing.
MeshLod Enables or disables mesh LOD drawing.
TessLevel Switches the vertex division level using tessellation.

Important Information

Nothing specific.

Execution Procedure

Build the sample program, select Tessellation from the menu, and then run it.

Description

This sample program uses tessellation to perform the following.

  • Builds a mesh terrain based on a height map.
  • Mesh LOD corresponding to the camera and target distance.

This sample program has the following flow.

  • Load and initialize the shader archive.
  • Load and initialize the model resource, and initialize the texture.
  • Register a sampler in the descriptor pool.
  • Use nns::g3d to set up models.
  • Begin the loop.
  • Execute the command list created in the previous frame.
  • Update each matrix.
  • Update models.
  • Generate the command list.
  • Return to the start of the loop.
