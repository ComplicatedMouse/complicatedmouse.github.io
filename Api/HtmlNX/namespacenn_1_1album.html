◆ AlbumReportOption

enum nn::album::AlbumReportOption

The settings for automatic notifications of the results of album operations.

Details
The settings for whether to notify automatically about calls to functions that operate on album files. The notifications to the user are the same as the values returned by the functions.

Enumerator
AlbumReportOption_NoReport 

Do not automatically notify the user about the results of operations on albums.

AlbumReportOption_ReportOnSuccess 

Automatically notify the user about successful operations on albums.

AlbumReportOption_ReportOnFailure 

Automatically notify the user about failed operations on albums.

AlbumReportOption_ReportAlways 

Always automatically notify the user about the results of operations on albums.

Enumerator

Indicates that orientation of the image is the same as the orientation of the display.

Indicates that the image is rotated 90 degrees. The right side of the display matched the up direction of the camera when the image was captured.

ImageOrientation_Rotate180 

Indicates that the image is rotated 180 degrees. The bottom of the display matched the up direction of the camera when the image was captured.

ImageOrientation_Rotate270 

Indicates that the image is rotated 270 degrees. The left side of the display matched the up direction of the camera when the image was captured.

◆ SaveScreenshot() [2/2]

nn::Result nn::album::SaveScreenshot ( const void *  pImageData,
size_t  imageDataSize,
ImageSize  screenshotSize,
ImageOrientation  imageOrientation,
AlbumReportOption  reportOption 
)

Saves image data with a specified orientation as a screenshot in the album.

Parameters
[in]pImageDataThe image data to save as a screenshot.
[in]imageDataSizeThe size of the image data, in bytes.
[in]screenshotSizeThe size of the screenshot to save.
[in]imageOrientationThe orientation of the image.
[in]reportOptionThe setting for automatically notifying the user about the save result.
Precondition
The album library is available to use.
pImageData != nullptr.
imageDataSize >= 4 × GetImageWidth(screenshotSize) × GetImageHeight(screenshotSize).
Return Values:

ResultAlbumFull Saving failed because the album does not have enough space.

ResultAlbumFileCountLimit Saving failed because the limit has been reached for the number of files that can be saved in the album.

ResultAlbumError Saving failed due to another error.

Details
Saves the image data as a screenshot in the album.

For pImageData, pass the pointer to the start of the image data to save. The format of the passed image is RGBA format, with 8 bits per channel expressed in sRGB color space. The image data has an image format specified as nn::gfx::ImageFormat_R8_G8_B8_A8_UnormSrgb, a tile mode of nn::gfx::TileMode_Linear, an origin at the upper left corner, and no padding (stride == 4 × GetImageWidth(screenshotSize).

Memory from the GPU's memory pool with caching enabled can be passed to pImageData.

For screenshotSize, specify the size of the screenshot to save. This function does not scale the image data, so the image size represented by pImageData must match the size of the screenshot.

For imageOrientation, use a nn::album:: ImageOrientation enumerated type value to specify the orientation (the rotational direction) of the image.

◆ SaveCurrentScreenshot()

void nn::album::SaveCurrentScreenshot ( AlbumReportOption  reportOption)

Saves the currently displayed image as a screenshot in an album.

Parameters
[in]reportOptionThe setting for automatically notifying the user about the save result.

Details
Saves the image that the program currently displays as a screenshot in the album. Note the following when saving images.

  • The screenshot being saved has the size nn::album::ImageSize_1280x720.
  • The <Screenshot> specification in the NMETA file is ignored.
  • The effect of nn::oe::DisableScreenShot() is ignored.
  • The image is rotated according to the value set by the nn::oe::SetAlbumImageOrientation() function.
  • The overlay notification indicating whether the save to the album succeeded adheres to the value specified for reportOption.
  • The processing by this function might be delayed if the user operates the Capture Button to save an image or a video clip immediately before the function has been called.

In addition, saving the screenshot might fail in the following kinds of situations. You cannot get the reason for the failure from the program, so we recommend setting reportOption to notify the user.

  • When the program is not in focus.
  • When the storage that is the save destination does not have free space.
  • When the limit has been reached on the number of album files that can be saved in the storage that is the save destination.
  • When some other album-related error has occurred.

Parameters
[in]pImageDataThe image data to save as a screenshot.
[in]imageDataSizeThe size of the image data, in bytes.
[in]screenshotSizeThe size of the screenshot to save.
[in]imageOrientationThe orientation of the image.
[in]reportOptionThe setting for automatically notifying the user about the save result.
[in]uidThe user ID to use for posting the screenshot.
[in]presetPostingMessageThe string to specify as the default text for the SNS posting message.
Precondition
The album library is available to use.
pImageData != nullptr.
imageDataSize >= 4 × GetImageWidth(screenshotSize) × GetImageHeight(screenshotSize).
Return Values:

ResultAlbumFull Saving failed because the album does not have enough space.

ResultAlbumFileCountLimit Saving failed because the limit has been reached for the number of files that can be saved in the album.

ResultAlbumError Saving failed due to another error.

ResultShareCanceled Posting was canceled.

ResultShareFailed Posting failed.

Details
Saves the image data as a screenshot in the album and posts it to an SNS.

This function synchronously posts to the SNS. Because the posting process uses the post screen and involves interaction with the user, the caller of the function will be blocked for a long time.

For pImageData, pass the pointer to the start of the image data to save. The format of the passed image is RGBA format, with 8 bits per channel expressed in sRGB color space. The image data has an image format specified as nn::gfx::ImageFormat_R8_G8_B8_A8_UnormSrgb, a tile mode of nn::gfx::TileMode_Linear, an origin at the upper left corner, and no padding (stride == 4 × GetImageWidth(screenshotSize).

Memory from the GPU's memory pool with caching enabled can be passed to pImageData.

For screenshotSize, specify the size of the screenshot to save. This function does not scale the image data, so the image size represented by pImageData must match the size of the screenshot.

For imageOrientation, use a nn::album:: ImageOrientation enumerated type value to specify the orientation (the rotational direction) of the image.

For reportOption, use a nn::album::AlbumReportOption enumerated type value to specify the preferred settings.

The application can specify an optional string in presetPostingMessage to use as a posting message. The maximum size that can be specified is defined by PresetPostingMessageSizeMax. The specified value is parsed as a UTF-8 encoded string. Depending on the social media you are posting to, you may not be able to set a message for the post.

If the screenshot is saved successfully, the system posts it to the SNS via the displayed UI. The saved screenshot will remain in the album even if posting is not completed, for example because the posting process was canceled by the user.
