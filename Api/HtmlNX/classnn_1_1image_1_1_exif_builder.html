Detailed Description

Creates Exif information.

Details
This class creates Exif-format binary to store JPEG metadata. You can select from among predefined fields to configure application-specific information. It supports Exif file format version 2.3.

Estimating an Upper Size Limit For the Creation of Exif Binary

The Exif file format specification sets a maximum of 65,527 bytes of binary output. When selecting the metadata to configure, do not exceed this size.

Object States

The ExifBuilder object can take the following two states.

  • Metadata unanalyzed
    • In this state, the metadata set in ExifBuilder is not analyzed.
  • Metadata analyzed
    • The state after the metadata set in ExifBuilder has been analyzed. The object in this state can be specified for the parameter of the nn::image::JpegEncoder::Encode() function.

◆ SetOrientation()

void nn::image::ExifBuilder::SetOrientation ( const ExifOrientation  orientation)

Sets the orientation of the image-capture device at the time of image capture.

Parameters
[in]orientationOrientation of the image.
Postcondition
The state is "metadata unanalyzed."

Details
Sets the 'image orientation (0x0112)' TIFF tag for the 0th IFD. Because standard viewer applications reference the image orientation, we recommend setting a value that is very close to the orientation of the image capture device. For example, if the device was rotated 90 degrees clockwise when the image was captured, set the orientation to ExifOrientation_Rotate270 (meaning that the image is rotated 270 degrees clockwise).

This setting is unnecessary if the orientation of the capture device is not recorded. In that case, the image is generally handled as not being rotated.

◆ SetDateTime()

void nn::image::ExifBuilder::SetDateTime ( const char *  dateTime,
const size_t  size 
)

Sets the date and time when the image was captured.

Parameters
[in]dateTimeThe date and time of image capture in the prescribed format. To delete the setting, specify nullptr.
[in]sizeThe size, in bytes, of the dateTime parameter, including the terminating null character.
Precondition
All of the following conditions must be satisfied.
  • If something other than nullptr is specified for dateTime, the string must be a contiguous set of 19 ASCII characters (0x00 to 0x7F) terminated with '\0'.
  • If something other than nullptr is specified for dateTime, size must be a value of 20 or larger.
Postcondition
The state is "metadata unanalyzed."

Details
Sets the creation date (0x0132) TIFF tag for the 0th IDF. The ExifBuilder class only inspects to check for the presence of terminating characters, and does not check the actual content.

To comply with the specification for Exif information, we recommend using the format YYYY:MM:DD hh:mm:ss for the string. However, the ExifBuilder class does not inspect the format.

  • YYYY ... The year of capture, specified as a four-digit integer.
  • MM ... The month of capture, specified as a two-digit integer.
  • DD ... The date of capture (0 to 31), specified as a two-digit integer.
  • hh ... The hour of capture (00 to 23), specified as a two-digit integer.
  • mm ... The minute of capture (00 to 59), specified as a two-digit integer.
  • ss ... The second of capture (00 to 59), specified as a two-digit integer.

The elements of any of these fields can be omitted. 

  "2015:04:09 12:00: "
If you omit elements, follow this example and enter spaces (0x20) for omitted fields.

The ExifBuilder object stores the buffer address to the capture date and time specified by this function. Be careful when altering the contents of this buffer.

This setting is unnecessary if a capture date and time is not recorded.

◆ Analyze()

JpegStatus nn::image::ExifBuilder::Analyze ( )

Analyzes the configured metadata and constructs the internal parameter.

Returns
Returns the execution result.
Return Values:

nn::image::JpegStatus_Ok Indicates that the process ended normally.

nn::image::JpegStatus_OutputOverabounds Indicates that the specified software name, maker note, and thumbnail were too big.

Postcondition
The state is "metadata analyzed." The ExifBuilder object can be specified as a parameter of the nn::image::JpegEncoder::Encode() function.

Details
Analyzes the configured metadata and calculates the internal parameter required to output the binary string of Exif information. The function returns JpegStatus_OutputOverabounds at this time if the configured metadata results in too large of a binary string for output. The Exif file format specification sets a maximum size of 65,527 bytes for the binary that can be created.

The ExifBuilder only checks the numbers and size of the configured metadata, with no consideration of its content. When you already know the sizes of all of the fields, you can register just the buffers at first and implement the Analyze() function to change the contents of the fields right before running the Build() function. This enables you to minimize the overall computational costs relating the Analyze() function in the application.

void nn::image::ExifBuilder::Build ( void *  outExifBuf,
const size_t  exifBufSize,
const Dimension &  dimension 
)

Outputs the Exif information to the specified buffer.

Parameters
[out]outExifBufOutput buffer for the Exif information.
[in]exifBufSizeSize of outExifBuf in bytes.
[in]dimensionSize (width and height) of the JPEG data referred to in the Exif information. This information is output as the 'effective image width' (0xA002) and the 'effective image height' (0xA003) Exif tags of the 0th IFD.
Precondition
All of the following conditions must be satisfied.
  • outExifBuf is at least as large as the value obtained by the GetAnalyzedOutputSize() function.
  • The state is "metadata analyzed."

Details
Generates the Exif information binary, based on the following standard metadata and the metadata that was configured before the call to the Analyze() function, and then outputs that binary to the specified buffer. For output, all strings are terminated with '\0' and any characters larger than 0x7F are replaced with 0x20.

  • 0th IFD TIFF tags
    • Maker name (0x010F) = "Nintendo co., ltd"
    • Model name (0x0110) = (undefined)
    • Resolution in the x-direction (0x011A) = 72
    • Resolution in the y-direction (0x011B) = 72
    • Unit of resolution (0x0128) = dpi
    • YCbCr sampling location (0x0213) = Center
  • 0th IFD Exif tags
    • Exif version (0x9000) = 2.3.0
    • Pixel composition (0x9101) = YCbCr
    • FlashPix version (0xA000) = 1.0.0
    • Color space (0xA001) = sRGB

This standard metadata cannot be changed. The following metadata is added if a thumbnail has been configured.

  • 1st IFD TIFF tags
    • Compression format (0x0103) = JPEG
    • Resolution in the x-direction (0x011A) = 72
    • Resolution in the y-direction (0x011B) = 72
    • Unit of resolution (0x0128) = dpi
