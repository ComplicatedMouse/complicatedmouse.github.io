Detailed Description

A handle class that controls sequence sounds that are played back.

Details
Sounds are associated with a general-purpose sound handle (SoundHandle) at playback time, but the processing unique to a sequence sound is not implemented in this general-purpose handle.

If you want to execute a unique process for the sequence sound, use a sequence sound handle that was created by using a constructor that takes the general-purpose sound handle as an argument.

Handles are used like the general-purpose sound handle SoundHandle.

Details
Stops the sound associated with the handle. If the handle is invalid, the function does nothing.

You can fade out over the number of frames specified by fadeFrames. If 0 has been specified, fade-out is not performed and sound stops after the sound for the release of the envelope is generated and all attenuation has been completed.

The volume control for fade-out is also used for fade-in. Number of elapsed frames during a fade out. The number of fade-out frames correlates to the speed of change from maximum volume to silence. If a fade out is specified during a fade in, or in other similar situations, it may complete sooner than the specified frame count.

Parameters

Pauses or restarts a sound.

Details
Pauses or restarts the sound associated with the handle. If the handle is invalid, the function does nothing.

You can pause the sound while fading out, or restart it while fading in, for the number of frames specified by fadeFrames.

If 0 is specified for fadeFrames, and the sound is paused, the volume is set to zero without a fade-out after the first SoundArchivePlayer::Update, and after the second SoundArchivePlayer::Update, the sound is paused. Note that the playback time advances in the silent state during the interval between the first and second calls to SoundArchivePlayer::Update. However, you do not need to track this issue because playback resumes after the first SoundArchivePlayer::Update without fade-in.

Fading when pausing and resuming sounds operates independently from the fade-in when resuming a sound and the fade-out when pausing a sound.

In addition, the number of frames that elapse during fading correlates to the speed of change from the maximum volume to silence, or from silence to the maximum volume. A fade operation can complete in less than the specified number of frames, for example, if a fade-out is started in the middle of a fade-in.

Parameters
[in]flagPlayback pauses if this parameter is true, and restarts if it is false.
[in]fadeFramesNumber of frames used to fade in and out. The number of frames is based on the number of calls to SoundArchivePlayer::Update. If you specify a negative value, the function works the same as if you had specified 0.

Changes the volume of the sound.

Details
Changes the volume of the sound associated with the handle. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other volume parameters, and they are all layered.

The volume parameter takes a ratio of 0.0 or greater. Volume does not change if 1.0 is specified. If 0.0 is specified, there is no sound. The default value is 1.0.

Note that after this value is combined with the other volume parameters, the final volume is clamped to the range from nn::audio::VoiceType::GetVolumeMin() to nn::audio::VoiceType::GetVolumeMax(). For more information about combining volume parameters, see Sound Parameter Control Operations.

The volume changes over the number of frames specified by frames. If this function is called during a volume change, the volume changes over the number of newly specified frames, using the volume value at that point as the base point.

Parameters
[in]volumeThe volume change ratio (0.0 or greater).
[in]framesThe number of frames that elapse when the volume changes. The number of frames is based on the number of calls to SoundArchivePlayer::Update. Specify a value of 0 or greater for the frames parameter. If you specify a negative value, the function works the same as if you had specified 0.

◆ SetPitch()

void nn::atk::SequenceSoundHandle::SetPitch ( float  pitch)

Changes the pitch of the sound.

Details
Changes the pitch of the sound associated with the handle. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other pitch parameters, and they are all layered.

pitch specifies the frequency ratio. In other words, pitch is not affected if 1.0 is specified. Specifying 2.0 doubles the playback frequency, increasing the pitch by one octave. If 0.5 is specified, the pitch is one octave lower. The default value is 1.0.

Note that after this value is combined with the other pitch parameters, the final pitch is clamped to the range from nn::audio::VoiceType::GetPitchMin() to nn::audio::VoiceType::GetPitchMax(). For more information about combining pitch parameters, see the atk Library Sound Parameter Control Operations documentation.

pan specifies a value that represents the relative change in the location of the sound. If 0.0 is specified, the pan value that is set within data is unchanged. Details
Changes the main send volume for sequence tracks of the sound associated with the handle for main (TV) output. If the handle is invalid, the function does nothing.

The value specified by this function operates independently of other surround parameters, and they are all layered.

Specifies the track to configure, using trackBitFlag. The tracks are represented in order from the lowest bit as Track 0, Track 1, Track 2, and so on. The main send volume changes for all tracks whose bits have been set. For example, to change the main send volume of Track 2 and Track 5, use (1 << 2) | (1 << 5); that is, 0x0024.

The send parameter specifies a value that represents the relative amount of change. The send volume is unchanged if 0.0 is specified. If –1.0 is specified, sound formerly sent at maximum send volume to the main bus is no longer sent to the main bus at all. The default value is 0.0.

Details
If the sound handle is not valid, a value of 0 is returned.

The value returned by the function is reset to 0 if the tick counter exceeds 32 bits.

A tick is a unit of time that serves as the unit for the time base. (For more information about the time base, see the Sequence Data Manual.) For example, if the time base is 48, the length of a quarter note is equivalent to 48 ticks.
