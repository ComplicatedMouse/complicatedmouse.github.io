This sample program shows putting a hat on a Mii character in g3d.

Overview

This sample program shows how to put a hat on the nn::g3d::ResModel assigned to the Mii model. Use g3d to switch between the nn::mii::CreateModelType for each part of the Mii.

File Structure

This sample program is located in Samples/Sources/Applications/MiiG3dHeadwear/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

To run this sample program, you must use the Mii library.

Execution Procedure

Build the sample program and then run it.

Description

This sample program assigns a Mii model to nn::g3d::ResFile, applies animation, and displays the Mii model. This sample program has the following flow.

Model Conversion Process

Convert the following source code to one BFRES file.

  • Samples/Sources/Applications/MiiG3dHeadwear/Resources/Model/MiiSeparateBone.fmdb

Empty model data, with data structures that contain no vertices or indices. The Mii model is assigned in the runtime. There is an individual bone for each part, so you can switch the visibility of each part. However, the CPU processing cost for bones is greater than when the bones are not joined.

  • Samples/Sources/Applications/MiiG3dHeadwear/Resources/Model/CapSample00.fmdb

The model of the hat to put on the Mii.

  • Samples/Sources/Applications/MiiG3dHeadwear/Resources/Model/MiiSeparateBoneFaceExpressionChange.fmab

Material animation for changing the expression of the Mii. In the runtime, this animation is applied to the nn::g3d::ResModel to which the Mii model has been assigned.

  • Samples/Sources/Applications/MiiG3dHeadwear/Resources/Model/MiiSeparateBoneModelTypeChange.fvbb

The animation for switching between nn:mii::CreateModelTypefor each part of the Mii.

  • Samples/Sources/Applications/MiiG3dHeadwear/Resources/Model/CapSample00WearOnOff.fvbb

The animation for putting on and taking off the hat.

  • Samples/Sources/Applications/MiiG3dHeadwear/Resources/Shader/SampleShader.glsl

The shader for displaying the Mii.

Processes of the Sample Program

The process flow of the running sample program is presented below.

  • Construct a model Mii.
  • Assign the Mii model to nn::g3d::ResModel.
  • Create an instance of the nn::g3d model.
  • Create an instance of the nn::g3d material and bone visibility animation.
  • Begin the rendering loop.
  • Update the animation.
  • Update models.
  • Execute the command list.
  • Swap the scan buffer.
  • Return to the beginning of the rendering loop.
