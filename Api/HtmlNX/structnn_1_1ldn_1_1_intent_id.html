Detailed Description

Identifier representing network objectives.

Details
The 128-bit identifier that indicates the purpose of the network. It is equivalent to the first half of NetworkId. The IntentId is comprised of the local communication identifier (localCommunicationId) and scene identifier (sceneId).

The local communication identifier is used to identify application titles or series. The general policy is to use the application ID as the local communication identifier. However, if the application communicates with other applications, a common local communication identifier shared between the communicating applications is required, so it is possible to use the application ID of one application for both. The local communication identifiers used by the application must all be registered in the application management data, and if they are not registered CreateNetwork() and Connect() will fail because the prerequisites have not been satisfied. If the local communication identifier was not set in the application management data, the application ID itself will be registered as the local communication identifier. DefaultLocalCommunicationId can be used for localCommunicationId if there is no need for the application to use multiple local communication identifiers.

In comparison, scene identifiers are intended to identify application scenes such as "online battles" or "data exchange", and may be freely configured by the application developer. Based on these characteristics, the same IntentId will be used repeatedly when the local network is configured for the same purpose.

When you need to create an IntentId, do not configure settings directly in the IntentId structure; make sure that you use the MakeIntentId function instead.
