This sample program shows how to use file system operations on ROM resource data.

Overview

This sample uses ROM file system operations.

For more information about file and directory operations, see the File System Library API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/FsRom/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log.

Execution Procedure

Build the sample program and then run it.

107  // Mounts the file system.
108  // Do not release the cache buffer until you unmount.
109  result = nn::fs::MountRom("rom", cacheBuffer, cacheSize);
110  // Always abort when a failure occurs.
111  NN_ABORT_UNLESS_RESULT_SUCCESS(result);

114  // Loads the resource data.

117 
118  nn::fs::FileHandle file;

120  if (nn::fs::ResultPathNotFound::Includes(result))
121  {
122  // The target file cannot be found.
123  // This error handling is not necessary when you are only opening existing files.
124  NN_ASSERT(false, "File not found.\n");
125  return;
126  }

133

Sample Program Description

This sample program has the following flow.

  • An example of file system operations on ROM resource data using the MountRom function.

Use the MountRom function to access the resource data file system.

In this sample program, a source file and a directory named dataSrc, which are both located in the same directory as is set for resource data references, as well as the test/testdata entries are loaded. Set the files and directories to be referenced by resource data in the project file. Also see the ROM Resource Data section in NintendoSDK Documents > Features > Basic Features > fs Library > Manual > Features.
