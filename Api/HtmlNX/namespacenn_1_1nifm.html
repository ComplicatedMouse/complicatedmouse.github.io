Detailed Description

Namespace for the network connection management library.

Feature Overview

The network connection management library provides various features, including the ability to arbitrate application and system requests, to select the network interface to use, and to determine whether network connections can be established and used. When applications use network connections, you can get the reason why an application could connect to the appropriate network yet be unable to use it when you submit use requests via this library.

Common Preconditions

To use the API provided by the network connection management library, you must initialize the library with the nn::nifm::Initialize function, prior to use, unless they are individually disclosed.

Registering Network Connection Settings

To connect a device to the network, you must add network connection settings using the Settings Manager tool ahead of time. For more information, see the Settings Manager manual.

Error Handling

Call the error handling function after submitting the network usage request to transition to the sequence for processing depending on the results of the application. For more information, see nn::nifm::HandleNetworkRequestErrorResult.

Behavior and Restrictions in Windows

When running on Windows and depending on the submission of the network use request via the library, the switching of Windows network connections and settings may not work. In addition, this library uses the fs library internally. Ensure that memory allocation can occur using the allocator set with the nn::fs::SetAllocator() function or with malloc(). For more information, see the fs library documentation.

Controls whether wireless communication is enabled or disabled.

Parameters
[in]isEnabledWhether wireless communication is enabled.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The process succeeded.
Precondition
  • The library is already initialized.

Details
This function is for testing, and before it can successfully run, you must select the Testing mode enabled check box on the Network page of the DevMenu. This function will always fail on products where this permission is not set. The Testing mode enabled operation is not necessary in the Windows environment.

The actual stop and restart time of wireless communication may be delayed.

The enable/disable wireless setting that is controlled by this function is the same as the enable/disable wireless setting that is controlled by the user.

The setting is not permanent, and will return to the setting specified by the user when the system is restarted.

◆ SubmitNetworkRequest()

void nn::nifm::SubmitNetworkRequest ( )

Requests a network connection of the system.

Precondition
  • The library is already initialized.

Details
Calling this function invokes an asynchronous process that checks whether a network connection can be used. During verification, the nn::nifm::IsNetworkRequestOnHold function returns true. After the application calls this function, have it periodically check the status of nn::nifm::IsNetworkRequestOnHold. When that function returns false, use the nn::nifm::IsNetworkAvailable function to see whether the network connection is available.
Verification of a network connection's use request occurs only once per call of this function. After verification, to request use of a network connection again, you must call this function again.
This function returns without blocking.
This function and the nn::nifm::SubmitNetworkRequestAndWait function are not thread-safe.

Windows Platform Only:
Calling the Windows version of this function has no effect on Windows network connections. When you do so, the network is available if Windows is connected to it. If not, it is unavailable.
For example, in a Windows environment with wireless connectivity, if you are only configured for wired NIFM connection settings, submitting a use request does not result in an available state. That current state will not meet any connection setting requirements. In this example, you would need to switch to development hardware (if possible) with a wired connection.
Examples
BcatPractical/BcatPractical.cpp, Libcurl/Basic/LibcurlBasic.cpp, Libcurl/SpeedTest/LibcurlSpeedTest.cpp, Libcurl/SslContextImport/LibcurlSslContextImport.cpp, NifmNetworkConnectionSimple/NifmNetworkConnectionSimple.cpp, SocketBasic/SocketBasic.cpp, SocketResolver/SocketResolver.cpp, SslSimple/SslSimple.cpp, and WebSocketChat/WebSocketChat.cpp.

◆ SubmitNetworkRequestAndWait()

void nn::nifm::SubmitNetworkRequestAndWait ( )

Makes a request to the system for a network connection, and blocks until the request is accepted or denied.

Precondition
  • The library is already initialized.

Details
Call this function to verify whether a network connection can be used. It blocks until the results of verification appear. After you leave this function, you can use the nn::nifm::IsNetworkAvailable function to check whether the network connection is in an available state.
Verification of a network connection's use request occurs only once per call of this function. After verification, to request use of a network connection again, you must call this function again.
Verification can occur very quickly, depending on the internal state of the system.
This function and the nn::nifm::SubmitNetworkRequest function are not thread-safe.

Windows Platform Only:
Calling the Windows version of this function has no effect on Windows network connections. When you do so, the network is available if Windows is connected to it. If not, it is unavailable.
For example, in a Windows environment with wireless connectivity, if you are only configured for wired NIFM connection settings, submitting a use request does not result in an available state. That current state will not meet any connection setting requirements. In this example, you would need to switch to development hardware (if possible) with a wired connection.
Examples
BcatSimple/BcatSimple.cpp, EcCatalog/Main.cpp, EcConsumableServiceItemManager/EcConsumableServiceItemManager.cpp, FriendsProfileImage/FriendsProfileImage.cpp, FriendsSimple/FriendsSimple.cpp, NifmNetworkConnectionSimple/NifmNetworkConnectionSimple.cpp, Openssl/Client/OpensslClient.cpp, SocketMulticast/Basic/SocketMulticastBasic.cpp, SocketStatisticsOverlay/SocketStatisticsOverlay.cpp, and WebSocketEcho/WebSocketEcho.cpp.

◆ CancelNetworkRequest()

void nn::nifm::CancelNetworkRequest ( )

Withdraws a network connection request.

Precondition
  • The library is already initialized.

Details
Withdraws a network connection request that was submitted to the system.
This function can be called regardless of whether a request is being verified. To clear the blocking from the nn::nifm::SubmitNetworkRequestAndWait function, call this function from a separate thread.
This function simply declares that you are no longer interested in a network connection. It is not a request to disconnect a network connection.

◆ IsNetworkAvailable()

bool nn::nifm::IsNetworkAvailable ( )

Determines whether a network connection is available.

Precondition
  • The library is already initialized.
Returns
Returns true if the network connection is available, or false otherwise.

Details
When this function returns true, you can use the network connection.
The return value can switch from true to false unexpectedly due to an external factor such as disconnection from the wireless LAN access point. Conversely, you would never see the return value change from false to true without submitting a use request via either the nn::nifm::SubmitNetworkRequest or nn::nifm::SubmitNetworkRequestAndWait function. There is no transition from false to true.

Examples
BcatPractical/BcatPractical.cpp, BcatSimple/BcatSimple.cpp, EcCatalog/Main.cpp, EcConsumableServiceItemManager/EcConsumableServiceItemManager.cpp, FriendsProfileImage/FriendsProfileImage.cpp, FriendsSimple/FriendsSimple.cpp, Libcurl/Basic/LibcurlBasic.cpp, Libcurl/SpeedTest/LibcurlSpeedTest.cpp, Libcurl/SslContextImport/LibcurlSslContextImport.cpp, NifmNetworkConnectionSimple/NifmNetworkConnectionSimple.cpp, Openssl/Client/OpensslClient.cpp, SocketBasic/SocketBasic.cpp, SocketMulticast/Basic/SocketMulticastBasic.cpp, SocketResolver/SocketResolver.cpp, SocketStatisticsOverlay/SocketStatisticsOverlay.cpp, WebSocketChat/WebSocketChat.cpp, and WebSocketEcho/WebSocketEcho.cpp.

◆ SetLocalNetworkMode()

void nn::nifm::SetLocalNetworkMode ( bool  isLocalNetworkMode)

The enabled or disabled state of the local network mode.

Parameters
[in]isLocalNetworkModeThe enabled or disabled state of the local network mode.
Precondition
  • The library is already initialized.
  • Before a use request was submitted or after it was canceled.

Details
Local network mode is a mode that you specify when using connections to a closed LAN to perform actions such as performing matchmaking within a local network. Network use requests are not processed when local network mode is not enabled and your environment only permits connections to a closed LAN.
Disable local network mode prior to submitting a network use request when you need to connect to the Internet. Using local network mode does not block communication with the Internet. However, that mode produces side-effects: You will eventually be selecting networks that cannot access the Internet and you will be unable to connect to the Internet from any public wireless LAN that requires user authentication or consent.
Local network mode is disabled by default.

◆ HandleNetworkRequestErrorResult()

bool nn::nifm::HandleNetworkRequestErrorResult ( )

Handles submitted use requests that result in failure.

Returns
Returns true if the error handling succeeds, or false if it does not.
Precondition
  • The library is already initialized.
  • The submitted network use request has failed.

Details
Depending on the error, some situations may require user operation during handling, so this function could block for a long time.
If the return value is true, the process has recovered from the error and if you resubmit the use request it might be immediately accepted.
You can call this function again even if the resubmitted use request is rejected.
If the return value is false, there is no way of predicting the result of the resubmitted use request, so it is better to return to the previous sequence.

This function performs the appropriate actions like displaying the error viewer. If the use request was explicitly withdrawn by the user or another operation, you do not need to call this function to try and resolve the error.

Windows Platform Only:
In the Windows version, this function does not return true if called when a use request has been rejected.
Examples
BcatPractical/BcatPractical.cpp, EcCatalog/Main.cpp, EcConsumableServiceItemManager/EcConsumableServiceItemManager.cpp, NifmNetworkConnectionSimple/NifmNetworkConnectionSimple.cpp, and WebSocketChat/WebSocketChat.cpp.

◆ GetCurrentProxySetting()

nn::Result nn::nifm::GetCurrentProxySetting ( ProxySetting *  pOutProxySetting)

Gets the proxy settings being used for the current connection.

Parameters
[out]pOutProxySettingPointer to the buffer that will store the obtained proxy settings.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The process succeeded.
Precondition
  • The library is already initialized.
  • pOutProxySetting != nullptr.

Details
This function returns a failure when not connected to the network, and proxy settings obtained at this point are undefined. Even if the system was verified to be connected to a network immediately beforehand, this situation may occur if the connection is disrupted by some external factor. For this reason, make sure that you verify the success of this function.

When using the libcurl library included in the NintendoSDK, the proxy settings are applied automatically. For this reason, you do not need to explicitly register the settings obtained from this function.
