◆ InitializeForCaptureDevice()

CAPTUREPROCESSOR_API void InitializeForCaptureDevice ( int  deviceId,
bool  isPreview,
CaptureResolution  captureResolution 
)

Initializes the CaptureProcessor library for capture devices.

Parameters
[in]deviceIdCapture device ID.
[in]isPreviewShow preview flag.
[in]captureResolutionCapture resolution.

Details
Sets the capture device connected to the PC as the capture target device. The capture device ID is a value assigned in order, starting from 0, to capture devices such as cameras and capture boards connected to the environment in use. For the deviceId, specify an integer of 0 or greater but less than the number of connected video devices. You can use the GetCaptureDeviceCount function to get the number of capture devices. If you specify true for isPreview, a window is generated in the library and a preview is displayed for the capture image in real time. If initialization has not completed, calling image operation functions does not work correctly.

◆ DetectObject()

CAPTUREPROCESSOR_API MatchingResult DetectObject ( int *  pOutX,
int *  pOutY,
ImageIdType  id,
double  threshold 
)

Performs template matching for the latest image obtained from the capture target device.

Parameters
[out]pOutXX-coordinate of the detection result.
[out]pOutYY-coordinate of the detection result.
[in]idTemplate image data ID.
[in]thresholdMatching threshold value (1.0 > x > 0.0)
Returns
Returns whether the operation succeeded.
Return Values:

MatchingResult_Success The value when a matching image is found.

MatchingResult_Failed The value when a matching image is not found.

MatchingResult_Unexpected The value when an unexpected error occurs.

MatchingResult_InvalidImageId The value when an invalid image data ID is specified.

MatchingResult_NotInitialized The value when called in an uninitialized state.

Precondition
The call to the Initialize function is complete.

Details
Gets the latest image from the capture target device, and detects the locations in that image that match the template image recorded in the specified ID. When MatchingResult_Success is returned, the upper-left coordinates of the area within the image that most matches the template image is passed to pOutX, pOutY. The higher the threshold, the harder it is to hit an image that does not closely match the template image.

Repeat template matching for the image being captured until the template image is detected.

Parameters
[out]pOutXX-coordinate of the detection result.
[out]pOutYY-coordinate of the detection result.
[in]idTemplate image data ID.
[in]thresholdMatching threshold value (1.0 > x > 0.0)
[in]timeoutTimeout time, in milliseconds.
Returns
Returns whether the operation succeeded.
Return Values:

MatchingResult_Success The value when a matching image is found.

MatchingResult_Failed The value when a matching image is not found.

MatchingResult_Unexpected The value when an unexpected error occurs.

MatchingResult_InvalidImageId The value when an invalid image data ID is specified.

MatchingResult_NotInitialized The value when called in an uninitialized state.

Precondition

Details
Repeatedly perform template matching for the latest image obtained from the capture target device until the template image is detected. The higher the threshold, the harder it is to hit an image that does not closely match the template image.

Binarization processing converts the image to grayscale using the NTSC weighted average rule (Gray = Red * 0.299 + Green * 0.587 + Blue * 0.114), and then converts pixels that exceed the binarizeThreshold to white and those that do not to black.
