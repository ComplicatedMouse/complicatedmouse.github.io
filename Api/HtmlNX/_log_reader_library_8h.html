LOGREADER_API LogReaderResult StartLogStoring ( const SerialNumberString &  serialNumber,
int  logMaxLineCount 
)

Starts the process of getting a log from the target.

Parameters
[in]serialNumberThe serial number of the target.
[in]logMaxLineCountThe maximum number of lines of the log to store.
Returns
Returns the result.
Return Values:

LogReaderResult_Success Succeeded.

LogReaderResult_ConnectFailed Connection failed.

LogReaderResult_AlreadyStarted Recoding has already started.

LogReaderResult_InvalidArgument An invalid value was passed to the argument.

Precondition
  • Connected to the target using TargetManager.
  • logMaxLineCount >= 0.

Details
Gets a log from the target and individually saves each line of the log in memory. If the newest log does not contain the newline control character, it will be saved when the next newline control character is generated. When the log is saved, the time when that line was saved is stored internally.
In addition, a cursor line is stored that represents which line of which log is currently being referenced in the library.
When the getting of the log starts, the first obtained log is set as the cursor line.
The getting of logs and the matching of regular expressions starts from this cursor line.
When the number of read lines exceeds logMaxLineCount, the oldest log is discarded and the new line is obtained. If the cursor line was pointing to this old discarded log, it is redirected to point to the log for the next line.

LOGREADER_API LogReaderResult WaitUntilMatched ( const char *  regexString,
int  timeoutMs 
)

Advances the cursor line until a log that matches the specified regular expression is found.

Parameters
[in]regexStringA regular expression string with a null terminator.
[in]timeoutMsThe time until timeout, in milliseconds.
Returns
The result.
Return Values:

LogReaderResult_Success Succeeded.

LogReaderResult_Timeout Timed out.

Details
Advances the cursor line until a log that matches the specified regular expression is found.
Specify a string encoded in UTF-8 as the regular expression string.
std::regex_match is used for the regular expression matching, and matching is performed with the default locale settings.
If no match is found through to the newest log, the function waits for the next log to be obtained.
If the time specified by timeoutMs has elapsed and there still is no match, the function returns the error LogReaderResult_Timeout.
If there is no match, the cursor line points to the newest log.

LOGREADER_API LogReaderResult SearchFormerLog ( const char *  regexString)

Moves back the cursor line until a log is found that matches the specified regular expression.

Parameters
[in]regexStringA regular expression string with a null terminator.
Returns
The result.
Return Values:

LogReaderResult_Success Succeeded.

LogReaderResult_NotFound No matching log was found.

Details
Moves back the cursor line until a log is found that matches the specified regular expression.
Specify a string encoded in UTF-8 as the regular expression string.
std::regex_match is used for the regular expression matching, and matching is performed with the default locale settings.
If no match is found even after reaching the oldest stored log, the function returns the error LogReaderResult_NotFound.
If there is no match, the cursor line points to the oldest log.
