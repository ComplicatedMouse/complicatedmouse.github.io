A sample program for the semaphore feature.

Overview

This sample shows the semaphore feature.

For information about using the semaphore feature, see the Semaphore Feature Manual and the OS API Reference.

File Structure

This sample program is located in Samples/Sources/Applications/OsSemaphore/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

This sample program does not display anything on the screen. The execution results are output to a log. This sample will also only work in operating environments where at least three CPU cores are available.

Execution Procedure

Build the sample program and then run it.

Sample Program Description

Semaphore functionality is a mechanism for controlling synchronization while safely excluding resources shared among multiple threads. It differs from mutex functionality in the following ways.

  • Multiple threads can acquire semaphores at the same time.
  • Threads other than those that acquired the semaphore can return the semaphore.

This sample comprises the following type of logic to realize the former ability above: Multiple threads can acquire semaphores.

  • Three threads are created with nnMain(). Priority cores are shared among 0 through 2 when creating.
  • The three threads all run ThreadSample() as a thread function.
  • Each thread repeats the following processes for a set number of times.
    • Acquire g_Semaphore.
    • Export own thread name to the log and stand by for a fixed period of time.
    • Export a line feed to the log and stand by again for a fixed period of time.
    • Return g_Semaphore.
  • nnMain() will end after standing by for all three threads to end.

Each semaphore uses the g_Semaphore initialized using a semaphore count of two for exclusion. As a result, a maximum of two thread names for each thread are exported, side by side.

The log output will vary for different operating environments and each time the function is run because the success of any given thread for acquiring a semaphore will depend on different factors, including their priorities, processing speeds, and the order in which they arrive at nn::os::AcquireSemaphore(). As a result, this sample will have threads that finish looping quickly and threads that are slower than the others, among the three threads.
