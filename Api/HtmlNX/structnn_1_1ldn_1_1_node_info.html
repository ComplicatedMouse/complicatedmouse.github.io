Details
Information for nodes that have joined a network.

The IPv4 addresses assigned to the nodes are stored in ipv4Address, and the socket library uses these addresses to perform communication. This will not change in the period between connecting to and then disconnecting from the LDN network. Even if the connection is disconnected, the same IPv4 address that was used previously will usually be allocated again if you reconnect to the same network. However, if the node connects to a variety of new stations while the original station was disconnected, the old IPv4 address may be reused for a different station, so it is not guaranteed that the old address will always be reused when reconnecting.

The node numbers are stored in nodeId, and correspond to the nodes index in the LdnNetworkInfo structure. The access point will always be node number 0, but the stations will be assigned node numbers 1 through StationCountMax. After being connected to the LDN network, this number will not change until disconnection. Unlike the IPv4 address, the same node number will not be prioritized for reuse when reconnecting.

If isConnected is true, the node is connected. If isConnected is false, there is no valid value stored for ipv4Address or nodeId.
