Detailed Description

The calculation engine class for 3D sound parameters.

Details
Calculates various types of parameters for sounds, such as volume and pan, from the position information of 3D sound actors and 3D sound listeners, among other information. Standard parameter calculations for 3D sounds are defined in this class.

An initialized instance of this class is registered in 3D Sound Manager.

If you want to customize the 3D sound engine, you can inherit from this class and create a unique engine class. You can replace the default engine class with a customized engine class by registering it in the 3D Sound Manager using Sound3DManager::SetEngine.

Operations of the Default Implementation

The implementation of this class calculates the following 3D sound parameters (SoundAmbientParam) passed by UpdateAmbientParam). Note that the method for calculating parameters varies depending on the number of 3D sound listeners registered to the 3D sound manager.

If you want to calculate any other parameters, override the UpdateAmbientParam function in a user class derived from this class.

Common Behavior

The following SoundAmbientParam parameters are not calculated.

  • volume
  • lowPassFilter
  • userData
  • outputLineFlag
  • effectSend of the tvParam group of parameters.

The following SoundAmbientParam parameters are calculated.

  • biquadFilterType

Calling the <tt>@ref Sound3DManager::AddListener</tt>

Function to Register Only One Listener to the 3D Sound Manager

The following SoundAmbientParam parameters are calculated.

  • pitch
  • biquadFilterValue
  • priority

For the following parameters, the values are calculated only for the relevant output lines, as indicated by Sound3DListener::GetOutputTypeFlag.

  • The volume tvParam for the relevant output line
  • The pan tvParam for the relevant output line
  • The surroundPan tvParam for the relevant output line

Common behavior when <tt>@ref Sound3DManager::AddListener</tt>

is run to register multiple listeners to the 3D sound manager.

The following SoundAmbientParam parameters are calculated, but only the largest value is used from the results calculated for the multiple registered listeners.

  • biquadFilterValue
  • priority

The behavior for an output line when <tt>@ref Sound3DManager::AddListener</tt> is run to

register multiple listeners to the 3D sound manager, but only one listener gets added for that line. (If only one listener is specified as OutputType_Tv, for example, this is the behavior for output to the TV.)

The following SoundAmbientParam parameters are calculated.

  • The volume tvParam for the relevant output line
  • The pan tvParam for the relevant output line
  • The surroundPan tvParam for the relevant output line

The function for calculating 3D sound parameters.

Details
This function is called when a sound is started, and when it is updated in each frame.

Override this function when you create a customized 3D sound engine class, and store the parameter calculation results in ambientParam. The stored values are applied to the sound processing at each frame.

3D sound listener and 3D sound actor information is used to calculate the parameters. The 3D sound listener can be obtained from sound3DManager. sound3DParam is the parameter structure used for calculations. It includes the actor coordinates and the 3D sound settings for each sound.

updateFlag is a bit flag that shows the parameters that must be calculated. The parameters in the ambientParam structure that correspond to each of the flags are updated based on the sound. Operations are undefined when a parameter that does not have its flag set is updated. 

       - UpdateVolume ... ambientParam.volume
       - UpdatePan ... ambientParam.pan
       - UpdatePriority UpdatePriority ... ambientParam.priority
       - UpdatePitch ... ambientParam.pitch

When this function is called to calculate player priorities before sound playback begins, UpdateStartPriority is set in updateFlag. (UpdatePriority is also set at this time.)

The results from the previous cycle of 3D sound calculations are stored in the ambientParam structure passed as an argument. When you override the UpdateAmbientParam function, you can compare the values passed in this argument with the values calculated in the current cycle to avoid sudden changes in parameters and suppress popping and crackling noises.
