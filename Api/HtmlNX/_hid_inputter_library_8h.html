◆ BeginHidInputRecording()

HIDINPUTTER_API HidInputterResult BeginHidInputRecording ( )

Starts recording HID input data for the target.

Returns
Returns whether the operation succeeded.
Return Values:

HidInputterResult_Success The value when the operation succeeds.

HidInputterResult_Unexpected The value when an unexpected error occurs.

HidInputterResult_NotInitialized The value when called in an uninitialized state.

Precondition
The Initialize() function must be called.

Details
Records the HID operations for the connected target. You can use the ExportHidInputRecordsWithPythonScript() function to export the recorded operations as a script. The following input data can be recorded as operation data.

  • GamePad input on development hardware through InputDirector
  • Input on a debug pad connected to the development hardware
  • Input on a Joy-Con connected to the development hardware
  • Input on a Switch Controller connected to the development hardware
  • Touch screen input on the development hardware

The functions for recording HID operations (BeginHidInputRecording, EndHidInputRecording, and ExportHidInputRecordsWithPythonScript) are not thread-safe. Operation is not assured if they are called from multiple threads. When these functions are called, existing HID input records are reset.
