Detailed Description

The class that performs the process of encoding PNG data.

Details
Encodes the specified pixel data based on the specified parameters and generates PNG data. The output PNG data is noninterlaced PNG data with the filter type set to None.

Output Size

The size of the encoded PNG data varies depending on the colors and patters of the specified pixel data, so it is difficult to calculate beforehand. Make sure that you allocate an output buffer that is large enough. The actual size of the output PNG data does not exceed the value obtained by the GetPngDataSizeMax() function.

Object States

The PngEncoder object can take the following three states.

  • Uninitialized Encoder
    • The PngEncoder object is not initialized.
  • Initialized Encoder
    • The PngEncoder object is initialized, but no pixel data has been registered to the object.
  • Registered Pixel Data
    • Pixel data has been registered to the PngEncoder object.

Parallel Processing

Multiple PngEncoder class objects can be created to encode multiple sets of pixel data in parallel.

Postcondition
The state of the object becomes Initialized Encoder and the SetPixelData() function can be called.

Details
Initializes the PngEncoder object. The specified functions for memory allocation and deallocation are used when data is encoded. If multiple PngEncoder objects are created to encode data in parallel, the functions for memory allocation and deallocation must be thread-safe. The memory returned by allocateFunc must have the same alignment as the platform standard malloc function.

  • In the Win64 environment, it must be aligned to 16 bytes.
  • In non-Win64 environments, it must be aligned to alignof(std::max_align_t).
  • The behavior is undefined if allocateFunc returns memory that does not comply with these alignment restrictions.

Specifies and registers the pixel data to be PNG encoded and the pixel format of that pixel data.

Parameters
[in]pixelDataThe pixel data to encode. The pixels in this pixel data must all be in the pixel format specified by pixelFormat. If the bit depth is 1, 2, or 4, the highest-order bit represents the left pixel, and each row of pixel data must be rounded up to the byte.
[in]pixelDataSizeThe size of the pixel data to encode.
[in]pixelFormatThe pixel format of the pixel data to encode.
[in]dimensionThe size of pixelData (the width and height, in pixels).
[in]rowStrideThe per-row stride of pixelData (in bytes). The padding region is ignored when the data is encoded. For example, specify 130 if an image with 128 bytes per row was being stored in a pixel buffer with 130 bytes per row.
Precondition
All of the following conditions must be satisfied.
  • The state is not Uninitialized Encoder.
  • pixelFormat is defined in the nn::image::PixelFormat enumerator, and the specified format is neither palette color nor nn::image::PixelFormat_None.
  • rowStride is equal to or larger than the stride required when pixel data in the format specified by pixelFormat is stored in the width specified by dimension.
  • pixelData is equal to or larger than the product of (the pixel data height specified by dimension) Ã— (the per-row stride specified by rowStride).
Postcondition
The state becomes Registered Pixel Data and the Encode() function can be called.

Details
Specifies and registers the pixel data to encode and the pixel format of the pixel data. After this function has been called, do not change the content of the pixel data registered in pixelData until the PngEncoder instance is destroyed or this function is called again.

Gets the maximum size of the PNG data that resulted from encoding the pixel data specified by SetPixelData().

Returns
Returns the maximum size of the encoded PNG data.
Precondition
The state must be Registered Pixel Data.

Details
This function calculates the maximum size of the PNG data generated after encoding, based on the format of the pixel data and the size of image being created. The actual size of the resulting PNG data varies depending on the colors and patterns of the specified pixel data, but the size does not exceed the value obtained by this function.

PngStatus nn::image::PngEncoder::Encode ( size_t *  pOutDataActualSize,
void *  outPngDataBuf,
size_t  outPngDataBufSize 
)

Encodes pixel data as PNG data.

Parameters
[out]pOutDataActualSizeThe actual size of the output PNG data, in bytes.
[out]outPngDataBufPointer to the buffer for storing the output PNG data. The size of the area needed for storing the output PNG data is undefined, because it varies depending on the pixel data being encoded.
[in]outPngDataBufSizeThe size of outPngDataBuf, in bytes.
Returns
Returns the execution result.
Return Values:

nn::image::PngStatus_Ok Indicates that the process ended normally.

nn::image::PngStatus_InternalError An internal error occurred.

nn::image::PngStatus_OutOfMemory Not enough memory.

nn::image::PngStatus_ShortOfOutputBuffer The output buffer was not large enough during encoding.

Precondition
The state must be Registered Pixel Data.
Postcondition
If PngStatus_Ok is returned, the encoded PNG data is written to outPngDataBuf, and the actual size of the output PNG data is written to pOutDataActualSize. If any other value is returned, it means that encoding has failed. If PngStatus_InternalError is returned, an internal error occurred. If PngStatus_OutOfMemory is returned, there is not enough memory for the allocator to use. If PngStatus_ShortOfOutputBuffer is returned, increase the size of the output buffer and call this function again.

Details
Encodes the specified pixel data, using the specified pixel format. You can use GetPngDataSizeMax() to get the theoretical maximum size of the output PNG data. The precise size of the output buffer cannot be calculated beforehand, but it does not exceed the value obtained by the GetPngDataSizeMax() function. If an internal error has occurred, you can check the details of the error in the log. If the error occurred because of incorrect pixel data or parameters, you can call Encode() again after setting the correct data.
