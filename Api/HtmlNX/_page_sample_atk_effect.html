A sample program for the effect features of the nn::atk library.

Overview

This sample applies atk library effects when playing audio data. The following effects can be applied.

  • Delay (nn::atk::EffectDelay)
  • Reverb (nn::atk::EffectReverb)
  • I3DL2 Reverb (nn::atk::EffectI3dl2Reverb)
  • Lightweight Limiter (nn::atk::EffectLightLimiter)
  • User-defined effects (nn::atk::EffectAux)
  • Sample data capture effects (nn::atk::EffectCapture)

File Structure

This sample program is located in Samples/Sources/Applications/AtkEffect/ .

Required Environment

Nothing specific.

How to Use

Operation instructions are displayed in the console.

Important Information

A log is displayed on the screen and on the console during execution, and immediately after startup this log is used to display the operation guide.

Note that in the PC version, a window does not open and the log is displayed only on the console screen.

Execution Procedure

Build the sample program and then run it.

250  virtual void ProcessSamples(const int32_t* pSamples, const ProcessSamplesArg& arg) NN_NOEXCEPT NN_OVERRIDE
251  {
252  std::lock_guard<decltype(m_Mutex)> lock(m_Mutex);
253  NN_ABORT_UNLESS_EQUAL(arg.readSampleCount % (arg.sampleCountPerAudioFrame * arg.channelCount), 0);
254  int readAudioFrameCount = arg.readSampleCount / (arg.sampleCountPerAudioFrame * arg.channelCount);
255  NN_ABORT_UNLESS_LESS_EQUAL(readAudioFrameCount, arg.audioFrameCount);
256 
257  for (auto frame = 0; frame < readAudioFrameCount; ++frame)
258  {
259  const int32_t* bufferBase = pSamples + frame * arg.sampleCountPerAudioFrame * arg.channelCount;
260  for (auto ch = 0; ch < arg.channelCount; ++ch)
261  {
262  const int32_t* pTargetSample = bufferBase + ch * arg.sampleCountPerAudioFrame;
263 
264  for (auto i = 0; i < arg.sampleCountPerAudioFrame; ++i)
265  {
266  auto floatSampleValue = static_cast<float>(pTargetSample[i]) / -std::numeric_limits<int16_t>::min();
267  m_Peak[ch] = std::max(m_Peak[ch], std::fabs(floatSampleValue));
268  }
269  }
270  }
271  m_ProcessedSampleCount += arg.sampleCountPerAudioFrame;
272  }

537 {
538  bool isSuccess = true;
539 
540  // Initialize SoundSystem.

542  std::size_t memSizeForSoundSystem = nn::atk::SoundSystem::GetRequiredMemSize( param );
543  g_pMemoryForSoundSystem = nns::atk::Allocate( memSizeForSoundSystem, nn::atk::SoundSystem::WorkMemoryAlignSize );
544  nn::atk::SoundSystem::Initialize(
545  param,
546  reinterpret_cast<uintptr_t>( g_pMemoryForSoundSystem ),
547  memSizeForSoundSystem );
548 
549  // Initialize SoundHeap.
550  g_pMemoryForSoundHeap = nns::atk::Allocate( SoundHeapSize );
551  isSuccess = g_SoundHeap.Create( g_pMemoryForSoundHeap, SoundHeapSize );
552  NN_ABORT_UNLESS( isSuccess, "cannot create SoundHeap" );

554  // Initialize SoundArchive.
555  const char* archiveAbsolutePath = nns::atk::GetAbsolutePath(ArchiveRelativePath);
556  isSuccess = g_SoundArchive.Open(archiveAbsolutePath);
557  NN_ABORT_UNLESS( isSuccess, "cannot open SoundArchive(%s)\n", archiveAbsolutePath );
558 
559  // Load the parameter data for SoundArchive into memory.
560  std::size_t infoBlockSize = g_SoundArchive.GetHeaderSize();
561  g_pMemoryForInfoBlock = nns::atk::Allocate( infoBlockSize, nn::atk::FsSoundArchive::BufferAlignSize );
562  isSuccess = g_SoundArchive.LoadHeader( g_pMemoryForInfoBlock, infoBlockSize );
563  NN_ABORT_UNLESS( isSuccess, "cannot load InfoBlock" );
564 
565  // Initialize SoundDataManager.
566  std::size_t memSizeForSoundDataManager = g_SoundDataManager.GetRequiredMemSize( &g_SoundArchive );
567  g_pMemoryForSoundDataManager = nns::atk::Allocate( memSizeForSoundDataManager, nn::atk::SoundDataManager::BufferAlignSize );
568  isSuccess = g_SoundDataManager.Initialize(
569  &g_SoundArchive,
570  g_pMemoryForSoundDataManager,
571  memSizeForSoundDataManager );
572  NN_ABORT_UNLESS( isSuccess, "cannot initialize SoundDataManager" );
573 
574  // Initialize the stream buffer for use with SoundArchivePlayer.
575  // This stream buffer must be allocated from the heap managed by the memory pool.
576  std::size_t memSizeForStreamBuffer = g_SoundArchivePlayer.GetRequiredStreamBufferSize( &g_SoundArchive );
577  g_pMemoryForStreamBuffer = nns::atk::AllocateForMemoryPool(memSizeForStreamBuffer);
578 
579  // Attach the specialized heap to the memory pool.

585  isSuccess = g_SoundArchivePlayer.Initialize(
586  &g_SoundArchive,
587  &g_SoundDataManager,
588  g_pMemoryForSoundArchivePlayer, memSizeForSoundArchivePlayer,

598 void FinalizeAtk()
599 {
600  // Clear the effect.
601  if ( !g_IsEffectClearing )
602  {
603  g_IsEffectClearing = PrepareEffectRemove( g_CurrentEffectType );
604  }
605  if ( g_IsEffectClearing )
606  {
607  // Wait until the effect can be deleted.
608  for ( ;; )
609  {
610  if ( CheckEffectRemovable( g_CurrentEffectType ) )
611  {
612  ClearEffect( g_CurrentEffectType );
613  g_IsEffectClearing = false;
614  break;
615  }
616  g_SoundArchivePlayer.Update();
617  }
618  }
619 
620  g_SoundArchivePlayer.Finalize();
621

Sample Program Description

The processes related the atk library in this sample program have the following flow. Places where this sample differs from the AtkSimple sample are marked as "additional."

  • Initialize the atk library (InitializeAtk()).
    • Perform initializations in AtkSimple.
  • Load the data needed to play wave sounds and sequence sounds in SoundHeap (LoadData()).
  • Main loop.
    • Process key operations.
      • Pause or play sounds.
        • Set the effect or main send volume (additional).
      • Change the type of effect applied (additional).
        • Add effects (AppendEffect()) and delete effects (ClearEffect()) (additional).
    • Update SoundArchivePlayer.
  • Finalize the atk library (FinalizeAtk()).

Effects can be applied to the sounds using key operations. The following types of effects can be applied.

  • Delay applied to the left and right channels (nn::atk::EffectDelay)
  • Delay applied to the right channel only (nn::atk::EffectDelay)
  • Reverb (nn::atk::EffectReverb)
  • I3DL2 Reverb (nn::atk::EffectI3dl2Reverb)
  • Lightweight Limiter (nn::atk::EffectLightLimiter)
  • User-defined effects that control the volume (nn::atk::EffectAux)
  • Sample data capture effects that get the peak value (nn::atk::EffectCapture)

To delete an effect, first use the PrepareEffectRemove() function to disable the effect. Next, use the CheckEffectClearable() function to check whether the effect can be deleted. Finally, delete the effect.
