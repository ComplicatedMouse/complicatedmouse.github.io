Detailed Description

Plays back and manages multiple sounds together.

Details
A sound actor is typically used in association with an in-game character or model. For example, one possible use is to change the sound orientation of the sound actor that is associated with a character to match the movements of that character.

The parameters set for a sound actor affect all sounds that the actor is playing.

Internally, a sound actor has four actor players. You can limit the number of sounds that can be simultaneously played by each actor player. If the configured sound count is exceeded, the player priorities that are set for each sound are compared, and the sounds with the lowest priority are stopped. The default values for the number of sounds that can be played back simultaneously are defined as follows. 

       - Actor Player 0 is unrestricted.
       - Others are 1.

The actor player in which to play a sound is set in the sound data. You can override this information at playback time by using nn::atk::SoundStartable::StartInfo.

Parameters by Output Line

SoundActor can have the following effects on sounds played on the sound actor.

Overall

  • Pitch: SetPitch.

TV output

  • Volume: SetVolume, StopAllSound, PauseAllSound
  • Pan: SetOutputPan.

In addition to these effects, these parameters can be affected by the following.

  • Effects of data specified by SoundMaker
  • SoundHandle or SoundPlayer
  • Effects of 3D sound calculations (Sound3DEngine))

◆ StopAllSound()

void nn::atk::SoundActor::StopAllSound ( int  fadeFrames)

Stops all sounds being played by the actor.

Details
Performs processing identical to that when stopping a sound by using the handle class.

You can fade out over the number of frames specified by fadeFrames. Fade-out is not performed when 0 is specified. However, for sounds playing as sequence sounds, the sounds stop after all attenuation has completed in the release stage of the envelope.

The volume control for fade-out is also used for fade-in. Number of elapsed frames during a fade out. The number of fade-out frames correlates to the speed of change from maximum volume to silence. If a fade out is specified during a fade in, or in other similar situations, it may complete sooner than the specified frame count.

Parameters
[in]fadeFramesNumber of elapsed frames during a fade-out. The number of frames is based on the number of calls to SoundArchivePlayer::Update. Specify a value of 0 or greater for the fadeFrames parameter. If you specify a negative value, the function works the same as if you had specified 0.

◆ SetPitch()

void nn::atk::SoundActor::SetPitch ( float  pitch)

Changes the pitch of the actor.

Details
The actor pitch affects all sounds that the actor is playing.

The value specified by this function operates independently of other pitch parameters, and they are all layered.

pitch specifies the frequency ratio. In other words, pitch is not affected if 1.0 is specified. Specifying 2.0 doubles the playback frequency, increasing the pitch by one octave. If 0.5 is specified, the pitch is one octave lower. The default value is 1.0.

Note that after this value is combined with the other pitch parameters, the final pitch is clamped to the range from nn::audio::VoiceType::GetPitchMin() to nn::audio::VoiceType::GetPitchMax(). For more information about combining pitch parameters, see the atk Library Sound Parameter Control Operations documentation.

Parameters
[in]pitchThe frequency ratio of the pitch change.

void nn::atk::SoundActor::SetBiquadFilter ( int  type,
float  value 
)

Changes the type and coefficient of the actor's biquad filter.

Details
The actor's biquad filter effect is applied to all sounds currently being played by the actor.

When the biquad filter is configured in multiple locations, the filter effect is not layered, but rather set to the configuration in the location with the highest priority.

type is the filter type, and it takes an enumerator of the BiquadFilterType enumerated type. In addition to the presets, you can also select a user-registered filter.

For type, specify a value between BiquadFilterType_Min and BiquadFilterType_Max. For more information about BiquadFilterType_Min and BiquadFilterType_Max, see BiquadFilterType.

value is the extent to which the filter is applied, specified in the range from 0.0f to 1.0f. The meaning of this value varies depending on the type of coefficient used by the filter.

When used together with the low-pass filter from SetLowPassFilterFrequency, the effects are overlaid.

Parameters
[in]typeThe filter type (BiquadFilterType_Min to BiquadFilterType_Max).
[in]valueHow to apply the filter (0.0f to 1.0f).

Sets the number of sounds that can be played at the same time.

Details
There are four actor players in a sound actor. You can set a limit on the number of sounds that can be simultaneously played by each actor player. This setting is independent of the simultaneous sound count that is set in the sound player.

If the number of sounds for playback exceed the configured sound count, the player priorities for each sound are compared, and the sounds with the lowest priority are stopped.

The initial value is unlimited for actor player 0, and 1 for all other actor players.
