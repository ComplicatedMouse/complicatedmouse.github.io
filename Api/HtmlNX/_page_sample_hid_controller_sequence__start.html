Overview

Describes how to call the controller support applet from the sample application start screen.

File Structure

This sample program is located in Samples/Sources/Applications/HidControllerSequence/ .

Required Environment

Pair the controller to the development hardware ahead of time.

How to Use

The controller support applet is called immediately after starting when no controller has been connected. The start screen appears when the controller is connected. You switch to the title screen by pressing a button on any of the connected controllers.

Important Information

Make sure that the controller is sufficiently charged before using it.

Execution Procedure

Build the application and run it.

Description

The following processing occurs at the Start screen.

  • Confirm that controllers are connected.
  • Confirm input for the connected buttons.
  • Change controllers.
  • Transition to the title screen.

The controller support applet is called at the Start screen under the following circumstances.

  • Neither nn::hid::NpadId::Handheld nor nn::hid::NpadId::No1.

If button input is received from a wireless controller other than nn::hid::NpadId::No1, the applicable controller replaces nn::hid::NpadId::No1 as the master player from that point.
