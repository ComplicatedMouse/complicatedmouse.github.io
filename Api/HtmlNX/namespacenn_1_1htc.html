Detailed Description

Namespace for host-target communication.

Feature Overview

Host target communication is abbreviated as HTC. The HTC library provides a means of communication with the host. You cannot ordinarily use HTC library features for a release build type. Define NN_ENABLE_HTC to use those features within a release build type.

How to Use

TargetManager.exe must be started before you can use the features of the HTC library. For more information about the connection procedure, see the manual.

Gets the size of buffer required to store the value of the target environment variable.

Parameters
[out]pOutValueThe obtained buffer size.
[in]variableNameName of the target environment variable.
Returns
The result of the operation.
Return Values:

ResultSuccess Succeeded.

ResultConnectionFailure Failure either to communicate with host or to load the target environment variable definitions file.

ResultNotFound The relevant target environment variable was not found.

Precondition
  • The library is initialized.
  • The specified target environment variable definitions file is in the correct format.
  • pOutValue != nullptr.
  • variableName != nullptr.

The target environment variable must have a terminating null character. If the target environment variable is a string type, the value returned as the buffer size is the size required including this terminating null character. This function always returns ResultConnectionFailure if the hardware environment is not connected to a host.

Result nn::htc::GetTargetEnvironmentVariable ( size_t *  pOutSize,
char *  outBuffer,
size_t  bufferSize,
const char *  variableName 
)

Gets the target environment variable value as a string.

Parameters
[out]pOutSizeBuffer size for the retrieved value.
[out]outBufferBuffer to store the retrieved value.
[in]bufferSizeSize of the buffer to store the retrieved value.
[in]variableNameName of the target environment variable to retrieve.
Returns
The result of the operation.
Return Values:

ResultSuccess Succeeded.

ResultConnectionFailure Failure either to communicate with host or to load the target environment variable definitions file.

ResultNotFound The relevant target environment variable was not found.

Precondition
  • The library is initialized.
  • The specified target environment variable definitions file is in the correct format.
  • pOutSize != nullptr.
  • outBuffer != nullptr.
  • variableName != nullptr.
  • - variableName is a string of no more than 64 characters.
  • bufferSize is at least as large as the value obtained by GetTargetConfigurationValueLength().

Details
The target environment variable must have a terminating null character. The obtained string is terminated with a null character and stored in pOutValue. The function can get strings up to a maximum of 2048 bytes, including the terminating null character. If the target environment variable definitions file defines a string that exceeds 2048 bytes, the function gets a value that is truncated to be 2048 bytes with a terminating null character. This function always returns ResultConnectionFailure if the hardware environment is not connected to a host.

◆ BindHostConnectionEvent()

void nn::htc::BindHostConnectionEvent ( nn::os::SystemEvent *  pOutValue)

Binds to an event object that notifies you that communication with the host has been established.

Parameters
[out]pOutValuePointer to an event object.
Precondition
  • The library is initialized.
  • pOutValue != nullptr.
  • *pOutValue is uninitialized.
Postcondition
  • *pOutValue is initialized.

Details
Binds to an event object that notifies you that communication with the host has been established. Events are set to be read-only and auto-cleared. You are constantly in a signaling state when connected to a host. That state is cleared upon disconnection. The event will be cleared by the system, so do not clear it on the user side. Nothing needs to be done in advance when it is time to destroy the event object initialized by this function.

Examples
HtcConnectionEvent/HtcConnectionEvent.cpp.

◆ RunOnHost()

Result nn::htc::RunOnHost ( int32_t *  pCommandResult,
const char *  command 
)

Runs the specified command on the host PC.

Parameters
[out]pCommandResultThe return value of cmd.exe /c "command."
[in]commandThe command to run.


Return Values:

ResultSuccess Succeeded.

ResultNotImplemented Target Manager could not recognize this command. Update Target Manager.

ResultOutOfMemory Memory was not allocated for running the process.

Precondition
  • The library is initialized.

Details
Executes the string specified by command using cmd.exe on the host PC. This function blocks until command processing is finished on the host. This function is thread-safe.

It is important to note that pCommandResult is not the return value of command itself. For example, if command is not a cmd.exe command, the value 9009 would be returned to cmd.exe, but the cmd.exe return value itself would be 1, so this function also returns 1.
