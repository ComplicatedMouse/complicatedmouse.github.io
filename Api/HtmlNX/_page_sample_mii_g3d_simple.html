This sample program shows the feature to assign Mii models to nn::g3d::ResModel.

Overview

This sample program uses nn::g3d::mii to assign a Mii model to nn::g3d::ResModel and displays that model. In this sample, the model data is optimized so that bone calculations are as light as possible. While it is suitable for rendering a lot of Mii characters, you cannot switch between the Mii rendering types (normal model and headwear model).

File Structure

This sample program is located in Samples/Sources/Applications/MiiG3dSimple/ .

Required Environment

Nothing specific.

How to Use

Nothing specific.

Important Information

To run this sample program, you must use the Mii library.

Execution Procedure

Build the sample program and then run it.

Description

This sample program assigns a Mii model to nn::g3d::ResFile, applies animation, and displays the Mii model. This sample program has the following flow.

Model Conversion Process

The following sets of source code are converted.

  • /Samples/Sources/Applications/MiiG3dSimple/Resources/Model/Mii.fmdb

Empty model data, with data structures that contain no vertices or indices. In the runtime, the Mii model is assigned to this.

  • /Samples/Sources/Applications/MiiG3dSimple/Resources/Model/MiiSimpleFaceExpressionChange.fmab

Material animation for changing the expression of the Mii. In the runtime, this animation is applied to the nn::g3d::ResModel to which the Mii model has been assigned.

  • /Samples/Sources/Applications/MiiG3dSimple/Resources/Shader/SampleShader.glsl

The shader for displaying the Mii.

Processes of the Sample Program

The process flow of the running sample program is presented below.

  • Construct a model Mii.
  • Assign the Mii model to nn::g3d::ResModel.
  • Create an instance of the nn::g3d model.
  • Create an instance of the nn::g3d material animation.
  • Begin the rendering loop.
  • Update the animation.
  • Update models.
  • Execute the command list.
  • Swap the scan buffer.
  • Return to the beginning of the rendering loop.
