Detailed Description

HTCS connection.

Details
This class represents a connection implemented using the htcs library.

HTCS can be used to communicate directly with a server started on the host-target. Before starting communications, you must complete the following steps.

  • Start the target manager.
  • Make the server register the port in Target Manager.

For more information, see the HTCS documentation.

Examples
WebSocketChat/WebSocketChat.cpp, and WebSocketEcho/WebSocketEcho.cpp.

Connects to the server.

Parameters
[in]pHostNameHost name.
[in]portNumberPort number.
[in]isSecureWhether to use SSL communication.
Returns
Returns a nn::Result that represents the process result.
  • nn::ResultSuccess The process succeeded.
  • nn::websocket::ResultNotSupported This feature is not supported.
  • nn::websocket::ResultOperationCanceled The connection has been canceled. (The cancel function returned true.)
  • nn::websocket::ResultCommunicationError, Communication error. This result will also be picked up by nn::websocket::ResultOperationCanceled, so be careful about the order of error handling. must be handled.
Precondition
  • pHostName != nullptr.
  • The port name for HTCS connections is already set.

Details
HTCS connections are a feature for communicating with a host-target, so the content of the parameters is ignored. The connection is established with the port name specified in the constructor or with SetPortName().

Attention
HTCS connections do not support SSL communication. This function returns nn::websocket::ResultNotSupported if true is specified for the isSecure parameter.
