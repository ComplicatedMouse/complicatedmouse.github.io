Changes in NintendoSDK NX Addon 7.3.0 (based on Generic 7.3.0)

Specification change that breaks your code

*7-3-0S01: (EC) Fixed a bug where the license information would be lost on rare occasions if the network connection was dropped while the license was being consumed.

On rare occasions, dropping the network connection while a license was being consumed (nn::ec::AsyncConsumeRightDataRequest) would create inconsistencies in the state of the license information between the client and the server, and the license information would disappear. This bug was fixed.
With this change, the nn::ec::ConsumableServiceItemManager::IsConsumptionRequired() function now returns true contingent on the outcome of license recovery (nn::ec::AsyncRecoverRightDataRequest). You must revise your application if your implementation depends on nn::ec::ConsumableServiceItemManager::IsConsumptionRequired() returning false after license recovery.

*7-3-0S02: (IRSENSOR) Changed the header structure that provides the Pointing features of the irsensor library.

Previously; the Pointing features were provided by the irsensor_PointingProcessor.h header included in irsensor.h, but now that header been separated from irsensor.h.
Note that as a result of this change, developers who have been using the Pointing features must now explicitly include the irsensor_PointingProcessor.h header.

*7-3-0S03: (LDN) Subdivided ResultImcompatibleVersion.

The new return values nn::ldn::ResultLowerVersionCannotUpdate and nn::ldn::ResultHigherVersionCannotUpdate have been added for the nn::ldn::Connect() function.

Previously, nn::ldn::Connect() returned nn::ldn::ResultLowerVersion or nn::ldn::ResultHigherVersion when the access point and the station had different local communication versions. Now those Result values are returned by the function only when the access point and the station have matching application IDs. If the application IDs do not match, the function now returns nn::ldn::ResultLowerVersion or nn::ldn::ResultHigherVersion.

Note that this change is not applied immediately. For applications built with NX Add-On 7.3.0 or later, the function will return these newly added Result values following a future firmware update. For applications that you plan to build with NX Add-On 7.3.0 or later, make sure that you add handling for these new Result values so your application can support the future firmware update.

This change will not affect your application if it does not perform local communication between titles with different application IDs, or if it does not adopt a local communication version mechanism.

*7-3-0S04: (TOOL) Moved the data for error detection in AuthoringTool from AuthoringToolKit to UnpublishableErrorKit.

The data that was included in the AuthoringToolKit for NX package for detecting errors when determining whether an NSP file meets the conditions for submission to Nintendo has been moved to the UnpublishableErrorKit for NX package.

Improvements

*7-3-0S11: (General) Reorganized the NintendoSDK package so now when you get only the SystemUpdater package you also get the Microsoft Visual C++ redistributable package, which you need for operating the tools.

The Microsoft Visual C ++ redistributable package included in the NintendoSDK package has been made a separate package.
Starting with this version, if you are using NDI you automatically get the Microsoft Visual C ++ redistributable package when you get either the NintendoSDK package or the SystemUpdater package.
Previously, when you got the SystemUpdater package the included tools sometimes did not work. This change resolves that problem.

*7-3-0S12: (OpenGL) Changed the timing of the planned removal of static links to OpenGL functions in the NX version.

Statically linking to functions in OpenGL ES 3.2 is currently deprecated in the NX version of OpenGL and will no longer be possible in NX Add-On 8.1.0 and later. Originally, it was to be ended in NX Add-On 8.0.0, but the timing has been changed to NX Add-On 8.1.0. In NX Add-On 8.1.0 and later, get function pointers dynamically. You can also use the nn::gll library or another OpenGL loader library such as GLEW to load these functions automatically.

*7-3-0S13: (TOOL) Added a feature to check for consistency of tool versions in the application and the patch.

Added a feature to AuthoringTool to check the version of the tool that was used when the NSP file was created for the application and for the patch.
An error is displayed when get-unpublishable-error is run if the NSP files for the application and the patch were created using different versions of the tool.
Also, the replace command fails if you run it to try to revise entries in those NSP files.
For more information about these commands, see Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

*7-3-0S14: (TOOL) Changed the specifications for ratings in the NMETA file and in AuthoringEditor.

In light of the following changes to the specifications for ratings, the specifications for ratings have been changed in the NMETA file and in AuthoringEditor.

  • For Japan’s CERO ratings, the age set for 審査予定 and 規定適合 was changed from 0 to 18.
  • For Taiwan’s GSRMR ratings, the age set for 審査中 was changed from 0 to 18.
  • For Australia’s ACB ratings, the age set for Check the Classification was changed from 0 to 18.
  • Ages have been standardized for the PEGI and PEGI Portugal ratings, so the entry field for PEGI Portugal has been deleted.

For more information, see NintendoSDK Documents > Features > Building > Creating Application Images > Writing the NMETA File > Rating.

*7-3-0S15: (TOOL) Creation of the NSP file now fails if the SDK used by the application and AuthoringTool have different major versions.

When AuthoringTool is used to create an NSP file, the process now fails if the major version of AuthoringTool is different from the major version of the SDK used by the application.

7-3-0S16: (FONT) Improved the error handling by BfttfMaker.

BfttfMaker error messages are now sent as standard errors, and when an error occurs the error level is set to a level other than level 0.

7-3-0S17: (BuildSystem) NintendoSdkVcProjectSettings.props no longer has the added /Zc:twoPhase- compile option.

When you build using Visual C++ 14.16 (Visual Studio 2017 15.9) for the Win32 and x64 platforms, the NintendoSdkVcProjectSettings.props file no longer has the added /Zc:twoPhase- compile option. This compile option was added as a workaround to a bug in Visual C++ 14.14 (Visual Studio 2017 15.7) and earlier.

7-3-0S18: (CODEC) Improved the processing speed of the DecodeAdpcm function.

Improved the processing speed of the nn::codec::DecodeAdpcm function.

* nn::codec::DecodeAdpcm()

7-3-0S19: (CODEC) Relaxed the preconditions for the nn::codec::DecodeAdpcm function.

The preconditions for the nn::codec::DecodeAdpcm function were relaxed in the following ways.
* The restrictions on <em>inputSize</em> and <em>sampleCount</em> were loosened to enable decoding multiple frames.
* The restrictions on <em>inputSize</em> and <em>sampleCount</em> were loosened to enable decoding data that is less than one frame in size.

* nn::codec::DecodeAdpcm()

7-3-0S42: (HID) Fixed a bug where the state of IsConnected in the attributes variable of the SixAxisSensorState structure did not match the state of the device.

For the attributes variable of the SixAxisSensorState structure, IsConnected is defined as the attribute value indicating that the sensor device is in the connected state. When a controller that is not equipped with the sensor device is connected to the system, this bit flag is expected to be false.
However, when a controller equipped with the sensor device was disconnected and an unequipped controller was then connected, the state sometimes did not properly reflect this change.

7-3-0S43: (MOVIE) Fixed a bug in MovieDecoderPlayer where an error would occur right after the start of the movie and the playback process would end.

Fixed a bug in MovieDecoderPlayer where the following kinds of errors would be added to the log immediately after the start of the movie, and the playback process would end.

ACodec: [OMX.Nvidia.vp9.decode] ERROR(0x80001020)
ACodec: signalError(omxError 0x80001020, internalError -2147483648)
MediaCodec: Codec reported err 0x80001020, actionCode 0, while in state 6

The program is implemented such that sometimes movie::Extractor::SeekTo is called from a different thread between the calls to movie::Extractor::Read and movie::Extractor::Advance immediately after the start of movie playback. In this case, an error can occur when the movie is being decoded. The implementation of the sample has been revised.

If the selection is changed by the Pre-Export Script, the Post-Export Script runs on the node that is the selected node after running the Pre-Export Script.
