Details

Changes in NintendoSDK NX Addon 9.6.0 (based on Generic 9.6.0)

Added Features

*9-6-0S01: (TOOL) Added a feature to SdkEnvironmentChecker for generating detailed information about the development environment.

Added a feature to SdkEnvironmentChecker for generating detailed information about the development environment. This information assists support by enabling the user's development environment to be shared smoothly.
This feature can be used together with the GUI version and the CUI version. The information includes the result of the check, information about the PC environment, information about the targets registered in Target Manager, information about NintendoSDK and other installed packages, and the set of log files created by the development hardware initialization and update tools.
For more information, see Checking the Suitability of the Operating Environment > Details Window, in addition to the descriptions of the --detail and --zip-path options.

Improvements

*9-6-0S02: (Compiler) Added an option to Clang that shortens the build time by skipping some dependency output information.

Updated Clang for NX to version 1.8.12.
This version includes the following change that helps to shorten build times.

  • Added a new compiler option called -fno-precompiled-file-deps, which skips over precompiled files in the dependency output. By specifying this option, precompiled files will not be checked, which promises to shorten build times.


9-6-0S03: (Compiler) Updated to Clang for NX 1.8.13.

Updated Clang for NX to version 1.8.13.
The main changes are as follows.

  • Added __llvm_profile_get_path_prefix() and __llvm_profile_get_filename() to profile/profile_interface.h for PGO.
  • When profile data is saved using __llvm_profile_set_filename(), the filename is now overwritten.
  • Improved the display of the debug information that is generated after abort() .
  • Updated to LLVM 8.0.1.
  • Fixed a bug where compiling files with large static buffers consumes a lot of memory.
  • Added the Clang tools clang-tidy.exe, clang-change-namespace.exe, clang-include-fixer.exe, and find-all-symbols.exe.
  • Added support for isatty().
  • Fixed a bug where the close() function tried to close the file handle a second time.
  • Fixed a bug where passing an invalid file descriptor to dup()/dup2() did not have the expected behavior.
  • Moved the declaration of nx_socket_init() from sys/socket.h to nx_socket.h.
  • Fixed a bug where a file descriptor was created if you opened an existing directory with open(O_CREAT | O_EXCL) even though -1 was returned.
  • Fixed a bug where some network header files were resolved in the wrong byte order.
  • Fixed a bug where the NRO instrumentation did not work with PGO. For more information about instrumentation, see <em>SDK PGO Additional Information</em>.
  • Fixed a race condition that occurred when a file descriptor was opened or closed.
  • Fixed a race condition that occurred with lseek.
  • Added support for some C++17 libraries. For more information, see the documentation.
  • Experimentally added support for ThinLTO.
  • Added support for dup() and dup2().
  • Fixed a race condition bug in pthread_exit().
  • Added support for POSIX network functions. For more information about the specific functions, see the documentation for POSIX Network Socket Functions.
  • Made revisions so that the -wide option for objdump works correctly.
  • Made revisions so that __nnmusl_pthread_rwlock_rdlock handles EDEADLK.
  • Fixed a bug where incorrect behavior occurred when NULL was passed to open_memstream().
  • Added support for clock_getcpuclockid(), clock_nanosleep(), and ftime().
  • Fixed a bug where errno was not set correctly when specifying an offset for lseek() that was out of bounds.
  • Fixed a bug where unnecessary header files were included in sys/types.h.


Bug Fixes

9-6-0S04: (BuildSystem) Adjusted the format of project files created using the wizard.

When project files (NintendoSDK application and NintendoSDK libraries) are created using the wizard, the WindowsTargetPlatformVersion property is now also registered in the files. (Only for the x86 and x64 builds.)

9-6-0S05: (TOOL) Fixed a bug where AuthoringTool unexpectedly detected differences when comparing NSP files.

Fixed a bug in the comparensp command of AuthoringTool where the process of comparing NSP files detected differences in the legal information file, even though there were no apparent changes to that file.

9-6-0S06: (TOOL) Fixed a bug where consistency between the SDK version and the AuthoringTool version was not checked when a subprogram was being created.

Fixed a bug where consistency between the SDK version and the AuthoringTool version was not checked when a subprogram was being created.
With this fix, creating the subprogram now fails if the SDK major version is higher than the AuthoringTool major version.

9-6-0S07: (TOOL) Fixed a bug where replacenspmeta succeeded even if you specified an NSP file that was created with a larger major version than the version of AuthoringTool you were using.

Fixed a bug where the replacenspmeta command succeeded even if the NSP file that was entered for the command was created with a larger major version than the version of AuthoringTool you were using.

9-6-0S08: (TOOL) Fixed the behavior of the AuthoringTool's replace command for applications with a configured ProgramIndex.

Fixed a bug where the AuthoringTool's replace command did not work correctly when run on an individual application with ProgramIndex set to a nonzero value.
