Details

Changes in NintendoSDK NX Addon 3.5.2 (based on Generic 3.5.2)

Added Features

3-5-2S01: (TOOL) Added the ability to import different NMETA files than those for the NMETA or NSP files in AuthoringEditor.

In AuthoringEditor, added the ability to import, using overwrite settings, another NMETA file while you are editing an application's NMETA or NSP file.

Improvements

3-5-2S02: (TOOL) Added the --error-unpublishable option to the createnspd command in the AuthoringTool.

Added the --error-unpublishable option to the createnspd command in the AuthoringTool.
For more information, see Features > Building > Creating Application Images > Tool Reference / AuthoringTool.

3-5-2S03: (TOOL) AuthoringTool has been improved to detect changes to the software legal information.

If it detects changes to the software legal information obtained from the Software Legal Information System (SLIM), AuthoringTool now returns an error when it is running.

3-5-2S04: (TOOL) Changed so that a warning is displayed when options are scheduled to be deprecated in AuthoringTool.

When running AuthoringTool, a warning now appears if the following options, which are scheduled to be deprecated, are specified.

  • --accessible-urls
  • --html-document
  • --legal-information
  • --icon
  • --nx-icon


For more information, see Features > Building > Creating Application Images > Tool Reference / AuthoringTool.

3-5-2S05: (TOOL) Deleted "DistributedByNintendo" from the logo data settings values in the NMETA file.

Deleted "DistributedByNintendo" from the logo data settings values in the NMETA file.
For more information about the values that can be set for logo data, see Features > Building > Creating Application Images > Writing the NMETA File.

Bug Fixes

*3-5-2S06: (IRSENSOR) Fixed a bug where regions, apart from the buffer specified by the application, were sometimes improperly written by the system when using Moment and Clustering modes.

Previously, a region, which would become a part of the buffer specified by the application, would sometimes be improperly rewritten when retrieving data using the Get function. This has been fixed.
For example, a portion of the nn::irsensor::MomentProcessorState structure is usually reserved for the nn::irsensor::GetMomentProcessorState function in the application, and passed to the function. However, there was a risk that the region next to that structure's region might also be rewritten by the system, resulting in two regions being rewritten together.
As a result of this fix, the workaround of reserving one extra portion of the structure is no longer required.

3-5-2S07: (FS) Updated FsAccessLogChecker to version 10.4.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 10.4. This includes the following changes.

  • Fixed a bug in the statistics for access logs in a check time interval boundary and for access logs with at least a minute's worth of gaps.
  • Fixed a bug where the next check time range would be off for access logs with at least a minute's worth of gaps.
  • Fixed a bug where some logs would be overlooked when there were multiple access logs for the same time.
  • Revised how the determination for read access overlapping occurs.
  • Fixed a bug where an exception would occur if a decimal value was set when specifying the --time_range option.
  • Added support to enable the checking passed load frequency values.
  • Added support to check file system access via nn::bcat.
  • Made revisions to check access logs after sorting them chronologically.
  • Made revisions to remove error access logs after checking them.


3-5-2S08: (VSI) Fixed a bug where VSI would enter an invalid state when downgrading the version of the NintendoSDK.

Fixed a bug where VSI would stop working normally when installing VSI for an earlier version of the NintendoSDK. This occurred for VSI versions 4.4.0 and beyond supporting NX Add-On 4.3.0 and beyond.
