Details

Changes in NintendoSDK NX Addon 4.4.0 (based on Generic 4.4.0)

Added Features

*4-4-0S01: (DevMenu) DevMenuApp is now installed when initializing the development hardware.

DevMenuApp, which previously needed to be installed separately, is now installed when initializing the development hardware.
This change also applies to ProdMode and makes it easier to install applications.

*4-4-0S02: (OE) Added a feature for temporarily enabling or disabling the crash report creation feature.

Added a function for temporarily enabling or disabling the crash report creation feature. Use this function to disable crash report creation in scenes in which sensitive data could potentially be captured. For more information, see the documentation and guidelines.

*4-4-0S03: (TOOL) Added a feature for getting crash information for crashes on retail devices.

You can now get information about application crashes that occur on retail devices. For more information, see NintendoSDK Documents > Features > System > Crash Reports.

4-4-0S04: (DevMenu) Added a feature for specifying a TV underscan value.

Added a feature in DevMenu and DevMenuCommand for specifying a TV underscan value. This feature is available on the Device tab in DevMenu and is accessible via the settings command in DevMenuCommand.

4-4-0S05: (MOVIE) Added support for multiple language tracks to the MoviePlayerSimple sample.

Added support for multiple language tracks to the MoviePlayerSimple sample. The sample plays the audio matching the current language setting on the system.

4-4-0S06: (MOVIE) Changed the minimum supported sampling rate in the MovieDecoderPlayer and MoviePlayerSimple samples.

Changed the minimum supported sampling rate in the MovieDecoderPlayer and MoviePlayerSimple samples to 16 KHz. Resources with a sampling rate of less than 16 KHz can no longer be played.

4-4-0S07: (MOVIE) Transitioned to using the native texture mode in the default rendering process in the MovieDecoderPlayer and MoviePlayerSimple samples.

Transitioned to using the native texture mode in the default rendering process in the MovieDecoderPlayer and MoviePlayerSimple samples.

4-4-0S08: (TOOL) Added a feature for importing NMETA files into other NMETA files and NSP files in AuthoringEditor.

In AuthoringEditor, added a feature for importing other NMETA files into the application NMETA file or NSP file that is currently being edited (this overwrites the existing file).

4-4-0S09: (TOOL) Added a save data-related item to AuthoringEditor.

The following save data-related settings can now be edited in NMETA files in AuthoringEditor.

  • UserAccountSaveDataSizeMax
    • The upper expansion limit on the save data region for user account save data.
  • UserAccountSaveDataJournalSizeMax
    • The limit on expansion of the journaling region for user account save data.
  • TemporaryStorageSize
    • The temporary storage size.
  • CacheStorageSize
    • The size of the cache storage save data region.
  • CacheStorageJournalSize
    • The size of the cache storage journaling region.


4-4-0S10: (TOOL) Added new crash report settings to AuthoringEditor.

You can now edit the following crash report settings (in the CrashReport element) in the NMETA file using AuthoringEditor.
For more information about crash reports, see NintendoSDK Documents > Features > System > Crash Reports.

4-4-0S11: (Keycode processing) Added a feature for removing the undefined portions of character code sequences during assembly.

Added the nn::kpr::KeyCodeComposer::Remove() member function. This function removes the undefined portions of character code sequences during assembly.
For more information, see the manual.

Improvements

*4-4-0S12: (NVN) Added support for several GPUs to NVIDIA Graphics Driver.

Added support for the following GPUs to NVIDIA Graphics Driver.

  • GeForce TITAN Xp - GP102
  • GeForce GTX 1080 Ti - GP102
  • GeForce GTX 1050 Ti - GP107
  • GeForce GTX 1050 - GP107
  • Geforce GT 1030 - GP108
  • Quadro P6000 - GP102
  • Quadro P4000 - GP104
  • Quadro P2000 - GP106
  • Quadro P1000 - GP107
  • Quadro P600 - GP107
  • Quadro P400 - GP107
  • Quadro GP100 - GP100


*4-4-0S13: (TOOL) Improved an issue where InitializeEdev failed to initialize the EDEV.

Improved an issue where InitializeEdev failed to initialize the EDEV.
If the EDEV is turned OFF or USB communication between EDEV and the host PC is interrupted while InitializeEdev is running, you can now restart InitializeEdev and then successfully initialize the EDEV.
This change only applies to situations in which you are using InitializeEdev to reinitialize an EDEV that was previously initialized using the version of InitializeEdev in NX Add-On 4.4.0 or later.
It has no effect when using the version of InitializeEdev in NX Add-On 4.4.0 to reinitialize an EDEV that was previously initialized using the version of InitializeEdev from a version of NX Add-On prior to 4.4.0.

4-4-0S14: (DevMenu) Reduced the time it takes to display a list of save data.

Reduced the amount of time it takes until the save data list is displayed when there is large save data present.

4-4-0S15: (DevMenu) SDEV units that need to have their memory data repaired now display a message indicating this.

On SDEV units that need to have their memory data repaired, "Needs Memory Repair" is now displayed on the Debug tab in DevMenu.
For more information, see DevMenu / Memory Mode / When an SDEV Requires Memory Data Repairs.

4-4-0S16: (FS) Added a feature for checking the read frequency region to FsAccessLogChecker 10.7.

Version 10.7 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) supports outputting read regions to the SameAddressRead category.

4-4-0S17: (OS) Improved the memory information displayed in the default user exception handler.

Added the following two items to the memory information displayed in the default user exception handler.

  • U, which indicates uncached memory (nv memory pool, etc.).
  • L, which indicates that nn::os::IsMemoryLocked() == true.


4-4-0S18: (TOOL) Added a feature for filtering resource data files included in NSPD files.

When creating NSPD files, you can now filter out which resource data files are included in the same way as when creating NSP files.

4-4-0S19: (TOOL) Added an option for getting the memory mode to DevMenuCommand.

Added the debug get-memory-mode option, which gets the current memory mode, to DevMenuCommand.

4-4-0S20: (TOOL) Added support for disabling execution of the development hardware system update tool from different firmware modes.

You can now disable execution of the development hardware system update tool (SystemUpdaterSdev) from different firmware modes.
If you are currently in a different firmware mode, use the initialization tool to initialize the development hardware in the desired mode.

4-4-0S21: (TOOL) Changed several options and added a new feature in the integrated command-line tool (TargetShell).

Changed how some options are specified in the integrated command-line tool.

  • Changed how some options in which a target must be specified are specified.
  • Added a feature for exporting, importing, and resetting settings files.

For more information, see the manual.

4-4-0S22: (TOOL) Deleted the DistributedByNintendo option in the LogoType element in AuthoringEditor.

Deleted the DistributedByNintendo option in the logo data settings that can be configured when editing NMETA files for applications in AuthoringEditor.

4-4-0S23: (TOOL) Updated the PEGIPortugal age classifications in AuthoringEditor.

Removed the "4 years or older" category from the PEGIPortugal age classifications in the AuthoringEditor rating information entry form. Use the new "3 years or older" category from now on.

Bug Fixes

*4-4-0S24: (DevMenu) Fixed a bug where DevMenuApp could not be started in PROD MODE.

Fixed a bug where DevMenuApp could not be started in PROD MODE.

*4-4-0S25: (TOOL) Fixed a bug in AuthoringEditor where the NMETA import feature could not be used while editing NSP files.

Fixed a bug in AuthoringEditor where the NMETA import feature could not be used while editing NSP files.
This fix makes it possible to import and apply the content of existing NMETA files even while an NSP file is open.

4-4-0S26: (APPLET) Fixed a bug where audio from an application sometimes was not stopped when a library applet was started from the application.

Fixed a bug where audio from an application sometimes was not stopped when a library applet was started from the application.

4-4-0S27: (AUDIO) Fixed a bug in nn::audio::GetActiveAudioDeviceChannelCountForOutput().

Fixed a bug where the nn::audio::GetActiveAudioDeviceChannelCountForOutput() function did not get channels correctly.
Specifically, the function returned 2 as the return value even when surround output was enabled.
However, even when this happened, the sound was still correctly output in surround mode.

This bug occurs in NintendoSDK 4.1 to 4.3.

4-4-0S28: (AUDIO) Fixed a bug where the specified device volume was not applied to audio renderers.

Fixed a bug where the specified device volume sometimes was not applied to audio renderers.
More specifically, when the device volume was set before at least 5 ms had elapsed since the nn::audio::StartAudioRenderer() function was called, sometimes the device volume was not correctly applied to the audio output from the audio renderer.


4-4-0S29: (DevMenu) Fixed a bug with starting applications for a selected user.

Fixed a bug where, when an application was started under the following conditions, the selected user information was not passed to the application.

  • Another application was already running.
  • The application was started after first selecting the user in DevMenu.


4-4-0S30: (FS) Fixed a bug where access logs were not output when an invalid mount name was specified.

Fixed a bug where access logs were not output when an invalid mount name was specified.

4-4-0S31: (FS) Fixed a bug where the thumbnail image region in save data was sometimes lost.

Fixed a bug where, when an application using temporary storage was closed, the thumbnail image region in existing user account save data was sometimes lost.

4-4-0S32: (FS) Fixed the behavior of the authoring tool for when a directory that does not exist is specified for the --cache-directory option.

Previously, when creating a patch with the authoring tool, if the directory specified for the --cache-directory option did not exist, an error was returned. Fixed this so that now, the specified directory is created and the process is successful.

4-4-0S33: (FS) Updated the input file character encoding detection process in FsAccessLogChecker 10.6.

There was a bug in version 10.2 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) that caused character encoding detection to fail for some encodings. This bug has been fixed in version 10.6.

4-4-0S34: (HID) Fixed a bug where the nn::hid::SetNpadAssignmentModeToSingle() function succeeded even when NpadStyleJoyLeft and NpadStyleJoyRight were both disabled.

There was a bug in NX Add-On 4.0.0 and later where the nn::hid::SetNpadAssignmentModeToSingle() function for changing Joy-Con assignments succeeded even when the Npad operation styles for single-controller grip modes were disabled.
Also fixed the nn::hid::SetNpadAssignmentModeToDual() function so that it does not succeed when the NpadStyleFullKey and NpadStyleJoyDual styles for dual-controller grip modes are disabled.

4-4-0S35: (IRSENSOR) Fixed a bug in Add-On 3.x and earlier where calling the Run function when switching between applets while using the IRSENSOR library sometimes caused the system to abort.

Fixed a bug where, while using the IRSENSOR library in Add-On 3.x, calling the Run function when returning the focus from an applet such as the controller support applet to the application sometimes caused the system to abort.

4-4-0S36: (IRSENSOR) Fixed a bug where dynamically calling the Run function repeatedly in image transfer mode caused the hid process to abort on a memory access violation.

Fixed a bug where depending on the call timing, dynamically calling the Run function repeatedly in image transfer mode caused the hid process to abort on a memory access violation in rare cases.

4-4-0S37: (IRSENSOR) Fixed a bug where quickly switching between parameters in image transfer mode caused the hid process to abort.

Fixed a bug where, in rare cases, quickly switching between parameters in image transfer mode caused the hid process to abort.

4-4-0S38: (IRSENSOR) Fixed a bug where the controller state could not be correctly obtained if the controller was disconnected and reconnected while using the IR Motion Camera.

Fixed a bug in NX Add-On 4.3.0 where the controller state could not be correctly obtained if the controller was disconnected while being used and was then reconnected.
Also fixed a problem where, due to this bug, in Windows builds the controller state stopped being obtained correctly even when the controller was not disconnected.

4-4-0S39: (IRSENSOR) Fixed a bug where, when using multiple IR Motion Cameras, finalizing just one of the handles caused the library to enter an invalid state.

Fixed a bug where, when working with multiple IR Motion Camera handles, if the nn::irsensor::Finalize function was called on just one of the handles and then you attempted to call nn::irsesor::StopImageProcessor on one of the other valid handles, the library failed an assertion.

4-4-0S40: (IRSENSOR) Fixed a bug with broken compatibility of the nn::irsensor::StopImageProcessor function in the SDK for NX Add-On 3.x and earlier.

Fixed a bug with broken compatibility of the nn::irsensor::StopImageProcessor function in the SDK for NX Add-On 3.x and earlier. This bug caused the system to abort when this function was called from an applet such as the controller support applet immediately after returning from an application.

4-4-0S41: (MOVIE) Fixed a bug in the MovieDecoderPlayer sample where, when playing looped videos, DevMenu was displayed between loops.

Fixed a bug in the MovieDecoderPlayer sample where, when playing looped videos, DevMenu was displayed between loops.

4-4-0S42: (MOVIE) Fixed a bug where the MovieDecoderPlayer sample crashed when attempting to load videos with very small or very large vertical or horizontal resolutions.

Fixed a bug where the MovieDecoderPlayer sample crashed when attempting to load videos with very small or very large vertical or horizontal resolutions.

4-4-0S43: (MOVIE) Fixed a bug where the MovieDecoderPlayer sample sometimes returned invalid memory values for corrupted media.

Fixed a bug where the MovieDecoderPlayer sample sometimes returned invalid memory values when the media file that was played was corrupted.

4-4-0S44: (MOVIE) Fixed a bug where using a CPU pass to render video data that had an extremely low resolution caused data corruption.

Fixed a bug where using a CPU pass to render video data that had an extremely low resolution caused data corruption. This bug was due to the value specified in the set bufferAlignment command.

4-4-0S45: (OE) Fixed a bug where, under certain conditions, the application startup logo continued to be displayed.

Fixed a bug where, under certain conditions, when a library applet was started from an application that was currently displaying a startup logo, the startup logo continued to be displayed.

4-4-0S46: (Socket) Fixed an issue where nn::socket::Finalize() did not return when SSL/HTTPS was used.

There was a timing issue which could prevent calls to nn::socket::Finalize() from returning when a TCP connection on a socket was lost while configuring an SSL/HTTPS connection.

4-4-0S47: (TOOL) Fixed a bug where the development hardware update tool failed when run on hardware with the NX Add-On 4.0.0 firmware.

Fixed a bug where the development hardware update tool (SystemUpdateSdev) failed when run on hardware with the NX Add-On 4.0.0 firmware.

4-4-0S48: (TOOL) Fixed a bug where the setup process failed in environments without the Microsoft Visual C++ 2015 runtime.

Fixed a bug where an error occurred when attempting to install the SDK in environments in which the Microsoft Visual C++ 2015 runtime (such as the Microsoft Visual C++ 2015 Redistributable package) was not installed.

Changes in NintendoWare NX Addon 4.4.0 (based on Generic 4.4.0)

Added Features

4-4-0W01: (3D-G3D) Added a feature for resetting just the parameters that are modified by material animations.

Added a feature (nn::g3d::MaterialAnimObj::RevertTo()) for resetting just the parameters that are modified by the following types of animations in material animations.

  • Shader parameter animations
  • Texture pattern animations
  • Material visibility animations


If you have multiple material animations that modify the same parameters, be mindful of the order and timing of applying the material animations and resetting the parameters.

4-4-0W02: (LayoutTool) Added a feature to LayoutEditor for changing the font while maintaining the font size scale.

Normally, when the font is changed in LayoutEditor, the size scale is automatically adjusted to keep the overall appearance the same. Added a new feature for changing the font while maintaining the font size scale.

To enable the feature for maintaining the size scale when changing fonts, add <KeepingSizeScaleEnabled>true</KeepingSizeScaleEnabled> to your project settings file.
The size scale is also maintained when layouts are re-opened in LayoutEditor after directly replacing a font file with another file of the same name.

Improvements

4-4-0W03: (LayoutTool) Filters defined in FFNT files are now ignored when displaying data in LayoutEditor.

When displayed LayoutEditor, filters defined in FFNT files are now ignored, and all of the characters are displayed.

Bug Fixes

4-4-0W04: (AudioTool) Fixed a bug in SoundMaker where no warning icon was displayed even after setting a maximum average loudness value.

Fixed a bug where, even after configuring a maximum average loudness value in the Project Settings dialog box, no warning icon was displayed for items exceeding that value.

4-4-0W05: (AudioTool) Fixed a bug in SoundMaker where the average loudness value was sometimes incorrect.

Fixed a bug where doing the following sometimes caused the average loudness value to be calculated incorrectly.

  • Clicking the Play button in the In-Game Editing panel while SoundMaker was not connected to an application.
  • Selecting Preview > Measure Reference Count of Instrument Waveform from the menu.
  • Selecting Preview > Measure Maximum Number of Voiced Sounds for Sequence Sound from the menu.


4-4-0W06: (FONT) Fixed a bug in FontConverter where, when converting FFNT files to BFFNT files, parameters that do not take a value were not reflected if specified.

Fixed a bug in FontConverter where, when converting FFNT files to BFFNT files, parameters that do not take a value were not reflected if specified. This occurred because the conversion result cache managed by FontConverter was not handled correctly. As a result, in some cases, when data was converted using a command in which only the parameters that do not take a value were different, the previously cached results were output.

4-4-0W07: (FONT) Fixed a bug where an error occurred when FCPX files did not include a scalable font.

Fixed a bug where when FCPX files did not include a scalable font, an error occurred during initialization.

4-4-0W08: (FONT) Fixed a bug where FontConverter did not convert data correctly if an uninstalled font was specified for -iface.

Fixed a bug where FontConverter did not convert data correctly if an uninstalled font was specified for -iface.

Also, previously, when the font for the face name specified for -iface could not be loaded, the data was converted using a system font. Now, FontConverter displays an error instead.

4-4-0W09: (FONT) Fixed a bug where, when strings including per-character transformation tags were displayed with the Divide Based on Character Width setting enabled, the interval between characters was strange.

Fixed a bug where, when strings including per-character transformation tags were displayed with the Divide Based on Character Width setting enabled, the interval between characters was strange.

To implement a custom tag processor, a process that returns renderable character positions must be implemented overriding the AcquireNextPrintableChar function. For more information about how to do this, see Sources\Libraries\font\font_ExtendedTagProcessorBase.cpp.

4-4-0W10: (LayoutTool) Fixed a bug in LayoutEditor when textures that were not a power of two in size were not clamped correctly when displayed.
