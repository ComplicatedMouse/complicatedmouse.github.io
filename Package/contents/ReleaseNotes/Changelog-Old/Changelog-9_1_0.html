Details

Changes in NintendoSDK NX Addon 9.1.0 (based on Generic 9.1.0)

Specification change that breaks your code

*9-1-0S01: (GFX) Changed the ImageFormatProperty structure used for the output of the GetImageFormatProperty function.

Changed the ImageFormatProperty structure used for the output of the GetImageFormatProperty function, adding the linearPropertyFlags variable to the structure to indicate the properties of a linear tiling texture.
In addition, added ImageFormatPropertyFlag_DepthStencil to the enumerations of ImageFormatPropertyFlag to indicate whether a texture can be used as a depth stencil.

*9-1-0S02: (WEB) Changed the path for offline HTML.

To open offline HTML from an SD card using WebCallerCommand, create a folder named NXWebApplet in the
root directory of the SD card and create a folder named html-document under this NXWebApplet folder.
For more information, see Features > Library Applets > Web Applet and Offline Web Applet > Information Common to Both > WebCallerCommand > Offline HTML File Paths.

*9-1-0S03: (WEB) Updated the web browser included in the web applet.

Updated the web browser included in the web applet. The web browser now supports such new web standards as ECMAScript 6 and CSS Grid Layout. Detailed documentation will be provided at a later date.
Due to the updates to the web browser specifications, in rare cases web content that was created for the web applet prior to the updates might not display correctly in the web applet after the updates. If this happens, deal with the situation by updating the web content.
As for the web browser that is included in the Offline Web Applet, compatibility is important so this web browser was not updated and remains the same as before.

Added Features

*9-1-0S04: (TOOL) Added an auto-test assist tool.

Added a tool for assisting the creation of automated tests for applications.
The auto-test assist tool facilitates the creation of scripts and tests to automate tasks, such as for getting screen captures from the development hardware, getting and searching logs, and entering HID operations.
For more information, see Auto Test Assist Tool.

*9-1-0S05: (VSI) Added a feature to Target Manager for filtering the logs.

Added a feature to Target Manager for filtering the logs. The feature uses the same syntax as LogViewer. For more information, see the Target Manager > Log.

*9-1-0S29: (VSI) When you make a release build in Visual Studio, the -fno-omit-frame-pointer compile option is now specified by default.

The -fno-omit-frame-pointer option must be specified when building the application in order to backtrace using the diag library, but for release builds, the -fomit-frame-pointer option was previously specified by default.
To avoid confusion among developers, the -fno-omit-frame-pointer compile option is also now specified by default for release builds in Visual Studio.

Added a feature (the --continue-on-error option) to the 3DTextureConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) so that when the tool is being used with the --job-list option it continues with processing even if errors occur in some jobs.

Added a feature to the 3DTexturePresetConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTexturePresetConverter.bat) for converting only the input file ending in the lowest number when converting to 3D textures or array textures. (The minimum_number_only attribute of the <preset_assign> element.)

Added a feature (the --continue-on-error option) to the 3DTexturePresetConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTexturePresetConverter.bat) to continue with processing even if errors occur during the conversion of some image files.

9-1-0W05: (3DTool) Added a setting to the config file in the Maya plug-in to export cameras that have the visibility attribute disabled.

Added a setting to the config file in the Maya plug-in to set whether to export cameras that have the visibility attribute disabled.
This setting is specified as follows.
export_invisible_camera="true"

9-1-0W06: (3DTool) Added an option to the 3dShaderConverter for deleting unnecessary shader cache.

Added the --remove-unused-shader-cache option to the 3dShaderConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) for deleting unnecessary shader cache. If you specify both this option and the --shader-cache-directory option, all cache other than the shader cache that has been loaded or written is deleted at the end of conversion.
This is designed for use with build machines and servers. For example, you can efficiently delete the cache that is unnecessary for the current environment by combining these options with the shader variation (FSV) file that contains all variations and options like --skip-convert and --force-variation.

9-1-0W07: (3DTool) Added config file keywords to the environment variables that can be obtained with pre-export and post-export scripts in the Maya plug-in.

Added a number of config file keyword values to the environment variables that can be obtained with pre-export and post-export scripts in the Maya plug-in.

9-1-0W08: (3DTool) Added the material reference settings for samplers to the default settings of 3DEditor.

Added the material reference settings for samplers to the default settings of 3DEditor.

Improvements

9-1-0W10: (3D-G3D) Improved the performance of communications in the viewer library.

Improved the performance of communications with 3DEditor in the viewer library. Previously, even if the nn::g3d::viewer::ViewerServer::Poll() function was called, communications waited for the nn::g3d::viewer::ViewerServer::ExecuteCommands() function to finish operations required for processing in the main thread. For operations that do not require waiting, communications now proceed if successive calls are made to nn::g3d::viewer::ViewerServer::Poll(). With this change, if nn::g3d::viewer::ViewerServer::Poll() is run at intervals shorter than the interval for nn::g3d::viewer::ViewerServer::ExecuteCommands(), operations sent from 3DEditor are now applied faster than before in some cases.

9-1-0W11: (3DTool) Changed the maximum size of the log that can be saved in the Debug Log window of 3DEditor.

The maximum size of the log that can be saved in the Debug Log window of 3DEditor has been increased from 100,000 characters to 1 million characters.

9-1-0W12: (3DTool) Changed the specification in 3DEditor for referencing sampler structures when there are multiple parent materials.

Changed the specification in 3DEditor for referencing sampler structures when there are multiple parent materials.
When Do not allow additional proprietary sample is selected for any of the parent materials, the sampler structures are now referenced for all of the parent materials.
Before this change, the sampler structure was referenced for only the highest propriety parent material among those for which Do not allow additional proprietary sample was selected.

9-1-0W13: (3DTool) Improved the operations of the animation time slider in 3DEditor.

The playback cursor in the animation time slider in 3DEditor has been adjusted for easier operation using a mouse.
In addition, when the mouse cursor is set on an animation in the time slider, the starting and ending positions of the animation are now shown in bold.

9-1-0W14: (3DTool) Improved the response to operations in 3DEditor for projects with many files.

Improved the response to operations in 3DEditor for projects with many files.

9-1-0W15: (3DTool) The Maya Import plug-in has been changed such that normals are now not locked if the values are the same as Maya's automatic calculations.

The Maya Import plug-in has been changed such that normals are now not locked if the values are the same as Maya's automatic calculations.
However, depending on the shape, the normals might be locked even you import an FMD file that was exported using Maya's automatically calculated normals.

9-1-0W16: (3DTool) The Maya plug-in for disabling node compression can now be set for the transform node of camera, light, and environment fog.

The Maya plug-in for disabling node compression can now be set for the transform node of camera, light, and environment fog.
If true has not been specified for export_invisible_camera in the config file, a camera is not exported if the visibility attribute is disabled, but by using this setting to disable node compression, the camera is exported.
Also, by using this setting to disable node compression you can export lights that have the template attribute disabled and environment fogs that have the visibility attribute disabled.

9-1-0W17: (3DTool) You can now use an environment variable in the 3DTexturePresetConverter to specify the number of files with filenames ending in numbers equal to or larger than the number of the input file.

For the 3DTexturePresetConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTexturePresetConverter.ba), when the discrete_number attribute of the <preset> element is true, the value set for the INPUT_FILE_DEPTH environment variable is taken as the number of files that have the same filename prefix as the input file but end in numbers equal to or larger than the number of the input file. Files are counted even if the ending numbers are not in sequential order.

9-1-0W18: (3DTool) You can now view the skeletal animation binary export settings in 3DEditor.

You can now view the skeletal animation binary export settings in 3DEditor. You can check the settings in the curve page of the property window.

9-1-0W19: (LayoutTool) Added settings related to the addition of segments to the LayoutEditor project configuration file.

Added settings related to the addition of segments to the LayoutEditor project configuration file. You can now set the default for creating an associated group when a segment is added, the default for automatically updating the types of animation to export, and the default for automatically updating the groups created when segments are added in separated mode. For more information, see Project Configuration File Format.

9-1-0W20: (LayoutTool) Improved the panel for checking playback in the Segment Tags window in LayoutEditor.

Improved the panel for checking playback in the Segment Tags window in LayoutEditor. Added a feature for recording playback in separated mode as the default.

9-1-0W21: (LayoutTool) In LayoutEditor, a key segment is now added when you paste panes between layouts created in separated mode.

In LayoutEditor, when you copy a pane in a layout in separated mode and paste it to a different layout also in separated mode, a segment with the same name as the key segment is now added if a segment with that same name does not already exist. When a group is set for the added segment, and the group with the same name is not in the layout to which the pane was pasted, the group is also added.

9-1-0W22: (LayoutTool) LayoutEditor no longer displays an error dialog box when the --save-thumbnail option has been specified.

When LayoutEditor was started with the --save-thumbnail option, an error message was displayed in some cases (such as because a parts layout was not found) and the process did not advance. Such error dialog boxes are no longer displayed.
