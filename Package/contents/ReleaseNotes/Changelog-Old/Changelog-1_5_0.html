Details

Changes in NintendoSDK NX Addon 1.5.0 (based on Generic 1.5.0)

Specification change that breaks your code

*1-5-0S01: (VI) Deleted some vi functions.

The following functions, specified as NN_DEPRECATED, have been deleted from vi_Display.private.h and vi_Layer.private.h

  • nn::vi::SetDisplayEnabled
  • nn::vi::GetDisplayResolution
  • nn::vi::CreateLayer(Layer*, Display, int, int, PixelFormat)
  • nn::vi::SetLayerAlpha


Added Features

1-5-0S02: (MII) Added database files for debugging.

Several sample Mii database files have been added for the convenience of development.
For more information, see Sample Database For Debugging.

1-5-0S03: (NGC) Added interfaces to the nn::ngc::ProfanityFilter class and the nn::ngc::CountNumbers() function that enable the use of UTF-16 strings as input.

Added interfaces that accept UTF-16 strings as input to the following functions.

  • nn::ngc::ProfanityFilter::CheckProfanityWords()
  • nn::ngc::ProfanityFilter::MaskProfanityWordsInText()
  • nn::ngc::CountNumbers()


Improvements

1-5-0S04: (GFX) Changed the nvn version to cache the GetGpuAddress result.

Made changes so that, in the nvn version, nvn is not called when calling nn::gfx::Buffer::GetGpuAddress and result obtained in advance is returned.
The size of nn::gfx::Buffer increased by 8 bytes in accordance with this change.

1-5-0S05: (TOOL) Added check items to the --error-unpublishable option for AuthoringTool.

Added whether to check the validity of the legal information to the check items when specifying --error-unpublishable and using AuthoringTool.

1-5-0S06: (TOOL) Improved the rating information input items in AuthoringEditor.

Revised the rating information input to be easier to understand.
After the region or sales method has been selected in AuthoringEditor, only input items for rating organizations required for that sales format are displayed.

Bug Fixes

1-5-0S07: (GFX) Fixed a bug where the nns_gfx library could not be built.

Fixed a bug where the nns_gfx library depended on the NintendoWare package and could not be built using only the NintendoSDK package.

1-5-0S08: (MOVIE) Fixed the memory leak in SoftAAC2.

Fixed a bug where some of the memory allocated in SoftAAC2 was not released after the end of video playback.

Changes in NintendoWare NX Addon 1.5.0 (based on Generic 1.5.0)

Specification change that breaks your code

*1-5-0W01: (3D-G3D) Updated the binary resource file format.

Adjusted the binary formats for BFRES files and BFSHA to maintain the fullest compatibility with future binary resource files. This change breaks the compatibility of binary files created using previous versions of 3D Binary Converter and 3D Shader Converter and so resource files must be reconverted.

Added Features

1-5-0W02: (AudioTool) Added Save project to the command features of SoundMaker.

Added a command to save the project to the command features that can be specified by the SoundMaker command-line options.

Bug Fixes

1-5-0W03: (AudioTool) Fixed a bug in SoundProjectWaveExporter that caused downsampling to fail.

Fixed a bug in SoundProjectWaveExporter where if a project file (FSP file), in which downsampling was enabled, was specified, the downsampling process would always fail and the tool would display an error and close.
