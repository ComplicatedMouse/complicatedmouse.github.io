Details

Changes in NintendoSDK NX Addon 9.5.0 (based on Generic 9.5.0)

Specification change that breaks your code

*9-5-0S01: (VSI) Made improvements to VSI and Target Manager.

  • VSI
    • Changed the VSI settings for PGO.
      • You can no longer specify the path for generating profraw from VSI. To specify a path, call __llvm_profile_set_filename from within the program.
      • Manually converting profraw files to profdata is now required.
      • Added properties to specify the profdata path.
      • For more information about using PGO, see SDK PGO Supplemental Information.
  • Target Manager
    • Deprecated nn::tm::GetAgentVersion().
      • To get the development hardware version, use nn::tm::GetFirmwareVersion() instead.


Added Features

*9-5-0S02: (General) Added Visual Studio 2019 support.

Added support for the following features in Visual Studio 2019.

  • VSI
  • VsHelp
  • Wizard
  • NintendoSDKVsExtension


Note that builds using MSVC v142 do not support these features.
For more information, see Important Information When Using Visual Studio 2019.
Operation with NintendoSDK is verified using Visual Studio 2019 Professional Edition, described on the Operating Environment page.

*9-5-0S03: (TOOL) Added support in Authoring Editor for changing application control data for NSP files created with an earlier major version of NX Add-On.

You can now edit application control data in Authoring Editor for NSP files created with an earlier NX Add-On with a different major version.
For more information about application control data, see Writing the NMETA.

*9-5-0S04: (VSI) Started supporting Visual Studio 2019.

  • Started supporting Visual Studio 2019.
    • Going forward, you can use Visual Studio 2019 as an IDE for development.


9-5-0S05: (DevMenu) Added functionality to DevMenuCommand for deleting rights of titles purchased on eShop.

Added the following commands to DevMenuCommand.

  • shop delete-all-rights <user_account_index> (Delete all rights for the specified account.)


For more information about these commands, see Features > Controlling the Development Hardware > DevMenuCommand.

9-5-0S06: (DevMenu) Added functionality to set production profile images for user accounts.

Added functionality to set production profile images for user accounts.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Managing Accounts > Edit profile image.


9-5-0S07: (VSI) Added VSI features.

  • VSI
    • Added official support for the Edit and Continue feature.
      • Started officially supporting the Edit and Continue feature. With this feature, changes to code can be applied while the application is running.
      • For more information, see the manual.


Improvements

*9-5-0S08: (TOOL) Added support in Authoring Tool for changing application control data for NSP files created with an earlier major version of NX Add-On.

You can now use the replacenspmeta command in Authoring Tool to fix application control data in NSP files created with an earlier NX Add-On with a different major version.
With this change, it is now possible to set strings containing Simplified Chinese, Traditional Chinese, and Hangul characters in the title of NSP files created with NX Add-On 4.x and earlier.

Bug Fixes

9-5-0S09: (BuildSystem) Fixed a bug where the path character encoding became scrambled in the wizard, causing project creation to fail.

Fixed a bug where, depending on the PC, character encoding for the path information became scrambled in the wizard for creating NintendoSDK application and Nintendo SDK library projects, causing project creation to fail.

9-5-0S10: (DevMenu) Fixed a bug where there were problems with handling multibyte characters when exporting save data to a destination other than the SD card of the development hardware.

Fixed a bug where, when exporting save data with DevMenuCommand or BackupSaveData to a destination other than the SD card of the development hardware, multibyte characters in file or directory names were unnecessarily replaced by default characters.

9-5-0S11: (FS) Updated FsAccessLogChecker to version 29.1.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 29.1. This version includes the following changes.

  • Added support for adding access logs for enabling global file data caching in the default nninitStartup() for NX Add-On 10.0.0.
  • Added support for changing the access log for nn::fs::MountSaveDataForDebug() in NX Add-On 10.0.0.
  • Added support for access log output for RamDisk in NX Add-On 10.1.0.
  • Added support to improve access log output in NX Add-On 10.2.0.
  • Fixed a bug where some types of nn::bcat access logs were judged to be Log Warnings.


9-5-0S12: (TOOL) Fixed a bug where the tool for getting crash reports sometimes crashed.

Fixed a bug where the tool for getting crash reports sometimes crashed if the ID of the thread where the exception is thrown was invalid.

9-5-0S13: (VSI) Fixed bugs for VSI and Target Manager.

  • VSI
    • Fixed a bug where adding a string containing an escape character into the AdditionalOptions metadata in the project file caused the escape character to be parsed incorrectly.
  • Target Manager
    • Fixed a bug where nn::tm::CreateTargetProgramDump() was not executed if the target was not the default.
    • Fixed a bug where Target Manager sometimes crashed when displaying tooltips.
    • Fixed a bug where screenshots sometimes could not be taken from Target Manager in environments where the network speed is slow.
    • Fixed a bug where changing the target name or IP address from the SDEV Settings in the Target Manager target list failed.
