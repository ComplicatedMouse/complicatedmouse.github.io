Details

Changes in NintendoSDK NX Addon 7.0.0 (based on Generic 7.0.0)

Specification change that breaks your code

*7-0-0S01: (GFX) The GL version of the gfx library now uses the nn::gll library.

Previously, the glew library was used to load OpenGL functions in the GL version of the gfx library. Now, the nn::gll library is used for this purpose instead.
Due to this change, the nn::gll library must now be linked in the GL version of the gfx library.
Also, it is no longer necessary to link the glew library.

*7-0-0S02: (RO) Modified the handling of duplicate global variables when unloading NRO modules.

In the previous version of the nn::ro::UnloadModule function, global variable symbols were resolved again when unloading a module. As a result, links to any duplicate symbols were changed in the remaining modules.
As a result, when some modules were unloaded, the symbols referenced by the remaining modules changed in unexpected ways, which created various bugs that were difficult to track down and fix.
Symbols are now restored to an unresolved state when a module is unloaded to prevent this type of confusing access to unexpected symbols.
This change has no effect in environments with no duplicated global variables.

*7-0-0S03: (TOOL) Removed the --swizzle option from Texture Converter.

Removed the --swizzle option from Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe).
Although continuing to specify this option does not cause any errors, the specified values are not applied in the binary texture files.

*7-0-0S04: (UTIL) Fixed nn::util::IntrusiveListMemberNodeTraits and nn::util::IntrusiveDicMemberNodeTraits to no longer allow incomplete types to be specified.

Previously, it was possible to pass incomplete types to nn::util::IntrusiveListMemberNodeTraits and nn::util::IntrusiveDicMemberNodeTraits, which is a violation of C++ guidelines. Fixed this bug to now produce an error.
If your code is now returning errors due to this change, change the order in which the types passed as template parameters are defined to produce complete types, or use nn::util::IntrusiveListBaseNodeTraits or nn::util::IntrusiveDicBaseNodeTraits instead.

As a temporary workaround, you can specify a fourth template parameter (such as nn::util::IntrusiveListMemberNodeTraits<ItemBase, &ItemBase::m_ListNode, ItemBase, 0>) to return to using the old behavior. However, this workaround does not always work, so implement support for the preceding approach as soon as possible.

*7-0-0S05: (UTIL) Memory regions are now initialized when assigned to nn::util::BitArray.

When a memory region is assigned to an nn::util::BitArray, that region is now initialized.

Due to this change, determination processes in the following functions may not always work correctly.
* nn::util::BitArray::all()
* nn::util::BitArray::none()
* nn::util::BitArray::any()

The contents of the assigned memory are indeterminate, so saving and restoring state using the persistence features might not work.
To explicitly persist the state, use nn::util::BitArray::iterator or similar.

*7-0-0S06: (Accounts) Added the new nn::account::TryOpenPreselectedUser() function, and changed the support status of the old nn::account::OpenPreselectedUser() function to scheduled for deprecation.

Added the new nn::account::TryOpenPreselectedUser() function, and changed the support status of the old nn::account::OpenPreselectedUser() function to scheduled for deprecation.
As described in Writing the NMETA File, when the IsOptional option is specified in StartupUserAccountOption, the presence of a user account selection made before the application is started can change dynamically. The API specification was changed to better support this behavior.

Improvements

*7-0-0S18: (OpenGL) Deprecated static linking of OpenGL functions on NX.

Currently, OpenGL for NX allows you to statically link functions that depend on the unextended version of OpenGL ES 3.2. This practice is now deprecated. In future releases, it will no longer be possible to statically link these types of functions. From now on, implement linking by getting function pointers dynamically. You can also use the nn::gll library or another OpenGL loader library such as GLEW to load these functions automatically.

*7-0-0S19: (TOOL) Added support for saving resource data redirect feature settings on the system.

Resource data redirection settings configured using DevMenuCommand are now persisted across system restarts and remain in effect until they are reconfigured.

*7-0-0S20: (VSI) Added a feature for checking the version when installing VSI and Target Manager.

A version check is now performed when installing VSI and Target Manager, and the installation is only allowed to proceed if the version of the installer included with that SDK is newer than the version that is already installed.

*7-0-0S21: (WEB) Changed the handling of offline HTML paths in the raw format (NSPD) to be case-sensitive.

Changed the handling of offline HTML paths in the raw format (NSPD) to be case-sensitive to match the case-sensitive handling of offline HTML paths used in the NSP format.

7-0-0S22: (AUDIO) Improved the precision of audio renderer volume processes and mix volume processes.

Improved the precision of audio renderer volume processes and mix volume processes so that the volume can now be changed more smoothly during volume change operations.

Due to this fix, waveform rendering results may sound slightly different than those obtained using an earlier version of the firmware.

*7-0-0W01: (3D-G3D) Modified the attributes of memory pools used when nn::g3d::ResFile::Setup() is called.

Previously, memory pools maintained by nn::g3d::ResFile had attributes of nn::gfx::MemoryPoolProperty_CpuUncached and nn::gfx::MemoryPoolProperty_GpuCached. The following attributes are now specified for these types of memory pools.
nn::gfx::MemoryPoolProperty_CpuCached and nn::gfx::MemoryPoolProperty_GpuCached

This change has the following effects.

  • The CPU can now access values in vertex buffers and index buffers faster than before.
  • If the CPU or GPU updates vertex buffers or index buffers stored in the resources region, subsequent processes may be unable to correctly read or write values in these buffers.


If you encounter situations in which values cannot be read or written correctly, check whether either of the following functions have been called.

  • nn::gfx::Buffer::FlushMappedRange()
  • nn::gfx::Buffer::InvalidateMappedRange()

*7-0-0W49: (Audio-ATK) Fixed a bug where the panning curve used in surround mode was different from the panning curve used in stereo mode.

Fixed a bug where the panning curve used in surround mode was different from the panning curve used in stereo mode.
The panning curve used in surround mode is now the same as the panning curve used in stereo mode.

More specifically, when you specify nn::atk::OutputMode_Surround in a call to nn::atk::SoundSystem::SetOutputMode, the sound volume is now increased if the pan value becomes less than -0.5f or greater than 0.5f.
Note that if you play stereo sound with dual panning specified as the panning mode, the pan value also satisfies these conditions in processes and cause the volume to be increased in the same way.

This bug was also reproducible in NW4F, so a NW4F-compatible mode has also been prepared.
For more information about porting NW4F data, see the API reference for the following function.

  • nn::atk::SoundSystem::SoundSystemParam::enableCompatiblePanCurve


For more information about panning curves in NW4F-compatible mode, see the NW4F-Compatible Mode Panning Curves section in NintendoWare > Audio > Guide > Sound System Manual > Pan Operations.
