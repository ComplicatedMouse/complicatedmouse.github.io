Details

Changes in NintendoSDK NX Addon 3.3.0 (based on Generic 3.3.0)

Specification change that breaks your code

*3-3-0S01: (IRSENSOR) Reverted the supported controller firmware version to Firmware 3.48.3.b

While bug fixes and added features related to controllers were implemented between NX Add-On 3.0.0 and NX Add-On 3.2.0 by updating the controller firmware, the only compatible controller firmware from NX Add-On 3.3.0 onwards is an earlier firmware.
The supported firmware version is Firmware 3.48.3.b.
Note that this change rolls back the following features from NX Add-On 3.2.0.
■ IR Motion Camera

  • Improved the data delay level, down to the order of two frames, during Moment and Clustering modes.
  • Updated the gestures library.
  • Fixed a bug where after the IR Motion Camera was started, the settings were not applied for several frames.
  • Fixed a bug where the data loss rate increased when a controller was reconnected after being disconnected, and while the IR Motion Camera was in use.
  • Added the ability to quickly switch modes and parameters as the processor is running.
  • Added a feature for checking whether a controller firmware update is required before starting the IR Motion Camera.

■ Other

  • Improved the firmware update speed for the Joy-Con (R) controller.


These removed features will be added in future SDKs.

Added Features

3-3-0S02: (BCAT) Added a working sample for using the data delivery feature

Added a working sample for using the data delivery feature.
The sample describes the cautions for use and appropriate error handling.

3-3-0S03: (DevMenu) Added a development feature for passing launch parameters to the application

Added a feature to pass files installed to the SD card, as launch parameters to the application.

For more information, see the Installing and Uninstalling Launch Parameters section in the DevMenu manual.

3-3-0S04: (WEB) Added a feature to disable pointer operations

Added the following function to disable pointer operations in the web applet.

  • nn::web::ShowWebPageArg::SetPointerEnabled


3-3-0S05: (WEB) Added a feature to enable JS extension

Added the following functions to enable JS extension in the web applet and the Offline Web Applet.

  • nn::web::ShowWebPageArg::SetJsExtensionEnabled()
  • nn::web::ShowOfflineHtmlPageArg::SetJsExtensionEnabled()


3-3-0S06: (WEB) Added a function to change the URL description to domain names

Added the following function to change the URL display in the web applet to the domain name.

  • nn::web::ShowWebPageArg::SetDisplayUrlKind()


3-3-0S07: (WEB) Added a startup feature that is specifically for video

Added the following function to perform startup, specifically for video for the offline web applet.

  • nn::web::ShowOfflineHtmlPageArg()


Improvements

*3-3-0S08: (AOC) Changed the encryption key for the add-on content NSP

Changed the encryption key used internally by the add-on content NSP from NX Add-On 3.2 onwards.

If add-on content that was submitted to the Nintendo Developer Portal is rebuilt in NX Add-On 3.X, the hash of the created content will be changed. Note that errors may occur when different content with the same version is registered.

When submitting content created with NX Add-On 3.X, make sure not to include non-updated content submitted in previous SDK versions in the same NSP.

3-3-0S09: (UTIL) Fixed an issue where a warning occurred from nn/util/detail/util_MatrixApi.neon.h when the compile option -Wfloat-conversion was enabled

Fixed an issue where a warning occurred from nn/util/detail/util_MatrixApi.neon.h when the compile option -Wfloat-conversion was enabled.

Bug Fixes

3-3-0S10: (APPLET) Fixed a bug where some applications could not exit correctly when being terminated from the menu

Fixed a bug where when using firmware with NX Add-On 3.0.0 and later, some applications could not be terminated correctly from the HOME Menu or DevMenu.

3-3-0S11: (AUDIO) Fixed a bug where DSP resource distribution would not work correctly when simultaneously using multiple audio renderers

There was a bug where DSP resource distribution would not work correctly when simultaneously using multiple audio renderers. This has been fixed.

Specifically, when the application simultaneously used multiple audio renderers, and the total DSP processing time for each audio frame exceeded 3500 microseconds, the application would sometimes use more than 3500 microseconds.

3-3-0S12: (FS) Fixed the mounting operation count in FsAccessLogChecker version 8.1

There was a bug where the mounting target in the access log was identified incorrectly and the loading operation was counted in FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe). This was fixed in FsAccessLogChecker version 8.1.

3-3-0S13: (HID) Added processing to stop the vibration when executing UnsetVibrationDeviceHandle

Added processing to the vibration wheel to stop the vibration when executing UnsetVibrationDeviceHandle.

3-3-0S14: (HID) Fixed a bug where attaching the controller to the console rail, while using the IR Motion Camera sensor in wireless communication mode, caused the IR to remain powered on

There was a bug in version NX Add-On 3.1.0 and later, where connecting the right controller to the rail, while using the IR Motion Camera sensor, would cause the IR to not power off. This has been fixed.

3-3-0S15: (LDN) Changed the nn::ldn::DefaultLocalCommunicationId definitions

nn::ldn::DefaultLocalCommunicationId was previously defined as 0, but has been changed to 0xFFFFFFFFFFFFFFFF in this SDK.

With this change, applications built using nn::ldn::DefaultLocalCommunicationId in NX Add-On 3.1.0 and NX Add-On 3.2.0, may no longer work in firmware containing NX Add-On 3.3.0 and later. Rebuild with NX Add-On 3.3.0.

3-3-0S16: (TOOL) Fixed a bug where some of the UI relating to rating offices, was not displayed correctly in AuthoringEditor

Fixed a bug where there were fewer options for the PEGIBBFC rating in AuthoringEditor, than described in the list in Writing the NMETA File.

3-3-0S17: (TOOL) Fixed a bug where the comparison results would be incorrect for URL lists that can be accessed from the web applet, when comparing NSP files in AuthoringEditor

There was a bug where, when comparing application NSP files in AuthoringEditor, the comparison results were incorrect for URL lists accessible from the web applet (AccessibleUrlsFilePath element in the NMETA file).

3-3-0S18: (TOOL) Fixed a bug where the intermediate file tool could not correctly load the intermediate files

When the NintendoWare package was not installed, sometimes an error occurred or the file could not be loaded correctly when using the intermediate file tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\IntermediateFile~.exe). This bug has been fixed.

Changes in NintendoWare NX Addon 3.3.0 (based on Generic 3.3.0)

Bug Fixes

3-3-0W01: (Layout-UI2D) Fixed a bug where the textures used in the Overwrite Textures Only feature, were treated as unused

Fixed a bug where the textures used in the Overwrite Textures Only feature, were treated as unused.

3-3-0W02: (LayoutTool) Fixed a bug where textures without any changes, would also get updated when the -u option was specified for LayoutConverter

There was a bug where textures without any changes, would also get updated when the -u option was specified for LayoutConverter. Though the <file_info> was previously included in FTXB files output by LayoutConverter, the file is no longer included.

3-3-0W03: (SPY) Fixed a bug where the Play button became enabled when data was closed

Fixed a bug where in Spy.exe, when Close All was clicked to close all data after opening data files or receiving data from the application, the Play button became enabled.
