Changes in NintendoSDK NX Addon 3.0.0 (based on Generic 3.0.0)

*3-0-0S04: (NVN) Breaking Changes in NVN

Made the following breaking changes in NVN.

The behavior of nvnWindowBuilderSetNativeWindow has changed on the Windows reference implementation, which could potentially break existing Windows NVN applications. In this version of NVN, passing a valid HWND as the native window parameter will cause the driver to create an internal GL context to swap buffers. Applications that create their own "donor" GL context must pass NULL as the native window parameter. If any applications create their own GL context and pass an NVWD as the native window parameter, there could be rendering corruptions as both the internal GL context and the application's GL context might conflict with each other. These applications must be modified to either pass NULL as the native window, or stop creating their own GL context and pass HWND as the native window to let the driver create its own internal GL context. For more information, see the API documentation of nvnWindowBuilderSetNativeWindow.

Fixed a bug where a memory overflow sometimes occurred if the size of the working memory was calculated incorrectly, and memory that did not satisfy the correct alignment restrictions was passed to the nn::font::RectDrawer::Initialize() function. In the new version, work buffers require that any passed memory satisfies the alignment restrictions. An assertion failure now occurs when incorrectly aligned memory is passed.
