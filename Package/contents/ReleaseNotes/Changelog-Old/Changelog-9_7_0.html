  • 9-7-0S07: (TOOL) Added multi-program applications to the NSP files that can be created for secure startup.
  • 9-7-0S08: (TOOL) Added support to AuthoringEditor for editing downloadable content settings that are required at startup (RequiredAddOnContentsSet).
  • 9-7-0S09: (TOOL) Added the --dummy option to the makesecure command of AuthoringTool to hide the application's information.

  • 9-7-0S17: (TOOL) Fixed a bug where there was a mismatch error in the major versions of the SDK and AuthoringTool if the application control data was modified in the NSP file.

Details

Changes in NintendoSDK NX Addon 9.7.0 (based on Generic 9.7.0)

Added Features

*9-7-0S01: (EC) Added a feature to the dummy EC applet for displaying a warning message if the running application uses a high amount of CPU.

Added a feature to the dummy EC applet for displaying a warning message if the running application uses a high amount of CPU.
If a warning message displays, see EC applet in NintendoSDK Documents and fix your application.

9-7-0S02: (EC) Added a feature to the AOC item list page in the EcCatalog sample to display installation information obtained from aoc_indexes.

A feature was added to get information from aoc_indexes in the screen after selecting AOC item list in the EcCatalog sample, and display the obtained AOC installation information.

9-7-0S03: (EC) Added a feature to the EcCatalog sample to get preorder sales information.

Added a feature to the EcCatalog sample to get all sale status information by specifying onsale_only = false when getting the AOC Item List and Related Title/Bundle List information.

9-7-0S04: (EC) Added processing to support the addition of monthly subscription tickets to the Subscription Service Item List page in the EcCatalog sample.

With this added feature, when Subscription service item list is selected in EcCatalog sample, the screen following the one where the list was selected supports displaying monthly subscription tickets, and displays course type information.

9-7-0S05: (HID) Added the functionality to assign the same function as the Capture Button to other buttons.

Added the nn::hid::SetNpadCaptureButtonAssignment() function to assign the same function as the Capture Button to other buttons.
This feature is designed to be used in games with special methods of operation where it is difficult to operate the Capture Button on the Joy-Con (L).

9-7-0S06: (TOOL) Added a feature to create NSP files for secure startup of patches and downloadable content.

Added a feature to AuthoringTool so you can use the makesecure command to create NSP files of patches and downloadable content for secure startup.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


9-7-0S07: (TOOL) Added multi-program applications to the NSP files that can be created for secure startup.

The makesecure command can now be used to create NSP files for secure startup of multi-program applications.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


9-7-0S08: (TOOL) Added support to AuthoringEditor for editing downloadable content settings that are required at startup (RequiredAddOnContentsSet).

Added support to AuthoringEditor for displaying and editing downloadable content settings that are required at startup (RequiredAddOnContentsSet), which has been supported since NX Add-On 10.3.
For more information about the NMETA elements, see Writing the NMETA File.

9-7-0S09: (TOOL) Added the --dummy option to the makesecure command of AuthoringTool to hide the application's information.

Added the --dummy option to the makesecure command of AuthoringTool to hide the application's information.
The application information that gets hidden are the title, publisher name, and icon.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


Improvements

9-7-0S10: (EC) Expanded the character types that can be displayed in the EcCatalog sample.

Added support for kanji (Traditional Chinese and Simplified Chinese), Hangul, and some symbols to the EcCatalog sample.

9-7-0S11: (TOOL) Revised AuthoringTool so you can now omit the Core element in the NMETA file used as input for the replacenspmeta command.

Revised AuthoringTool so you can now omit the Core element in the NMETA file that is specified for the --meta option of the replacenspmeta command.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Revise Application Control Data for NSP Files


9-7-0S12: (TOOL) You can now create NSP files for secure startup from NSP files that were created with earlier versions of the SDK.

The makesecure command for creating an NSP file for secure startup now succeeds even if the major version of the SDK that was used to create the input NSP file is earlier than the major version of AuthoringTool. Previously, the major versions needed to be the same for both.

Bug Fixes

9-7-0S13: (DevMenu) Fixed a bug where changes to the focus state of the application were not notified when the system was in sleep mode.

Fixed a bug where DevMenu did not report nn::oe::MessageFocusStateChanged to the application if the application was in any focus state other than background focus when the system entered sleep mode. With this fix, the behavior of the HOME Menu and DevMenu are now the same when notifying the application of changes in the focus state.

9-7-0S14: (TOOL) Fixed a bug where it was possible to downgrade the version of the SDK or AuthoringTool when creating a patch.

Fixed a bug where it was possible to downgrade the version of SDK or AuthoringTool when creating a patch. An error now occurs in the following situations.

  • The version of AuthoringTool used when creating the patch is earlier than the SDK version or the AuthoringTool version used when creating the revised application.
  • The SDK version of the revised application is earlier than the SDK version of the previous version of the application.

For more information, see the following.

  • Features > Patches > Building and Running a Patch > Creating the Patch.


9-7-0S15: (TOOL) Fixed a bug where patch creation with AuthoringTool failed.

Fixed a bug where the message "[Error] Failed to Build RelocatedIndirectStorageBuilder 0x001F4002" was displayed and the creation of the patch NSP file failed on rare occasions, depending on where the delta from the previous version was located.

9-7-0S16: (TOOL) Fixed a bug where patches could not be created for multi-program applications.

There was a bug where using a different DESC file for each sub-program in a multi-program application resulted in the following error because only a single file was passed to the --desc option of the makepatch command.
Wrong desc file. Acid in the desc file and main.npdm don't match.
To address this issue, multiple DESC files are now passed to the --desc option of the makepatch command when you create a patch for a multi-program application.

9-7-0S17: (TOOL) Fixed a bug where there was a mismatch error in the major versions of the SDK and AuthoringTool if the application control data was modified in the NSP file.

Fixed a bug where a mismatch error occurred in the major versions of the SDK and AuthoringTool when the NSP file was loaded in AuthoringEditor if the application control data had been modified. This bug applied to the following NSP files.

  • NSP files subjected to the replacenspmeta command in AuthoringTool
  • NSP files whose application control data was edited in AuthoringEditor
