Details

Changes in NintendoSDK NX Addon 5.2.0 (based on Generic 5.2.0)

Specification change that breaks your code

*5-2-0S01: (GFX) Deprecated the root signature features.

The nn::gfx::RootSignature class and related features are now deprecated. From now on, use the binding functions such as nn::gfx::CommandBuffer::SetTextureAndSampler to bind shader resources.

*5-2-0S02: (TOOL) Added a feature in the AuthoringTool for checking accessible URLs.

Improvements were made so that when the --error-unpublishable option is specified in the AuthoringTool, an error occurs if something other than ^https:// or ---- is set in the accessible URL.
For more information, see Features > Library Applets > Web Applets and Offline Web Applets > Web Applets.

*5-2-0S03: (UTIL) Updated zlib from version 1.2.8 to 1.2.11.

Updated the version of the zlib package used in the SDK from 1.2.8 to 1.2.11. Increased the size of the necessary work buffers by 8 to 16 bytes as part of this change. If you are using any features that depend on the zlib package provided in the SDK or the Compress module in nn::util, you must recompile.

*5-2-0S04: (Album) Album image and video files saved using the firmware in NX Add-On 5.0 or 5.1 can no longer be referenced.

Album image files and video files saved using the firmware in NX Add-On 5.0 or 5.1 are no longer compatible with the firmware in NX Add-On 5.2 and as a result can no longer be accessed in NX Add-On 5.2. (Question marks (?) are now displayed if you attempt to access such files from the album system feature.)
Album image and video files saved using the firmware in NX Add-On 4.x and earlier can still be accessed normally.

Added Features

*5-2-0S05: (AOC) Made the on-card downloadable content feature public.

The on-card downloadable content feature for distributing downloadable content via game cards rather than via Nintendo eShop is now public.
For more information, see aoc Library > On-Card Downloadable Content.

*5-2-0S06: (General) Added support for writing multiple applications to a game card.

Previously, it was only possible to write a single application to a game card. Starting in NX Add-On 5.2.0, you can write multiple applications to a game card.
For more information, see Features > NX Development Cards > Multi-Application Cards.

For more information about specific usage of this utility, see the AudioEffectAuxUtility sample.

As part of this change, also added nn::audio::SetReverbParameters, which is now also used in the existing nn::audio::ReverbType class.
This structure is used to batch-update settings for ReverbType objects.
