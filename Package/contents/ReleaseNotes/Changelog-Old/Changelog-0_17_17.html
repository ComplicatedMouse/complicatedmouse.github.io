Details

Changes in NintendoSDK NX Addon 0.12.17 (based on Generic 0.17.17)

Specification change that breaks your code

*13-18S01: (IRSENSOR) Fixed a bug in applications that use multiple IR Motion Cameras, where if you transition to an applet, and then return to the application with controller connections that exceed the upper limit on the number of controllers, an abort would occur.

When using multiple IR Motion Cameras, for example when you have four controllers and two controllers are using IR Motion Cameras at the same time, if you transition to the menu or elsewhere and connect an additional controller and then return to the application with five controllers connected, even if the application conforms to the limit on the number of controllers, a bug would cause the application to abort.
Changed the specification to avoid this problem. When a data retrieval function is called when the limit on IR Motion Camera connections is violated, the Hid Resource Error is now output to the log in the library and the nn::irsensor::ResultIrsensorNotReady error is returned.
This error creates a state in which data is not received, the same as before, so there is no need to change the handling policy.
This fix avoids the abort, but if the upper limit on the number of controllers is not complied with when entering the same problematic sequence, only one IR Motion Camera will work after resuming the application. If you are using multiple controllers, consider the setting on the upper limit of controllers.
In the future, we plan to release a fix so that an error is not returned even when entering this problematic sequence.

*13-18S02: (OE) Deleted the definitions for performance configurations that are scheduled to be deprecated.

Deleted the definitions for the following performance configurations that can be set using nn::oe::SetPerformanceConfiguration because they are scheduled to be deprecated.

  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu307MhzEmc1600Mhz
  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1600Mhz


Added Features

13-18S03: (TOOL) Added the ability to specify environment variables using DataPath from nmeta.

Added the ability to specify an environment variable for DataPath in the nmeta option, when creating downloadable content. Disabled by default.
You need to set UseEnvironmentVariable="true" for the DataPath attribute to enable it. For more information, see AOC Library / Creating Add-On Content.

Improvements

*13-18S04: (General) Removed SDEV1.6 from the list of hardware that can be initialized using InitializeSdev.

The SDEV1.6 can no longer be initialized using InitializeSdev.
Also, note that although SDEV1.6 can still be selected in the UI as the hardware to initialize, selecting this option will not actually initialize the hardware.

13-18S05: (BuildSystem) Changed the display order for the SDK root selection list box in the wizard.

Reversed the display order of the SDK root selection list, such that the sort is based on the latest install date, in the project creation wizard for NintendoSDK applications and NindendoSDK libraries.

13-18S06: (TOOL) Added the get-unpublishable-error option and the --error-unpublishable option check item to AuthoringTool.

Added whether to use DebugAPI to the check item when specifying --error-unpublishable and using AuthoringTool.
Added the ability to check whether DebugAPI is in use by specifying the get-unpublishable-error option using AuthoringTool as well.

13-18S07: (TOOL) Improved Redundant Processing in InitializeEdev

The system restart process in InitializeEdev is no longer needed and so it has been omitted.

Bug Fixes

13-18S08: (BuildSystem) Fixed a bug where an SDK installation failed in environments that included parentheses in user names.

Fixed a bug where using the wizard to install the SDK would fail when logging in with a user that included parentheses in the user name.

13-18S09: (General) Fixed a bug where offline HTML content could not be accessed when the NSPD file was started.

The problem was that when the path to the offline HTML content was specified in the NMETA file, that path was not being applied.

13-18S10: (TOOL) Fixed a bug where InitializeEdev would fail as it ran.

Fixed a bug where the <em>Parse GetSerialNumber output failed.</em> error was exported to the error log and InitializeEdev would fail.

13-18S11: (TOOL) Fixed a bug where the default Target Manager target would change after initializing using the setup tools.

Fixed a bug where the default Target Manager target settings would not change to the development hardware, even though specifications indicate that the settings should change, after initializing using the InitializeSdev and InitializeEdev setup tools.

13-18S12: (TOOL) Fixed a bug where the GUI versions of the setup tools would throw an exception.

Implemented a workaround to address the appearance of an exception window when using the GUI versions of the InitializeSdevWin and InitializeEdevWin setup tools.

Changes in NintendoWare NX Addon 0.12.17 (based on Generic 0.17.17)

Specification change that breaks your code

*13-18W01: (FONT) Fixed a bug where an overrun would occur if, after mistakenly calculating the size of the RectDrawer working memory, memory that did not meet the correct alignment restrictions was passed.

Fixed a bug where an overrun would occur if you passed memory that did not meet the correct alignment restrictions when calculations for working memory size were erroneous in nn::font::RectDrawer::Initialize(). Changed specifications to require alignment restrictions for the work buffer in the new version. Will now fail on an assertion when memory with an incorrect alignment is configured. Enable the acquisition of the necessary alignment value using nn::font::RectDrawer::GetWorkBufferAlignment(). The bug occurs when adjusting an alignment at or above 1524 bytes. A memory attribute violation occurred after either having memory corruption or unintentionally overwriting the memory attribute. This bug existed prior to version 0.16 of the NintendoSDK. However, we project that few instances surfaced.

Bug Fixes

13-18W02: (EffectTool) Fixed a bug where the normal was not calculated correctly if you configure a stripe to Perfectly follow emitter in EffectMaker.

Fixed a bug in EffectMaker where the normal was not calculated correctly when enabling Perfectly follow emitter for the Historical Stripe and Historical Stripe2 settings.
