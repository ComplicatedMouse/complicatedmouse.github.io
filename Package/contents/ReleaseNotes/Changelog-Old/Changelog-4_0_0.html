Specification change that breaks your code

*4-0-0S01: (GFX) Revised the rasterizer state's DepthClip to be enabled by default.

Fixed a known bug that caused nn::gfx::RasterizerStateInfo::SetDepthClipEnabled( false ) to be called from nn::gfx::RasterizerStateInfo::SetDefault instead of nn::gfx::RasterizerStateInfo::SetDepthClipEnabled( true ).
Note that some behavior will be different from usual if nn::gfx::RasterizerStateInfo::SetDepthClipEnabled is not called after nn::gfx::RasterizerStateInfo::SetDefault is called.


*4-0-0S02: (Graphics) Changed the format of the GPU crash dump that is output whenever an application causes a GPU error to occur.

The format of the GPU crash dump file that is output whenever an application causes a GPU error to occur has been changed.
These files used to be output as text, but have been changed to a binary format. In addition, a tool has been added that interprets these binary files and converts them to text files.
For more information, see GPU Crash Dump.

*4-0-0S03: (LDN) Features that display an internal error message in nn::ldn::Connect() are scheduled to be deprecated.

The following features will be deprecated in future SDKs.

* nn::ldn::ConnectOption_ShowError
* nn::ldn::ConnectOption_Default
* nn::ldn::ConnectOption_All

We also plan to change the specification so that an error message will not appear for nn::ldn::Connect(), which cannot take option as an argument. If you are planning to move to the new SDK, use instances of Connect() that can take option as an argument because error handling has already been added to nn::ldn::ResultIncompatibleVersion so that it will continue to work without any issues after the specification change.

*4-0-0S04: (TIME) Deprecated the feature that establishes the validity of the network clock.

The following functionality that establishes the validity of the network clock is only effective in limited situations: when the system's auto-correction continues to fail. It was slated to be deprecated because there is no guarantee that network clock validity will continue after using this function, and has been completely removed from this version.

  • nn::time::EnsureNetworkClockAvailability(), nn::time::CancelEnsuringNetworkClockAvailability()
  • nn::time::AsyncEnsureNetworkClockAvailabilityTask class


Added Features

*4-0-0S05: (NVN) Added a feature that builds a margin into the graphics firmware memory in the development emulator.

Added a feature to set a graphics firmware memory margin in the development emulator.

The graphics firmware memory size available to the development emulator using this feature will be about 160 KB smaller than the size set by nv::InitializeGraphics().

This feature is enabled by default as a safeguard against the risk of throwing an "Out of firmware memory" GPU exception by inadvertently exceeding the maximum firmware memory in the production environment.

Note that this margin will not be applied to applications that were built in a pre-NX Add-On 4.0.0 environments, even if this firmware has been applied to the hardware.
