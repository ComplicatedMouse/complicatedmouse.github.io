Details

Changes in NintendoSDK NX Addon 9.3.0 (based on Generic 9.3.0)

Specification change that breaks your code

*9-3-0S01: (AUDIO) Changed the naming conventions in Nintendo.AudioTool.SystemAudioCapture.

Changed the names specified in constructors that take names in Nintendo.AudioTool.SystemAudioCapture in Nintendo.AudioTool.dll from the target names used in TargetManager to peer names.

Due to this change, when a target in TargetManager that has a name that is not a serial number is used when specifying a name in SystemAudioCapture, the capture process fails for that target.
When specifying names, the implementation must be changed to specify serial numbers for NX devices.

*9-3-0S02: (EC) Added a minimum size requirement for the working buffer for the eShop service access utility feature.

Added a minimum size requirement for the working buffer for the eShop service access utility feature. At least 1 MB of memory is required for using the eShop service access utility feature.

Added Features

*9-3-0S03: (FRIENDS) Added a function for calling an applet to send friend invitations.

Added a function for calling the My Page applet to send a friend invitation.

9-3-0S04: (DevMenu) Added a feature for starting applications from issued notifications on the Notifications page.

Added a feature for starting applications from notifications issued by those applications that are displayed on the Notifications page.

9-3-0S05: (DevMenu) Added new regions that can be set in DevMenu and DevMenuCommand.

Added Hong Kong, Taiwan, and Korea as regions that can be set in DevMenu and DevMenuCommand.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Changing Device Settings > How to Use.
  • Features > Controlling the Development Hardware > DevMenuCommand > Changing the System Settings > Region Settings.


9-3-0S06: (EC) Added support for registering multiple responses to the EC response simulation feature.

The shop register-response-of-ec-response-simulation command in DevMenuCommand now supports registering multiple responses.

For more information about this command, see Features > Controlling the Development Hardware > DevMenuCommand > Shop Features > Registering a Response for EC Response Simulation.

9-3-0S07: (FRIENDS) Added a sample for the friend invitation feature.

Added a sample demo (FriendsFriendInvitation) showing how to use the friend invitation feature.

9-3-0S08: (IRSENSOR) Added a feature for testing states in which the IR Motion Camera is not present.

The IR Motion Camera feature built into controllers sometimes cannot be used when a user is using devices such as the following.
* Nintendo Switch Lite
*Joy-Con-compatible licensed controllers

When using a device that is not equipped with an IR Motion Camera, the behavior of nn::irsensor functions is different.
Added a feature for testing those states virtually using Joy-Con controllers.

For more information, see Features > Controlling the Development Hardware > DevMenuCommand > Managing Controllers.

9-3-0S09: (MOVIE) In the movie library sample, added support for video files with dynamically changing resolution.

The movie library sample now supports video files in which the video resolution changes during playback.
More specifically, added this support to Samples\Sources\Libraries\mm\mm_MovieVideoOutputHandler.cpp.
Also added an H.264 file with dynamically changing resolution to the MovieDecoderPlayer sample folder as an example.

9-3-0S10: (NFC) Added a feature for testing states in which NFC is not present or not working.

On controllers that have built-in NFC, it is sometimes not possible to use the NFC features in the following kinds of situations.
* When the Switch Pro Controller is connected via USB
* When using Joy-Con- or Switch Pro Controller-compatible licensed controllers (which are not equipped with NFC)

When a controller is connected that is not equipped with NFC or NFC is not working on that controller, the behavior of nn::nfp::ListDevices() is different and does not list those devices.
Added a feature for testing those states using a Joy-Con controller or Switch Pro Controller.

For more information, see Features > Controlling the Development Hardware > DevMenuCommand > Managing Controllers.

9-3-0S11: (TOOL) Added support for specifying IARC Generic ratings in AuthoringEditor.

In AuthoringEditor, added support for specifying IARC Generic ratings, which are not specific to any particular region.
Note that currently (as of September 2019), master ROMs cannot be submitted for applications for which an IARC Generic rating has been specified.
For more information about when master ROM submission for such applications will be supported, see Nintendo Developer Portal.

9-3-0S12: (TOOL) Added support for using DevKitSetup.ps1 on the HDEV.

In NX Add-On 9.3.0 and later, DevKitSetup.ps1 can also be used on the HDEV.
The HDEV can be specified as a type of EDEV. This information has been added to the script documentation.

9-3-0S13: (TOOL) In the integrated command-line tool (TargetShell), added a feature for performing system updates on the development hardware.

Added a feature for performing system updates on the development hardware.
For more information, see the manual.

9-3-0S14: (UTIL) Added macros for suppressing insertion of implicit padding.

Added the following macros for suppressing insertion of implicit padding to nn/nn_Macro.h.

  • NN_PACK (for environments other than MSVC)
  • NN_PACK_BEGIN (for the MSVC environment)
  • NN_PACK_END (for the MSVC environment)


Also added the following macros with the same functionality for use in the C language.

  • NNC_PACK
  • NNC_PACK_BEGIN
  • NNC_PACK_END


9-3-0S15: (VSI) New Features in VSI and Target Manager

  • Target Manager
    • Added the nn::tm::GetFirmwareVersion function.


Improvements

*9-3-0S16: (Display) Reduced the amount of memory required when calling nn::vi::CreateLayer.

Reduced the amount of memory allocated by the allocator set with nv::SetGraphicsAllocator when nn::vi::CreateLayer is called.
The amount of memory allocated when a layer is created with nn::vi::CreateLayer is now approximately 320 bytes less than when using the SDK 8.3 HOME Menu environment.

*9-3-0S17: (General) Improved the search feature in NintendoSDK Documents.

Added Sample Descriptions to the search targets that can be specified on the search page in NintendoSDK Documents.

Due to this change, documents that were previously treated as being API references can be further divided into API references and sample descriptions when searching.

The default search targets are API References, Sample Descriptions, and All Other Documents.

*9-3-0S18: (TOOL) Integrated the HDEV setup tools into the EDEV tools.

Integrated the HDEV setup tools (InitializeHdev.exe and SystemUpdateHdev.exe) into the EDEV setup tools (InitializeEdev.exe and SystemUpdateEdev.exe).
Due to this change, the EDEV setup tools now support HDEV device setup.
The HDEV setup tools will remain separately available for the time being but are displayed as EDEV tools in log output and the like as a result of these tools being integrated with the EDEV tools.

9-3-0S19: (Compiler) Updated to Clang for NX 1.8.7

Updated Clang for NX to version 1.8.7.
The main changes are as follows.

  • Implemented a fix for an fstack-protector vulnerability (VU#129209).
  • Added thrd_create(), thrd_exit(), thrd_detach(), thrd_join(), thrd_sleep(), thrd_yield(), thrd_current(), thrd_equal(), call_once(), mtx_init(), mtx_destroy(), mtx_lock(), mtx_trylock(), mtx_unlock(), cnd_init(), cnd_destroy(), cnd_broadcast(), cnd_signal(), cnd_timedwait(), cnd_wait(), tss_create(), tss_delete(), tss_set(), and tss_get().
  • Fixed a bug where errno was set even when posix_memalign() succeeded.
  • Fixed a bug where the wrong errno was set when the value specified for the alignment parameter in posix_memalign() was not a power of 2.


9-3-0S20: (CPU Profiler) Nintendo NX CPU Profiler Version 1.1

  • Added an Options window.
  • Added keyboard shortcuts to Start and Stop profiling.
  • Added optional Affinity and Priority columns to the Threads tab.
  • Added a Start page.
  • Added screenshots to profiles.
  • Added Checker categories and filtering.
  • Added new Unity Checkers.
  • Added a new Checker.
  • Added support for high DPI monitors.
  • Added support for pressing CTRL and rotating the wheel button to scroll up and down in Sample Graph.
  • Added the ability to adjust the height of items in the Icicle Graphs.
  • Added the option to use either addr2line or llvm-symbolizer in the Assembly tab.
  • Restored the choice of classification or sequential function selection colors.
  • Changed the scripting cmdlets that accept -Tab to show the tab automatically.
  • Changed the thread state tracing to support the profiler library integrated into another NSO rather than just the main NSS.
  • Improved the use of Autofill NSS files with Unity applications.
  • Improved the filtering time on the Code Coverage tab when RegEx is used.
  • Fixed a bug with Load per Frame Analysis on a Checker putting the Sample Graph into a state that is difficult to recover from.
  • Fixed the code blocks tooltip to show duration with a thousands separator.
  • Fixed a crash that could occur on the dev kit when profiling if a thread exits and there are threads that have affinity to more than one core.
  • Fixed Garbage Collection Spike Checker to detect spikes in Unity's garbage collection.
  • Fixed Profiler-ShowTab to accept any tab as a target.
  • Fixed a bug that could cause the Counters group filtering to show a hash.
  • Fixed crashes that could occur after loading a profile that was already loaded and syncing to a dev kit.
  • Fixed a bug that left the selected Instrumented function after an Unsync.
  • Fixed a bug that caused the computation of a Checker's time to always increase.
  • Fixed a bug that could allow for two profiles to try opening at the same time.
  • Fixed some minor text placement issues in the Sample Graph.
  • Fixed Code Coverage filtering with an invalid RegEx to show no results.


9-3-0S21: (DevMenu) Only DevMenuApp can be used to work with NSAVE files.

Importing and exporting NSAVE files in DevMenu and DevMenuCommand sometimes causes errors, depending on the size of the NSAVE file.
For this reason, from now on only DevMenuApp can be used to work with NSAVE files.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Save Data Management > Backup
  • Features > Controlling the Development Hardware > DevMenuCommand > Managing Save Data > Exporting Save Data


9-3-0S22: (FS) Added support for reducing the size of temporary storage and deleting temporary storage with patches.

Patches that update the size of temporary storage specified in NMETA files can now be used to reduce the size of temporary storage and to delete temporary storage.

9-3-0S23: (WEB) Added a feature for checking the amount of memory consumed by web content displayed in the web applet and Offline Web Applet.

Added a feature for checking the amount of memory consumed by web content displayed in the web applet and Offline Web Applet.

Bug Fixes

9-3-0S24: (AUDIO) Fixed a bug where SystemAudioMonitor did not start.

Fixed a bug in NX Add-On 9.2.0 where SystemAudioMonitor did not start.
In NX Add-On 9.2.0, you can work around this bug by creating a SystemAudioMonitor directory in the directory specified for the TEMP environment variable.

9-3-0S25: (EC) In the EcCatalog sample, fixed a bug with getting subscription service items that was caused by an incorrect path.

In the EcCatalog sample, fixed a bug where Subscription service item list did not work when selected because the wrong path was accessed.

9-3-0S26: (HID) Fixed a bug where HidShell did not generate the correct timestamp when the locale was set to fr-FR.

Fixed a bug where HidShell did not generate the correct timestamp when the locale was set to fr-FR.

9-3-0S27: (MOVIE) Fixed a bug in the Windows (32-bit) version of the movie library where performance was inadequate.

Fixed a bug where the performance of the movie library was inadequate when the target platform was 32-bit Windows.

9-3-0S28: (MOVIE) Fixed a bug where in rare cases, calling movie::Decoder::Stop() while a video with dynamically changing resolution was playing caused the system to stop responding.

Fixed a bug where in rare cases, calling movie::Decoder::Stop() while a video with dynamically changing resolution was playing prevented the Stop() function from ever returning program control.

9-3-0S29: (MOVIE) Fixed a bug where processing slowed down when using the Windows version of the movie library to play 1080p content.

Fixed a bug where playing 1080p video files in the Windows environment slowed down processing, resulting in unexpectedly poor playback performance.

9-3-0S30: (TOOL) Fixed a bug where NSAVE files failed to be extracted due to the path length of entries contained in the file.

Fixed a bug where NSAVE files containing entries with full paths of 260 characters or greater after extraction would fail to be extracted.
For more information, see the following documentation.

  • Creating Application Images > Tool Reference > AuthoringTool > Extracting the NSAVE File


9-3-0S31: (VSI) VSI and TargetManager Bug Fixes

  • VSI
    • Fixed a bug where variables were not visible while debugging apps built with clang 1.8.4 or later.
    • Fixed a bug where values were sometimes not displayed currently in the local window.
    • Fixed a bug where breakpoints set in dynamically loaded code sometimes did not work.
    • Fixed a bug where when a NSS file with a path containing Japanese characters was loaded, the call stack was not displayed correctly.
    • Fixed a bug where no warning message was displayed when an unsupported NSS file was specified while debugging.
    • Fixed a bug where the NX64 register window on the disassembly screen was sometimes not updated.
  • Target Manager
    • Fixed a bug where calling the htc::Initialize function on the Windows platform resulted in an internal call to new.
    • Fixed a bug where the EDEV took a long time to wake up from sleep mode.
    • Fixed a bug where the power did not turn on automatically when connected after turning off the power from Target Manager.
    • Fixed a bug where patches installed on an SD card failed to be uninstalled.
    • Fixed a bug where nn::tm::StartSavingLog() was called multiple times.
    • Fixed a bug where in rare cases, nn::tm::ResetTarget took a long time to execute.
    • Fixed other minor bugs.


Changes in NintendoWare NX Addon 9.3.0 (based on Generic 9.3.0)

Added Features

9-3-0W01: (Audio-ATK) Added a feature for using a high-quality resampler in SoundProjectWaveExporter.

Added the --enable-high-quality-resampler option to SoundProjectWaveExporter. Using this option causes SrcQuality_High to be specified for nn::audio::SetVoiceSrcQuality() in all Voice processes and makes it possible to reduce aliasing noise during pitch conversion.

9-3-0W02: (3DTool) Added Maya 2019 support to the Maya plug-in.

The Maya plug-in is now compatible with Maya 2019.

9-3-0W03: (LayoutTool) Added support for partially overwriting text position and shadows in text panes.

Added support for partially overwriting text position and shadows in text panes within parts.

Improvements

9-3-0W04: (LayoutTool) In LayoutEditor, added support for overwriting scissor panes and align panes within parts.

In LayoutEditor, added support for overwriting scissor panes and align panes within parts. Scissor panes and align panes can be set to be overwritten in the Parts and Control Settings window.

9-3-0W05: (LayoutTool) Modified how animation markers are displayed in LayoutEditor.

In LayoutEditor, modified how animation markers are displayed when a curve is muted or locked.

9-3-0W06: (EffectTool) Fixed a bug in EffectMaker where an error message was always displayed during conversion.

Previously, a loading error for a DLL file unrelated to the conversion results was displayed. Fixed the bug so that the DLL file is now loaded correctly.

Bug Fixes

9-3-0W07: (3D-G3D) Fixed a bug where calculation results for skeletal animations using the motion mirroring feature contained invalid values under some conditions.

Fixed a bug where calculation results for skeletal animations using the motion mirroring feature contained invalid values under some conditions.

More specifically, if any bones satisfied the following conditions, the local matrices for bones beneath them in the hierarchy would sometimes take invalid values.

  • The motion_mirroring_type attribute of the <bone> element in the intermediate model file (FMDA or FMDB file) was set to auto.
  • The bone was not in the yz-plane (that is, it did not have an x-coordinate value of 0).
  • There was no bone that was left-right symmetric to the bone.


9-3-0W08: (LayoutTool) Fixed a bug in LayoutEditor where clicking the Play button in the playback testing panel in the Animation Segment Tags window caused the program to crash.

Fixed a bug in LayoutEditor where clicking the Play button in the playback testing panel in the Animation Segment Tags window caused the program to crash.

9-3-0W09: (LayoutTool) Fixed a bug in LayoutEditor where data failed to be transferred to the viewer when there was only a single segment for which a segment weight had been set in separated mode.

Fixed a bug in LayoutEditor where data failed to be transferred to the viewer when there was only a single segment for which a segment weight had been set in separated mode.

9-3-0W10: (LayoutTool) Fixed a bug in LayoutEditor where muting curves that would be deleted when a file was loaded caused LayoutEditor to crash.

Fixed a bug in LayoutEditor where muting curves that would be deleted when a file was loaded caused LayoutEditor to crash.
