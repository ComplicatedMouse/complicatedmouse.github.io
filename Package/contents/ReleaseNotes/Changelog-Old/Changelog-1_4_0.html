Details

Changes in NintendoSDK NX Addon 1.4.0 (based on Generic 1.4.0)

Added Features

1-4-0S01: (DevMenu) Added a mode for events (exhibition mode).

For more information, see the Automatic Startup Application Settings section in Controlling Development Hardware / DevMenu.

1-4-0S02: (MII) Added features for supporting Mii development to DevMenu.

For more information, see the Mii Testing Mode section in Controlling the Development Hardware / DevMenu.

1-4-0S03: (MOVIE) Added a sample implementation for calling Decoder::Reset for MovieDecoderPlayer.

Added the implementation of a usage example of movie::Decoder::Reset to the MovieDecoderPlayer sample.
To apply this implementation, pass -reset when starting up the sample.

1-4-0S04: (MOVIE) Added functions for specifying affinity for Decoder and Extractor threads.

When using Decoder and Extractor, the threads are created inside the movie library.
The movie::SetCoreMask() function was added to specify which CPU core these threads should be created in.

1-4-0S05: (OS) Added a feature for testing whether the memory in the specified address space is locked.

Added the nn::os::IsMemoryLocked() function as a debugging-only feature for developing applications. This function checks whether the memory in the specified address space is locked. This feature cannot be used in retail units.

1-4-0S06: (TOOL) Added a feature for comparing NMETA or NSP files to AuthoringEditor.

It is now possible to check for differences between two files in AuthoringEditor.
Comparisons are supported between the following file types.

  • Between application NMETA files.
  • Between application NSP files.
  • Between add-on content NMETA files.
  • Between patch NSP files.
  • Between patch NSP files and application NSP files.


1-4-0S07: (TOOL) Added support for add-on content NSP files to AuthoringEditor.

The following operations are now available for add-on content NSP files in AuthoringEditor.

  • Create new add-on content NMETA files.
  • Create new add-on content NSP files.
  • Check the content of add-on content NSP files.


1-4-0S08: (TOOL) Added support for patch NSP files to AuthoringEditor.

The following operations are now available for patch NSP files in AuthoringEditor.

  • Create new patch NSP files (operate from the screen immediately after starting AuthoringEditor).
  • Check patch NSP file content.


Improvements

1-4-0S09: (Compiler) Updated Clang for NX to version 1.2.3.

Updated Clang for NX to version 1.2.3.
The changes are described below.

To improve cases where linking time increased greatly or links failed when building applications, the output option specified as the default for outputting bug information in the compiler was reverted to -fno-standalone-debug.

1-4-0S10: (DevMenu) Added a feature to prevent starting an application from the DevMenu or DevMenuCommand if the initial version application and patch combination is incorrect.

If the combination of the initial version application and the patch is incorrect, the previous behavior produced a fatal error in the application due to an fs error after the application was started.
However, this behavior made it difficult to determine the cause of the fatal error. As a result, whether the initial version application and patch combination is correct is now checked when starting the application with DevMenu or DevMenuCommand and, if it is incorrect, the message The application is not one that is used to create the patch is displayed and the application will not start.
However, this check does not work on the HOME Menu.

1-4-0S11: (DevMenu) Added information about the Open and Close state display to the description of the Accounts page in the SDK documentation.

The following description of the Accounts page in DevMenu was added to the Controlling the Development Hardware > DevMenu section in the SDK documentation.

* Information displayed in the user accounts list.
* Of these items, created a section about the account guide in the description for the Open state.

1-4-0S12: (MOVIE) Deleted the extra memcpy that was performed by MovieDecoderPlayer.

Reduced the number of times memcpy was performed until the decoding results obtained from movie::Decoder were copied to NVNTexture.

Bug Fixes

1-4-0S13: (DevMenu) Fixed a bug where a fatal error would occur when starting up DevMenu if multibyte characters are included in the application title.

Fixed a bug where a fatal error would occur when starting up DevMenu if the application title satisfies the following conditions.

  • The title name is more than 64 bytes including the terminating character.
  • The title contains multibyte characters.


1-4-0S14: (MOVIE) Fixed a bug where videos played in the MovieDecoderPlayer would stutter at the beginning of playback.

There were cases where the picture or sound would cut out immediately after starting playback. This has been fixed so that the symptoms no longer occur.

1-4-0S15: (OS) Fixed a bug where the alignment of the stack frame in fiber functions was incorrect.

Fixed a bug where the alignment of the stack frame in the fiber function specified by nn::os::InitializeFiber() was incorrect. This fix also resolves the bug where, for example, adding alignment restrictions of greater than 8 bytes for NX32 and greater than 16 bytes for NX64 to the automatic variables inside the fiber function would cause the variables to not be placed in the correct alignment positions.

1-4-0S16: (TOOL) Changed the specifications relating to file path input in AuthoringEditor.

The file paths entered into the NMETA file in AuthoringEditor are converted into indirect references (relative paths) from the NMETA file path that is written when the file is saved.
If the file path uses a different drive letter or specifies a location on the network, it will be saved in the NMETA file as an absolute reference (absolute path).

1-4-0S17: (TOOL) Fixed a bug where converting from a cube map DDS file caused images to become reversed on the +z and –z axes.

There was a bug where converting a DDS file for a cube map with no mipmap in the texture converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe) would cause the images to be reversed in the +z and –z directions. This has been fixed.

1-4-0S18: (TOOL) Fixed a bug where the shader compiler may become unresponsive when running in a Windows 7 environment.

Fixed a bug where using the shader compiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderCompiler.dll) and shader converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe) to compile shaders in a Windows 7 environment had a certain probability of causing the tool to become unresponsive.

1-4-0S19: (TOOL) Fixed a bug with the AuthoringTool parameter check for NX cards.

Fixed a bug where executing the replace and makepatch commands in AuthoringTool would, if certain conditions are satisfied, cause an inappropriate error to be detected in the parameter check for NX cards.

Changes in NintendoWare NX Addon 1.4.0 (based on Generic 1.4.0)

Specification change that breaks your code

*1-4-0W01: (EffectTool) Updated the version of the binary output by EffectMaker.

Adjusted the binary region to maintain binary compatibility as much as possible with future releases of the SDK.
This breaks compatibility for binaries created with previous versions of EffectMaker, so the resources must be reconverted.

Added Features

1-4-0W02: (AudioTool) Added an item filter feature to SoundProjectConverter.

It is now possible to filter the items and bank instruments in the sound project and decide at the time of conversion whether to include them in the sound archive.

To use this feature, specify the --exclude-list and --include-list options in SoundProjectConverter.
For more information, see the SoundProjectConverter description page.

Note: This feature is currently under evaluation and the specifications may change without warning.

1-4-0W03: (Effect-VFX) Added the nn::vfx::Emitter::GetCustomShaderSwitch() method.

Added the nn::vfx::Emitter::GetCustomShaderSwitch() method.
This method can be used to obtain the selection state of the custom shader radio buttons.

1-4-0W04: (Effect-VFX) Added the nn::vfx::System::Calculate( nn::vfx::EmitterSet* pEmitterSet ) member function.

Added the nn::vfx::System::Calculate( nn::vfx::EmitterSet* pEmitterSet ) member function. If calculation cannot be performed using an individual EmitterSet, use this member function to also update debugging information such as the number of emitter sets currently being played back that are counted by the System class.

Improvements

1-4-0W05: (Effect-VFX) Improved the warning messages output by the vfx runtime.

Improved the warning messages output by the vfx runtime and prepared documentation about the warnings.

1-4-0W06: (GFX) Merged the project that generates shader binaries for libnns_gfx use with the libnns_gfx project.

Previously, the project that generates shader binaries for libnns_gfx was separated as a NnsGfxShader project, but this project has been merged with libnns_gfx. The shader binaries used for libnns_gfx will be built automatically when building libnns_gfx.

Bug Fixes

1-4-0W07: (Audio-ATK) Fixed a bug where rebuilding the atk library failed.

Fixed a bug where rebuilding the atk library failed due to insufficient headers required for building.

1-4-0W08: (3D-G3D) Fixed a bug where the write destination of the pointer to the dependency shader parameter was incorrect.

If shader annotation depend is described in the shader parameters, the library writes the pointer to the shader parameters that the function depends on to the end of the shader parameters during runtime, but this write destination was incorrect. This pointer was written to a location that is 4 bytes misaligned after the correct location, so there was an issue where it was not possible to reference the correct pointer to the dependency shader parameters, and other shader parameters placed after it would be corrupted.

This problem would not occur if the depend annotation specified itself because the library would not write the pointer.

For more information about the behavior when describing the depend annotation, see Shader Parameters.

1-4-0W09: (Layout-UI2D) Fixed a bug where the texture mapping settings were not rendered correctly when using the detailed combiner.

There was a bug where, if the detailed combiner is specified in LayoutEditor and the texture mapping settings are set to something other than UV mapping, it would not be rendered correctly in the runtime. This has been fixed.

1-4-0W10: (LayoutTool) Fixed a bug where textures could not be loaded in LayoutEditor when there is no project settings file in the project root directory.

There was a bug where starting LayoutEditor when there is no project settings file in the project root folder for LayoutEditor made it impossible to load textures. This has been fixed.

1-4-0W11: (LayoutTool) Fixed a bug where the Color Sample item would increase when restoring the workspace in LayoutEditor.

There was a bug where the Color Sample item would increase when restoring the workspace in LayoutEditor. This has been fixed.

1-4-0W12: (LayoutTool) Fixed a bug where the pane name and segment tags could not be changed in the LayoutEditor Parts and Control Settings.

There was a bug where it was not possible to change the pane column in the Feature Pane list or segment tags in the animations list in the LayoutEditor Parts and Control Settings. This has been fixed.

1-4-0W13: (Effect-VFX) Fixed a bug when calling nn::vfx::viewer::ViewerSystem::UpdateModelInfo.

Fixed a bug where models in the application that were already registered in the viewer runtime could not be deleted when using nn::vfx::viewer::ViewerSystem::UpdateModelInfo to update the model information in the application.

1-4-0W14: (Effect-VFX) Fixed a bug where the real-time editing of child emitters did not work.

Fixed a bug where some of the parameters could not be edited when using EffectMaker to edit child emitters in real time that were currently being played back in EffectViewer or by the application.

1-4-0W15: (EffectTool) Fixed an issue where converting a large number of EffectMaker emitter set files would cause conversion to fail.

Fixed an issue where batch converting more than approximately 1400 emitter set files would cause a texture conversion error to occur.

1-4-0W16: (EffectTool) Fixed an issue where replacing the EffectMaker emitter rendering caused the viewer display to become corrupted.

Fixed an issue where changing the emitter node order specified for primitives and textures to change the draw order in EffectMaker causes the display in the viewer to become corrupted.

1-4-0W17: (EffectTool) Fixed the problem where the EffectMaker custom action callback ID is rewritten.

Fixed the problem where, in EffectMaker, if there are more custom action user defined values than existed when the eset file was saved, the callback ID would be overwritten by a different ID when the eset file is opened because the custom action callback ID "does not use a callback function".
