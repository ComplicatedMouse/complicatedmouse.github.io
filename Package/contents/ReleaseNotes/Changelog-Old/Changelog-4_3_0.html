Details

Changes in NintendoSDK NX Addon 4.3.0 (based on Generic 4.3.0)

Specification change that breaks your code

*4-3-0S01: (IRSENSOR) Fixed a bug where, if the Run function was called unaccompanied by a mode transition while the processor was running, old data would continue to be retrieved.

Previously, if the Run function was called unaccompanied by a mode transition while the processor was running, the old data from before the parameter was set would continue to be retrieved until the setting was applied.
That has been fixed such that now the nn::irsensor::ResultIrsensorNotReady error is returned until the Run function can get the correct data.
This revision breaks the compatibility with applications created with NX Add-On 4.0.0 through 4.2.0 versions of the SDK.
Rebuild these applications using NX Add-On 4.3.0.

*4-3-0S02: (VSI) Breaking changes to Target Manager.

  • Definitions have been changed for nn::tm::TargetHandle and nn::tm::DiscoveryHandle.
    • The definitions for nn::tm::TargetHandle and nn::tm::DiscoveryHandle have been changed to end mutual implicit conversions. With this change, misuses are now detected at compilation time.
  • Support for TMAPI.h will effectively end from NX Add-On 5.1.
    • There is now official support for the tm library, which since NX Add-On 4.1, could use the functionally of Target Manager. In step with this, the functions defined in C:\Program Files\Nintendo\Oasis\include\TMAPI.h will become unusable after the release of NX Add-On 5.1. Going forward, use C:\Program Files\Nintendo\Oasis\include\tm.h.
  • The Target Manager option for enabling JIT debugging is now disabled by default.
    • To use JIT debugging, select Target Manager > Tools > Options > Enable JIT debugging.


Added Features

4-3-0S03: (DevMenu) Added support for SD card cache storage data.

The Save Data page now displays the cache storage data saved to an SD card.

4-3-0S04: (FS) Added a feature that shortens the time it takes to create a patch.

Added the --cache-directory option to AuthoringTool to speed up the creation of a patch.
If you specify this option and create a cache, patches will be created faster the next and subsequent times.
For more information about the option, see Building and Running a Patch.

4-3-0S05: (MOVIE) Added a feature to return the supported range for color space and H264 video.

Added a feature to return the supported range for color space and H264 video.

4-3-0S06: (OE) Added a function for writing video to files for applications for which video capture is enabled.

Added the nn::oe::RequestToSaveRecordingForDebug() function for writing video to files for applications for which video capture is enabled.
Calling this function has the same effect as a long press of the Capture Button.
However, note that this function cannot be used in the retail unit.
Use it only for debugging purposes during application development.

4-3-0S07: (OE) Added a sample program for getting the application launch parameters.

Added a sample program for getting the application launch parameters. The sample program presents an example implementation and descriptions, including a description of how to use the nn::oe::TryPopLaunchParameter() function, and notes regarding the method of defining the launch parameter format.

4-3-0S08: (TOOL) Added a feature for specifying the name of the EDEV when initializing the hardware using InitializeEdev.

Added a feature for specifying the name of the EDEV when initializing the hardware by using InitializeEdev.

4-3-0S09: (TOOL) Added a tool for updating the development hardware, and a tool that lets you update the development hardware using an NX development card or an SD card.

The following tools have been added for updating the development hardware.
1. SystemUpdateSdev(SystemUpdateSdev): A command-line tool for updating the development hardware, while keeping the settings and the same procedures used for a system update.
2. DevKitUpdater*.nsp: A tool that updates the development hardware. Similar to the preceding command-line tool, but runs on the development hardware and updates that hardware using an NX development card or an SD card.
Note that presently, there are a number of restrictions on these update methods. Read the SDK documentation prior to using these tools.

4-3-0S10: (TOOL) Added new features to the integrated command-line tool (TargetShell).

Added the following features to the integrated command-line tool.

  • Install and uninstall applications, patches,and downloadable content.
  • Switch the mute setting.
  • Switch menus.
  • Switch shops.
  • Delete save data.

For more information, see the manual.

4-3-0S11: (VSI) New Features in VSI and Target Manager

  • VSI
    • Added an option to generate a build error when ROM submission is not possible.
  • Target Manager
    • Added a feature to Target Manager to issue a warning when file operations by HostFs fail due to insufficient Windows permissions.
    • Added menus items to turn the power on/off and blink the LEDs on multiple devices.


4-3-0S12: (WEB) Added a command to WebCallerCommand to enable Web Audio in the web applet and the offline web applet.

Added the --web-audio-enabled command to WebCallerCommand.
Use this command to enable Web Audio for the content in the web applet and the offline web applet, launched from WebCallerCommand.
For more information on usage, use the --help command.

4-3-0S13: (WEB) Added a command to WebCallerCommand to enable/disable the page cache in the web applet and the offline web applet.

Added the --page-cache-enabled command to WebCallerCommand.
You can use this command to enable or disable the page cache for the content in the web applet and the offline web applet, started from WebCallerCommand.
For more information about how to use this, use the --help command.

4-3-0S14: (WEB) Added functionality for adding an optional character string to UserAgent.

Added the following function for setting an optional character string to UserAgent in the web applet.

  • nn::web::SetUserAgentAdditionalString()


4-3-0S15: (WEB) Added functionality for enabling and disabling page cache.

Added functionality for enabling and disabling page cache in the web applet and the offline web applet.

  • nn::web::ShowWebPageArg::SetPageCacheEnabled()
  • nn::web::ShowOfflineHtmlPageArg::SetPageCacheEnabled()


4-3-0S16: (WEB) Added functionality for enabling and disabling Web Audio.

Added functionality for enabling and disabling Web Audio in the web applet and the offline web applet.

  • nn::web::ShowWebPageArg::SetWebAudioEnabled()
  • nn::web::ShowOfflineHtmlPageArg::SetWebAudioEnabled()


4-3-0S17: (Keycode processing) Added a feature for getting the character codes corresponding to the keyboard key codes.

Added a feature for converting USB HID Keyboard/Keypad Page Usage IDs to character codes (UTF-16), based on the system settings. For more information, see here{.

4-3-0S18: (Keycode processing) Added a feature for resolving dead keys and romaji input.

Added a feature for resolving character input comprised of the input from multiple keys, such as for processing dead keys and romaji, accepting the input, and assembling the character code strings. For more information, see here{.

Improvements

4-3-0S19: (AUDIO) Increased the amount of DSP resources that can be allocated to the application.

It is now possible to allocate 75% of DSP resources to the application, increased from the previous 70%.

For more information about the amount of DSP resources that can be allocated to the application, see DSP Resource Limitations.

4-3-0S20: (DevMenu) Overlay notifications about video capture are now also made when using DevMenu.

Overlay notifications regarding the save results when video is captured, are now also made when DevMenu is being used. Notifications are presented in English text only. Thumbnails are not displayed and sound effects are not played, so switch to the HOME Menu to check operations in the retail environment.

4-3-0S21: (DIAG) LogViewer now supports escape sequences to specify 24-bit foreground color and background color.

LogViewer now supports escape sequences to specify 24-bit foreground color and background color.

4-3-0S22: (FS) Added output of the nn::bcat access log, and added support for checking that access to FsAccessLogChecker version 10.0.

Added output of the log regarding fs access by the nn::bcat API, and added support to version 10.0 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) for checking fs access via nn::bcat. For more information, see fs Library / Debugging / Checking the Frequency of Access.

4-3-0S23: (FS) Added support to FsAccessLogChecker version 9.6 for checking the value of the read frequency that got a PASS result.

Added support to version 9.6 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) for outputting an upper value to SameAddressRead when the read frequency gets a PASS result. In step with this revision, the appearance of the output has changed.

4-3-0S24: (FS) FsAccessLogChecker version 10.2 has been improved to sort the access log in time series order before checking.

Depending on the thread configuration of the application, the fs access log might not be output in time series, so version 10.2 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) has been improved to resort the access log in time series order before checking.

4-3-0S25: (FS) In version 10.3 of FsAccessLogChecker, the access logs for errors are not checked.

It is presumed that the access logs for errors returned to the application do not involve access to storage. For this reason, in version 10.3 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe), the access logs are not checked for anything other than result: 0x00000000.

4-3-0S26: (General) Stabilized the impact on performance of applications enabled for video capture.

When the video capture feature introduced with NX Add-On 4.2.0 was enabled, the performance of the application would be erratically effected in terms of CPU and GPU processing time. The irregularity of the impact has been smoothed out and the load has been stabilized.
For more information, see this page.

4-3-0S27: (TOOL) AuthoringTool has been improved to detect changes to the software legal information.

If it detects changes to the software legal information obtained from the Software Legal Information System (SLIM), AuthoringTool now returns an error when it is running.

4-3-0S28: (TOOL) Deleted "DistributedByNintendo" from the logo data settings values in the NMETA file.

Deleted "DistributedByNintendo" from the logo data settings values in the NMETA file.
For more information about the values that can be set for logo data, see Features > Building > Creating Application Images > Writing the NMETA File.

4-3-0S29: (TOOL) Deprecated the AuthoringTool options --legal-information, --html-document, --accessible-urls, --icon , and --nx-icon.

The following AuthoringTool options have been deprecated.

  • --legal-information
  • --html-document
  • --accessible-urls
  • --icon
  • --nx-icon


Going forward, use the following NMETA file elements.

  • LegalInformationFilePath
  • HtmlDocumentPath
  • AccessibleUrlsFilePath
  • IconPath
  • NxIconPath


For more information, see Features > Building > Creating Application Images > Writing the NMETA File.

4-3-0S30: (VSI) Improvements to VSI and Target Manager.

  • Target Manager
    • If an NSP file is dragged to Target Manager when the target is not connected, that target will not be automatically connected.
    • When Target Manager is run with a specified NSP path, it can correctly parse the path, even if it ends with a double quote (").


4-3-0S31: (VSI) Suppressed the message that resembled an error during installation.

The message that was displayed in red text, resembling an error, is no longer displayed when you install the VSI for NX package on a PC, to which Visual Studio 2013 or 2015 is not installed.
If Visual Studio 2013, 2015, or 2017 has been installed, VSI for NX can be installed and used regardless of whether the NX Add-On version is 4.2.0 or earlier, or 4.3.0 or later.

Bug Fixes

4-3-0S32: (DevMenu) Fixed a bug where acting to show the legal information caused a fatal error.

Fixed a bug where a fatal error sometimes occurred when you clicked the "Show Legal Info" button, in the screen with details about the application, and then attempted to display legal information by clicking on "Support Info," "Important Info," or "IP Notice".

4-3-0S33: (DevMenu) Fixed a bug where it was not possible to start the application from the details screen shown by nn::friends::ShowUserDetailInfo.

Fixed a bug where the application would not start, and you could advance no further, when you selected the application icon in the details screen shown by nn::friends::ShowUserDetailInfo.

4-3-0S34: (DIAG) Fixed a bug in Log Viewer where the background color could not be changed using an escape sequence.

Fixed a bug in Log Viewer where the background color could not be changed by using an escape sequence.

4-3-0S35: (DIAG) Fixed a bug where Log Viewer would stop responding when long logs were output on multiple threads.

Fixed a bug where Log Viewer would stop responding if logs were output on two or more threads, and over 761 characters were output on each call to NN_LOG and NN_VLOG.

When the bug is recreated, the log output of RunOnTarget and Target Manager stop, and control does not return from NN_LOG and NN_VLOG, called by the program running on the development hardware.
You can escape these symptoms by exiting Log Viewer.

4-3-0S36: (Display) Optimized LCD brightness.

When the setting for LCD brightness is set to a low value, the screen is now darker than before.
The result is that screen brightness has been optimized to better correspond with the value set for LCD brightness.

4-3-0S37: (IRSENSOR) Fixed a bug where if the application was in the background when the irsensor library was initialized, and then returned to the foreground, the system would abort.

Fixed a bug where if the application was running in the background when the irsensor library was initialized, and then returned to the foreground, the system would abort.

4-3-0S38: (IRSENSOR) Fixed a bug where the Run function would block for several hundred milliseconds if called immediately after calling the nn::irsensor::StopImageProcessor function.

Fixed a bug where the Run function would block for several hundred milliseconds if called immediately after calling the nn::irsensor::StopImageProcessor function.

4-3-0S39: (IRSENSOR) Fixed a bug where the system would sometimes abort on dynamic mode switching in an environment with heavy packet loss.

Fixed a bug where system processes would sometimes abort on dynamic mode switching in an environment with heavy packet loss.

4-3-0S40: (libcurl) curl_easy_perform() may cause an assertion failure.

In certain situations, the call to the curl_easy_perform() function may cause an assertion failure.

4-3-0S41: (MOVIE) Fixed a bug where media containing certain metadata would crash or stall.

Fixed a bug where media containing certain metadata, such as menus or track information, would crash or stall.

4-3-0S42: (MOVIE) Fixed a bug where the decoder would crash when you played Theora/OGG media.

Fixed a bug where the decoder would crash when you played Theora/OGG media.

4-3-0S43: (MOVIE) Fixed a bug where WebM video files, without an audio track, could not be played.

Fixed a bug that appeared in NX Add-On 4.2.0 where WebM video files, without an audio track, could not be played.

4-3-0S44: (MOVIE) Introduced changes that eliminate the superfluous output buffer alignment for the movie::decoder API and the CPU path, reducing the size of the output buffer.

Eliminated the superfluous output buffer alignment for the movie::decoder API and the CPU path, reducing the size of the output buffer.

4-3-0S45: (NIFM) Fixed a bug that resulted in a 2110-4000 error when trying to make a wireless connection.

Fixed a bug that resulted in a 2110-4000 error when trying to connect to a wireless access point. After successfully connecting to the wireless access point, it is still possible for a 2110-4000 error to occur.

4-3-0S46: (OE) Fixed a bug where a fatal error would sometimes occur subsequent to the call to the nn::oe::GetDefaultDisplayResolutionChangeEvent() function.

Fixed a bug where sometimes FATAL ERROR 2001-0105 would be triggered after the application called the nn::oe::GetDefaultDisplayResolutionChangeEvent() function.

4-3-0S47: (OE) Fixed a bug where the callers of the functions to enable/disable a screen capture would be blocked for an extended time.

Fixed a bug where the callers of the following functions would be blocked for an extended time in applications where video capture was enabled.
nn::oe::EnableScreenShot(), nn::oe::DisableScreenShot(), nn::oe::EnableRecording(), nn::oe::DisableRecording()

4-3-0S48: (SWKBD) Fixed a bug where the nn::swkbd::ShowKeyboard() function returned an invalid string when in utf-8 mode.

Fixed a bug where the nn::swkbd::ShowKeyboard() function, if called after the mode was set to utf-8 by nn::swkbd::SetUtf8Mode(true), would return an invalid string. This occurred if the user entered characters in excess of 1002 bytes, when converted to utf-8 on the software keyboard displayed by the function.


4-3-0S49: (TOOL) Fixed a bug where PROD MODE could not be used with InitializeSdev and InitializeEdev.

Fixed a bug in PROD MODE, so now when you initialize using InitializeSdev or InitializeEdev, the PROD MODE firmware with +Prod and +ProdWithLog are displayed in the "Select firmware variation" list.

4-3-0S50: (TOOL) Fixed the incorrect detection of an error preventing submission of a master ROM in AuthoringEditor and AuthoringTool.

There was a bug where "Feature that requires application for use: Bundle_Private_NetworkMiddleware_Generic" would be incorrectly detected when you used AuthoringEditor to open an NSP file made using libcurl, or when you got the error preventing ROM submission with AuthoringTool. This bug has been fixed.

4-3-0S51: (VSI) VSI and TargetManager Bug Fixes

  • VSI
    • Fixed a bug where values would not change in the Memory window of Visual Studio.
    • Fixed a bug where even if the NX32 disassembly was open, it would display as NX64 registers.
    • Fixed a bug where the debugging of a library project would not proceed beyond the loading stage.
  • Target Manager
    • Fixed a bug where the behavior was unstable when multiple devices were connected.
    • Fixed a bug where the tool would occasionally remain on the error screen when the EDEV was connected or disconnected.
    • Fixed a bug where an application that was running before Target Manager was connected, could not be terminated from Target Manager.
    • Fixed a bug where though two or more instances were running, the log would be displayed for only the first instance of Log Viewer.
    • Fixed a bug where the log would not be displayed if Log Viewer was started before Target Manager.
    • Fixed a bug where one log would not display if large logs were sent from multiple threads at the same time.
    • Fixed a bug where tm::GetTargetProgramState would return a result of failure if called when the application was not running.
    • Fixed a bug where a target, that was unregistered immediately after Target Manager was started, would remain registered.
    • Fixed a bug where incorrect exception code would be displayed when you debugged or dumped an application that was started from Target Manager.
  • Fixed other minor bugs.


Changes in NintendoWare NX Addon 4.3.0 (based on Generic 4.3.0)

Improvements

4-3-0W01: (Audio-ATK) Improved the increased processing load and memory requirements that accompanied the addition of evaluation features in NX Add-On 4.2.0.

Improved the increased processing load and memory requirements that accompanied the addition of evaluation features in NX Add-On 4.2.0.
Processing load and memory requirements increased, regardless of whether the evaluation features were actually being used. That situation has been improved.

Bug Fixes

*4-3-0W02: (3DTool) Fixed a bug where the Thumbnail Explorer extension would access invalid memory.

Fixed a bug where the Thumbnail Explorer extension would access invalid memory when displaying the thumbnails for ESET, FLYT, FMDA, and FMDB files.

4-3-0W03: (AudioTool) Fixed bugs involving playback in the Preview Player panel when the automatic measurement of average loudness is enabled in SoundMaker.

Fixed the following bugs involving playback in the Preview Player panel when the automatic measurement of average loudness is enabled.

    • The lack of sound during playback.
    • The incorrect measurement of average loudness.


4-3-0W04: (Layout-UI2D) Fixed a bug where constant color did not display correctly in the ui2d runtime when an advanced combiner mode was used.

Fixed a bug where constant colors, beyond constant color 1, did not display correctly in the ui2d runtime when an advanced combiner mode was used and a value of less than 254 was set for opacity or vertex color.

4-3-0W05: (LayoutTool) Fixed a bug in LayoutEditor where animations could not get pasted using the Pane Information Paste Controls.

Fixed a bug in LayoutEditor where animations could not get pasted using the Pane Information Paste Controls.

4-3-0W06: (LayoutTool) Fixed a bug in LayoutEditor where the name for a new picture pane would take the name of the Window pane.

Fixed a bug in LayoutEditor where a new picture pane would take the name of the Window pane when it was created from the menu.

4-3-0W07: (LayoutTool) Fixed a bug where the selected segment would not be applied in the viewer, even when set for the animation playback range.

Fixed a bug where, even if the option was enabled to 'set the selected segment as the animation playback range' and the data was sent to the viewer, the playback range in the viewer would not reflect the configured values.

4-3-0W08: (EffectTool) Fixed so that the binary no longer includes resources assigned to invalid emitters in EffectMaker.

The process of binary conversion in EffectMaker has been revised such that the binary data no longer contains the resources for textures and primitives assigned to invalid emitters.
