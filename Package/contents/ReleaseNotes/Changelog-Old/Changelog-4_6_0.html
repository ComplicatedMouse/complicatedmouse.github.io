Details

Changes in NintendoSDK NX Addon 4.6.0 (based on Generic 4.6.0)

Specification change that breaks your code

*4-6-0S01: (General) Ended support for Visual Studio 2013.

Ended support for Visual Studio 2013, as scheduled.

*4-6-0S02: (TOOL) Added a feature in the AuthoringTool for checking accessible URLs.

Improvements were made so that when the --error-unpublishable option is specified in the AuthoringTool, an error occurs if something other than ^https:// or ---- is set in the accessible URL.
For more information, see Features > Library Applets > Web Applets and Offline Web Applets > Web Applets.

*4-6-0S03: (TOOL) Made changes so that an error now occurs if 1 is set in CardSpec/Size of the NMETA file when running AuthoringTool.

Made changes so that an error now occurs if 1 is set for CardSpec/Size of the NMETA file when running AuthoringTool.
For more information about values that can be set for CardSpec/Size, see NintendoSDK Documents > Features > NX Development Card.

Added Features

*4-6-0S04: (AUDIO) Added a feature for audio input.

Added an audio input feature for recording audio data from the device.

We recommend using the Nintendo Switch Online application voice chat feature, provided by the NEX library, to implement voice chat features. For more information, see the voice chat guide for Nintendo Switch Online.
Use this feature if you need to implement voice chat on the Nintendo Switch system itself.

4-6-0S05: (DevMenu) Added a feature for deleting save data by specifying the Application ID.

Added a feature to DevMenuCommand for deleting save data by specifying the Application ID.
In step with this, changed the existing method of deleting save data by specifying the save data ID.
These two methods of deleting can now be done with the following subcommands.

  • savedata delete --application-id <application_id>
  • savedata delete --savedata-id <savedata_id>


4-6-0S06: (DevMenu) Added a feature for forcing shutdown of applications.

Added a feature for forcing shutdown of applications. When running applications cannot be terminated using normal shutdown methods, it is possible to force shutdown of applications by simultaneously pressing the L, R, and X buttons.

4-6-0S07: (DevMenu) Added a feature for selecting where to import the cache storage data.

Added a feature to select whether to import the cache storage data to system memory or an SD card.

4-6-0S08: (EC) Added a function for displaying the subscription course detail screen.

Added the nn::ec::ShowShopSubscriptionList() function for displaying subscription course details.

4-6-0S09: (ERR) Added an API for handling errors returned under certain conditions by NEX and other middleware.

Added an API for handling errors returned under certain conditions by NEX and other middleware.

4-6-0S10: (WEB) Added a web inspector for web content development.

Added a web inspector for use in when developing web content.
For more information, see NintendoSDK Documents / Features / Library Applets / Web Applet and Offline Web Applet / Information Common to Both / Web Inspector.

Improvements

4-6-0S11: (DevMenu) Deleted the Sound page and moved the various items to the Device page.

Moved various items on the Sound page over to the Device page. In conjunction with this, deleted the Sound page.

4-6-0S12: (DevMenu) The progress is now displayed when Verify is run on large volumes of save data.

The progress is now displayed when Verify is run on large volumes of save data so it does not look like the screen has frozen.

4-6-0S13: (TOOL) Added a feature in the AuthoringTool to check for use of the icon included in the NintendoSDK.

Added a check item for cases where the --error-unpublishable option is specified when running the AuthoringTool.
An error now occurs if the icon included in the NintendoSDK (NintendoSDK_Application.bmp) is set in the icon in the application image.

4-6-0S14: (TOOL) Added a feature to AuthoringTool for editing application control data in an NSP file.

Added the replacenspmeta command to AuthoringTool. This command can be used to edit the application control data included in an NSP file.
For more information, see Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

4-6-0S15: (TOOL) Added a feature to check for use of NEX with vulnerabilities in the AuthoringTool.

Improvements were made so that when the --error-unpublishable option is specified when running the AuthoringTool, an error occurs if there are vulnerabilities in the NEX being used.
For more information, see Technical Information / Master ROM Submission Information.

4-6-0S16: (TOOL) Added a feature to the AuthoringTool for checking icon consistency.

Added a check item for cases where the --error-unpublishable option is specified when running the AuthoringTool.
An error now occurs if the small icon in the application image has an image different from the one specified in the large icon.
For more information about icons, see Features > Build > Creating Application Images > Icons in Application Images.

4-6-0S17: (TOOL) You can now set the application's own ID in the SaveDataOwnerIds element of the NMETA file.

You can now set the application's own ID in the NintendoSdkMeta/Core/FsAccessControlData/SaveDataOwnerIds element of the NMETA file without causing a build error.
Only one ID can set for the application's own ID.
For more information about the NintendoSdkMeta/Core/FsAccessControlData/SaveDataOwnerIds element, see NintendoSDK Documents > Features > Building > Creating Application Images > Writing the NMETA File.

Bug Fixes

4-6-0S18: (DevMenu) Corrected the time zone notation.

Corrected old information in the time zone notation for America/Caracas.

4-6-0S19: (DevMenu) Deleted the option for writing NSP files for downloadable content to game cards.

To prevent a fatal error that occurred when writing downloadable content NSP files to game cards, that option was deleted.

4-6-0S20: (DevMenu) Fixed a bug where applications started during user selection.

Fixed a bug where an application would run regardless of whether user selection was complete if the HOME Button was pressed on the user-selection screen prior to starting up the application.

4-6-0S21: (DevMenu) Fixed a bug where specifying an incorrect NSP file when writing to a game card would cause a fatal error.

Fixed a bug where a fatal error would occur if you specified an invalid Patch when writing an NSP file containing a patch to a game card.

4-6-0S22: (NFP) Fixed a bug where NfpDebugTool would crash.

Fixed a bug where NfpDebugTool would crash when loading tags.

4-6-0S23: (TOOL) Adjusted the functionality of the SDK root selection page in NintendoSDK application and library projects.

There was a bug in the window displayed in Visual Studio for creating projects for NintendoSDK applications and libraries. On the Specify SDK Location page, the Next button could be pressed before the page's display preparations were complete. This has been fixed.

In addition, changes were made so that the page cannot transition if the Next button is pressed while an invalid password is specified as the SDK root.

4-6-0S24: (TOOL) Fixed a bug where errors were erroneously detected when using Pia or NEX.

Fixed a bug where errors such as "11-003: Access to this feature is restricted - Bundle_Private_XXX" were erroneously detected when using Pia or NEX.

4-6-0S25: (TOOL) Fixed a bug where it was possible to specify progressive JPEGs to small icons in the AuthoringTool.

There was a bug where it was possible to specify progressive JPEGs to small icons when creating application images in the AuthoringTool. This has been fixed.
For more information, see Features > Build > Creating Application Images > Icons in Application Images.

4-6-0S26: (TOOL) Fixed a bug where the AuthoringTool would terminate with an error when run if .NET Framework 4.7.1 was installed.

If .NET Framework 4.7.1, Windows 10 Fall Creators Update (Version 1709), or .NET Framework 4.7.1 was installed, the AuthoringTool would terminate with an error when run. This has been fixed.

Changes in NintendoWare NX Addon 4.6.0 (based on Generic 4.6.0)

Bug Fixes

4-6-0W01: (3D-G3D) Fixed a bug where binary conversion of intermediate model files sometimes failed after applying polygon reduction in Intermediate File Optimizer.

Fixed a bug in Intermediate File Optimizer (%NITENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe) where an error sometimes occurred during polygon reduction and binary conversion of intermediate model files to which the quantization analysis feature had been applied.

Quantization settings are now deleted after polygon reduction, so the quantization analysis feature must be used again to reconfigure any desired settings.

4-6-0W02: (3DTool) Fixed a bug in the polygon reduction feature of intermediate file optimizer where the length of the binormal would not become 1.

Fixed a bug in the polygon reduction feature of intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe) where the length of the binormal after reduction would not be 1.

4-6-0W03: (3DTool) Fixed a bug in the polygon reduction feature of the intermediate file optimizer where an error occurred if there was a submesh with a total surface area of zero.

Fixed a bug in the polygon reduction feature of the intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe) where an error would occur if there was a submesh with a total surface area of zero.

4-6-0W04: (3DTool) Fixed a bug in the polygon reduction feature of the intermediate file optimizer where hard edges would appear in 3D models.

Fixed a bug in the polygon reduction feature of the intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe) where hard edges would appear when a model intermediate file (FMDB file) that contained only one shape or a model intermediate file that cannot combine all shapes was simplified.

4-6-0W05: (3DTool) Fixed a bug where meshes that include normal vectors oriented in incorrect directions, were sometimes output in the polygon reduction feature of the intermediate file optimizer.

There was a bug where meshes that include normal vectors oriented in incorrect directions, were sometimes output when the bone value associated with shapes fulfilled the following conditions in the polygon reduction feature of the intermediate file optimizer (%NINTENDO_SDK_ROOT%/Tools/Graphics/3dTools/3dIntermediateFileOptimizer.exe).

  • An anisotropic value is set to the scale.
  • A value other than 0 is set to the rotation.


4-6-0W06: (LayoutTool) Fixed a bug in LayoutEditor where the display did not switch when you switched tabs in the Layout window.

Fixed a bug in LayoutEditor where the display did not switch when you switched between overlapping tabs in the Layout window.
