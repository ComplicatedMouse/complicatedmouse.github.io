Added HLSLCC_FLAG_FORCE_DECLARED_ORDER_ATTRIB_LOCATION (0x800000), which assigns numbers to the location of the vertex shader's in variable in order of the dcl_input, to HlslCrossCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe).

In NX Add-On 5.1.0, the location for the in variable for the vertex shader ends up as a semantic index, which breaks compatibility. However, you can treat the order as arranged by dcl_input as the location, which is the original behavior of this flag.

5-3-0S12: (HID) Added a vibration demo collection, containing models that introduce ways to effectively use the HD rumble feature.

The vibration demo collection is a demo that can be run on the EDEV or SDEV, which introduces ways to effectively use HD rumble.
TargetTools/NX-NXFP2-a64/VibrationDemo/Release/VibrationDemo.nsp

In this collection of vibration demos, users can experience vibrations attached
to UI components such as buttons, sliders, knobs, and switches.
These demos show how vibrations can make UI easier to understand and improve the feel of operations.

They demonstrate these improvements using a simple implementation that associates straightforward playback of vibration waveforms from the vibration collection for each UI component.
Hopefully these demos will be a useful reference when using vibrations in games.

For more information, see NintendoSDK Documents > Features > Input Device > hid Library > Tool Reference > Vibration Demo Collection.

5-3-0S13: (HID) Added an HD rumble sample program that easily generates vibration waveforms from parameters.

Added an HD rumble sample program that easily generates vibration waveforms from parameters.
Samples/Sources/Applications/HidVibrationGenerator

It is possible to design various vibration waveforms by directly including parameters in the source code.
Asset management is not required because vibration files are not used.
For that reason, it enables vibration features to be used more easily.

DumpedVibrationGenerator, which is included in the sample, is a class that inherits nn::hid::VibrationNode.
This class can generate various vibration waveforms by setting the amplitude, frequency, time, and attenuation curve parameters.
Similar to nn::hid::VibrationPlayer, play back vibration using Play(), and stop vibration using Stop().

In the sample, there is vibration if the pointer touches the line during gyroscope operations.
Multiple preset waveforms are provided, so switch between them to check how different vibrations feel.

For more information, see SDK Sample Backgrounder＞HID＞Generating Simple Vibration Waveforms From Parameters in NintendoSDK API Reference.

5-3-0S18: (ThermalManagement) Added a feature to DevMenu for enabling and disabling the feature for avoiding low-temperature burns.

Added Thermal Burn Prevention Mode to the Development Support Features settings in DevMenu.
You can enable or disable the feature for avoiding low-temperature burns.
When Thermal Burn Prevention Mode is set to Enabled, the system is forced to sleep when the all of the following conditions apply for a prolonged period of time.

  • You are operating in handheld mode.
  • There are no button presses or other obvious user operations.
