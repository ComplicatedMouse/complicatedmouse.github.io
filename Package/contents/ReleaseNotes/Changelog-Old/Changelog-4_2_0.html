Details

Changes in NintendoSDK NX Addon 4.2.0 (based on Generic 4.2.0)

Specification change that breaks your code

*4-2-0S01: (General) Added a video capture feature for the application.

Added a video capture feature for the application. In applications with the video capture feature enabled, the system always automatically stores a maximum of 30 minutes of video data in memory. By holding down the Capture Button, that video data can be saved to a file. This video capture feature is enabled by default in applications built with this version of the SDK.
For more information, such as the format of the saved video, also see this page.

For applications with the video capture feature enabled, in an NX64 environment, the memory that can be used by the application is reduced by 96 MiB, and that portion of memory is used by the system. There are no changes to the size of usable memory in an NX32 environment. Also, it affects the performance of the application. For more information, see this page.

In cases where you do not want to permit this functionality, of for other planned reasons, you can disable the video capture feature by setting Video Capture in the NMETA file to Disable. You can also use AuthoringEditor to configure this setting.
See Video Capture.

You can temporarily stop video capture in applications with the video capture feature enabled. For more information, see the newly added nn::oe::EnableRecording() and nn::oe::DisableRecording().

*4-2-0S02: (IRSENSOR) Fixed a bug where nn::irsensor::StopImageProcessor() compatibility was corrupted.

Fixes were made to make nn::irsensor::StopImageProcessor() asynchronous in NX Add-On 4.0.0 and later versions. Due to these fixes, there was a bug where nn::irsensor::StopImageProcessor(), called by NX Add-On 3.x and earlier versions, would also be asynchronous, changing its behavior.
StopImageProcessor specifications were reverted to the former synchronous behavior for NX Add-On 3.x and earlier versions. Also, an asynchronous version of nn::irsensor::StopImageProcessorAsync() was added so that the suspended state of the internal processor can be obtained using nn::irsensor::GetImageProcessorStatus(), which was added at the same time. This made it possible to wait for the stop process to complete on the application side.

*4-2-0S03: (TOOL) Deleted the NintendoSDK Portal.

Deleted the NintendoSDK Portal application, which was scheduled for deletion, from NintendoSDK 1.1.0.

From now on, call a similar feature provided in NDI, or use the various Install.bat files in the NintendoSDK\Installers folder. In most cases, administrator rights are required to run Install.bat.

4-2-0S18: (NVN) Added new NVN features.

Added the following features to NVN.

  • Added the function nvnDeviceGenerateDebugDomainId to obtain an incrementing globally unique ID.

  • Added the device property NVN_DEVICE_INFO_DEBUG_GROUPS_MAX_DOMAIN_ID to query the maximum value of the IDs which can be generated.

  • Added a new API for event synchronization between the GPU and CPU.
    • Added the function nvnEventBuilderSetDefaults to set the default state for an NVNeventBuilder object.
    • Added the function nvnEventBuilderSetStorage to set the backing GPU-accessible memory pool storage for a 32-bit unsigned integer value associated with an NVNeventBuilder object.
    • Added the functions nvnEventInitialize and nvnEventFinalize to initialize and finalize an NVNevent object, respectively.
    • Added the function nvnEventGetValue to get the value associated with an NVNevent object.
    • Added the function nvnEventSignal to use the CPU to signal an NVNevent object.
    • Added the function nvnCommandBufferSignalEvent to use the GPU to signal an NVNevent object.
    • Added the function nvnCommandBufferWaitEvent to pause work on the GPU until an event is signaled.
