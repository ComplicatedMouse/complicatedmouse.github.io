Details

Changes in NintendoSDK NX Addon 8.1.0 (based on Generic 8.1.0)

Specification change that breaks your code

*8-1-0S01: (ERR) Deprecated nn::err::ShowNetworkServiceError.

Merged the features of nn::err::ShowNetworkServiceError into nn::err::ShowError, and deprecated nn::err::ShowNetworkServiceError.

*8-1-0S02: (OpenGL) Deleted static links to the OpenGL functions in NX.

As planned, you can no longer statically link to functions in OpenGL ES 3.2 from the NX version of OpenGL, which we no longer recommend. To call OpenGL functions, you must use a function pointer to get them dynamically. You can also use the nn::gll library or another OpenGL loader library such as GLEW to load these functions automatically.

*8-1-0S03: (WEB) Deprecated headers, functions, and types that were scheduled for deprecation.

The following headers, functions, and types that were scheduled for deprecation in NintendoSDK 5.3.0 have been deprecated.
Replacement headers, functions, and types are available for each of the deprecated headers, functions, and types other than nn::web::SetFooterEnabled.

Reference the following information for replacement.

  • offline_Api.h is web_OfflineApi.h
  • offline_ShowArg.h is web_OfflineShowArg.h
  • offline_Types.h is web_OfflineTypes.h
  • nn::web::ShowApplicationHtmlDocumentPage() is nn::web::ShowOfflineHtmlPage()
  • nn::web::ShowApplicationHtmlDocumentPageArg is nn::web::ShowOfflineHtmlPageArg
  • nn::web::OfflinePageReturnValue is nn::web::OfflineHtmlPageReturnValue


*8-1-0S04: (WebSocket) Improved error handling during connection.

Added and changed the results returned when the nn::websocket::WebSocket::Open() fails.

The following results are now returned when HTTP status 300 errors occur.
Before this change, nn::websocket::ResultHttpStatus3xx was returned regardless of the status.

  • nn::websocket::ResultHttpStatus301
  • nn::websocket::ResultHttpStatus302
  • nn::websocket::ResultHttpStatus303
  • nn::websocket::ResultHttpStatus305
  • nn::websocket::ResultHttpStatus307
  • nn::websocket::ResultHttpStatus308
  • For any other status: nn::websocket::ResultHttpStatus3xx


The following result is now returned when HTTP status 200 errors occur.
Before this change, nn::websocket::ResultHttpStatus2xx was returned.

  • nn::websocket::ResultProtocolUpgradeRequestIgnored


Added Features

*8-1-0S05: (EC) Added a notification feature for shop purchases.

Added a shop purchase notification feature (nn::ec::GetPurchasedEvent and others) to the ec library.
Applications can now detect product purchases when users purchase consumables or passes through the shop application on NX devices or the Nintendo.com website.
Also added the EcPurchasedEvent sample demonstrating how to use this feature.
Note: Testing with firmware version 7.0.0-5.0 or later is required.

*8-1-0S06: (HID) Expanded the maximum number of users that can be specified in the Controller Support Applet.

Expanded the maximum number of users that can be specified in the Controller Support Applet from 4 to 8.

*8-1-0S07: (OE) Added a feature to create and run one system thread immediately after starting the application.

During the runtime initialization immediately after a program application is started, the system now automatically generates and starts running one thread. For more information about this thread, see Automatically Starting the System Thread in nnMain() and Library Initialization.

8-1-0S08: (BCAT) Added comparison operators for nn::bcat::Digest structures.

Added the equal to (operator==) and not equal to (operator!=) operators for nn::bcat::Digest structures.

8-1-0S09: (DevMenu) Added features to import and export network settings to and from SD cards, and to delete network settings.

For more information, see Features > Controlling the Development Hardware > DevMenu > Network.

8-1-0S10: (NVN) New NVN Features

Added the following features to NVN.

  • Added the nvnEventBuilderGetMemoryPool and nvnEventBuilderGetMemoryOffset functions to get the memory pool and offset used for an NVNeventBuilder.

  • Added the nvnEventGetMemoryPool and nvnEventGetMemoryOffset functions to get the memory pool and offset used for an NVNevent.


8-1-0S11: (OE) Added a function to enable and disable CPU overclocking.

Added SetCpuOverclockEnabled to the oe library as a function for enabling and disabling CPU overclocking.

  • Unlike the previous configuration methods using DevMenu and DevMenuCommand, you can change the settings from the application.
  • The CPU clock rate increases if either the SetCpuOverclockEnabled settings or the DevMenu and DevMenuCommand settings are enabled, and the development hardware support CPU overclocking.
  • The SetCpuOverclockEnabled settings are destroyed when the application is terminated.


8-1-0S12: (RunOnTarget) Added the --print-host-time option to add a time stamp to RunOnTarget and ControlTarget logs.

Added the --print-host-time option to add a time stamp to RunOnTarget and ControlTarget logs.
You can combine the time stamps with other logs for comparison.

8-1-0S13: (UTIL) Added an overload that does not take size to the nn::Span::subspan() function.

Added an overload that does not take size to the nn::Span::subspan() function.

8-1-0S14: (UTIL) Added features for memory resources.

Added a new nn::util::pmr namespace and added the following functions and classes.

  • nn::util::pmr::NewDeleteResource()
  • nn::util::pmr::GetDefaultResource()
  • nn::util::pmr::SetDefaultResource()
  • nn::util::pmr::PolymorphicAllocator


These functions now provide the same functionality as the functions and classes under the std::pmr namespace.

8-1-0S15: (VSI) New Features in VSI and Target Manager

  • VSI
    • Added properties for PGO.
      • Added options for using profile-guided optimization.
      • Select C/C++ > Code Generation > Profile-Guided Optimization > PGO Data Generation to build and run the application, and then use the same properties for Apply PGO Data to build.
      • For more information, see LTO/PGO Additional Information and VSI > Project Parameters.
  • Target Manager
    • Added functions to start saving logs.
      • Use nn::tm::StartSavingLog, nn::tm::StopSavingLog, nn::tm::StartSavingJsonLog, and nn::tm::StopSavingJsonLog.
    • Added functions for handling remote video.
      • Added functions to start and stop remote video recording, and to get image data.
      • Use functions such as nn::tm::StartVideoRecording and nn::tm::GetScreenImageData.

Added a feature to the Offline Web Applet to specify the video UI permissions flag.

Added the --media-player-ui-enabled command to WebCallerCommand.
You can use this command to enable and disable video UI and operations in Offline Web Applets opened by WebCallerCommand.

Improvements

*8-1-0S18: (OE) Changed the crash report creation feature to be enabled by default.

Changed the crash report creation feature to be enabled by default.

8-1-0S19: (LayoutTool) In LayoutEditor, changed the behavior when moving the selected pane with the arrow keys so that the constant value changes regardless of the scale of the parent pane.

In LayoutEditor, changed the behavior when moving the selected pane with the arrow keys so that the constant value changes regardless of the scale of the parent pane. Previously, this value was changed by applying a conversion so that it became a constant value in world coordinates with consideration of the effects of the scale of the parent pane. As a result, the coordinates of the pane itself were set as incomplete decimals.

Clarified the error message when nn::fs::Unmount() and nn::fs::Commit() fail due to the open state of files and directories. This error message is generated before the Develop built aborts.

8-1-0S27: (FS) Strengthened the illegal access check in FsAccessLogChecker version 23.1.

NINTENDO_SDK_ROOTIn FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 23.1, access logs with the following unexpected patterns are now considered as failure.

  • Invalid offset and size values in ReadFile or WriteFile access.
  • fetched is included in access outside of ReadFile.
  • A mount name for which Unmount access name is not mounted
  • A value that is not opened by handle in CloseFile and CloseDirectory


8-1-0S28: (GFX) Revised the description for ImguiSimple.

Revised incorrect information in the description for ImguiSimple.

8-1-0S29: (HID) Improved the behavior of the Controller Support Applet when started from an application that does not support handheld mode.

Even when started from an application that does not support handheld mode, the applet now supports cancel operations in handheld mode.
In addition, calling the single-player mode when handheld mode is disabled is no longer supported, and now returns nn::hid::ResultControllerSupportNotSupportedNpadStyle.

8-1-0S30: (NIFM) Reduced the amount of time from waking up to starting the wireless connection.

Reduced the amount of time from waking up to starting the wireless connection. However, even in the earlier specification there were situations where wireless was connected initially even when a wired connection was detected.
Whether wireless or wired is used first when waking up depends on the network connection status before entering sleep mode.
