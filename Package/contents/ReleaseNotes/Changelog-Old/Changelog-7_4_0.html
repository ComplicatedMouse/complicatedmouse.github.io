Details

Changes in NintendoSDK NX Addon 7.4.0 (based on Generic 7.4.0)

Specification change that breaks your code

*7-4-0S01: (TOOL) Added a feature to the makepatch command of AuthoringTool that checks whether the NSP file can be submitted.

Added a feature to the makepatch command of AuthoringTool that checks whether the NSP file can be submitted.
To check whether the NSP file can be submitted, specify the --error-unpublishable option when you run the makepatch command.
This option checks for the same items that are checked by the get-unpublishable-error command of Features > Building > Creating Application Images > Tool Reference > AuthoringTool.
The creation of the NSP file fails if any items require fixing.
If there are any fatal errors, the creation of the NSP file fails even if the --error-unpublishable option has not been specified.
For more information about the makepatch command, see Features > Patches > Building and Running a Patch.

Added Features

*7-4-0S02: (EC) Added a notification feature for shop purchases.

Added a shop purchase notification feature (nn::ec::GetPurchasedEvent and others) to the ec library.
Applications can now detect product purchases when users purchase consumables or passes through the shop application on NX devices or the Nintendo.com website.
Also added the EcPurchasedEvent sample demonstrating how to use this feature.
Note: Testing with firmware version 7.0.0-5.0 or later is required.

7-4-0S03: (TOOL) Added support for multi-program applications to the tool for getting crash reports.

The tool for getting crash reports now supports multi-program applications.

  • Added the --program_index option for specifying the program index inside the multi-program application.
  • Along with the --program_index option, you can now specify multiple ---nss options and --module_id options.


This change was included in NX Add-On 7.3.0 but was omitted from that revision history.

7-4-0S04: (TOOL) Added the --error-unpublishable option to the mergeapplication command of AuthoringTool.

Added the --error-unpublishable option to the mergeapplication command of AuthoringTool.
For more information, see Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating the Multi-Program Application NSP File.

Improvements

7-4-0S05: (EC) Removed release_date used by AOC item shelf in the EcCatalog sample.

AOC item shelf in the EcCatalog sample no longer uses release_date, so the places where release_date was used have been removed.

This change was included in NX Add-On 7.3.0 but was omitted from that revision history.

7-4-0S06: (TOOL) AuthoringEditor can now display master ROM submission errors grouped by importance.

You can now choose to display master ROM submission errors grouped by their level of importance when you open the NSP file in AuthoringEditor.
This change was included in NX Add-On 7.3.0 but was omitted from that revision history.

7-4-0S07: (TOOL) mergeapplication now fails if the legal information of all the programs included in a multi-program application do not match.

When you create a multi-program application using the mergeapplication command in AuthoringTool, the command now fails if the legal information of all the programs included in the multi-program application do not match.
For more information, see Features > Multi-Program Applications > Legal Information Restrictions.
