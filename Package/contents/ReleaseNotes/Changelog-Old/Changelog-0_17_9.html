Details

Changes in NintendoSDK NX Addon 0.12.9 (based on Generic 0.17.9)

Added Features

13-10S01: (TOOL) Added a feature to BackupSaveData for specifying the application ID of the save data to export.

Added a feature to BackupSaveData for qualifying which application's save data to export. (The --export-title option.)

13-10S02: (TOOL) The authoring tool's replace command now supports replacing multiple entries.

The authoring tool's replace command now supports replacing multiple entries. Replacement of the NCA files in the NSP file is also supported now.

Improvements

13-10S03: (NIFM) Added support for nn::nifm::HandleNetworkRequestResult() log output.

In environments such as Windows where a UI for error handling is not displayed, error codes are now output to a log when the nn::nifm::HandleNetworkRequestResult() function is called (except in the Release build).

Bug Fixes

13-10S04: (MOVIE) Fixed a bug where some MKV and WEBM video files could not play correctly.

When video files in MKV or WEBM format were specified for the Extractor::SetDataSource() function, the following error would occur and the files could not be processed correctly.
MatroskaExtractor: Corrupt webm source: <no-scheme URI suppressed>
This bug has been fixed.

Changes in NintendoWare NX Addon 0.12.9 (based on Generic 0.17.9)

Bug Fixes

13-10W01: (3D-G3D) Fixed a bug where an assertion failure would occur in the viewer library when a model with a material with a uniform block size of 0 was sent.

Fixed a bug where an assertion failure would occur in the viewer library when a model with a material with a uniform block size of 0 was sent from 3DEditor.
