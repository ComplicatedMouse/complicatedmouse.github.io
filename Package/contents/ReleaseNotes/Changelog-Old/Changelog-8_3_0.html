Details

Changes in NintendoSDK NX Addon 8.3.0 (based on Generic 8.3.0)

Added Features

*8-3-0S01: (DevMenu) Extended the Exhibition Mode feature.

Extended the Exhibition Mode feature to make the following possible.

  • Displaying images before applications start
  • Displaying images after applications are closed
  • Automatically closing applications
  • Forcibly closing applications
  • Starting applications safely to reduce the risk of resource data being misappropriated
  • Freezing save data when applications are started

For more information, see Features > Controlling the Development Hardware > DevMenu > Exhibition Features.

*8-3-0S02: (OE) Added support for getting GPU crash reports.

GPU crash dump information is now included in the crash report when a GPU crash occurs. It can be displayed using CrashReport.exe --detail.

8-3-0S03: (DevMenu) Added a feature for exporting play records to a CSV file.

Added a feature to DevMenuCommand to export play records for an application to an external device using an CSV file.
For more information, see Features > Controlling the Development Hardware > DevMenuCommand > Getting the Play Record.

8-3-0S04: (DevMenu) In DevMenu, added a feature for batch-installing NSP files in a directory.

In DevMenu, added a feature for batch-installing NSP files in a directory.
For more information, see Features > Controlling the Development Hardware > DevMenu > Managing Applications > Installing All NSP Files Directly in a Directory.

8-3-0S05: (EC) Added a function for displaying subscription details screens.

Added the nn::ec:: ShowShopSubscriptionDetails() function for displaying subscription details screens.

8-3-0S06: (EC) Added a sample to get information about other titles related to the EcCatalog samples.

Added the Related item list option to the EcCatalog start screen and added a sample to get and display information about other related titles.

8-3-0S07: (EC) Added an EcCatalog sample that shows how to get a list of downloadable items.

Added the AOC item list option to the start screen of EcCatalog and added a sample to get and display the downloadable items list.

Improvements

8-3-0S08: (Compiler) Updated to Clang for NX 1.7.8.

Updated Clang for NX to version 1.7.8.
The main changes are as follows.

  • Fixed a bug where the junction destination path was being exported in the VSI build error output.
  • Fixed a bug where differing object files were generated even when compiling the same source file.

Fixed a bug where symbols within the library specified in '#pragma comment(lib, "...")' could not be resolved when building with LTO enabled.

Bug Fixes

8-3-0S09: (FS) Updated FsAccessLogChecker to version 24.7.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 24.7. This version includes the following changes.

  • Added support to check the frequency of functions with the nn::fs::MountHostOption argument.
  • Added support for adding handle information to generate access logs.
  • The FS Access Log Details area for analysis results now also includes analysis range information.
  • Fixed a bug where the tool sometimes used the wrong path when the same handle value was assigned to different targets before and after the start value.
  • Enhanced the check for invalid access logs.


8-3-0S10: (MOVIE) Fixed a bug where a frame rate of 0 was returned for some WebM videos.

Fixed a bug where a frame rate of 0 was returned for some WebM videos.

8-3-0S11: (MOVIE) Fixed a bug where the last frame or more of VP8 and VP9 videos was not decoded.

Fixed a bug where during playback of VP8 and VP9 videos, the final frame or more of the video files were not decoded even if movie::Decoder had already returned an EOF.

8-3-0S12: (NIFM) Fixed a bug where you could not connect to an AP using WEP on Windows.

Fixed a bug where attempting to connect to an access point using WEP encryption from Windows resulted in error 2110-1100.

8-3-0S13: (TOOL) Fixed a bug where a public function was falsely detected when checking whether Master ROM submission was possible for an NSP file.

Fixed a bug where the public function nn::oe::SetCpuBoostMode() was falsely detected when the get-unpublishable-error command was executed for an NSP file in AuthoringTool.

Changes in NintendoWare NX Addon 8.3.0 (based on Generic 8.3.0)

Bug Fixes

8-3-0W01: (LayoutTool) Fixed a bug in Layout Binary Converter where separated mode UV matrix animations were not exported.

Fixed a bug in Layout Binary Converter where separated mode UV matrix animations were not exported to the binary file containing the overall animation.
