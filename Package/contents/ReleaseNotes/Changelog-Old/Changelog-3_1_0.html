Changes in NintendoSDK NX Addon 3.1.0 (based on Generic 3.1.0)

Specification change that breaks your code

*3-1-0S01: (BuildSystem) Revised the library structure.

Made the following changes to the structure of the library.
If you are using a custom build system, you may need to update it appropriately.

  • Moved the NVN library into SDK DLL.
  • Moved the VI library into SDK DLL.
  • Moved the SWKBD library into SDK DLL.
  • Moved the IRSENSOR library into SDK DLL.
  • Changed the ZLIB library from a dynamic library to a static library.
  • Changed the LIBCURL library from a dynamic library to a static library.
  • Changed the static library name of the INIT library.


*3-1-0S02: (IRSENSOR) Updated the gestures library.

Updated the gestures library.
Several structures in the library have been changed. Developers who are using the gestures library will need to update their code accordingly.

  • Implemented a machine learning process to make detection of hand information more accurate than before.
  • You can now get new information indicating whether arms or fingers are touching.


Added Features

*3-1-0S03: (FS) Added a cache storage feature.

Added a Cache Storage Feature for storing data that can be regenerated.

*3-1-0S04: (FS) Added a temporary storage feature.

Added a Temporary Storage Feature that makes it possible to temporarily save data that does not need to be made persistent.

*3-1-0S05: (FS) Added an application save data expansion feature.

Added a feature for increasing the size of save data from an application. There are restrictions on use of this feature. For information, see the FS library documentation.

3-1-0S06: (BuildSystem) Added support for enabling nlib when creating a NintendoSDK Application project.

In environments that have the nlib package installed, nlib can now be enabled in the wizard that is displayed when creating a new NintendoSDK Application project.

3-1-0S07: (BuildSystem) Added support for precompiled headers for NintendoSDK Application and NintendoSDK Library projects.

Added support for precompiled headers when generating NintendoSDK Application and NintendoSDK Library projects.

To use this feature, select the “Precompiled headers” option on the advanced application settings page in the project creation wizard.

3-1-0S08: (CODEC) Added the nn::codec::HardwareOpusMultiStreamDecoder class for decoding Opus multistream data without using the CPU.

Added the nn::codec::HardwareOpusMultiStreamDecoder class to nn::codec. This class makes it possible to decode Opus multistream data using Audio DSP.

3-1-0S09: (DevMenu) Added a feature for deleting the authentication token cache, for debugging purposes.

Added the [Debug] Invalidate token cache button to the user account details screen on the Accounts page.
For more information about this button, see Controlling the Development Hardware / DevMenu in the SDK documentation.

3-1-0S10: (DevMenu) Added an option to DevMenuCommand for setting the graphics API Vsync settings to the settings used on retail systems.

Added an option to DevMenuCommand for setting the graphics API Vsync settings to the settings used on retail systems. For more information, see the Graphics API VSync Behavior Settings section in the DevMenuCommand documentation.

3-1-0S11: (DevMenu) Added the Auto option to TV Sound in DevMenu.

Added the Auto option to the TV Sound page in DevMenu.
Auto behaves identically to the previous Surround option. More specifically, the Auto option enables surround output when the output device supports 5.1ch output. If the device does not support 5.1ch output, the Auto option uses stereo output.
Also changed the behavior of the Surround option. More specifically, the Surround option now uses surround output, even when the output device does not support 5.1ch output.

3-1-0S12: (EC) Added a function for displaying a list of consumable service items.

Added the nn::ec::ShowShopConsumableItemList() function, which displays a list of the consumable service items associated with the specified application.

3-1-0S13: (FS) Added a feature for expanding save data with patches.

Added a feature for expanding the space allocated for save data by applying a patch to update the NMETA file.
For more information, see the FS Library Manual.

3-1-0S14: (FS) Added a feature for getting file timestamps for debugging purposes.

Added the nn::fs::GetFileTimeStampForDebug() function for getting file timestamps. This function makes it possible to get the timestamps of files on the host PC.

3-1-0S15: (HID) Added a feature for rolling back the controller firmware.

Added a feature to ControllerFirmwareUpdater for rolling back the controller firmware to the NX Add-On 3.0.0 version, or earlier. This is to facilitate debugging of features that require the controller firmware to be updated.
To use this firmware rollback feature, run the UpdateControllerFirmwareForDebug.bat file located in TargetTools/NX-NXFP2-a64/ControllerFirmwareUpdater/Release.

3-1-0S16: (HID) Added HidShell support to DebugPad.

HidShell can now be used for DebugPad logging and automation.

3-1-0S17: (HID) Added several new startup modes to the controller support applet.

Added the following functions.
