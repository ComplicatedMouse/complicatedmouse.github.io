Details

Changes in NintendoSDK NX Addon 3.4.0 (based on Generic 3.4.0)

Specification change that breaks your code

*3-4-0S01: (BCAT) Revised the Result definitions

Revised the Result definitions as follows.

  • nn::bcat::ResultTooManyHandle → nn::bcat::ResultTooManyHandles
  • nn::bcat::ResultTooManySyncRequest → nn::bcat::ResultTooManySyncRequests


*3-4-0S02: (FS) Deleted nn::fs::MountSaveData(const char* name) for mounting save data for debugging

Deleted nn::fs::MountSaveData(const char* name) for mounting save data for debugging.
Use the nn::fs::MountSaveDataForDebug() function to mount save data for debugging.

*3-4-0S03: (General) Removed SDEV 1.8 and preMP from the list of hardware that can be initialized using InitializeSdev.

The SDEV 1.8 and preMP can no longer be initialized using InitializeSdev.
Also, note that although these items can still be selected in the UI as the hardware to initialize, selecting this option will not initialize the hardware.

Added Features

3-4-0S04: (APPLET) Added a feature to DevMenuCommand to switch an eShop

Added a feature to DevMenuCommand to switch between the eShop for development and the DummyShop.

3-4-0S05: (DevMenu) Added backup and restore features for the cache storage data

Added to DevMenu, backup and restore features for the cache storage data.

3-4-0S06: (DevMenu) Added commands for TV settings and Auto-Sleep settings to DevMenuCommand

Added the following commands for TV settings and Auto-Sleep settings to DevMenuCommand.

  • TV resolution settings
  • RGB range settings
  • TV Screen burn-in reduction settings
  • CEC feature settings
  • Auto-Sleep settings


For more information, see DevMenuCommand Manual.

Improvements

*3-4-0S07: (AUDIO) Added to the documentation, the cautions for audio features that handle SystemEvents

Added cautions for audio features, that handle SystemEvents, to the following functions. These issues may cause the application to stop working. Make sure that they are verified.

* nn::audio::OpenAudioIn()
* nn::audio::OpenDefaultAudioIn()
* nn::audio::OpenAudioIn()
* nn::audio::OpenDefaultAudioOut()
* nn::audio::OpenAudioRenderer()

*3-4-0S08: (Graphics) Added a description to explain the content of the logs recorded when GPU crashes occur

Added a description to explain the content of the logs recorded when GPU crashes occur.
For more information, see the GPU Error Guide in NVN Library > Manual.

*3-4-0S09: (TOOL) Deleted UserAccountSaveDataOperation from the NMETA file settings

Deleted UserAccountSaveDataOperation from the NMETA file settings.
If the UserAccountSaveDataOperation tag is described in the NMETA file, the AuthoringTool will fail to create the application image.

*3-4-0S10: (TOOL) Increased the amount of memory available to the application within the game card

Increased the available capacity in the game card from 128 MB to 144 MB, if the allocated memory for the application exceeds the guaranteed region.

*3-4-0S11: (WEB) Changed the function prerequisites for getting the URL that was last opened, from the result information displayed on the webpage.

Added that the nn::web::WebPageReturnValue::GetLastUrl() function has the prerequisite that the value obtained with the nn::web::WebPageReturnValue::GetWebExitReason() is nn::web::WebExitReason::WebExitReason_CallbackUrlReached.

Added that the nn::web::OfflineHtmlPageReturnValue::GetLastUrl() function has the prerequisite that the value obtained with the nn::web::OfflineHtmlPageReturnValue::GetOfflineExitReason() is nn::web::OfflineExitReason::OfflineExitReason_CallbackUrlReached.

3-4-0S12: (DevMenu) Improved the issue of the system entering auto sleep during installation

Though the system used to automatically enter sleep mode if the installation took an extended time, it has been improved to no longer enter sleep mode.

3-4-0S13: (FS) Added rules for specifying entries for filtering targets, used when using the resource data filter output feature in the AuthoringTool

Adding rules when specifying entries for filtering targets for the filter description file, enables more efficient filtering processing by omitting the majority of the entries.
For more information, see the Authoring Tool Documentation.

3-4-0S14: (TOOL) Added check items when creating the application image in the AuthoringTool

The following errors now occur when creating the application image in the AuthoringTool.

  • The language settings for the Title and Icon in the NMETA file do not match.
  • Only one of BcatDeliveryCacheStorageSize and BcatPassphrase is set in the NMETA file.


3-4-0S15: (TOOL) Deleted settings for using the touch screen for NMETA files

Deleted the settings for specifying the use of the touch screen for NMETA files.
A policy has been determined to no longer use these settings for the system features, so the application NMETA settings are no longer needed.


3-4-0S16: (TOOL) Revised specifications so that the included XML elements are retained as much as possible when saving NMETA files loaded in AuthoringEditor

Revised specifications so that the XML elements, originally included in the NMETA file, are retained as much as possible when saving NMETA files loaded in AuthoringEditor.

3-4-0S17: (WEB) Added the web API sample

Added sample programs that use the web API.

Bug Fixes

*3-4-0S18: (MOVIE) MovieDecoderPlayer sample now supports full-range RGB

Updated the shaders so that full-range YCbCr is converted to full-range RGB in the MovieDecoderPlayer sample.

*3-4-0S19: (TOOL) Fixed a bug where the --filter option was not applied when creating the NSP file

There was a bug where no filtering would be performed, even when the --filter option was specified, when creating NSP files. This has been fixed.

3-4-0S20: (AUDIO) Fixed a bug where there was noise mixed with the audio when starting the audio renderer

There was a bug where noise (approximately 5 ms to 40 ms) was mixed with the audio when starting the audio renderer. This has been fixed.

3-4-0S21: (AUDIO) Fixed a bug with the nn::audio::Resampler headers

Fixed a bug where the header files, related to the nn::audio::Resampler feature, were not included in the package.

3-4-0S22: (DIAG) Fixed a bug where the SD card log was not recorded unless the development hardware was connected to the host PC

Fixed a bug where the SD card log was not recorded unless the development hardware was connected to the host PC. This bug occurred when using NX Add-On 3.0.0 and later versions of the firmware.

3-4-0S23: (DIAG) Fixed a bug with CapstoneSimple

The cs_disasm_iter() function was used incorrectly in CapstoneSimple. The issue has been fixed.

3-4-0S24: (MEM) Fixed a bug where the GetTotalFreeSize() and GetAllocatableSize() functions sometimes returned a size that could not be mapped by the physical address when using the virtual memory management feature in nn::mem::StandardAllocator

Though the GetTotalFreeSize() and GetAllocatableSize() functions previously returned a size for the virtual address space when using the virtual memory management feature in nn::mem::StandardAllocator, they have been revised to return a size that can be mapped by the physical address (in the empty area for the physical address).

3-4-0S25: (MOVIE) Deleted Type.h from the MovieDecoderPlayer sample

Deleted the unused file Type.h from the MovieDecoderPlayer sample.

3-4-0S26: (MOVIE) Fixed a bug where the MovieDecoderPlayer sample did not end immediately after the final frame

The MovieDecoderPlayer no longer continues to display the final frame for a period of time, after playback ends.

3-4-0S27: (MOVIE) Fixed a bug where the MovieDecoderPlayer sample did not work in builds that suppressed ASSERT

Fixed a bug where the MovieDecoderPlayer sample did not work in builds that suppressed ASSERT.

3-4-0S28: (SWKBD) Fixed a bug where specifying the maximum character count for UTF-8 only with the swkbd string settings functions, caused a fatal error

There was a bug where a fatal error occurred when specifying the exact maximum string in the UTF-8 version string settings functions nn::swkbd::SetOkTextUtf8, nn::swkbd::SetHeaderTextUtf8, nn::swkbd::SetSubTextUtf8, nn::swkbd::SetGuideTextUtf8, and nn::swkbd::SetInitialTextUtf8. This has been fixed.

3-4-0S29: (TOOL) Fixed a bug where providing the path to "previous patch (NSP)" in the AuthoringEditor patch window, caused the check for patch positional relationship to be constantly run.

There was a bug where, if the patch NSP path was entered in the Previous Patch (NSP) field, in the patch creation window in the AuthoringEditor, the patch positional relationship check would constantly run, even when the Use Previous Patch check box was cleared. This has been fixed.
This bug is related to a warning message display and does not affect the patch creation process itself.

3-4-0S30: (TOOL) Fixed a bug where the NintendoSDK Documents for NX package could not be installed with NDICmd

There was a bug where, when using NDICmd to install the NintendoSDK Documents for NX package, the process would return termination code 0 and appear as though it had installed correctly. However, the package was not installed (= NintendoSDK Documents for NX package was not enumerated in NDICmd.exe list environments). This has been fixed.

3-4-0S31: (TOOL) Fixed an issue where there were insufficient rating classifications for PEGIPortugal in the rating input form in AuthoringEditor

There were not enough rating classifications for PEGIPortugal in the AuthoringEditor rating input form, so additional classifications were added.
The added ratings were ages 3 and 7, options for the PEGIPortugal rating classifications.

3-4-0S32: (VSI) Fixed a bug where VSI for NX could not be installed

There was a bug where VSI for NX could not be installed on the system if multiple values were included in the PSMODULEPATH environment variables. This has been fixed.

Changes in NintendoWare NX Addon 3.4.0 (based on Generic 3.4.0)

Specification change that breaks your code

*3-4-0W01: (Effect-VFX) Deprecated the option features for the function to emit the VFX runtime manually

For the nn::vfx::EmitterSet::EmitParticle function, it was previously possible from NX Add-On 3.0.0, to specify up to the 3rd parameter for the options (optional) for searching for empty particles internally. However, in specific cases, this was discovered to cause unintended behaviors, so it has been deprecated.
The function and behavior will be returned to the same specification in 1.x.x.
Compatibility is maintained if the 3rd parameter is not explicitly specified. If the 3rd parameter is specified, delete the 3rd parameter.
