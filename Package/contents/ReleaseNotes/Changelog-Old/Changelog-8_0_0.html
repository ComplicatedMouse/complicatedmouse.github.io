Details

Changes in NintendoSDK NX Addon 8.0.0 (based on Generic 8.0.0)

Specification change that breaks your code

*8-0-0S01: (AUDIO) Removed deprecated functions and constants.

Removed deprecated functions and constants from the nn::audio namespace.

Functions that work the same as the removed functions have been prepared and are available for you to use instead.


The following functions were removed.

* nn::audio::SetAudioInBufferInfo(nn::audio::AudioInBuffer*, void*, size_t)
* nn::audio::SetAudioOutBufferInfo(nn::audio::AudioOutBuffer*, void*, size_t)
* nn::audio::AcquireAudioDeviceSwitchNotification()
* nn::audio::LoadI3dl2ReverbPreset(nn::audio::I3dl2ReverbType*, nn::audio::I3dl2ReverbType::Preset)
* nn::audio::GetPolyphaseResampledOutBufferSize(const nn::audio::PolyphaseResamplerType*, size_t)
* nn::audio::GetResampledOutBufferSize(const nn::audio::ResamplerType*, size_t)
* nn::audio::ResamplerType::MaxChannels()
* nn::audio::SetVoiceBiquadFilterParameter(nn::audio::VoiceType*, const nn::audio::BiquadFilterParameter&)
* nn::audio::GetVoiceBiquadFilterParameter(const nn::audio::VoiceType*)

*8-0-0S02: (CODEC) Removed deprecated functions.

Removed deprecated functions from the nn::codec namespace.

Functions that work the same as the removed functions have been prepared and are available for you to use instead.


The following functions were removed.

* nn::codec::GetOpusPacketSampleCountPerFrame(const uint8_t[], size_t, int)
* nn::codec::GetOpusPacketChannelCount(const uint8_t[], size_t)
* nn::codec::GetOpusPacketSampleCountInPacket(const uint8_t[], size_t, int)
* nn::codec::GetOpusPacketFrameCount(const uint8_t[], size_t)

*8-0-0S03: (DevMenu) Removed battery draining commands from the DevMenuCommand development support features.

Removed the following three commands from the DevMenuCommand development support features.

  • get-battery-draining-performance-enabled
  • enable-battery-draining-performance
  • disable-battery-draining-performance

The targets that could be operated using these commands had no effect on the development hardware at any time since their initial preparation.

*8-0-0S04: (DIAG) NN_LOG and NN_ASSERT are now disabled when the NintendoSDK build type is the release build.

If you want NN_LOG or NN_ASSERT to be enabled when the NintendoSDK build type is the release build, define the NN_ENABLE_LOG macro or the NN_ENABLE_ASSERT macro.

Previously, regardless of the NintendoSDK build type, NN_LOG and NN_ASSERT were enabled when the NN_DISABLE_LOG and NN_DISABLE_ASSERT macros were not defined.

Note that the following bugs might occur due to this change.

  • By disabling NN_ASSERT, the program might not stop on an assertion failure and end up reaching some unintended code.
    • ⇒ If you want the program to stop on an assertion failure even when the NintendoSDK build is the release build, use NN_ABORT_UNLESS instead of NN_ASSERT.

  • If code for side effects has been written in NN_LOG or NN_ASSERT, that code is deleted, producing unintended program behavior.
    • ⇒ Do not write code for side effects in NN_LOG or NN_ASSERT.


In the sample programs, the NN_ENABLE_LOG macro is defined and NN_LOG is enabled even when the NintendoSDK build type is the release build.

*8-0-0S05: (SSL) Disabled Symantec root certificates.

The ssl library now distrusts Symantec root certificates by default. As a result, the SSL/TLS handshake with servers that use Symantec server certificates now returns nn::ssl::ResultSslErrorUnknownCertificate and fails.

Although we do not recommend it, the application can use the ssl library to explicitly import a Symantec certificate and make the SSL/TLS handshake succeed. For more information about the root CA certificates that the ssl library trusts, see the documentation for the ssl library.

*8-0-0S06: (TOOL) Added a feature to the makepatch command of AuthoringTool that checks whether the NSP file can be submitted.

Added a feature to the makepatch command of AuthoringTool that checks whether the NSP file can be submitted.
To check whether the NSP file can be submitted, specify the --error-unpublishable option when you run the makepatch command.
This option checks for the same items that are checked by the get-unpublishable-error command of Features > Building > Creating Application Images > Tool Reference > AuthoringTool.
The creation of the NSP file fails if any items require fixing.
If there are any fatal errors, the creation of the NSP file fails even if the --error-unpublishable option has not been specified.
For more information about the makepatch command, see Features > Patches > Building and Running a Patch.

*8-0-0S07: (TOOL) The texture converter now adjusts the alignment of the texture data when it exports a BNTX file that has linear tiling.

The texture converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe) now adjusts the alignment of the texture data when it exports a BNTX file that has linear tiling.
The tool now aligns the address and size of the data for each mipmap level to 4 bytes (the number of bytes in one block in compressed format) for export.
The size of the BNTX file or the required alignment of the buffer for reading the BNTX file may be larger than before in some cases.

Added Features

8-0-0S08: (AUDIO) Added to the information that can be obtained using Nintendo.AudioTool.SystemAudioCapture.

The information that can be obtained using the SystemAudioCapture tool included in Nintendo.AudioTool.dll now includes the capture waveform start time.

Along with this added information, incompatible changes were made to the process of communication between target and host.
Make sure that you use a combination of firmware and DLL with the same SDK version.
The same goes for SystemAudioMonitor.exe.

  • Clarified the documentation about debug groups.

  • Added debug layer checks to ensure that new CPU-accessible memory pools do not overlap with existing memory pools.

Added nn::os::TryWaitThread() as a function for polling for the thread to exit. For more information, see the API Reference.

8-0-0S17: (TOOL) Added the --use-sv-position-w option to HlslCrossCompiler.

*8-0-0S19: (TIME) Deprecated nn::time::SteadyClockTimePoint and the functions that use the structure.

The following structure and functions have been deprecated.

  • The nn::time::SteadyClockTimePoint structure
  • nn::time::GetSpanBetween()
  • nn::time::ClockSnapshot::GetStandardSteadyClockTimePoint()
  • nn::time::StandardSteadyClock::GetCurrentTimePoint()
  • nn::time::PeriodicBenefitClaimContext::GetEndOfPenaltyAtLastClaimForDebug()


To calculate the time that elapses while the application shuts down, use the nn::time::CalculateSpanBetween() function that takes nn::time::ClockSnapshot as an argument. This function can be used in more situations than nn::time::GetSpanBetween() to calculate the elapsed time.
Use nn::time::PeriodicBenefitClaimContext::GetSpanToEndOfPenaltyAtLastClaimForDebug() in place of the nn::time::PeriodicBenefitClaimContext::GetEndOfPenaltyAtLastClaimForDebug() function.

Specification change that breaks your code

*8-0-0W01: (FONT) Fixed a bug where the second font of PairFont would be used if the first font was not a plotted font.

Fixed a bug where the second font of PairFont would be used if the first font was an unplotted scalable font.

In addition, added IsGlyphExistInFont as a virtual function to the Font class to check whether glyphs exist in the font, regardless of whether the font is plotted.

*8-0-0W02: (3DTool) Changed the exit code values for 3dBinaryConverter.exe and 3dShaderConverter.exe.

Changed the exit code values for the 3D binary converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe) and the 3D shader converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe).
Previously, the two exit code values were 0 for success, and 1 for failure. In SDK 8.0.0 and later, the exit code value for success is still 0, but the exit code for failure is an integer value of 1 or greater.

If you have batch processes and the like that are designed to make determinations based on whether the exit code is 1 or not, you must revise those processes.

*8-0-0W03: (3DTool) The 3D texture converter now adjusts the alignment of the texture data when it exports a BNTX file that has linear tiling.

The 3D texture converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) now adjusts the alignment of the texture data when it exports a BNTX file that has linear tiling.
The tool now aligns the address and size of the data for each mipmap level to 4 bytes (the number of bytes in one block in compressed format) for export.
The size of the BNTX file or the required alignment of the buffer for reading the BNTX file may be larger than before in some cases.

*8-0-0W04: (Layout-UI2D) Removed the vector graphics feature.
