Details

Changes in NintendoSDK NX Addon 0.12.8 (based on Generic 0.17.8)

Bug Fixes

13-9S01: (AUDIO) Fixed a bug in the nn::audio::SetDelayInputOutput() function.

Fixed a bug where the audio renderer would become unstable and the system would crash if the input/output was switched by the nn::audio::SetDelayInputOutput() function when the nn::audio::DelayType structure was in use.

This bug occurred if nn::audio::RequestUpdateAudioRenderer() was called without the input/output having been set by the nn::audio::SetDelayInputOutput() function after the addition of nn::audio::DelayType.

As part of this bug fix, the state of nn::audio::DelayType now is reset when nn::audio::SetDelayInputOutput() is called.
Note that noise can get mixed in the waveforms if nn::audio::SetDelayInputOutput() is called while nn::audio::DelayType is active.

13-9S02: (FS) Fixed a bug in the write size calculated by FsAccessLogChecker.

The FsAccessLogChecker tool (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) erroneously includes the size specified by nn::fs::SetFileSize() in its calculation of the write size. This bug has been corrected in FsAccessLogChecker version 1.2.
