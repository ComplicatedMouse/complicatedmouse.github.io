Details

Changes in NintendoSDK NX Addon 6.0.0 (based on Generic 6.0.0)

Specification change that breaks your code

*6-0-0S01: (FONT) Removed the fontll2 library.

Removed the fontll2 library from the package because there was a problem with how the library was being provided.

If you were previously using the fontll2 library, use the fontll library instead from now on.

*6-0-0S02: (AUDIO) Changed the name and namespace for Nintendo.Audio.dll.

Changed the name of Nintendo.Audio.dll to Nintendo.AudioTool.dll.
As part of this change, also changed the namespace for the functions in Nintendo.Audio.dll from Nintendo.Audio to Nintendo.AudioTool.

Nintendo.Audio.dll can still be referenced in the current version, but the associated API functions are now deprecated, and will be removed in a future version.

*6-0-0S03: (CRYPTO) Improved performance of CMAC mode and CCM mode.

Improved performance of the CMAC mode and the CCM mode provided in the nn::crypto namespace.

Note that member variables of the following classes have also been changed as part of this change.

  • nn::crypto::Aes128CmacGenerator
  • nn::crypto::Aes128CcmDecryptor
  • nn::crypto::Aes128CcmEncryptor
  • nn::crypto::CcmDecryptor
  • nn::crypto::CcmEncryptor
  • nn::crypto::CmacGenerator


*6-0-0S04: (RO) In the UnloadModule function, added a check that the function is not being called from the same thread that called the constructor for the global TLS object in the NRO file to be unloaded.

In the UnloadModule function, added a check that cancels the function call if the constructor for the global TLS object in the NRO file to be unloaded has been called, and the function is being called from the same thread from which that constructor was called.
Never call this function from a thread from which a function that calls the constructor for the global TLS object was called.

*6-0-0S05: (TOOL) Removed AlbumDownloader.nsp from the package.

Deprecated AlbumDownloader and removed it from the SDK package, as previously announced. The same functionality is now provided via the album subcommand in DevMenuCommand. Use this subcommand to access this functionality from now on.

6-0-0S20: (Graphics) Added the Maxwell PerfWorks Profiling Guide.

6-0-0S21: (HID) Added support for automatic operations and operation logging for the Switch Pro Controller and the Joy-Con.

6-0-0S22: (NGC) Added an interface that takes char16_t and char32_t parameters to the profanity checking library.

Previously, when working with UTF-16 and UTF-32 encoded data in the profanity checking library, only the interface supporting uint16_t and uint32_t parameters was available. Now, an additional interface is available that supports char16_t and char32_t parameters.

6-0-0S23: (NVN) New NVN Features

Added the following features to NVN.

  • Added new debug enum values to NVNdebugCallbackSource and NVNdebugCallbackType.

  • Added the nvnSyncInitializeFromFencedGLSync and nvnSyncCreateGLSync functions to allow sync interoperability between OpenGL and NVN.

  • Added the new enum value NVN_BARRIER_INVALIDATE_L2_CACHE_BIT to NVNbarrierBits.

  • Added the nvnTextureBuilderGetRawStorageClass and nvnTextureGetRawStorageClass functions for querying the RAW storage class associated with an NVNtextureBuilder object or an NVNtexture object, respectively.

  • Added internal NVN extensions to allow for mapping memory pools to predefined addresses on Windows. No public API changes with this version.
