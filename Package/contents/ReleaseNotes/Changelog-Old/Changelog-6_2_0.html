Details

Changes in NintendoSDK NX Addon 6.2.0 (based on Generic 6.2.0)

Specification change that breaks your code

*6-2-0S01: (AUDIO) Deprecated nn::audio::GetResampledOutBufferSize() and nn::audio::GetPolyphaseResampledOutBufferSize().

Deprecated the following functions in the nn::audio namespace.

* nn::audio::GetResampledOutBufferSize()
* nn::audio::GetPolyphaseResampledOutBufferSize()

Use the following functions instead.

* nn::audio::GetResamplerOutputSampleCount()
* nn::audio::GetPolyphaseResamplerOutputSampleCount()

(The buffer size parameter can be obtained by multiplying the number of samples by the number of channels and the value returned by sizeof(int16_t).)

*6-2-0S02: (OS) Improved the user exception handler.

Made the following improvements to the user exception handler.

  • When the user exception handler returns program control, it now restores the context in which the user exception was thrown before it stops running.
    • Also, the context displayed when data is attached or dumped after starting the user exception handler is this same context where the user exception was thrown.
    • However, note that memory overwritten by the user exception handler remains overwritten.
    • This change also increases the amount of stack memory consumed by the user exception handler.
  • Added the fpscr, fpcr, and fpsr values to nn::os::UserExceptionInfoDetail.


Added Features

*6-2-0S03: (EC) Added an eShop service access utility feature.

Added an eShop service access utility feature to the ec library. This feature makes it possible to get a list of consumable service items that are currently available on Nintendo eShop for the application from within the application itself. Also added the EcCatalog sample demonstrating how to use this feature.

6-2-0S04: (AUDIO) Added a feature for recording two-channel audio in SystemAudioMonitor.

Added a feature for recording two-channel audio in SystemAudioMonitor. For more information, see Audio Library > Tool Reference > SystemAudioMonitor.

6-2-0S05: (AUDIO) Added an option for skipping voice pitch conversion and sampling rate conversion.

Added an option for skipping the pitch conversion and sampling rate conversion processes executed when playing waveforms using voices in an audio renderer.
Enabling this option limits the playable types of waveforms to those that have a sampling rate equal to the audio renderer's rendering rate, and also disables the ability to change playback pitch. However, skipping these conversion processes yields a proportional reduction in the processing load associated with rendering voices.


For more information, see the function reference.

* nn::audio::AcquireVoiceSlot()
* nn::audio::VoiceType::BehaviorOptions
* nn::audio::SetVoicePitch()

6-2-0S06: (CODEC) Added functions for resetting Opus decoders.

Added the following functions to the nn::codec namespace. These functions reset Opus decoders to the state corresponding to immediately after calling the initialization function.

* nn::codec::ResetOpusDecoder()
* nn::codec::ResetOpusMultiStreamDecoder()
* nn::codec::ResetHardwareOpusDecoder()
* nn::codec::ResetHardwareOpusMultiStreamDecoder()
* nn::codec::OpusDecoder::Reset()
* nn::codec::OpusMultiStreamDecoder::Reset()
* nn::codec::HardwareOpusDecoder::Reset()
* nn::codec::HardwareOpusMultiStreamDecoder::Reset()

6-2-0S07: (CODEC) Added the --disable-prediction option to OpusEncoder.exe.

Added an option for disabling the Opus prediction feature when using the celt mode in OpusEncoder.exe. This option makes it possible to more easily reduce noise when randomly seeking or using looped playback. For more information, see codec Library > Manual > Feature Description > Opus Decoder and codec Library > Tool Reference > OpusEncoder.

6-2-0S08: (DevMenu) Added a feature for preconfiguring the user account selected when using exhibition mode.

Added a feature for preconfiguring the user account selected when using exhibition mode. This feature can be selected when configuring applications to be started automatically for exhibition purposes.

6-2-0S09: (Display) Added a function for specifying a display range within a layer's textures.

Added a function for specifying a display range within a layer's textures to the nn::vi library.
This function (nn::vi::SetLayerCrop) provides the same functionality as nvnWindowSetCrop to applications using a graphics API other than NVN such as OpenGL.

6-2-0S10: (GFX) Added info structures that depended on the low-level graphics API.

Added versions of the following info structures which are specific to the low-level graphics API being used.

  • nn::gfx::BufferInfo
  • nn::gfx::MemoryPoolInfo
  • nn::gfx::QueueInfo
  • nn::gfx::SamplerInfo
  • nn::gfx::TextureInfo

For more information about these API-specific info structures for low-level graphics, see Info and Objects.

6-2-0S11: (HID) Added a feature for sending key lock events to keyboards.

Added the nn::hid::SendKeyboardLockKeyEvent() function, which sends key lock events to keyboards. This function can be used to change the lock state of external keyboards. For more information, see this page.

6-2-0S12: (MOVIE) Added a buffer swapping function.

Added a buffer swapping function (movie::Buffer::Swap) which makes it possible to transfer movie::Buffer content and ownership between instances.

6-2-0S13: (MOVIE) Updated the multimedia sample to show how to use the new nn::audio::PolyphaseResampler class in the movie::Player library.

Updated the multimedia sample to show how to use the new nn::audio::PolyphaseResampler class in the movie::Player library.

6-2-0S14: (NIFM) Added a new error handling function.

Added a new error handling function called nn::nifm::HandleNetworkRequestErrorResult(). Note that the existing nn::nifm::HandleNetworkRequestResult() function will be deprecated in the future.

6-2-0S15: (NVN) New NVN Features

Added the following features to NVN.

  • Added the nvnQueueBuilderSetQueuePriority and nvnQueueBuilderGetQueuePriority functions for specifying queue priority to the queue builder.


6-2-0S16: (RunOnTarget) Added the --working-directory option to RunOnTarget.

Added the --working-directory option to RunOnTarget.
This option makes it possible to run an application from RunOnTarget in a mount directory specified using a relative path, in the same way that you might use the MountHost function to do something like MountHost("host", "./SomeDirectory");.
For example, you can use it as follows.
"RunOnTarget.exe app.nsp --working-directory D:/SomeDirectory"

6-2-0S17: (SSL) Added VerifyOption_EVCertFingerprint as a new verification option in the nn::ssl library.

Added VerifyOption_EVCertFingerprint as a new verification option in the nn::ssl library. This option verifies the fingerprints of CAs found in the received certificate chain that are EV qualified with a built-in list. This new option also verifies the policy OID the same way VerifyOption_EvPolicyOid does.

6-2-0S18: (TOOL) Added the auto parameter to the --shift-patch-location option in HlslCrossCompiler.

Added the auto parameter to the --shift-patch-location option in HlslCrossCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe).

When this auto parameter is passed to the -- shift-patch-location option, an offset is applied to the patch in and patch out variables to prevent them from colliding with the locations of the in and out variables for hull shaders and domain shaders. Also, location numbers can be reserved as the location numbers for the patch in and patch out variables to prevent gaps from being created between the locations as a result of truncating system semantics.

6-2-0S19: (TOOL) Added the --enable-packoffset option to HlslCrossCompiler.

Added the --enable-packoffset option to HlslCrossCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe). Use this option to enable consideration of a packoffset value for cbuffer objects when converting to GLSL. For more information, see the documentation.

Improvements

*6-2-0S25: (HID) Changed the behavior of calls made to the controller support applet while NpadJoyHoldType_Vertical is set.

Previously, enabling NpadStyleHandheld when making a call to the controller support applet while NpadJoyHoldType_Vertical was set caused the applet to return ResultControllerSupportNotSupportedNpadStyle and close immediately. Starting in FW 6.0.0-2.0, the applet can be successfully started by making this call.
Due to this change, it is also possible to leave the controller support applet while using handheld mode.

*6-2-0S26: (HID) Removed all internal use of the winmm.dll library.

The internal implementations of the hid library and Input Director no longer depend on winmm.dll. This improves overall stability and makes applications less likely to crash when a USB gamepad is connected or disconnected. Due to this change, the Input Director gamepad settings for DirectInput mode must be reconfigured.

*6-2-0S27: (TOOL) Added support for getting and sending crash reports in the development environment.

Added support for getting and sending crash reports in the development environment.

ProdMode, DevMenu, or DevMenuCommand must be used to enable the feature for sending crash reports in the development environment. For more information, see Enabling the Feature for Sending Crash Reports in the Development Environment.

To get a crash report for an event that occurred in the development environment, specify the --dev option when running the command. For more information, see Crash Reports.

*6-2-0S28: (Keycode processing) Keyboard layout settings are now referenced when getting character code conversion tables.

Now that the system settings support keyboard layout configuration, the conversion tables for conversion of character codes corresponding to key input are selected based on the keyboard layout settings rather than the language settings. For more information, see this page. Keyboard layout settings can be changed in DevMenu and Settings Manager.

6-2-0S29: (AUDIO) Changed the minimum and maximum supported sampling rate ratios in PolyphaseResampler.

Changed the values returned by the following functions. This change makes it possible to convert a wider range of sampling rates than before.

* nn::audio::PolyphaseResamplerType::GetConversionRatioMax()
* nn::audio::PolyphaseResamplerType::GetConversionRatioMin()

6-2-0S30: (AUDIO) Relaxed the preconditions that must be satisfied to successfully call functions for working on FinalMix and SubMix objects while an audio renderer is running.

Relaxed the preconditions that must be satisfied to successfully call certain functions for working on FinalMix and SubMix objects while an audio renderer is running.

* nn::audio::AcquireSubMix()
* nn::audio::ReleaseSubMix()
* nn::audio::SetSubMixDestination()

Fixed a bug in the Photoshop plug-in where the texture export plug-in did not work correctly when a slash-delimited PSD file path was specified as a parameter to Photoshop.exe.

6-2-0W43: (3DTool) Fixed the conversion times in the Shader Converter and Binary Converter logs.

After a Shader Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) or Binary Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe) conversion is complete, the conversion time is saved to a log. Fixed a bug where the values after the decimal value in these conversion times was incorrect.
For example, a conversion time of 1.6 seconds was displayed when the actual conversion time was 1.006 seconds.

6-2-0W44: (3DTool) In 3DEditor, shaders are now exported again when Automatic Shader Regeneration is enabled in the toolbar.

In 3DEditor, shaders are now exported again when Automatic Shader Regeneration is enabled in the toolbar.
If there were any parameters that were edited while the automatic generation was disabled, the latest shaders at the point the feature is enabled are exported.
