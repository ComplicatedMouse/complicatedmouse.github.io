Details

Changes in NintendoSDK NX Addon 4.5.0 (based on Generic 4.5.0)

Added Features

4-5-0S01: (DevMenu) Added a feature for entering user names using the software keyboard.

User names can now be entered freely using the software keyboard.

4-5-0S02: (DevMenu) Added a feature to DevMenu for displaying the audio output target and the number of channels.

Added a feature to DevMenu for displaying the audio output target and the number of channels.

4-5-0S03: (Sample) Added a sample to get the supported languages.

Added a sample that gets supported languages from a program to the Samples/Sources/Applications/OeGetLanguage folder.

Improvements

4-5-0S04: (DevMenu) Expanded the maximum size for launch parameters when using DevMenu to 16 KiB.

Expanded the maximum size for launch parameters when using DevMenu from 4 KiB to 16 KiB.

For more information about launch parameters, see Features > Basic Features > oe Library > Manual > Features > Getting Launch Parameters.

4-5-0S05: (TOOL) Added a feature for checking the character encoding for tag names when creating downloadable content.

When creating downloadable content in the AuthoringTool, an error now occurs if characters that cannot be specified in NMETA file tags are set.
For more information, see Features > System > AOC Library > Creating Downloadable Content.

4-5-0S06: (TOOL) Added a feature for checking the string length of tag names when creating downloadable content.

When creating downloadable content in AuthoringTool, an error now occurs if a string that exceeds 128 characters is specified in NMETA file tags.
For more information, see Features > System > AOC Library > Creating Downloadable Content.

4-5-0S07: (TOOL) Added a feature to AuthoringTool for displaying diff information between patches.

Added the diffpatch command to AuthoringTool. This command displays file-based diff information between patches.

4-5-0S08: (TOOL) Improved patch generation times in AuthoringTool.

Optimized the data conversion processes in AuthoringTool, which yielded particularly large improvements in patch generation times.

4-5-0S09: (TOOL) Improved the checks for content with size updates in save data, temporary storage, and cache storage when creating patches.

When updating the size of save data, temporary storage, and cache storage using patches, in some cases invalid values were entered or updated when creating patches. For this reason, the check contents have been improved.

4-5-0S10: (TOOL) Improved the usability of several features in AlbumDownloader.

Improved the usability of several features in AlbumDownloader.

  • AlbumDownloader can now run in the background while an application is running. (The --suppress-auto-kill option must be specified when starting the application with RunOnTarget.exe.)
  • When downloading data, if any files (such as video files) are currently being saved on the target side, AlbumDownloader now automatically waits until the file saving process is complete.
  • After waiting for 60 seconds, if the file saving process is still not complete, AlbumDownloader cancels the download and records a download error to the log.


Bug Fixes

4-5-0S11: (DevMenu) Added a feature for checking NSP files to install.

Strengthened the check for whether the NSP file to install, specified by DevMenuCommand, is appropriate.
The appropriate logs, such as when it is not an absolute path or if an NSP file is not being specified, are now displayed.

4-5-0S12: (DevMenu) Fixed a bug where a fatal error occurred when a large number of downloadable content packages were installed.

Fixed a bug where installing a large number of downloadable content packages and attempting to open the content management screen in DevMenu by using the Manage button caused a fatal error.
As part of this bug fix, also improved installation times for NSP files including a large number of downloadable content packages.

4-5-0S13: (DevMenu) Fixed a display bug on SDEV systems that need a memory repair.

Fixed a bug where SDEV systems that had already undergone a memory repair displayed "Needs Memory Repair" on the Debug page in DevMenu, but only immediately after the system was initialized using InitializeSdev.
As part of this bug fix, the "Needs Memory Repair" message is now only displayed in DevMenuApp. It is no longer displayed in DevMenu.

4-5-0S14: (FS) Fixed a bug with expanding cache storage on SD cards.

Fixed a bug where the cache storage could no longer be opened if the process of using a patch to expand cache storage on an SD card failed because the SD card had insufficient free space.

4-5-0S15: (FS) Updated FsAccessLogChecker to version 12.3.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 12.3. This version includes the following changes.

  • Fixed where FS Access Log Details is placed in the analysis results.
  • Added support for CR as a newline code.
  • Added support for Shift-JIS, including multiple-byte characters.
  • Added support for checking nn::fs::GetSaveDataSize() and the indexed cache storage feature.
  • Changed the output format for read operations that are not related to files for SameAddressRead items.
  • Added the --warning_ratio option, and support for the continuous output of top values for SameAddressRead that do not exceed the threshold.


4-5-0S16: (IRSENSOR) Fixed a bug where pointing mode output values were not normalized correctly.

Previously, the <em>position</em> data in nn::irsensor::PointingProcessorState objects returned by the nn::irsensor::GetPointingProcessorStates function was not normalized. The upper-left corner of the screen was at (0, 0) and the lower-right corner of the screen was at (320.0, 240.0). This data is now correctly normalized such that the center of the screen is at (0, 0), the upper-right corner of the screen is at (1.0, 1.0), and the screen aspect ratio is 4:3.

4-5-0S17: (MOVIE) Fixed a bug where Matroska files containing only AAC audio data and no video data could not be played.

Matroska files containing only AAC audio data and no video data can now be played.

4-5-0S18: (MOVIE) Fixed a bug where the amount of memory used steadily increased during playback of Matroska and WebM files.

Fixed a bug where the amount of memory used steadily increased during playback of Matroska and WebM files.

4-5-0S19: (TOOL) Added a workaround for a bug in the Nintendo NX CPU Profiler where sampling was sometimes not possible.

Reverted the sampling method to the earlier format as a workaround for a bug in the Nintendo NX CPU Profiler where sampling was sometimes not possible. Because threads with affinities set in multiple cores could not be observed in the earlier method, set affinities only in a single core when using the profiler.

4-5-0S20: (TOOL) Fixed a bug where createnspd sometimes failed when specifying filter description files in AuthoringTool.

Fixed a bug where creating NSPD would fail when the program0.ncd/data directory already existed in the .nspd directory when filter description files were specified, and the createnspd subcommand in AuthoringTool specified options such as --program for files or directories in the output directory set with the -o option.

4-5-0S21: (TOOL) Fixed a bug where error logs were not output even when serial numbers that do not exist in SystemUpdateEdev were specified.

Previously, the error log was not output even when serial numbers that did not exist in the EDEV update tool (SystemUpdateEdev) were specified. It is now output.

4-5-0S22: (TOOL) Fixed a bug where file loading became slow in some cases when performing edits, such as when the replace command in the AuthoringTool changed file sizes.

Fixed a bug where loading resource data and HTML document files slowed down for applications or patches created based on those applications.
This bug occurred when resource data and HTML document files included in the application were edited for each file using the replace command in the AuthoringTool,
and the file size was changed from the original size.

Also, due to this fix, patch sizes no longer increase unintentionally when the replace command is used.

4-5-0S23: (TOOL) Fixed a bug where installation of the document package failed if Visual Studio was uninstalled but the Help Viewer remained.

Fixed a bug where installation of a document package failed if Microsoft Visual Studio was uninstalled, but the Microsoft Help Viewer remained.

4-5-0S24: (TOOL) Fixed a bug where the behavior of the ControlTarget commands for controlling the system power was incorrect.

Fixed a bug where in NX Add-On 4.3, the behavior of the power-off, power-on, and reset commands in ControlTarget was incorrect.

4-5-0S25: (TOOL) Fixed a bug where the list command in AuthoringTool did not display deleted files.

Fixed a bug with the list command in AuthoringTool where using the --patched-only option to only display files included in a patch sometimes caused files that had been deleted in the patch to not be displayed in the "Listing removed files from original" section.

Changes in NintendoWare NX Addon 4.5.0 (based on Generic 4.5.0)

Added Features

4-5-0W01: (Layout-UI2D) Added a feature for setting the LOD bias.

Added a feature for setting the LOD bias using nn::ui2d::GraphicsResource::Setup.

For more information, see Preventing Texture Blurring When Scaling.

Bug Fixes

4-5-0W02: (LayoutTool) Fixed a bug in LayoutEditor where parts were not loaded correctly when a batch process was run on all files in a folder.

Fixed a bug in LayoutEditor where parts were not loaded correctly when executing the processes for re-exporting the layout files under the parts root folder, batch-synchronizing the layout files in the parts folder, or updating thumbnail images under a specified folder.

4-5-0W03: (LayoutTool) Fixed a bug in LayoutEditor where shortcuts did not work in curve editor text boxes.

Fixed a bug in LayoutEditor where shortcuts did not work in curve editor text boxes.

4-5-0W04: (LayoutTool) Fixed a bug in LayoutEditor where specifying an argument and opening a layout file would cause an error to occur.

Fixed a bug in LayoutEditor where minimizing the Layout window, closing a layout file, and then starting LayoutEditor with that same layout file specified for the argument caused the creation of the D3D device to fail.

4-5-0W05: (LayoutTool) Fixed a bug in LayoutEditor where text box panes that used a scalable font were not displayed.

Fixed a bug in LayoutEditor where text box panes that used a scalable font were not displayed.

4-5-0W06: (LayoutTool) Fixed a bug in LayoutEditor where the constant color of the detailed combiner would not be displayed correctly in the editing screen when the option to specify color parameters in linear color space was not selected.

Fixed a bug in LayoutEditor where the constant color of the detailed combiner would not be displayed correctly in the editing screen (inconsistent with the display in the viewer and the game program) when the option to specify color parameters in linear color space was not selected.

4-5-0W07: (LayoutTool) Fixed a bug where LayoutEditor would stop responding if you clicked the icon for related parts after a font failed to load.

Fixed a bug where LayoutEditor would stop responding if you clicked the icon to display related parts in the Layout window after a font failed to load.
