Changes in NintendoSDK NX Addon 5.0.0 (based on Generic 5.0.0)

Specification change that breaks your code

*5-0-0S01: (FONT) Moved BfttfMaker from platform-specific packages into the standard package.

Moved BfttfMaker from platform-specific packages into the standard package.

As part of this change, BfttfMaker is now located at Tools/CommandLineTools/Fontll/BfttfMaker.exe instead of Tools/Graphics/LayoutTools/BfttfMaker.exe.

*5-0-0S02: (General) Ended support for prototype Switch Pro Controllers.

Ended support for prototype Switch Pro Controllers.

*5-0-0S03: (General) Ended support for Visual Studio 2013.

Ended support for Visual Studio 2013, as scheduled.

*5-0-0S04: (GFX) Increased the amount of memory consumed by the nv::SetGraphicsAllocator heap.

Changed the assignment patterns used in the nv::SetGraphicsAllocator heap. In particular, memory consumption has increased by 264 bytes for each texture and is now displayed on the screen. You can measure the increase in memory consumption using one of the following functions that corresponds to the graphics API you are using (NVN, OpenGL, or Vulkan).

  • nvnWindowInitialize
  • eglSwapBuffers
  • vkCreateSwapChainKHR


*5-0-0S05: (MII) Fixed a bug where the nn::mii::Database::UpdateLatest() function did not return nn::mii::ResultBroken.

Fixed a bug where the nn::mii::Database::UpdateLatest() function returned a value other than nn::mii::ResultBroken when passed a corrupted instance of nn::mii::CharInfo.

*5-0-0S06: (TOOL) Removed the features for intermediate texture files from Texture Converter.

Removed the features for intermediate texture files from Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe).
Also removed the Texture Preset Converter tool from GraphicsTools.

*5-0-0W01: (Audio-ATK) Changed the specifications for converting stream sound binary files.

Previously, loop information was disabled if both region information and loop information was set in the stream sound waveform file. The specification has been changed such that now both the region information and the loop information are output.

Previously, if you wanted to read the region information with nn::atk::SoundArchivePlayer::ReadStreamSoundRegionDataInfo() from a stream sound being looped without using the stream jump feature, you needed to prepare two waveforms: one with only the loop information, and one with only the region information. With this change, you can do this using a single waveform.

However, like before, it is not possible to use both the stream jump feature and looped playback at the same time.
Due to this change, if your code includes any uses of the stream jump feature for waveforms including loop information, the library will stop on an assertion failure during playback.
More specifically, the library stops on an assertion failure if you specify a region callback when playing a stream sound including loop information.
