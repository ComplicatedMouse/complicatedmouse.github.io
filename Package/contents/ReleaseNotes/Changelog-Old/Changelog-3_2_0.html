Details

Changes in NintendoSDK NX Addon 3.2.0 (based on Generic 3.2.0)

Specification change that breaks your code

*3-2-0S01: (AUDIO) Added an I3dl2Reverb effect utility for use with nn::audio::AuxType.

Added the nn::audio::AuxI3dl2ReverbType utility, which makes it possible to apply the same effect process as nn::audio::I3dl2ReverbType on buffers obtained with nn::audio::AuxType.

For more information on usage, see the AudioEffect sample.

Also added the nn::audio::I3dl2ReverbParameterSet structure for use with the existing nn::audio::I3dl2ReverbType class.
This structure makes it possible to batch-update the settings for I3dl2ReverbType objects.
Note that due to this addition, the parameters of the nn::audio::LoadI3dl2ReverbPreset() function are slightly different.

*3-2-0S02: (FS) Added a size restriction violation check for extended save data.

Added a restriction on the size that can be specified when extending save data by installing a patch or calling nn::fs::ExtendSaveData(). The program will now stop if this restriction is violated. For more information, see FS Library / Manual / Feature Descriptions / Save Data / Save Data (NX Add-On).

Added Features

*3-2-0S03: (AUDIO) Added functions for getting audio output device and audio input device event notifications.

Added the nn::audio::AcquireAudioDeviceNotificationForOutput() and nn::audio::AcquireAudioDeviceNotificationForInput() functions for getting audio output device and audio input device event notifications.

Because of this addition, nn::audio::AcquireAudioDeviceSwitchNotification() is now deprecated.
For more information, see the API Reference.

*3-2-0S04: (GFX) Added HlslCrossCompiler.

Added HlslCrossCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe) for converting HLSL data to GLSL data. For more information, see Graphics / Tools / HlslCrossCompiler.

*3-2-0S05: (TOOL) Separated the graphics resource file converters into the new NintendoSDK GraphicsConvertersKit package.

Moved the graphics resource file converters and the Photoshop plug-in, from the standard NintendoSDK package into the new NintendoSDK GraphicsConvertersKit package.
When creating application patches, use the same version of the GraphicsConvertersKit package corresponding to the tools used for the initial version of the application. This is to minimize the changes in the resource files and reduce patch size.

3-2-0S06: (APPLET) Added a feature for setting a rotation direction for screenshot captures.

Added the following function for setting a rotation direction to apply to screenshots, captured by short-pressing the Capture Button.

  • nn::oe::SetScreenShotImageOrientation()


3-2-0S07: (APPLET) Added a function for accessing system albums from applications.

Added the ALBUM library, which provides features for accessing system albums. More specifically, this library provides features for saving screenshots of images generated by the application and for posting such screenshots to social networking services from the application.

3-2-0S08: (APPLET) Added a function for getting launch parameters passed to applications.

Added the nn::oe::TryPopLaunchParameter() function. This function is intended for use in applications and makes it possible to get the launch parameters passed from the caller when an application is started. For more information about the types of situations in which the system features (such as the HOME Menu) pass launch parameters, see the system feature specifications.

3-2-0S09: (AUDIO) Added functions for converting sampling rates.

Added several sampling rate conversion functions for use with nn::audio::AudioOut. For more information, see the following function references. All internal processes for these functions are executed by the CPU.

  • nn::audio::GetRequiredBufferSizeForResampler()
  • nn::audio::GetResampledOutBufferSize()
  • nn::audio::InitializeResampler()
  • nn::audio::ProcessResamplerBuffer()


3-2-0S10: (DevMenu) Added a feature for calibrating the accelerometer and the gyroscope.

Added a feature to the Controller page for calibrating the accelerometer and the gyroscope.
For more information, see the DevMenu Manual.

3-2-0S11: (DevMenu) Added a feature for toggling the exhibition mode setting.

Added a feature for toggling the exhibition mode setting on the Debug page (development support features) in DevMenu.
For more information, see the DevMenu Manual.

3-2-0S12: (DevMenu) Added features for toggling the Operation Mode and Performance Mode to DevMenu.

Among the development support features in DevMenu, added features for toggling the following settings.

  • Operation Mode Policy
  • Performance Mode Policy
  • CPU Overclock


For details about usage and other important information for these features, see the DevMenu Manual.

3-2-0S13: (DevMenu) Added several application management features.

Added the following features for managing content (applications, downloadable content, and patches) by application ID.

  • Delete all content associated with a given application ID.
  • Delete single sets of content.
  • Display ROM ID hash on a per-content set basis.
  • Reset required version.


For more information, see the DevMenu Manual.

3-2-0S14: (ERR) Added a function for displaying a message prompting the user to update downloadable content.

Added the nn::err::ShowUnacceptableAddOnContentVersionError function, which displays a message prompting the user to update downloadable content.

3-2-0S15: (FS) Added new check targets in FsAccessLogChecker version 8.0.

In FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 8.0, access logs are now checked for write operations from the following functions.

  • nn::fs::ExtendSaveData()
  • nn::album::SaveScreenshot()
  • nn::album::SaveAndShareScreenshot()

For more information, see FS Library / Debugging Features / Checking Access Frequency.

3-2-0S16: (GFX) Added the separate texture and separate sampler modes as shader interface query types.

Added the nn::gfx::ShaderInterfaceType_SeparateTexture and nn::gfx::ShaderInterfaceType_SeparateSampler modes, which can be specified for the nn::gfx::ShaderInterfaceType parameter in nn::gfx::Shader::GetInterfaceSlot.
As part of this change, added the nngfxToolShaderCompilerSeparateTexture and nngfxToolShaderCompilerSeparateSampler elements to the nngfxToolShaderCompilerShaderReflection reflection information in shader compiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderCompiler.dll).

Also, specifying the --reflection-full option in the EXE version of Shader Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe) or specifying a value other than 0 for nngfxToolShaderConverterConvertArg::enableFullReflection in the DLL version of Shader Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.dll) makes it possible to get this reflection information at runtime by calling the nn::gfx::ResShaderProgram::GetShaderCompilerReflection() function.


3-2-0S17: (HID) Added a feature for determining whether a keyboard is connected.

You can now determine whether a keyboard is connected by checking the value of the nn::hid::KeyboardAttribute::IsConnected bit flag in the attributes member variable of the nn::hid::KeyboardState structure. For more information, see the manual.

3-2-0S18: (HID) Added a feature for determining whether a mouse is connected.

You can now determine whether a mouse is connected by checking the value of the nn::hid::MouseAttribute::IsConnected bit flag in the attributes member variable of the nn::hid::MouseState structure. For more information, see the manual.

3-2-0S19: (HID) Added a function for getting whether vibration is enabled in system settings.

Added a function for getting whether vibration is enabled in system settings, for use in applications that need to get the current vibration setting.

When this setting is disabled, vibration is disabled on all controllers connected to the system. Also, this setting can be changed in DevMenu > Controller > Enable Vibration.

3-2-0S20: (TOOL) Added a filtering feature for use when creating downloadable content.

You can now specify data directory filters, on a per-item basis, when creating NSP files for downloadable content in AuthoringTool.
For more information, see Features / System / AOC Library / Creating Downloadable Content.

Improvements

*3-2-0S21: (FRIENDS) Deprecated the functions for setting/getting application-specific information in the friend presence library.

Deprecated the following functions to prevent high server loads.

  • nn::friends::UserPresence::SetAppValue
  • nn::friends::FriendPresence::GetAppValue


3-2-0S22: (AOC) Changed the ReleaseVersion of the downloadable content included in the AocSimple sample.

Changed the ReleaseVersion value of the downloadable content included in the AocSimple sample, from 1 to 0.

3-2-0S23: (BCAT) Added additional information about background synchronization.

Added additional information to the Initialize function reference about background synchronization.

3-2-0S24: (DevMenu) Added support for displaying an error message when applications crash.

When the current running application crashes, an error message indicating the reason for the crash is now displayed.

When the Terminate Application on GPU Error setting is enabled in the development support features, the following message is now displayed when an application crashes on a GPU error.

  • Application has been abnormally exited.


3-2-0S25: (FRIENDS) Improved the presence state update process.

When the application updates the presence state, it no longer sends any data to the server if the state has not changed.

3-2-0S26: (FS) Added support for outputting access operations from additional functions to access logs.

Access operations from the following functions are now output to access logs when FS Access Log is enabled.

  • nn::fs::SetAllocator()
  • nn::fs::SetSaveDataRootPath()
  • nn::fs::QueryMountRomCacheSize()
  • nn::fs::QueryMountAddOnContentCacheSize()
  • nn::fs::EnsureSaveData()
  • nn::fs::ExtendSaveData()
  • nn::fs::GetEntryType()
  • nn::album::SaveScreenshot()
  • nn::album::SaveAndShareScreenshot()

Among these functions, the nn::fs::ExtendSaveData(), nn::album::SaveScreenshot(), and nn::album::SaveAndShareScreenshot() functions are now checked for write operation access frequency. For more information, see FS Library / Debugging Features / Checking Access Frequency.

3-2-0S27: (FS) Improved the fs access log library and added support for this improvement in FsAccessLogChecker version 7.1.

Added support for outputting values, specified for the mode parameter of the nn::fs::OpenFile() function in the fs access log library, to access logs. Added support for the corresponding log format to FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 7.1.

3-2-0S28: (FS) Improved the patch creation algorithm.

To improve read speed, improved the algorithm used for the second and subsequent times a patch is created.

3-2-0S29: (FS) The nn::fs::ReadDirectory() function no longer stores invalid entries in the buffer.

Child entries obtained by the nn::fs::ReadDirectory() function are no longer stored in entryBuffer if their names include characters that cannot be used.

3-2-0S30: (GFX) The NVN version of the GFX library no longer requires an OpenGL context when using devices in interoperation mode.

Previously, the nn::gfx::InteroperationNvn::ConvertToGfxDevice function required the OpenGL context to be set to the current context when called in the Windows environment. This is no longer required.

3-2-0S31: (GFX) The NVN version of the GFX library now allows NULL to be specified for CPU pointers when using memory pools for interoperation purposes.

You can now specify NULL for CPU memory when creating memory pools using nn::gfx::InteroperationNvn::ConvertToGfxMemoryPool.
However, a value other than NULL must be specified for memory pools used for the following purposes.

  • Memory pools used by buffer descriptor pools.
  • Memory pools that do not have the NVN_MEMORY_POOL_FLAGS_CPU_CACHED attribute and are used when using textures tiled for the development hardware on PC.


3-2-0S32: (Graphics) Updated the Maxwell instruction set architecture documentation.

Updated the Maxwell instruction set architecture documentation. Added several additional notes about accessing resources, such as constant buffers and vertex attributes, to the bottom of the top page of the documentation.
For more information, see the Maxwell Instruction Set Architecture section in NVN Library / Manual.

3-2-0S33: (HID) When using Joy-Con controllers in the Windows environment, a button now has to be pressed, after starting an application, for the controllers to be recognized by the application.

Previously, Joy-Con controllers connected via Bluetooth were automatically recognized by applications in the Windows environment.
Starting in NX Add-On 3.2.0, connected Joy-Con controllers are no longer automatically recognized after an application is started. Instead, they are recognized in the order in which a button is pressed on the controllers.

3-2-0S34: (LDN) Changed the link level threshold values for local communications.

Fixed a bug where low link levels were returned, even when the communication quality was good enough that the game should be playable without any issues. This change ensures that appropriate link levels can now be obtained.

3-2-0S35: (NGC) Accelerated the nn::ngc::ProfanityFilter::MaskProfanityWordsInText() function.

Accelerated the nn::ngc::ProfanityFilter::MaskProfanityWordsInText() function.
The resulting speed improvement depends on the text being processed, but in most cases, the function should run approximately 5 to 30 times faster than previously.

3-2-0S36: (NIFM) Changed the link level threshold values for Internet communications.

Fixed a bug where low link levels were returned, even when the communication quality was acceptable enough that the game should be playable without any issues. This change ensures that appropriate link levels can now be obtained.

3-2-0S37: (OS) Improved the OsMemoryHeap sample.

Simplified the OsMemoryHeap sample to address several issues where the content or implementation, was difficult to understand.

3-2-0S38: (PCTL) Changed the return value of the nn::pctl::CheckFreeCommunicationPermission function.

Previously, when the nn::pctl::CheckFreeCommunicationPermission function was called while a message was displayed, it returned the permission state from when the message had been displayed. Now, this function rechecks the internal state after the message is displayed, and returns the permission state from the point in time at which the message was closed. This makes it possible to proceed to scenes that require the permission, without having to implement any special handling in the application when the application transitions from a state that does require the permission to a state that does not require the permission while a message is displayed, for example.

This change only applies to applications using this version of the SDK or later. It will not affect applications built using earlier versions of the SDK.

3-2-0S39: (Sample) Added support for handheld mode (NpadId::Handheld) in nns::hid.

Added support for NpadId::Handheld in nns::hid.
When using a GamePad in nns::hid, the GamePad with an Id of 0 can now be used with the controller assigned to NpadId::Handheld or NpadId::No1.

3-2-0S40: (TOOL) AuthoringEditor now displays a notification when languages selected as supported languages are not included in the title information.

When specifying the supported languages for an application in AuthoringEditor, a notification is now displayed if a selected language is not included in the title information.
This notification is only intended to be reference information from the tool. It does not constitute an input error.

3-2-0S41: (TOOL) In Texture Converter, when the input file is a DDS file with two RG components, rg01 is now selected as the default component.

In Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe), when the input file is an unsigned DDS file with two RG components, rg01 is now selected as the default component.

3-2-0S42: (TOOL) Texture Converter now supports output of all DDS file formats.

In Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe), you can now specify the --format option to output the data in all of the DDS file formats.
Also, specifying the --no-encoding option when converting intermediate texture files to DDS files, now also results in encoded data being output to the DDS files.

3-2-0S43: (TOOL) The system now prohibits applications authored for older versions of the firmware from being started.

The system now prohibits applications authored with AuthoringTool, from being started if the Firmware Version setting is for a version older than 3.0.

Updated the description of the RequiredWithNetworkServiceAccountAvailable element, in the StartupUserAccount setting for application NMETA files, in AuthoringEditor.
More specifically, deleted the “The application does not need to handle the process of what to do when there is no associated Nintendo Account” text.

3-2-0S45: (TOOL) When an application NSP file is opened, AuthoringEditor now displays whether there are any items that will cause an error during master ROM submission.

When an application NSP file is opened, AuthoringEditor now displays a list of any items that will cause an error during master ROM submission.
The content of the application's NSP file can still be modified regardless of any errors that are displayed.

3-2-0S46: (VSI) Standardized installation behavior in NDI and NDICmd.

Previously, NDI and NDICmd exhibited different behavior when installing VSI for NX.

  • NDI: Started the VSI for NX installer and installed VSI for NX on the system.
  • NDICmd: Did not install VSI for NX on the system.


This was because NDICmd was originally intended to make it possible to build the development environment without having to close any currently running instances of Visual Studio that had been started for application development, while NDI exhibited different behavior than NDICmd, which made it difficult to determine the correct installation behavior.




Therefore, standardized the installation behavior so that both tools now start the VSI for NX installer and install VSI for NX on the system.

In the account library API reference and documentation, updated the information about error handling.
Added information about errors that occur during SSL communication.

Bug Fixes

3-2-0S48: (BuildSystem) Fixed a bug that caused a build error to occur if nlib was enabled when a NintendoSDK application project was created.

Fixed a bug where if nlib was enabled when creating a NintendoSDK application project, a linking error occurred when building the application for NX32 or NX64.

Fixed a bug where when installing applications to an SD card from DevMenu/DevMenuCommand, the installation speed was approximately 0.6 times that previously achieved in NintendoSDK Add-On 3.0.0.

Fixed a bug where, when an application was launched, if there was insufficient free space in memory to create save data, the resulting error dialog box sometimes remained displayed indefinitely and could not be closed.

3-2-0S51: (Display) Fixed a bug where on some televisions, when the system entered sleep mode while docked, the television would, by itself, sometimes switch input channels to the NX.

There was a bug where, depending on the type of TV and splitter the NX was connected to, the HDMI signal output from the dock was automatically recognized and the TV automatically switch the input channel.
Note: This feature is specific to the TV and splitter and is not related to the CEC standards. It may occur even if you disable CEC.

This bug was caused by the fact that the NX is designed to wake up intermittently to run certain background system processes, even when in sleep mode, and would output an HDMI signal during these intermittent periods.
Fixed this bug by modifying system behavior so that no HDMI signal is output until the system is fully woken up from sleep mode.

3-2-0S52: (Display) Improved HDMI connectivity.

Improved an issue where in rare cases, nothing was displayed on the TV when the system was connected via HDMI.

Specification change that breaks your code

*3-2-0W01: (LayoutTool) Fixed a bug where gamma conversion did not work correctly for font shadow colors and per-character transformation colors when the --degamma-parameter option was specified.

Fixed a bug in LayoutConverter where gamma conversion did not work correctly for font shadow colors and per-character transformation colors when the degamma-parameter option was specified. Note that as a result of this bug fix, any adjustments made to layout data to make it display correctly, may now have the opposite effect and cause layout data to be displayed incorrectly. (The degamma-parameter option is specified in order to use ported products, and other past layout data that was created on the basis of a SRGB gamma workflow, as-is.)

*3-2-0W02: (LayoutTool) Picture pane shape settings are now only output to the intermediate file when a value other than rectangle is specified for the shape setting.

Previously, all shape-related parameters were output to the intermediate file regardless of the picture panes’ shape settings. Now, the tags related to these parameters are only output when a value other than rectangle is specified as the shape setting.
Due to this change, if you save an intermediate file while picture panes are set to have the rectangle shape, the previously saved parameters for shape settings, other than rectangle, will be reset.
