Details

Changes in NintendoSDK NX Addon 6.5.1 (based on Generic 6.5.1)

Added Features

6-5-1S01: (TOOL) Added support for checking the integrity of NSP file data from AuthoringEditor.

An NSP file data integrity check is automatically run when opening NSP files (applications, patches, and downloadable content).
The check is performed by the checkintegrity subcommand in AuthoringTool. (The --simple option is not applied.)
If the integrity check fails, AuthoringEditor notifies the user.

Improvements

6-5-1S02: (TOOL) AuthoringEditor can now display master ROM submission errors grouped by importance.

You can now choose to display master ROM submission errors grouped by their level of importance when you open the NSP file in AuthoringEditor.

6-5-1S03: (TOOL) Made improvements so that ROM information can only be verified from the patch NSP in AuthoringEditor.

You can now verify the ROM identification hash and size of individual patch NSP files.
The original application NSP is only requested when verifying the file configuration within the program.

6-5-1S04: (TOOL) When you run AuthoringTool, the message that is displayed for a function that is not required in the production ROM is now tagged with Note instead of Warning.

Previously, when you ran AuthoringTool with the --error-unpublishable option specified, a Warning message would be output for any function you were using that was not required in the production ROM. That output has been changed to a Note message.
Uploading the ROM to the NDP does not generate an error if this message has been output, but use of the listed function in the retail product is not recommended, so it would be best to fix the issue.
