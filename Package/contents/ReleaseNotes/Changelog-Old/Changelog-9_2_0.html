*9-2-0S01: (HID) Fixed a bug with the flag indicating the Npad cable connection status.

Fixed a bug with the flag indicating the Npad cable connection status.

You can use nn::hid::GetNpadState(NpadHandheldState *<em>pOutValue</em>, const NpadIdType &<em>id</em>) to get the nn::hid::NpadHandheldState structure, which indicates the controller input status. It was discovered that the IsWired flag detection in the attributes (nn::hid::NpadAttribut) indicating the Npad status was incorrect.

IsWired indicates that the Npad is connected via a cable or other equipment. For nn::hid::NpadStyleHandheld, it indicates whether an AC adapter is connected. In earlier SDK versions, a bug in this detection returned true regardless of the cable connection status. To maintain compatibility, this flag has been changed to always return true.

9-2-0S30: (HID) Made improvements to the nn::hid::VibrationPlayer function names.

nn::hid::VibrationPlayer is a vibration node that provides the function to cause vibrations by referencing the vibration binary data extracted to application memory. The application must extract the binary data of the vibration file to play into its own memory in advance and not overwrite it until playback is complete.

However, as a result of this functionality, the Load(), Unload() and IsLoaded() functions in nn::hid::VibrationPlayer gave developers the impression that vibration file data is copied to the instance, leading to bugs. For this reason, the SetVibrationFileAddress(), UnsetVibrationFileAddress(), and IsPlayable() functions have been added to explicitly indicate that they are referencing the vibration file. You can continue to use the old functions, but use the new functions when possible.

For more information, see the nn::hid::VibrationPlayer API documentation.
