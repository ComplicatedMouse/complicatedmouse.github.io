Details

Changes in NintendoSDK NX Addon 1.7.0 (based on Generic 1.7.0)

Specification change that breaks your code

*1-7-0S01: (FS) Deleted nn::fs::MountSaveData(const char* name) for mounting save data for debugging.

Deleted nn::fs::MountSaveData(const char* name) for mounting save data for debugging.
Use the nn::fs::MountSaveDataForDebug() function to mount save data for debugging.

*1-7-0S02: (General) Removed SDEV 1.8 and preMP from the list of hardware that can be initialized using InitializeSdev.

The SDEV 1.8 and preMP can no longer be initialized using InitializeSdev.
Also, note that although these items can still be selected in the UI as the hardware to initialize, selecting this option will not initialize the hardware.

Improvements

*1-7-0S03: (TOOL) Increased the amount of memory available to the application within the game card

Increased the available capacity in the game card from 128 MB to 144 MB, if the allocated memory for the application exceeds the guaranteed region.

1-7-0S04: (TOOL) Deleted settings for using the touch screen for NMETA files

Deleted the settings for specifying the use of the touch screen for NMETA files.
A policy has been determined to no longer use these settings for the system features, so the application NMETA settings are no longer needed.


Bug Fixes

1-7-0S05: (TOOL) Fixed a bug where a different EDEV were initialized using InitializeEdev when multiple EDEV consoles were connected

Fixed a bug where a different EDEV would be initialized using InitializeEdev when multiple EDEV consoles were connected.

1-7-0S06: (TOOL) Fixed a bug where you were able to edit an NSP file built with an SDK of a different major version on AuthoringEditor

It was possible to edit an application NSP file built with a different major version of the SDK containing AuthoringEditor that you used for opening and editing the NSP file. This bug has been fixed.
An NSP file with an SDK of a different major version is always read-only when opened.

1-7-0S07: (TOOL) Fixed an issue where there were insufficient rating classifications for PEGIPortugal in the rating input form in AuthoringEditor

There were not enough rating classifications for PEGIPortugal in the AuthoringEditor rating input form, so additional classifications were added.
The added ratings were ages 3 and 7, options for the PEGIPortugal rating classifications.
