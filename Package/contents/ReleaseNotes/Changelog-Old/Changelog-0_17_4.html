Details

Changes in NintendoSDK NX Addon 0.12.4 (based on Generic 0.17.4)

Specification change that breaks your code

*13-5S01: (FS) Deleted the functions that were scheduled to be deleted.

Deleted the following functions that had been scheduled to be deleted.

  • Result nn::fs::MountRom(const char* name) NN_NOEXCEPT;
  • void nn::fs::SetRomRoot(const char* rootPath) NN_NOEXCEPT;


*13-5S02: (HID) Added support for mixed use of XInput and DirectInput for connected controllers.

Previously, XInput controllers were treated as being DirectInput controllers. XInput controllers are now handled using the XInput API in order to improve system stability. Due to this change, the nn::settings::DebugPadGenericPadMap structure and the nn::settings::BasicXpadGenericPadMap structure are now interpreted differently.
For more information, see the HID emulation settings documentation (%NINTENDO_SDK_ROOT%\Documents/Package/contents/Pages/Page_89983199.html).

Added Features

*13-5S03: (BuildSystem) Added a feature to AuthoringTool for checking the NRO information registered in an NRR file, when creating NSP files.

Added a feature to AuthoringTool for checking whether the NRO files registered in an NRR file, are present in the directory specified using the --nro option, when creating NSP files.
To use the NRO files in the ro library, the directory in which the NRO files are located must be specified using the --nro option in AuthoringTool.
You can skip this procedure if you do not require the ro library feature.

*13-5S04: (NVN) Added NVN_DEVICE_FLAG_DEFERRED_FIRMWARE_MEMORY_RECLAIM_BIT.

Previously, the region of firmware memory passed to the nv::InitializeGraphics() function for the nvnMemoryPoolInitialize() function to use for GPU paging tables, was not freed until the nvnDeviceFinalize() function was called. Now, only the GPU paging tables that are not shared by other memory pools are freed when the nvnMemoryPoolFinalize() function is called. However, you can return to the previous behavior by passing NVN_DEVICE_FLAG_DEFERRED_FIRMWARE_MEMORY_RECLAIM_BIT to the nvnDeviceBuilderSetFlags() function. This feature will be removed in a future release and should only be used if your support contact at Nintendo instructs you to do so.

13-5S05: (DevMenu) Added a feature for batch-installing all of the NSP files on a microSD Card at one time.

Added a feature for batch-installing all of the NSP files on a microSD Card at one time.
To batch install the NSP files, go to the Application page and then select Install from SD → Install All.

13-5S06: (FS) Added the nn::fs::IsSaveDataExisting() function.

Added the nn::fs::IsSaveDataExisting() function as a method to check whether any user account save data exists.

13-5S07: (HID) Added a sample demo that makes it possible to switch between several Npad features.

Added the HidNpadIntegrate sample, which allows the user to switch between using the following Npad features.

  • Display pad input state.
  • Display IR sensor state.
  • Control vibration device.
  • Display six-axis sensor state.


13-5S08: (TOOL) Added a command to ControlTarget for emulating battery levels.

Added a command to ControlTarget for emulating battery levels.
When production systems and EDEV systems are docked and the battery level is low, the operation mode switches to TV mode and the performance mode switches to normal mode.
These states can now be dynamically emulated on the SDEV.
For more information, see Controlling the Development Hardware / ControlTarget.

13-5S09: (TOOL) Added a command to DevMenuCommand for enabling/disabling the high battery consumption performance configurations.

Added a command to DevMenuCommand (%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nca) for enabling/disabling the high battery consumption performance configurations.

  • For more information about using this feature, see Controlling the Development Hardware / DevMenuCommand.
  • For more information about the high battery consumption performance configurations, see OE Library / Manual / Features / Selecting Performance Configurations.


13-5S10: (TOOL) Added AuthoringEditor, which is a GUI tool for editing META files.

Added AuthoringEditor, which is a GUI tool for creating and editing META files.
META files created using this tool can be used to generate NSP and NSPD files.
The tool is located in the %NINTENDO_SDK_ROOT%\Tools\AuthoringEditor\AuthoringEditor.exe directory.
For more information, see Tools Guide / AuthoringEditor.

Improvements

*13-5S11: (HID) The keyboard state can now be retrieved using the <code><code>nn::hid</code></code> function in the production environment as well.

The following functions were originally provided for debugging purposes only, but can now be used to get the state of USB keyboards in the production environment as well.

  • nn::hid::InitializeKeyboard()
  • nn::hid::GetKeyboardState()
  • nn::hid::GetKeyboardStates()

The nn::swkbd function is recommended when getting text input, therefore, also check the nn::swkbd API documentation.

13-5S12: (AUDIO) Fixed portions of the API reference related to system event postconditions.

Previously, the function references for functions such as nn::audio::OpenAudioRenderer, that wait for an nn::os::SystemEvent object as a parameter, reported that the system event returns to an uninitialized state when the audio renderer is closed. Removed this information from the API reference because the system event does not return to an uninitialized state when the audio renderer is closed.
Also updated the audio, codec, and spy sample demos so that the system event used when closing the samples is explicitly destroyed.

This change has been applied to the API reference for the following functions.

  • nn::audio::OpenAudioRenderer
  • nn::audio::OpenDefaultAudioIn
  • nn::audio::OpenAudioIn
  • nn::audio::OpenDefaultAudioOut
  • nn::audio::OpenAudioOut


13-5S13: (DIAG) Added support for opening files in Log Viewer by dragging them in.

You can now open NXBINLOG log files by dragging them onto the Log Viewer icon or by dragging them into the Log Viewer window.

13-5S14: (FRIENDS) Added a process for verifying the validity of language codes for in-application display names.

Functions that take an in-application display name as a parameter (such as nn::friends::AddPlayHistory) now check the validity of language codes for the in-application display name when executed.

13-5S15: (NIFM) Modified publication status of Result values.

Among the Result values related to network connection management, only those that actually need to be handled when an application is running are now public.

13-5S16: (TIME) Fixed the value of nn::time::InputPosixTimeMin.

Changed the value of nn::time::InputPosixTimeMin from 2000/1/1 00:00:00 (UTC) to 1999/12/31 00:00:00 (UTC).
This fix was required because depending on the system time zone setting, it is possible to set a time older than nn::time::InputPosixTimeMin (the minimum value for which the nn::time::ToCalendarTime() function is guaranteed to work), and the nn::time::StandardUserSystemClock::GetCurrentTime() function was returning time values that did not satisfy the nn::time::InputPosixTimeMin condition.

13-5S17: (TOOL) Added support of log output for exceptions that occur in features other than the development hardware initialization feature in the GUI versions of the setup tools.

Exceptions that occur in features other than the development hardware initialization feature of the GUI versions of the setup tools (InitializeSdevWin and InitializeEdevWin), can now be viewed as logs.

The Microsoft DebugView program is required to view these logs.

Bug Fixes

13-5S18: (BuildSystem) Fixed a bug where the NintendoSdkVsExtension package would sometimes fail to install.

Fixed a bug where the NintendoSdkVsExtension package would fail to install in the following situations.

  • If Visual Studio had not been installed.
  • If Visual Studio 2015 had not been installed, but Visual Studio 2013 was installed and the NintendoSdkVsExtension had already been installed at an earlier point.


Also improved the installation process so that even when the installation itself fails, the NintendoSdkVsExtension installer is placed in the NintendoSDK/Installers/NintendoSdkVsExtension directory.

13-5S19: (HID) Fixed a bug where gamepad assignments in Input Director were reset when a gamepad was connected or disconnected.

Fixed a bug where gamepad assignments in Input Director were reset when a gamepad was connected or disconnected.

13-5S20: (HID) Fixed a bug where gamepad names were not displayed in gamepad settings in Input Director.

Fixed a bug where gamepad names were not displayed in gamepad settings in Input Director.

13-5S21: (HID) Fixed a bug where simultaneous presses of the ZL and ZR Buttons did not work when using XInput supported controllers.

Fixed a bug where simultaneous presses of the ZL and ZR Buttons did not work when using XInput supported controllers.

13-5S22: (HID) Fixed a bug where XInput controllers were not recognized when multiple controllers were connected.

Fixed a bug where XInput supported controllers were not recognized when multiple controllers were connected.

13-5S23: (TOOL) Fixed a bug where InitializeSdev would fail to run when network latency was large.

Fixed a bug where InitializeSdev would more often return the “[ERROR] Update HostBridge firmware failed.” error when network latency was large.

13-5S24: (TOOL) Fixed a bug where the GUI versions of the setup tools would throw an exception when closed.

Fixed a bug where the GUI versions of the setup tools (InitializeSdevWin and InitializeEdevWin) would occasionally throw an exception when closed.
This exception had no effect on the setup tools themselves, even when it occurred.

13-5S25: (TOOL) Fixed an AuthoringTool bug that occurred when the number of resource data files was extremely large.

Fixed a bug where creation of NSP/NCA files would fail, or take an especially long time to complete, if the number of resource data files and directories was unusually large.

13-5S26: (VSI) Fixed a bug where a large number of IntelliSense errors would occur.

Fixed a bug where IntelliSense did not work correctly for the NX32 or NX64 platforms. This occurred when certain SDK header files, such as util_Vector.h, were included.

13-5S27: (VSI) Updated NX-VSI and Target Manager to version 0.14.13.

Updated NX-VSI and Target Manager to fix the following issues.

  • Improved the instability that occurred when multiple devices were connected.
  • Fixed a bug where when symbols included in an NSS file were overwritten with a .o file, those symbols would be resolved in an unexpected way.


Changes in NintendoWare NX Addon 0.12.4 (based on Generic 0.17.4)

Specification change that breaks your code

*13-5W01: (Layout-UI2D) Changed the behavior of LayoutConverter when a 2-channel format such as BC5 is specified for a TGA file that does not include alpha information.

Changed the behavior of LayoutConverter when a 2-channel format such as BC5 is specified for a TGA file that does not include alpha information. Previously, a value of 255 was used for the alpha values. The same values as in the color information are now used for the alpha values. This behavior is the same as the behavior for legacy platforms and will make it easier to migrate data to the new platform. For more information, see the sample code for the new utility tool that lists files that will be affected by this change.

Added Features

*13-5W02: (Effect-VFX) Added a feature for restricting the number of texture mipmaps that can be used.

Added a feature for limiting the number of texture mipmaps that can be used. Selecting the Limit Mipmaps check box in the Textures section of the Emitter tab in EffectMaker, limits the number of mipmap textures that can be used to 2. Due to the addition of this feature, the effect binary version has been updated.

13-5W03: (3DTool) Added a feature for specifying the options used when generating optimized shaders in 3DEditor.

Added a feature for specifying the <ShaderConverterAdditionalArgsForOptimizedShader> option in the 3DEditor team settings file or platform settings file. Shader Converter uses the value specified for this option when generating optimized shaders. For more information, see Team Settings File Format.

13-5W04: (Layout-UI2D) The Project on Entire Pane option can now be specified for objects other than window panes.

The Project on Entire Pane option can now be specified for objects other than window panes. Previously, this option could only be used with window panes.

13-5W05: (Effect-VFX) Added the EmitterSet::IsAnyEmitterFadeRequest() function, which detects whether any of the Emitter objects in an EmitterSet are being faded.

Individual emitters can be faded by using the Emitter::Fade() function in a custom callback function. Therefore, added the EmitterSet::IsAnyEmitterFadeRequest() function, which detects whether any of the Emitter objects that belong to an EmitterSet are being faded.
Note, however, that the check for whether any emitters are currently being faded is performed as the last calculation for the EmitterSet, therefore, this function cannot detect anything for the frame in which the Emitter::Fade() function is called.

Improvements

13-5W06: (3D-G3D) Improved the nn::g3d::viewer::ViewerServer::QueueAttachModel feature.

Multiple nn::g3d::ModelObj objects that were created from the same nn::g3d::ResModel object can now be attached using nn::g3d::viewer::ViewerServer::QueueAttachModel.
Any edits made in 3DEditor are reflected in all of the attached nn::g3d::ModelObj objects.

13-5W07: (Layout-UI2D) LayoutConverter now displays more detailed error messages when it cannot find texture files.

LayoutConverter now displays more detailed error messages when it cannot find texture files. More specifically, when textures that reference materials or textures that reference animation segment tags cannot be found from the layout texture list, an error message that includes the names of the texture and the layout file is now displayed.

Bug Fixes

13-5W08: (Audio-ATK) Fixed a bug where the Fix Priority at Release feature could not be disabled.

Fixed a bug in SoundMaker where even if the Fix Priority at Release option was disabled, the priority would not be set to the minimum value when released.

13-5W09: (Audio-ATK) Fixed a bug where the processes implemented in the functions for attaching/detaching memory pools were not locked to be mutually exclusive, relative to renderer update processes.

Fixed a bug where the processes implemented in the nn::atk::SoundSystem::AttachMemoryPool and nn::atk::SoundSystem::DetachMemoryPool functions were not locked to be mutually exclusive, relative to the process used by the atk sound system to update audio renderers.

13-5W10: (Audio-ATK) Fixed a bug where the SoundSystem finalization process would cause the system to crash.

Fixed a bug where when attempting to destroy a memory pool before that memory pool was detached, using the nn::atk::SoundSystem::Finalize function, would sometimes cause the system to crash.

13-5W11: (FONT) Fixed a bug with the calculation of proportional metric information for line breaks in text, rendered using OpenType fonts.

Fixed a bug with calculation of proportional metric information for line breaks in text, rendered using OpenType fonts.

Previously, when multiple lines of text were rendered, the width of lines that included line breaks, would be different than the width of the final line of text. This has been fixed.

13-5W12: (FONT) Fixed an invalid memory deallocation process in the nn::font::TextureCache::Finalize() function.

Fixed an invalid memory deallocation process in the nn::font::TextureCache::Finalize() function. Previously, the memory managed by a memory pool was freed before the memory pool finalization process was complete. This could cause issues if the system attempted to use that freed memory again, before the finalization process was actually complete.

13-5W13: (Layout-UI2D) Fixed a bug in LayoutEditor where when overwriting a parts pane, depending on the data being used, the UI would not be initialized correctly and the tool would crash on a NULL reference if only the color interpolation information was overwritten.

Fixed a bug in LayoutEditor where when overwriting a parts pane, depending on the data being used, the UI would not be initialized correctly and the tool would crash on a NULL reference if only the color interpolation information was overwritten.

13-5W14: (Layout-UI2D) Fixed a bug where hidden panes would occasionally be displayed when the nn::ui2d::DrawInfo::SetInvisiblePaneCalculateMtx() function was called.

Fixed a bug where hidden panes would occasionally be displayed when the nn::ui2d::DrawInfo::SetInvisiblePaneCalculateMtx() function was called.

13-5W15: (LayoutTool) Fixed a bug in LayoutEditor where the snap to grid feature was not enabled.

Fixed a bug in LayoutEditor where the snap to grid feature in the View menu was not enabled.

13-5W16: (Effect-VFX) Fixed a bug in the VFX runtime where applying a Speed Scale Y value to particles with a speed of 0, that were emitted by a GPU+SO emitter, would cause those particles to stop being displayed.

Fixed a bug where applying a Speed Scale Y value to particles with a speed of 0, that were emitted by a GPU+SO emitter, would cause those particles to stop being displayed. For particles with a speed of 0, the correct behavior is that the size should not change.

13-5W17: (Effect-VFX) Fixed a bug where emitter position was not reflected correctly when a frame rate of 0 was passed to the nn::vfx::System::Calculate() function.

Fixed a bug where if a frame rate of 0 was passed to the nn::vfx::System::Calculate() function, the DirtyFlag object maintained internally by EmitterSet objects would get deleted, which would cause emitter positions to stop being reflected correctly when changed using a function such as nn::vfx::EmitterSet::SetPos().

13-5W18: (SPY) Fixed a bug where data collected using Spy.exe could not be saved to a file.

Fixed a bug where data collected using Spy.exe could not be saved to a file because the save button in the Spy.exe menu was always disabled.
