Details

Changes in NintendoSDK NX Addon 0.12.5 (based on Generic 0.17.5)

Improvements

13-6S01: (TOOL) Improved the quality of files encoded in float_bc6 format by the texture converter tool.

Improved the quality of files encoded in float_bc6 format in the GPU by the texture converter tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe).

Bug Fixes

13-6S02: (TOOL) Fixed a bug where the system sometimes would crash when the texture converter tool was converting a large image to a file in ASTC format.

Fixed a bug where the system sometimes would crash when a large image was being encoded in ASTC format in the GPU by the texture converter tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe).

Changes in NintendoWare NX Addon 0.12.5 (based on Generic 0.17.5)

Added Features

13-6W01: (Layout-UI2D) Added options to LayoutArchiver for specifying the texture tiling method.

Added options to LayoutArchiver for specifying the texture tiling method (tile-optimize and tile-size-threshold). The specifications conform to the same options of the TextureConverter tool. There will be differences in size for textures that are small and not a size that is a power of 2. For example, if tile-optimize=size_auto is specified for all textures in the layout file Samples\Sources\Resources\Ui2d\cannonButton\CannonButton_00.flyt, the size will be 96.1 KB when tile-size-threshold=5, compared to 72.1 KB when tile-size-threshold=100. The larger the value set for tile-size-threshold, the more the size is prioritized during conversion.

Bug Fixes

13-6W02: (3D-G3D) Fixed a bug where the nn::g3d::mii::AssignCharModel() function did not calculate the radius of the bounding sphere correctly.

Fixed a bug where the nn::g3d::mii::AssignCharModel() function did not calculate the radius of the bounding sphere correctly. With this fix, the correct value now is entered for the radius held by nn::g3d::ResShapeData::pRadiusArray.

13-6W03: (Effect-VFX) Fixed a bug where particle z-sorting was calculated in local coordinates in the VFX runtime.

Fixed a bug where particle z-sorting was calculated in local coordinates in the VFX runtime, causing effects to look wrong depending on the relationship between the camera and the position. To correct this problem, z-sorting is now calculated in world coordinates

13-6W04: (Effect-VFX) Fixed a bug where the vertex state buffer could be insufficient depending on the structure of the FMDB file.

Fixed a bug where the vertex state buffer could be insufficient depending on the structure of the FMDB file, causing an assertion failure.
