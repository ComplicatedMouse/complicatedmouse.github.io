Details

Changes in NintendoSDK NX Addon 9.0.0 (based on Generic 9.0.0)

Specification change that breaks your code

*9-0-0S01: (Compiler) Updated to Clang for NX 1.8.3.

Updated Clang for NX to version 1.8.3.
The main changes are as follows.

  • Updated to LLVM 8.0.0.
  • llvm-objdump for armv7l now supports the -Mreg-names-raw and Mreg-names-std flags.
  • Fixed a bug where SafeStack was dependent on the alignment of memory returned by calloc.
  • Deleted std::bad_array_length.
  • Fixed a bug where pthread_detach() did not free thread resources.
  • Modified the internal implementation of libc++ to support removal of the ABI.


*9-0-0S02: (libcurl) Updated libcurl to 7.64.1.

Curl has been updated from 7.55.1 to 7.64.1. This update brings many improvements but has the potential to be a breaking change.

One example is CURLE_SSL_CACERT; which was previously part of the CURLcode enum type with its own independent value but in this later version it is a preprocessor macro replaced with the CURLE_PEER_FAILED_VERIFICATION token, which is itself part of the CURLcode enum. Consequently, if you have both in a single C switch statement, your source file will not compile because it is illegal to have repeating values in switch statements.

*9-0-0S03: (UTIL) Changed the type of nn::Span::index_type from ptrdiff_t to size_t.

Changed the type of nn::Span::index_type from ptrdiff_t to size_t. This change may cause new warning messages to occur in locations where the size of nn::Span is used. If this happens, use static_cast to cast to the appropriate type. This change was made to reflect the fact that std::span::index_type is of type size_t in C++20.

9-0-0S11: (FS) Added a function for getting the amount of free space available in file systems.

Added the nn::fs::GetFreeSpaceSize(int64_t* <em>outValue</em>, const char* <em>name</em>)function for getting the amount of free space available in file systems.
In all file systems, metadata and block management occupy a certain amount of space, so it may not necessarily always be possible to create files equal in size to the amount of free space returned by this function.
For file systems that use space allocated in advance (such as save data file systems), the file system should be designed to not depend strictly on the amount of free space available or should be designed to be able to manage increases in the number of data elements.
