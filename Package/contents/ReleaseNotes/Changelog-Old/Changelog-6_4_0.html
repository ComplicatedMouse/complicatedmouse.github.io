Details

Changes in NintendoSDK NX Addon 6.4.0 (based on Generic 6.4.0)

Specification change that breaks your code

*6-4-0S01: (DevMenu) The DevMenu Application from NX Add-On 6.3 and earlier now closes during startup.

In firmware version 6.0.0-4.0 and later, the DevMenu Application that is packaged with NX Add-On 6.3.0 and earlier now closes during startup and can no longer be used.
To use DevMenu Application, install the DevMenuApp.nsp application that is packaged with the version of NX Add-On corresponding to the firmware version used on your development hardware.

Added Features

*6-4-0S02: (BCAT) Added documentation for the delivery by area operators feature.

Added documentation describing how to implement and use the delivery by area operators feature.

<em>Delivery by area operators</em> is one of the mechanisms for BCAT data delivery. This mechanism can be used for collaboration between an application and the stores of the operators of Nintendo Zones and other such public wireless LAN services for the Switch.

*6-4-0S03: (DevMenu) Added a feature that makes it possible to communicate between the target and host over USB 3.0.

DevMenu and DevMenuCommand now support communication between the development hardware and PC over USB 3.0.
This feature can be used in DevMenu from the Debug page, and in DevMenuCommand with the debug commands.
However, this feature is subject to some limitations when wireless communication is also in use. Only enable this feature after you are familiar with the associated considerations. For more information, see Features > Controlling the Development Hardware > DevMenu > Development Support Feature Settings and Features > Controlling the Development Hardware > DevMenuCommand.
Depending on the environment, enabling this feature can increase maximum communication speed by as much as 3.3 times.

*6-4-0S04: (HID) Added a feature for extracting vibration values from audio data in real time, and added an associated sample program.

Added a feature for extracting vibration values from audio data to the vibration API.
Used the nn::hid::VibrationConverter feature, which makes it possible to convert audio data to vibration values in real time.
This feature was implemented by porting the Vibration Converter tool for PC to the development hardware.
For more information, see the nn::hid::VibrationConverter API documentation.
This feature can be used to trigger vibrations corresponding to any form of audio data played in game applications, which makes it useful in a wide variety of game scenes. Avoid excessive use of strong, extended vibration effects in game applications because this type of use negatively affects the user experience.
Audio data that does not include any low-frequency components on the order of 500 Hz or less produce vibration values with an amplitude of approximately zero when converted.

Also added an HD vibration sample program demonstrating how to use this feature to convert sound effects to vibration values in real time during playback.
%NINTENDO_SDK_ROOT%¥Samples¥Sources¥Applications¥HidVibrationRealTimeConverter

This sample can be used to convert sound effects that are actually playing to vibration values in real time and to confirm that the controller vibrates accordingly. The sample also allows you to apply effects to the audio data and see how the resulting vibration patterns change.
For more information, see the following section in the NintendoSDK API Reference: SDK Sample Backgrounder > HID > Converting Audio to Vibration Values in Real Time.

*6-4-0S05: (OE) Added a feature for getting how long a program has been running since startup (not including time spent in states like sleep mode).

Added the nn::oe::GetProgramTotalActiveTime() function. This function gets the total actual time that a program has been running since being started. The nn::TimeSpan-typed time object returned by this function does not include time during which the program was suspended due to changes in focus state or the system being in sleep mode.

*6-4-0S06: (Accounts) Added a setting for use in NMETA files that restricts calls of the nn::account::OpenUser() function on user accounts other than the one selected prior to starting an application.

Added a setting for use in NMETA files that restricts calls of the nn::account::OpenUser() function on user accounts other than the one selected prior to starting an application. The default setting is OFF (no restrictions).
For more information, see Restricting the Switching of User Accounts in Writing the NMETA File.

6-4-0S07: (AUDIO) Added a feature for displaying true peaks in SystemAudioMonitor.

Added a feature for displaying true peaks in SystemAudioMonitor. For more information, see audio Library > Tool Reference > SystemAudioMonitor.

6-4-0S08: (AUDIO) Added an AuxLimiter effect.

Added a Limiter effect that can be used in combination with Aux. For more information, see the following functions.

* nn::audio::GetRequiredBufferSizeForAuxLimiter()
* nn::audio::InitializeAuxLimiter()
* nn::audio::ResetAuxLimiter()
* nn::audio::GetAuxLimiterInputOutput()
* nn::audio::SetAuxLimiterInputOutput()
* nn::audio::GetAuxLimiterChannelCountMax()
* nn::audio::SetAuxLimiterParameters()
* nn::audio::GetAuxLimiterParameters()
* nn::audio::GetAuxLimiterAttackTimeMax()
* nn::audio::GetAuxLimiterReleaseTimeMax()
* nn::audio::ProcessAuxLimiter()

6-4-0S09: (DevMenu) Added auto-sleep settings.

Added an item to the Device page for configuring the auto-sleep settings. For more information, see DevMenu > Changing Device Settings.

6-4-0S10: (DevMenu) Added save data import and export features to DevMenuCommand.

Added the following commands to DevMenuCommand.

  • savedata list-importable-savedata: Displays a list of importable backups on SD cards and host PCs.
  • savedata import: Imports save data from an SD card or host PC.
  • savedata export: Exports save data to an SD card or host PC.


For more information about these commands, see Features > Controlling the Development Hardware > DevMenuCommand.

6-4-0S11: (EC) Added a function for displaying a title's own product information.

Added a function for displaying a title's own product information.

6-4-0S12: (GFX) Added a feature to nn::gfx::util::MemoryPoolAllocator for selecting the allocation/deallocation mode.

Added a feature for selecting between a mode for prioritizing execution efficiency and a mode for prioritizing minimizing fragmentation when allocating and deallocating memory using nn::gfx::util::MemoryPoolAllocator.
Use the nn::gfx::util::MemoryPoolAllocator::SetAllocationMode() and nn::gfx::util::MemoryPoolAllocator::GetAllocationMode() functions to set and get the allocation/deallocation mode.

6-4-0S13: (HID) Added a function for creating a key config to use when using a generic game pad or key config as an NpadStyleFullKey controller in the Windows environment.

Previously, when using an NpadStyleFullKey controller via a generic gamepad or key config in the Windows environment, the Xpad key config settings in nn::settings were simply repurposed as appropriate. Now, a dedicated function is available for this use case.
The Xpad-based function will be deprecated in a future release. Developers who are currently using this function should migrate to the new function.

6-4-0S14: (HID) Added several new features to the HD vibration sample program for generating vibration waveforms from specified parameters.

Added the following new features to the HD vibration sample program for generating vibration waveforms from specified parameters.
%NINTENDO_SDK_ROOT%¥Samples¥Sources¥Applications¥HidVibrationGenerator

  • Added a vibration binary file output feature.
  • Added vibration curve types and frequency curve types.
  • Added a parameter that makes it possible to change the starting strength.
  • Added a key repeat feature to use when parameters are changed.
  • Added several preset waveform types.


This sample program can be used to design various types of vibration waveforms by changing several parameters in the source code.
For more information, see the following section in the NintendoSDK API Reference: SDK Sample Backgrounder > HID > Generating Simple Vibration Waveforms From Parameters.

6-4-0S15: (NVN) New NVN Features

Added the following features to NVN.

  • Added the nvnProgramSetSampleShading function to override the fragment shader's sample shading mode.


6-4-0S16: (RunOnTarget) In ControlTarget, added a feature for checking whether the development hardware is connected to a PC.

Added a feature for checking the connection between the development hardware and PC to ControlTarget.
This feature makes it possible to check the connections between PCs and a large number of connected targets in a short period of time.
In addition, the check-alive command can be used to check individual development unit-PC connections.
For more information, see the ControlTarget documentation.

6-4-0S17: (TIME) Added commands for displaying the current time to DevMenuCommand.

Added the following commands for displaying the current time to DevMenuCommand.

  • DevMenuCommand time get-user-clock
  • DevMenuCommand time get-network-clock


6-4-0S18: (TOOL) Added support for editing restrictions on user account switching in AuthoringEditor.

Added support for editing restrictions on user account switching (the UserAccountSwitchLock setting) in NMETA files in AuthoringEditor.
For more information, see Writing NMETA Files.
The AuthoringEditor tool is located in %NINTENDO_SDK_ROOT%\Tools\AuthoringEditor\AuthoringEditor.exe.

Improvements

*6-4-0S19: (FS) Standardized handling of resource data (ROM) paths to be case-sensitive.

Path handling for resource data (ROMs) on the Windows platform and raw NSPD-formatted resource data (ROMs) on the NX platform is now case-sensitive to match path handling for NSP-formatted resource data (ROMs) on the NX platform. This change was made to standardize path handling behavior for the various different types of resource data (ROMs).

6-4-0S20: (AUDIO) Changed the minimum values that can be specified for volume and mix volume in VoiceType, SplitterType, and SubMixType.

Changed the minimum values that can be specified for volume and mix volume in VoiceType, SplitterType, and SubMixType objects in audio renderers from 0.0f to -128.0f.
Specifying a negative volume inverts the phase of the waveform.

* nn::audio::VoiceType::GetVolumeMin()
* nn::audio::SplitterType::GetVolumeMin()
* nn::audio::SubMixType::GetVolumeMin()
* nn::audio::SetVoiceVolume()
* nn::audio::SetVoiceMixVolume()
* nn::audio::SetSplitterMixVolume()
* nn::audio::SetSubMixVolume()
* nn::audio::SetSubMixMixVolume()

6-4-0S21: (Compiler) Updated Clang for NX to version 1.5.8.

Updated Clang for NX to version 1.5.8.
The main changes are as follows.

  • Improved the performance of the NX32 version of the crypto library.
  • Fixed a bug where the --symbol-ordering-file option was ignored when using the LLD linker if the --symbol-ordering-file option and the --script option were both specified.


6-4-0S22: (DevMenu) Added a feature to DevMenuCommand for installing NSP files stored in a network location.

NSP files stored in a network location can now be specified as installation targets for the application install command in DevMenuCommand.
Use the following command to install NSP files on the network on the development hardware.
RunOnTarget.exe DevMenuCommand.nsp -- application install \network\Application.nsp

6-4-0S23: (DevMenu) DevMenu Application now displays a warning message when started on development hardware using a version of the firmware that is not assured to be compatible.

DevMenu Application now performs a firmware version check on startup and displays a warning message if compatibility with the detected firmware version is not assured.

6-4-0S24: (DIAG) The nn::diag::GetAllModuleInfo() function now supports getting build IDs.

The nn::diag::GetAllModuleInfo() function now supports getting build IDs.

The build ID is stored in the id member of the returned nn::diag::ModuleInfo object.

For more information about build IDs, see Compiler > Toolchain for NX.

In the Windows environment, the GUID and Age information of the PDB file is obtained.

6-4-0S25: (EC) Added a message to use when the response language is different from the language of the EcCatalog sample.

Revised to display a message when getting the item list in the EcCatalog sample if the response and request languages are different.
Although it is not necessary to offer similar steps in separate applications, it is implemented as one example of how to respond to this issue.

6-4-0S26: (EC) Added the EcCatalog sample to display value-added tax data.

Revised the EcCatalog sample to display value-added tax data when displaying the consumable item list.

In applications, when inc_tax_message is not null, you must include value-added tax data (inc_tax_message) with price data.

6-4-0S27: (EC) Deleted the description of the known bug about currency symbols in the EcCatalog sample.

Previously, when displaying price values, a ? symbol was displayed before or after the value instead of the appropriate currency symbol (such as £). This bug was caused by lack of support for currency symbols in the debug font library used in the sample. The debug font library now supports display of currency symbols, so the description of this known bug has been deleted.

6-4-0S28: (FS) Improved the handling of input file character encoding in FsAccessLogChecker 20.6.

Made the following improvements to handling input file character encoding inFsAccessLogChecker 20.6 (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe).

  • Improved automatic determination of input file character encoding, and added support for UTF-8 without BOM, UTF-16LE, UTF-16BE, UTF-32LE, and UTF-32BE.
  • Added the --encoding option for explicitly specifying the input file encoding.


For more information, see fs Library > Tool Reference > FsAccessLogChecker.

6-4-0S29: (FS) In FsAccessLogChecker 20.5, added support for UTF-32 with BOM and also added support for matching the character encoding of output files to the character encoding of input files.

In FsAccessLogChecker 20.5 (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe), added support for UTF-32 with BOM and also added support for matching the character encoding of output files to the character encoding of input files.

6-4-0S30: (FS) In FsAccessLogChecker 20.8, empty output files are no longer created when invalid options are specified.

In FsAccessLogChecker 20.8 (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe), empty output files are no longer created when invalid options are specified.

6-4-0S31: (FS) Made the check performed by the --encoding option stricter in FsAccessLogChecker 20.7.

In FsAccessLogChecker 20.7 (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe), when the --encoding option is specified, an error now occurs if the BOM of the specified encoding does not match the BOM of the input file.

6-4-0S32: (GFX) Added a function for deleting commands to the nns::dbgui library.

Added the EndFrame() function to the nns::dbgui library.
The EndFrame() function can be used to delete commands without allowing them to be rendered by Render().

6-4-0S33: (GFX) Added support for single-byte variants of the €, £, ł, and č symbols in nn::gfx::util::DebugFontTextWriter.

By developer request, added support for single-byte variants of the €, £, ł, and č symbols in nn::gfx::util::DebugFontTextWriter.

6-4-0S34: (GFX) Fixed a bug where applications using the gfx library sometimes could not use NVNGD in some environments.

In the nvnTutorial sample, changed the mask value to 255 so that the stencil buffer gets cleared correctly.

This change was included in NX Add-On 6.2.0 but inadvertently omitted from the revision history for that version.

The API reference for methods in the nn::ssl::Connection class was updated to include missing nn::Result codes that can be returned from them. The behavior did not change; these classes were missing only in the documentation. For more information, see the documentation.

*6-4-0W03: (3DTool) Added a note to the documentation that the intermediate file formats, the binary resource file format, and the shader archive file format are scheduled to be updated in a future release.

The upcoming release of NintendoSDK 7.0.0 will include changes to the intermediate file formats, the binary resource file format (BFRES), and the shader archive file format (BFSHA).

The following tasks must be performed to migrate assets for use in SDK 7.0.0 and later.

  • Update intermediate files using Intermediate File Updater (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileUpdater.exe).
  • Regenerate the binary resource file (BFRES file) from the intermediate files using 3D Binary Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe).
  • Regenerate the shader archive file (BFSHA file) from the intermediate files using 3D Shader Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe).
