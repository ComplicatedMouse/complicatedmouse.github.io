Details

Changes in NintendoSDK NX Addon 5.5.0 (based on Generic 5.5.0)

Specification change that breaks your code

*5-5-0S01: (FONT) Removed the fontll2 library.

Removed the fontll2 library from the package because there was a problem with how the library was being provided.

If you were previously using the fontll2 library, use the fontll library instead from now on.

Added Features

*5-5-0S02: (EC) Added an eShop service access utility feature.

Added an eShop service access utility feature to the ec library. This feature makes it possible to get a list of consumable service items that are currently available on Nintendo eShop for the application from within the application itself. Also added the EcCatalog sample demonstrating how to use this feature.

5-5-0S03: (TOOL) Added a feature for changing the duration of the inactive state allowed by RID applications.

Added a feature for changing the allowed duration of the inactive state individually for each RID application.

5-5-0S04: (TOOL) Added a feature to DevMenuCommand for redirecting resource data.

Added the debug set-host-romfs-redirection-target option to DevMenuCommand for configuring the resource data redirect feature.
When running the application on the host PC, this feature can switch the resource data (ROM) that the application references to the resource data installed on the development hardware.
For more information, see Controlling the Development Hardware > DevMenuCommand.

Improvements

*5-5-0S05: (General) Updated the EDEV USB driver certificate.

Updated the EDEV USB driver certificate.
As of June 28, if you attempt to install the earlier version of the EDEV USB driver, the USB driver is not installed successfully, and the EDEV is not recognized correctly.

If you look in Device Manager after this happens, NintendoSdkDebugger is displayed under Other Devices together with a warning message.

In most cases, if the EDEV is already successfully connected to your PC or you have already installed the SystemUpdater package on your PC, the EDEV USB driver has already been installed and there is no need to reinstall it.

*5-5-0S06: (TOOL) Added support for getting and sending crash reports in the development environment.

Added support for getting and sending crash reports in the development environment.

ProdMode, DevMenu, or DevMenuCommand must be used to enable the feature for sending crash reports in the development environment. For more information, see Enabling the Feature for Sending Crash Reports in the Development Environment.

To get a crash report for an event that occurred in the development environment, specify the --dev option when running the command. For more information, see Crash Reports.

5-5-0S07: (FS) Updated FsAccessLogChecker to version 19.1.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 19.1. This version includes the following changes.

  • Added support for access logs when individual file data caching is enabled.
  • Added support for determining access logs including invalid fetched entries as FAIL.
  • Added support for determining access logs including invalid fetched and fetch_exceeded entries as FAIL.


5-5-0S08: (TOOL) Added a feature to AuthoringTool for editing application control data in an NSP file.

Added the replacenspmeta command to AuthoringTool. This command can be used to edit the application control data included in an NSP file.
For more information, see Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

5-5-0S09: (TOOL) Improved compression quality for the bc1, bc2, bc3, bc7, and astc formats in Texture Converter.

Improved compression quality for the bc1, bc2, bc3, bc7, and astc formats in Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe). Also, the bc7 format now maintains the A component better than before.

5-5-0S10: (TOOL) Improved the crash report retrieval tool.

Extended the default timeout when the crash report retrieval tool communicates with the server to 180 seconds. It is also now possible to set any timeout time using the --timeout option.

Improved the mechanism that the crash report retrieval tool uses to get stack trace symbols, and sped up the process.

Bug Fixes

*5-5-0S11: (AUDIO) Fixed a bug that caused bias components to be included in audio.

Playing audio at a high volume and then suddenly stopping it, or cutting off audio due to a voice drop mixed the bias components into the input to nn::audio::SubMixType or nn::audio::FinalMixType for the currently playing waveform in nn:::audio::VoiceType.
Fixed this bug so that this bias is no longer included.

Due to this fix, waveform rendering results may sound slightly different than those obtained using an earlier version of the firmware.

5-5-0S12: (FS) Fixed access log output for the global file data caching feature.

Fixed a bug where the information for the size item was not correctly added to access logs generated after calling the nn::fs::EnableGlobalFileDataCache() function in 32-bit applications.

5-5-0S13: (General) Fixed a rare bug where unplugging the cable from the USB Type-C port while the system was being recognized as a USB device and then plugging the cable back in displayed a fatal error.

Fixed a rare bug where unplugging the cable from the USB Type-C port and then plugging the cable back in while the system was connected to a PC or similar device via USB and was being recognized as a USB device displayed a fatal error.

5-5-0S14: (Graphics) Fixed a bug where a white screen was displayed and the firmware stopped responding when an application started outputting video data.

Fixed a bug where a white screen was displayed and the firmware stopped responding when an application attempted to start displaying video.
This bug typically occurred when the application attempted to display a texture created in a memory pool larger than 1 GB.

5-5-0S15: (HID) Fixed a bug where the screen burn-in reduction feature and the auto-sleep feature stopped working after a Joy-Con controller was connected to or disconnected from the system rails.

Fixed a bug in NX Add-On 5.1.0 where in rare cases, after a Joy-Con controller was connected to or disconnected from the system rails, the screen burn-in reduction feature and the auto-sleep feature were no longer triggered even when the system remained inactive for the required period of time.
The inactive state is now correctly detected even after a controller is connected to or disconnected from the rails.

5-5-0S16: (IRSENSOR) Fixed a bug where attempting to start a broken IR Motion Camera caused the internal system state to become invalid, and the broken IR Motion Camera could not be properly detected until the system restarted.

Error 2205-0123 is designed to be displayed upon attempting to start a broken IR Motion Camera. Fixed a bug where previously, after this error was displayed, the system entered an invalid internal state, and error 2205-0123 continued to be displayed (even if a working IR Motion Camera was connected) until the system was restarted.


5-5-0S17: (IRSENSOR) Fixed a bug where the system aborted if the IR Motion Camera was in use, a controller was disconnected and reconnected, and then image transfer mode was immediately started and quickly ended.

Fixed a bug that occurred if, while the IR Motion Camera was in use in any mode, the controller using the IR Motion Camera was disconnected and reconnected and then image transfer mode was started.
Specifically, after reconnecting the controller and starting image transfer mode, if the nn::irsensor::StopImageProcessor or nn::irsensor::StopImageProcessorAsync function was called before image data had been successfully obtained by at least one call to nn::irsensor::GetImageTransferProcessorState, depending on the timing, the system sometimes aborted and returned error code 2162-0002.

5-5-0S18: (MOVIE) Fixed a bug where playback of high-bitrate video tracks in WebM/Matroska files sometimes stopped midway.

Fixed a bug where playing a WebM or Matroska file containing a track using the VP8/VP9 video codec sometimes caused the playback to stop midway.

5-5-0S19: (MOVIE) Fixed a bug where playback of Vorbis tracks having more than two channels stopped midway.

Fixed a bug where adding a Vorbis track with more than two channels to a WebM/Matroska container caused playback to stop midway when the data was played using the movie library.

5-5-0S20: (NFC) Fixed a bug that sometimes prevented listing of NFC devices during local communication.

Fixed a bug that prevented listing of NFC devices by functions such as nn::nfc::ListDevices() when the conditions of "conducting local communication" and "one wirelessly connected controller" were met.

5-5-0S21: (NIFM) Fixed a bug where a fatal error occurred if the system shut down due to a low battery level.

Fixed a bug where the system shut down due a low battery level, the NIFM transition stopped responding, and a fatal error occurred due to the resulting timeout.

5-5-0S22: (OE) Fixed a bug where system events from nn::oe::GetDefaultDisplayResolutionChangeEvent() were sometimes not signaled.

Fixed a bug where events for detecting changes to the display resolution, as obtained from the nn::oe::GetDefaultDisplayResolutionChangeEvent() function, were sometimes not appropriately signaled. Now an event is properly signaled when the default display resolution is changed.

5-5-0S23: (SWKBD) Fixed a bug where bufSize in nn::swkbd::String was overwritten when nn::swkbd::TextCheckCallback was configured and nn::swkbd::ShowKeyboard() was called.

Fixed a bug where configuring nn::swkbd::TextCheckCallback and calling nn::swkbd::ShowKeyboard() caused the <em>bufSize</em> in the nn::swkbd::String specified in the argument to be overwritten with the length of the text entered on the keyboard.

As a result of this bug, if nn::swkbd::ShowKeyboard() was called with <em>bufSize</em> in nn::swkbd::String overwritten, and the user entered text longer than <em>bufSize</em>, a problem occurred where only a portion of <em>bufSize</em> starting from the beginning of the text was returned to the application.

5-5-0S24: (TOOL) Fixed a bug where errors were erroneously detected when using Pia or NEX.

Fixed a bug where errors such as "11-003: Access to this feature is restricted - Bundle_Private_XXX" were erroneously detected when using Pia or NEX.

5-5-0S25: (TOOL) Fixed a bug where partially exported data could not be exported while following an account.

Fixed a bug where data that is partially exported (using --export-title) in BackupSaveData could not be exported while the account following feature (--follow-exist-account) was enabled.

5-5-0S26: (TOOL) Modified checks of setting values performed when AuthoringTool generates a patch.

Modified checks of setting values for save data, cache storage, temporary storage, and indexed cache storage performed when AuthoringTool generates a patch. There were excesses and deficiencies in the checks.

Changes in NintendoWare NX Addon 5.5.0 (based on Generic 5.5.0)

Improvements

5-5-0W01: (3DTool) Improved compression quality for the bc1, bc2, bc3, bc7, and astc formats in 3D Texture Converter.

Improved compression quality for the bc1, bc2, bc3, bc7, and astc formats in 3D Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe). Also, the bc7 format now maintains the A component better than before.

Bug Fixes

5-5-0W02: (Effect-VFX) Fixed a bug where an incorrect buffer size was specified when the constant buffer was configured.

Fixed a bug where the buffer size specified when configuring the constant buffer was miscalculated, causing an important notification about the size specification to be output in some cases, depending on the state of the binary.
