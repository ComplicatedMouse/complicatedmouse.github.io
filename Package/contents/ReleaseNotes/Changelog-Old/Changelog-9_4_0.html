Details

Changes in NintendoSDK NX Addon 9.4.0 (based on Generic 9.4.0)

Added Features

*9-4-0S01: (DevMenu) Added a feature for backing up save data as a batch process.

Added a feature to DevMenu and DevMenuCommand for backing up all of the save data associated with an application at the same time.
This feature enables all of the user save data for multiple users to be backed up at the same time. For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Save Data Management > Backup
  • Features > Controlling the Development Hardware > DevMenuCommand > Managing Save Data > Save Data Batch Backup.


*9-4-0S02: (General) Added features that are necessary for developing applications for the Switch system for China.

Added features that are necessary for developing applications for the Switch system for China.
You can submit master ROMs of applications for the Switch system for China if developed with NX Add-On 9.4.0 or later of the SDK.
For more information about these features and related notes, see For Switch Systems For China (NX Add-On).

*9-4-0S03: (TOOL) Added subcommands to AuthoringTool to merge and split downloadable content.

Added subcommands to AuthoringTool to merge and split downloadable content.

  • mergensp: Merges multiple pieces of downloadable content into one file.
  • splitnsp: Splits a single piece of downloadable content into multiple files.

For more information, see the following sections in Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

  • Merging Multiple Pieces of Downloadable Content Into a Single NSP File
  • Separating a Single Downloadable Content NSP File by Index


9-4-0S04: (CRYPTO) Added functions for getting the hash value for data subject to verification after RSA signature verification.

Added functions to the nn::crypto namespace for getting the hash value for data subject to verification after RSA signature verification.
For more information, see the following functions in the API reference.

  • nn::crypto::RsaPssVerifier::GetHash
  • nn::crypto::RsaPkcs1Verifier::GetHash


9-4-0S05: (EC) Added price information to the information obtained for other related titles in the EcCatalog sample.

The EcCatalog sample now displays price information on the screen for getting information for other related titles.

9-4-0S06: (NVN) New NVN Features

Added the following features to GLSLC.

  • Implemented support for the EXT_texture_shadow_lod GLSL extension.
    • Supported in both GLSLC/NVN and OpenGL release.


Improvements

9-4-0S07: (DevMenu) Added thumbnails for user save data that are backup and restore targets.

Added thumbnails for user save data that are backup and restore targets in DevMenu and DevMenuCommand.

9-4-0S08: (EC) Expanded the subscription service item functionality in the EcCatalog sample.

Added the following functionality for subscription service items in the EcCatalog sample.

  • Added support for displaying the price.
  • Added handling for automatically renewed items that are free for the first time.
  • Added support for using item_tags to change the display order.


9-4-0S09: (TOOL) Added a feature for saving the error logs for the GUI setup tools.

The GUI setup tools (InitializeSdevWin and InitializeEdevWin) already have a feature for saving a log at the time of the final setup, but now a feature has been added for saving a log of errors under a different name.

This feature is used when these tools are supported.

9-4-0S10: (TOOL) Changed the output content of getproperty in AuthoringTool.

Changed the output content when getproperty in AuthoringTool is executed. The output content includes the following items.

  • Application ID
  • Type
  • Index (only when the Type is downloadable content)
  • Release Version
  • Digest (ROM identification hash)
  • Size Info

For more information, see Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Revise Application Control Data for NSP Files.

9-4-0S11: (TOOL) Revised the logs when using InitializeHdev and SystemUpdateHdev.

In NX Add-On 9.3.0 and later, the HDEV setup tools (InitializeHdev and SystemUpdateHdev) have been merged with the EDEV tools. As a result, the logs have also been merged. However, logs that showed EDEV when running an HDEV setup tool have been revised to now show HDEV.
Either the EDEV or HDEV setup tools can be used to initialize and update HDEV.

9-4-0S12: (TOOL) Updated the confirmed operating environment for Retail Interactive Demo applications.

Updated the confirmed operating environment for Retail Interactive Demo applications geared toward NintendoSDK Firmware for NX 9.x.x or later.
Along with this change, updated the tool for creating the asset data for a Retail Interactive Demo application. The following content was updated.

  • Made improvements so that the titles for all languages in the input NSP file are included in the package title in the Asset Data.
  • Fixed a bug where creating Asset Data could fail depending on the drive where the input files were located.


Bug Fixes

9-4-0S13: (DevMenu) Fixed a bug where a fatal error sometimes occurred when attempting to delete user accounts while the application was running.

Fixed a bug where a fatal error occurred if alarm settings were linked to a user account when attempting to delete that account while the application was running.
This fix has changed the specifications so that user accounts cannot be deleted while the application is running regardless of the situation.

9-4-0S14: (DevMenu) Fixed a bug where pressing the HOME Button while a library applet was running could cause a fatal error.

Fixed a bug where running a library applet from DevMenu and then pressing the HOME Button caused a fatal error.

9-4-0S15: (DevMenu) Fixed a bug where registering a primary console in DevMenuApp failed under certain conditions.

Fixed a bug where registering a primary console from Registering an Active Console in DevMenuApp failed under the following conditions.

  • A different console was already been registered as the primary console for the Nintendo Account linked to the selected account.


9-4-0S16: (DevMenu) Fixed a bug where save data could not be properly backed up if a filename or a directory name included multibyte characters.

Fixed a bug where save data could not be properly backed up if it included a filename or a directory name with multibyte characters.
For more information about handling multibyte characters, see Features > Controlling the Development Hardware > DevMenu > Save Data Management > Backup.

9-4-0S17: (MOVIE) Fixed a bug in the video decoder where sometimes the movie became corrupted after the resolution was changed in CPU rendering mode.

With this fix, if the resolution is changed while a movie is being played in CPU rendering mode, the new resolution is calculated correctly and the movie no longer becomes corrupted. You can confirm this fix by playing a movie with the MovieDecoderPlayer sample program and changing the resolution from 720p to 1080p while the movie is playing.

9-4-0S18: (NVN) NVN Bug Fixes

Fixed the following NVN bugs.

  • Fixed a bug where some debug layer operations were not thread-safe.

  • Fixed a bug where nvnQueueBuilderSetCommandMemorySize could fail in some cases.

  • Fixed a bug where nvnCommandBufferBarrier could cause memory corruption in the out-of-memory routine for command memory.


Fixed the following bugs in glslc.

  • Fixed a bug where SPIR-V would not properly compile when the image type was Unknown and the capability was StorageImageReadWithoutFormat.


9-4-0S19: (TOOL) Fixed a bug where it was no longer possible to set privileges for other applications to access save data in AuthoringEditor.

Fixed a bug where, in AuthoringEditor in NintendoSDK AuthoringToolKit for NX 9.2.0 or later, it was not possible to configure (add, delete, or change) the access privileges for other applications setting in Details > File System Access.

9-4-0S20: (TOOL) Revised the environments checked by the SDK environment checker.

Windows 7 has been removed from the confirmed operating environments for NintendoSDK, so Windows 7 has been removed from environments checked by the SDK environment checker tool (Tools\SdkEnvironmentChecker\SdkEnvironmentChecker.exe),

9-4-0S21: (TOOL) You can no longer revise the application control data for a patch using the replacenspmeta command.

An error now occurs if the NSP file for a patch is entered as the file for the replacenspmeta command.

9-4-0S22: (WEB) Fixed a bug in the web inspector that is used for developing web content.

Fixed the following bugs with the web inspector.

  • Styles could not be turned on and off in the Styles window.
  • Operating the color picker and other editing tools caused an internal error in the web inspector.


Changes in NintendoWare NX Addon 9.4.0 (based on Generic 9.4.0)

Bug Fixes

9-4-0W01: (LayoutTool) Fixed a bug in LayoutEditor where using a capture pane caused the tool to crash.

Fixed a bug in LayoutEditor where the tool crashed if you opened a layout that used a capture pane and then moved the main window.
