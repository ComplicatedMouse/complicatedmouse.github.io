Fixed a bug where a different EDEV would be initialized, using InitializeEdev, when multiple EDEV consoles were connected.

Changes in NintendoWare NX Addon 3.5.0 (based on Generic 3.5.0)

Bug Fixes

3-5-0W01: (3D-G3D) Fixed a bug where lodLevel in pRangeArray would be 0, even when any value other than 0 had been assigned to meshIndex as an argument of nn::g3d::ShapeObj::MakeSubMeshRange().

The bug that caused lodLevel in pRangeArray to always be 0, even when any value other than 0 had been assigned to meshIndex as an argument of nn::g3d::ShapeObj::MakeSubMeshRange() has been fixed.

This fix can speed up the rendering process by rendering models at their specified lodLevel.
However, there is also the possibility that your rendering results will change because models are now rendered at their set lodLevel instead of the unintentional rendering of IodLevel 0 models.

3-5-0W02: (LayoutTool) Fixed a bug that caused the tool to exit unexpectedly due to an increase in memory usage when the Parts Palette in LayoutEditor was reloaded repeatedly.

Fixed a bug that caused the tool to exit unexpectedly due to an increase in memory usage when the Parts Palette in LayoutEditor was reloaded repeatedly.
