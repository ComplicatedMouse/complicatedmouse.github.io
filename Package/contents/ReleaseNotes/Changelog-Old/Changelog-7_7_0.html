Details

Changes in NintendoSDK NX Addon 7.7.0 (based on Generic 7.7.0)

Specification change that breaks your code

*7-7-0S01: (WebSocket) Improved error handling during connection.

Added and changed the results returned when the nn::websocket::WebSocket::Open() fails.

The following results are now returned when HTTP status 300 errors occur.
Before this change, nn::websocket::ResultHttpStatus3xx was returned regardless of the status.

  • nn::websocket::ResultHttpStatus301
  • nn::websocket::ResultHttpStatus302
  • nn::websocket::ResultHttpStatus303
  • nn::websocket::ResultHttpStatus305
  • nn::websocket::ResultHttpStatus307
  • nn::websocket::ResultHttpStatus308
  • For any other status: nn::websocket::ResultHttpStatus3xx


The following result is now returned when HTTP status 200 errors occur.
Before this change, nn::websocket::ResultHttpStatus2xx was returned.

  • nn::websocket::ResultProtocolUpgradeRequestIgnored


Added Features

7-7-0S02: (EC) Added a sample to the EcCatalog samples to get subscription service item information.

Added the Subscription service item list option to the EcCatalog start screen, and added a sample for getting and displaying subscription service item information.

7-7-0S03: (TOOL) Added the comparensp command to AuthoringTool.

Added the comparensp command to AuthoringTool.
comparensp compares two NSP files and displays the result.
For more information, see NintendoSDK Documents > Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

7-7-0S04: (WebSocket) Added a connection function to specify the sub-protocol list.

Added an overload function to nn::websocket::WebSocket::Open to specify the sub-protocol list.

7-7-0S05: (WebSocket) Added a handshake completion event handler.

Added nn::websocket::WebSocket::HandshakeCompletionEventHandler.
You can now check the response header and sub-protocol during a handshake.

Improvements

7-7-0S06: (GFX) Added support for Latin characters with diacritical marks to nn::gfx::util::DebugFontTextWriter.

In response to a developer request, added support for the following Latin characters with diacritical marks to nn::gfx::util::DebugFontTextWriter.

7-7-0S07: (WebSocket) Improved error handling when there is an abnormal disconnection.

When there is an abnormal disconnection, such as when the server is down, the error is now nn::websocket::ResultConnectionClosed instead of nn::websocket::ResultSendError or nn::websocket::ResultReceiveError.

7-7-0S08: (WebSocket) Improved internal processing when receiving close requests.

The connection is now automatically disconnected when a close request is received from the server.
This happens smoothly when the server is waiting for a client to disconnect after that server sent a close request.

Bug Fixes

7-7-0S09: (Compiler) Updated to Clang for NX 1.7.10.

Updated Clang for NX to version 1.7.10.
The main changes are as follows.

  • Implemented a fix for an fstack-protector vulnerability (VU#129209).


7-7-0S10: (EC) In the EcCatalog sample, fixed a bug with getting subscription service items that was caused by an incorrect path.

In the EcCatalog sample, fixed a bug where Subscription service item list did not work when selected because the wrong path was accessed.

7-7-0S11: (TOOL) Fixed a bug where an invalid application ID was displayed when submitting the master ROM if you specified PlayLogQueryableApplicationId.

Fixed a bug where an invalid Application ID (0x0) was displayed when submitting the master ROM if PlayLogQueryableApplicationId was described in the NMETA file.
