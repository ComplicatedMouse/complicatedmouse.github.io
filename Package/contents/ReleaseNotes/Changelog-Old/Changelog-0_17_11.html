Summary

Changes in NintendoSDK NX Addon 0.12.11 (based on Generic 0.17.11)

Added Features

  • *13-12S01: (OE) Changed the performance configurations that the retail product can use (nn::oe::PerformanceConfiguration).
  • *13-12S02: (OE) Changed the performance configurations that the retail product can use (nn::oe::PerformanceConfiguration).
  • 13-12S03: (DIAG) Added a feature to Log Viewer for converting files in NX Binary Log format files to text files.

  • 13-12S04: (DIAG) Fixed a bug where Log Viewer would display an invalid timestamp when a file in NX Binary Log format was opened.

  • 13-12S06: (MOVIE) Fixed a bug where program control would not return from the movie API while video was playing.
  • 13-12S07: (TOOL) Fixed a bug where BackupSaveData would generate an error when importing save data if the save data contained a file that was larger than the journal size.

Details

Changes in NintendoSDK NX Addon 0.12.11 (based on Generic 0.17.11)

Added Features

*13-12S01: (OE) Changed the performance configurations that the retail product can use (nn::oe::PerformanceConfiguration).

Changed the performance configurations that can be set with nn::oe::SetPerformanceConfiguration.

  • The following performance configuration is now available in the production environment (whereas before it could only be used in the development environment).
    • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1331Mhz
  • In the near future, the following performance configurations will no longer be available in either the production environment or the development environment.
    • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu307MhzEmc1600Mhz
    • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1600Mhz


*13-12S02: (OE) Changed the performance configurations that the retail product can use (nn::oe::PerformanceConfiguration).

Changed the performance configurations that can be set with nn::oe::SetPerformanceConfiguration.

  • The following performance configuration is now available in the production environment (whereas before it could only be used in the development environment).
    • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1331Mhz
  • In the near future, the following performance configurations will no longer be available in either the production environment or the development environment.
    • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu307MhzEmc1600Mhz
    • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1600Mhz


13-12S03: (DIAG) Added a feature to Log Viewer for converting files in NX Binary Log format files to text files.

You can now convert the file format by selecting Menu > Convert NX Binary Log and specifying the target NX Binary Log file and the output destination for the converted text file.

Bug Fixes

13-12S04: (DIAG) Fixed a bug where Log Viewer would display an invalid timestamp when a file in NX Binary Log format was opened.

NX Binary Log files do not contain a timestamp, but Log Viewer would mistakenly display the time that the file was opened as its timestamp. That has been fixed, and now nothing is shown.

13-12S05: (DIAG) Fixed a bug where Log Viewer would end abnormally when a huge file in NX Binary Log format was opened.

Fixed a bug where Log Viewer would become inoperable and end abnormally due to insufficient memory if a file larger than 1 GB in NX Binary Log format was opened.

13-12S06: (MOVIE) Fixed a bug where program control would not return from the movie API while video was playing.

Fixed a bug where, on rare occasions, program control would not return from the movie API while video was playing.

13-12S07: (TOOL) Fixed a bug where BackupSaveData would generate an error when importing save data if the save data contained a file that was larger than the journal size.

Fixed a bug where BackupSaveData would generate an error when importing save data that had been exported if the save data contained a file that was larger than the journal size.

Changes in NintendoWare NX Addon 0.12.11 (based on Generic 0.17.11)

Bug Fixes

13-12W01: (Layout-UI2D) Fixed a bug where previewing would fail in LayoutEditor if the process of binary conversion for previewing took a long time, due to the determination that previewing had timed out unexpectedly.

Fixed a bug where previewing would fail in LayoutEditor if the process of binary conversion for previewing took a long time, due to the determination that previewing had timed out unexpectedly.
