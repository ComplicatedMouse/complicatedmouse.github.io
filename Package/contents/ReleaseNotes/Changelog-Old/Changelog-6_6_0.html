Details

Changes in NintendoSDK NX Addon 6.6.0 (based on Generic 6.6.0)

Specification change that breaks your code

*6-6-0S01: (IRSENSOR) Changed the header structure that provides the Pointing features of the irsensor library.

Previously; the Pointing features were provided by the irsensor_PointingProcessor.h header included in irsensor.h, but now that header been separated from irsensor.h.
Note that as a result of this change, developers who have been using the Pointing features must now explicitly include the irsensor_PointingProcessor.h header.

*6-6-0S02: (TOOL) Added a feature to the makepatch command of AuthoringTool that checks whether the NSP file can be submitted.

Added a feature to the makepatch command of AuthoringTool that checks whether the NSP file can be submitted.
To check whether the NSP file can be submitted, specify the --error-unpublishable option when you run the makepatch command.
This option checks for the same items that are checked by the get-unpublishable-error command of Features > Building > Creating Application Images > Tool Reference > AuthoringTool.
The creation of the NSP file fails if any items require fixing.
If there are any fatal errors, the creation of the NSP file fails even if the --error-unpublishable option has not been specified.
For more information about the makepatch command, see Features > Patches > Building and Running a Patch.

*6-6-0S03: (TOOL) Moved the data for error detection in AuthoringTool from AuthoringToolKit to UnpublishableErrorKit.

The data that was included in the AuthoringToolKit for NX package for detecting errors when determining whether an NSP file meets the conditions for submission to Nintendo has been moved to the UnpublishableErrorKit for NX package.

Added Features

*6-6-0S04: (EC) Added a notification feature for shop purchases.

Added a shop purchase notification feature (nn::ec::GetPurchasedEvent and others) to the ec library.
Applications can now detect product purchases when users purchase consumables or passes through the shop application on NX devices or the Nintendo.com website.
Also added the EcPurchasedEvent sample demonstrating how to use this feature.
Note: Testing with firmware version 7.0.0-5.0 or later is required.

Improvements

*6-6-0S05: (TOOL) Added a feature to check for consistency of tool versions in the application and the patch.

Added a feature to AuthoringTool to check the version of the tool that was used when the NSP file was created for the application and for the patch.
An error is displayed when get-unpublishable-error is run if the NSP files for the application and the patch were created using different versions of the tool.
Also, the replace command fails if you run it to try to revise entries in those NSP files.
For more information about these commands, see Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

*6-6-0S06: (TOOL) Creation of the NSP file now fails if the SDK used by the application and AuthoringTool have different major versions.

When AuthoringTool is used to create an NSP file, the process now fails if the major version of AuthoringTool is different from the major version of the SDK used by the application.

*6-6-0S07: (TOOL) Enabled using Hangul, Simplified Chinese, and Traditional Chinese for title names, publisher names, and tags in NMETA files.

Enabled using Hangul, Simplified Chinese, and Traditional Chinese for the following elements in NMETA files.

  • Application/Title/Name
  • Application/Title/Publisher
  • AddOnContent/Tag

For more information, see Features > Building > Creating Application Images > Writing the NMETA File and Features > System > aoc Library > Creating Downloadable Content.

6-6-0S08: (EC) In the EcCatalog sample, added support for the new sales_status specifications and deleted parsing of discount_price.

Because sales_status from the default behavior of the function for getting a list of consumables/AOC was changed to return only onsale, the processing for branching in sales_status was deleted.
In addition, added support for outputting errors when sales_status is not onsale.
Because there may be changes to the content obtained by discount_price when getting the list of consumable items, the parsing processing and member variables within the DiscountPrice class have been deleted.

6-6-0S09: (EC) Removed release_date used by AOC item shelf in the EcCatalog sample.

AOC item shelf in the EcCatalog sample no longer uses release_date, so the places where release_date was used have been removed.

Bug Fixes

6-6-0S10: (BuildSystem) Fixed a bug where nlib was not available even if installed when the wizard for creating a NintendoSDK application was run.

Fixed a bug where even though the nlib package was installed, nlib was not shown as a candidate in the screen for setting the configuration options when the wizard for creating a NintendoSDK application was run.

6-6-0S11: (MOVIE) Fixed a bug in the MovieDecoderPlayer sample where movie playback did not resume immediately after returning from the HOME Menu.

Fixed a bug in the MovieDecoderPlayer sample where movie playback did not resume immediately after returning from the HOME Menu.
The unmodified Tick value is used to calculate the amount of time the movie has played, and the procedure failed to take into account the elapsed time while the sample application was out of focus.

6-6-0S12: (MOVIE) Fixed a bug where calling AppendAudioOutBuffer in MovieDecoderPlayer occasionally caused a crash.

In MovieDecoderPlayer samples, there was a dependency issue with the timing of termination processing for AppendAudioOutBuffer and AudioOutRenderer, and calling AppendAudioOutBuffer occasionally caused a crash.
This bug was fixed.

Changes in NintendoWare NX Addon 6.6.0 (based on Generic 6.6.0)

Added Features

6-6-0W01: (Audio-ATK) Added a feature to AtkPlayer to decode using HardwareOpusDecoder.

Added a feature to AtkPlayer to decode using the HardwareOpusDecoder class. With this feature, you can now play and check Opus stream sounds even when Specifies using the hardware to decode Opus in the target is enabled in the SoundMaker project settings.
