Details

Changes in NintendoSDK NX Addon 1.2.0 (based on Generic 1.2.0)

Specification change that breaks your code

*1-2-0S01: (FS) Deprecated the version of the nn::fs::MountAddOnContent() function that has no cache buffer parameter.

Followed through with plans to deprecate nn::fs::MountAddOnContent(const char * name, nn::aoc::AddOnContentIndex targetIndex), the version of the function that has no cache buffer parameter.
Switch to nn::fs::MountAddOnContent(const char * name, nn::aoc::AddOnContentIndex targetIndex, void * pCacheBuffer, size_t cacheBufferSize), which has a parameter for specifying the cache buffer.

*1-2-0S02: (OE) Deleted the definitions for performance configurations that are scheduled to be deprecated.

Deleted the definitions for the following performance configurations that can be set using nn::oe::SetPerformanceConfiguration because they are scheduled to be deprecated.

  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu307MhzEmc1600Mhz
  • nn::oe::SetPerformanceConfiguration_Cpu1020MhzGpu384MhzEmc1600Mhz


Added Features

1-2-0S03: (APPLET) Added a new focus handling mode.

Added a new focus handling mode called In Focus Only mode. To change to this mode, specify nn::oe::FocusHandlingMode_InFocusOnly for the nn::oe::SetFocusHandlingMode() function. For more information, see Handling Changes in the Focus State in Notifications For Programs.

1-2-0S04: (AUDIO) Added six-channel support to nn::audio::ReverbType.

nn::audio::ReverbType now supports six channels of audio.

1-2-0S05: (DevMenu) Added Suspend background activity temporarily to the Firmware settings menu on the Accounts page.

Added Suspend background activity temporarily to the Firmware settings menu on the Accounts page.
When this item is enabled, the following background processes related to accounts are suspended.

  • Immediate synchronization with network-service accounts when profiles are updated.


The suspension lasts for the following durations.

  • Until the item is disabled.
  • In the case of DevMenu: Until a reboot or the development hardware is reset.
  • In the case of DevMenuApp: Until DevMenuApp ends.
  • In the case of both DevMenu and DevMenuApp: Until the various suspended processes become invalid.


1-2-0S06: (DevMenu) Added a feature to DevMenuCommand for enabling and disabling automatic adjustment of the user clock.

Added the following commands to DevMenuCommand to control whether to automatically adjust nn::time::StandardUserSystemClock via the network.

  • time set-user-clock-auto-correction <enable|disable>
  • time get-user-clock-auto-correction


1-2-0S07: (DevMenu) Added an item to DevMenu for enabling and disabling CEC (the feature for controlling power with the TV).

Added an item to DevMenu for enabling and disabling CEC, which is the feature for controlling power with the TV. The feature can be set using the Consumer Electronics Control (CEC) item on the Device page of DevMenu.

1-2-0S08: (DevMenu) The original storage location of the running application is now displayed.

While an application is running, the storage location where the application and any patch was booted is now displayed.

1-2-0S09: (General) Added a mechanism for periodically conducting system processes while the system is sleeping.

Added a mechanism for the system to periodically connect to the network and conduct background processes while the system is sleeping. This addition does not require any particular consideration when developing applications.

1-2-0S10: (GFX) Added a sample application using Vulkan in the development hardware environment.

Added the Samples/Applications/Sources/VkSimple sample application, which demonstrates the use of the Vulkan API in the development hardware environment.

1-2-0S11: (OS) Extended nn::os::SetMemoryPermission() to take nn::os::MemoryPermission_None for its parameter.

nn::os::MemoryPermission_None can now be specified for the nn::os::SetMemoryPermission() function. With this memory access permission setting, read/write access to changed memory regions is not allowed.

1-2-0S12: (TOOL) Added a feature for writing on-card patches to DevMenuCommand.

You can now use the gamecard command with DevMenuCommand to create development cards that have an on-card patch. For more information, see Building and Running a Patch.

1-2-0S13: (TOOL) Added an option to the shader converter for specifying GLSLC option flags.

Added the --glslc-option-flags option for specifying GLSLC option flags to the shader converter tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe).
Each flag is coded as a suboption in this option, using the same option notation.
The following GLSLCoptionFlags members can be specified as suboptions.

  • spillControl
  • tessellationAndPassthroughGS
  • enableFastMathMask
  • optLevel
  • unrollControl
  • errorOnScratchMemUsage
  • enableCBFOptimization
  • enableWarpCulling

The following are a few examples.
--glslc-option-flags="--optLevel=GLSLC_OPTLEVEL_NONE --errorOnScratchMemUsage=1 --spillControl=NO_SPILL --unrollControl=GLSLC_LOOP_UNROLL_NONE"

1-2-0S14: (TOOL) Added flags for specifying GLSLC options to the shader compiler.

Added the following option flags to nngfxToolShaderCompilerCompileOptionNvn for the shader compiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderCompiler.dll). The same values can be set for these as can be set for the GLSLCOptionFlags members.

  • tessellationAndPassthroughGS
  • prioritizeConsecutiveTextureInstructions
  • enableFastMathMask
  • optLevel
  • unrollControl
  • errorOnScratchMemUsage
  • enableCBFOptimization
  • enableWarpCulling


1-2-0S15: (TOOL) The shader converter can now add GLSLC option flag information to the dump file.

If you specify --ApiType=nvn and the --dump-directory option when you run the shader converter tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe), information about the GLSLC option flags will be included in the dump file.

Improvements

*1-2-0S16: (FRIENDS) Made changes so that the declaration of the end of online play is performed automatically during sleep mode.

Because it is self-evident that the Internet connection is dropped when the system is sleeping, the declaration of the end of online play (a process comparable to nn::friends::DeclareCloseOnlinePlaySession) is now performed automatically when the system enters sleep mode.

1-2-0S17: (Effect-VFX) Enlarged the sort buffer for particles in the VfxSimple sample.

Increased the size of the sort buffer for particles in the VfxSimple sample so a warning about insufficient buffer size is no longer output.

1-2-0S18: (DevMenu) Improved the operation of scrolling to select a user.

Fixed a bug where the Classic Controller and the Joy-Con could not be used to select a user on the Accounts page if the list of user accounts needed to be scrolled down to display the user.

1-2-0S19: (FS) Added a check of the card size when creating a patch.

Added a feature when creating a patch to calculate the size required of an NX card to store that patch as an on-card patch and to display an error message when the configured card size will not be large enough.
If this error message is displayed, handle it using the procedure described in Building and Running a Patch.

1-2-0S20: (FS) Added support for using regular expressions in the authoring tool's extract and list commands.

Added options to the authoring tool's extract and list commands that support using regular expressions to filter entry targets.

1-2-0S21: (GFX) Improved performance by not using Flush and Invalidate on CpuUncached buffers.

To prevent a significant drop in performance in the NVN version when FlushMappedRange or InvalidateMappedRange is called on a buffer created from a CpuUncached memory pool, the processes are no longer passed to the low-level APIs Flush and Invalidate.

1-2-0S22: (HTC) Added samples and documentation for the functions for getting notification events for connection and disconnection with Target Manager.

Added samples and documentation for the functions for getting notification events for connection and disconnection with Target Manager.

1-2-0S23: (HTCS) Made revisions to the function reference for nn::htcs::GetLastError().

The reference for the nn::htcs::GetLastError() function was out of step with the current specification, so the following revisions were made.

  • Deleted the sentence that stated that at the current time the function returns only HTCS_EWOULDBLOCK (HTCS_EAGAIN) or HTCS_EUNKNOWN.
  • Noted that the errors obtained by the function are errors that occurred on the same thread.


1-2-0S24: (NIFM) Alleviated a problem where sometimes the wired connection was not available when the system started or immediately after it woke from sleep.

There was a bug where sometimes the wired connection was not available when a network use request was made when the system started or immediately after it woke from sleep. To alleviate the problem, the system now waits a while for the USB Ethernet adapter to be discovered. One byproduct of this tweak is that the review of the network use request is now suspended for as long as around 10 seconds even when the system is not connected to a USB Ethernet adapter.

1-2-0S25: (NVN) Added directions in the Troubleshooting section to deal with cases where the Environment Checker tool displays "Driver not installed" for NVN API Version even though the NVIDIA Graphics Driver is installed.

Added directions in the Troubleshooting section to deal with cases where the Environment Checker tool displays "Driver not installed" for NVN API Version even though the NVIDIA Graphics Driver is installed. For more information, see Troubleshooting.

1-2-0S26: (NVN) Revised the NvnTutorial sample to make use of the nvnWindowAcquireTexture() function.

Revised the NvnTutorial sample to make use of the nvnWindowAcquireTexture() function, replacing the nvnQueueAcquireTexture() function that is scheduled to be deprecated.

1-2-0S27: (TOOL) Added a process for checking errors when creating downloadable content.

When you create downloadable content using AuthoringTool, an error is now generated if duplicate Tag values are discovered in the NMETA file.

Bug Fixes

*1-2-0S28: (GFX) Fixed a problem with implementations of the sample applications that used aligned_alloc() for the allocation function passed to nv::SetGraphicsAllocator() where sometimes aligned_alloc() was called with a size that did not satisfy the requirements.

Fixed a problem with implementations of the sample applications that used aligned_alloc() for the allocation function passed to nv::SetGraphicsAllocator() where sometimes aligned_alloc() was called with a size that did not satisfy the requirements.
The size passed to aligned_alloc() must be an integer multiple of the requested alignment. However, the memory size requested by nv::SetGraphicsAllocator() sometimes did not satisfy that requirement. When aligned_alloc() is used in the implementation, the size must be adjusted so it is an integer multiple of the requested alignment.

1-2-0S29: (AOC) Fixed a bug where aoc::ListAddOnContent did not work correctly.

Fixed a bug where aoc::ListAddOnContent sometimes returned incorrect results if <em>count</em> <= <em>offset</em>.

1-2-0S30: (AUDIO) Fixed a bug in SplitterType.

There was a bug where SplitterType would not correctly mix the output to the destinations if the input was a multichannel VoiceType or SubMix object.
This bug has been fixed.

1-2-0S31: (AUDIO) Fixed a bug with the nn::audio::CopyeMemoryPoolData() function.

There was a bug with the nn::audio::CopyeMemoryPoolData() function where the specification of neighboring memory spaces for the input and output addresses would wrongly cause an assertion failure. This bug has been fixed.

1-2-0S32: (Display) Fixed a bug where on rare occasions the video would continue to be output to the console screen even when the system was docked.

Fixed a bug where on rare occasions the video would continue to be output to the console screen even when the system was docked.

1-2-0S33: (FRIENDS) Fixed a bug where an unexpected result occurred when the network-service account ID list to be passed to GetProfileList was destroyed immediately.

Fixed a bug where there was an unexpected result when the network-service account ID list to be passed to GetProfileList was destroyed immediately.

1-2-0S34: (FRIENDS) Fixed the problem of a libcurl.a link error when building the FriendsProfileImage sample.

Fixed the problem of a libcurl.a link error when building the FriendsProfileImage sample.

1-2-0S35: (FS) Fixed a bug where intermediates would remain when the second and subsequent patches were created.

Fixed a bug where intermediates would remain when a patch was created with the --previous option specified.

1-2-0S36: (FS) Fixed a bug where patches sometimes took a long time to create.

Fixed a bug where it could take a long time to create a patch if the application contained files with very little compression.

1-2-0S37: (FS) Fixed a bug where the development hardware would not start if you inserted an SD card with corrupted metadata.

Fixed a bug where the development hardware would not start if you inserted an SD card with corrupted metadata.

1-2-0S38: (FS) Fixed a bug where the mount functions would return an unexpected Result value to the application when they failed.

Fixed a bug where the mount functions of the fs library would return an unexpected Result value to the application when they failed.

1-2-0S39: (General) Fixed a bug where fatal error 2162-0002 would occur when running an application while connected to Target Manager.

Fixed a bug where the following fatal error would occur when connected to Target Manager and running an application.
ErrorCode: 2162-0002
Program: 01000000000000017

1-2-0S40: (General) Fixed a bug where offline HTML content could not be accessed when the NSPD file was started.

The problem was that when the path to the offline HTML content was specified in the NMETA file, that path was not being applied.

1-2-0S41: (GFX) Fixed a bug where an assertion failure would occur if four scan buffers were specified by the nn::gfx::SwapChain::Initialize() function when using the nvn low-level API.

Fixed a bug where an assertion failure would occur if four scan buffers were specified by the nn::gfx::SwapChain::Initialize() function when using the nvn low-level API.
When you checked to make sure there were four or fewer scan buffers, only values of three or less could be accepted.

1-2-0S42: (GFX) Fixed a bug where the shader compiler header files could not be loaded in the NX version.

Fixed a bug where nn/gfxTool/gfxTool_ShaderCompilerApi.h and nn/gfxTool/gfxTool_ShaderConverterApi.h could not be loaded in the NX version.
These header files are required when the shader converter is used with the --full-reflection feature.

1-2-0S43: (HID) Fixed a bug where auto-dimming mode was sometimes not canceled in system applications.

The problem was that the play formats provided for system applications did not detect changes in input.

1-2-0S44: (HID) Fixed a bug where NFC could not be used during local communication.

Fixed a bug where the nn::nfp::StartDetection() and nn::nfc::mifare::StartDetection() functions would fail during local communication.

1-2-0S45: (HID) Fixed a bug where the End flag sometimes was not set when the HID Shell was used for automated touch operations.

Fixed a bug where the nn::hid::TouchAttribute::End flag sometimes was not set when HidShell was used for automated touch operations and the touch on the screen is finally released.

1-2-0S46: (LDN) Fixed a bug that sometimes caused the nn::ldn::Scan() function to generate FATAL 2162-0002.

Fixed a bug where on rare occasions the nn::ldn::Scan() function would generate FATAL 2162-0002.

1-2-0S47: (LDN) Fixed a bug where local communication stopped working correctly after Internet connection settings with an incorrect security key were registered.

If Internet settings with the incorrect security key were registered, nn::ldn::Connect() would return nn::ldn::ResultConnectionFailed and fail.

1-2-0S48: (LDN) Fixed a bug where the ldn library was slow to change states after In-Fight mode was selected.

The event obtained by the nn::ldn::AttachStateChangeEvent() function is signaled when In-Fight mode is selected during local communication. If nn::ldn::GetState() was called directly after this event, the function is supposed to return nn::ldn::State_Error, but in some cases it would return nn::ldn::State_AccessPointCreated or nn::ldn::State_StationConnected because the ldn library was slow to change states due to the bug.

1-2-0S49: (MOVIE) Fixed a bug where decoding would stop when multiple movies were playing simultaneously.

Fixed a bug where decoding would stop after a while if multiple instances of movie::Decoder were created and multiple clips were decoded at the same time.

1-2-0S50: (NFC) Fixed a bug where sometimes a screen transition would halt for around three seconds if the screen transition occurred while the nn::nfp::StartDetection() function was running.

Fixed a bug where sometimes a screen transition would halt for around three seconds if the screen transition occurred while the nn::nfp::StartDetection() function was running.

*1-2-0W01: (LayoutTool) Specifying target groups is now required when using animation segment tags.

Specifying target groups is now required when using animation segment tags. Also made the following changes in accordance with this change.

- If groups are not specified, a warning icon is now displayed in the Edit Animation Segment Tags window in LayoutEditor.
