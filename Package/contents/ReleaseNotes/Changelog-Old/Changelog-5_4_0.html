Details

Changes in NintendoSDK NX Addon 5.4.0 (based on Generic 5.4.0)

Specification change that breaks your code

*5-4-0S01: (TOOL) Made changes so that an error now occurs if 1 is set in CardSpec/Size of the NMETA file when running AuthoringTool.

Made changes so that an error now occurs if 1 is set for CardSpec/Size of the NMETA file when running AuthoringTool.
For more information about values that can be set for CardSpec/Size, see NintendoSDK Documents > Features > NX Development Card.

Added Features

*5-4-0S02: (FS) Added file data cache feature.

Added a file data cache feature to automatically cache file data accessed by the application in the fs library.
For more information, see NintendoSDK Documents > Features > Basic Features > fs Library > Manual > Features > File Data Cache.

*5-4-0S03: (VSI) New Features in VSI and Target Manager

  • Target Manager
    • Added a feature to collect usage statistics data.
      • A feature to collect usage statistics data was added to improve the NintendoSDK. Thank you for your cooperation.

⁻ Other

    • Added a library for analyzing NXDMP files.
      • For more information, see CDR Library.


5-4-0S04: (DevMenu) Added support for indexed cache storage.

Support for indexed cache storage has been added. Index review and restore can now be performed, which were not possible in versions earlier than NX Add-On 5.3.0.

5-4-0S05: (FS) Added support in the access log for when the file data cache is enabled in FsAccessLogChecker version 16.0.

Added support in the access log for when the file data cache is enabled in FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 16.0. For more information, see NintendoSDK Documents > Features > Basic Features > fs Library > Manual > Features > File Data Cache.

5-4-0S06: (VSI) New Features in VSI and Target Manager

  • Target Manager
    • Added a feature for displaying log metadata as a Log View column.
    • Added a feature for changing some options using arguments at startup.
      • Send options for usage statistics data can be changed by providing arguments at startup.


Improvements

*5-4-0S07: (RO) Improved the speed of the nn::ro::UnloadModule function.

It was established that processing time for the nn::ro::UnloadModule function was longer than expected. This has been improved.

5-4-0S08: (DevMenu) Added a feature for checking compatibility with applications when installing patches.

When there is an application installed on the development hardware, and there is an attempt to install a patch for that application, there is now a check for consistency between the application and the patch.

5-4-0S09: (TOOL) Added a feature in the AuthoringTool for checking setting values of indexed cached storage.

Added a check item for cases where the --error-unpublishable option is specified when running the AuthoringTool.
A warning is now displayed when non-default values are set for the following items in the NMETA file.

  • CacheStorageDataAndJournalSizeMax
  • CacheStorageIndexMax

For more information, see NintendoSDK Documents > Features > Building > Creating Application Images > Write the NMETA Files.

5-4-0S10: (TOOL) Added the --error-unpublishable option to the bundleup command in the AuthoringTool.

Added the --error-unpublishable option to the bundleup command in the AuthoringTool.
For more information, see NintendoSDK Documents > Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

5-4-0S11: (TOOL) Improved stability of the feature for saving information related to running setup tools.

Improved exclusion for operations of the feature for saving information related to running the various setup tools (InitializeSdev, InitializeEdev, SystemUpdateSdev, SystemUpdateSdev).

Bug Fixes

*5-4-0S12: (TOOL) Fixed a bug where previous information was retained when ratings were changed in AuthoringEditor.

Fixed a bug where settings were rolled back when opening a file if ratings were changed while the NMETA or NSP file of an application was being edited in AuthoringEditor.
As a result, ratings displayed in the AuthoringEditor UI are now correctly reflected.

*5-4-0S13: (VSI) Fixed a bug where files directly under the drive would be deleted when uninstalling VSI 2015.

A bug existed in specific versions (4.3.4 through 5.2.2.22477_Hotfix1) where files directly under the drive would be deleted when uninstalling VSI 2015. This has been fixed.
In addition, even when installing a fixed version, the older VSI 2015 is uninstalled, which causes this issue to occur. When this issue occurs, a warning dialog box appears which pauses the installation process, so perform the appropriate processing.
The fix for this bug is the same as the fix for 5.2.2+REV22884+Hotfix2.1 that was released on 2018-2-28.

5-4-0S14: (FS) Fixed a bug where FsAccessLogChecker version 16.1 sometimes caused a crash.

Fixed a bug where overlapping of read accesses for multiple patterns in FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) sometimes caused a crash. The bug was fixed in FsAccessLogChecker version 16.1.

5-4-0S15: (MEM) Fixed a bug where GetTotalFreeSize() and GetAllocatableSize() returned a size smaller than the actual value when the virtual address memory management feature was enabled in nn::mem::StandardAllocator.

It is possible to use the virtual address memory management feature by specifying NULL to the addr parameter when Initialize() is called for nn::mem::StandardAllocator. However, when allocation or release was performed with this specific pattern, there was a bug where the value returned by GetTotalFreeSize(), and GetAllocatableSize(), was substantially smaller (by a few tens or hundreds of megabytes) than the actual size. This has been fixed.

5-4-0S16: (MOVIE) Fixed a deadlock that occurred when a vector had to be assigned during push_back that another thread was retrieving.

When trying to get an index that has already been released by another internal thread, if an internal vector has to be assigned during push_back, a deadlock no longer occurs when calling ReleaseOutputBufferIndex().

5-4-0S17: (NVN) Revisions to nvn.

The following fixes were made to nvn.

  • Fixed a bug where calling the bootstrap loader after devtools entry points were already initialized, would have disabled those entry points. nvnBootstrapLoader is modified such that calling with a NULL device skips initialization of the global state if the global state has already been initialized. In cases where initialization is skipped, the base nvn entry points are still returned.


These changes were included in NX Add-On 5.2.0, but omitted from the revision history for that version.

5-4-0S18: (VSI) VSI and TargetManager Bug Fixes

  • VSI
    • Fixed a bug where the NATVIS file was not loaded correctly.
      • Fixed an issue where Result and the like were not parsed correctly with VSI.
    • Fixed a bug where VSI would crash if debugging was restarted during NXDMP file debugging.
    • Fixed a bug where source code was not displayed correctly when NSS files were not automatically loaded.
    • Fixed a bug where classes that include bool and long int would not always parse correctly with NATVIS.
  • Target Manager
    • Fixed a bug where Target Manager would crash if relative paths were specified when creating dump files with the ControlTarget and TM libraries.
    • Fixed a bug where memory usage of Target Manager would increase drastically when structures with specific configurations were displayed using VSI.
    • Fixed a bug where Target Manager would sometimes freeze when the LTO beta version was enabled.
    • Fixed a bug where Target Manager would crash when htcs::Send/Recv was called multiple times in a short period of time.
    • Fixed other minor bugs.


Changes in NintendoWare NX Addon 5.4.0 (based on Generic 5.4.0)

Specification change that breaks your code

*5-4-0W01: (FONT) Increased the maximum number of fonts that can be combined in a single font-face for multiple scalable fonts.

The maximum number of fonts that can be combined in a single font-face, for multiple scalable fonts, was increased from 8 to 16.

With this change, the size of nn::font::TextureCache::InitializeArg will increase from 45 KB to 90 KB, so a stack overflow may occur if it is allocated in the stack. In such cases, make changes so that it is allocated in the heap.

Bug Fixes

5-4-0W02: (3DTool) Fixed a bug where meshes that include normal vectors oriented in incorrect directions, were sometimes output in the polygon reduction feature of the intermediate file optimizer.

There was a bug where meshes that include normal vectors oriented in incorrect directions, were sometimes output when the bone value associated with shapes fulfilled the following conditions in the polygon reduction feature of the intermediate file optimizer (%NINTENDO_SDK_ROOT%/Tools/Graphics/3dTools/3dIntermediateFileOptimizer.exe).

  • An anisotropic value is set to the scale.
  • A value other than 0 is set to the rotation.


5-4-0W03: (LayoutTool) Fixed a bug where the standard value of the window pane's window size was not set correctly in LayoutEditor.

Fixed a bug where the standard value of the window size was not set correctly when the window pane type was horizontal in division mode.

5-4-0W04: (Effect-VFX) Fixed a bug where particle shapes would be corrupt when the Speed Scale Y feature was used at low frame rates.

Fixed a bug where particle shapes would be corrupt when the Speed Scale Y feature was used at low frame rates, and in CPU emitters (similar to when using custom actions).
