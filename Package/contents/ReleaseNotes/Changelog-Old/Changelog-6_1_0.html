Details

Changes in NintendoSDK NX Addon 6.1.0 (based on Generic 6.1.0)

Specification change that breaks your code

*6-1-0S01: (BuildSystem) Changed the default linker to LLD when building applications for the NX development hardware.

The default linker has been changed to LLD when building applications for the NX development hardware in VSI. If you want to use the same linker as before this change, explicitly specify the linker to use (Link.LinkerType metadata) in the Visual C++ project files.
This change is only valid for builds using the NintendoSDK property sheet.

*6-1-0S02: (Compiler) Changed the default linker of Clang for NX to LLD.

As a result of the update of Clang for NX to version 1.5.4, the default linker is now LLD. When using Clang for NX without specifying the -fuse-ld command-line option, the LLD linker is used.

*6-1-0S03: (CRYPTO) Changes to GCM mode class members.

Changes were made to the following GCM mode class members in the nn::crypto namespace.

  • nn::crypto::Aes128GcmDecryptor
  • nn::crypto::Aes128GcmEncryptor
  • nn::crypto::GcmDecryptor
  • nn::crypto::GcmEncryptor


*6-1-0S04: (FS) Enhanced the checking of the mount name.

Regarding the mount name to specify, the following conditions are now checked when mounting a file system for use.

  • The name is not a single alphabetical character.
  • The name is not UTF-8 encoded.


For more information about the conditions for valid mount names, see the API reference.

*6-1-0S05: (Album) Deleted features for accessing album files.

Deleted the following functions for accessing newly-added album files in NX Add-On 6.0.0. Contact Nintendo if you need to use these functions.

  • nn::album::GetAlbumFileCreatedTime()
  • nn::album::GetAlbumScreenshotFileList()
  • nn::album::GetAlbumMovieFileList()
  • nn::album::LoadAlbumImage()
  • nn::album::LoadAlbumThumbnailImage()
  • nn::album::OpenAlbumMovieStream()
  • nn::album::CloseAlbumMovieStream()
  • nn::album::GetAlbumMovieStreamSize()
  • nn::album::ReadAlbumMovieStream()
  • nn::album::DeleteAlbumFileForDebug()

Added an option for changing the behavior of the played sample count when playing a looped waveform using a voice of the audio renderer.
If this option is enabled, the played sample count, which can be obtained using the nn::audio::GetVoicePlayedSampleCount() function when looping, is reset.

* nn::audio::AcquireVoiceSlot()
* nn::audio::VoiceType::BehaviorOptions

Added PolyphaseResampler, which under certain conditions operates faster than the existing Resampler, to nn::audio. For more information, see the API reference.

Added a feature for getting the region setting in DevMenu and DevMenuCommand.
This feature can be used in DevMenu from the Device page, and in DevMenuCommand with the settings commands.
For more information, see Features > Controlling the Development Hardware > DevMenu > Changing Device Settings and Features > Controlling the Development Hardware > DevMenuCommand.

6-1-0S11: (FS) Added a function to the multi-commit feature for simultaneously committing multiple valid file systems.

The nn::fs::Commit(const char* const* <em>nameArray</em>, int <em>nameCount</em>) function was added to the multi-commit feature to simultaneously commit updated content for multiple valid file systems.
NX Add-On 6.1.0 does not ensure file system content when the program stops during multi-commit execution. Internally, the file systems are committed individually.
In addition, NX Add-On 6.1.0 does not support access log output or access frequency checks.
We plan to complete the internal processing, and support access log output and access frequency checks in NX Add-On 6.2.0 and later.

6-1-0S12: (FS) Added support for the access log of FsAccessLogChecker version 17.0 when individual file data caching is enabled.

Added support for checking nn::fs::EnableIndividualFileDataCache() as a read operation in FsAccessLogChecker (<em>%NINTENDO_SDK_ROOT%</em>\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 17.0. For more information, see fs Library > Debugging > Checking the Frequency of Access.

6-1-0S13: (GFX) Added a flag to the image format properties to indicate whether the format can be used for images.

Added the flag nn::gfx::ImageFormatPropertyFlag_Image to nn::gfx::ImageFormatPropertyFlag to indicate whether the image format can be used for images.
You can now use nn::gfx::ImageFormatProperty, obtained with nn::gfx::GetImageFormatProperty, to determine whether a given image format can be used for images.

6-1-0S14: (MEM) Added a function for cleaning up the nn::mem::StandardAllocator management space at any time.

The nn::mem::StandardAllocator management space dynamically increases and decreases in size. Previously, StandardAllocator determined when to clean up this management space, but now the function nn::mem::StandardAllocator::CleanUpManagementArea(), which the user can call at any time, has been added.

Effective from NintendoSDK 7.0.0, scheduled for future release, the input option --glsl-version will be required for the 3D shader converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) when converting shader definition files (FSDB) into shader archive files (BFSHA).

If the --glsl-version option is not specified, a conversion error occurs. Use this option to specify the GLSL version, rather than write it in the shader code.

6-1-0W24: (3D-G3D) Revised the G3dDemo sample program, changing the location of the parent material referenced by MaleA.fmdb and FemaleA.fmdb to the material intermediate file.

Revised the G3dDemo sample program, changing the location of the parent material referenced by MaleA.fmdb and FemaleA.fmdb to the material intermediate file (FMTB), instead of the model intermediate file (FMDB).

6-1-0W25: (3DTool) Added a feature to 3DEditor for forcibly enabling and disabling the automatic output of an optimized shader when editing materials.

Added a feature to 3DEditor for forcibly enabling and disabling the automatic regeneration of an optimized shader when editing materials. Enable and disable by using the Automatic Shader Regeneration settings item in the toolbar.
By using this feature to temporarily disable automatic shader regeneration, you can eliminate the time spent for unnecessary shader generation when batch editing shader parameters.

6-1-0W26: (3DTool) Added an environment variable that can be used for the NTCP file attribute by the 3D texture preset converter.

Added %INPUT_FILE_PATH% (the absolute path of the input file) to the environment variables that can be used by the NTCP file attribute in the 3D texture preset converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTexturePresetConverter.bat).

6-1-0W27: (3DTool) Added the Group settings item to the Parent Material page of the material Property window in 3DEditor.

Added the Group settings item to the Parent Material page of the material Property window in 3DEditor.

6-1-0W28: (3DTool) An order of priority is now set in 3DEditor for reference paths when there are multiple reference materials or textures with the same name.

An order of priority is now set in 3DEditor for reference paths when there are multiple reference materials or textures with the same name.
When reference paths are resolved, if multiple files with the same name exist in subfolders, the file paths will be sorted alphabetically (not case sensitive), and the files higher in the sort will take precedence.

6-1-0W29: (3DTool) Changed how the Normal Map Filter plug-in for Photoshop processes transparent (semi-transparent) pixels when height maps are converted to normal maps.

When the Normal Map Filter plug-in converts height maps to normal maps, it now blends the pixel opacity of transparent (semi-transparent) pixels with the gray background (RGB = 128) before performing the conversion.
By rendering depressions in black and elevations in white, the transparent layer can be used as a height map.

6-1-0W30: (3DTool) Changed the Normal Map Filter plug-in for Photoshop so that now, when the normal map is merged with the normal map of another file, the impact is lessened by making the pixels of the other file almost transparent.

Changed the Normal Map Filter plug-in for Photoshop so that now, when the normal map is merged with the normal map of another file, the impact is lessened by making the pixels of the other file almost transparent.

6-1-0W31: (3DTool) Changed where the Thumbnail Explorer Extension is installed.

The Thumbnail Explorer Extension is now installed to %ProgramFiles%\Nintendo\ThumbnailExplorerExtensions. Previously, it was installed to %USERPROFILE%/AppData/Local/Nintendo/ThumbnailExplorerExtensions, meaning that multiple users could not use the Thumbnail Explorer Extension on the same PC.

6-1-0W32: (3DTool) If the option uniform block is not being used, the 3D shader converter now sets the size of the option uniform block to 0 in the shader binary (BFSHA).

The 3D shader converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) now sets nn::g3d::ResUniformBlockVarData::size (corresponding to the option uniform block) to 0 if the option uniform block is not being used. In the runtime, you can get this value by using the nn::g3d::ResUniformBlockVar::GetSize() function. Example cases where the size will be 0 are presented as follows.

• All static shader option branch values are false
• The --force-variation option is specified

With this change, the nn::g3d::ShadingModelObj::CalculateBlockBufferSize() function will calculate a size of 0 if the shader does not use the option uniform block, resulting in nn::g3d::ShadingModelObj no longer consuming GPU memory.

6-1-0W33: (3DTool) Improved the color space converted precision of the 3DEditor color picker.

The conversion between sRGB and Linear by the 3DEditor color picker has been improved to be more accurate, and is no longer simple conversion using a gamma value of 2.2.

6-1-0W34: (3DTool) Improved the compression quality of the A component in the bc3, bc7, and astc formats in 3D Texture Converter.

Improved the compression quality of the A component in the bc3, bc7, and astc formats in 3D Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe).
There is now less noise when the A component is 255 (1.0 as a real-number type).

6-1-0W35: (3DTool) Improved the compression quality of the RGB components of the bc1, bc2, and bc3 formats in 3D Texture Converter.

Improved the compression quality of the RGB components of the bc1, bc2, and bc3 formats in 3D Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe).

6-1-0W36: (3DTool) Textures with unresolvable reference paths are now set to the sampler for previewing in 3DEditor.

Textures with unresolvable reference paths are now set to the sampler for previewing in 3DEditor.
Only used for previewing with 3DEditor, and the edited content is not saved to an intermediate file.

6-1-0W37: (3DTool) The 3D shader converter and the 3D binary converter now support the displaying of Japanese error messages.

When conversion errors occur in the 3D shader converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) and the 3D binary converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe), the error messages can now be output in either the English or Japanese language, depending on the language set in the operating system.

6-1-0W38: (3DTool) The 3D shader converter now ignores the choice value of the option variable for uniform variable branching written in the shader variation file (FSVA).

When the 3D shader converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) creates the shader archive file (BFSHA), it now ignores the choice value of the option variable for uniform variable branching written in the shader variation file (FSVA).
Previously, the choice value of the option variable for uniform variable branching written in the shader variation file needed to include the default values defined in the shader definition files (FSDB and FSDA).

6-1-0W39: (3DTool) The GLSL version used by 3DEditor can now be specified in the team settings.

The GLSL version used by 3DEditor can now be specified in the team settings.
To specify the version, specify the --glsl-version <version> option in the Shader Converter Additional Arguments area of the team settings dialog box.
If nothing is specified, the version is set automatically by checking which version can be used in the current environment. However, specify the option because effective from NintendoSDK 7.0.0, it must be explicitly specified. For more information about the option, see 3D Shader Converter.

6-1-0W40: (3DTool) You can now import the material settings when multiple textures with the same name are opened in 3DEditor.

You can now import the material settings when multiple textures with the same name are opened in 3DEditor.
If a texture with the same name as the imported texture is already resolved in the reference path, that referenced texture is overwritten.

6-1-0W41: (LayoutTool) Added support in LayoutEditor for dragging texture properties.

Added a feature for dragging textures from the Texture Manager window, and dropping them on texture property thumbnails.
To switch the order, you can also drag thumbnails onto each other.

6-1-0W42: (LayoutTool) Added support in LayoutEditor for the simultaneous editing of multiple panes.

The following properties can be edited simultaneously in multiple panes.
Base panes, masks, drop shadows, fragment operations, and textures.

6-1-0W43: (LayoutTool) Changed the default position of null panes created using the G key in LayoutEditor.

The null pane is now created in the center of the parent pane if the selected panes have the same parent pane.
If the parent panes are different, the null pane is created at the origin of the layout.

6-1-0W44: (LayoutTool) Changed the display of texture thumbnails in the Property windows of LayoutEditor.

Thumbnail images displayed in the texture Property window for Picture and Window panes, now include the alpha channel.
