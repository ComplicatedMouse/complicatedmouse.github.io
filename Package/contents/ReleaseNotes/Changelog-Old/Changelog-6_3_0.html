Details

Changes in NintendoSDK NX Addon 6.3.0 (based on Generic 6.3.0)

Specification change that breaks your code

*6-3-0S01: (OS) Deleted deprecated functions in the OS API.

Deleted the following deprecated functions in the OS API. From now on, use the substitute functions that have been prepared to replace these deprecated functions.

  • os::GetCurrentStackInfoForDebug() (Use os::GetCurrentStackInfo() instead.)
  • os::QueryMemoryInfoForDebug() (Use os::QueryMemoryInfo() instead.)


*6-3-0S02: (TOOL) Shader Converter now returns an error when file output fails.

Previously, when the output of optional files (such as debugging information, dump information, and shader cache files) failed in Shader Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderConverter.exe), no error or other indication was returned. Now, an error is appropriately returned.

Added Features

*6-3-0S03: (APPLET) Added an inline software keyboard that can be displayed directly within applications.

Added an inline software keyboard (nn::swkbd::InlineKeyboard) that allows the application to continue executing other processes even while the keyboard is displayed.

*6-3-0S04: (AUDIO) Added a feature for audio input.

Added an audio input feature for recording audio data from the device.

We recommend using the Nintendo Switch Online application voice chat feature, provided by the NEX library, to implement voice chat features. For more information, see the voice chat guide for Nintendo Switch Online.
Use this feature if you need to implement voice chat on the Nintendo Switch system itself.

*6-3-0S05: (EC) Feature for providing consumable service items without an independent server

In the ec library, added a function that makes it possible to use consumable service items by communicating directly with the eShop server instead of having to set up an independent server.

*6-3-0S06: (Accounts) Added a feature for restricting which user accounts can be opened.

In NX Add-On 6.2.0 and later, there are states where restrictions on which user accounts can be opened get applied. Added this feature to facilitate debugging these states.
For more information, see Open User Accounts Restriction Feature (NX Add-On 6.3.0 and Later) in Controlling the Development Hardware > DevMenuCommand and Account Restricted Mode in Controlling the Development Hardware > DevMenu > Development Assistance Settings.
For more information about restricting which user accounts can be opened, see System > account Library > User Accounts.

6-3-0S07: (AUDIO) Added functions for clearing the output destinations for VoiceType, SplitterType, and SubMixType objects.

Added functions for clearing the output destinations for VoiceType, SplitterType, and SubMixType objects to the nn::audio namespace.

* nn::audio::ClearVoiceDestination()
* nn::audio::ClearSplitterDestination()
* nn::audio::ClearSubMixDestination()

6-3-0S08: (DevMenu) Added a feature for starting an application with a specified user account selected.

Added the --user-index option for use with the application launch command in DevMenuCommand.
Specifying the --user-index makes it possible to start an application with the specified user account selected.
For more information, see Features > Controlling the Development Hardware > DevMenuCommand.

6-3-0S09: (EC) Added a feature for displaying a screen for entering redemption codes.

Added the nn::ec::ShowShopEnterCodeScene() function for displaying a redemption code entry screen.

6-3-0S10: (ERR) Added an API for handling errors returned under certain conditions by NEX and other middleware.

Added an API for handling errors returned under certain conditions by NEX and other middleware.

6-3-0S11: (FS) Added support for displaying analysis progress in FsAccessLogChecker 20.2.

Added support for displaying analysis progress in FsAccessLogChecker 20.2 (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe). To disable the progress display, use the --disable_progress option.

6-3-0S12: (FS) Added support for multicommit access log output in addition to support for checking these logs in FsAccessLogChecker 20.1.

Added support for access log output in multicommit operations performed using nn::fs::Commit(const char* const* <em>nameArray</em>, int <em>nameCount</em>) in addition to support for checking these logs using FsAccessLogChecker 20.1 (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe). For more information, see fs Library > Debugging > Checking the Frequency of Access.

6-3-0S13: (GFX) Added the GetRowPitch function for getting API-dependent information about low-level graphics.

Added overloads of the nn::gfx::Texture::GetRowPitch function for getting API-dependent information about low-level graphics.

6-3-0S14: (GFX) Improved the Imgui supplementary library.

Added a library that is compatible with nns::gfx::GraphicFramework.
Also added the ImguiSimple sample.

6-3-0S15: (Graphics) Added documentation for RenderDoc For NX.

Added documentation for the RenderDoc For NX package for working with Vulkan Standard Validation Layers and RenderDoc resources in the NX environment.
For more information, see RenderDoc For NX.

6-3-0S16: (HID) Added a feature for generating binary vibration files from arrays of vibration values.

Added a file output feature to hid_VibrationFileApi.h.

Added a feature for generating binary vibration files from arrays of vibration values in addition to the existing feature for extracting vibration values from binary vibration files.

This feature can be used, for example, to save vibrations generated during in-game sequences to file for easy playback when viewing replays at a later time.

For more information, see the API Reference.

* nn::hid::VibrationValueArrayInfo
* nn::hid::GenerateVibrationFile()
* nn::hid::CalculateVibrationFileSize()

6-3-0S17: (HID) Added a function for checking for firmware updates for improving the performance of the six-axis sensor.

Added the nn::hid::IsFirmwareUpdateAvailableForSixAxisSensor function. This function checks whether any firmware updates for improving the performance of the six-axis sensor are available.


6-3-0S18: (HID) Added support for using DebugPad in the gamepad input transfer feature in Input Director.

Input from a USB gamepad connected to the host PC can now be transferred to the development hardware as DebugPad input. For more information, see the manual.

6-3-0S19: (MEM) Added a feature to nn::mem::NumberLineAllocator for selecting allocation/deallocation modes.

Added a feature for selecting between a mode for prioritizing execution efficiency and a mode for prioritizing minimizing fragmentation when allocating and deallocating memory using nn::mem::NumberLineAllocator.
Use the nn::mem::NumberLineAllocator::SetAllocationMode() and nn::mem::NumberLineAllocator::GetAllocationMode() functions to set and get the allocation/deallocation mode.

6-3-0S20: (NVN) New NVN Features

Added the following features to NVN.

  • Added functions to allow setting and getting the active framebuffer count at run time.
    • Added the nvnWindowBuilderSetNumActiveTextures and nvnWindowBuilderGetNumActiveTextures functions.
    • Added the nvnWindowSetNumActiveTextures, nvnWindowGetNumActiveTextures, and nvnWindowGetNumTextures functions.


Added the following features to GLSLC.

  • Added a function for setting debug labels for output GLSL semantic errors/warning messages via the GLSLCoptions::<em>debugLabels</em> parameter.

  • Added the GLSLCincludeInfo::<em>useIncludePathsFirst</em> option to allow search paths to be searched before the absolute file path inside the #include statements within GLSL code. This option is useful for specifying any mount names needed for loading up files on NT without operating system exceptions.
