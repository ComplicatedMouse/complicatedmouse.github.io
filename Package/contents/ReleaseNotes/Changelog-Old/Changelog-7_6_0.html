Summary

Changes in NintendoSDK NX Addon 7.6.0 (based on Generic 7.6.0)

Added Features

  • *7-6-0S01: (OE) Added a CPU boost feature that temporarily increases CPU processing capabilities at the cost of lowering GPU processing capabilities.
  • 7-6-0S02: (EC) Added a sample to get information about other titles related to the EcCatalog samples.
  • 7-6-0S03: (EC) Added an EcCatalog sample that shows how to get a list of downloadable items.
  • 7-6-0S04: (TOOL) Added a feature to AuthoringTool to create NSP files for secure startup.

Improvements

  • *7-6-0S05: (Compiler) Updated to Clang for NX 1.7.9.
  • 7-6-0S06: (CPU Profiler) Nintendo NX CPU Profiler Version 1.0
  • 7-6-0S07: (EC) Added a feature to display the discount price and discount period in the Consumable item list in the EcCatalog sample.

Bug Fixes

  • 7-6-0S08: (FS) Fixed a bug where the first access log was not generated.
  • 7-6-0S09: (MOVIE) Fixed a bug where a frame rate of 0 was returned for some WebM videos.
  • 7-6-0S10: (MOVIE) Fixed a bug where the last frame or more of VP8 and VP9 videos was not decoded.
  • 7-6-0S11: (NIFM) Fixed a bug where you could not connect to an AP using WEP on Windows.
  • 7-6-0S12: (TOOL) Fixed a bug where the check of the NSP file to determine whether the master ROM could be submitted used a size limit for demo software that was different from the size stipulated in the guidelines.

Details

Changes in NintendoSDK NX Addon 7.6.0 (based on Generic 7.6.0)

Added Features

*7-6-0S01: (OE) Added a CPU boost feature that temporarily increases CPU processing capabilities at the cost of lowering GPU processing capabilities.

Added a CPU boost feature that temporarily increases CPU processing capabilities at the cost of lowering GPU processing capabilities. For more information, see CPU Boost Functionality.

7-6-0S02: (EC) Added a sample to get information about other titles related to the EcCatalog samples.

Added the Related item list option to the EcCatalog start screen and added a sample to get and display information about other related titles.

7-6-0S03: (EC) Added an EcCatalog sample that shows how to get a list of downloadable items.

Added the AOC item list option to the start screen of EcCatalog and added a sample to get and display the downloadable items list.

7-6-0S04: (TOOL) Added a feature to AuthoringTool to create NSP files for secure startup.

A feature has been added to AuthoringTool for creating NSP files for secure startup. For more information, see the makesecure command in Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

To use secure startup NSP files, you must initialize the development hardware with NintendoSDK SystemUpdater for NX 8.1.0+Build1.0 or later.

Improvements

*7-6-0S05: (Compiler) Updated to Clang for NX 1.7.9.

Updated Clang for NX to version 1.7.9.
The main changes are as follows.

  • Fixed a bug where the fopen() function did not work correctly in a multithreaded environment.
  • Fixed a bug where SafeStack was dependent on the alignment of memory returned by calloc.


7-6-0S06: (CPU Profiler) Nintendo NX CPU Profiler Version 1.0

  • Improved loading times.
  • Improved drawing times of the Sample Graph.
  • Added a new Checker.
  • Added impact bars to the Partial Call Tree.
  • Added a keyboard shortcut to the corresponding button's tooltip.
  • Added keyboard shortcuts for Start and Stop profiling.
  • Changed Selected Only to automatically uncheck if no functions are selected.
  • Reordered the Load Graph to show systems with less variability at the bottom.
  • Removed an unused DLL file from the package.
  • Fixed a crash that occurred on the Call Tree if many nodes were expanded and the tree was quickly scrolled.
  • Fixed Total/Self/Sub on the Partial Call Tree for recursive functions.
  • Fixed a bug that allowed the Partial Call Tree to be taller than the window.
  • Fixed loading errors due to unbalanced NVN debug group pushes and pops.
  • Fixed a loading error that occurred if nn::profiler::InstallGraphicsHooks() was called after NVN functions were loaded.
  • Fixed excessive slowdown in the Sample Graph when moving the mouse.
  • Fixed a bug that could cause the Sample Graph tooltip to show details for a thread state group for an extra microsecond.
  • Fixed a crash that could occur when Deselect All was used for code blocks quickly after selecting code blocks.
  • Fixed high memory usage and slowdown when a heartbeat has a quick period.
  • Fixed the Unable to Effectively Profile Thread checker to not trigger for the thread NxSystemThread.
  • Fixed a bug that could cause the system idle thread to trigger some checkers.
  • Fixed a bug that could cause thread state tracing to cause checker failures.
  • Fixed checker failures if thread names failed to transfer.
  • Fixed a bug causing the Instrumented function to not clear properly.
  • Fixed a bug that could cause the Stop button to disappear on the Instrumented Profile tab.
  • Fixed a bug that showed GPU Trace on the Instrumented Profile tab when profiling was not available.
  • Fixed a bug where taking a GPU trace profile and then attempting an out-of-process profile would result in data parsing failures.
  • Fixed a crash when clicking and pressing SHIFT on the Functions tab after filtering.
  • Fixed a crash when clicking and pressing SHIFT on the Threads or Counters tab without first clicking somewhere in the tab.
  • Fixed a bug that could cause parts of the Overview tab to disappear.
  • Fixed some issues with 32-bit ARM disassembly.
  • Fixed a bug where 64-bit ARM disassembly would not show data sections, causing instructions after the section to show at the wrong address.


7-6-0S07: (EC) Added a feature to display the discount price and discount period in the Consumable item list in the EcCatalog sample.

Added a feature to display the discount price and discount period on the item details page of the Consumable item list in the EcCatalog sample.

Bug Fixes

7-6-0S08: (FS) Fixed a bug where the first access log was not generated.

Fixed a bug where overlapping initial fs access operations only generated a log for one of the access operations.

7-6-0S09: (MOVIE) Fixed a bug where a frame rate of 0 was returned for some WebM videos.

Fixed a bug where a frame rate of 0 was returned for some WebM videos.

7-6-0S10: (MOVIE) Fixed a bug where the last frame or more of VP8 and VP9 videos was not decoded.

Fixed a bug where during playback of VP8 and VP9 videos, the final frame or more of the video files were not decoded even if movie::Decoder had already returned an EOF.

7-6-0S11: (NIFM) Fixed a bug where you could not connect to an AP using WEP on Windows.

Fixed a bug where attempting to connect to an access point using WEP encryption from Windows resulted in error 2110-1100.

7-6-0S12: (TOOL) Fixed a bug where the check of the NSP file to determine whether the master ROM could be submitted used a size limit for demo software that was different from the size stipulated in the guidelines.

When AuthoringTool ran the get-unpublishable-error command on an NSP file, it checked demo software using a size limit of 1 GiB. That has been corrected to use the size limit of 2 GiB, as stipulated in the guidelines.

Changes in NintendoWare NX Addon 7.6.0 (based on Generic 7.6.0)

Bug Fixes

7-6-0W01: (3DTool) Fixed a bug in 3DShaderConverter where an error occurred when converting a shader that included vertex attributes without the layer qualifier.

Fixed a bug in 3DShaderConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) where an error occurred when converting a shader that included vertex attributes without the layer qualifier.
With this modification, conversion succeeds even if the vertex attribute does not have a layout qualifier, but switching between shader variations may change the location of the vertex attributes, which may lead to performance degradation when re-creating the vertex state and other data. For this reason, we recommend using the layout qualifier to fix the vertex attributes in g3d.
For more information, see Shader Resource Workflow > 1. Creating the Shader Source > Vertex Attributes.
