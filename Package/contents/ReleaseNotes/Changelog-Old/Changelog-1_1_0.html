*1-1-0S01: (FS) Added a restriction on the amount of downloadable content that can be mounted simultaneously.

The amount of downloadable content that can be mounted simultaneously has been restricted to maximum 128 sets of content.

*1-1-0S02: (IRSENSOR) Fixed a bug in applications that use multiple IR Motion Cameras, where if you transition to an applet and then return to the application with controller connections that exceed the upper limit on the number of controllers, an abort would occur.

When using multiple IR Motion Cameras, for example when you have four controllers and two controllers are using IR Motion Cameras at the same time, if you transition to the menu or elsewhere and connect an additional controller and then return to the application with five controllers connected, even if the application conforms to the limit on the number of controllers, a bug would cause the application to abort.
Changed the specification to avoid this problem. When a data retrieval function is called when the limit on IR Motion Camera connections is violated, the Hid Resource Error is now output to the log in the library and the nn::irsensor::ResultIrsensorNotReady error is returned.
This error creates a state in which data is not received, the same as before, so there is no need to change the handling policy.
This fix avoids the abort, but if the upper limit on the number of controllers is not complied with when entering the same problematic sequence, only one IR Motion Camera will work after resuming the application. If you are using multiple controllers, consider the setting on the upper limit of controllers.
In the future, we plan to release a fix so that an error is not returned even when entering this problematic sequence.

Enabling the CEC feature and then connecting the system to a TV that supports CEC, makes it possible to use the following features.

  • Put the development hardware to sleep when the TV is turned off while the system is docked (system standby).
  • Turn the TV on and change to the correct channel when the development hardware is woken up from sleep mode or turned on while docked (one-touch play).
  • Display the NX product name in the list of channels (connected devices) displayed on the TV.


Currently, there are several known bugs that occur when CEC is enabled.
Make sure that you read Known bugs/NintendoSDK 1.1.0 before enabling this feature.

Also, note that enabling the CEC feature on the SDEV PreMP or earlier version sometimes negatively affects connectivity with the TV. For more information, see "On the SDEV PreMP or earlier, sometimes no image is displayed on the TV when the system is docked if the Controlling Power With the TV (CEC) feature is enabled" in Troubleshooting (NX Add-On).

*1-1-0W01: (3D-G3D) Support for Resource Data Created With NX Add-On 0.12.x

Shader archive files (BFSHA) created with NX Add-On 0.12.<em>x</em> are not compatible because the major version of the binary is different, but support has been added to handle them as resource data. This change means that shader archive files that were created with NX Add-On 1.0.0 are no longer supported. They must be reconverted in the new 3D shader converter. The following resource data is supported by the g3d runtime in this package.

  • Resource data created with NX Add-On 0.12.<em>x</em>
  • Resource data created with NX Add-On 1.1.0
