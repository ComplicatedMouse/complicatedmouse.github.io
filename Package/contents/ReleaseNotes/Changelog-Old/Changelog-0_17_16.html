Details

Changes in NintendoSDK NX Addon 0.12.16 (based on Generic 0.17.16)

Added Features

*13-17S01: (PCM) Added the pcm library for development use. The library includes features for measuring power consumption inside the development hardware.

Added the pcm library for development use. The library includes features for measuring power consumption inside the development hardware.
These features can be used to measure power consumption inside the development hardware while the application is running.
By checking how the values change from scene to scene, you can determine which places are relatively large drains on battery power.

Notes

  • The absolute values measured for voltage will vary due to individual hardware differences. Keeping this in mind, use the feature to determine the scenes that consume relatively large amounts of power.
  • Power consumption can only be measured on the SDEV.
  • The functions of this library cannot be used in the production environment.


For more information, see PCM Library in the NintendoSDK documentation.

This change was included in NX Add-On 0.12.15 but omitted from the revision history for that version.

13-17S02: (NVN) New NVN Features

Added the following features to NVN.

  • Added new device flag NVN_DEVICE_FLAG_DEFERRED_FIRMWARE_MEMORY_RECLAIM_BIT to control when firmware memory, used to store GPU page table entries, is freed

  • Added new API functions to obtain total queue memory usage from queue commands, from when the queue was last initialized or the usage counters were reset.
    • Added the functions nvnQueueGetTotalComputeMemoryUsed, nvnQueueGetTotalControlMemoryUsed, and nvnQueueGetTotalCommandMemoryUsed to get the total queue compute, control, and command memory used (in bytes).
    • Added function nvnQueueResetMemoryUsageCounts to reset the usage counters for tracking queue memory usage.


These changes were included in NX Add-On 0.12.10 but omitted from the revision history for that version.

Improvements

*13-17S03: (CODEC) Deprecated use of nn::codec::HardwareOpusDecoder.

A bug was found where putting the system into sleep mode and then docking it while using nn::codec::HardwareOpusDecoder would cause the system to stop. There is no method to avoid this issue on the application side, so it has been deprecated from use.

13-17S04: (FS) FsAccessLogChecker now checks whether to overwrite files.

Version 3.1 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) has been modified to check, when outputting the results to a file, whether to overwrite the file if it already exists.
In addition, added the -f,--force options to specify overwriting without checking.

13-17S05: (FS) Updated FsAccessLogChecker to version 3.2.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 3.2. This includes the following changes.

  • Support for confirming file overwrite.
  • Fixed the problem where invalid timestamp order would be incorrectly detected.


13-17S06: (General) Added NMETA files.

Added NMETA files to the sample resources.
Also, when generating applications using the Wizard in VSI, the NMETA files are now copied to the project.

13-17S07: (VSI) Improved the compatibility for NX Add-On 0.12 versions of NX VSI and Target Manager.

Installing VSI for NX 1.1.1 Hotfix2 removed the necessity of updating the VSI when switching between NX Add-On 1.1.x <-> NX Add-On 0.12.16.
While there are no problems with NX Add-On 0.12.16 and later versions, install the NX VSI included in each package when using NX Add-On 0.12.15 and earlier versions.

Bug Fixes

13-17S08: (FRIENDS) Fixed a bug where an unexpected result occurred when the network service account ID list to be passed to GetProfileList was destroyed immediately.

Fixed a bug where there was an unexpected result when the network service account ID list to be passed to GetProfileList was destroyed immediately.

13-17S09: (FS) Fixed a bug where FsAccessLogChecker incorrectly detected Invalid timestamp order.

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) incorrectly detected Invalid timestamp order and generated a parsing error. This bug was fixed in FsAccessLogChecker version 3.2.

13-17S10: (SWKBD) Fixed a bug where incorrect strings would be returned by TextCheckCallback on the NX32 platform.

Fixed a bug where incorrect strings would be returned by TextCheckCallback on the NX32 platform.

13-17S11: (TOOL) Enhanced the error checking conducted when add-on content is created.

Enhanced the error checking conducted by AuthoringTool when add-on content is being created. Errors will now be generated in the following cases.

  • The value of Index is not within the range of 1 to 2000.
  • Duplicate Index values have been specified in an NMETA file.


Changes in NintendoWare NX Addon 0.12.16 (based on Generic 0.17.16)

Added Features

13-17W01: (Effect-VFX) Added a masking check feature to some of the stripes in EffectMaker.

Expanded features so that the Masking Check feature now works for some stripes. The Masking Check can be specified in the same manner as particles in the Color tab of EffectMaker. If the starting point of the stripe is masked, the entire stripe will be subjected to alpha fade.
Applicable stripes are described below. No other stripes are supported.

  • Historical Stripe
  • Historical Stripe 2


13-17W02: (Effect-VFX) Added a z-axis rotation feature to some stripes in EffectMaker.

Expanded features so that the Z-Axis Rotation feature now works for some stripes. Z-Axis Rotation can be specified in the same manner as particle rotation in the Rotation tab of EffectMaker. The bands of the stripe will twist along the axis in which the strip advances.
Applicable stripes are those with the following specifications. Other combinations are not supported.

  • For Historical Stripe, the Calculation Method is set to Emitter Matrix Stripe or Extend up/down from emitter
  • For Historical Stripe2, the Calculation Method is set to Emitter Matrix Stripe or Extend up/down from emitter
