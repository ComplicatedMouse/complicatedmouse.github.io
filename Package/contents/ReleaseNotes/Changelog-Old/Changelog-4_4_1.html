Details

Changes in NintendoSDK NX Addon 4.4.1 (based on Generic 4.4.1)

Added Features

*4-4-1S01: (TOOL) Added the EDEV system update tool (SystemUpdateEdev).

Added the SystemUpdateEdev tool to perform EDEV system updates from a PC.

The advantage of this tool over the initial one (InitializeEdev) is that it updates the firmware version while keeping various settings.
For more information about using it, see the SystemUpdateEdev Command Description page.

Also refer to SystemUpdateSdev, the tool for updating SDEV systems.

4-4-1S02: (DevMenu) Added a feature for writing NSP files to a game card.

Added a feature for writing microSD card NSP files to a game card. Also see the Known Bugs page because there are known bugs where fatal errors occur.

4-4-1S03: (FS) Added support for adding nn::nfp access log output, and checks in FsAccessLogChecker version 11.0.

Added access log output in nn::nfp functions for FS access, and support for checking FS access via nn::nfp in FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 11.0. For more information, see fs Library > Debugging > Checking the Frequency of Access.

Bug Fixes

*4-4-1S04: (TOOL) Fixed a bug where the AuthoringTool failed patch generation.

Fixed a bug in the initial version application where patch generation failed for some data patterns.

4-4-1S05: (FS) Fixed overlapping judgment of read access in FsAccessLogChecker version 10.8.

Fixed a bug where overlapping of read access for multiple patterns in FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) was overlooked. The bug was fixed in FsAccessLogChecker version 10.8.
