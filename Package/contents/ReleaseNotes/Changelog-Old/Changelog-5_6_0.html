Details

Changes in NintendoSDK NX Addon 5.6.0 (based on Generic 5.6.0)

Added Features

*5-6-0S01: (AUDIO) Added a feature for audio input.

Added an audio input feature for recording audio data from the device.

We recommend using the Nintendo Switch Online application voice chat feature, provided by the NEX library, to implement voice chat features. For more information, see the voice chat guide for Nintendo Switch Online.
Use this feature if you need to implement voice chat on the Nintendo Switch system itself.

*5-6-0S02: (EC) Added a feature for providing consumable service items without an independent server.

In the ec library, added a function that makes it possible to use consumable service items by communicating directly with the eShop server instead of having to set up an independent server.

5-6-0S03: (ERR) Added an API for handling errors returned under some conditions by NEX and other middleware.

Added an API for handling errors returned under some conditions by NEX and other middleware.

Improvements

5-6-0S04: (DevMenu) Added the ability to create accounts when starting applications.

Added the ability to create accounts when starting applications. A Create button appears when user selection is needed before starting.
These changes were included in NX Add-On 5.5.0 but omitted from the revision history for that version.

5-6-0S05: (EC) Added a message to use when the response language is different from the language of the EcCatalog sample.

Revised to display a message when getting the item list in the EcCatalog sample if the response and request languages are different.
Although it is not necessary to offer similar steps in separate applications, it is implemented as one example of how to respond to this issue.

5-6-0S06: (EC) Added the EcCatalog sample to display value-added tax data.

Revised the EcCatalog sample to display value-added tax data when displaying the consumable item list.

In applications, when inc_tax_message is not null, you must include value-added tax data (inc_tax_message) with price data.

5-6-0S07: (FS) Updated FsAccessLogChecker to version 20.4.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 20.4. This version includes the following changes.

  • Added support for multi-commit access logs.
  • Added support for displaying the progress of analysis.
  • Organized the analysis results of access via nn::nfp.
  • Organized the SameAddressRead enumerators.


5-6-0S08: (TOOL) Added a feature for checking for use of a version of PiaFramework that cannot be used for master submission.

Added the --error-unpublishable option to AuthoringTool. When this option is specified, AuthoringTool returns an error upon detecting use of a version of PiaFramework that cannot be used for master submission.
For more information, see Technical Information / Master ROM Submission Information.

5-6-0S09: (TOOL) Added a feature for checking play data retrieval settings to AuthoringTool.

Added a check for when the --error-unpublishable option is specified when running AuthoringTool.
This check displays a warning message if the NMETA file contains any settings that would allow play data for application IDs other than the current application to be retrieved.
For more information about play data retrieval settings, see Features > Building > Creating Application Images > Writing the NMETA File.

5-6-0S10: (TOOL) An error now occurs if the version of AuthoringTool used to create a specified NSP file is newer than the version of AuthoringTool used to execute the bundleup and convertaoc commands.

Added a feature for checking the version of AuthoringTool in the specified NSP input file when executing the bundleup and convertaoc commands in AuthoringTool.
If the version of AuthoringTool used to create the specified NSP input file is newer than the version of AuthoringTool being used to execute the commands, an error occurs.
For more information about these commands, see Features > Building > Creating Application Images > Tool Reference > AuthoringTool.

5-6-0S11: (TOOL) You can now set the application's own ID in the SaveDataOwnerIds element of the NMETA file.

You can now set the application's own ID in the NintendoSdkMeta/Core/FsAccessControlData/SaveDataOwnerIds element of the NMETA file without causing a build error.
Only one ID can set for the application's own ID.
For more information about the NintendoSdkMeta/Core/FsAccessControlData/SaveDataOwnerIds element, see NintendoSDK Documents > Features > Building > Creating Application Images > Writing the NMETA File.

Bug Fixes

5-6-0S12: (AUDIO) Fixed a loudness measurements bug in SystemAudioMonitor.

Fixed a bug in the loudness measurement feature in SystemAudioMonitor where the displayed loudness values were incorrect.

5-6-0S13: (DIAG) Fixed a bug where it took a long time to use ConvertNxbinlog to convert some logs.

Fixed a bug where it took a long time to use ConvertNxbinlog to convert logs to an NXBINLOG file, if multiple logs larger than several KiB were all exported at the same time using NN_LOG(), NN_VLOG(), and NN_PUT().

5-6-0S14: (EC) Fixed a bug in the EcCatalog sample where some values could not be loaded if they were null.

Fixed a bug where an error occurred when the EcCatalog sample was executed if the values obtained for disclaimer and regular_price were null.
