Details

Changes in NintendoSDK NX Addon 5.1.0 (based on Generic 5.1.0)

Specification change that breaks your code

*5-1-0S01: (VSI) VSI and Target Manager breaking changes.

  • End of support for the TMAPI library.
    • The TMAPI library is no longer supported. Use the TM library instead.
  • Changed the default value for HtcsPeerName on the host.
    • The default value for HtcsPeerName has been changed from 127.0.0.1 to the name of PC.


Added Features

*5-1-0S02: (General) Added a feature to display the application log as an overlay.

A feature was added for using an overlay to display a log of the same information shown in TargetManager. For more information, see Controlling the Development Hardware/DevMenuOverlay.

*5-1-0S03: (Socket) New nn::socket API

All global-scope socket structures and macros (for example struct sockaddr_in and AF_INET) have been copied into the nn::socket namespace. Overloaded functions that take these new types have also been created. In the future, use these new functions for all sockets programming because the old ones may be deprecated in a future SDK release.

5-1-0S04: (APPLET) Added a feature for listing all the files in an album.

Added the nn::album::ShowAlbumFiles() function for listing all the files in an album.
For more information, see Library Applets > Album Applet.

5-1-0S05: (AUDIO) Added a tool for getting the system's final audio output.

Added the SystemAudioMonitor tool for getting the system's final audio output to %NINTENDO_SDK_ROOT%\Tools\Audio\SystemAudioMonitor.
When multiple audio outputs are being used in an application, you can use this tool to check diverse results.
For more information, see SystemAudioMonitor.

5-1-0S06: (AUDIO) Added the AudioRedirector tool.

Added a tool for sending sounds output by the PC to the development hardware for playback. For more information, see AudioRedirector.

5-1-0S07: (BCAT) Added a function for canceling immediate synchronization.

Added the function nn::bcat::CancelSyncDeliveryCacheRequest for canceling immediate synchronization.
In addition, added ResultCanceledByUse so cancellation by the user can be detected.

5-1-0S08: (BCAT) Added an immediate synchronization function that specifies a directory.

Added a function that specifies a directory and performs immediate synchronization.
If you plan to use this function, read the notes in the API Reference.
