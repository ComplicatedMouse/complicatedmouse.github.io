Details

Changes in NintendoSDK NX Addon 3.7.0 (based on Generic 3.7.0)

Improvements

3-7-0S01: (TOOL) Added a feature for checking the character encoding for tag names when creating downloadable content.

When creating downloadable content in the AuthoringTool, an error now occurs if characters that cannot be specified in NMETA file tags are set.
For more information, see Features > System > AOC Library > Creating Downloadable Content.

3-7-0S02: (TOOL) Added a feature for checking the string length of tag names when creating downloadable content.

When creating downloadable content in AuthoringTool, an error now occurs if a string that exceeds 128 characters is specified in NMETA file tags.
For more information, see Features > System > AOC Library > Creating Downloadable Content.

3-7-0S03: (TOOL) Added a feature to AuthoringTool for displaying diff information between patches.

Added the diffpatch command to AuthoringTool. This command displays file-based diff information between patches.

3-7-0S04: (TOOL) Improved patch generation times in AuthoringTool.

Optimized the data conversion processes in AuthoringTool, which yielded particularly large improvements in patch generation times.

3-7-0S05: (TOOL) Improved the checks for content with size updates in save data, temporary storage, and cache storage when creating patches.

When updating the size of save data, temporary storage, and cache storage using patches, in some cases invalid values were entered or updated when creating patches. For this reason, the check contents have been improved.

Bug Fixes

3-7-0S06: (FS) Updated FsAccessLogChecker to version 12.3.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 12.3. This version includes the following changes.

  • Fixed where FS Access Log Details is placed in the analysis results.
  • Added support for CR as a newline code.
  • Added support for Shift-JIS, including multiple-byte characters.
  • Added support for checking nn::fs::GetSaveDataSize() and the indexed cache storage feature.
  • Changed the output format for read operations that are not related to files for SameAddressRead items.
  • Added the --warning_ratio option, and support for the continuous output of top values for SameAddressRead that do not exceed the threshold.


3-7-0S07: (TOOL) Added a workaround for a bug in the Nintendo NX CPU Profiler where sampling was sometimes not possible.

Reverted the sampling method to the earlier format as a workaround for a bug in the Nintendo NX CPU Profiler where sampling was sometimes not possible. Because threads with affinities set in multiple cores could not be observed in the earlier method, set affinities only in a single core when using the profiler.

3-7-0S08: (TOOL) Fixed a bug where file loading became slow in some cases when performing edits, such as when the replace command in the AuthoringTool changed file sizes.

Fixed a bug where loading resource data and HTML document files slowed down for applications or patches created based on those applications.
This bug occurred when resource data and HTML document files included in the application were edited for each file using the replace command in the AuthoringTool,
and the file size was changed from the original size.

Also, due to this fix, patch sizes no longer increase unintentionally when the replace command is used.

3-7-0S09: (TOOL) Fixed a bug where installation of the document package failed if Visual Studio was uninstalled but the Help Viewer remained.

Fixed a bug where installation of a document package failed if Microsoft Visual Studio was uninstalled, but the Microsoft Help Viewer remained.

3-7-0S10: (TOOL) Fixed a bug where the list command in AuthoringTool did not display deleted files.

Fixed a bug with the list command in AuthoringTool where using the --patched-only option to only display files included in a patch sometimes caused files that had been deleted in the patch to not be displayed in the "Listing removed files from original" section.
