Details

Changes in NintendoSDK NX Addon 6.5.0 (based on Generic 6.5.0)

Added Features

*6-5-0S01: (Compiler) Added support features for profile-guided optimization.

Support for profile-guided optimization (PGO) is now possible with Clang for NX.
You can us this feature starting with NX Add-On 6.4.0 (Clang for NX 1.5.8).
For more information, see Features > Building > Compiler > Toolchain for NX > SDK LTO/PGO Supplemental Information.

*6-5-0S02: (EC) Added a function to display a list of products in the shop for things like downloadable content.

Added the nn::ec::ShowShopProductList() function to display a list of products in the shop for things like downloadable content.

*6-5-0S03: (EC) Added a function to display detailed product information in the shop for things like downloadable content.

Added the nn::ec::ShowShopProductDetails() function to display detailed product information in the shop for things like downloadable content.

6-5-0S04: (EC) Added a sample to get a list of downloadable content to the EcCatalog samples.

Added the AOC item shelf list option to the start screen of EcCatalog and added a sample to get and display the downloadable content shelf list.

6-5-0S05: (EC) Added user selection to the EcCatalog samples.

Revised the EcCatalog samples to enable user selection after startup.
(Until now, the user in the 0th position was used automatically.)
Revised the start screen notations. (retrive -> retrieve)

6-5-0S06: (ERR) Added a function for creating error codes from an error code category type and number.

Added the nn::err::MakeErrorCode() function for creating error codes from an error code category type and number.

6-5-0S07: (TOOL) Added a feature for displaying warning messages about using API functions that are unnecessary in production ROMs.

Added a new item for AuthoringTool to check when the --error-unpublishable option is specified.
In particular, added support for displaying warning messages about using functions that are unnecessary in production ROMs.

6-5-0S08: (TOOL) Added the checkintegrity command to AuthoringTool.

Added the checkintegrity command to AuthoringTool to check the integrity of data in NSP files. For more information, see Creating Application Images > Tool Reference > AuthoringTool.

6-5-0S09: (TOOL) Updated AssetDataCreator.

Updated AssetDataCreator to add options that are available when creating data for Retail Interactive Demo applications. For more information, see Setting Up a Testing Environment for Retail Interactive Demos.

Improvements

6-5-0S10: (CPU Profiler) Nintendo NX CPU Profiler Version 0.14.1

  • Added Umbra to the known middleware lists.
  • Improved the user interaction with Checkers.
  • Changed how Checkers are sorted.
  • Fixed a bug where a crash occurred if a custom build ID was specified when building. There is still a 20 byte limit on the supported length of build IDs.
  • Fixed extremely long loading times that could occur when using a very large number of threads with similar call stacks.
  • Fixed potential erroneous identification of AES functions in Checkers.
  • Fixed errors that could occur when disabling Checkers.
  • Fixed a bug where a negative number could be displayed in Wastefully Short Sleep Checker.
  • Fixed some potential misclassification of functions.
  • Fixed a crash that would occur if characters that were not paths or relative paths were entered in Scenario Selection and the ellipsis (...) button was clicked.
  • Fixed a crash that could occur if clicking Unsync while taking a profile and then specifying an automatic stop in the GUI.


6-5-0S11: (EC) Fixed a bug in the EcCatalog sample where some null values could not be loaded, and also added an associated error message.

Fixed a bug where an error occurred if the end_datetime values of the description and discount_price objects obtained when the EcCatalog sample is started were null.
In addition, if the value of sales_status was unreleased or sales_termination and regular_price was not null, it was previously possible to get these values. Fixed this bug so that a parsing error occurs instead.
Also added an error message that is displayed when the parsing process fails.

6-5-0S12: (TOOL) Moved the included AssetDataCreator package to AuthoringToolKit.

Moved the included AssetDataCreator package from the NX Add-On package to the AuthoringToolKit package.

Bug Fixes

6-5-0S13: (AUDIO) Fixed a bug where changing the filter coefficient of an nn::audio::BiquadFilterType filter created erroneous noise.

Fixed a bug where using the nn::audio::SetBiquadFilterParameter() function to change the filter coefficient of an nn::audio::BiquadFilterType filter for an audio renderer created erroneous noise.

* nn::audio::SetBiquadFilterParameter()

6-5-0S14: (DevMenu) Fixed a bug where tapping a screen with scrollbars moved you to an unintended screen.

Fixed a bug where tapping a DevMenu screen with scrollbars moved you to an unintended screen.

6-5-0S15: (DIAG) Fixed a bug where functions for getting module information caused memory access violations in Windows 7 environments.

Fixed a bug where calling the following functions in a Windows 7 environment caused memory access violations.

* nn::diag::GetModulePath()
* nn::diag::GetRequiredBufferSizeForGetAllModuleInfo()
* nn::diag::GetAllModuleInfo()
* nn::diag::GetReadOnlyDataSectionRange()

6-5-0S16: (libcurl) Increased the accuracy of the upload and download speeds reported by libcurl.

Increased the accuracy of the upload and download speeds reported by libcurl for quick transfers by increasing the precision from seconds to milliseconds.

6-5-0S17: (MOVIE) Fixed a bug in MoviePlayerSimple where an assertion was triggered in rare cases immediately after starting.

Previously, the MovieDecoderEventHandler thread was started before any of the Event objects were linked, so the WaitAny process and Event object linking processes ran in parallel within the thread, which sometimes caused an assertion to be triggered depending on the timing.
Fixed the implementation such that now the thread is always started after the Event object linking processes have started.

6-5-0S18: (MOVIE) Fixed a bug where the channel mapping for six-channel Vorbis audio data was inconsistent with the Vorbis audio specification.

In the movie sample, the FR, CF, RF, RL, RR, and LFE channel mapping for six-channel Vorbis audio data was not handled correctly.
Changed the mapping to be consistent with the Vorbis audio specification so that this type of data can now be played correctly.

6-5-0S19: (NIFM) Fixed a bug where applications sometimes stopped with an assertion failure when handling NIFM errors.

Fixed a bug where in Develop and Debug builds of applications, externally closing an applet started with a call to nn::nifm::HandleNetworkRequestErrorResult() and nn::nifm::HandleNetworkRequestResult() caused the application to crash.

6-5-0S20: (TOOL) Fixed a bug where AuthoringTool failed to create multi-application cards including NSP files that had been created using an earlier version of AuthoringTool.

Fixed a bug where creating multi-application cards in AuthoringTool failed if you attempted to include any NSP files that had been created using an earlier version of AuthoringTool.

Changes in NintendoWare NX Addon 6.5.0 (based on Generic 6.5.0)

Improvements

6-5-0W01: (EC) In the EcCatalog sample, added support for jumping to a title's own shop page.

The ShowShopConsumableItemDetail function called in the EcCatalog sample is now called without specifying an application ID (to jump to the title's own shop page).
Revised the associated comments and documentation accordingly.
