Details

Changes in NintendoSDK NX Addon 7.2.0 (based on Generic 7.2.0)

Specification change that breaks your code

*7-2-0S01: (libcurl) libcurl now supports compressed accept encoding

Libcurl now supports compression through the libz library. Applications that link with libcurl are required to link with libz. To use this feature (where 'curl' is the handle) use curl_easy_setopt(curl, CURLOPT_ACCEPT_ENCODING, "");.

Added Features

*7-2-0S02: (Compiler) Added support for the PGO (profile-guided optimization) feature.

Support for profile-guided optimization (PGO) is now possible with Clang for NX.
You can us this feature starting with NX Add-On 6.4.0 (Clang for NX 1.5.8).
For more information, see Features > Building > Compiler > Toolchain for NX > SDK LTO/PGO Supplemental Information.

*7-2-0S03: (EC) Added a function to display a list of products in the shop for things like downloadable content.

Added the nn::ec::ShowShopProductList() function to display a list of products in the shop for things like downloadable content.

*7-2-0S04: (EC) Added a function to display detailed product information in the shop for things like downloadable content.

Added the nn::ec::ShowShopProductDetails() function to display detailed product information in the shop for things like downloadable content.

Added functionality to SystemAudioMonitor for configuring user settings.
The following user settings can be configured.

  • Whether to maintain the level meter peak value


For more information, see Audio Library > Tool Reference > SystemAudioMonitor.

7-2-0S06: (CODEC) Added a function to return the processing time required for decoding HardwareOpusDecoder and HardwareMultiStreamDecoder.

Added a function that returns the processing time required in the hardware accelerator for the decoding required immediately prior to nn:::codec::HardwareOpusDecoder and nn::codec::HardwareMultiStreamDecoder.

* nn::codec::HardwareOpusDecoder::GetProcessingTime()
* nn::codec::HardwareOpusMultiStreamDecoder::GetProcessingTime()

Added the OpusDecoder.exe tool for decoding Opus files on PC.

You can use this notification for verifying compliance with guidelines when accessing the consumable item server.
This support feature is only available when the startup menu setting is DevMenu.

Added the following log functions for setting module name, severity, and redundancy as metadata.

Added the following features to NVN.

  • Added more documentation about the alpha test to the NVN Programming Guide detailing conditions for alpha test being disabled and adding examples.


Added the following features to GLSLC.

  • Added per-stage definitions __VERTEX_SHADER, __FRAGMENT_SHADER, __TESSELLATION_CONTROL_SHADER, __TESSELLATION_EVALUATION_SHADER, __GEOMETRY_SHADER, and __COMPUTE_SHADER, which are each defined as 1 in their corresponding shader stage and undefined in other stages.

  • Added global definitions __GLSLC_VER_GPU_MAJOR and __GLSLC_VER_GPU_MINOR, corresponding to the major and minor versions of the GLSLC's GPU code, respectively.


Added the following features to the texture packager.

The following features were added to the texture packager in NX Add-On 7.1.0, but were omitted from the revision history.

7-2-0S30: (Display) nn::vi::SetLayerScalingMode() has been deprecated.

nn::vi::SetLayerScalingMode() is deprecated. If you are using this function, a warning message is displayed during compile.

From now on, the default scaling mode (nn::vi::ScalingMode_FitToLayer) is always applied. If any of the following conditions applies to the application, simply delete the function call.

* nn::vi::SetLayerScalingMode() was not called.
* The default value (nn::vi::ScalingMode_FitToLayer) was called.
* The aspect ratio of the framebuffer matches the platform default aspect ratio (16:9).

For other applications, note that the display range is automatically reduced according to the aspect ratio, and you might not be able to insert pillarboxing and letterboxing. The application must call the nn::vi::SetLayerCrop() function (or a function equivalent to the various graphics functions) to explicitly restrict the display range, or explicitly render the pillarboxing or letterboxing.

7-2-0S31: (EC) In the EcCatalog sample, added support for the new sales_status specifications and deleted parsing of discount_price.

Because sales_status from the default behavior of the function for getting a list of consumables/AOC was changed to return only onsale, the processing for branching in sales_status was deleted.
In addition, added support for outputting errors when sales_status is not onsale.
Because there may be changes to the content obtained by discount_price when getting the list of consumable items, the parsing processing and member variables within the DiscountPrice class have been deleted.

7-2-0S35: (GFX) Added support for reduction filters in the GL version.

In the GL version, when the reduction mode of the filter set for nn::gfx::SamplerInfo::SetFilterMode was Minimum or Maximum, it was ignored and set to an average reduction. However, environments that support the GL_ARB_texture_filter_minmax extension now set a reduction filter corresponding to the specified values.

Changes in NintendoWare NX Addon 7.2.0 (based on Generic 7.2.0)

Specification change that breaks your code

*7-2-0W01: (FONT) Improved the behavior of nn::font::RegisterGlyphFromTextBoxRecursive.

Previously, nn::font::RegisterGlyphFromTextBoxRecursive performed RegisterGlyph for all scalable fonts within a composite font when run for a text box set with a composite font. It has been improved to only configure fonts that will actually be rendered.

Added a function to skip RegisterGlyph processing for hidden panes.

*7-2-0W02: (Layout-UI2D) Changed the method for managing dynamic rendering texture resources.

Previously, dynamic rendering textures such as capture textures or vector graphics textures were managed by registering them with names to ResourceAccessor. However, because texture names within ResourceAccessor must be unique, some cases that were difficult to handle, such as setting root names when copying layouts or not being able to create multiple instances of the same layout data without cloning it.
To resolve these types of situations, implementation was changed to manage dynamic rendering textures internally in the pane instance without using ResourceAccessor.

The following specifications were changed for this purpose.

  • Dynamic rendering textures can no longer be registered in ResourceAccessor. The ResourceAccessor::AcquireDynamicGenerateTexture() or ResourceAccessor::AcquireDynamicGenerateTextureWithResolvePrefix() methods for dynamic rendering textures are not used.
  • Functionality for specifying the root name or prefix for capture textures, such as those within copy constructors or functions, have been deprecated as they are no longer needed.
