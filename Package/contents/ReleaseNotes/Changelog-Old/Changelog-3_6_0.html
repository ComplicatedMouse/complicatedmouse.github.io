Details

Changes in NintendoSDK NX Addon 3.6.0 (based on Generic 3.6.0)

Specification change that breaks your code

*3-6-0S01: (General) Added a video capture feature for the application.

Added a video capture feature for the application. In applications with the video capture feature enabled, the system always automatically stores a maximum of 30 minutes of video data in memory. By holding down the Capture Button, that video data can be saved to a file. This video capture feature is enabled by default in applications built with this version of the SDK, when launched on consoles that have version 4.0.0 or later of the firmware installed.
For more information, such as the format of the saved video, also see this page.

For applications with the video capture feature enabled, in an NX64 environment, the memory that can be used by the application is reduced by 96 MiB, and that portion of memory is used by the system. There are no changes to the size of usable memory in an NX32 environment. Also, it affects the performance of the application. For more information, see this page.

In cases where you do not want to permit this functionality, of for other planned reasons, you can disable the video capture feature by setting Video Capture in the NMETA file to Disable. You can also use AuthoringEditor to configure this setting.
See Video Capture.

You can temporarily stop video capture in applications with the video capture feature enabled. For more information, see the newly added nn::oe::EnableRecording() and nn::oe::DisableRecording().

*3-6-0S02: (TOOL) Separated the VSI for NX and Target Manager packages.

The packages (zarf) for VSI for NX and for Target Manager have been reorganized.

* Before the change: NX Visual Studio Integration and Target Manager were both included in the VSI for NX package.
* After the change: NX Visual Studio Integration is included in the VSI for NX package, and Target Manager is in the Target Manager package.

The Target Manager package is required for running applications on the NX development hardware.
The new VSI for NX package is required for building and debugging applications using Visual Studio.

Install the packages required for your purposes.

For both, if you use NDI to install the package, the installation to the Windows system will also be performed. If you want to install the package to the system at another time, execute the following batch files as the administrator.

* For VSI for NX, run Installers\Vsi\Install.bat.
* For Target Manager, run Installers\TargetManager\Install.bat.

*3-6-0S03: (VSI) Breaking changes in Target Manager.

  • Support for the TMAPI.h file ended with NX Add-On 5.1.
    • Official support for the TM library, enabling use of Target Manager features, began in NX Add-On 4.1. As a result, the functions that had been defined in C:\Program Files\Nintendo\Oasis\include\TMAPI.h are no longer available, from NX Add-On 5.1 onwards. Use C:\Program Files\Nintendo\Oasis\include\tm.h moving forward.
  • The option to enable just-in-time debugging for Target Manager is disabled by default.
    • To use just-in-time debugging, select Target Manager > Tools > Options > JIT Debugger.


Added Features

3-6-0S04: (AOC) Added a sample for season passes that use downloadable content functionality.

Added a sample, which shows how to implement season passes, using downloadable content functionality.

3-6-0S05: (FS) Added the exporting of nn::nfp access logs, and updated FsAccessLogChecker to version 11.0.

Added the exporting of access logs with nn::nfp functions related to FS access, and updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 11.0. This includes the following changes.

  • Fixed the analysis of character encoding for input files.
  • Added support for checking regions subject to frequent read access.
  • Revised how overlapping read access is determined.
  • Added support for checking FS access via nn::nfp.


3-6-0S06: (General) Added functions to pause and resume video capture for applications.

Added the following functions to temporarily pause and resume video capture in applications with the video capture feature enabled.

  • nn::oe::EnableRecording()
  • nn::oe::DisableRecording()


Use the VideoCapture option in the NMETA file to enable video capture. For more information on video capture, see this page.
For more information on topics such as the format of saved videos, also see this page.
For information about impacts on application performance, see this page.

3-6-0S07: (IRSENSOR) Added a pointing feature to the IR Motion Camera.

Added a screen pointing feature using the IR Motion Camera.
Note that dedicated hardware is required to use this feature.
For more information, see the IRSENSOR library manual.

3-6-0S08: (Sample) Added a sample to get the supported languages.

Added a sample that gets supported languages from a program to the Samples/Sources/Applications/OeGetLanguage folder.

3-6-0S09: (TOOL) Added functionality to get crash reports from incidents that occur in the market.

You can now get data when an application crashes in the market. For more information, see Features > System > Crash Reports in the SDK documentation.

3-6-0S10: (TOOL) Support for developing an application that can handle Chinese (Simplified), Chinese (Traditional), and Korean.

Enabled the creation of applications that support Chinese (Traditional), Chinese (Simplified), and Korean.
Added the method for getting supported languages to oe Library / Manual / Feature Description / Getting the Language, and added a sample to get the settings for supported languages.
Also revised AuthoringTool to stop generating errors when the --error-unpublishable option is used while creating applications that support Simplified and Traditional Chinese, and Korean.

3-6-0S11: (VSI) Added new features to VSI and Target Manager.

  • VSI
    • Added a feature to individually pause and resume threads.
    • Added a feature to implicitly include compiled headers.
      • Previously, you needed to specify files to force include, but with the NX platform that operation is not necessary.
      • Along with this change, the content of the compiled headers is now displayed in IntelliSense.
    • Added an option to use build errors when ROM submissions are not possible.
  • Target Manager
    • Added a feature for capturing screenshots.
      • Right-click a target and on the shortcut menu, select Capture Screenshot.
    • Added a feature for showing a Log View for each target.
      • Right-click a target, and on the shortcut menu, select View Log.
    • Added a feature to install applications and a feature to start the installed applications.
      • Right-click a target and on the shortcut menu, select Install Application or Start/Manage Application.
    • Added a feature to create dump files.
      • Right-click a target and on the shortcut menu, select Dump.
    • Added a feature to write to a game card.
      • Right-click a target and on the shortcut menu, select Install Application.
    • Added a feature to enable the use of Target Manager on its own.
      • Previously, Target Manager needed copies of dependency files, but now it is possible to use just NintendoTargetManager.exe.
    • Added features for resetting multiple devices at the same time and for ending the program on multiple devices.
  • Other
    • Added support for compatibility.
      • Earlier firmware and the RunnerTools provided in NX Add-On 4.1 and later versions can be used without re-installing Target Manager.
    • Added support for host libraries that use some Target Manager features.
    • Association of the .nsp extension.
      • It has a unique icon, and double-clicking runs the file on the default Target Manager target.
    • Added the ability to send warnings to Target Manager when attempts to control files, using HostFs, fail because of insufficient Windows permissions or other such issues.
    • Added menu items to control power and LED flashing for multiple devices.


Improvements

3-6-0S12: (DevMenu) Updated for the changes in NX Add-On in 4.x.

Applied some of the DevMenu changes in NX Add-On in 4.x to NX Add-On 3.x. The following is a list of relevant changes. For more information, see the revision histories of the respective versions.

• NX Add-On 4.0.0

  • Fixed a bug that caused a fatal error to occur when a button was pressed on the User Details screen.


• NX Add-On 4.1.0

  • Added a feature to DevMenuCommand for referencing and changing the system's time zone.
  • Improved error handling during network connection for DevMenu.


• NX Add-On 4.2.0

  • Fixed a bug where Error 2110-111 was displayed when submitting a network use request.
  • Added a feature to get the accurate current time from the server, and correct the network clock.
  • Made a change such that an error is now displayed if an SD card that cannot be used in the NX environment is inserted.


• NX Add-On 4.3.0

  • Fixed a bug where applications could not start from the details screen displayed with nn::friends::ShowUserDetailInfo.


• NX Add-On 4.4.0

  • Added the ability to configure television underscan values.
  • Fixed a bug when launching applications that involve user selection.
  • Improved the amount of time needed to display the save data list.


3-6-0S13: (TOOL) Removed DistributedByNintendo from the LogoType options in AuthoringEditor.

Removed DistributedByNintendo from the Logo Data Setting option that can be configured when editing application NMETA files in AuthoringEditor.

3-6-0S14: (VSI) Revisions to VSI and Target Manager

  • Target Manager
    • Changes have been made to automatically connect to the target when dropping an NSP, in an unconnected state, to Target Manager.
    • Changes have been made to correctly parse paths enclosed in quotation marks, when specifying the path for an NSP to run it, using Target Manager.


Bug Fixes

*3-6-0S15: (TOOL) Fixed a bug where importing NMETA files did not work when editing NSP files in AuthoringEditor.

Fixed a bug where importing NMETA files could not be performed when editing application NSP files in AuthoringEditor.
This fix enables you to import existing NMETA file content and apply it while an NSP file is open.

3-6-0S16: (BuildSystem) Fixed a bug where building would fail if there was a space in the path specified during NSPD creation.

There was a bug when creating an NSPD in AuthoringTool, where the build would fail if there was a space in the path. This has been fixed.
This bug similarly occurred when there was a space in the TargetName properties of the project when building an application in VSI.

3-6-0S17: (TOOL) Fixed a bug where setup would fail in environments without a Microsoft Visual C++ 2015 runtime.

Fixed a bug where an error would result in environments without an installed Microsoft Visual C++ 2015 runtime, which includes items such as the Microsoft Visual C++ 2015 redistributable package.

3-6-0S18: (TOOL) Fixed the erroneous detection for errors blocking master ROM submission for AuthoringEditor and AuthoringTool.

Fixed an issue where <em>Functionality requiring a usage request - Bundle_Private_NetworkMiddleware_Generic</em> was erroneously detected when either opening NSP files that use libcurl in AuthoringEditor, or getting errors that block ROM submission in AuthoringTool.

3-6-0S19: (VSI) Bug fixes for VSI and Target Manager.

  • VSI
    • Fixed a bug where IntelliSense would wrongly output errors.
    • Fixed a problem where stepping took a long time.
    • Fixed a bug where some NatVis syntax did not work right.
    • Fixed a problem with the UI when loading symbol files.
      • The required menu option for loading NSS files prior to VSI for NX 4.1.2 was Symbol Load Information. From VSI for NX 4.1.2 onward, use the Load Symbols menu option.
    • Fixed a bug where functions pointers to member functions were not displayed correctly.
    • Fixed the inability to specify multiple project properties for an NRO file.
    • Fixed a bug where NMETA files could not be opened in VSI.
    • Fixed an issue in the Visual Studio memory window where changes to values could not occur.
    • Fixed an issue when an NX32 disassembly appears as NX64 registers when opening the NX32 disassembly.
    • Fixed an issue where you could not proceed beyond loading when debugging a library project.
  • Target Manager
    • Fixed a bug where repeatedly running an application and resetting the EDEV led to a situation where the hardware could no longer be reset from Target Manager.
    • Fixed a bug with HTCS ports remaining when the system was asleep.
    • Fixed a bug where file read/write operations would fail when the system was asleep.
    • Fixed a bug where Target Manager became unstable when the system was put to sleep.
    • Fixed an issue where behavior would become unstable when connected to multiple devices.
    • Fixed an issue where an error screen would appear when connecting or disconnecting an EDEV.
    • Fixed an issue where you could not quit any applications from Target Manager that had been running prior to the Target Manager connection.
    • Fixed an issue where logs would only appear in the first log viewer when two or more log viewer windows were open.
    • Fixed an issue where logs would not appear when the Log Viewer was launched prior to launching Target Manager.
    • Fixed an issue where some logs would not appear when simultaneously submitting logs of a certain length from multiple threads.
    • Fixed an issue where a tm::GetTargetProgramState failure was returned prior to running an application.
    • Fixed an issue where a target would remain registered, even after removing that registration immediately after launching Target Manager.
    • Fixed an issue where erroneous exception codes would appear when debugging or dumping applications launched from Target Manager.
  • Fixed other minor bugs.
