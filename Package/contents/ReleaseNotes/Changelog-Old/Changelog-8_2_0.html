Summary

Changes in NintendoSDK NX Addon 8.2.0 (based on Generic 8.2.0)

Added Features

  • *8-2-0S01: (DevMenu) Added a feature to start DevMenu by using button operations on the HOME Menu.
  • *8-2-0S02: (OE) Added a CPU boost feature that temporarily increases CPU processing capabilities at the cost of lowering GPU processing capabilities.
  • 8-2-0S03: (HID) Added the batch file that performs VibrationConverterConsole conversion.
  • 8-2-0S04: (OS) Added the busy mutex feature as a mutual exclusion feature between threads.
  • 8-2-0S05: (VSI) New Features in VSI and Target Manager

Improvements

  • *8-2-0S06: (Accounts) Added support for StartupUserAccountOption settings in the NMETA file to the application startup processing in the HOME Menu.
  • 8-2-0S07: (CPU Profiler) Nintendo NX CPU Profiler Version 0.16
  • 8-2-0S08: (General) Fixed a bug where mounting save data sometimes failed when running applications that contained single sub-programs.

Bug Fixes

  • 8-2-0S09: (Display) Fixed a bug where the nn::vi::ListFrameHistory() function returned an invalid nn::vi::FrameInfo.
  • 8-2-0S10: (FS) Fixed a bug where the first access log was not generated.
  • 8-2-0S11: (GFX) Fixed a bug where an error would occasionally occur when multiple processes attempt to initialize gfx simultaneously in the PC version of the nvn version.
  • 8-2-0S12: (HID) Made changes so that the system aborts if the nn::hid::GetVibrationDeviceHandles preconditions are violated.
  • 8-2-0S13: (OpenGL) Fixed a bug where OpenGL 1.1 functions could not be loaded, depending on the environment, in the Windows version of gll.
  • 8-2-0S14: (OS) Revised the error codes that are left in the crash report when the application aborts.
  • 8-2-0S15: (Sample) Fixed a bug where OpenGL samples failed to build.
  • 8-2-0S16: (SSL) Fixed the operation of nn::ssl::SetDebugOption.
  • 8-2-0S17: (TOOL) Fixed a bug in the texture converter where converting an ATI2A2XY format DDS file switched the R and G components.
  • 8-2-0S18: (TOOL) Fixed a bug where upscaling vectors to scalar vectors with bitcasts was not added in HlslCrossCompiler.
  • 8-2-0S19: (VSI) VSI and TargetManager Bug Fixes
  • 8-2-0S20: (Vulkan) Fixed a bug where Vulkan did not work correctly in NX32.

Details

Changes in NintendoSDK NX Addon 8.2.0 (based on Generic 8.2.0)

Added Features

*8-2-0S01: (DevMenu) Added a feature to start DevMenu by using button operations on the HOME Menu.

Added a feature to start DevMenu by using button operations on the HOME Menu.
You can start DevMenu by using button operations while a specific dialog box is displayed in the HOME Menu.
For more information, see Controlling the Development Hardware > DevMenu > Starting DevMenu.

*8-2-0S02: (OE) Added a CPU boost feature that temporarily increases CPU processing capabilities at the cost of lowering GPU processing capabilities.

Added a CPU boost feature that temporarily increases CPU processing capabilities at the cost of lowering GPU processing capabilities. For more information, see CPU Boost Functionality.

8-2-0S03: (HID) Added the batch file that performs VibrationConverterConsole conversion.

Added the batch file that performs VibrationConverterConsole conversion. You can drag Wave files to the batch file to convert them to vibration binary files.

VibrationConverterConsole location.
%NINTENDO_SDK_ROOT%\Tools\Hid\VibrationConverterConsole

For more information, see Features > Input Device > hid Library > Tool Reference > Vibration Converter.

8-2-0S04: (OS) Added the busy mutex feature as a mutual exclusion feature between threads.

Added the busy mutex feature as an os library feature for feature mutual exclusion between threads. For more information, see the manual and API reference.

8-2-0S05: (VSI) New Features in VSI and Target Manager

  • Target Manager
    • Added functions that can register events signaled when the application crashes.
      • Use Nintendo.Tm.TargetManager.OnApplicationError.


Improvements

*8-2-0S06: (Accounts) Added support for StartupUserAccountOption settings in the NMETA file to the application startup processing in the HOME Menu.

Added support for StartupUserAccountOption settings in the NMETA file to the application startup processing in the HOME Menu. For applications with IsOptional set for StartupUserAccountOption, the behavior has been changed to be identical to when nn::account::TryOpenPreselectedUser() is called from DevMenu.

8-2-0S07: (CPU Profiler) Nintendo NX CPU Profiler Version 0.16

  • Added a code block push/pop API.
  • Added a code block hierarchy view to the Sample Graph.
  • Added new checkers.
  • Disabled the display of the number of ztest passed pixels because it is always 0.
  • Fixed a display bug where GPU Trace was drawn with the wrong offset.
  • Fixed a bug that used more memory than expected when opening profiles.
  • Fixed a crash that occurred when attempting to view details on a thread destruction event.
  • Fixed an error that could occur when a profile saved with Notes was opened.


8-2-0S08: (General) Fixed a bug where mounting save data sometimes failed when running applications that contained single sub-programs.

Fixed a bug where starting an application containing a single sub-program from VSI or TargetManager sometimes caused mounting save data to fail.

Bug Fixes

8-2-0S09: (Display) Fixed a bug where the nn::vi::ListFrameHistory() function returned an invalid nn::vi::FrameInfo.

Fixed a bug where the element values in the nn::vi::FrameInfo returned by the nn::vi::ListFrameHistory() function in NX Add-On 8.1 were not valid.

8-2-0S10: (FS) Fixed a bug where the first access log was not generated.

Fixed a bug where overlapping initial fs access operations only generated a log for one of the access operations.

8-2-0S11: (GFX) Fixed a bug where an error would occasionally occur when multiple processes attempt to initialize gfx simultaneously in the PC version of the nvn version.

Fixed a bug where an error would occasionally occur when multiple processes attempt to call nn::gfx::Initialize simultaneously in the PC version of the nvn version.

8-2-0S12: (HID) Made changes so that the system aborts if the nn::hid::GetVibrationDeviceHandles preconditions are violated.

Made changes so that the system aborts if the nn::hid::GetVibrationDeviceHandles preconditions are violated.

NpadStyleSet must satisfy the following conditions.
style.IsAnyOn() == true
style.CountPopulation() == 1

If the preconditions are not met, an invalid nn::hid::VibrationDeviceHandle is generated and vibrations cannot be controlled normally. This change was made to fix this bug.

8-2-0S13: (OpenGL) Fixed a bug where OpenGL 1.1 functions could not be loaded, depending on the environment, in the Windows version of gll.

Fixed a bug where OpenGL 1.1 functions could not be loaded with nn::gll::InitializeGL, depending on the environment, in the Windows version of gll.

8-2-0S14: (OS) Revised the error codes that are left in the crash report when the application aborts.

Until this change, when the application closed with NN_ABORT, the error code was 2000-0000. With this change, the error code is 2162-0001, which indicates that the application aborted.

8-2-0S15: (Sample) Fixed a bug where OpenGL samples failed to build.

Fixed a bug where GlSimple and NvnTutorial10 failed to build on the NX32 and NX64 platforms.

8-2-0S16: (SSL) Fixed the operation of nn::ssl::SetDebugOption.

Fixed the behavior of nn::ssl::SetDebugOption.
The function is intended to allow bypassing certificate validation when used together with nn::ssl::Connection::SetVerifyOption, but only when used with Debug or Develop builds. On Release builds, it is supposed to ignore nn::ssl::SetDebugOption.
This change fixes the bug where nn::ssl::SetDebugOption was ignored for all NX build types.

8-2-0S17: (TOOL) Fixed a bug in the texture converter where converting an ATI2A2XY format DDS file switched the R and G components.

Fixed a bug in the texture converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe) where converting a ATI2A2XY format DDS file switched the R and G components.

8-2-0S18: (TOOL) Fixed a bug where upscaling vectors to scalar vectors with bitcasts was not added in HlslCrossCompiler.

Fixed a bug where upscaling vectors to scalar vectors with bitcasts was not added in HlslCrossCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe).

8-2-0S19: (VSI) VSI and TargetManager Bug Fixes

  • VSI
    • Fixed a bug where .o files were sometimes retained after cleanup.
  • Target Manager
    • Fixed a bug where the target state was incorrect when waking from sleep mode.
    • Fixed a bug where logs were occasionally not generated during sleep mode.
    • Fixed a bug where the remote video mute status was not correctly saved.
    • Fixed a bug where nn::tm::ProgramExitEvent was not signaled by events except for the prescribed target.
  • Fixed other minor bugs.


8-2-0S20: (Vulkan) Fixed a bug where Vulkan did not work correctly in NX32.

Fixed a bug where Vulkan sometimes did not work correctly in NX32.

Changes in NintendoWare NX Addon 8.2.0 (based on Generic 8.2.0)

Specification change that breaks your code

*8-2-0W01: (FONT) Fixed a bug where WordWrapping wrapped in the middle of a tag when there was a tag at the start of a line and a line feed was inserted immediately after it.

Fixed a bug where WordWrapping wrapped in the middle of a tag when there was a tag at the start of a line and a line feed was inserted immediately after it.

Added Features

8-2-0W02: (LayoutTool) Expanded LayoutEditor to enable selecting whether to start the standard PC viewer automatically.

Expanded LayoutEditor to enable selecting whether to start the standard PC viewer automatically. Select this option from Menu > Tools > Automatically Start the Standard PC Viewer.

8-2-0W03: (EffectTool) Added pass-through options for 3dShaderConverter to EffectMaker.

Added --3d-shader-converter-options as options for passing options to 3dShaderConverter.

Improvements

8-2-0W04: (LayoutTool) Expanded animation segment tags that do not generate any content in LayoutEditor to display a warning message.

Expanded animation segment tags that do not generate any content in LayoutEditor to display a warning message.

8-2-0W05: (LayoutTool) Improved the time scale adjustments in Curve Editor in LayoutEditor.

Improved the time scale adjustments in Curve Editor in LayoutEditor. You can now adjust the time scale in Curve Editor. In addition, the key tilt during the scale can also be adjusted. Two or more keys must be selected for each curve to adjust the time scale.

8-2-0W06: (LayoutTool) Made changes so that rendering results for panes to which capture effects were applied can now be captured.

Made changes so that you can place a pane to which capture effects have been applied under the capture pane and capture the rendering results.
Note that you can only render correctly within the same layout, and cannot render correctly if there are dependencies that cross multiple layouts, such as overwriting.
In addition, the Standard (Max Alpha) blending mode was added to the various feature modes for Capture Effects to control the alpha value when converting to a texture.

8-2-0W07: (LayoutTool) Made changes so that the Show Only Enabled Items option in Curve Editor in LayoutEditor is always enabled.

Made changes so that the Show Only Enabled Items option in Curve Editor in LayoutEditor is always enabled.

8-2-0W08: (LayoutTool) Made improvements to display an indent depending on the hierarchy level in the Show Parts Layout in Parts Pane feature in LayoutEditor.

Made improvements to display an indent depending on the hierarchy level in the Show Parts Layout in Parts Pane feature in LayoutEditor.

8-2-0W09: (EffectTool) Changed the --team-setting-file option in EffectMaker to not recommended, and added the --team-settings-file option.

For unifying the options with other tools, changed the --team-setting-file option in EffectMaker to not recommended, and added the --team-settings-file option.

Bug Fixes

8-2-0W10: (3D-G3D) Fixed a bug where enabling OverDraw in the Town scene in the G3dDemo Samples caused the sample to crash.

Fixed a bug where enabling OverDraw on certain platforms in the Town scene in the G3dDemo Samples caused the sample to crash.

8-2-0W11: (3DTool) Fixed a bug in the 3D texture converter where converting an ATI2A2XY format DDS file switched the R and G components.

Fixed a bug in the 3D texture converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) where converting an ATI2A2XY format DDS file switched the R and G components.

8-2-0W12: (Layout-UI2D) Fixed a bug where getting the constant buffer slot for the vertex shader failed under some conditions when the mask shader variations were not correctly specified.

Fixed a bug where getting the constant buffer slot for the vertex shader failed under some conditions when the mask shader variations were not correctly specified.
When creating a layout archive that includes multiple layout data, a bug occurred when data using vector graphics textures was ahead of data using masks in the filename order in the layout binary.

8-2-0W13: (Layout-UI2D) Fixed a bug where memory was illegally accessed when calling Pane::FindExtUserDataByName() for a pane for which there was extended user data for the system, such as capture effects.

Fixed a bug where calling Pane::FindExtUserDataByName() for a pane for which there was extended user data for the system, such as capture effects, exceeded the range of extended user data included in the resource and caused memory to be illegally accessed.

8-2-0W14: (LayoutTool) Fixed a bug in LayoutEditor where a thumbnail creation error message was displayed when operating the UI of a pane using a capture texture.

Fixed a bug in LayoutEditor where the "invalid thumbnail detected" message was displayed when operating the UI of a pane using a capture texture.

8-2-0W15: (LayoutTool) Fixed a bug where BFFNT files in drive C were not displayed in LayoutViewer.

Fixed a bug where text panes that used BFFNT files in the C drive were not displayed when transferring from LayoutEditor to LayoutViewer.

8-2-0W16: (LayoutTool) Fixed a bug where canceling or reverting changing the time for multiple keys in the Curve Editor in LayoutEditor did not work correctly.

Fixed a bug where canceling or reverting changing the time for multiple keys in the Curve Editor in LayoutEditor did not work correctly.

8-2-0W17: (LayoutTool) Fixed a bug where specifying the layout file as an argument in LayoutEditor causes a crash.

Fixed a bug where specifying the layout file as an argument in LayoutEditor causes a crash.

8-2-0W18: (LayoutTool) Fixed a bug where the capture texture content was not correctly displayed when using perspective projection mode in LayoutEditor.

Fixed a bug where the capture texture content was not correctly displayed when using perspective projection mode in LayoutEditor.

8-2-0W19: (LayoutTool) Fixed a bug where the clear color of the Capture Pane in the preview in LayoutEditor was not affected by gamma settings.

Fixed a bug where the clear color of the Capture Pane in the preview in LayoutEditor was not affected by gamma settings.

8-2-0W20: (LayoutTool) Fixed a bug where unwanted tags were added to the intermediate file when a pane was deleted in LayoutEditor while Capture Effects were enabled in LayoutEditor.

Fixed a bug where unwanted tags were added to the intermediate file when a pane was deleted in LayoutEditor while Capture Effects were enabled in LayoutEditor.
