Details

Changes in NintendoSDK NX Addon 0.12.15 (based on Generic 0.17.15)

Added Features

13-16S01: (General) Added integrated sample.

Added a Samples\Sources\Applications\Demo1\ sample to use multiple NintendoSDK features at the same time. (Described as integrated sample below.)
Build and execute the integrated sample to operate.

<Description>

The integrated sample is comprised of the following features.

  • Primitive rendering using the nns::gfx::PrimitiveRenderer library.
  • Using the nn::hid library to get the state of the human interface device.
  • Audio playback using the audio renderer.
  • Loading files using the nn::fs library.
  • Writing user save data using the nn::fs library.
  • Displaying the current time and detecting time operations using the nn::time::StandardUserSystemClock library.
  • Getting the state of the SixAxisSensor using the Npad.
  • Getting the state of the IrSensor using the Npad.
  • Displaying the string input by the software keyboard.


Improvements

13-16S02: (FS) Updated FsAccessLogChecker to version 3.0.

Updated FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) to version 3.0. This includes the following changes.

  • Added support for the access log SD card output format for NintendoSDK Firmware for NX 2.0.0-3 and later.
  • When there are multiple start logs for the input file, each log is now analyzed, and a warning that there are multiple logs is displayed.
  • The tool now returns an error if the timestamps in an access log are not in chronological order.
  • The tool now returns an error if the FS_ACCESS log is not complete.
  • FS_ACCESS logs for which the "result" entry contains an error are not included in the read/write operation count.


13-16S03: (TOOL) Added an error condition for the creation of add-on content.

An error is now generated if the Tag element has not been specified in the add-on content's NMETA file and --error-unpublishable has been specified.

Bug Fixes

13-16S04: (DevMenu) Fixed a bug where it was not possible to use DevMenu to install applications in PROD MODE.

Fixed a bug where it was not possible to install NSP files in the SD card to the console from the DevMenu Application page if the development hardware was initialized in PROD MODE.

13-16S05: (APPLET) Fixed a bug where the system would stop after attempting to start multiple library applets simultaneously from the application.

Fixed a bug where the system would stop after attempting to start multiple library applets simultaneously from the application. This issue occurred in sequences such as the following.

  • From the application, start the error viewer at nearly the same time as starting any library applet (however, the error viewer will wait to start).
  • Transition to the HOME Menu in this state.
  • Start any library applet from the HOME Menu (this causes the system to stop responding).


13-16S06: (TOOL) Fixed a bug where the +Prod revision number was not displayed in the combo box for selecting firmware type.

Fixed a bug where the +Prod (and +ProdWithLog) revision number was not displayed in the combo box for selecting firmware type.
