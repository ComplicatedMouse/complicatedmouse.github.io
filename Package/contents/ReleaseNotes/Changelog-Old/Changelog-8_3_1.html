Details

Changes in NintendoSDK NX Addon 8.3.1 (based on Generic 8.3.1)

Added Features

8-3-1S01: (MOVIE) Some formats are now supported on the Windows platform.

Playback of WebM containers (with VP9 video and Vorbis audio) is now also supported on the Windows platform.
MoviePlayerSimple is the only sample that supports Windows.

Some bugs remain. For more information, see Known Bugs.
The timing of support for other formats is to be determined.

8-3-1S02: (TOOL) Updated the confirmed operating environment for Retail Interactive Demo applications.

Updated the confirmed operating environment for Retail Interactive Demo applications geared toward NintendoSDK Firmware for NX 8.x.x or later.
Along with this change, updated the tool for creating the asset data for a Retail Interactive Demo application and made it possible to set a master volume.
For more information, see Guide to Setting Up the Environment > Setting Up a Testing Environment for Retail Interactive Demos.

Improvements

*8-3-1S03: (Compiler) Updated to Clang for NX 1.7.9.

Updated Clang for NX to version 1.7.9.
The main changes are as follows.

  • Fixed a bug where the fopen() function did not work correctly in a multithreaded environment.
  • Fixed a bug where SafeStack was dependent on the alignment of memory returned by calloc.


Bug Fixes

8-3-1S04: (DevMenu) Fixed a bug where the installation and removal of content failed in PROD MODE.

Fixed a bug where the installation and removal of content using DevMenu failed if the development hardware was initialized in PROD MODE.

8-3-1S05: (TOOL) Fixed a bug where the check of the NSP file to determine whether the master ROM could be submitted used a size limit for demo software that was different from the size stipulated in the guidelines.

When AuthoringTool ran the get-unpublishable-error command on an NSP file, it checked demo software using a size limit of 1 GiB. That has been corrected to use the size limit of 2 GiB, as stipulated in the guidelines.
