Changes in NintendoSDK NX Addon 7.1.0 (based on Generic 7.1.0)

Added Features

*7-1-0S01: (TOOL) Expanded the remote operation features in Target Manager.

  • Added a remote video feature.
    • In Target Manager, you can now right-click a target and select Remote Video to view a video of the development hardware screen.
    • You can also start or stop recording the video at any time.
  • Added a remote operation feature.
    • You can now right-click a target and select Remote Operation to send button input to the selected target.
    • By using this feature along with the remote video feature, you can also perform screen touch operations from within Target Manager.


7-1-0S02: (DevMenu) Added a feature for getting the IP address assigned to the development hardware.

Added the following commands to DevMenuCommand.

  • network get-current-primary-ip-addr (Gets the IP address assigned to the development hardware.)


For more information about this command, see Features > Controlling the Development Hardware > DevMenuCommand.

7-1-0S03: (DevMenu) Added a feature that makes it possible to download just a single file using the album command in DevMenuCommand.

In the album command in DevMenuCommand, added a feature that makes it possible to download just a single specified file to complement the existing feature for batch-downloading all files.

7-1-0S04: (DevMenu) Added a system setting that controls whether the system wakes up from sleep mode when the charging state changes.

Added a system setting that controls whether the system wakes up from sleep mode when the charging state changes.
This setting is true by default (wake up from sleep mode when the charging state changes). Specify false to prevent the system from waking up from sleep mode when the charging state changes for reasons such as the system being undocked.
You can configure this setting using the following menus and commands in DevMenu and DevMenuCommand.

* In DevMenu, from Device > Sleep Settings > Wake Up on Charging Condition Change.
* In DevMenuCommand, by using the settings set-wakes-at-power-state-change and settings get-wakes-at-power-state-change commands.

7-1-0S05: (EC) In DummyECApplet, added support for displaying the selected user when starting the shop.

In DummyECApplet, added support for displaying the selected user when starting the shop.

Added a feature for enabling or disabling airplane mode to QuickMenu in DevMenuOverlay.
For more information, see Features > Controlling the Development Hardware > DevMenuOverlay.

7-1-0S07: (HID) Added several preset vibration files to the vibration collection.

The vibration collection is a tool for previewing how vibrations and sounds work together on EDEV and SDEV hardware that can play vibrations and audio at the same time.
TargetTools/NX-NXFP2-a64/VibrationCollection/Release/RunVibrationCollection.bat

The vibration collection already provides a set of vibration files and sound files as presets. This update includes several new vibration files that bring the total number of vibration file presets to more than 80.


All of the provided preset vibration files can be used freely in game applications for Switch.
TargetTools/NX-NXFP2-a64/VibrationCollection/Release/Resources/bnvib

For more information, see NintendoSDK Documents > Features > Input Device > hid Library > Tool Reference > Vibration Collection.

Vibration files can also be created using the HidVibrationGenerator sample.
%NINTENDO_SDK_ROOT%¥Samples¥Sources¥Applications¥HidVibrationGenerator
For more information, see the following section in the NintendoSDK API Reference: SDK Sample Backgrounder > HID > Generating Simple Vibration Waveforms From Parameters.

Added a feature for notifying programs that a screenshot was captured or a video was saved by pressing the Capture Button. Use the following functions to enable or disable this notification feature.

  • nn::oe::EnableAlbumImageTakenNotification()
  • nn::oe::DisableAlbumImageTakenNotification()


7-1-0S09: (RunOnTarget) Added the read-log and stop-read-log commands to ControlTarget.

Added the read-log command to ControlTarget. This command can be used to continue receiving logs from the development hardware independently of RunOnTarget execution. You can continue receiving logs across multiple runs of an application.
Also added the stop-read-log command, which stops the log acquisition process initiated by the read-log command.

7-1-0S10: (RunOnTarget) RunOnTarget now returns explicit exit codes.

RunOnTarget now returns explicit exit codes that indicate why a particular process stopped, such as when an application aborts, a network disconnection occurs, or a timeout time is reached.
Also added a one-line log entry to the log output generated when RunOnTarget stops and added the --output-result option, which can be used to write the execution results to file in YAML format.

7-1-0S11: (TOOL) Added a feature for displaying warning messages about using API functions that are unnecessary in production ROMs.

Added a new item for AuthoringTool to check when the --error-unpublishable option is specified.
In particular, added support for displaying warning messages about using functions that are unnecessary in production ROMs.

Added the --system-buffer-binding option to HlslCrossCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe). This option makes it possible to specify bindings used in an HlslccSystemBuffer object when such an object is included.

Bug Fixes

*7-1-0W50: (Audio-ATK) Fixed a bug where pan mode and pan curve settings for stream sounds were not applied.

Fixed a bug where pan mode and pan curve settings for stream sounds were not applied.

Previously, during playback of stream sounds, the pan mode and pan curve settings remained set to Dual and Sqrt(-3dB) regardless of the parameters configured in SoundMaker. Now, the pan mode and pan curve settings configured in SoundMaker are applied.
Note that because the default pan mode and pan curve settings for stream sounds created in SoundMaker are Balanced and Sqrt(-3dB), stream sounds for stereo sound sources exhibit the following changes by default.

  • Stream sounds sound slightly different when panning is applied using a function like nn::atk::SoundHandle::SetPan.
  • Playback volume is 3dB lower.


This bug was also present in NW4F, so now this behavior can be switched on and off while using NW4F compatibility mode.
For more information about porting NW4F data, see the API reference for the following function.

  • nn::atk::SoundSystem::SoundSystemParam::enableCompatiblePanCurve
