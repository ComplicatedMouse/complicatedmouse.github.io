Details

Changes in NintendoSDK NX Addon 1.3.0 (based on Generic 1.3.0)

Specification change that breaks your code

*1-3-0S01: (General) Changed the file structure for the SD card.

The SD card file structure has changed from the format used between NX Add-On 1.1.0 and 1.2.0.

When using an SD card that was used from NX Add-On 1.1.0 to 1.2.0 with a system that was initialized with an NX Add-On 1.3.0 or later package, a dialog box indicating that loading failed will be displayed after entering the Entrance screen in the HOME Menu.
The SD card can be made usable again by executing either the DevMenuCommand sdcard cleanup or the DevMenuCommand sdcard format.
This change does not affect SD cards that were used before NX Add-On 1.0.0.

Added Features

*1-3-0S02: (GFX) Added a tool for HLSL-to-GLSL conversion.

Added a tool SLConverter to convert from HLSL to GLSL.
For more information, see SLConverter.

1-3-0S03: (FS) Added the --minimum-matching-size option for adjusting matching size.

You can now specify the minimum matching size to avoid the problem where matching is overly effective and files become too fragmented when creating patches in the AuthoringTool.
Use --minimum-matching-size to specify a power of 2 greater than 16. If this option is not specified, it will be 32 as with previous versions.

1-3-0S04: (TOOL) Added patch support for the list and extract commands in AuthoringTool.

You can now show and extract information for files that have been patched by specifying options for the list and extract commands in the AuthoringTool. You can also use the list command to show information for files contained only in the patch.

1-3-0S05: (TOOL) Added the getproperty command to AuthoringTool.

The getproperty command can be used in AuthoringTool to display the download size of the title including NSP and the minimum required save data size for playing the title.

Improvements

*1-3-0S06: (General) Restriction on Starting Applications Created With NX Add-On 1.3.x Using Earlier Firmware

Applications created with NX Add-On 1.3.x will not start if the firmware is NintendoSDK Firmware for NX 1.0.0 series firmware. When you try to start an application with this combination, a message is displayed stating that a system update is required. This combination is not present in the retail market, so to test your applications created with NX Add-On 1.3.x, use NintendoSDK Firmware for NX 2.0.0 or a later version.

1-3-0S07: (Display) Improved the connectivity of the CEC feature.

Improved the connectivity of the CEC feature so that various features such as one-touch play and system standby work more reliably.

However, it is not guaranteed that the CEC features would work reliably for all TVs that support CEC.

1-3-0S08: (ERR) Revised the description of nn::err::ShowApplicationError, adding omitted information about the preconditions for calling the function.

Added text noting that the call to nn::err::ShowApplicationError will fail if a language has not been set for the nn::err::ApplicationErrorArg passed to the function's parameter.

1-3-0S09: (FRIENDS) Improved the API documentation associated with communication processing.

Noted that an Internet use request must be made and accepted beforehand.

1-3-0S10: (FS) Added comments for sample demo error handling.

Added comments to describe how to perform error handling for each function in the fs sample demo.

1-3-0S11: (FS) Updated FsAccessLogChecker to version 4.0.

Made the following changes in FsAccessLogChecker 4.0 (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe).

  • Added access log support for nn::fs::MountSaveDataReadOnly().
  • Changed the save data access type to Nand.
  • Improved the message indicating that analysis of invalid access logs has failed.


1-3-0S12: (NVN) Made improvements to the NVN library.

The following improvements have been made to the glslc compiler library.

  • Added an error if applications try to use assembly dumps (the outputAssembly option) in conjunction with multi-threading (enableMultithreadCompilation option). This is currently a limitation of GLSLC.

  • GLSL shaders using NV_fragment_shader_interlock now consume less scratch memory than similar shaders that do not use the feature.

  • Made various programming guide updates.
    • Added new entries to Comparing Compiled Programs for Functional Equality.
    • Clarified that atomic_uint functionality is unsupported.
    • Documented the GLASM limits in the programming guide.


1-3-0S13: (TOOL) Changed the error handling policy for updating firmware with InitializeSdev.exe.

If InitializeSdev.exe is used to initialize various areas and initialization fails for areas that do not directly affect program behavior, a warning is now displayed and the operation continues instead of an error occurring.

Bug Fixes

1-3-0S14: (AUDIO) Fixed a bug where an assertion failure occurred in the PC environment due to the erroneous detection of a precondition violation when two or more channels of audio were played.

Fixed a bug where an assertion failure occurred in the PC environment due to the erroneous detection of a precondition violation when two or more channels of audio were played.
This problem only occurred when audio was played using nn::audio::AudioRenderer. It did not occur when audio was played using nn::audio::AudioOut.
In addition, it did not occur in the Release build because assertions were not enabled for that build target.

1-3-0S15: (DevMenu) Fixed a bug where installing a patch caused the application to no longer start.

Fixed a bug where installing a patch caused the application that was patched to no longer start.

1-3-0S16: (DevMenu) Revised the DevMenuCommand network command.

Fixed a bug where executing the DevMenuCommand network command would initialize the socket library multiple times and cause it to stop working.

1-3-0S17: (Display) Fixed a bug where the development hardware stopped responding when the CEC was enabled and outputting to a specific TV, the channel was changed, and then the TV was turned off.

Fixed a bug where the development hardware stopped responding when the CEC was enabled and outputting to a specific TV, the channel was changed, and then the TV was turned off.

1-3-0S18: (Display) Fixed a bug where the system sometimes stopped responding when docking or undocking the console.

Fixed a bug where docking or undocking the console occasionally caused the system to stop responding.

1-3-0S19: (Display) Fixed a bug where the system would hang if repeatedly changed to sleep mode and then awakened while in TV mode.

There was a bug where a small memory leak occurred when the system was put into sleep mode in TV mode, and repeating this process several thousand times would cause the system to stop responding. This bug has been fixed.

1-3-0S20: (HID) Fixed a bug where Npad could not be operated via InputDirector.

Fixed a bug where Npad could not be operated using an XInput-compatible controller connected via InputDirector.
Official support for Npad by the InputDirector tool will be implemented in a future SDK version.

1-3-0S21: (NFC) Fixed a bug where nn::nfp::ResultNfcDeviceNotFound was returned when other NFC devices were disconnected while nn::nfp::StartDetection() was running.

Fixed a bug where, if the nn::nfp::StartDetection() function was called for a specific NFC device, nn::nfp::ResultNfcDeviceNotFound was returned if other NFC devices were disconnected while the function was running.

1-3-0S22: (NFP) Fixed a bug where nn::nfp::ResultNeedRetry or nn::nfp::ResultNeedRestart was returned when accessing multiple NFC devices simultaneously.

Fixed a bug where nn::nfp::ResultNeedRetry or nn::nfp::ResultNeedRestart was returned when accessing multiple NFC devices simultaneously.

1-3-0S23: (NFP) Fixed a bug where performing restart operations caused by nn::nfp::ResultNfcDeviceNotFound in NfpMulti for multiple NFC devices at the same time caused an abnormal exit.

Fixed a bug where performing restart operations caused by nn::nfp::ResultNfcDeviceNotFound in NfpMulti for multiple NFC devices at the same time caused an abnormal exit.

1-3-0S24: (NFP) Fixed a bug where the nn::nfp::AttachActivateEvent() and nn::nfp::AttachDeactivateEvent() functions returned nn::nfp::ResultNfcDeviceNotFound when the NFC feature was disabled.

There was a bug where nn::nfp::AttachActivateEvent() and nn::nfp::AttachDeactivateEvent() returned nn::nfp::ResultNfcDeviceNotFound instead of nn::nfp::ResultNfcDisabled when the NFC feature was disabled.

1-3-0S25: (NIFM) Fixed a bug where waking from sleep mode took a long time.

Fixed a bug where it took a long time to wake from sleep mode if the USB Ethernet adapter was not connected.

1-3-0S26: (PowerManagement) Fixed a bug where repeatedly pressing the POWER Button or HOME Button during sleep mode could cause the system to be unable to wake from sleep mode.

Fixed a bug where repeatedly pressing the POWER Button or HOME Button during sleep mode could cause the system to be unable to wake from sleep mode.

1-3-0S27: (PowerManagement) Fixed a bug where the charging icon did not disappear even if the PC and development hardware were disconnected.

There was a bug where, if the PC and development hardware were connected using a TypeA-C conversion cable, etc., and the cable was then disconnected, the icon indicating that the device is charging would not disappear.
In addition, putting the development hardware into sleep mode when this symptom occurs causes the system screen to fade to black and stop responding.
This bug has been fixed.

1-3-0S28: (PowerManagement) Fixed a bug where the system would occasionally be unable to wake from sleep mode.

Fixed a bug where the system would continue to show a black screen and be unable to wake from sleep mode even when attempting to wake it.

1-3-0S29: (Socket) Fixed a bug where nn::socket::SendTo() did not return.

Fixed a bug where the function did not return if nn::socket::SendTo() was called frequently.

1-3-0S30: (Socket) Fixed a bug where the nn::socket::Close() function did not return processing.

Fixed a bug where nn::socket::Close() would occasionally fail to return processing.

1-3-0S31: (VSI) Fixed a bug where unnecessary builds were performed when building in VSI.

Fixed a bug where unnecessary builds were performed when building in VSI.

1-3-0S32: (Accounts) Fixed a bug where sometimes character output would not terminate in the sample.

Fixed a bug where strings output by the sample would not terminate and show meaningless data.

Changes in NintendoWare NX Addon 1.3.0 (based on Generic 1.3.0)

Added Features

1-3-0W01: (3D-G3D) Added nn::g3d::ResShaderProgram::IsInitialized().

Added nn::g3d::ResShaderProgram::IsInitialized() to determine whether nn::g3d::ResShaderProgram is initialized.

Bug Fixes

1-3-0W02: (3D-G3D) Fixed a bug where an assertion failure occurred when connected to 3DEditor for the Edit scene of the G3dDemo sample.

Fixed a bug where an assertion failure with the following message would occur when the model or shader was sent from the connected 3DEditor for the Edit Scene of the G3dDemo sample.

Cannot find the appropriate shader type.

1-3-0W03: (3D-G3D) Fixed a bug where an assertion failure occurred when nn::g3d::viewer::ViewerServer::QueueClearMaterialSelectionCommand() was run.

Fixed a bug where an assertion failure would occur when nn::g3d::viewer::ViewerServer::QueueClearMaterialSelectionCommand() was run.

1-3-0W04: (Layout-UI2D) Fixed a bug where the global matrix set by the user was overwritten when nn::ui2d::Pane::CalculateGlobalMatrix was called.

Fixed a bug where the global matrix set by the user was overwritten by the default global matrix when nn::ui2d::Pane::CalculateGlobalMatrix was called.

1-3-0W05: (SPY) Fixed a bug where an exception would cccur if the size of the Spy.exe data file exceeded 2 GB.

Fixed a bug where an exception would occur when disconnecting communication or closing Spy.exe if the Spy.exe data file exceeded 2 GB.

1-3-0W06: (SPY) Fixed a bug where it was not possible to record for more than one hour and 11 minutes in Spy.exe.

Fixed a bug where data could no longer be updated, exceptions occurred, Spy.exe could no longer run normally, and invalid content was recorded in the data file when Spy.exe was connected to the application for more than one hour and 11 minutes.

For this release, all invalid data files recorded up to now cannot be opened normally.
(There is no schedule for fixing the underlying bug.)

1-3-0W07: (SPY) Fixed a bug where Spy.exe would end abnormally when AtkSpy was in use.

Fixed a bug where Spy.exe would end abnormally when Spy.exe was connected to the application, and multiple sequence sounds were played back using nn::atk.
