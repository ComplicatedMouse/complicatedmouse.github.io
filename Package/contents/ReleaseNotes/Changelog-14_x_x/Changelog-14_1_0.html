Details

Changes in NintendoSDK NX Addon 14.1.0 (based on Generic 14.1.0)

Specification change that breaks your code

*14-1-0S01: (EC) Deleted the ec_Result.http.h header file.

The errors defined by ec_Result.http.h do not need to be referenced, so that header file was deleted.

*14-1-0S02: (FS) Delete operations on the root of the drive in the host file system are now precondition violations.

Calling either nn::fs::DeleteDirectory() or nn::fs::DeleteDirectoryRecursively() on the root of the drive in the host file system is now a precondition violation.
In versions earlier than SDK 14.1.0, nothing happened when a delete operation was performed on the root of the drive in the host file system.
To delete all entries in a drive, use the nn::fs::CleanDirectoryRecursively() function.

Added Features

*14-1-0S03: (TOOL) Added a setting to the NMETA file for the target age for applications for consoles for China.

Added a setting to the NMETA file for the target age for applications for consoles for China. A CADPA icon (target age icon) corresponding to this setting is displayed in the HOME Menu when the application is started. This is supported in the HOME Menu in NintendoSDK Firmware for NX 13.2.0-1.0 and later. For more information, see Writing the NMETA File > Target age setting (for China).

14-1-0S04: (AOC) Changed the default setting for the policy for changes to available content to Stable.

The default setting for the policy for changes to available content has been changed to Stable.
For more information, see Writing the NMETA File > Policy Setting for Changes to Available Content.

14-1-0S05: (AOC) In light of the easing of restrictions on user accounts, a setting was added to the NMETA file for setting the policy for changes to available content.

A setting was added to the NMETA file for setting the policy for changes to available content.
With the easing of the restrictions on user accounts, it is now possible to set a policy for changes to available content while the application is running.
For more information, see Writing the NMETA File > Policy Setting for Changes to Available Content.

14-1-0S06: (HID) Added a feature for connecting the Nintendo Switch Pro Controller via a USB connection in the PC environment.

Added support for connecting the Nintendo Switch Pro Controller via a USB connection in the PC environment.
There is no support for connecting the same Nintendo Switch Pro Controller simultaneously via both Bluetooth and USB.

14-1-0S07: (INIT) Added a function that can be used inside nninitStartup().

Added nn::oe::SetCpuBoostMode() as a function that can be used inside nninitStartup().

14-1-0S08: (NVN) New nvn Features

Added the following features to GLSLC.

  • Added support for SPV_EXT_demote_to_helper_invocation and GLSL_EXT_demote_to_helper_invocation.txt.


14-1-0S09: (TOOL) Added a feature for creating an NSP file for secure startup from an NSP file that merges a compacted application and a patch.

Added a feature to the makesecure command of AuthoringTool for creating an NSP file for secure startup from an NSP file that merges a compacted application and a patch. For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


14-1-0S10: (TOOL) Added a feature to display library applets started from the application in remote video.

Added a feature to display library applets started from the application in remote video. For more information, see DevMenuCommand > Development Support Features.

14-1-0S11: (TOOL) Added a feature to Target Manager 2 for adjusting the remote video volume.

Added a feature to Target Manager 2 for adjusting the remote video volume.

14-1-0S12: (TOOL) Added a hardware management log feature to Target Manager 2.

Added a hardware management log to Target Manager 2 for saving a log of the date and time that hardware is connected.

14-1-0S13: (TOOL) Added a link to the manual page in the SDK environment checker.

Added a link to the manual page in the GUI version of the SDK environment checker.
If NintendoSDK Documents is located in the same directory as the SDK, clicking the Help link displays the relevant page in that directory. If it is not located in that directory, the relevant page for the online documentation on the Nintendo Developer Portal is displayed. In either case, the page is displayed in the default browser.

14-1-0S14: (TOOL) Added DumpViewer as a tool for listing NXDMP files.

Added a new development tool called DumpViewer that can be used to list information and screenshots of NXDMP files in the host environment.
For more information, see DumpViewer.

14-1-0S15: (TOOL) Added the application ID to the target log.

When you use Target Manager 2, the application ID can now be displayed in the log that is output from the target (the development hardware).
The application ID can be used for a log filter and also a log trigger.

To use the log's application ID, you must be using version 14.1.0 or later for both the target (development hardware) firmware and Target Manager 2.

Improvements

*14-1-0S16: (BuildSystem) Changed the supported version of MSVC v142 to v14.29-16.11.

The supported version of MSVC v142 was changed to v14.29-16.11 (VcToolsVersion 14.29.30133). Use this version if you are using MSVC v142.

*14-1-0S17: (Compiler) Notice of removal of the ld and gold linkers.

The ld linker and the gold linker, which have been deprecated since NintendoSDK 10.4.0, will be removed from the package starting with NintendoSDK 15.0.0.
If you are still using ld or gold, you must change to the use of lld.

*14-1-0S18: (TOOL) Changed the behavior when installing Target Manager to forcibly close any running instance of Target Manager ahead of time.

Previously, the installation of Target Manager failed if an instance of Target Manager was running. The behavior had been changed, and the running instance of Target Manager now forcibly closes before installation.

14-1-0S19: (Compiler) Updated to Clang for NX 1.11.6.

Updated Clang for NX to version 1.11.6.
The main changes are as follows.

  • clang-shared.dll is no longer included in the package because it is unnecessary for compilation.
  • Several linker scripts in the binutils area have been removed from the package.
  • llvm-objcopy has been improved to make its behavior more like objcopy.
  • Fixed a bug where the build could be non-deterministic when link-time optimization (LTO) or profile-guided optimization (PGO) was used.
  • std::random_device() now uses getentropy().


14-1-0S20: (CPU Profiler) Nintendo NX CPU Profiler Version 1.7

  • Added splitting for "_internal (unresolved)" functions. Function numbers are not comparable between SDK versions or build configurations.
  • Added support for Ring Buffer profiling when taking profiles.
  • Added support for File System Tracing when taking profiles.
  • Added support for Thread-Centric Data Views to the Counters and Instrumented tabs.
  • Added a new memory API to signal an event when Heap Inspector connects.
  • Switched to using Windows PowerShell 5 instead of Windows PowerShell 2.
  • Improved the response time to showing the Select a Scenario window in some situations.
  • Changed the layout of the Sampled and Instrumented Profile tabs.
  • Improved profile length estimates for Sampled profiles.
  • Changed all tabs in the left window to only appear if they contain data.
  • Split data on the Counters tab into Counters and Logs tabs when using Thread-Centric Data Views.
  • When Thread-Centric Data Views is enabled, Core 3 Impact no longer shows in the Counters tab and always renders on the Timeline tab.
  • Increased the number of heaps that can be tracked by the memory API.
  • A small number of memory allocations can now be tracked before Heap Inspector connects.
  • Removed Performance Counters from the instrumentation. Thread execution can move between cores, but the counters are tied to specific cores.
  • Fixed a crash that could occur if an error happened while loading profile data.
  • Fixed various text issues in tooltips.
  • Fixed a bug where some tabs would not fully update the sorted column when changed via scripting.
  • Fixed a bug where the left window tabs had slightly different layouts for the buttons and drop-down menus above the data list view.
  • Fixed a display bug on the Threads tab when a thread was manually moved between cores while profiling.
  • Fixed a crash that could occur on the Instrumented tab when a non- default sort option was used with Show Only Sort Value.
  • Fixed a bug that could cause data on the Overview tab and Checkers tab to use the selected time range instead of the full profile duration.
  • Fixed a bug on the Overview tab that could stop Info from displaying.
  • Fixed a bug that could cause tooltips to be slightly out of place when resting the mouse pointer on code blocks in the Timeline.
  • Fixed sorting issue of items on the Timeline tab when reselecting items.
  • Fixed a display issue for Sample Gaps that made it look like the gap occurred earlier than it did.
  • Fixed a timeout that occurred if no allocations were tracked and Heap Inspector was disconnected and then reconnected.
  • Fixed a bug where heap names were sometimes not visible in Heap Inspector.


14-1-0S21: (DevMenu) Added support to Install via HTTP > Automatic for deleting all history.

Added support to Install via HTTP > Automatic for deleting all history.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Managing Applications > Installing HTTP Server NSP Files > Automatic.


14-1-0S22: (DevMenuCommand) Added support for launch parameters when using DevMenuCommand to start NSP and NSPD files.

You can now used the application launch command of the DevMenuCommand tool to specify launch parameters when starting NSP and NSPD files.
For more information, see DevMenuCommand > Running and Controlling Applications.

14-1-0S23: (FS) Improved the error message in FsAccessLogChecker version 41.9 when analysis results are entered.

In version 41.9 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe), the error message that appears when analysis results are entered as the input file has been modified to be more understandable.

14-1-0S24: (FS) When nn::fs::CreateFile() fails, the file that was being created is now deleted in the host file system.

Previously, when the nn::fs::CreateFile() function failed, the file that was being created remained in the host file system. That has been changed and that file is now deleted.

14-1-0S25: (Sample) Added a way to exit the HidGesture sample.

Added support for existing the HidGesture sample by selecting the Exit button.
Along with this change, log entries are added when the application starts and ends.

14-1-0S26: (Sample) Added support for C++14 features to the HidNpadIntegrate sample.

Added support for C++14 features to the HidNpadIntegrate sample.

  • Replaced some of the for statements with range-based for statements.
  • Converted some of the const constants to constexpr.
  • Replaced the NULL macro with nullptr.
  • For some variable declarations, replaced the specification of a specific type name with auto.
  • Replaced NN_NOEXCEPT with the C++ standard noexcept keyword.
  • Replaced NN_OVERRIDE with the C++ standard override keyword.
  • Replaced the conventional enum type with enum class.
  • Changed to the use of the default declaration instead of empty constructors and destructors.
  • Some of the variables that were dynamically allocated and released using the new and delete operators have been replaced with unique_ptr and shared_ptr.


14-1-0S27: (SSL) OpenSSL has been updated to version 1.1.1l.

The OpenSSL library has been updated to version 1.1.1l to include the latest vulnerability and bug fixes.

14-1-0S28: (TOOL) Added a feature to Target Manager 2 for exporting a ZIP file that includes the collection of dump files for bug reports and the log files.

Added a feature to Target Manager 2 for exporting a ZIP file that includes the collection of dump files for bug reports and the log files.

14-1-0S29: (TOOL) Added a feature to Target Manager 2 for restoring the target name to the default value in the SDEV settings and the USB development hardware settings.

Added a feature to Target Manager 2 for restoring the target name to the default value in the SDEV settings and the USB development hardware settings.

14-1-0S30: (TOOL) Added support to Target Manager 2 for displaying an icon for unsupported targets.

A warning icon is now displayed when you connect development hardware with an unsupported firmware version to Target Manager 2.

14-1-0S31: (TOOL) Changed the default addr2line in the crash report retrieval tool to llvm-addr2line.exe.

Changed the default setting for addr2line used for symbol resolution from the conventional aarch64-nintendo-nx-elf-addr2line.exe to the faster llvm-addr2line.exe.

14-1-0S32: (TOOL) In AuthoringEditor, the list of middleware no longer shows NintendoSdk_BuildTarget-64Bit or NintendoSdk_BuildTarget-32Bit.

In SDK 13.0.0 and later, the internal value NintendoSdk_BuildTarget-64Bit or NintendoSdk_BuildTarget-32Bit is used for determining the platform type (NX32 or NX64). Previously, those values were shown in the middleware list in AuthoringEditor.
That has been revised and the middleware list in AuthoringEditor no longer shows those values.

14-1-0S33: (TOOL) The analyze-patch command of AuthoringTool now shows whether application compaction has been performed.

The analyze-patch command of AuthoringTool has been improved to show whether application compaction has been performed.

14-1-0S34: (TOOL) USB targets are now found if newly recognized while searching with the detect-target subcommand of ControlTarget in the htc-gen2 environment.

USB targets that are newly recognized while searching with the detect-target subcommand of ControlTarget in the htc-gen2 environment can now be found, whereas previously they were not found.

Bug Fixes

14-1-0S35: (DevMenu) Fixed a bug in DevMenu where a fatal error occurred when content in an application was deleted.

Fixed a bug where on rare occasions DevMenu ended on a fatal error (error code 2162-0002) when content in an application was deleted using the screen for managing application content in the Application tab of DevMenu.

There was a bug in FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) where an incorrect warning message was added to the access log of the nn::fs::IsMounted() function prior to mounting. This bug was fixed in version 41.10 of the tool.

Fixed a bug where an access log was not generated when the nn::fs::ExtendSaveData() function was called on save data that was being mounted.

14-1-0S39: (IRSENSOR) Corrected the omission of the irsensor library's pointing mode functions from the API Reference.

14-1-0S40: (Sample) Fixed a bug where Visual Studio 2019 could not build the Release x64 configuration of the TmRemoteVideo sample.

14-1-0S41: (TOOL) Corrected typos in the error messages of RunOnTarget and ControlTarget.

Corrected typos in the error messages of RunOnTarget and ControlTarget.

The wrong symbols were displayed when the crash report retrieval tool resolved symbols, due to a problem in the parsing of inline functions that caused symbols at other addresses to also be included. The parsing process was revised so symbols are displayed correctly even if there are inline functions.

14-1-0S43: (TOOL) Fixed a display bug that occurred when Target Manager 2 was used in a high DPI environment.

Fixed a bug where parts of the UI were not drawn correctly when Target Manager 2 was used in an environment with a high DPI monitor.

    • Windows were displayed in shifted positions if set to hide automatically.
    • Windows did not match the screen size when maximized.


14-1-0S44: (TOOL) Fixed the error that occurred when the remote video API was called right after nn::tm::CloseTargetVideoStream().
