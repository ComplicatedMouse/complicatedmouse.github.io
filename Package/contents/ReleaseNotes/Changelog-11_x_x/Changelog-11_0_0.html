Details

Changes in NintendoSDK NX Addon 11.0.0 (based on Generic 11.0.0)

Specification change that breaks your code

*11-0-0S01: (DIAG) Eliminated the Log Viewer tool.

The Log Viewer tool has been eliminated.

Use Target Manager instead of Log Viewer from now on.

Initially, Target Manager only supported viewing logs generated in real time, and it did not support access to the log's metadata, viewing logs saved to files, or filtering logs.

Because Target Manager did not support those features, they were provided by the Log Viewer tool. However, Target Manager now supports those features, and so Log Viewer now has been removed from the SDK package.

*11-0-0S02: (General) Ended support for Visual Studio 2015.

Ended support for Visual Studio 2015, as scheduled.

*11-0-0S03: (Socket) Call nn::socket::Initialize() before using the socket API.

For Windows and Linux builds, you now must call the nn::socket::Initialize() function before using the functions of the socket library. In addition, after calling that function, you must call nn::socket::Finalize().

*11-0-0S04: (Socket) Removed the inclusion of old nn::socket APIs from the current nn::socket API headers.

Previously, if an application included nn/socket.h, nn/socket/socket_Api.h and nn/socket/socket_ApiDeprecated.h were also included. nn/socket/socket_ApiDeprecated.h is no longer included in nn/socket/socket_Api.h.

This change fixes an issue where developers would sometimes run into symbol collisions on Windows builds because ws2tcpip.h ended up being included. This change affects all platforms, not just Windows. In addition to fixing symbol collisions, the goal of this change was to separate the older API from the newer API for deprecation purposes.

Developers may notice a build failure if they are using functionality defined in nn/socket/socket_ApiDeprecated.h. We recommend changing the code to use only the newer API defined in nn/socket/socket_Api.h. However, if that option is not feasible, another solution is to include nn/socket/socket_ApiDeprecated.h directly. We do not recommend this solution. Only use it temporarily, because the plan is to deprecate everything in nn/socket/socket_ApiDeprecated.h.

*11-0-0S27: (CRYPTO) Renamed functions for hash computations and HMAC computations.

The functions in the nn::crypto namespace for hash computations and HMAC computations have been renamed, and for this reason the following functions have been deprecated.

  • nn::crypto::GenerateMd5Hash
  • nn::crypto::GenerateSha1Hash
  • nn::crypto::GenerateSha256Hash
  • nn::crypto::GenerateSha512Hash
  • nn::crypto::GenerateHmacSha1Mac
  • nn::crypto::GenerateHmacSha256Mac


From now on, use the functions with these new names instead.

  • nn::crypto::GenerateMd5
  • nn::crypto::GenerateSha1
  • nn::crypto::GenerateSha256
  • nn::crypto::GenerateSha512
  • nn::crypto::GenerateHmacSha1
  • nn::crypto::GenerateHmacSha256

11-0-0S30: (Compiler) Updated to Clang for NX 1.9.4.

Updated Clang for NX to version 1.9.4.
The main changes are as follows.

  • Updated to LLVM 9.0.0.
  • Added official support for ThinLTO.
  • Fixed a bug where an assertion failure occurred on rare occasions if std::condition_variable::wair_for() was called with a timeout specified.

- The source of this fix was that libcxx was changed to get the time using nn::os::GetSystemTickOrdered() instead of nn::os::GetSystemTick().

  • Reduced memory consumption by UBSan. All memory required at startup is now allocated ahead of time.
  • Fixed a bug where the program crashed if a negative value was passed to _nnmusl_socket_RemoveNativeSocketFileDescriptorFromNnmuslTable().
  • Fixed a bug where sometimes EBUSY instead of EEXIST was set for errno when the open() function failed.
  • Added the definition for the deprecated symbol _ALIGN.
  • Updated MAX_SOCKETS_PER_CLIENT to match the NintendoSDK definition.
  • Fixed a bug where ENOENT was not set for errno when access() was called on an invalid path.
  • Fixed a bug where the nx_socket_init() function did not work in the NintendoSDK environment.
  • Crash reports are now generated when the sanitizer fails to allocate memory.
  • NN_ABORT is now called when finalization is triggered by the sanitizer.
  • symtab is now stripped by llvm-objcopy when --strip-unneeded is specified.
  • lld now uses an implementation of Visual Studio-style diagnostics via upstream LLVM, making for some minor differences in format.

11-0-0S39: (SSL) Enhanced the nn::ssl::Connection:: SetServerCertBuffer() and DoHandshake() functions of the SSL API.

These changes are documented in the nn\ssl\ssl_Connection.h header file.
No new functions were added and the number of parameters did not change.
