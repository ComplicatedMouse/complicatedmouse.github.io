Details

Changes in NintendoSDK NX Addon 11.3.0 (based on Generic 11.3.0)

Specification change that breaks your code

*11-3-0S01: (libcurl) Disabled DNS cache in libcurl.

Observations indicate that libcurl uses its own DNS cache and subsequent calls use this cache. By default, the curl cache expires every 60 seconds, or the time indicated by the value set in CURLOPT_DNS_CACHE_TIMEOUT. This improvement disables the curl DNS cache and uses the expected Nintendo SDK behavior where records are expired from the Nintendo resolver cache at the lowest TTL. It also disables the previously mentioned option by returning CURLE_NOT_BUILT_IN if it is used.

*11-3-0S02: (libcurl) Revised the handling of proxy settings in libcurl to not implicitly reference environment variables.

Modified libcurl so the proxy settings are not implicitly obtained via environment variables when CURLOPT_PROXYAUTOCONFIG is enabled, which is the default behavior. NIFM (system) proxy settings are preferred over environment variables.

This preference may affect applications that explicitly set proxy settings via environment variables. Application developers are strongly encouraged to always use CURLOPT_PROXYAUTOCONFIG.

*11-3-0S03: (TOOL) Changed the format of the configuration files for the htc-gen2 tools.

The format of the configuration files for Nintendo SDK Daemon and Target Manager 2 was changed in preparation for the coming version upgrades.

If you have changed from version 11.2.0 or earlier to version 11.3.0 or later, the settings are migrated to the new format automatically when you start the tool for the first time.
The migration process requires no particular steps on your part.

Note the following information if you intend to use an earlier version after using version 11.3.0 or later.

  • The settings for versions 11.2.0 and earlier are managed separately from the settings for versions 11.3.0 and later. The settings configured with versions 11.3.0 or later of the tool are not applied to versions 11.2.0 and earlier of the tool.

The layout files created from the View menu of Target Manager 2 11.3.0 and later cannot be used in Target Manager 2 11.2.0 and earlier.

Added Features

*11-3-0S04: (APPLET) Added a function the program can call to restart itself.

Added the oe::RestartProgram() function as a way for the program to restart itself. For more information, see Restarting the Program.

11-3-0S05: (DIAG) Added a class for getting the information saved in stack frames.

Added the nn::diag::Backtrace class for getting the information saved in stack frames while following the backtrace.

For more information, see the API reference for nn::diag::Backtrace.

11-3-0S06: (HID) Added a feature for pseudo attachment and detachment of the Joy-Con while it is attached to the rail.

Added a feature for pseudo attachment and detachment of the Joy-Con while it is attached to the rail of the SDEV or EDEV. You can use this feature when developing remotely to test operations without physically detaching the Joy-Con. For more information, see DevMenuCommand > Managing Controllers.

11-3-0S07: (HID) Added the functionality to assign the same function as the Capture Button to other buttons.

Added the nn::hid::SetNpadCaptureButtonAssignment() function to assign the same function as the Capture Button to other buttons.
This feature is designed to be used in games with special methods of operation where it is difficult to operate the Capture Button on the Joy-Con (L).

11-3-0S08: (NVN) New NVN Features

Added the following features to NVN.

  • Added functions that provide GPU progress marker support for NVN Aftermath, but these are no-op until NVN Aftermath is released.
    • nvnCommandBufferInsertCheckpointStatic
    • nvnCommandBufferInsertCheckpointDynamic
    • nvnDeviceSetAutomaticCheckpointsEnable
    • nvnDeviceSetAutomaticCheckpointsSamplingInterval
    • nvnDeviceSetAutomaticCheckpointsByDebugGroup
    • nvnDeviceSetCheckpointCallStacksEnable
    • nvnDeviceSetCheckpointMaxCallStacksDepth


11-3-0S09: (TOOL) Added the set-host-htc-generation command to ControlTarget.

Added the set-host-htc-generation command to ControlTarget for setting the htc-generation on the host side.

11-3-0S10: (UTIL) Added a function to the nn::util::BitArray class for running specified functions for indexes set with the 1 bit.

Added the nn::util::BitArray::ForeachIfOne() function to the nn::util::BitArray class for running specified functions for indexes set with the 1 bit. For more information, see the API Reference.

Improvements

*11-3-0S11: (MOVIE) Fixed a bug where an error occurred during video playback due to depletion of the allocatable global heap.

Decreased the amount of core heap memory used by Movie::Extractor in the movie library. Memory was previously allocated from the core heap, but that has been changed to use the multimedia heap. Along with this change, the programming guide has also been changed.

This change was included in NX Add-On 11.2.0 but was inadvertently omitted from the revision history.

11-3-0S12: (Compiler) Updated to Clang for NX 1.9.8.

Updated Clang for NX to version 1.9.8.
The main changes are as follows.

  • Reorganized the library's internal implementation.


11-3-0S13: (DevMenu) Made an improvement to update the content of the Debug page of DevMenu when you switch to the page.

Made an improvement to update the content of the Debug page of DevMenu when you switch to the page. If the settings in the Debug page have been changed using an external tool like DevMenuCommand or DevMenuApp, those changed settings are applied when you switch to the Debug page.
If the applied settings include settings that require the development hardware to be restarted, a message to that effect is displayed.

11-3-0S14: (DevMenuOverlay) CPU overclocking is now indicated in DevMenuOverlay.

The specification has been changed so that now, when the CPU clock rate increases due to CPU overclocking, that increase is indicated in the ClockRateGraph of DevMenuOverlay.

11-3-0S15: (FS) Organized the fs library Result values, and made it possible to search for private error Result values in NintendoSDK Documents.

Organized the fs library Result values, and modified the Result Value Search feature of the NintendoSDK Documents package to enable searches for some error Result values of privately defined functions of the fs library.
Although private Result values cannot be used in the program, the generated error Result values can be used to investigate the causes of errors.

11-3-0S16: (NVN) Improvements in NVN

Made the following improvements to the nvn library.

  • Added NVN support for the Windows 452.44 driver. This build is based on NVIDIA driver 451.48. This package implements version 53.313 of the NVN API.

  • Added debug layer validation of depth/stencil format.


11-3-0S17: (RenderDoc) Updated the UI.

  • Changed the Default SDEV label to Default Target.


11-3-0S18: (SSL) Support for RSA-PSS has been added for the device client certificate.

Support for RSA-PSS has been added for the device client certificate (when using nn::ssl::Context::RegisterInternalPki()). This change enables the device client certificate to be used with TLS 1.3.

11-3-0S19: (TIME) Publicly released the location name representing Coordinated Universal Time (UTC).

Publicly released the following location name (nn::time::LocationName) representing UTC.

  • nn::time::UtcLocationName


11-3-0S20: (TOOL) A warning message is now displayed if the application's own ApplicationId is set for SaveDataOwnerIds.

Added a check when AuthoringTool commands are run with the --error-unpublishable option, and when the get-unpublishable-error command is run.
A warning message is now displayed when the NSP file meets all of the following conditions.

  • The application's own ApplicationId is set in the main program's SaveDataOwnerIds.
  • The application is not a multi-program application.

For more information about SaveDataOwnerIds, see fs Library > Manual > Feature Description > Save Data > Save Data (NX Add-On) > Referencing Save Data From Other Applications.

11-3-0S21: (TOOL) Added a feature to Target Manager 2 for transferring software keyboard operations.

Added a feature to Target Manager 2 for transferring software keyboard operations to the remote video view.

11-3-0S22: (TOOL) Added functionality for changing the Nintendo SDK Daemon server setting.

Previously, the setting for the server that the Nintendo SDK Daemon opens on the host PC had been fixed to 127.0.0.1:20182. That process has been revised, and you can now change the setting from the Settings menu in the task tray.
In addition, a feature was added for changing the connection target setting of the Nintendo SDK Daemon to which Target Manager 2 connects.

11-3-0S23: (TOOL) Added support for displaying information about on-card downloadable content NSP files and multi-application card NSP files in AuthoringEditor.

When the following types of NSP files are opened in AuthoringEditor, information related to the application, patch, or downloadable content now is displayed.

  • On-card downloadable content NSP file
  • Multi-application card NSP file


11-3-0S24: (TOOL) The crash report retrieval tool now expands and displays inline functions.

The crash report retrieval tool now expands and displays inline functions when it displays stack traces.

Bug Fixes

*11-3-0S25: (Settings) Fixed a bug where the language code comparison method was incompatible.

Restored the former method for comparing nn::settings::LanguageCode objects. This method compares strings in the internal data, rather than comparing all of the internal data. For more information, see the manual.

11-3-0S26: (DevMenu) Fixed a bug where a UI button on the screen remained highlighted if an input operation for that button was performed with a different system.

Fixed a bug where a UI button on the screen remained highlighted if the action of that UI button was executed via a controller hardware button while that UI button was being touched. The following example shows this kind of operation.

  • The A Button of the controller is pressed while some button on the Application page (such as the Install from SD button) is being touched.


11-3-0S27: (DevMenu) Fixed a bug where account management information was not updated.

Fixed a bug where the following account management information was not updated.

  • If Add an user account and launch was selected, the application was started, and then you returned to DevMenu, the added user account was not displayed in the list.
  • When the Accounts tab was selected from another tab, the user information was not updated.


11-3-0S28: (DevMenu) Fixed a bug where sometimes the highlighting remained after you tapped a UI button on the screen.

Fixed a bug where sometimes the highlighting remained after you tapped a UI button on the screen.

11-3-0S29: (DevMenu) Fixed a bug where unintended actions occurred when you tapped the application list on the Application page.

Fixed a bug where unintended actions occurred when you tapped the application list on the Application page.
For example, if you pressed and released the + Button on the controller when the Application tab was in focus and then tapped the application list, the action of tapping the list did not execute, and instead the action of pressing the + Button executed and the Options screen appeared.

11-3-0S30: (DIAG) Fixed a bug in the Generic version of the SDK where the end of the log was not exported if a long module name was specified for NN_STRUCTURED_LOG().

Fixed a bug in the Generic version of the SDK where the end of the log was not exported if a long module name was specified for NN_STRUCTURED_LOG().

11-3-0S31: (EC) Fixed a bug in the EcCatalog sample where items were not displayed when Related title / bundle list was set to on sale items only.

Fixed a bug in the EcCatalog sample where no items were displayed if on sale items only was selected for Related title / bundle list, or the sequence returned to the same selection (on sale items only).

11-3-0S32: (FS) Fixed a bug where FsAccessLogChecker crashed on the access log of the nn::fs::GetFreeSpaceSize() function.

Fixed a bug where FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) crashed on the access log of the nn::fs::GetFreeSpaceSize() function. This bug was fixed in version 39.0.

11-3-0S33: (libcurl) Fixed a bug where a HTTP/2 transfer timeout caused all future transfers using Curl_easy to time out.

Previously, transfers sometimes repeatedly timed out on one Curl_easy session and then continue to time out until the application was restarted.

11-3-0S34: (MOVIE) Fixed a bug where resource leaks occurred in the Windows version.

Fixed a bug where resource leaks occurred in the Windows version of the movie API.
Memory leaks no longer occur even if movie:Extractor is deleted before the media is played.
This fix also resolved a bug where multiple threads (named ucrtbase.dll) remained instead of being destroyed.

You can check the Windows version of the MoviePlayerSimple sample program to see how mm_NvnVideoRenderer.cpp has been changed to affect this bug fix.

11-3-0S35: (NVN) NVN Bug Fixes

Fixed the following NVN bugs.

  • Fixed some dead links.


Fixed the following bugs in glslc.

  • Fixed a few dead links in the programming guide.


11-3-0S36: (PL) Fixed a bug that made nn::pl::SetUsbDetachedForDebug() unusable.

Fixed a bug where the nn::pl::SetUsbDetachedForDebug() function did not work when htc-gen2 was used.

11-3-0S37: (TOOL) Fixed a bug in remote operations with Target Manager 2 where controllers that were not selected were connected when reconnecting.

Fixed a bug in remote operations with Target Manager 2 where selecting a controller that was not a Switch Pro Controller, disconnecting it, and then reconnecting it instead connected a controller that was not selected.

11-3-0S38: (TOOL) Fixed a bug in remote operations with Target Manager 2 where the Joy-Con dual-controller grip stick operations did not work correctly after the Joy-Con was switched to solo horizontal grip.

Fixed a bug in remote operations with Target Manager 2 where the Joy-Con dual-controller grip stick operations did not work correctly after the Joy-Con was switched to solo horizontal grip.

11-3-0S39: (TOOL) Fixed a bug in the filtering process of the analyze-patch command of AuthoringTool.

Fixed a bug in the analyze-patch command of AuthoringTool where files and directories smaller than the size specified by the filter were displayed.

11-3-0S40: (TOOL) Fixed a bug where AuthoringTool mistakenly detected an unpublishable error when the debug API was used in the earlier version.

Fixed a bug where AuthoringTool mistakenly detected an unpublishable error in patch NSP files created under the following conditions.

  • An NSP file is specified for the initial application or the previous patch in which the debug API was used.
  • An NSP file is specified for a revised version of the application in which the debug API was not used.


11-3-0S41: (TOOL) Fixed a bug where patch sizes were larger than expected.

Fixed a bug where patches ended up being larger than expected if there were a lot of changes between versions of the revised application and it was difficult to perform binary matching on the parts with updated content.

11-3-0S42: (TOOL) Fixed a bug where Target Manager 2 crashed when a log was copied.

Fixed a bug where Target Manager 2 sometimes crashed if you copied a log when the target log had reached the maximum number of display lines.

11-3-0S43: (TOOL) Fixed a bug where Target Manager 2 did not start properly if multiple remote video windows were docked.

Fixed a bug where Target Manager 2 did not start properly after multiple remote video windows were docked to the main window.

11-3-0S44: (TOOL) Fixed a bug where the set-target-htc-generation and set-target-htc-generation commands failed.

Fixed a bug where the set-target-htc-generation and get-target-htc-generation ControlTarget commands failed if the target was not registered in both Target Manager and Target Manager 2 when the commands were run.



11-3-0S45: (TOOL) TargetManager and VSI Bug Fixes

  • Fixed a bug where the memory for use by Target Manager was not released correctly when using VSI 2019.
  • Fixed a bug where inserting breakpoints and running Step Over led to odd behavior.
  • Fixed a bug where the CDR document did not link correctly to the page for use of the symbol server.


11-3-0S46: (UTIL) Fixed a bug in nn::util::DivideUp().

Added a precondition that dividends and divisors be in ranges that do not cause computation overflow.
Fixed a bug in the PC environment where, depending on the argument type, the constexpr nn::util::DivideUp() function returned incorrect values when values of 0x80000000 or greater were passed to the arguments.

11-3-0S47: (UTIL) Fixed a bug where nn::util::pmr::NewDeleteResource() sometimes did not satisfy the alignment request.

Fixed a bug where the pointer to the nn::MemoryResource instance returned by the nn::util::pmr::NewDeleteResource() function did not satisfy the alignment request if the requested memory region had a size of 0 and an alignment that was greater than alignof(std::align_max_t).

Changes in NintendoWare NX Addon 11.3.0 (based on Generic 11.3.0)

Added Features

11-3-0W01: (3DTool) Added a feature for adding a user-customized UI to the options dialog box in the Photoshop texture export plug-in.

Added a feature for adding a user-customized UI to the options dialog box in the Photoshop texture export plug-in.

11-3-0W02: (3DTool) Added a feature to the Maya plug-in to convert polygons to triangles using the same pattern as Maya.

Added a feature to the Maya plug-in to convert polygons with four or more sides to triangles using the same pattern as Maya.
To enable this feature, specify optimize_triangulation="dcc" in the config file.

11-3-0W03: (3DTool) Added a feature to the Maya plug-in to prioritize the Maya texture SRT when merging FMD files.

Added a feature to the Maya plug-in to prioritize the Maya texture SRT when merging FMD files.
Specify texture_srt_merge_priority="true" to prioritize the Maya texture SRT.

Improvements

11-3-0W04: (3DTool) In the Maya plug-in, you can now specify a preset name with the third argument of NintendoExportByFdes3.

Added -preset (-prs) to the options that can be specified with the third argument of NintendoExportByFdes3, making it possible to specify present names in the Maya plug-in.

11-3-0W05: (3DTool) Revised the Photoshop texture export plug-in so Post-Export Script can now be run before the PSD file is saved.

Revised the Photoshop texture export plug-in so Post-Export Script can now be run before the PSD file is saved.
To run Post-Export Script before saving the PSD file, set save_psd and execute_script_before_saving_psd to true in the config file.
If the PSD file is large and takes time to save, these settings can shorten the time until the script is run.

11-3-0W06: (3DTool) The Maya multi-export plug-in can now process the FDES filename as the preset name when you use an FDES file in the presets folder.

The Maya multi-export plug-in now can process the FDES filename as the preset name when you use the FDES file in the presets folder (the preset="" file).
To process the FDES filename as the preset name, specify multi_export_use_fdes_name_as_preset="true" in the config file.
It is used for branching the processing of Pre-Export Script and Post-Export Script based on the preset name.

11-3-0W07: (LayoutTool) Added a feature for specifying the kinds of panes to exclude from Align Pane processing in the LayoutEditor project settings.

Added a feature for specifying the kinds of panes to exclude from Align Pane processing in the LayoutEditor project settings. For more information, see {{DocLink＝124016278}}Project Configuration File Format{{/DocLink}}.

11-3-0W08: (3DTool) Fixed a bug in the Photoshop texture export plug-in where some environment variables became undefined when SetEnv() was called from Pre-Export Script or Post-Export Script.

Fixed a bug in the Photoshop texture export plug-in where environment variables like NINTENDO_EXPORT_TGA_FOLDER became undefined when SetEnv() was called from Pre-Export Script or Post-Export Script.

11-3-0W09: (LayoutTool) Fixed a bug in LayoutEditor where textures were added to the layout when the overwriting of parts involved objects other than textures.

Fixed a bug in LayoutEditor where unused textures were added to the layout when the overwriting of parts involved objects other than textures. In addition, fixed a bug where LayoutEditor crashed when unused textures were deleted.

11-3-0W10: (LayoutTool) Fixed a bug where LayoutEditor crashed if a combination of a large variety of groups was set to animation segment tags.

Fixed a bug where LayoutEditor crashed if a combination of a large variety of groups was set to animation segment tags.

Fixed a bug where the appearance in LayoutEditor was different from that in LayoutViewer when different constant colors were set for RGB and A using an advanced combiner in LayoutEditor.
