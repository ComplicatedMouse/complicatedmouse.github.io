Changes in NintendoSDK NX Addon 11.2.0 (based on Generic 11.2.0)

*11-2-0S01: (Socket) Deleted the Nn_So_ShutdownExempt option.

Deleted the Nn_So_ShutdownExempt socket option. This feature will no longer be available, so revise any applicable code that uses it.

*11-2-0S02: (SSL) Deleted nn::ssl::BuiltInManager.

Deleted the nn::ssl::BuiltInManager class. This feature will no longer be available, so revise any applicable code that uses it.

Added functionality to use DevMenuApp for secure startup in the HOME Menu environment.

  • Features > Controlling the Development Hardware > DevMenuCommand > Managing Downloadable Content > Installing and Uninstalling.

Added the delete subcommand to the album command in DevMenuCommand as a feature to delete specific album files in the target device.

Added a feature to the EcCatalog sample to get all sale status information by specifying onsale_only = false when getting the AOC Item List and Related Title/Bundle List information.

Updated Clang for NX to version 1.9.7.

  • Fixed a bug where an error occurred when the number of characters in the command line was exactly 32,768.


11-2-0S14: (CPU Profiler) Fixed an out-of-process profiling issue with threads that have affinity to more than one core.

When using out-of-process profiling, it was possible for a thread with a core affinity to more than one core to move between cores between when the profiler received notification of which thread was executing and its ability to read the call stack. As a result, the thread could resume execution while sampling was in progress, potentially resulting in recording incorrect call stacks.

There is a similar known issue when one core is executing a thread using significantly more stack space than the other executing threads. We are still working on a fix for the stack disparity issue.

11-2-0S15: (CPU Profiler) Nintendo NX CPU Profiler Version 1.4.1

  • Changed the assignable classifications to be consistent throughout the product.
  • Fixed a compatibility issue with compiler version 1.9.6.
  • Fixed a bug that could cause Target Manager to crash on Sync.
  • Fixed a bug where the default sort of Self on the Functions tab did not sort correctly.
  • Fixed a bug where the Functions (new) tab would reselect deselected functions when changing the grouping mode.
  • Fixed a bug caused by the Functions (new) tab where selecting a function could result in extra functions also getting selected.
  • Fixed a bug with the Functions (new) tab where filtering could show functions that did not match the filter.
  • Fixed a bug where the Functions (new) tab would not expand to show functions selected by a script.
  • Fixed a bug where the Functions (new) tab would not change highlight colors after overriding a classification.
  • Fixed memory leaks on the Timeline tab.


11-2-0S16: (DevMenu) Changed the method for displaying the error description when account operations fail.

11-2-0S21: (FS) Added support to FsAccessLogChecker version 35.1 to not count write operations to resource data (ROM) and downloadable content.

Because writing to storage media does not actually occur when nn::fs::FlushFile is called on resource data (ROM) and downloadable content, support was added to FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 35.1 to not count these as write operations.
However, from a performance standpoint, we recommend reducing calls to nn::fs::FlushFile() on resource data (ROM) and downloadable content.

Bug Fixes

*11-2-0S37: (SSL) Restricted SSL negotiations for SSL versions earlier than the version specified in nn::ssl.

Previously, nn::ssl did not restrict connections using SSL versions earlier than the version specified in nn::ssl::Context. For example, even if nn::ssl::Context::SslVersion_TlsV12 was specified, it was possible to negotiate with SslVersion_TlsV11 or even earlier versions. Applications built with SDK 11.2.0 and later only perform negotiation with the specified version, and negotiation fails if that version is not supported.

11-2-0S38: (AUDIO) Fixed the behavior of nn::audio::OpenAudioRenderer() when there is no audio device in Windows.

Fixed a bug where the nn::audio::OpenAudioRenderer() function did not return nn::audio::ResultNoAudioDeviceFound() when no audio device was available in the Windows environment.
This bug first appeared in NintendoSDK 10.0.0.

11-2-0S39: (DevMenu) Fixed a bug where the system could not enter sleep mode or shut down while an application was starting.

Fixed a bug where the development hardware could not enter sleep mode or shut down while an application was starting. This fix applies to the following screens.

  • The user-selection screen when starting
  • The passphrase screen for secure startup

  • Fixed a bug with reporting higher than required scratch memory for some shaders.

  • Fixed a bug where querying an array within a struct using a dynamic index could return the wrong value.

  • Fixed a bug where GLSLC spun in an infinite loop while compiling certain SPIR-V shaders generated from other tools.


11-2-0S41: (TOOL) Fixed a bug in the analyze-patch command of AuthoringTool.

Fixed a bug in the analyze-patch command of AuthoringTool where the incorrect size would sometimes be reported.

Fixed a bug where it was possible to downgrade the version of SDK or AuthoringTool when creating a patch. An error now occurs in the following situations.

Changes in NintendoWare NX Addon 11.2.0 (based on Generic 11.2.0)

Added Features

11-2-0W01: (3D-G3D) Added a feature for mirroring bone visibility animations.

Added a feature to bone visibility animations to flip the animated bones horizontally. A model's skeleton must satisfy certain conditions for this feature to be available. For more information, see Bone Visibility Animation > Mirroring, (Maya) Motion Mirroring, and Motion Mirroring Settings Plug-In.

11-2-0W02: (3DTool) Added a feature for previewing mirrored bone visibility animations.

Added a feature to 3DEditor for previewing the mirroring of bone visibility animations.

Improvements

11-2-0W03: (3D-G3D) Revised the LOD export feature of the Maya Plug-in so the number of submesh divisions of the LOD model is always fewer than the number of submesh divisions of the base model.

Revised the LOD export feature of the Maya Plug-in so when submesh division is performed, the LOD model always ends up with fewer submesh divisions than the base model.

11-2-0W04: (3DTool) Added an environment variable to the Maya multi-export plug-in for using Pre-Export Script and Post-Export Script to get the export item index that is currently being exported.

Added an environment variable (NINTENDO_MULTI_EXPORT_CURRENT_ITEM_INDEX) to the Maya multi-export plug-in for using Pre-Export Script and Post-Export Script to get the export item index that is currently being exported.

11-2-0W05: (3DTool) Made exporting a little faster when the output of FTX files is disabled in the Photoshop Texture Export plug-in.

Made exporting a little faster when the output of FTX files is disabled in the Photoshop Texture Export plug-in.
This is more effective when the image size is larger.

11-2-0W06: (3DTool) Made improvements to the intermediate file optimizer so submesh division is applied to the LOD mesh when submesh division is preformed at the same time that the LOD models are combined.

Made improvements to the intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe) so submesh division is applied to the LOD models that are being combined when --divide-submesh and --unite-lod-model are executed simultaneously. The meshes at higher LOD levels have fewer submesh divisions.

Bug Fixes

11-2-0W07: (Audio-ATK) Fixed a bug where noise occurred when the stream jump feature and the file read cache feature were used together.

Fixed a bug where noise occurred in the channel when the stream jump feature and the file read cache feature were used together if the block at the end of the region was small.

11-2-0W08: (Audio-ATK) Fixed a bug where using the prefetch feature on small stream sound files caused an assertion failure.

Fixed a bug where using the prefetch feature for stream sound files smaller than the size of the stream buffer (40 KB × number of waveform channels in the stream sound × stream buffer size multiplier) caused an assertion failure.
This fix changes the conversion results of BFSTM, BFSTP, and BFSAR files, so reconvert your sound projects.
Reconverting the BFSTM and BFSTP files may increase the file sizes by as much as 512 bytes.
You can still use the BFSTM, BFSTP, and BFSAR files from sound projects that were converted with previous versions, but the bug will be reproduced.

11-2-0W09: (3DTool) Fixed a bug in Photoshop plug-ins where it was not possible to save (export) an image with a width and height of 16,384.

Fixed a bug in the Photoshop FTX File Format plug-in and Texture Export plug-in where images with a width and height of 16,384 could not be saved (exported).
