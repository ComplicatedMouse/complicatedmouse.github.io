Details

Changes in NintendoSDK NX Addon 11.4.0 (based on Generic 11.4.0)

Specification change that breaks your code

*11-4-0S01: (FS) FsAccessLogChecker version 40.0 supports self-restart.

FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 40.0 supports self-restart.
For more information about this self-restart feature, see oe Library > Manual > Features > Switching Between Programs.
FsAccessLogChecker input assumes an access log obtained from SD card, output by development hardware initialized with NintendoSDK Firmware for NX 11.0.0-6 or later.
Access logs obtained from development hardware initialized with versions earlier than NintendoSDK Firmware for NX 11.0.0-6 and access logs obtained from log output can be analyzed, but do not include logs for Switching Between Programs. As a result, a warning message is displayed.

*11-4-0S02: (FS) Revised the description of fs Result in the API reference.

Removed a reference to nonexistent results in the details for fs Result in the API reference.
nn::fs::ResultHandledByAllProcess and nn::fs::ResultHandledBySystemProcess were removed in SDK 11.3.0. Do not use them for handling.

*11-4-0S03: (General) Updated the version of Microsoft .NET Framework required for the Nintendo SDK operating environment to 4.7.2.

Updated the version of Microsoft .NET Framework required for the Nintendo SDK operating environment from 4.6.1 to 4.7.2.
As part of this change, the version of the Microsoft .NET Framework checked in the SdkEnvironmentChecker was also updated from 4.6.1 to 4.7.2.

For more information about SdkEnvironmentChecker, see Checking the Suitability of the Operating Environment.

*11-4-0S04: (NGC) Renamed the South American Portuguese pattern list available in nn::ngc to the "Brazilian Portuguese" list.

As part of this language settings change to Brazilian Portuguese, the pattern list for South American Portuguese of invalid strings that can be handled in nn::ngc has been renamed to Brazilian Portuguese. Similarly, the nn::ngc::ProfanityFilterPatternList enumerated type ProfanityFilterPatternList_SouthAmericanPortuguese has been changed to ProfanityFilterPatternList_BrazilianPortuguese.

Added Features

*11-4-0S05: (TOOL) Added a crash screenshot to the crash report.

Screenshots can now be captured when the application crashes and kept in the crash report. The image files are saved in JPEG format.
Saving screenshots is disabled by default. For more information, including enabling it, see Crash Reports > Screenshots.

Screenshots can be retrieved using the crash report retrieval tool (CrashReport.exe). For more information about using the tool, see Crash Reports > Crash Report Retrieval Tool.

*11-4-0S06: (TOOL) Added Brazilian Portuguese to the system settings languages.

Added Brazilian Portuguese to the system settings languages.
As a result of this fix, the nmeta file title, icon, and supported language can be configured for each language. In addition, setting the preceding language in system settings causes nn::oe::GetDesiredLanguage() to return the preceding language for applications where it is set as a supported language.

For more information about configuring using DevMenu and DevMenuCommand, see the following pages.

  • DevMenu > Changing Device Settings
  • DevMenuCommand > Changing the System Settings


Brazilian Portuguese is available for NX Add-On 11.4.0 and NX Add-On 10.9.0 and later.
However, if the target distribution format includes packaged version, see the separate Nintendo Developer Portal notifications when applications and patches using Brazilian Portuguese can be submitted.

11-4-0S07: (DevMenu) Added a feature to control settings for capturing screenshots for crash reports.

Added a feature to control settings around capturing screenshots for crash reports to each of DevMenu and DevMenuCommand.
For more information, see Crash Reports > Screenshots.

11-4-0S08: (DevMenu) Added a function to temporarily allow a development device to sleep while in a secure startup state.

Bug Fixes

*11-4-0S31: (libcurl) Fixed the issue with curl_easy_pause() that could potentially block other threads.

Fixed a bug where curl_easy_pause() could block another thread from running for too long if a calling thread is given the same thread priority as the other thread.

*11-4-0S32: (TOOL) Deprecated initialization through nn::socket in the library when using htc or htcs with htc-gen2 in a PC environment.

In the library, nn::socket was used to initialize when using htc or htcs in a Windows environment set to use Target Manager 2 (htc-gen2).
However, this process was not appropriate because nn::socket initialization parameters different from those intended by the developer of the application could be used.
To solve this issue, initialization through nn::socket in the library has been deprecated. Applications are responsible for handling initializing nn::socket.
To make existing code work, as a temporary measure htc and htcs will continue to initialize nn::socket if the app has not initialized nn::socket. In this case, a warning message is displayed when the application is run.

Changes in NintendoWare NX Addon 11.4.0 (based on Generic 11.4.0)

Added Features

11-4-0W01: (3DTool) Added a feature for editing vertex colors to NintendoLiveEditingTestTool.

Added a feature to the NintendoLiveEditingTestTool (%NINTENDO_SDK_ROOT%\Tools\Graphics\MayaPlugins\Scripts\NintendoLiveEditingTestTool.py) that allows edits to a color set that are made while mesh editing is enabled to be applied in the corresponding vertex colors for the transferred model.

11-4-0W02: (3DTool) Added assign-original-curve subcommand to AssignUtility.

Added an assign-original-curve subcommand to AssignUtility (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe). You can use this subcommand to assign original curves in the material animation intermediate file to the specified shader parameter animation.

Improvements

11-4-0W03: (3DTool) Changed the procedure for running the multi-export Maya plug-in to return the number of export items for which export failed.

Changed the procedure for running the multi-export Maya plug-in to return the number of export items for which export failed. It returns -1 if the process was canceled by the overwrite verification or pre_multi_export_script.

  • NintendoMultiExportDirect()
  • NintendoMultiExportDirect2()

In addition, added a procedure to get the index array for successful and failed exports when doing the final export.

  • NintendoMultiExportGetSucceededItemIndices()
  • NintendoMultiExportGetFailedItemIndices()


11-4-0W04: (3DTool) Modified the Maya multi-export plug-in to show an error instead of a warning if the FDES file or node does not exist.

Instead of a warning message, an error is now displayed in the Maya multi-export plug-in if the FDES file or node specified for the export item does not exist.

11-4-0W05: (Layout-UI2D) Added a function to specify the number of frames to advance when updating the layout.

You can now specify the number of frames to advance in the arguments for nn::ui2d::viewer::Viewer::UpdateLayout.

Bug Fixes

11-4-0W06: (Audio-ATK) Fixed a bug where the single playback function for sounds in the additional sound archive did not work.

Fixed a bug where the single playback function for sounds in the additional sound archive did not work.

11-4-0W07: (3DTool) Fixed a bug in 3DEditor where texture transfer sometimes did not finish when sending a model in 3DEditor.

Fixed a bug where progress sometimes stopped with a message about texture transfer in progress when submitting a model in 3DEditor.

11-4-0W08: (3DTool) Fixed a bug where disconnecting and reconnecting after performing specific actions in 3DEditor sometimes resulted in an error.

Fixed a bug in 3DEditor where disconnecting from an application after attaching a model or shader and then reconnecting sometimes caused an error and failure to connect.

11-4-0W09: (3DTool) Fixed a bug where the 3D Binary Converter would not convert material animation user data.

Fixed a bug where user data (<user_data_array> in <material_anim>) for material animation intermediate files (FMAA, FMAB) was not converted by the 3DBinaryConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe).

11-4-0W10: (3DTool) Fixed a bug where the getLastError command could not get the last error when using the Maya export plug-in in batch mode.

Fixed a bug where the getLastError command could not get the last error when using the Maya export plug-in in batch mode.

11-4-0W11: (3DTool) Fixed a bug where updating material animation intermediate files with an additional search path set in the intermediate file updater failed.

Fixed a bug in the intermediate file updater (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFIleUpdater.exe) where the "global name 'AddAttr' is not defined" error occurred if material animation intermediate files (FMAB and FMAA files) with an additional search path configured were updated.

11-4-0W12: (EffectTool) Fixed a bug where using EffectMaker with a system locale other than Japan generated incorrect shader code.

When generating EffectMaker shader code, some locations used the code page CP932 and some used the code page matching the system locale. Shader code was sometimes not generated correctly for system locales other than Japan.
These locations were unified to CP932, and shader code is now generated correctly.
