Details

Changes in NintendoSDK NX Addon 11.4.3 (based on Generic 11.4.3) FW 11.0.1-1.2

Improvements

*FW11-0-1-1-2S01: (TOOL) Added a feature that checks which controller buttons need to be pressed to unlock the device when you are using DevMenuCommand to disable the device lock feature.

Added a feature to use when you are disabling the device lock feature with DevMenuCommand that checks which controller buttons were set as the buttons that need to be pressed to unlock the device when the device lock feature was enabled.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenuCommand > Changing the System Settings > Device Lock Setting.

If you have forgotten which controller buttons were set for unlocking the device, initialize the firmware and disable the device lock feature.

FW11-0-1-1-2S02: (DevMenu) Made improvements so that secure startup can also be used in PROD mode.

Made improvements so that secure startup can also be used in PROD mode.
Although the procedures for creating, installing, and running content for secure startup are the same as before, some restrictions have been placed on other operations.
For more information, see Features > Controlling the Development Hardware > DevMenu > Managing Applications > Secure Startup.

Bug Fixes

FW11-0-1-1-2S03: (DevMenu) Fixed a bug where the application did not restart when selecting to restart the application from Error Viewer in the DevMenu environment.

Fixed a bug where the application did not restart when selecting to restart the application from Error Viewer in the DevMenu environment.
