Details

Changes in NintendoSDK NX Addon 12.3.0 (based on Generic 12.3.0)

Added Features

*12-3-0S01: (General) Added Visual Studio 2019 MSVC v142 support.

Added Visual Studio 2019 MSVC v142 support.
Libraries built with Visual Studio 2019 MSVC v142 have been added to the NintendoSDK, and builds using Visual Studio 2019 MSVC v142 are now possible.
For more information, see Important Information When Using Visual Studio 2019.

*12-3-0S02: (TOOL) Added application compaction feature.

Added a feature to compact applications and create applications with low storage consumption by removing the data from the initial version of an application that is no longer referenced due to patch fixes.
For more information, see Application Compaction.

12-3-0S03: (DevMenu) Added a feature for outputting a log on Target Manager when the main unit sleeps due to high temperature of the unit or other reasons.

In the DevMenu environment, when the main unit sleeps due to high temperature of the unit or other reasons, a record of this is output to the log in Target Manager.

12-3-0S04: (DevMenuCommand) Added the ability to measure network bandwidth to the network command.

Added the following commands as a way to measure network bandwidth.

  • network measure-band-width-for-download [--timeout <em>time_in_msec</em>]
  • network measure-band-width-for-upload [--timeout <em>time_in_msec</em>]


For more information about these commands, see Features > Controlling the Development Hardware > DevMenuCommand > Getting Network Information.

12-3-0S05: (TOOL) Added support for associating NSP files with Target Manager 2.

NSP files can now be associated with Target Manager 2, allowing NSP files to be run directly on the target from Explorer, even in the htc-gen2 environment.

NSP files can be associated with Target Manager 2 on starting the application.

12-3-0S06: (Album) Added a feature to change the save location for screenshots and video clips to the host PC.

Added a feature to DevMenuCommand to change the album file save directory to a directory on the host PC. For more information, see DevMenuCommand > Managing Saved Album Files.

12-3-0S07: (Album) Added a feature to save 0-byte application screenshots and video clips.

Added a command to DevMenuCommand to force file size for JPG and MP4 files created as screenshots and video clips to 0 bytes. For more information, see DevMenuCommand > Development Support Features.

Improvements

12-3-0S08: (CODEC) Included libnn_codec.lib and libnn_codec_aac.lib in libnnSdk.lib.

libnn_codec.lib and libnn_codec_aac.lib are no longer provided individually, and are instead now provided as part of libnnSdk.lib.
libnnSdk.lib is linked automatically by the NintendoSDK property sheet.
For more information, see How the Libraries Are Provided and How to Use Them.

12-3-0S09: (Compiler) Updated to Clang for NX 1.10.5.1.

Updated Clang for NX to version 1.10.5.1.
The main changes are as follows.

  • Fixed a bug where weak symbols were sometimes not linked correctly when using ThinLTO.


12-3-0S10: (DevMenu) Improved procedures for temporarily allowing sleep while in a secure startup state.

Unified the secure startup procedures for temporarily allowing sleep in DevMenu and DevMenuApp.
For more information, see Features > Controlling the Development Hardware > DevMenu > Managing Applications > Secure Startup.

12-3-0S11: (DevMenu) Save data for secure startup is now created only when using secure startup.

Save data for secure startup is now created only if the NSP file for secure startup has been installed.
Save data for secure startup configuration is displayed on the save data management screen with the name "Setting data for Secure Startup."

12-3-0S12: (FRIENDS) Removed friends library sample for Windows.

The friends library sample for Windows did not behave as expected and has been removed.

12-3-0S13: (FS) Improved error message on abort.

Tweaked the error message output on abort.
The error message is added to the access log for the SdCard settings before the Develop build aborts.

12-3-0S14: (IMAGE) Included libnn_image.lib in libnnSdk.lib.

libnn_image.lib is no longer provided individually, and is instead now provided as part of libnnSdk.lib.
libnnSdk.lib is linked automatically by the NintendoSDK property sheet.
For more information, see How the Libraries Are Provided and How to Use Them.

12-3-0S15: (SSL) Adding common root CA to the SSL built-in store.

Adding common root CA to the SSL built-in store. For more information, see the SSL documentation.

12-3-0S16: (TOOL) A warning message is now displayed if Target Manager 2 connects to an instance of the Nintendo SDK Daemon started by a different user.

A warning message is now displayed if Target Manager 2 connects to an instance of the Nintendo SDK Daemon started by a different user.

12-3-0S17: (TOOL) Added support for displaying IARC Generic in AuthoringEditor only if the sales region is Japan.

AuthoringEditor now displays IARC Generic only if the sales region is Japan. In addition, Japan is added to the sales regions if IARC Generic is set in the NSP file.

12-3-0S18: (TOOL) Changed window/tab display for Target Manager 2.

To make identification easier when there are multiple tabs, the way windows and tabs are displayed in Target Manager 2 has been changed as follows.

  • Display icon on window/tab
  • For windows/tabs displayed for a specific target, display the target name first


12-3-0S19: (TOOL) DevMenuApp is now also updated via SystemUpdateXdev.exe.

When updating with SystemUpdateXdev.exe, DevMenuApp is also now updated.

12-3-0S20: (TOOL) Tightened NMETA file check when running AuthoringTool.

Tightened NMETA file check when running AuthoringTool and modified it so that an error occurs if no valid value is configured.

Bug Fixes

12-3-0S21: (AUDIO) Fixed a bug where a portion of the constants in Audio Renderer could not be used.

A constant indicating the maximum amount of time the audio coprocessor may use to process a single audio frame was added in NX Add-On 12.0.0, but could not be used. This bug has been fixed and the constant may now be used.

This constant was descried in the API reference, but the defining header file was not included in the SDK.

  • nn::audio::AudioCoprocessorFrameTimeLimit


12-3-0S22: (AUDIO) Fixed a bug where the work buffer used by AuxLightLimiterType was not aligned correctly.

Fixed a bug where the work buffer used by nn::audio::AuxLightLimiterType was not aligned correctly.
This fix increases the work buffer size required by nn::audio::GetRequiredBufferSizeForAuxLightLimiter() to nn::audio::BufferAlignSize - 1 bytes.

12-3-0S23: (NIFM) Fixed a bug where the transition to wired connection link down was not correctly detected.

Fixed a bug where FW11 systems could not detect when a wired connection went link down due to a failure on the connected device or LAN cable disconnect.

12-3-0S24: (NVN) Fixed a bug where some files were not included in the API reference file list.

The following files were not included in the API reference file list and have now been added to the file list.

  • %NINTENDO_SDK_ROOT%\Include\nvn\nvn_CppFastpath.h
  • %NINTENDO_SDK_ROOT%\Include\nvn\nvn_DeviceConstantsNX.h
  • %NINTENDO_SDK_ROOT%\Include\nvn\nvn_Fastpath.h
  • %NINTENDO_SDK_ROOT%\Samples\Sources\Applications\NvnTutorial02OfflineBuildTools\WaveFrontObjFileManager.cpp
  • %NINTENDO_SDK_ROOT%\Samples\Sources\Applications\NvnTutorial02OfflineBuildTools\WaveFrontObjFileManager.h


12-3-0S25: (Sample) Fixed a bug where the memory used by nn::profiler was not allocated when using the NvnSimpleSharedLib sample library's USE_NXCP pre-processor.

Fixed a bug where the memory used by nn::profiler was not allocated when using the NvnSimpleSharedLib sample library's USE_NXCP pre-processor.

12-3-0S26: (TOOL) Fixed a bug in AuthoringEditor that caused an error when opening an NSP file on a multi-application card that did not contain any downloadable content.

Fixed a bug in AuthoringEditor that caused an error when opening an NSP file on a multi-application card that did not contain any downloadable content.

12-3-0S27: (VSI) VSI and Target Manager Bug Fixes

  • VSI
    • Fixed a bug where thread priority core was not displayed correctly in the dump file.
    • Fixed usage of the ApplicationDataDir property in other project properties (NRODeployPath in particular).
    • Fixed a bug where an incorrect thread status was displayed when using the cdr library
    • Fixed a bug where the status of a waiting thread was incorrectly displayed as running in the VSI NX thread window.
    • Fixed a bug that made step debugging in Debug builds slow when using the -fsanitize=float-divide-by-zero, integer-divide-by-zero, or null options.
  • Target Manager
    • Fixed application crashes caused by improper file resource management.


Changes in NintendoWare NX Addon 12.3.0 (based on Generic 12.3.0)

Added Features

12-3-0W01: (3DTool) Added support for material intermediate file input to the assign-search-path subcommand in the assign utility.

Added support for material intermediate (FMTB, FMTA) file input to the assign-search-path subcommand in the assign utility (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe).

Bug Fixes

12-3-0W02: (Audio-ATK) Fixed a bug where the initial portion of a sequence sound exported using SoundMaker's waveform conversion feature is sped up.

Fixed a bug where the initial portion of a sequence sound exported using SoundMaker's waveform conversion feature is sped up.
This bug first appeared in NintendoSDK 6.0.0.

12-3-0W03: (3DTool) Fixed a bug in Intermediate File Converter where TGA files could not be converted.

Fixed a bug in Intermediate File Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileConverter.exe) where opened TGA files were determined to have an unsupported file extension.

12-3-0W04: (3DTool) Fixed a bug that caused an error when 3dShaderConverter was copied out of the NintendoSDK directory.

Fixed a bug that caused an error when copying 3dShaderConverter ( %NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.dll) out of the NintendoSDK directory and then running it.

12-3-0W05: (3DTool) Fixed a bug where the <material> elements of model intermediate files exported using the intermediate file converter were not sorted according to specifications.

Fixed a bug in the intermediate file converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileConverter.exe) where the <material> elements contained in exported model intermediate files were not sorted according to the intermediate file specifications.

12-3-0W06: (Layout-UI2D) Fixed a bug where an unintended rectangle was rendered when the alpha value for the capture effect boundary was zero.

Fixed a bug where an unintended rectangle was rendered in some situations even when the capture effect boundary setting had an alpha value of zero.

Changes in Target Manager NX Addon 12.3.0 (based on Generic 12.3.0)

Bug Fixes

12-3-0T01: (FS) Fixed a bug where you could not specify a specific path in the file system on the host PC.

Fixed a bug where the correct processing was not executed if you used "/" as the UNC path delimiter for functions operating in the host PC system.
