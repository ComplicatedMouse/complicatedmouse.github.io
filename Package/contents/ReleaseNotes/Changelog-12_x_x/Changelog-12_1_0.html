Details

Changes in NintendoSDK NX Addon 12.1.0 (based on Generic 12.1.0)

Specification change that breaks your code

*12-1-0S01: (NIFM) Split up the libnn_nifm library and added the libnn_ndim library in the Windows environment.

Split up the libnn_nifm library and added the libnn_ndim library in the Windows environment.
Developers who construct their own build systems without using the standard SDK property sheet and are linking libnn_nifm.lib now also need to link libnn_ndim.lib.
Developers using the SDK standard property sheet do not need to do anything in particular because the added library is included as a target of automatic linking in the property sheet.

This change was included in NX Add-On 12.0.0 but was omitted from that revision history.

*12-1-0S02: (NVN) Breaking Changes in the nvn Library

The following breaking changes were made to the nvn library.

  • Removed nvnDeviceApplyDeferredFinalizes and the corresponding device flag bit (DEFERRED_FINALIZE).

  • Removed nvnQueueAcquireTexture.

  • Removed DEFERRED_FIRMWARE_MEMORY_RECLAIM_BIT.

  • Removed the DEBUG_ENABLE, DEBUG_DRAW_VALIDATION, and DEBUG_DRAW_VALIDATION_HEAVY device flag bits.

  • Introduced a separate DLL for NVN in the Windows reference implementation. This DLL allows the NVN driver to be updated without needing to install the complete driver.
    • For more information about using the nvn DLL, see the programming guide section "Bootstrapping NVN on Windows."

    • Changed texture fetch for depth/stencil to return (Z, 0, 0, 1.0) or (S, 0, 0, 0x00000001), respectively.


*12-1-0S03: (Socket) nn::socket::Connect() now triggers EInProgress on nonblocking sockets in Windows.

Previously, calling nn::socket::Connect() on a nonblocking socket in Windows could trigger nn::socket::Errno::EAgain. That bug was fixed, and instead it triggers nn::socket::Errno::EInProgress to more accurately correspond to the NX behavior.

Added Features

12-1-0S04: (AUDIO) Added a feature to display the final output volume of the development hardware in an overlay.

Added a feature to display the final output volume of the development hardware in an overlay.
For more information about using the overlay feature, see Controlling the Development Hardware > DevMenuOverlay.

12-1-0S05: (Compiler) Added documentation about the feature for converting symbol names into hash values.

Added the Compiler > Toolchain for NX > Converting Symbol Names Into Hash Values page to the documentation to describe the compiler feature for converting any function name or variable name into a hash value.

12-1-0S06: (Graphics) Added the aftermath library.

Added the aftermath library, which is a tool for getting a detailed GPU crash dump.

  • Include/aftermath/aftermath.h
  • aftermath-target.nso
  • aftermath-target.nss


For more information about the aftermath library, see the documentation in the Low-Level Graphics Debugger package.

12-1-0S07: (HID) Input Director now supports the transfer of input from the six-axis sensor of the Nintendo Switch Pro Controller.

Input Director now supports the transfer of input from the six-axis sensor of the Nintendo Switch Pro Controller. This makes it possible to use a Nintendo Switch Pro Controller connected to the host PC to operate the six-axis sensor for the application on the development hardware.

12-1-0S08: (NVN) Added a NATVIS file that visualizes the contents of NVN API objects.

Added a NATVIS file to %NINTENDO_SDK_ROOT%/Resources/VisualStudio/NX/Visualizers/nvn.natvis that visualizes the contents of NVN API objects.
This information is automatically referenced by VSI for NX if you are using a version of NintendoSDK that has this file.

12-1-0S09: (NVN) Added nvn.dll and nvn.lib from the nvn library for Windows.

Added nvn.dll and nvn.lib from the nvn library for Windows to %NINTENDO_SDK_ROOT%/Tools/Graphics/NvnLoader.
For more information about using these, see the NVN Programming Guide.

12-1-0S10: (NVN) New NVN Features

Added the following features to NVN.

  • Ampere GPU support for the Windows reference implementation.

  • Inlined versions of several NVN API calls that result in calling _fastpath versions.

  • Addition of the nvnTextureBuilderGetViewOffset functionality.

  • Added Visual Studio debugger support to view NVN data structures on both Windows and NX with the addition of Natvis files to the release distribution.


Added the following features to GLSLC.

  • Added the #pragma directive packunpackpreservedenorm.

  • Added new define GLSLC_COMP_GPU_CODE_SECTION_DATA_MAGIC_NUMBER for compute shader GPU data to the public GLSLC API.


12-1-0S11: (TOOL) Added a feature to CrashReport.exe to display the actual cumulative operating time of the application when there is a crash.

Added a feature to CrashReport.exe to display the actual cumulative operating time of the application when there is a crash.
When you use CrashReport.exe to get crash reports recorded with version NintendoSDK 12.1.0 or later of the firmware, the application's cumulative operating time at the time of the crash (equivalent to nn::oe::GetProgramTotalActiveTime is displayed.

12-1-0S12: (TOOL) Added the Microsoft DirectX Shader Compiler to the SDK.

The Microsoft DirectX Shader Compiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\dxc.exe) has been added to the SDK. This tool allows you to convert HLSL shaders to SPIR-V, which can then be consumed by other tools. It is an alternative to the HLSLCrossCompiler. Which tool is better for converting HLSL depends on the specific shader. Generally, dxc may work better for recent shaders using DirectX 11 features, and hlslcc may work better for earlier shaders.

12-1-0S13: (UTIL) Added an overload of nn::util::VectorIsZero(const Vector4fType&).

Added an overload of the nn::util::Vector4fType::VectorIsZero() function that specifies a margin of error and determines whether a vector is a zero vector.

  • Added support for BuildTools 2019.

    • VSI now supports the use of normal operations to open the new NXDMP file format. In the future, this format of NXDMP will be included in crash reports.

Improvements

12-1-0W09: (3DTool) Improved NintendoLiveEditingTestTool to apply the attribute information described in the config file.

Revised NintendoLiveEditingTestTool (%NINTENDO_SDK_ROOT%\Tools\Graphics\MayaPlugins\Scripts\NintendoLiveEditingTestTool.py) so that the material_attr_info_ノードタイプ keyword settings described in the config file of the export plug-in are applied during synchronization of texture SRT animations.

Bug Fixes

12-1-0W10: (3D-G3D) Fixed a bug where entering the Simple scene in G3dDemo caused an assertion failure.

Fixed a bug where entering the Simple scene on the Generic Win32 platform in G3dDemo caused an assertion failure.

12-1-0W11: (3DTool) Fixed a bug in 3D Binary Converter where conversion failed for models that include vertices that are not referenced by shapes.

Fixed a bug in 3D Binary Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe) where an error occurred when converting model intermediate (FMDA and FMDB) files that include vertices not referenced by shapes (there is a <vertex> that was never referenced by <vertex_index> in <shape_info>).

12-1-0W12: (3DTool) Fixed a bug in 3DEditor where loading files used in material references caused the settings to not be applied.

Fixed a bug in 3DEditor where, when loading files for which the material reference feature is set to Require for Child Materials only, the setting was disabled without being applied.

12-1-0W13: (3DTool) Fixed a bug in the Maya multi-export plug-in where the Get Clip button did not work correctly for clips configured with speed curve.

Fixed a bug in the Maya multi-export plug-in > Input Export Items window where the Get Clip button did not work correctly for clips configured with speed curve.

12-1-0W14: (3DTool) Fixed a bug in the Photoshop normal map filter plug-in where specifying a file path that included the % symbol caused it to crash.

Fixed a bug in the Photoshop normal map filter plug-in where, when combing with a normal map from a different file, specifying a file path that included the % symbol caused it to crash.
Added the ability to use file paths that include %-enclosed environment variables.

12-1-0W15: (3DTool) Fixed a bug where the binormals in the model intermediate file exported by the intermediate file converter were incorrect.

Fixed a bug in the intermediate file converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileConverter.exe) where, when converting from FBX files to model intermediate files, the binormals in the model converted model became incorrect depending on the input file.

12-1-0W16: (3DTool) Fixed a bug where vertex_skinning_count was not converted when merging shapes in Intermediate File Optimizer.

Fixed a bug in the shape compression feature of the intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe) where processing did not run for converting shapes with the same vertex_skinning_count even when original_bone_name matches when specifying --ignore-skinning-count. This could create gaps between the same shape with different materials.

12-1-0W17: (Layout-UI2D) Fixed a bug where the process stopped when the size of the pane for which capture effects was set was negative.

Fixed a bug where the process of generating a texture was stopped when the size of the pane for which capture effects were set was set to a negative value.

12-1-0W18: (LayoutTool) Fixed a bug in LayoutEditor where the key frames of the reference color for the color animation of the color overlay of the rounded corners in the Picture Pane were not saved.

Fixed a bug in LayoutEditor where the key frames of the reference color for the color animation of the color overlay of the rounded corners in the Picture Pane were not saved.
