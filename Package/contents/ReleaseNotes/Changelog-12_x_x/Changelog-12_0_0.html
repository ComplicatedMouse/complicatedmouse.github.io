Changes in NintendoSDK NX Addon 12.0.0 (based on Generic 12.0.0)

Specification change that breaks your code

*12-0-0S01: (GFX) Improved the interface of the nn::gfx::util::PrimitiveShape and its related classes.

The following changes have been made to the interface of nn::gfx::util::PrimitiveShape and its related classes.

  • Deprecated the nn::gfx::util::PrimitiveShapeAttribute enumerated type.
  • Changed the <em>vertexFormat</em> type accepted by the constructor of the PrimitiveShape class group from the nn::gfx::util::PrimitiveShapeFormat enumerated type to the int type.
    • Deprecated constructors that accept the existing nn::gfx::util::PrimitiveShapeFormat enumerated type.
  • Added <em>stackCount</em> as the number of slices for the height direction to the constructor arguments in the nn::gfx::util::HemiSphereShape class.
    • Deprecated constructors that only specify the existing <em>sliceCount</em>.
  • The IsPrimitiveShapeTopologySupported() function determines the primitive topology supported by the PrimitiveShape class group, excluding the nn::gfx::util::PrimitiveShape class. Added this function as a static member function of each class.
  • NN_NOEXCEPT is specified for all member functions included in the PrimitiveShape class group and the nn::gfx::util::DebugFontTextWriter class.
    • When inheriting these classes and overriding virtual functions, NN_NOEXCEPT must also be specified in the derivative parts.


The preceding PrimitiveShape class group refers to the following classes.

  • nn::gfx::util::PrimitiveShape
  • nn::gfx::util::SphereShape
  • nn::gfx::util::CircleShape
  • nn::gfx::util::CubeShape
  • nn::gfx::util::QuadShape
  • nn::gfx::util::HemiSphereShape
  • nn::gfx::util::PipeShape
  • nn::gfx::util::CylinderShape
  • nn::gfx::util::ConeShape


*12-0-0S02: (GFX) Modified the normal line of the top and bottom surfaces of the nn::gfx::util::CylinderShape class.

Lines normal to the circumference of the top and bottom surfaces of cylinders generated with the nn::gfx::util::CylinderShape class were implemented in a way that caused them to tilt toward the outside of the circle instead of being parallel to the y-axis, causing a bug where the rendered object appeared incorrect.
To fix this bug, the implementation was changed so that all normal lines fitting the preceding criteria are now parallel to the y-axis.

Note that due to this change, rendering results of codes that use the normal line information of primitives generated by the nn::gfx::util::CylinderShape class are expected to change.

*12-0-0S03: (GFX) Modified the UV specification for child classes that inherit the nn::gfx::util::PrimitiveShape class.

Modified the UV specification of the following class.

  • nn::gfx::util::PipeShape
    • Modified the UV of each vertex so that the cylindrical projection follows the circumference.
  • nn::gfx::util::Cylinder class
    • Modified the UV of each vertex so that the cylindrical projection follows the circumference.
    • Modified the UV of each vertex so that the UV of the top and bottom surfaces are equivalent to the nn::gfx::util::Cube class.
  • nn::gfx::util::Cone class
    • Modified the UV of each vertex so that the cylindrical projection follows the circumference.
    • Modified the UV of each vertex so that the UV of the bases are equivalent to the nn::gfx::util::Cube class.


Note that due to this change, rendering results of codes that use the UV information of primitives generated by the applicable classes are expected to change.

The new UV specification and previously unlisted specifications have been added to the API reference of the following classes.

  • nn::gfx::util::PrimitiveShape
  • nn::gfx::util::SphereShape
  • nn::gfx::util::CircleShape
  • nn::gfx::util::CubeShape
  • nn::gfx::util::QuadShape
  • nn::gfx::util::HemiSphereShape
  • nn::gfx::util::PipeShape
  • nn::gfx::util::CylinderShape
  • nn::gfx::util::ConeShape


*12-0-0S04: (TOOL) Calling nn::tm::OpenTargetVideoStream() in advance is now a requirement for using htc-gen2 nn::tm::SaveRecentVideo().

Changed htc-gen2 so that calling the nn::tm::OpenTargetVideoStream() function in advance is now a requirement for using the nn::tm::SaveRecentVideo() function.
The same behavior applies to the Nintendo.Tm.Target.SaveRecentVideo() function in Nintendo.Tm.dll.

Calling the nn::tm::OpenTargetVideoStream() function is listed as a necessary precondition of nn::tm::SaveRecentVideo().
However, nn::tm::SaveRecentVideo() could be used without satisfying the precondition in each of the following situations.

  • The period after calling the nn::tm::StartVideoRecording() function but before calling the nn::tm::StopVideoRecording() function
  • While the remote video window is open in Target Manager


By allowing the API to function without satisfying the preconditions, the API succeeded or failed depending on the situation, which sometimes caused issues during debugging.
For this reason, in htc-gen2, satisfying preconditions is now a requirement.

Note that when migrating to htc-gen2, code that does not satisfy the preconditions will no longer work.

*12-0-0S12: (SSL) Changed the behavior of nn::ssl::Context::SslVersion_Auto to include TLS 1.3 for SDK 12.0.0 or later.

Even though SslVersion_TlsV13 has been available to choose explicitly, SslVersion_Auto previously only selected up to TLS 1.2. Now SslVersion_Auto selects up to TLS 1.3 if the SDK version is 12.0.0 or later and the FW version is 12.0.0-1.0 or later. If either the SDK or FW versions are earlier than version 12, TLS 1.2 is the latest version that will be selected.

Changed preconditions for the <em>count</em> in the function used to obtain the input/output index for the following effects. The preconditions have been relaxed so that values other than those supported by the effect can be specified for the number of channels.

12-0-0S14: (BuildSystem) Made improvements so that an error is reported if malloc_usable_size() is insufficiently defined.

Made improvements so that a link error is reported if the application does not define malloc_usable_size() in the memory acquisition function, but uniquely defines malloc(), free(), calloc(), realloc(), or aligned_alloc(). Link errors are reported as duplicate symbol errors with libnn_init.a (init_Malloc.o).

If this link error occurs, define malloc_usable_size(). For more information about important considerations when independently defining memory acquisition functions, see Starting and Initializing Programs.

12-0-0S15: (BuildSystem) Made improvements to no longer require link options such as -Wl, -u, and malloc.

The following link options no longer need to be specified when linking (independent build system) application programs.

  • -Wl,-u,aligned_alloc
  • -Wl,-u,calloc
  • -Wl,-u,free
  • -Wl,-u,malloc
  • -Wl,-u,malloc_usable_size
  • -Wl,-u,realloc

12-0-0S21: (GFX) Changed the buffer size configuration method for the nn::gfx::util::PrimitiveShape class.

The size of the vertex buffer needed in the nn::gfx::util::PrimitiveShape class was configured using nn::gfx::util::PrimitiveShape::SetVertexBufferSize(). Now an appropriate size is automatically configured at the time the vertex count is set using nn::gfx::util::PrimitiveShape::SetVertexCount().

Note that due to these changes, nn::gfx::util::PrimitiveShape::SetVertexBufferSize() and nn::gfx::util::PrimitiveShape::SetIndexBufferSize() have been deprecated.
