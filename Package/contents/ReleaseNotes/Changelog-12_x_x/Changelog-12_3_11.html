Details

Changes in NintendoSDK NX Addon 12.3.11 (based on Generic 12.3.11)

Added Features

*12-3-11S01: (TOOL) Added a setting to the NMETA file for the target age for applications for consoles for China.

Added a setting to the NMETA file for the target age for applications for consoles for China. A CADPA icon (target age icon) corresponding to this setting is displayed in the HOME Menu when the application is started. This is supported in the HOME Menu in NintendoSDK Firmware for NX 13.2.0-1.0 and later. For more information, see Writing the NMETA File > Target age setting (for China).

Improvements

12-3-11S02: (TOOL) Changed the default addr2line in the crash report retrieval tool to llvm-addr2line.exe.

Changed the default setting for addr2line used for symbol resolution from the conventional aarch64-nintendo-nx-elf-addr2line.exe to the faster llvm-addr2line.exe.

Bug Fixes

12-3-11S03: (TOOL) Fixed a bug in AuthoringTool where creating the second or later version of a patch for a multi-program application failed under certain conditions.

Fixed a bug where, depending on the combination of AuthoringTool versions, the creation of the second or later version of a patch for a multi-program application failed if a subprogram was being added that was created with a version of AuthoringTool different from the version that was used to create the previous patch.

12-3-11S04: (TOOL) Fixed a bug in AuthoringTool where, on rare occasions, the creation of the patch failed if the patch specified for --previous was created with application compaction enabled.

Fixed a bug where, on rare occasions, the creation of the patch NSP file failed if the file specified for --previous was a patch NSP file that was created with application compaction enabled in the past.

12-3-11S05: (TOOL) Fixed a bug where the wrong symbols were displayed when the crash report retrieval tool resolved symbols.

The wrong symbols were displayed when the crash report retrieval tool resolved symbols, due to a problem in the parsing of inline functions that caused symbols at other addresses to also be included. To work around this bug, inline function expansion has been disabled.
