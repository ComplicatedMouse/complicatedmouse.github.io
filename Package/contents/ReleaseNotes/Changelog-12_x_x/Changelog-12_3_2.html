Details

Changes in NintendoSDK NX Addon 12.3.2 (based on Generic 12.3.2)

Specification change that breaks your code

*12-3-2S01: (TOOL) Fixed the color tones when displaying remote video with htc-gen2.

When displaying remote video, some color tones differed from the actual image output and image recording results.
The video decoding process for remote video was revised to bring color tones in the htc-gen2 environment closer to the actual image output and image recording results.

With this fix, the pixel values obtained by the nn::tm::GetScreenImageData() function, which is a feature that depends on remote video, have also changed.
If you are referencing pixel values obtained using this function, note that these values might change as a result of this fix.

Added Features

12-3-2S02: (HTC) Added a feature for setting the serial number for the Windows target.

Added a feature for setting the serial number specified for the SDK_HTC_WIN_SERIAL_NUMBER target environment variable when a Windows target using nn::htc or nn::htcs is registered in Target Manager 2 (Nintendo SDK Daemon).
For more information, see Target Environment Variables.

Improvements

12-3-2S03: (FS) Improved the error message on an abort.

Added a link to the error code list to the error message that occurs when the application aborts.
The error message is added to the SD card access log before the develop build aborts.
The error code list provides a detailed description of each error.

Bug Fixes

12-3-2S04: (TOOL) Fixed a bug where the exception address in the crash report contained an incorrect value.

Fixed a bug where the exception address in the crash report obtained by the crash report retrieval tool sometimes contained an incorrect value.
The fix was implemented on the server side, so the previous version of the crash report retrieval tool can still be used.
