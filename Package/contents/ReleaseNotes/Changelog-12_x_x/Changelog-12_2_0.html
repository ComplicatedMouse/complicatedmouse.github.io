Details

Changes in NintendoSDK NX Addon 12.2.0 (based on Generic 12.2.0)

Specification change that breaks your code

*12-2-0S01: (BuildSystem) Added libnnSdk.lib, which consolidates the libraries automatically linked by the build system in the Windows environment.

Added libnnSdk.lib, which consolidates the libraries automatically linked by the build system in the Windows environment.
Developers using the NintendoSDK property sheets do not need to do anything in particular because libnnSdk.lib is linked automatically.
Developers who construct their own build systems without using the NintendoSDK property sheets need to link to libnnSdk.lib instead of the static library files for each feature. For more information about the features included in libnnSdk.lib, see How the Libraries Are Provided and How to Use Them.

*12-2-0S02: (General) Added .NET Core SDK 3.1 to the operating environment required by NintendoSDK.

Added .NET Core SDK 3.1 to the operating environment required by NintendoSDK.
If not installed, use the installer from the following path.

  • %NINTENDO_SDK_ROOT%\Installers\DotNetSdk\v3.1


As part of this change, .NET Core SDK 3.1 has been added to the environment that is checked by SdkEnvironmentChecker.

For more information about SdkEnvironmentChecker, see Checking the Suitability of the Operating Environment.

*12-2-0S03: (TOOL) Changed some of the command-line arguments of MakeMeta.exe.

Changed some of the command-line arguments of MakeMeta.exe. (--kernel_version -> --kernel-version)

Added Features

*12-2-0S04: (General) Made preparations for adding Visual Studio 2019 MSVC v142 support.

We plan to support Visual Studio 2019 MSVC v142 in NX Add-On 12.3.0 and later.
To prepare for this support, some of the files for Visual Studio 2019 MSVC v142 support were added to NX Add-On 12.2.0.
However, note that NX Add-On 12.2.0 is not yet supported.

12-2-0S05: (AUDIO) Added a feature for getting stats for the processing results of nn::audio::LightLimiterType.

Added a feature for getting stats for the processing results of nn::audio::LightLimiterType.
You can get the information described in nn::audio::LightLimiterChannelStatistics.

Accordingly, the required size when initializing the AudioRenderer work buffer will increase based on the number of effects.
Specifically, the required size for nn::audio::GetAudioRendererWorkBufferSize() is nn::audio::AudioRendererParameter.effectCount * 256 bytes, while the required size for nn::audio::GetAudioRendererConfigWorkBufferSize() increased by nn::audio::AudioRendererParameter.effectCount * 128 bytes.

For more information, see the following API references.

  • nn::audio::LightLimiterChannelStatistics
  • nn::audio::IsLightLimiterStatisticsEnabled()
  • nn::audio::SetLightLimiterStatisticsEnabled()
  • nn::audio::GetLightLimiterChannelStatistics()
  • nn::audio::ResetLightLimiterStatistics()


12-2-0S06: (DevMenu) Added a feature for verifying the serial numbers of paired Joy-Con controllers.

You can now use Features > Controlling the Development Hardware > DevMenu > Controller Management to check the serial numbers of paired Joy-Con controllers.

12-2-0S07: (NVN) New NVN Features

Added the following features to NVN.

  • Added the NVNdeviceInfo flag VERTEX_ATTRIBUTE_STATE_MAXIMUM_RELATIVE_OFFSET.


12-2-0S08: (PREPO) Added a setting to allow saving reports from users who have not linked their Nintendo Account.

Added a setting to allow saving reports from users who have not linked their Nintendo Account.

This setting is available just by updating the firmware, even if you do not update the application SDK.

For more information, see DevMenu > Play Report Settings.

12-2-0S09: (TOOL) Added a feature to TextureConverter to convert various textures in the texture binary file to image files.

Added a feature to TextureConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe) to convert the various textures in a texture binary file to image files (the --extract-image-files option).
You can also specify the type of texture file (the --extract-ldr-type and --extract-hdr-type options).

12-2-0S10: (TOOL) Added a feature to the automated testing support tools to get screenshot images.

Added a feature to the automated testing support tools to get screenshot images.
For more information, see the API reference and the Features page of the documentation for automated testing support tools.

12-2-0S11: (VSI) New Features in VSI

  • Added a dialog box to TM1 for verifying the NSP file associations.
    • If the NSP file is not associated, a dialog box now asks the user to confirm where they want to associate the NSP file with TM1.
  • Added linker options with dependencies without culling.
    • Added linker options where the user can specify the list of dependencies without culling. The specified library is linked using the whole-archive and -no-whole-archive options.


Improvements

12-2-0S12: (AUDIO) Improved the processing load when nn::audio::SampleFormat_PcmFloat is specified for nn::audio::VoiceType.

Improved the processing load when nn::audio::SampleFormat_PcmFloat is specified for nn::audio::VoiceType on the NX32 and NX64 platforms.
In particular, the processing load has been improved when using nn::audio::SetVoicePitch() to set a high pitch.

12-2-0S13: (BuildSystem) NintendoSDK property sheets now have the -fno-precompiled-file-deps compile option added by default.

When building for the NX32 and NX64 platforms, NintendoSDK property sheets now have the -fno-precompiled-file-deps compile option added by default.

12-2-0S14: (Compiler) Updated Clang for NX to 1.10.4.

Updated Clang for NX to version 1.10.4.
The main changes are as follows.

  • Fixed a bug where std::aligned_alloc() was unavailable even if you included <cstdlib>.


12-2-0S15: (DevMenu) Made improvements so that secure startup can also be used in PROD mode.

Made improvements so that secure startup can also be used in PROD mode.
Although the procedures for creating, installing, and running content for secure startup are the same as before, some restrictions have been placed on other operations.
For more information, see Features > Controlling the Development Hardware > DevMenu > Managing Applications > Secure Startup.

12-2-0S16: (DevMenu) The firmware version is now displayed in a location that can be confirmed immediately after startup.

The firmware version is now displayed in a location (at the bottom of the screen) that can be confirmed immediately after startup.

12-2-0S17: (DevMenuCommand) Made improvements to the DevMenuCommand application launch.

Previously, if --user-index was specified for the application launch DevMenuCommand, the user selection settings was sometimes retained, and it was necessary to run disable-auto-user-selection.
This update makes it so that the user settings are no longer retained, and you no longer have to run disable-auto-user-selection.

12-2-0S18: (NGC) Made fixes so that the nn::ngc::ResultInvalidCharacter error Result is now returned when a string in an invalid format is provided when using the profanity filtering API.

Previously when using the following functions, providing a string in an invalid format caused the program to abort, but it has been revised to return the nn::ngc::ResultInvalidCharacter error result instead.

  • nn::ngc::ProfanityFilter::CheckProfanityWords()
  • nn::ngc::ProfanityFilter::MaskProfanityWordsInText()


12-2-0S19: (NVN) Improvements in NVN

Made the following improvements to the nvn library.

  • Added NVN support for the Windows 458.01 driver. This build is based on NVIDIA driver 456.38. This package implements version 55.8 of the NVN API.

  • Made function declarations consistent between the fastpath functions defined in nvn_FuncPtrInline.h and nvn_Fastpath.h and in nvn.h.
    • Added the #error directive to nvn_FuncPtrInline.h and nvn_Fastpath.h if NVN_PROTOTYPES is defined.
    • Added 'extern "C"' linkage specifiers to nvn.h function prototypes.

  • Fastpath functions fall back to function pointers if the debug layer or other internal function pointer substitutions are used.

  • Updated the documentation.
    • Added information about the supported Ampere GPUs.
    • Removed old documentation about accessing NVN entry points through the OpenGL context.


12-2-0S20: (RenderDoc) RenderDoc for NX

  • Added support for VK_EXT_external_memory_host in the RenderDoc layer.


12-2-0S21: (RenderDoc) RenderDoc for NX

  • Added glslc.exe to the RenderDoc package
  • Added spirv-as.exe to the RenderDoc package
  • Added spirv-cross.exe to the RenderDoc package
  • Added spirv-dis.exe to the RenderDoc package


12-2-0S22: (RenderDoc) RenderDoc for NX

  • Added support for HTC Gen 2.


12-2-0S23: (TOOL) Added indicators to Remote Video in Target Manager 2 to show when video recording is prohibited and when Input Director is connected.

Added status bar indicators to the Remote Video window in Target Manager 2 to show when video recording is prohibited and when Input Director is connected.

12-2-0S24: (TOOL) Fixed a bug where an error occurred when the Nintendo SDK Daemon server port and HTCS control port were set to the same port.

Fixed a bug where an error occurred when the Nintendo SDK Daemon server port and HTCS control port were set to the same port.

12-2-0S25: (TOOL) Improved the error display for the Target Manager 2 Settings window.

Added and improved the error messages displayed when the input content is incorrect in the Target Manager 2 Settings window.

12-2-0S26: (VSI) VSI Improvements

  • Fixed a bug where Intellisense made incorrect error judgments.


12-2-0S27: (Keycode processing) Added C++14 support to nn::kpr.

Replaced NN_NOEXCEPT with the C++ standard noexcept keyword.

Granted final to classes and structures that are not inherited.

Bug Fixes

12-2-0S28: (LayoutTool) Fixed a bug where the accuracy of colors displayed in the PC version of LayoutViewer was reduced when specifying linear color space mode in LayoutEditor.

Fixed a bug where the accuracy of colors displayed in the PC version of LayoutViewer was reduced when specifying linear color space mode in LayoutEditor.

12-2-0S29: (AUDIO) Fixed a bug where a waveform overflow occurred in nn::audio::VoiceType for which nn::audio::SampleFormat_PcmFloat was specified.

Fixed a bug where using nn::audio::AppendWaveBuffer() to add nn::audio::WaveBuffer with absolute amplitude greater than 1.0, when nn::audio::SampleFormat_PcmFloat was specified in nn::audio::VoiceType, caused a waveform overflow.
The modified firmware subjects this to clipping, but the noise itself still occurs. Note that you need to avoid this noise at the stage of specifying waveform data for nn::audio::WaveBuffer.

12-2-0S30: (AUDIO) Fixed a bug where sometimes noise was mixed in with the audio when attaching and detaching large memory pools.

Fixed a bug where sometimes noise was mixed in with the audio when attaching and detaching large audio renderer memory pools.

  • nn::audio::RequestDetachMemoryPool()
  • nn::audio::RequestAttachMemoryPool()


12-2-0S31: (AUDIO) Revised the nn::audio::AcquireAudioDeviceNotificationForOutput() API reference.

Revised the nn::audio::AcquireAudioDeviceNotificationForOutput() API reference.

As described in the API reference for nn::audio::GetActiveAudioDeviceChannelCountForOutput(), in the NX environment you can use nn::os::SystemEvent, registered by nn::audio::AcquireAudioDeviceNotificationForOutput(), to get changes in the output channel count as events.

However, the API reference for nn::audio::AcquireAudioDeviceNotificationForOutput() did not sufficiently describe this feature.
For this reason, the description was revised to add this information.

* nn::audio::AcquireAudioDeviceNotificationForOutput()
* nn::audio::GetActiveAudioDeviceChannelCountForOutput()

12-2-0S32: (AUDIO) Revised the nn::audio::GetActiveAudioDeviceName() API reference.

Revised the nn::audio::GetActiveAudioDeviceName() API reference.

In the NX environment, nn::audio::GetActiveAudioDeviceName() processing sometimes took time.
For this reason, we recommend calling this function only when nn::os::SystemEvent, registered with nn::audio::AcquireAudioDeviceNotificationForOutput(), is signaled.

The important notes did not sufficiently describe this recommendation, so the API reference was revised.

* nn::audio::GetActiveAudioDeviceName()

12-2-0S33: (DevMenu) Fixed a bug where invalid save data was created when installing secure startup content.

Fixed a bug where invalid save data was created when installing secure startup content.
When you want to update from NintendoSDK Firmware for NX 11.0.1-1.2 to a new firmware version to use secure startup, use the SystemUpdater package to initialize the console.

12-2-0S34: (DevMenu) Fixed a bug where save data with invalid application names was displayed in the Save Data Manager.

Fixed a bug where "Unable to load Application Name" was displayed for application data that was correctly managed in Displaying Application Save Data in Features > Controlling the Development Hardware > DevMenu > Save Data Management.

12-2-0S35: (DevMenu) Fixed a bug where the application did not restart when selecting to restart the application from Error Viewer in the DevMenu environment.

Fixed a bug where the application did not restart when selecting to restart the application from Error Viewer in the DevMenu environment.

12-2-0S36: (MII) Fixed a bug where an assertion failure occurred when running the EXE file for Windows version Mii samples.

Fixed a bug where an assertion failure occurred when running the EXE file for NX Mii samples built for Windows.

12-2-0S37: (NGC) Fixed a bug where some words were not masked when setting nullptr for pProfanityWordCount in nn::ngc::ProfanityFilter::MaskProfanityWordsInText().

Fixed a bug in nn::ngc::ProfanityFilter::MaskProfanityWordsInText() where setting nullptr for <em>pProfanityWordCount</em> always resulted in the processing in the same state as when nn::ngc::ProfanityFilter::SkipAtSignCheck(nn::ngc::ProfanityFilter::SkipMode_SkipAtSign) is called, such that some of the words were not correctly masked.

12-2-0S38: (NVN) NVN Bug Fixes

Fixed the following NVN bugs.

  • Fixed a GPU exception when using nvnCommandBufferBeginRecording_fastpath.


Fixed the following bugs in glslc.

  • Fixed a bug where incorrect GLASM assembly errors would be produced on GLSLC_OPTLEVEL_NONE but not on GLSLC_OPTLEVEL_DEFAULT.


12-2-0S39: (SSL) Fixed a bug with the OpenSSL API SSL_CTX_new() when using built-in certificates.

Fixed a bug that happens when using built-in certificates and creating more than one OpenSSL context with SSL_CTX_new().

12-2-0S40: (TOOL) Fixed a bug in Target Manager 2 where old logs were still displayed when the maximum number of rows in the target log was reduced.

Fixed a bug in Target Manager 2 where using the settings to reduce the maximum number of rows displayed in the target log while filters were applied did not work and all of the rows were still displayed.

12-2-0S41: (TOOL) Fixed a bug where an invalid string was sometimes set by the feature for changing the target name.

Fixed a bug where an invalid string was sometimes set by the feature for changing the target name.

12-2-0S42: (TOOL) Fixed a bug where it was not possible for RunOnTarget and ControlTarget to specify a target other than the subnet if Target Manager 2 was set for use.

Fixed a bug where operations failed when using RunOnTarget and ControlTarget with Target Manager 2 if the target was connected to a network other than the subnet of the host PC and was not registered in Target Manager 2.

12-2-0S43: (TOOL) Fixed a bug where the large size icon and small size icon generated from the same BMP image sometimes did not match.

Fixed a bug where the large size icon and small size icon generated from the same BMP image were judged as not matching when creating an application image in AuthoringTool.
With this fix, an error now occurs if the large size icon and small size icon do not match.

12-2-0S44: (TOOL) Fixed a bug where the tool for getting crash reports crashed if the NINTENDO_SDK_ROOT environment variable was not configured.

Fixed a bug where the tool for getting crash reports crashed if the NINTENDO_SDK_ROOT environment variable was not configured.

12-2-0S45: (VSI) VSI Bug Fixes

  • Fixed a bug where NSS files were not loaded correctly and VS2019 stopped responding.
  • Fixed a bug in htc-gen2 where it took time for NSP files to load in VSI.

Changes in NintendoWare NX Addon 12.2.0 (based on Generic 12.2.0)

Added Features

12-2-0W01: (3DTool) Added a feature to 3D TextureConverter to convert various textures in the texture binary file to image files.

Added a feature to 3D TextureConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) to convert the various textures in a texture binary file to image files (the --extract-image-files option).
You can also specify the type of texture file (the --extract-ldr-type and --extract-hdr-type options).

12-2-0W02: (3DTool) Added the --3d-texture-converter-options option to Intermediate File Converter.

Added the --3d-texture-converter-options option to Intermediate File Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileConverter.exe). It allows additional options to be specified for 3D Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) during conversion.

12-2-0W03: (LayoutTool) Added a feature for displaying animation segment tags and playback frames for the default playback target in the Animation Segment Tags window when using separated mode in LayoutEditor.

Added a feature for displaying animation segment tags and playback frames for the default playback target in the Animation Segment Tags window when using separated mode in LayoutEditor.

Improvements

12-2-0W04: (3DTool) Ended support for 3ds Max 2017 in the 3ds Max plug-in.

Support has been ended for 3ds Max 2017 in the 3ds Max plug-in.

12-2-0W05: (3DTool) Ended support for Maya 2017 in the Maya plug-in.

Support has been ended for Maya 2017 in the Maya plug-in.

12-2-0W06: (3DTool) Ended support for Photoshop CC 2017 in the Photoshop plug-in.

Support has been ended for Photoshop CC 2017 in the Photoshop plug-in.

12-2-0W07: (3DTool) Improved the precision for handling correlation between LOD models and base models in LOD merging in the Intermediate File Optimizer.

In the LOD merging feature of the Intermediate File Optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe), made improvements so that when a specific shape in an LOD model corresponds to multiple base model shapes, it is merged with the most relevant shapes. Previously, the first shape that satisfied the corresponding conditions was used as the shape to merge with.

Bug Fixes

12-2-0W08: (3DTool) Fixed a bug in the CMDL Converter where it crashed if the Usage attribute for the texture coordinates was not numbered starting from 0.

Fixed a bug in the CMDL Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dCmdlConverter.exe) where it crashed if the Usage attribute for the texture coordinates (the <Vector2VertexStreamCtr> element of the CMDL file) was not numbered starting from 0.

12-2-0W09: (3DTool) Fixed a bug in the Maya Import Plug-In where an intermediate file that was not an export target was exported during LOD export.

Fixed a bug in the Maya Import Plug-In where an intermediate file that was not an export target was exported during LOD export if intermediate files remained in the temporary folder that was used during the previous export.

12-2-0W10: (3DTool) Fixed a bug in the Maya Import Plug-In where importing an FMD file with integer types for the vertex colors caused the program to crash.

Fixed a bug in the Maya Import Plug-In where importing an FMD file with integer types for the vertex colors caused the program to crash.
