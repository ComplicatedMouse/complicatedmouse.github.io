Changes in NintendoSDK NX Addon 10.1.0 (based on Generic 10.1.0)

  • GLDEBUGPROCAMD
  • GLDEBUGPROCARB

Added Features

*10-1-0S02: (DevMenu) Added a feature for backing up save data as a batch process.

Added a feature to DevMenu and DevMenuCommand for backing up all of the save data associated with an application at the same time.
This feature enables all of the user save data for multiple users to be backed up at the same time. For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Save Data Management > Backup
  • Features > Controlling the Development Hardware > DevMenuCommand > Managing Save Data > Save Data Batch Backup.


*10-1-0S03: (General) Added features that are necessary for developing applications for the Switch system for China.

Added features that are necessary for developing applications for the Switch system for China.
For more information about these features and related notes, see For Switch Systems For China (NX Add-On).

*10-1-0S04: (PREPO) Added the Play Report library.

Added the Play Report library.

Play Report provides a means by which information about events inside applications can be collected by a server and analyzed.

For more information, see NintendoSDK Documents.

*10-1-0S05: (VSI) Added the Edit and Continue feature (beta version).

Added the Edit and Continue feature (beta version). With this feature, changes to code can be applied while the application is running.
For more information, see the manual.

10-1-0S06: (CRYPTO) Added functions for calculating SHA-512.

Added functions for calculating SHA-512 to the nn::crypto namespace.
For more information, see the following functions in the API reference.

  • nn::crypto::Sha512Generator
  • nn::crypto::GenerateSha512Hash


10-1-0S07: (EC) Added price information to the information obtained for other related titles in the EcCatalog sample.

The EcCatalog sample now displays price information on the screen for getting information for other related titles.

10-1-0S08: (FS) Added the RamDisk access log output feature, and added support for that log in FsAccessLogChecker version 28.0.

Added access log output to the API for operating RamDisk, and also added support for the preceding access log to FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 28.0.
RamDisk does not access storage media, so its frequency of access is not something that is checked by the checker.

10-1-0S09: (HID) Added an API for implementing digitizer operations.

Added an API for implementing digitizer operations for debugging purposes. For more information, see this page.

10-1-0S10: (MOVIE) Added support for playback of 144p video to the VP9 decoder.

Added support for playback of 144p video to the VP9 decoder. Previously, the lowest resolution supported by VP9 was 180 × 180.

10-1-0S11: (Settings) The Settings Manager tool can now be used to change the capture feature and remote video settings.

The Settings Manager tool can now be used to change the capture settings and the settings for the remote video frame rate and bit rate. For more information, see the manual.

Added the following features for managing save data.

10-1-0S13: (TOOL) Added features for managing user accounts to the integrated command-line tool (TargetShell).

Added the following features for managing user accounts.
・Creating user accounts

・Deleting user accounts
・Configuring the account system
・Configuring individual user accounts
・Synchronizing the user account status with the server
For more information, see the manual.

10-1-0S14: (VSI) Added Target Manager features.

  • Target Manager
    • Added functions for saving the previous video.
      • Use nn::tm::SaveRecentVideo or Nintendo.Tm.Target.SaveRecentVideo.


Improvements

*10-1-0S15: (Compiler) Updated to Clang for NX 1.8.11.

Updated Clang for NX to version 1.8.11.
The main changes are as follows.

  • Fixed a bug where the NRO instrumentation did not work with PGO. For more information about instrumentation, see <em>SDK PGO Additional Information</em>.
  • Fixed a race condition that occurred when a file descriptor was opened or closed.
  • Fixed a race condition that occurred with lseek.
  • Added support for some C++17 libraries. For more information, see the documentation.


*10-1-0S16: (Socket) Increased the network socket limit for applications from 32 to 128.

The network socket limit for applications has been increased from 32 to 128.

10-1-0S17: (AOC) Changed the behavior of the application when it does not satisfy the required application release version for the add-on content.

Previously, when the application did not satisfy the required application release version for a particular set of add-on content, the application was prohibited from starting.
That behavior has been changed, and now the application can start, but it cannot access that add-on content (which is the same behavior as if that add-on content did not exist).

10-1-0S18: (BuildSystem) Enabled assertions that were mistakenly disabled in the develop version of libraries for the Windows environment.

Enabled assertions that were mistakenly disabled in the develop version of SDK libraries for the Windows environment.
Assertion failures might now occur that previously did not.
In particular, many assertions now have been enabled in libnn_mem, libnn_codec_aac, and libnn_multimedia.

10-1-0S19: (DevMenu) Added thumbnails for user save data that are backup and restore targets.

Added thumbnails for user save data that are backup and restore targets in DevMenu and DevMenuCommand.

10-1-0S20: (HID) Improved the practicality of the HidVibrationBasic vibration values.

The HidVibrationBasic sample generates a vibration pattern that corresponds to the number of times the button is pressed.
The vibration values represented extreme vibration patterns, so the values have been improved to be more practical.

%NINTENDO_SDK_ROOT%¥Samples¥Sources¥Applications¥HidVibrationBasic

For more information, see the following section in the NintendoSDK API Reference: Descriptions of Samples＞HID＞Simple Vibration Motor Controls Sample.

10-1-0S21: (NVN) Improvements in NVN

Made the following improvements to the glslc library.

  • Added internal improvements to support the ability in the future to debug SPIR-V shaders using the debugging tools.


10-1-0S22: (RenderDoc) Updated the UI.

  • Added additional controls in the Resource Inspector tree.


10-1-0S23: (RenderDoc) Updated the UI.

  • Copy and pasting from tree views now preserves the correct item order.


10-1-0S24: (TOOL) Added a feature for saving the error logs for the GUI setup tools.

The GUI setup tools (InitializeSdevWin and InitializeEdevWin) already have a feature for saving a log at the time of the final setup, but now a feature has been added for saving a log of errors under a different name.

This feature is used when these tools are supported.

10-1-0S25: (TOOL) The CPU boost feature can now be used even when GPU resource limitations have been disabled.

You can now use the CPU boost feature at the same time that GPU resource limitations have been disabled by the gpu-resource-limitation command, which is a debug category command in DevMenuCommand.

10-1-0S26: (VSI) Made improvements to VSI and Target Manager.

  • VSI
    • Changed the color of link text in the Dark theme for Visual Studio.
    • Links in Visual Studio representing a hexadecimal address now start with 0x.
  • Target Manager
    • Increased the file transfer speed for small files.
      • For files of 32 KiB and smaller, data is now transferred to console system memory at the same time the file is opened, omitting the step of communicating with Target Manager when loading a file.
      • Data that is transferred at the same time a file is opened is stored as cache in console system memory, and that cache is invalidated the next time the file is opened.
      • For files of 32 KiB and smaller, there is no assurance that the time the file is closed on the host side is synchronized with the time the file is closed on the target side.
      • For files of 32 KiB and smaller, if a file has been opened and then the file on the host side is switched, the switch is not applied in the loaded data until the content that is cached in console system memory has been invalidated.


Bug Fixes

10-1-0S27: (AOC) Fixed a bug where operations on add-on content failed during installation of add-on content.

There was a bug where the running application aborted if it operated on add-on content at a certain time while the installation of add-on content was underway.
This bug has been resolved so that this no longer occurs if the --dynamic option is specified for the installation of add-on content.

10-1-0S28: (AUDIO) Fixed a bug where an assertion failure occurred after the call to nn::audio::FlushWaveBuffers() in the Windows environment.

Fixed a bug where the program stopped on an assertion failure in the nn::audio::RequestUpdateAudioRenderer() function when called after the call to nn::audio::FlushWaveBuffers() in the Windows environment.

10-1-0S29: (AUDIO) Fixed a bug where microphones that do not sample at 48 kHz could not be used for AudioIn on Windows.

Fixed a bug in NX Add-On 10.0.0 where microphones that do not sample at 48 kHz could not be used for AudioIn on Windows.

10-1-0S30: (DevMenu) Fixed a bug where save data could not be properly backed up if a filename or a directory name included multibyte characters.

Fixed a bug where save data could not be properly backed up if it included a filename or a directory name with multibyte characters.
For more information about handling multibyte characters, see Features > Controlling the Development Hardware > DevMenu > Save Data Management > Backup.

10-1-0S31: (FS) Fixed a bug in RamDisk where the path was not verified.

Fixed a bug where the path specified in the argument for functions that operate RamDisk was not verified.

10-1-0S32: (HID) Fixed a bug where Input Director entered capture mode from the Windows lock screen.

Fixed a bug where Input Director entered capture mode from the Windows lock screen.

10-1-0S33: (HID) Fixed a problem with Input Director where the locked state of the keyboard was not applied.

Fixed a bug where the locked state of the keyboard on the host side was not applied on the target side when Input Director was used to capture keyboard input.

10-1-0S34: (MOVIE) Fixed a bug in the MovieDecoderMultiInstance sample where a timeout occurred during playback.

Changed MovieVideoOutputHandler.cpp in the MovieDecoderMultiInstance sample such that movie::Status_EndOfStream is now checked if the video decoder has sent an EOS buffer that does not have valid video data.

10-1-0S35: (MOVIE) Fixed a bug in the video decoder where sometimes the movie became corrupted after the resolution was changed in CPU rendering mode.

With this fix, if the resolution is changed while a movie is being played in CPU rendering mode, the new resolution is calculated correctly and the movie no longer becomes corrupted. You can confirm this fix by playing a movie with the MovieDecoderPlayer sample program and changing the resolution from 720p to 1080p while the movie is playing.

10-1-0S36: (NIFM) Fixed a bug where an invalid error was displayed when nn::nifm::HandleNetworkRequestErrorResult was called.

Fixed a bug where the 2000-0000 error was displayed when nn::nifm::HandleNetworkRequestErrorResult was called if an application that was built with SDK 8.x.x or an earlier version of NintendoSDK was run with the 9.0.x version of the firmware.

10-1-0S37: (NS) Fixed a bug where a fatal error occurred when the development hardware was restarted after performing a system update with the development hardware update tool.

Fixed a bug where a fatal error occurred when the development hardware was restarted after completing a system update using one of the development hardware update tools (SystemUpdateSdev or SystemUpdateEdev).
Note that for systems initialized with an SDK version earlier than NX Add-On 9.x.x, this bug can still occur despite the bug fix.
If it occurs, you can still use the hardware without problems by restarting it manually.

10-1-0S38: (NVN) NVN Bug Fixes

Fixed the following NVN bugs.

  • Fixed a bug where writing linear texture data to regions of native 1D array textures using nvnTextureWriteTexels caused an error.

  • Fixed a bug where an error sometimes occurred when copying one texture to another texture with a different format even if the same formats were explicitly specified in the API texture views.

  • Fixed a bug in Windows where sometimes queues that required flushing could be submitted without all of the prerequisite driver internal queues being submitted first.

  • Fixed a bug where it was possible for the NVN driver to merge segments for submission to the GPU in sizes that exceeded the physical limitation of the hardware for submissions.

  • Fixed a bug where nvnCommandBufferBarrier could cause memory corruption in the out-of-memory routine for command memory.


Fixed the following bugs in glslc.

  • Fixed a bug where SPIR-V would not properly compile when the image type was Unknown and the capability was StorageImageReadWithoutFormat.


10-1-0S39: (TOOL) Fixed a bug in the shader compiler tool where an error occurred if the common and glsl options were not specified.

Fixed a bug in the shader compiler tool ((%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\ShaderCompiler.dll)) where an error occured if you called nngfxToolShaderCompilerCompile without specifying the nngfxToolShaderCompilerCompileOptionCommonand nngfxToolShaderCompilerCompileOptionGlsl options.

10-1-0S40: (TOOL) Fixed a bug where AuthoringTool ended abnormally when creating a patch.

Fixed a bug when creating a patch with AuthoringTool with the --cache-directory option specified where, depending on the content revised in the application and the cache, the tool ended abnormally and the patch could not be created.

10-1-0S41: (TOOL) Fixed a bug where AuthoringTool unexpectedly detected differences when comparing NSP files.

Fixed a bug in the comparensp command of AuthoringTool where the process of comparing NSP files detected differences in the legal information file, even though there were no apparent changes to that file.

10-1-0S42: (TOOL) Revised the environments checked by the SDK environment checker.

Windows 7 has been removed from the confirmed operating environments for NintendoSDK, so Windows 7 has been removed from environments checked by the SDK environment checker tool (Tools\SdkEnvironmentChecker\SdkEnvironmentChecker.exe),

10-1-0S43: (TOOL) You can no longer revise the application control data for a patch using the replacenspmeta command.

An error now occurs if the NSP file for a patch is entered as the file for the replacenspmeta command.

10-1-0S44: (VSI) Fixed bugs for VSI and Target Manager.

  • VSI
    • Fixed a bug where sometimes you could not debug-run the application in Visual Studio.
  • Target Manager
    • Fixed a bug where sometimes the Target Manager CUI options did not work.
    • Fixed a bug where sometimes an NXDMP file created from Target Manager was not entered in the log.
    • Fixed a bug where NOT filtering was performed unintentionally when a string containing a hyphen was specified in the log filter.
    • Fixed a bug where the previously displayed screen remained on display when the Remote Video screen was started.
    • Fixed a bug where graphics broke down in the Remote Video screen.

As a result of this improvement, if you add the shader parameter animations from 3DEditor to the material animations exported from the DCC tool and then perform a merge-export from the DCC tool, the shader parameter animations created in 3DEditor remains behind.

10-1-0W12: (3DTool) Improved the process for calculating the slopes of keys when creating animation curves from baked data in the Maya plug-in.

Improved the process for calculating the slopes of keys when creating animation curves from baked data in the Maya plug-in.

Bug Fixes

*10-1-0W18: (Audio-ATK) Fixed a bug where effects like envelope decay and fade-in took longer to complete than the specified time.

Fixed a bug where effects like envelope decay and fade-in took longer to complete than the specified time.
This bug occurred in both the runtime and in SoundMaker (excluding the feature to create waveforms).
Note that as a result of this bug fix, fades and other effects complete faster than in earlier versions, and this changes how things sound.

This bug fix affects the following.

  • The envelope attack, decay, release, and hold times
  • The LFO cycle
  • The sweep time
  • The fade time specified by the nn::atk::SoundSystem::SetMasterVolume() function.
  • The fade time specified by the nn::atk::SubMix::SetBusVolume(), nn::atk::SubMix::SetChannelVolume(), and nn::atk::SubMix::SetSubMixVolume() functions.
