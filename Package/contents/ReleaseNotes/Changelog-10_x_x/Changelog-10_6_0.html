Details

Changes in NintendoSDK NX Addon 10.6.0 (based on Generic 10.6.0)

Added Features

10-6-0S01: (DevMenu) Added patches and downloadable content to the types of content that can be used with secure startup.

Added patches and downloadable content to the types of content that can be used with secure startup. This change enables combinations of application, patch, and downloadable content to be used with secure startup.

10-6-0S02: (FS) Added access log output to the nn::fs::IsMounted() function, and added support for that access log in FsAccessLogChecker version 33.0.

Added access log output to the nn::fs::IsMounted() function, and also added support for that access log to version 33.0 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe).
The nn::fs::IsMounted() function does not access storage media, so the checker does not check its frequency of access.

10-6-0S03: (FS) Added the function nn::fs::IsMounted() for determining whether the mount name is already mounted.

Added the function nn::fs::IsMounted() for determining whether the mount name is already mounted.

10-6-0S04: (SSL) Added the ISRG Root X1 root CA certificate to the built-in certificate store.

Added the ISRG Root X1 root CA certificate to the built-in certificate store. With this addition, server certificate chains that have ISRG Root X1 as the root are now trusted by default by the nn::ssl library.

10-6-0S05: (TOOL) Added a feature to create NSP files for secure startup of patches and downloadable content.

Added a feature to AuthoringTool so you can use the makesecure command to create NSP files of patches and downloadable content for secure startup.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


10-6-0S06: (TOOL) Added multi-program applications to the NSP files that can be created for secure startup.

The makesecure command can now be used to create NSP files for secure startup of multi-program applications.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


10-6-0S07: (TOOL) Added the --dummy option to the makesecure command of AuthoringTool to hide the application's information.

Added the --dummy option to the makesecure command of AuthoringTool to hide the application's information.
The application information that gets hidden are the title, publisher name, and icon.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


10-6-0S08: (TOOL) Suppressed the automatic installation of Target Manager 2.

Previously, when you installed the NintendoSDK Target Manager 2 package on NDI, the installer (MSI) automatically performed the installation.
With this version of the SDK, you can specify 1 or TRUE for the NINTENDOSDK_NOINSTALL_TARGET_MANAGER environment variable to suppress automatic execution by the installer.

The name of this environment variable is the same as the environment variable used previously to suppress automatic installation of Target Manager.
For more information, see the following.

  • Nintendo Developer Portal > Developer Support > Nintendo Switch > Technical Information > One Point TIPS > Suppressing automatic installation of VSI for NX and Target Manager (https://developer.nintendo.com/group/development/g1kr9vj6/tech-info/one-point-tips?articleId=297136093)


Improvements

*10-6-0S09: (DevMenu) Added options to DevMenu and DevMenuCommand for showing overlay notifications and the like in remote video in the SDEV environment.

In the SDEV environment, overlay notifications, library applets, and the like are not displayed in the remote video if the video capture feature is enabled in the application. On the other hand, if the video capture feature is disabled in the SDEV environment, overlay notifications and the like are displayed in the remote video, albeit in alternate mode and at a slower frame rate.

Given these circumstances, it was necessary to change the application with workarounds such as disabling the video capture feature in order to display overlays in the remote video. To allow this display of overlays without needing to change the application, a feature was added to both DevMenu and DevMenuCommand that forcibly disables the application's video capture feature.

You can use either of the following methods to use this feature.

  • Using DevMenu, open the Debug tab and in Misc Settings, select Force Disable Game Recording (needs restart application).
  • Using DevMenuCommand, call DevMenuCommand debug set-force-disable-game-recording <true|false>.


In either case, the setting is applied the next time the application is started. If the setting is changed while the application is running, the change is not applied at that time.

Note that in the EDEV environment and other non-SDEV environments the remote video feature cannot be used if the video capture feature has been disabled.

10-6-0S10: (DevMenu) Added support to DevMenu and DevMenuCommand for restoring save data for debugging.

Added support to DevMenu and DevMenuCommand for restoring save data for debugging.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Save Data Management > Restore.
  • Features > Controlling the Development Hardware > DevMenuCommand > Managing Save Data > Restoring Save Data.


10-6-0S11: (SSL) Added Sectigo root certificates to the nn::ssl built-in data.

The following root certificates have been added to the CA built-in data list for nn::ssl.
- USERTrust ECC Certification Authority
- COMODO RSA Certification Authority
- COMODO ECC Certification Authority

10-6-0S12: (TOOL) Revised AuthoringTool so you can now omit the Core element in the NMETA file used as input for the replacenspmeta command.

Revised AuthoringTool so you can now omit the Core element in the NMETA file that is specified for the --meta option of the replacenspmeta command.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Revise Application Control Data for NSP Files


Bug Fixes

10-6-0S13: (DevMenu) Fixed a bug where changes to the focus state of the application were not notified when the system was in sleep mode.

Fixed a bug where DevMenu did not report nn::oe::MessageFocusStateChanged to the application if the application was in any focus state other than background focus when the system entered sleep mode. With this fix, the behavior of the HOME Menu and DevMenu are now the same when notifying the application of changes in the focus state.

10-6-0S14: (DevMenu) Fixed a bug where deleting applications using DevMenuApp failed after the console was initialized using the HOME Menu.

Fixed a bug where deleting applications using DevMenuApp failed and DevMenuApp aborted if you initialized the console from the system settings in the HOME Menu and did not restart DevMenu at all before using DevMenuApp.

10-6-0S15: (DevMenu) Fixed a bug where the system could not enter sleep mode or shut down while an application was starting.

Fixed a bug where the development hardware could not enter sleep mode or shut down while an application was starting. This fix applies to the following screens.

  • The user-selection screen when starting
  • The passphrase screen for secure startup


10-6-0S16: (TOOL) Fixed a bug in the analyze-patch command of AuthoringTool.

Fixed a bug in the analyze-patch command of AuthoringTool where sometimes the wrong filter conditions were displayed and the wrong filter process, size, and ratio were displayed.

10-6-0S17: (TOOL) Fixed a bug where patches could not be created for multi-program applications.

There was a bug where using a different DESC file for each sub-program in a multi-program application resulted in the following error because only a single file was passed to the --desc option of the makepatch command.
Wrong desc file. Acid in the desc file and main.npdm don't match.
To address this issue, multiple DESC files are now passed to the --desc option of the makepatch command when you create a patch for a multi-program application.

10-6-0S18: (TOOL) Fixed a bug where TargetManager2 NX Add-On was not installed by default.

Fixed a bug where TargetManager2 NX Add-On was not installed as part of the standard installation of NintendoSDK NX Add-On.
