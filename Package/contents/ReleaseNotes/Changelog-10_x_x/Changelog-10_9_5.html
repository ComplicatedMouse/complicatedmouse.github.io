Details

Changes in NintendoSDK NX Addon 10.9.5 (based on Generic 10.9.5)

Added Features

*10-9-5S01: (TOOL) Added an application compaction feature.

Added a feature to compact applications and create applications with low storage consumption by removing the data from the initial version of an application that is no longer referenced due to patch fixes.
For more information, see Application Compaction.

10-9-5S02: (TOOL) Added the ability to split up an NSP file created by merging a patch and a compacted application to the AuthoringToolKit splitnsp command.

Added the ability to split up an NSP file created by merging a patch and a compacted application to the AuthoringToolKit splitnsp command.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Splitting NSP Files.


Improvements

10-9-5S03: (TOOL) Enabled a consistency check between the SDK version and the AuthoringTool version when creating a subprogram.

Re-enabled the consistency check between the SDK version and the AuthoringTool version when a subprogram is being created. This checking process was temporarily disabled in AuthoringTool 10.4.0.
As a result, subprogram creation now fails if any of the following conditions are met.

  • The SDK major version is different from the AuthoringTool major version.
  • The SDK version is earlier than the AuthoringTool version.


10-9-5S04: (TOOL) Made fixes to multi-application card documentation.

Fixed documentation to specify pre-conversion additional content instead of on-card downloadable content when creating a multi-application card with DLC.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating an NSP File for the Production of a Multi-Application Card.
  • Features > NX Development Cards > Multi-Application Cards


10-9-5S05: (TOOL) Setting SaveDataOwnerIds to its own ApplicationId when creating a patch now causes an error.

When creating a patch with AuthoringTool or AuthoringEditor, an error now occurs if the revised version of the application's NSP file meets all of the following conditions.

  • The application's own ApplicationId is set in the main program's SaveDataOwnerIds.
  • The application is not a multi-program application.

For more information about SaveDataOwnerIds, see Features > Basic Features > fs Library > Manual > Feature Description > Save Data > Save Data (NX Add-On) > Referencing Save Data From Other Applications.

Bug Fixes

10-9-5S06: (TOOL) Fixed a bug in AuthoringTool where files remained when a patch generation error occurred.

Fixed a bug in AuthoringTool where the compacted application remained when patch generation using the --do-application-compaction option resulted in an error.

10-9-5S07: (TOOL) Fixed a bug in AuthoringTool where NSP files for secure startup could not be created from a compacted application.

Fixed a bug in AuthoringTool where NSP files for secure startup could not be created from a compacted application.

10-9-5S08: (TOOL) Fixed a bug in AuthoringTool where the path specified in the -o option was ignored when running the splitnsp command.

Fixed a bug in AuthoringTool where the output file was always placed in the folder where the input source NSP file was located when running splitnsp for NSP files like the following.

  • NSP files merging a compacted application and patch


10-9-5S09: (TOOL) Fixed a bug where patches could not be created when multiple programs with different major versions of the SDK existed in a multi-program application.

Fixed a bug where patches could not be created when multiple programs with different major versions of the SDK existed in a multi-program application.
For more information, see the following documentation.

  • Features > Patches > Building and Running a Patch > Creating the Patch
