Details

Changes in NintendoSDK NX Addon 10.9.0 (based on Generic 10.9.0)

Specification change that breaks your code

*10-9-0S01: (General) Updated the version of Microsoft .NET Framework required for the Nintendo SDK operating environment to 4.7.2.

Updated the version of Microsoft .NET Framework required for the Nintendo SDK operating environment from 4.6.1 to 4.7.2.
As part of this change, the version of the Microsoft .NET Framework checked in the SdkEnvironmentChecker was also updated from 4.6.1 to 4.7.2.

For more information about SdkEnvironmentChecker, see Checking the Suitability of the Operating Environment.

*10-9-0S02: (NGC) Renamed the South American Portuguese pattern list available in nn::ngc to the "Brazilian Portuguese" list.

As part of this language settings change to Brazilian Portuguese, the pattern list for South American Portuguese of invalid strings that can be handled in nn::ngc has been renamed to Brazilian Portuguese. Similarly, the nn::ngc::ProfanityFilterPatternList enumerated type ProfanityFilterPatternList_SouthAmericanPortuguese has been changed to ProfanityFilterPatternList_BrazilianPortuguese.

Added Features

*10-9-0S03: (TOOL) Added a crash screenshot to the crash report.

Screenshots can now be captured when the application crashes and kept in the crash report. The image files are saved in JPEG format.
Saving screenshots is disabled by default. For more information, including enabling it, see Crash Reports > Screenshots.

Screenshots can be retrieved using the crash report retrieval tool (CrashReport.exe). For more information about using the tool, see Crash Reports > Crash Report Retrieval Tool.

*10-9-0S04: (TOOL) Added Brazilian Portuguese to the system settings languages.

Added Brazilian Portuguese to the system settings languages.
As a result of this fix, the nmeta file title, icon, and supported language can be configured for each language. In addition, setting the preceding language in system settings causes nn::oe::GetDesiredLanguage() to return the preceding language for applications where it is set as a supported language.

For more information about configuring using DevMenu and DevMenuCommand, see the following pages.

  • DevMenu > Changing Device Settings
  • DevMenuCommand > Changing the System Settings


Brazilian Portuguese is available for NX Add-On 11.4.0 and NX Add-On 10.9.0 and later.
However, if the target distribution format includes packaged version, see the separate Nintendo Developer Portal notifications when applications and patches using Brazilian Portuguese can be submitted.

10-9-0S05: (DevMenu) Added a feature to control settings for capturing screenshots for crash reports.

Added a feature to control settings around capturing screenshots for crash reports to each of DevMenu and DevMenuCommand.
For more information, see Crash Reports > Screenshots.

10-9-0S06: (TOOL) Added Brazilian Portuguese to languages and titles supported by AuthoringEditor.

Brazilian Portuguese can now be set for supported languages and titles (including icons) in AuthoringEditor.

10-9-0S07: (TOOL) Added the ability to convert a main program NSP file to a multi-program application subprogram.

In AuthoringTool, added the ability to convert a main program NSP file to a multi-program application subprogram.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Converting a Main Program NSP File to a Subprogram.


10-9-0S08: (TOOL) Added the ability to include DLC in multi-application cards.

You can now add DLC to a multi-application card, using the bundleup command in AuthoringTool.
For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating an NSP File for the Production of a Multi-Application Card.


Improvements

10-9-0S09: (DevMenu) Made improvements to enable operations with the L Stick.

Made improvements to enable operations with the L Stick.

10-9-0S10: (EC) Added a CPU usage display function to the dummy ec applet.

Added a feature to the dummy ec applet to display the usage rate for each CPU core used in the application.

10-9-0S11: (HID) Added an Apply button to the Input Director configuration form.

Added an Apply button to the Input Director configuration form.
Previously when applying changes to gamepad settings, clicking the OK button closed the window. This change makes it possible to change the gamepad settings without closing the configuration window.

10-9-0S12: (TOOL) A warning message is now displayed if the application's own ApplicationId is set for SaveDataOwnerIds.

Added a check when AuthoringTool commands are run with the --error-unpublishable option, and when the get-unpublishable-error command is run.
A warning message is now displayed when the NSP file meets all of the following conditions.

  • The application's own ApplicationId is set in the main program's SaveDataOwnerIds.
  • The application is not a multi-program application.

For more information about SaveDataOwnerIds, see fs Library > Manual > Feature Description > Save Data > Save Data (NX Add-On) > Referencing Save Data From Other Applications.

10-9-0S13: (TOOL) Corrected the Japanese-language text for the AuthoringEditor ratings configuration "IARC Generic" to "IARC汎用レーティング."

The "IARC Generic" rating that can be set in AuthoringEditor has been corrected to show "IARC汎用レーティング" when using the Japanese-language UI.

Bug Fixes

10-9-0S14: (DevMenu) Fixed a bug where a fatal error occurred in rare cases when installing an NSP file.

Fixed a bug where a fatal error (error code: 2162-0002) occurred in rare cases when installing an NSP file from the Application page.

10-9-0S15: (DevMenu) Fixed a bug where a UI button on the screen remained highlighted if an input operation for that button was performed with a different system.

Fixed a bug where a UI button on the screen remained highlighted if the action of that UI button was executed via a controller hardware button while that UI button was being touched. The following example shows this kind of operation.

  • The A Button of the controller is pressed while some button on the Application page (such as the Install from SD button) is being touched.


10-9-0S16: (DevMenu) Fixed a bug where account management information was not updated.

Fixed a bug where the following account management information was not updated.

  • If Add an user account and launch was selected, the application was started, and then you returned to DevMenu, the added user account was not displayed in the list.
  • When the Accounts tab was selected from another tab, the user information was not updated.


10-9-0S17: (DevMenu) Fixed a bug where only the touch screen could be used to operate.

Fixed a bug where only touch screen operations were available if Register NSA failed in account management.

10-9-0S18: (DevMenu) Fixed a bug where sometimes the highlighting remained after you tapped a UI button on the screen.

Fixed a bug where sometimes the highlighting remained after you tapped a UI button on the screen.

10-9-0S19: (DevMenu) Fixed a bug where the button to delete content and the button to show ROM identifier hash did not respond.

Fixed a bug in Feature > Controlling the Development Hardware > DevMenu > Managing Applications > Managing Content where the button to delete content (A: Delete) and the button to show ROM identifier hash (Y: Show Hash) did not respond. These buttons are disabled if no content is selected.

10-9-0S20: (DevMenu) Fixed a bug where unintended actions occurred when you tapped the application list on the Application page.

Fixed a bug where unintended actions occurred when you tapped the application list on the Application page.
For example, if you pressed and released the + Button on the controller when the Application tab was in focus and then tapped the application list, the action of tapping the list did not execute, and instead the action of pressing the + Button executed and the Options screen appeared.

10-9-0S21: (DevMenu) Fixed a bug where, in exhibition mode, the microSD card cleanup screen was shown without the application having been started automatically.

Fixed a bug where the application did not start automatically in exhibition mode when an uncleaned microSD card was inserted into development hardware, and the microSD card cleanup screen was displayed.

10-9-0S22: (TOOL) Fixed a bug where AuthoringEditor sometimes crashed while starting.

Fixed a bug where AuthoringEditor sometimes crashed while starting for one of the following reasons. The cause of the error is also indicated in the error dialog box that appears on an unexpected exit.

  • The configuration file was corrupted (%APPDATA%/Nintendo/AuthoringEditor/config.yaml).
  • The environment does not meet the requirements for the Nintendo SDK to operate.


10-9-0S23: (TOOL) Fixed a bug where AuthoringTool falsely detected an unpublishable error when using the Debug API in earlier versions.

Fixed a bug where AuthoringTool mistakenly detected an unpublishable error in patch NSP files created under the following conditions.

  • An NSP file is specified for the initial application or the previous patch in which the debug API was used.
  • An NSP file is specified for a revised version of the application in which the debug API was not used.
