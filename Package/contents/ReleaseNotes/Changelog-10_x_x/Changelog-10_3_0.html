Details

Changes in NintendoSDK NX Addon 10.3.0 (based on Generic 10.3.0)

Specification change that breaks your code

*10-3-0S01: (EC) Applications are now certified when you use ShopServiceAccessor.

Applications are now certified when you use ShopServiceAccessor.
If the version of the application is an earlier version that does not allow the use of network features, ShopServiceAccessor now returns an error that an update is required.
When you release a patch, you can configure the versions of the application to allow. For more information, see the description of the NCMS mandatory update flag.

Along with this change, applications that have not submitted OMAS requests return the 2164-1500 error when they use ShopServiceAccessor.
If you encounter this error, submit a request on OMAS to use the e-commerce features and wait for approval from Nintendo.

*10-3-0S02: (OE) The default focus-handling mode on program startup has been changed back to nn::oe::FocusHandlingMode_Suspend.

In NX Add-On 10.2.0, the focus handling mode that is the default mode immediately after the program has started was changed from nn::oe::FocusHandlingMode_Suspend to nn::oe::FocusHandlingMode_InFocusOnly. It has been changed back now to nn::oe::FocusHandlingMode_Suspend, which was the mode in NX Add-On 10.1.0 and earlier.

10-3-0S20: (libcurl) NintendoSDK libcurl now supports HTTP/2

Support for HTTP/2 was added in Nintendo SDK 10.3.0.

Some advantages of HTTP/2 include:

  • Single Connection: Only one connection to the server is used to load a website, and that connection remains open as long as the website is open. This reduces the number of round trips needed to set up multiple TCP connections.
  • Multiplexing: Multiple requests are allowed at the same time, on the same connection. With HTTP/1.1, each transfer has to wait for other transfers to complete.
  • Server Push: Additional resources can be sent to a client for future use.
  • Prioritization: Requests are assigned dependency levels that the server can use to deliver higher priority resources faster.
  • Binary: Makes HTTP/2 easier for a server to parse, more compact, and less error-prone. No additional time is wasted translating information from text to binary.
  • Header Compression: HTTP/2 uses HPACK compression, which reduces overhead. Many headers are sent with the same values in every request in HTTP/1.1.

10-3-0S35: (TOOL) Fixed a bug in the HlslCrossCompiler tool where there was a possibility that NaN values were propagated in the GLSL after conversion if the DXCB included 'and' instructions.

Fixed a bug in the HlslCrossCompiler tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe) where there was a possibility that NaN values were propagated in the GLSL after conversion if the DXCB included AND operators.

The HlslCrossCompiler tool converts using a mix() function that selects two values conditioned on the optimization of the AND operators.

As an example of NaN propagation, if you call mix(1.f, INF, float(false)), the return value is a NaN value, and the behavior changes from the behavior of the HLSL that had the AND operators. This bug fix gets rid of the float constructor, and by doing so eliminates the possibility of NaN propagation.

10-3-0S37: (TOOL) Fixed a bug in the texture converter where the results varied when the CPU converted the same image to BC1, BC2, and BC3 formats.

Fixed a bug in the texture converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\TextureConverter.exe) where the results varied when the CPU converted the same image to BC1, BC2, and BC3 formats.
This bug occurred when the image width or height was not a multiple of 4, and when mipmaps were created at levels smaller than 4 × 4 pixels.

Changes in NintendoWare NX Addon 10.3.0 (based on Generic 10.3.0)

Added Features

10-3-0W01: (3D-G3D) Added the ShapeAnimBlender class for blending shape animations.

Added the ShapeAnimBlender class for blending shape animations.
You can use this class to blend two or more nn::g3d::ShapeAnimObj objects and apply the result to nn::g3d::ModelObj.
For more information, see Shape Animation > Blending.

10-3-0W02: (3DTool) Added a feature to the 3D binary converter that lets multiple shapes share an index buffer.

Added a feature that lets multiple shapes share an index buffer.
If you specify the --share-duplicated-mesh-array option and use the 3D binary converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dBinaryConverter.exe) to convert multiple model intermediate (FMD) files that have matching content below the <mesh_array> element of the <shape> element, a binary resource file is generated in which the nn::g3d::ResMesh array is shared by multiple nn::g3d::ResShape objects. nn::g3d::ResMesh maintains an index buffer, so this index buffer is also shared. The multiple shapes that reference the shared mesh have the same value for nn::g3d::ResShapeData::pMeshArray. This feature can help reduce the size of the binary.

To prevent multiplex operations on the nn::g3d::ResMesh array and the index buffer in the runtime, one of the multiple of nn::g3d::ResShape objects that reference the shared nn::g3d::ResMesh array retains ownership of that nn::g3d::ResMesh array. To determine whether an object has ownership, use the nn::g3d::ResShape::IsMeshOwner() function. When performing operations that involve changes to subordinate nn::g3d::ResMesh objects, you can prevent multiplex operations by performing the operation via the nn::g3d::ResShape object for which nn::g3d::ResShape::IsMeshOwner() returns true.

10-3-0W03: (3DTool) Added a feature to the Maya Import plug-in for suppressing the creation of material nodes.

Added a feature to the Maya Import plug-in for suppressing the creation of material nodes. (Added the Create Material and Create Texture Node options.)

Improvements

10-3-0W04: (3DTool) 3DEditor now reads files faster when parent materials are used and responds to tool operations better.

3DEditor now reads files faster when parent materials are used and responds to tool operations better.

10-3-0W05: (3DTool) Improved the 3D texture converter's process for encoding data in BC3 format.

Improved the process used by the 3D texture converter tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) to encode data in BC3 format.
Improved the image quality of the A component of GPU encoding, and accelerated CPU encoding.
In some situations, the image quality of the A component of CPU encoding may be slightly lower than before.

10-3-0W06: (3DTool) Improved the intermediate file read/write speed of the 3D command-line tools.

The 3D command-line tools (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools) can now read and write intermediate files faster.

10-3-0W07: (3DTool) The Maya Import plug-in now has tabs for selecting the processing mode.

The Maya Import plug-in now has tabs for selecting the processing mode.

10-3-0W08: (LayoutTool) A warning about segments without any specified related groups is no longer displayed in the Segment Tags window of LayoutEditor when binary export is disabled.

A warning about segments without any specified related groups is no longer displayed in the Segment Tags window of LayoutEditor when binary export is disabled.

10-3-0W09: (LayoutTool) Added a menu (Synchronize the playback states of all layouts) to LayoutEditor.

Added the Synchronize the playback states of all layouts menu to the Edit menu of LayoutEditor. When this feature is enabled, all layouts opened in LayoutEditor are played at the same time. When the mode is separated mode, only the animations in segments with the same name are played.

10-3-0W10: (LayoutTool) Added a menu item to the Tools menu of LayoutEditor for selecting whether to send the first layout that was opened.

Added a menu item to the Tools menu of LayoutEditor for selecting whether to send the first layout that was opened. If Send the first layout that was opened is enabled, the first body layout that was opened is sent to the viewer when you open a parts layout that is opened from a body layout.

10-3-0W11: (LayoutTool) Added a setting in the LayoutEditor project configuration file for specifying the default for the Capture First Frame Only feature for Capture Pane and Capture Effect.

Added the <DefaultCaptureEffectFirstFrameOnlyFlag> element to the LayoutEditor project settings to specify the default setting for the Capture First Frame Only feature for Capture Pane and Capture Effect. Treated as true when omitted.

10-3-0W12: (LayoutTool) Added a setting to LayoutEditor for using the state when the layout was saved in separated mode as the initial state.

Added the <SaveStateAsInitialState> element to the LayoutEditor project settings. This element sets the state of the layout when it saved in separated mode as the initial state.

10-3-0W13: (LayoutTool) Added an option to the layout binary converter for not copying the font resource with the specified font name to the output destination.

Added the --no-copy-font-name option to the layout binary converter. By specifying this option, the font resource with the specified font name was not copied to the output destination.
