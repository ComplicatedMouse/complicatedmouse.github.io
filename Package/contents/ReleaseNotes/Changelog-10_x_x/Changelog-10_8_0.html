Details

Changes in NintendoSDK NX Addon 10.8.0 (based on Generic 10.8.0)

Added Features

10-8-0S01: (EC) Added a feature to the EcCatalog sample to get preorder sales information.

Added a feature to the EcCatalog sample to get all sale status information by specifying onsale_only = false when getting the AOC Item List and Related Title/Bundle List information.

10-8-0S02: (HID) Added the functionality to assign the same function as the Capture Button to other buttons.

Added the nn::hid::SetNpadCaptureButtonAssignment() function to assign the same function as the Capture Button to other buttons.
This feature is designed to be used in games with special methods of operation where it is difficult to operate the Capture Button on the Joy-Con (L).

Improvements

10-8-0S03: (Compiler) Updated to Clang for NX 1.8.14.

Updated Clang for NX to version 1.8.14.
The following changes were made.

  • Fixed a bug where an error occurred when the number of characters in the command line was exactly 32,768.


10-8-0S04: (CPU Profiler) Fixed an out-of-process profiling issue with threads that have affinity to more than one core.

When using out-of-process profiling, it was possible for a thread with a core affinity to more than one core to move between cores between when the profiler received notification of which thread was executing and its ability to read the call stack. As a result, the thread could resume execution while sampling was in progress, potentially resulting in recording incorrect call stacks.

There is a similar known issue when one core is executing a thread using significantly more stack space than the other executing threads. We are still working on a fix for the stack disparity issue.

10-8-0S05: (FS) Added support to FsAccessLogChecker version 35.1 to not count write operations to resource data (ROM) and downloadable content.

Because writing to storage media does not actually occur when nn::fs::FlushFile is called on resource data (ROM) and downloadable content, support was added to FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 35.1 to not count these as write operations.
However, from a performance standpoint, we recommend reducing calls to nn::fs::FlushFile() on resource data (ROM) and downloadable content.

10-8-0S06: (TOOL) Added support for displaying information about on-card downloadable content NSP files and multi-application card NSP files in AuthoringEditor.

When the following types of NSP files are opened in AuthoringEditor, information related to the application, patch, or downloadable content now is displayed.

  • On-card downloadable content NSP file
  • Multi-application card NSP file


10-8-0S07: (TOOL) Advice about defragmentation is now displayed for the analyze-patch command in AuthoringTool.

Made improvements so that advice about defragmentation is displayed for the analyze-patch command of AuthoringTool if performance worsens because of patch fragmentation.

Bug Fixes

10-8-0S08: (TOOL) Fixed a bug in the analyze-patch command of AuthoringTool.

Fixed a bug in the analyze-patch command of AuthoringTool where the incorrect size would sometimes be reported.

10-8-0S09: (TOOL) Fixed a bug in the filtering process of the analyze-patch command of AuthoringTool.

Fixed a bug in the analyze-patch command of AuthoringTool where files and directories smaller than the size specified by the filter were displayed.

10-8-0S10: (TOOL) Fixed a bug where it was possible to downgrade the version of SDK or AuthoringTool when creating a patch.

Fixed a bug where it was possible to downgrade the version of SDK or AuthoringTool when creating a patch. An error now occurs in the following situations.

  • The version of AuthoringTool used when creating the patch is earlier than the SDK version or the AuthoringTool version used when creating the revised application.
  • The SDK version of the revised application is earlier than the SDK version of the previous version of the application.

For more information, see the following.

  • Features > Patches > Building and Running a Patch > Creating the Patch.


10-8-0S11: (TOOL) Fixed a bug where NSP files could be created even if the AuthoringTool version was earlier than the SDK version.

Fixed a bug where NSP files could be created even if the minor version of AuthoringTool was earlier than the SDK version of the application.

10-8-0S12: (TOOL) Fixed a bug where patch sizes were larger than expected.

Fixed a bug where patches ended up being larger than expected if there were a lot of changes between versions of the revised application and it was difficult to perform binary matching on the parts with updated content.

10-8-0S13: (TOOL) Fixed a bug where there was a mismatch error in the major versions of the SDK and AuthoringTool if the application control data was modified in the NSP file.

Fixed a bug where a mismatch error occurred in the major versions of the SDK and AuthoringTool when the NSP file was loaded in AuthoringEditor if the application control data had been modified. This bug applied to the following NSP files.

  • NSP files subjected to the replacenspmeta command in AuthoringTool
  • NSP files whose application control data was edited in AuthoringEditor
