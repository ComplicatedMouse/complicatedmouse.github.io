Details

Changes in NintendoSDK NX Addon 10.2.0 (based on Generic 10.2.0)

Specification change that breaks your code

*10-2-0S01: (OE) Changed the focus handling mode that is the default mode immediately after the program has started.

The focus handling mode that is the default mode immediately after the program has started is now nn::oe::FocusHandlingMode_InFocusOnly instead of nn::oe::FocusHandlingMode_Suspend.
For more information, see the API Reference for the nn::oe::SetFocusHandlingMode() function.

*10-2-0S02: (VSI) VSI and Target Manager Changes

  • VSI
    • Changed the VSI settings for PGO.
      • You can no longer specify the path for generating profraw from VSI. To specify a path, call __llvm_profile_set_filename from within the program.
      • Manually converting profraw files to profdata is now required.
      • Added properties to specify the profdata path.
      • For more information about using PGO, see SDK PGO Supplemental Information.
  • Target Manager
    • Deprecated nn::tm::GetAgentVersion().
      • To get the development hardware version, use nn::tm::GetFirmwareVersion() instead.


Added Features

*10-2-0S03: (General) Added Visual Studio 2019 support.

Added support for the following features in Visual Studio 2019.

  • VSI
  • VsHelp
  • Wizard
  • NintendoSDKVsExtension


Note that builds using MSVC v142 do not support these features.
For more information, see Important Information When Using Visual Studio 2019.
Operation with NintendoSDK is verified using Visual Studio 2019 Professional Edition, described on the Operating Environment page.

*10-2-0S04: (HID) Added a vibration framework to facilitate the use of nn::hid::VibrationNode.

Added a framework to the HidVibrationPlayer sample program named VibrationNodeFramework that facilitates the use of the nn::hid::VibrationNode class.

This framework facilitates the use of the following features.

  • Basic features like the initialization of vibration motors
  • Simultaneous playback of multiple vibration files
  • Panning left and right
  • The nn::hid::VibrationPlayer features for changing speed and delaying playback (see note 1)
  • The nn::hid::VibrationMixer compressor feature (see note 1)

Note 1: These new features of nn::hid::VibrationNode were added in NX Add-On 10.2.

For more information, see NintendoSDK API Reference > Sample Descriptions > Human Input Device Library Samples > Controlling the Vibration Motor Using a Vibration File.

*10-2-0S05: (HID) Added new features to VibrationPlayer and VibrationMixer.

The following features have been added to nn::hid::VibrationPlayer and nn::hid::VibrationMixer, which are derived classes of nn::hid::VibrationNode.

  • Features to nn::hid::VibrationPlayer to change the playback speed and delay the playback of vibration files
  • A compressor feature to nn::hid::VibrationMixer to compress the vibration amplitude


For more information, see the reference pages for the following functions in the NintendoSDK API Reference.

  • nn::hid::VibrationPlayer::SetDelayTime
  • nn::hid::VibrationPlayer::SetPlaySpeed
  • nn::hid::VibrationMixer::SetCompressor
  • nn::hid::VibrationMixer::CompressorParameter


*10-2-0S06: (TOOL) Added a feature to SdkEnvironmentChecker for generating detailed information about the development environment.

Added a feature to SdkEnvironmentChecker for generating detailed information about the development environment. This information assists support by enabling the user's development environment to be shared smoothly.
This feature can be used together with the GUI version and the CUI version. The information includes the result of the check, information about the PC environment, information about the targets registered in Target Manager, information about NintendoSDK and other installed packages, and the set of log files created by the development hardware initialization and update tools.
For more information, see Checking the Suitability of the Operating Environment > Details Window, in addition to the descriptions of the --detail and --zip-path options.

You can now edit application control data in Authoring Editor for NSP files created with an earlier NX Add-On with a different major version.
For more information about application control data, see Writing the NMETA.

Global IP addresses can now be used to restrict access to files on a file-by-file basis for delivery by area operators.

10-2-0S12: (IRSENSOR) Added a mode to the irsensor library called adaptive clustering mode.

Added a mode to the irsensor library called <em>adaptive clustering mode</em>, where the camera settings changed adaptively.
If you are going to use this mode, make sure you understand how it compares with the conventional clustering mode.
For more information, see the manual.

10-2-0S13: (IRSENSOR) Added IrLed mode to the irsensor library.

Added IrLed mode to the irsensor library as a mode to control the lighting of the IR LEDs on the IR Motion Camera.
This feature makes it possible for the IR Motion Camera on one controller to detect whether the IR LEDs are lit on another IR Motion Camera.
For more information, see the manual.

10-2-0S14: (MOVIE) Fixed a bug in the movie decoder samples where a memory leak was reported if a negative value or 0 was passed for memory allocation.

Fixed a bug in the MovieDecoderMultiInstance, MovieDecoderPlayer, and MoviePlayerSimple movie decoder samples where a memory leak was reported if a negative value or 0 was passed for memory allocation. This was a bug in the samples, and memory leaks did not actually occur. The occurrence rate was low, but the possibility of occurrence was high for movies longer than 5 minutes. You can check this bug fix in mm_MoviePlayerUtils.h, where MemoryTracker is used.

10-2-0S15: (NVN) New NVN Features

Added the following features to NVN.

  • Added documentation about framebuffer compression.

  • Implemented the NVN API to support shader subgroup functionality (shaders compiled using the KHR_shader_subgroup family of extensions).


Added the following features to GLSLC.

    • Added a new section to the GLSLC programming guide about how to use shader subgroup features with NVN. The title of the new section is "Shader subgroup support."

The AuthoringEditor tool now supports editing the PlayLogPolicy element in the NMETA settings, which is the element that sets whether to make application usage public to users on other consoles.

Improvements

*10-2-0S19: (TOOL) Added support in Authoring Tool for changing application control data for NSP files created with an earlier major version of NX Add-On.

You can now use the replacenspmeta command in Authoring Tool to fix application control data in NSP files created with an earlier NX Add-On with a different major version.
With this change, it is now possible to set strings containing Simplified Chinese, Traditional Chinese, and Hangul characters in the title of NSP files created with NX Add-On 4.x and earlier.

10-2-0S20: (AUDIO) Improved the behavior of nn::audio::ListAudioOuts() and nn::audio::ListAudioDeviceName() in the Windows environment.

Improved the behavior of the nn::audio::ListAudioOuts() function and the nn::audio::ListAudioDeviceName() function.

In NintendoSDK 9.x.x or earlier, these functions had the following behavior.

* nn::audio::ListAudioOuts() returned the name of the sound device on Windows.
* nn::audio::ListAudioDeviceName() did not return anything.

In NX Add-On 10.2.0, these functions were changed as follows.

* nn::audio::ListAudioOuts() returns MainAudioOut.
* nn::audio::ListAudioDeviceName() returns the name of the sound device on Windows.

Also, along with these changes, nn::audio::SetAudioDeviceMapping(const char*, const char*) was deprecated.
From now on, use nn::audio::SetAudioDeviceMapping(const char*, const AudioDeviceName*).

10-2-0S21: (AUDIO) Increased the file-splitter chunk size for the recording feature of SystemAudioMonitor.

When you use the recording feature of SystemAudioMonitor for long-duration recordings, the file is now split into units of 4 GB instead of 2 GB.
With this change, files are now split less frequently.

10-2-0S22: (BuildSystem) NintendoSDK property sheets now have the -Wover-aligned option added by default.

The -Wover-aligned option is now added by default to the Clang compile options in the NintendoSDK property sheet.

The -Wover-aligned warning is generated when the default new operator is used and the assigned type requires an alignment that is greater than the ensured alignment. This kind of assignment is an illegal operation because the required alignment cannot be met.

10-2-0S23: (DIAG) Actual values are now added to the log when an assertion failure occurs.

When the evaluations of the NN_ASSERT macros and the NN_ABORT macros lead to an assertion failure, the actual value passed to the macro is now added to the log.

10-2-0S24: (DIAG) Added support for calling the nn::diag::GetBacktrace() function inside user exception handlers.

You can now use the nn::diag::GetBacktrace() function to get a backtrace even inside a user exception handler.

10-2-0S25: (FS) Deleted the access logs for the nn::bcat::DeliveryCacheDirectory and nn::bcat::DeliveryCacheFile destructors.

Previously, when the destructor was run for the nn::bcat::DeliveryCacheDirectory class or the nn::bcat::DeliveryCacheFile class without calling the Close() function, the access logs were generated for both function: "Close" and function: "~DeliveryCacheDirectory" or function: "~DeliveryCacheFile". With this change, now only the function: "Close" access log is generated.

10-2-0S27: (General) Enlarged display of images in NintendoSDK Documents.

The images in NintendoSDK Documents are now always displayed in their original size when you click to enlarge them.

10-2-0S28: (HID) Added a command for automatically updating the controllers when UpdateControllerFirmware.bat is run.

If you run UpdateControllerFirmware.bat with --auto specified, all connected controllers are now automatically updated to the latest firmware after the tool starts.

10-2-0S29: (NVN) NVN Changes

Made the following improvements to the glslc library.

  • Removed an optimization for speculatively hoisting textures based on some recent bugs until it can be made more stable.

  • Added an optimization to eliminate dead code under certain circumstances.


10-2-0S30: (PCTL) The feature for communicating freely with others can now be explicitly started and ended.

Added nn::pctl::TryBeginFreeCommunication() and nn::pctl::EndFreeCommunication() to start and end the feature for communicating freely with others.
In addition, added nn::pctl::IsFreeCommunicationAvailable() for determining whether the feature for communicating freely with others is available.
The nn::pctl::CheckFreeCommunicationPermission() function, which was the conventional way for checking whether the feature for communicating freely with others could be used, has been deprecated.

10-2-0S31: (PREPO) Play reports now support strings larger than 64 bytes.

String values larger than 64 bytes can now be saved in play reports.

10-2-0S32: (SWKBD) Added options for changing the behavior of the inline keyboard when the hardware keyboard is being used.

<em>miniaturizationMode</em> has been added to nn::swkbd::AppearArg, and if you specify MiniaturizationMode_Auto or Miniaturization_Forced for this option, the display area of the inline keyboard becomes miniaturized if characters are entered with the hardware keyboard while the inline keyboard is being used.
In this state, the nn::swkbd::InlineKeyboard::Calc() function now returns State_Miniaturized. When the state is State_Miniaturized, the inline keyboard is in a form that only receives input from the hardware keyboard, and all input from the controllers is processed by the application.
By supporting this function, you can operate the application with the controller in tandem with entering text with the hardware keyboard.
