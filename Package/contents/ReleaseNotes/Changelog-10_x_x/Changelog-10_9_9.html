Details

Changes in NintendoSDK NX Addon 10.9.9 (based on Generic 10.9.9)

Added Features

*10-9-9S01: (TOOL) Added a setting to the NMETA file for the target age for applications for consoles for China.

Added a setting to the NMETA file for the target age for applications for consoles for China. A CADPA icon (target age icon) corresponding to this setting is displayed in the HOME Menu when the application is started. This is supported in the HOME Menu in NintendoSDK Firmware for NX 13.2.0-1.0 and later. For more information, see Writing the NMETA File > Target age setting (for China).

Improvements

10-9-9S02: (TOOL) Tightened the NMETA file check when running AuthoringTool.

Tightened the NMETA file check when running AuthoringTool and modified it so that an error occurs if no valid value is configured.

Bug Fixes

10-9-9S03: (TOOL) Fixed a bug in AuthoringTool where creating the second or later version of a patch for a multi-program application failed under certain conditions.

Fixed a bug where, depending on the combination of AuthoringTool versions, the creation of the second or later version of a patch for a multi-program application failed if a subprogram was being added that was created with a version of AuthoringTool different from the version that was used to create the previous patch.

10-9-9S04: (TOOL) Fixed a bug in AuthoringTool where large and small icons created from the same BMP image were sometimes judged as not matching.

Fixed a bug where large and small icons created from the same BMP image were judged as not matching when creating an application image in AuthoringTool.
With this fix, an error now occurs if the large and small icons do not match, regardless of whether the --error-unpublishable option has been specified.

10-9-9S05: (TOOL) Fixed a bug in AuthoringTool where on rare occasions the creation of the patch failed if the patch specified for --previous was created with application compaction enabled.

Fixed a bug where on rare occasions the creation of the patch NSP file failed if the file specified for --previous was a patch NSP file that was created with application compaction enabled.
