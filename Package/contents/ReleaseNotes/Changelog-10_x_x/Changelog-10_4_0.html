Details

Changes in NintendoSDK NX Addon 10.4.0 (based on Generic 10.4.0)

Added Features

*10-4-0S01: (DevMenu) Added a secure startup feature when starting applications in DevMenu.

Added a secure startup feature when starting applications in DevMenu. Previously, exhibition mode supported secure startup when the application started automatically. Now, secure startup is also available when starting the application on DevMenu.
For more information about secure startup, see Features > Controlling the Development Hardware > DevMenu > Managing Applications > Secure Startup.
Multi-program applications do not support secure startup. Support is planned in a future version.

*10-4-0S02: (EC) Added a feature to the dummy EC applet for displaying a warning message if the running application uses a high amount of CPU.

Added a feature to the dummy EC applet for displaying a warning message if the running application uses a high amount of CPU.
If a warning message displays, see EC applet in NintendoSDK Documents and fix your application.

*10-4-0S03: (TOOL) Added the analyze-patch command to AuthoringTool tool for analyzing patch information, such as patch size.

Added the analyze-patch command to AuthoringTool tool for analyzing patch information, such as patch size.
For more information, see the analyze-patch command in Optimization Information.

*10-4-0S04: (TOOL) The pre-release of Target Manager2, the successor to the Target Manager tool, is now available.

The pre-release of Target Manager2, the successor to the Target Manager tool, is now available.
This SDK package still uses the original Target Manager by default, but we plan to switch to Target Manager 2 as the default in a future SDK package.

For more information, see Using Target Manager 2.

10-4-0S05: (AOC) Added a function to notify users when downloadable content is lost.

Added the nn::aoc::ShowAddOnContentLostError() function to notify users when downloadable content is lost.

10-4-0S06: (AUDIO) Added a feature for SystemAudioMonitor to connect to the Windows version of the application.

SystemAudioMonitor can now monitor the final output of the Windows version of the application.
For more information, see audio Library > Tool Reference > SystemAudioMonitor.

10-4-0S07: (DevMenu) Added a feature for downloading NSP files from an HTTP server and installing them.

Added a feature for downloading NSP files from an HTTP server and installing them.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Managing Applications > Installing HTTP Server NSP Files


10-4-0S08: (EC) Added a feature to the AOC item list page in the EcCatalog sample to display installation information obtained from aoc_indexes.

A feature was added to get information from aoc_indexes in the screen after selecting AOC item list in the EcCatalog sample, and display the obtained AOC installation information.

10-4-0S09: (EC) Added processing to support the addition of monthly subscription tickets to the Subscription Service Item List page in the EcCatalog sample.

With this added feature, when Subscription service item list is selected in EcCatalog sample, the screen following the one where the list was selected supports displaying monthly subscription tickets, and displays course type information.

10-4-0S10: (FRIENDS) Added a friend list cache deleting feature and a blocked-user list cache deleting feature to DevMenuCommand.

Added a friend list cache deleting feature and a blocked-user list cache deleting feature to DevMenuCommand. For more information, see the Deleting the Friend List Cache and Deleting the Blocked-User List Cache sections in DevMenuCommand > Friends Feature.

10-4-0S11: (FRIENDS) Added functions to ensure that the friend list and blocked-user list can be obtained.

Attempts to get the friend list or the blocked-user list immediately after linking a Nintendo Account may not be successful due to the cache not being created yet.
The following functions have been added to ensure that lists can be obtained.

  • nn::friends::EnsureFriendListAvailable()
  • nn::friends::EnsureBlockedUserListAvailable()

To reproduce the state where no cache has been created, use the delete-friend-list-cache or the delete-blocked-user-list-cache command in DevMenuCommand.

10-4-0S12: (INIT) Added a feature for calling static object destructors for testing purposes.

Added a feature that allows calling static object destructors when returning from the nnMain() function, and also added linker scripts ApplicationWithAtexit.aarch64.lp64.ldscript and ApplicationWithAtexit.arm.ilp32.ldscript that enable this feature.
Do not use this feature in the release version.

10-4-0S13: (TOOL) Added support to AuthoringEditor for displaying information for subprograms in a multi-program application.

When NSP files for multi-program applications are opened in AuthoringEditor, information for subprograms are displayed in addition to the information for the main program.
The following information is displayed.

  • Master ROM submission errors
  • Program information
    • SDK version
    • AuthoringTool version
    • Build type
    • List of middleware
  • Offline HTML documentation
  • List of accessible URLs

For more information about multi-program applications, see NintendoSDK Documents > Multi-Program Applications.

10-4-0S14: (TOOL) Added support to AuthoringEditor for editing downloadable content settings that are required at startup (RequiredAddOnContentsSet).

Added support to AuthoringEditor for displaying and editing downloadable content settings that are required at startup (RequiredAddOnContentsSet), which has been supported since NX Add-On 10.3.
For more information about the NMETA elements, see Writing the NMETA File.

10-4-0S15: (TOOL) Added the --enable-min-precision option to HlslCrossCompiler for converting min16f/min16i to mediump.

Added the --enable-min-precision option to HlslCrossCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\HlslCrossCompiler.exe) for converting min16f/min16i to mediump.

When this option is selected, the following operations occur.

  • mediump is assigned as the in16f/min16i (including vector) type.
  • GL_NV_desktop_lowp_mediump is enabled.


10-4-0S16: (VSI) Added a feature for VSI and Target Manager.

  • VSI
    • Added configuration properties that can specify the C standard language.
    • Added configuration properties so users can specify the C standard language. Currently, the only default option available is Gnu99.


Improvements

10-4-0S17: (CPU Profiler) Nintendo NX CPU Profiler Version 1.3

  • Added an experimental setting to the group Functions tab entries by function, thread, or core.
  • Reworked the Assembly tab to make it easier to use.
  • Added a collection of anonymous telemetry data.
  • Added an NX setting to change the maximum wait for the Launch scenario.
  • Added support for profiler library initialization from modules other than the main application NSP.
  • Changed the profile mode to try selecting the last chosen option when syncing.
  • Changed the timeline headers to make it easier to read the frame numbers.
  • Fixed a crash that occurred in the timeline when resting the mouse pointer on thread state data when the call stack only contained functions provided by Nintendo.
  • Fixed a bug that stopped the Wastefully Short Sleep checker from triggering.
  • Fixed a bug that stopped Select Frame and Select Time in the timeline from working with Thread State or Code Blocks by Thread.
  • Fixed a bug in the timeline for Thread State and Code Blocks by Thread where a heartbeat applied to all cores did not properly show up for those areas.
  • Fixed a bug that erroneously allowed function Instrumentation.
  • Fixed a bug where the Last tab was using values from the current profile rather than the previous profile.
  • Fixed a bug where functions did not reclassify after changes were made in the Options window.
  • Fixed a bug where warning messages on the Overview tab could not be viewed while a profile was being taken or loaded.


10-4-0S18: (DevMenu) Added settings for restricting application features to the DevMenu.

Settings for restricting application features that were previously only available in the DevMenuCommand are now also available in the DevMenu.
For more information, see DevMenu > Development Assistance Settings.

10-4-0S19: (DevMenu) Improved blacklist debugging by feature.

Improved the content that is generated by the command used to verify the settings of the blacklist debugging by feature (DevMenuCommand application get-function-authorization-error-simulation-settings).

10-4-0S20: (Display) Fixed a bug where the screen went dark when the display buffer used was greater than 1080p and the TV output resolution was 480p.

Fixed a bug where the screen went dark when a size greater than 1080p was specified for the texture passed to nvn::WindowBuilder and the TV output resolution was 480p.
In the same case with this fix, the scalable range is displayed and anything outside the range is not displayed.

10-4-0S21: (FRIENDS) Changed the behavior of the API used for automatic selection of user accounts.

Previously, an assertion failure occurred if the number of user accounts in an open state was any value except 1. The behavior has been changed to abort.
This change was made to prevent unexpected behavior in the release build.

10-4-0S22: (MOVIE) Deleted unnecessary logs from the logs generated during video playback.

A log was generated while video playback was in progress in the movie library, even if there was no actual problem.
This log has been suppressed.

10-4-0S23: (NFP) Added the enumerated type nn::nfp::NfpType, which represents the NFP type.

Added the enumerated type nn::nfp::NfpType, which represents the value stored in nfpType in nn::nfp::ModelInfo.

10-4-0S24: (TOOL) Disabled checking the consistency between the SDK version and the AuthoringTool version when a subprogram is being created.

Disabled checking the consistency between the SDK version and the AuthoringTool version when a subprogram is being created. This checking process was introduced in AuthoringTool 10.2.0.
This change is temporary so that we can provide an environment that allows changing the following type of application into a subprogram.

  • Applications created using an earlier version of the SDK that does not support multi-program applications

A future version will provide a different method to convert this type of application to a subprogram. At that time, the version consistency check will be enabled again.

10-4-0S25: (VSI) Improved VSI and Target Manager.

  • VSI
    • Updated the description to clarify that we do not recommend using gold and bdf.
  • Target Manager
    • Disabled (grayed out) htc-gen2 target display.
    • The ALT+F4 shortcut can now be used to close an active window.
    • Updated the API to send NOTIFY_TARGET_MANAGER_EXIT when Target Manager 1 exits unexpectedly.


10-4-0S26: (WebSocket) Added the definition of the constant value for the status code.

Added nn::websocket::StatusCode.

Bug Fixes

10-4-0S27: (AUDIO) Fixed a bug where an access violation could occur in the nn::audio library for Windows.

Fixed a bug where an access violation could occur when the nn::audio library for Windows was linked to Windows DLL.

10-4-0S28: (DevMenu) Fixed a bug where an error would occur while writing a patch to the game card when the file path of the specified NSP file contained 36 or more characters.

Fixed a bug where an error would occur while writing a patch to the game card when the file path of the specified NSP file contained 36 or more characters.
The text of this path includes sd:/, which is the string used for mounting microSD cards.

10-4-0S29: (OS) Fixed a bug where Writer lock requests are not prioritized for nn::os::ReaderWriterLock in Windows environments.

Fixed a bug in the Windows environment where the Writer lock sometimes would not be prioritized and the Readers lock was obtained instead when the lock was released if nn::os::ReaderWriterLock was in an exclusive locked state and there were both Readers and Writer lock requests from multiple threads other than the owner.

10-4-0S30: (TOOL) Fixed a bug in AuthoringTool where it was possible to pass an incorrect combination of application and patch to the diffpatch command.

Fixed a bug where it was possible to specify an incorrect combination of application and patch to the diffpatch command in AuthoringTool.

10-4-0S31: (TOOL) Fixed a bug where the link destination of the Read the documentation button on AuthoringEditor was not accessible.

Fixed a bug where the URL linked to the Read the documentation button on the AuthoringEditor startup screen was not accessible.

10-4-0S32: (TOOL) Fixed bugs in the options for creating multi-program applications.

When specifying the --nsp-list-file option in mergeapplication in AuthoringTool, an error occurred when there was no position argument even if the specified file specified a subprogram.
An error no longer occurs when there is no position argument as long as --nsp-list-file is specified.

10-4-0S33: (VSI) Fixed a bug where the DirectoryHandle opened for the host PC file system was no longer valid after waking up from sleep mode.

Fixed a bug in Target Manager 2where the DirectoryHandle opened for the host PC file system was no longer valid after waking up from sleep mode.

10-4-0S34: (VSI) Fixed bugs in VSI and Target Manager.

  • VSI
    • Fixed a bug where sometimes step execution is slow.
    • Fixed a bug in the display of the watch window data for the pointer-to-member-functions array and the pointer-to-member-functions size.
    • Fixed a bug where debug information for the thread_local variable specified by Class is not displayed.
    • Fixed a bug in the memory window where you could not jump to the destination address when the base address was a destructor.
    • Fixed a bug in version compatibility when there was no LibAdditionalDependencies.
    • Fixed a bug in IntelliSense errors.
    • Fixed a bug in toolbar localization.
    • Fixed a bug where linker output was missing.
    • This fix generates the paths of all linked files, but this issue will be fixed soon.
    • Fixed a bug where application execution failed after performing a special debug procedure.
  • Target Manager
    • Fixed a bug where Target.GetConnectionType() and nn::GetTargetconnectionType() returned unknown values for htc-gen2 targets.
    • Fixed a bug where a TM1 crash occurred when either htc-gen2 EDEV or htc-gen2 HDEV was found.


Changes in NintendoWare NX Addon 10.4.0 (based on Generic 10.4.0)

Specification change that breaks your code

*10-4-0W01: (3D-G3D) Changed the conditions that enable the viewer library.

Changed the conditions that enable nn::g3d::viewer to the same conditions as nn::htcs. Previously, NN_ENABLE_HTC had to be defined to enable nn::g3d::viewer. Now, nn::g3d::viewer can be enabled for Debug and Develop builds without having to define NN_ENABLE_HTC. Note that the behavior is different when NN_ENABLE_HTC is not defined.

Added Features

10-4-0W02: (3DTool) Added a feature to the 3D texture preset converter that can take strings found in the input folder path and use them for the output folder path.

Added a feature to the 3D texture preset converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTexturePresetConverter.bat) that can take strings found in the input folder path and use them for the output folder path.
Use the search_input attribute in the <output_folder> element to specify the string to search for.
If the string cannot be found, the input file is not converted.

10-4-0W03: (3DTool) Added a feature to the Maya plug-in for exporting any type of surface material.

Added a feature to the Maya plug-in for exporting any type of surface material.
In the config file, specify attribute information to export (material_attr_info_*).
For more information, see Extended Materials in the materials documentation.

10-4-0W04: (3DTool) Added a feature to the Maya plug-in that suppresses output from <material array> elements in FMD files.

Added a feature to the Maya plug-in that suppresses output from <material array> elements in FMD files.
This feature is used for referencing existing material intermediate (FMT) files.
Specify the following option in the config file to suppress output.
export_material_array="false"

10-4-0W05: (3DTool) Added a setting value to the feature that assigns behavior for referencing materials in the assign utility.

Added ValuePolicyToRefer as a value that can be specified for the --child-value-restriction option in the assign-material-reference-behavior subcommand in the assign utility (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe). Also, values that were previously ForceOverride have been changed to ValuePolicyToOverride. ForceOverride is still available, so compatibility is maintained.

10-4-0W06: (3DTool) Added an option to the shader update feature in the assign utility to ignore materials if no shader definitions are found.

In the update-model and update-material subcommands in the assign utility (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe), the --ignore-unfound-shader option was added to ignore materials when shader definitions cannot be found for them.

10-4-0W07: (3DTool) Added Require the child material to reference the value to 3DEditor's material referencing feature.

Added Require the child material to reference the value to the child material restrictions in 3DEditor's material referencing feature.

10-4-0W08: (Layout-UI2D) Added a feature to render text that extends beyond the pane area when capture effects are applied to text panes.

There was a bug where applying capture effects to text panes caused italics to extend beyond the pane area and be truncated. Added nn::ui2d::Layout::BuildOption::isSpreadPaneSizeOfItalicEnabled and added a feature that adjusts pane size to ensure text is not truncated. Set <IsSpreadPaneSizeOfItalicEnabled> to true in the Project Configuration File in LayoutEditor to enable confirming it in LayoutViewer. Automatic wrapping based on per-character transformations and pane size is not supported.

Improvements

10-4-0W09: (3D-G3D) Changed the behavior of additive blending for nn::g3d::SkeletalAnimBlender::Blend().

Changed the behavior of additive blending for nn::g3d::SkeletalAnimBlender::Blend().

When calculating the difference of pSrcAnimObj minus pReferenceAnimObj in nn::g3d::SkeletalAnimBlender::Blend(), the bone animation difference is no longer calculated when a bone animation in pSrcAnimObj has no corresponding bone animation in pReferenceAnimObj.
Previously, the difference was calculated by substituting the scale rotate translate value from nn::g3d::ResBone in place of pReferenceAnimObj when no bone animation existed in pReferenceAnimObj.

10-4-0W10: (3DTool) Eased the restrictions on bone structure for the base model and the LOD models in the intermediate file optimizer's LOD merging feature.

In the feature for merging LOD models in intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe), an error tolerance can now be set for the discrepancy in rotation values between the bones of the LOD model and bones of the base model, and LOD merging can proceed if this discrepancy is less than 1.0e-5.

10-4-0W11: (3DTool) Environment variables used to get information from Pre-Export/Post-Export Scripts in the Maya plug-in are now deleted after export is complete.

Environment variables used to get information from Pre-Export/Post-Export Scripts in Maya plug-in previously remained defined, but are now deleted after export is complete.

10-4-0W12: (3DTool) Fixed a bug in the quantization analysis feature in the intermediate file optimizer where processing times were unusually long for model intermediate files with numerous shapes.

Fixed a bug in the quantization analysis feature in intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimzier.exe) where processing times were unusually long for model intermediate file inputs that have numerous shapes.

10-4-0W13: (3DTool) Improved the load speed of intermediate files in the 3D command-line tool.

The 3D command-line tool (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFile*.exe) that is used to export intermediate files can now load intermediate files faster.

10-4-0W14: (3DTool) The Maya plug-in now supports textures that are connected to materials via the aiNormalMap node.

The Maya plug-in now supports textures that are connected to materials via the aiNormalMap node.

Bug Fixes

10-4-0W15: (3D-G3D) Fixed a bug where the nn::g3d::ResSceneAnim::Bind() function did not work correctly.

Fixed a bug where nn::g3d::ResSceneAnim::Bind(), nn::g3d::ResLightAnim::Bind(), and nn::g3d::ResFogAnim::Bind() did not work correctly.

These Bind() functions resolve index values that can be obtained from the following functions, but there was a bug where incorrect index values were set.

  • nn::g3d::ResLightAnim::GetLightTypeIndex()
  • nn::g3d::ResLightAnim::GetDistAttnFuncIndex()
  • nn::g3d::ResLightAnim::GetAngleAttnFuncIndex()
  • nn::g3d::ResFogAnim::GetDistAttnFuncIndex()


Also fixed a bug where the index value obtained by nn::g3d::ResLightAnim::GetAngleAttnFuncIndex() had an initial value of 0. The correct initial value is -1.

10-4-0W16: (3DTool) Fixed a bug in 3DEditor where materials were not corrected when the shader definition and material revision were the same.

Fixed a bug in 3DEditor where materials were not corrected when the shader definition and material revision were the same.

10-4-0W17: (3DTool) Fixed a bug in the configuration plug-in for Maya sampler attributes where using invalid characters did not trigger an error.

Fixed a bug in the configuration plug-in for Maya sampler attributes where using invalid characters, such as periods and hyphens, in the sampler name or hint information did not trigger an error.
