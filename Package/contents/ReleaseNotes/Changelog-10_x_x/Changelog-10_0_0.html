Details

Changes in NintendoSDK NX Addon 10.0.0 (based on Generic 10.0.0)

Specification change that breaks your code

*10-0-0S01: (DevMenu) Deprecated the CPU overclocking features in DevMenu and DevMenuCommand.

Deprecated the CPU overclocking features in DevMenu and DevMenuCommand.
From now on, use the nn::oe::SetCpuOverclockEnabled function in the oe library instead.

*10-0-0S02: (EC) Added a minimum size requirement for the working buffer for the eShop service access utility feature.

Added a minimum size requirement for the working buffer for the eShop service access utility feature. At least 1 MB of memory is required for using the eShop service access utility feature.

*10-0-0S03: (FS) Standardized the method for handling save data paths to be case-sensitive.

Standardized path handling for save data on the Windows platform and save data placed on the PC by the nn::fs::SetSaveDataRootPath() function so that it is case-sensitive in both cases. This change was made to standardize the path handling behavior for save data.

*10-0-0S04: (GFX) Deleted the root signature feature, which we did not recommend.

Deleted nn::gfx::RootSignature and nn::gfx::RootSignatureInfo, which we did not recommend. Use binding functions like nn::gfx::CommandBuffer::SetTextureAndSampler instead.
In addition, deleted the following command buffer functions for the root signature, which we did not recommend.

  • nn::gfx::CommandBuffer::SetRootSignature
  • nn::gfx::CommandBuffer::SetRootBufferDescriptorTable
  • nn::gfx::CommandBuffer::SetRootTextureAndSamplerDescriptorTable
  • nn::gfx::CommandBuffer::SetRootConstantBuffer
  • nn::gfx::CommandBuffer::SetRootUnorderedAccessBuffer
  • nn::gfx::CommandBuffer::SetRootTextureAndSampler


*10-0-0S05: (IMAGE) Added functionality to handle PNG files to nn::image.

Added the following classes to handle PNG files to nn::image.

  • nn::image::PngDecoder
  • nn::image::PngEncoder

For more information, see image Library.
In addition, new pixel formats were added to nn::image::PixelFormat, so enumerator compatibility is lost.
For more information, see the API reference for nn::image::PixelFormat.

*10-0-0S06: (INIT) Enabled global file data caching with the default nninitStartup().

Made changes so that global file data caching for the fs library is enabled by the default nninitStartup() defined by the init library.
This change has the following effects on applications that have not replaced the nninitStartup() definitions.

  • The amount of memory heap available to the application is reduced by 32 MB.
  • If you call nn::fs::EnableGlobalFileDataCache(), global file data caching is enabled twice, causing an abort.
  • Due to the changes in file access processing times, problems may occur based on the application's dependency on timing.


To revert the available memory heap size and file access behavior to the specifications for prior to NX Add-On 9.x.x, the application must replace the definition of nninitStartup().
For more information, see Starting and Initializing Programs and Global File Data Cache.

This change only affects NX32 and NX64 applications.

*10-0-0S07: (TOOL) Changed the directory when the options for the settings for the integrated command-line tool (TargetShell) were not configured.

If directories were not specified with the export-setting and import-setting commands, the directories that are created and used have been changed to the following.

  • The directory has been changed from TargetShellSetting directly under the SDK to the TargetShellSetting for the working directory that runs TargetShell.

For more information, see the manual.

10-0-0W40: (3D-G3D) Fixed a bug in the G3dDemo sample where the sample program crashed if the demo shader was detached from a certain model in the Edit scene.

Fixed a bug in the G3dDemo sample where if you connected the Edit scene to 3DEditor and sent a model and attached the demo shader to that model, the sample program crashed if you then detached that shader.

10-0-0W41: (3D-G3D) Fixed a bug where 3DEditor could not connect to the viewer library if nn::fs::MountHostOptionFlag_PseudoCaseSensitive was enabled.

Fixed a bug where if 3DEditor sent a model to the viewer library when nn::fs::MountHostOptionFlag_PseudoCaseSensitive was enabled, an error was generated and 3DEditor became disconnected from the library.

10-0-0W42: (3D-G3D) In the G3dDemo sample, an absolute path is now used to specify the path of the arguments file in the 3DEditor team settings file.

Previously, the path to the arguments file in the team settings file was described using a relative path, and, depending on the current directory, shader conversion failed if 3DEditor was started by running Start3dEditor.bat.
This bug was fixed by changing to use an absolute path instead of a relative path when specifying the path.

Fixed a bug in 3DEditor where an error occurred if the tool was connected to the viewer with TextureCompositor enabled.

Fixed a bug in 3DEditor where connecting the tool to an application and modifying the sampler parameters did not properly sent those modifications to the application.

10-0-0W45: (3DTool) Fixed a bug in 3DEditor where edits to a material were wrongfully applied to a model already sent to the viewer.

Fixed a bug in 3DEditor where edits made to a material that had not been sent to the viewer were also applied to a model already sent to the viewer (which does not use that material).

10-0-0W46: (3DTool) Fixed a bug in 3DEditor where material references became unresolved if Save All in Specified Location was executed.

Fixed a bug in 3DEditor where material references became unresolved if Save All in Specified Location was executed to save materials loaded from the parent material reference path.

Fixed a bug in 3DEditor where if multiple files were all loaded together, notifications of the loading of the second and subsequent files were not sent to the connected application.

Fixed a bug in 3DEditor where selecting Restore Group to Defaults for the material that was set as the parent material in the material property windows fixed the parameters to their default values and they could no longer be edited.
