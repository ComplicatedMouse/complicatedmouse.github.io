Changes in NintendoSDK NX Addon 13.1.0 (based on Generic 13.1.0)

Added Features

*13-1-0S01: (AUDIO) Changed the specifications for the device names specified for the functions for setting and getting the audio output device volume.

Changed the specifications for the device names specified for the functions for setting and getting the audio output device volume.

For more information about the changes to device name specifications, see the revision history for nn::audio::ListAudioOutputDeviceName().

  • nn::audio::ListAudioOutputDeviceName()
  • nn::audio::SetAudioDeviceOutputVolume()
  • nn::audio::GetAudioDeviceOutputVolume()


*13-1-0S02: (AUDIO) Changed the specifications of the function for getting the list of audio output devices supported by the platform.

This version changed the specifications of the function for getting the list of audio output devices supported by the platform.

We no longer recommend the nn::audio::ListAudioDeviceName() function, which previously provided this functionality. Use the newly provided nn::audio::ListAudioOutputDeviceName() function instead.

In the function we do not recommend, the device that outputs audio from the headphone/microphone jack and the device that outputs audio from a USB audio output device were treated as separate audio output devices.
The newly provided function consolidates them as external audio output devices connected to the console.

For more information, see the descriptions of each function.

  • nn::audio::ListAudioOutputDeviceName()
  • nn::audio::ListAudioDeviceName()


*13-1-0S03: (AUDIO) Changed the specifications of the functions for getting the name of the device to which audio is currently being output.

The specification of the function for getting the name of the device to which audio is currently being output has been changed in this version.

We no longer recommend the nn::audio::GetActiveDeviceName() function, which previously provided this functionality. Use the newly provided nn::audio::GetActiveAudioOutputDeviceName() function instead.

In the function we do not recommend, the device that outputs audio from the headphone/microphone jack and the device that outputs audio from a USB audio output device were treated as separate audio output devices.
The newly provided function consolidates them as external audio output devices connected to the console.

The specifications of the function for getting the list of devices supported by the platform were also changed accordingly.

For more information, see the descriptions of each function.

  • nn::audio::GetActiveAudioOutputDeviceName()
  • nn::audio::GetActiveDeviceName()
  • nn::audio::ListAudioOutputDeviceName()
  • nn::audio::ListAudioDeviceName()

13-1-0S06: (DevMenu) Added a feature to automatically mute the speakers.

In Sound Settings, use the Auto Speaker Mute check box to enable and disable this feature.

Accordingly, the feature for automatically muting the speakers only when switching audio output from headphones to speakers (Mute when headphones are Disconnected) has been deprecated.

13-1-0S09: (OS) Added the light message queue feature.

Added the light message queue feature as a synchronization feature for sending and receiving arbitrary data.
The light message queue is functionally similar to the message queue, but with the following differences.

  • The thread does not go into a waiting state even if a light message queue function is called by another core.
  • You cannot specify it as the target for multiple wait.


13-1-0S10: (OS) Added the light semaphore feature.

Added the light semaphore feature as a synchronization feature that uses a numerical counter to enable exclusive use of limited common resources.
Functionally, the light semaphore is similar to the semaphore, but with the following differences.

  • The thread does not go into a waiting state even if a light semaphore function is called by another core.
  • You cannot specify it as the target for multiple wait.

Changes in NintendoWare NX Addon 13.1.0 (based on Generic 13.1.0)

Added Features

13-1-0W01: (Audio-ATK) Added a feature for specifying the rounding direction of ticks for sequence sounds.

Added a feature to SoundMaker for specifying the rounding direction of ticks for sound sequences.
With the addition of this feature, an update is required when reading earlier versions of the FSPJ files.

Set the tick rounding direction for sequence sounds to either the new playback method, rounding up, or the playback method for maintaining compatibility with the previous version, rounding down.
New FSPJ files created with this version or a later version of SoundMaker are set to rounding up by default, creating a different waveform output when playing back a sequence sound.
FSPJ files updated using this version or a later version of SoundMaker or SoundProjectUtility, or project files from previous platforms, are imported by default as rounding down, so the waveform output results do not change when playing back the sequence sound.
These settings can be changed in SoundMaker.

For more information about the rounding direction of ticks for sequence sounds, changes in behavior, and how to change this setting in SoundMaker, see the following pages.

  • NintendoWare > News > atk News ~Siglo1310~
  • NintendoWare > Audio > Tools > SoundMaker Documentation > Option Settings > Sound Archive Settings 1.


13-1-0W02: (Audio-ATK) Added the nn::atk::EffectCapture effect for running the processing for getting sample data.

Added the nn::atk::EffectCapture effect for running the processing for getting sample data.
For more information about using this feature, see the AtkEffect sample. For more information about its behavior, see the nn::audio documentation.

Added settings for the rounding direction of ticks for sequence sounds to the Project > Project Settings > Sound Archive Settings 1 tab.

Improvements

13-1-0W05: (3DTool) Changed the behavior in AssignUtility so that invalid <sampler_assign> elements are not deleted when applying the parent material.

When applying parent materials in AssignUtility (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe) <sampler_assign> elements that reference sampler names that do not exist were previously deleted. However, this practice made it difficult to notice errors in the input data, so these elements are no longer deleted.

During the LOD export in the Maya Plug-In, influences that were not used for some base polygons are now available for LOD polygons.
Influences that are not used for any of the base polygons are still unavailable for LOD polygons.

13-1-0W07: (3DTool) Relaxed the restrictions on the LOD models that can be combined when using the LOD model combination feature in Intermediate File Optimizer.

Relaxed the restrictions on LOD model skinning when using the LOD model combination feature in Intermediate File Optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe). Specifically, previously only the bones referenced by the base shape of an LOD shape could be referenced. However, this has been changed so that bones that are not referenced by the base shape can also be referenced as long as you do not change the matrix table used for skinning.

Added a feature to show a warning in LayoutEditor if the drivers do not support preview with linear color spaces.

Added processing in LayoutEditor for creating a directory when saving thumbnails if the thumbnail save target is a symbolic link and there is no actual directory at that link.
