Details

Changes in NintendoSDK NX Addon 13.3.2 (based on Generic 13.3.2)

Added Features

*13-3-2S01: (TOOL) Added a setting to the NMETA file for the target age for applications for consoles for China.

Added a setting to the NMETA file for the target age for applications for consoles for China. A CADPA icon (target age icon) corresponding to this setting is displayed in the HOME Menu when the application is started. This is supported in the HOME Menu in NintendoSDK Firmware for NX 13.2.0-1.0 and later. For more information, see Writing the NMETA File > Target age setting (for China).

13-3-2S02: (AOC) In light of the easing of restrictions on user accounts, a setting was added to the NMETA file for setting the policy for changes to available content.

A setting was added to the NMETA file for setting the policy for changes to available content.
With the easing of the restrictions on user accounts, it is now possible to set a policy for changes to available content while the application is running.
For more information, see Writing the NMETA File > Policy Setting for Changes to Available Content.

13-3-2S03: (TOOL) Added a feature for creating an NSP file for secure startup from an NSP file that merges a compacted application and a patch.

Added a feature to the makesecure command of AuthoringTool for creating an NSP file for secure startup from an NSP file that merges a compacted application and a patch. For more information, see the following documentation.

  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.


13-3-2S04: (WEB) Added a feature to WebAppletCaller for displaying the settings.

Added a feature to WebAppletCaller for displaying the settings.
For more information, see NintendoSDK Documents > Features > Library Applets > Web Applet and Offline Web Applet > Information Common to Both > WebAppletCaller.

Improvements

13-3-2S05: (TOOL) Added support in AuthoringEditor for displaying the compression ratio of NSP files for which the ROM compression feature was enabled.

Added support in AuthoringEditor for displaying the compression ratio of NSP files for which the ROM compression feature was enabled.
For more information, see Creating Application Images > Resource Data (ROM) Compression.

13-3-2S06: (TOOL) Changed the default addr2line in the crash report retrieval tool to llvm-addr2line.exe.

Changed the default setting for addr2line used for symbol resolution from the conventional aarch64-nintendo-nx-elf-addr2line.exe to the faster llvm-addr2line.exe.

13-3-2S07: (TOOL) In AuthoringEditor, the list of middleware no longer shows NintendoSdk_BuildTarget-64Bit or NintendoSdk_BuildTarget-32Bit.

In SDK 13.0.0 and later, the internal value NintendoSdk_BuildTarget-64Bit or NintendoSdk_BuildTarget-32Bit is used for determining the platform type (NX32 or NX64). Previously, those values were shown in the middleware list in AuthoringEditor.
That has been revised and the middleware list in AuthoringEditor no longer shows those values.

13-3-2S08: (TOOL) The mergensp command in AuthoringTool now supports all compacted applications created in any SDK version.

You can now enter compacted applications created with earlier SDK versions for the mergensp command in AuthoringTool.
As a result, you can now enter compacted applications created with any SDK version for the mergensp command.

Bug Fixes

13-3-2S09: (GFX) Fixed a bug where eglSetBlobCacheFuncsANDROID() did not work correctly.

Fixed a bug where calling eglSetBlobCacheFuncsANDROID() caused eglGetError() to return EGL_BAD_ACCESS, and the provided callback function was not called.

13-3-2S10: (TOOL) Fixed a bug where DirectXShaderCompiler could not compile files larger than 64 KiB.

Fixed a bug where the DirectXShaderCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\dxc.exe) could not compile files larger than 64 KiB. When trying to compile files larger than 64 KiB, the following error occurred.
Internal compiler error: access violation. Attempted to read from address 0x[address].

13-3-2S11: (TOOL) Fixed a bug where the wrong symbols were displayed when the crash report retrieval tool resolved symbols.

The wrong symbols were displayed when the crash report retrieval tool resolved symbols, due to a problem in the parsing of inline functions that caused symbols at other addresses to also be included. To work around this bug, inline function expansion has been disabled.
