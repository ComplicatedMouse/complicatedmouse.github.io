*13-0-0S01: (AUDIO) Removed the nn::audio::SetAudioDeviceMapping(const char* virtualName, const char* physicalName) function because we no longer recommend using it.

Removed the nn::audio::SetAudioDeviceMapping(const char* <em>virtualName</em>, const char* <em>physicalName</em>) function because we no longer recommend using it.

The following significant changes were made in the Clang for NX 1.11.1 update.

  • Made the following ABI changes.
    • time_t is now int64_t in armv7l.
    • pid_t is now 64-bit.
    • Fixed the type sizes and alignments of the pthread types.
    • Changed _IO_FILE, the __locale_struct structure, and the __libc structs structure.
    • The musl-local structs structure might also be different.
    • Changed thread priorities for threads handled by the pthread API from a range of -16 (highest) to 15 (lowest), to a range of 4 (lowest) to 35 (highest).

*13-0-0S05: (Socket) Redefined ssize_t.

In previous versions ssize_t for Windows was defined in nn/socket/socket_Types.h and the Nintendo socket API used this type. However, in this release ssize_t from socket_Types.h is deprecated and replaced with nn::SignedSize for C++ or nnSignedSize for C code. You will also see a difference in the socket API where functions that used ssize_t now use nn::SignedSize. If you want to silence this deprecation warning please enable the NN_SOCKET_DISABLE_SSIZE_T_TYPEDEF preprocessor macro. However, note that you must eventually remove the instances causing the deprecation warnings, because the typedef will be removed in a future SDK version. We recommend replacing any instances of ssize_t with nn::SignedSize in your code. Another option could be to introduce your own typedef for ssize_t.

Added Features

*13-0-0S06: (UTIL) Added the NN_UTF8_TEXT macro for creating UTF-8 strings.

NN_UTF8_TEXT(<em>str</em>) returns the UTF-8 byte representation of the <em>str</em> string literal as a reference to an array. The type is const char (&)[N]. (N is the byte size + 1 of the UTF-8 representation of <em>str</em>.)

This macro was added to internalize the difference in the meaning of the u8 string literal prefix between C++17 and earlier and C++20 and later. In C++17 and earlier, string literals with the u8 string literal prefix have the const char[N] type, whereas in C++20 and later, the type is const char8_t[N]. This difference can lead to compatibility problems. By using this macro instead of the u8 string literal prefix, all of the string literals are treated as the const char (&)[N] type, regardless of the C++ version.

13-0-0S29: (OE) Added a grace period so that auto-sleep does not activate immediately when nn::oe::SetUserInactivityDetectionTimeExtended(false) or nn::oe::EndAutoSleepDisabledSection() is called.

Even if called, functions that change the auto-sleep timer like nn::oe::SetUserInactivityDetectionTimeExtended() or nn::oe::BeginAutoSleepDisabledSection()~nn::oe::BeginAutoSleepDisabledSection() do not reset the inactivity timer.

As a result, when policy changes that shortened the duration for the nn::oe::SetUserInactivityDetectionTimeExtended(false) and nn::oe::EndAutoSleepDisabledSection() auto-sleep timers were made, specifications were such that sleep mode would be entered immediately if the inactive time exceeded the specified time.
The policy is suspended from going into effect for a maximum 90 seconds to prevent entering sleep mode immediately after the function is called, only in cases where the policy change shortens the time before automatically entering sleep mode.

For more information about this feature, see Control Over Idle State Actions.

Because this is a modification to the firmware, this feature will be applied even to apps built with NX Add-On 12.xx or earlier if running on a main unit whose firmware is NintendoSDK Firmware for NX 13.0.0 or later.
