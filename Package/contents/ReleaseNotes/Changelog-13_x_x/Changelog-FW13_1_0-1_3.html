Details

Changes in NintendoSDK NX Addon 13.3.0 (based on Generic 13.3.0) FW 13.1.0-1.3

Specification change that breaks your code

*FW13-1-0-1-3S01: (DevMenuCommand) Fixed a bug where gamecard write failed for NSP files on multi-application cards that contain DLC.

Fixed a bug where the gamecard write DevMenuCommand failed for NSP files on multi-application cards that contain DLC.
This fix allows you to write the file created by the bundleup command in AuthoringTool as is.
In exchange, you can no longer write the NSP files used as the workaround before the fix. These NSP files were the files for multi-application cards that include DLC that were converted to on-card DLC in advance.

Improvements

FW13-1-0-1-3S02: (TOOL) Added support for the update to system version 13.1.0.

Added support for the update to system version 13.1.0.

Bug Fixes

FW13-1-0-1-3S03: (DevMenu) Fixed a bug where it was not possible to start an application under some conditions, even though it should be available to start.

Fixed a bug where attempting to start an application under the following conditions caused the "Please turn Airplane Mode off" message to be displayed and the application could not be started.

  • Cannot connect to the Internet
  • The account specified to start the application can use the application even when not connected to the Internet.
  • The account specified to start the application cannot use (at least some) DLC if it cannot connect to the Internet.

Due to this fix, the following verification message is now displayed when you start an application under these conditions.
"You can launch this application, but you don't have some rights of add-on contents. Launch anyway?"
If you select Launch at this point, the application starts, but the applicable DLC is not available.

FW13-1-0-1-3S04: (DevMenu) Fixed a bug where the system crashed while transitioning between screens during account management.

Fixed a bug where the system crashed during screen transition after increasing the number of user accounts from 7 to 8 during account management.

FW13-1-0-1-3S05: (DevMenuCommand) Revised the message that is displayed when starting a compacted application without the required patch.

Fixed a bug where the wrong message was displayed when starting a compacted application without the required patch. The revised message is as follows.
"This application is compacted. To launch, please install an appropriate patch."
