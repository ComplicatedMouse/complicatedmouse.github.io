Details

Changes in NintendoSDK NX Addon 13.2.0 (based on Generic 13.2.0)

Added Features

*13-2-0S01: (TOOL) Added compression functionality for resource data (ROMs) in AuthoringTool.

Added compression functionality for resource data (ROMs) in AuthoringTool.

You can add the compression option in AuthoringTool to create NSP files with compression enabled.
Decompression of compressed resource data is performed by the system, so the application does not have to pay attention to compression.

For more information, see the following documentation.

  • Creating Application Images > Tool Reference > AuthoringTool
  • Creating Application Images > Resource Data (ROM) Compression


13-2-0S02: (AUDIO) Added a feature to get the total number of samples lost from the buffer managed by nn::audio::CaptureType.

Added a feature to get the total number of samples lost from the buffer managed by nn::audio::CaptureType.
For more information, see the following API references.

  • nn::audio::GetCaptureLostSampleCount()
  • nn::audio::ResetCaptureLostSampleCount()


13-2-0S03: (DevMenu) Added functionality to manage DLC availability.

Added functionality to manage DLC availability to application management in DevMenu.
For more information, see DevMenu > Managing Applications > Managing DLC Availability.

13-2-0S04: (DevMenu) Added functionality to manage user restriction status.

Added functionality to manage user restriction status to application management in DevMenu.
For more information, see DevMenu > Managing Applications > Managing User Restriction Status.

13-2-0S05: (DevMenu) Made changes so that you can write NSP files for secure startup to a game card and start it.

Made changes so that you can write NSP files for secure startup to a game card and start it.
For more information, see Features > Controlling the Development Hardware > DevMenu > Managing Applications > Secure Startup.

13-2-0S06: (HID) Added a feature to DevMenuOverlay to display the sample receiving rate for the six-axis sensor.

Added a feature to DevMenuOverlay to display the sample receiving rate for the six-axis sensor.
For more information, see Controlling the Development Hardware > DevMenuOverlay.

13-2-0S07: (NVN) New nvn Features

Added the following features to GLSLC.

  • Added support for SPV_NV_bindless_texture.


13-2-0S08: (TOOL) Added SDEV-D and EDEV-D to the HardwareType display in ControlTarget.

SDEV-D and EDEV-D are now displayed for the HardwareType shown when using the --csv and --xml options for ControlTarget list-target and detect-target.

13-2-0S09: (TOOL) Added SDEV-D, EDEV-D, and ADEV to the models supported by the integrated command-line tool (TargetShell).

Added SDEV-D, EDEV-D, and ADEV to the models supported by the integrated command-line tool (TargetShell).
For the available subcommands, see the manual.

13-2-0S10: (VSI) New Feature in Target Manager

  • Added support for .NET Core to the Nintendo.Tm.dll library.


Improvements

*13-2-0S11: (TOOL) Enabled Target Manager 2 to remotely operate multiple controllers.

Enabled Target Manager 2 to remotely operate multiple controllers.
The operating method and behaviors have changed due to this feature, so see Remote Video and Remote Input in the Target Manager 2 documentation.

13-2-0S12: (Compiler) Updated to Clang for NX 1.11.3.

Updated Clang for NX to version 1.11.3.
The main changes are as follows.

  • The -Wundefined-var-template warning is now enabled by default. To disable this warning, add -Wno-undefined-var-template to the compile options.
  • The compiler now preferentially uses the specially configured function alignment (-falign-functions) over the default alignment. This change reverts the expanded code size that occurred since Clang for NX 1.11.2.
  • Changed the behavior for cases where pread returns zero.
  • Improved the error code behavior for clock_nanosleep() to more closely reflect POSIX. In addition, fixed some rare timer overflows.
  • errno for rename() is now set to ENOENT if the source path cannot be found.


13-2-0S13: (DevMenu) Added support for HTTP server redirect settings to the Install via HTTP feature.

Added support for HTTP server redirect settings to the Install via HTTP feature in DevMenu.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Managing Applications > Installing HTTP Server NSP Files


13-2-0S14: (FS) Added the --enable_total_time option with FsAccessLogChecker version 41.5.

In FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 41.5, the --enable_total_time option has been added to display the overall elapsed time for the access log even if the result is PASS.

13-2-0S15: (FS) In FsAccessLogChecker version 41.7, added elapsed time to the scope of analysis fpr the "FS Access Log Details" analysis results.

In FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) version 41.7, the time elapsed since the first access log has been added to the scope of analysis for the "FS Access Log Details" analysis results. For applications that immediately access the log when run, you can get information about the time elapsed since the application was started and the access frequency. In addition, the --disable_elapsed_time option was added to suppress the addition of elapsed time. For more information, see fs Library > Tool Reference > FsAccessLogChecker.

13-2-0S16: (LDN) Added nn::ldn::ResultDeviceNotFound.

Added nn::ldn::ResultDeviceNotFound as an error returned when problems occur with the wireless communication functionality of the Switch console.

This error is equivalent to a lower order Result of nn::ldn::ResultDeviceNotAvailable and nn::ldn::ResultDeviceDisabled. For that reason, if you perform appropriate error handling for any one of these errors, there is no need to change the application.

13-2-0S17: (Sample) Added support for C++14 in the Demo1 sample.

Added support for C++11 for the Demo1 sample.

  • Some of the variables that were dynamically allocated and released using new/delete operators have been replaced with unique_ptr.


13-2-0S18: (TOOL) Added line number info for the HLSL wrapper generated by DirectXShaderCompiler with -fspv-debug=line.

Added line number info for the HLSL wrapper generated by DirectXShaderCompiler (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\dxc.exe) with -fspv-debug=line.

13-2-0S19: (TOOL) Added support for hardware acceleration to the remote video codec for Target Manager 2.

Hardware acceleration is not available for the remote video codec for Target Manager 2.

You can enable or disable hardware acceleration in the Target Manager 2 settings.
For more information, see the settings section in Controlling the Development Hardware > Target Manager 2.

13-2-0S20: (TOOL) Added support in AuthoringTool for NSP files for secure startup related to application compaction.

The NSP files for secure startup of the following can now be merged with the mergensp command in AuthoringTool.

  • Compacted applications
  • Patches that support application compaction

In addition, the ROM identification hash has been changed to be the same for input/output NSP files for the makesecure command in AuthoringTool.
You can use this feature to identify the source NSP file that was used to create the output NSP file for secure startup.

13-2-0S21: (TOOL) Changed EDEV/HDEV Settings to USB Development Hardware Settings in Target Manager 2.

Changed the term for EDEV/HDEV Settings to USB Development Hardware Settings in Target Manager 2.

13-2-0S22: (TOOL) Changed the BackupSaveData timeout period.

Extended the maximum time for importing or exporting BackupSaveData on the development hardware.

13-2-0S23: (TOOL) Improved the installation log for the Visual Studio Wizard feature.

Improved the output of the installation log (_InstallVsWizardLog.txt) for the Visual Studio Wizard files that are installed along with the NintendoSDK package.

13-2-0S24: (TOOL) Installing EDEV drivers in advance is no longer needed.

When connecting EDEV via USB on a Windows PC, the required setup is now performed automatically according to WinUSB specifications, and installing drivers in advance is no longer needed.

Currently, drivers are installed automatically when installing NintendoSDK in NDI, but the drivers will be deleted in future SDK versions.

13-2-0S25: (TOOL) Made changes so that the "old SDK version" error is always displayed for the makepatch command in AuthoringTool.

The "old SDK version" error is now displayed even if you do not add the --error-unpublishable option to the makepatch command in AuthoringTool.
For more information, see the following documentation.

  • Features > Patches > Building and Running a Patch > Creating the Patch.


13-2-0S26: (TOOL) Made changes to Target Manager 2 so that the screen does not become black when remote video is stopped in TV mode.

When using Target Manager 2 and remote video is displayed for an SDEV in TV mode, the screen no longer turns dark when the remote video stops.
When remote video stops, the image that was last displayed continues to be displayed.

You can determine whether the remote video has stopped based on the recording prohibited icon displayed on the status bar of the remote window. For more information, see Remote Video and Remote Input in the Target Manager 2 documentation.

13-2-0S27: (TOOL) Reduced the processing load when displaying remote video in Target Manager 2 while connected to a remote desktop.

Previously, displaying remote video in Target Manager 2 while connected to a remote desktop caused a high load on the host PC and increased the display lag for the remote video. This behavior has been improved.

13-2-0S28: (VSI) Target Manager Improvements

  • Made improvements to differentiate between SDEV-D and EDEV-D information in Target Manager (TM1).
  • Added support for ADEV to Target Manager (TM1).


13-2-0S29: (VSI) VSI Improvements

  • Added a change where the output pane becomes debug when a build succeeds in Edit and Continue.
  • Changed the length of the string supported by SetWorkingDirectory so that it matches the maximum length in Windows.
  • Improved the text visualizer for string arrays.


Bug Fixes

13-2-0S30: (AUDIO) Fixed a bug where system behavior was unstable when an invalid memory region was specified for the AudioOut/AudioIn waveform buffer.

Fixed a bug where system behavior was unstable when an invalid memory region, such as a memory region that does not actually exist, was specified for the AudioOut/AudioIn buffer.

This bug was caused by the system firmware of the console.
For this reason, even for applications built in an earlier SDK, this bug does not occur as long as the console on which the application is running has a system firmware version later than the version attached to this SDK.

  • nn::audio::AppendAudioOutBuffer()
  • nn::audio::AppendAudioInBuffer()


13-2-0S31: (AUDIO) Fixed a bug where the system became unstable if you specified an invalid memory region when attempting to attach a memory pool.

Fixed a bug where system behavior was unstable when an invalid memory region, such as a memory region that did not actually exist, was specified when attempting to attach an audio renderer memory pool.

This bug was caused by the system firmware of the console.
For this reason, even for applications built in an earlier SDK, this bug does not occur as long as the console on which the application is running has a system firmware version later than the version attached to this SDK.

  • nn::audio::AcquireMemoryPool()
  • nn::audio::RequestAttachMemoryPool()


13-2-0S32: (CODEC) Fixed a bug where some of the symbol names in AacDecoder were not included in the nn::codec namespace.

Fixed a bug where some of the symbol names in AacDecoder were not included in the nn::codec namespace, causing symbol conflicts when building the application.

13-2-0S33: (Compiler) Fixed a bug where memory could become corrupted with pthread_create.

Unallocated memory can no longer be used as a stack even if the memory alignment returned by malloc is not 4096.

13-2-0S34: (DevMenu) Fixed a bug where the file list was sometimes not displayed correctly in the Install via HTTP feature.

Fixed a bug where the file list could not be displayed correctly if the slash (/) at the end of the URL was omitted when using the Install via HTTP feature in DevMenu.

13-2-0S35: (MOVIE) Deleted the NVN debug layer callback function, which was commented out, from the Movie decoder sample.

Implemented fixes so that the assertion that sometimes occurred when rendering in native texture mode in develop or debug mode in the Movie decoder sample no longer occurs. For that reason, the NVN debug layer callback function for mm_NvnVideoRenderer.cpp, which was commented out in SDK 13.1.0, has been deleted.

13-2-0S36: (MOVIE) Fixed a bug where an error occurred with the AAC decoder due to symbol name conflicts.

Fixed a bug where an error occurred with the AAC decoder due to symbol name conflicts. This error occurred in CodecAacDecoder if you defined the global variable int BitMask;.

13-2-0S37: (MOVIE) Fixed a bug where vp9 playback stopped before it was finished without an error message.

Fixed a bug where vp9 playback stopped before it was finished without an error message when the buffer was insufficient for the files being played back. Changed the input buffer size calculation to provide the correct allocation, and if the buffer is too small, the system returns movie::Status_BufferTooSmall and the error is also displayed in the log.

13-2-0S38: (NGC) Fixed a bug where masking failed when strings with a certain string pattern were input for nn::ngc::ProfanityFilter::MaskProfanityWordsInText().

Fixed a bug where masking failed when strings with a certain pattern were passed to the <em>pText</em> argument of the nn::ngc::ProfanityFilter::MaskProfanityWordsInText() function.

13-2-0S39: (NVN) Fixed a bug where the Environment Checker tool could not check whether an appropriate NVIDIA graphics driver was installed.

Fixed a bug where the Environment Checker tool did not appropriately check whether an NVIDIA graphics driver was installed that is compatible with the NVN API required by NintendoSDK.

This fix has no effect on the usage or display content of the Environment Checker tool.

13-2-0S40: (NVN) NVN Bug Fixes

Fixed the following NVN bugs.

  • Fixed compilation warnings generated by nvn_Fastpath.h in NX-Win32.

  • Fixed a bug where global debug callback could report errors regardless of the error level set by debug layer initialization.


13-2-0S41: (OS) Fixed a bug where nn::os::GenerateRandomBytes and nn::util::TinyMt::GenerateRandomBytes were written to areas outside of the buffer.

Fixed a bug where, when using nn::util::TinyMt::GenerateRandomBytes, if the buffer was not 4-byte aligned and the size was 0 through 2, data was written to areas outside of the buffer.

Because nn::os::GenerateRandomBytes also uses nn::util::TinyMt::GenerateRandomBytes internally, it had the same problem, and this bug was also fixed.

13-2-0S42: (TOOL) Fixed a bug in Target Manager 2 where Nintendo SDK Daemon sometimes did not start.

Fixed a bug in Target Manager 2 where Nintendo SDK Daemon sometimes did not start due to the settings file being corrupted.

13-2-0S43: (TOOL) Fixed a bug in Target Manager 2 where the display name was reverted when changing the target name from the USB development hardware.

Fixed a bug in Target Manager 2 where, when changing the target name from the setting on the USB development hardware, the display reverted to the previous name once and then changed to the new name after reconnecting.

13-2-0S44: (TOOL) Fixed a bug where input via mouse operations on the remote video screen was misaligned when using Target Manager 2 in a high DPI environment.

Fixed a bug where the position of the mouse on the remote video screen was misaligned from the input values passed to the target when using Target Manager 2 on a high DPI environment PC.

13-2-0S45: (TOOL) Fixed a bug where the cache data was not updated appropriately when generating patches in AuthoringTool.

Fixed a bug in AuthoringTool where, when --cache-directory was specified when generating a patch, the generated cache data was not updated correctly.

13-2-0S46: (TOOL) Fixed a bug where the installation log for the Visual Studio Wizard feature sometimes had scrambled encoding.

Fixed a bug where the character encoding was sometimes scrambled for some parts of the installation log (_InstallVsWizardLog.txt) in a multi-byte user name environment. This log is for Visual Studio Wizard files that are installed along with the NintendoSDK package.

13-2-0S47: (TOOL) Fixed a bug where, when Target Manager 2 was set for use and the target was sleeping, the correct status could not be obtained by the tm library.

Fixed a bug where, when Target Manager 2 was set for use and the target was sleeping, the correct status could not be obtained by the tm library.

13-2-0S48: (VSI) VSI Bug Fixes

  • Fixed a bug where memory use suddenly increased from the time when debug information is initially loaded.
  • Fixed a bug where selecting Undo during Edit and Continue does not revert to the correct state.
  • Fixed a bug where, when the number of temporary files in the temp directory exceeds the maximum, an error occurs and the build fails.
  • Revised the order of the input and output libraries when specifying LibAdditionalDependencies in Visual Studio 2019.
  • Revised the library project build so that if a relative path is specified for LibAdditionalDependencies, the library path specified for VC++ directory > library directory is used.
  • Fixed a bug where the NRS file was mistakenly passed to MakeNrr, and improved the NRO path include functionality.
  • Fixed a bug where ControlTarget create-target-program-dump sometimes stopped responding.
  • Fixed a bug where DebugCore crashed when displaying local variables.
  • Fixed a bug where, after disconnecting from the target with htc-gen2, the TMAPI GetTargetStatus() function returned TargetStatus_Connected.


Changes in NintendoWare NX Addon 13.2.0 (based on Generic 13.2.0)

Added Features

13-2-0W01: (3DTool) Added the --ignore-revision option for ignoring revision comparisons to the feature for updating shaders in the AssignUtility.

Added the --ignore-revision option for the update-model, update-material, assign-parent-material, and apply-parent-material subcommands in the AssignUtility (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe) to update shaders without comparing the material revision and shader definition revision.

13-2-0W02: (LayoutTool) Added a feature to the LayoutEditor project settings to specify the maximum size of the drop shadow capture effect.

Added a feature to the LayoutEditor project settings to specify the maximum size of the drop shadow capture effect. For more information, see Project Configuration File Format.

Improvements

13-2-0W03: (3DTool) Changed the sorting method when saving FMD files in 3DEditor to sort materials by name.

Changed the sorting method when saving FMD files in 3DEditor to sort materials by name.

13-2-0W04: (3DTool) Made changes so that double-clicking the material reference icon in 3DEditor changes the settings.

Made changes so that double-clicking the Value Reference icon in the material reference in 3DEditor changes the setting to Overwrite.

13-2-0W05: (3DTool) Made changes so that up to 20 layers can be merged in the Photoshop normal map filter plug-in.

Made changes so that up to 20 layers can be merged in the Photoshop normal map filter plug-in.
In addition, made changes so that you can select filter processing type with tabs.

13-2-0W06: (EffectTool) Changed the folder used for file conversion in EffectMaker.

Changed the temporary folder used when converting textures and shaders in EffectMaker from [EffectMaker.exe folder]\Converter to the %TEMP%\Nintendo_EffectMaker\Work folder.

Bug Fixes

13-2-0W07: (AudioTool) Fixed a bug where SoundMaker forcibly closed during use.

Fixed a bug where repeatedly showing and hiding the parameters panel caused SoundMaker to forcibly close.

13-2-0W08: (AudioTool) Fixed a bug where version and other information was not displayed on the splash screen when starting SoundMaker.

Fixed a bug where the version and copyright information was not displayed on the splash screen at startup.

13-2-0W09: (3D-G3D) Fixed a bug where the G3dCalculateArchiveSize() function threw an exception.

Fixed a bug where a LoaderLock exception was thrown when compilation failed when using 3dShaderConverter.dll, causing the application to crash.

13-2-0W10: (3D-G3D) Fixed a bug where the Model drop-down list for G3dDemo Town and SkeletalAnimation was not displayed correctly.

Fixed a bug where the drop-down list was not rendered correctly if you built G3dDemo (G3dDemo-spec.NX.vs2019.sln, G3dDemo-spec.Generic.vs2019.sln) with x64 using a Develop or Release configuration. This bug occurred in cases where the compiler settings in the imgui project settings used by G3dDemo were MSVC Toolset Version=14.28.29910 or later, and something other than -Od was used for optimization.

13-2-0W11: (3DTool) Fixed a bug in the AssignUtility where update-material did not update <attrib_assign>.

Fixed a bug in the update-material subcommand of the AssignUtility (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileAssignUtility.exe) where <attrib_assign> of the input material intermediate file was not updated.

13-2-0W12: (3DTool) Fixed a bug where contiguous shapes could not be determined correctly in the quantization analysis feature of the intermediate file optimizer.

Fixed a bug in the quantization analysis feature of the intermediate file optimizer (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileOptimizer.exe) where contiguous shapes could not be determined correctly.
