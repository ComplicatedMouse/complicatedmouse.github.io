Details

Changes in NintendoSDK NX Addon 13.3.0 (based on Generic 13.3.0)

Added Features

*13-3-0S01: (AUDIO) Added support for Bluetooth audio devices as an audio output target for the console.

Added new support for Bluetooth audio devices as an audio output target for the console.
For more information about features and important considerations, see the Nintendo Switch System Features Guide.

Bluetooth audio devices are handled as external audio output devices (AudioExternalOutput) in the audio API, tge same as when handling the built-in headphone jack and so on.
For more information about external audio output, see audio Library > Manual > Features > Other > Setting the Output Devices and the API references for the following audio functions.

  • nn::audio::ListAudioOutputDeviceName()
  • nn::audio::GetActiveAudioOutputDeviceName()


For considerations when creating audio for external audio device output, see the Nintendo Switch Sound Design Guide.

*13-3-0S02: (TOOL) Added a save data report feature.

Added a save data report feature, which is a mechanism for collecting save data from a specific user (user save data) through the internet.

You can use the save data report feature to collect save data from users and use it to investigate application bugs.
In addition, if there are bugs in the recovered save data, you can directly fix the save data and write it back to the user's Switch.

For more information, see the Save Data Report Manual.

13-3-0S03: (Compiler) Added support for Dwarf5 and Split-Dwarf on an experimental basis.

Added support for Dwarf5 and Split-Dwarf on an experimental basis.

Dwarf5 (Dwarf debugging information file format version 5) is an update to how debugging information is generated.
It includes multiple features to shrink the size of the debugging symbol that is generated and improve the debugging experience.
For more information about the changes, see the following.
http://www.dwarfstd.org/doc/DWARF5.pdf
To enable Dwarf5, specify -gdwarf-5 in the compiler command line.

The Split-Dwarf feature moves the debugging information from the ELF file to the external DWO file. This change improves the incremental build time and slightly reduces the overall size of the symbol, and may slightly simplify symbol stripping.
In particular, the debugging information will ultimately be stored outside of the ELF file, meaning that operations would become more efficient as Linker does not have to process debugging information that was not changed in files that were not changed.
You can use the llvm-dwp.exe utility to merge multiple individual DWO files into a single DWP file for the entire application. As a result, you can place scattered debugging files into a single file for much simpler management.
To check the original reasons behind creating Split-Dwarf, see the following.
https://gcc.gnu.org/wiki/DebugFission
To enable Split-Dwarf, specify -gsplit-dwarf in the compiler command line.

This feature was implemented on an experimental basis in SDK 13.0.0.

13-3-0S04: (DevMenu) Added a feature to automatically download and install NSP files from an HTTP server.

Added a feature to automatically download and install NSP files from an HTTP server.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Managing Applications > Installing HTTP Server NSP Files > Automatic.


13-3-0S05: (DevMenu) Added Bluetooth audio management features.

Added Bluetooth audio management features to DevMenu.
For more information, see DevMenu > Managing Bluetooth Audio.

13-3-0S06: (DevMenuCommand) Provided a debugging feature to disable the required version check during startup.

We have provided a debugging feature to disable the required version check that is performed when you start DevMenuCommand.
For more information, see DevMenuCommand > Running and Controlling Applications.

13-3-0S07: (NET) Added nn::net::Oui::FromAscii() and nn::net::MacAddress::FromAscii().

Added the nn::net::Oui::FromAscii() function to generate instances of the nn::net::Oui class from ASCII strings, and the nn::net::MacAddress::FromAscii() function to generate instances of the nn::net::MacAddress class from ASCII strings.

13-3-0S08: (SSL) Added nn::ssl::FixedBuiltInManager to retrieve the bundled CA certificate data.

Added a to nn::ssl method to retrieve the bundled CA certificate data. There are limitations on using this feature. Using this feature also requires Nintendo approval. For more information, see the API reference of nn::ssl::FixedBuiltInManager.

13-3-0S09: (TOOL) Added support in AuthoringEditor for editing the NMETA setting values for compressed configuration files.

The NMETA setting CompressionFileConfigurationFilePath supports editing in AuthoringEditor.
For more information about the NMETA elements, see Writing the NMETA File.

Improvements

13-3-0S10: (Compiler) Updated to Clang for NX 1.11.4.

Updated Clang for NX to version 1.11.4.
The main changes are as follows.

  • Reverted the change to Sanitizer implemented in Clang for NX 1.9.3. This change means that dynamic allocation is now used instead of static allocation, which reduces the initial memory overhead.


13-3-0S11: (CPU Profiler) Nintendo NX CPU Profiler Version 1.6.1

  • Changed time selection and measuring on the Timeline tab to limit based on the profile duration.
  • Fixed a data loading error that would occur when thread state tracing is enabled and the code does not have frame pointers.
  • Fixed a bug where the Threads tab would sometimes fail to fill in the Self column when Thread-Centric Data Views was enabled.
  • Fixed a bug where the right-clicking on the Checkers tab could show the wrong tab when Thread-Centric Data Views was enabled.
  • Fixed a bug where a crash occurred when attempting to select leaf functions from the Core 3 Impact row on the Timeline tab.
  • Fixed a display bug on the Timeline tab for Load Graphs when changing the selected Heartbeat.
  • Fixed a bug on the Timeline tab where the Framerate graph would sometimes not appear when it was supposed to, claiming MainHeartbeat needed to be recorded.
  • Fixed some typos in the Assembly tab tooltips.


13-3-0S12: (Sample) Added support for C++14 in the Demo1 sample.

The Demo1 sample now supports C++14.

  • Some of the variables that were dynamically allocated and released using new/delete operators have been replaced with unique_ptr and shared_ptr.


13-3-0S13: (TOOL) Added an option to the shortcut menu to save the Target Manager 2 target log.

Added an option to the shortcut menu to save the Target Manager 2 target log.

13-3-0S14: (TOOL) Added support for the update to system version 13.0.0.

Added support for the update to system version 13.0.0.

13-3-0S15: (TOOL) Expanded the save target display functionality in the screenshot capture window in Target Manager 2.

The following features have been added to the screenshot capture window in Target Manager 2.

  • Added an option to the file list shortcut menu to open the save destination.
  • When opening the save destination, the file that was selected from the file list is now selected in Explorer.


13-3-0S16: (TOOL) For remote operations in Target Manager 2, enabled lining up controllers vertically in the controller status view.

For remote operations in Target Manager 2, enabled lining up controllers vertically in the controller status view.

13-3-0S17: (TOOL) Integrated the ADEV series setup tools into the EDEV series tools.

In NX Add-On 13.2.0, InitializeAdev was used for initialization and SystemUpdateAdev for system updates for the ADEV. However, in NX Add-On 13.3.0 and later, you can also perform these processes with InitializeEdev and SystemUpdateEdev. These two tools now support the same development hardware.

13-3-0S18: (TOOL) Made revisions so that Target Manager 2 retries connections.

Made revisions so that Target Manager 2 retries connections if connection fails.

13-3-0S19: (TOOL) Some improvements have been made to the SPIR-V debug output produced by DirectXShaderCompiler.

Some improvements have been made to the SPIR-V debug output produced by dxc.exe (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\dxc.exe) and dxcompiler.dll (%NINTENDO_SDK_ROOT%\Tools\Graphics\GraphicsTools\dxcompiler.dll). If no source file is available for the HLSL string provided to the compiler, the original source string is passed through for the debugger. This only applies if the -fspv-debug flag is included.

13-3-0S20: (VSI) VSI Improvements

  • Made improvements to the descriptions for "additional NRO files" and "NRR file name" on the Linker Properties page.


Bug Fixes

13-3-0S21: (DevMenu) Fixed a bug where a fatal error occurred if you enabled network service accounts from DevMenu while the library applet was running in the application.

Fixed a bug where a fatal error occurred if you enabled network service accounts from DevMenu while the library applet was running in the application.
When enabling network service accounts from DevMenu, the library applet being run by the application is terminated.

13-3-0S22: (EC) Made improvements to the way the dummy EC applet's CPU usage rate is calculated.

Changed the CPU usage rate, which is used to display a warning for the dummy EC applet, to be based on the average value for the most recent set time period.
This change fixes the bug where the warning is displayed because processing that causes a high load is performed only for a short time, even if the overall load is light.

13-3-0S23: (FS) Fixed a bug where access sometimes failed when the total size of the extended resource data (ROM) exceeded 4 GB.

Fixed a bug in the resource data (ROM) compression feature where access sometimes failed if the total size of the extended resource data (ROM) exceeded 4 GB.

13-3-0S24: (FS) Fixed a typo in the output of the --help option in FsAccessLogChecker version 41.8.

Fixed a typo in the output of the --help option of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) in version 41.8.

13-3-0S25: (General) Fixed a bug where some symbols in the public documentation were not replaced with links to the API references.

Fixed a bug where some of the C++ symbols (functions, classes, macros, and so on) in the NintendoSDK Documents pages were not correctly replaced by links to the API references.

13-3-0S26: (HTC) Fixed a bug where HTC Disconnection Emulation did not work in htc-gen2.

Fixed a bug where HTC Disconnection Emulation did not work in htc-gen2.

13-3-0S27: (Sample) Fixed a bug where starting LdnSimple with airplane mode enabled and then trying again caused the application to abort.

Fixed a bug where an error code (2203-0023) was displayed when starting LdnSimple with airplane mode enabled, and retrying caused the application to abort.

13-3-0S28: (TOOL) Fixed a bug in AuthoringEditor where an error occurred when comparing patch NSP files.

Fixed a bug in AuthoringEditor where using the Compare two files feature to compare NSP files caused an error.

13-3-0S29: (TOOL) Fixed a bug in AuthoringEditor where NSP files with ROM compression enabled were not displayed correctly.

Fixed a bug in AuthoringEditor where opening an NSP file with ROM compression enabled and selecting Program Internal File Structure did not correctly display the information for the compressed source data.

13-3-0S30: (TOOL) Fixed a bug where an error occurred when opening some patch NSP files in AuthoringEditor.

Fixed a bug in AuthoringEditor where an error occurred when opening patch NSP files created from ROM compression application NSP files.

13-3-0S31: (TOOL) Fixed a bug where creating multi-program applications in AuthoringTool failed.

Fixed a bug where, when creating the second or later version of a multi-program application, using a version of AuthoringTool to create it different from the version used to create the preceding patch caused patch creation to fail depending on the version combinations.

13-3-0S32: (TOOL) Fixed a bug where running RunOnTarget/ControlTarget without specifying a target, when there are no targets registered, resulted in an error.

Fixed a bug in Target Manager 2 where, when configuring settings, an error occurred if you ran RunOnTarget/ControlTarget without specifying a target when no target was registered.

13-3-0S33: (TOOL) Fixed a bug where the touch operations cursor was sometimes not displayed in the remote video window of Target Manager 2.

Fixed a bug in Target Manager 2 where, when touch operations are enabled, moving the mouse pointer in the remote window causes a cursor not used for touch operations to be displayed.

Changes in NintendoWare NX Addon 13.3.0 (based on Generic 13.3.0)

Added Features

13-3-0W01: (3DTool) Added a feature to the Maya import plug-in to import materials as extended materials.

Added a feature to the Maya import plug-in to import materials as extended materials.
If you specify the extended material attribute information in the config file export plug-in, you can select the node type of the material in the Material Type options.

Improvements

13-3-0W02: (3DTool) Made revisions in the Maya plug-in so that nodes that are not skinning influences are also exported in the bind pose.

Made revisions in the Maya plug-in so that even if a node is not a skinning influence, if influences exist for child (descendant) nodes and it has a bind pose, it is now exported to the FMD file in the bind pose.

13-3-0W03: (LayoutTool) Made changes in LayoutEditor so that ASTC format textures are converted to BC7 format when sent to the PC Viewer.

Made changes in LayoutEditor so that ASTC format textures are converted to BC7 format when sent to the PC Viewer. Previously, textures were converted to BC3 and transferred.

Bug Fixes

13-3-0W04: (3D-G3D) Fixed a bug in 3dShaderConverter where BFSHA files could not be merged correctly.

Fixed a bug that occurred when using 3dShaderConverter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dShaderConverter.exe) to merge multiple BFSHA files.
Previously, applications sometimes crashed depending on the combination of BFSHA files entered.

13-3-0W05: (LayoutTool) Fixed a bug in LayoutEditor where the SaveStateAsInitialState option is enabled even when it is omitted in the project configuration file.

Fixed a bug in LayoutEditor where the SaveStateAsInitialState option is enabled even when it is omitted in the project configuration file, causing the layout in separated mode to be updated to the default state when saved.
