Details

Changes in NintendoSDK NX Addon 13.3.0 (based on Generic 13.3.0) FW 13.2.0-1.0

Specification change that breaks your code

*FW13-2-0-1-0S01: (DevMenuCommand) Changed the name of the command for setting the number of channels in the audio output target.

Changed the name of the command for setting the number of channels in the audio output target from audio set_output_mode to audio set-output-mode.
For more information about the feature, see DevMenuCommand > Audio Settings.

The old command, audio set_output_mode, can still be used, but it is deprecated and will be deleted in the future.

Added Features

FW13-2-0-1-0S02: (DevMenuCommand) Added a command for getting the current audio output target.

Added a command for getting the current audio output target.
For more information, see DevMenuCommand > Audio Settings.

Improvements

FW13-2-0-1-0S03: (DevMenu) Added support to Install via HTTP > Automatic for deleting all history.

Added support to Install via HTTP > Automatic for deleting all history.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Managing Applications > Installing HTTP Server NSP Files > Automatic.


FW13-2-0-1-0S04: (DevMenu) Added support to Install via HTTP > Automatic for stopping and resuming the automatic process.

Added support to Install via HTTP > Automatic for stopping and resuming the automatic process.
For more information, see the following documentation.

  • Features > Controlling the Development Hardware > DevMenu > Managing Applications > Installing HTTP Server NSP Files > Automatic.


FW13-2-0-1-0S05: (FS) Improved the error message in FsAccessLogChecker version 41.9 for when analysis results are entered.

In version 41.9 of FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe), the error message that appears when analysis results are entered as the input file has been modified to be more understandable.

FW13-2-0-1-0S06: (TOOL) Added support for the system update to system version 13.2.0.

Added support for the system update to system version 13.2.0.

Bug Fixes

FW13-2-0-1-0S07: (DevMenu) Fixed a bug where a fatal error occurred in the DevMenu environment when an application was started and ended from the host PC.

Fixed a bug where starting an application from a program like Visual Studio or Target Manager on the host PC and then later ending the application caused DevMenu to close on rare occasions on a fatal error (error code 2162-0002).

FW13-2-0-1-0S08: (DevMenu) Fixed a bug where the number of channels for the audio output target, specified in DevMenuCommand, was not correctly displayed.

Fixed a bug where the number of channels for the audio output target, specified in DevMenuCommand, was not correctly displayed.
With this fix, when you reopen DevMenu > Device, the number of channels for the audio output target is displayed for both TV Sound and System Sound (Speaker & HP) for the Sound Settings.

FW13-2-0-1-0S09: (DevMenuCommand) Fixed a bug where setting target to sphp in the audio set-output-mode command caused a fatal error.

Fixed a bug where setting target to sphp in the audio set-output-mode command caused a fatal error.

FW13-2-0-1-0S10: (FS) Fixed a bug in FsAccessLogChecker version 41.10 where an incorrect warning was added to the nn::fs::IsMounted() access log prior to mounting.

There was a bug in FsAccessLogChecker (%NINTENDO_SDK_ROOT%\Tools\FsAccessLogChecker\FsAccessLogChecker.exe) where an incorrect warning was added to the access log of the nn::fs::IsMounted() function prior to mounting. This bug was fixed in version 41.10 of the tool.
