Set Selected Segment to Animation Playback Range is a feature that links the range of segment tags created in the Edit animation segment tags window with
the ranges of the time slider, and the range slider displayed in the Curve Editor window.
Enabling and disabling this feature facilitates editing of animations in the Curve Editor window, as shown in the following image.

Automatically Adjust Playback Range When Sending to the Viewer

When this feature is enabled in separated mode, the playback range is adjusted automatically for the viewer.
The entire playback range of the animation created in separated mode is calculated and applied to the viewer.
The following table compares the display results in LayoutViewer and the curve editor when this feature is enabled or disabled.

Editing Animation Segment Tags

Automatically Adjust Playback Range When Sending to the Viewer OFF

Automatically Adjust Playback Range When Sending to the Viewer ON
LayoutViewer

The scale on the slider is the viewer's playback range.

The range is automatically adjusted to the scale of the entire animation.
Curve Editor

No effect on the slider in the curve editor.
Info

When the entire animation is played in the viewer, there is a 10-frame margin in the boundary between segments.
This margin is taken into account when automatically calculating the playback range of the entire animation.

For example, consider a 50-frame segment and a 100-frame segment in the Edit Animation Segment Tags window.
The following scale is calculated when you want to play the entire animation in the viewer.

Automatically Adjust Playback Range When Sending to the Viewer ON: 50 + 10 + 100 = 160

Animation Conflict Warning Feature

An animation conflict occurs when multiple animations are played simultaneously in the game, and different animations try to update the same target (setting).
When this conflict occurs, each animation overwrites the results of the previous animations and the resulting animation is wrong.

Use the Animation Conflict Warning feature to display a warning symbol when this data is created and a comment indicating the location to fix.
This warning symbol does not necessarily indicate the data where the issue lies.
If there are no issues with the displayed game application, there is no need to remove the warning symbol.
