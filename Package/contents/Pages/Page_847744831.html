Cannot connect to target.

The target may not be set to htc-gen2.
See Using Target Manager 2and verify that the target is set to htc-gen2.

Also note that targets initialized with firmware earlier than NintendoSDK 10.4 cannot be connected with Target Manager 2.
Initialize or update the target with NintendoSDK 10.4 or later.

The target list is blank, and targets cannot be added.

Something may have caused Nintendo SDK Daemon to crash.
From the Start menu, restart Nintendo SDK Daemon.

It is also possible that Nintendo SDK Daemon may not be able to communicate normally for some reason.
Close Nintendo SDK Daemon from the task tray or Task Manager.

Cannot perform remote input due to "Connecting to HidShell ...” being displayed.

To run remote input, you must configure the host PC to use Target Manager 2.
See Using Target Manager 2 and enable the Enable htc-gen2 pre-releases version setting on the host PC.

Also, if InputDirector was started before the Enable htc-gen2 pre-releases version setting was enabled on the host PC, you must restart it.
After closing InputDirector once in the task tray, open the remote input window again.

"Ephemeral port" is displayed in the About window of Nintendo SDK Daemon.

Info

This functionality is available in Nintendo Target Manager 2 version 11.3.0 and later.
In versions earlier than 11.2.0, the Nintendo SDK Daemon server settings were fixed to 20182 at localhost (127.0.0.1), so if any other processes have this port open, end them before starting Nintendo SDK Daemon.


Because Nintendo SDK Daemon uses the applicable port when establishing the server for communicating with various tools, in this state an unused port is selected automatically from the ephemeral ports and a server is established.
In this state, programs such as Target Manager 2 and ControlTarget may not be able to communicate with Nintendo SDK Daemon and as a result may not be available, so use one of the following handling procedures.

End processes using the port used by Nintendo SDK Daemon

  1. In the task tray, right-click Nintendo SDK Daemon and click Settings.
  2. In Settings, verify the port number. (The default is 20182.)
  3. Use tools such as netstat to check which process opened the applicable port on localhost (127.0.0.1), and end that process.
  4. Restart Nintendo SDK Daemon.
    • Because the port that was in use may remain for a short period of time after ending the process, we recommend restarting after verifying (with a tool such as netstat) that the port has been closed.
  5. In the task tray, right-click Nintendo SDK Daemon and click About.
  6. Check that the port that was verified in the About window in Step 2 is displayed, and that ephemeral port is no longer displayed.

Changing the Nintendo SDK Daemon Server Settings

  1. In the task tray, right-click Nintendo SDK Daemon and click Settings.
  2. In New Value, enter the unused localhost (127.0.0.1) port number, and click OK.
  3. Restart Nintendo SDK Daemon.
  4. Check that the port that was specified in the About window in Step 2 is displayed, and that ephemeral port is no longer displayed.
  5. Start Target Manager 2. (If it was already started, restart it.)
  6. The connection to Nintendo SDK Daemon fails, and the server select dialog box is displayed. Add the connection destination set in step 2 and connect.
    • If the program starts normally without displaying the dialog box, open Tools > Settings and check the settings in the Nintendo SDK Daemon tab.

Cannot operate Joy-Con controllers attached to the console when using remote input.

If the Joy-Con are attached to the rails of the development hardware after the controller has been set to handheld mode using the remote input feature of Target Manager 2, the Joy-Con will not be recognized.
When attaching the Joy-Con to the development hardware rails, change the remote input controller mode to something other than handheld mode.

If the Joy-Con are already attached to the development hardware rails when you switch the remote input to Handheld mode, you can also operate the system with the actual Joy-Con.

VSI data breakpoints do not work.

For targets that use NintendoSDK 11.x.x or earlier versions of the firmware, VSI data break points do not work if set using Target Manager 2.
This problem is fixed in NintendoSDK 12.x.x and later.
If you are using a NintendoSDK 11.x.x or earlier version of the firmware and want to use data break points, set them using Target Manager, and not Target Manager 2.

When Target Manager 2 starts, a message is displayed stating that it is connected to Nintendo SDK Daemon started by another user (user name).

Nintendo SDK Daemon was started by the user with the shown user name, and the tool has connected to that.
If this result is unintended, take either of the following actions.

Have the user with the shown user name end the Nintendo SDK Daemon.

Switch to the user, and end the Nintendo SDK Daemon that is in the task tray.
When you go back to the original user after that and restart Target Manager 2, the error disappears.

Change the Nintendo SDK Daemon server settings.

Use the following procedure to change the Nintendo SDK Daemon server settings.

  1. From the Start menu, select Nintendo SDK Daemon.
  2. In the task tray, right-click Nintendo SDK Daemon and click Settings.
  3. In New Value, enter the unused localhost (127.0.0.1) port number, and click OK.
  4. End Nintendo SDK Daemon and restart it.
  5. Start Target Manager 2.
  6. Open Tools > Settings and select the Nintendo SDK Daemon tab.
  7. If settings for the port number set in step 3 do not exist, select Add and add the settings for that port.
  8. Make the port set in step 3 the default port.
  9. Restart Target Manager 2 and confirm that the error message does not appear.

Remote Video Is Not Displayed Correctly

Remote video might not be displayed correctly depending on the application running on the target or the host PC environment.
Perform the following handling based on the symptoms.

The video frame rate is low, and the screen does not update.

The remote video might have switched to an alternate mode.
Check whether the running application supports Video Capture, and if so, whether video capture is prohibited within the application.

For more information about alternate modes, see Remote Video and Remote Input.

Images are not displayed or are scrambled.

Info

This applies to Nintendo Target Manager 2 version 13.2.0.0 and later.

If you move the remote video window or resize it, the display might become temporarily scrambled.
This behavior is by design.

Hardware acceleration might not be working correctly for video decoding.
Try the following methods.

  • Restart the host PC.
    • If a problem occurred with the graphics driver, restarting the PC might resolve it.
  • Disable hardware acceleration
    1. Open Tools > Settings and select the Remote Video tab.
    2. Clear the Use hardware acceleration if available check box.
    3. Restart Nintendo SDK Daemon and Target Manager 2.
