Creating Downloadable Content

To create downloadable content, you must perform the following procedures.

  • Create downloadable content metadata.
  • Prepare data contained in the downloadable content.
  • Package the downloadable content.

This section describes these steps in order.

Info

Starting with NX Add-On 3.0.0, the default root directory for NMETA files has been changed from Meta to NintendoSdkMeta.
Although you can continue to use Meta, note that samples and documentation assume that NMETA files are using NintendoSdkMeta.

Warning

Downloadable content that has already been submitted to the Nintendo Developer Portal cannot be rebuilt and resubmitted with a different major version of SDK, even if there are no changes to downloadable content, for example.
When submitting downloadable content with a different major version of SDK, it cannot contain downloadable content that has been submitted in the past.
To include downloadable content that was not previously submitted, use the splitnsp command and mergensp in AuthoringTool, and split and merge the downloadable content.

Creating Downloadable Content Metadata

Create metadata for the downloadable content.

The metadata uses the following format in XML format.

You can write multiple items of information about downloadable content to metadata. One <AddOnContent> directive corresponds to one item of downloadable content.

<?xml version="1.0"? >
<NintendoSdkMeta>
  <!-- Index 1 Add-On Content -->
  <AddOnContent>
    <Index>1</Index>
    <ReleaseVersion>0</ReleaseVersion>
    <ApplicationId>0x0005000c10000000</ApplicationId>
    <RequiredApplicationReleaseVersion>0</RequiredApplicationReleaseVersion>
    <Tag>Index1 Tag</Tag>
    <DataPath>Index1DataPath</DataPath>
    <FilterDescriptionFilePath>FdfForIndex1.fdf</FilterDescriptionFilePath>
  </AddOnContent>

  <!-- Index 2 Add-On Content -->
  <AddOnContent>
    <Index>2</Index>
    <ReleaseVersion>0</ReleaseVersion>
    <ApplicationId>0x0005000c10000000</ApplicationId>
    <RequiredApplicationReleaseVersion>0</RequiredApplicationReleaseVersion>
    <Tag>Index2 Tag</Tag>
    <DataPath>Index2DataPath</DataPath>
    <FilterDescriptionFilePath>FdfForIndex2.fdf</FilterDescriptionFilePath>
  </AddOnContent>

  <!-- Writes multiple items of content information in the same area -->
</NintendoSdkMeta>

The following table describes each item.

For more detailed descriptions of the terminology, see Downloadable Content Terminology.

Setting Description Setting Restrictions
Index

This option specifies the index of downloadable content.

For more information, see here.

Required.
ReleaseVersion

Specifies the release version of the downloadable content.
For the initial release, this is set to 0. For each subsequent update that is released, the value is incremented by 1.


For more information, see here.

Required.
ApplicationId

The application ID of the application that uses this downloadable content.

For more information, see here.

 Required.
RequiredApplicationReleaseVersion

Specifies the required application release version.

For more information, see here.

 Can be omitted. (Set to 0 when omitted.)
Tag

Specifies the content.

For more information, see here.

Required.

Must not overlap with the Tag of a different AddOnContent in the same NMETA file.

The maximum number of characters that can be specified is 128.

You can only use the character codes supported by the Japan, US, and Europe fonts, and the Korean, Simplified Chinese, and Traditional Chinese fonts described in Shared Data > Internal Fonts in the System Features Guide.

Info

Korean, Simplified Chinese, and Traditional Chinese fonts can be configured in NX Add-On 7.2.0 and later.
DataPath

Specifies the path of the data directory included in the downloadable content.

When a relative path is specified, it is the relative path from the directory that ran AuthoringTool.


To make use of an environment variable, specify UseEnvironmentVariable="true" for the DataPath attribute.


For example, if the environment variable is AOC_ROOT and the path is set to D:\Project\Aoc, DataPath will be evaluated as D:\Project\Aoc\Index1Data.


<DataPath UseEnvironmentVariable="true">%AOC_ROOT%\Index1Data</DataPath>
Required.
FilterDescriptionFilePath

Specifies the applicable FDF file path for DataPath. This path is prioritized over paths specified using the --filter option in AuthoringTool.

The path specification rules are the same as for DataPath.

For more information about FDF files, see AuthoringTool.

 Can be omitted (but in that case, FDF files are not applied).


Preparing Data to Include in Downloadable Content

Because downloadable content can include any directory on a PC, place the data you want to include in the downloadable content in the directory specified using the DataPath of the metadata.

Packaging the Downloadable Content

Packaging is required to install the downloadable content on production devices.
Use the AuthoringTool included in the SDK to package the content.
You can package the downloadable content using the following command.

%NINTENDO_SDK_ROOT%Tools\CommandLineTools\AuthoringTool\AuthoringTool.exe creatensp -o <OutputFilePath> --save-adf --type AddOnContent --meta <MetaFilePath>

Identifying Downloadable Content

One package (NSP file) contains multiple items of downloadable content.
A ROM identification hash value is specified for each of these content items.

For more information about the ROM identification hash, see Identifying the Build Output.
A new value is obtained for the ROM identification hash for downloadable content each time the content or metadata other than tags is changed.
Note that revising tags does not affect the ROM identification hash value.

Use the AuthoringEditor to check the ROM identification hash for each item of content.
