Reference Colors

This section provides a description of reference colors, which are used to assign a consistent set of colors across multiple layouts.

Overview

The reference colors feature allow you to assign a consistent set of colors across multiple layouts.

Reference colors are saved in reference color (FLRC) files under the parts root directory. Both reference colors and reference color files can be created from the Color Sample window.
When LayoutEditor is started, it automatically loads reference color files placed under the parts root directory.
Reference color features can be summarized as follows.

  • Creating and editing reference colors
  • Assigning reference colors to and removing them from properties

Important Information Consider the following important information for this version.

  • If you assign a reference color to a layout and then either rename or delete the color from the Color Sample window, it may no longer be possible to resolve references to the color when the layout is opened in LayoutEditor or during binary conversion, resulting in an error.
  • The personal settings displayed in the Color Sample window are not reference colors. When they are assigned via a drag-and-drop operation, only the color values are overwritten.


Info

You must specify a directory that contains a reference color file (FLRC file) in the --reference-color-directory argument when performing binary conversion of a layout that uses a reference color.
A binary version of the layout is generated using the color values that have been assigned to it from the reference color file.
