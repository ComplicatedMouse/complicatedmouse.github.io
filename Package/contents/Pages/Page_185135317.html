Provided Features

NX patches provide the following functionality.

  • Fix bugs and add features to card-based applications that have already been released.
  • Fix bugs and add features to downloadable applications that have already been released.
  • Incorporate patches into game cards to manufacture the game cards with bug fixes (so-called on-card patches).

For any of these applications, patches can make use of common parts.

Partial File Patch

With NX patches, if only some files need revising, just those revised files can be incorporated in the patch.

On past platforms, if an archive with several files was created for fast loading, the patch had to incorporate the entire archive even if only some of the files needed revising. As a result, sometimes the patches were huge.

The NX system supports partial patches, so this problem does not arise very often. The trade-off is that there are a number of restrictions when using differences to incorporate the parts that need revising. For more information, see Optimization Information.

Patch Updates

If multiple patches have been released, the update between patches corresponds to the difference in the amount of revision implemented in each. For example, if patch v1 is 100 MB and patch v2 is 110 MB and the difference in the amount of revision between patches is 20 MB, users who downloaded v1 only need to download that 20 MB. (This example is simple. The size of the download varies depending on the sizes of the patches and the content of the revisions.)

Developers do not need to pay close attention to the updating of patches.

Patches and File System Performance

If a patch is applied to the application, the performance of the file system which the application references will change. For more information about what is affected, see Patches and Performance

Special care is required with Firmware for NX 1x because a bug makes the reading speed slower than expected.

If the change in performance of the file system affects the performance of the game, test the state in which the patch is applied.

SDK Versions That Can Develop Patches

To create patches, you must migrate to NintendoSDK NX Add-On 1.0.0 or later.

However, the patches can be applied without problem to applications that were created using earlier versions of NintendoSDK.
