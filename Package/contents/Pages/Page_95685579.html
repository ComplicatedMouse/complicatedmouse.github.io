Playback Method - Loop

This section describes the Loop playback method for texture pattern animations, which can loop playback.

Overview

The Loop playback method animates each pattern for the frame interval set for Sequential frames per pattern.
If one cycle is shorter than the lifespan, you can loop the animation.
The following overview describes the settings ().

Select to display the Loop () and Pattern Tables () settings. The group has the following settings.

Sequential Frames per Pattern

Sets the number of frames to display for each pattern.
Related Links For more information, see here.

Number of Pattern Tables To Use

Sets the number of patterns to use for the animation.
Related Links For more information, see here.

Random Starting Frame

Select or clear the check box () to set whether to randomize the pattern that is displayed when the animation starts.

Example Configurations

The following image compares what happens when the playback method is Loop and the value for the lifespan is changed. It also shows details about the settings.

Texture Pattern Image
Sequential Frames per Pattern 5
Number of Pattern Tables To Use 16
Table Easy Input 0 to 15
A

Lifespan = 50

Playback time per pattern (5 frames) / Particle lifespan (50 frames) => Only 10 patterns can be played.
After the 10 patterns have been played, the animation plays starting from the first pattern.
B Lifespan = 100

Playback time per pattern (5 frames) / Particle lifespan (100 frames) => Up to 20 patterns can be played.

All 16 patterns are played in the cycle, after which the animation loops back to play starting from the first pattern.
C Lifespan = 200

Playback time per pattern (5 frames) / Particle lifespan (200 frames) => Up to 40 patterns can be played.

All 16 patterns are played in the cycle, after which the animation loops back to play starting from the first pattern.

The following figure compares what happens when Lifespan = 100 and Random starting table is enabled and disabled.

ON The starting pattern of the animation varies randomly each time a particle is emitted.
OFF The 0th pattern is always the starting pattern of the animation each time a particle is emitted.
