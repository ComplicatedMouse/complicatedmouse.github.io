Overview

These settings are for adding and deleting users and getting network service accounts.

How to Use

Changing User Account List Order

Info

This feature is available in NintendoSDK Firmware for NX 10.0.0-1.0 and later.

With the focus on a specific user account in the accounts list screen, press the + Button or - Button to change the user position in the list.
Press the + Button to move the user one position toward the front of the list, or the - Button to move it one position toward the back of the list.

These operations are applied automatically to the behavior of functions that enumerate user accounts, such as nn::account::ListAllUsers() and nn::account::ListOpenUsers().

Information Displayed in Account Management

Item Name Description
Nickname The specified nickname is displayed within quotation marks (").
Profile Image The specified profile image.
UID An ID that is automatically issued to each user account.
Open Status When the user account is open, the text is displayed in white.
When the account is closed, the text is displayed in gray.
LastOpened Flag The last user account that was opened (which can be the current account) is displayed in white text.
Other accounts are displayed in gray text.
Info

The UID is only displayed to facilitate development.
This ID is not displayed on the retail devices and is never seen by end users.

Note

The account guides for each platform describe recommended ways to control how and when user accounts are opened and closed.
Application developers can use the Open Status and LastOpened Flag items to check whether their applications are controlling user accounts as intended.


Firmware Settings

You can configure the following settings.

Item Name Description
Requires Nintendo Account for use of network service You can change whether a linked Nintendo Account is needed to use network services.
The default is Enabled.
Enable Nintendo Account license verification You can change whether to verify credentials when signing in to a network-service account.
The default is Enabled. (This behavior is effective in NX Add-On 5.0.0 and later.)
Enable user qualification verification

Do not disable this feature if you have not received a detailed description about it.

The default is Enabled.
Skip user account selection if possible Specifies whether to skip the selection of a user account when the application starts.
The default is Disabled.
Suspend background activity temporarily Temporarily pauses autonomous system actions, including the synchronization of profiles and Nintendo Account status, for user accounts.
The default is Disabled.
Enable inter-program open user retention Do not use this feature if you have not received a detailed description of the feature.
The default is Disabled.

Adding User Accounts

You can add up to eight user accounts.

Editing User Information

When you select a user, you can change information for that user or delete the user entirely. The information that you can edit depends on whether a Nintendo Service Account has been registered for that user.

Viewing Play Data

You can use the View play data button to display the most recently played application and the play duration for the target account.

The only applications that appear are those that have been installed and those started from game cards. Applications started from a PC, using RunOnTarget or TargetTool, do not appear.

Edit profile image

You can use the Edit profile image button to change the user profile image.

The following profile images are available in DevMenu.

Profile Images for Retail Product
Info

This feature is available in NintendoSDK Firmware for NX 10.0.0-3.0 and later.

Select Edit with account manager to open the profile image editing screen for retail products.

Change Profile Image Color

This option changes the color of the profile image. You can select from the following colors.

  • Indigo (the default value)
  • Blue
  • LightCyan
  • Green
  • GreenYellow
  • Yellow
  • Orange
  • Red
  • LightPink
  • Magenta
Setting Invalid JPEG Data for Profile Image

Select Corrupted JPEG to set invalid JPEG data as the profile image.

Settings Editable Before NSA Registration

Register NSA

This option registers a Nintendo Service Account for the user.

Info

(NintendoSDK Firmware for NX 6.x.x and earlier)
If you are using the Friend Presence feature, you must run Upload notification token every time an NSA is registered.

Delete user account

This option deletes a user.

Settings Editable After NSA Registration

Unregister NSA

This option unregisters a Nintendo Service Account.

Sync user profile

This option syncs a user profile with the server.

Upload notification token (NintendoSDK Firmware for NX 6.x.x and earlier)

This option uploads an NSA notification token to the server.

Info

The NSA notification token is required for notifications to reach the device where the NSA is registered.
The token is used primarily with the Friend Presence feature.

Info

You must have a network connection already established on the network settings page.
If you get a 1540413 error, check the network connection state.

Force upload user profile

This option uploads the current profile to the server.

Update NA info cache

This option updates the Nintendo Account information cache.

[Debug] Invalidate token cache

This option destroys the authentication token cache for this user.

Info

Users are not differentiated, and information at the development hardware level, and the application level, is not destroyed.

[Debug] Set user state to invalid

This feature is for debugging. It invalidates a user. The state can be set to the following states.
Differentiating these states is not required in standard application development.

  • NetworkServiceAccount: "CredentialBroken"
  • NetworkServiceAccount: "Unmanaged"
  • NetworkServiceAccount: "Banned"
  • NintendoAccount: "LinkageBroken"
  • NintendoAccount: "StateOtherButInteractionRequired"
  • NintendoAccount: "StateDleted"
  • NintendoAccount: "StateBanned"
  • NintendoAccount: "StateSuspended"
  • NintendoAccount: "StateWithdrawn"
  • NintendoAccount: "StateTermsAgreementRequired"
  • NintendoAccount: "StateReauthorizationRequired"
Info

You cannot generate errors on the application side on consoles with NintendoSDK Firmware for NX 8.0.0-1.0 and later, even when this debugging feature is used.
To check whether an application is properly handling a state where the user is disabled, use the procedure in 0121: Implementing handling when your application is unable to use a network service account in the Nintendo Switch Guidelines.
