Overview

Updates intermediate files to the latest version.

Only use this tool to update the version of intermediate files, because old intermediate files cannot be directly converted to binary files.
Note, however, that there are some cases depending on the workflow where this updater is unnecessary because the intermediate files are updated when re-exported from the DCC tool, edited using 3DEditor, and then merged.

Dragging an intermediate file or folder to the executable file (3dIntermediateFileUpdater.exe) or a shortcut to that file updates the intermediate file and overwrites it.

Note that files are not overwritten if there is nothing to update.
Files other than intermediate files are ignored.

Use this tool after making a copy of intermediate files to ensure there are no problems with intermediate files being overwritten.

For a listing of updater operations performed on each version, see the Revision History in the Intermediate File Specification manual.

Command-Line Options

3dIntermediateFileUpdater.exe [--args-file ""] [--version] [-h] [-s] [-j ""]
[--enable-extension ""] [--disable-extension ""] [--disable-file-info] [--error-detail-output ""] [intermediate_file|folder]...

Updates intermediate files to the latest version.

--args-file ""

Specifies the file.
You can use the contents in the parameter file to specify arguments.
The parameter file allows you to specify each option over several lines.
Lines starting with a number sign (#) are treated as comments.
Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--version

Displays version information.

-h --help

Displays Help.

-s --silent

Suppresses the display of messages during processing.

-j "" --jobs ""

Specifies the number of jobs. When unspecified, the processor count is assigned.

--enable-extension ""

Specifies the extensions to be processed among text intermediate files. 　
Use a comma (,) as a delimiter when listing more than one filename extension. 　
Example: --enable-extension "fmda,ftxb"

--disable-extension ""

Specifies the extensions to be processed among text intermediate files. 　
Use a comma (,) as a delimiter when listing more than one filename extension. 　
Example: --disable-extension "fmda,ftxb"

--disable-file-info

<file_info> is not updated when updating intermediate files.

--error-detail-output

Specifies a folder for saving detailed information when an error occurs. If not specified, this information is written to the default folder.

Intermediate File | Folder

Specifies the file or folder to be processed.

Files located under the specified folder are recursively processed when a folder is specified.
Files other than intermediate files are ignored.

Supports wildcards (?: Replaces one character; *: Replaces one or more characters).
Example: "a?b*"
Converts files with a first letter "a" and a third letter "b”.

Usage Examples

This section describes how to use the intermediate file updater.

Updating an Intermediate File

Updates old_version.fmdb to the latest version of specifications.

3dIntermediateFileUpdater.exe old_version.fmdb

Updating Folders

Updates the intermediate files contained within the folder old_version and its subfolders to the latest version of specifications.

3dIntermediateFileUpdater.exe old_version\

Updating User Tool Data

The intermediate file updater cannot perform user-specific processes such as updating user tool data.
In such scenarios, use the intermediate file filter tool to update.
