Info Window

The Info window displays general information about the profile.

Info Window Toolbar

The following controls are in the Info window toolbar.

Expand All Button: Expands all possible items in the view.

Collapse All Button: Collapses all possible items in the view.

Copy Button: Copies to the clipboard the contents of the window exactly how it appears in its expanded or collapsed state.

Environment

The Environment section has information on the profiler's operating environment at the time of the profile. This includes the following information:

  • User: The name of the user logged into the computer.
  • Computer: The name of the computer running the profiler.
  • Time: The time at which the profile was taken.
  • Game: The path to the executable file on the hard drive.
  • Captured By: The name and type of dev kit used to capture the profile.

Sampling Settings

The Sample Settings section shows the settings that were used to generate the profile.

  • Listed per Core:
    • The rate at which sampling was requested, either by time or by performance counter event trigger.
    • Whether Callstack profiling was enabled.
    • Whether performance counters were recorded with the profile, and if so, which performance counter group was selected.
    • Whether user logged frame markers were seen on the core.
  • Profiling Mode: Whether In-Process or Out-of-Process profiling was used to take the profile.

Profile Statistics

The Profile Statistics section shows general information about the profile. This includes the following:

  • Buffer Used: The size of the raw data buffer.
  • Listed per Core:
    • Samples: The number of samples seen on the core.
    • Duration: The duration of the profile on the core. It is possible for there to be a slight variance between durations on each core.
    • Functions Seen: The number of functions seen executing on the core.
    • Threads Seen: The number of threads seen executing on the core.
    • Effective Sampling Rate: The rate at which the game was actually sampled (number of samples per frame, given an assumed 60Hz framerate).
    • Max callstack depth: The deepest callstack at any time while profiling (only visible if a callstack profile was taken).
    • Call tree nodes: The total number of nodes in the Call Tree tab (only visible if a Callstack profile was taken).
  • Profiler Overhead:
    • Per sample overhead: The amount of time it took to take each sample.
    • Overhead per 60Hz, 30Hz frame: The amount of time sampling took with respect to each framerate.
    • Overhead per frame: The amount time sampling took as a percentage of all frame time.

Module Statistics

The Module Statistics section shows the various modules that were seen during the profile. This is split into Recorded Modules, Resolved Modules, and Module References.

Recorded Modules are those relocatable modules that the profiler saw loaded and/or unloaded. For each recorded module there should an entry in the Resolved Modules list.

Resolved Modules are the actual modules that were loaded into the profiler in order to resolve symbol address and names. For static modules there will be an entry in this list, but not in the recorded modules list.

Module References are the references actually obtained from profiler before they move into Recorded Modules. These are now shown primarily for troubleshooting purposes.

Each module listing shows the following:

  • The name of the module.
  • The path which the module was loaded from (only for Resolved Modules).
  • Base: The address at which the module was loaded into memory.
  • Size: The reported size of the module in memory (only for Recorded Modules and Module References).
  • Functions: The number of functions loaded from the module (only for Resolved Modules).
  • Lifetime: The times during which the module was loaded into memory in the format "[time loaded, time unloaded]".

Inferred Heartbeats

The Inferred Heartbeats section shows any inferred heartbeats that were automatically detected while analyzing the profile data. The following information is provided per inferred heartbeat:

  • The core the inferred heartbeat was found on.
  • Period: The period at which the function was found to execute.
  • Variance: How close the function was to a consistent period.
  • Thread: The ID of the thread which the function was running on.
  • The name of the function.

Application Information

The Application Information section provides general data about the running application. The following items are recorded:

  • Average Frame Rate: The average frame rate seen while profiling. This relies on the user logged main heartbeat.
  • Application Build SDK: The SDK that was used to build the application that was profiled.
  • Threads: The thread list returned from the profiler.
    • ID: The ID of the thread.
    • Name: The name of the thread.
    • Affinity: The core affinity of the thread. If this is empty, then the data was not able to be obtained for the thread.
    • Priority: The thread's priority. If this is empty, then the data was not able to obtained for the thread.

Power Information

The Power Information section provides data gathered when Power Stats was enabled for profiling. A variety of information is recorded and displayed. Most of the values shown are averages followed by the minimum and maximum values seen during profiling.

  • Estimated Battery Life: The estimated amount of time in hours the application could run on battery power.
  • Temperatures: The internal and external temperatures.
  • Frequencies: The CPU, GPU, and EMC frequencies. In general, these values will not change during a profile.
  • GPU Load: The load the GPU was under during the profile.
  • Watt Usage: A breakdown by component on where power is being used by the system.

Profiler Information

The Profiler Statistics section shows information about the profiler runtime that was used to record the data in the profile. While not directly useful most of the time, these values are useful when reporting problems seen with the profiler.

  • Profiler Version: The version of the profiler used to record the data.
  • Profiler Build Date/Time: The date and time that this version of the profiler library was built.
  • Profiler Build SDK: The SDK used to build the profiler.

Notes

The Notes text box at the bottom of the Info window allows you to store user notes with a profile. These notes are saved along with the profile when it is saved with the Save button. Notes can only be added with a newly taken profile.

If there are no notes, the Notes section will be collapsed with only the header showing. To expand the Notes section, either click the up arrow on the right side of the Notes header or drag the Notes header upward with the mouse.
