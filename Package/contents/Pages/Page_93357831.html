Run on CPU

This section describes the Run on CPU feature, which lets you switch between using the CPU and the GPU for calculating particle behavior.

Overview

This feature is for switching between the CPU and GPU for particle behavior calculations. Its availability depends on the features being used by the emitter.
The following is an overview of the setting.

Run on CPU

Select or clear this check box () to enable or disable this method of calculating particle behavior.
Run on CPU is forced to ON when a custom action or stripes are added.

ON

The emitter type is run on the CPU and the particle computation is performed by the CPU.
OFF

The emitter type is run on the GPU or on the GPU+SO depending on the settings and the particle computation is performed by the GPU.

The table describes characteristics and the display of emitter nodes in the workspace.

CPU

Uses the CPU to process the position and speed of particles.
It is used when retrieving particle information from the application and when controlling with in-game information.
We recommend that you turn off Run on CPU when there are a large number of particles, as it will consume a large amount of CPU processing.
GPU Uses the GPU to process the position and speed of particles.
Large numbers of particles do not consume large amounts of CPU time. Emitter processing, does consume CPU time.
GPU+SO (GPU + StreamOut)

Processes particle position and speed using the GPU, and saves the processing results to memory to use in the next frame.
Depending on the feature in use, this setting may be automatically enabled when Run on CPU is set to OFF in EffectMaker.
Large numbers of particles do not consume large amounts of CPU time, but this setting does consume large amounts of working memory.
