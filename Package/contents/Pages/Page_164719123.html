nn::vfx News

News for developers about changes to nn::vfx in NintendoSDK 0.14.0.

Added support for placing an effect binary in a user-created memory pool.

You can now place an effect binary in a user-created memory pool and then initialize that effect binary.

Related Links: For more information, see the following links.

Memory Pools

Describes memory pools created within the VFX runtime and how to place effect binaries using memory pools prepared by the user.

Optimized GPU processing by adding support for compute shader variations.

Accelerated GPU processing by adding support for compute shader variations. The speed for compute shader processing was increased between ten and thirty percent, based on support for variation. Compute shader binaries for the number of used feature combinations will be created. However, the size of the effect binary that is ultimately output will increase based on the number of compute shader variations generated.

nn::ui2d News

News for developers about changes to nn::ui2d in NintendoSDK 0.14.0.

Optimization of the nn::ui2d::Material Class Size

We have streamlined the member variables within the nn::ui2d::Material class and optimized the class size. This optimization has reduced the nn::ui2d::Material class size by about 60-70%. Due to this optimization, some API interfaces have changed and the application codes may need to be changed in some cases. For more information about the changes, see the Revision History.

Advanced Combiner Mode Implementation

You can switch to an advanced combiner code in LayoutEditor and check the edited content in the runtime.

Added Compare (Darken) and Compare (Lighten) to the Color Blending Settings in the Texture Combiner.

Added Compare (Darken) and Compare (Lighten) to the Color Blending Settings in the Texture Combiner.

Support for the BC7 Format

Expanded LayoutEditor to be able to use BC7 format.

BC7 is an RGBA color format, and there is 1 byte of texture format (1/4 of RGBA, twice of BC1) for each pixel.
Compared to BC3, the format has less compression noise and displays at a slightly higher quality. However, be aware that conversion takes more time (particularly if the size is large).

Fixed a bug where a different binary file is output every time the LayoutConverter outputs binary files.

Fixed a bug where a different binary file is output every time the LayoutConverter outputs binary files. Differences occurred in the *.arc, *.bflyt, and *.bflan files.
