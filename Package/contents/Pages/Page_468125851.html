Executor

Overview

nn::async::Executor is a abstract base class representing the asynchronous processing executor.

When performing asynchronous processing within an application process, the async library does not automatically start a new thread. Instead, it requests the Executor object from the user of the async library. The Executor class manages resources such as threads and memory, assigns threads, and starts the asynchronous processing. The async library requests the assigned Executor object to schedule asynchronous processing. This starts the asynchronous processing.

The Executor object must be created and managed by the application itself. In other words, the application process thread for performing the asynchronous processing must be managed by the application itself.

If the asynchronous API function used requires an Executor object, select the appropriate Executor object and pass it to the function based on the thread asynchronously executing that function.

Executor is an abstract base class, and the actual function is provided by its derived class. Refer to Implementation Class List. Note that an application cannot implement any Executor derived class.

The Executor class does not currently have a public interface.
