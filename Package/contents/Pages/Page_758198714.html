Command Line Options

In order to provide more control over some of the behavior of the profiler GUI, some command-line options have been provided. The format of the command line should be as follows:

Command Line Format

"Nintendo CPU Profiler.exe" [options] [NPROF] [ELF] [Executable]

[options]

  • Zero or more of the options (see below).

[NPROF]

  • Enter in a filename of a previously saved NPROF (C:\path\file.nprof) to have it loaded automatically when the profiler starts.
  • Important: The NPROF parameter should not be specified with the ELF or Executable parameters.

[ELF]

  • Enter in the path to an ELF file that you would like to use for syncing (C:\path\example.nss). This tells the GUI that it should perform a Sync with the specified file as soon as the Sync option becomes available. If Executable is not specified, the profiler will attempt to find it automatically.

[Executable]

  • Enter the path to an executable file to be used for syncing (C:\path\example.nsp). This tells the GUI that it should perform a Sync with the specified file as soon as the Sync option becomes available. If ELF is not specified, the profiler will attempt to find it automatically.
  • Important: The NPROF parameter should not be specified with the ELF or Executable parameters.

Options

--autosave[=filename]

  • This tells the GUI that it should save out the results of the profile immediately, rather than requiring a button press. Only the NPROF will be saved if this option is used.
  • The filename in the command line is optional. If not present, the filename will be based on the currently loaded ELF and will be saved into the same directory as the ELF. The final filenames used will be in the form: filename-YYYYMMDD-hhmmss.nprof

--loadelf=filename

  • Works the same as specifying [ELF] on the command line directly.

--script=filename

  • When the profiler starts up, it will automatically run the specified script. This operation will be performed after a specified NPROF is loaded, but may occur before --loadelf has completed.

--shutdown

  • This option will shutdown the profiler GUI after a profile has been received. It is recommended that this option only be used with the --autosave option.

--targetdevkit=identifier

  • Specifies a unique identifier for a devkit that should be used when attempting to automatically Sync with a devkit. This is useful when there may be more than one valid target and a specific devkit is needed. The identifier should be an IP address, serial number, or devkit name.
