Align Pane

The section describes the Align Pane, which is used for aligning and adjusting the positions and sizes of subordinate panes in the Hierarchy window.

Overview

The align pane () is a pane for aligning and adjusting the positions and sizes of subordinate panes in the Hierarchy window with their arrangement relative to one another being maintained.
The processing of this feature takes place when the game application is running, so it can be used to dynamically change the sizes of panes in the program.

For example, if the string in a text box pane () changes when the application is localized, this feature helps to maintain the relative position of the icon placed to the side, as shown below.

A The Align Pane feature is not used.

B The Align Pane feature is used. The spade icon (♠) moves, following exactly along the string.

Normally, panes below the Align Pane () are not rendered in the Hierarchy window.
The following table shows the Align Pane settings.

Base Pane Sets information including the pane name, position, shape, and center position.
Align Pane

Sets how to adjust the panes in the Align Pane () hierarchy.
Extended User Information User-defined extended user information for the pane.

Align Pane

The following table shows the Align Pane () settings.

Alignment Method (Horizontal) Sets the position in the Align Pane to which to align the panes in the horizontal direction in the hierarchy.
Alignment Method (Vertical) Sets the position in the Align Pane to which to align the panes in the vertical direction in the hierarchy.
Alignment Direction Set the standard direction of position alignment for the alignment process.
Default Margin Sets the spacing, in pixels, between child panes when the panes in the hierarchy are being aligned.
Stretch End Pane

Sets whether to stretch the width of the pane at the end, when the panes in the hierarchy are being aligned.

The pane that gets stretched will vary, depending on the setting for Alignment Method (Horizontal).
Info

You can enable and disable the Align Pane feature in the project configuration file. The feature is enabled by default.

Alignment Process

This section uses an example to demonstrate the alignment process.

Hierarchy Window Layout Window


Calculating the Area Occupied by the Pane

First, calculate the alignment area for each of the two panes below the Align Pane, A and B.
For the null pane A, the size of a below that pane in the hierarchy is used as the area of alignment processing. Because the string varies, the size of the text box pane B gets adjusted during the alignment process. For this reason, the size of this pane is not used.

Info

Only the panes directly below the Align Pane in the hierarchy are the targets of the alignment process.
Panes that are child panes of a pane below the Align Pane are not reordered by the functioning of the Align Pane.
However, panes are targets of the alignment process in the following cases.

  • Null Pane
    Search for panes that are child panes of the Null Pane, and use the sum of the pane sizes in the hierarchy as the size of the Null Pane.
  • Parts Pane
    Do not use the size of the Parts Pane itself, but rather, analyze the positions of the panes inside the Parts Pane to calculate the size.
Adjusting Position Based on the Calculated Area

Next, adjust the positions of the panes so they are aligned in the calculated area.
If Alignment Direction is Left or Center, align in the order of A → B from the top of the hierarchy. If the direction is Right, align in the order of B → A from the bottom of the hierarchy.

Restrictions

Align Pane () Restrictions

The hierarchical structure in the Hierarchy window affects the alignment process.
There are hierarchical structure restrictions if you want the panes to be adjusted as intended.

Restrictions Related to Center Position of Panes Placed in the Hierarchy

The Center of Parent Pane for Base Pane () in the property window is set to Center.
If a setting other than Center is used, the pane will move to an unintended position.

Restrictions Related to the Position of Panes Placed in the Hierarchy

Panes in the Align Pane hierarchy should be placed inside the region of the Align Pane.
If you place a pane outside the region of the Align Pane, the pane may not be adjusted correctly, or the display of the pane may become corrupted.

Restrictions Related to Translate Animation for Panes Placed in the Hierarchy

Even though you have set an animation that moves in the direction of the alignment for the pane below the Align Pane, the pane does not move because the position is overwritten by the alignment process.

Related Links For more information about the workaround for this situation, see I Can't Animate a Translate Animation for a Pane Below the Align Pane.
