SystemAudioMonitor

Overview

SystemAudioMonitor is a tool for capturing the following audio data and verifying it on a host PC.

  1. Final output from development hardware
  2. Final output from game applications built for Windows using NintendoSDK (Windows applications)

Final output refers to the result of mixing all audio output from the nn::audio library.

  • With Windows applications, this output is generated from game applications.
  • On the NX, final output includes game application output and audio played back by the library applet, DevMenu, and HomeMenu.

In any of these cases, the final output can be considered to be the audio data immediately before it is output to the device.
For example, if the application uses 5.1 ch output and the device uses 2 ch output, then SystemAudioMonitor captures the 5.1 ch audio data.

How to Use

Connect the tool to the development hardware using the Target Manager.

Start SystemAudioMonitor.exe after a connection to the development hardware has been established.

When the tool is running, click the Connect button in the upper-left part of the window to change the status from No Connection to Connection, and commence monitoring.
In this state, audio is played using the PC's default audio player, the waveforms of each channel are rendered, and the true peak and loudness values are measured.
The current true peak and loudness values are shown on the left side. The peak values are shown on the right side.
The monitoring volume can be controlled using the volume buttons under the Connect button.
To end monitoring, click the Disconnect button next to the Connect button.

If you want to save output as a waveform file, specify an Output Directory and click the Record button after monitoring has begun.
The numbers on the lower right indicate the number of channels when recording.

If you want to stop recording, click the Stop button next to the Record button.

Terminology

This section describes the meaning of the labels marked to the right of the loudness value, true peak value, and waveform rendering.

Loudness Value Terms Description
MMT Abbreviation for "momentary loudness." Indicates the loudness value per 400 milliseconds.
SHT Abbreviation for "short-term loudness." Indicates the loudness value per 3 seconds.
INT Abbreviation for "integrated loudness." Indicates the loudness over the overall monitoring segment; "loudness value" generally refers to this value.
Channel Terms Description
FL Abbreviation for "front left." Indicates the front left channel.

FR

 Abbreviation for "front right." Indicates the front right channel.
FC Abbreviation for "front center." Indicates the front channel.
LFE Abbreviation for "low frequency." Indicates the channel that emits low-pitched audio.
RL Abbreviation for "rear left." Indicates the rear left channel.
RR Abbreviation for "rear right." Indicates the rear right channel.

Tips

  • Right-clicking the area near the right end of the Connect button displays a menu, allowing you to specify a connection target and to connect.
    • Windows applications, and devices that can be connected to by Target Manager, are displayed as connection targets.
    • The number in parentheses after the .exe filename extension for Windows applications indicates the process ID for that Windows application.
      • The process ID appears only when multiple Windows applications with the same process name are running.
    • Clicking the Connect button normally connects to the default target configured in Target Manager.

  • You can open the specified output directory in explorer using the shortcut menu that appears when you right-click in the Record button area.


  • To configure user settings, click the gear button to the left of the minimize button.
    You can configure the following settings, and the configured settings are applied even if you restart the application.
    • Maintain level meter peak values
      • Sets whether to maintain the peak values for the loudness meter and true peak meter.
    • Displaying Frequency Attributes
      • Displays the frequency attributes in the waveform rendering area for each channel.
    • Record Channel Count
      • Specifies the number of channels when recording. You can set either 2 channels or 6 channels.
        • When recording is performed on 2 channels, RL, RR, FC, and LFE output waveforms are downmixed in the same way as System default downmixing.
    • Output Device
      • Selects the audio output device to use during monitoring
        • By default, output is directed to the Windows default audio playback device.

  • You can reset the peak value by clicking the SystemAudioMonitor loudness meter or true peak meter.
  • Audio for a specific channel can be monitored by clicking the M and S buttons in the waveform rendering area for each channel.
    • M is mute, and S is the solo playback function.
      • Muted and solo playback functions are only usable for playback via an audio playback device.
      • They are not usable in other audio functions, such as recording or loudness measurement.

  • The various operations of the SystemAudioMonitor have been assigned to shortcut keys.
    The following is a list of keyboard shortcuts.

Important Information

Important Information When Connecting NX

  • The system firmware processing load increases and may affect application behavior.
  • The device volume settings configured by nn::audio::SetAudioDeviceOutputVolume() are applied to acquired data.

Using the Capture Feature in Nintendo.AudioTool.dll

The capture feature used by SystemAudioMonitor can be built into your own C# tools by using Nintendo.AudioTool.dll.

The procedure for using it in a .NET application is as follows.

  1. Nintendo.AudioTool.dll is included in Tools/Audio/AudioToolLibraries. Add it with Solution Explorer > Reference > Reference (B).
  2. Implement capture functionality. For more information about implementation, see the SystemAudioCapture sample in Samples > Sources > Applications > AudioToolUtilities.
