Overview

Determines whether intermediate files are valid.

This checker is used to determine whether there are problems with intermediate files when an error has occurred.

Make sure that you use this tool to check for problems in intermediate files that you have created using your own tools.

To check an intermediate file, drag it or its folder to the executable file (3dIntermediateFileChecker.exe) or the executable's shortcut.
Files other than intermediate files are ignored.

If a problem is found, an error message appears and waits for key input.
If no problems are found, the checker exits silently.

Command-Line Options

3dIntermediateFileChecker.exe [--args-file ""] [--version] [-h] [-s] [-j ""] 
[--enable-extension ""] [--disable-extension ""] [--error-detail-output ""] [intermediate_file|folder]...

Determines whether intermediate files are valid.

--args-file ""

Specifies the file.
You can use the contents in the parameter file to specify arguments.
The parameter file allows you to specify each option over several lines.
Lines starting with a number sign (#) are treated as comments.
Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--version

Displays version information.

-h --help

Displays Help.

-s --silent

Suppresses the display of messages during processing.

-j "" --jobs ""

Specifies the number of jobs. When unspecified, the processor count is assigned.

--enable-extension ""

Specifies the extensions to be processed among text intermediate files.
Use a comma (,) as a delimiter when listing more than one filename extension.
Example: --enable-extension "fmda,ftxb"

--disable-extension ""

Specifies the extensions to be processed among text intermediate files.
Use a comma (,) as a delimiter when listing more than one filename extension.
Example: --disable-extension "fmda,ftxb"

--error-detail-output

Specifies a folder for saving detailed information when an error occurs. If not specified, this information is written to the default folder.

Intermediate File | Folder

Specifies the file or folder to be processed.
Files located under the specified folder are recursively processed when a folder is specified.
Files other than intermediate files are ignored.

Supports wildcards (?: Replaces one character; *: Replaces one or more characters).
Example: "a?b*"
Converts files with a first letter "a" and a third letter "b”.

Usage Examples

This chapter describes procedures for using the intermediate file checker.

Checking Intermediate Files

Checks files of the form check.fmdb.

3dIntermediateFileChecker.exe check.fmdb

Checking Folders

Checks all intermediate files in the check folder and its subfolders.

3dIntermediateFileChecker.exe check\

Check Items

Checks the following items.

  • Whether the XML schema is correct.
  • Whether the floating-point value in the stream is infinity or a non-numeric value.
  • Whether normals, tangents, or binormals in the model have a length of 1.
