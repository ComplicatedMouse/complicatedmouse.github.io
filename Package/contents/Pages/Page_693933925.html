Unity Setup

Using an SDEV

  1. You will be profiling your game from another process (using the  Out-of-Process profiling mode). Do not use In-Process profiling as this will result in extremely inaccurate data.
  2. Run a Release (non-Development) build of your Unity game. Do not profile Development builds, since they have code inefficiencies that make optimization difficult (the profile data is deceiving).
  3. In the profiler, after you  Sync, use the Attach scenario to attach to your running Unity Release build.


Directions for running a Release (non-Development) build:

  1. Go to  Build Settings > Nintendo Switch and uncheck the Development Build  checkbox.
  2. Run your game as normal.


In a Unity game, you will not be able to perform Full Integration Setup for a Release build. Unity has integrated Full Integration Setup, but only for Development builds. Unfortunately, Development builds provide inaccurate and deceiving profile data. Do not profile Unity Development builds.

Using an EDEV (not recommended)

It is NOT recommended to profile your game using an EDEV, since you can only profile Development builds. In this case, function percentages are inaccurate/misleading and will be missing data from high priority threads and migrating threads. If possible, obtain an SDEV to perform profiling. If you only have access to an EDEV, there is still some value in profiling. There are Checkers that automatically detect problematic Unity-specific issues, so you still might want to profile your game.


The following steps allow you to take an  In-Process profile of a Development build of your Unity game on an EDEV.

  1. In Unity, enable the Nintendo NX CPU Profiler:
    1. Set your game to a Development build ( Build Settings > Nintendo Switch: check the Development Build  checkbox).
    2. Open the Project Settings  ( Edit > Project Settings ).
    3. Select Player  on the left menu.
    4. Under Settings for Nintendo Switch  open Other Settings .
    5. Under the  Application  section, check the Use Nintendo CPU Profiler  checkbox.
  2. Run the Development build of your Unity game.
  3. In the profiler application, click  Sync and then use the In-Process Only scenario to connect to your running game.
  4. On the Sampled Profile tab, click  Start to begin profiling and  Stop to end profiling.
  5. After the profile loads, click on the Checkers tab to see Unity specific issues.
  6. Do not trust any function percentages as those will be inaccurate and misleading.

Note: An EDEV is designed to be a test dev kit and is not intended for development or for profiling your game using the Nintendo NX CPU Profiler.
