Useful EffectMaker Features

EffectMaker

This page introduces EffectMaker features that may be particularly useful for developers.

Opening Emitter Set Intermediate Files Specified at Startup

To open a particular emitter set intermediate file when EffectMaker starts, specify that file in the EffectMaker startup argument, as shown in the following example.

Workspace files (.work) and preview files (.prev) can be opened in the same manner.

EffectMaker.exe c:\xxx\xxx\xxx\EmitterSet1.eset

Opening the Specified Project Settings File on Startup

To open a particular project settings file when EffectMaker starts, specify that file in the EffectMaker startup argument, as shown in the following example.

EffectMaker.exe --team-settings-file c:\xxx\xxx\xxx\ProjectConfig.xml

You can also specify an emitter set intermediate file to be loaded on startup.

EffectMaker.exe EmitterSet1.eset EmitterSet2.eset --team-settings-file ProjectConfig.xml

The project settings file can be specified either as an environment variable or as a relative path.

EffectMaker Bridge

EffectMaker Bridge is a command-line tool that can be used to link EffectMaker to external tools.

Opening Emitter Set Intermediate Files Specified While EffectMaker Is Running

By specifying one or more emitter set intermediate files as parameters of EffectMaker Bridge, as follows, you can open those specified files while EffectMaker is running.

Workspace files (.work) and preview files (.prev) can be opened in the same manner.

EffectMakerBridge.exe load-files c:\xxx\xxx\xxx\EmitterSet1.eset c:\xxx\xxx\xxx\EmitterSet2.eset
