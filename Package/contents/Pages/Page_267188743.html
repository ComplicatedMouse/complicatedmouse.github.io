Updating With System Update
Note

This feature was added in NX Add-On 4.3.0.

System Update Tool

The System Update tool has been added for development hardware, starting with NX Add-On 4.3.0. (However, the EDEV PC tool is only available for NX Add-On 4.4.1 and later.)
When upgrading the firmware, the settings are saved, which was not the case for initializing with the initialization tool.
Only updates from NX Add-On 4.0.0 on are possible, however. Note: You cannot update development hardware with firmware that predates it.

Major Revisions

Support for the HDEV was added with the release of NX Add-On 9.3.0. The EDEV tools also support the HDEV.

Support for updating DevMenuApp by running SystemUpdateXdev.exe was added with the release of NX Add-On 12.3.0.
DevMenuApp is not updated in updates performed using earlier versions of SystemUpdateXdev.exe or DevKitUpdarXXX.nsp, so you must install it separately. For more information, see here.

Support for ADEV was added with the release of NX Add-On 13.2.0. In addition, the EDEV tools also support the ADEV with the release of NX Add-On 13.3.0.

Update Strategy

Install the DevKitVersionUpdater and SystemUpdater for NX packages to enable updates with any of the following methods.

Updating the SDEV From a PC

You can update using the SystemUpdateSdev.exe tool, included in the DevKitVersionUpdater package.

For more information, see the SDK documentation.

Updating the EDEV Series Development Hardware From a PC

You can update the EDEV, HDEV, and ADEV (collectively referred to as the EDEV series development hardware) using the SystemUpdateEdev.exe tool that is included with the DevKitVersionUpdater package.

For more information, see the SDK documentation.

Updating From NX Development Cards for SDEV and EDEV Series Development Hardware

You can update after writing the DevKitUpdarXXX.NSP file to an NX development card.

  1. Install the DevKitVersionUpdater and NX Add-On packages.
  2. Insert the development card into the development hardware.

  3. Run the following commands to write an image to the development card.

    Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- gamecard write %NINTENDO_SDK_ROOT%\Resources\Firmwares\NX\DevKitUpdaterEdevK5.nsp
  4. Insert this development card with the written image into the development hardware that you want to update.
  5. When starting, the green icon shown in the following screenshot appears. It also appears for the HOME Menu.

  6. The update screen appears on the LCD screen. Wait briefly until the following type of screen appears, which indicates the update is complete.

This method, however, cannot be used to update DEV MODE for the SDEV. Use the procedure in Updating From a PC instead.

Note

DevMenuApp is not updated in updates performed using DevKitUpdarXXX.nsp. You must install it separately. For more information, see here.

Updating From microSD Cards for SDEV and EDEV Series Development Hardware

You can also use a microSD card to update.

This method requires the NX Add-On package.

  1. Insert the microSD card into a PC and copy the %NINTENDO_SDK_ROOT%\Resources\Firmwares\NX\DevKitUpdaterXXX.NSP file to that card.
    1. The XXX part of this filename differs depending on the circumstances. These different circumstances include the following.
      1. For EDEV series development hardware (DEV MODE), it is EdevK5.
      2. For SDEV and EDEV series development hardware (PROD MODE), it is NxProd.
      3. For SDEV and EDEV series development hardware (log output PROD MODE), it is NxProdLog.
  2. Insert the copied microSD card into the development hardware you want to update.
  3. Use the Install from SD option in DevMenu to install the DevKitUpdaterXXX.NSP file you wrote to the microSD card.
  4. A green icon appears for the installed title. Select it.
  5. The update screen appears on the LCD screen. Wait briefly until the following type of screen appears, which indicates the update is complete.

This method, however, cannot be used to update DEV MODE for the SDEV. Use the updating from PC method described previously.

Note

DevMenuApp is not updated in updates performed using DevKitUpdarXXX.nsp. You must install it separately. For more information, see here.

Updating in a Remote Environment (EDEV Series Development Hardware)

When using the EDEV series development hardware in a remote environment, such as when using RemoteFX, you cannot initialize the development hardware using InitializeEdev.exe. In this case, you can use the following commands to update the firmware.

ControlTarget devmenu -- application install %NINTENDO_SDK_ROOT%\Resources\Firmwares\NX\DevKitUpdaterEdevK5.nsp
ControlTarget devmenu -- application launch 0x01000000000020d4
Note

DevMenuApp is not updated in updates performed using DevKitUpdarXXX.nsp. You must install it separately. For more information, see here.

Restrictions

Various Restrictions

Also see the important considerations for the Tool Reference.

  • You cannot update to an earlier version or the same version.
    • The tool will end successfully, but no update will be performed.
  • DevMenuApp is not updated if the update is performed using DevKitUpdaterXXX.nsp or a version of SystemUpdateXdev.exe earlier than NX Add-On 12.3.0. Update separately after running the update tool.
  • Cannot be used on development hardware installed with NX Add-On 3.X or earlier firmware.

Running the Update Tool Does Not Update DevMenuApp

As mentioned previously, DevMenuApp is not updated if the update is performed using DevKitUpdaterXXX.nsp or a version of SystemUpdateXdev.exe earlier than NX Add-On 12.3.0.
For this reason, use the following command to install DevMenuApp separately after you have run the tool.

ControlTarget devmenu -- application install %NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuApp\Release\DevMenuApp.nsp

If you have NX Add-On 6.4.0 or later and an earlier version of DevMenuApp, a "Version mismatch!" error appears when you try to run DevMenuApp. If that occurs, install the latest version of DevMenuApp.
