Graphics Subsystem Allocator

Overview

The nn::vi library also makes use graphics subsystem allocator.  As a result, nv::InitializeGraphics and nv::SetGraphicsAllocator must be called even before calling nn::vi::Initialize.  Failure to do so will result in the application aborting.

Donation Requirements

As part of the graphics subsystem allocator initialization, the application must donate a portion of memory for this exclusive use.  Please note that any recommendations in this section pertain to nn::vi only.  Additional memory will be required for the graphics libraries.

Allocation Timing

Allocations can be expected to occur at a few different points during application lifetime:

  • nn::vi::Initialize
  • nn::vi::CreateLayer
  • Acquiring textures
  • Presenting textures

If there is not enough memory available, these operations will fail.

Quick Recommendation (32K)

For development of an application, a 32K donation will be enough for nn::vi usage.  This will account for a single layer with four associated textures (quadruple buffering).
