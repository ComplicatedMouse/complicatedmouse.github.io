Viewport-Scissor State

Viewport-Scissor State

Viewport-scissor state collectively represents several states for the fixed features of viewport transformation and the scissor test in the pipeline.

Information

Use nn::gfx::ViewportScissorStateInfo to define viewport-scissor state information.

Scissor Test

To enable or disable the scissor test, use the SetScissorEnabled function.
This setting must match the scissor test parameter of the rasterizer state.

Viewport State

Set the array of viewport states using the SetViewportStateInfoArray function.
If the scissor test has been enabled, this array must be the same length as the scissor state array.

Viewport Scope

The scope of the viewport is set using the following functions.

Function Description of Settings
SetOriginX The origin of the x-coordinate.
SetOriginY The origin of the y-coordinate.
SetWidth Width
SetHeight Height
SetMinDepth The minimum depth.
SetMaxDepth The maximum depth.


Scissor State

Set the array of scissor states using the SetScissorStateInfoArray function.
If the scissor test has been enabled, this array must be the same length as the viewport state array.

Scissoring Scope

The scope of the scissoring is set using the following functions.

Function Description of Settings
SetOriginX The origin of the x-coordinate.
SetOriginY The origin of the y-coordinate.
SetWidth Width
SetHeight Height

Object

Use nn::gfx::ViewportScissorState to define the viewport-scissor state object.

Initialization and Destruction

The viewport-scissor state is initialized and finalized according to the rules for objects.

Non-Array Viewport State

The object's additionally requested memory size is always 0 when the viewport state is not an array, where the viewport state and scissors state is 1. For this reason, SetMemory does not need to be called when it is known that the viewport state is not an array.

Setting the Viewport-Scissor State

Set the viewport-scissor state by calling the SetViewportScissorState function of the nn::gfx::CommandBuffer class.

Unlike other states, viewport-scissor state is not included in the pipeline.
