Major Changes to atk

Adjusted the alignment of the waveform data included in the binary resource file.

Due to the nn::audio waveform data alignment being changed to 64 bytes, we adjusted the alignment of the waveform data included in the ATK resource file.

Resource files created in NintendoSDK 0.8.0 and earlier cannot be used with the new atk library, so reconvert them using the new converter.
The following resource files are affected.

  • bfwav
  • bfwar
  • bfgrp
  • bfsar

Separated the atk::viewer functionality out from the libnn_atk library to the libnn_atkviewer library.

Moved the Viewer feature (the nn::atk::viewer namespace's functionality) from the libnn_atk library to a separate library, the libnn_atkviewer library.
Although there are no changes to the header location or the namespace, applications using the Viewer feature will need a new link to the libnn_atkviewer library.
