Rolling Back the Controller Firmware For Debugging Purposes

  • Overview
  • Necessary Tools
  • Procedure

Overview

This page describes the rolling back of the controller firmware to a previous version, for the purpose of debugging the features required for updating the controller firmware.

With the releases of NX Add-On 3.1.0, it has become necessary, in some cases, to update the controller firmware in order to use the latest features of the IRSENSOR library.
The controller support applet for updating the controller firmware is called from the IRSENSOR library, while the application is running.
In order to determine the behavior when the controller support applet is called, you can use the procedure described on this page for rolling back the controller firmware and create a state where a firmware update is required.

For more information about firmware updates when using the IRSENSOR library, see irsensor Library.

Info

The feature for rolling back the controller firmware is supported by NX Add-On 3.2.0 and later versions.

Necessary Tools

  • Controller Firmware Updater (Debug Mode)

    • TargetTools/NX-NXFP2-a64/ControllerFirmwareUpdater/Release/UpdateControllerFirmwareForDebug.bat in the NintendoSDK folder.

    • A tool for rolling back the NX controller firmware to a previous version.

Procedure

  1. Start the controller firmware update tool.
    Run UpdateControllerFirmwareForDebug.bat located at TargetTools/NX-NXFP2-a64/ControllerFirmwareUpdater/Release in the NintendoSDK folder.

  2. After the tool has started, it is operated as described in Updating Controller Firmware.

When you run ControllerFirmwareUpdaterForDebug.bat and use it for rolling back the firmware, the controllers revert to the following firmware versions at, or prior to, NX Add-On 1.x.x.

Controller Firmware Version
Joy-Con (L) 03.48.00.00
Joy-Con (R) 03.48.03.0b
Switch Pro Controller 03.48.03.05
