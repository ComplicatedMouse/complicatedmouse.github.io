Initializing the HDEV

Turning the HDEV On

Press the POWER Button on the top of the HDEV console.

Power to the HDEV automatically turns on when connected to a PC when the power is off.

Note

The HDEV must be sufficiently charged in advance. We recommend charging for at least an hour.

If the battery runs low during initialization, the update might fail.

Connecting the HDEV to a PC

Connect the USB charging cable to the PC's USB port and the USB Type-C terminal on the bottom of the HDEV console.

Note

Do not use the relay box during initialization.

If the relay box is used during initialization, the update might fail.


The USB driver might update when the HDEV is connected to a PC.

When the HDEV is connected to a PC, wait for a while after turning on the HDEV's power.

If the USB driver is updated during initialization, the update might fail.

Initializing the HDEV

Initialize the HDEV firmware using a special tool. (This process usually takes about five minutes.)

Info

Warning messages about the Windows Firewall are displayed several times during this process. Select Allow access.


Procedure Operation
1

Note the code written on the sticker applied to the bottom of the HDEV.

(This code represents the HDEV's serial number. It is XXX00000000000 in the following example.)

Info

The serial number is used to select the HDEV in step 3.
2

On the NintendoSDK management screen, click Initialize the HDEV.


Info

To display the NintendoSDK management screen, select Dev Environments > Manage from the Nintendo Dev Interface Welcome screen.
3

Select the HDEV you want to initialize and begin initialization.


From the HDEV connection list under 1. Select HDEV actual equipment, click the address noted in step 1 for the serial number information.


Under 2. Select firmware variation, in the combo box, select the firmware version to use.

Info

We strongly recommend selecting the firmware that was initially suggested, without any changes.

Before selecting + Prod or + ProdWithLog, read NintendoSDK Documents > Guide to Setting Up the Environment > More Information > Firmware That Approximates Retail Hardware > Initialization for PROD MODE.

To start HDEV initialization, under 3. Start HDEV initialization, click Start initialization.


Info

When the HDEV console is in safe mode, a Safe Mode entry appears in the Name column.


To start initialization, select that entry and click Start initialization.


Info

The log pauses for 80 to 90 seconds during this process. Wait for the log to resume.

The process may have failed if the log does not resume after several minutes.


Info

If the HDEV console you want to initialize does not appear on the HDEV connection list even after restarting and clicking Re-Scan, something might be wrong with the HDEV. For more information, see When HDEV Initialization Fails.
4

If Success is displayed at the lower left of the log window, press the Exit button to close the tool.


Info

Fan Operating Noise

While executing step 3, the cooling fan inside the HDEV starts running.
The fan is somewhat noisy, but that does not mean the unit is malfunctioning.


Info

For more information about the setup, see NintendoSDK Documents > Guide to Setting Up the Environment > More Information > Other Information About Setting Up the Development Hardware.


Info

A sample has been built to check whether the HDEV was initialized correctly.

Run the sample as follows.

  1. On the NintendoSDK management screen, click the Run Sample button.
  2. A logo like the following is displayed if the HDEV was initialized correctly and VSI for NX was installed correctly.

Note: This sample does not exit automatically. Return to the Menu screen with the HOME Button.

Troubleshooting

If HDEV initialization fails or the console you want to initialize is not on the HDEV connection list, see When HDEV Initialization Fails and try initializing several more times.

If initialization is still unsuccessful, contact Nintendo support.

HDEV Drivers

HDEV drivers must be installed for a PC to recognize the HDEV.

In most cases, the HDEV device driver is installed automatically when you use NDI to install NintendoSDK. However, automatic installation sometimes fails depending on the environment.

Use the following procedure to confirm whether HDEV device drivers are installed.

  1. Turn the HDEV on and connect it to the PC with a USB cable.
  2. Open Device Manager on the PC.
  3. Verify that the NintendoSdkDebugger is shown in Device Manager.

If HDEV device drivers are not installed, use the following procedure to install the drivers manually.

  1. Find \DevkitDriver\dpinst.exe in the installation directory for NintendoSDK, and run it.

The procedure is now complete.
