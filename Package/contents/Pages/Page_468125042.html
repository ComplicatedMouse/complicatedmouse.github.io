Overview

Introduction

The async library enables handling of the asynchronous API functions provided by NintendoSDK via a unified interface. Each library in the NintendoSDK provides asynchronous API functions by using the async library. The goal is to allow application developers to handle asynchronous API functions in different libraries in a shared manner.

The async library is not intended to provide a means of asynchronous processing to applications. As such, the current async library does not provide features for applications to perform arbitrary asynchronous processing. Use of other libraries (such as operating system libraries) is required for performing arbitrary asynchronous processing.

How the Library Is Provided

The async library is not currently linked automatically from the build system provided by NintendoSDK. Explicitly link the following libraries from the application.

Windows Environment Development Hardware Environment
libnn_async.lib libnn_async.a

The async library public header file is Include/nn/async.h.
