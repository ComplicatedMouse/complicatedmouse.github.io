Policy Settings for System-Based Automatic Correction Processing of the Network Clock

$ RunOnTarget.exe DevMenuCommand.nsp -- time set-auto-network-clock-ensure-policy enable
*** Please reboot the target to allow new settings to take effect.
[SUCCESS]
  • When enable is specified, the system automatically adjusts the network clock.
  • When disable is specified, the system does not automatically adjust the network clock.

To update the settings after changing them, restart the development hardware.

Getting the Policy for System-Based Automatic Correction Processing of the Network Clock

$ RunOnTarget.exe DevMenuCommand.nsp -- time get-auto-network-clock-ensure-policy
enabled
[SUCCESS]

Invalidating the Network Clock

This operation puts nn::time::StandardNetworkSystemClock in the unadjusted state.

$ RunOnTarget.exe DevMenuCommand.nsp -- time invalidate-network-clock
*** Please reboot the target to allow new settings to take effect.
[SUCCESS]

To update the settings after changing them, restart the development hardware.

Resetting the Steady Clock

This command resets nn::time::StandardSteadyClock.
This operation has the following effects.

  • nn::time::StandardUserSystemClock reverts to its initial value.
  • nn::time::StandardNetworkSystemClock reverts to the unadjusted state.
  • The elapsed time cannot be calculated by two nn::time::SteadyClockTimePoint instances straddling the reset.
    Note: The nn::time::GetSpanBetween function returns failure.
$ RunOnTarget.exe DevMenuCommand.nsp -- time reset-steady-clock
*** Please reboot the target to allow new settings to take effect.
[SUCCESS]

To update the settings after changing them, restart the development hardware.

Adding an Offset to the Steady Clock

You can make nn::time::StandardSteadyClock advance faster than the actual passage of time.

The elapsed time, as calculated by nn::time::SteadyClockTimePoint instances straddling this operation, is increased by this offset value.
Note: Use the nn::time::GetSpanBetween function to calculate the elapsed time. 

$ RunOnTarget.exe DevMenuCommand.nsp -- time add-steady-clock-test-offset 60
Added 1 minutes. Total offset is 60 minutes.
*** Please reboot the target to allow new settings to take effect.
[SUCCESS]

To update the settings after changing them, restart the development hardware.

Getting the Offset Added to the Steady Clock Value

$ RunOnTarget.exe DevMenuCommand.nsp -- time get-steady-clock-test-offset
60 minutes.
[SUCCESS]

Getting and Setting User Clock Time

The following example shows how to get the time.

$ RunOnTarget.exe DevMenuCommand.nsp -- time get-user-clock
2018-09-14 01:31:00
StandardTimeName UTC
DaylightSavingTime disabled
UtcOffsetSeconds 0
[SUCCESS]

The following example shows how to set the time.

$ RunOnTarget.exe DevMenuCommand.nsp -- time set-user-clock --year 2016 --month 1 --day 1 --hour 12 --minute 23
[SUCCESS]
