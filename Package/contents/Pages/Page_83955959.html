Point 1: In Principle, Use a Compression Format

Consider using a compression format for textures, because they have many merits, including small size and less processing load on the hardware.
Only use a non-compression format if compression creates problems such as image degradation.

Point 2: Use BC3 or BC7 for RGB Color Textures

For color textures with an alpha component, we recommend using BC3 or BC7. (BC3 is the default format for LayoutEditor.)
In some situations, you may want to try the following.

  • BC7 if you are worried about alpha compression noise and want to try higher quality compression (however, note that conversion takes time).
  • BC1 if 0 or 1 bit of alpha is sufficient.
  • BC2 if alpha changes sharply or you are worried about alpha compression noise.

BC1 with or without alpha is selected automatically based on whether the image file has an alpha component.

Point 3: For Grayscale Textures, Select Based on the Number of Channels

If you use two channels (brightness and alpha), select BC5. For one channel, select BC4A|BC4L.
In LayoutEditor, the default changed based on whether the image file has an alpha component.

To generate the texture for the hardware, BC4A extracts the alpha value of the image file (or the R value if there is no alpha), and BC4L extracts the R value.

Point 4: Special Uses (for Indirect Textures)

With indirect textures, the amount of image distortion is different depending on which format you choose.
Related Links For more information about indirect textures, see here.

Point 5: Actually Try a Variety of Formats

We recommend that you actually try different formats with LayoutEditor and see how they look in Viewer.
You can change the format in Detail View mode in the Textures window.
