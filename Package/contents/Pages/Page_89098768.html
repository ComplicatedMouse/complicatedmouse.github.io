Features

This section describes the features provided by the mem library and how to use the API resources.

Features

Features Overview
StandardAllocator

The StandardAllocator class is a generic allocator class that allocates and deallocates memory space.
The class can allocate and deallocate memory much faster than the expanded heap.

NumberLineAllocator The NumberLineAllocator class is an integer allocator that can separately maintain managed memory, and allocated and deallocated memory.
Expanded Heap

The expanded heap (ExpHeap) manages the supplied memory space and provides memory allocation and deallocation features for that space.

This heap has fewer limitations than frame heaps and unit heaps and therefore is suitable to be used as a generic memory allocator.

From the perspective of allocation and deallocation speeds, we recommend using StandardAllocator instead of the expanded heap.
Frame Heap

The frame heap is a memory allocator that offers simpler features than the expanded heap.

In general terms, it can allocate memory blocks, but it can only deallocate all allocated memory blocks simultaneously.

It has good memory efficiency because no management information is contained in the memory blocks.
Unit Heap

A unit heap is a memory allocator that can only allocate and free memory blocks of a predetermined size.

Although it is not as flexible as an expanded heap, it runs faster.

Like a frame heap, it has good memory efficiency because its memory blocks have no management regions.
