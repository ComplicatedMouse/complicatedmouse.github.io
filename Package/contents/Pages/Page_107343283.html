File Format and Conversion Processes

This section describes the file formats and conversion processes in the layout.

Conversion Processes


  • In the figure, data is processed from left to right.
  • It is ultimately converted to a layout archive (ARC) file and loaded to the runtime library.
  • The classifications are as follows.
    • Deep blue squares are tools.
    • Blue squares are placements and animations.
    • Red squares are textures.
    • Pale blue squares are fonts.
    • Green squares are shader variation files.
    • Purple squares are shaders.
    • Gray squares are shader variation tables.
  • In broad terms, the designer is responsible for everything before LayoutConverter, and the developer is responsible for everything after LayoutConverter.

Placements, Animations (FLYT and FLAN Files)

  • Placements and animations are managed with FLYT and FLAN files.
  • Those files can be edited in LayoutEditor.
  • After binary conversion in LayoutConverter, FLYT and FLAN files become BFLYT and BFAN files.

Textures (FTXB, BNTX)

  • Texture (TGA) files are converted using LayoutConverter to binary texture intermediate files (FTXB files).
  • Multiple FTXB files can be combined into a single BNTX texture binary file by converting them with LayoutArchiver. When you combine them, the FTXB files are deleted.
  • To include some textures in an archive file, include FTXB files as source data for LayoutArchiver.
    • FTXB files are exported by the 3D texture converter and the Photoshop plug-in.

Shaders (BNSH Files)

  • In conjunction with the material settings in an FLYT file, LayoutConverter can export embedded shaders.
  • Among the uses of user shaders, is that you can include them in the source data for LayoutArchiver after exporting BNSH files from the GFX ShaderConverter when you want to include a shader in an archive file.
    • For more information about user shaders, see the Ui2dUserShader sample.

Shader Variation Tables (BUSHVT Files)

  • Shader variations tables define the associations between material blending types and shader variation indices.
  • Users do not need to be particularly aware of these files.

Fonts (BFFNT, OTF and TTF, BFTTF Files)

  • For fonts, you can use either a bitmap format (BFFNT files) or a scalable font format (OTF and TTF, and BFTTF files).
    • When using a bitmap format, use FontConverter to export.
      • Use input BMP files (that define character shapes) and XLOR files (that define character order) as conversion-source information.

Layout Archives (ARC Files)

  • Layout archives combine the data necessary for a layout.
  • The runtime library associates an ARC file with the ResourceAccessor class and loads it.
