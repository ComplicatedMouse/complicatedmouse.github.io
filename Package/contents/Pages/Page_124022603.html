Features Added by NX Add-On

Support for NX Controllers

The NX Add-On enables the use of the following controllers.

  • Joy-Con
  • Switch Pro Controller
    A Nintendo licensed USB controller can also be used as a Switch Pro Controller-compatible controller.

Styles of Operation Added for the NX

The API resources have been augmented to support the following styles of operation that are supported by the NX system.

  • Joy-Con dual-controller grip operations (NpadStyleJoyDual)
  • Joy-Con (R) operations (NpadStyleJoyRight)
  • Joy-Con (L) operations (NpadStyleJoyLeft)

API Resources Added Specifically for the Joy-Con

The API resources have been augmented with the following Joy-Con-specific features.

  • A feature for controlling Joy-Con assignments to Npad objects.
  • A feature for changing how the Joy-Con is held.
  • A feature for getting the color information for a Joy-Con or a Switch Pro Controller.

Controllers for NX Add-On

Using NX Add-On adds support for controllers specific to the Nintendo Switch platform, such as the Joy-Con and Switch Pro Controller. NX controllers have the following special characteristics.

Prohibited Operations for the +Control Pad and Directional Buttons

Some operations are prohibited for the +Control Pad and directional buttons when using NpadStyleFullKey or NpadStyleHandheld.
The Up and Left Buttons are prioritized when the Up/Down Button or the Left/Right Button is pressed.

Clamping the Input Range for the Control Sticks

The input values for control sticks are clamped to a circular circumference to ensure a range from -nn::hid::AnalogStickMax to nn::hid::AnalogStickMax.

The center is also clamped to a circle to ensure stable input when there are no operations.

Notes When Treating the Joy-Con Dual-Controller Grip as NpadStyleFullKey (Switch Pro Controller Operations)

The Joy-Con dual-controller grip and the Switch Pro Controller differ in the following ways.

Item Differences
Connection recognition

When the Joy-Con dual-controller grip is treated as the operation of a Pro Controller, the right and left Joy-Con controllers are both connected and recognized together as a single controller.

If one of the two Joy-Con controllers becomes disconnected because of a drained battery or for any other reason, the entire controller is viewed as disconnected. However, the HOME Button and the Capture Button are enabled.
+Control Pad

The +Control Pad is available with the Pro Controller. On the Joy-Con, however, directional buttons that split the four directions into separate buttons are used because there is no +Control Pad.

The +Control Pad and the directional buttons differ it the following ways.

  • On the +Control Pad, the up and down buttons and left and right buttons cannot be pressed simultaneously.

  • The four buttons on the +Control Pad are integrated, so when quickly switching between left and right the user may on rare occasions unintentionally generate up or down input, which is not a worry with the Joy-Con directional buttons.
Control sticks

The range of motion of the control sticks on the Joy-Con is narrower than that of the Pro Controller.

The maximum and minimum values for the control sticks are the same, but the resolution for the control stick values is different.
Accelerometer
Gyroscope

An accelerometer and gyroscope are built into the Pro Controller.
When using the Joy-Con dual-controller grip or Joy-Con attached to the Joy-Con grip, the sensor for the right Joy-Con is used.
HD rumble

The Pro Controller has a vibration motor in the left grip and in the right grip, for a total of two.

The state of the two Joy-Con controllers attached to the Joy-Con grip provides the same sensations as using the Pro Controller.

In contrast, the two Joy-Con controllers are separated left and right in the state of the Joy-Con dual-controller grip, meaning that the vibration sensations are also separated.

Joy-Con Assignment Settings to Npad (NpadJoyAssignmentMode)

Dual Assignment and Single Assignment

For each Npad object specified by an Npad ID, you can set whether the Joy-Con is assigned as "a set of two controllers" or as "separate, individual controllers."

  • Assignment as a set of two controllers (NpadJoyAssignmentMode_Dual)
    In this mode, the left and right Joy-Con controllers are connected as a set for a single Npad object.
    When the pair is connected as a set, the valid styles of operation are Joy-Con dual-controller grip operations (NpadStyleJoyDual) and Switch Pro Controller operations (NpadStyleFullKey).
    When operating in the Joy-Con dual-controller grip play format, you can get all of the input from both the Joy-Con (L) and Joy-Con (R) controllers. Even when the left and right controllers are both connected, you can get input from the two devices separately. If only one or the other is connected, you can get the input from that one controller.
    When operating in the Switch Pro Controller play format, you can start to get the input state of the Joy-Con after both controllers are connected. If only the left or right controller is connected, the behavior when operating in the Switch Pro Controller play format is as if the controller is not connected. If you want to handle the connection status of the left and right Joy-Con controllers separately, use the Joy-Con dual-controller grip play format (NpadStyleJoyDual).

  • Assignment as separate, individual controllers (NpadJoyAssignmentMode_Single)
    In this mode, each Npad is assigned either just the left or just the right Joy-Con controller. If the left and right Joy-Con controllers are both connected, Joy-Con (L) and Joy-Con (R) get separate Npad assignments.
    When the mode is set to single mode and only the left or the right Joy-Con controller is assigned, the following play formats are enabled: Joy-Con (R) play (NpadStyleJoyRight)) or Joy-Con (L) play (NpadStyleJoyLeft).

Npad Style of Operation Settings and Joy-Con Assignments

In some cases, the Joy-Con assignment mode is changed automatically when the Npad play format (NpadStyleSet) is changed.

If the only valid play formats are those that correspond to separately assigned controllers (NpadStyleJoyRight and NpadStyleJoyLeft), the Joy-Con assignment mode is automatically changed to NpadJoyAssignmentMode_Single.
If the only valid styles of operation are those that correspond to controllers assigned as a pair (NpadStyleJoyDual and NpadStyleFullKey), the Joy-Con assignment mode is automatically changed to dual mode.

If a change in assignment mode is required when the play format is changed, all connected Joy-Con controllers are temporarily disconnected.
Example: Switching From a Single Assignment Play Format to a Dual Assignment Play Format

  1. The application enables only NpadStyleJoyRight and NpadStyleJoyLeft.
  2. The system sets the Joy-Con assignment mode for all Npad objects to NpadJoyAssignmentMode_Single.
  3. The user connects a Joy-Con (R) that is assigned to NpadId::No1 as an individually assigned controller.
  4. The application switches to only having NpadStyleJoyDual enabled.
  5. The system sets the Joy-Con assignment mode for all Npad objects to NpadJoyAssignmentMode_Dual.
  6. The system automatically disconnects the Joy-Con (R) connected in step 3.

To switch from single assignment mode to dual assignment mode without disconnecting the Joy-Con, temporarily enable both the dual-mode and single-mode play formats and then switch the Joy-Con assignment mode.

Example: Switching From a Single Assignment Play Format to a Dual Assignment Play Format Without Disconnecting the Joy-Con

  1. The application enables only NpadStyleJoyRight and NpadStyleJoyLeft.
  2. The system sets the Joy-Con assignment mode for all Npad objects to NpadJoyAssignmentMode_Single.
  3. The user connects a Joy-Con (R) that is assigned to NpadId::No1 as an individually assigned controller.
  4. The application switches to having NpadStyleJoyDual, NpadStyleJoyRight, and NpadStyleJoyLeft enabled.
    At this point, the single assignment play format is still enabled, so the Joy-Con assignment mode is not changed yet.
  5. The application calls the function for changing the Joy-Con assignment mode to dual mode.
    For more information about changing the assignment mode, see Changing the Assignment Mode.
  6. The application switches to only having NpadStyleJoyDual enabled.

Changing the Assignment Mode

Use SetNpadJoyAssignmentModeSingle() to change from dual assignment to single assignment.
If the left and right Joy-Con controllers associated with the specified Npad ID are both connected already, you can use the second parameter to specify the controller type, indicating which controller to keep assigned to that Npad object.
If nothing is specified for the second parameter, then Joy-Con (L) will be assigned by default and Joy-Con (R) will be automatically reassigned to a free Npad object.
The Npad object to which the Joy-Con is newly assigned takes it as a dual mode assignment if nothing is done.
If the play format is changed to single mode for both controllers, the assignment for this other Npad object is changed to single mode.

Use SetNpadJoyAssignmentModeDual() to change from single assignment to dual assignment.
This function changes the assignment from single mode to dual mode without changing the correspondence between the Joy-Con and the Npad object.
You can use MergeSingleJoyAsDualJoy() to take left and right Joy-Con controllers connected to separate Npad IDs and merge them to a single Npad ID.
The merged Npad ID will automatically switch to dual assignment.

Setting How the Joy-Con Is Held

With the Npad interface you can configure how the Joy-Con is held (the Joy-Con grip).
Configuring the Joy-Con grip has the following effects.

  • Changes the positions at which the player LEDs light up in a way that is intuitive for the user.
    • For the horizontal grip, the LEDs light up in order from the left. For the vertical grip, the LEDs light up in order from the top.
  • Makes it possible to just use the system features for whatever grips or play formats are supported by the game application.
Note

Changing the grip does not change the input state retrieved by the GetNpadState() and GetNpadStates() functions.
Support for rotating control stick input when the controller is held horizontally must be implemented in the application.

NintendoSDK NX Add-On supports the following Joy-Con grips.

  • Solo remote grip (NpadJoyHoldType_Vertical)
    In this state, the Joy-Con controller is held vertically with the L/ZL buttons or R/ZR buttons facing up.
  • Solo horizontal grip (NpadJoyHoldType_Horizontal)
    In this state, the Joy-Con controller is held horizontally with the SL/SR buttons facing up.

If the application supports other grips, have it select a style similar to one of these styles.

Detecting Attachment Status

You can detect when the Joy-Con controllers are attached to the Switch console.
To get input from the Joy-Con controllers when they are attached to the system, enable NpadStyleHandheld using SetSupportedNpadStyleSet() and then get the input state using NpadId::Handheld.
To check whether both Joy-Con controllers are attached to the console, check the state of NpadAttributes::IsConnected in NpadHandheldState::attributes.

Getting Controller Color Information

Using the NX Add-On enables you to get the color information that differentiates controllers.

Controller color information consists of two colors: the main color and the sub-color. Each of these colors is obtained as an 8-bit RGB value.
The controller color information is stored in the Joy-Con and in the Switch Pro Controller. This controller color information is available when the application has enabled NpadStyleJoyDual , NpadStyleJoyLeft, or NpadStyleJoyRight.
