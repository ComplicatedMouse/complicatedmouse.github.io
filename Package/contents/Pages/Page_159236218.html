Lower Processing Load Child Emitter

Added a child emitter with a low processing load.
To enable or disable the option to make particles emitted from the parent emitter become child emitters, select or clear the check box (). This option is disabled by default.
The following table shows the differences in behavior when this option is enabled and disabled.

ON

While the child emitters each have an emitter, this causes computational load for each emitter.

Enable the setting if you want to use child features from before this feature was supported.
OFF
The features supported in this version can be used.
The computational load of emitters is reduced because the child emitters inherit a single shared emitter.
While it is particularly effective when many child emitters are emitted, it is subject to the following restrictions.
  • Emitter-time animations for child emitters are not supported. While it is possible to set them in the UI, they will be ignored during execution.
  • Cannot follow the movement of the parent particles after the child emitters are released.
Info

If data using child emitters from before this feature was supported is loaded, the data will be loaded in the state where Emitter Assignment for Particles is enabled.

ON Child emitters follow the particles emitted from the parent emitter.
OFF While child emitters are emitted based on the location of the parent particle, they do not follow the movement of the parent particle.

Link the Angle of View With the Scaling Restrictions

Added a feature that suppresses the effects of changing the camera angle-of-view on the particle display size when using the Camera Distance Scale Restriction feature.
You can reduce the change in size of particles on the screen when the angle of view is dynamically changing in the game.

The following figures compare when the link the angle of view with the scaling restrictions features are enabled and disabled when changing the angle of view from 60 to 100.

Angles of view: Changed from 60 (default)... ...to 100.
ON The particle size is not affected even if the angle of view is changed.
OFF The particle size is affect if the angle of view is changed.

In-Game Editing Workflow

Added a page about the new recommended in-game editing workflow for NintendoWare effects. This workflow makes it possible to create effects while checking how they look in real time directly in the game application.

In-game editing provides a way to create and edit effects while checking them in the game application.
Embed the viewer runtime in the game application to allow EffectMaker to be connected to the game application and make the following features available.
Related Links For an overview of the viewer runtime and how to use it, see Embedding the Viewer Runtime.

A Creating and Editing New Effects

When you create a new emitter set in EffectMaker, it is applied to the game application.
The size and color of effects can be adjusted in the game.
B Editing Implemented Effects

Effects embedded in a game application can be loaded in EffectMaker and edited in the game.
The effects can be adjusted and refined so that they can be correctly viewed in the game.
Info

If you need to adjust game elements like custom shaders and custom actions, we recommend adjusting them within the game.
