Terminology

This section is a glossary of the terminology used in the gfx library.

General GFX Terminology

The following terms have specific meanings in the gfx library.

Term Description
Variations The type and version of the low level graphics API.
Data

Structures guaranteed to be POD (plain old data).

Corresponds to an accessor.
Accessor The class that inherits the corresponding data.
Information

Class that contains the parameters for building an object.

Not dependent on the low-level graphics API.
Object

Class for conversing with the GPU.

Multiple exist for each variation of the low-level graphics API.
