2. Loading Samples

This page describes the procedure for loading shaders, models, and animations.
Load the files (model and skeletal animation) from the sample in the package.

This tutorial involves the following UI.

Drag

 Loads the file.


Loading Models

Load the sample model SampleTripod.fmdb from the following location.
Drag the SampleTripod.fmdb file from the folder to the 3DEditor screen.
SampleTripod.fmdb will be displayed in the File view.

Model

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod.fmdb
Info

If you set the additional reference path in Team Settings to the path for the FSDB files, the required FSDB files will be loaded automatically when the FMDB file is loaded.
The Start3dEditor.bat file that this tutorial starts has been configured to automatically load the shaders (demo.fsdb and town.fsdb).

Related Links For more information, see Team Settings File.

Loading Skeletal Animations

Load the three sample animations from the following location.
Drag the files from the folder into the 3DEditor screen.
They are displayed in the File view.

Skeletal animation (Move left)

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod_MoveLeft.fskb

Skeletal animation (move right)

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod_MoveRight.fskb

Skeletal animation (Wait)

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\SampleTripod\SampleTripod_Wait.fskb

The screen will appear similar to the following when all the files are loaded.

Verifying Shaders

Verify whether shaders are assigned to the material.

From the schematic display, double-click the material to open the Properties window.

Verify that the shader definitions for BodyMt and EyeMt are demo, as shown in the following diagram.

If the definitions are Unconfigured, click the red box and select demo.

Note

Nothing is displayed in the viewer if the shader definitions are not configured.

The batch file that this tutorial uses to start 3DEditor automatically loads the shader (demo.fsdb and town.fsdb).
