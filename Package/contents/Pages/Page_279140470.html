Creating Animations for Background Decorations

The following sample procedure creates an animation in LayoutEditor to animate texture coordinates for a background decoration.
The following image shows a layout window with completed background decoration.

Preparation

To prepare for creating the layout data, start LayoutEditor in advance.

Related Links For more information about starting LayoutEditor, see Workflow.

Opening Windows

Begin the task after opening the following windows.

  • Hierarchy
  • Group
  • Animation Segment Tags
  • Curve Editor

Related Links For more information about the UI of each window, see Windows.

Key Auto-Configuration

Use either of the following operations to disable the key auto-configuration feature.

  • In the toolbar, set Key Auto-Configuration to OFF.
  • In the menu, select Edit >Key Auto-Configuration > OFF.

Related Links For more information about each window's UI, see LayoutEditor Window.

Open Layout

From the Parts Palette, open Menu.flyt.

In the Parts Palette list, right-click Menu.flyt, and in the shortcut menu, select Open.
Menu.flyt is loaded in the LayoutEditor.

The Edit Animation Segment Tags Window Setting

This works with the Edit Animation Segment Tags window configured in merged mode.

If the mode is separated mode, perform the following operation.

  • Open the Tools menu in the Edit Animation Segment Tags window and select Switch to merged mode.

Related Links For more information about the Edit Animation Segment Tags window, see here.

Configuring Groups

The following procedure uses the Groups window to create a group of panes for which animation is set.
Use the following procedure to configure fade-in and fade-out animation and create a pane group.

1

In the Hierarchy window, select the following nodes.
BG Deco_Scroll Deco N_Deco N_Samples N_InOut
2 Right-click, and in the shortcut menu, select Create Group.
3

In the Groups window, right-click G_group_00 (), and in the shortcut menu, select Edit Selected Group.
The Edit Selected Group dialog box is displayed.
4 Change the group name to InOut and click OK to close the Edit Selected Group dialog box.
5

In the Groups window, check for InOut ().

Use the following procedure configure fade-in to wait animation and create a pane group.

1

In the Hierarchy window, select the following nodes.
BG Deco_Scroll Deco Samples
2 Right-click, and in the shortcut menu, select Create Group.
3

In the Groups window, right-click G_group_00 (), and in the shortcut menu, select Edit Selected Group.
The Edit Selected Group dialog box is displayed.
4 Change the group name to Wait and click OK to close the Edit Selected Group dialog box.
5

In the Groups window, check for Wait ().

Related Links For more information about the group feature, see here.

Configuring Animation Segment Tags

Configure segment tags in the Edit Animation Segment Tags window.
For each segment tag, set the playback range for animation and the pane in which to play animation.

Click the Add button, and then in the Add Tag window, set the tags in the following order.

1

For Segment Tag, specify the name and segment.
2

In the related groups, select Select From Existing Group, select a group, and click OK to add a segment tag.

Add the following tags.

Number Name Segment Comments Related Groups Playback mode

0

 Wait 100 - 300 Wait animation after the fade-in. Wait Loop
1 In 10 - 18 Fade-in animation. InOut One Time
2 Out 0 - 4 Fade-out animation. InOut One Time

Related Links For more information about the Edit Animation Segment Tags window, see here.

Set an animation.

Create animation according to the animation playback range set in the Edit Animation Segment Tags window.
Configure the following setting in the Edit Animation Segment Tags window.

  • In the Edit Animation Segment Tags window, select Tools > Set Selected Segment to Animation Playback Range.

Using the Animation Conflict Warning Feature

An animation conflict occurs when multiple animations are played simultaneously in the game, and different animations try to update the same target (setting).
When this conflict occurs, each animation overwrites the results of the previous animations and the resulting animation is wrong.
Use the Animation Conflict Warning feature to display a warning symbol when this data is created and a comment indicating the location to fix.

Complete the following steps to avoid animation conflicts.

1

If you use the Animation Conflict Warning feature, warning symbols are displayed for the following animation segment tags.

  • Wait
  • In
  • Out
2

Select the Wait animation segment tag, right-click Vertex Color / Transparency to open the shortcut menu, and select Display the collision animation in curve editor.
3

Delete the following key from the transparency animation shown in Curve Editor.

  • 100f = 255
4 Confirm that the warning symbol on the animation segment tag has disappeared.
