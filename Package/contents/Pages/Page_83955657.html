Semaphore Class

Feature Overview

The Semaphore class is a synchronization feature that uses a counter to control exclusive access to a common resource from multiple threads.

The Semaphore class wraps the semaphore feature using the nn::os::SemaphoreType object and contains the following member functions. For more information, see the API reference.

Name Description Remarks
Semaphore Constructor Executes initialization equivalent to InitializeSemaphore.
~Semaphore Destructor Executes destruction equivalent toFinalizeSemaphore.
Acquire Waits until getting semaphore Executes wait for getting semaphore equivalent to AcquireSemaphore.
TryAcquire Tries to get semaphore Tries to get semaphore equivalent to AcquireSemaphore.
TimedAcquire Waits for a limited time until getting semaphore Executes time limited wait to get semaphore equivalent to TimedAcquireSemaphore.
Release Return semaphore

Executes semaphore return equivalent to ReleaseSemaphore.

The following two have been provided for returning 1 count and returning multiple counts

Release

Release( int count )

GetCurrentCount Gets the semaphore count Executes get semaphore count equivalent to GetCurrentSemaphoreCount.


Examples of Using the Semaphore Class

This example implements a SemaphoreSample class using the Semaphore class.

#include <nn/os.h>

class SemaphoreSample
{
public:
    explicit SemaphoreSample(int count) : m_stockCount(    0, count),
                                          m_emptyCount(count, count)  {}

    void Consume()
    {
        m_stockCount().Acquire();
        DequeueAndConsume();
        m_emptyCount().Release();
    }

    void Produce()
    {
        m_emptyCount().Acquire();
        ProduceAndEnqueue();
        m_stockCount().Release();
    }

private:
    nn::os::Semaphore   m_stockCount;   // Stock count
    nn::os::Semaphore   m_stockCount;   // Empty count
};


The SemaphoreSample class implements various problems of producers and consumers by using semaphores.

The constructor parameter specifies limited buffer count, count. In this way two semaphores, m_stockCount and m_emptyCount, are initialized. m_stockCount shows the stock count in the buffer with the semaphore count, with an initial value of 0 and a maximum value of count. m_emptyCount shows the empty count in the buffer with the semaphore count, with an initial value and a maximum value of count.

The processing on the consumer side is with the Consume member function. The first m_stockCount.Acquire waits until the stock is one or more, and when the wait is released, data is obtained from the buffer by using DequeueAndConsume. Finally, a single buffer is emptied using m_emptyCount.Release.

The processing on the producer side is with the Produce member function. The first m_emptyCount.Acquire waits until there are one or more free buffers, and when the wait is released, data is accumulated in the buffer with ProduceAndEnqueue. Finally, the stock buffer count is increased by one by using m_stockCount.Release.

Consume and Produce operate properly regardless of the timing when they are called from multiple threads. In other words, these operate to correctly count the data items from a minimum of zero to a maximum of count, and make the consumer or the producer wait as needed.

Separate queue management and mutual exclusion are required to implement ProduceAndEnqueue to actually accumulate data and DequeueAndConsume to get the data.
