NintendoWare Graphics refers to the following modules.

  • nn::g3d
  • nn::ui2d
  • nn::font
  • nn::vfx

For more information about the major changes to NintendoWare Graphics implemented in NintendoSDK 0.8.0, see Changes to NintendoWare Graphics in 0.8.0.

Common Graphics Runtime Changes

3D ui2d/fontVFX for Changes to the Command-Line Converter Options Specifications

The specifications for the command-line converter options were changed

The converter options specified using 3D, Ui2d, Font, and Vfx, respectively, are in line with the converter options for graphic tools like ShaderConverter and TextureConverter that each command-line converter calls. The p converter option, which had been supported by the command-line tools like Ui2D, Font, and Vfx, to date, can be used in this release. It will be deprecated, however, so we are asking developers to transition to its replacement.

For more information, see the documentation for each converter.

Main Changes in ui2d/font Runtime

Optimized the Font Library and Removed Consumed Memory

Changed the internal implementation of the font library to not use vertex buffers. This reduced the amount of consumed memory.
In conjunction with this change, the following changes were also made to specifications.

  • The implementation resulted in deprecating the common functionality, CharAttribute.
  • Removed the interface and implementation for vertex buffers because they are no longer needed.
  • Removed nn::font::VertexBufferAdditionalContent, and relocated the same settings to nn::font::ConstantBufferAdditionalContent.

Although CharAttribute has been deprecated, even when the resulting increase is included, the amount of consumed memory is less than before.

The following public interfaces were removed.

  • nn::font::CharWriter::GetDispStringBufferSizeWithSharedCharAttrs()
  • nn::font::CharWriter::GetVertexBufferSize()
  • nn::font::CharWriter::InitializeDispStringBufferWithSharedCharAttrs()
  • nn::font::DispStringBuffer::InitializeArg::SetVertexBuffer()
  • nn::font::DispStringBuffer::InitializeArg::SetSharedCharAttrsEnabled()
  • nn::font::DispStringBuffer::GetRequiredVertexBufferSize()
  • nn::font::DispStringBuffer::IsUseSharedCharAttrs()
  • nn::font::DispStringBuffer::SetUseSharedCharAttrs()
  • nn::font::DispStringBuffer::CalculateVertexBufferSize()
  • nn::font::DispStringBuffer::SetVertexBuffer()
  • nn::font::DispStringBuffer::GetVertexBufferGpuAddress()
  • nn::font::DispStringBuffer::CopyVertexBufferData()
  • nn::font::DispStringBuffer::BuildVertexBufferData()
  • nn::font::DispStringBuffer::BuildAndBindVertexBuffer()
  • nn::ui2d::DrawInfo::SetFontVertexBuffe()
  • nn::ui2d::DrawInfo::GetFontVertexBuffer()
  • nn::ui2d::Layout::BuildOption::isShareTextBoxCharAttributeBuffer
  • nn::ui2d::Layout::GetSharedCharAttrs()
  • nn::ui2d::Layout::SetCharAttributeCountMax()
  • nn::ui2d::Layout::GetCharAttributeCountMax()
  • nn::ui2d::Layout::SetSharedCharAttrs()
  • nn::ui2d::TextBox::IsShareCharAttributeBuffer()

The parameters for the following public interface were changed.

  • nn::ui2d::TextBox::AllocateStringBuffer()

To support these changes, remove all eliminated interfaces and parameters.
Also, because nn::font::VertexBufferAdditionalContent was removed and its content was shifted to nn::font::ConstantBufferAdditionalContent,
call the constant buffer update process, nn::font::DispStringBuffer::BuildConstantBuffer, when updating those shifted parameters.
