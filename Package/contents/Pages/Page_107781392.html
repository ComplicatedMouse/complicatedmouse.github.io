Initializing the SDEV

Initialize the SDEV firmware using a special tool. (This process usually takes about two minutes.)

Info

Warnings about the Windows Firewall will be displayed several times during this process. Select Allow access.


Procedure Operation
1

Take note of the code written on the sticker applied to the bottom of the SDEV.

(This code represents the SDEV's serial number. It is XAW02121013686 in the following example.)

Info

The serial number is used to select the SDEV in step 3.
2

On the NintendoSDK management screen, click the Initialize the SDEV button.


Info

To display the NintendoSDK management screen, select Dev Environments > Manage from the Nintendo Dev Interface Welcome screen.
3

Select the SDEV you want to initialize and begin initialization.


From the SDEV connection list under 1. Select SDEV actual equipment, use the mouse to select the address noted in step 1 for the serial number information.


Under 2. Select firmware variation, in the combo box, select the firmware version to use.

Info

We strongly recommend selecting the firmware that was initially suggested, without any changes.

When selecting + Prod or + ProdWithLog, first read Initialization for PROD MODE, the Firmware That Approximates Retail Hardware section in NintendoSDK Documents > Guide to Setting Up the Environment > More Information.


To start SDEV initialization, in 3. Start SDEV initialization, click Start initialization.
4

If Success appears at the lower left of the log window, click the Exit button to close the tool.


Info

Fan Operating Noise

When the login window in step 4 is displayed, the cooling fan inside the SDEV starts operating.
The fan is somewhat noisy, but that does not mean the unit is malfunctioning.


Info

For more information about the setup, see NintendoSDK Documents > Guide to Setting Up the Environment > More Information > Other Information About Setting Up the Development Hardware.



Info

A sample has been built to check whether the SDEV was initialized correctly.

Run the sample as follows.

  1. On the NintendoSDK management screen, click the Run Sample button.
  2. A logo like the following will be displayed if the SDEV was initialized correctly and VSI for NX was installed correctly.

Note: This sample does not exit automatically. Return to the Menu screen with the HOME Button.

Troubleshooting

  • If the GUI application does not start in step 2, verify that the NintendoSDK\Tools\CommandLineTools\InitializeSdevWin.exe file exists.
    If this file does not exist, it might have been deleted by a malfunction in the anti-virus software. Check that the anti-virus software has not malfunctioned.
  • When using an SDEV MP from the initial lot, no serial number may be printed and Serial in the SDEV connection list confirmed in step 3 may be blank. In this case, verify the SDEV by using the following procedure.
    1. Flip the SDEV over so that the LEDs on the back are visible.
    2. Right-click any line in the SDEV connection list.
    3. Select Blink LEDs.
    4. Check whether the LEDs (GC/SD) on the back are blinking.

    If the LEDs are blinking, that terminal is the SDEV selected in the SDEV connection list.

  • If the SDEV initialization fails, Fail appears instead of Success in step 4.
    If that happens, try taking the following steps, repeating them several times. If initialization is still unsuccessful, contact Nintendo support.
    1. Exit Target Manager if it is running.
      • Target Manager might be in the task tray, so also check there.
    2. Insert and remove the Wii U console's AC adapter connected to the SDEV.
    3. Retry the procedure starting from step 2.
