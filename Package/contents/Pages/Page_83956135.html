Manual

This document describes how to use the features provided by the gfx library.

First, start with the location of the gfx library.

The gfx library is described by the Programming Overview, and the individual features are described in Features.

The namespace.

All features provided by the gfx library are contained in the nn::gfx:: namespace.

The nn::gfx::details namespace is an internal feature that is not intended for use. Do not use it, unless you have a special need.

Header

All features of the gfx library are provided by including the nn/gfx.h header file.

nn/gfx.h is located in the following directory.

$NINTENDO_SDK_ROOT/Include/nn/gfx.h

Other GFX header files are located in the following directory, and are included in nn/gfx.h.

$NINTENDO_SDK_ROOT/Include/nn/gfx/

The header file is written in C++03. Features from C++11 or later are not used.

These header files may be used in a C++03 or higher environment.

Library File

See the following pages.

  • How the Libraries Are Provided and How to Use Them

Library File Structure

The libnn_gfx library file includes shared features that are independent of the low-level graphics API, in addition to the features in the default low-level graphics API for the target environment.
To use a low-level graphics API other than the default, the dedicated libnn_gfx* file for that library must be linked in addition to the libnn_gfx file.
For example, to use OpenGL in an environment in which OpenGL is not the default low-level graphics API, the libnn_gfx and libnn_gfxGl files must be linked.

Source

The source code is included to make rebuilding and debugging possible.

All of the source code of the gfx library is included in the following location.

 $(NINTENDO_SDK_ROOT)/Sources/Libraries/gfx/

The source code, like the headers, are written using C++03.

The project files are included in the following location.

$(NINTENDO_SDK_ROOT)/Sources/Libraries/gfx/libnn_gfx/libnn_gfx.vcxproj

Open the project file in Visual Studio to rebuild the gfx library.

Use the distributed library unless there is a specific reason to conduct a rebuild.
