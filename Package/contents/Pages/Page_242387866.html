tm Library

Introduction

The tm library enables you to directly use the various Nintendo Target Manager features from tools that run on the PC.

Because it links to Target Manager rather than the NintendoSDK itself, it is installed under C:\Program Files\Nintendo\Oasis\.

The various library functions assume that Target Manager is running. Take any necessary measures, such as prompting the user to start Target Manager.

Structure

The four libraries based on the combinations of Visual Studio 2015 or Visual Studio 2017 and x86 or x64 are contained in the C:\Program Files\Nintendo\Oasis\lib directory.
Use C:\Program Files\Nintendo\Oasis\include\tm.h as the common header for all libraries.

Nintendo.Tm.dll is also provided as a .NET library.


How to Use

Use From a C++ Application

This section describes what kind of TM libraries can be used from a C++ console application.

  1. From Project Properties > General > Additional Library Directories, add C:\Program Files\Nintendo\Oasis\lib\.
  2. From Project Properties > Linker > Input > Additional Dependency Files, add libnn_tm_v120_x86.lib;.
    1. Replace the version and architecture sections according to the Visual Studio version or platform configuration that you are using.
  3. Include tm.h and implement.
  4. For more information about using each function, see the nn::tm API reference and the sample.
Info

For debug builds, the warning message "warning LNK4098: defaultlib 'MSVCRT' will conflict with the use of other libraries. Use /NODEFAULTLIB:library instead." will be displayed.
In this case, specify MSVCRT in Project Properties > Linker > Input > Ignore Specific Existing Library.

Use From a .NET Application

  1. Add C:\Program Files\Nintendo\Oasis\lib\Nintendo.Tm.dll from Solution Explorer > Reference > Reference (B).
  2. Implement the application. For information about using each function, see the references you can jump to from the function definitions in Visual Studio, Samples, and C:\Program Files\Nintendo\Oasis\lib\Nintendo.Tm.xml.

Using Target Manager 2

The tm library can also be used with Target Manager 2.
If the host PC has been configured to use Target Manager 2, the tm library communicates with Nintendo SDK Daemon rather than with Target Manager. Nintendo SDK Daemon must be running when you use the tm library.
The API specifications do not change.

For more information about using Target Manager 2, see Guide to Setting Up the Environment > Using Target Manager 2.
