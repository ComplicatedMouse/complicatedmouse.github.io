Important Information

Important information for this version.


Scripts for Rewriting the Quantization Type of Model Files

A model file (FMDB) will not load if Primitive is specified as the particle shape and the quantization type (the quantize_type attribute) of uv0 and uv1 do not match.
They can be made to match by rewriting the quantization type using the intermediate file filter.
This version of the package now includes scripts (.py and .bat) for rewriting the quantization type of model files using the intermediate file filter.

EffectMaker

This section introduces new features that have recently been added to EffectViewer.


EffectViewer Now Supports Docking

EffectViewer can now be placed inside the docking area of EffectMaker when you click Launch PC Viewer ().
The following image shows the new EffectViewer.

The window's toolbar shows features that are frequently used for debugging with the viewer.
The following table describes the features in the toolbar.


Show Ground

Shows the ground so it is easier to calculate the position of effects when you operate the camera.
You have the following options.

  • Hide ground ( ): Hides the ground.
  • Show ground using grid (): Displays a grid.
  • Show ground using blocks (): Displays translucent blocks.

Specify Size

Sets the size for individual grid points or blocks.

Lets you standardize the particle size when creating effects.

Specify the Number of Divisions

Sets the number of divisions when the ground is being shown by using a grid.


Glare Intensity

Sets the intensify of glare.


Show Emitters and Fields

  • Show Child Emitter Shapes
  • Display Field Information



Overlap Check for Particles

Use this feature to check the extent of particle overlapping.

Info

The EffectViewer debug information is displayed using a meter bar (█) to depict the processing load of an effect.
You can use it as a load indicator when using the EffectMaker features.
A red meter bar (█ ) is used to indicate the processing cost of calculations by the CPU. A green meter bar (█) is used to indicate the processing cost of rendering by the GPU.

Added Feature To Capture Thumbnails

A new feature named Capture Thumbnail () has been added as a Preview menu item and to the toolbar in the main window.
The feature is used to capture thumbnails of emitter set files and model files shown in the viewer.

When you select Capture Thumbnail (), a folder named .NwThumbs is created automatically in the folder where emitter set files are stored, and the captured images are saved there as PNG files.

The saved PNG files display as thumbnails in the Effect Browser.
The following figure shows the Effect Browser displaying the VfxTemplate emitter set that is in Favorites. Thumbnails of the individual effects are shown.

Curve Editor Snap Settings Are Now Remembered

When snap-to values for Time (horizontal axis) and Value (vertical axis) are configured for Snap width when moving keys () in the Curve Editor UI, those settings are now remembered.
