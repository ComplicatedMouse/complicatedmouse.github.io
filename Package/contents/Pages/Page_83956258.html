User Settings

You can check and edit user settings.

Related Links For more information about user settings, see here.

A Displays the description of Additional Reference Path settings.
B Displays the description of Startup Path settings.
C You can check reload settings.
D You can display and edit settings for Edit History.
E You can display and edit External Program settings.

G

Changes the visibility of slider controls on the shader page in the material property window.

Additional Reference Path Settings

Add and edit additional reference paths.

Related Links For more information about additional reference paths, see here.

A List of additional reference paths. Displays a list of information about the additional reference paths.
B When there are multiple additional reference paths, changes the order of additional reference paths by moving the selected additional reference path up and down in the list.
C Adds an additional reference path.
D Deletes the selected additional reference path from the list.
E Enter a name for the additional reference path selected from the list. This name can be accessed from the startup path.
F Enter a folder path for the additional reference path selected from the list. You can set an absolute path or a relative path from the reference source file.
G Enables/disables subfolder searches. This setting determines whether searches proceed down the folder hierarchy.
H Confirm any changes made and close the window.
I Cancel any changes made and close the window.

Editing Edit History

You can specify when to clear the edit history, and the number of edit history items to save.

Related Links For more information about the edit history, see here.

A Select when to clear the edit history.
B Specify the number of items to save in the edit history.
At present, you can set a value between 0 and 20000.

External Program Settings

You can set the name of a text editor that can be started from 3DEditor whenever you want to edit shader source code or similar text files.

Related Links For more information about external program settings, see here.

A Displays the path to the text editor that is currently set.
B Select a text editor, and set it as the text editor you want to use.
When you press this button, the file selection window opens. You can then choose any text editor.
C Specifies options used when calling up the text editor.
D Specifies whether to automatically close the log dialog box when the user command is done.
E Specifies whether to generate a debugging log for the connected application.

I/O Settings

Use this submenu to configure file input/output settings.

Related Links For more information about file I/O, see here.

A Specifies whether to automatically revise materials when a model is loaded.
B Sets whether light animation adjustment is performed automatically.
C Specifies whether to automatically reload files that are currently loaded if they have been updated when they are reopened.
D This feature is intended for use by developers. Specifies whether to output and display compiling information during shader optimization.
The information is displayed in the shader variation information on the General Page of the Material Property Window.
E Note: Do not use for features being verified.
F Note: Do not use for features being verified.
G
Allows you to change the number of recently used files displayed in the File menu in the main menu.
H Allows you to change the number of recently used projects displayed in the File menu in the main menu.

Auto Animation Bind Mode

Auto animation bind mode is a feature that automatically creates an animation set when an animation file is loaded using the name of that file, and then binds the animation set created to a model.
Related Links You can also enable or disable auto animation bind mode by using an icon on the toolbar.

Auto Animation Bind Mode Description Image
Enabled Automatically creates an animation set with the same name as the animation files and binds it to the model.

In the screenshot shown on the right, several animation files named SampleTripod_MoveLeft and SampleTripod_Wait are loaded. An animation set is automatically created for each animation filename and bound to the model.
The individual animations are displayed in a collapsed state below the animation set in the file view.
Enabled
Load animation files while holding CTRL		 Automatically creates an animation set with the same name as the animation files and binds it to the model.

If you load the animation files while holding the CTRL key, they are displayed in an expanded state below the corresponding animation set.
Disabled All of the SampleTripod_MoveLeft and SampleTripod_Wait animation files are put into the default animation set.
