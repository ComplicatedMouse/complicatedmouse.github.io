Displays

Overview

A display object is an interface to query capabilities and create layers for use on a physical display.  Note these objects may map to more than one physical display!

Default Display

The default display's functionality will vary based on the platform.  For NX, this is the only display available to applications and will always direct content to the correct display.  Applications are not responsible for switching content between possible output paths (i.e. HDMI or LCD).

Info

Applications are not required to recreate content when operation modes change unless they wish to take advantage of the potential resolution change.

 

Display Policies

An application has particular rights on displays it can open.  Applications primarily need to be concerned with the number of layers it can submit to the display process.  Additionally, application's layer is limited to certain dimensions.

Info

Each display that can be enumerated may have a different set of policies.

The DisplayInfo structure details the limitations of a display.  If the amount of content is limited, then layerCountMax will contain the maximum number of layers the process can create.  Content dimensions are restricted to layerWidthPixelCountMax by layerHeightPixelCountMax.

 The policies are expected to be fairly static throughout the platform lifecycle.
