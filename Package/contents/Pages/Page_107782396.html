Differences From Previous Development Environments

NintendoSDK has the following changes from previous development environments.

API

  • API names and features have been changed compared to the previous development environment APIs.
  • APIs are shared across platforms with future Nintendo platforms.
  • A Windows development environment is provided that can use the same APIs as the target platforms.

Compiler and Debugger

  • The compiler tool chain is shared between platforms.
  • The debugger software is shared between platforms.
  • The environment is assumed to be Visual Studio.
    • The debugger software is only available in through Visual Studio Integration (VSI).
    • Build systems like make are not provided.
    • Visual Studio is used as the compiler tool chain and debugger software in the Windows program development environment.

Releases

  • Packages are distributed through a dedicated Windows application called Nintendo Dev Interface (NDI).
    • NDI supports the installation of packages over a network
      and offline installation where the required files are downloaded in advance.
    • The SDK will not be distributed as ZIP files divided by package, as in previous SDKs.
