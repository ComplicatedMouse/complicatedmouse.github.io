Target Environment Variables (NX Add-On)

Overview

NX applications can reference the same target environment variables as the Windows platform.

Using Target Environment Variables

Warning

There are important restrictions on the use of this feature. Read about the restrictions in the NintendoSDK's Target Environment Variables section.

Preparing a Target Environment Variables Definitions File

Prepare a target environment variables definitions file according to the method described in the NintendoSDK's Target Environment Variables section.

Specifying the Target Environment Variables Definitions File and Running the Application

For NX applications, the following methods are used to specify the target environment variables definitions file when running the application.
As with Windows applications, the running application can get the values for target environment variables using functions like nn::htc::GetTargetEnvironmentVariableLength() and nn::htc::GetTargetEnvironmentVariable().

Running an Application Using VSI

In Debug > Target Environment Variable File, in the project properties, specify the path indicating the prepared target environment variable definitions files, and then run the application.

Running NX Applications Using Target Manager

When running applications using Target Manager, in the Load executable file window, in Target Environment Variable File, specify the absolute path indicating the prepared target environment variable definitions file.

Running an NX Application Using RunOnTarget

Specify the path (relative or absolute) to the prepared target environment variables definitions file for the --env option of the RunOnTarget command-line tool.

$ RunOnTarget.exe --env ./app.tdf ./app.nsp

Running an Installed NX Application

There is currently no support for using target environment variables in installed applications. This support will be provided in a future update.

Restrictions

In addition to the restrictions described in the NintendoSDK's Target Environment Variables section, there are the following restrictions.

  • This feature can only be used with Windows and NX development hardware. It cannot be used with NX retail devices.
    • In NX retail devices, all functions related to target environment variables will return errors when they are called.
  • The hardware must communicate with the host PC to get target environment variables, so there is no guarantee of how much time the process will take. For this reason, avoid making function calls to get target environment variables in scenes with real-time constraints.
