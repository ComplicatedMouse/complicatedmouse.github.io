My Page Applet

The My Page applet is a system application that brings together functionality and data specific to the user.

It includes the friend list feature, which can be used to check the friend list and add friends.

Friends Features Included in the My Page Applet

The My Page applet provides the following Friend features and the following functions for using each feature.

Feature Corresponding Function

Call the friend list screen

 nn::friends::ShowFriendList
Call the screen showing details about users n::friends::ShowUserDetailInfo
Call the send friend request process nn::friends::StartSendingFriendRequest
Call the screen listing ways to send friend requests nn::friends::ShowMethodsOfSendingFriendRequest
Call the screen to schedule friend requests during local communication nn::friends::StartFacedFriendRequest
Call the screen that lists received friend requests nn::friends::ShowReceivedFriendRequestList
Call the screen for managing the blocked-user list nn::friends::ShowBlockedUserList
Call the screen for friend invitations. nn::friends::StartFriendInvitation
Call the process for sending friend invitations. nn::friends::StartSendingFriendInvitation


Info

For more information about the Friend features, see Nintendo Switch Online Play Guide > Friends > Using From Applications. This documentation provides detailed specifications about the provided features.

For more information about friend invitations, see Nintendo Switch Online Play Guide > Friends > Friend Invitation.


Important Usage Information

Effect on Communication

Internet communications, local communications, or both may be available for use while the My Page applet is running.

Effect on Performance

The system internally uses a web browser for network service account processes (such as logging in to associate accounts). As a result, in some cases the system will use the same CPU core being used by the application.
 When this occurs, if the application continues to run background processes while the account management applet is displayed, the system may exhibit reduced performance when displaying UIs or executing other operations.

Including Header Files

Add the following code to your program.

#include <nn/friends.h>
or
#include <nn/friends/friends_Applet.h>

Starting the My Page Applet

Call the My Page applet API functions to access the supported features.
Note that functions that start the My Page applet block program control until the applet is closed. 

nn::Result result = nn::friends::ShowFriendList( uid );

API Processing Results

There are no processes that must be run as a result of the function that calls the My Page applet.
Although you can run optional processes in response to results, if necessary, do not implement additional calls to the My Page applet when there is a failure, because failures are returned when the My Page applet quits for external factors as it starts.

Criteria for the Calling Function to Return a Failure

The function that calls the My Page applet returns a failure in the following circumstances.

  • The My Page applet closed because of external factors.
  • The call to nn::friends::StartSendingFriendRequest() to send a friend request failed.
  • The call to nn::friends::StartSendingFriendInvitation() to send a friend invitation failed.
  • A user operation closed the applet after the call to nn::friends::StartSendingFriendInvitation() without the friend invitation being sent.

External Factors That Result in My Page Applet Closing

The following are examples of situations where the My Page applet might close because of external factors.

  • User operations result in a transition to the HOME Menu as the My Page applet is opening, and another library applet is opened.
  • User operations from Play Activity or Friend Invitation in the My Page applet result in the selection and execution of an application that is already running.
