Changes for Developers

This section describes the changes made to NintendoWare audio (nn::atk) in NintendoSDK 0.14.0.

Contents

  • Added a feature for simultaneous playback of multiple Opus files.
  • Added sound profile features.
  • Added a command-line option to SoundProjectConverter for specifying the output destination.
  • Updated the version for sound archives.

Added a feature for simultaneous playback of multiple Opus files.

Until now, no more than one sound could not be played at a time using Opus files.
Now multiple sounds can be played simultaneously. For more information, see the AtkStreamSound sample and the following function references.

  • nn::atk::InitializeOpusDecoder
  • nn::atk::GetRequiredOpusDecoderBufferSize

Added sound profile features.

Added the sound profile feature for getting information about each frame of audio, including information about the CPU processing time of the sound thread, the processing time of the nn::audio audio renderer, and the number of voices that are playing.
For more information, see the AtkProfile sample, Measuring Processing Time in the Programming Guide, and the following function references.

  • nn::atk::SoundProfile
  • nn::atk::ProfileReader

Added a command-line option to SoundProjectConverter for specifying the output destination.

Added the command-line options -o and --output to SoundProjectConverter for specifying the output destination.
The setting specified with this option has priority over the setting in the sound project file (FSPJ).

For more information, see SoundProjectConverter.

Updated the version for sound archives.

For the purpose of features that are under review, the version for the binary archive (BFSAR) file was updated to 0.2.3.0.
Compatibility is not broken by this latest update. It has no effect on existing features and you do not need to convert the project again.
