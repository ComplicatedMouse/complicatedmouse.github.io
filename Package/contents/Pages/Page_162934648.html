DescriptorPoolAllocator

Introduction

DescriptorPoolAllocator is an allocator for allocating the slots in a descriptor pool.

Memory space for management purposes is allocated and freed by user-specified callback functions.

DescriptorPoolAllocator

Header File

Include nn/gfx/util/gfx_DescriptorPoolAllocator.h.

How to Use

1. Prepare functions for allocating and freeing the memory used by the allocator.

The descriptor pool itself must already be initialized at this time.

#include <nn/gfx/util/gfx_DescriptorPoolAllocator.h>

void* UserAllocateFunction(size_t size, void* pParam)
{
    NN_UNUSED(pParam);
    return malloc(size);
}

void UserFreeFunction(void* ptr, void* pParam)
{
    NN_UNUSED(pParam);
    free(ptr);
}

nn::gfx::DescriptorPool g_DescriptorPool;


2. Initialize DescriptorPoolAllocator.

Specify the functions for allocating and freeing memory, the descriptor pool, the index of the starting slot, the number of slots, and the thread-safe flag.

When the thread-safe flag is enabled, the Allocate and Free functions are called in a thread-safe way.


For the starting slot, specify a value of 0 or greater.

For the number of slots, specify a value that is no larger than nn::gfx::util::DescriptorPoolAllocator::SlotCountMax.

int baseSlotIndex = 0;    // baseSlotIndex >= 0
int slotCount = 1000;    // (slotCount >= 1) && (slotCount <= nn::gfx::util::DescriptorPoolAllocator::SlotCountMax)
bool threadSafe = true;

nn::gfx::util::DescriptorPoolAllocator allocator;
allocator.Initialize(
    UserAllocateFunction,
    nullptr,
    UserFreeFunction,
    nullptr,
    &g_DescriptorPool,
    baseSlotIndex,
    slotCount,
    threadSafe
);


3. Use the Allocate function to allocate slot indexes.

You can allocate a block of as many as the count number of consecutive indexes.


If allocation fails, the function will return nn::gfx::util::DescriptorPoolAllocator::InvalidIndex.

int count = 10;
int indexSlotA = allocator.Allocate(count);
int indexSlotB = allocator.Allocate(count);
 
if(indexSlotA == nn::gfx::util::DescriptorPoolAllocator::InvalidIndex){
    // Allocation fails.
}
if(indexSlotB == nn::gfx::util::DescriptorPoolAllocator::InvalidIndex){
    // Allocation fails.
}


4. Use the Free function to free the allocated block of indexes.

allocator.Free(indexSlotA);


5. Call the Finalize function to finalize the allocator.

If any allocated blocks have not been freed at this time, they will be freed automatically inside this function.

// indexSlotB is freed automatically by the Finalize() function.
allocator.Finalize();


For more information about other functions, see the API reference for each.

Dump Format

When the Dump() function is called, the allocator's internal data is displayed in the following XML format.

 <heapinfo>
  <offset>0</offset>
  <factor>1</factor>
  <max_allocatable_size>16777214</max_allocatable_size>
  <free_memory>16777214</free_memory>
  <allocated_memory>2</allocated_memory>
  <freelists>
    <free index='255' first='0000000000000002' last='0000000001000000' size='16777214'/>
  </freelists>
  <allocs>
    <alloc level0='0' level1='0' level2='0' first='0000000000000000' last='0000000000000002' size='2'/>
  </allocs>
</heapinfo>

Descriptions of each element are the same as for nn::mem::NumberLineAllocator::Dump(). For more information, see the nn::mem::NumberLineAllocator documentation.
