3D News

This section introduces new features added to the 3D module.

  • 3DEditor
    • Added 3DBinaryConverter additional arguments to the team settings.
    • Enabled control saving of the parent material.
    • All files loaded at startup are now saved by Save All in Specified Location.
    • The UIs for setting the sampler parameters can now be locked.
    • Intermediate material files can now be duplicated.
  • Photoshop Plug-In
    • The filename for output can now be automatically adjusted when Layer Comp Export is enabled.
  • Maya Plug-In
    • In the Import plug-in, you can now specify the image file type when converting FTX files.
    • You can now set a flag to export the binary for the bone's skeletal animation data.
    • Custom attributes now keyable in the plug-in for setting animation binary output.
    • The animation layer can now be set from the shortcut menu that opens when you right-click an export item.
    • The enable or disable state of the Merge fmd File option and the path to the FMD files to merge can now be locked separately.
    • Updated the version of the FDES files exported by the Maya plug-in to 4.1.0.

Enabled control saving of the parent material.

Prevented saving parent material FMT files that are not actively open.

The next five actions are considered “active."

  • Creating a new FMT file
  • Opening an FMT file using Open File
  • Dragging an FMT file to File View
  • Reloading an FMT file using Reload File
  • Creating an FMT file via Duplicate

Whether a file can be saved is determined based on the following criteria.

  1. Actively open FMT files can be saved.
  2. Passively opened, parent material FMT files cannot be saved.
  3. Other FMT files (FMT files not being referenced by any material) can be saved.

Photoshop Plug-In

The filename for output can now be automatically adjusted when Layer Comp Export is enabled.

In the Photoshop texture export plug-in, when the Layer Comp Export option is changed to On, it now automatically adjusts to <PSD>_<LayerComp> if the Output File Name option is <Input>.
If the Slice Export option is On and the Output File Name option is <Input>, it automatically adjusts to <PSD>_<LayerComp>.<Slice>.

Maya Plug-In

In the Import plug-in, you can now specify the image file type when converting FTX files.

Added a feature to the Maya Import plug-in for specifying the file type when converting FTX files to image files.

For LDR, the choices are Auto, TGA, PNG, and DDS. For HDR, the choices are Auto, DDS, and EXR. In either case, the default is Auto (the same extension as the value of the src_path_r attribute in the FTX file). The setting from the last import time is retained in the Maya preferences.

You can now set a flag to export the binary for the bone's skeletal animation data.

The Maya plug-in for setting animation binary output now has an added feature for setting a flag to export the binary for the bone's skeletal animation data.

If Bone Animation is cleared, that node's skeletal animation data is not exported to the binary file, just as if that node did not exist.
If Bone Animation is selected and Scale, Rotate, and Translate are all set to OFF, the animation becomes a fixed animation that uses the SRT values of the FMD file. This affects animation blending.

Custom attributes now keyable in the plug-in for setting animation binary output.

In the Maya plug-in for setting animation binary output, the custom attributes added by the plug-in are now keyable.
This is an improvement to change the binary output flag depending on the animation start frame (primarily for the multi-export plug-in).

If a key with a different value is set on a custom attribute between the start and end frame, only the value in the start frame is applied in the intermediate file.
If the custom attributes NW4F_BinarizeTranslate, NW4F_BinarizeRotate, and NW4F_BinarizeScale have already been added in an existing scene, selecting the node while starting the binary output setting plug-in changes them to Keyable. (The node itself selected when the plug-in was started is also changed to Keyable.)
In Windows > General Editors > Channel Control, existing custom attributes can also be changed to Keyable.
