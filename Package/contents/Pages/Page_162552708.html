Model Data Structure

You must prepare an intermediate file that can be used to assign a Mii. This page describes the various elements of this intermediate file and how they correspond to the different parts of the Mii.

The sample data that can assign Mii characters are at this location:

$NINTENDO_SDK_ROOT/Samples/Sources/Applications/MiiG3dSimple/Resources/Model/Mii.fmdb

By loading the data into 3DEditor, you can view the relationships between shapes, materials, and bones.

  • render_info, shader_option, shader_param
    Use render_info, shader_option, and shader_param, etc., to apply the Mii parameters (in the following table) to the material.

    Mii Parameter Description
    int value for culling Use the nn::mii::DrawParam::GetCullMode() function to get this value.
    int value for transparency determination This value is determined by nn::mii::CharModel::DrawType. Parts for which the draw type is nn::mii::CharModel::DrawType_XluMin or greater are translucent.
    Constant color.

    Set the value returned by the nn::mii::DrawParam::GetConstantColor() function.

    For float3, values equal to the number defined by nn::mii::DrawParam::ConstantColorNum are required.
    Modulation type Set the value returned by the nn::mii::DrawParam::GetModulateType() function.

  • sampler
    A sampler element is attached to the material elements of the following nn::mii::CharModel::DrawType instances, which have textures. The settings are values that can be obtained using the nn::mii::GetSamplerInfo() function. The parameters under the sampler element are not dependent on the Mii, so you can put the following values in your intermediate file ahead of time if you want.

vec4 GetAlbedo()
{
    vec4 texel = texture(s_Tex,In.texCoord);
    vec4 albedo;
    switch(modulate)
    {
    case MODULATE_TYPE_CONSTANT:
        albedo = vec4(constColor[0],1.0f);
        break;
    case MODULATE_TYPE_TEXTRUE:
        albedo = texel;
        break;
    case MODULATE_TYPE_LAYERED:
        albedo = vec4(constColor[0] * texel.r
            + constColor[1] * texel.g
            + constColor[2] * texel.b
            , texel.a);
        break;
    case MODULATE_TYPE_ALPHA:
        albedo = vec4(constColor[0] * texel.r ,texel.r);
        break;
    case MODULATE_TYPE_LUMINANCE_ALPHA:
        albedo = vec4(constColor[0] * texel.r ,texel.g);
        break;
    case MODULATE_TYPE_ALPHA_OPA:
        albedo = vec4(constColor[0] * texel.r ,1.0f);
        break;
    default:
        albedo = vec4(1.0f);
        break;
    }
    return albedo;
}

void main()
{
    /// Consider the ModulateType and get the albedo.
    vec4 albedo = GetAlbedo();
    if(albedo.a == 0.0f)
    {
        discard;
    }
    o_Color = albedo;
}

  • name
    Create a name based on the naming convention of "[bone_name]__[mat_name]". Because of the logistics of linking with other tools, we do not recommend allowing users to decide on the shape name at their discretion.
  • mat_name
    The name of the material corresponding to the shape. Even if the shapes of nn::mii::CreateModelType and nn::mii::CreateNoseType are different, they will still have a common material.

  • bone_name
    The name of the bone corresponding to the shape.

  • mesh

    • count
      Set an appropriate count, because it is necessary to include in the stream for the dummy.
      In the sample, one polygon is included in the dummy so count="3" is written.

    • stream_index
      A stream for dummy vertex indices is required. The stream with stream_index=”5” is the stream for dummy vertex indices.

  • bone
    Prepare one parent bone, and then prepare other bones corresponding to the nn::mii::CreateModelType and nn::mii::CreateNoseType.

  • visibility
    Rendering Mii characters using nn::g3d uses the bone visibility to switch between nn::mii::CreateModelType and nn::mii::CreateNoseType.

    Example 1: Switching between a standard forehead and a forehead for use with hats.
    Example 2: Switching between a standard height nose and a nose with the height adjusted.

    Set the visibility of the (bone corresponding to) nn::mii::CreateModelType and nn::mii::CreateNoseType that the user wants to show in the runtime.

    Info

    For parts that do not exist for Mii characters (see the conditions in the following table), the bone visibility will be set to false when the library performs assignment.

    nn::g3d::mii::DrawParamCallbackOutput Member Value
    isMiiFace true
    pDrawParam nullptr
