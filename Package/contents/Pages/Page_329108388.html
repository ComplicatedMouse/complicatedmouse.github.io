Capture Effect (Drop Shadow)

This section describes the Capture Effect (Drop Shadow) feature, which provides an easy way to change the appearance of panes by generating various effects from the pane shapes.

Use Knockouts Configure whether the drop shadow effect where panes overlap will be knocked out.
Render Effects Only Configure whether to render only the drop shadow effect.
Capture First Frame Only Set whether to continue using the frame rendering results from the first pass.
This can reduce the processing load by applying the effect only to the initial rendering results for the pane.
Enable Outlines Add an outline to the pane. Useful when rendering shapes with hard edges.
Enable Brilliance Add an effect where it appears the pane is emitting light.
Enable Drop Shadows Add a shadow behind the pane.

Overview

The Capture Effect (Drop Shadow) feature generates and displays effects like a blurry shadow or border lines from the shape of the pane.
It allows you to easily modify the general appearance of the pane.

The settings for Opaque and Multiplicative Blend presets in the Text Box pane are not accurately applied.

Restrictions

The following restrictions apply to Capture Effect (Drop Shadow).

  • Display Restrictions
    • The display of the Layout window is not updated.
      To check the display, verify in LayoutViewer.
  • Runtime Restrictions
    • If the drawing result of the Capture Effect (Drop Shadow) feature is captured by the capture pane, or if the Capture Effect (Drop Shadow) feature is applied to a pane that uses a texture created in the capture pane, you can only get the correct effect if the actions are done in the same layout.
  • Using With a Pane-Size Animation
    • The capture effect uses the capture function, so on the included LayoutViewer, create a capture texture with the pane size of the starting frame. (The roughness is noticeable if enlarged.)
    • However, if you change the size of each frame on the game application side and then re-create the texture, it can be scaled cleanly. For more information, see here.
  • The capture texture result varies because blending is applied twice: once by the blending process of the capture effect Rendering Mode, and once by the blending process of the capture pane.
