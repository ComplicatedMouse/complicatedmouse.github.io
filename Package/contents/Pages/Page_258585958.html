Restrictions on Frequency of Function Calls

From a server load perspective, some functions have restrictions on the frequency at which they may be called.
The restricted functions are described in the following sections.

Info

For more information about the restrictions, see the guidelines.

Changing Presence Status

  • nn::friends::DeclareOpenOnlinePlaySession()
  • nn::friends::DeclareCloseOnlinePlaySession()

Communication occurs when these functions are called because the server is notified of the status change.

Updating presence

  • nn::friends::UserPresence::Commit()

Communication occurs when this function is called because the server is notified of the presence update.
The content set with the nn::friends::UserPresence::SetAppValue() or nn::friends::UserPresence::SetDescription() functions is updated when nn::friends::UserPresence::Commit() is called.

Getting Profile List

  • nn::friends::GetProfileList()

Communication occurs because getting the profile list sends an inquiry to the server.
