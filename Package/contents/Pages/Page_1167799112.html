Overview

InitializeAdev is a command-line tool for initializing the firmware of the ADEV development hardware.
(The tool can also initialize predecessor EDEV development hardware.)

Package

InitializeAdev is included in the SystemUpdater for NX package.

It is available in SystemUpdater for NX 13.0.0-3.0 and later.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/InitializeAdev.exe

In addition to this EXE file, there are also several dependency files for running the tool.

Command Definitions

Option Format

InitializeAdev [target options] [other options]

target options:
    - target <serial number> Specifies the serial number of the target to be initialized.

other options:
    --target-version                                   Specify the type of firmware to use for initialization.
    --display-available-version                        Display all available firmware types.
    --verbose                                          Generate detailed logs.
    --version                                           Display to tool version.

When specifying the serial number, initialization works with or without the check digit.

Exit Code

You can use the exit code to check whether the tool succeeded.

0: Successful completion
Nonzero value: Terminated with error

How to Use

This tool is used the same way as InitializeEdev.
For more information, see InitializeEdev.

Important Information

  • You can use InitializeAdev to initialize firmware on the following devices.
    • ADEV DP1
    • ADEV DP2
    • ADEV DP3
    • HDEV DP2
    • HDEV FinalDP
    • HDEV MP
    • EDEV MP
  • In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after initialization is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.
    • For more information, see Using Target Manager 2.
