NX

This section describes the basic ways of using the NX Add-On for developing NX applications.

Building and Running the Samples

For information about building and running the samples included in the NX Add-On package, see Quick Start.

To run the sample applications on the SDEV and EDEV development hardware, you must first prepare the hardware as described in Initializing the SDEV / Initializing the EDEV.

You must have a dedicated driver already installed to run the sample applications that use the graphics features on Windows. For the detailed procedure, see Installing the GPU Driver in nvn Library.

Creating an NX Application Project

Use NX Add-On to create a project file for the development of your NX application.
You can also configure an existing project file so it is set for an NX application.
For a detailed procedure, see Creating New Projects.

Developing an NX Application

NX Add-On allows you to develop NX applications using the NX API set.
By changing the platform that is the build target for your NX application, you can use basically the same code to debug and run the application in both the Windows environment and on the NX development hardware.
For a detailed procedure, see Application Development.

Installing and Running an NX Application on the Development Hardware

You can test the NX application you have developed without connecting to a host PC by installing and running it on the development hardware.
For a detailed procedure, see Installing Applications on Development Hardware.

Writing NX Applications to NX Development Cards, and Running Applications From NX Development Cards

NX applications can be written to and executed from NX development cards to test applications without needing to connect to a host PC.
For a detailed procedure, see Writing to NX Development Cards.
