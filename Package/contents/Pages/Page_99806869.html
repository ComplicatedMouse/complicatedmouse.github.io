Overview

The viewer library links to 3DEditor for the real-time editing of models and shader archives. The library does not manage or render models and shader archives, so the game application must perform these tasks. You can use the viewer library to edit registered models in real time, regardless of whether the models are implemented in the game application or were transferred from 3DEditor. Shader archives that are implemented in the application can also be edited in real time. A separate class does not need to be prepared to register edit targets. You can use model instances and shader archives that are registered in the same way as in a game.

Sample

The following sample uses an embedded viewer library.

%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo

The viewer library is embedded in the G3dDemo samples Edit and Town. These sample applications can be connected with 3DEditor.

For more information, see the respective sample backgrounder documents.

Embedding

To enable the viewer library features, NN_ENABLE_HTC must be added to the preprocessor definitions when building the application.

If this preprocessor definition is not added, all of the viewer library API functions will be empty implementations.

To run the viewer library inside the game application, you must add calls to the following five locations.

Initialization

Initializes the viewer library. Adds a callback to receive requests to create models and the like from the viewer library. 

nn::g3d::viewer::ViewerServer::InitializeArg arg(
    pDevice,
    AllocateWithUserData,
    FreeWithUserData);
 
// Callback to g3d::viewer can be set optionally.
arg.SetCallbacks(&callbacks);
 
// If you want to customize the texture bind, set a texture bind callback.
arg.SetTextureBindCallback(SampleViewerTextureBindCallback);

nn::g3d::viewer::ViewerResult initResult = nn::g3d::viewer::ViewerServer::Initialize(arg);
NN_ASSERT(initResult == nn::g3d::viewer::ViewerResult_Success);

The following processes must be completed before initializing the viewer library.

  • Run the nn::htcs::Initialize() function.
  • Run the nn::fs::MountHostRoot() function.

Open Server

Move to a state where connections between the server and 3DEditor are possible. To connect to 3DEditor, you must call nn::g3d::viewer::ViewerServer::Poll() after calling this function. 

nn::g3d::viewer::ViewerResult openResult = nn::g3d::viewer::ViewerServer::GetInstance().Open();
NN_ASSERT(openResult == nn::g3d::viewer::ViewerResult_Success);

Animation Calculations

To overwrite the animation being executed by the game application with the animation executed by the viewer library, you must run the following process after the game application.

In addition, to apply the animation results to the rendering, the process must be called before any of the Calculate functions of the nw::g3d::ModelObj class.

nn::g3d::viewer::ViewerServer::GetInstance().CalculateAnimations();

Communicating With 3DEditor

Communicates between the viewer library and 3DEditor. The viewer library does not internally create a thread, so use a separate thread for polling if necessary.

Commands that require communication with 3DEditor are not processed unless this function is called.

Internal states specific to communication, including connection state, change when this function is called.

nn::g3d::viewer::ViewerServer::GetInstance().Poll();

Command Execution

Execute commands received from 3DEditor or enqueued by the game application in nn::g3d::viewer::ViewerServer.

The callback registered during initialization is called by this function internally.

Even when not connected to the 3DEditor (if nn::g3d::viewer::ViewerServer::IsConnected() returns false), processes such as callback execution will be called during release processing when disconnecting, or when freeing resources that are initialized for real-time editing performed by the user in 3DEditor. For this reason, make sure to call this function.

Because the viewer library constructs and destroys resources, it must be called after enabling the OpenGL context.

The process overwrites memory referenced by the GPU. It must be executed when the GPU is not referencing memory. 

nn::g3d::viewer::ViewerServer::GetInstance().ExecuteCommands();

Closing the Server

Disconnects the connection between the server and 3DEditor.

For the editing resources possessed by the viewer library to be freed when the disconnection is made, this process must be called after enabling the OpenGL context.

nn::g3d::viewer::ViewerServer::GetInstance().Close();

To disconnect from 3DEditor immediately after closing the connection to the server, the nn::g3d::viewer::ViewerServer::GetInstance().Poll() function must also be called.

You can use the nn::g3d::viewer::ViewerServer::GetInstance().IsOpened() function to determine whether the connection to 3DEditor is still open. The following code shows how to call nn::g3d::viewer::ViewerServer::GetInstance().Poll() before disconnecting from 3DEditor.

while (nn::g3d::viewer::ViewerServer::GetInstance().IsOpened())
{
    nn::g3d::viewer::ViewerServer::GetInstance().Poll();
}

If you are handling all of your communication processes on a completely separate thread, you do not need to do this after closing the connection to the server.

Finalization

Finalizes the viewer library. The following function must be called after enabling the OpenGL context because the server will be closed within this function if it has not already been closed. nn::htcs must be finalized separately.

The connection to 3DEditor is closed several seconds after the server finalization process.

nn::g3d::viewer::ViewerServer::Finalize();

Registering and Unregistering Models as Edit Targets

Any model can be registered as an edit target, regardless of whether the model is implemented in the game application or transferred from 3DEditor.

Registering Models as Edit Targets

From the game application, you can register instances of nn::g3d::ModelObj as edit targets for 3DEditor.

nn::g3d::ModelObj* pModelObj = ...;
if (!nn::g3d::viewer::ViewerServer::GetInstance().IsModelAttached(pModelObj))
{
    nn::g3d::viewer::ViewerServer::GetInstance().QueueAttachModelCommand(pModelObj);
}
nn::g3d::viewer::ViewerServer::GetInstance().ExecuteCommands();
nn::g3d::viewer::ViewerServer::GetInstance().Poll();

Unregistering Models as Edit Targets

From the game application, you can also unregister the instances of nn::g3d::ModelObj registered as edit targets of 3DEditor.

nn::g3d::ModelObj* pModelObj = ...;
if (nn::g3d::viewer::ViewerServer::GetInstance().IsModelAttached(pModelObj))
{
    nn::g3d::viewer::ViewerServer::GetInstance().QueueDetachModelCommand(pModelObj);
}
nn::g3d::viewer::ViewerServer::GetInstance().ExecuteCommands();
nn::g3d::viewer::ViewerServer::GetInstance().Poll();

Registering and Unregistering Shader Archives as Edit Targets

A shader archive can be implemented in the game application or it can be registered as an edit target.

Registering Shader Archives as Edit Targets

You can register instances of nn::g3d::ResShaderArchive as edit targets for 3DEditor from the game application. If already connected to 3DEditor, a dialog box appears with an option to specify fsdb or fsda after this process is performed. Specify the FSD file that corresponds to the shader archive. 

nn::g3d::ResShaderArchive* pArchive = ...;
if (!nn::g3d::viewer::ViewerServer::GetInstance().IsShaderArchiveAttached(pArchive))
{
    nn::g3d::viewer::ViewerServer::GetInstance().QueueAttachShaderArchiveCommand(pArchive);
}
nn::g3d::viewer::ViewerServer::GetInstance().ExecuteCommands();
nn::g3d::viewer::ViewerServer::GetInstance().Poll();

Unregistering Shader Archives as Edit Targets

From the game application, you can also unregister the instances of nn::g3d::ModelObj registered as edit targets of 3DEditor. 

nn::g3d::ResShaderArchive* pArchive = ...;
if (nn::g3d::viewer::ViewerServer::GetInstance().IsShaderArchiveAttached(pArchive))
{
    nn::g3d::viewer::ViewerServer::GetInstance().QueueDetachShaderArchiveCommand(pArchive);
}
nn::g3d::viewer::ViewerServer::GetInstance().ExecuteCommands();
nn::g3d::viewer::ViewerServer::GetInstance().Poll();

Other Processes

Calculating Only the Specified Model Skeletal Animation

Performs calculations only for the skeletal animation bound to the specified model. If you called the nn::g3d::viewer::ViewerServer::CalculateAnimations() function after running this process, calculations for the skeletal animations bound to the specified models will be skipped. This state of being skipped will be cleared when nn::g3d::viewer::ViewerServer::ExecuteCommands() is called. 

nn::g3d::ModelObj* pModelObj = ...;
nn::g3d::viewer::ViewerServer::GetInstance().CalculateSkeletalAnimations(pModelObj);

Selecting Materials for Models Loaded in 3DEditor

Selects the specified materials of the models loaded in 3DEditor. Materials are specified by an index.

nn::g3d::ModelObj* pModelObj = ...;
int materialIndex = ...;
nn::g3d::viewer::ViewerServer::GetInstance().QueueSelectMaterialCommand (pModelObj, materialIndex);

Clearing the 3DEditor Material Selection State

Clears the selection of all currently selected materials in 3DEditor.

nn::g3d::viewer::ViewerServer::GetInstance().QueueClearMaterialSelectionCommand();

Sending Model Preview Scale and Rotation/Position Information to 3DEditor

Sends the preview scale, rotation information, and position information for a model to 3DEditor, and updates the model loaded in 3DEditor accordingly.

The preview information can be viewed on the Preview tab of the model property window in 3DEditor.

ModelObj* pModelObj = ...;
nn::util::Vector3f scale = nn::util::Vector3f(1.0f, 1.0f, 1.0f);
nn::util::Vector3f rotate = nn::util::Vector3f(0.0f, 0.0f, 0.0f);
nn::util::Vector3f translate = nn::util::Vector3f(0.0f, 0.0f, 0.0f);
LayoutModelArg arg(pModelObj, scale, rotate, translate);
nn::g3d::viewer::ViewerServer::GetInstance().QueueLayoutModelCommand(arg);

Embedding Rendering Information

As an example of how to use rendering information, insert the render priority value into the rendering information.

This assumes that the following rendering information code has already been inserted into the shader file. 

// @@ renderinfo="render_priority" type="int" label="render priority"

3DEditor will prompt the application to send definitions data if a choice entry is not included. The viewer library calls the SendRenderInfoRequested callback when it receives a request from 3DEditor to send the definitions data.

You must implement the SendRenderInfoRequested callback processes to send rendering information definitions to 3DEditor.

// Sends the definition information for render priority to 3DEditor.
virtual void SendRenderInfoRequested(const nn::g3d::viewer::SendRenderInfoRequestedArg& arg) NN_OVERRIDE
{
    // Sets a render priority from 0 through 255.
    nn::g3d::viewer::ViewerServer::GetInstance().QueueSetRenderInfoIntRangeCommand("render_priority", 0, 255);
 
    // Set 255 as the default value.
    nn::g3d::viewer::ViewerServer::GetInstance().QueuePushRenderInfoIntDefaultCommand("render_priority", 255);
}

The RenderInfoUpdated callback is called if you edit rendering information in 3DEditor.

The following denotes the processes that update the rendering information in the application. 

// Executed when rendering information is edited from 3DEditor.
virtual void RenderInfoUpdated(const nn::g3d::viewer::RenderInfoUpdatedArg& arg) NN_OVERRIDE
{
    const nn::g3d::ModelObj* pModel = arg.pModelObj;
    const nn::g3d::MaterialObj* pMaterial = pModel->GetMaterial(arg.materialIndex);
    const nn::g3d::ResMaterial* pResMaterial = pMaterial->GetResource();
    const nn::g3d::ResRenderInfo* pResRenderInfo = pResMaterial->FindRenderInfo(arg.name);
 
    if (pResRenderInfo == NULL)
    {
        return;
    }
 
    if (strcmp(arg.name, "render_priority") == 0 &&
        pResRenderInfo->GetType() == nn::g3d::ResRenderInfo::Type_Int &&
        pResRenderInfo->GetArrayLength() > 0)
    {
        // Render priority setting: To use this value, store it in the user model for the application.
        int renderPriorityValue = pResRenderInfo->GetInt()[0];
    }
}

Running Optional Processing on Certain Models From Within 3DEditor

Use the 3DEditor RuntimeUserScript to run optional processing on certain models from within 3DEditor. When the commands specified using RuntimeUserScript are run in 3DEditor, the viewer library calls the nn::g3d::viewer::CallbackType_ModelUserScriptExecuted callback. Script parsing and execution will occur within this callback because you can get the target model, the script text data, and the selected bones, shapes, and materials from the nn::g3d::viewer::ModelUserScriptExecutedArg parameter passed to this callback.

The following is an implementation of the callback that simply exports data sent from 3DEditor to the log.

virtual void ModelUserScriptExecuted(const nn::g3d::viewer::ModelUserScriptExecutedArg& arg) NN_OVERRIDE
{
    NN_LOG("ScriptText: \n%s\n", arg.scriptText);

    const nn::g3d::ResModel* pResModel = arg.pTargetModelObj->GetResource();
    NN_LOG("Target Model Name: %s\n", pResModel->GetName());

    NN_LOG("selectedBoneCount = %d\n", arg.selectedBoneCount);
    for (int i = 0; i < arg.selectedBoneCount; ++i)
    {
        int boneIndex = arg.pSelectedBoneIndices[i];
        NN_LOG("pSelectedBoneIndices[%d] = %d\n", i, boneIndex);
    }

    NN_LOG("selectedShapeCount = %d\n", arg.selectedShapeCount);
    for (int i = 0; i < arg.selectedShapeCount; ++i)
    {
        int shapeIndex = arg.pSelectedShapeIndices[i];
        NN_LOG("pSelectedShapeIndices[%d] = %d\n", i, shapeIndex);
    }

    NN_LOG("selectedMaterialCount = %d\n", arg.selectedMaterialCount);
    for (int i = 0; i < arg.selectedMaterialCount; ++i)
    {
        int materialIndex = arg.pSelectedMaterialIndices[i];
        NN_LOG("pSelectedMaterialIndices[%d] = %d\n", i, materialIndex);
    }
}


Callbacks

By adding the nn::g3d::viewer::ViewerCallback callback function or the nn::g3d::viewer::ViewerCallbacks class when the viewer library is initialized, the results of editing with 3DEditor can be applied to the model in real time.

Either callback will be called inside the nn::g3d::viewer::ViewerServer::ExecuteCommands() function when it runs.

This documentation includes only the callbacks that are necessary for the minimum implementation when embedding the viewer library. For the full list of callbacks, see nn::g3d::viewer::CallbackType.

ModelFileLoaded

Called when a model file is opened in 3DEditor. The game application must copy and accept, manage, and render resource files.

To register an edit target, call the nn::g3d::viewer::ViewerServer::QueueAttachModelCommand() function. You can also delay and register edit targets outside of the callback.

ModelFileUnloaded

Called when a model file is closed in 3DEditor. A game application can discard a resource file at the time it is no longer necessary.

TextureFileLoaded

Called when a texture file is opened in 3DEditor. Initializes the received resource file. Resources obtained by this callback function cannot be bound to application-managed resources such as models. They can be used only for resources managed by the viewer library or for attached resources. The viewer library automatically performs binding so the application does not have to do so.

TextureFileUnloaded

Called when a texture file is closed in 3DEditor. Finalizes the obtained resource files. This callback function is called several frames after the viewer library–managed resources are no longer being accessed, so even if the command buffer is double-buffered, you do not need to manually delay finalization in the game application.

ShaderAssignUpdated

Called when the editing with 3DEditor involves a change in the structure of the model resources, such as changing shader bindings or adding or deleting samplers. Because the viewer library changes model resources dynamically, use this callback for construction and destruction when the model resources require attached data.

MaterialUpdated

Called after ShaderAssignUpdated has been called and the viewer library has updated the model instance using the new model resources sent from 3DEditor. Use it when the model instance needs to be changed in conjunction with ShaderAssignUpdated.

Notes

The viewer library does not have a process for delaying the destruction of model resources and the calling of callbacks. For this reason, depending on the game framework, the GPU may reference memory regions that have already been freed. In this case, when the callback associated with freeing resources or instances is called, you must be careful not to reference memory regions that have already been freed either by waiting for rendering to complete as necessary or by skipping the render process that follows immediately.

The following callbacks are associated with the destroying resources and instances managed internally by the viewer library.

  • TextureFileUnloaded
  • MaterialUpdated
  • ShapeUpdated
  • ShaderProgramUpdated
  • ShaderAttached
  • ShaderDetached

For the model preview feature, the model that is implemented by the game application is not edited. This feature is enabled only for models transferred from 3DEditor.

We do not recommend using processing that uses rendering commands related to models, being edited in the viewer library, across different frames. The model may not display correctly when using this kind of processing.

FAQ

The resources that can be obtained by the callback function are not updated after editing the material in 3DEditor.

If nn::g3d::ResMaterial, nn::g3d::ResShape, and so on are obtained from nn::g3d::ResModel as follows, it is not possible to get the resources updated in 3DEditor.

nn::g3d::ModelObj* pModelObj = pArg->pModelObj;
const nn::g3d::ResModel* pResModel = pModelObj->GetResource();
const nn::g3d::ResMaterial* pResMaterial = pResModel->GetMaterial(materialIndex);

Get these resources directly from the object as follows.

nn::g3d::ModelObj* pModelObj = pArg->pModelObj;
nn::g3d::MaterialObj* pMaterialObj = pModelObj->GetMaterial(materialIndex);
const nn::g3d::ResMaterial* pResMaterial = pMaterialObj->GetResource();
