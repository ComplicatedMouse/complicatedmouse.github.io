Introduction

Purpose of This Documentation

This documentation describes the aoc library features included in NintendoSDK.

Intended Audience

This documentation is intended for people developing applications using NintendoSDK.

Related Documentation

Also see the aoc API Reference.

How the aoc Library Is Provided

See the following pages.

  • How the Libraries Are Provided and How to Use Them

The aoc library is currently only provided for NX.

Features Provided by the aoc Library

The aoc library provides the following features.

  • Investigate the number of downloadable content indexes for downloadable content installed in the console and for which the console owns the rights.
  • Enumerate the downloadable content indexes for downloadable content installed on the console and for which the console owns the rights.
  • (For NX Add-On 4.2 and later) Notification when available downloadable content changes while the application is running.
  • (For NX Add-On 13.3.2 and later) Policy Setting for Changes to Available Content

For more information about each feature, see the API Reference for the library. For more information about terminology such as downloadable content index, see Downloadable Content Terminology.

Information about handling the downloadable content, that is available while the application is running, is available separately at Enabling the Use of Downloadable Content Downloaded in the Background While the Application Is Running. Also refer to the preceding section.

Policy setting for changes to available content is summarized separately in Policy Setting for Changes to Available Content. Also refer to the preceding section.

The downloadable content library does not provide features to access the contents of the downloadable content. For more information about the contents of the downloadable content, use the separate fs library.

Detailed procedures for using the downloadable content library and accessing downloadable content using the fs library are described in the the downloadable content sample program.

Enumeration of Downloadable Content Indexes

Enumerated Sequence

When the system version is 7.0.0 or later, it sorts in ascending order of the downloadable content indexes.
For earlier system versions, the order is not assured.

Features for Downloadable Content Provided Outside the aoc Library

Specifying Downloadable Content Required at Startup

If there is downloadable content that you want to ensure is available to the application when it starts, you can specify that content in the NMETA file. For more information, see Writing the NMETA File.

As an example of how to use this feature, if you have a large amount of language data and have divided it up in the downloadable content, you can make sure that the application does not start unless some set of language data has been installed.

Note that, depending on how the feature is used, it might lead to situations where the user gets an application but cannot play it. If you plan to use this feature, confirm that this problem does not occur, including confirming that there are no issues with how the application is being sold. For example, sell the application together as a set with the required downloadable content, or in the case of a game card, include it in the on-card downloadable content.

Info

If you have downloadable content that is required under all conditions, consider including it in the application itself.

For example, if (1 and 2) or (1 and 3) or (1 and 4) is a startup condition, consider including the content of index 1 in the application itself.

Info

In NX Add-On 10.3.0, the error message that appears when starting an application from the HOME Menu when the required downloadable content is not present is not appropriately matched to the issue. It will be updated in NX Add-On 10.4.0 and later.

Debugging Feature for Setting Whether DLC is Available

In the development environment, all installed DLC is available to all accounts. On the other hand, in the production environment, the availability of DLC differs for each user.

Even in the development environment, you can use a debugging feature that configures whether DLC is available for each user.

For more information, see Managing DLC Availability in Development Support Features.
