Overview

The Maya plug-in is a group of tools for generating graphics intermediate files from Autodesk Maya.

Notation

This description uses the following simplified notation.

Official Name Simplified Name
Autodesk Maya Maya
Graphics Intermediate Files Intermediate File

Operating Environment

The Maya plug-in has been confirmed to run under the following environments.

Maya Maya 2018.7
Maya 2019.3.1
Maya 2020.4
OS Microsoft Windows 10 Pro (64-bit)

The Maya plug-in runs in both the English and Japanese versions of Maya.

Note that the menus and dialog boxes of this plug-in are displayed in English, even if the Japanese language environment is being used. (Some parts that use the Maya standard user interface are displayed in Japanese.)
This manual only shows English menus and attribute names used by Maya.

Info

Maya 2020 is supported by NintendoSDK 12.1.0 and later.


Starting the Maya Plug-In

You can use the Maya plug-in by starting Maya with the startup batch file in the Tools\Graphics\MayaPlugins folder.
Different batch files are run for different versions of Maya. Make sure that you use the batch file that matches the version of Maya you are using.

Maya Version Batch File
Maya 2018 StartMaya-2018.bat
Maya 2019 StartMaya-2019.bat
Maya 2020 StartMaya-2020.bat

Startup is successful if you see "Nintendo" added to the main menu after Maya starts.

If you pass the name of a scene file as an argument to the batch file for starting Maya, that scene opens after Maya starts.

StartMaya-2020.bat scene1.mb

If you specify -proj project_name after the scene filename, the specified project opens.

StartMaya-2020.bat scene1.mb -proj C:\Project1

You can specify a MEL command to execute after opening a scene by including the -command MEL command in the command line after the scene filename.
Leave a single space between the -command and the next argument.

StartMaya-2020.bat scene1.mb -command "select modelA"

Add -batch as the first argument of the Maya startup batch file to start Maya with mayabatch.exe.

StartMaya-2020.bat -batch -proj "C:/Project1" -file "Scene1.mb" -command "NintendoExportByFdes \"C:/Data/Scene1.fdes\""

When you start Maya using a startup batch file, the path to the folder containing the startup batch file is set in the NINTENDO_MAYA_ROOT environment variable.

If the first argument of the batch file used for Maya startup is --environment-only only execute the plug-in environment variable settings, and do not run Maya.
You can specify this option when using the plug-in with mayapy.exe. 

call StartMaya-2020.bat --environment-only
"%NINTENDO_MAYA_APP_ROOT%\bin\mayapy.exe" "UserScript.py"


Maya Environment Settings

Set the following preferences in Window > Settings/Preferences > Preferences.

  • Select Settings > Up axis > Y to set the y-axis as the up axis.
  • Select Settings > Angular > Degrees to set angular measurements in degrees.
  • In Settings > Animation, we recommend setting the Weighted Tangents option to OFF.
    When you create animations with the Weighted Tangents option set to ON, animation curve data is always automatically created and exported from data baked with the values in each frame. As a result, the playback results for a small number of frames may be different than in Maya.
Note

In Maya 2018, we recommended setting the MAYA_NO_TBB environment variable to 1 when using Maya.
The MAYA_DET_TANGENT environment variable was set to 1 in StartMaya-2018.bat to handle the issue where tangent and binormal values changed for each export, but this setting might also cause Maya to crash during export or when performing UV operations.
You can avoid the crash by setting the MAYA_NO_TBB environment variable to 1.


Restrictions on Usable Characters

This section describes the characters that can be used in the names of nodes and materials.

Prohibited Characters

Only single-byte uppercase and lowercase alphabetic characters, numeric characters, and underscores (_) can be used in node names and material names.

Use of any other character or symbol is prohibited. If a prohibited character is used, it is converted to an underscore (_) at time of output.
Node names and other objects sometimes include a colon (:) when scene loading or references are used. Note that the colon is a prohibited character and is converted to an underscore in the intermediate files.

Info

Maya handles Test:Cube1 and Test_Cube1 as different names, but NintendoWare handles them as the same name and changes the name of the node in the intermediate files.

Info

When the Remove Namespace Export plug-in option is set to ON, names with the namespace portion deleted can be exported to the intermediate files.


Uppercase and Lowercase Characters

Although either uppercase or lowercase characters can be entered for notation used in Windows, they are overwritten because ABC.fmdb and abc.fmdb are recognized as the same file during internal processing.

Maya, however, treats the names ABC and abc as different names. In other words, such names can co-exist.
Because of this, names might collide during data clipping.

When you export data from Maya to the external operating system as files, make sure that you use names that do not conflict even when uppercase characters are treated as lowercase characters.
