No sound is voiced even though StartSound succeeded.

If no sound is voiced even though nn::atk::SoundArchivePlayer::StartSound succeeded, the following may be occurring.

  • The nn::atk::SoundArchivePlayer::Update function is not called.

The process for detaching memory pools never completes.

If a looped sound is left playing, attempts to detach the memory pool containing the memory region where that sound is loaded will fail.
First, check that you are not attempting to detach the memory pool without first stopping the sound.

For more information about resources that require memory pool management, see the following documentation.

  • AUDIO library memory pool documentation
  • Programming guide memory management documentation

SoundMaker

For more information about troubleshooting problems in SoundMaker, see Troubleshooting in the SoundMaker documentation.

PC applications in which the nn::atk::viewer library is embedded cannot be connected to SoundMaker.

  • There are different SoundMaker connection settings for applications in which the generic version of the viewer library is embedded, and applications in which the NX version of the viewer library is embedded.
    For applications in which the NX version of the viewer library is embedded, make sure that the Connection Target in the toolbar is set to PC application and Platform is set to NX.
