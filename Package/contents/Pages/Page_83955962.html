Linear Space in the Layout

Linear workflows are supported in layouts.

What Are the Advantages of Layouts?

For 3D rendering there are many benefits, while for layouts there are few readily apparent advantages.
The biggest advantages of aligning 3D and gamma are as follows.

  • You can avoid unnecessary additional processes for the ultimate combining of data.
  • Aligning colors in the layout with 3D models is easier.

The minimization of this sort of excess effort can be called its greatest advantage.

What Is the Effect of Switching the Method During the Middle of a Project?

This changes the way gradation is applied and how animations change, so you must adjust the layout data.
The color settings and animation settings must be fixed for almost all layouts.

  • Textures
  • Vertex color
  • Color interpolation
  • Various color animations
  • Panes configured for transparency


Calibrating Color Spaces at the Asset Level

If it has been set as a linear space, then the design assets in the layout will be subjected to gamma correction.
The following sections describe the method of gamma correction that we recommend for each type of design asset.

Textures

The designer creates textures in the normal way for use with LayoutEditor.
The binary converter performs the preparations for conversion to linear color space.
The developer specifies the options for the converter.
When options are specified, the converter exports textures that use hardware features (sRGB fetch) that reference the textures while performing linear conversion.
(Because of hardware limitations, only the RGBA8, BC1, BC2, and BC3 formats can use the sRGB fetch feature. Correction occurs within the converter.)
The converter does not perform gamma correction if the texture is for indirect texture use.
The determination of whether a texture is for indirect texture use is done automatically, so the designer does not need to worry about it.

For these operations, the designer does not need to configure gamma settings for each individual texture.
Textures have only limited use in layouts, so the converter can make the determinations automatically.

Vertex Color and Color Correction

There are two methods.

  • The designer creates the data in the usual way, and then it is all converted to linear gamma at the time of binary conversion.
  • The data is created as linear gamma data from the start.

If the data is created in the usual way, the linear-conversion option is used with the converter. (The developer specifies the option for the converter.)
The result does not strictly match for animation (because the slope is not converted).

To create the data as linear data, specify linear color space mode in the Tool Environment Settings (Menu > Edit > Tool Environment Settings).
The display in the layout window and in the viewer changes to linear color space.
If you create resources based on colors viewable in LayoutEditor and the Viewer, you can make pictures in a linear environment.

To batch-convert these resources when converting binary data, clear the Specify color parameters in linear color space check box. This option is specified to the converter when previewing data to convert the data to the linear space.

Fonts

The different formats are handled differently.

  • 1-channel format: No action The image is handled as an alpha value, so gamma has no effect.
  • 2-channel format: Apply gamma correction as needed on the image file being input to the font converter.
  • Full color: Creates the font using a linear color space format (_SRGB is appended to the format starting in version 1.10.0).
    Gamma correction is usually applied to image files input to FontConverter, but we recommend using a linear color space format to avoid the loss in precision arising from the gamma correction calculations.

Checking on the Viewer

In the Tool Environment Settings, you can change between linear mode and sRGB mode (operations as normal).
The binary conversion setting for previewing also changes in tandem.
