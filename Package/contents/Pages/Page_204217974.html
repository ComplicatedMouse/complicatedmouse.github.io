Test Delivery

A mechanism called Test Delivery is provided for distributing data experimentally.

The Test Delivery mechanism allows the following features.

  • Distributing data that ignores the actual delivery start date and time configured for each file
  • Distributing data with a customized combination to a particular Switch console

Test Delivery is used in the following cases.

  • When performing final testing in the production environment prior to distribution to end users
    • For example, to test immediately before releasing an application or starting an online event
  • When debugging an application with multiple distributions in parallel in the development environment
    • For example, to debug a summer online event and winter online event in parallel

Receive Settings for Test Delivery

The following operations are required for receiving test deliveries.

For more information, see the BCAT Management Tool Manual.

Specifying Delivery Target Systems

Use the BCAT Management Tool to specify the serial number of the target console and the Nintendo Account associated with the target console.

Switching the System Connection Target

Configure the system appropriately to allow it to connect to the test delivery server.

Test Delivery Priority

When you configure a console to receive test delivery, it no longer receives standard delivery.

When data received from prior standard delivery exists in the console, it is overwritten by the contents of test delivery.
