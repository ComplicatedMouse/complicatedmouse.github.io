Polygons

Polygon Shapes

When polygons are output to the intermediate file, the Export plug-in operates internally to divide the polygons into triangles for output.

Vertex

The number of vertices that can be exported to an intermediate file is greater than the number or vertices that can be used with the DCC tool. Although this differs depending on the mesh structure, it can be as much as three times the maximum number of triangular polygons.

Normal

Normal vectors can be exported.

If the shading of models that are exported to an intermediate file differs greatly from the shading in 3ds Max, data may be created incorrectly.
If shading is not displayed correctly, perform the following procedure.

  • On the Utilities panel, select Reset XForm.
  • Use the STL-Check modifier to determine whether the geometry is correct.
  • Use the Vertex Weld modifier to clean up the mesh if there are any inappropriate, unconnected vertices.
  • On the Edit Polygons panel, use Remove Isolated Vertices to delete vertices that do not belong to any polygons.

Vertex Color

To export multiple vertex colors, prepare color sets like the ones shown below, with the names nw4f_colorN (where N is an integer between 0 and 7).

nw4f_color0 → exports to color0
nw4f_color1 → exports to color1
nw4f_color2 → exports to color2
　......

nw4f_color7 → exports to color7

(Up to 8 colors can be exported for each vertex.)

If the color set named nw4f_color0 does not exist, the values calculated based on the standard vertex color, vertex alpha, and vertex illumination are exported for color0.

Conversion from a 3ds Max vertex color to an intermediate file vertex color is carried out as follows.

  • Intermediate File Vertex Color (RGB) = 3ds Max Vertex Color * 3ds Max Vertex Illumination
  • Intermediate File Vertex Color (A) = 3ds Max Vertex Alpha ((R + G + B ) / 3)

The type of the vertex color value is defined by the ending of the color set name: If the name ends in "_i," then the type is an integer. The type is an unsigned integer if the name ends in "_u," and it is a real number if the name ends in "_f." If nothing has been specified, the type is a real number. So, for example, if you want to export color1 as an integer, name the color set as nw4f_color1_i. When the values are specified as integers and unsigned integers, 3ds Max rounds the real number to the closest integer for export.

You can specify the number of vertex color components by inserting a numeral, 1, 2, 3, or 4, before the letter that designates the value type at the end of the color set name.
Specify 1 to export the R component, 2 to export the RG components, and 3 to export RBG. You can export four components (RGB and A) by specifying 4, but the value for A will always be 1. So, for example, if you want to export color3 as two components that are real number values, then name the color set as nw4f_color3_2f.

If the number of components is not specified, four components (RGBA) are exported for standard vertex color if vertex alpha is being used, but only three components (RGB) are exported otherwise.
Three components (R, G and B) will be exported for other vertex colors with names of the form nw4f_colorN.

Instances

Instances are exported as separate objects in an individualized state.

However, the plug-in does not support instances of models for which skin has been configured.
