Host PC

Feature Overview

The fs library includes features for mounting the host PC’s file system.

Warning
  • These features cannot be used in the production environment.
    • For more information, see Using Debugging APIs in fs Library > Development Overview.

Sample Programs

See the File System Operations on a Host PC section of the Sample Backgrounder.

Mounting the Host PC’s File System

You can access data on the host PC by mounting the host PC’s file system using the nn::fs::MountHost() and nn::fs::MountHostRoot() functions.

Journaling is not supported. nn::fs::Commit() does not need to be executed.

Due to the way the fs library is designed, the behavior of mounted file systems may be slightly different from the original host PC file systems. For more information, see fs Library > Development Overview > fs Library Restrictions.

Path Restrictions

Prohibited characters and path length restrictions depend on the host PC.

Specifying Relative Paths

You can specify a relative path for the parameter of the nn::fs::MountHost() function that specifies the root path of the host PC’s file system.
In this case, the base path for the relative path takes one of the following values according to the development environment.

Note that relative paths that point to a higher level than the base path (such as “../” and “./dir/../../”) cannot be specified.

Windows Environment

  • Normal execution
    • Current working directory
  • Execution from VSI
    • The working directory specified in Project Properties (Project Properties > Debugger > Working Directory).

Development Hardware Environment

  • Normal execution (using RunOnTarget, Target Manager, DevMenu, etc.)
    • The working directory specified in Target Properties in Target Manager (right-click a target in the target list and then select Target Properties > Working Directory).
    • If a working directory is not specified, it defaults to the same directory as the running instance of TargetManager.exe.
  • Execution from VSI
    • The working directory specified in Project Properties (Project Properties > Debugger > Working Directory).
