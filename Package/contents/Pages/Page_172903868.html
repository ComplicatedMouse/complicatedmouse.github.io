Using float Color Output

Introduction

This document describes how to use float colors in layout data.

Usually, colors are specified in LayoutEditor by using values between 0 and 255, but enabling float color output allows you to set the color as a float value.
float colors support the range from 0.0 to 1.0 for values previously set from 0 to 255. You can also set values greater than 1.0.

Enabling Features

This feature is disabled by default. The following settings are required to use the feature.

Enabling float Colors in LayoutEditor

Set the following items in the project settings file to enable float color output in LayoutEditor.

<FloatColorWriteEnabled>true</FloatColorWriteEnabled>

For more information about the project settings file, seeProject File Format.

Glare Effect Using the LayoutEditor Preview

A feature for applying a glare effect is included when previewing using the LayoutEditor viewer (LayoutViewer).

Enabling a float color does not necessarily display a glare even though the glare effect is implemented in LayoutViewer using that float color functionality.

You must implement things like effects that use data from within the application to use data exported using the float color.

Considerations for Handling float Colors in Applications

To handle float color data in application code, you must select a format that can appropriately handle color values greater than 1.0, such as setting the color buffer format to ImageFormat_R16_G16_B16_A16_Float.

Considerations for Using This Feature

Take care with intermediate files that are saved when the feature is enabled by changing the expression format of the internal data.
When data using float colors is output in a state where the feature is enabled, the data that was previously stored internally as values from 0 to 255 will be saved as floating-point values from 0.0 to 1.0 (or greater).
While data with the same meaning is output so that there are no calculation errors during data conversion, the saved tags will be different from the existing tags. As a result, the files cannot be opened with versions of LayoutEditor that do not support float colors.
