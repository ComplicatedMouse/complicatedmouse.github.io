nn::g3d News ()

Notice of Making the --glsl-version 3D Shader Converter Option a Required Option

When converting from shader definition files (FSDB files) to shader archive files (BFSHA files) in the future planned release of NintendoSDK 7.0.0, the --glsl-version 3D shader converter option will become a required option.
Currently a warning message is displayed when --glsl-version is not specified, but this message will become an error after migrating to NintendoSDK 7.0.0.

When the version is defined in the shader source code, specify the version using the --glsl-version option.

Added the --merge-shader-archive-file Option for Merging Multiple Shader Archive Files (BFSHA Files) in the 3D Shader Converter

A feature to merge multiple shader archive files (BFSHA files) into a single shader archive file has been added.
You can use the --merge-shader-archive-file option to specify multiple shader archive files to merge. 

3dShaderConverter.exe --merge-shader-archive-file="A.bfsha" --merge-shader-archive-file="B.bfsha" --output="AB.bfsha"

Only shader archive files that satisfy the following requirements can be merged.

  • The version of the shader archive file matches the newest binary version supported by the shader converter that is being used.
  • All shader archive files being merged were created from the same shader definition file.

Added 3D Texture Preset Converter features.

The following features were added to the 3D Texture Preset Converter.

  • A preset that inherited other presets can now be defined.
<preset name="Albedo1" format="unorm_bc1" ... />

<!-- Option addition -->
<preset name="Albedo2" base_name="Albedo1"
    replace_r="%MATCH_1%.R.%INPUT_FILE_EXT%"
    replace_g="%MATCH_1%.G.%INPUT_FILE_EXT%"
    replace_b="%MATCH_1%.B.%INPUT_FILE_EXT%"
/>

<!-- Option overwriting and addition -->
<preset name="Albedo3" base_name="Albedo2" format="unorm_bc3"
    replace_a="%MATCH_1%.A.%INPUT_FILE_EXT%"
/>
  • A different output format is now possible when there is no replacement file specified with replace_r/g/b/a.
<!-- Where there is no a, use alternative_format_rgb. When there is no b and a, use alternative_format_rg. When there is only r, use alternative_format_r -->
<preset name="Packed1" format="unorm_bc7" comp_sel="rgba"
    alternative_format_rgb="unorm_bc7,rgb1"
    alternative_format_rg="unorm_bc5,rg01"
    alternative_format_r="unorm_bc4,r001"
    replace_r="%MATCH_1%.R.%INPUT_FILE_EXT%"
    replace_g="%MATCH_1%.G.%INPUT_FILE_EXT%"
    replace_b="%MATCH_1%.B.%INPUT_FILE_EXT%"
    replace_a="%MATCH_1%.A.%INPUT_FILE_EXT%"
/>
  • If you specify channel_packing="true", you only need to convert once even if multiple files are converted at the same time.
<preset_assign file_name="(.*)\.(R|G|B|A)" preset_name="Packed1" channel_packing="true" />
  • Added %INPUT_FILE_PATH% to the available environment variables.

The following feature was added to the 3D texture converter.

  • Added the --disable-merge-error option to prevent an error when a file specified with the --merge option does not exist.

nn::atk News ()

Added a feature to specify the presence of a loop during playback in the runtime.

A feature has been added to specify the presence of a loop during playback in the runtime.
For more information, see nn::atk::SoundStartable::StartInfo and nn::atk::SoundStartable::StartInfo::LoopInfo.

Added a feature to get marker information included in waveforms.

A feature has been added to get marker information included in source waveforms from stream sound binary data.
Only marker information written using specified naming conventions can now be included in the binary data.

For more information about naming conventions, see the stream sound documentation.
For more information about loading, see the AtkStreamSound sample.

As a result of this change, if you convert without making any changes, small differences appear in the BFSAR, BFSTP, and BFSTM binary data.
If you do not use this feature, there is no need to change the binary data.
