Plug-In for Setting Animation Binary Output

This plug-in sets the flags to output the animation binary for transform nodes and joint nodes.

You can also search for the settings.

It can be used for the following purposes.

  • To clear the flag to output the binary for skeletal animation data for nodes for which you do not want to play skeletal animations.
  • To clear the flag to output skeletal animation translation binary to make common use of a rotation animation for characters with different physiques.
  • To clear the flag to output the binary for visibility animation data for nodes for which you do not want to play visibility animations.

How to Use

Select Nintendo > Set Animation Binarization to open the window.

Selected transform nodes and joint nodes are displayed under Selected Transform Node.
Click a node name to select that node.

Setting Method

When one or more transform nodes or joint nodes are selected, the current setting for the selected node is displayed under Set. If you select multiple nodes, the setting for the first selected node is displayed.

Change the binary output flags by selecting or clearing the check boxes.
You can select multiple nodes to change them all at the same time. (Only the changed binary output flags are applied.)

You can configure the setting for a different node by keeping the window open and selecting a different node.

Binarizing Skeletal Animation

Bone Animation sets the flag to output the binary for the bone's skeletal animation data. (This feature is available in Nintendo SDK 10.0.0 and later.) If Bone Animation is cleared, that node's skeletal animation data is not output to the binary file, just as if that node did not exist (and does not have any influence on animation blending).

Scale, Rotate, and Translate set the individual flags to output the binary for the skeletal animation's scaling, rotation, and translation. If cleared, the translation, rotation, and scaling values of the FMD file are used.

By default, all of the flags are selected.

If Bone Animation is selected and Scale, Rotate, and Translate are all set to OFF, the animation becomes a fixed animation that uses the SRT values of the FMD file. This affects animation blending.

Binarize Visibility Animation

Visibility sets the flag to export the binary for the visibility animation data for the bone. (This feature is available in NintendoSDK 10.0.0 and later.) If Visibility is cleared, that node's skeletal animation data is not exported to the binary file, just as if that node did not exist. This option is enabled by default.

When export_bone_visibility_all="false" has been specified in the config file, this binary is exported if the new flag is set and the animation has been set as visible.

Custom Attributes

When changing the binary output flag, custom attributes are added to the selected nodes if they do not already exist.
Custom attributes are displayed with the following names in the channel box.

  • NW Binarize Bone Anim
  • NW4F Binarize Translate
  • NW4F Binarize Rotate
  • NW4F Binarize Scale
  • NW Binarize Visibility Anim

If you set keys for custom attributes, you can change the binary output flag based on the animation starting frame (this feature is primarily intended for multi-export plug-ins).
However, if a key with a different value is set on a between the start and end frame, only the value in the start frame is applied in the intermediate file.

It was not possible to set keys for custom attributes added in NintendoSDK 9.x.x and earlier. When you select a node while the binary output selection plug-in is running in NintendoSDK 10.0.0 and later, if keys could not be set for the NW4F Binarize Translate, NW4F Binarize Rotate, and NW4F Binarize Scale custom attributes, they are automatically changed to allow keys to be set.
