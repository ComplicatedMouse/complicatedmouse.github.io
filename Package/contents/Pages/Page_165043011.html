Checking for Profanity

Introduction

The ngc library provides the following profanity check features.

  • Checking strings against a profanity list to determine whether the string contains profanity
  • Counting the number of numeric characters in strings

Checking for Profanity

In applications where users exchange user-generated content (UGC), you can check whether the UGC contains profanity.

Implementing a Profanity Check

Use one of the following functions to check for profanity.

  • nn::ngc::ProfanityFilter::CheckProfanityWords
  • nn::ngc::ProfanityFilter::MaskProfanityWordsInText

Your application must take countermeasures such as the following so that the text that the CheckProfanityWords function determined as problematic will not be displayed to communication partners.

  • Replace the text with alternate characters such as *.
  • Do not send the text, or discard it if it is received.

On the other hand, the MaskProfanityWordsInText function replaces any problematic portions of input text with * (asterisks). For this reason, the application itself does not need to perform these countermeasures.

In addition, a profanity check for profanity in Mii nicknames is not necessary because Mii characters that have nicknames that contain profanity cannot be created.

The ngc library checks for profanity using different profanity pattern lists, which are dependent on the system language settings. You are free to perform the check using lists for any languages in addition to the preset lists in the ngc library.

The profanity check can be performed either on the sender or the receiver side. When the sender and receiver have a different system language setting, it is acceptable for the application to choose the side on which the check is performed.
It is acceptable to check on both the sender and the receiver side. You can also perform the check while the text is input. Note that in doing so, there is a possibility that the language setting could be changed after the check but before the message is sent. However, no particular handling is required for such cases.

You are welcome to perform your own independent profanity check. This assumes that you first perform a profanity check using the library provided by Nintendo.

Pattern lists stored outside of the ngc library are dynamically referenced.
As a result, the checked content may change, depending on the SDK version or development hardware initialized with the SDK version.
From a compatibility perspective, do not write code that assumes that the result will always be constant for specific strings.
