NintendoWare Audio News

This page introduces the changes in the NintendoWare audio development environment for NintendoSDK 0.7.0.

The name of the NintendoWare sound development environment changed from "NintendoWare sound" to "NintendoWare audio."

The NintendoWare sound development environment had previously been called "NintendoWare sound." As of NintendoSDK 0.7.0, this name has been changed to "NintendoWare audio."

The lower layers of the NintendoSDK are called "graphics" (nn::gfx) and "audio" (nn::audio). This change was made to align those layers, but the role and scope have not changed.

Changed the placement directory for command-line tools.

The location for command-line tools for individual sound resources, such as WaveConverter, has been changed as follows.
If you are using these tools, please use this new location.

Directory Description
Tools/Audio/SoundMaker/

The location for tools for sound projects.

  • SoundProjectConverter.exe
  • SoundProjectWaveExporter.exe
  • Other

Tools/Audio/AtkTools/

The location for tools for individual sound resources.

  • WaveConverter.exe
  • Other

Changed the command-line interface.

Made changes to the command-line interface for NintendoWare audio as a means of bringing the NintendoSDK command-line interface specification into line.
We ask those who use the command-line tools to comply with the new specifications.

For a detailed list of changes, see NintendoSDK Migration Guide for NintendoWare Sound.

Compatibility With NintendoSDK 0.6.0 Binary Resource Files

Changed the endianness of the binary resource files (such as BFSAR and BFSTM files) used by the nw::snd and nn::atk libraries to little-endian. This change makes them incompatible with NintendoSDK 0.6.0.

Binary resource files converted with NintendoSDK 0.6.0 will not be available for use in the nw::snd and nn::atk runtimes for NintendoSDK 0.7.0.
Please re-convert using the NintendoSDK 0.7.0 converter.
