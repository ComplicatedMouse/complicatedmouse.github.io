NMETA File

The NMETA file describes the settings for an application or a set of downloadable content.

This file is written in XML format using UTF-8 with BOM.

In AuthoringEditor, you can use the GUI to create and edit NMETA files.

NMETA Files for Applications

The NMETA file for an application is used when creating the NPDM and NCA/NSP files. For more information, see Creating an Independent Build System (NX Development Hardware Environment).

The NMETA file to be used as a template is placed in the following location. Copy and edit it for each application.

Type of Application Metadata Source
64-bit applications %NINTENDO_SDK_ROOT%\Resources\SpecFiles\Application.aarch64.lp64.nmeta
32-bit applications %NINTENDO_SDK_ROOT%\Resources\SpecFiles\Application.arm.ilp32.nmeta

When using Visual Studio, reference the edited file when building and referencing Project Parameters. If a new application project is created from the Visual Studio wizard, a template NMETA file is automatically created in the same directory as the project file. This file is already configured to be referenced during the build.

The NMETA file for an application has three broad groups of settings.

  • Core
    • Configures the settings required to start and run applications.
  • Application
    • Application control data.
    • Configures the settings of the system the application uses.
  • CardSpec
    • Configure the parameters when writing applications to NX cards.
Note

Currently, if the application format is set to Raw as a VSI parameter, you must run the createnspmeta command in AuthoringTool to have the application management data applied.

$ AuthoringTool.exe createnspmeta -o sample.nspd --meta $NINTENDO_SDK_ROOT/Resources/SpecFiles/Application.aarch64.lp64.nmeta --type Application

If you register this command as a Post-Build Event, it eliminates the risk of forgetting to run it.

For more information about the command, see Creating Metadata to Place in NSPD in AuthoringTool.

This procedure might be fixed in a future update.

Core Settings

Application ID

  • A unique ID assigned to each application.
  • Element name: ApplicationId
  • Value
    • 64-bit ID string
  • For more information about the value to set, see Application ID.

The application ID is required.

Program Index

  • An identifier for each program in a multi-program application.
  • Element Name: ProgramIndex
  • Value
    • A value between 0 and 15
  • For more information, see Multi-Program Application.

Main Thread Stack Size

  • Specifies the stack size for the main thread of the application.
  • Element name: MainThreadStackSize
  • Value
    • String expressing a uint32 number
      • Use a hexadecimal value.
  • The default is 0x100000.

Process Address Space

  • Specifies the address space of an application.
    • For more information about address space size differences and reserved regions, see Application Operating Environments (Development Hardware Environment).
  • Element name: ProcessAddressSpace
  • Values
    • AddressSpace32Bit
      • Uses a 32-bit address space.
      • Can be specified when using Application.arm.ilp32.nmeta.
    • AddressSpace32BitNoReserved
      • Uses a 32-bit address space.
      • Uses the reserve memory region as the region for the memory heap.
      • Can be specified when using Application.arm.ilp32.nmeta.
    • AddressSpace64Bit
      • Uses a 64-bit address space.
      • Can be specified when using Application.aarch64.lp64.nmeta.
  • Set from the beginning in the NMETA file that is used.
  • Do not revise this setting except when specifying AddressSpace32BitNoReserved.

System Resource Size

  • Specifies the memory size available to the system.
    • When this value is set to something larger than 0, less memory becomes available to the application but more memory can be used by the system.
    • This setting can be used only when the address space is set to AddressSpace64Bit.
  • Element name: SystemResourceSize
  • Value
    • String expressing a uint32 number
      • Use a hexadecimal value.
      • Specify as an integer multiple of 0x200000.
  • The default is 0.

Virtual Address Memory Allocation Compatibility Mode

  • Specifies whether to make the virtual address memory allocation method the same as in versions earlier than NX Add-On 7.2.
    • In NX Add-On 7.2 and later, when an application re-allocates memory that has been released once, it can now allocate at high speed without initializing the memory.
    • Set this value to True to disable the preceding optimization and always initialize the memory when allocating virtual memory addresses.
  • Element name: MemoryAllocationCompatibilityMode
  • Values
    • False
      • Does not use compatibility mode.
    • True
      • Uses compatibility mode.
  • The default is False.

File System API Usage Permissions

  • Sets the usage permissions for a specific file system API.
  • Currently, this setting must be described only in the following cases.
    • When preparing an environment for application testing in which the fs library's functions for debugging cannot be used.
      • For more information, see Using Debugging APIs in fs Library > Development Overview.
    • When mounting another application's save data.
      • For more information, see Mounting Another Application's Save Data in Save Data (NX Add-On).
      • You must apply in advance to set the save data from another application. Contact Nintendo.
    • When it is a multi-program application and mounting a resource contained in another program.
      • For more information, see Multi-Program Application > Resource Data.

Application Settings

Title

  • The name for the application, as shown by the system features.
  • Titles can be set individually for multiple languages.
  • Settings for a single title include the title name and language, and the publisher name.
  • Element name: Title
  • Values
    • Title name (Element name: Name)
      • Must be limited to 127 characters.
      • You can only use the character codes supported by the Japan, US, and Europe fonts, and the Korean, Simplified Chinese, and Traditional Chinese fonts described in Shared Data > Internal Fonts in the System Features Guide.
      • Korean, Simplified Chinese, and Traditional Chinese fonts can be configured in NX Add-On 7.2.0 and later.
    • Title language (Element name: Language)
      • AmericanEnglish, BritishEnglish, Japanese, French, German, LatinAmericanSpanish, Spanish, Italian, Dutch, CanadianFrench, Portuguese, Russian, SimplifiedChinese, TraditionalChinese, Korean, BrazilianPortuguese
      • Make sure that the title language is configured when you specify a title.
      • BrazilianPortuguese is available in NX Add-On 11.4.0 and in NX Add-On 10.9.0 and later.
    • Publisher name (Element name: Publisher)
      • A publisher name can be up to 63 characters in length.
      • You can only use the character codes supported by the Japan, US, and Europe fonts, and the Korean, Simplified Chinese, and Traditional Chinese fonts described in Shared Data > Internal Fonts in the System Features Guide.
      • Korean, Simplified Chinese, and Traditional Chinese fonts can be configured in NX Add-On 7.2.0 and later.
  • The default value is all null characters.
  • To submit the package, entries must be made for all items.
  • If the system settings use a language not specified by the title, an appropriate language will be selected and displayed among the available languages.
    • For more information about the priorities for selected languages, see How UI Languages Work in the System Features Guide.

Icons

  • The icon that will be displayed in the system features.
    • For more information about icons, see Icons in Application Images.
  • An icon path can be set for each of the multiple languages.
    • For more information about the priorities for selected languages, see How Display Languages Work in the System Feature Guide.
  • An icon path must be set for every language set for the title.
  • Element name: Icon
  • Values
    • Icon language (Element name: Language)
      • AmericanEnglish, BritishEnglish, Japanese, French, German, LatinAmericanSpanish, Spanish, Italian, Dutch, CanadianFrench, Portuguese, Russian, SimplifiedChinese, TraditionalChinese, Korean, BrazilianPortuguese
      • BrazilianPortuguese is available in NX Add-On 11.4.0 and in NX Add-On 10.9.0 and later.
    • Path for large icon (Element name: IconPath)
      • Set an absolute path, or an indirect path, from the directory containing the NMETA file.
    • Path for small icon (Element name: NxIconPath)
      • Set an absolute path, or an indirect path, from the directory containing the NMETA file.
      • Under normal circumstances, do not specify this setting.
  • The default is no settings.
  • You can describe the environment variables, using the %environment_variable_name% format after setting UseEnvironmentVariable to true as an attribute value for IconPath and NxIconPath.

Release Version

  • Specifies the application release version.
  • When submitting applications and patches to the Nintendo Developer Portal, configure the release version issued by the Nintendo Developer Portal to this.
  • Element name: ReleaseVersion
  • Value
    • Specifies a string that represents the UInt16 value.
  • The default is 0.

Version Notation

  • The version number of the application as shown in the system.
  • Element name: DisplayVersion
  • Value
    • The value is set as an alphanumeric string (example: 3.1.5b) that is displayed after Ver. in the system.
    • An ASCII string of at least 1 character, up to 15 characters
  • Multiple languages are not supported.
  • The configured string is shown on the screen with details about the software that was selected from HOME Menu > System Settings > Data Management > Advanced Software Management.
  • Make sure that you configure a different string for each different release version of an application and patch.
  • The default value is all null characters.

Console Account Selection at Startup

  • Used for the system to select a console account when the application starts.
  • Element name: StartupUserAccount
  • Values
    • Required
      • A console account will be determined at startup even if the application does not act to select a console account. The application does not need to handle the process of selecting a console account.
    • RequiredWithNetworkServiceAccountAvailable
      • The console account that is selected when the application starts is always associated with a Nintendo Account.
      • The application cannot start from a console account that is not associated with a Nintendo Account.
    • None
      • The system ensures nothing for console accounts at startup.
  • The default is None.

Option for Console Account Selection at Startup

  • Specify the option based on the StartupUserAccount setting.
  • Element name: StartupUserAccountOption
  • Value
    • IsOptional
      • Enable this option in cases where you want to implement your own user selection functionality in an application, but use that information if it is selected because the console account requires it.
      • Effect of IsOptional

        If this option is enabled, user selection is omitted for StartUpUserAccount if it is not required for operating the console.

Restricting the Switching of User Accounts

  • Specify whether to restrict the users accounts that can open while the application is running.
    • By restricting the users accounts that can open, you can gain exemption from some of the guidelines.
      (Reference: Prohibition of opening restricted (unqualified) user accounts.)
  • Element name: UserAccountSwitchLock
  • Values
    • Disable
      • No restrictions on which user accounts can open
    • Enable
      • All user accounts are restricted from opening except the one that is selected according to the StartupUserAccount setting when the application starts.
      • If None has been set for StartupUserAccount, no user accounts can open.
  • The default is Disable.

Parental Controls Settings

  • Used to describe the Parental Controls elements used by the application.
    • When you configure the elements, the applications are listed on the Parental Controls settings screen, and enable and disable options can be selected.
    • The processing to enable and disable element features must be implemented by the application.
    • Whether the configured content has been properly applied in the implementation is verified during Lotcheck.
  • Element name: ParentalControl
  • Value
    • FreeCommunication
      • Free communication with others is available. For the pctl library, specify this element when using features related to Free Communication With Others.
  • The default is No Settings.

  • Notes About the Settings
    • You must set a rating for the release region.
      • To set a rating, use AuthoringEditor to import the NMETA file downloaded from the Nintendo Developer Portal.
      • If the application is exempt from game rating evaluation, set the rating classification to Not Required (Age Value 0) as the setting for the rating organizations in the regions where the application will be released.
    • Do not set ratings for regions where your application will not be released.


Note
  • Mexican rating (SMEC) information is automatically converted from the ESRB rating setting. In order to configure a Mexican rating, please configure the age value through the ESRB rating.
  • In NX Add-on 9.3.0 and later, the IARC generic rating (IARC Generic) settings are supported. For more information about submitting master ROMs for applications that use the IARC generic rating. see the notifications in the Nintendo Developer Portal.
  • Reconfigure the rating using the new AuthoringEditor when using an NMETA where the rating was configured using NX Add-On 5.6.1, NX Add-On 6.5.2, or with AuthoringEditor for a version of NX Add-On earlier than 7.3.0.
    • Open the NMETA file using the new AuthoringEditor and reconfigure the settings.
      • Any options that were configured in AuthoringEditor using ratings organizations or categories that were subsequently changed in specifications are configured in separate options. Reconfigure for an appropriate rating category.
      • For more information, see Changes Since NX Add-On 5.6.1, NX Add-On 6.5.2, and NX Add-On 7.3.0.
    • No revisions required for previously submitted NSP files.
    • Ratings for PEGI Portugal remain in the XML exported data under My Products in Nintendo Developers Portal, but they can be used as is.
Changes Since NX Add-On 5.6.1, NX Add-On 6.5.2, and NX Add-On 7.3.0
  • For Japan’s CERO ratings, the age set for 審査予定 and 規定適合 was changed from 0 to 18.
  • For Taiwan’s GSRMR ratings, the age set for 審査中 was changed from 0 to 18.
  • For ACB (Australia) ratings, the age set for Needs Confirmation was changed from 0 to 18.
  • Standardized PEGI and PEGI Portugal age values into PEGI.
    • A PEGI Portugal rating is not needed when using AuthoringEditor for NX Add-On 5.6.1, 6.5.2, and 7.3.0 or later. PEGI Portugal has been removed.

Target Age Setting (for China)

  • Added target age settings for applications for Chinese consoles.
  • Game products for mainland China are required to indicate a target age.
  • A CADPA icon (target age icon) corresponding to this setting is displayed when the application is started.
  • Element name: AppropriateAgeForChina
  • Setting Value

    • Set one of the following values.

      Setting ValueTarget Age
      NoneNot specified
      Age88 or older
      Age1212 or older
      Age1616 or older
  • The default is None.
  • Unlike ratings, this setting does not restrict starting the app.
  • It can be configured in NX Add-On 13.3.2, NX Add-On 12.3.11, NX Add-On 11.4.9, and NX Add-On 10.9.9 or later.
  • A CADPA icon (target age icon) is displayed in the HOME Menu in NintendoSDK Firmware for NX 13.2.0-1.0 and later.

Supported Languages

  • If supported languages have been specified and the system's language setting is set to another language that is not supported, the application sets one of the supported languages using the following priority.
  • Element name: SupportedLanguage
  • Value
    • AmericanEnglish, BritishEnglish, Japanese, French, German, LatinAmericanSpanish, Spanish, Italian, Dutch, CanadianFrench, Portuguese, Russian, SimplifiedChinese, TraditionalChinese, Korean, BrazilianPortuguese
    • BrazilianPortuguese is available in NX Add-On 11.4.0 and in NX Add-On 10.9.0 and later.
  • Use nn::oe::GetDesiredLanguage() to get the language among the supported languages that was set to the highest priority in the system's language setting.
  • At least one supported language must be set.
  • For more information about the priorities for selected languages, see System Features Guide > Appendix > How Display Languages Work.

Presence Group ID

  • This ID is used when Friend Presence is shared among multiple applications.
    • For more information about presence group IDs, see Presence Group ID.
  • Element name: PresenceGroupId
  • Value
    • Specify a 64-bit ID string.
  • The default value is the ApplicationId.

Screenshot

  • Sets whether screenshots of the game can be captured.
  • If Deny is specified for this option or set by another API, screenshots cannot be captured.
    • For more information about the APIs, see Operating Environment Controls.
  • Element name: Screenshot
  • Values
    • Allow
      • Allows screenshots within the game.
    • Deny
      • Denies screenshots within the game.
  • The default is Allow.

Video Capture

  • You can configure whether an application enables video captures.
  • It is only permitted when all of the following conditions are met.

Runtime Parameter Delivery

  • In some cases, you might want the system to pass parameters while an application is running.
  • This setting specifies the timing for receiving those parameters.
  • If you are not using features associated with runtime parameter delivery, this setting has no effect on the behavior of the application.
  • Element name: RuntimeParameterDelivery
  • Values
    • Always: Always receive parameters.
    • AlwaysIfUserStateMatched: Always receive parameters if the user state is matched (*).
      • *A user state match occurs when the users that have the application in the Open state (0 or more) and the users for which the parameters need to be received (0 or more) matches exactly.
        • In other words, if no users that the parameters need to be received for have been specified (0 users), the parameters are only passed to the running application if there are no users that have the application in the Open state.
      • If the user state does not match, the parameters are passed when the application is restarted.
    • OnRestart: Always receive parameters when the application is restarted.
      • With this value, parameters are never received while the application is running.
  • The default is Always.
  • Restarting the application has disadvantages such as the following.
    • In many cases, additional time is required before the user can use the application.
    • Application progress from before restarting might be lost.

Public Settings for Play Reports

  • Sets whether to make application usage status publicly available to users on other consoles.
  • This setting is available in NX Add-On 10.0.0 and later.
  • Element name: PlayLogPolicy
  • Values
    • Open
      • Setting for standard applications.
      • The system follows the user's Friend settings and appropriately makes public when the user is playing the application, or has played it, to users on other consoles.
    • Closed
      • This setting is for applications, such as closed beta versions, where it is not desirable for a large number of unknown users to know of the application's existence or running state.
      • The system is not shared when an application is being played or when it has been played with users on other consoles.
        • Specifically, the friend presence status while playing the application becomes always offline, and no play records are retained.
        • It cannot be exempted from the Nintendo Switch Parental Controls. If the console the application is played on is subject to Nintendo Switch Parental Controls, the play status is shared with the smart device application.
      • You can send friend requests or friend invitations freely from within the application.
        • If these features are implemented, data must be appropriately managed as needed via implementation or operation.
  • The default is Open.
  • You must make a request in advance to use Closed. Contact Nintendo.

Setting Downloadable Content Required at Startup

  • Set downloadable content that you want to ensure is available to the application when it starts. Use this setting if you intend to always distribute the downloadable content together with the application as a set and do not support starting the application without it.
  • This setting is available in NX Add-On 9.7.0, and in NX Add-On 10.3.0 and later.
  • Element name: RequiredAddOnContentsSet
    • Index
      • Specifies the index of the required downloadable content.
      • You can specify multiple Index values in RequiredAddOnContentsSet.
  • You can describe multiple RequiredAddOnContentsSet elements. The application can start providing that all of the Index values set in any one of RequiredAddOnContentsSet elements are available to the application.
    • In other words, you can use this for And/Or startup conditions. For example, if you want to allow startup in the case of (1 and 2) or (1 and 3) or (4), write as follows.
    • <Applicaction>
  :
  <RequiredAddOnContentsSet>
    <Index>1</Index>
    <Index>2</Index>
  </RequiredAddOnContentsSet>
  <RequiredAddOnContentsSet>
    <Index>1</Index>
    <Index>3</Index>
  </RequiredAddOnContentsSet>
  <RequiredAddOnContentsSet>
    <Index>4</Index>
  </RequiredAddOnContentsSet>
  :
</Application>

  • The total number of Index values is limited to 32. As a result, when the maximum RequiredAddOnContentsSet elements that can be listed is 32 and also multiple Index values are specified for one RequiredAddOnContentsSet element, the number of the RequiredAddOnContentsSet elements you can list decreases.
  • If this element is not specified, no required downloadable content is specified and the application can start without any.

Policy Setting for Changes to Available Content

  • Set the policy for changing the availability of content while the application is running.
    • The required behavior of the application when using downloadable content varies depending on this setting. For more information, see Policy Setting for Changes to Available Content.
    • The behavior regarding user account restrictions will change depending on this setting. For more information, see User Account.
  • This setting is available in NX Add-On 13.3.2 and later.
  • Element name: ContentsAvailabilityTransitionPolicy
  • Setting values
    • Legacy
      • Available downloadable content does not change when opening or closing a user account.
      • Whether or not each user account is restricted (Qualified/Unqualified) is determined when the application starts, and does not change until the application is closed.
    • Stable
      • Available downloadable content does not change when opening or closing a user account.
      • The existence of restrictions (Qualified or Unqualified) on each user account changes while the application is running.
        • The user account can be changed to Qualified and opened at a time when the application and all of the DLC that was available at startup is available to that user account.
    • Changeable
      • Available downloadable content changes when user accounts are opened or closed.
        • If there was downloadable content that was not available when the user account was opened, that downloadable content will not be available from the application. In such a case, downloadable content will become available when that user account is closed.
      • The existence of restrictions (Qualified or Unqualified) on each user account changes while the application is running.
        • The user account can be set to Qualified and opened if the application is available.
      • If DLC that is required at startup has been specified, selecting this policy setting might make that DLC unavailable while the application is running. Select Stable if you need to maintain the required DLC after starting the application.
  • The default value is Legacy for NX Add-On 14.0.0 and earlier, and Stable for NX Add-On 14.1.0 and later.

CardSpec Settings

Info

You can omit the code for CardSpec settings. For more information, see NX Development Card.

Capacity

  • Sets the capacity available on the NX development card.
  • Element name: Size
  • Value
    • 2, 4, 8, 16, or 32
  • Default Value
    • Automatically set according to the size of the application data. For more information, see NX Development Card.

Operating Frequency

  • Sets the operating frequency of the NX development card.
  • Element name: ClockRate
  • Value
    • Either 25 or 50
    • The obtainable value is restricted according to the setting value for the capacity.
  • Default Value
    • Automatically set according to the size of the application data. For more information, see NX Development Card.

The following is an example of a pattern to change the setting from the automatic setting and check the speed. (Be careful not to leave this setting in place.)
