Overview

The Vibration Collection is a tool for previewing how vibrations and sounds work on EDEV or SDEV hardware that can play vibrations and audio at the same time.


The Vibration Collection provides a set of vibration files and sound files as presets.

Simply select a vibration and a sound from the menu

to try different combinations and test out how they feel.


The impression of the vibration is greatly influenced by the sounds that are playing at the same time, and you can experience how this impression changes by testing various sounds with the same vibration pattern.


This tool is ideal for people who are trying the vibration feature for the first time or want to try combining vibrations with audio.

Using the Tool for Game Production

The Vibration Collection tool displays a menu showing the vibration files and sound files in the specified folder on an SD card or the host PC.


If you import the files for sounds that are used in your game to this folder, you can test vibration and sound combinations.

For example, if you have an audio waveform for something like a gunshot or an explosion that makes an attenuating sound, a suitable vibration would be one that attenuates with the same waveform.

In this case, select a preset with an attenuated vibration.

(For Gun1.wav, a suitable preset would be P01_Damped1.bnvib.)


The tool allows you to do more than just selecting a vibration file. You can also change playback parameters like the playback speed, pitch, gain, and offset.

Using these parameters, it is possible to simply create a vibration file that is slightly weaker than it is now, or a vibration file with a slightly higher pitch.

The vibration waveform with the changed parameters can be saved as a new vibration file.

For example, if the vibration feels too strong for the sound, you can make an adjustment by lowering the gain.

(For Gun1.wav, the P01_Damped1.bnvib preset works more effectively with Gain-Low set to 0.5.)

By selecting different vibration files and adjusting the parameters you can check how different vibrations feel with a particular sound.


The sound file that is playing at the same time as the vibration has a huge impact on the perception of the vibration.

To experience this, play P02_Damped2.bnvib with the audio files Metal1.wav, Metal2.wav, and Metal3.wav, in that order.

You sense that the same metal material is being hit, but the feeling is different simply by changing the sound that is played at the same time.

To better understand the relationship between sound and vibration, try playing the vibration with the sound muted.

Of course, it is not necessary to prepare different vibrations files for every sound in the game.

To reduce development costs, consider using general-purpose vibration files for all but the scenes that matter most.


The preset vibration files prepared in the Vibration Collection are not a comprehensive set of vibrations for all sounds.

Create new vibration files if you are not satisfied with the feel of the presets for the scenes that matter most.

You can use the Vibration Converter tool to convert sound files to vibration files.


It is difficult to describe how vibrations feel.

Because this tool provides an easy way to preview combinations of vibrations and how they feel to different people, it serves as a communication tool when many people are assessing and adjusting vibrations.

Location

This tool is included with NintendoSDK NX Add-On and is found in the following location.

TargetTools/NX-NXFP2-a64/VibrationCollection/Release/VibrationCollection.nsp


The preset sound files and vibration files are found in the following location.

TargetTools/NX-NXFP2-a64/VibrationCollection/Release/Resources
Info

Important Information

The vibration files provided with this tool, and the vibration files generated by editing them, can be used only with games for the Nintendo Switch.

The sound files are only for use with this tool.


How to Use

The tool provides the following features.

  1. Load vibration files and sound files.
  2. Play a vibration file and a sound file at the same time.

  3. Play vibration files after adjusting the playback parameters (playback speed, pitch, and gain).

  4. Edit the loop settings of a vibration file.
  5. Write a vibration file after editing the playback parameters.

Preparations

To start the Vibration Collection tool, run the following script. When running the tool for the first time, press 0 and ENTER.

TargetTools/NX-NXFP2-a64/VibrationCollection/Release/RunVibrationCollection.bat


How to Operate

After it is started, an image like the following is displayed on the NX screen.

The list of vibration files is shown in the center of the screen and the list of sound files is to the right.


You can operate this tool using a controller and also via the touch screen.

Select files to play by pressing the A Button.

The selected vibration file is highlighted with green and the selected sound file is highlighted with red.



Press the X Button to play a vibration and a sound. (Or click Play Vibration in the lower-left part of the screen.)

The waveform of the active vibration is shown on the left side of the screen.

The preceding image shows changes in the amplitude time series, and the image in the following section shows the current amplitude and frequency.



Click Open in the upper right of the screen to edit the vibration file and sound effect file before loading them.

Before loading, you can select whether to load from the SD card inserted in the Switch or from the host PC.

For more information about the limitations on the path set before loading, see Features.


After making the changes before loading, click Reload and close the file or

Click Reload in the upper right of the screen to reload and run the file.


Setting the Playback Parameters


If you want to change parameters while a vibration is playing, press the + Button or the - Button.

(Or click Vibration Setting in the lower-left part of the screen.)


Change the vibration playback parameters by changing the following settings.

Parameter Name Description Setting Range Notes
Gain-Low The gain in the low-frequency band. 0.00 - 5.00 The vibration is stronger for values above 1.0.
Gain-High The gain in the high-frequency band. 0.00 - 5.00
Pitch-Low The pitch in the low-frequency band. 0.00 - 5.00 The frequency is higher for values above 1.0.
Pitch-High The pitch in the high-frequency band. 0.00 - 5.00
Speed The playback speed of the vibration. 0.01 - 5.00 Set to 0.5 to double the playback time.
Offset The time from the audio playback until the vibration playback. 0.00 - 2.50 sec Set to 0.5 to start playing the vibration 0.5 seconds after the audio.

To reset the parameters, press the Y Button.


To play the vibration with the newly configured parameters, press the X Button from the Vibration Setting screen.

The parameters for sound playback do not change. However, you can adjust the volume using the system volume buttons.


To close the Vibration Setting screen, press the + Button or the - Button again.


Info

Important Information

The data in the vibration files contained in the Vibration Collection has been normalized. As a result, vibrations may be overly intense when used directly in games.

In this case, adjust the gain to a value in the range from about 0.2 to 0.5 before using.

Loop Playback Feature Settings


You can change the loop playback settings for a vibration by changing the following settings in the parameter settings screen.

Parameter Name Description Notes
LoopInterval Interval from the end of the loop range until the vibration is played again. Can be edited in units of 1 sample (5 milliseconds).
LoopFlag Enables or disables loop playback.

When loop playback is enabled, the loop range is displayed on the vibration waveform in the parameter settings screen.

The loop range can be edited by stick operations or touch screen operations while the vibration waveform is selected.


The loop playback settings can be reset by pressing the Y Button.

Vibration File Output Feature


You can export the vibration file that was configured in the parameter settings screen by clicking the Export button on the parameter settings screen.


When changing a vibration file before exporting it, you must change the following settings.

  • OutputDir: The directory name to export the file to.
  • FileName: The name of the exported file.

(For more information about the limitations on the paths that can be set as export destinations, see Features.)


Click the Export Bnvib File button to execute the file output.

File Structure

The vibration files and sound files used with this tool have the following formats.

Vibration files *.bnvib
Sound files *.wav

To use this tool to play vibration files and sound effects, you must place those files in the correct directories before you start the tool.

Vibration files must be placed in the bnvid directory, and sound files must be placed in the se directory.

Argument Settings

To change where files are loaded from, specify an argument when starting the tool.

Change the target on the SD card. -sd *****
Change the target on the host. -host *****

Example: -host D:/Test/Test -> Loads the files in D:/Test/Test/bnvib and D:/Test/Test/se.


Example of Starting the Tool

$ Tools/CommandLineTools/RunOnTarget.exe TargetTools/NX-NXFP2-a64/VibrationCollection/Release/VibrationCollection.nsp -- -host D:/test


You can also install the NSP file on the development hardware and use the tool. For more information about installing the NSP file, see Installing and Running an NX Application on the Development Hardware.

-- For -host D:/test, substitute the place where you copied the preset files.
