Auto Test Assist Tool

Introduction

Purpose of This Documentation

This documentation describes the Auto Test Assist Tool included in NintendoSDK.

Intended Audience

This documentation is intended for people developing applications using NintendoSDK.

Important Information

Note

The Auto Test Assist Tool is available for use in NX Add-On 9.1.0 and later.

Features of the Auto Test Assist Tool

The Auto Test Assist Tool is a library for the automated testing of the Nintendo Switch system, implemented in the form of a set of DLLs for automatic control of the development hardware.

The primary functions are to get log information output from the development hardware and video data via the capture board, and to enable the use of HID input operations.

Operating Environment

The operating environment basically complies with the NintendoSDK operating environment.

As a standalone environment, it verifies operation of the following items.

Capture Boards
  • Compatible with DirectShow, and resolutions of 1280 × 720 and 1920 × 1080.
Info

Confirmed to run specifically under the following environments.

  • Micomsoft SC-510N1/AVS
  • Micomsoft SC-500N1/DVI
  • AVerMedia GC550


Image quality may become unstable when using a capture board that performs hardware encoding as a standard.

Note: A capture board is unnecessary when you are not using CaptureProcessor.dll, or when you are using the capture function only via the network.
Python
  • Python 3.10.0 (x64)

Note: Unnecessary when using the Auto Test Assist Tool library from anything other than a Python script.
Notes Only supported in 64-bit environments.
