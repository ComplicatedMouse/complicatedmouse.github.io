New Effect Expressions

This section introduces features that broaden the scope of what can be depicted with effects.

Camera Distance Scale Restriction (Far)

Added Camera Distance Scale Restriction (Far) () to Emitter Node () > Scale () > Other Scale Settings ().
This feature keeps particles from becoming smaller than necessary as they move away from the camera.

This allows you to set display sizes for game character information, for example, or other information needed for the game, that can be recognized by the player without respect for the camera distance.

The following images compare the differences between when the Camera Distance Scale Restriction (Far) check box () is selected and when it is cleared.
When this feature is used, the particle marked as '1P' maintains a fixed size even when it is beyond the specified distance of 120 from the camera.

Refraction Effect That Distorts the Image More the Farther It Is From the Camera

Added Distance-from-Camera-Based Distortion () to Emitter Node () > Combiner () > Particle Shader ().
The farther away a particle is from the camera, the stronger the applied distortion.

The following images compare the same distortion coefficient when the Distance-from-Camera-Based Distortion check box () is selected to when it is cleared.
When this feature is used, distortion appears fixed, regardless of distance from the camera.

Emitter Extended Parameter

Added an Emitter Extended Parameter () feature that can attach specific data to effects to be used in game applications.

Values that can be used as game-application-specific features can be configured. A UI can also be added in EffectMaker in a format that fits those values.
Users can edit values set in the UI and save them as effect data.

Emitter Extended Parameter () performs one of the following actions.

  • Right-click, and from the shortcut menu, select Add Emitter Extended Parameter ().
  • On the toolbar, click Add Emitter Extended Parameter ().

The following image shows the Emitter Extended Parameter properties screen.
New settings items that use a UI that can be handled in EffectMaker have been added.

The following table describes the elements of the basic EffectMaker UI that the emitter extended parameter can use.

A Check Box UI element for selecting one of two selections.
Has the advantage of not taking up much space.
B Option Button

UI element for selecting one of two or more selections.
Has the advantage of clearly showing the selections.
C Color Type UI element for intuitively setting color and alpha.
Has the advantage of being able to see the actual color while setting it.
D Multiple Type UI element that allows you to set a dial () for multiple items, such as each axis (x, y, and z) in a set of coordinates.
E Integer Type UI element that allows you to set a dial () for integer values.
F Decimal Number Type UI element that allows you to set a dial () for decimal number values.
G
Combo Box

UI element for selecting one of two or more selections.
Has the advantage of taking up less space than option buttons if there are a lot of selections.

Removing the Parent-Child Relationship With Remove Hierarchy

You can now easily remove the hierarchy in emitters that are in a parent-child relationship in the workspace.
This section describes the operations when Emitter3 (), a child emitter under the hierarchy of Emitter1 (), is selected.

Do one of the following.

  • Right-click, and from the shorcut menu, select Remove Hierarchy ().
  • You can also press CTRL+SHIFT+P as a keyboard shortcut.

The parent-child relationship between Emitter1 (), the higher emitter in the hierarchy, and Emitter3 (), the child emitter in a child hierarchy, is removed.
The node is placed in the workspace as follows. Emitter3 () is in the same hierarchy level as Emitter1 ().

Support for the Model Settings Node

You can now confirm the model data added to the Model Settings node in the Viewer.
The size and color of the model can be adjusted with the registered model and effects linked together in the viewer.

Model data added to Model Node () > Basic Settings () > Models () > Specifying Files () is displayed in the Viewer.
In addition, by selecting animations from the animation files () within folders added to Character Animation () > Specifying Folders (), you can now reflect the animations to the models in the Viewer.

Emitter Node () > Combiner () > Particle Shader () > Deform vertex based on speed () is no longer dependent on the Viewer's playback speed. A constant scale is now always applied.
