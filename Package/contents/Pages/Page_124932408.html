WindowsFont

This section describes the WindowsFont tab UI.
Windows font files can be used as input font files.

A Font Name

Specifies the Windows font name to use as the conversion source.

Click to view a list of all fonts installed on the PC.

Fonts that have the tool tip background color as the background color are raster fonts (bitmap fonts), and fonts with normal background colors are vector fonts (outline fonts).
B Import Internal System Font (BFTTF) Cannot be used currently outside the Cafe platform. Use FontConverterConsole when reading Switch system fonts outside the Cafe platform.
C Size

Specifies the conversion source font size in pixels.

Because the size of the font that is actually output depends on each font, consider the value as for reference only.
D Specify Average Width

Enables specification of the average width for conversion source fonts. When enabled, the average width can be specified so that height-width ratios can be changed.
E Average Width

Specifies the average width of the conversion source font in pixels. Consider the value as for reference only.
F Use nn::fontll for the source from which the kerning is extracted.

Use this option when slight differences in display are a problem with BFFNT fonts and TTF and OTF fonts. For example, when using nn::font, nn::fontll, or nn::ui2d to draw TTF and other BFFNT source fonts.

The default is Off.
G Bit Depth

Specifies the number of levels in the gray scale output.

The information in parentheses is the texture format used when the data is saved as a BFFNT file.
H Enable Soft Antialiasing

Softens the anti-aliasing process on glyphs that are output in several gradations (other than 2-gradation).

Depending on the font, the quality of the output glyph may be improved.
I Character Width
Specifies how to calculate the character width of the font to output.
Related Links For more information, see here.
J Character Width

This setting can only be configured when Fixed width for all characters is selected for the character width.

The common width for all characters is specified in pixels.
K Weight

Specifies the character weight.

Specify a value in the range from 0 through 900. 400 is normal, and 700 is bold.
Info

Characters in the font that is in Unicode 0x0000 through 0xFFFD are passed to the output side. When the output side is an image, the output characters are restricted by the character order file, but for BFFNT, if no character filter is specified, many characters in the font are output. Depending on the font, as many as several tens of thousands of characters may be output.

  • Bit depth Specifies the number of levels in the gray scale output.
    The information in parentheses is the texture format used when the data is saved as a BFFNT file.
  • Deprecated Options
    The deprecated options decrease the number of gradations without changing the memory size for the font.
    They have been retained for backward compatibility. Unless you have a specific reason, use L8 or LA8 instead.


  • Character Width
    Specifies how to calculate the character width of the font to output.
    • Use the glyph width as character width
      Makes the character width equal to the glyph width.
      In other words, no space is inserted before or after the glyph. Consequently, it is proportional.

      Blank space characters are output with a character width of 0, and if used that way, it displays as if there are no blank spaces.
      To use blank space characters, either output using the Use the glyph width (keep spaces) setting, or output the image file and then adjust the character width of the blank space character.
    • Use the glyph width (keep spaces)
      Similar to Use the glyph width as character width, but the character width for blank spaces is the same as when Use character width specified in font is specified.
    • Use character width specified in font
      Uses the character width of each font as is. As a result, proportional fonts are proportional and fixed-width fonts are fixed-width.
    • Fixed width for all characters
      By having the appropriate amount of space before and after the glyph, all characters have the same character width.
      The common width for all characters is specified by Specify Width, described later. Output is fixed-width.
