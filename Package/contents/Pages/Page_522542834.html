When HDEV Initialization Fails

Factors such as the HDEV status can cause initialization to fail.

  • Initialization fails.
  • The HDEV console you want to initialize does not appear (or it no longer appears) on the HDEV connection list in InitializeHdevWin.

If this happens, verify the following.

Using InitializeHdev.exe (or InitializeHdevWin.exe)

We recommend directly calling InitializeHdev.exe or InitializeHdevWin.exe.

The following files also call these functions directly.

  • %NINTENDO_SDK_ROOT\InitializeHdev.bat

Verifying the Specified Parameters

If you are using the InitializeHdev.exe command-line tool, verify that the specified parameters match. (The serial number works with or without a check digit.)

Repeating Initialization

Sometimes initializing the HDEV several times is successful. Try to initialize again.

Fully Charging the HDEV Console

Try initialization after fully charging the HDEV console.

To charge the device you must attach the AC adapter. Note that simply connecting the USB cable does not charge the console.

Reconnecting the USB Cable and Turning On the Power

Try initializing the HDEV console again after reconnecting the USB cable and turning the power on.

If initialization is particularly unsuccessful, reconnect the USB cable with the following procedure.

  1. Disconnect the USB cable from the HDEV system.
  2. Hold down the POWER Button on the upper left of the HDEV console for approximately 12 seconds to turn off the power.
  3. Reconnect the USB cable to the HDEV console while the power is off.
  4. The HDEV turns on automatically when it is connected. Try initialization again.

If you are using the device connected to a relay box, try connecting it directly to the PC without using the relay box.

Exiting Target Manager if It Is Running

Exit the Target Manager if it is running, and then try to initialize HDEV.

Sometimes the Target Manager stays in the system tray, so also check there.

Verifying That the HDEV Console Is Recognized by the Device Manager

Verify that the HDEV is recognized as a USB device in Device Manager.
For more information, see Initializing the HDEV > HDEV Drivers.

Manually Activating Safe Mode

Sometimes initialization is successful after activating safe mode.

Info

Safe mode is a mode that runs with minimum features.

Use safe mode in the following situations.

  • The console is not operating properly and must be restored.
  • HDEV initialization fails.
  • The power is on but the system menu does not appear.

Safe mode can be activated with user operations.

After safe mode is activated, you may not be able to start in normal mode until the EDEV is initialized, so we recommend using safe mode only when you must restore or initialize.

Entering Safe Mode

  1. Hold down the POWER Button (for approximately 12 seconds) and turn off the HDEV console with the USB cable connected. (The system will restart after the button is held down for 20 seconds, so do not go much longer than 12 seconds.)
  2. Press the buttons on the upper left of the console in the following order.
    1. Press the POWER Button while holding down the volume down button.
    2. While holding down the POWER Button, release the volume down button and press the volume up button.
    3. While holding down the volume up button, release the POWER Button.
    4. Release the volume up button.
  3. To check whether safe mode is running, see Method for Verifying Whether the System Has Entered Safe Mode. If the system did not enter safe mode correctly, restart the process from step 1.
Info

For cases where the system enters the standard startup sequence and displays the menu despite attempting to enter safe mode, sometimes adjusting the intervals between button operations will fix the issue. (This is because some operations may not be detected if the intervals between button operations is too fast.)

As a guide, try using an interval of 0.6 seconds. It is equivalent to 4 beat intervals at 100 beats per minute (bpm). We recommend searching for a video with "BPM=100" that you can use as a guide.

Verifying Whether the System Has Entered Safe Mode

Use the following procedure to verify whether the system has entered safe mode.

  1. Press the Re-Scan button to display the systems that are in safe mode "(Safe Mode)" in the list of connected systems. (However, only do this when using InitializeHdevWin.exe.)
  2. "NX DevKit Safe Mode" is displayed for four seconds on the HDEV console screen.

Using USB 2.0 When USB 3.0 Connection Fails

HDEV setup may fail depending on the compatibility with the USB 3.0 controller.
Try the following.

  • Update the USB 3.0 controller driver to the latest version.
  • Update the USB 3.0 controller firmware to the latest version. (Sometimes this firmware is not publicly available.)
  • Perform setup by connecting the device to a USB 2.0 port.

Setup has been confirmed to fail when using a PC with the following USB 3.0 controller.

  • Renesas Electronics USB 3.0 Host Controller

Contacting Support

If initialization is unsuccessful after attempting these procedures, contact Nintendo support.

When contacting support, the process will be smoother if you include the following information.

  • Serial number of the HDEV unit
  • SDK version
  • Initialization log
    • For InitializeHdevWin, copy the log that appears in the log window to a text file and send in the text file.
    • For InitializeHdev, execute a log with the --verbose option added and save it to a file.
