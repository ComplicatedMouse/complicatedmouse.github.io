Package Structure

NintendoSDK Packages

NintendoSDK is divided into multiple packages for delivery.

This makes it possible to get only the development environment you need while avoiding the risk of using features you do not need.

Example:

  • Not installing NintendoWare to make sure that NintendoWare is not used

List of Packages

Common Packages

The following packages are provided in the standard NintendoSDK package.

Basics
NintendoSDK

These libraries and tools are required for application development.

If this package is not installed, applications cannot be built.
NintendoSDK Samples

These samples are for NintendoSDK.
NintendoWare
NintendoSDK NintendoWare This middleware allows development of 3D and 2D graphics, effects, and sound. For more information, see NintendoWare . Depends on the NintendoSDK package.
NintendoSDK NintendoWare Samples These samples are for NintendoWare.

Package Contents

All packages in NintendoSDK are installed in the same directory with the following structure.

NintendoSDK/ Build/
Includes the Visual Studio project templates and other resources for application development.

Common/
Contains the information required for building applications.

Compilers/
Contains the compilers required in specific environments.

Documents/ API/

Includes the API reference.


Licenses/ Contains the license for the external libraries on which NintendoSDK depends.


Package/ Includes the other documentation.


index.html The index page for the documentation.

Externals/
Contains libraries external to NintendoSDK on which NintendoSDK depends.

Include/
Contains the header files.

Installers/
Contains the installers for software required in specific environments.

Libraries/
Contains the library files.

Resources/
Contains the resources required in specific environments and for specific samples.

Revisions/
Contains the revision information for identifying the NintendoSDK package.

Samples/
Contains NintendoSDK samples.

Sources/
Contains the source code (only that which can be provided).
Operations cannot be ensured when the source code was revised. Carefully consider this fact when using modified libraries within retail products.

TargetTools/
Contains the tools (executable files) for hardware in specific environments.

Tools/
Includes tools for the host PC and the target development hardware.

NintendoSdkRootMark
This file indicates the NintendoSDK root directory. It is referenced from within the build system provided by NintendoSDK.
