Effect News

NintendoWare Effect News

This document is a graphical introduction to the features that were added in NintendoSDK 0.5.0.

Multi-Platform Support

Multi-platform support starts with this version of the effect module.
Up to now, both the Cafe version and Cafe (Windows version) existed, but now the latter has become an independent Windows version.

Info

Multi-platform support for NintendoWare Effect is in progress.
The specifications may change in subsequent versions.

Changes to the Effect Binaries

The binary format has changed as follows.
The binary endianness is now output to match the endianness of the platform.

Version 0.4.0 Binary Endianness Graphics Library
Cafe Version Big GX2
Windows Version Big OpenGL
Version 0.5.0 Binary Endianness Graphics Library
Cafe Version Big GX2
Windows Version Little OpenGL

There are also changes to the binary platforms that can be used with the Cafe runtime and Windows runtime.

Changes to Command-Line Conversions

There are changes to the parameters used in command-line conversions because of multi-platform support.
As shown in the following example, you must use the --platform option to specify the target for the conversion.

Example: Creating effect binaries for Cafe
EffectMaker.Application.exe --convlist ConvertList.xml -o EffectBinary.ptcl --platform cafe

Changes to EffectMaker

Added a combo box for specifying the platform to EffectMaker.
You must correctly specify the platform you are developing for to connect to the viewer.
Users of previous SDKs (including NW4F) can select Cafe as the platform and either DEV-KIT or PC as the viewer to connect to.

Changing this platform selection does not affect the intermediate file that gets saved.

Changes to EffectViewer

EffectViewer, which connects to EffectMaker, has also changed as follows.
Platform-specific viewers are for that platform only. The Windows viewer plays back binaries for each platform.
You can use the Cafe viewer and the Windows viewer when developing for the Cafe platform, but only the Windows viewer when developing for the Windows platform.

New Effect Expressions

Added a Loop-In-Range feature.

Added a Loop-In-Range feature that loops particles within the range of the "box" specified for displaying them.
When particles go beyond the range, they return to the opposite side of the box and are shown again.

Usage Examples

You can use this for effects positioned in the specified world space, such as rain, snow, or dust.
The particle behavior is looped inside the specified box, so a small volume of particles can be positioned for great effect.

A Clip bottom This setting can be used for effects such as having rain stop at the waterline.
B Clip top This setting can be used for effects such as having bubbles stop at the waterline.

Related Links For more information about loop-in-range, see Loop-In-Range.

EffectMaker

Tool Hints on Startup

The "Hint of the Day" is shown each time EffectMaker starts.
New EffectMaker operations are illustrated.

Color Adjustments All in One Place

The color picker is now located on the property window, making it even easier to make frequent color adjustments to particles.

Simultaneous Editing of Homogeneous Nodes

You can now select multiple homogeneous nodes on the workspace and edit them simultaneously.

In the same way as with images, you can select three emitter nodes and register a texture for one of them to have the texture registered for the other nodes as well.

Feature for Showing Where Changes Have Occurred

Tabs and labels that have had changes from the defaults are now colored orange.

In the image in the example, the contents of the Particles and Textures tabs have changed from the defaults.
If a texture is registered, the File Settings label also changes from yellow to orange.

This can be convenient for checking samples or effects created by other people to see what settings must be made when starting with a new emitter.

You can now also restore tabs and groups to their default values.

In the example in the image, this menu is opened by right-clicking in the tab.
Select Restore Tab Defaults to restore the default state when new emitters are created.

Effect Browser Feature Enhancements

Show a Windows Explorer-like list of multiple items of effect data in EffectMaker with "Effect Browser."
There is also a feature for searching and showing the assets used by each item of effect data, such as textures and model files.
The following features have been added to Effect Browser.

Search for Strings in the Search Box

Enter strings in the search box to show only files that have that string as part of their name, just as you would with the search box in Windows Explorer.

In the image in the example, the word "Fire" was entered into the search box, and of the 310 items available, only the one that have "Fire" in its name is shown in the list.

Changing the Size of Thumbnail Displays

File display sizes can now be switched between two levels.

To switch the display size of the thumbnails, in Options, click the Scaling button ().

Info

The effect of this Scaling button () differs, depending on the tab, as in the following table.

Computer or Favorites tab Files inside the Textures and Primitives folders are displayed in their large sizes.
Search tab Files showing the search results are displayed in their large sizes.

Add Label Colors and Comments Anywhere

You can add label colors and comments to textures and model files in addition to emitter sets.
The locations where label colors and comments can be added have also increased.

In the example image, the label color is being changed for a file in the list.
