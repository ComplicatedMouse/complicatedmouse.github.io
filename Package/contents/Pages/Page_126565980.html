Major Changes to atk

Memory Pool Support

Some memory regions managed by atk now support use of the memory pools feature in the audio library.
However, for certain types of memory regions, the memory pool must be managed on the application side.

For more information, see the Memory Management documentation in the Programming Guide.

Data for Which atk Manages Memory Pools

atk handles attaching/detaching memory pools as needed for data for which allocation/deallocation of the corresponding memory regions is managed using an atk sound system.

  • Sound heap, player heap
  • ADPCM context info for WSD, STRM, and SEQ

Data for Which atk Does Not Manage Memory Pools

For data for which allocation/deallocation of the corresponding memory regions is managed on the application side, manage memory pools on the application side as well.

  • Stream buffers
  • Effect buffers
  • Addresses of waveform data specified for StartInfo in wave sounds
