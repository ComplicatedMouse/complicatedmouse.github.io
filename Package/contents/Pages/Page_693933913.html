Analyze your Profile

We recommend the following workflow after taking a profile. However, once you get comfortable and better understand the profiler, you can more easily pursue clues and deeply inspect interesting areas on your own, deviating from the guide below. Ultimately, understanding your game will be an exploratory process and there are many ways to do this.

1. Check that your Profile Data is Accurate

Before looking at the profile data, please answer these two questions:

  1. Did you profile a Debug build?
    1. If yes, this is a problem because the profile data will be inaccurate and misleading.
    2. Problem: Profiling a Debug build is not helpful since the code is not optimized. If your purpose in profiling is to find optimization opportunities, a Debug build will be misleading. Do not waste your time profiling a Debug build.
    3. Solution: Profile a Release build.
    4. Note: If you are a Unity developer and only have an EDEV kit, then you will not be able to profile a Release build. While the profile data is misleading, you might still be able to get some value by looking at the data. In particular, there are specific Checkers for Unity builds that you should pay attention to if they were triggered (look at the Checkers tab).
  2. Are any Critical Checkers listed in the Checkers tab? (they have a black  arrow next to them)
    1. If yes, then there are known issues causing inaccurate results. The image below shows one black Critical Checker that needs to be fixed.
    2. Problem: Critical Checkers are issues that cause inaccurate or misleading profile data. Do not waste time looking at the profile if any Critical Checkers were triggered.
    3. Solution: Fix all Critical Checker issues and then profile again.


2. Look at the Top Functions

In the left Functions tab, you will see a list of the top functions by time.



By default, the Functions tab is sorted by Self, which is the time spent inside the function itself (but not functions it called). If you want to sort based on the time spent in the function and its children, then click on the header Total. The figure below visually shows the difference between Total time and Self time for a "Function" that calls two child functions.



In the Functions tab, start at the top of the list and move downward looking for suspicious functions that should not be taking very long. Examples of functions that shouldn't take much time are memory, string, or math functions. To better understand a suspicious function, select it with a left click. Once selected, you can see where it's called in the Call Tree tab or you can view the raw data within each frame in the Timeline tab.

3. Explore the Call Tree

By exploring the Call Tree, you can follow the hot path to identify areas to focus on. Start at the root of the tree and expand the highest percentages. To do this quickly, you can right click on a function and select Hot Path from Here.


4. Visually Inspect your Frames in the Timeline Tab

In the Timeline you can explore the raw data of the profile. Sometimes its easy to miss clues when you look at averages in the Functions or Call Tree tabs. There is real value to looking at the raw data to understand the outliers, the spikes, and the details in the data.

To start, select one interesting function in the Functions tab and then look at the Timeline tab. By default, the System Load graph is shown (seen in the image below), which represents the load of various systems over the duration of the profile. The alternating gray/white vertical stripes show the individual frames. Looking at this System Load graph, you can see that most of the time is spent in Idle (shown in red) with Graphics (shown in purple) greatly varying frame-by-frame. The color key at the top explains the color coding (please see the System Classifications section for an explanation of this automatic classification).



The function you initially selected will be shown as a single row with many short vertical lines that represent every sample recorded.

To explore this further, deselect the System Load graph in the toolbar and instead select the Icicle Graph. The Icicle Graph is best viewed if you zoom into an individual frame. Use the mouse wheel to zoom in until it looks like the image below.
