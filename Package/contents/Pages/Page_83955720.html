Custom Action Nodes

This section describes the Custom Action feature, which can be used to configure user-defined actions for game applications.

Custom actions

The Custom Action group () is a set of features for selecting one of the callback actions that were registered in the effect system by the game developer.
The action can be used for game application features and workflow purposes.
This group has the following settings.

Action ID This ID can be used to select and execute one (potentially from among many) action that was registered in the effect system by the game developer.

Bits

The Bits group () is used to set custom fields using flags.
This group has the following settings.

Bits 0-15 A group of 16 flags that can be individually selected using the check boxes ().

Numbers

The Numbers group () is used to set custom fields using sliders.
This group has the following settings.

Numbers 1-5

 Use the dial () to set the values in the range from -1000000 to 1000000.

Number (float)

The Numbers group () is used to set custom fields using sliders.
This group has the following settings.

Number 0 & Number 7 Use the dial () to set the values in the range from -1 to 1.
Info

Here, an “action” refers to a group of custom features created for a game application by the game developer.
To use these actions, the game developer prepares settings files inside the Custom Actions folder in advance.
Using these actions, you can perform behaviors such as detect collisions in the game and control particle behavior.

Custom actions are processed by the CPU. The emitter type switches to CPU emitter () when you create these nodes.

Example Configurations

Next, an example of configuring a custom action for implementing collision detection in a game application will be described.
Here, “collision detection” refers to an action that defines what happens when particles collide with objects or the map in a game.

UI Example Image Used in Game

A Action ID = “Use collision detection” The emitter uses the “collision detection” action.
B Detect collisions with map = ON
Detect collisions with object = OFF		 As shown in the image, the particles collide with the map.
They do not collide with objects (characters).
C Bounce coefficient = 0.8 When particles collide with the map, they bounce off at 0.8x speed.
