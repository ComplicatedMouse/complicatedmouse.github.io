Emitter Extended Parameter

Extended Emitter Parameters

User data definitions make it possible to add extended parameters on a per-emitter basis. Extended emitter parameters edited in EffectMaker can be accessed from the effect runtime and used to link applications and effect playback, among other purposes.

Preparations for Editing and Saving Extended Emitter Parameters and Converting to Binary

The following preparations are required for editing and saving extended emitter parameters in EffectMaker and then converting them to binary data.

  • Creating a User Data Definition File
  • Creating an Extended Emitter Parameters Settings File
  • Add these files to the project configuration file.

Creating a User Data Definition File

You can create a user data definition file that includes parameter definitions that you create and enables the custom UI to be recognized and displayed in EffectMaker. For more information, see the User Data Definitions page.

Include the following block at the beginning of the created extended emitter parameter structure, set the UserDataDefinition element's Kind attribute to EmitterExtParams, and declare it as an extended emitter parameter.

Creating an Extended Emitter Parameters Settings File

Create an extended emitter parameter settings file to import the user data definitions file that you created into EffectMaker.

1
2
3
4		 
<?xml version="1.0"?>
<ArrayOfEmitterExtParamsDefinition xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
    <EmitterExtParamsDefinition Udd2Definition="EmitterExtParamsDefinition.h" />
</ArrayOfEmitterExtParamsDefinition>
Info

The custom data configuration file must be saved using UTF-8 text encoding.

Registering in the Project Settings File

To use the custom data in EffectMaker, you must enter the path of the custom data configuration file that you created in the Project Settings file. For more information, see Project Settings File.
You can also check how things work without using the Project Settings file by setting an emitter extended parameter settings file in EffectMaker's options.

Verifying the User Interface in EffectMaker

Start EffectMaker and create a new emitter set. Right-click the emitter, and select Add extended emitter parameter to add an extended emitter parameter to the emitter.

// Custom data structure.
struct ResCustomData
{
    nn::util::Float4    color;
    nn::util::Float3    rgb;
    float               alpha;
    nn::util::Float3    position;
    nn::util::Float2    texCoord;
    float               coefficient;
    int                 paramInt;
    uint32_t            flags;
    uint32_t            choices;
    uint32_t            choices2;
    char                usrString[32];
};

// Create emitter set.
vfxSystem->CreateEmitterSetId( &handle, 0, resId, groupId, heap );
nn::vfx::EmitterSet* emitterSet = m_handle.GetEmitterSet();
 
// Get custom data.
for ( int i = 0; i < emitterSet->GetCreatedEmitterCount(); i++ )
{
    const nn::vfx::Emitter* emitter = emitterSet->GetEmitter( i );
    const ResCustomData* customData = reinterpret_cast<const ResCustomData*>( emitter->GetCustomDataParameter() );
}

/// <?xml version="1.0" encoding="UTF-8"?>
/// <UserDataDefinition Kind="EmitterExtParams"
///                     Name="Extended emitter parameters sample"
///                     Version="1">
#include <nn/util/util_MathTypes.h>
struct ResCustomData
{
    /// <ParameterDefinition Name="Color"
    ///                      Type="ColorRgba">
    /// <ColorSetting Default="1 1 1 1" />
    nn::util::Float4  color;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Color only"
    ///                      Type="ColorRgb">
    /// <ColorSetting Default="1 0.7 0.5" />
    nn::util::Float3  rgb;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Alpha only"
    ///                      Type="ColorA">
    /// <ColorSetting Default="0.3" />
    float   alpha;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Position coordinates"
    ///                      Type="Slider3f">
    /// <SliderSetting Default="0.1 0.2 0.3" Minimum="-1000000.0" Maximum="1000000.0" Step="1" />
    nn::util::Float3  position;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Texture offset"
    ///                      Type="Slider2f">
    /// <SliderSetting Default="0.5 0.5" Minimum="0.0" Maximum="1.0" Step="0.01" EnableTransform="False" />
    nn::util::Float2  texCoord;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Parameters ( Float )"
    ///                      Type="Slider1f">
    /// <SliderSetting Default="0.5" Minimum="0.0" Maximum="1.0" Step="0.01" />
    float   coefficient;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Parameters ( Integer )"
    ///                      Type="Slider1i">
    /// <SliderSetting Default="10" Minimum="0" Maximum="1000000" Step="1" />
    int     paramInt;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Flag"
    ///                      Type="CheckButtons"
    ///                      NeedReload="true">
    /// <OptionGroup>
    ///   <Option Value="Switch 1" />
    ///   <Option Value="Switch 2" />
    ///   <Option Value="Switch 3" />
    /// </OptionGroup>
    unsigned int flags;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Option"
    ///                      Type="RadioButtons">
    /// <OptionGroup Default="1">
    ///   <Option Value="Option 1" />
    ///   <Option Value="Option 2" />
    ///   <Option Value="Option 3" />
    ///   <Option Value="Option 4" />
    /// </OptionGroup>
    unsigned int choices;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="Option 2"
    ///                      Type="RadioButtons">
    /// <OptionGroup Default="1">
    ///   <Option Value="Option 2-1" />
    ///   <Option Value="Option 2-2" />
    ///   <Option Value="Option 2-3" />
    ///   <Option Value="Option 2-4" />
    /// </OptionGroup>
    unsigned int choices2;
    /// </ParameterDefinition>
    /// <ParameterDefinition Name="User-defined string"
    ///                      Type="StringComboBox">
    /// <OptionGroup>
    ///   <Option Value="UserDefinitionString1" />
    ///   <Option Value="UserDefinitionString2" />
    ///   <Option Value="UserDefinitionString3" />
    ///   <Option Value="UserDefinitionString4" />
    /// </OptionGroup>
    char    usrString[32];
    /// </ParameterDefinition>
};
/// <UILayout>
///   <Group Name="Flags">
///     <Parameter Member="flags" />
///   </Group>
///   <Group Name="Radio">
///     <Parameter Member="choices" />
///     <Parameter Member="choices2" />
///   </Group>
///   <Group Name="Color groups">
///     <Parameter Member="color" Comment="Groups including color will be automatically placed in the color picker." />
///     <Parameter Member="rgb" />
///     <Parameter Member="alpha" />
///   </Group>
///   <Group Name="Group">
///     <Parameter Member="position" />
///     <Parameter Member="texCoord" />
///     <Parameter Member="paramInt" />
///     <Parameter Member="coefficient" />
///     <Parameter Member="usrString" />
///   </Group>
/// </UILayout>
/// </UserDataDefinition>
