History Change Period(±)

This section describes the History change period (±) feature for setting the period of change on each axes for the historical samples.

Overview

The History change period (±) feature sets the waveform period using angles on each of the x-, y-, and z-axes when a waveform is used to move the history.
By setting a history change period for each of the x-, y-, and z-axes, you can make the end of the stripe sway.

The following figure demonstrates how the configured angle is replaced by coordinates on the waveform.

A The angle set for History change period determines the position coordinates on the arc.

B The position of A is replaced by a position on the waveform. The width of the sway changes over time.
Info

This feature can be used to combine two waveforms that have different periodicities, as shown in the following figure.
This technique makes the end of the stripe appear to sway randomly.


The following overview describes the settings ().

History Change Period(±)

Set the change period for the history on each of the x,y, and z axes by either using the dials () or entering values in the text boxes ().
The width of the swaying of the end of stripe varies depending on the value of (History change period × Direction change ratio).
The speed of the rate of change of the history is calculated with the following expression.

  • History change rate: sin (Time × History change period × Direction change ratio) × Initial rate of history

Blue For more information, see the following pages.

Initial Rate of History

Sets the initial speed with which to move the history in a random direction.
Direction Change Ratio Sets the change ratio for the waveform for moving the end of the stripe.
