VariableIconBodyShader

VariableIconBodyShader

Overview

nn::mii::VariableIconBodyShader is the shader class for rendering a Mii icon texture with a body with a certain physique. It uses the shader provided in the Face Library and provides features for rendering a Mii icon texture with a body with a certain physique to any texture.

Without using VariableIconBodyShader, you can use CharModel and VariableIconBody to render an icon texture with a body with a certain physique, using your own shader.

IconShaderInfo

The nn::mii::IconShaderInfo class sets the information required to use VariableIconBodyShader.

The following values can be set in IconShaderInfo.

  • Image format of the texture to write to
  • Image format of the texture for depth-stencil

Preparing the Shader File

Before initializing VariableIconBodyShader, you must prepare the shader file that is used.

Load the shader file below.

$(NINTENDO_SDK_ROOT)/Resources/Mii/$(platform)/shader/VariableIconBodyShader.bnsh

Cast the loaded data to nn::gfx::ResShaderFile and initialize.

Initializing and Finalizing

The following are notes regarding initialization and finalization. You can also view the API reference for each function.

Initialization

Use the nn::mii::VariableIconBodyShader::Initialize() function to perform initialization.

Following the shader file preparation procedure, pass the initialized nn::gfx::ResShaderFile and nn::gfx::ShaderCodeType obtained in the process of initialization to the Initialize() function.

You can use the nn::mii::VariableIconBodyShader::CalculateMemorySize() and nn::mii::VariableIconBodyShader::GetMemoryAlignment() functions to get the memory size and alignment required for initialization. nn::gfx::ResShaderFile, nn::gfx::ShaderCodeType, and IconShaderInfo passed to these functions must be the same as those during initialization. nn::gfx::ResShaderFile must be initialized.

The shader for rendering icon textures is composed from nn::gfx::ResShaderFile and IconShaderInfo, passed during initialization. The initialized content cannot be changed until the object is finalized.

nn::gfx::ResShaderFile and the memory used during initialization, must be maintained until you call termination processing.

Finalization

Call the nn::mii::VariableIconBodyShader::Finalize() function to perform termination processing. After termination processing, you can destroy nn::gfx::ResShaderFile and the memory used in initialization.

Rendering Icons with a Certain Physique

First, initialize nn::mii::VariableIconBody containing the body shape data.

Initialize VariableIconBody using the following procedure.

  1. Load the IconBody body resource.
  2. Initialize IconBody.

Next, prepare nn::mii::VariableIconBodyGpuBuffer, which contains the constant buffer and color target view required for rendering an icon texture with a certain physique.

Using the following procedure, prepare to pass VariableIconBodyGpuBuffer to the nn::mii::IconShader::DrawIcon() function. It is assumed that nn::mii::CharModel is initialized and that rendering to the Faceline and Mask textures in CharModel is complete.

  1. Call the nn::mii::VariableIconBodyGpuBuffer::Initialize() function to perform initialization.
    Use the Calculate functions for VariableIconBodyGpuBuffer for the memory and memory pool size.
  2. Use the nn::mii::VariableIconBodyGpuBuffer::SetBackgroundColor() function to specify the background color.
  3. Use the nn::mii::VariableIconBodyGpuBuffer::SetCharModelAndBody() function to set the CharModel and model type you want to render, the mask texture index that is used, and VariableIconBody.
    The specified model type must be a type included in the generation flags set during CharModel initialization. If the memory pool passed during initialization is CpuCached, you must flush the memory pool after calling this function.
  4. Use the nn::mii::IconGpuBuffer::SetColorTargetAndDepthStencil() function to set the texture to render to, and the depth-stencil texture that is used.
    Set the texture to render to width, height, and IconShaderInfo passed during initialization of the VariableIconBodyShader that is used. If the memory pool passed during initialization is CpuCached, you must flush the memory pool after calling this function.

Between calling the nn::gfx::CommandBuffer::Begin() and the nn::gfx::CommandBuffer::End() functions, call the nn::mii::VariableIconBodyShader::DrawIcon() function.

Pass the command buffer, VariableIconBody, VariableIconBodyGpuBuffer, and sampler to the DrawIcon() function. For the sampler, pass the sampler created using nn::mii::GetSamplerInfo(). In the DrawIcon() function, add a command for rendering the icon texture to the command buffer passed as a parameter.

When command execution for rendering the icon texture is complete, you can destroy VariableIconBody, VariableIconBodyGpuBuffer, and CharModel if they are not needed.
