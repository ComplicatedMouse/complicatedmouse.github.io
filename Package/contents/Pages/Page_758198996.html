Assembly Tab

The Assembly tab shows the assembly for a given function along with sample hit counts per line.

The simplest way to view a function's assembly code is to click on the Assembly tab and then click on a function in the Functions  tab. You can also go to the Code Coverage tab, right click on any function, and select Show Assembly.

The right click option to Show Assembly is provided anywhere a function is displayed, including:

  • Functions and Last tabs
  • Checkers tab (in the Results Table of each checker)
  • Call Tree tab (including the Partial Call Tree)
  • Timeline tab (on a function row or Icicle Graph)
  • Code Coverage tab (both Seen and Not Seen functions)
  • Function Details dialog

Assembly Tab Toolbar

The following controls are on the Assembly tab toolbar.

Copy Button

The Copy button copies the assembly as it currently appears to the clipboard.

View Previous Line Hit Button

This button brings into view the previous non-visible assembly line that has a non-zero Self or Sub count.

View Next Line Hit Button

This button brings into view the next non-visible assembly line that has a non-zero Self or Sub count.

Show Source Toggle Button

Enable this button to show source code that corresponds with each line of assembly. This will only work if the profiler can locate the source code, which might require setting a directory in the Options (found on the Home tab of the Ribbon Bar).

Assembly Function Information

The function information shows details about the currently selected function.

  • Function: Contains the name of the function that is being viewed along with the core and thread the function was seen on. The hit counts in the assembly view body are based on which core and thread is shown here. To see details for other threads or cores, select the function from that core and thread in the Functions tab.
  • Location: Shows the location reported by the source code lookup tool specified in the NX Options. If the link is clicked on, the profiler will attempt to open the file using an external editor as specified in the General Options.

Assembly View

The assembly for the selected function is shown.

Each line of assembly has a color associated with it. Lines with hits on Self are shades of red and lines with hits in Sub are shades of yellow-green. The saturation of the color shows the number of times a line was hit in the function. The line with the most Self or Sub will have the most intense color. The color fades to white as the number of hits approaches zero.

The following is a description of the columns. Note that clicking on the columns will resort the values.

  • Address: The address in memory of the instruction.
  • Self: The number of times this instruction was sampled.
  • Sub: The number of times code branched from this instruction was sampled.
  • ByteCode: The byte code that represents this instruction.
  • Assembly: The assembly for this instruction.

Right Click Menu

Within the Assembly Table you can right click on a row for the following options:

  • Select Target Function: Selects the target function in the Functions tab.
  • Jump to Target: Makes the assembly of the target address visible in the Assembly View. If the jump occurs within the same function, the Assembly View scrolls to bring the target address to the top of the window. If the jump is to another function then the Assembly View changes the selected function to the target function.
  • Show Target Details: Brings up the Function Details dialog, showing details on the target function.
  • Copy Target Function Name: Copies the target function name to the clipboard.
  • View data as...: This is only applicable for 32-bit NX applications where data is stored within a function.
