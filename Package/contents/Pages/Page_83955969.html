Extended User Information Presets

Advantages

Using the preset definitions for extended user information enables you to enter information by simply selecting items from a list.
This helps avoid typos when entering filenames, and allows you to read the descriptions written by the definer of the presets to check the specifications for the extended user information.

Defining Presets

Include extended user data presets in the project definition file (NW_LytProject.flpj) located in the parts root folder.

<?xml version="1.0" encoding="utf-8"?>
<ProjectSettings xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
<ExUserDataPresets>
  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions> Describes the user data. </Descriptions>
  </ExUserDataPreset>
</ExUserDataPresets>
</ProjectSettings>

Presets are then displayed in a list on the Extended User Data page in the Property Window.

ExUserDataPreset Elements

Contains a Name element and a Type element.
You can specify any of the following for the Type element: Float, Int, String, AnmInt, AnmFloat, AnmIntVec2, AnmFloatVec2, AnmFloatVec3, AnmByteRGBA4, or AnmByteRGBA4Degamma.

Placing Limitations on Target Panes

By coding the PaneKindLimitations element, you can limit the preset target to only certain kinds of panes.
If this element is omitted from your coding, all of the kinds of panes will be targets.
However, if ControlNameLimitations is coded, then panes deemed to be presets for controls are not targets.
For PaneKind, specify Picture, Textbox, Window, Parts, Bounding , or Null.
When RestrictedToOverwrite is true, the target is limited to overwriting within the part. If omitted, this is treated as false.

  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions> Describes the user data. </Descriptions>
    <PaneKindLimitations> <PaneKind>Null</PaneKind> <PaneKind>Picture</PaneKind> </PaneKindLimitations>
    <RestrictedToOverwrite>false</RestrictedToOverwrite>
  </ExUserDataPreset>
</ExUserDataPresets>

Placing Limitations on Target Controls

By coding the ControlNameLimitations element, you can limit the preset target to only certain kinds of controls.
Specify the control name for the string element.
However, if this option is omitted, and at the same time, PaneKindLimitations is coded or RestrictedToOverwrite is set to true, all of the controls are considered to be presets for panes and none are targets.

<ExUserDataPresets>
  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions> Describes the user data. </Descriptions>
    <ControlNameLimitations> <string>Locator</string> <string>NormalButton</string> </ControlNameLimitations>
  </ExUserDataPreset>
</ExUserDataPresets>
Note

By simultaneously specifying PaneKindLimitations and ControlNameLimitations, you can configure presets that target both certain panes and certain controls.


Making It Possible To Apply Layout and Animation Segment Tags

You can specify whether to apply tags to the layout using the ApplicableToLayout element. You can also specify whether to apply the animation segment tags using the ApplicableToAnimTag element. Set to true or false.

<ExUserDataPresets>
  <ExUserDataPreset>
    <Name>DataName</Name>
    <Type>String</Type>
    <Descriptions> Describes the user data. </Descriptions>
    <ApplicableToLayout>true</ApplicableToLayout>
    <ApplicableToAnimTag>true</ApplicableToAnimTag>
  </ExUserDataPreset>
</ExUserDataPresets>
Note

In addition to descriptions like ApplicableToLayout and ApplicableToAnimTag, if you add a description like <ControlNameLimitations> <string>__Dummy__</string> </ControlNameLimitations> to target a control that does not exist, you can create presets that can be applied only to layouts or animation segment tags.


Selection List

Using the EnumValues element allows you to enter values that you have prepared ahead of time directly from the selection list.
This allows you to easily enter values and fix any typos that may be present.
Specify a selection as a string in the EnumValue element. (You can also specify numbers.) Each selection is described in the Description attribute (may be omitted).
When MultiSelect is true, you can select one or more elements. If multiple elements have been selected, the String type values in the FLYT file are saved with line delimiters, and the Float and Int type values are saved with space delimiters.

Extended User Information Presets Definition File (FLUP File)

In addition to FLPJ files, you can also write presets to FLUP files. Use this approach to split the definition files into multiple files for ease of management.

All FLUP files in the parts root folder (the same folder as the FLPJ files) are loaded.

The setting for loading later files first when there are multiple presets with the same name.
For example, if you divide files into a FLUP file that determines the overall rules, and a FLUP file that determines individual team rules, you can use the individual rules that are loaded later to override the overall rules.

The files are loaded in alphabetical order (case-insensitive), so change the filenames to adjust the order.
