General Options Tab

This tab contains general options for working with the profiler.

Functions

Options in this group relate to the Functions Tab.

Function Selection Colors

By default, functions are highlighted using a color based on its inferred classification (i.e. which system the function is thought to belong to, such as Graphics = dark purple, Physics = yellow, and so on). The option here overrides this behavior to use a color based on a rotating set of 16 colors. This can make it slightly easier to view data in the Timeline if you are more interested in looking at the function level rather than the system level.

Timeline

Options in this group relate to the Timeline Tab.

DPI Scaling

This setting supports Windows DPI scaling, which is common with 4K monitors. It is recommended to leave this on its default setting of Use OS Value. If you have two monitors with different DPIs, the OS value is automatically updated based on the monitor where main profiler window is located. The OS value is shown in case you wish to change the scaling to be either larger or smaller. For example, using a smaller DPI scaling value can make it easier to see large amounts of data on the Timeline, but might make it difficult to read the text or hover the mouse pointer over individual items. Using a larger DPI scaling makes it easier to read the text or click on individual items.

Icicle Graph Line Height

There are two types of icicle graphs that are shown in the Timeline: functions and code blocks.

This setting controls how tall each line will be in pixels. Smaller values allow you to see more information on the screen, at the cost of making it more difficult to hover the mouse pointer over a particular item.

Overview

Options in this group relate to the Overview Tab.

Blink on Warning

This setting will control whether the Overview tab item will blink if another tab is currently focused. This provides a visual indicator that a warning has occurred.

Assembly

Options in this group relate to the Assembly Tab.

External Source Editor

These settings control how the profiler attempts to open source files when attempting to open them from the Assembly tab. By default, the profiler will just pass the source file to Windows Explorer to open the file with your default editor. The additional settings are provided to allow you to either choose a different editor or to pass in additional information, such as to open the file to the line number shown.

  • Command: What will be passed to Windows Explorer to run. By default will be the file to open. However, programs or scripts can also be selected to run.
  • Arguments: The arguments to pass to pass when attempting to run the specified Command.

Example

Visual Studio 2017

The following will open a new instance of Visual Studio 2017 each time a file is clicked on and scroll to a specific line number.

Command: C:\Program Files (x86)\Microsoft Visual Studio\2017\Professional\Common7\IDE\devenv.exe

Arguments: $1 /command "edit.goto $2"
