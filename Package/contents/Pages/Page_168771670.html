Creating Documents That Can Accompany Applications

Types of Documents That Can Accompany Applications

The following three types of documents can be specified by the application.

  • Offline HTML
    • The data accessed by the Offline Web Applet.
    • For more information about using this data, see Offline Web Applet.
  • List of Internet Site URLs Accessible by the Web Applet
    • For more information, see Web Applet.
  • Legal information
    • Information that must be presented primarily for legal reasons.
    • Uses the information created by the Software Legal Information System (SLIM) on the Nintendo Developer Portal.

Creating Documents That Can Accompany Applications


  • When creating the application's NSP file with AuthoringTool, write the paths to the directories for the various documents or the ZIP file in the NMETA file.
    • For more information about writing a NMETA file, see Writing the NMETA File.
    • If there are redundant specifications in the AuthoringTool options and the NMETA file, the description in the NMETA file takes precedence.

Example of Running AuthoringTool

$ AuthoringTool.exe creatensp --type Application --desc Application.desc --meta Application.meta --program programDirectory dataDirectory


When using VSI to build the application, specify each of the directories and ZIP files using Configuration Properties > Authoring.
