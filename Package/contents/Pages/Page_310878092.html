Overview

You can delete save data, back it up to a microSD card, and restore it from a microSD card.
For more information about creating save data, see Save Data and the fs library reference.

Save data is listed for each application separately. For the application name, "Unable to load application name" is displayed in the following cases.

  • The application was started and save data was created on the host PC.
    • Applies to applications started using VSI, Target Manager, or RunOnTarget.
  • The save data is for an application with an unspecified name.
  • The application was deleted after the save data was created.

How to Use

Displaying Application Save Data

Press the + Button or - Button to display all save data for an application.

Viewing Details

Press the + Button or - Button to display details for the save data.

Verifying

Click the Verify button to check whether save data has been corrupted.

Deleting

Click the Delete button to delete data.

Backup

Use one of the following methods according to the use case to back up save data to a microSD card.

a. Back up one specific save data file.

    • In the list of application save data, highlight the save data and press the A Button.
    • On the save data details screen, click Export.

B. Back up all save data associated with a specific application (NintendoSDK Firmware for NX 9.1.0-1.0 or later).

    • In the list of application save data, click Export All.

Option A is expected to be used in cases where you want to separately back up only some save data. B is expected to be used in cases where you want to back up user save data for multiple users at the same time.
With option B, you can select whether to perform a batch backup including cache storage.

Backup save data is saved as binary data in the following paths containing the date and application ID information.

sd:/NintendoSDK/BackupSaveData/<yyyymmddhhmmss>_<applicationId>/<type>/<applicationId>/<xxxx>/

One of the following strings is entered for <type>, according to the save data type. <xxxx> is a number, and it is currently fixed at 0001.

Type String
User Account Save Data user
bcat bcat
Cache Storage cache

If the full path of the backup destination exceeds 260 characters, it is saved as an NSAVE archive file.

sd:/NintendoSDK/BackupSaveData/<yyyymmddhhmmss>_<applicationId>.nsave
Note
  • To back up save data to an archive file in NintendoSDK Firmware for NX 9.0.0-5.0 or later, use DevMenuApp. You cannot back up the data using DevMenu. Also use DevMenuApp to restore the data.
  • To extract the archive file on a Windows PC, use AuthoringTool. For more information, see AuthoringTool.
Info

Some of the following characters are copied to the SD card after being replaced in a format where they are preceded by aa6B4S8D9aP and followed by a. Any replaced characters are converted back to their original format when restoring save data.

  • Characters with an ASCII value equal to or less than 0x20
  • Characters that cannot be used in paths and multibyte characters, as listed in fs Library > Manual > Features > SD Card
    • Replacing multibyte characters is only supported in NintendoSDK Firmware for NX 9.1.0-1.0 or later. In earlier versions, none of these characters could not be replaced with an appropriate character, and they could not be restored to the original state.

Replacing Non-Multibyte Characters

For example, a half-width space (0x20) is replaced with 0x61 0x61 0x36 0x42 0x34 0x53 0x38 0x44 0x39 0x61 0x50 0x20 0x61.
When multiple characters needing replacement continue in sequence, they are all preceded (as a string) with aa6B4S8D9aP and followed with a.

Replacing Multibyte Characters

When a filename or directory name has only one multibyte character, that character is replaced in a format where it is preceded by aa6B4S8D9aP and followed by a.
When there are multiple multibyte characters, the whole name is replaced to a format where it is preceded by aa6B4S8D9aP and followed by a.

Restore

Select Backup in SD card from the drop-down menu, select the target save data, and press the A Button to restore that data.

Note: NSAVE file operations are the same as the operations for normal save data.

  • User Account Save Data
    • Displays the screen for selecting the account to link during restore. Data cannot be restored if there is no account on the system.
  • Cache Storage
    • Displays the screen for selecting the storage device to restore to. If data with the same index exists in storage that is not the restore target, that data will be deleted.

    • Data backed up in versions earlier than NintendoSDK Firmware for NX 5.0.0-4 cannot be restored in NintendoSDK Firmware for NX 5.0.1-1 and later.
      To restore, directly edit the __BackupSaveDataInfo.xml file.

      • Replace SaveDataType with Cache.

      • Replace CacheIndex with the target hexadecimal number.

        sd:/NintendoSDK/BackupSaveData/<yyyymmddhhmmss>_<applicationId>/<type>/<applicationId>/<xxxx>/__BackupSaveDataInfo.xml

<?xml version="1.0" encoding="utf-8"? >
<BackupSaveData>
  <SaveDataType>Cache</SaveDataType>
  ...
  <CacheIndex>00000000</CacheIndex>
</BackupSaveData>
  • Save Data for Debugging
    • Versions earlier than Nintendo SDK Firmware for NX 11.0.0-1
      • As with user account save data, an account selection screen is displayed prior to restoration.
      • After being restored, it is handled as user account save data.
    • NintendoSDK Firmware for NX 11.0.0-1 and later
      • There is no account selection screen prior to restoration.
      • Even after being restored, it is handled as debug save data.
      • Debug save data that was backed up in versions earlier than Nintendo SDK Firmware for NX 11.0.0-1 cannot be restored as debug save data. It is handled as user account save data.
Info

For more information about debug save data, see Save Data (NX Add-On).
