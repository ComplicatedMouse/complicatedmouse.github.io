NX Development Card Features

Use the gamecard command to write an NSP file to an NX development card inserted in the development hardware, or to get the status of that NX development card.
For more information about NX development cards, see the following documentation.

  • Nintendo Switch Overview > Hardware > Peripheral Hardware (Available on the Nintendo Developer Portal.)
  • NX Development Card 
$ RunOnTarget.exe DevMenuCommand.nsp -- gamecard --help
usage: DevMenuCommand gamecard status
       DevMenuCommand gamecard write <absolute_nsp_path> [--timeout <seconds>] [--verify] [--on-card-patch <absolute_nsp_path>] [--on-card-aoc <absolute_nsp_path>] 
                                                         [--multi-application-card <absolute_nsp_path>] [--nsp-list-file <absolute_text_file_path>] [--region <global|china>]
       DevMenuCommand gamecard erase [--timeout <seconds>]
       DevMenuCommand gamecard mount-status
       DevMenuCommand gamecard ensure-access
[SUCCESS]

Writing the Application

For more information about NSP files, see Creating Application Images and Guide to Setting Up the Environment.

$ RunOnTarget.exe DevMenuCommand.nsp -- gamecard write "C:\application.nsp"
---- given Parameter ----
target nsp : C:/application.nsp

---- Card Parameter ----
CardSize     : 1 GB
WriteSize    : 4107264 Byte

---- Write Start ----
112128 / 4107264
4044800 / 4107264
Write rest sectors.
Finished
[SUCCESS]
Option Description
--timeout

Writing times out if a game card is not inserted within the specified number of seconds.

If this option is not specified, the process waits for a game card to be inserted.
--verify Verifies that CRCs match after writing. Processing takes approximately 1.3 times longer when compared to when this option is not specified.
--on-card-patch

Use this option to write a patch at the same time. You must specify the NSP file for the patch that is the write target.

For more information about patches, see Building and Running a Patch.
--on-card-aoc

Use this option to write on-card add-on content at the same time. You must specify the NSP file for the on-card add-on content that is the write target.

For more information about on-card downloadable content, see On-Card Downloadable Content.
--multi-application-card

Use this option to write multiple applications at the same time. You must specify the NSP file for uploading.
For more information about multi-application cards, see Multi-Application Cards and Writing to NX Development Cards.

Info

In NintendoSDK Firmware for NX 9.0.0-5.0 and later, you can also include on-card downloadable content when writing to multi-application cards.
--nsp-list-file Use this option if you want to write multiple applications at the same time. You must specify the text file that was specified when the NSP file for uploading was created.
--region

Use this option to specify a region when writing. When this option is not used, the system is not a console for China.
Specify china to write to the card with settings for consoles for China.
For more information about the behavior of the --region option and cards, see For Switch Consoles For China (NX Add-On) > Important Testing Considerations for Consoles for China.

Info

This option is available in NintendoSDK Firmware for NX 9.1.0-1.0 and later.

Getting the Insertion Status

Use the following command to determine whether an NX development card is inserted.

$ RunOnTarget.exe DevMenuCommand.nsp -- gamecard status
Inserted
[SUCCESS]

Initialization

Use this command to initialize the NX development card. Everything on the card will be deleted.
To overwrite the card, you can use the command to write an application rather than initializing the card.

$ RunOnTarget.exe DevMenuCommand.nsp -- gamecard erase
Erase gamecard.
[SUCCESS]

By including the --timeout option, writing times out if a game card has not been inserted in the specified number of seconds. (If this is not specified, the process waits for a game card to be inserted.)

Getting the Mount Status

Even if an NX development card is inserted, that does not necessarily mean it is available to use.
The corresponding command can get information related to the current firmware mount.

$ RunOnTarget.exe DevMenuCommand.nsp -- gamecard mount-status
{
    "sequence": 1,
    "isAttached": true,
    "lastMountFailureResult": "0x00000000"
}
[SUCCESS]
Item Description
sequence Indicates how many times a card has been inserted and ejected from when the system started.
isAttached If true, the card is available for use. If false, the card is not available for use.
lastMountFailureResult Indicates the Result value from the last time the mount failed.

Empty Access

To check whether the NX development card can be accessed, perform an empty access.

$ RunOnTarget.exe DevMenuCommand.nsp -- gamecard ensure-access
[SUCCESS]
