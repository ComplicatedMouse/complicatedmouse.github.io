account Library

Introduction

Purpose of This Documentation

This documentation describes the account library included in NintendoSDK.

Intended Audience

This documentation is intended for people developing applications using NintendoSDK.

Related Documentation

Also see the account guides for target platforms in the Nintendo Developer Portal, and the API reference for the account library.

Features of the Account Library

The account library provides functionality for identifying the users of devices via user accounts and their associated features.
For overviews of user accounts and peripheral features, see the Account Guide for each target platform in the Nintendo Developer Portal.

Provided Features

The account library provides feature to do the following.

  • Use user accounts.
    • Access user account IDs.
    • Get user account states.
  • Use user profile information.
  • Use network service accounts.
    • Access network service account IDs.
    • Authenticate network service accounts.
    • Get the credentials (NSA ID token) for network-service accounts.

For more information about these features, see the API Reference for the library.

Prohibitions

The following are contraindicated for the account library in the current version.

Using List Position (Index) for User Identification

It is prohibited to use the user account list position (index), obtained by functions such as nn::account::ListAllUsers(), for identifying users on the console.

Use the user account ID (nn::account::Uid) to identify users on the console.
Use an ID that is unique across all consoles, such as the network service account ID, to identify users on the network.

The list position (user account list order) on the user account list may change regardless of whether an application is running.
For example, the user account position on the list may change due to the following operations.

  • Adding and deleting user accounts
  • Sorting user accounts

Using User Account IDs on a Network

The use of user account IDs, nn::account::Uid, is prohibited on networks.
Only use user account IDs to identify users within the same device.

When a unique ID is required in a network environments, use one of the following methods.

  • Use the network service account ID (nn::account::NetworkServiceAccountId).
    • Note that the network service account ID associated with a user account can be changed by a user operation.
  • Issue a proprietary sequential ID for users from the application, and use that ID.

Restrictions

The following restrictions exist for the account library in the current version.

Availability of Network Service Accounts Depends on Nintendo Account Attribute

Specifications for SDK 6.1.0 and Later

The account library does not determine whether the network service is available based on the Nintendo Account attributes.
Note however that you might be able to determine whether the service is available, for each network service used, based on different standards.

Specifications for SDK 6.0.0 and Earlier

When a user account is linked to a Nintendo Account, the ability to get the credentials (NSA ID tokens) for network-service accounts depends on the Nintendo Account attribute.

In the current version of the SDK, you cannot use features that use internet communications in the Windows environment.
