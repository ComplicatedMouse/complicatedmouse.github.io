Introduction

NintendoSDK supports operations in the environments described in Operating Environment.

This document describes how to use this environment checker tool. There is a tool you can use to check whether the environment on your PC is satisfactory for NintendoSDK.

Using the Environment Checker

You can run either a GUI application or a CUI application to use the tool that checks whether your PC environment satisfactorily supports NintendoSDK.

GUI Version SdkEnvironmentChecker.bat or
Tools\SdkEnvironmentChecker\SdkEnvironmentChecker.exe
CUI Version

Tools\SdkEnvironmentChecker\SdkEnvironmentCheckerConsole.exe

The following sections describe how to use each version of the tool.

GUI Tool

Directly under the NintendoSDK folder, double-click SdkEnvironmentChecker.bat to open the following dialog box.

Each item checks the following content.

Checks whether the first and second values of the installed .NET Core SDK version match what supports the target, and whether the third value is a number that supports the target or later.

NVN API Version, GLSLC GPU Code Version

Checks whether a NVIDIA Graphics Driver compatible with the NVN API required by NintendoSDK is installed.

In addition, for SDK 13.0.0 and later, check whether an NVIDIA graphics driver is installed that is compatible with the GPU code version of the glslc.dll installed in NintendoSDK.

Info

Needed when using NVN in a PC environment.

For more information, see NVN Description.



If "Driver not installed", "Not compatible", or "API version not compatible" is displayed, following the instructions for Installing the GPU Driver in nvn Library to install the driver.


If "Driver not installed" is displayed despite the driver being installed, check your PC settings, referring to the item in Troubleshooting called "In the Environment Checker tool, NVN API Version shows "Driver not installed" even though the NVIDIA Graphics Driver is installed."


If "GPU not supported" is displayed, verify the supported GPU and supported features in nvn Library, and prepare a supported GPU.


If "GPU code version not compatible" is displayed, check the GPU code version supported by the driver and GPU code version supported by glslc.dll, and make adjustments so that the appropriate combinations are used.

Details Window

In the lower-right corner of the window, next to the OK button, clicking Show detail displays the following items.

  • Environment checker results
  • PC environment information
  • Information about targets registered to Target Manager
  • Information about installed NintendoSDK packages and other packages


The Save button on the lower right can also be used to save a ZIP file with the information from this details window in JSON format and the set of log files generated by the development hardware initialization and update tools (%USERPROFILE%\AppData\LocalLow\Nintendo).

Info

Submit the ZIP file created using this functionality when making inquiries about bugs and other issues.

This ZIP file allows you to smoothly share information about your development environment, which can be helpful when support representatives investigate.


The content is the same as the content in the ZIP file created by the CUI version using the --detail and --zip-path options.

--exclude Option

Use the --exclude option to exclude specific items from being checked.
If --exclude is not used, all items will be checked.

If you are using the --exclude option, rather than using SdkEnvironmentChecker.bat in the NintendoSDK folder, add the option to Tools\SdkEnvironmentChecker\SdkEnvironmentChecker.exe and run it.

Calling example:

> Tools\SdkEnvironmentChecker\SdkEnvironmentChecker.exe --exclude=nvn

You can currently specify the following values.

Value Description
nvn The NVN API version is excluded from being checked.


CUI Tool

Run Tools\SdkEnvironmentChecker\SdkEnvironmentCheckerConsole.exe on the command line to output content resembling the following.

>Tools\SdkEnvironmentChecker\SdkEnvironmentCheckerConsole.exe
[NG] (Not compatible)   NVN API Version 55.8
[NG] (Not installed)    Microsoft Visual C++ 2013 Redistributable (x86) - 12.0.30501
[NG] (Not installed)    Microsoft Visual C++ 2013 Redistributable (x64) - 12.0.30501

[OK] (New version installed)    .NET Framework 4.7.2
[OK] (New version installed)    Microsoft Visual C++ 2017 Redistributable (x86) - 14.16.27024
[OK] (New version installed)    Microsoft Visual C++ 2017 Redistributable (x64) - 14.16.27024

If you run this command without any arguments, only the [NG] lines will be written to standard error output.
The [NG] lines indicate that your PC environment does not meet the requirements.
If the output includes [NG] lines, install the lacking environment components such as boxed articles, as described in GUI Tool.

The lines that read [OK] (New version installed) are standard output.
It indicates that the version of the installed program is newer than the version supported by the SDK.

If even one [NG] instance is exported, SdkEnvironmentCheckerConsole.exe will return 1 as a termination code.
If no [NG] instances are exported, 0 is returned as a termination code.


--verbose Option

Run the CUI tool with the --verbose option to include the satisfied environment components in the output.

Lines with [OK] are environment components that your PC has installed. These lines are written to standard output.
Lines with [NG] are environment components that your PC is lacking. These lines are written to standard error output.

--zip-path Option

By using the --zip-path option along with the --detail option, you can save a ZIP file including the information displayed with the --detail option in JSON format and the set of log files generated by the development hardware initialization and update tools (%USERPROFILE%\AppData\LocalLow\Nintendo).

Info

Submit the ZIP file created using this functionality when making inquiries about bugs and other issues.

This ZIP file allows you to smoothly share information about your development environment, which can be helpful when support representatives investigate.


The contents are the same as in the ZIP file created by using Save in the details window of the GUI version.
