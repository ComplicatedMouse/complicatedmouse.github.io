ReaderWriterLock Class

Feature Overview

The ReaderWriterLock class is a synchronization feature of exclusive locking for individual purposes of reading and writing of shared resources used among multiple threads.

The ReaderWriterLock class wraps the Readers-Writer Lock feature using the nn::os::ReaderWriterLockType object and has the following member functions. For more information, see the API reference.

Function Name Description Remarks
ReaderWriterLock() Constructor Performs initialization equivalent to the InitializeReaderWriterLock() function.
~ReaderWriterLock() Destructor Performs destruction equivalent to the FinalizeReaderWriterLock() function.
AcquireReadLock() Gets a read lock. Gets a read lock equivalent to the AcquireReadLock() function.
TryAcquireReadLock() Tries to get a read lock. Tries to get a read lock equivalent to the TryAcquireReadLock() function.
ReleaseReadLock() Releases a read lock. Releases a read lock equivalent to the ReleaseReadLock() function.
AcquireWriteLock() Gets a write lock. Gets a write lock equivalent to the AcquireWriteLock() function.
TryAcquireWriteLock() Tries to get a write lock. Tries to get a write lock equivalent to the TryAcquireWriteLock() function.
ReleaseWriteLock() Releases a write lock. Releases a write lock equivalent to the ReleaseWriteLock() function.

IsReadLockHeld()

Returns whether there is a read lock. Equivalent to the IsReadLockHeld() function.
IsWriteLockHeldByCurrentThread() Returns whether the current thread has a write lock.
 Equivalent to the IsWriteLockHeldByCurrentThread() function.
IsLockOwner() Determines whether the current thread is the lock owner. Equivalent to IsReaderWriterLockOwnerThread().
lock() Gets a write lock. Equivalent to the AcquireWriteLock() member function.
try_lock() Tries to get a write lock. Equivalent to the TryAcquireWriteLock() member function.
unlock() Releases a write lock. Equivalent to the ReleaseWriteLock() member function.
lock_shared() Gets a read lock. Equivalent to the AcquireReadLock() member function.
try_lock_shared() Tries to get a read lock. Equivalent to the TryAcquireReadLock() member function.
unlock_shared() Releases a read lock. Equivalent to the ReleaseReadLock() member function.


The ReaderWriterLock class meets the C++ BasicLockable and Lockable requirements, so it may be passed as type parameters for class templates like std::lock_guard and std::unique_lock. In this case, you get and release write locks. If, on the other hand, it is passed as the type parameter for the std::shared_lock class template, you get and release read locks.


Example of Using the ReaderWriterLock Class

The ReaderWriterLock class is often used as follows as some kind of class member variable. In the ValueSetting class, it is provided to a function for setting two int-type numbers, and to a function for getting those configured numbers.

#include <nn/os.h>

class ValueSetting
{
public:
    ValueSetting() : m_Value1(0), m_Value2(0) {}
    void Set(int v1, int v2)
    {
        m_Rwlock.AcquireWriteLock();
        m_Value1 = v1;
        m_Value2 = v2;
        m_Rwlock.ReleaseWriteLock();
    }
    void Get(int* p1, int* p2)
    {
        m_Rwlock.AcquireReadLock();
        *p1 = m_Value1;
        *p2 = m_Value2;
        m_Rwlock.ReleaseReadLock();
    }
private:
    int  m_Value1;
    int  m_Value2;
    nn::os::ReaderWriterLock   m_Rwlock;
};


First, the constructor of the ValueSetting class initializes private members. An instance of the ReaderWriterLock class is also initialized.

The Set() function acquires a write lock and then writes to m_Value1 and m_Value2. While Set() is running, calls to both the Set() and Get() functions from other threads are blocked.
The Get() function gets a read lock and returns the values of the m_Value1 and m_Value2 variables. While Get() is running, the Get() function called from other threads can run in parallel, but the threads that call Set() are blocked.

With lock functions used for separate purposes of reading and writing this way, the reading processes can run in parallel without blocking, while exclusive control can be safely implemented for just the writing processes.
