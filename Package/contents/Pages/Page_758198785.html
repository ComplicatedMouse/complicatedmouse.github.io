Scenario Selection

After clicking the Sync button, the Scenario Selection window will appear. These Scenarios list how to connect the profiler to your game.

Scenarios

Attach

Attaches the profiler to an already running application.


Do not launch the application with a debugger attached. If a debugger is attached, the profiler will be unable to attach and Out-of-Process profiling will not be available. Launching from the debugger is okay as long as 'Start Without Debugging' is used.


When selecting this scenario, the following items need to be specified:

  • Select NSS: The path to the NSS file for the application.

Attach on Launch

Waits for an application to launch. If an application is already running, the profiler attaches to that process.


Do not launch the application with a debugger attached. If a debugger is attached, the profiler will be unable to attach and Out-of-Process profiling will not be available. Launching from the debugger is okay as long as 'Start Without Debugging' is used.


When selecting this scenario, the following items need to be specified:

  • Select NSS: The path to the NSS file for the application.

Launch

In this scenario, the profiler handles launching the application through Target Manager and then attaches to it. This option guarantees that Out-of-Process profiling is available. Not all of the options that are available in Target Manager for launching executables are available here, nor are they intended to be. If additional options are required when launching, please use either the Attach or Attach on Launch scenario, instead.


This scenario will automatically perform the following tasks:

  1. Kill any already running applications using the ControlTarget.exe tool provided in the SDK.
  2. Start the requested application using the RunOnTarget.exe tool provided in the SDK.

To find these tools, the profiler looks first for the NINTENDO_SDK_ROOT environment variable. If that environment variable is not found, the profiler looks for the root of the SDK using the expected relative path from itself.


When selecting this scenario, the following items need to be specified:

  • Select NSS: The path to the NSS file for the application.
  • Select Executable: The path to either the NSP or NSPD_ROOT for the application.
  • Arguments: Anything entered into the arguments box will be passed to the launched application. The box can be left empty.
  • Wait for NSS parsing to finish before launching: Using this option will have the profiler wait to finish loading the provided NSS file before attempting to Launch the application. This allows for profiling application start times (the profiler will be ready to start profiling when the application starts, but you must still click the Start button to begin the profile).

In-Process Only

Taking a profile with only in-process profiling is equivalent to the profiling available in Nintendo NX CPU Profiler version 0.10 and earlier. In order to select this scenario, an already-running application on the dev kit must have called nn::profiler::Initialize(). If no application has made this call, it will not be possible to select this scenario.


This option explicitly does not allow Out-of-Process profiling. To enable it, please use one of the other scenarios.


When selecting this scenario, the following item need to be specified:

  • Select NSS: The path to the NSS file for the application.

Autofill

When selecting a Scenario, there may be Autofill buttons that are available, depending on the currently selected scenario.

Autofill NSS

Available for In-Process and Attach.

This works by sending a query to the running application to return the first command-line argument (argv[0], i.e. the path to the running executable). This will only work if the running application has called nn::profiler::Initialize(). If a path is found, the path will automatically change the extension to ".nss" and use that value as the NSS path. If no path is found, the current NSS path is cleared.

Note

If the application does not use ".nss" for the extension of the ELF file, that path will need to manually corrected after attempting the autofill.

Autofill Executable

Available for Launch.

This works by taking the existing NSS path and changing the extension. The profiler first attempts to change the file extension to ".nsp". If a file with that extension is not found, the extension ".nspd_root" is used. If one of these files is found, that file is used for the executable path. If no file is found, the current executable path is cleared.
