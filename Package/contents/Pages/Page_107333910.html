Platform Presets Files

LayoutEditor Platform Presets Files

The platform presets file designates the target platform and viewer.

.preset files are placed in the Tools\Graphics\LayoutEditor\Platforms\ directory.


─ Platformss ┬ Generic.preset
               └ PlatformAAA.preset

LayoutEditor Platform Presets File Format

Platform presets files (.preset files) are XML files.

<PlatformPreference>

The root element of a platform presets file.

Description

The user config file includes the following elements.

  • <PlatformName>: Sets the platform name. At most one may appear.
  • <UiPlatformName>: Sets the UI platform name. At most one may appear.
  • <ArchiverPlatformName>: Sets the platform name designated with the archiver. At most one may appear.
  • <AcceptableBntxPlatformNames>: Sets the BNTX platform name in the BFFNT file loaded into LayoutEditor. At most one may appear.
  • <PcViewerPath>: Sets the PC viewer path. At most one may appear.
  • <TargetViewerRunnnerPath>: Sets the development hardware viewer path. At most one may appear.
  • <PreviewStarterPath>: Sets the preview starter path. At most one may appear.
  • <PcTransferProtocol>: Sets the PC viewer and communication protocol. At most one may appear.
  • <TargetTransferProtocol>: Sets the development hardware viewer and communication protocol. At most one may appear.
  • <PcPeerType>: Sets the peer type used to communicate with the PC viewer. At most one may appear.
  • <TargetPeerType>: Sets the peer type used to communicate with the Hardware viewer. At most one may appear.
  • <PCConverterPath>: Sets the path for the converter that creates the layout binary to send to the PC viewer. At most one may appear.
  • <TargetConverterPath>: Sets the path for the converter that creates the layout binary to send to the Hardware viewer. At most one may appear.
  • <ArchiverPath>: Sets the path for LayoutArchiver. At most one may appear.
  • <ArchiverAdditionalArguments>: The number of additional arguments for the archiver. At most one may appear.
  • <PreArchiverCommandAdditionalArguments>: The number of additional arguments for the layout archiver pre-execution command. At most one may appear.
  • <FontConverterConsolePath>: Sets the path for the console version font converter. At most one may appear.
  • <SearchPCViewerByTitle>: Sets the PC Viewer search method.
  • <IsNW4FMode>: NW4F mode settings. At most one may appear.

<PlatformName>

<PlatformName> is the platform name setting.

Description

Specifies the platform name.

<UiPlatformName>

<UiPlatformName> is the UI platform name setting.

Description

Specifies the UI platform name. Must be different from the value specified for the UiPlatformName element in other .preset files.

<ArchiverPlatformName>

<ArchiverPlatformName> is the platform name setting designated with the archiver.

Description

Specifies the platform name setting designated with the archiver.

<AcceptableBntxPlatformNames>

Includes the BNTX platform name settings in the BFFNT files loaded into LayoutEditor.

Description

The user config file includes the following elements.

  • <AcceptableBntxPlatformName>: Sets the BNTX platform name in the BFFNT file loaded into LayoutEditor.

<AcceptableBntxPlatformName>

Sets the BNTX platform name settings in the BFFNT files loaded into LayoutEditor.

Description

Sets the BNTX platform name settings in the BFFNT files loaded into LayoutEditor.

<PcViewerPath>

<PcViewerPath> is the PC viewer path setting.

Description

Specifies the executable file or batch file path that starts the PC viewer.

<TargetViewerRunnnerPath>

<TargetViewerRunnerPath> is the development hardware viewer path setting.

Description

Specifies the executable file or batch file path that starts the development hardware viewer.

<PreviewStarterPath>

<PreviewStarterPath> is the preview starter path setting.

Description

Specifies the executable file or batch file path that starts the preview starter.

<PcTransferProtocol>

<PcTransferProtocol> is the PC viewer and communication protocol setting.

Description

Specifies the PC viewer and communication protocol. Specifies HTCS or HIO.

<TargetTransferProtocol>

<TargetTransferProtocol> is the development hardware viewer and communication protocol setting.

Description

Specifies the development hardware viewer and communication protocol. Specifies HTCS or HIO.

<PcPeerType>

Sets the peer type used to communicate with the PC viewer.

Description

Sets the peer type used to communicate with the PC viewer.

<TargetPeerType>

Sets the peer type used to communicate with the Target viewer.

Description

Sets the peer type used to communicate with the Target viewer.

<PCConverterPath>

Sets the path for the converter that creates the layout binary to send to the PC viewer.

Description

Sets the path for the layout converter that creates the layout binary to send to the PC viewer.

<TargetConverterPath>

Sets the path for the converter that creates the layout binary to send to the Hardware viewer.

Description

Sets the path for the layout converter that creates the layout binary to send to the Hardware viewer.

<ArchiverPath>

Sets the LayoutArchiver path.

Description

Sets the LayoutArchiver path.

<ArchiverAdditionalArguments>

Sets additional arguments for the layout archiver.

Description

Specifies additional arguments when the layout archiver executes.

<PreArchiverArchiverCommandAdditionalArguments>

Specifies additional arguments for the layout archiver pre-execution command.

Description

Specifies additional arguments for the layout archiver pre-execution command. The layout archiver pre-execution command is specified in the <PreArchiverCommandPath> element of the layout project (FLPJ) file. The first argument of the command is passed by the archive target folder. These additional arguments are passed for the second and subsequent arguments.

<FontConverterConsolePath>

Sets the console version LayoutArchiver path.

Description

Sets the console version LayoutArchiver path.

<SearchPCViewerByTitle>

Sets the PC Viewer search method.

Description

Specifies the PC Viewer search method. If true, attempts to connect to the PC viewer by referencing the window title, but not limited to the PC viewer specified in <PcViewerPath>.

<IsNW4FMode>

NW4F mode settings.

Description

Specifies true or false.

Generic.preset Example

<?xml version="1.0"?>
<PlatformPreference>
  <PlatformName>Generic</PlatformName>
  <UiPlatformName>Win</UiPlatformName>
  <ArchiverPlatformName>Win</ArchiverPlatformName>
  <AcceptableBntxPlatformNames>
    <AcceptableBntxPlatformName>Gen</AcceptableBntxPlatformName>
  </AcceptableBntxPlatformNames>
  <PcViewerPath>LayoutViewer\Win64\lytviewerGfx.exe</PcViewerPath>
  <TargetViewerRunnnerPath>LayoutViewer\Cafe\run_viewer.bat</TargetViewerRunnnerPath>
  <PcTransferProtocol>HTCS</PcTransferProtocol>
  <PcPeerType>Generic-Windows</PcPeerType>
  <TargetTransferProtocol>HIO</TargetTransferProtocol>
  <TargetConverterPath>LayoutTools\LayoutConverter.exe</TargetConverterPath>
  <PCConverterPath>LayoutTools\LayoutConverter.exe</PCConverterPath>
  <ArchiverPath>LayoutTools\LayoutArchiver.exe</ArchiverPath>
  <FontConverterConsolePath>FontConverter\FontConverterConsole.exe</FontConverterConsolePath>
  <IsNW4FMode>false</IsNW4FMode>
  <PcConverterAdditionalArguments></PcConverterAdditionalArguments>
  <ArchiverAdditionalArguments>--tile-mode linear --api-type Gl --code-type Source</ArchiverAdditionalArguments>
  <FontConverterAdditionalArguments>-tile-mode linear</FontConverterAdditionalArguments>
</PlatformPreference>
