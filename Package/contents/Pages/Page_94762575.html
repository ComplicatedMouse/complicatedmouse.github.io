Features Added by NX Add-Ons

The following features become available when the various NX add-on packages in the NintendoSDK family are added.

Item Overview Limitations and Plans Required Packages
General
  • Support for the NX API set.
  • Support for Joy-Con and Nintendo Switch Pro Controller.


  • NintendoSDK NX Add-On
Controlling the Development Hardware

Provides tools for controlling SDEV / EDEV.
  • NintendoSDK NX Add-On
  • NintendoSDK System Updater for NX
album This library provides various features for accessing albums from applications.
  • Supports builds on the Windows platform, but album functionality is not available.
  • NintendoSDK NX Add-On
aoc Library that provides features to handle downloadable content.
  • Currently not supported on the Windows platform.
  • NintendoSDK NX Add-On
ec

The following features are available.

  • Utility features for accessing eShop services
  • Feature to jump to a purchase screen for downloadable content and subscription service items
  • Feature for providing consumable service items without an independent server
  • Currently not supported on the Windows platform.
  • NintendoSDK NX Add-On
fs

The following features are available.

  • Read/write FAT32 file systems on SD cards for development.
  • NintendoSDK NX Add-On
gfx

A fast-running gfx library implemented with NVN
  • NintendoSDK NX Add-On
  • NVIDIA graphics driver
hid

The following additional features are included with the hid library.

  • Joy-Con support

Capture Button support

Taking screenshots and videos from within running applications is supported as functionality for the Capture Button.
  • There is a performance hit for applications that have enabled video capture.
    For more information, see Important Implementation Information (NX Add-On).
irsensor The IR Motion Camera features can be used.

Near Field Communication Provides features for handling near field communication, including accessing amiibo accessories.
  • NintendoSDK NX Add-On
ldn A local communications library to handle wireless communication not using a relay.
  • Currently not supported on the Windows platform.
  • NintendoSDK NX Add-On
movie The library that provides features for handling video files.
  • 8.3.1 and 9.2.0 only support some containers and codecs (WebM, VP9/Vorbis).
  • 8.x.x versions earlier than 8.3.1 and 9.x.x versions earlier than 9.2.0 do not support the Windows platform.
  • NintendoSDK NX Add-On
nvn
  • This API provides low-level graphics features for the NX.
  • Requires a GPU supported on the Windows platform and the installation of the NintendoSDK graphics libraries.
  • NintendoSDK NX Add-On
  • NVIDIA graphics driver
NvPerf The debugging library for measuring GPU performance.
  • Currently not supported on the Windows platform.
  • NintendoSDK NX Add-On
oe An API that restricts the operating environment for programs.
It includes the following features.
  • Features for notification messages to applications
  • Features for program operations
  • Features for performance selection
  • Currently not supported on the Windows platform.
  • NintendoSDK NX Add-On
OpenGL A graphics API created by the Khronos Group.
  • NintendoSDK NX Add-On
pcm

Measures power consumption on the board.
  • For more information about the restrictions in the current version, see pcm Library Restrictions.
  • NintendoSDK NX Add-On
pl

The library that provides platform-specific features.

It includes the following features.

  • Features for shared fonts
  • A feature for disabling connection to the USB interface


  • NintendoSDK NX Add-On
ro

Library that provides features to handle dynamic modules.
  • Currently not supported on the Windows platform.
  • NintendoSDK NX Add-On
vi

An API providing features for accessing the display.
It includes the following features.

  • Basic display output features, such as layer creation and format and size settings
  • NintendoSDK NX Add-On
Vulkan A graphics API created by the Khronos Group.
  • NintendoSDK NX Add-On
NintendoWare
  • Faster implementations of the graphics modules (such as nn::g3d, nn::vfx, and nn::ui2d) are available in NVN and the gfx library implemented with NVN.


  • NintendoSDK NX Add-On
  • NintendoSDK NintendoWare NX Add-On
  • NVIDIA graphics driver
System menus

The menus that are called by the HOME and POWER Buttons are collectively referred to as system menus.

The following list shows the types of system menus that are available.

  • Entrance (includes the unlock screen)
  • HOME Menu
  • Full Launcher
  • Quick Settings
  • POWER Menu
  • NintendoSDK NX Add-On
System applications

Applications built into the system that can be started from the HOME Menu are called system applications.

  • My Page
  • News
  • Album
  • System Settings

The following system applications will be added in the future.

  • Nintendo eShop
  • NintendoSDK NX Add-On
Library Applets

Library applets:
These are small, modularized applications created for specific purposes.
You can call a library applet while playing an NX application, and then return to the NX application when the applet closes.

  • account management applet
  • controller support applet
  • software-keyboard applet
  • My Page applet
  • Create or Manage Mii Applet (separate application required for ROM submission)
  • amiibo settings applet
  • ec applet
  • web applet
  • Offline Web Applet
  • error viewer applet
  • Album Applet (from NX Add-On 5.1.0)


Startup library:
The library for starting library applets.


  • NintendoSDK NX Add-On
Note

To submit a ROM that uses Mii features (the Face library and the Create or Edit Mii applet), you must use a separate application form. Contact Nintendo at support@noa.com when you want to submit this type of ROM.
