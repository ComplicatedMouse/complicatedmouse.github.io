Introduction

Purpose of This Documentation

This document describes the audio and video coding codecs included in NintendoSDK.

Intended Audience

This documentation is targeted at developers who write code relating to audio and video compression and decompression using NintendoSDK.

Related Documentation

Also see the API reference for audio and video codec.

codec Library Features

The codec library provides codec features for audio and video.

The following audio codec features are currently available.

  • AAC
  • ADPCM
  • Opus

How the codec Library Is Provided

See the following pages.

  • How the Libraries Are Provided and How to Use Them

Features Provided by the codec Library

The codec library provides the following features.

  • ADPCM Decoder
  • Opus Decoder
  • Opus Encoder

In addition, the following related PC tools are provided.

  • OpusEncoder

For more information about these features, see the manual, the API reference, and the sample program.

License Terms

AAC

The AAC decoder that is included in the codec library uses the Fraunhofer FDK AAC open-source software. When planning and developing applications that use the features of this open-source software, you must follow the provisions stipulated in the software license terms.

The following summary describes the FDK AAC license terms.

  • Copyright
    • When redistributing FDK AAC execution code or binary data that contains the FDK AAC execution code, the redistributed item must contain the following text, identical to that found in the Fraunhofer FDK AAC License Terms.

The full text of the license terms is provided in the Fraunhofer FDK AAC License Terms Review the full terms and follow its requirements.

Info

The features of the codec library are included in the binary for all applications, so the license terms are in the system settings. The license notice does not need to be shown in the application.

Opus

The opus decoder and opus encoder PC tools included in the codec library use libopus version 1.1.5, which is open source software (OSS). This license requires applications using the features found within libopus to follow the OSS licensing terms.

The following summary describes the libopus license terms.

  • Copyright
    • When redistributing libopus execution code or binary data that contains the libopus execution code, the redistributed item must contain the following text, identical to that found in the Full libopus License Terms.
  • Exclusions
    • You cannot use, without advance permission, the names of the libopus copyright holders in publicity or advertising.

The full text of the license terms is provided in the Full libopus License Terms. Review the full terms and follow its requirements.

Info

The features of the codec library are included in the binary for all applications, so the license terms are in the system settings. The license notice does not need to be shown in the application.
