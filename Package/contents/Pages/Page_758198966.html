Overview Tab

The Overview tab displays a summary of the profile results at a glance.

Warnings

When a Warning is logged, the Warnings Banner appears at the top of the Overview tab. The banner can be opened by clicking it once, and closed by clicking it again. Clicking on the Click here to view warnings in more detail link will open the Warnings Window where warnings show their complete text and the text can be copied.

Left Pane

The left pane of the Overview tab contains elements that display a visual representation of the profile results.

Hot Functions by Self

The Hot Functions by Self section displays functions which took the most time, not including the functions they call. This element supports the following interactions:

  • Hovering over a function will display its full name in a tooltip. 
  • Expanding the width of the window will reveal more of each function's name.
  • Clicking on a function will select it in the Functions tab. All instances of the function will be selected in the functions list.
Note

The percentages shown for each function are added up across all threads and cores the function was found on. As such, it is possible for a function to have a percentage over 100%.

Work Breakdown

The Work Breakdown section displays the systems that took the most CPU time. Clicking on one of the systems will select all functions belonging to that system in the Functions tab.

The percentages shown are found by adding up the self time for every function in each classification group. More details about the function classification system can be found at System Classifications.

Checkers

The Checkers section contains elements that show the state of various checkers.

Checkers Icons

The Checkers Icons display the checker severity levels, marked with how many checkers of each severity were triggered. Clicking on an icon will open the Checkers tab and scroll so that a checker of that severity will be visible.

High-Severity Checkers

The High-Severity Checkers section shows the names of all high-severity checkers that triggered. Click on a high-severity checker to open the Checkers tab and expand the checker that was selected.

Right Pane

The right pane of the Overview tab contains extra information about the profile.

Screenshot Preview

The Screenshot Preview displays a screenshot that was taken at the end of a profile. If no screenshot data is included in the profile, the preview will instead show a solid gray rectangle with a white X indicating that there is no screenshot.

Clicking the red X button in the upper-right corner of the Screenshot Preview removes the screenshot from the profile. If the profile was loaded from disk, it will need to be resaved to permanently remove the screenshot from the saved profile.

Clicking the Screenshot Preview itself will open a larger screenshot viewer window. In this window, use the mouse to pan and zoom.


Profile Info

The Profile Info section contains a short list of information about the profile. The View Detailed Info link at the bottom of the list opens a window with a more complete set of information and statistics for the profile. See the Info Window section for more detail. If there is no data for one of the below items, it will not be displayed.

  • Profile Duration: The duration of the profile according to an arbitrary core. It is possible for there to be a slight variance between durations on each core.
  • Total Samples: The total number of samples seen across all cores.
  • Average Framerate: The average frame rate seen while profiling. This relies on the user-logged main heartbeat.
  • Threads Seen: The total number of unique threads seen executing across all cores.
  • Max Callstack Depth: The depth of the deepest callstack found at any time while profiling across all cores (only visible if a callstack profile was taken).
  • Functions Seen: The number of unique functions seen executing across all cores.
  • Functions Not Seen: The number of unique functions which were present in the executable but not seen executing at any point during the profile.
  • Build SDK: The SDK that was used to build the application that was profiled.
  • Buffer Used: The size of the raw data buffer.
  • Game: The name of the executable file on the hard drive.
  • Computer: The name of the computer that ran the profiler.
  • Captured By: The name of the user logged into the computer that ran the profiler.
  • Captured On: The time and date at which the profile was taken.

Profiler Overhead

The Profiler Overhead element shows the average overhead for the entire profile across all cores. For a detailed breakdown of the overhead per core, click the View Detailed Info link in the Profile Info section.

The Sample Rate is located immediately under the Profiler Overhead element. It displays the requested sample rate at the time the profile was taken.
