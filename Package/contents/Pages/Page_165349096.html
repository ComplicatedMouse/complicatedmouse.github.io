BackupSaveData

Overview

BackupSaveData is a command-line tool to back up and list the save data saved in the SDEV or EDEV development hardware.

Packages

BackupSaveData is included in the NintendoSDK NX Add-On package.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/BackupSaveData.exe

There are dependent files in addition to this EXE file for operations.

Command Definitions

Option Format

BackupSaveData.exe <command> directory-path target-options [other-options]

command:
    export                        Exports save data from development hardware to a PC.
    import                        Imports exported data from a PC to development hardware.

directory-path:                   Specifies the directory path where the export data is located.

target-options:
    --target <IP address | serial number | MAC address | name> Specify the IP address, serial number, MAC address, or name of the target to update.
                                                                (You cannot, however, use an IP address or MAC address for EDEV.)

other-options:
    --export-title <Application Id>  Restricts the save data to export to data from the specified application. Only valid when exporting.
    --verbose                        Outputs the detailed log.
    --version                        Displays the tool's version.

The following new options have been added in NX Add-On 4.0.0.

Note

These options are not available in earlier versions.

other-options:
    --follow-exist-account       Changes the user account save data to import to save data associated with the account IDs on the development hardware. Only valid when importing data.
    --exist-account-only         Limits the user account save data to export to save data associated with accounts on the development hardware. Only valid when exporting.

Exit Code

You can use the exit code to check whether the tool succeeded.

Nonzero value: Terminated with error.

How to Use

The following shows actual examples of BackupSaveData corresponding to usage cases.

Backing Up Save Data Saved in the Development Hardware to Under the PC C:\dir Folder

BackupSaveData export C:\dir --target 169.254.0.1

Export save data saved in the SDEV with the IP address 169.254.0.1 to folders under the C:\dir folder on the PC.
This example specifies the IP address, but you can also specify the serial number, MAC address, and name. (For EDEV, you can only specify the serial number and name.)

Listing the Data Exported to Under the PC C:\dir Folder in the Specified Development Hardware

BackupSaveData import C:\dir --target 169.254.0.1

Write the data exported to under the PC C:\dir folder to the SDEV with the IP address 169.254.0.1.

Only Backing Up the Save Data Stored in the Development Hardware For Application ID 0x0100000000000000

BackupSaveData export C:\dir --target 169.254.0.1 --export-title 0100000000000000

Add --export-title <Application Id> when you want to export save data only for a particular application. Note that you do not include the 0x part in Application Id.

Restoring User Account Save Data Associated With Account IDs on the Development Hardware From Exported Save Data

BackupSaveData import C:\dir --target 169.254.0.1 --follow-exist-account

Specify the --follow-exist-account option to import only the user account save data associated with accounts on the target development hardware from the exported save data. This specification enables you to avoid an issue where imported save data cannot be accessed if the user accounts are different (described in the next section).

Only Backing Up Save Data for Usable Accounts Among the Save Data on the Development Hardware

BackupSaveData export C:\dir --target 169.254.0.1 --exist-account-only

Specify the --exist-account-only option to limit the exported user account save data to save data associated with accounts on the development hardware.
In some cases, the development hardware may have leftover save data for accounts that no longer exist. Use this option to exclude such save data from the exported save data.
If you experience difficulty attempting to import save data when the --follow-exist-account option is specified, re-exporting the save data with the --exist-account-only option and then re-importing the save data may help.

Specification Restrictions

Even If Save Data is Imported, It Cannot Be Accessed From a Different User Account

BackupSaveData requires using the same user account for importing and exporting. As a result, if data is exported when there are no user account IDs and this data is then imported, the import will succeed but the imported data cannot be accessed because there is no account that can access that data. This problem can arise in the following kinds of situations.

  • After data was exported, the development hardware was initialized.
  • The data was imported to a different development hardware unit.

In these kinds of cases, you can use the following procedure to import the save data. (Note that this procedure may be deprecated at a future date.)

  1. Create the same number of user accounts as there were when the save data was exported.
  2. Start the application and create save data.
  3. Use BackupSaveData to export the save data.
  4. Overwrite the export data you created in step 3 with the export data you to import.
  5. Use BackupSaveData to import the export data you created in step 4.
Note

In addition, some improvements have been made for the bugs that caused import to fail in the following situations.

  • The total size of the files stored in the save data was larger than the size of the journaling region.
  • The save data included a file larger than the size of the journaling region.

Also, starting in NX Add-On 4.0.0, you can make sure that imported save data is usable without having to follow this procedure by simply specifying the --follow-exist-account option when importing the save data.

Notes

  • This tool can back up application save data. It does not back up system setting values. Use the SettingsManager if you want to back up setting values.
  • Operations are not guaranteed if the exported data has been modified.
  • Save data backup features are also available in DevMenu. That feature only allows backup and restore on development hardware.
