Introduction

DevMenu is a development hardware-specific menu, with tabs providing various features for operating the development hardware.
DevMenu operates as a standalone menu, which means the following features can be used even if the development hardware is not connected to a host.

Tab Name Description Page Main Provided Features
Application Managing Applications
  • List installed applications, and start, exit, delete, and install applications.
  • Write applications to game cards.
Save Data Managing Save Data
  • List save data, and delete save data.
  • Backup save data to a microSD card.
  • Restore save data from a microSD card.
Controller Controller Management
  • Display the number of paired controllers and the status of connected controllers.
  • Pair with new controllers.
  • Delete the pairing information.
Accounts Managing Accounts
  • Add and delete users, and modify user information.
  • Register network service accounts.
Device Changing Device Settings
  • The various settings for the development hardware.
Network Network
  • Check the network connection status.
  • Various test features.
PlayReport Play Report Settings
  • Play Report Settings
Debug Development Assistance Settings
  • The settings for the development assistance features.
Time Test Time Test Settings
  • The network clock settings.
  • The steady clock settings.
Error Error Information
  • Display and export information about errors previously generated.
  • Configure features related to crash reporting.
BT Audio Managing Bluetooth Audio
  • Pair with Bluetooth audio devices.
  • Connect and disconnect Bluetooth audio devices.
  • Delete the pairing information.
Shop Shop Settings
  • Switch shops.
  • Register and unregister the consoles that can download content.
Firmware Firmware Information
  • Display the firmware version information.

If you want to use the Exhibition features, see the Exhibition Features Manual.

Info

DevMenuCommand is also available for executing the DevMenu features by using the command line.

How to Use

Starting DevMenu

DevMenu starts automatically when the development hardware is powered on. DevMenu resides in the background after an application has been started.

Info

In NintendoSDK Firmware for NX 8.0.0-5.0 and later, you can start DevMenu start from HOME Menu by using the following procedure.

  1. Tap the Sleep icon in the lower-right part of the HOME Menu.
  2. While the "going to sleep" message is displayed, simultaneously press the Left Stick or Right Stick and the + Button or - Button.
    • With the Left or Right Stick and the + or - Button held down, short press the other button.
      • While holding down the Left Stick, short press the + Button.
      • While holding down the - Button, short press the Right Stick.
        etc.
    • Note that a fatal error occurs if you perform these button operations while the application is running. (This bug will be fixed in a future update.)
  3. DevMenu starts up.

If you restart the development hardware after DevMenu has started, the HOME Menu is display again. The initial startup menu is unchanged by this operation.

How to Operate

DevMenu is operated using various controllers (debug pad, Joy-Con, or Switch Pro Controller) or by tapping the touch screen.
The following descriptions assume a controller is being used for operations.

General Operations

To display the content of a page, set the focus to that page in the frame on the left side of the screen.　
You can move the focus between the Page tab and the pages using any of the following.

  • Left and right on the +Control Pad
  • Left and right on the Left Stick
  • A Button (move from the Page tab to the page)
  • move from the page to the Page tab)

Overlay Notifications for Developers

Overlay notifications for developers appear in the DevMenu environment.

  • Overlay notifications for developers appear in English, regardless of the language settings.
  • Icons and thumbnails are not displayed, and sound effects are not played.

Switch to the HOME Menu environment to verify how operations behave on a retail unit.
