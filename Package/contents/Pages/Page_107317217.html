NintendoWare includes platform-specific information.

Although common packages are not supposed to depend on specific platforms, NintendoWare includes some platform-specific implementations, documentation, and samples.
These resources will be removed from the common package in future releases.

Note that the information about these resources is hard to use and understand in some parts of the package.

Operation is not assured when the system locale is changed to anything other than the default value.

The operation when the system locale is changed to anything other than the default value is not assured. (For example, if the locale is set to anything other than "Japanese [Japan]" in the Japanese version of Windows.)

Bugs Related to Visual Studio Help

If you try to uninstall the Documents package after uninstalling Visual Studio but with the Microsoft Help Viewer still installed, the uninstall will fail.

If you try to uninstall the Nintendo SDK Documents package after uninstalling Visual Studio (any version) but with the Microsoft Help Viewer still installed, the uninstall will fail.

Uninstall Microsoft Help Viewer first, and then uninstall the package.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer. If you try to use it, you cannot enter anything in the text box, making search impossible.

Use a web browser to use the Find Result page.

When the Documents package is installed without an Internet connection, the Visual Studio Help format API reference is sometimes not installed.

When the NintendoSDK Documents package is installed in an environment without an Internet connection, the installation succeeds, but sometimes the API reference cannot be viewed from the Microsoft Help Viewer.
This problem occurs because it cannot connect to the expiration server for the digital certificate and check the validity of the signature, so the Help file is not installed.

You can handle this problem by connecting to the Internet or by installing the intermediate certificate issued by an authority higher than the digital certificate (CN = DigiCert SHA2 Assured ID Code Signing CA) and then reinstalling the NintendoSDK Documents package.

Bugs Related to HTCS

When htcs::Initialize is called while Target Manager is not running, the function is blocked until a connection with Target Manager is established.

We plan to change this function in the future, so that the call is successful regardless of the connection status with Target Manager.

audio Bugs

Exporting sound waveforms does not work in NintendoWare SoundMaker and SoundProjectWaveExporter.

Attempts to export sound waveforms using the NintendoWare tools SoundMaker and SoundProjectWaveExporter always result in the creation of blank waveform files.

This problem is expected to be fixed by the next release.

When you convert a sound project imported using NintendoWare's SoundMaker, an invalid sound archive is exported.

When, in SoundMaker, an NW4C/NW4F sound project (one for which the cache file remains for converting purposes) is imported and (normally) converted, NW4C/NW4F binary source files become mixed in with the NintendoSDK binary sound archive (BFSAR) files. As a result, they cannot be correctly read during the nn::atk and nw::snd runtimes. This is a problem.

As a workaround to this problem, make sure to reconvert the sound project after importing it. This conversion makes the old cache file unusable.

Some of the APIs for nn::atk do not work.

APIs related to LowPassFilter do not work.

APIs related to LowPassFilter, such as nn::atk::SoundHandle::SetLowPassFilterFrequency(), can currently be set, but they are not applied.

The LowPassFilter value set in SoundMaker is not applied either.
BiquadFilter can still be used, as before.

If some of the parameters in Biquad filter are set, the audio may be cut off.

If the following parameters are set in nn::atk::SoundHandle::SetBiquadFilter() or nn::atk::SoundPlayer::SetBiquadFilter(), audio from some sources may be cut off.

  • When the filter in nn::atk::BiquadFilterType_LowPassFilter is selected and the value is set to 1.0
  • When the filter in nn::atk::BiquadFilterType_HighPassFilter is selected and the value is set to 0.1

An exception is thrown when you start the NintendoWare AtkPlayer when there are no available playback devices.

If AtkPlayer is started when there are no available playback devices on the PC, an exception is thrown, and the program halts. The workaround is to have a playback device available.

The problem that results in the exception being thrown is expected to be fixed in either the next release or the one after that.

When playing a sound in NintendoWare AtkPlayer from NintendoWare SoundMaker, the SoundParameters panel values are not applied.

When you connect SoundMaker and AtkPlayer and play a sound from SoundMaker in AtkPlayer, the values in the SoundParameters panel in AtkPlayer are not applied. However, if you change the values in the SoundParameters panel during playback, the changed parameters will be applied.

For example,if you play a sound from SoundMaker and then change Volume in the SoundParameters panel in AtkPlayer, the Volume value will be applied, but the other parameters remain at the default values, so there will be a gap between what is shown and the actual behavior.

Sometimes effects are not processed in the order they were added in nn::audio.

The intended specification is for effects to be processed in the order in which they were added to SubMix or FinalMix. However, if you have two or more effects registered to a single SubMix or FinalMix, and you then delete an effect and add another new effect, the effects sometimes run in the wrong order.

You can avoid this problem by deleting all effects from SubMix and FinalMix and then adding the effects.

This bug will be fixed in an upcoming release.

hid Bugs

An exception is thrown when the USB gamepad is detached during use in the Windows 8.1 environment.

When the USB GamePad is removed during use in the Windows 8.1 environment, an exception is thrown and the program may stop.

Accurate speed cannot be calculated for touching using Windows Touch Messages.

When functions such as nn::hid::GetTouchScreenState() read the touch status using Windows Touch Messages, the nn::hid::TouchState::deltaTime value does not consider the report rate and becomes a simple sampling interval.

Based on this, if the touch moving speed is calculated using this value, a value around the actual speed may be obtained as a result.

ui2d Bugs

Cannot use Render in two cycles under Special border format in the Text Box Pane.

In this package, the runtime does not support the Text Box Pane > Special border format > Render in two cycles option, so it cannot be used.

Alpha Test does not work.

Alpha Test does not work in this package.

This problem is expected to be fixed by the next release.

Texture display cannot be performed correctly for LayoutViewer in some graphics driver environments.

In some graphics driver environments, the textures in LayoutViewer are not displayed correctly. (We have seen this bug in the Quadro2000 driver 341.05 environment.)

3D Bugs

The G3dDemo Edit demo is not displayed correctly in Visual Studio 2013 Release build.

When you run the Visual Studio 2013 Win32/x64 Release build for the G3dDemo Edit demo, the aspect ratio is not calculated correctly when rendering, and the display of the demo is corrupted.

Either set the compiler optimization options in the Visual Studio project file properties to disabled (/Od), or build in something other than Visual Studio 2013 Release build.

This bug will be fixed in an upcoming release.

Material animation files exported by the NintendoWare 3D tool have the same name.

The NintendoWare 3D Binary Converter has restrictions against converting shader parameter animations (FSP files), texture pattern animations (FTP files), texture SRT animations (FTS files), material visibility animations (FVM files), or color animations (FCL files) with the same name simultaneously. However, the Maya Export Plug-In and 3dsMax Export Plug-In export intermediate files with the same name as the models.

Also, when creating a new animation for a model from 3DEditor, the default name is the same as the model.

When collecting these animation files with the same name into one binary file for conversion, change the names of the intermediate files before converting.

This bug will be fixed in an upcoming release.

An assertion failure occurs when an application with nn::g3d::viewer embedded and 3DEditor are connected, and the sample parameter filtering settings are edited in 3DEditor.

An assertion failure occurs in nn::g3d::viewer when an application with nn::g3d::viewer embedded and 3DEditor are connected, and the sample parameter filtering settings are edited in 3DEditor.

This bug will be fixed in an upcoming release.

An assertion failure occurs when an application with nn::g3d::viewer embedded and 3DEditor are connected, and the camera animation parameters for a scene animation are edited.

An assertion failure occurs in nn::g3d::viewer when an application with nn::g3d::viewer embedded and 3DEditor are connected, and the camera animation parameters for a scene animation are edited.

This bug will be fixed in an upcoming release.

3DEditor sometimes crashes when changing the platform.

When the platform is changed while the Property Window is open in 3DEditor, 3DEditor sometimes crashes.

Close the Property Window before changing the platform.

This bug will be fixed in an upcoming release.

gfx Bugs

In the GL version, textures with a width that is not a multiple of 4 bytes are not initialized correctly.

When initializing textures in the GL version, if the width is not a multiple of 4 bytes, the texture is not initialized correctly, and the content is corrupted and initialized.
For example, for a texture with image format nn::gfx::ImageFormat_R8_G8 and width 7, the width is 14 bytes, so this problem occurs.

This bug will be fixed in an upcoming release.

Strings in a Non-UTF-8 Encoding Result in Corrupted Text When Used With the Log and Assertion API.

Strings in a non-UTF-8 encoding result in corrupted text when used with the log and assertion API.

Use the following when using strings in a non-UTF-8 encoding.
Considerations for Log and Assertion Character Codes

The alphabet list is not displayed correctly in functions such as API references.

In API references, under Namespace > Namespace Members, the alphabet lists displayed under All, Functions, and Enumerated Values do not display the correct lists when selected.

Jumping to change log details does not work.

In the change log list, when you try to click each history to jump to the details, it may jump to the wrong location.

This problem does not occur with Internet Explorer.

Sample demo batch processing fails when () is included in the path for the NintendoSDK.

Batch processing fails when building the sample demo that performs batch processing if () is included in the path to the NintendoSDK. You can avoid this problem by not using () in the NintendoSDK path.

Response is slow when using the mouse wheel to scroll on the Nintendo Dev Interface Environment Structure screen.

We have received reports that the response is slow when using the mouse wheel to scroll on the Nintendo Dev Interface Environment Structure screen in some PC environments.

If this problem occurs, please use the scrollbar instead of the mouse wheel to scroll.

Sometimes there is no response when deleting an environment in Nintendo Dev Interface.

When the separately installed Nintendo TargetManager is running, environment deletion processing may stop running.

You can avoid this problem by closing Nintendo TargetManager.

Applications started from DevMenu or DevMenuCommand cannot be closed with Visual Studio or RunOnTarget.

Applications started from DevMenu or DevMenuCommand cannot be closed with Visual Studio or RunOnTarget. To close these applications, you must disconnect and reinsert the power cable to restart the hardware.

This problem will be fixed in an upcoming release.

When an application is uninstalled from DevMenuCommand while that application is running from DevMenu or DevMenuCommand, it becomes impossible to install applications with the same Application ID.

When an application is uninstalled from DevMenuCommand while that application is running from DevMenu or DevMenuCommand, it becomes impossible to install applications with the same Application ID. To recover from this state, use InitializeSdev on the hardware to reinitialize it.

This problem will be fixed in an upcoming release.
