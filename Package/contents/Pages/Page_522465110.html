What is Exhibition mode?

Exhibition mode provides features that assume exhibition of an application.

  • When the development hardware power is turned on, an application is automatically started according to preset user-selected settings.
    • Settings are possible within the Application page in DevMenu only when exhibition mode is enabled.
  • Only screens associated with the application to be exhibited are displayed.
    • The DevMenu screen is not displayed (except when the application cannot start due to insufficient setup).
    • Only the following screens can be displayed.
      • The waiting for demo screen before the application starts
      • Screens while the application is running
      • After the application ends, the Demo exited screen
    • The HOME button is disabled.
    • Overlay displays other than “Power status monitor" and “Volume level" are disabled.
  • Automatic termination of the application is supported.
    • Play timer
    • Controller Idle Timer

To enable exhibition mode, either set the corresponding item (Launch Mode Settings > Exhibition Mode) on the DevMenu Debug page or run the DevMenuCommand exhibition command.

DevMenuCommand exhibition set-exhibition-mode-enabled <true|false>

Features

The following features can be used when Exhibition mode is enabled.

Starting an Application

Automatic Application Startup

A target application set in advance can be automatically started after the development hardware power is turned on.
If a waiting for demo image has been set, the waiting for demo image is displayed without starting the application.

The application to be automatically started can be set by pressing the Y Button in the application list on the Application page.
To change the application that starts automatically, turn on the development hardware and enter the following commands within one to four seconds after the Nintendo Switch logo disappears.
If the command is entered correctly, the auto-start is canceled and the DevMenu screen is displayed.

Controller Command to Enter
Joy-Con Press the ZL Button, the R Button, and the Down Button in the directional buttons at the same time.
Classic Controller Pro Press the ZL Button, the R Button, and the Down Button on the +Control Pad at the same time.

Pro Controller

For applications that require selecting an account at startup, at least one account must be created ahead of time.

Waiting for Demo Screen

A Waiting for demo screen can be displayed without auto-starting the application.
Only PNG format (1920 × 1080, RGBA32) is supported for image files displayed in the waiting for demo screen.
Press any of the following buttons in the waiting for demo image to start the application.

  • A, B, X, Y Button
  • L/R, ZL/ZR Button
  • -/+ Button
  • Directional buttons

By default, no waiting for demo image is set.

Save Data at Startup Snapshot

You can take a snapshot of the save data at startup for an application that is automatically started.
The following snapshot formats are available.

  • Include existing save data from all save data for the application.
  • Include existing save data only for the user selected at startup.

You can also delete all save data for the application by adjusting the settings.

Secure Startup

Info

If new secure startup features have been added, you must initialize the development hardware and update the firmware using the SystemUpdater package.
For more information about the cases where this requirement applies, see Managing Applications > Secure Startup.

Secure startup is a method of startup that reduces the risk of your application data being leaked if the development hardware is stolen.

If you install an NSP file for secure startup on the development hardware and set that file as the application that starts automatically, you can require a passphrase to be entered when the application starts.
The data contained in the application is decryptable and the application can be started (in the secure startup state) only if the correct passphrase is entered.
The secure startup state is maintained until the development hardware is powered off. If the development hardware is shut down and restarted, the passphrase must be entered again.

The secure startup state in exhibition mode has the following characteristics. These characteristics help prevent content from being stolen in a decryptable state.

  • Cannot put development hardware to sleep.
    • If a normal sleep operation such as tapping the POWER Button is performed on the development hardware, it shuts down instead of going to sleep.
  • Cannot restart the development hardware
    • If you attempt to restart the development hardware using the Reboot button in DevMenu, it shuts down instead of restarting.
  • The development hardware shuts down if it shifts from being charged with a genuine AC adapter to not being charged.
Note

Use this feature only in TV mode. In handheld mode, changes in the charging status cannot be detected under certain circumstances.

For more information about creating NSP files for secure startup, see the following documentation.

  • Features > Building > Creating Application Images > Creating NSP Files for Secure Startup.
  • Features > Building > Creating Application Images > Tool Reference > AuthoringTool > Creating NSP Files for Secure Startup.
Info

The maximum amount of content that can be installed to the development hardware is as follows.

  • Maximum of 10 (earlier than NintendoSDK Firmware for NX 10.1.0-3.0)
  • Maximum of 96 (NintendoSDK Firmware for NX 10.1.0-3.0 and later)
    • Downloadable content is counted by index.
Note
  • For versions earlier than NintendoSDK Firmware for NX 10.1.0-3.0, you cannot use secure startup for NSP files for patches or downloadable content.
  • The information contained in the application management data is what is passphrase-encrypted in an NSP file for secure startup.
    • Elements under the Application tag in NMETA files such as the title or icon correspond to application management data.
    • To protect information in the application control data, use the AuthoringTool replacenspmeta command or AuthoringEditor to change values in the original NSP file to an extent that does not impact the behavior of the application before you make the secure startup version of the NSP file.
  • Save data created by an application is stored in the same way as during normal startup. It is not related to the encryption for secure startup.

Terminating the Application

Demo Exited Screen

A Demo exited screen can be displayed after the application terminates.
Only PNG format (1920 × 1080, RGBA32) is supported for image files displayed in the Demo exited screen.
A minimum display time (2.5 seconds) and timeout time (5 seconds) are set for the demo exited screen.
After the minimum display time has elapsed, you can end the display by pressing any of the following buttons.

  • A, B, X, Y Button
  • L/R, ZL/ZR Button
  • -/+ Button
  • Directional buttons

By default, no demo exited image is set. If no demo exited image has been set, the application is restarted.

Play Timer

If the play timer is enabled, the application can be made to automatically terminate after a set time (in seconds) has passed since starting the application.
The demo exited screen is always displayed if the application is ended by the play timer. If no demo exited image is specified, a default image is shown.
The timer continues to count even while the development hardware is in sleep mode.

Controller Idle Timer

If the controller idle timer is enabled, the application can be terminated when a state in which there is no input from the controller continues for a set time (in seconds).
The timer continues to count even while the development hardware is in sleep mode.

Info

The lack of input from the sticks is not used to determine whether the controller is idle. This design accounts for situations where the controller is left unattended with a stick in contact with the exhibition stand.

Forced Close

You can forcibly close a running application with any of the following operations.

  • Hold down the POWER Button for three seconds.
  • Hold down the HOME Button for 0.5 seconds and then, within 10 seconds, simultaneously press the A Button, Y Button, + Button, and ZR Button. (For versions of the NintendoSDK Firmware for NX earlier than 10.0.0-4.0, you must hold down the HOME Button for up to two seconds.)

The Demo Exited screen is not displayed if the application has been forcibly closed.

Unprompted Exits

Implement this feature according to the procedure described in NintendoSDK Documents > Features > Program Flow > Exiting the Program.

Setting Method

Exhibition mode settings can be made in DevMenuCommand in the DevMenu. For more information about configuring the setting using DevMenuCommand, see DevMenuCommand > Exhibition Features.

Settings for Auto-Startup Applications

Highlight the application to be automatically started in the applications list on the Application page and press the Y Button to set that application to auto-startup.
The user selection screen appears when targeting applications that need user selection at start.

Option Settings

On the Application page, click Exhibition Settings to transfer to the Option Settings screen.
An auto-startup application must be configured in advance to use the Option settings.
The following operations can occur on this screen.

  • Confirmation of the name of the application configured for auto-startup
  • Confirmation of the snapshot status for fixed save data
  • Various option settings

Startup User

The user loaded when starting an application can be changed.
(not required) appears for applications that do not need user selection at start.

Playable Time

You can enable the play timer and configure how much time is available for an application's demo. When the configured time elapses, the application automatically ends.
The default is 0 (disabled). The playable time can be set to any time (seconds) by changing the Playable Time.

Images Before and After Play

You can configure images to use before and after the demo, and can remove them. These image files are stored in the Exhibition mode save data (0x010000000000212F).
Click the Edit button to display a list of PNG images saved to the microSD card.

Tab Name Category
Waiting for play image Waiting for demo image
Play exited image Demo exited image

Setting Image Files to Display

  1. Highlight the PNG image on the microSD card inserted in the development hardware displayed on the screen and press the A Button to confirm.
  2. Select the type of image to set and click Import to confirm.
Note

The PNG file format is successfully set, even if it is not in the supported format (1920 × 1080, RGBA32).
Make sure the images appear as expected after configuring them.

Deleting Image Files to Display

  1. Determine the images to delete in Delete imported image.
  2. Select the type of image to delete and click Delete to confirm.

Idle Detection Time

You can change the enabled state for the controller idle timer.
The default is 0 (disabled). You can change the configuration of Idle Detection Time at any time, in seconds, to determine when a controller is idle.

Stored Save Data Snapshot

You can select the type of the fixed save data.

Item Description
Not apply This is the default. Maintains the save data status without taking a snapshot.
Apply to a startup use only Includes the save data stored in the snapshot save data area for only the user selected at startup.
This item cannot be selected if a user is not selected at startup in the settings for an application to be automatically started.
Apply all data Includes the save data stored in the snapshot save data area for all user save data for the application.

Storing Save Data in the Snapshot Save Data Area

  1. Highlight the save data for the application to be automatically started on the Save Data page and press the Y Button.
  2. Confirm that all save data for the application has been copied and saved to the exhibition mode save data (0x010000000000216D).

Deleting Save Data at Startup

If no save data will be stored in the snapshot save data area, start the application after deleting the user save data targeted for snapshot.
Set either Apply to a startup use only or Apply all data, and then confirm that the Exhibition mode save data contains no user save data.

Play Data Output

You can use the playdata command of DevMenuCommand to collect play data when exhibiting an application.
For more information, see DevMenuCommand > Getting the Play Activity
