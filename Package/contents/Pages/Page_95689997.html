Introduction

About This Page

This page includes the following information about the property sheets provided by the NintendoSDK build system.

  • Directories where the property sheets are installed and features of each property sheet
  • List of properties referenced and provided when these property sheets are built

Notation

There are places in this document where $(NintendoSdkRoot) is used to denote the directory where NintendoSDK is installed.

Property Sheets

Property sheets provided by NintendoSDK are located in the following directories.

Directory Path Description
$(NintendoSdkRoot)\Build\Vc Contains the property sheet that determines the build settings for NintendoSDK. The file structure in this directory may change with future NintendoSDK releases. Do not directly import the property sheet in this directory to the project, except when there are specific instructions to the contrary.
$(NintendoSdkRoot)\Build\VcProjectUtility This location contains the property sheet that projects import using the NintendoSDK build system. Developers can freely import the property sheet located in this directory to a project under development.

The following types of property sheets are located in the VcProjectUtility directory.

Filename
Description
ImportNintendoSdk.props Import the other property sheets that are required for using NintendoSDK. These sheets are at $(NintendoSdkRoot)\Build\Vc. The properties cited in Properties Referenced by the Build System must be defined ahead of time.

Properties List

Properties Referenced by the Build System

The following properties must be properly defined before importing NintendoSDK property sheets.

ConfigurationType

Used to determine the binary type of the build target. The value of this property must be one of the following.

  • Application
  • StaticLibrary

The following values may also be used. However, this is limited to cases of using the NX Add-On package when the properties Platform value is NX32 or NX64.

  • DynamicLibrary

NintendoSdkBuildType

Used to determine the build type of the NintendoSDK library in use. For more information, see Build Targets and Build Types.

The value of this property must be one of the following.

  • Debug
  • Develop
  • Release

NintendoSdkRoot

Used to get the root path of the NintendoSDK to use. The value set for this property must be the path to the directory where NintendoSDK has been installed.

NintendoSdkSpec

Used to determine the API set of the NintendoSDK in use. If the project is using shared packages, you must set this property to Generic to indicate the general-purpose API set.

If the project is using the NX Add-On packages, you can set this property to NX to indicate the NX API set. When building for the NX development hardware environment or its Windows compatible environment, select NX.

Platform

Used to specify the current build target platform. When using a shared package, the value of this property must be one of the following values.

  • Win32
  • x64

In addition, you can use the following property values when using the NX Add-On package. These platforms build the respective 32-bit and 64-bit binaries for the NX development hardware environment.

  • NX32
  • NX64

PlatformToolset

Used to determine the version of the Visual C++ tool chain in use, only when the build uses Visual C++. The value of this property must be one of the following.

  • v141
  • v142

Properties Provided by the Build System

The developer can reference the following properties in the Visual Studio project.

NintendoSdkBuildTarget

Identifier indicating the binary type of the library or application to build. For more information, see Build Targets and Build Types.

Values for the NintendoSdkBuildTarget property are used by some of the NintendoSDK include directory paths and library directory paths.

NintendoSdkDllSuffix

The suffix that must be appended to the library name when searching a dynamic library.

Normally, this property is the extension, including the period, of the dynamic library file. For example, it would be .dll when building for the Win32 and x64 platforms.

NintendoSdkIncludePath

Lists the NintendoSDK include directories.

NintendoSdkLibraryPath

Lists the NintendoSDK library directories.

NintendoSdkLibSuffix

The suffix that must be appended to the library name when searching a static library.

Normally, this property is the extension, including the period, of the static library file. For example, it would be .lib when building for the Win32 and x64 platforms.

NintendoSdkNXAddonVersionMajor

Major version number of the NintendoSDK NX Add-On.

NintendoSdkNXAddonVersionMinor

Minor version number of the NintendoSDK NX Add-On.

NintendoSdkNXAddonVersionMicro

Micro version number of the NintendoSDK NX Add-On.

NintendoSdkVcRuntimeLibrary

Represents the type of Visual C++ runtime used by NintendoSDK. It is only defined when the program was compiled using Visual C++.

Values are the same as the RuntimeLibrary metadata of ClCompile. This property takes one of the following values.

  • MultiThreadedDLL
  • MultiThreadedDebugDLL

NintendoSdkVersionMajor

Major version number of the NintendoSDK.

NintendoSdkVersionMinor

Minor version number of the NintendoSDK.

NintendoSdkVersionMicro

Micro version number of the NintendoSDK.

Other Properties

Properties not included in this document are defined in the NintendoSDK property sheet. These properties are used internally by the build system. Avoid referencing them. Normally, these property names are prefixed with either NintendoSdk or _NintendoSdk.
