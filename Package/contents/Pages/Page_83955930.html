Layout Binary Converter

Introduction

The Layout Binary Converter (LayoutConverter) is a command-line tool to convert layout data files to binary files that are Layout library resources. This document describes how to use the Layout Binary Converter and the features that it offers.

Using the Layout Binary Converter

Executing the Layout Binary Converter

The following two methods are available to use the Layout Binary Converter.

  • From a command line, by executing a LayoutEditor save file (also called a layout data file).
  • From LayoutEditor

We do not recommend using Layout Binary Converter from LayoutEditor, because that method is a more experimental one for developers. For more information, see the LayoutEditor documentation.
The following section describes the command-line method.

Executing From the Command Line

When using the Layout Binary Converter from the command line, it executes in the following format.

  • LayoutConverter.exe [options] input layout data file (multiple files permitted) output directory
  • LayoutConverter.exe [options] input directory output directory

Example:

LayoutConverter.exe layout.flyt animarion.flan outDir 
LayoutConverter.exe -u -g inDir outDir

Multiple input layout data files can be specified.

In addition, when specifying an input directory, the layout data files in the specified directory are found and used as input layout data files.

When layout data and animation data are in the same directory, binary conversion can be performed in a batch without specifying each type of file.

When FCPX files are specified as the input, the converted BFCPX files are output to the output directory.

Example:

LayoutConverter.exe multi.fcpx outDir

FLYT and FLAN Files

We recommend saving FLYT and FLAN files in the same folder and giving them the same name for the following reasons.

  • When converting FLAN files, the converter looks in the same folder for FLYT files with the same name. If it can find a corresponding FLYT file, it only exports animations that are actually used.
  • The converter only exports textures that are actually used and does not work correctly for texture pattern animations if there are no corresponding FLYT files. (To avoid conversion issues, you must specify the cvtr-all-tex option.)

Layout Binary Converter Options

LayoutConverter.exe options are specified using a combination of hyphens and characters. When specifying two or more options such as -u and -g, they must be specified as separate arguments such as -u -g rather than -ug. The following table shows the option types.

For more information about each option, see the next section.

-h --help

Display a Help message and exit.

--tile-mode <arg>

Specifies the target tiling mode to use when converting FCPX files using FFNT.

Specify linear when exporting a binary for Windows or OpenGL.

For more information about exporting a binary for platforms other than Windows or OpenGL, see Platform-Specific Information.

--api-type <arg>

Specifies the target rendering API type. This option will be deprecated. Use LayoutArchiver's --api-type and --code-type options instead.

--code-type <arg>

Specify the target shader code type. This option will be deprecated. Use LayoutArchiver's --api-type and --code-type options instead.

-p --platform <arg>

Specify a target platform name. This option will be deprecated. Use LayoutArchiver's --api-type and --code-type options instead.

-u

Only files with changed content are updated.

-g

The animation binary file is output in units of animation section tags.

--tag-split-and-whole

Generates animation binary files in units of animation section tags, together with an animation binary file that contains the entire animation.

--no-convert-cvtrchar

Hexadecimal strings of ^ + 00 through 1F in a text box string, are not converted to characters in the character code range of 00 through 1F.

--discard-textboxtext

Text box strings are not output to the binary. If Restrict String Length is enabled, the specified buffer is allocated.

--discard-muted-animation

Muted animations are not exported to the binary.

--omit-samekey

This option is enabled when the -g option is specified. Excluding the first animation segment tag, when the value in the frame segment specified with the animation segment tag does not change (between the start and end time of the interval), the key is not generated.

--omit-samekey-all

This option is enabled when the -g option is specified. When the value in the frame segment specified with the animation segment tag does not change (between the start and end time of the interval), the key is not generated.

--omit-nokey-all

This option is enabled when the -g option is specified. When no key is in the frame segment specified (between the start and end time of the interval) using the animation segment tag, no animation data is generated.

--bake-infinity

When the values outside the key frame segment are not "constant" values, repeatedly generate the key so that the curve is reproduced.

--no-taginfo

The file does not contain animation segment tag information. Enabled when exporting in units of the animation segment tag.

--no-check-version

The version of the layout file is not checked. When this option is set, the layout library might not run properly, depending on the file version.

--cvtr-all-tex

Processes and exports all texture files registered in the FLTY texture list. If this option is not specified, only the texture files actually used in the material or texture pattern animation are processed.

--no-copy-font

During conversion, the font resource is not copied to the destination. This option is enabled when the font is not inserted in the layout archive and is managed separately.

--no-copy-font-name <arg>

At conversion time, the specified font resource is not copied to the destination. Omit the filename extension when you specify the font file. Multiple fonts can be excluded by specifying this option multiple times.

--extra-frame-for-onetime

The one-time animation frame count is set to "1 + the value specified by the animation segment tag." There is an issue with this. The animation may not stop on the final frame if the number of frames is different from the looped animation and the final frame does not have a key. This issue also occurs in NW4R and NW4C.

--cvtr-all-anim-content

For animation tags specified with an associated group, the animation is output even for a pane where the animation target is not included in the associated group. If this option is not specified, only animation of panes that are included in the group are output for animation tags for which an associated group is specified. Provided for compatibility with the behavior in past versions.

--allow-nogroup-animtag

Allows the use of animation tags not specified by the related group. This option is ignored when cvtr-all-anim-content is specified. Provided for compatibility with the behavior in past versions.

--degamma-texture

Converts the texture color space from sRGB to linear. Only RGB is converted and the alpha value is left alone. Used when you want to draw a layout using a linear color space. However, it does not convert any textures that are used indirectly. The colors of indirectly used textures are actually displacements, so they become difficult to use if converted as colors. Specifically, textures are considered to be used indirectly in the following cases.

  • Textures that are being used in a layer specified as indirect [distortion source = self, object = previous stage] in the texture combiner.
  • Textures that are being used in a layer specified as indirect [distortion source = self and mixing result of previous stage, object = previous stage] in the texture combiner.
  • Textures that are being used in a layer after a stage specified as indirect [distortion source = self and mixing result of previous stage, object = previous stage] (3rd stage) in the texture combiner.

--degamma-parameter

Converts a black-white corrected color space, or a vertex color space, from sRGB to linear.
Used when you want to use a linear color space for display, but also want to enter sRGB gamma values in the tool.
When this option is used, the values entered by the designer in the LayoutEditor and the values obtained from the pane parameters at run time are different. We do not recommend using this option because of this type of complexity.

--degamma-vector-graphics-color

Converts the colors within a vector graphics scene from sRGB to linear when vector graphics textures are used within a layout scene.
Specify this option when layouts are used in a linear color space.

--gpu-encoding <type>

Specifies whether to perform encoding in the GPU when converting textures.
For type, specify true, false, or auto. If omitted, the default is true.

The specified option is passed to the internally called 3D texture converter.
For more information, see gpu-encoding in the 3D texture converter manual.

--mesh-trimming-optimization <type>

Applies mesh trimming optimization, analyzing the textures for a picture pane and trimming the meshes to create shapes where pixels that are completely transparent are not rendered, thereby reducing the drawing area.
For more information about mesh trimming optimization, see here.
For type, specify one of the values shown in the following table.

type Usefulness
auto Apply all available optimization types in each picture pane and automatically select the type that reduces the pixels to draw by the most.
AABB

Use AABB to create the optimized mesh for all picture panes.
OBB

Use OBB to create the optimized mesh for all picture panes.

--mesh-trimming-optimization-margin <size>

Sets a fixed margin, in pixels, for the size of the mesh that is created by mesh trimming optimization.
Mesh trimming optimization is performed on the textures that are set as layout data input, so when compressed textures and the like are later converted, there will be cases where transparent pixels become opaque and stick out from region of the optimized mesh.
This option can help to avoid this problem by enlarging the size of the mesh by a few pixels.

--output-optimization-statistics

Generates a report about the optimization of the layout data.
When this option is specified, statistics about the results of optimizing all layout data below the input directory is saved to a file called OptimizationReport.html in the specified directory without converting the layout data.
If the --mesh-trimming-optimization option is specified at the same time, information about mesh trimming optimization will be included in the optimization results.

--skip-if-srctexture-not-exist

Not recommended. Skips the conversion process without reporting an error if the conversion source texture file cannot be found. We no longer recommend this option as of version 1.3.0 of the binary converter. When this option is specified, conversion is confirmed to fail in window panes where the source texture file does not exist. Use a method to convert in a state where textures exist and then delete the unnecessary textures from the post-conversion file.

-s --silent

Displays only error messages.

--ftxb-cache-directory

Caches the FTBX file that is being created in the specified directory. Layouts that use textures are converted faster on the second and subsequent occasions.

--no-output-layout-anim-tag-name

Suppresses the output of the animation segment tag names linked to the layout.

--args-file <arg>

Specifies the file. You can use the contents in the parameter file to specify arguments. The parameter file allows you to specify each option over several lines. Lines starting with a number sign (#) are treated as comments. Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--keep-font-scale

When this option is specified, the font scale is maintained even if the font file is switched after conversion of the BLFYT file. Use this option when you want to switch fonts depending on the language setting.

--reference-color-directory <arg>

Specifies the directory containing the reference color file (FLRC file). When converting a layout using reference colors, this folder and its subdirectories are searched for the reference color file. Multiple directories can be specified by specifying this option multiple times.

--vector-graphics-unsupported-function-error

If you specify --unsupported-function-error when converting vector graphics data used in the layout data, an error occurs if unsupported features are used in the vector graphics data.



Layout Binary Converter Features

Convertibility to the Layout Binary File

The extension for the layout data file is related to the extension of the layout binary files as shown in the following table.

Table 4-1. Extension Relation

Extension of the Layout Binary File Created After Conversion Layout File to Convert Using the Layout Binary Converter
.bflyt flyt
.bflan flan

Resource Type

The Layout Binary Converter creates a directory for each resource type in the destination directory and saves the layout file after binary conversion, and the fonts and texture images used for the layout.

Table 4-2. Resource Types and Save Directory Names

Resource Type Filename Extension Save Directory Name
Layout data .bflyt blyt
Animation data .bflan anim
Font resource .bffnt font
Texture image FTXB timg
Composite Font .fcpx .bfcpx
  • The Layout library gets resources according to these directory names. For that reason, the converted files must be saved in the directory corresponding to the type.
  • Files with the .ftxb extension are image files converted from TGA files.
    • Set the TGA file as a format supported by the layout editor.
    • Ultimately FTXB files will be combined in LayoutArchiver into one BNTX file.
  • The scale, position, rotation, and color curves for per-character transformation in the text box pane are contained in the BFLYT file, and not in the BFLAN file.

Storing Names in Resources

As with the layout data file, the layout binary file stores the names of panes and groups. The Layout library looks at these names, searches for data, and applies animation.

However, texture and other resource names in the layout binary file must be unique. Because converted files are saved for each resource type, even if the save location before conversion is different, only the most recently converted file remains for resources that are of the same type and have the same name.

However, as an exception, texture images have a unique character appended to the filename for each format. If the same texture is used as a different format (for example, A8 and L8), it is output with different filenames.

Binary Conversion of Animation Data

Animation Data Format

When converting a layout data file to a layout binary file, the data format of the animation that is saved changes depending on the animation type.

Step Format

Saves key data that is a set of (frame, value).

frame is saved as a single precision floating decimal, and value is saved as a unsigned 16-bit integer.

Hermite Interpolation Format

Saves key data that is a set of (frame, value, slope).

frame, value, and slope are all saved as single precision floating decimals.

Animation Types and Data Formats

The animation type determines whether animation is saved in step format or Hermite interpolation format.

Table 4-3. Animation Types and the Data Format to Save

Animation Type Data Format
Pane SRT Animation Hermite Interpolation Format
Visibility Animation Step Format
Vertex Color Animation Hermite Interpolation Format
Material Color Animation Hermite Interpolation Format
Texture SRT animation Hermite Interpolation Format
Texture Pattern Animation Step Format
Indirection SRT Animation Hermite Interpolation Format
Alpha Compare Animation Hermite Interpolation Format

File Conversion Options

-u

By specifying the -u option, a binary comparison of the file after conversion and the existing file is performed, and the file is not updated if the contents are the same.
This specification enables you to prevent a careless change in the file time stamp.

-g

By specifying the -g option, an animation binary file is created for each tag struck in an animation.
This specification is used when extracting and using a segment from a sequence of animations.

Naming Rule

The following naming convention is used when exporting the animation's binary file: Layout name_Animation segment tag name.

--no-convert-cvtrchar

When the --no-convert-cvtrchar option is not specified, a string made up of ^ followed by two hexadecimal characters in the range of 00 through 1F in a text box is converted to a character with the corresponding character code (00 through 1F). If this option is specified, this conversion is not performed.

--omit-samekey, --omit-samekey-all, --omit-nokey-all

The --omit-samekey, --omit-samekey-all, and --omit-nokey-all options are enabled when the -g option is specified.

(We recommend specifying -- omit-nokey-all. For more information, see this page.)


Normally, when even a single key is set for the entire frame segment, the single key is generated even if the values do not change in the frame segment specified with the animation segment tag (between the start and end time of the interval).

If the --omit-samekey or --omit-samekey-all option is specified, the key is not output if there is no change in the value in the frame segment specified with the animation segment tag. However, for the --omit- samekey option, the first animation segment tag in the animation segment tags with Output Binary selected always exports the key.

If the -- omit-nokey-all option is specified, when no key is struck in the frame segment specified with the animation segment tag, no animation data is output. This case ignores whether the value changes. To play an animation for the number of frames in the animation segment tag without having the animation change midway, you can reduce the size of the animation binary by specifying these options.

--bake-infinity

The --bake-infinity option exports by repeating the key so that the curve is reproduced when outside the key frame segment is something other than a constant. Note that the size of the output animation binary file is increased by the portion being repeated.

--no-taginfo

This --no-taginfo option is enabled when the -g option is specified. Normally, when exporting a binary file for each animation segment tag, information about the animation segment tag is included in the binary file. When this option is specified, the animation segment tag information is not included in the binary file.

--no-check-version

When --no-check-version is specified, the version of the layout file is not checked. Normally, the version of the layout file is checked and processing is ended if the version is not one that the binary converter recognizes. When this option is set, the layout library might not run properly, depending on the file version.

--cvtr-all-tex

When --cvtr-ref-tex-only is specified, all texture files registered in the layout file (the FLYT file) are converted. When this option is not specified, of the texture files registered in the layout file, only the texture files that are actually used with the material or texture pattern animation are converted. When this option is not specified, if texture pattern animation is being used in the FLAN file, the corresponding FLYT file must have the same name and be in the same folder as the FLAN file.

-no-copy-font

When -no-copy-font is specified, the font resource is not copied to the destination during conversion. Because fonts become larger with Cafe, if each layout archive has a copy, a large amount of memory is consumed. This option is useful when fonts are managed separately from the layout archive to prevent this problem.
