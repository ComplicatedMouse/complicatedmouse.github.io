This section details the features of the gfx library.

New Features and Legacy Features

Each new generation of low-level graphics API brings several major changes, including the addition of brand-new concepts.

The gfx library provides support for these new methods while also maintaining support for legacy interfaces to assist with migration and for cases where the program does not need to run on a new low-level graphics API. Note that while the earlier methods might not run on a new low-level graphics API, Nintendo strives to maintain compatibility with all low-level graphics APIs when using the new methods. In other words, when using the new methods on old low-level graphics APIs, emulation takes place.

Compatibility Between New Features and Legacy Features

The following table lists new features and the corresponding legacy features.
Some variations of low-level graphics APIs will not work with legacy features.

New Features Legacy Feature
Memory Pool Create objects without using a memory pool.
Descriptor Pool Set resources directly in the command buffer.
Resource State Flush and invalidate the cache.
Pipelines Shader and various states.

Alpha Tests

The alpha test feature is not currently available via the gfx library. You can perform something similar to an alpha test. Discarding fragments using discard in the pixel shader in combination with a low-level graphics API that has an alpha test feature can work as a substitute.
