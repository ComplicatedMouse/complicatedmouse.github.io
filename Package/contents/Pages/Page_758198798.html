NX Options Tab

This tab contains options specific to development for NX.

Directories

In the case where the build environment differs from the profile environment, you can specify the search paths for the profiler to use. For example, if a build server makes the application to be profiled, the profiler looks for files based on the location of the build server and not on your local computer. In this case, you'll need to set the search directories so the profiler can find the correct files.

NSO Files

The NSO Files directory is the local location containing the NSO and NSS files that were linked against when building the application. For example, if a build server is used, download the NSO and NSS files onto your local computer and set the NSO Files directory to this location. This directory is added to the beginning of the search path. The remaining directories that the profiler searches will still be searched.

Directory Search Order

  1. The NSO Files directory specified here.
  2. The directories pulled from the path specified to the linker for an NSO or NSS.
  3. The NintendoSDK library directories.
  4. The current directory.
  5. The directories pulled from the NSO running on the dev kit.

Source Code

The source code directory is the common root location where source code is stored. For example, the directory where the source repository is checked out on your local computer. The source code directory is currently only used to interleave source code on the Assembly Tab.

To find the source file, the profiler makes use of a source code lookup tool. Once the profiler has a path, if there is no source code directory specified the profiler will just attempt to use that path. If a source code directory is specified, the profiler will first attempt to find the file in that source code directory.

Search Method

  1. Get just the file name from the path.
  2. Search the source code directory for that file, and if found, return the file location.
  3. If the file is not found, add in the next parent directory from the original path.
  4. Repeat 2 and 3 until the file is found or the whole original path has been used.
  5. If the file is not found, return the original path.

Assembly

Settings in this section relate to the Assembly Tab.

Source Code Lookup Tool

This is the tool used when interleaving source code is enabled. There are two options:

  • addr2line.exe
  • llvm-symbolizer.exe

The default setting is currently addr2line.exe.

Info

If you are using NX Addon 9.x or later, is recommended that you switch this setting to llvm-symbolizer.exe. For earlier SDKs, it is recommended to continue using addr2line.exe.
