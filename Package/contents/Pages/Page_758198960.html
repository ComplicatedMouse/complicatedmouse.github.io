Start Tab

The Start tab is the first tab that appears when the profiler launches. It provides easy access to useful actions and information.

Quick Actions

The Quick Actions section provides easy shortcut buttons to actions you may want to take when first opening the profiler.

The buttons labeled Sync to a Devkit, Open a Profile, and Read the Manual behave the same way as the corresponding buttons in the Home Tab..

Recent Profiles

The Recent Profiles list displays a list of recently opened or saved profiles. The most recently accessed profiles appear at the top of the list.

Each recent profile shows its filename, date taken, and a screenshot. Click on a profile to open it.

Helpful Tips

The Helpful Tips section shows a short tip to help you learn new things about the profiler. A new tip will appear each time the profiler is launched or whenever you press the Next button.

Utilization

Info

This feature requires NX Addon 11.4.0 or later and Nintendo NX CPU Profiler 1.5 or later.

The Utilization section shows a live view of the CPU cores when you are synced with a dev kit. Until the Sync operation is completed, the section will remain empty. After the Sync operation is finished, the graphs appear and will update once a second with data streamed from the dev kit.


Within the view, there are three graph styles of note:

  • Application Cores
  • Individual Application Core
  • System Core

Application Cores

The Application Cores shows the combined CPU load of cores 0, 1, and 2. The individual values of each core are averaged together to provide a total CPU load.

Individual Application Core

Cores for application use (cores 0-2) display their real-time load. Use this information to determine if any of the cores are underutilized. Ideally the application is maximally using all of the application cores for the benefit of the user.

System Core

Unlike the individual application cores, the system core graph shows the impact your application has upon the system core. The values being shown are not representative of the CPU load itself. This graph is composed of three values: green, yellow, and red.

Green (Low Impact): If the core is only showing a green bar graph, your game's impact upon the core is low and no performance issues are likely to be seen.

Yellow (Medium Impact): If the core is showing a yellow bar above the green bar, your game has begun to have an impact upon the core. This impact can mean that some calls, including call for graphics, audio, networking, and file system, may take longer. There is more of a risk for noticeable issues appearing to users playing the game.

Red (High Impact): If the core is showing a red bar above both the yellow and green bars, your game is having a serious impact upon the core. At this point, some of the systems on the core, such as graphics, audio, networking, and file system, may be impacted. Users may notice issues with any of these systems while playing.

Please see the page Tracking Down High Core 3 Impact for more information on identifying and resolving high system core usage.
