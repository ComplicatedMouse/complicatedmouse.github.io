Overview

Switches shops and configures the active consoles that can download content.

How to Use

Switching Shops

Valid Shop

Use this setting to switch shops for development purposes.

When the value is set to Dev Shop, the dummy EC applet starts when a shop is selected.

When the value is set to Shop, an actual shop starts when a shop is selected.

The default is Dev Shop.

Registering an Active Console

Initial Settings and Registering an Active Console

Configures settings the first time a connection is made to the shop, and registers an active console that can download content. User accounts linked to a Nintendo Account are presented as the choices.

You can use this feature to execute a process that is run the first time the shop is accessed. The console that runs the process is registered as an active console that can download content.

Other settings besides the registration of an active console must be configured when the shop is accessed for the first time, so this process must be run once by one console.

Register an Active Console

Registers an active console that can download content. User accounts linked to a Nintendo Account are presented as the choices.

The registration process will fail if you specify a Nintendo Account for which another system already has been registered as the active console.

To change the active console, first unregister the system that is now registered as the active console and then register the new system as the active console.

Unregister an Active Console

Unregisters an active console. User accounts linked to a Nintendo Account are presented as the choices.

Info

You cannot unregister using the Application page of DevMenu.

Unregister All Active Consoles

Unregisters all consoles that have been registered as active consoles. User accounts linked to a Nintendo Account are the targets.

Info

You cannot unregister using the Application page of DevMenu.

Check the Active Console Registration Status

Checks the registration status of active consoles. User accounts linked to a Nintendo Account are presented as the choices.

If the console that ran the process is registered as an active console for the specified account, the following message is displayed.

This console is registered as an active console of the account user index [user account index number]

If another console is registered as an active console for the specified account, the following message is displayed.

Another console is registered as an active console of the account user index [user account index number]

If no console is registered as an active console for the specified account, the following message is displayed.

Unregistered

EC Response Simulation

This development support feature simulates a response from the shop service server.

For more information, see ec Library > Debugging > Response Simulation Feature.

JSON Compatibility Check

This development support feature can verify whether the application is handling JSON properly and whether there would be any issues with adding JSON elements in a future server update.

For more information, see ec Library > Debugging > JSON Compatibility Check Feature.
