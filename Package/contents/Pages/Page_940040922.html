Save Bytes

When Debugging Using VSI

When stopped on an assertion failure or stopped at a breakpoint, you can go to any of the 4 memory windows and in the toolbar, look for "Bytes to Save" section and type in the size of the section (in bytes) you would like to save and given the address in the toolbar, TM will spend some time to dump the memory into a filepath of your choice (the File Explorer opens up right after hitting the save icon button). 




  • Max size for the request is 4GB (4294967296 bytes).
  • If request is cancelled, the file that was being saved into, is deleted. 
  • Address must be valid number.
  • Dump does not consider the memory ranges, just starting point and size. 
  • Dump will save out the same number of bytes that is requested even if there is nothing in that range.
