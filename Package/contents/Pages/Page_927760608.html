cdr Library

Introduction

The core dump report (cdr) library is a library for performing nxdmp operations. It is installed at the C:\Program Files\Nintendo\Oasis\ location.
Information that can be obtained from NXDMP files by using this library includes exception information and descriptions, loaded modules, a list of threads, and the call stack for those threads.

Structure

The files libnn_cdr_x64.dll and libnn_cdr_x64.lib are installed at the C:\Program Files\Nintendo\Oasis\lib location.
Use the header C:\Program Files\Nintendo\Oasis\include\cdr.h.
The LIB file is an import library, so tools that use the LIB file must be able to reference the DLL at execution time.

We also plan to provide a .NET library in the future.

How to Use

Use From a C++ Application

This section describes what kind of TM libraries can be used from a C++ console application.

  1. From Project Properties > General > Additional Library Directories, add C:\Program Files\Nintendo\Oasis\lib\.
  2. From Project Properties > Linker > Input > Additional Dependencies, add libnn_tm_x64.lib;.
  3. Select Project Properties > Build Events > Post-Build Events > Command Line, and add copy "C:\Program Files\Nintendo\Oasis\lib\libnn_cdr_x64.dll" "$(OutDir)\".
  4. Include cdr.h and implement it.
  5. For more information about using each function, see the nn::cdr API reference and the sample.

Troubleshooting

Cannot properly get the source file, line number, function name, and other information as a call stack entry.

If the proper NSS file cannot be found, the call stack entry can only get the instruction pointer address. To get other values, you must use AddSymbolPath to specify the directory containing the relevant NSS file. If using NXSymStore, you may also use AddSymbolStorePaths to specify the symbol store path instead.
