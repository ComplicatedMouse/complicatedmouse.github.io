Documentation Guide

About This Documentation Guide

The documentation guide provides an introduction to the overall structure of NintendoSDK.

It also explains how to use the documentation.

Document Structure

List of Documentation for the Overall Package

Document Name Overview
NintendoSDK Overview Describes NintendoSDK.
Documentation Guide (this page) Describes how to use the documentation.
Guide to Setting Up the Environment Describes how to set up the development environment.

Tutorial

Describes various aspects of using the NintendoSDK for development.

It also includes a Quick Start section for those who want to run through the samples.
Features Describes the features provided by NintendoSDK in detail.
API Reference

Describes the API resources.

Do not use functions that are not included in the API Reference.
SDK Sample Descriptions Describes the SDK's sample programs.
Notes

Provides important notes and considerations for application development.

Please review this content before starting development.
Known Bugs Describes bugs that are planned to be fixed in a future version.
Troubleshooting Describes workarounds to try when specific problems are encountered.
In general, topics covered in Troubleshooting involve the specifications, so unlike the topics addressed in Known Bugs, they will not be fixed.
Revision History Provides a timeline of the changes made to the SDK.

Structure of the Feature-Related Documentation

The basic structure of the documentation provided for each feature is as follows.

Document Name Overview
Overview

Describes the main points.

Includes details intended for developers about the design and implementation of the application.
Manual Describes how to design and implement applications that use the system's features.
Sample Describes the source code and how to run the samples.

Reading the Documentation

Introduces how to read each type of documentation in various situations.

When You Want an Introduction to the Package

  1. NintendoSDK Overview
  2. Documentation Guide (this document)
  3. Guide to Setting Up the Environment

When You Want to Try Out a Specific Module

  1. Overview (for each module)
  2. Tutorial
  3. Sample documentation

When You Want to Check Something at the Beginning of or During an Implementation

  • Tutorial, sample documentation
    • You can learn by reading the documentation while running the samples and checking the actual behavior.
    • You can browse code fragments that provide a reference for the implementation.
  • Manual
    • You can check the detailed specifications.
  • API References
    • You can check the specifications for API functions.
