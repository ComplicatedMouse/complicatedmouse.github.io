Sound Parameter Initial Settings

Initial Settings File

You can configure the initial settings for sound parameters by editing the project_name.fspj.defaultparam initial settings file located in the same folder as the sound project file.

Outputting the Initial Settings File

  1. Open the sound project.
  2. Close the sound project.
  3. The project_name.fspj.defaultparam initial settings file located in the same folder as the sound project file is output.

If an initial settings file already exists, it cannot be overwritten.

Even if you delete the initial settings file, when you close the sound project, the initial settings file is output again.

Editing the Initial Settings File

The output initial settings file is written in YAML format.

All parameters are commented out, so remove the comments for the required parameters and configure the initial values.


  • Example: Setting the initial value for stream sound volume to 127.

"# Volume: 96" in the StreamSound: stream sound label is the editing target.

## Stream Sound
StreamSound:
　　:
　～～～
　　:
#  # Volume
#  Volume: 96

In the following code, remove the # starting comment out character and make the initial value 127.

## Stream Sound
StreamSound:
　　:
　～～～
　　:
#  # Volume
  Volume: 127
Note

Do not delete the space (blank character) at the beginning of the Volume: parameter. Indentation using a space is required for parameter recognition.


  • Example: Making the Wave Sound's Item Color Pink

"# Color: None" in the wave sound's WaveSound: label is the editing target.

## Wave Sound
WaveSound:
　　:
　～～～
　　:
#  # For the item color, specify None, Red, Orange, Yellow, Green, Cyan, Blue, Purple, or Pink.
#  Color: None

In the following code, remove the # starting comment out character and make the initial value Pink.

## Wave Sound
WaveSound:
　　:
　～～～
　　:
#  # For the item color, specify None, Red, Orange, Yellow, Green, Cyan, Blue, Purple, or Pink.
  Color: Pink
Note

Do not delete the space (blank character) at the beginning of the Color: parameter. Indentation using a space is required for parameter recognition.

Loading the Initial Settings File

When you open a sound project, the initial settings file is automatically loaded. When a new sound item is created, the initial value is applied.

If you edit the initial settings file, reopen the sound project.

If there is an error in the initial settings file code, an error occurs and the initial values are not applied.
