Overview

InitializeSdev is a command-line tool to initialize the firmware of the SDEV development hardware.

The GUI version of this tool is InitializeSdevWin.

Packages

InitializeSdev is included in the SystemUpdater for NX package.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/InitializeSdev.exe

There are several dependent files in addition to this EXE file for operations.

Command Definitions

Option Format

InitializeSdev [target options] [other options]

target options:
    --target <IP address | serial number | SDEV name> Specify the IP address, serial number, or name of the target to update.

other options:
    --target-version                                   Specify the type of firmware to use for initialization.
    --display-available-version                        Output all available firmware types.
    --connect-ip-direct                                When initializing by specifying an IP address, if this option is also specified, initializes while skipping segments.
    --verbose                                          Output detailed logs.
    --version                                           Display to tool version.

Exit Code

You can use the exit code to check whether the tool succeeded.

Nonzero value: Terminated with error.

How to Use

The following sample based on a usage case shows how to run InitializeSdev.

Specifying the Development Hardware Name and Initializing

InitializeSdev --target (target name)

Specifying the Development Hardware IP Address and Initializing

InitializeSdev --target 192.168.10.11

Initializing by Specifying the Development Hardware Serial Number

InitializeSdev --target XAW03121000000


Examples of other options are omitted because they are not currently used.

Descriptions of Other Features

Initializing Using Firmware Approximating the Production Hardware

You can also initialize the EDEV using firmware approximating the production hardware.

InitializeSdev.exe --target XAL0210000XXXX --target-version "0.X.Y-Z + Prod"

As shown in the preceding example, to initialize the SDEV using firmware approximating the production hardware, add "+ Prod" after the --target-version option.

In addition:

InitializeSdev.exe --target XAL0210000XXXX --target-version "0.X.Y-Z + ProdWithLog"

To initialize the SDEV using firmware approximating the production hardware with functionality for exporting logs from a microSD memory card, add "+ ProdWithLog" after the --target-version option.

Note

The Initialize using firmware approximating the production hardware feature is only available with the SDEV MP. Note that an error results if it is executed on other SDEV systems.


Important Information

  • When specifying the development hardware serial number, specification is possible with or without the check digit.
  • In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after initialization is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.
    • For more information, see Using Target Manager 2.
  • The later iteration of the SDEV development hardware is supported starting with the following version.
    • SDEV-D: NX Add-On 12.3.2 (FW 12.0.2-1.0)
