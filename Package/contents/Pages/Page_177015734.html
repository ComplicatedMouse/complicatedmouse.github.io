err Library

Introduction

Purpose of This Documentation

This documentation describes the error library (err) included in NintendoSDK.

Intended Audience

This documentation is intended for people developing applications using NintendoSDK.

Related Documentation

Refer to the Error Library API Reference along with this documentation.

Error Library Features

The error library provides the following error display functionality.

  • The display of error messages based on the nn::Result and nn::err::ErrorCode values defined in the libraries provided in NintendoSDK and by Nintendo.
  • The display of application-specific error messages.

How the Tool Is Provided

See the following pages.

  • How the Libraries Are Provided and How to Use Them

Displaying Application-Specific Errors

The application can use the nn::err::ShowApplicationError function to show application-specific error messages together with error codes.

To use this feature, you must enter an ApplicationErrorCodeCategory in the NMETA file when the application is created. Typically you would specify the application's initial code for ApplicationErrorCodeCategory. As an exception, if you want use the same ApplicationErrorCodeCategory for a series, it is all right to specify the initial code for another application.

For more information about writing the NMETA file, see Writing the NMETA File.

Info

The nn::err::ShowApplicationError function was provided to display errors. This function is not for displaying ordinary messages. Do not use it for displaying messages that are not errors, such as warnings and notifications.

Restrictions

The following restrictions apply to the error library.

Displaying Errors in the Windows Environment

Errors are not displayed in the Windows environment.
