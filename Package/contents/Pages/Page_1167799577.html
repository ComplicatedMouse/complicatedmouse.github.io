Overview

InitializeAdevWin is a GUI tool for initializing the firmware of the ADEV development hardware.
The tool can also initialize predecessor EDEV development hardware (EDEV, HDEV, and ADEV).

The command-line version of this tool is InitializeAdev. If you are using the auto-test features, use that version instead.

Package

InitializeAdevWin is included in the SystemUpdater for NX package.

It is available in SystemUpdater for NX 13.0.0-3.0 and later.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/InitializeAdevWin.exe

In addition to this EXE file, there are also several dependency files for running the tool.

Important Information

When you initialize the development hardware with this tool, all installed applications and save data are deleted.

Option Format

There are no command-line options that must be specified in this executable file.

A window is displayed when you run EXE files without arguments.

Display Screen

Main Window

This screen is displayed when InitializeAdevWin is started.

EDEV devices are shown with an E icon, HDEV devices are shown with an H, and ADEV devices are shown with an A.

To use the tool, specify the development hardware to initialize (1), select the firmware version (2), and then click Start initialization (3).
When you click the Start Initialization button, a log window is displayed and initialization starts.

If the development hardware you want to initialize is not displayed on the screen in 1, click Re-Scan to search for it again.
Nothing is registered yet in the Tools menu, so nothing is displayed when you click there.

Log Window

This screen is displayed when you perform initialization.

The progress is displayed in the progress bar, and the log is displayed in the log window.
When initialization is finished, the result is displayed in Result in the lower left of the screen, and the Exit button in the lower right of the screen becomes enabled.


Descriptions of Other Features

After you have selected the type of firmware for initialization from the combo box, the following features become available.

Initializing Using Firmware Approximating the Production Hardware

To initialize the development hardware using firmware approximating the production hardware, select a firmware version in the combo box that includes + Prod in its name.

To initialize the development hardware using firmware approximating the production hardware with functionality for exporting logs from a microSD memory card, select a firmware version that includes + ProdWithLog in its name.

Warning

If you have initialized the development hardware using firmware that approximates the production hardware, the only way to reinitialize the device is to do it manually in safe mode.



Important Information

Initializing With a Downgraded Version

If you have initialized the development hardware using the current version of the tool, you cannot initialize the device with a downgraded version of the firmware using NX Add-On 0.10.x and earlier versions of InitializeEdevWin (or InitializeEdev).

Initializing the ADEV in Safe Mode

If the development hardware unit (EDEV, HDEV, or ADEV) is running in safe mode when you start this tool, the name for that unit in the main window is shown in Name with (Safe Mode).



Select that line and click Start initialization to initialize that development hardware unit.

Target Communication Protocols

In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after initialization is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.

  • For more information, see Using Target Manager 2.
