Creating Sound Projects

What Is a Sound Project?

A sound project is a project file in SoundMaker. A sound project must be created first when you begin to work with SoundMaker.

A sound project consists of one sound project file (FSPJ file) and one or more sound set files (FSST files).
The actual task of creating sounds is primarily performed on sound sets (by using sound set files). Sounds can be classified into separate categories at the level of sound set files, and these can be managed by more than one staff member.

Creating Sound Projects

Select File > Create New Project.
A new project creation wizard starts. Follow the instructions on the wizard to create a sound project.

New Project Creation Wizard

First, specify a project name and path.
The project name that you specify is the sound project filename, and is also used as the sound archive filename at the time of conversion.
To create a sound set at the same time as the sound project, select Create a new sound set.
Make sure that you select Specify existing sound set when you use an existing sound set file.

After you have specified these settings, click Complete to create the sound project.

Creating Sound Sets

What Is a Sound Set?

A sound set is a small-scale collection of all the information that is required to create a sound archive.
The actual task of creating sounds is primarily carried out on sound sets (by using sound set files).
Sound sets are saved in sound set files (FSST files). More than one sound set file can be registered as a reference for a sound project.

Creating Sound Sets

You can skip this procedure if you selected the Create new sound set option in the Create New Project wizard when you created a sound project, because a sound set was created at that time.

Creating New Sound Set Files

Select Project > Add Sound Set > New Sound Set.
The Create New Sound Set dialog box appears. Specify a sound set name and the path to use for the new sound set file.
The sound set name that you enter is used as the filename. A sound set file is created at the specified path location. Simultaneously, the created sound set file is registered with the sound project as a reference.

Using Existing Sound Set Files

When using a sound set file that has already been created, select Project > Add Sound Set > Existing Sound Set.
In the dialog box, select the sound set file to register with the sound project.
The specified sound set file is registered with the sound project as a reference.

Sound Set Names

As with comments, sound set names can use any characters, including Japanese characters.
(However, the characters \, /, :, *, ?, ", <, >, and | cannot be used.
Other item names can use only single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9_].
(Note that numeric characters cannot be used to start a name.)
This name is exported to the HTML sound list.

Sound set names can be changed in the project tree.

Elements of a Sound Set

This section describes the elements that make up a sound set.

Stream Sounds (STRM)

Sounds that are played back using software streaming from storage are called "stream sounds."
Instruments reference external waveform files as resources.

Stream sounds are suitable for playback of content, such as background music, using a large waveform file.

Wave Sound (WSD)

Sounds that are played by using waveform data that is loaded into memory are called wave sounds.
Instruments reference external waveform files as resources.

Wave sounds are suitable for the playback of effects that use, for example, one-shot waveform files.

Sequence Sounds (SEQ)

Sounds that are played by using sequence (MIDI) performance information and tones (banks) loaded into memory are called sequence sounds.
Either an external standard MIDI file that is created with a sequencer, or a text sequence file that is created with a text editor, are referenced as a resource.

Because sequence sounds provide a greater freedom to change the tempo and make other musical manipulations, they are well suited for background music and musical-type sound effects.

Banks (BANK)

Sounds that are used by a sequence are called a bank.
A bank file that is created under SoundMaker is referenced as a resource.

Waveform Archives (WARC)

Collections of more than one instance of waveform data are referred to as a waveform archive.
Waveform data is much larger than bank data. Accordingly, bank data and waveform archive data are handled separately.
This allows for more efficient memory usage, such as referencing a single waveform archive used by multiple different banks.

Players (PLAYER)

The entity that controls playback of stream, wave, sequence sounds, and so on is called a player.
A player must be specified for all sounds.

A sound limit can be specified for a player. This sound limit value can control the specified number of sounds that can be played back with a player.

Groups (GROUP)

Waveform data and playback data must be loaded into memory to play sequence sounds (banks), wave sounds, and so on.
The items included in a sound set can be registered and gathered together in a group.
The application can then operate at a group level to load the waveform data and playback data into memory.

If you use only SoundMaker to confirm sounds, group creation and registration of items to groups is not necessary.
