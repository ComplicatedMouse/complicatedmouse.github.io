Testing Environment

Introduction

This section describes factors that must be considered when testing applications for Chinese-market Nintendo Switch consoles (consoles for China).

What Is the Platform Region?

The platform region indicates the device usage region that a console is configured for.
Changing the platform region causes the console to act as if it is for a different specific region.

There are two platform region settings: Global and China.
Setting the console to Global uses the settings used for all countries other than China. The China setting is used for systems intended for China.

Switching the Platform Region

Use DevMenuCommand or DevMenu to switch the platform region to China.
Switching the platform region reverts development hardware to its default state.
For more information, see either DevMenuCommand > Changing the System Settings or DevMenu > Changing Device Settings.

To switch the platform region on development hardware in PROD MODE, burn DevMenuApp.nsp to an NX development card and then use that card.

Writing to Game Cards

To write an application for China to a game card, use DevMenuCommand or DevMenu.

DevMenuCommand

Run the gamecard command with china specified for the -–region option. This command writes the application to the game card for the China region.

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- gamecard write "C:\application.nsp" --region china
Info

If you do not specify china for the --region option, the application is written to the game card for all countries other than China.
Although this game card can be started on consoles for China, the Health and Safety warning message for For Switch Consoles for China (NX Add-On > Important Implementation Information does not appear.

DevMenu

See DevMenu > Managing Applications > Writing to Game Cards.
