Profiler-WaitFor

Overview

Causes the profiler to wait for a specific event before continuing execution. As of version 1.4, the command waits indefinitely unless a timeout is supplied.

Profiler-WaitFor [-WaitEvent] <ProfilerWaitEvents> [-Timeout <Double>]


[-WaitEvent] <ProfilerWaitEvents>

Events that can be waited for in a script.

    • CanSync: Waits until the Sync button lights up in the GUI.
    • CheckersFinished: Waits until the Checkers have finished running.
    • IsSynced: Waits until the profiler is connected and synced to a dev kit. This is the state the GUI will be in after clicking Sync and selecting a Scenario.
    • NotDisabled: Waits until the GUI is back into a state where most of the UI is enabled. For example, the UI is disabled while opening or parsing a profile.
    • ProfileInProgress: Waits until the a profile is in progress and the Stop button can be clicked.
    • ReadyToProfile: Waits until the Start button can be used in the GUI.
    • Connected: (Added in 1.4) Waits until a dev kit is connected. This is very similar to the IsSynced event and follows the same distinction as Profiler-Connect vs Profiler-Sync.
    • Disconnected: (Added in 1.4) Waits until a dev kit is no longer connected. This is the state the GUI will be in after clicking Unsync.
    • ProfileLoaded: (Added in 1.4) Waits until any profile is loaded, either by opening an existing profile or taking a new one. Effectively waits until the Save button can be clicked.
    • ProfileReceived: (Added in 1.4) Waits until the next profile is received from a dev kit, even if there is already a profile open.


[-Timeout <Double>]

Optional. If not specified, the command will wait indefinitely.

Sets the time in seconds that the command should wait for the event to occur before timing out and causing an error to occur. The specified value should be either be greater than or equal to 0 or be -1.0. A value of -1.0 is used to specify an infinite timeout, though [System.Double]::PositiveInfinity can also be specified to indicate an infinite timeout.


Info

Prior to the introduction of the Timeout value in version 1.4, the timeouts were all hardcoded and different values, ranging from 10 seconds to 8 minutes.


Examples

Basic Usage

Provided here are some basic usage examples.

Profiler-WaitFor ReadyToProfile
Profiler-WaitFor -WaitEvent NotDisabled
Profiler-WaitFor ProfileReceived -Timeout 120.0
