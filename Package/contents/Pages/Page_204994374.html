Overview of Features

The Nintendo NX CPU Profiler is a tool to collect and visualize data to understand the performance of your game. Primarily it is a statistical sampling profiler that detects which functions are using significant CPU resources. It uses time-based interruption to sample which functions are active at the time of interruption. The tool also traces discrete events, such as thread state changes, GPU events, and user logged events. With all aspects presented together, you can better understand the behavior of your game and find opportunities to improve the performance.

Features

  • Sampling of function callstacks at a rate of 5x to 4000x per 60Hz frame (300Hz to 240,000Hz).
  • Call Tree view and partial call tree view (forward and reverse from selected functions).
  • Counters  that capture CPU Performance Counters and user-logged numbers over time.
  • User logging of Marked blocks of code (enter and exit).
  • User logging of frame rates and Heartbeats (functions that are periodically executed).
  • Live CPU Utilization graphs.
  • Timeline : Graphing of data using DirectX to draw millions of data points:
    • Icicle Graph: Shows the callstack at each sample, color coded by system.
    • Stack Graph: Shows the stack depth with each sample.
    • Load Graph: Shows the function load per frame.
    • System Load: Shows the load per frame of each system (Physics, Graphics, AI, etc).
    • Heatmap Mode: Overlays all frames to get an aggregate view of the data.
    • Timeline Overview: Shows the frame rate and window view.
    • Graphing of user logged counter data, marked blocks of code, and heartbeats.
  • Checkers  to analyze and quickly detect issues within an application.
  • Automatic analysis
    • Inferred Heartbeats: Detects the period of called functions, inferring various heartbeat rhythms per core.
    • System Classification: Functions are classified as belonging to one of 16 different systems.
    • Story Analysis: Identifies key functions during each frame that tell a sequential story of the frame.
    • Spike Detection: Identifies anomalous functions that spike on a single frame or burst over multiple frames.
  • Code Coverage  of seen and not seen functions during profiling.
  • Assembly  view of samples per opcode.
