FAQ for Animation Segment Mode Switching

The following are frequently asked questions about the animation segment modes merged mode () and separated mode ().

Question Answer
What are merged mode and separated mode?

Merged mode is the mode for the conventional way to work with animations, using a number of segment tags as demarcations in a single animation.

Separated mode is a new LayoutEditor feature that creates a 1:1 association between animations and segment tags.


By switching to separated mode, you can create a separate animation for each segment tag.

Related Links For more information about animation segment modes, see here.
Can merged mode and separated mode coexist? You can set the mode to merged mode or separated mode for each individual layout file.
Which mode do you recommend?

We recommend separated mode for the following reasons.

  • The animation is displayed divided up, making it easier to grasp and manage.
  • It is easier to adjust the lengths of segments.

We suggest you try incorporating separated mode in the following partial ways.

  • Use separated mode for only the new animation components.
  • Switch to separated mode for some files only.
Why can't I set the per-character transformation SRT for each segment tag in separated mode?

With per-character transformation, the SRT (scale, rotate, transform) parameters are special parameters for which you cannot set an animation curve for each segment tag.

If you want to use per-character animation with a number of segment tags, set a time offset and a duration for each segment tag.

Related Links For more information about per-character transformation animations, see here.
In separated mode, why can't I edit the per-character scale, position, rotation, and color curve values if I don't select a segment tag?

If there are no segments in the layout, you must prepare one.

A warning message might appear if only the per-character scale, position, rotation, and color curve is set in the segment.

If you leave the related groups for the segment empty and disable binary output, the warning message disappears.

Even if binary output has been disabled, the per-character scale, position, rotation, and color curve values are exported to the BFLYT file.
When I switch the segment tag in separated mode, the position of the pane moves.

If animation has been set to the pane, the next animation is applied to the pane each time you switch the segment tag.

If animation is not set to the pane, the pane reverts to the initial state it was in when the file was loaded.
FAQ About Switching From Merge Mode to Separated Mode
Will the layout animation be remade?

The layout animation can continue to be used, without any modification, when you switch to separated mode.

The operations for separating and merging both involve transformations designed to preserve the shape of the curve within the segment tag range.
For this reason, you get the same result when exporting binary files for each segment tag in either separated mode or merged mode.
Why can I not add animation keys any more?

To add keys in separated mode, first select a segment tag.
When I revert to merged mode, why does the animation curve shape change?

After you have changed to separated mode in a layout file, the curve might not revert to its original shape when you switch back to merge mode.

However, there is no difference in the execution result because, as previously mentioned, the binary file that is output for each segment tag takes the same result in either mode.


When you switch to separated mode, if a group is set for the segment tag, only curves in the pane included in that group will be set to the segment tag.
When I switched a number of segment animations to separated mode, why did the playback result differ from merged mode?

In separated mode, the Do not export animation if there are no keys in the frame segment of the segment tag option is enabled by default.
This is why the behavior changes during playback in both modes.

FAQ About Switching From Separated Mode to Merged Mode

Why are there gaps between segments?

 That is how the merge operation is designed. An interval of 10 frames is set between segments so that you can distinguish one from the next.
