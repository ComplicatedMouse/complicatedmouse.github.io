Introduction

DevMenuOverlay is a feature for displaying information on development hardware using an overlay.

This feature can be used regardless of application build type or whether there is a connection with the host PC.

In the following image, items such as the amount of memory or GPU used, power consumption graph and debug log, and controller input when the application is running are displayed as overlays.

How to Use

Starting DevMenuOverlay

DevMenuOverlay is started automatically when the development hardware is turned on with DevMenu as the boot menu.

How to Operate

Info

The DevMenuOverlay operations differ from the HOME Menu operations and internal processing.
Use the HOME Menu for final verification of operations.

Operations When the Menu Is Hidden

Operation Description
Long Press of HOME Button Starts the QuickMenu.
Long Press of POWER Button (See the note.) Starts the PowerMenu.

Note: A long press of the POWER Button is about five seconds. When you press the button for a longer time, the development hardware shuts down.

Operations During Menu Display

Operation Description
Directional buttons Moves the cursor.
A Button Selects.
B Button Closes the current menu.
HOME Button Closes all menus.
(QuickMenu only) Right Stick Scrolls the LogViewer.
(DebugMenu only) Right Stick Scrolls the menu.
Touch Moves the cursor or selects.
(QuickMenu only) Slide Scrolls the LogViewer.
(DebugMenu only) Slide Scrolls the menu.

QuickMenu

A long press of the home button opens the QuickMenu.


The following items can be selected from the QuickMenu.

Item Description
Debug Opens the DebugMenu.
Sleep Puts the development hardware into sleep mode.
Shutdown Shuts down the development hardware.
Airplane Mode Enables or disables airplane mode on the development hardware.
Close Closes the menu.

PowerMenu

Hold the POWER Button down to open the PowerMenu.


The following items can be selected from the PowerMenu.

Item Description
Debug Opens the DebugMenu.
Sleep Puts the development hardware into sleep mode.
Shutdown Shuts down the development hardware.
Close Closes the menu.

DebugMenu

Selecting Debug from the QuickMenu or the PowerMenu opens the DebugMenu.


The following items can be selected from the DebugMenu.

Item Description
Debugging Displays Enables or disables debugging display and changes the display position.
OK

Closes the DebugMenu.

If you select □Save debug settings to SD card, the debugging display settings are saved to an SD card.

Debugging Display

The debugging display feature can be enabled or disabled from the DebugMenu.

Audio

Displays the final output volume (peak value) of the development hardware.

Displays the peak value for each of the final output channels.

  • Units are in dBFS.
  • There is a white line for each 5 dBFS interval.
  • The red line indicates -3 dBFS. If the peak value exceeds this value, the true peak may exceed 0 dBTP and noise might occur.
  • The audio waveform (final output) used for measurement is equivalent to the waveform obtained by SystemAudioMonitor.
  • For more information about the final output and channels, see SystemAudioMonitor.

Clock

Displays the time on the development hardware.

Item Value Description
Type
  • SystemClock
  • SystemTick

For SystemClock, the current time is displayed in YYYY/MM/DD format.

For SystemTick, the current SystemTick is displayed as a decimal.

Log

Displays the logs. The displayed logs are the same logs displayed by Target Manager.

Item Value Description
Line
  • Numerical value
  • Background

If a numerical value has been specified, only the specified number of lines of the log are displayed on the screen.

Background collects the logs but does not display them on the screen. By changing the mode to Show, you can check the past logs that were collected.

If logs are displayed, while QuickMenu is open, you can use Up and Down on the Right Stick (or the slide operation on the touch screen) to display past logs.

If a large log is generated in a short amount of time, part of the log may not be displayed in LogViewer.

Specifies the display size.

Specifies the number of controllers to display.

Scale

  • Large
Specifies the display size.

FrameRate

Displays the frame rate of the currently running application, in frames per second (fps).

Item Value Description
Mode
  • Text
  • Text+Graph

In the case of Text, only displays a string.

In the case of Text + Graph, changes in frame rate are displayed in a graph.

Displays the instantaneous and average fps of the application. The graph shows changes in fps over the last 1,800 frames.

  • DevMenuOverlay samples the application's framebuffer submission time while FrameRate display is enabled.
    The displayed fps is a value calculated from the sampled submission time. In most cases, this value corresponds to the period of the application's main loop.
  • Instantaneous FPS is the frame rate calculated from the interval of the last two samples.
  • Average FPS is the frame rate calculated from the exponential smoothing average of the sample interval.
  • The framebuffer submission time is sampled while the application is in the foreground. Sampling does not occur while the application is running in the background or changing states.
  • The x-axis of the graph shows the number of frames sampled. Note that if the HOME Button is pressed or the development hardware is put into sleep mode, the application frame rate may not be measured correctly.

PowerGraph

Info

The power consumption and battery life are only available for SDEV units in DEV MODE. Cannot be used with EDEV or SDEV in PROD MODE.

Displays the GPU usage time, power consumption, (estimated) battery life when using handheld mode at this time, and memory bandwidth consumption.

Because the GPU usage time also includes the overlay processing time, a value approximately 3% higher than the GPU time used by the actual application is displayed.

The red graph indicates the CPU power consumption, the green graph indicates the GPU power consumption, and the blue graph indicates the main memory power consumption.

ClockRateGraph

Displays the CPU, GPU, and EMC clock rates of the development hardware in a graph.

SocketStatistics

Displays statistical information about socket communication.


Saving and Loading Debugging Display Settings

To save the debugging display settings to an SD card, insert an SD card and select □Save debug settings to SD card next to the OK button.

The settings file is automatically loaded when the development hardware starts.

The settings file is saved in SD:\DevOverlayDisp.setting.txt. By copying this file, you can use the same settings on multiple development hardware devices.

Notes About the Debugging Display

Consider the following information when using the debugging display.

Effects on Application Performance

When the debugging display is enabled, the system CPU and GPU load and the memory access increase as a result of collecting and rendering debugging information.

We recommend disabling the debugging display if you want to accurately measure application load.

Restrictions on Screenshots and Video Capture

The content displayed by DevMenuOverlay is not captured in screenshots or videos.

To take screenshots or video that contain the debugging display, use a feature such as HDMI capture for recording.
