FontConverter (Console Version)

FontConverter (Console Version)
FontConverterConsole.exe

The following only addresses features specific to the command-line version.

Overview

FontConverterConsole is a command­line version of FontConverter (hereafter referred to as the "GUI version").
Functionality is the same as the GUI version.
This tool is intended for use in Makefiles and for processing in batch files.

Font Licenses

FontConverter does not include any font licenses.
To sell software that uses proprietary fonts, you must sign a license agreement with each font publisher.

Command-Line Options

FontConverterConsole.exe -tile-mode <tile mode> -i <input format> <input options>
 -o <output format> <output options> [-f <character filter file path>] [<other options>]
Options Required Remarks
-tile-mode <tile mode> Yes Tile mode. The platform name installed by linear or the add-on.

Specify linear when exporting a binary for Windows or OpenGL.

For more information about exporting a binary for platforms other than Windows or OpenGL, see Platform-Specific Information.
-p <Target Platform> Yes The platform installed by Windows or the add-on. This option will be deprecated. Use -tile-mode.
-i input format Yes One of image, win, or ffnt.
-o output format Yes image or bffnt.
-f character filter file path - No filter if omitted.
-ignore-filter -

Specify this option to ignore the filters in the FFNT. Filters that have been explicitly specified with the -f option are not ignored.

LayoutEditor uses this option internally, so the option does not need to be specified by the user.

-gpu-encoding <type>

-

Specifies whether to perform encoding in the GPU when converting textures.
For type, specify true, false, or auto. If omitted, the default is true.

The specified option is passed to the internally called 3D texture converter.
For more information, see gpu-encoding in the 3D texture converter manual.
  • input options and output options vary based on the specified input format and output format (described later).
  • Enclose elements that contain spaces between words with quotation marks (").
    Quotation marks cannot be used inside of quotation marks.

Tile mode

Specifies the tile mode of the output font.
Specify linear to output a linear tiling font.
When a value other than linear is specified, the add-on for the target platform must be installed.

Target Platform

(This option will be deprecated. Use -tile-mode.)

Specifies the target platform of the output font.
Specify Win to output generic data (where tiling is linear).
When a value other than Win is specified, the add-on for the target platform must be installed.

Input Options

Input: Image (-i image)

Options Required Remarks
-if Image file path Yes -
-io Character order file path Yes Character order file
-ic Color format Yes One of A4, A8, LA4, LA8, LA4Packed, LA4PackedNoCompress, RGB565, RGB5A1,RGBA4, RGB8, RGBA8, RGBA8_BC7, RGB565_SRGB, RGB5A3_SRGB, RGBA8_SRGB, RGBA8_BC7_SRGB.
Parsed internally as follows:
(A4,A8 => BC4) (LA4,LA8 => BC5)
(LA4Packed => BC4 (Brightness and alpha stored in a special format. Stored in the BC4 compression format.) )
(LA4PackedNoCompress => A8 (Stores brightness and alpha in a special format. Stored in noncompressed A8 format. Twice the size of LA4Packed.) )
(RGB565,RGB8 => BC1)
(RGB5A1,RGB5A3 => BC3) (RGBA8 => RGBA8)
(RGBA8_SRGB => RGBA8) (RGB565_SRGB => BC1) (RGB5A3_SRGB => BC3)

The _SRGB suffix represents the linear gamma format. Specifying _SGRB enables sRGB fetches when the program runs.
-ia - Specify to deal with linear interpolation.
When selected, corrects bleeding into surrounding colors caused by linear interpolation of glyphs.
This process is effective only for color formats that have alpha channels.

Example

Read the BMP file font.bmp as input according to the character order file xlor/cp1252.xlor. Interpret BMP color for the LA8 format.

-i image -if font.bmp -io xlor/cp1252.xlor -ic LA8

Input: Windows Font (-i win)

Options Required Remarks
-in <font path, or the font-face name if it is an installed font> Yes

Specifies either the path for the font file, or the font-face name if it is an installed font (example: Tahoma). You can specify files in the formats TTF, OTF, BFTTF, and BFOTF.
If you specify a font face name, that font must be installed on the computer. Also, even if a font is specified, if a font with the same font face name is installed on the PC, the installed font will be used.

A font-face name is required for loading some OTF and BFOTF fonts. If a read error occurs, abide by the error content and specify a font-face name using the -iface option.
-isysfont Internal system font (BFTTF) file path - May be specified when using the system font for input. When this option is specified, the -in option is ignored. This option remains for compatibility. Normally, also use the -in option when loading system fonts.
-is Font size Yes Specify in pixels.
-isw Average width - Specify in pixels.
-iwg Font weight/thickness - Specify a value in the range from 0 through 900. 400 is normal. 700 is bold.
-ic Texture format/number of gradations Yes Either A6, A8, LA6, or LA8. Select LA if you want separate images for luminance and alpha.
A4 and LA4 are deprecated. (They decrease the number of gradations without changing the memory size for the font.)
A6 and LA6 operate the same way A8 and LA8.
A1 and LA1 have been discontinued.
Analyzed internally as follows:
(A4 A6 A68 => BC4) (LA4 LA6 LA8 => BC5)
-ia - When specified, soft anti-aliasing is enabled.
-isck -

Specify this option to extract kerning information using nn::fontll and minimize the differences between the displayed text and nn::fontll.
Use this option when slight differences in display when drawing BFFNT font and TTF and OTF fonts are a problem.
-it Glyph/character width type - glyph, keepsp, char, or fixed. If omitted, char is used.
-iw fixed monospace width - Specify the pixel value. Required when -it fixed is specified. Specifications in all other cases are invalid.
-iface <font-face name> -

If the -in option was used to specify a file path, loading may fail because some OTF and BFOTF fonts require a font-face name for loading. If a read error occurs, use this option to specify a font-face name.

Example

Read using the Windows MS Gothic font with a size of 16 pixels and a 64 grayscale of alpha only (=A6). Use the glyph width for the character width.

-i win -in "MS Gothic" -is 16 -ic A6 -it glyph

Output Options

Output: Image (-o image)

Options Required Remarks
-of image file path Yes -
-oo character order file path Yes Character order file
-oi Output image format - -
-ia - ext, bmp, tga, or tiff. If ext is specified, determines based on the image filename extension. If omitted, ext is used.
-og - If specified, no grid is drawn.
-ow Cell width - If omitted, it is analyzed as a sufficiently large value. It cannot be specified simultaneously with -or -ob.
-oh Cell height - If omitted, it is analyzed as a sufficiently large value. It cannot be specified simultaneously with -or -ob.
-ol Cell left margin width - If omitted, it is analyzed as 0.
-or Cell right margin width - If omitted, it is analyzed as the same value as -ol. It cannot be specified simultaneously with -ow -oh.
-ot Cell top margin width - If omitted, it is analyzed as 0.
-ob Cell bottom margin width - If omitted, it is analyzed as the same value as -ot. It cannot be specified simultaneously with -ow -oh.
-ocg Grid color - Specifies the color as a 6-digit hexadecimal. If omitted, 000000 is used.
-ocm Margin color - Specifies the color as a 6-digit hexadecimal. If omitted, 99AA99 is used.
-ocw Width line colo - Specifies the color as a 6-digit hexadecimal. If omitted, FF0000 is used.
-ocn null color - Specifies the color as a 6-digit hexadecimal. If omitted, FF7F7F is used.

Example

Apply a one-pixel margin to the left and right and a three-pixel margin to the top and bottom of each cell according to the character order file xlor/cp1252.cxlor, and then output the image to font.bmp without a grid.

-o image -of font.bmp -oo xlor/cp1252.xlor -og -ol 1 -ot 3

Output: BFFNT (-o bffnt)

Options Required Remarks
-of Output file path Yes Outputs using the specified filename.
-oe Supported character code - unicode, sjis, or cp1252. If omitted, unicode is used.
-op Glyph group file - Discontinue option. If omitted,there is no grouping (no compression).
-os Sheet size - Sheet sized specified in kilopixels. A power of 2 greater than 16 and less than 8192. If omitted, it is determined automatically.
-oa Alternate character - If omitted, it uses the default value depending on input.
-oh Line feed width - If omitted, it uses the default value depending on input.
-ol Default left space - If omitted, it uses the default value depending on input.
-ow Default glyph width - If omitted, it uses the default value depending on input.
-or Default right space - If omitted, it uses the default value depending on input.
-ok - Whether to output kerning information. If this option is omitted, the kerning information is output by default. (This change is effective from 1.11.2)
When specified, a warning is displayed if the input does not have kerning information.
-onok - Prohibits the output of kerning information.

Example

Output to font.bffnt using a sheet size of 256 kilopixels, Shift_JIS encoding, and "?" as the alternate character.

-o bffnt -of font.bffnt -os 256 -oe sjis -oa ?

Other Options

Options Required Remarks
-usedtd - Enables input file validation using a DTD file.
-silent - Prevents log output.
