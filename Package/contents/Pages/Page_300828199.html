Reverse Lookup Guide

Introduction

This section describes a reverse lookup reference for reviewing the detailed procedure for enabling nn::audio functionality, using concrete examples.

Audio Renderer

I want to see the DSP processing load from a tool that works on a PC.

You can do this by using the Spy tool included in the SDK. For more information, see Spy and the SpyAudio sample.

I want to connect to multiple SubMix and FinalMix output destinations from a single Voice object.

Use the nn::audio::SplitterType function to take the input from a single Voice object to multiple SubMix and FinalMix output destinations. For more information, see the API reference for nn::audio::SplitterType and the AudioSplitter sample.

I want to reduce the processing load of effects run on a DSP.

You can run effects processing on the CPU to reduce the processing load of effects run on a DSP. For more information, see the API references for nn::audio::AuxI3dl2ReverbType and nn::audio::AuxReverbType and the AudioEffectAuxUtility sample.
I want to provide a way to run all rendering processing for the audio renderer on the CPU. For more information, see the API reference for nn::audio::AudioRendererHandle and the AudioCpuRenderer sample.

Audio Output

I want to play sample rate waveform data not supported by audio output.

Use nn::audio::ResamplerType to perform resampling. For more information, see the API reference for nn::audio::ResamplerType and the AudioOutWithResampler sample.
