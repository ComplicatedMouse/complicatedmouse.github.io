Overview

InitializeEdevWin is a GUI tool to initialize the firmware of the EDEV development hardware.

The command-line version of this tool is InitializeEdev. If you are using automatic tests, use those.

Packages

InitializeEdevWin is included in the SystemUpdater for NX package.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/InitializeEdevWin.exe

There are several dependent files in addition to this EXE file for operations.

Important Information

When you initialize EDEV with this tool, all installed applications and save data are deleted.

Option Format

There are no command-line options that must be specified in this executable file.

A window is displayed when you run EXE files without arguments.

Display Screen

Main Window

This screen is displayed when you start InitializeEdevWin.

EDEV devices are shown with an E icon, HDEV devices are shown with an H, and ADEV devices are shown with an A.

To use the tool, specify the development hardware to initialize (1), select the firmware version (2), and then click Start initialization (3).
When you click the Start Initialization button, a log window is displayed and initialization starts.

If the development hardware you want to initialize is not displayed on the screen in 1, click the Re-Scan button to search for it again.
Nothing is registered yet in the Tools menu, so nothing is displayed when you click there.

Log Window

This screen is displayed when you perform initialization.

The progress is displayed in the progress bar, and the log is displayed in the log window.
When initialization is finished, the result is displayed in Result in the lower left of the screen, and the Exit button in the lower right of the screen becomes enabled.


Descriptions of Other Features

The following features becomes available after you have selected the type of firmware for initialization from the combo box.

Initializing Using Firmware Approximating the Production Hardware

To initialize the SDEV using firmware approximating the production hardware, select the item that includes the "+ Prod" firmware version selection combo box.

To initialize the EDEV using firmware approximating the production hardware with functionality for exporting logs from a microSD memory card, add "+ ProdWithLog".

Warning

When reinitializing a system that was initialized using Firmware approximating the production hardware, if you do not manually activate Safe Mode, reinitializing is not possible.



Important Information

Downgrading the Version and Initializing

Prior to NX Add-On 0.10, if you have already initialized EDEV using the current version of InitializeEdevWin, you cannot downgrade and initialize EDEV using InitializeEdevWin (and InitializeEdev).

EDEV Initialization in Safe Mode

When starting InitializeEdev with the EDEV console in Safe Mode, a line with (Safe Mode) in the Name column is displayed on the main screen.



After selecting this line, click Start initialization to initialize EDEV.

Target Communication Protocols

In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after initialization is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.

  • For more information, see Using Target Manager 2.
