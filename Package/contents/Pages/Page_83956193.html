NintendoSDK

NintendoSDK provides the following features.
Note that some features have restrictions and there is a possibility that the features may be changed in the future.

Build System

A system for creating applications using the NintendoSDK libraries and tools is provided.

The following platforms and platform toolsets are currently supported.

Platform
  • Win32
  • x64
Platform Toolsets
  • v141 (Visual Studio 2017) (only supported in NX Add-On 4.x and later)
  • v142 (Visual Studio 2019) (only supported in SDK 12.3.0 and later)

For more information about the build system, see here.

Library

The libraries provided by NintendoSDK are as follows.
Link these libraries to get access to basic features such as the operating system and file system, gamepad controllers, communication, graphics, and audio.

Info

This collection of features is called the generic API set.

Features that are only supported on specific platforms are provided as a collection of add-on packages called the [hardware name] API set.

Library Supported Features Limitations of Current Version or Notes
account
  • Getting a list of user accounts created on the system
  • Features for network service accounts
  • Features for Nintendo Accounts


audio
  • Input or output for audio devices
  • Pitch conversion, filtering, and mixing on a "voice" basis
  • Effect processing and auxiliary processing on a "sub-mix" or "final mix" basis


bcat
  • Data Delivery
codec
  • Tools for encoding and decoding audio data in formats such as ADPCM, Opus, and AAC on the target and encoding the data on the PC
  • The Opus and AAC features are implemented using open source software (OSS).
    • You must agree to the terms of the applicable OSS license to use this library.
crypto
  • Date encryption and decryption
  • Generation and authentication of digital signatures
  • Calculation of hash values using a cryptographic hash function
curl
  • HTTP/HTTPS communication using cURL
diag
  • Logging, assertions, and stack trace retrieval
err
  • Displaying errors
fontll
  • Loading scalable fonts included in the system fonts and renders them in main memory
friends
  • Getting the friend list
  • Getting the blocked-user list
  • Updating presence
  • Getting any user's user profile (nickname and profile image URL)
  • Adding users played with
fs
  • Loading application resource data
  • Reading and writing save data
  • Accessing the file system on the host PC for reading and writing purposes for development
  • Limited to the usable types of save data.
gfx
  • Graphics features that abstract low-level graphics APIs
  • Currently, there are several limitations in the implementation of the GL version.
gll
  • Loading OpenGL.
hid
  • Controlling the input of human interface devices such as gamepads, keyboards, touch screens, and accelerometers
  • Vibration Feature
  • The following devices are supported.
    • Gamepad
    • Touch screen
    • Debugging gamepad
    • Keyboard
    • Debugging mouse
  • The following features are scheduled to be added.
    • Debug features like auto-input and recording and playback of input (some are already supported)
htc
  • Various features for working with the host PC by using Target Manager
  • For more information about restrictions, see htc Library Restrictions.
htcs
  • Socket communication between the application on the host and the application on the target by using Target Manager
  • For more information about restrictions, see htcs Library Restrictions.
init
  • (Initializing memory allocators)
  • Also see Starting and Initializing Programs.
image
  • JPEG decoding and Exif data retrieval
  • JPEG encoding for adding Exif data
  • PNG decoding
  • PNG encoding


kpr
  • Encoding conversion for key input.
  • Encoding assembly for key input, which includes dead key processing, Roman letter input, and more.
mem
  • Thread-safe memory allocator feature for the allocation of memory space
    • Existing expanded, frame, and unit heaps
    • Standard allocator with faster allocation and deallocation than an expanded heap (StandardAllocator class)
  • Thread-safe integer allocator functionality
    • NumberLineAllocator class


Mii (Face Library)
  • Getting Mii data from the system
  • Getting drawing resources from the system
  • Rendering Mii characters using Mii data
  • Rendering features equivalent to the Create or Edit Mii applet
  • Creating Mii icons
  • There is no support in the Windows environment for getting drawing resources from the system.
ngc
  • Checking for profanity
nifm
  • Network connection control
  • For more information about the restrictions associated with the current version, see Considerations and Restrictions.
openssl
  • SSL/TLS communication and encryption using OpenSSL.
os
  • Multithreading with support for multiple cores
  • Fiber
  • Various synchronization features
  • Memory heap
  • Ticks
pctl
  • Check for feature restrictions set in parental controls
  • In the Windows environment, the settings cannot be changed to verify what is actually restricted.
pdm
  • Getting play data
  • In the Windows environment, this information cannot actually be retrieved.
perf
  • Performance metrics tabulation
  • CPU performance metrics
  • GPU performance metrics
  • Provides features for metrics at the application level.
settings
  • Getting and setting system settings, debug settings, and emulation settings


socket
  • Berkeley sockets-compatible TCP/IP communication
    • On Windows, this library operates as a WinSock wrapper.
ssl
  • SSL/TLS communication
time
  • Getting the time
  • Conversion between POSIX time and time in YYYYMMDDHHMMSS format


util

  • Bit operations

  • String operations

  • Containers (lists and dictionaries)

  • Arithmetic operations (trigonometric functions, vectors, matrices)

  • Compression and decompression
vi
  • Basic display output features, such as layer creation and format and size settings
zlib
  • Compression and decompression by zlib
  • The gzip file access functions cannot be used.


Target Manager

Target Manager is a tool to manage applications on a target connected to a PC or LAN and applications running on Windows.

We plan to provide various features, such as central management of development hardware, automatic control of targets using scripts, and HTCS management.

For more information about Target Manager, see here.

NintendoWare

NintendoWare is middleware that can be used to develop graphics and sounds for game software.

For more information about NintendoWare, see here.

Warning
  • Major changes were made to the NintendoWare module in NintendoSDK 0.7.0. For more information, see About the New NintendoWare Libraries.
  • NintendoWare is currently being authored.

Features Added by Add-On

Platform-specific features can be added by installing a collection of packages, called an add-on, for that platform.

For a list of features provided by each add-on, see Features Added by [Add-On].
