Overview

NfpDebugTool is a target-oriented tool to assist the debugging of applications that use the nfp library and check that they conform to the guidelines.

The following features are available.

  • Displaying and editing data written to NFP tags
  • Deleting application-specific regions and initial registration information
  • Corrupting data on NFP tags
  • Initializing amiibo backup data saved in system storage
  • Calling amiibo Settings

Location

This tool is included with NintendoSDK NX Add-On and is found in the following location.

TargetTools/NX-NXFP2-a64/NfpDebugTool/Release/NfpDebugTool.nsp

How to Use

Starting the Tool

Use RunOnTarget to start the tool.

$ Tools/CommandLineTools/RunOnTarget.exe TargetTools/NX-NXFP2-a64/NfpDebugTool/Release/NfpDebugTool.nsp

You can also install the NSP file on the development hardware and use the tool.
For more information about installing the NSP file, see Installing and Running an NX Application on the Development Hardware.

When the tool starts, a list of available operations is displayed on the Main menu. Select the feature you want.

Info

Operate the tool by using the Joy-Con or the Nintendo Switch Pro Controller.

Info

When reading and writing NFP tags, hold the tag over the controller used in tool operations.

Displaying and Editing Data Written to NFP Tags

From the Main menu, select Edit tag data, press the A Button, and hold the NFP tag over the controller.

When tag reading is complete, the available operations will be displayed in the Tag menu.
After the Tag menu is displayed, you can move the NFP tag away from the controller.

Display current data

Displays data written to the NFP tag.

If a tag other than an NFP tag is read, or if reading the NFP tag failed, only TagInfo is displayed.

Note

Nicknames in languages other than English and Japanese, and nicknames with special characters, cannot be displayed correctly.

Info

If the NFP tag has never been written to after initialization, Last write date and Write counter will not be displayed correctly. This problem is due to NFP tag specifications and is not a bug.


Edit admin info

Confirms and changes the access ID of the application-specific region.

Edit register info

Info

This feature can be used in NX Add-On 1.0.0 and later.

Changes the nickname registered for the amiibo.

As the language for the nickname, you can choose English, Japanese, Chinese (Traditional or Simplified), Korean, or For guideline check. There is a fixed nickname for each language that is selected. If you change the nickname, FontRegion also changes accordingly.

If you select For guideline check, the nickname becomes ĈĉĔĕĜĝ and FontRegion changes to nn::nfp::FontRegion_JpUsEu. Select this option when checking the guidelines.

Set application area

Overwrites the application-specific region in incremented series from 0x00, or all to 0xFF.

Flush current data

Writes the modified content to an NFP tag.

After selecting Flush current data, hold the NFP tag that was read immediately beforehand over the controller again.

Deleting Application-Specific Regions and Initial Registration Information

From the Main menu, select Delete info to delete the application-specific region and the initial registration information written to the NFP tag.

Delete application area deletes the application-specific region, and Delete register info deletes the initial registration information.
After selecting either one and pressing the A Button, hold the NFP tag over the controller.

Corrupting Data on NFP Tags

Break amiibo in the Main menu creates a state equivalent to corrupting data written to the NFP tag and failing to write data to it.
NFP tags with corrupted data must be recovered or initialized using the amiibo Settings.

After selecting Break amiibo and pressing the A button, hold the NFP tag with the corrupted data over the controller.

Info

Of the shared region, information such as character ID details that cannot be overwritten is not destroyed.

Initializing amiibo Backup Data Saved in System Storage

From the Main menu, select Backup data to display Initialize backup data, and delete all backup data for recovering amiibo saved in system storage.
After this backup data has been deleted, the result nn::nfp::ResultNeedFormat will be returned when an NFP tag with corrupted data is mounted.

Select Initialize backup data and press the A Button to immediately delete data.

Calling amiibo Settings

From the Main menu, Call amiibo settings calls the amiibo settings applet.

Select the feature you want to call and press the A Button.

Menu Item Calling Function
Nickname & Owner Register owner and nickname
Erase Game Data Delete game data
Restore Restoring Data
Format Resetting the amiibo
