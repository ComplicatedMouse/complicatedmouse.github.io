AudioRedirector
Warning

The AudioRedirector is currently provided as a release candidate. Note that disruptive specification changes may be introduced without warning.

Overview

The AudioRedirector is a tool for sending playback sound from the host PC to the development hardware for playback. The AudioRedirector’s intended use is to play back sounds that are in development, by tools like DAW, through the speakers of the console for verification.

The tool is located in the following locations.

  • Tools/Audio/AudioRedirector/AudioRedirector.exe
  • TargetTools/NX-NXFP2-a64/AudioRedirectorServer/AudioRedirectorServer.nsp

How to Use

Development Hardware Side Setup

Run AudioRedirectorServer.nsp.

PC Side Setup

Start the Target Manager and connect to the development hardware. Confirm that AudioRedirector is displayed in the HTCS entry for Port Name.

Start AudioRedirector.exe after a connection to the development hardware has been established. The following log is displayed upon success.

start redirecting, press any key to exit

Additionally, AudioRedirectorServer.nsp displays a screen similar to the following image.

Sounds played on the PC are played through the development hardware. Pressing any key terminates AudioRedirector.exe.

Other

Both AudioRedirector.exe and AudioRedirectorServer.nsp display errors if any have occurred. Perform the previously listed steps again from the beginning if an error occurs.

Limitations

  • AudioRedirector.exe only supports default PC sound devices with a sample rate of 48kHz/2ch.
  • AudioRedirector.exe connects to the default target in Target Manager.
