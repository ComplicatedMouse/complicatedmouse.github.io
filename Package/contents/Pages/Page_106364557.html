Customization

Introduction

You can customize the features provided by default to display more advanced shader expression, or link effects more tightly with applications rather than simply displaying the effects.
You can customize the effect runtime in two ways.

  • Custom Shaders
  • Custom actions

Custom Shaders

The custom shader allows you to portray lighting and other application-specific effects for effects rendering by customizing the standard vertex/pixel shader.

Left: Lighting using a custom shader.
Right: Rendered with the standard shader.

Custom actions

You can customize the behavior of emitters and the behavior and rendering of particles.

For example, you can customize rendering a 3D model on the application side in a location to render with particles, or implement collision detection between objects and particles in the game to realize more interactive effects.

User Data Definitions

EffectMaker includes features for working with user data definitions, which are user-defined structures for binary data. EffectMaker also includes a UI for editing such user data definitions to achieve the desired customizations. The binary data structures configured here can be accessed from the effect runtime to implement various customizations.

Callback Functions

Callback functions can be called at various times while using the effect runtime to customize the behavior of emitters and particles. Custom shaders make it possible to access parameters that were edited and saved by artists using the user data definitions feature in EffectMaker from within these callback functions, and then use these parameters from a constant buffer in vertex and pixel shaders.
