movie Library


Introduction

Purpose of This Documentation

This documentation describes the movie library included in NintendoSDK.

Intended Audience

This documentation is intended for developers who would like to support multimedia playback in their applications.

Associated Documentation

Please use this documentation in conjunction with the movie namespace API reference.

Overview

The movie library is a software API that you can include in your applications to provide real-time playback of multimedia content. The library can be used to extract and decode audio / video files or elementary content streams into uncompressed audio and video data suitable for presentation.

There are two groups of APIs available in the library: 

  • Extractor APIs: Parse, de-multiplex, and output compressed audio / video data from multimedia files
  • Decoder APIs:  Decode audio / video compressed data into presentable content

Data Processing

Movie decoding can be thought of a data processing operation that takes compressed data as input, and generates uncompressed audio and video data as output. The application is responsible for feeding compressed data to the decoder, either from a local file source using the extractor APIs, or through an external mechanism. The application is also responsible for consuming output data from the decoder and rendering it or performing additional post-processing operations as needed.

It is important to note that the movie library video decoder requires complete picture data within a frame buffer. Currently, it is not possible to decode partial data (for a slice). The movie decoder outputs ITU-R BT.601 YCbCr color space pixels in NV12 format. The movie audio decoder outputs 16-bit PCM samples.

Memory requirement

The movie library makes a variety of memory allocation calls and also requests allocations from the application's global heap. This is in addition to allocations from memory functions set using movie::SetAllocator API. The actual memory usage varies by content specification.

Dependencies

The movie library is packaged as a dynamic linkable library (.nso/.nss files). For the movie library to work correctly, applications must also link to these additional dynamic libraries:

  • nvn
  • nnSDK
