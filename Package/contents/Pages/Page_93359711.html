Introduction

This documentation describes the Target Manager included in NintendoSDK.

Target Manager is a GUI application for managing development hardware connected to the PC.

Target Manager can manage Windows applications just like it can manage development hardware.

Package

Target Manager is provided in the Target Manager package.

You must download and install the preceding package to connect to the NX development hardware.

Note that this package can be obtained even if SystemUpdater is selected in NDI.

Placement

Target Manager is installed as a Windows application in the following path.

C:\Program Files\Nintendo\Oasis

After you install it, Target Manager can be started from the Start menu.

Supported Platforms

Supports Windows and NX.

Functionality

View

Target

Displays information for each added target.
You can right-click and use the shortcut menu to add and remove targets, and to load executable files.

Log

Application > Development Hardware > Target Manager displays the log output.
You can open a log window that only displays this target's log by right-clicking the target from the target list and selecting Display Log.

Saving the Log

You can start saving a log from Menu > File > Save Log or by right-clicking the log and selecting Save Log.
There are also four selection trees that combine whether to include a log that is already being output or to save in a JSON format that includes metadata.
To end saving, select it again from the menu.

Viewing the Log

A JSON file saved by the preceding procedure, or a NXBINLOG file saved by Development Assistance Settings can be opened using Menu > File > Open Log. You can also open these files by dragging them to the Target Manager.

Displaying Metadata

For any log window, you can select a displayed column by right-clicking the item name.

Fonts

You can change the font used in the log view by right-clicking the log view and selecting a font or from Menu > Tools > Options > General > Fonts.

Get With the Command Line

For more information about using the command line to get logs for individual targets, see Receiving Separate Logs for Each Target.

Console Control Using Escape Codes

Supports text color changes only.
The color specification format supports only ANSI colors.

Limitations

There is a limit to the size of logs maintained by Target Manager. This limit can be changed from Menu > Tools > Options > General > Maximum Lines.
If a high maximum is set, the amount of memory used by Target Manager increases according to the size of the log output.
While there are limits to memory usage, use the Save Log feature if you want to retain a lot of log data.

Filter Feature

To display only logs matching certain criteria, enter the criteria into the box at the upper left of the log view and press ENTER.
The syntax available are those supported by Log Viewer and those supported in KQL.
For more information about the former, see Log Viewer. For more information about the latter, see the page opened by clicking the question mark (?) icon at the right of the box.

Search Feature

Entering text into the box at the upper right of the log view results in those words being highlighted in real time.
Press ENTER to move to the next candidate, and press SHIFT+ENTER to shift backwards to the preceding candidate.

HTCS Entry

Displays a list of HTCS ports currently opened by Target Manager.
You can change the items displayed by right-clicking item names.

Peer Name Supports nn::htcs::HtcsPeerName and nn::tm::HtcsPeerName. The same as the serial number.
Port Name Supports nn::htcs::HtcsPortName and nn::tm::HtcsPortName.
IP Address
Port Number The TCP port number.
Target Name The name set on the corresponding equipment. If no name is set, the serial number is displayed.

Tools

Add Target

Selecting Add Target displays a list of targets that are available to connect on the network.
Select the target to add, and then double-click or click OK to add to the target view.

Info

The Target Manager detects SDEV by broadcasting.
If you want to detect SDEVs connected outside the subnet, add a broadcast address from Add target > Broadcast address and run detection again.
This additional broadcast address feature has been supported since Target Manager 9.0.2.


Options

You can change the following settings.

General

  • Maximum Number of Log Lines
    • The log output will be maintained until it reaches this number of rows.
    • If the number of rows in the log exceeds this value, the excess number of rows are deleted in order from the oldest.
    • Used to limit the amount of memory used by the Target Manager.
    • You can specify a value in the range of 1000 to 100000000.
  • Display Log Metadata With Mouse Pointer
    • Specify whether to show the log metadata pop-up window.
  • Font
    • Specify the log font.
  • Minimize to System Tray When Minimize Icon Is Clicked
    • Specify the behavior to use when the Minimize icon is clicked.
  • Display Dialog Box When Executable File Fails to Start
    • Switch between showing and hiding the dialog box when an executable file fails to start.
  • Enable JIT Debugger
    • Selects whether the VSI JIT debugging dialog box appears when the application crashes when executed without debugging.

HTCS

  • Current Control Port
    • Displays the port number that is currently used as the htcs Library control port.
  • Specify Control Port
    • Specifies the port number that is used as the htcs Library control port.
    • You must restart Target Manager to enable the settings.
    • Specify 0 to return to automatic configuration.

The following priority is followed for using port numbers as the control port. The system attempts to use the next lowest priority port, in order, if a port is already in use by another service or is otherwise unavailable.

  1. The specified port if a value other than zero (0) is specified as the control port.
  2. 8003
  3. 7974
  4. An available ephemeral port

The nn::tm::GetHtcsControlPort() function in tm Library can be used to get the port that is currently open as the control port.

Screenshots

  • Screenshot Naming Conventions
    • Specify the naming conventions when saving. For the legend, see the bottom of the same screen.
  • Screenshot Output Folder
    • Specify the destination to save screenshots.
  • Screenshot Format
    • For the output format, specify BMP, JPG, PNG, or TIF.

Videos

  • Video File Naming Conventions
    • Specify the naming conventions when saving. For the legend, see the bottom of the same screen.
  • Output Destination Folder for Video Files
    • Specifies the video file save destination.
  • Action When Saving Video Files
    • Select Nothing, Open in Media Player, or Open in Explorer.

Symbol Cache

These settings are for the debugging information file (NSS file) caching performed while debugging in NXSymStore and VSI.

  • Symbol Cache
    • Specify the save destination for symbol caches.
  • Maximum Size for Symbol Cache
    • Items are deleted beginning with the oldest so that the size of the directory specified in Symbol Cache does not go over the size specified in this option.
  • Clear Symbol Cache
    • Delete the contents of the Symbol Cache directory by pressing this button.
    • The on-memory cache may not be destroyed until Target Manager or VSI is restarted.

Layout Hotkey

  • Links keys to the *.layout files where the view position and size information have been recorded.
  • *. To create a *.layout file, select View > Save Layout As.

Hardware Management

  • Enable Hardware Logging
    • If this check box is selected, the initial and final connection date and time and serial number of the target connected to from Target Manager on the device.
  • Log File Path
    • Specifies a path to save hardware management logs to a path other than the default.

Data Collection

Some data will be collected to improve the SDK Development Environment. We request your cooperation.
These settings can be changed with this option.

Usage Statistical Data

Switches between sending and not sending SDK usage statistical data. You can see a sample of the data that is actually sent from Usage Statistical Data (Sample) in this dialog box.

Incident Reporting

Switches whether to send crash information when the Target Manager crashes.
Data is actually sent when Target Manager is next started.

HTCS Port Mapping

Open the settings file to specify the HTCS port map in Editor.
For the settings file format, see Fixed Port Mapping in the HTCS Manual.

Development Hardware Operation

Preparation

  1. Connect the development hardware to the PC and turn it on.
  2. From the Start menu, start Target Manager.
  3. If a warning message appears stating it is being blocked by the firewall, allow communications as needed.
    • Target Manager performs network communications using the connected SDEV.
  4. Add the development hardware to connect to in target view.
    • On the shortcut menu, or from Tools, select Add Target and then select the development hardware that was turned on in the previous step.
    • Ignore these steps if the target you will be connecting to is already added to the target view.
  5. Set the development hardware that you will be connecting to as the default target.
    • Right-click the target to connect to (development hardware) and select Set Default Target from the shortcut menu.
    • The default target is indicated by the bold text.
  6. Connect to the development hardware.
    • Right-click and select Connect to target from the shortcut menu. Confirm that the status indicates an active connection.
    • When other users are using that target, you must determine whether to forcibly connect.
      • If a forced connection is selected, it is disconnected from the original user.

Running and Stopping Your Application

Using VSI

Running the Tool

  1. Confirm that the NX32 or the NX64 platform is selected in Visual Studio.
  2. You can also run the tool by pressing F5, the debug start button, or from Tools > Debug.

Stop.

  1. You can stop the tool by pressing SHIFT + F5, the debug stop button, or from Tools > Debug.

Related Items

For more information, see Features - VSI.

Using Target Manager's Load Executable File Feature

Running the Tool

  1. Right-click the target, and select Load executable file.
  2. Select either an NSP executable file or the nspd_root file.
    • To create an executable file in NSP format, you must create an NSP file in advance by using VSI. For more information, see Features - VSI.
    • Creating an nspd_root file requires building for Raw mode with VSI. For more information, see Features - VSI.
  3. Enter when specifying a command-line argument.
    • These command-line arguments can be obtained using the nn::GetHostArgc() and nn::GetHostArgv() functions defined in nn/os/os_Argument.h.
  4. To use a target environment variable, specify the TDF file in the target environment variable files.
    • For more information about target environment variables, see Target Environment Variables.
Info

You can directly execute NSP and NSPD_ROOT files by dragging them.


Finalization

  1. In Target View, select the target that is running.
  2. Right-click and select Force Quit from the shortcut menu,.
    • You can determine whether the application stops by checking for whether the name of the application that is running disappears.

Installing, Running, and Uninstalling Applications

Target Manager supports installing, running, and uninstalling applications in system save memory, game cards, and SD cards.
To install an application, right-click the target and select Install Application or Patch. To run or uninstall an application, select Start/Manage Application.

Installing, Uninstalling, and Managing Patches

To install a patch, right-click the target and select Install Application or Patch. To verify or uninstall a patch, select Manage Patches.

Creating Dump Files

To get a dump of the running application, right-click the target while the application is running and select Dump of Running Application.
To perform a full dump, select the Quick Core Dump check box.
You do not need any tools other than Target Manager to create a dump file, but to analyze the dump file you need VSI and the tm library. For more information, see Using the Dump Feature.

Screenshot

Capture a screenshot while the application is running by right-clicking the target, and from the shortcut menu, selecting Take Screenshot. The default file format is 32-bit BMP.
An image of the application is always output, regardless of whether the application is running in the foreground or the background. A screenshot cannot be captured if the application is not running.
You can specify the naming convention, folder, and format for the saved files in Tools > Options > Screenshot.

Select View > Screenshot to open a view for screenshots.
In this view, in addition to capturing screenshots, you can change the magnification for each image and perform operations such as deleting and copying.

Specifying the Working Directory

By specifying the working directory for each target, you can use a relative path in the root directory to specify by MountHost.

  1. Right-click the target and select Target property.
  2. Specify the directory in the working directory.
  3. You can reference the specified directory using "." from the MountHost function.
    • For example, for the structure C:/Directory/SubDirectory, by specifying the working directory as C:/Directory, you can specify MountHost("host", "./SubDirectory").

Reset Target

Right-click the target and select Software Reset or Hardware Reset.
The software reset process is equivalent to Reboot from DevMenu.
The hardware reset forcibly resets and restarts the hardware, even when the target system has become unresponsive.
The hardware reset is only supported in SDEV.

Changing the SDEV Settings

If an SDEV is connected, you can change its settings by right-clicking the line showing that SDEV in the target view and then selecting SDEV Settings.

Rename

  1. Select Rename and enter the name that you want to change to in Name.
    • Names can have a maximum of 63 characters and, of the ASCII characters, can use alphanumeric characters, hyphens (-), and underscores (_).
  2. To apply the specified settings, click OK and restart the SDEV.
    • You can specify this name using the -t and --target options in RunOnTarget and ControlTarget for displaying and filtering in Task Manager.

Setting a Static IP Address

  1. Select the specific IP address to use and specify valid values for the IP address, subnet mask, and default gateway.
  2. To apply the specified settings, click OK and restart the SDEV.
Warning

If you specified an IP address that cannot be connected to from your network, check the IP address from the Firmware tab in DevMenu and try reconnecting and reconfiguring the settings.

The ControlTarget set-ip command can also be used if the IP address is in range of receiving broadcast packets.

DNS Server Address Settings

  1. Select the option to use a specific DNS server address, and specify the address for the preferred DNS server and the alternative DNS server.
  2. To apply the specified settings, click OK and restart the SDEV.

Using Jumbo Frames

  1. To use jumbo frames, select the Enable jumbo frame option.
  2. To apply the specified settings, click OK and restart the SDEV.

Communications may be faster if your network configuration supports jumbo frames.
Jumbo frames on the SDEV are fixed to 4,096 bytes.

Changing the EDEV Settings

If an EDEV is connected, you can change its settings by right-clicking the line showing that EDEV in the target view and then selecting USB Settings.

Rename

  1. Select Rename and enter the name that you want to change to in Name.
    • Names can have a maximum of 63 characters and, of the ASCII characters, can use alphanumeric characters, hyphens (-), and underscores (_).
  2. Click OK to apply the specified setting.
    • You can specify this name using the -t and --target options in RunOnTarget and ControlTarget for displaying and filtering in Task Manager.

LED Flashing

The LED in the back of the SDEV, can be made to flash to identify which hardware is registered in Target Manager.
This feature is only supported in SDEV.

Log

Collects the logs output from all targets connected to this instance of Target Manager and displays them in Log View.
If you want to display the log from each target in its own separate window, right-click a target, and from the shortcut menu, select View Log.

Remote Video

You can save the video output for the application currently running on Target Manager. You can also save that video as an MP4.
Two capture features are provided for saving videos: one that records the last 30 seconds of a video, and one that records any interval using the Start and Stop recording buttons.

Note that the remote video feature depends on the Video Capture feature. When the Video Capture feature is disabled, Target Manager switches to an alternate mode but is subject to the following limitations. This alternate mode is only available with the SDEV.

  • The frame rate drops significantly for video played in Target Manager.
  • The video recording feature using Target Manager is disabled.
  • In TV output mode, the screen is black.
    • As a workaround, you can use DevMenCommand to run a command like debug set-operation-mode-policy handheld to send the output to the LCD screen.

Also note that all applets disable the video capture feature. While DevMenu is an applet, the DevMenuApplication that can be started from HomeMenu is an application rather than an applet, and the video capture feature is enabled.

Info

In remote video mode, the overlays for changes in volume and the like are not rendered.
In the alternate mode, the overlays also are rendered.

Info

By specifying debug set-force-disable-app-chain-la-recording false in DevMenuCommand (available in NX Add-On 14.1.0 or later environments), library applets started from the app can also be rendered on the EDEV remote video.

For more information about this command, see Development Support Features.


In addition, some of the features are not available in environments that do not have the following DLLs.

  • mf.dll
  • mfplat.dll
  • mfplay.dll
  • mfreadwrite.dll

Info

Input Director starts automatically when you open this window. Depending on the Input Director settings, moving the pointer off the screen while this window is open could activate mouse capture mode. To disable capture mode, in the Input Director options, clear the Toggle when mouse pointer moves to edge of screen check box.


Remote Input

You can perform target operations from the Target Manager.
To operate the controller, click the controller image displayed in the view or connect it to the PC.
That input is recognized as either the input coming from the existing controller or the additional controller, depending on the connection status of the controllers.
If the controller type is the Joy-Con dual-controller grip, it is treated as operations with the Joy-Con attached to the console.
Also, if you start it with the Remote Video screen, you can also operate touch controls by clicking the screen.

Input Director starts automatically when you start the screen.
You cannot use this feature if Input Director cannot be found, but you can use this feature by starting Input Director manually in advance.

  • Pressing multiple buttons simultaneously
    1. Hold down the CTRL key and simultaneously click the buttons you want to press.
    2. Release the CTRL key.
    3. When you click one of the buttons you want to press simultaneously, all buttons selected with CTRL are entered.
    4. The selection is cleared when you click a button other than one you wanted to press simultaneously.
  • Holding down the stick
    1. By clicking the stick with the mouse wheel, you can hold down the stick.

To use controllers connected to the PC, xinput9_1_0.dll must be installed.

Info

When the development hardware is in handheld mode with the Joy-Con controllers attached and remote input is set to Joy-Con dual-controller grip, input can be merged with both the attached Joy-Con controllers and the remote input as 1P. These controllers are recognized as separate controllers in all other combinations.

Info

Input Director starts automatically when this window opens. Depending on the Input Director settings, moving the pointer off the screen while this window is open could activate mouse capture mode. To disable capture mode, in the Input Director options, clear the Toggle when mouse pointer moves to edge of screen check box.

For more information about common problems, see Remote Input Troubleshooting.

Power ON and Power OFF

You can control the hardware power from the Power ON and Power OFF menus.
Power ON is not currently supported for EDEV. Start hardware by pressing the POWER Button.

Using From Windows Applications

Preparation

  1. From the Start menu, start Target Manager.
  2. If a warning appears that it is being blocked by the firewall, allow communications.
    • Target Manager conducts network communications with applications.

Connecting to Windows Applications

  1. Start a Windows application that uses HTC / HTCS features.
  2. When nn::htc::Initialize() or nn::htcs::Initialize() is called, it is automatically registered with Target Manager. A "Connection Established" message is displayed after a successful connection is made.

Compatibility

Introduction

Target Manager and the modules that depend on the application are included in different packages and updated at different times.
As a result, environments that deal with multiple versions may not work properly when versions become mismatched.
This section describes how to update to a properly working environment.

Structure and Compatibility

The figure depicts the structure of Target Manager installed in the Program Files directory and the dependent modules, and their compatibilities.
If you have the latest versions of both VSI and Target Manager, you can use other versions of the various components.

However, only compatibility between the three major versions is supported. For example, when using Target Manager 7.x, it may not be possible to use FW 4.x or earlier.
When such a target is connected, a warning icon is displayed in the information column of Target Manager. Consider changing the Target Manager or firmware version if these conditions are applicable.

Operating Multiple Devices

The specifications place no upper limit on the number of hardware devices that can be operated by Target Manager.
However, operations can be tested for up to 14 devices, and problems may arise when more than this number of devices are operated at the same time.

Troubleshooting

An "Unable to load 'Application'" message is output to the log, and I can't start the application.

Right-click the target, select Reset, and then try starting the target again.

It says it cannot find msvcp120.dll and msvcr120.dll.

Install those files from the Visual C++ Redistributable Packages for Visual Studio 2013.

SDEV disconnect is not detected when a LAN cable is disconnected.

To prevent accidental disconnection due to LAN congestion, Target Manager does detect LAN cables being disconnected. To disconnect properly, right-click the applicable device, select Disconnect Target, and then disconnect the cable.

Remote input is not working.

See Remote Input Troubleshooting.

Tips

Receiving Log Metadata

You can connect to the @JsonLog htcs port to receive a log, in JSON format, containing the metadata. For the port number of the @JsonLog htcs port, either see the HTCS entry view, or use the Nintendo.Tm.TargetManager.GetHtcsEntryEndPoints.

Receiving Separate Logs for Each Target

Behavior Effective in NX Add-On 7.1.0 and Later

Use ControlTarget read-log [-t Target Name] .

Behavior Effective From NX Add-On 3.0.0

  1. The HTCS entry port can be fixed by using HTCS Port Mapping. Open Views > HTCS Entries.
  2. Using telnet, connect to the open port that is indicated by the entry with a port name of @Log.
  3. The logs corresponding to the target can now be received.

The @Log port can also be discovered by using the ControlTarget list-service command.

You can also use Log Viewer filtering to separate and receive logs for each target.

Reporting Bugs

If you encounter a bug while using Target Manager, please submit a bug report and include the .dmp file that was generated in the %TEMP%\Nintendo\Oasis\TargetManager directory.
