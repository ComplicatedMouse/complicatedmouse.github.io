Device

Overview

Device indicates a target physical GPU device.
Because various other objects in gfx request a device for initialization, a device must first be initialized to use them.

Unlike several low level graphics APIs, the gfx library does not have a global state such as "current devices." A device in gfx is explicitly managed on the application side as an object.

When a device is used for initialization of another object, the initialized object is associated with that device. Using that object with another device causes an error to occur.

Information

The device information is defined as nn::gfx::DeviceInfo.

API Version

Specify nn::gfx::ApiMajorVersion and nn::gfx::ApiMinorVersion in SetApiVersion.

Debug Mode

Use SetDebugMode to specify whether to initialize the device in debug mode.

Use DebugMode_Disable to disable the debug feature. Use DebugMode_Full to enable all debug features.
When DebugMode_Enable is specified, only the standard debugging features are enabled. Debug features that significantly affect performance are not enabled.

Object

The device object is defined as nn::gfx::Device.

Initialization and Destruction

Unlike most other gfx classes, devices do not require a device for initialization.

Initialization is performed with Initialize, and destruction is performed with Finalize.

Single Device Restriction

Currently, no feature has been provided for handling multiple physical GPU devices, and a device indicates a specific physical GPU device.
There is no feature for specifying a physical GPU device to be initialized and listing the physical GPU devices that can be used.
