Project Configuration File

Overview

The project configuration file is written and managed by the programmer and describes the EffectMaker settings that are shared throughout the project.
The configuration file is the starting point for loading various effect settings that are used in the application.
The file is used both when editing data in EffectMaker and during binary conversion.

The main settings that can be configured are as follows.

  • Custom feature settings
    • Path to the custom shader settings file
    • Path to the custom action settings file
    • Path to the extended emitter parameters settings file
  • UI label settings
    • Emitter set user data
    • Custom settings
  • Shader compilation settings
    • Additional preprocessors
  • Draw path settings
  • Application rendering settings
  • File paths
    • Path to the preset folder
    • Setting for linking with the version management tool
  • Other
    • Path to documentation
    • Standard gravity settings
    • The binary name for previewing to the file path

For more information about settings, see Settings.

How to Use

Configuring Settings From the EffectMaker Option Dialog Box

You can configure settings from the project configurations file path in the EffectMaker option dialog box.
For more information, see Project Settings in the EffectMaker documentation.

Even if the following settings are changed with options,

  • Linear Workflow
  • Custom Shaders
  • Custom Actions

the project configuration file will not be updated when you exit EffectMaker.
When changing these settings, you need to edit the configuration file itself.

Info

You can export the current settings in the project configuration file using Project Settings in the EffectMaker options.
You can use this feature to create a template of the project configuration file.

Configuring Settings From Arguments at EffectMaker Startup

Use the --team-settings-file option to pass the project configuration file when EffectMaker starts.

EffectMaker.exe --team-settings-file ProjectConfig.xml

For more information, see the EffectMaker Features page.

Configuring Settings From Binary Converter Arguments

You can also specify the project setting file using the --team-settings-file option at binary conversion time.

EffectMaker.exe --team-settings-file ProjectConfig.xml --convert-list ConvertList.xml --output EffectBinary.ptcl --api-type Gl --code-type Source --tile-mode Linear

For more information about binary conversion, see Binary Converter.

Notation Method

File Format

The file format must satisfy the following.

  • XML format
  • UTF-8 character code
Info

The file path descriptions can use the following.

  • Absolute paths
  • Relative paths from the project configuration file


Environment variables can also be used. Write descriptions as follows, using the same format used for batch files.

%APPLICATION_ROOT%

Texture Origin Settings 
TextureOriginMode

Configures the texture origin. Possible values are as follows.

LowerLeft
 Lower-Left Origin for Texture Coordinates 
UpperLeft
 Upper-Left Origin for Texture Coordinates

Changing the setting changes the projection transformation matrix used in the shaders.
If the setting is not configured correctly to match the setting in the application, rendering that uses the refraction shader framebuffer and the like will be corrupted.

Info

For the foreseeable future, compatibility will be maintained if you switch the origin using AdditionalShaderPreprocessorDefinitions from NX Add-0n 4.1 or earlier.
However, using the new method of specifying TextureOriginMode is recommended.


Related Links

  • Switching the UV Origin of the Framebuffer and Depth Buffer 
<TextureOriginMode>LowerLeft</TextureOriginMode>
Depth Mode 
DepthMode

Configures the depth mode. Possible values are as follows.

NearIsMinusW

Z-position of the near clipping plane is -W.
NearIsZero

Z-position of the near clipping plane is 0.

Changing the setting changes the projection transformation matrix used in the shaders.
If the setting is not configured correctly to match the setting in the application, rendering of soft particles and the like using depth buffer textures will be corrupted.

 
<DepthMode>NearIsMinusW</DepthMode>
File Paths
  • Path to the preset folder.
  • Setting for linking with the version management tool.
 
FilePaths

Specifies user-specified file paths.

  • Preset folder path
    • Path to the preset folder directory.

The following file paths for hook processing are used for purposes such as connecting to the version management tool.

  • Pre-File Save Command
    • Path to the batch file to call before the file is saved.
  • Post-File Save Command
    • Path to the batch file to call after the file is saved.
  • Pre-File Delete Command
    • Path to the batch file to call before the file is deleted.
  • Post-File Delete Command
    • Path to the batch file to call after the file is deleted.
  • Specifying the pre-file open command in EffectMaker
    • Path to the batch file to call before the file is opened.
 
<FilePaths>
  <!-- Path to the preset folder -->
  <PresetFolderPath>%PRESET_DIRECTORY_PATH%</PresetFolderPath>
  <!-- Setting for linking with the version management tool -->
  <PreSaveCommandFilePath>%EFFECT_HOOK_SCRIPT%\PreSaveCommand.bat</PreSaveCommandFilePath>
  <PostSaveCommandFilePath>%EFFECT_HOOK_SCRIPT%\PostSaveCommand.bat</PostSaveCommandFilePath>
  <PreDeleteCommandFilePath>%EFFECT_HOOK_SCRIPT%\PreDeleteCommand.bat</PreDeleteCommandFilePath>
  <PostDeleteCommandFilePath>%EFFECT_HOOK_SCRIPT%\PostDeleteCommand.bat</PostDeleteCommandFilePath>
  <PreOpenCommandFilePath>%EFFECT_HOOK_SCRIPT%\PreOpenCommand.bat</PreOpenCommandFilePath>
</FilePaths>
Other Document Path DocumentPath Use to set the path associated with a document.
If no value has been set, Documents/index.html in the NintendoSDK package will be referenced automatically. However, if the document has been placed elsewhere, you can specify this path to switch to the document associated with that path.		 
<DocumentPath>%NINTENDO_SDK_ROOT%\Documents\index.html</DocumentPath>
