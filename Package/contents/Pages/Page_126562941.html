Composite Fonts

Composite fonts are multiple fonts combined and handled as a single font.

Composite fonts are one of the most basic features used when working with scalable fonts in LayoutEditor and game applications and can be used even when working with scalable fonts that only include a single font.
Fonts that can be combined are scalable fonts (TTF/OTF/BFTTF) and bitmap fonts (BFFNT).

Example of Using Composite Fonts

For example, it is possible to realize the following combinations.

  • Combine only the number part of a system font (BFTTF) with a scalable font and use them as a single font.
  • Simply load one scalable font.
  • Combine the extended characters area of a scalable font with bitmap font pictograms.

Terminology

The following table summarizes terminology descriptions and filename extensions for composite fonts.

Composite Fonts A font that handles a combination of multiple fonts as a single font. (Overall term for multiple scalable fonts and font pairs.)
Multiple Scalable Fonts A composite font that combines multiple scalable fonts as a set.
Font Pairs A font comprised of two fonts combined. Scalable fonts and bitmap fonts can be combined.
Scalable Fonts When displaying scalable fonts, you can scale the font to an optional size. TTF/OTF/BFTTF
Bitmap Font

Bitmap fonts are displayed at a fixed size that is specified when the font is created.

Using Composite Fonts

This section describes how to use scalable fonts.

Creating Composite Font Definition (FCPX) Files

To use a composite font, a dedicated composite font definition file (FCPX file) that defines the fonts and font settings to load must be created.

File Content

The following items should be defined for multi-scalable fonts, for example.

  • Overall font size
  • For each font:
    • Bold weight
    • Border thickness
    • Font index (font number used for TTC fonts)
    • Font file path
    • Range of characters used

Composite Font Definition File Example

The composite font definition file must be written in XML format and saved in UTF-8 encoding.
The paths can be either absolute paths or relative paths from the font definition file.

<?xml version="1.0"?>
<complexFontDescription version="1.0.0">
  <subFont>
    <multiScalableFont size="40">
      <scalableFontDescriptionSet>
        <scalableFontDescription boldWeight="0" borderWidth="2" ttcFontIndex="0">
          <path>font/alphabet.ttf</path>
          <charCodeRangeSet>
            <charCodeRange first="65" last="90" />
            <charCodeRange first="97" last="122" />
          </charCodeRangeSet>
        </scalableFontDescription>
        <scalableFontDescription boldWeight="0" borderWidth="4" ttcFontIndex="0">
          <path>font/kanji.ttf</path>
          <charCodeRangeSet />
        </scalableFontDescription>
      </scalableFontDescriptionSet>
    </multiScalableFont>
  </subFont>
</complexFontDescription>

For more examples, see the composite font sample files (FCPX files) in the Samples\Sources\Resources\Ui2d\ComplexFont\ directory.

Font Selection

When multiple fonts are listed, the font at the top of the fonts matching the range of the <charCodeRange> that includes the character glyphs to be rendered is selected.

For example, in the case of the previous example, alphabet.ttf is used for characters with text codes in the ranges from 65 to 90 and 97 to 122 (corresponding to the alphabet), and kanji.ttf is used for all other characters.

kanji.ttf is used for characters that are in the ranges from 65 to 90 and 97 to 122, but are not included among the glyphs in alphabet.ttf.

Using the Load Composite Font Definition File (FCPX) by Loading It Into LayoutEditor

After a composite font definition file (FCPX file) has been created, you can specify it for text box panes just like with BFFNT files.
Use the following procedure to load a composite font definition file.

  • When creating a text box pane, specify the composite font definition file as the font filename.
  • Open the property window of the created Text Box pane and specify the composite font definition file using the button to the right of Font Information - Font Filename.

If a composite font definition file has already been loaded in the layout, it is also possible to specify that font.

Using the FCPX File by Loading It Into the Game Application

Convert the FCPX file to binary and use it by loading it into the game program.
