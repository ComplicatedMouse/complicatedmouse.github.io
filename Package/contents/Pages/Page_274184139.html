nn::g3d News ()

Updated Intermediate File Version

The file version for 3D intermediate files was updated to version 4.0.0. For the major changes, see the update history in 3D Intermediate File Specification Manual. This includes changes that affect intermediate file filter scripts, such as deletions of elements and additions of required attributes.

Make sure that you update existing intermediate files using Intermediate File Updater. This intermediate file version update does not affect the binary resource files (BFRES files) output by 3D Binary Converter.

nn::ui2d News ()

Fixed Bug Where Child Panes Disappeared Under Some Conditions

There was a bug where child panes that were supposed to be drawn disappeared from the runtime under the following conditions.

This fix goes into effect in NintendoSDK 5.0.0 and may cause panes that had previously disappeared to be displayed.

Child panes disappeared from the runtime in versions earlier than NintendoSDK 5.0.0 if all of the following conditions were satisfied.

  • The Apply transparency to children option is enabled in the parent pane.
  • The transparency of the parent pane is a value other than 255.
  • The Apply transparency to children option is disabled in the child pane.
  • The transparency of the child pane is 0, or the pane is hidden.
  • The grandchild pane is displayed.

The following figure is an example of rendering a layout in the 4.4.0 runtime that satisfies the conditions.

Grandchild panes that should be displayed are hidden.

Detecting Affected Panes

Use Tools/Graphics/LayoutTools/SearchInfluenceAlphaChild.exe to detect panes that are potentially affected.

When you specify all folders that contain all flyt and flyt parts in the root folder, all grandchild panes that satisfy the following conditions are listed.

  • The Apply transparency to children option is enabled in the parent pane.
  • The Apply transparency to children option is disabled in the child pane.

Transparency and hide settings are ignored because these settings could be changed by the animation or program. Panes that are not affected are listed.

Of the panes that are detected, check that there are no cases in which the transparency and hide settings would cause problems.

In reality, cases that satisfy conditions that would result in a problem are rare. In many cases, almost all of the listed panes cause no issues.

Fix

To fix panes that would cause a problem, enable the Apply transparency to children option for the listed panes. If the appearance does not change, this is the simplest solution.

However, note that the transparency of the child panes may be affected when you enable the Apply transparency to children option.

You can also disable the Apply transparency to children option for the parent pane of the listed panes.

Note that doing this may also affect the transparency of the panes under that parent pane.

nn::atk News ()

Added Feature for Making Bus Routing Configuration More Flexible (for Evaluation Purposes)

Previously, bus routing settings only supported four 6ch buses. Added a feature that allows users to configure bus routing settings in a more flexible way using submixes (for evaluation purposes).
This feature allows users to increase the number of buses and configure bus routing settings that use more than four buses.

For more information, see the Submix page in the Programming Guide, the AtkSubMix sample, the Sample Backgrounder, and the following API references.

  • nn::atk::SubMix
  • nn::atk::FinalMix
  • nn::atk::OutputReceiver
  • nn::atk::SoundSystem::SoundSystemParam::enableCustomSubMix
  • nn::atk::SoundArchivePlayer::SetDefaultOutputReceiver()
  • nn::atk::SoundHandle::SetSend()

Modified the Stream Sound Binary Conversion Specifications

Previously, loop information was disabled for stream sound waveform files in which both region information and loop information were configured. Now, both the region information and the loop information are output.

Previously, if you wanted to call nn::atk::SoundArchivePlayer::ReadStreamSoundRegionDataInfo() to read the region information from a stream sound using looped playback without using the stream jump feature, it was necessary to prepare a waveform that only included the loop information and a waveform that only included the region information. Now, you can do this using a single waveform.

However, like before, it is not possible to use both the stream jump feature and looped playback at the same time.
Due to this change, if your code includes any uses of the stream jump feature for waveforms including loop information, the library will stop on an assertion failure during playback.
More specifically, the library stops on an assertion failure if you specify a region callback when playing a stream sound including loop information.

Updated Some Binary File Versions

Versions of the following binary files have been updated.

  • The binary stream file (BFSTM) version has been updated to 0.6.2.0.
  • The binary prefetch stream binary (BFSTP) version has been updated to 0.5.2.0.

Compatibility is not broken by this latest update. You do not need to convert the binaries again. If you want to use both region information and loop information that were covered in "Modified the Stream Sound Binary Conversion Specifications," you will need to reconvert the binary.
