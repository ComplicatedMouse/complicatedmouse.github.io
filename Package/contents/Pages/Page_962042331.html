Experimental Features Tab

This tab contains options that control the profiler's experimental features.

Enable Thread-Centric Data Views

Controls the Thread-Centric Data Views feature. When this feature is enabled, experimental versions of the Functions and Threads tabs will be displayed in the left window, and the experimental Call Tree and Timeline tabs will be displayed in the right window when a profile has been loaded. The data can be grouped into one of three function group modes, allowing the data to be summarized and analyzed in multiple ways. 

Function Grouping Modes

The function grouping mode can be changed in the mode dropdown of the new Functions and Call Tree experimental tabs, or by clicking on the Type column header in the Functions tab. Changing the mode will control how data in the Functions and Call Tree tabs are organized. The modes are listed below.

Function then Threads

In this mode, the items at the root of the list are functions. The function percentage or time represents the overall totals for each function, regardless of what thread or core it was running on. The items in the next level of the hierarchy are threads. The thread percentage or time represents the function's data by thread, regardless of which core the thread ran on. The items in the last level of the hierarchy are cores, which have the data for the specific function on the specific thread for the specific core.

Thread then Functions

In this mode, the items at the root of the list are threads. The thread percentage or time represents the overall totals for each thread, regardless of what functions it was running or what core it was running on. The items in the next level of the hierarchy are functions. The function's data represents the thread's data by function, regardless of which core the thread and function were running on. The items in the last level of the hierarchy are cores. The core data represents the data for the specific thread running the specific function on the specific core.

Core then Threads

In this mode, the items at the root of the list are cores. The core percentage or time represents the overall totals for each core, regardless of what threads or functions were running on each core. The items in the next level of the hierarchy are threads. The thread's data represents the aggregate of all functions that were observed to be running on the specific core and thread. The items in the last level of the hierarchy are functions. The function data represents the data for the specific function running on the specific thread and core.
