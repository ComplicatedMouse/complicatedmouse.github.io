Introducing SoundMaker

Introducing SoundMaker

SoundMaker is a Windows application for creating sound data to use in the NintendoWare audio development environment.

If an application includes the atk::viewer library, you can use in-game editing by connecting the application to SoundMaker.

Info

For more information about atk::viewer and in-game editing, see the In-Game Editing page.

SoundMaker Overview

In the NintendoWare audio development environment, sound data is normally handled as a single binary file called a sound archive.
Sound data used as resources for the sound archive is created by using a wave editor tool or the sequencer of your choice.
SoundMaker creates a sound archive from sound data which is then used by the runtime library of the NintendoWare audio development environment.
SoundMaker can check sounds on actual system hardware that is connected to a PC by using a dedicated sound program called AtkPlayer. In addition, you can use emulation on the computer to check some sounds during the creation process.

Main Features of SoundMaker

This section describes the main features and characteristics of SoundMaker.

Creating Sound Projects and Exporting Sound Archives

Projects in SoundMaker are called sound projects. Each sound project can reference more than one collection of sounds, called a sound set. A single sound project corresponds to a single sound archive.
Sound projects are saved in files having the extension .fspj (indicated hereafter as sound project files), while sound sets are saved in files having the extension .fsst (indicated hereafter as sound set files).

Creating Stream Sounds

This type of sound is created using sound sets. Waveform files are the resources for stream sounds. Use an existing waveform editing application or DAW software to prepare the waveform files as AIFF or WAV files.
Waveform data is stereo or monaural, 16-bit or 8-bit. Any sample rate can be selected. Stream sounds are played back (by using the nn::atk library) while loading the minimum amount of data necessary from the files.

Creating Wave Sounds

This type of sound is created using sound sets. First create a wave sound set group, and then create wave sounds for the individual wave sound sets. The wave data used as a wave sound resource is the same data that is used for stream sounds.
To play wave sounds using the nn::atk library, you must first load the waveform data as wave sound sets into memory.

Creating Sequence Sounds

This type of sound is created using sound sets. The sequence data used as the resources can be either standard MIDI files or text sequence files (FSEQ files), which are written in plain text.
Prepare standard MIDI files by using your sequencer, and prepare text sequence files by using your text editor. For information about coding control changes for MIDI files and sequence commands for text sequence files, see the Sequence Data Manual.

Creating a Bank and Specifying Sequence Sounds

Use sound sets to create banks. Specify a bank for each sequence sound. The bank functions as a tone when playing a sequence sound.
Bank data to be used as a resource supports special bank files (with the .fbnk filename extension).

Editing Bank Files

Bank files are edited on the Bank tab. The Bank tab is a pane within the main window and is dedicated to editing bank files that are the materials of the banks. For each bank file, you can create more than one voice, each called an instrument. Instruments are placed on the instrument list. The program number that corresponds to the MIDI program change is defined.
Instruments reference waveform files as resources. The waveform data in resources for bank files is the same data that is used with stream sounds or wave sounds.
To play banks using the nn::atk library, you must first load the waveform data as instruments into memory, similar to the requirement for playing wave sounds.

Creating 3D Sounds

3D sounds are created by using a sound set. Operations when playing a sound using 3D sound are set for each sound.

Creating a Player and Specifying a Player for Each Sound

Create a player by using the sound set, and then specify a player for each sound.

Creating Groups and Registering Sound Data With Groups

Create a group by using a sound set, and then register each sound with the created group as an item.
The nn::atk library loads the playback data and waveform data required for playing sounds into memory at the group level.

Importing Existing File Formats

Imports projects, sound sets, bank files, and effect sound definition files (STXT files) from different platforms to the sound project.

An STXT file is an effect sound definition file that is used by the sysdolphin audio library. This library is used as the development environment for the Nintendo GameCube game console. This feature is used when porting game projects created using Sysdolphin.

Creating Sound Archives

A sound archive is created from a Sound Project. All referenced files use a special converter.

Sound Verification Through PC Emulation

SoundMaker supports the playback of instruments by using PC emulation. This allows you to check sounds to a certain degree even without a development kit.
PC emulation can also be used to play MIDI signals connected to the computer.

Starting AtkPlayer and Checking Sounds on Actual Hardware

Sound archives created with SoundMaker can be checked by starting AtkPlayer from inside the tool.
