Composite Font Definition File Format

Composite Font Definition File Format

The composite font definition file is an XML file.

This page describes elements included in the composite font definition file.

Related Links For an overview of composite fonts, see Composite Fonts.

<complexFontDescription>

The root element of the file.

Attributes

Name Type Description
Version string The version of the composite font definition file. Specify 1.4.0.

Description

The user config file includes the following elements.

  • <subFont>: Composite font settings. One setting will always appear.

<subFont>

Composite font settings.

Description

Include one of the following elements.

  • <pairFont>: The pair font setting.
  • <bitmapFont>: The bitmap font setting.
  • <scalableFont>: The scalable font setting.
  • <multiScalableFont>: Multiple scalable font settings.

<pairFont>

The pair font setting.

Description

The user config file includes the following elements.

  • <firstFont>: Setting for the first pair font element. One setting will always appear.
  • <secondFont>: Setting for the second pair font element. One setting will always appear.

<firstFont>

Setting for the first pair font element.

Description

Include one of the following elements.

  • <pairFont>: The pair font setting.
  • <bitmapFont>: The bitmap font setting.
  • <scalableFont>: The scalable font setting.
  • <multiScalableFont>: Multiple scalable font settings.

<secondFont>

Setting for the second pair font element.

Description

Include one of the following elements.

  • <pairFont>: The pair font setting.
  • <bitmapFont>: The bitmap font setting.
  • <scalableFont>: The scalable font setting.
  • <multiScalableFont>: Multiple scalable font settings.

<bitmapFont>

The bitmap font setting.

Description

Includes one of each of the following elements.

  • <comment>: Comments. One of these elements at most.
  • <path>: Specifies the font file path.
  • <charCodeRangeSet>: Specify the range used to display the characters in this font.

<scalableFont>

Scalable font settings.

Attributes

Name Type Description
size float Font size. Cannot be omitted.
alternateChar unsignedInt The alternate character as a decimal value. When omitted, it uses 63 (the character for '?').
lineFeedOffset int Specifies the offset value between rows. Set to 0 if omitted.

Description

The user config file includes the following elements.

  • <scalableFontDescription>: Scalable font settings. One setting will always appear.

<multiScalableFont>

Multiple scalable font settings.

Attributes

Name Type Description
size float Font size. Cannot be omitted.
alternateChar unsignedInt The alternate character as a decimal value. When omitted, it uses 63 (the character for '?').
lineFeedOffset int Specifies the offset value between rows. Set to 0 if omitted.

Description

The user config file includes the following elements.

  • <comment>: Comments. One of these elements at most.
  • <scalableFontDescriptionSet>: Scalable font settings. One setting will always appear.

<scalableFontDescriptionSet>

Array elements for scalable fonts.

Description

The user config file includes the following elements.

  • <scalableFontDescription>: Scalable font settings. One or more settings will appear.

<scalableFontDescription>

Scalable font settings.

Attributes

Name Type Specifiable Range Description
boldWeight float 0.0 to 1.0 Sets the bold weight. (0.0 indicates normal thickness.) Cannot be omitted.
borderWidth int 0 to 10 Sets the frame weight (drawing the frame is turned off if this is set to 0). Cannot be omitted.
ttcFontIndex int
Sets the font index when using TCC fonts. Cannot be omitted.
scaleWidth float
Specifies the scale of character width. If omitted, it will become 1.0.
scaleHeight float
Specifies the scale of the character height. If omitted, it will become 1.0.
ignorePalt boolean
Specifies whether to ignore the information used to make double-byte characters in OpenType fonts proportional when rendering text. If omitted, it will be false.
deleteBearingX boolean

Specifies whether to delete the bearings (margins) to the left and right of the glyph. For fonts with overly large margins, set true to adjust bearingOffsetX and mitigate this issue.

If omitted, it will be false.
bearingOffsetX int
Adjusts the bearing (margins) to the left and right of the glyph. If omitted, it becomes 0.
forceMonospacedEnabled bool
If true, forcibly renders as a monospaced font. If omitted, it will be false.
forceMonospacedWidth int
Specifies the character width when forceMonospacedEnabled is true. If omitted, it becomes 0.
baselineOffset int
Adjusts the vertical baseline offset. If omitted, it will become 0.
excludeFromHeightCalculation bool
The specified font is excluded from the calculation of line height. If omitted, it will be false.

Description

The user config file includes the following elements.

  • <comment>: Comments. One of these elements at most.
  • <path>: Specifies the font file path. One setting will always appear.
  • <charCodeRangeSet>: Specifies the range of characters representing this font. One setting will always appear.

<path>

Sets the font file path.

Description

Specifies the font file path. You can use the FFNT and BFFNT formats when the parent element is <bitmapFont>. You can use the TTF, TTC, OTF, BFTTF and BFOTF formats when the parent element is <scalableFontDescription>.

The paths can be either absolute paths or relative paths from the composite font definition file. The absolute path may include environment variables surrounded by percent signs (%).

<charCodeRangeSet>

Array elements for setting the displayed character range.

Description

The user config file includes the following elements.

  • <charCodeRange>: Specifies the range of displayed characters. Zero or more may appear. Must not exceed 16.

<charCodeRange>

Sets the displayed character range.

Attributes

Name Type Description
first unsignedInt Sets the start of the character range as a decimal value. Cannot be omitted.
last unsignedInt Sets the end of the character range as a decimal value. Cannot be omitted.

<comment>

Comments. This element has no effect on the binary file.

Description

Strings used for comments.
