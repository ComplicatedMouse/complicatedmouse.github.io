Profiler-NX-Scenario

Overview

Sets the profiling scenario to use for profiling. The options provided will match up with the options that are specified manually in the Scenario Selection Window.

Profiler-NX-Scenario [-ScenarioType] <ScenarioType> -NssFile <String> [-Autofill]
Profiler-NX-Scenario [-ScenarioType] <ScenarioType> -NssFile <String> [-Autofill] -NspFile <String> [-Args <String>] [-WaitForParsing]
Profiler-NX-Scenario [-ScenarioType] <ScenarioType> -NssFile <String> [-Autofill] -ProgramId <String>


[-ScenarioType] <ScenarioType>

The types of scenarios that can be set. These values correspond with the options available in Scenario Selection.

    • AttachToProgramId: Used to specify the Attach scenario from the Scenario Selection window.
    • AttachOnLaunch: Used to specify the Attach on Launch scenario from the Scenario Selection window.
    • Launch: Used to specify the Launch scenario from the Scenario Selection window.
    • InProcess: Used to specify the In-Process Only scenario from the Scenario Selection window.

-NssFile <String>

The path to the NSS file that should be synced with. This field can be an empty string.

[-Autofill]

Optional. A flag specifying that the files for the scenario should be filled in automatically. This is similar to pressing the Autofill button for each entry, if available. If attempting to autofill the NSS file, Profiler-NX-WaitFor InProcessAvailable should be used prior this.


AttachOnLaunch, AttachToProgramId

The form of the Cmdlet when ScenarioType is AttachOnLaunch or AttachToProgramId.

Profiler-NX-Scenario [-ScenarioType] <ScenarioType> -NssFile <String> [-Autofill] -ProgramId <String>


-ProgramId <String>

The program ID that the profiler should attach to. This should be a 16-digit hexadecimal number. The number may start with "0x".


Launch

The form of the Cmdlet when the ScenarioType is Launch.

Profiler-NX-Scenario [-ScenarioType] <ScenarioType> -NssFile <String> [-Autofill] -NspFile <String> [-Args <String>] [-WaitForParsing]


-NspFile <String>

The path to either the NSP or NSPD_ROOT file that should be launched.

[-Args <String>]

Optional. The string of arguments that will be passed to the launched executable.

[-WaitForParsing]

Optional. Request the profiler wait for the NSS parsing to occur before attempting to start the executable. This option is useful when attempting to profile the start of an application.


InProcess

The form of the Cmdlet when the ScenarioType is either InProcess.

Profiler-NX-Scenario [-ScenarioType] <ScenarioType> -NssFile <String> [-Autofill]


There are no changes from the common arguments.


Examples

Basic Usage

Provided here are some basic usage examples.

Profiler-NX-Scenario InProcess -NssFile $nssFile -Autofill
Profiler-NX-Scenario AttachToProgramId -NssFile "C:\example.nss" -ProgramId "0x1234567890abcdef"
Profiler-NX-Scenario Launch -NssFile "C:\example.nss" -NspFile "C:\example.nsp" -Args "--example-argument"
