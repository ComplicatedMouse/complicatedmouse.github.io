Getting Target Manager Connection and Disconnection Events

Overview

Functionality for getting events notifying connection and disconnection with Target Manager.

You can use this feature to wait for connection to or disconnection from Target Manager.

The obtained synchronization wait event is a SystemEvent class object. For more information about the SystemEvent class, see the following page.

  • SystemEvent Class

How to Use Connection and Disconnection Events

Use the following functions to get connection and disconnection events.

  • To get connection events
    • nn::htc::BindHostConnectionEvent
  • To get disconnection events
    • nn::htc::BindHostDisconnectionEvent

The htc library must be initialized to use these functions. For this reason, call nn::htc::Initialize() first.

For more information, see the following sample program that demonstrates how to use these functions.

  • Sample program for getting the connection status with the host

Behavior During Sleep Mode

A connect/disconnect event will not be signaled if the system enters sleep mode while connected to TargetManager.
However, a disconnect event is signaled if the system is disconnected while sleeping, such as due to a network disconnection.

Important Information

The timing at which the system events, obtained by using this feature, are cleared, is controlled by the system. Do not clear the events in a user-controlled way.
