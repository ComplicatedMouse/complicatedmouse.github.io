Glossary

Buffer

The memory on the dev kit used to store the profile data.

Call Tree

A tree constructed from individual sample callstacks, showing the calling structure of the game during the time it was profiled. This is slightly different from a call graph, since there are no cycles. In a call tree, the same function can appear multiple times, based on where it was called.

Callstack

The list of function calls stored in the application's stack.

Counter

Data from a CPU counter (performance counters) or user data manually logged through the profiler API.

Heartbeat

A periodic event on a particular core that can be manually logged (through the profiler API) or inferred automatically by the profiler. A heartbeat is similar to a frame marker, but more generic in meaning since it can represent any type of periodic event. Heartbeat markers can be viewed in the Timeline and can be selected in the Heartbeats tab of the Ribbon Bar.

Icicle Graph

An icicle graph is a hierarchical display of the sampled callstack over time, with the root of the callstack at the top and the leaf function at the bottom.

In-Process

Refers to operations that happen within a user's application. Most commonly refers to taking a profile from within the user's application. See In-Process and Out-of-Process Profiling Modes for more information on In-Process profiling.

Out-of-Process

Refers to operations that happen in another process outside of the user's application. Most commonly refers to taking a profile from the profiler process. See In-Process and Out-of-Process Profiling Modes for more information on Out-of-Process profiling.

Sampling

Periodically stopping the game and recording the current function or callstack.

Sampling Rate

The rate at which samples are taken.

Self

The time spent exclusively in a function, but not any functions it calls (Self = Total - Sub).

Sub

The time spent in the functions called by this function (Sub = Total - Self).

Sync

Connecting with the dev kit and loading the ELF file. Once the profiler is synced, a profile can be taken.

System Classification

Functions are classified as belonging to a particular game system, such as Animation, Graphics, Physics, and so on.

Total

The time spent in a function and all of the functions it calls (Total = Self + Sub).

Tracing

Tracing is a type of data recording that happens when an event occurs or is logged periodically (as opposed to statistical sampling). Examples of tracing in the profiler are Thread State, GPU, Audio, and Power data recording. Instrumentation is a type of tracing, but the term instrumentation is used exclusively in the profiler to refer to function tracing. Counters, when logged within the code, are another type of tracing that happens in the profiler, but note that CPU Performance Counters are periodically sampled along with function sampling.

Units

The type of measured value used to show how much time a function has taken. This can be either Percent, Time, Avg, or Samples.

Unsync

Disconnecting from the dev kit.
