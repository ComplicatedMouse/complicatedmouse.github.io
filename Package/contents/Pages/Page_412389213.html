Debugging Features

Overview

The debugging features support debugging applications using the GFX features. Settings for using the debugging features include the use of external tools such as a graphics debugger.

To use the debugging features, the device must be initialized in debug mode.

Debugging Labels

Debugging label strings can be set for various types of objects. These label strings can be used by external tools such as a graphics debugger.

For objects that can have debug labels set, call SetDebugLabel() to set a label. When SetDebugLabel() is called, any label set previously using SetDebugLabel() is deleted and overwritten by the new label string. Specify a null label to delete a label setting.

If the low-level graphics API for the object does not support debugging labels, SetDebugLabel() may not have any effect.

Debugging Groups

Notes can be attached to the particular scope of commands set in a command buffer, as a debugging group. Such debugging group strings added as notes can be used by external tools such as a graphics debugger.

A debugging group is specified using nn::gfx::CommandBuffer::PushDebugGroup or nn::gfx::CommandBuffer::PopDebugGroup. nn::gfx::CommandBuffer::PushDebugGroup is used to push a debug group on the debug group stack with the specified string, and nn::gfx::CommandBuffer::PopDebugGroup is used to pop the currently active debug group off of the debug group stack. Notes are attached to commands set between these two functions, indicating the debug group. You do not have to alternate between nn::gfx::CommandBuffer::PushDebugGroup and nn::gfx::CommandBuffer::PopDebugGroup, and you can specify hierarchal debug groups. In a single command buffer, use of nn::gfx::CommandBuffer::PushDebugGroup and nn::gfx::CommandBuffer::PopDebugGroup does not have to be balanced, but they must be balanced when finally presented to the queue, otherwise an error will occur.
