Profiler-NX-WaitFor

Overview

Causes the profiler to wait for a specific event before continuing execution. These wait events require a prior call to either Profiler-Connect or Profiler-Sync for proper operation. As of version 1.4, the command waits indefinitely unless a timeout is supplied.

Profiler-NX-WaitFor [-WaitEvent] <NxWaitEvents> [-Timeout <Double>]


[-WaitEvent] <NxWaitEvents>

Events that are NX-specific that can be waited for in the scripting engine.

    • InProcessAvailable: Waits until the In-Process profiling mode is enabled on the Sampled Profile tab.
    • OutOfProcessAvailable: Waits until the Out-of-Process profiling mode is enabled on the Sampled Profile tab.
    • ThreadStateAvailable: Waits until the Thread State option is enabled on the Sampled Profile tab.


[-Timeout <Double>]

Optional. If not specified, the command will wait indefinitely.

Sets the time in seconds that the command should wait for the event to occur before timing out and causing an error to occur. The specified value should be either be greater than or equal to 0 or be -1.0. A value of -1.0 is used to specify an infinite timeout, though [System.Double]::PositiveInfinity can also be specified to indicate an infinite timeout.



Info

Prior to the introduction of the Timeout value in version 1.4, the timeouts were all hardcoded to a value of 1 minute.



Examples

Basic Usage

Provided here are some basic usage examples.

Profiler-NX-WaitFor InProcessAvailable
Profiler-NX-WaitFor -WaitEvent OutOfProcessAvailable
Profiler-NX-WaitFor OutOfProcessAvailable -Timeout 60
