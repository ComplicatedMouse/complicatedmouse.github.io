Instrumented Tab

The Instrumented tab displays all instrumented functions and all instrumented code blocks that were recorded during a profile. The Instrumented tab will only appear if either of these types of data were recorded.

Note

These notes point out differences in the Instrumented tab when the Thread-Centric Data Views experimental feature is enabled. See Experimental Features Tab for additional information.

Instrumented Tab Toolbar

The following controls are in the Instrumented tab toolbar.

Deselect All: Deselects all selected items.

Expand All: Expands all items in the view.

Collapse All: Collapses all items in the view.

Show Only Sort Value: Filters out all items in the view except those that are being sorted on.

Select Similar: When enabled, the selecting or deselecting of an item causes similarly named items to be selected or deselected. Only expanded blocks are evaluated for Select Similar.

Copy: Copies to the clipboard whatever is currently shown using the current sorting and filtering. You can then paste the content into an email, document, or Excel spreadsheet. Because the copied content uses tab delimiters, it will retain the column organization when pasted into an Excel spreadsheet.

Filter: 

  • Cores: Filters the items in the view so that only items found on the selected core will be shown. Only cores with data available in this tab are shown in this drop-down.
  • Threads: Filters the items in the view so that only items found on the selected thread will be shown. Only threads with data available in this tab are shown in this drop-down.

Sort: Items in the instrumented tab can be sorted by the data they contain. The sorting methods that are available depend on the type of data recorded in the instrumented tab.


Note

The Show Only Sort Value, Core Filter, and Sort toolbar controls are not present due to the changes in the tab. Instead, the following toolbar controls have been added.

Expand Selected Only: Collapse and Expand all items in the view so that only parents that have children selected are expanded.

Details: When toggled on, show additional columns in the display.

Selected Only: When toggled on, only show selected items and their parents in the display. This button can only be toggled on if something is selected. If everything is deselected, the button will automatically toggle off.


Instrumented Items

Instrumented items can be expanded to view all of the data that was recorded during the profile. The following measurements should appear for each instrumented item.

  • Function Calls or Blocks: The number of times the function or code block was entered and exited during the profile.
  • Max Recursive Depth: Displays the max recursive depth that was detected during the profile. This measurement only appears if the value is greater than zero.
  • Time per Call or Time per Block: This is the average time per call or block that was measured (time = total time / number of calls).


Note

The view when Thread-Centric Data Views has been significantly changed. Rather than having data displayed as value/name pairs, the data is displayed in a table much like those seen on the Functions and Threads tabs.

Columns

Some of the columns are only displayed when the Details button is toggled on. Clicking on the column header will sort by that column, except for the Type column.

Type

Shows what kind of data is represented on this line of the view.

Block: Shows the combination of the data for named Code Block or instrumented function, regardless of the core or thread that it was used on. It can be expanded to show a breakdown at a per-thread level.

Thread: Shows the named Code Block or instrumented function data based on the thread it was running on, regardless of the core that was executing that thread.

This value does not updating when selecting a time range.

Blocks

The number of times the Code Block or instrumented function was entered and exited. When profile data is loaded, if there is a mismatch in the number of enters and exits, the profiler will automatically balance the code blocks by inserting enters at the beginning of the profile and exits at the end of the profile.

When selecting a time range on the Timeline tab, the values in this column will update. Only blocks that are both entered and exited within the time range are shown in the column.

Avg Time

The average time spent executing the Code Block or instrumented function. When profile data is loaded, if there is a mismatch in the number of enters and exits, the profiler will automatically balance the code blocks by inserting enters at the beginning of the profile and exits at the end of the profile. This may cause a slight reduction in the average time.

When selecting a time range on the Timeline tab, the values in this column will update. Only blocks that are both entered and exited within the time range are used to calculate the average time.

Recursions

Only visible when the Details button is toggled on.

The maximum number of recursions seen by this Code Block or instrumented function. A recursion is defined as a block occurring on the same thread appearing within the time range a block with the same name is already covering.

When selecting a time range on the Timeline tab, the values in this column will update. Only blocks that are both entered and exited within the time range are considered, though the recursive parents do not need to be fully selected.

Cores

Only visible when the Details button is toggled on.

A list of the cores which executed the Code Block or instrumented function.

This value does not update when selecting a time range.

Name

The name associated with the Type column.

Block: The name of the Code Block or instrumented function.

Thread: The name of the thread on which the Code Block or instrumented was seen.

This value does not update when selecting a time range.


Selecting Instrumented Items to Graph

Selecting data fields within each instrumented item will cause them to be graphed within the Timeline tab.

To select similar data fields within all expanded items, enable the Select Similar button in the Instrumented tab toolbar and click on a single data field.

Note

When blocks are drawn in the Timeline tab, they will appear in two sections.

Code Blocks by Thread: A hierarchy view of Code Block and instrumented functions showing the relationship between the blocks on the threads over time.

Code Blocks: A view of when the selected Code Block or instrumented function was executing. If there were any recursions, the height of this graph will increase to show those recursions.

The Select Similar button has been removed.


Alternative Selection Methods

Multiple items can be selected or deselected with Shift+Click (select/deselect an item without holding the Shift key, then hold the Shift key and click on a second item). The range between the two clicks will be either selected or deselected.

To quickly select only a single item and deselect everything else, Alt+Click can be used. This deselects all and then selects the clicked upon item.

Filtering the Instrumented tab

The Instrumented tab can be filtered using the Filter text box at the bottom of the tab. This behaves similar to the Filter text box on the Functions tab.
