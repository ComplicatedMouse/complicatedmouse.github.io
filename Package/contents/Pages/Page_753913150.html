Profiler-NX-ThreadState

Overview

Controls whether the Thread State tracing should be recorded in the profile data or not. This will only enable the feature if the Thread State button is enabled for selection at the time this cmdlet is called. If the Thread State button is disabled, thread state tracing will be disabled.

Profiler-NX-ThreadState [-EnableThreadState] <Boolean>


[-EnableThreadState] <Boolean>

If true, record thread state tracing in the profile data. If false, turn off recording of thread state tracing.

Provided here are some basic usage examples.
