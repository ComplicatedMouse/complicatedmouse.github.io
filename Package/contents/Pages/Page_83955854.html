Overview

FDES (3D DCC plug-in export settings) files are text format files that store the option settings to be used by the export plug-ins provided by NintendoWare for various commercially available 3D DCC tools.
FDES files have a shared format independent of any 3D DCC tool.
The filename extension for FDES files is .fdes.
FDES files can be input/output from the export plug-in. They are also used when exporting intermediate files using a Windows batch file.

Structure

FDES files are made up of keywords that correspond to option settings of the export plug-in and values that represent the setting.
Lines that start with @ represent comments and are ignored when importing to the export plug-in.
Comment lines in the fdes file sample are always output.

The first line of an FDES file must be a comment of the following type.

# NW4F_Export settings

(Include one ASCII space character after the #)

Info

If the first line is invalid, an error results when the Export plug-in loads the file.

Keywords names cannot be used more than once. The earlier table shows the output order.
A forward slash ("/") is used as the delimiter in file and folder paths.
Any number of digits can be output for the fractional part of a floating point number.

Info

In the case of values for poly_reduction_mode, pre_export_script, and post_export_script, single quotation marks (') can be used instead of double quotation marks ("), and exclamation points (!) can be used instead of semicolons (;). Also, &#x27! can be used instead of single quotation marks, and &#x21! can be used instead of exclamation points.

I/O

When exporting an FDES file from the export plug-in, output to an FDES file is possible for all items that can be set in the options window (including items displayed as unavailable).

When importing an FDES file to the export plug-in, only options listed in the FDES file using a valid combination of keyword and value are applied to the export plug-in option settings.
If an invalid keyword or value is included in the FDES file, that keyword is ignored.

If an FDES file in which playback is specified for the frame_range keyword is input to a 3D DCC tool that does not support the concept of playback, the value all is assumed instead.
