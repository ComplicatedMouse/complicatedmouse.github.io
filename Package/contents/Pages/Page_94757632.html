Creating an Independent Build System (Windows Environment)

Introduction

This section is for developers who intend to create their own build system without using the Visual Studio property sheets provided by NintendoSDK. It presents the information needed to create applications for the Windows environment.

Visual Studio Settings Required by the SDK

Runtime Libraries

The Visual C++ runtime library settings used by the NintendoSDK libraries differ depending on the library build type. For more information, see Build Targets and Build Types.

Character Set

All the NintendoSDK libraries are built with settings configured to use the Unicode character set.

SDK Include Directories

The include directories of the NintendoSDK are located in the following places. You must specify these include directories at compilation time.

  • %NINTENDO_SDK_ROOT%\Common\Configs\Targets\BuildTarget\Include
  • %NINTENDO_SDK_ROOT%\Include

For more information about build targets, see Build Targets and Build Types.

SDK Library Directories

The NintendoSDK library directories are found in the following locations. These library directories must be specified at link time.

  • %NINTENDO_SDK_ROOT%\Libraries\<BuildTarget>\<BuildType>

For more information about build targets and build types, see Build Targets and Build Types.

For a list of the libraries in the NintendoSDK, see How the Libraries Are Provided and How to Use Them.

Preprocessor Macros Required by the SDK

Several preprocessor macros are required for switching among the provided features in NintendoSDK. Specify the following preprocessor macros as compiler options.

Required Macros

Always define the following preprocessor macros.

  • NN_NINTENDO_SDK

Marcos Requiring Selection

Based on your build settings, define one of the following preprocessor macros.

Build Type

Define one of these, based on the build type of the NintendoSDK libraries you are using.

  • NN_SDK_BUILD_DEBUG
  • NN_SDK_BUILD_DEVELOP
  • NN_SDK_BUILD_RELEASE
