Notes

Controlling Panes Programmatically

While you can manually configure values and animations for pane parameters directly in LayoutEditor, you can also control panes and pane parameters programmatically by using the functions in the classes for each pane.
For example, you can use the SetTranslate function in the Pane class to move a pane within a layout.

This functionality enables interactive experiences such as one where a pane moves to the position selected by the user.

Rendering States

The layout library specifies blend settings and logical operation settings internally.

Note that all other settings that were configured before drawing with the Draw member function are used as is.
The settings that this has the largest effect upon are the culling and depth test settings.
By default, layouts are drawn without culling and without a depth test, but there are cases where the depth test is preferred if you want to create special effects.

Multithreading

There are no processes for synchronizing threads in the layout library.
In other words, threads are not locked or unlocked in the library.

As a result, you must use separate instances of the layout library for each thread when you want to process multiple threads.
For example, you can simultaneously use functions such as Animate and Draw in different threads as long as there are separate instances of the nn::ui2d::Layout class for each thread.
When doing this, you must also have separate instances of the nn::ui2d::DrawInfo class to pass to the Draw function in each thread.

The nn::ui2d::GraphicsResource class is an exception to this rule, and you can share a single instance of this class among different threads.
For example, the layout library functions normally even if you have only a single global instance of the nn::ui2d::GraphicsResource class in the application and you use the SetGraphicsResource function to set it to multiple instances of the nn::ui2d::DrawInfo class.
