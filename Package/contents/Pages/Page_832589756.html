Content Save Locations

Overview

This section describes the locations where content downloaded by the user is saved.
For more information about the locations where content is saved in the development environment, see Running and Controlling Applications.

Terminology Definitions for Content

This page uses "content" as an overall term for the following three types of data.

  • Application
  • Downloadable content
  • Patches

Content Save Locations

You can currently use the system memory or microSD cards to save content data on the Switch system.
The user cannot select where the content is saved. If a microSD card is inserted into the Switch, the microSD card is prioritized as the destination for saving content. If there is not enough space on the microSD card to save the applicable content, it is saved in the system memory. However, this specification may change in the future.

For applications and patches, the preceding specifications might cause applications and patches to be saved in different locations. If a microSD card is inserted after an application was already downloaded to system memory, the patch data may be saved to the microSD card. As a result, there may be situations where the patch becomes unavailable when the microSD card is removed. However, due to the required version system, an application that has been started with the patch applied does not start without the patch. For more information, see Deleting Patches.

There are also situations where multiple pieces of downloadable content are saved in the Switch. Based on the preceding specifications, these pieces of downloadable content may be saved either to system memory or to the microSD card.
In some cases the storage locations might be different, such as one piece of downloadable content being downloaded to system memory while another piece of downloadable content is saved to the microSD card. As a result, in some situations, removing the microSD card might cause some downloadable content to become unavailable.

Transferring Content

In versions earlier than Firmware 10.0.0-4.0, it was not possible for the user to transfer content between the system memory and the microSD card.

In Firmware 10.0.0-4.0 and later, the user can transfer content between the system memory and the microSD card.
In this case, the application, downloadable content, and patch cannot be transferred individually. Instead, only transferring all content associated with the application together is supported.

You can transfer content from Data Management in System Settings.

Deleting Content

Users cannot delete downloadable software, downloadable content, and patches individually. To delete the content, you must delete all content associated with that application.

To delete content, use Options in the HOME menu or Data Management in System Settings.

Deleted content can be downloaded again from Nintendo eShop if the account that owns the license for that content is associated with the Switch.
