Functionality Differences for Texture Formats of Indirect Textures

How Does Functionality Differ for Texture Formats of Indirect Textures?

The functionality of indirect textures varies depending on their texture formats (or, more accurately, the number of format channels they have).

Number of Texture Channels Format Behavior
1-channel BC4, L8, A8 The same values for s and t are used as indirect offset values.
2-channel BC5, LA8 For indirect offset values, R and A are used as s and t, respectively.
3-channel BC1, RGB565_INDIRECT For indirect offset values, R and G are used as s and t, respectively. The B value is used as the value for alpha elimination after applying indirect.
4-channel BC2, BC3, RGBA8 For indirect offset values, R and G are used as s and t, respectively. The A value is used as the value for alpha elimination after applying indirect.
The B value is not used.

Optimizing Processing of Masks or Alphas in a Pane That Uses Indirect Textures

In some cases, you must add additional textures for the alpha channel in multi-textures to apply masks or alphas to panes that use indirect textures.

Adding additional multi-textures substantially increases rendering loads. You can keep rendering load in check by creating indirect textures that include alpha channels instead of adding textures.

When using 3-channel (BC1) or 4-channel (BC3, RGBA8) texture-format files as indirect textures, the third channel (B component) and fourth channel (A component) are used as alpha channels.

The characteristics of compressed textures may prevent recreation of alpha channels as expected in BC1 textures. Size increases in such cases, so use a 4-channel format such as BC3 textures.
