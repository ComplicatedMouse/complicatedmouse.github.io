Photoshop Plug-In


The Texture Export plug-in now allows you to specify the default output slice.

Specifies the default output target slice when no output option is set within the configuration.

var default_target_slice = "user"; // all / user / selection

If user is specified but the user-specified slice does not exist, it defaults to all.
In the case of selection, all user-defined slices are enabled by default.
(If no user-defined slice exists, only the first automatic slice is enabled.)

Describes the available features and how to use them.

In the Texture Export plug-in, you can now specify <Slice> for Output File Name when Layer Comp Export and Slice Export are both selected.

Use this option when the slices exported from each layer comp do not overlap and you do not want to include the layer comp name in the exported filename.

Texture Export Plug-In

Describes the available features and how to use them.

Improved the export plug-in UI of the dialog box for selecting the slices to export.

This makes it easier to search for target slices when there are multiple slices to export.

  • Added a feature for searching for slices (supports regular expressions, case-insensitive)
    Changed the color of the name of the matched slice.
    When searching using the Search button, the matched check boxes take focus.
    Pressing ENTER in the search box leaves the search box in focus.


  • When slices do not have the default names, indexes are no longer shown in the check boxes.
    (Display the index in the tooltip.)

  • A vertical scrollbar is now shown when there are numerous slices in the vertical direction.
    Previously, if the number of slices in the vertical direction was 33 or greater, the positions of the slices in Photoshop were ignored and the check boxes rearranged to fit in 32 rows.


  • The Enable Specified button now ignores case when determinations are made.

Maya Plug-In

The Import plug-in now has a feature for specifying a namespace for the node being created.

The Namespace option has been added to the Maya Import plug-in as a feature for specifying a namespace for the node being created.
In addition, if only animation intermediate files are being imported and a node has not been selected, the files are imported preferentially to a node in the namespace specified by the Namespace option.

Behavior When Creating the Namespace
Leave blank Create as :model Created using the current namespace.
Specify "test" Create as :model:test Created using test added to the current namespace.
Specify ":test" Create as :test Created using the specified namespace.
Specify "＠file" Create as :model:intermediate file name The namespace is comprised of the current namespace plus the name of the intermediate file.

Click Browse to select an existing namespace, @file, or the filename of the FMD, FSK, or FSN file.

Import Plug-In: Camera and Light Attributes Key Slope Calculation Improvement

Improved how the slopes of keys are calculated for some camera and light attributes in the Maya Import plug-in.
- Camera focal distance
- Light rotation, center of illumination, and penumbra angle
This improvement reduces the margin of error when the animation is played in Maya at a higher frame precision than that at export time.

The improvement is the same as described in the following News item for SDK 10.1.0:Improved the key tilt calculation when creating an animation curve from baked data.

Improved the process for calculating the slopes of keys when creating animation curves from baked data (3D News)

 The process for calculating the slopes of keys when creating animation curves from baked data has been improved.
