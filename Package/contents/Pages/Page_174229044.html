NintendoWare Audio News

This page introduces the changes in the NintendoWare audio development environment for NintendoSDK 0.16.0.

Audio

New Features Under Evaluation

The usefulness of the following new features is currently under evaluation.

Note

Note that there is the possibility that destructive specification changes may be introduced without warning.

  • Downsampling
  • Single playback

As a part of this evaluation, the project (FSPJ) file, sound (FSST) file, and binary sound archive (BFSAR) file versions are updated.
However, as long as you do not use these features, there is no effect on the data size of the binary sound archive (BFSAR) file or the behavior of the nn::atk library.

Downsampling Feature

This feature allows the sample rate set for the sound to be downsampled and included in the sound archive without changing the original wave file.

This feature uses the Downsampling and Sample rate columns in the SoundMaker list.
In addition, the SoX settings are added to the Project Settings dialog box.

Single Playback Feature

This feature restricts simultaneous playback of the specified sound.

This features uses the Single playback and Single playback time columns in SoundMaker.


Spy

The average loudness of waveform data can now be calculated.

Added a loudness meter panel that calculates and displays the loudness of waveform data received from an application.

For more information, see Spy/Spy.exe/Audio Loudness Meter Panel.

Note
  • This feature is currently only supported for waveforms with a sampling rate of 48,000 Hz.
  • Output device volume settings configured using the nn::audio library are not considered in the loudness calculation.
