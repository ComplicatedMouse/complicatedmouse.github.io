Switching Between Programs

Switching Between Programs

NintendoSDK can switch between running programs using the following features.

  • Using nn::oe::RestartProgram() to restart the program itself
  • Using nn::oe::ExecuteProgram() to switch to sub-programs in multi-program applications

When the program calls these functions, the system closes the program that was previously running, and then loads the next program. As a result, switching programs takes the same amount of time as starting the application from scratch from the system menu.

Note

If you call these functions for switching programs, the currently running program is terminated as is, without returning processing, so perform termination processing for the program before calling the function.

Note

The process of switching between programs does not work correctly on programs started from the host PC using tools like Target Manager, VSI, and RunOnTarget, so when you call these functions, such programs fail to restart. To test the operation of program switching, write the programs to system storage or a game card and start the programs from there.

Orchestrating the Switching of Programs

It is important to make it appear to the user that a single application is running, even while it is switching between programs.

Take the time spent switching between programs into account, implement it in a way that appears natural to the user, and be sure to test the user experience.

Visuals During Program Switching

The program stops once when you switch between programs.

The final visual output by the program prior to switching continues to be displayed until the new program starts and begins to be rendered.

Make sure that what is displayed while programs are switching does not seem strange and unnatural.

Sounds During Program Switching

You cannot play audio while programs are switching.

Make sure that you terminate sounds before switching programs.

Keeping the User Account Open

You must keep the user account in the Open state when programs are switching, so there is no change in the user's play status.

Call the nn::account::PushOpenUsers() function before switching programs to have the system save the information for user accounts that are currently in the Open state.

After program switching has completed, call the nn::account::PopOpenUsers() function in the program switched to so that the saved information for user accounts is in the Open state.

Launch Parameters

When switching programs, you can pass launch parameters from the program that was switched from to the program you are switching to.

The program you are switching to can get the launch parameters by using the nn::oe::TryPopLaunchParameter() function.

For more information about launch parameters, see Getting Launch Parameters.

To debug program switching when developing the programs themselves, use DevMenu or DevMenuCommand to set the launch parameters and simulate program switching.
