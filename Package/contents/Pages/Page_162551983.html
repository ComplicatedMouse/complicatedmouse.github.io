NintendoWare 3D News

This is a graphical introduction to the features that were added in NintendoSDK 0.13.0.

3DEditor

This section describes features newly added to 3DEditor.

Added a Model Merging Feature

The previously deprecated model merging feature is now available again.
You can now apply the settings from other model files to the selected model.

Related Links For more information, see the following pages.

File View

Describes the file view, which enables you to view and select intermediate files loaded into 3DEditor.

See Model Files.

3DEditor Team Settings File

Added a team settings file feature.

Added a feature for hiding bone visibility animations for which binarization is not enabled in Curve Editor.

Added a feature for hiding bone visibility animations for which binarization is not enabled.

Enable the <ShowOnlyBinarizeEnabledBoneVisibilityAnimCurve> setting in the team settings to hide the applicable curves.

If export_bone_visibility_all is set to false in the Maya export plug-in config file, bone visibility animations for which keys were not set in Maya will not be displayed in Curve Editor in 3DEditor.

Related Links For more information, see the following pages.

Team Settings File Format

This section describes the components of a team settings file.

See the <ShowOnlyBinarizeEnabledBoneVisibilityAnimCurve> section.

Added platform-specific connection settings to the team settings file.

You can now customize the connection settings for each platform.

For example, you can limit the platforms to which an application can connect and configure additional 3D shader converter and texture converter parameters for each platform.

Related Links For more information, see the following pages.

Team Settings File Format

This section describes the components of a team settings file.

See the <PlatformPresets> section.

Added a feature for calling user-defined commands prior to sending data when sending intermediate files to a connected application.

Added a feature for injecting user-defined processes (such as applying an intermediate file filter) prior to sending data when sending intermediate files to a connected application.

Related Links For more information, see the following pages.

Team Settings File Format

This section describes the components of a team settings file.

See the <PreBinarizeCommand> section.

Added a feature for skipping shader variation creation during shader optimization.

Added a feature for skipping shader variation creation for certain materials during shader optimization and allocating shader archives that are maintained in the connected application.

This shortens the shader conversion time when 3DEditor is optimizing shaders.

Related Links For more information, see the following pages.

Team Settings File Format

This section describes the components of a team settings file.

See the description of the AllowNoOutput attribute in the <CreateShaderVariationCommand> section.

User settings and application settings can now be configured independently for each team settings file.

Previously, user settings and application settings were configured on a per-machine basis and shared across all team settings files. With this change, user settings and application settings can be configured on a per-team settings file basis.

Specify a name for the settings configuration in the <SettingName> element in the team settings file to create user settings or application settings with that name.

Related Links For more information, see the following pages.
