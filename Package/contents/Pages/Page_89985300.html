WaveConverter

Introduction

WaveConverter is a command-line tool for converting waveform files (WAV and AIFF files) into resource files (BFWAV and BFSTM files).
The resource files created by WaveConverter can be used by the nn::atk library.

This document describes how to use WaveConverter and its features.

Using WaveConverter

How to Run

To use WaveConverter, run it from the command line in the following format.

WaveConverter.exe [Options] Input_file_path

Examples:

WaveConverter.exe --format=bfwav --encoding=adpcm yoshi.wav
WaveConverter.exe --format=bfstm --encoding=pcm16 -o bgm.adpcm.bfstm bgm.txt

If the conversion process is successful, the BFWAV and BFSTM resource files are output to the file path specified by the --output option.
If the --output option is omitted, then the resource files are output to the input file path, but with the filename extensions changed to .bfwav and .bfstm.

Command-Line Arguments

Input File Path

Specifies the file path of the input waveforms.
If -format=bfstm has been specified, you can output a multi-track stream binary file by specifying the path to a waveform list file.

A waveform list file is a text file like the following with waveform file paths separated by newline characters.

Examples:

piano.wav
bass.wav
drum.wav

Command-Line Options

Option

Description

-h / --help

Display a Help message and exit.

-v / --verbose

Displays detailed internal operations.
--format <format-type>

Specifies the format of the waveform binary files to output (as BFSTM or BFWAV).

This option must be specified.
--encoding <encoding-type>

Specifies the output encoding as pcm16 or adpcm.

This option must be specified.
-o, --output <output-filepath> Specifies the output file path.
If omitted, the resource files are output to the input file path, but with the file extensions changed to .bfwav and .bfstm.
--loop-start <frame>

Specifies the starting frame of the loop. If the sound is not being looped, do not specify this option.
--loop-end <frame>

Specifies the ending frame of the loop. If you specify --loop-start but omit --loop-end,

the sound loops between the position of --loop-begin and the end of the waveform.
--stream-prefetch Creates the stream prefetch binary file. Only valid when --format=bfstm is specified.
--stream-buffer-scale <arg>

Specify the stream buffer stream prefetch multiplier.
Only valid when --format=bfstm and --stream-prefetch are specified.
