Initializing the EDEV

Turning the EDEV On

Press the POWER Button on the top of the EDEV console.

If the EDEV is off and you connect it to a PC, the power turns on automatically.

Note

The EDEV must be sufficiently charged (we recommend at least an hour) in advance.

If the battery runs low during initialization, the update might fail.

Connecting the EDEV to a PC

Connect the USB charging cable to the PC's USB port and to the USB Type-C terminal on the bottom of the EDEV console.

Note

Do not use the relay box during initialization.

If the relay box is used during initialization, the update might fail.


The USB driver might update when the EDEV is connected to the PC.

When the EDEV is connected to a PC, wait for a while after turning on the EDEV's power.

If the USB driver is updated during initialization, the update might fail.

Initializing the EDEV

Initialize the EDEV firmware using a special tool. (This process usually takes about five minutes.)

Info

Warning messages about the Windows Firewall are displayed several times during this process. Select Allow access.


Procedure Operation
1

Note the code written on the sticker on the bottom of the EDEV.

(This code represents the SDEV's serial number. It is XAW03121003905 in the following example.)

Info

The serial number is used to select the EDEV in step 3.
2

On the NintendoSDK management screen, click Initialize the EDEV.


Info

To display the NintendoSDK management screen, select Dev Environments > Manage from the Nintendo Dev Interface Welcome screen.
3

Select the EDEV you want to initialize and begin initialization.


From the list of connected EDEV units shown in 1. Select EDEV actual equipment, click to select the one whose Serial value matches the serial number you noted in step 1.


Under 2. Select firmware variation, in the combo box, select the firmware version to use.

Info

We strongly recommend selecting the firmware that was initially suggested, without any changes.

When selecting + Prod or + ProdWithLog, first read Initialization for PROD MODE, the Firmware That Approximates Retail Hardware section in NintendoSDK Documents > Guide to Setting Up the Environment > More Information.

Next, to start EDEV initialization, in 3. Start EDEV initialization, click Start initialization.


Info

When the EDEV console is in safe mode, an entry that reads (Safe Mode) appears in the Name column.


To start initialization, select that entry and click Start initialization.


Info

The log pauses for 80 to 90 seconds during this process. Wait for the log to resume.

The process may have failed if the log does not resume after several minutes.


Info

If the EDEV console you want to initialize does not appear in the list of connected EDEV units even after restarting and clicking Re-Scan, something might be wrong with the EDEV. For more information, see When EDEV Initialization Fails.
4

If Success is displayed at the lower left of the log window, press the Exit button to close the tool.


Info

Fan Operating Noise

While executing step 3, the cooling fan inside the EDEV starts running.
The fan is somewhat noisy, but that does not mean the unit is malfunctioning.


Info

For more information about the setup, see NintendoSDK Documents > Guide to Setting Up the Environment > More Information > Other Information About Setting Up the Development Hardware.



Info

A built sample has been prepared to check whether the EDEV was initialized correctly.

Use the following method to run this sample.

  1. On the NintendoSDK management screen, click the Run Sample button.
  2. The following screen is displayed on the EDEV LCD screen if the EDEV was initialized correctly and VSI for NX was installed correctly.

Note: This sample does not exit automatically. Return to the Menu screen with the HOME Button.

Troubleshooting

If EDEV initialization fails or the console you want to initialize is not on the EDEV connection list, see When EDEV Initialization Fails and try initializing several more times.

If initialization is still unsuccessful, contact Nintendo support.

EDEV Drivers

The EDEV drivers must be installed for a PC to recognize the EDEV.

In most cases, the EDEV device drivers are installed automatically when you use NDI to install NintendoSDK. However, automatic installation sometimes fails depending on the environment.

Use the following procedure to confirm whether the EDEV device drivers are installed.

  1. Turn the EDEV on and connect it to the PC with a USB cable.
  2. Open Device Manager on the PC.
  3. Verify that NintendoSdkDebugger is shown in Device Manager.

If the EDEV device drivers are not installed, use the following procedure to install the drivers manually.

  1. Find \DevkitDriver\dpinst.exe in the installation directory for NintendoSDK, and run it.

The procedure is now complete.
