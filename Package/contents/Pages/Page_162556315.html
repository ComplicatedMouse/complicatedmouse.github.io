ArchiveShaderCombiner

Introduction

ArchiveShaderCombiner merges the variation definition files for GfxShaderConverter that LayoutArchiver created into one file. That tool also generates shaders for inclusion in the layout archive.

There is no need to directly call this tool. It is automatically called within LayoutArchiver.

Using ArchiveShaderCombiner

Location of the Executable File

The ArchiveShaderCombiner executable file is in the following path.

  • Tools\Graphics\LayoutTools\ArchiveShaderCombiner.exe

Command-Line Format

ArchiveShaderCombiner is used in the following format.

ArchiveShaderCombiner.exe bgsh<directory path> ApiTypeName CodeTypeName

See the following example.

ArchiveShaderCombiner.exe "arcsrc\bgsh" Gl Source

Output

Running ArchiveShaderCombiner merges all the XML files within the bgsh directory and creates two files: __ArchiveShader.bnsh and __ArchiveShader.bushvt.

__ArchiveShader.bnsh is the merged shader binary. __ArchiveShader.bushvt is a table that lists the correspondence between the values, or blend types, specified using the variation definition files and the variation indices.

__ArchiveShader.bushvt is a binary file with the following data format.

The nth byte, using leading zeros. Data Content Data Size in Bytes
0 The Variation Count 2
2 Data for the first variation. The first blend type. 2
4 The second blend type. 2
6 Variation Indices 2
8 The second variation type. The first blend type. 2
10 The second blend type. 2
12 Variation Indices 2
... ... ... ...
