Conversion Log

To open the Search Result window, on the View menu select Search Result, or click Show Search Result on the toolbar.
You can also use the shortcut key ALT+9 for this operation.
These operations are used to switch between showing and hiding windows.

The Conversion Log window displays a log of conversion activity.
Successful and failed conversion processes are kept in this log, so that you can use this window to check on the latest status of registered items.

The Conversion Log window appears as a docking window in the lower pane of the main window.
You can make the Conversion Log window float, dock, or hide by selecting Floating, Docking, or Hide in the Conversion Log tab at the bottom of the window or from the shortcut menu for the title bar.
You can also switch between Floating and Docking by double-clicking the title bar.

When the window is docked, you can minimize it so that only the tab is displayed by clicking the Minimize icon on the right side of the title bar, or by double-clicking the splitter.

When an error occurs during conversion, you can select the error item in the Conversion Log window and click Jump to Item to select and display the warning or error in the Sound Set tab.
To perform the same operation, you can double-click an error item or right-click and select Jump to Item from the shortcut menu.

Warnings are displayed in the following cases.

  • There is an item that has not been registered to the group (targets are sounds, banks, and subgroups).
  • There is an unused item (targets are players, groups, and banks).

The warning feature can be enabled or disabled by using the Project Settings dialog box.

You can right-click an item selected in the Conversion Log and select Copy from the shortcut menu, during normal operation or when an error occurs.
