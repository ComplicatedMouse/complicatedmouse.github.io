Introduction

Application programs can exit unexpectedly inside and outside of the SDK libraries for many different reasons.

This document describes the fixed character strings that are output when an application unexpectedly exits and the reasons for the output.

This understanding will help you search for the causes of unexpected exits.
Additional usage is as a list of strings to monitor for during automatic testing.

Fixed Strings Output On Unexpected Exits

String Reason
SDK Assertion Failure Conditions were not satisfied in the SDK code.
Precondition not met Some precondition was not met when a function was called in the user code.
Assertion Failure Conditions were not satisfied in the NN_ASSERT APIs in the user code.
Unexpected Default

The default label was reached where it should not have been in a switch statement in the SDK code or the user code.
Abort Some NN_ABORT API was called in the SDK code or the user code.
UserException handler is called at pid=0x... An exception was thrown in the SDK code or the user code.

In the following cases, the user can fix the bug. Make corrections so all of the conditions are satisfied.

  • Precondition not met
  • Assertion Failure
  • Unexpected Default if NN_UNEXPECTED_DEFAULT was reached in the user code.
  • Abort if an NN_ABORT API in the user code did not satisfy conditions.
  • UserException handler is called at pid=0x... if an exception was thrown in the user code.

In all other cases (SDK Assertion Failure and condition violations and exceptions in SDK code), the issue cannot be fixed by the user.
Contact Nintendo support and provide an error log.
