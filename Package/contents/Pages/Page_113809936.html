Introduction

Initializing the SDEV describes how to set up the SDEV for the first time.

This section describes how to update the SDEV firmware that was set up for the earlier SDK.

Updating the SDEV Firmware

Follow these steps to update the SDEV firmware.

  1. Install the firmware package.
  2. Connect the SDEV to a PC and attach the SDEV power cord.
  3. Back up the system settings (optional).
  4. Run InitializeSdev.bat.

The following sections describe the details of each step.


Installing the Firmware Package

Install NintendoSDK System Updater for NX.

If the package is being installed in the form of a MegaZarf package, it is installed automatically.
If the package is being installed by accessing the NDI server and selecting a package, explicitly choose to install NintendoSDK System Updater for NX.

Connecting the SDEV to a PC and Attaching the SDEV Power Cord

Connect the SDEV to a PC and attach the power cord to the SDEV in the regular way.

Backing Up the System Settings

When you update the hardware, the system settings revert to their default values.

If you backup the system settings using the Settings Manager tool, you can continue to use the previous settings, even after you have updated the hardware.

Backing Up the Save Data

The various save data will also be erased, but if you use the backup feature of DevMenu, you can create a backup of the save data that will be available after the system is initialized.
(You can also backup the save data using the BackupSaveData tool.)

Running InitializeSdev.bat

Run the InitializeSdev.bat file located directly under the NintendoSDK folder.

This procedure writes the new firmware to the SDEV.

Run this batch file using the same procedure described in Initializing the SDEV.


For NX Add-On 4.3.0 On Only: Using the Update Tool to Update the SDEV Firmware

In NX Add-On 4.3.0 and later, you can use the SDEV console update tool to update firmware.

For more information, see the section about the development hardware update tools in Information About Setting Up Development Hardware.

Note

The system update tool does not update DevMenuApp. For more information, see here.

Limitations

Retaining Information Stored in the SDEV

Updating the SDEV firmware deletes all information that was stored to the device using previous versions of the SDK and firmware. The following information gets deleted.

  • Installed applications
  • Save data
  • Account information


Updating When Downgrading to a Version Earlier Than NX Add-On 0.5.0

InitializeSdev.bat may fail to execute if you downgrade the SDEV to a version prior to NX Add-On 0.5.0 after updating the firmware for that or a later version.

When this happens, restore the SDEV and then run InitializeSdev.bat again. For more information about restoring the SDEV, see Troubleshooting on the SDEV Console Startup Fails page, in the SDEV Operations Manual.
