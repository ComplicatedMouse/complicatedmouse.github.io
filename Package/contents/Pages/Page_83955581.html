Creating Effect Binaries

This section describes how to convert an emitter set intermediate file into an effect binary file.
Use EffectMaker.exe to create the effect binaries. Create them by running EffectMaker.exe with a convert list file (a list of emitter set intermediate files) as the input.

Example: Creating effect binaries for Windows
EffectMaker.exe --convert-list ConvertList.xml --output EffectBinary.ptcl --api-type Gl --code-type Source --tile-mode linear

Convert List File

This file describes the emitter set intermediate files that will be the target of binary conversion. There are two ways to specify emitter set intermediate files.

  • To convert specified emitter set intermediate files.
  • Converting Emitter Set Intermediate Files Inside a Specified Folder

Converting Specified Emitter Set Intermediate Files

Describe the file paths in the <EmitterSetList> – <file> tag. In addition to specifying absolute paths, you can specify relative paths from an XML file for conversion (as in the preceding example), and also use environment variables. To specify an environment variable, you can use either $(ENV_NAME) or %ENV_NAME% for the notation.

Tags Used in the XML File for Conversion

Tags Description
EffectMakerCommandLineConverterArg XML for conversion. The following tags are used to specify information for the conversion process.
BinaryName Specifies the name of the binary. The specified name is binarized and can be referenced from the runtime.
EmitterSetList The list of emitter sets to convert. Specify using the <dir> and <file> tags. You can make multiple specifications.

dir The emitter sets inside the specified directory will be converted.
file The specified emitter set files will be converted.
name Specify the namespace to use with the header file.

Command-Line Options

Options Description
-? Shows the command-line Help.

--silent

-s

 This function suppresses the log output.

--output-c-header [header file path]

-h [header file path]

 Specifies the output path of the header that enumerates the emitter set IDs.

--convert-list [convert xml file path]

--convlist [convert xml file path]

 Specifies the convert list file.

--team-settings-file [project config file path]

 Specifies the project settings file.
The --prjconfig and --team-setting-file options were replaced by this --team-settings-file option.

--output [binary file path]

--o [binary file path]

 Specifies the output binaries.
--output-texture-binary [texture binary file path]

Specifies the output path of the texture binary.
Use if you want to output the texture binary separately.

The texture binary will be output in BNTX format.
For more information about using divided and output texture files, see Effect Binary.
--output-g3d-primitive-binary [primitive binary file path]

Specifies the g3d primitive binary output path.
Use if you want to output the g3d primitive binary separately.

The g3d primitive binary that is output will be in BFRES format.
For more information about using divided and primitive files, see Effect Binary.
--output-shader-binary [shader binary file path]

Specifies the output path of the shader binary.

The vertex and pixel shader binaries are output to a separate file than the effects binary.
The texture binary will be output in BNSH format.

For more information about using divided and output shader files, see Memory Pools.
--output-compute-shader-binary [compute shader binary file path]

Specifies the output path of the compute shader binary.

The compute shader binary is output to a separate file than the effects binary.

The texture binary will be output in BNSH format.

For more information about using divided and output shader files, see Memory Pools.
-d Displays detailed conversion information.
Omit this option to view the number of shaders, textures and primitives and their sizes, and the progress.
--update [convert xml file path] Updates emitter set intermediate files by overwriting them with the latest versions.
If you are using a custom shader or custom action, specify the project settings file.
Using the same method used for binary conversion, write the target file in the convert list file and specify the list file path.

--update-nw4c-to-nsdk [convert xml file path]

Updates NintendoWare for CTR emitter set intermediate files by overwriting them with the latest versions.

Using the same method used for binary conversion, write the target file in the convert list file and specify the list file path.

For more information, see Guide for Migrating From NW4C Effect to NintendoSDK.
-j [job number] Specifies the job number for parallel shader compilation.
If the specified job number exceeds the number of CPU cores, the process takes (number_of_CPU_cores – 1) as its value.
If the -j option has not been specified, the process takes (number_of_CPU_cores) as its value.
--api-type

Follows along with the converter options you specify for ShaderConverter.

Specify GI when exporting a binary for Windows or OpenGL.

For more information about exporting a binary for platforms other than Windows or OpenGL, see Platform-Specific Information.
--code-type

Follows along with the converter options you specify for ShaderConverter.

Specify either Binary_Source or Source when exporting a binary for the Windows or OpenGL platforms.

For more information about exporting a binary for platforms other than Windows or OpenGL, see Platform-Specific Information.
--shader-cache-directory

Follows along with the converter options you specify for ShaderConverter.

There are restrictions on the platforms that are supported. --api-type Gl is not supported.
--tile-mode

Conforms to the converter options specified in 3dTextureConverter.

Specify linear when exporting a binary for Windows or OpenGL.

For more information about exporting a binary for platforms other than Windows or OpenGL, see Platform-Specific Information.
--tile-optimize

Conforms to the converter options specified in 3dTextureConverter.
There are restrictions on the platforms that are supported. No support for export of Windows / OpenGL binary.

--tile-size-threshold

Conforms to the converter options specified in 3dTextureConverter.
There are restrictions on the platforms that are supported. No support for export of Windows / OpenGL binary.

--platform [platform]

This option will be deprecated in the future.

Specifies the platform that will use the output binaries.

Binary conversion does not run if this option is not specified.

The platform specified by [platform] is the name of the platform supported by the package you are currently using.

The platform name is not case-sensitive

For more information about exporting binary for other platforms, see Platform-Specific Information.
Win or Windows

Generates the Windows 64-bit version of the binary.

Shader Converter is called with the following options.

ShaderConverter.exe --api-type Gl --code-type Source --tile-mode linear

Win32 or Windows32　

Generates the Windows 32-bit version of the binary.

Shader Converter is called with the following options.

ShaderConverter32.exe --api-type Gl --code-type Source --tile-mode linear

--include-emitter-set-file-path-array

Adds the file path table of the emitter set to binary output.
Add this to search for the emitter set ID using the file path, and
to get the file path using the emitter set ID in the runtime.

The emitter set file path becomes the relative path from the project settings file.

As a result, you must specify the project settings file using the --team-settings-file option.
Also see Effect Binary.

Note

Including emitter set file paths in binaries is performed for binaries under development. Do not use this feature when converting binaries that are included in retail devices.
When effect binaries are parsed, there is a risk that information such as the directory structure and emitter set names in the development environment will be leaked.
--3d-shader-converter-options Specifies the 3dShaderConverter options.
The specified string is passed to 3dShaderConverter without any modification as the options.
Info

Option notation has been standardized. Any option comprising two or more characters is now prefixed with two hyphens (--).
The earlier notation is still accepted, but it will be deprecated at some future date, so make sure that you comply with this later notation.

Important Features

  • Updating the intermediate files

  • Exporting the header file listing the emitter set IDs when the effect binary is created

  • Contents of the output binary


Updating the Intermediate Files

The file version of the emitter set intermediate files may be updated when features are added and bugs are fixed.
When EffectMaker loads the emitter set files of the old intermediate file versions, they are updated to the new intermediate file versions internally, so the user can load the old intermediate files without performing any particular operation. Because the old intermediate files are also updated during command-line conversion, loading all of the emitter set files can shorten the time it takes for command-line conversion.

You can update the intermediate files at binary conversion time by adding and executing the --update command-line option.

Exporting the header file listing the emitter set IDs when the effect binary is created

Use the --output-c-header option to output a header file for the runtime listing the emitter set IDs.

EffectMaker.exe --output-c-header test.h --convert-list ConvertList.xml --output EffectBinary.ptcl --api-type Gl --code-type Source --tile-mode linear
