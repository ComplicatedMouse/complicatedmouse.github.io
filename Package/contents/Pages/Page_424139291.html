Attempting to Make Wireless Connections

Use this command to connect all paired controllers.
Note that the success of connecting depends on the surrounding wireless environment, so the connection is not ensured.
If connection fails, the process will be retried for a maximum of five times.

$ RunOnTarget.exe DevMenuCommand.nsp -- controller connect-all

Disconnecting Wireless Connections

Use this command to disconnect all controllers that are connected wirelessly.

$ RunOnTarget.exe DevMenuCommand.nsp -- controller disconnect-all
 [SUCCESS]

Disabling Built-In Controller Features

Info

Available in NintendoSDK Firmware for NX 9.0.0-3.0 and later.

This feature disables features built into the controllers (NFC and the IR Motion Camera). It can be useful when testing controllers or systems that are not equipped with these features.

The features can be configured on a per-controller number basis (Handheld, No1, No2, No3, No4, No5, No6, No7, No8), which makes it possible to test combinations such as NFC being disabled on the controller connected as controller No1 while NFC remains enabled on the controller connected as controller No2.

NFC Settings

$ RunOnTarget.exe DevMenuCommand.nsp -- controller configure-nfc <enabled|disabled> --npadId <No1|No2|No3|No4|No5|No6|No7|No8|Handheld>
 [SUCCESS]

IR Motion Camera Settings

$ RunOnTarget.exe DevMenuCommand.nsp -- controller configure-irSensor <enabled|disabled> --npadId <No1|No2|No3|No4|No5|No6|No7|No8|Handheld>
 [SUCCESS]

Feature to Connect Registered Controllers

Info

Available in NintendoSDK Firmware for NX 11.0.0-1.0 and later.

You can list Joy-Con and Switch Pro Controllers that have been registered (paired) in advance and then run the connection process for the specified device. For example, you can use this during testing or other situations to connect a controller without pressing any buttons on it.

List Registered Devices

$ RunOnTarget.exe DevMenuCommand.nsp -- controller list-registered-devices
 [SUCCESS]

Connect Devices Listed Earlier

$ RunOnTarget.exe DevMenuCommand.nsp -- controller connect <index>
 [SUCCESS]


Feature to Disable and Enable Joy-Con Attached to the Rail

Info

Available in NintendoSDK Firmware for NX 11.0.0-4.0 and later.

You can disable the Joy-Con while it is attached to the rail of the console. (This state is equivalent to detaching the Joy-Con.) When the Joy-Con is in this disabled state, you can use the controller connect command to wirelessly connect the specified Joy-Con.
You can use the --suppress-auto-kill option for RunOnTarget to execute without terminating the running application. 


$ RunOnTarget.exe --suppress-auto-kill DevMenuCommand.nsp -- controller configure-rail <enabled|disabled>
 [SUCCESS]
