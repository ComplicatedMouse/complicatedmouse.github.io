Texture SRT Animation

Texture SRT animations are set by animating the Repeat UV, Rotate Frame, and Translate Frame attributes of the place2dTexture node.
Connecting Repeat UV and Translate Frame to other combined attributes is also supported (supported by NintendoSDK 14.0.0 and later).

Make sure that you set the Coverage attribute of the place2dTexture node to (1, 1), Offset to (0, 0), and Rotate UV to 0.

The method of calculating the texture SRT can be specified by using the Texture SRT Mode export option.

For texture SRT animation, data is exported only for attributes for which animation has been set. Attributes for which data was not output are shown in the FMD file.
Even if an attribute has a set value, if it differs from the state in the FMD file, make sure that you set at least one key to the attribute before exporting the texture SRT animation.

Textures connected to the reflected color attribute of the standard material (Reflected Color) do not export the texture SRT animation.

When the Texture SRT Mode option is set to Maya, the lower-left corner of the texture is the scale center. However, if you set the following expressions for the place2dTexture node, you can scale around the center of the texture.

translateFrameU = 0.5 - 0.5 / repeatU;
translateFrameV = 0.5 - 0.5 / repeatV;
Info

If the Material Compression export option is enabled, even materials for which texture SRT animation is not set are sometimes animated as a result of united materials.
If this happens, use the plug-in for setting material attributes to change settings so that materials for which texture SRT animation is set are not compressed.

Info

This is not supported if the animation is set directly in the attribute of the file node.
