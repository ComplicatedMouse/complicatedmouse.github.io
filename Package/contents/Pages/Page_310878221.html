Overview

You can change settings for the development hardware.

How to Use

System Settings

Language

This setting changes the language. You can specify the following languages. The default is American English.

  • American English
  • British English
  • Japanese
  • French
  • German
  • Latin American Spanish
  • Spanish
  • Italian
  • Dutch
  • Canadian French
  • Portuguese
  • Russian
  • Simplified Chinese
  • Traditional Chinese
  • Korean
  • Brazilian Portuguese (NintendoSDK Firmware for NX 11.0.0-6.0 and later)

Region

You can change region settings. You can set the region to any of the following values. The default is Usa.

  • Japan
  • USA
  • Europe
  • Australia
  • Hong Kong, Taiwan, Korea (NintendoSDK Firmware for NX 9.0.0-5.0 and later)

Time Zone

Click Edit to set the time zone for the network clock.

Platform Region

Info

Available in NintendoSDK Firmware for NX 9.1.0-1.0 and later.

You can change the platform region. You can set the following platform regions. The default is Global.

  • Global
  • China

When the platform region is changed, the development hardware automatically restarts and reverts to its default state.

When the platform region is changed to China, the two other settings are also changed and fixed to the following values.

Option Setting Value
Language Simplified Chinese
Region China
Time Zone Asia/Shanghai
Note

To switch the platform region on development hardware initialized in Prod mode, write DevMenuApp to an NX development card and then use that card to start DevMenuApp.
Use the following procedure to switch the platform region and install the application in development to development hardware.

  1. Insert the microSD card containing the NSP file of the application in development into the development hardware.
  2. Write DevMenuApp to the NX development card.
  3. Use the NX development card to start DevMenuApp.
  4. Switch the platform region.
  5. Restart the development hardware and restart DevMenuApp on the NX development card.
  6. Use the DevMenuApp to install the application in development.

Attempting to switch the platform region using the pre-installed DevMenuApp causes DevMenuApp to forcibly close with an error, and the platform region switch does not complete.
If this happens, manually restart the development hardware and use the preceding procedure to switch the platform region.

Date Time

This setting specifies the date and time. To use the network clock, the system must be able to use the network.

Automatic Correction

Select whether to automatically adjust the date and time based on the network clock. When Automatic Correction is selected, you cannot manually adjust the date and time. It is enabled by default.

DW Filter for PlatformRegion=China

Info

Available in NintendoSDK Firmware for NX 9.1.0-1.0 and later.

This setting is only available when the platform region is set to China.
It allows you to set whether to enable the DirtyWord filter, which is expressly enabled for consoles for China.

Setting Method Setting Value Details Filter Enabled?
Auto (default) -

The enabled state changes based on platform region.

  • Global: Disabled
  • China: Enabled
Manual Enabled Enabled
Disabled Disabled (default)

For more information about the behavior of the DirtyWord Filter for consoles for China, see For Switch Consoles For China (NX Add-On) > DirtyWord Filter.

Display Settings

Screen Brightness

This setting adjusts the screen brightness of the development hardware. When Auto Brightness Control is enabled, which it is by default, the screen brightness is adjusted automatically based on the ambient brightness as detected by the brightness sensor.

Screen Resolution

You can change the display resolution when using HDMI. The default is Auto.

Consumer Electronics Control (CEC)

This setting determines whether controlling power with the TV (CEC) is enabled. Specify On to enable the feature, or Off to disable it. This is enabled by default.

This setting is applied immediately.

TV Underscan Settings

You can change the underscan percentage value when using HDMI. You can specify values in the range of 0 to 20. The default is 0.

Color Remapping Control (CRC)

Use this setting to change the vividness of the console screen. Select Vivid to apply a brilliant color tone to the OLED. (This setting is only displayed on the Switch OLED model). Select Standard to apply a color tone similar to the console screen of a conventional Switch model. The default is Vivid.

Sound Settings

TV Sound

Sets the number of channels of audio to send as output to the television. You can set the number of channels to any of the following values. The default is Stereo.

  • Mono (1 ch)
  • Stereo (2 ch)
  • Surround (5.1 ch)
  • Auto (5.1 ch if supported by the TV, 2 ch if not)

System Sound (Speaker & HP)

You can configure the number of channels for the audio output of the development hardware (the system speakers, headphone output, and USB audio output). You can set the number of channels to any of the following values. The default is Stereo.

  • Mono (1 ch)
  • Stereo (2 ch)

Mute When Headphones Are Disconnected

This setting specifies whether to automatically mute the speakers when the audio output is switched from the headphones to the speakers. It is enabled by default.

Info

Available in versions of NintendoSDK Firmware for NX earlier than 13.0.0-2.0.

Speaker Auto Mute

This setting specifies whether to automatically mute the speakers when the audio output is switched from an output other than TV to the speakers. It is enabled by default.

Info

This feature is available in NintendoSDK Firmware for NX 13.0.0-2.0 and later.

Limit Headphone Output Level

Use this option to set whether to limit the audio output level of the headphones. This option is disabled by default.

Sound Test (Voice)

Plays the test sound (phoneme) from the development hardware console or TV.

  • Plays a phoneme continuously on each channel.

Sound Test (Sine Wave)

Plays the test sound (sine wave) from the development hardware console or TV.

  • A sine wave with a different frequency is played on each channel.
  • A sine wave is played on all channels at the same time.
  • The test sound continues to play until you select the Stop button or play the Sound Test (Voice).

Output Device

Displays the current audio output destination and the number of channels.

Volume Setting

Info

Available in NintendoSDK Firmware for NX 11.0.0-2.0 and later.

Use this setting to set and check the volume and mute settings for the audio that is output from the development hardware (the system's internal speakers, headphone output, USB audio output, Bluetooth audio output). By default, the volume is 10 and mute is disabled.

Security Settings

The device lock feature is the same as the device lock setting described in DevMenuCommand.

Device Lock

Info

Available in NintendoSDK Firmware for NX 11.0.0-3.0 and later.

You can change the device lock setting. The default is Disabled.
By enabling device lock, DevMenu and the HOME Menu display a lock screen and do not accept operations when the hardware is started or awakened from sleep mode.
The lock screen is displayed until the controller buttons configured for unlocking are pressed simultaneously. If the screen is not unlocked within a certain amount of time, the system shuts down.

Info

The device may be unlocked at the following times.

  • For DevMenu, the device displays "Locked" on startup and when waking from sleep mode, and accepts controller button input at this time. When the device is unlocked, "Unlocking" is displayed and it then transitions to DevMenu.
  • For the HOME Menu, controller button input is accepted after the Nintendo Switch logo disappears after startup, or during the 10 seconds when the screen goes dark after pressing the POWER Button when waking from sleep mode. When the device is unlocked, it transitions to the entrance.
Note

Device lock only restricts operations in DevMenu and the HOME Menu. To protect application data more securely, we also recommend following the procedure in AuthoringTool > Creating NSP Files for Secure Startup.


When this setting is changed to Enabled, a dialog box appears to register controller buttons to press simultaneously when unlocking the device. Specify and register two or more buttons.

Sleep Settings

Sleep Mode and Idle Time Detection work the same as the DevMenuCommand commands for controlling the sleep mode and controlling the idle state detection mode.

Sleep Mode

You can change how the system operates in sleep mode. Select Default for the default sleep mode behavior.

  • When the sleep mode is set to Default, the development hardware operates just like the retail device.
    • The system is put entirely to sleep, so the connection between the host PC and the target will be lost.
  • When the sleep mode is set to Pseudo, the development hardware does not enter the actual sleep state even if the POWER Button is pressed, and it wakes from sleep after about one second.
    • The connection between the host PC and the target is maintained before and after sleep.
    • Because sleep notifications and other aspects of the interface with the application operate the same way as with the retail device, this feature is useful for debugging the application straddling the sleep state.

The setting becomes enabled after a change from the sleep state after the command is executed.

Idle Time Detection

For development purposes, you can enable and disable the Auto-Sleep and Screen Burn-In Reduction features that are triggered when the idle state is detected.
Currently, this option is set to Off by default. This feature is always enabled in retail devices.
For purposes such as burn-in tests, you must use this command to set the automatic sleep mode feature to disabled.

Note

When the screen burn-in reduction feature is disabled, prolonged display of a screen pattern where the RGB values change in every frame can cause ghosting and flickering.
If this happens, stop changing the RGB values in every frame and the problem will gradually resolve itself.

You must restart the development hardware to apply the new setting.

Auto-Sleep (Playing on Console Screen)

You can change the amount of time before auto-sleep is triggered when playing in handheld mode. The default is 10min.

Auto-Sleep (Connected to TV)

You can change the amount of time before auto-sleep is triggered when playing in TV mode. The default is one hour.

Suspending Auto-Sleep While Playing Media Content

You can configure whether to suspend auto-sleep when playing back content such as videos. The default is Off.
Even when disabled, sleep does not occur until four hours have passed.

Waking Up When a Charging Condition Changes

You can configure whether the system wakes up when charging conditions change. The default value is set to Enabled (wake up).
If disabled, the system no longer wakes up from sleep mode when it is undocked or if the AC adapter is connected or disconnected while it is asleep.

Info

Some of the behavior during the disabled state for SDEV (such as displaying the icon when re-entering sleep mode) differs from the retail device. Use the EDEV to check the behavior equivalent to the retail device.
