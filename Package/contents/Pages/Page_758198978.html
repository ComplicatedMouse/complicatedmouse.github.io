Call Tree Tab

The Call Tree tab shows the discovered call tree based on the profiled sample data. This tab only appears if the profile was taken with Callstack data enabled.

To change the units displayed before each item, use the Unit buttons in the Home tab of the Ribbon Bar.

Each function that is not a root item in the Call Tree or Partial Call Tree will show an impact bar as the left-most item. This impact bar shows how much impact that function had based on the current selection of the Bar graph data source drop down.

Note

These notes point out differences in the Call Tree tab when the Thread-Centric Data Views experimental feature is enabled. See Experimental Features Tab for additional information.

Function Grouping

Note

This feature is only available when Thread-Centric Data Views are enabled.

The call trees in this tab are grouped according to the the viewing mode. See Experimental Features Tab for a description of function grouping.

Call Tree Tab Toolbar

The following controls are on the Call Tree tab toolbar.

Function Grouping Mode Drop Down

Note

This feature is only available when Thread-Centric Data Views are enabled.

This drop down list controls the function grouping mode. Note that it will also control the function grouping mode of the Functions Tab. A description of each mode can be found on the page Experimental Features Tab.

Prefix Selection

The prefix before each function name is determined by the three Prefix toggle buttons in the Call Tree toolbar. By default, the Total and Self prefixes are turned on. To toggle a prefix on and off, click on the particular prefix button. If all buttons are set to off, just the function name will be shown. The prefix choices are:

  • Total: Time spent in each function and the functions it calls.
  • Self: Time spent in each function, not including the functions it calls.
  • Sub: Time spent in functions that are called by this function.

Note that the functions at each level of the tree are sorted based on the prefix. If Total, Self, or Sub are selected, the sorting is from highest to lowest of the leftmost prefix. If no prefix is selected, the sorting is alphabetical, based on the function name.

Expand All Button

The Expand All button expands all possible items in the view.

Collapse All Button

The Collapse All button collapses all possible items in the view.

Auto Expand Drop Down

By default, the Call Tree will auto-expand and auto-collapse based on which functions are selected in either the Functions tab or the Call Tree tab, as well as the Find text box at the bottom of the Call Tree tab.

The Auto Expand drop down box has three options:

  • Auto Expand Off: The Call Tree will not expand or collapse when functions are selected, deselected, or highlighted by Find.
  • Auto Expand Only: The Call Tree will expand (but not collapse) when functions are selected or highlighted by Find.
  • Auto Expand/Collapse: The Call Tree will expand and collapse when functions are selected/deselected or highlighted by Find.

You can also quickly expand and collapse the entire tree with the Expand All and Collapse All buttons.

Copy Button

The Copy button will copy to the clipboard all selected or found items in their current expanded or collapsed state. If there are no selected or found items, only the root nodes of the tree will be copied.

Copy All Button

The Copy All button will copy to the clipboard all items in their current expanded or collapsed state. All items are copied regardless of what is currently selected or found.

Bar graph data source

Changing the Bar graph data source option changes how the profiler displays the bars that appear to the left of functions in the Call Tree.

The drop down has two options:

  • Total impact: The impact bar height gradually decreases the farther down the tree you get. The height is based on this function's Total value compared against the root node's Total value.
  • Local impact: The impact bar height shows relative busyness within a function. The height is based on this function's Total value compared against its parent's Total value.

The values in this drop down are applied to both the full Call Tree and the Partial Call Tree views.

Find Feature

The Find text box at the bottom of the Call Tree tab will auto-expand and highlight all matching functions (regardless of the Auto Expand drop down option selection). This behaves similar to the Filter text box in the Functions tab.

Function Selection

Clicking on a function in the Call Tree will highlight the function, similarly to how function selection works in the Functions tab. All instances of the highlighted function will be highlighted in the Call Tree. If auto-expanding is on (Auto Expand/Collapse or Auto Expand Only), then the tree will be expanded to show all instances of the selected function.

Right-Click Context Menu

A right-click in the Call Tree tab will bring up a context menu containing the following options:

Hot Path From Here

Recursively expands all nodes from this point that meet the criteria for being on a hot code path.

Expand from Here

Recursively expand all nodes from this point.

Collapse to Here

Recursively collapse all nodes to this point.

Copy Function Name to Clipboard

Copy the function name to the clipboard.

Show Assembly

Move focus to the Assembly Tab, showing the assembly of the clicked function.

Show Details

Open a new Function Details window, showing the details of the clicked function.

Partial Call Tree

When a function is selected, it will also appear in the Partial Call Tree at the bottom of the Call Tree tab.

There are three different views available in this sub-window: Forward from Selected, Reverse from Selected, and Legacy Reverse.

In each of the three views, the Total, Self, and Sub values are calculated as the percentage of the profile where the specific tree view was present.

The Partial Call Tree window can be resized by dragging the header up and down. It can also be collapsed by clicking on the down arrow on the right side of the header and similarly expanded by clicking on the up arrow.

Forward from Selected

This view treats all selected functions as the root. Expanding each root allows you to travel down the call tree, producing a view similar to that of the Call Tree. The difference, though, is that if there are multiple identical functions in the Call Tree, this view will combine the selected function into a single entry (adding up all of their times, similar to the Functions tab). This allows for easier viewing of the calls made by the selected function.

Reverse from Selected

This view treats all selected functions as the root. Expanding each root allows you to travel up the call tree. However, this is not just a simple reversal of the call tree, but instead separates and groups functions based on how they are called in order to show relationships that would otherwise be hard to see. For example, a vector normalize function might appear 10 times in the Call Tree, but it might be called from only 3 different functions. In this case, the vector normalize function will be listed once, with three child nodes (one for each calling function).

Legacy Reverse

This view is similar to Reverse from Selected. The difference is that there will be a unique root for each time the selected function appeared in the Call Tree, with the list ordered by time spent. This is the legacy behavior from previous versions of the profiler and can be useful in certain situations.
