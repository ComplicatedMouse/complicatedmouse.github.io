Accessories

This section describes creating accessories.

Accessories that you can create are headbands, crowns, and similar items that can be attached to hair. Accessories that are not for hair are outside the scope of these accessories.

Verifications Before Creating an Accessory

All hair models have three Null locations for attaching accessories. Accessories can be attached at set_front, set_side, and set_top.

The following are examples of attaching accessories at each Null location. You can also create accessories used at all Null locations at the same time.

set_front (Headband) set_side (Cat Ears) set_top (Crown)

Creating an Accessory

This section describes the procedure for creating accessories.

First, we describe the procedure where the designer creates a model of an accessory, and the developer implements it.

Creating the Accessory Model Data (Designer Task)

First, display the Hair_Sample bald head sample using 3DCG software.

Create an accessory that can be attached to the Null of the bald head. You can reference the HairAcce_Sample accessory sample at this time.

Details of HairAcce_Sample

  • Acce_front (Headband)
  • Acce_Side (Cat ears)
  • Acce_top (Crown)

The bald head (Hair_Sample) is the only hair model sample available. No other hair type models are provided. As a result, when the application developer decides to create an accessory shape for a certain hairstyle, it must be verified independently.

Displaying Accessories (Programmer Task)

Use the Face Library to display a Mii face model and display the accessory created by the designer.

When attaching the accessory to the face model, get the accessory rotation and movement information and attach the accessory based on this information. Use the nn::mii::CharModel::GetHeadPartsTransform() function to get the rotation and movement information.

In Hair_Sample, the Null value for Acce_Side is only for the right ear side. To get the rotation and movement information for the left ear side, specify nn::mii::HeadPartsType_Left in the parameter of the nn::mii::CharModel::GetHeadPartsTransform() function.

Verifications After Creating an Accessory

For verification purposes, attach the accessory to all of the hairstyles of Mii characters in the actual application.

With certain hairstyles, the accessory may sink down, or the hair and accessory may overlap. If this occurs, fix the accessory shape at the discretion of the application developer.
