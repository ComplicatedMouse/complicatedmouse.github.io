Performance Guide

This chapter provides technical information to help you achieve optimal performance.

Increasing Speed Through Data Settings

The layout runtime can be made to render faster according to data settings.

This list contains hints for increasing speed.

  • Disable vertex color and transparency.
    • Set the vertex color to white (255, 255, 255).
    • Set the transparency to 255.
    • Note that changing either of these settings increases the processing load.
  • Disable blend settings.
    • From the fragment operation Preset, select Opacity.
    • The blend settings color become "Disabled" and the alpha settings become "Same setting as color."
  • Reduce the number of multi-textures.
    • Reduce the number of multi-textures by combining textures ahead of time if you can.
  • Reduce the number of times large textures are repeated.
    • When textures with large file sizes are repeatedly displayed, texture cache mistakes tend to occur frequently, and the processing load increases.
    • You can fix this problem by decreasing the size of the texture, either by decreasing its height and width or by changing the texture's format.
    • Reduce the number of times the texture is repeated by modifying the mapping settings or the texture coordinates.

Increasing the Speed of Window Panes

There are several methods of reducing the processing load associated with window panes.

  • Make frames as small as possible.
    • Window types are layered one on top of another in the following order.
      • Texture type: No horizontal content, Texture count: １
      • Texture type: No horizontal content, Texture count: ２
      • Texture type: Horizontal, Texture count: 1
      • Texture type: Horizontal, Texture count: 2
      • Texture type: Omnidirectional, Texture count: 1
      • Texture type: Omnidirectional, Texture count: 4
      • Texture type: Omnidirectional, Texture count: 8
  • Enable the Share Top Left Material feature for window panes.

Increasing the Speed of a Text Box Pane

Note the following points to reduce the processing load associated with text box panes.

  • Reduce the frequency at which text is updated.
  • Do not use character color animation (Top color, Bottom color).
    • Vertex color is used internally, and causes a vertex recreation process.
    • Consider substituting a black and white color complement.
  • Reduce the frequency of calls to TextBox::SetDirtyFlag.
    • Use TextBox::SetDirtyFlag for actions such as changing the pane size.

Optimizing Processing of Masks or Alphas in a Pane That Uses Indirect Textures

In some cases, you must add additional textures for the alpha channel in multi-textures to apply masks or alphas to panes that use indirect textures.

Adding additional multi-textures substantially increases rendering loads. You can keep rendering load in check by creating indirect textures that include alpha channels instead of adding textures.

When using 3-channel (BC1) or 4-channel (BC3, RGBA8) texture-format files as indirect textures, the third channel (B component) and fourth channel (A component) are used as alpha channels.

The characteristics of compressed textures may prevent recreation of alpha channels as expected in BC1 textures.
Size increases in such cases, so use a 4-channel format such as BC3 textures.
