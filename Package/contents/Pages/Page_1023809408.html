I Can't Animate a Translate Animation for a Pane Below the Align Pane

Question

If, as shown in the following example, you set a position animation in a null pane that is a target for alignment, and a pane that is lower in the hierarchy is made to fade out off the screen, the animation is not animated.

Hierarchy LayoutViewer

Text is moved horizontally by TranslateAnimation ().

In LayoutViewer, where you can confirm the alignment by the Align Pane (), you can see that the translate animation does not start.

Answer

Even though you have set an animation that moves in the direction of the alignment for the pane below the Align Pane (), the pane does not move because the position is overwritten by the alignment process.
If you use a dummy picture pane (), as shown in the following image, you can widen the alignment area with the set animation.
The pane moves as a consequence of the resulting alignment process.

Hierarchy LayoutViewer

"Dummy" () is a dummy to widen the alignment area.
Because you do not want rendering in this area, set the pane location and size to zero.

You can confirm the translate animation.
In this example, Alignment Direction (horizontal) is set to Left for standard alignment in the Align Pane (), so the alignment causes the text to move to the right.
Info

The workaround introduced on this page has the following issues.

  • Alignment Direction is set to either Left or Right, and you cannot move in both directions.
  • This workaround uses a dummy picture pane, so the hierarchy becomes more complicated the more extensively it is used.
