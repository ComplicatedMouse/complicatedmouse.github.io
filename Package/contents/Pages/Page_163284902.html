Changes to Graphics for Developers

This section describes the changes made to NintendoWare Graphics for developers in NintendoSDK 0.11.0.
NintendoWare Graphics refers to the following modules.

  • nn::g3d
  • nn::ui2d
  • nn::font
  • nn::vfx

For more information about the major changes to NintendoWare Graphics implemented in NintendoSDK 0.8.0, see Changes to NintendoWare Graphics in 0.8.0.

Changes to the Font Runtime

Fixed a bug where old kerning files (BFFKN files) could not be loaded.

Fixed a bug where FontConverter corrupted old kerning files (the BFFKN files output in NintendoSDK versions before 0.10.0) upon loading them.

The kerning file format has also been updated accordingly. Please re-output any kerning files you are using that were created in NintendoSDK 0.10.0 to 0.12.0.

For more information, see the following table.

NintendoSDK Version Used to Generate BFFKN File Supported Character Encoding Loaded in FontConverter 0.13.0
Prior to 0.10.0 16bit Loads correctly.
Changes from 0.10.0 to 0.12.0. 32bit Gets corrupted upon loading (re-output files).
0.13.0 32bit Loads correctly.

Changes to the ui2d Runtime

Added extended user data for layouts and animation segment tags.

Extended user data can now be configured for layouts and animation segment tags.
Use the following functions to access the extended user data.

  • nn::ui2d::AnimResource::GetExtUserDataList
  • nn::ui2d::Layout::GetExtUserDataList

For an overview of the tool settings, see here.

Archive shaders are now merged together into a single shader.

Previously, in ui2d it was sometimes necessary to include multiple shader binary files in an archive file according to your intended use case. These shaders are now merged together into a single shader.

Increasing the number of shader binaries used also increased memory usage due to the alignment process. However, merging the shader binaries together into a single shader solves this problem.
