3dIntermediateFileDependencyEnumerator

This feature is currently being verified. When using this feature, note that its operation and specifications are subject to change without notice.

This page describes the 3dIntermediateFileDependencyEnumerator (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dIntermediateFileDependencyEnumerator.exe) and considerations when using the tool.

  • Overview

Overview

This command-line tool enumerates the dependency files of the model intermediate file (FMDA and FMDB files) and writes them to a JSON file.

The tool creates a JSON file that stores information about all of the dependency files of the entered file.

Command-Line Options

3dIntermediateFileDependencyEnumerator.exe [-h] [--version] [--args-file ""]... [-j ""] [-s] [-o ""]
[--disable-xsd-check] [--recursive] [--continue-on-error]
[--shader-search-path ""] [--material-search-path ""] [--shader-graph-search-path ""] [--sub-graph-search-path ""] [--texture-search-path ""] [intermediate file|folder]...

Enumerate the absolute paths of the dependency files for the model intermediate file in a JSON file.

--args-file ""

Specifies the file.
You can use the contents in the parameter file to specify arguments.
The parameter file allows you to specify each option over several lines.
Lines starting with a number sign (#) are treated as comments.
Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--version

Displays version information.

-h --help

Displays Help.

-s --silent

Disables the log.

-j "" --jobs ""

Specifies the number of jobs. When unspecified, the number of processors is assigned.

--recursive

When searching for reference files, also search in the subfolders of the additional reference paths.

--continue-on-error

If dependency files cannot be found or a similar error occurs that does not prevent processing from continuing, treat that error as a warning and keep going.

--disable-xsd-check

Disables format checking of intermediate files using schemas (XSD files).
When this option is enabled, only the latest version of the intermediate files can be processed.

--shader-search-path ""

Specifies the shader definition search path.

--material-search-path ""

Specifies the folder to search for the referenced material intermediate file.

--shader-graph-search-path ""

Specifies the folder to search for the referenced shader graph file.

--sub-graph-search-path ""

Specifies the shader sub-graph search path.

--texture-search-path ""

Specifies the texture search path.

-o "" --output ""

Specifies the path of the JSON file to which the dependency information is saved. The paths inside the JSON file are absolute paths.

Intermediate File | Folder

Specifies the intermediate file, or the folder containing the file, that you want to check dependencies for. Only model intermediate files are checked this way.

Usage Examples

Enumerating the Dependencies of a Model Intermediate File

Save the dependencies of the file named Input.fmdb to the file named /.Output.json.

3dIntermediateFileDependencyEnumerator.exe "./Input.fmdb" -o "./Output.json" --shader-search-path "./Shaders" --material-search-path "./ParentMaterials" --shader-graph-search-path "./ShaderGraphs" --sub-graph-search-path "./ShaderSubgraphs" --recursive

With this command, the input.fmdb.json file saved to the Output folder has the following structure.

{
  "DependencyInfos": [
    {
      "Path": "C:\\Input.fmdb",
      "Dependencies": [
        "C:\\material.fmtb",
        "C:\\textures\\texture.ftxb"
      ]
    },
  ]
}


Enumerating the Dependencies of Intermediate Files in a Specified Folder

Save a list of the dependencies of the model intermediate files inside the ./Input folder to the file named ./Output.json.

3dIntermediateFileDependencyEnumerator.exe "./Input" --shader-search-path "./Shaders" --material-search-path "./ParentMaterials" -o "./Output.json" --recursive
 
{
  "DependencyInfos": [
    {
      "Path": "C:\\Input\\ModelA\\ModelA.fmdb",
      "Dependencies": [
        "C:\\Input\\ModelA\\ModelA_Material.fmtb",
        "C:\\Input\\ModelA\\textures\\texture.ftxb"
      ]
    },
    {
      "Path": "C:\\Input\\ModelB\\ModelB.fmdb",
      "Dependencies": [
        "C:\\Input\\ModelB\\ModelB_Material.fmtb",
        "C:\\Input\\ModelB\\textures\\texture.ftxb"
      ]
    },
  ],
}
