Checking Access Speed Dependency

Overview

Guidelines specify that applications must operate correctly, even if the access speed to the storage media changes.
The FS speed emulation feature allows you to debug applications using three patterns of access speed.

Test Method

Note

Use development hardware initialized with NintendoSDK Firmware for NX 5.0.0-4 (NX Add-On 5.3.0) or later.

  1. Do not remove the SD card from the development hardware.
  2. Install the application, add-on content, or patch that you want to test behavior for, to the system memory.
    1. For more information about installing applications, see Installing Applications on Development Hardware.
    2. For more information about installing downloadable content, see Managing Downloadable Content.
    3. For more information about installing patches, see Building and Running a Patch.
  3. From DevMenu, select the Debug menu tab to configure the FS Speed Emulation Mode.
    Effective from NX Add-On 1.1.0, you can also use DevMenuCommand to configure the setting. For more information, see DevMenuCommand > Development Support Features.
  4. Play through the application and check the behavior.

The FS Speed Emulation Mode settings return to the initial state when the development hardware is restarted or reset.

Info

If not enough memory is available when installing all of the application, patches, and downloadable content to the development hardware system memory, use an SD card that supports UHS-I (Ultra High Speed Phase I, loading speed 60 to 95 MB per second).
The speed emulation feature is implemented by delaying access to the storage media, so more optimal testing can be performed by using storage media with a faster access speed. Make sure to place the test subject in system memory, and also place the other data in system memory as much as possible. Also use an SD card that is as fast as possible.

Speed Emulation Feature

The speed emulation feature slows down system memory access speed, with the fastest access speed in the storage media, to emulate storage media with slow access speed.
The speed emulation feature has the following modes.

  • Faster mode
    Operates with no delay in access to system memory and with a simplified version of the security processing of the retail product.
    Represents operations on development hardware where access speeds are as fast as they can be. Covers a wider range of testing conditions for faster environments, including when hardware is faster because of individual console differences or because of system improvements.
    Effective from NintendoSDK SystemUpdater for NX 2.0.0-3 (NX Add-On 1.0.0), Faster mode simplifies the security processing of the retail product. As a result, it operates faster than OFF, which operates in a manner that closely resembles the retail product.
    In versions earlier than NintendoSDK SystemUpdater for NX 2.0.0-3 (NX Add-On 1.0.0), Faster mode and OFF had the same behavior because there was no support for the security processing of installed applications in the retail device.
  • Slower mode
    Introduces a certain delay, corresponding to the method used for accessing system memory.
    Emulates a storage media with unvarying slow access speed, comparable to the types of slower external storage devices that can be encountered in the commercial market.
    In versions earlier than NintendoSDK SystemUpdater for NX 5.0.0-1 (NX Add-On 5.1.0), access to data stored only in system memory, was restricted to a slower speed equivalent to a slow-operating external storage device.
  • Random mode
    Creates a random delay corresponding to the access method for system memory access.
    This emulates responsiveness fluctuations of storage media internally.

For actual measurements of performance in Faster mode and Slower mode, see Performance.

You can use Slower mode in combination with background download stress testing.
However, do not use Faster mode or Random mode in combination with this test. This is because access is slowed by the download stress test, so you cannot conduct the testing as intended using steadily fast or randomly changing access speeds.

In NintendoSDK Firmware for NX 5.0.0-1 (NX Add-On 5.1.0) and later, accessing the SD supports slowing down access depending on the mode. However, the speed emulation feature is implemented by slowing down access to storage media, so more appropriate emulation can be performed by placing the data in system memory with a fast access speed rather than on the SD card.

Note that the speed emulation feature does not assure the maximum and minimum speeds of the NX in the future. In the future, smaller storage media and system improvements may cause the access speed from the application to be faster than during verification. Users may also use retail external storage media that is a slower speed.
