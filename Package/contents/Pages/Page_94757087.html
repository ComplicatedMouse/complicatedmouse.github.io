Purpose of the Revisions

In NintendoSDK 0.7.0, changes were made to the low-layer libraries on which the graphics and audio modules depend for the realization of more optimal settings and more optimal performance on future Nintendo platforms. Specifically, the implementations were changed so the graphics module uses the nn::gfx library and the audio module uses the nn::audio library. In addition, the namespaces were transitioned from the nw namespace to the nn namespace.

In addition, the system modules were deprecated with the release of NintendoSDK 0.7.0. These were prepared as NintendoSDK internal modules to be a more optimal module for future Nintendo platforms. However, not all features that were provided until now have been prepared in these internal modules.

This document refers to versions NintendoSDK 0.7.0 and later as New NintendoWare.

About the Old NintendoWare Library

During the transitional period, the old NintendoWare was made available along with the NintendoSDK 0.7.0 release. We planned to deprecate it with the release of NintendoSDK 0.8.0. As a transitional measure, we announced the inclusion of graphic and audio modules that would provide compatibility with old NintendoWare libraries for NintendoSDK 0.8.0 in its release. However, the release is being delayed so that we can focus on improving the quality of the new NintendoWare. The schedule based on this delay must still be decided.

Namespace Transition

In NintendoSDK 0.7.0, the graphics and audio modules were transitioned from the nw namespace to the nn namespace.
In step with this namespace transition, the namespace names were reviewed and integrated.

System Module Deprecation

Developers using the deprecated system modules (nw::sys, dev, math, and util) must switch to the replacement modules provided in the NintendoSDK module by the release of NintendoSDK 0.8.0.

The following table shows the main replacement modules.

Linking the Libraries

Info

This is not a permanent change, but a temporary change in line with the change in NintendoWare modules.

The NintendoWare libraries are not linked when the application is built. You must revise your project file in the ways shown in the following table.

What You Want to Link What Is Coded in the Additional Dependencies

Linked Libraries
New NintendoWare libraries (libnn_*) $(NintendoSdkNintendoWareLibraries)

  • libnn_font

  • libnn_ui2d

  • libnn_vfx

  • libnn_g3d

  • libnn_ige

  • libnn_atk
Info

The libnn_ige runtime is a communications library newly added in NintendoSDK 0.7.0. For more information, see NintendoWare / Version 0.7.0 News.

Notes

  • The New NintendoWare cannot be used together with the Old NintendoWare.
