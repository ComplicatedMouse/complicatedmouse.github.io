Overview

  • The following actions can be performed in the NintendoWare Audio development environment.
    • Playing sounds using the NintendoWare Audio Toolkit library (the atk library)
    • Converting data using NintendoWare SoundMaker
  • This document describes these features and how to build the sample program.

Advance Preparations

To use the NintendoWare Audio development environment, you must, at minimum, install the following packages.

  • NintendoSDK
  • NintendoSDK NintendoWare

This documentation uses sample demos for its descriptions, so also install the following sample packages.

  • NintendoSDK Samples
  • NintendoSDK NintendoWare Samples

Creating Sound Data

Starting SoundMaker and Converting

  • In the NintendoWare Audio development environment, you use the tool called SoundMaker to create data that can be played using the atk library.
  • SoundMaker is located in \Tools\Audio\SoundMaker\SoundMaker.exe.
  • The sample data is located in \Samples\Sources\Resources\Atk\Common.
    • Load the Common.fspj file located in this same directory by dragging it to the project tree of SoundMaker, or by selecting File > Open Project from the SoundMaker menu.
  • Convert the data either from the toolbar using Convert or Reconvert, or from the menu by selecting Project > Convert or Project > Reconvert.
  • The data that results from the conversion process is sent to the output folder inside the project folder.
  • You can also import data from NW4F SoundMaker, NW4C SoundMaker, and NW4R SoundMaker.
    • NW4F SoundMaker, NW4C SoundMaker, and NW4R SoundMaker are the respective tools belonging to the development environments for Cafe (Wii U), CTR (Nintendo 3DS), and Revolution (Wii).
  • For more information, see the SoundMaker Manual.

atk Library

Overview

  • Use libnn_atk.lib when using the atk library (the API associated with the nn::atk namespace).

Building the Sample Program

  • To build and run the sample program, open the VisualStudio solution file.
    For NintendoSDK 4.0.0 and later, it is
    　\Samples\Sources\Applications\AtkSimple\AtkSimple-spec.Generic.autogen.vs2015.sln
    and for earlier versions it is
    　\Samples\Sources\Applications\AtkSimple\AtkSimple-spec.Generic.vs2015.sln
  • After VisualStudio starts, build the relevant solution from the Build menu.
  • Run the sample program by selecting Debug > Start Debugging from the menu.
  • Run the sample. The command prompt window appears.
    • This audio sample program does not draw anything to the screen.
    • You can use the PC keyboard to perform operations like playing and stopping.
    • The command prompt window shows log output, such as NN_LOG.

  • To stop the program, on the menu, select Debug > Stop Debugging.
