Graphics News

Added the Guide for Migrating From NW4C to NintendoSDK.

The document describes the update procedure (and so forth) for making it possible to use the NintendoWare for CTR intermediate asset files in NintendoSDK.

nn::g3d News

Added a CMDL File Converter

Added a converter for converting NintendoWare for CTR model intermediate files (CMDL files) into NintendoSDK model intermediate files (FMDB files).

Use this converter when you are migrating from NintendoWare for CTR.

Ability to Use uniform and renderinfo as Condition Determinations in the Condition Parameter of the Group Element in Shader Annotation

Previously, option was the only condition determination available in the condition parameter of the group element in shader annotation, but now, uniform and renderinfo are also available.

// @@ group="render_state_opaque_group" label="Opaque mode" condition="$renderinfo.render_state_mode == Opaque"

For information about formatting, see the Shader Annotation documentation.

nn::vfx News

Added a Feature to Avoid Collisions of Emitter Sets With the Same Name in the Viewer Runtime

When editing effect data in the application using the viewer runtime, there was a bug that caused emitter sets with the same name as the editing target in the application, to also be recognized as the editing target. To avoid this problem, we added a feature to the binary converter to convert emitter set file paths to binary.

Two steps are required to use this feature.

  • Change the effect binaries to load in the game to effect binaries with paths
  • Change the effect binaries to transfer from EffectMaker to effect binaries with paths

To change effect binaries to effect binaries with paths, specify --include-emitter-set-file-path-array in the conversion options of the effect binaries to load in the application.

To change binaries to transfer from EffectMaker to effect binaries with paths, you must write the <IncludeEmitterSetFilePathArrayInEditorMode> tag in the project settings file of EffectMaker. Note: This setting is not associated with binary conversion.

Note

Including emitter set file paths in binaries is performed for binaries under development. Do not use this feature when converting binaries that are included in retail devices.
When effect binaries are parsed, there is a risk that information such as the directory structure and emitter set names in the development environment will be leaked.

Added a Member Function for Operations on Primitive LOD

You can now draw using LOD meshes included in primitive files (FMDB).

You can configure the LOD level settings with the void Emitter::SetPrimitiveLodLevel(uint8_t lodLevel) operation only from the effect runtime.
