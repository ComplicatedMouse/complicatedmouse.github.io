GL Initialization

Overview

GL initialization is a feature to batch load GL cores and extension commands.

When GL initialization is performed, each core and extension command is loaded depending on the current context. Even if a feature could not be loaded, it is not handled as a failure and the remaining features continue to be loaded.

When GL initialization is successful, you can call the GL commands that could be loaded, determine whether a specific GL command could be loaded, and determine whether a specific extension is supported.

If you want to re-initialize with a different context, re-initialize the GL after setting the other context as the current context.

How to Use

GL initialization is performed by nngllInitializeGl.

Before calling this function, you must configure the GL context to be used in loading.

After loading is successful, call GL functions in the C interface format to use the GL commands.

Usage Examples

For example, you can use this feature as follows.

// The GL context is configured.
nngllResult result = nngllInitializeGl(); // Loads the GL functions.
NN_ASSERT_EQUAL( result, nngllResult_Succeeded ); // Checks whether loading was successful.
glColorMaski( 0, GL_TRUE, GL_TRUE, GL_TRUE, GL_TRUE ); // Loading was successful, so GL functions can be called.

Including Header Files

nn/gll.h contains the types, enumeration, and functions needed to use the GL. When initializing the GL using the gll library, instead of gl.h, include nn/gll.h in the locations where the GL is used. Including nn/gll.h when gl.h is included causes a compilation error.
