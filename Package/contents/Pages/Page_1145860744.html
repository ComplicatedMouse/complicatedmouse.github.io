Transparency animation is not applied to character capture effects

Question

When I set an Outline capture effect in a text box pane and set a transparency animation, the animation is not applied correctly to the capture effect part.
An outline was set around text using a capture effect, and then a transparency animation was set to make the characters appear and disappear.

A A transparency animation with the Outline capture effect set.
The transparency is not applied correctly to the effect part.
B The transparency animation is applied correctly if the capture effect is not used.

Answer

The Outline capture effect does not support transparency animations.
You can achieve a similar effect using the Drop Shadow capture effect.
In the following example, transparency animation is applied to text that has been outlined using Drop Shadow.
The outlined characters appear without looking strange in any way.

LayoutViewer Property Window Settings

Set Spread to 0 to reduce the unnaturalness of the animation compared to the Outline capture effect.

Outline Settings

Text pane ()
　・Capture effect ()
　　• Enable Drop Shadows: On
　　・Angle: 0
　　・Distance: 0
　　・Spread: 0
　　・Size: 10
Info

The border of the capture effect is equivalent to drop shadow distance (=0) and spread (=100).
As a result, this workaround does not look exactly like a border.
However, if it displays acceptably with a border with spread=0 attached, it is possible to eliminate any unnaturalness seen when the characters appear (or disappear).
