Enabling Crash Reports in Development Environments

Introduction

This section describes how to enable sending crash reports in development environments to check the information that can be obtained with the Crash Report Retrieval Tool.

Using Crash Reports in the Development Environment

To use crash reports in the development environment, the following are required.

  • Setting configured when building the application
  • Settings for development hardware


This page describes the configuration method for each item.

DEV MODE and PROD MODE each require different settings. For more information about DEV MODE and PROD MODE, see Initialization for PROD MODE, the Firmware That Approximates Retail Hardware.

Info

The development hardware must be configured before creating crash reports.
Crash reports created before settings are configured are not obtained by the Crash Report Retrieval Tool even if sending is enabled later.

Settings for development hardware

To use crash reports in the development environment, you must configure the development hardware in addition to the application build settings.
Even if you enable crash reporting and sending screenshots when building the application, reports are not actually sent unless you enable sending on the development hardware.

In addition, the development hardware must be configured before creating crash reports.
Crash reports created before settings are configured are not obtained by the Crash Report Retrieval Tool even if sending is enabled later.

The following sections describe how to configure the development hardware for both DEV MODE and PROD MODE.

Configuring Settings in DEV MODE

You can configure settings for creating and sending crash reports in DEV MODE using DevMenu (DevMenuApp) or DevMenuCommand.

Info

When crash reporting is enabled in SDK versions earlier than NX Add-On 11.1.0, the feature to automatically generate CPU dumps is disabled.
In NX Add-On 11.2.0 and later SDK versions, you can use both crash reports and automatic CPU dump generation during crashes simultaneously.

Using DevMenu (DevMenuApp)

From the Error page, select Settings and configure the following settings. You must restart after you have finished updating these settings.

  • Select the Enable Upload check box to enable them to be sent.
  • To also enable sending screenshots during crashes, select the Enable Crash Screenshot check box.


Info

For the 10 series earlier than NX Add-On 10.8.0 and 11 series earlier than NX Add-On 11.3.0, there is no Enable Crash Screenshot check box.
To enable creating crash reports in SDK versions earlier than NX Add-On 11.1.0, you must select the Enable CrashReport Creation (= Disable JIT debug) check box.

Using DevMenuCommand

  • Run the DevMenuCommand error enable-report-upload command to enable sending reports.

  • To enable sending screenshots during crashes, run the DevMenuCommand error enable-crash-screenshot.

Info

For the 10 series earlier than NX Add-On 10.8.0 and 11 series earlier than NX Add-On 11.3.0, you cannot run the DevMenuCommand error enable-crash-screenshot command.
In SDK versions earlier than NX Add-On 11.1.0, run the DevMenuCommand debug disable-jit-debug before running the DevMenuCommand error enable-report-upload to disable JIT debugging. This command allows crash reports to be generated when the application crashes.

Configuring Settings in PROD MODE

For PROD MODE, do the following to enable sending crash reports.

  1. Link a Nintendo Account.
  2. Configure the Share Error Information setting.

Info

In PROD MODE, unlike DEV MODE, you cannot use the hardware settings to only disable sending screenshots.
In PROD MODE, if you want to send crash reports but disable sending screenshots, set CrashScreenshotForProd in the NMETA file to Deny.

1. Linking a Nintendo Account

Refer to Creating a Nintendo Account in the Development Environment and account Library > Linking With Nintendo Accounts and link the device's user account with a Nintendo Account.

When using a Nintendo Account in some European countries, you must agree to share user information from the Nintendo Account home page.

2. Configuring the Share Error Information setting

From System Settings, select System and then enable Share Error Information (as shown in the following screenshot).

When Avoiding Sending Screenshots

For in-development applications, there are cases where you might want to get crash reports but avoid sending screenshots to Nintendo's servers. In a case like that, you can use one of the following methods to avoid sending screenshots.

For DEV MODE (screenshots are not sent if any of the following conditions are satisfied) For PROD MODE
In the NMETA file, set CrashScreenshotForDev to Deny. In the NMETA file, set CrashScreenshotForProd to Deny.

Clear the Enable Crash Screenshot check box in the Settings on the Error page of DevMenu.
Run the DevMenuCommand error disable-crash-screenshot command.

If you disable sending crash reports itself, screenshots taken during a crash are not sent even if screenshot-related settings are enabled.

When Crash Reports Are Sent

After you have enabled sending crash reports as described previously, a crash report is automatically created and sent whenever the application crashes while connected to the Internet. If the application is not connected to the Internet when it crashes, the crash report is sent when the system is restarted with the appropriate network settings to establish an Internet connection.

In NintendoSDK NX Add-On 6.2.0 and later, it is possible to verify that a crash report has been sent by selecting the corresponding error report from the list on the DevMenu/DevMenuApp error page.

Info

The crash report database update interval is one hour. As a result, it can take up to an hour for a crash report to be obtained by the Crash Report Retrieval Tool after it has been sent by development hardware.
