NintendoWare Effect News

This document is a graphical introduction to the features that were added in NintendoSDK 0.6.0.

New Effect Expressions

This section introduces features that broaden the scope of what can be depicted with effects.

Scale Fade

Added scale fade when executing termination processing!

Usage Examples

Termination processing can set the behavior when executing processing (Fade command) to force emissions from the emitter to be stopped on the developer side.
The newly added "scale fade" is a feature by which a particle gradually becomes smaller and then disappears over the time set in Fadeout Time.

The following figure compares what happens when the various Termination Processing settings are selected or cleared.

A Stop emission: ON, Fade: OFF Click Execute fade command to stop particle emission.
Already emitted particles are displayed for their lifespans.
B Stop emission: ON, Alpha fade: ON Particle emission stops when you click Execute Fade command.
The particles grow gradually more transparent over the time specified in Fade-out duration and then die.
C Stop emission: ON, Scale fade: ON Particle emission stops when you click Execute Fade command.
The particles gradually shrink over the time specified in Fade-out duration and then die.
Info

You can check how the Termination Processing features look in the Viewer by clicking the Fade button () in the Preview controls.

Feature Improvements to the Equal Division Emitter Shape

Emission Type was added to Emitter Shapes with (evenly divided) in the preset name.
Also, Number of divisions was added to Circle (evenly divided) and Line (evenly divided).
With this improvement, the Number of divisions and Number of emissions can be separately specified when the Emission Type is Random or By Index.

The following settings are for Circle (evenly divided). The portion in the red frame includes the items added this time.

Example of a Together Emission Type

When the Emission Type is Together, all locations divided into Number of divisions emit particles at the same time.
Using the example of the Number of divisions set to 6, a circle emitter is divided into six divisions as follows, and particles are emitted from all locations at the same time every time.

Example of Random Emission Type

When the Emission Type is Random, particles are randomly emitted every time from a number of locations set with Emission rate when the locations are divided into the Number of divisions.
Using the example of the Number of divisions set to 6 and the Emission rate set to 2, two randomly selected locations from the six divisions emit every time, as follows.

Example of By Index Emission Type

Particles are emitted every time from a number of locations set with Emission rate in the order that the locations were divided using the Number of divisions.
Using the example of the Number of divisions set to 6 and the Emission rate set to 2, two locations from the six divisions emit at a time in order, as follows.

Info

The following notes are about compatibility.
The emission rate is a fixed value for Circle (evenly divided) and Line (evenly divided) for emitter Shapes and data using emission rate animation.
Also, the value the emission rate at the beginning of the animation is inherited by Number of divisions as a fixed value.

Related Links For more information, see the following pages.

Circle (Evenly Divided) Particles emit in a disc shape from locations evenly divided around the circumference of a circle.
Sphere (evenly divided) Particles emit in a sphere shape from equidistant locations around a divided sphere.
Sphere (Evenly Divided 64) Particles emit in a sphere shape from locations around the perimeter of a sphere divided into anywhere from 4 (minimum) to 64 (maximum) sections.
Line (Evenly Divided) Particles emit from locations along an evenly divided line in the direction in which the line is drawn.

Setting a Constant Texture Density

Added a Set a Constant Texture Density feature to display the texture display size in the same size, unaffected by the particle scale.

Usage Examples

When particles are scale animated, the animation is not influenced by the particle texture coordinates.
For example, this feature may be used in the physical fitness cage as shown in the figure, or to display above a game character's head in 3D space.

Limit Scaling in Front of Camera

The Limit Scale in Front of Camera feature applies a limit to the scaling of particles.

Usage Examples

In the following example of using the Limit Scale in Front of Camera feature, A is the emitter before the effect is applied, and B is the emitter after the effect is applied.
This feature can reduce rendering costs by controlling particle visibility based on proximity to the camera.
By using this feature with the Near Alpha feature, the particle transparency can be changed and disappear naturally as it approaches the camera.

GPU Noise

Added GPU Noise that allows regulated movement that appears to be random in the field.
Particles move on a trajectory (the waveform) determined at the time of emission.
For this reason, the trajectories on which the particles move can, to an extent, be more controlled than the Random field.

The waveform that defines the trajectory of movement of a particle is formed from a combination of four waveform components.
In the following figure, two waveforms are added together to form a single waveform. The particles move on this waveform.

Usage Examples

In the following example, GPU Noise is used for the movement of a group of fish moving along a determined trajectory and on the wobbling motion of rising bubbles along with scale animation expansion and contraction.
Because it is difficult to control with the Random field, sometimes animations probably have long dimensions such as groups of fish where the animation goes off screen and does not return.

EffectMaker

This section introduces UI elements newly added to EffectMaker.

Displays information about registered primitives for primitive emitter shapes and particle shapes.
You can check various aspects of a primitive being used, such as whether it has the intended data structure, and whether it has any unnecessary information.

Changing Values Handled by Gravity and Curl Noise

Revised so that the Gravity and Curl Noise values can be edited to large values.

The following values are from the previous version or have been changed to have the same behavior.
When loading a previous intermediate file, this content is automatically converted.

Item Name Previous Value Changed Value Description
Gravity 0.001 0.1

The Gravity emitter node has the same behavior at 100 times the previous value.
Curl Noise 0.05 0.5 The "strength of influence" of the Curl Noise node has the same behavior at 10 times the previous value.

Infinite Lifespan Is Limited Only When One Time Settings Are Enabled

Placed limitations so that Infinite lifespan functions only when the emission setting One Time is enabled.
This prevents an user from increasing the number of particles in a game application unintentionally by a mistaken operation such as carelessly disabling the One Time setting.

Viewer Control Features Added to Editor

Added a new Viewer node Debug tab.
The following items are arranged in this tab to control view features.

Linear Edit Mode When enabled, the view display is changed to linear.
Playback Speed The playback speed on the Viewer is changed by multiplying by the set value.
Ground Display Type Selects the ground display type to display on the Viewer from the following presets:
Grid

This grid is displayed by default. Each unit of the grid is a rectangle equivalent in size to 10 × 10 particles.
Hi-Grid This grid is a high-mesh grid. Each unit of the grid is a rectangle equivalent in size to 1 × 1 particles.
Blocks

This grid type has a grid face filled with a gray color.

It is useful for confirming the effect of color buffer refraction or soft particles.
No Hides the ground. Useful to confirm particles only.
Binary Transfer Feature Effects can be confirmed on the Viewer by loading the exported binary file.
