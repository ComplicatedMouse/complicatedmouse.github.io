nn::g3d News ()

3D Texture Converter

Changed the specification to produce an error if GPU encoding is not possible when false is not specified for --gpu-encoding.

The specification has been changed to produce an error if GPU encoding is not possible when false is not specified for the --gpu-encoding option. If you want to encode in the GPU, if possible, and in the CPU otherwise, specify --gpu-encoding auto.

--gpu-encoding Value Behavior
Not specified Same behavior as when true is specified.
true GPU encoding if possible, or an error otherwise.
false Encoding always takes place in the CPU.
auto Encoding takes place in the GPU if possible, or in the CPU otherwise (matching the past behavior of true).

You can change the behavior in the Photoshop Plug-in or Maya Plug-in using gpu_encoding="true"/"false"/"auto" in the config file.

Improved the quality of the alpha component when compressing to the BC3, BC7, and ASTC formats.



The quality of the alpha component when compressing to the BC3, BC7, and ASTC formats using Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) has been improved.

Before Improvement After Improvement

It is now possible to determine whether GPU encoding is possible in the current environment.

Specify the following option in Texture Converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTextureConverter.exe) to determine whether GPU encoding is possible in the current environment.

3dTextureConverter.exe --check-gpu-encoding

If GPU encoding is possible, the 3D Texture Converter exit code (%ERRORLEVEL%) is 0.

3D Shader Converter

Added a feature for using machine distributed compilation.

The following options have been added to perform distributed compilation on machines running in parallel: the --variation-begin-ratio option and the --variation-end-ratio option to compile only some shader variations, and the --skip-convert option to skip non-compilation processing. For more information, see 3D Shader Converter and Distributed Compilation.

G3dDemo Samples

Changed the resource used in the ShapeAnimation demo.

Updated the resource that runs shape animations in the ShapeAnimation demo.

  • The number of vertices and key shapes have been increased, making it easier to know the difference between CPU and GPU processing.
  • In addition to vertex positions, normals are also animated.

nn::atk News ()

Added a feature to add sound archives after initializing the sound archive player.

The Add-On Sound Archive feature has been added to allow sound archives to be added after the sound archive player is initialized.

This feature allows you to add sound archives to the sound archive player when they are needed, and delete them when they are no longer needed.
The feature is suitable for use with sounds that are needed only in specific situations, and sounds that are included in downloadable content.

For more information, see the following pages.

  • Guide > Add-On Sound Archives
  • Programming Guide > Loading Resources

nn::spy News ()

Support for the Add-On Sound Archive Feature of nn::atk

The addonSoundArchiveCountMax member function was added to the nn::spy::atk::AtkSpyModule::InitializeArg structure to support the nn::atk feature for add-on sound archives.
