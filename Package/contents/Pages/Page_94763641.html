Version 0.7.0 News

This section presents news about NintendoSDK 0.7.0 that affects NintendoWare overall.

Switch to Use of TargetManager for Host-Target Communications

With the release of NintendoSDK 0.7.0, communication between the host PC and the various viewers and applications is now done via the Windows tool called TargetManager. With this change, TargetManager now serves as an intermediary when Windows tools like 3DEditor, LayoutEditor, and EffectMaker communicate with the various viewers and applications.

Warning

To use TargetManager for communication between a Windows tool and a viewer, you must start TargetManager first, and then start the viewer, and then start the tool, in that order.
If you installed the NintendoSDK package via NDI, TargetManager has been added to Windows startup.

Info

In NintendoSDK 0.7.0, Winsock is used for connections between the following tools and runtimes.

  • Connections between SoundMaker and nn::atk. 　(The plan is to add support for connections via TargetManager in the next version of the SDK.)
  • Connections between SoundMaker and SoundPlayer / nw::snd.
  • Connections between EffectMaker and EffectViewerNw4f/EffectViewer.exe / nw::eft2.
  • Connections between LayoutEditor and LayoutViewer-Nw4f/Win32/lytviewer.exe / nw::lyt.

Addition of the nn::ige Library

The nn::ige library is used for communications via TargetManager between host and target. As a result, if you plan to implement viewers in your application using the viewer libraries of the various modules, you must link to the nn::ige library and build the application.

  • nn::g3d::viewer
  • nn::ui2d::viewer
  • nn::vfx::viewer
  • nn::atk::viewer (Support planned in the next version.)
