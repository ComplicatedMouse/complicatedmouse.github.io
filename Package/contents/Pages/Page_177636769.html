Introduction

This page introduces the various IDs used for NintendoSDK NX Add-On.

The meanings of the terms used on this page and the ways in which those terms are used may differ from the generally known meanings.

Procedural Issues

Product Code

Product Category Code - Form Code - Initial Code (Market Code)
HAC - x - xxxxx (xxx)
  • Product Category Code
    • HAC will be assigned.
  • Format Code
    • A single alphanumeric character will be assigned based on the product type.
  • Initial Code
    • A string of five alphanumeric characters.
  • Market code
    • A 3-character string indicating the region.

Program-Related IDs

Application ID

A unique ID assigned to each application.
Specified using a value in the range from 0x0100 0000 0001 0000 to 0x01FFF FFFF FFFF FFFF.
For retail applications, the application ID is the value that was specified on the Nintendo Developer Portal.
For development tools and prototypes, use the value 0x0100 XXX0 0348 8000, where XXX is any value between 000 and FFF.

For the NX, the application ID is used for the following identifiers.
If the application ID for another product is mistakenly used, it will have the noted effects.

  • The ID for content installed on the NX system
    • If you install content with the same ID as content that is already installed, the older content will be overwritten.
  • The ID when the ROM is submitted on the Nintendo Developer Portal
    • Submittable IDs are managed at the NDID level, so the wrong ID cannot be submitted.
  • The ID of the application when accessing network services
    • Each application is issued a separate access key, so an application without the right access key cannot access another application's network service.
