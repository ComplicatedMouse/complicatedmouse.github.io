Errors and Warnings

This section describes the errors and warnings of the export plug-in.

Verifying Errors

If an error or warning occurs during export, it is exported to the listener window.

Some errors are also displayed in a message box.

Error Types and Handling Errors

The following list of current errors is presented in alphabetical order.

When an error is detected, the export process stops.

  • Animation file to merge is not found
     No animation files to merge could be found even though the export option Merge Animation Folder is enabled and a folder has been specified.
  • Can't open file
     The file does not exist, cannot be opened, or cannot be overwritten.
  • Comment is wrong
     The Comment export option includes one or more prohibited characters. Comments cannot include semicolons (;) or quotation marks (").
  • DDS file is wrong
    The content or format of the DDS file is invalid. See Supported Formats.
  • Image type is not supported
     An unsupported image file is being used. Currently, only TGA files and DDS files can be used as texture files.
  • Influence object is not outputted
    Nodes influenced by skinning are not output. This error arises when Export Target is set as Selection and nodes influenced by skinning are not selected, or when no envelope has been assigned to a vertex even though a skinning modifier and Physique modifier are being applied.
  • Merge Animation Folder is wrong
    This occurs if the Merge Animation Folder export option includes prohibited characters. Full-space characters, half-space kana characters, and semicolons (;) cannot be used when specifying the Merge Animation Folder.
  • Merge Animation Name is wrong
    This occurs if the Merge Animation Name export option includes prohibited characters. Full-space characters, half-space kana characters, and semicolons (;) cannot be used when specifying the Merge Animation Name.
  • No effective node
     There are no active nodes to output.
  • No output file
    All export specifications for the intermediate file are cleared, or only the export specification for texture data is selected but there is no texture to export.
  • Output File Name is wrong
    The Output File Name Export option is blank, or the filename contains invalid characters.
  • Output Folder is wrong
    The Output Folder Export option is blank, or the name contains invalid characters.
  • Path of fmd file to merge is wrong
     This error is generated when the Merge fmd File export option is selected and also an FMD file path is specified, but the FMD file to be merged does not exist or the path includes one or more prohibited characters. Full-space characters, half-space kana characters, and semicolons (;) cannot be used in the path.
  • Post-Export Script failed
    An error occurred in the post-export script (MAX Script). See the listener window for a description of the error.
  • Pre-Export Script failed
  • An error occurred with the pre-export script (MAX Script). See the listener window for a description of the error.
  • Preset Name is wrong
     The Preset Name field was left blank or contains prohibited characters. Preset names cannot use the characters that cannot be used for filenames, nor double-width characters or single-width katakana characters or semicolons. Also, None cannot be used as the name for a preset.
  • Sampler hint is wrong
     The sampler hint includes one or more prohibited characters.
  • Sampler hint is duplicate
    The same sampler hint is used more than once by the same material. Use the Plug-In for Setting Sampler Attributes to set hints that are not duplicates.
  • Sampler name is wrong
     The sampler name includes one or more prohibited characters.
  • Sampler name is duplicate
    The same sampler name is used more than once by the same material. Use the Plug-In for Setting Sampler Attributes to set names that are not duplicates.
  • Texture size is wrong
    The target platform cannot use textures with this size height and width.
  • TGA file is wrong
     The TGA file contents are invalid.
  • The number of weighted influence is over
    The skinning weight is being distributed to more nodes than specified in the config file.
  • Wrong weighted vertices exist
     Vertices have a total skinning weight that is less than 1.0. Make sure that you correct the weight value.

Warning Types and Handling Warnings

The following list of current warnings is presented in alphabetical order.

The export process continues even if a warning is detected.

  • ftx file to merge is not found
     No FTX files to merge could be found even though the Merge ftx File export option is selected. Files are exported without merging.
  • Inheritance of node link info is off
    One or more of the Hierarchy, Link Information, or Inheritance settings are cleared. Objects do not display correctly unless all of these settings are selected.
  • Map channel does not exist
    A channel n map is set in an object for which map channel n does not exist.
    Because data cannot be exported in this state, the export forcibly pastes the UV information for map channel 1.
  • Material name is changed
     A material name exported to the intermediate file was changed because two or more materials had the same name.
  • Morpher modifier must be under skin (physique) modifier
    The morpher is above skin (or physique).
    The morpher modifier must be below the skin or physique modifier.
  • The shape animation is not output.
  • Morpher target topology is different from original mesh
    This error is generated when the morpher target shape and the base shape differ in terms of the number of vertices, number of surfaces, or number of vertex colors.
  • Node has no material
  • No material has been set for a node. A dummy material is exported.
  • Node name is changed
    If two or more nodes have the same name, or if prohibited characters are included in node names, the names are changed when the nodes are exported to the intermediate file.
  • Shader is not supported
     A shader not supported by the Export plug-in is being used. When this warning occurs, the plug-in automatically adds and exports default material to the intermediate file.
  • Texture file doesn't exist.
    The assigned texture does not exist. If a path is included in the filename for a texture assigned to a material, the image saved in that path is referenced during export.
    This warning occurs when no image exists in that path. If the texture filename does not include a path, the image is referenced in the directory where the current MAX file is stored.
  • Texture name is changed
     The texture name contains prohibited characters. Prohibited characters are converted to an underscore (_), and then exported.
  • Texture type is not supported
  • The only texture maps that can be used are bitmaps, normal maps, and composite maps.
  • There are multiple texture files with the same name
     Two or more texture files have the same filename as the export target, but are in different folders or have a different extension. Note that only one FTX file will be exported.
  • Zero normal exist
    There is a normal vector with a length of zero. Lighting effects will not work properly. Fix this by using the Edit normal modifier or a smoothing group.
  • Zero tangent or binormal exists
    There was a tangent or binormal vector with a length of zero, so it was replaced with (0, 1, 0) and exported. Be sure to correct the normal vector and UV coordinates.
