Filter Syntax

Write a conditional expression in the target log filter condition box, with a data key for the log specified to the left of the operator and an operand specified to the right, as shown below.
If you enter a filter conditional expression and press ENTER, only those logs that conform to the filter will be displayed.

(Data key) (Operator) (Operand)
PeerName == "MyTarget"

To cancel the entered filter, clear the filter condition box, leaving it blank, and press ENTER.

Data Keys

Info

The TargetName key is available in Nintendo Target Manager 2 version 11.4.0.0 and later.

Info

The ApplicationID key is available in Nintendo Target Manager 2 version 14.1.0.0 and later.
The firmware version on the development machine side must also be 14.1.0 or later.

Key Type Description
TargetName String The name of the development hardware system that generated the log.
PeerName String The peer name of the development hardware system that generated the log.
ThreadId Number The ID of the thread that generated the log.
Severity Number The severity of the log.
Verbosity Number The log verbosity level.
LineNumber Number The number of the line that generated the log.
UserSystemClock Timestamp The time on the development hardware when the log was generated.
FileName String The name of the file that generated the log.
FunctionName String The name of the function that generated the log.
ModuleName String The name of the module that generated the log.
ThreadName String The name of the thread that generated the log.
Timestamp Timestamp The time when the host PC received the log.
ApplicationId Number Application ID of the program that output the log
TextLog String The body text of the log.

Operands

Operand Description
Log key The data key of the log to use for filtering.
Numeric literal An integer in decimal notation, or in hexadecimal notation and starting with 0x
String literal A string enclosed in double quotation marks.
Example: "Text"
Regular expression literal A string pattern enclosed in forward slashes.
Example: /RegexPattern/
Timestamp literal

A data stamp enclosed in quotation marks, written in either YYYY-MM-DD HH:MM:SS or MM-DD-YYYY HH:MM:SS format.

Example: "2020-06-15 09:58:33"

Operators

Operator Description
( ) Groups conditional expressions, specifying that the conditional expression written in parentheses be evaluated first.
! Inverts the evaluation result of the filter conditional expression.

>, >=, <, <=

 Comparison operators for numerical values and timestamps. Specify the same type of data key or literal on both sides.

==, !=

 Evaluates the equivalence of strings, numerical values, and timestamps. Specify the same type of data key or literal on both sides.

=~, !~

 Tests the regular expression match of a string. Specify a string-type data key on the left and a regular expression literal on the right.
&& Describes a condition that is true when two filter conditional expressions are combined and both are satisfied.
|| Describes a condition that is true when two filter conditional expressions are combined and either is satisfied.
Info

Use regular expression matching if you want to extract the log with partial string matches.
The equality comparison checks the entire string for a match, so a log cannot be extracted if it includes a different string.

Filter Examples

Displaying Logs That Include the String Hello

TextLog =~ /Hello/

Displaying Logs for the Thread Named abc

ThreadName == "abc"

Displaying Logs With a Severity Less Than 4

Severity < 4

Displaying Logs for the Thread Named report or Logs With a Severity of 3 or Higher

(ThreadName == "report") || (Severity >= 3)

Displaying Logs That Include the TestApp Filename and Also Contain the Words OK or NG in the Body

(FileName =~ /TestApp/) && (TextLog =~ /OK|NG/)

Displaying Logs Received on or After 9:58:33 on June 15, 2020

Timestamp >= "2020-06-15 09:58:33"
