VI on Windows

Overview

A major goal of the VI library is to minimize the differences in using display functionality when targeting Nintendo platforms.  When there are any differences, they will be detailed here.

The VI library may be used for any Generic and NX specifications.  Both Windows and Horizon operating systems are fully supported.

For any Windows implementation of the VI library, please note the following behaviors:

  • Certain functions will always succeed, but they will not appear to have any effect.  Only the preconditions are verified in these cases.  The following functions are not fully supported on Windows:
    • nn::vi::SetLayerCrop()
    • nn::vi::GetLatestFrameNumber()
    • nn::vi::ListFrameHistory()
  • nn::vi::GetDisplayVsyncEvent()
    • While NX only supports certain resolutions with a refresh rate of 60 Hz, the vsync event on Windows will signal according to the monitor's refresh rate.  It will not emulate a 60 Hz signal.
    • Only the primary monitor's vsync is used for event signalling.  The signal time will not be adjusted if the window is moved to a secondary monitor.  If the primary and secondary monitors have different refresh rates, then the vsync signal and texture acquire time may be out of sync.

Generic on Windows

  • nn::vi::CreateLayer()
    • The width and height parameters are used to determine the window size.
  • nn::vi::LayerCreationSettings for nn::vi::CreateLayer()
    • The width and height settings are used to determine the window size when nn::vi::LayerCreationSettings::SetFullscreen() is set to false.  (This setting is false by default.)
    • If nn::vi::LayerCreationSettings::SetFullscreen() is set to true, the window is guaranteed to be borderless fullscreen on some monitor.  The width and height settings are used to find an exact match for a monitor resolution.  If such a monitor with that exact resolution does not exist, the window will be borderless fullscreen on the main display.
    • If nn::vi::LayerCreationSettings::SetVisibility() is set to false, the window will be created, but not appear on any display.

NX on Windows

  • nn::vi::CreateLayer()
    • The width and height parameters are ignored.  The client area (i.e. the portion of the window where application content is displayed) will always be 1280x720p.
  • nn::vi::LayerCreationSettings for nn::vi::CreateLayer()
    • The width and height settings are ignored.
    • The nn::vi::LayerCreationSettings::SetFullscreen() option is ignored and will have no effect.
    • The nn::vi::LayerCreationSettings::SetVisibility() option is ignored and will have no effect.
