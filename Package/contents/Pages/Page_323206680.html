Introduction

This section describes how to configure and use retail interactive demo (RID) mode while providing important information about its use.
You must enable RID mode to test the behavior of RID demo applications.

Note
  • The asset data NSP file created on this page is a dedicated file for validating operations in the development environment. It cannot be submitted to the Nintendo Developer Portal.
  • Creation and submission to Lotcheck of the retail environment asset data files used in stores is handled by Nintendo personnel.
    • The information and data needed to create the asset data file must be provided to Nintendo. For more information, contact your Nintendo representative.

Preparations

You must complete the following steps to enable RID mode.

  • Finish the HOME Menu’s initial startup sequence.
  • Create one or more user accounts.
  • Install the retail interactive demo menu.
  • Get the latest NintendoSDK SystemUpdater for NX package.
Info

The NintendoSDK SystemUpdater for NX package includes the files that are needed to enable RID mode.
Testing in RID mode must occur on the latest version of NX firmware.

To start your own demo application, you need the NintendoSDK NX Add-On package and NintendoSDK AuthoringToolKit for NX, which are provided separately.

How to Use

Enabling Retail Interactive Demo Mode

  1. Install the retail interactive demo menu and DevMenu Application Asset Data on the devkit.
    • Running the BAT file installer starts DevMenuCommand and begins the installation.
      • $NINTENDO_SDK_ROOT\TargetTools\NX-NXFP2-a32\InStoreDemoMenu\Release\InstallNsps.bat
    • The following NSP files are installed.
      • $NINTENDO_SDK_ROOT\TargetTools\NX-NXFP2-a32\InStoreDemoMenu\Release\InStoreDemoMenu.nsp
      • $NINTENDO_SDK_ROOT\TargetTools\NX-NXFP2-a32\InStoreDemoMenu\Release\AssetData_DevMenuApplication.nsp
  2. Enable RID mode.

    • Either open the DevMenu Debug page and in In-Store Demo Mode, select Enabled, or run the following DevMenuCommand.

      $NINTENDO_SDK_ROOT\Tools\CommandLineTools\RunOnTarget.exe $NINTENDO_SDK_ROOT\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- debug enable-in-store-demo-mode
  3. Restart the system.
    • The RID menu starts automatically with the demo application’s sample packages listed in the launcher.

Creating the Demo Application’s Asset Data

Asset Data is the NSP file for showing a demo application on the RID menu.
There are two ways to create these Asset Data files: the batch file included in AuthoringToolKit or a PowerShell script.

Using the Batch File

Creating With Default Settings

  1. Drag the demo application's NSP file to the batch file for creating Asset Data.
    • $NINTENDO_SDK_ROOT\Tools\CommandLineTools\AssetDataCreator\CreateAssetData.bat
  2. AssetData.nsp is created in the following path in the same directory as the batch file.
    • __output_yyyymmdd_hhmmss\AssetData.nsp

Creating With Custom Settings

Edit CreateAssetData.bat as needed. All of the configuration items are optional, and default values are used if they are not changed.

Settings Default Value Overview Details
PLAYABLE_TIME 900 The elapsed time, in seconds, after starting the demo software and before the software automatically closes. Specify an integer greater than or equal to 0.
Specifying a value of 0 prevents the application from exiting automatically.
EXIT_MESSAGE_Validity enabled State where the termination message displayed by the RID menu is enabled when the demo application closes. When set to enabled, the image specified by EXIT_IMAGE_FILE_PATH is displayed.
Changing this setting to disabled prevents a message from being displayed when the demo application closes.
Disable this option if the demo application displays its own message when it closes.
EXIT_IMAGE_FILE_PATH Thank you for playing image The image displayed on the demo application exit screen. PNG format (1920 × 1080 × 32, RGBA) is supported.
Specify an absolute path. The specified image is not displayed when EXIT_MESSAGE_Validity is disabled.
IDLE_TIME_TO_EXIT Value set by Nintendo The amount of time, in seconds, that the controller is allowed to be in the idle state. In the production environment, the default value set by Nintendo is different for every unit.
The system closes the application after the set amount of idle time has passed.
Specify 1 or more if you want to change this value.
PACKAGE_IMAGE_FILE Application ID image The package image of the demo application that is displayed in the RID menu. PNG format (264 × 430 × 32, RGBA) is supported.
Specify an absolute path.
MOVIE_FOR_IDLE_TIME Switch logo image video Demo application video that is played if the RID is left idle for a certain amount of time. MP4 format is supported. (For more information, see Video File Formats later on this page.)
Specify an absolute path.
RELEASE_DATE Empty string The release date displayed in the demo application description (YYYY-MM-DD). You must specify a date in 2017 or later, in the format of YYYY-MM-DD.
If not specified, the release date information is not displayed.
RATING_VALUE 0 Rating values for each rating organization. Specify the rating values for each rating organization together as an integer greater than 0.
If not specified, the rating value is 0 and the state is unreviewed.
For more information, see Rating Settings later on this page.
VOLUME_LEVEL 100 Percentage of the master volume.

Set a percentage of master volume for the running demo application, specifying a value between 0 and 100.
If this setting is not specified, the audio is not at 100% of the master volume.

Info

Available only with AuthoringToolKit 8.3.1 and later.
Example Configurations
set PLAYABLE_TIME=900
set EXIT_MESSAGE_Validity=enabled
set EXIT_IMAGE_FILE_PATH=C:\exit.png
set IDLE_TIME_TO_EXIT=600
set PACKAGE_IMAGE_FILE=C:\package.png
set MOVIE_FOR_IDLE_TIME=C:\movie.mp4
set RELEASE_DATE=2017-03-03
set RATING_VALUE=5
set VOLUME_LEVEL=90

Using a PowerShell Script

powershell -ExecutionPolicy Unrestricted -File $TOOLDIR_PATH\CreateAssetData.ps1 -inputNsp <input nsp path> 
          [-outputPath <output nsp path>] [-workDir <working directory path>] [-index <number>]
          [-playableTime <seconds>] [-idleTime <seconds>]
          [-exitMessage <enabled|disabled>]  [-exitImage <image file path>]
          [-packageImage <image file path>] [-idleTimeMovie <movie file path>] [-releaseDate <YYYY-MM-DD>] [-rating <number>] [-volume <number>]
  • $TOOLDIR_PATH indicates the directory ($NINTENDO_SDK_ROOT\Tools\CommandLineTools\AssetDataCreator) that contains CreateAssetData.ps1.
  • inputNsp is the only required setting.

Options

For items that have the same settings as those used for batch files, see Using Batch Files > Creating With Custom Settings.

Options Default Value Overview Details
inputNsp
Path to the NSP file for the demo application.
outputPath $TOOLDIR_PATH\__output_yyyymmdd_hhmmss\AssetData.nsp Output path for the asset data (including the name of the NSP file).
workDir $TOOLDIR_PATH\__work Work directory path. If the work directory does not exist, it is created automatically with the default values.
If the work directory uses the default values, it is deleted after exiting the tool.
index 1001 Asset data index (50–1999). Specify unique indices to show multiple demo applications with different application IDs.
playableTime See PLAYABLE_TIME in Using Batch Files > Creating With Custom Settings.
idleTime See IDLE_TIME_TO_EXIT in Using Batch Files > Creating With Custom Settings.
exitMessage See EXIT_MESSAGE_Validity in Using Batch Files > Creating With Custom Settings.
exitImage See EXIT_IMAGE_FILE_PATH in Using Batch Files > Creating With Custom Settings.
packageImage See PACKAGE_IMAGE_FILE in Using Batch Files > Creating With Custom Settings.
idleTimeMovie See MOVIE_FOR_IDLE_TIME in Using Batch Files > Creating With Custom Settings.
releaseDate See RELEASE_DATE in Using Batch Files > Creating With Custom Settings.
rating See RATING_VALUE in Using Batch Files > Creating With Custom Settings.
volume See Using the Batch File > Creating With Custom Settings > VOLUME_LEVEL.

Installing the Demo Application

  1. Install the demo application and the generated asset data (NSP file) on a devkit.
  2. Restart the system.
Info

If you used DevMenuCommand to install the NSP file with RID mode enabled, the HOME Menu displays an error.
The installation itself has succeeded, so it is safe to restart the system.

Starting and Terminating the Demo Application

  1. Select and run the demo application from the RID menu.

    • Before the application starts, the RID menu displays the rating screen and a description of how to exit the application.

  2. Close the application.

    • There are multiple methods for closing the application. When the application exits, you return to the RID menu.

      • Press the HOME Button to exit the application from the menu.
      • The application closes automatically if there is no controller input for 60 seconds.
      • The application closes automatically after a certain period of time has elapsed.
      • The application itself calls nn::oe::ExitApplicationAndGoBackToInStoreDemoMenu() and exits.

Disabling Retail Interactive Demo Mode

Either open the DevMenu Debug page and in In-Store Demo Mode select Disabled, or run the following DevMenuCommand.

$NINTENDO_SDK_ROOT\Tools\CommandLineTools\RunOnTarget.exe $NINTENDO_SDK_ROOT\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- debug disable-in-store-demo-mode

Testing in PROD MODE

You cannot enable and disable the RID menu using DevMenuCommand when the development hardware is initialized with the PROD MODE firmware. For this reason, you must enable or disable it using the DevMenu application.
From the HOME Menu, start the DevMenu application and install its asset data on the devkit by following the instructions under Installing the Demo Application.

Important Information

RID mode is intended to allow application developers to check the behavior of their demo applications.
As a result, it exhibits slightly different behavior than the RID menu that is actually used by retailers.

  • The description video from About Nintendo Switch cannot be played.
  • The demo application’s Information screen is mostly empty.

The following behaviors can be adjusted by changing the input when asset data is being created.

  • The package image displayed in the RID menu launcher is the sample image for the Application ID only.
  • The Nintendo Switch logo appears when the system is left unattended and you have not installed any asset data containing a movie.
