3D News

This document is a graphical introduction to the features that were added in NintendoSDK 0.17.0.

3DEditor

This section describes features that are newly added to 3DEditor.

Ability to Run Optional Processing for Model Intermediate Files From the GUI

Now able to run optional processing on certain model intermediate files from the shortcut menu, including removing the quantization settings for certain materials within certain models.

Related Links For more information, see <UserCommand>.

Ability to Send Any Script to Models for Applications to Connect

Now able to run optional processing on certain models within applications to connect to, from the shortcut menu, including focusing on certain models.

Related Links For more information, see <RuntimeUserScript>.

Note

User Command and Runtime User Script will not appear in the UI unless commands are configured using team settings.


Maya Plug-In

This section describes the features newly added to the Maya plug-in.

Bone Visibility, as Set in Maya, Is Now Applied When Models Are Merged for Export

By writing the config file as shown below, the bone visibility as set in Maya is now applied when models are merged for export.

bone_visibility_merge_priority="true"

Related Links For more information, see bone_visibility_merge_priority in Using the Export Plug-In.

Submesh Division Now Can Be Used When Exporting LOD

Up until now, submesh division could only be used together with NintendoWare's polygon reduction. But now, submesh division can also be applied when exporting LOD using any LOD model.

Using this feature in the runtime has restrictions compared to using submesh division together with NintendoWare's polygon reduction, therefore, consult with your programmers before use. For more information about restrictions, see the LOD section in the Nintendo 3D Programming Guide.

Related Links For more information on submeshes, see Model Geometry Data Structure in Models.
