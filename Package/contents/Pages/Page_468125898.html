Preparing to Use the Library

Overview

This page summarizes common items that are required before using the async library. Make sure that you read this before actually using the async library.

nn::MemoryResource Instance Preparations

The async library requests objects that implement the nn::MemoryResource interface for dynamic memory allocation. Prepare an appropriate nn::MemoryResource instance in the application in advance.

We recommend using the nn::MemoryResource instance that is available using nn::util::pmr::NewDeleteResource() if the application has no particular need for memory allocation. If a unique nn::MemoryResource instance is generated in the application, note the following.

  • Ensure that alignment requests larger than alignof(std::max_align_t) are correctly satisfied. For example, the memory area allocated by std::malloc() usually does not satisfy this requirement.
  • Each member function must be thread-safe. In other words, make sure that it works correctly even if multiple threads call the same member function at the same time for the same MemoryResource instance.
