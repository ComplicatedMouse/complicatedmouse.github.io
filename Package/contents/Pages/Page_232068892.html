Notes When Re-exporting

This section presents notes about re-exporting the data. It describes notes about the workflow when modeling and when creating textures.

In the following cases, you must configure the merge export settings.

  • When re-exporting model data (the FMD file) from Maya after editing the model's material and other properties in 3DEditor.
  • When re-exporting texture data (the FTX file) from Maya after changing the format of the texture or adding comments in Photoshop and exporting it as an FTX file.

If you re-export the data without properly configuring the merge settings, some of the edited content will be overwritten and lost.

Info

About Merging

Broadly speaking, the data in the model intermediate file can be classified into two groups: data representing the shape and structure of the model, and data for the material settings (the settings for the game's shader). The former data is created using a DCC tool, while the latter data is configured using 3DEditor after exporting the created data from the DCC tool.
When you merge export, the material settings of the model intermediate file into which the data is being exported are retained, and only the model's shape and structure data is overwritten.

In the table below, an FMD file is exported by way of example.

The Model's Shape and Structure Data
(Created in Maya)

The Material Settings
(Set Using 3DEditor)

Normal Export The model's shape and structure data. Overwritten. The material settings (shader settings). Overwritten.
Merge Export The model's shape and structure data. Overwritten. The material settings (shader settings). Retained.

Merge Settings

This section describes the merge settings.

The settings are configured as depicted in the figure below. If there is no file to merge with, the data are exported in the normal way, so the merge settings can be enabled from the start.

Item Description
A Merge FMD File

When this option is selected, the FMD files will be merged.

The model will be updated, retaining the edits made to materials and other properties in 3DEditor.

(If this option is cleared, the edits made to shader definitions and the like in 3DEditor will cease to be in effect.)
B Merge fmd File Path

If the path field is left blank, the data will be merged with the FMD file specified by Output File Name in the folder specified by Output Folder.

(If you specify an FMD file path, the data will be merged with the FMD file in that specified path and written to the path specified for Output Folder.)
C Merge ftx File

Updates the texture while retaining edits made to the format setting, added comments, and the like.

(If this option is cleared, the settings for the format and the like that were configured in Photoshop when the FTX file was exported will cease to be in effect.)
Note

When the Merge fmd File option is selected, some of the Maya settings will cease to be in effect in the FMD file. These settings will need to be configured using 3DEditor.

  • Example 1: Texture replacement.
  • Example 2: Texture repeat UV.
  • Example 3: Texture mirror and wrap.
  • Example 4: Material color (varies depending on shader).

Materials are merged if they have the same name. If you change the material name in Maya they will not be merged.

Info

In the workflow for some projects, the texture data is not exported from Maya.
In such projects, the Merge ftx File option can be cleared and kept cleared.
