CharModel

CharModel

Overview

nn::mii::CharModel is the class with the shapes, textures, and other data of the parts that comprise a Mii face model.

For more information about the parts that make up a Mii face model, see Mii Face Models.

CharModelInfo

nn::mii::CharModelInfo stores the information that is needed to create a Mii face model, including the resolution of the face outline textures and the mask textures, and the coordinate system.

Dynamic Texture Settings

The texture format, the resolution, and the number of mipmaps, are all properties that can be freely specified for face outline textures and mask textures.

Use the following functions to specify these properties.

  • nn::mii::CharModelInfo::SetDynamicTextureFormat()
  • nn::mii::CharModelInfo::SetDynamicTextureMipCount()
  • nn::mii::CharModelInfo::SetDynamicTextureResolution()

Use the nn::mii::CharModelInfo::SetMaskCount() function to specify how many mask textures to use. If you want CharModel to hold mask textures for a certain number of facial expressions, prepare mask textures for that number of expressions, and render an expression mask to each of these textures.

Conditional Creation of Mii Face Models

By setting a combination of flags, you can specify what type of Mii face model to create. For more information about the types of Mii face models, see Mii Face Models.

To specify the type, pass a logical OR of nn::mii::CreateFlag values to the nn::mii::CharModelInfo::SetCreateFlag() function. You must specify at least one among nn::mii::CreateFlag_Normal, nn::mii::CreateFlag_Hat and nn::mii::CreateFlag_FaceOnly, and at least one among nn::mii::CreateFlag_NoseNormal and nn::mii::CreateFlag_NoseFlatten.

Coordinate System

The default coordinate system when the Mii face model is initialized, has the top of the head pointing in the positive direction along the y-axis, and the face pointing in the positive direction along the z-axis.

If the application uses a different coordinate system than this default system, you can use the nn::mii::CharModelInfo::SetCoordinateUp() and nn::mii::CharModelInfo::SetCoordinateFront() functions to specify the coordinate system when rendering the model. You can also rotate the Mii face model in advance to match the coordinate system used by the application.

The values passed to SetCoordinateUp() and SetCoordinateFront() must be at right angles to each other. Initialization takes longer if you are not using the default coordinate system because the coordinates must be transformed.

Scaling

When the Mii face model is initialized, it is set to a scale value of 1.0.

If the application uses a different scale, you can use the nn::mii::CharModelInfo::SetScale() function to set the scale when it is time to render the model.
The Mii face model is created in a size that matches the scale used by the application.

Initialization takes longer if you are not using the default scale value because the scale must be transformed.

Initialization and Finalization

The following cautions are for initialization and finalization. You can also view the API reference for each function.

Initialization

Use the nn::mii::CharModel::Initialize() function to initialize the CharModel object.

Use the functions nn::mii::CharModel::CalculateMemorySize() and nn::mii::CharModel::CalculateMemoryAlignment() to determine the size and alignment of memory for initialization. The CharModelInfo object passed to these functions must be the same as that passed when initializing CharModel.

Two types of memory pools are passed when CharModel is initialized.

One is a memory pool for initializing the textures that are the rendering targets held by CharModel (the face outline textures and mask textures). This memory pool is for dynamic texture initialization, so it is called the dynamic memory pool.

Use the functions nn::mii::CharModel::CalculateDynamicMemoryPoolSize() and nn::mii::CharModel::CalculateDynamicMemoryPoolAlignment() to determine the size and alignment of this dynamic memory pool. The CharModelInfo object passed to these functions must be the same as that passed when initializing CharModel. As the properties of this dynamic memory pool, specify nn::gfx::MemoryPoolProperty_CpuCached and n::gfx::MemoryPoolProperty_Compressible.

The second memory pool is for initializing shapes and textures other than those that are the rendering targets held by CharModel. When the memory pool for these textures and shapes has been initialized, it will not change. For this reason, it is called the static memory pool.

Use the functions nn::mii::CharModel::CalculateStaticMemoryPoolSize() and nn::mii::CharModel::CalculateStaticMemoryPoolAlignment() to determine the size and alignment of this static memory pool. The nn::mii::Resource object passed to these functions must be the same as that passed when initializing CharModel. As the properties of this static memory pool, specify nn::gfx::MemoryPoolProperty_CpuCached. You do not need to specify nn::gfx::MemoryPoolProperty_Compressible.

Info

It is also possible for a CharModel object to be initialized with a single memory pool.

Use the functions nn::mii::CharModel::CalculateMemoryPoolSize() and nn::mii::CharModel::CalculateMemoryPoolAlignment() to determine the size and alignment of this single memory pool. The CharModelInfo and nn::mii::Resource objects passed to these functions must be the same as those passed when initializing CharModel. As the properties of the memory pool that is passed when CharModel is initialized, specify nn::gfx::MemoryPoolProperty_CpuCached and nn::gfx::MemoryPoolProperty_Compressible.

When the object is initialized with a single memory pool, the memory pool set by nn::gfx::MemoryPoolProperty_Compressible is larger in size. The memory pool set by nn::gfx::MemoryPoolProperty_Compressible also has larger overhead when CharModel is initialized, so we recommend passing the two types of memory pools for initialization of the object.

Depending on the setting that was configured when Resource was initialized, the resource data in the Resource object might be in a compressed format. If the resource data is compressed, the needed resources are expanded when CharModel is initialized. A temporary buffer is required for expanding the compressed resources. This temporary buffer is defined as the dedicated structure nn::mii::TemporaryBuffer.

The data for rendering the Mii face model is composed from nn::mii::CharInfo, CharModelInfo, and the Resource passed when CharModel was initialized. The initialized content cannot be changed until the object is finalized.

You must maintain the memory and memory pool used in initialization until you call termination processing. You can destroy the temporary buffer, nn::mii::TemporaryBuffer, after initialization.

Note

The size of the memory pool used for initialization and the alignment size are both relatively large in the Win/Nvn environment compared to other platforms (Win/Generic and the NX development hardware) because the implementations take platform-specific memory pool restrictions into account. The following functions are affected.

  • nn::mii::CharModel::CalculateStaticMemoryPoolSize()
  • nn::mii::CharModel::CalculateStaticMemoryPoolAlignment()
  • nn::mii::CharModel::CalculateMemoryPoolSize()
  • nn::mii::CharModel::CalculateMemoryPoolAlignment()

Be careful when running the same implementation on Win/Nvn and other platforms.

Finalization

To finalize the CharModel object, call the nn::mii::CharModel::Finalize() function. When the finalization process has competed, you can free the memory and the memory pool that was used for initialization.

Getting the TextureView Objects

You can get the nn::gfx::TextureView objects needed for drawing the Mii face model from the initialized CharModel object. The TextureView objects that you can get from CharModel are associated with enumerators of the nn::mii::CharModel::TextureType enumerated type. Use the nn::mii::CharModel::GetTextureView() function to get the pointer to the TextureView of the specified TextureType.

Depending on the parameters set in CharInfo, passed during initialization, there may be TextureType enumerators for which there is no TextureView object. If you pass such a TextureType as a parameter, nullptr is returned. For example, if you are rendering a Mii that has no eyeglasses, in this case, nullptr would be returned if TextureType_Glass was specified for the parameter. Ignore nullptr if it is returned by the GetTextureView() function.

If the CharModel object holds multiple mask textures, you can get the TextureView for a certain mask texture by specifying its mask index. You can get TextureView objects for as many masks as were set in the CharModelInfo that was used when CharModel was initialized.

Configuring DescriptorSlot

Set nn::gfx::TextureView, obtained from CharModel, to the descriptor pool, and set the obtained descriptor slot back to CharModel. Perform this by using the nn::mii::CharModel::SetTextureDescriptorSlot() function. Specify and set the same type as nn::mii::CharModel::TextureType for the obtained TextureView. If the type is nn::mii::CharModel::TextureType_Mask, specify this together with the mask index as a set.

Face Outline Textures

You can get the face outline texture from the initialized CharModel object. To get the face outline texture, use the nn::mii::CharModel::GetFacelineTexture() function. Use the Faceline class to render the obtained face outline texture.

Mask Textures

You can get as many mask textures that were specified at initialization time from the initialized CharModel object. To get the mask textures, use the nn::mii::CharModel::GetMaskTexture() function. Use the Mask class to render the obtained mask textures.

As the parameter of the GetMaskTexture() function, specify the index of the mask texture to get. If CharModel has mask textures for multiple facial expressions, get them by specifying the relevant mask indexes.

Getting DrawParam

You can get the DrawParam objects for drawing the Mii face model from the initialized CharModel object. The DrawParam objects that you can get from CharModel are associated with enumerators of the nn::mii::CharModel::DrawType enumerated type. Draw the Mii face model in the order of the enumerators.

Use the nn::mii::CharModel::GetDrawParam() function to get the pointer to the DrawParam of the specified DrawType. Depending on the parameters set in nn::mii::CharInfo passed during initialization, there may be DrawType-associated objects that do not need rendering. If you pass such a DrawType as a parameter, nullptr will be returned. For example, if you are rendering a Mii that has no beard, nullptr would be returned if DrawType_Beard was specified for the parameter. If the GetDrawParam() function returns nullptr, do not render that DrawType.

In addition to DrawType, you must also specify the Mii face model type and the nose type in the GetDrawParam() function. The values passed to the parameters for the Mii face model type and the nose type, must be types that were set in the CharModelInfo that was used when CharModel was initialized.

If CharModel has mask textures for multiple facial expressions, specify the relevant mask index when you have DrawType_Mask get the DrawType.

Bounding Box

You can get the bounding box for the Mii face model from the initialized CharModel object. Perform this by using the nn::mii::CharModel::GetBoundingBox() function.

Parts Transform

The parts transform stores information about where to position embellishments on the Mii. It is defined as the nn::mii::PartsTransform structure, and you can get it from the initialized CharModel object using the nn::mii::CharModel::GetHeadPartsTransform() function.
