Using Controls

Creating User-Specific Controls (Parts)

Describes the procedure for creating user-specific controls (parts).

Planning and Designing a User-Specific Control and Creating the Control Definitions File (Developer Task)

Talk with the developers about the specifications and get them to create a control definitions file (FLCT file).
The control definitions file is described in XML. (There is a sample file in /Samples/Sources/Resources/Ui2d/sampleNW_DefaultControl.flct.)

Creating a Layout

When you place the control definitions that have been created under the parts root and restart Tools, the user-specific controls are added as options in the Parts and Control Settings window.
The designer selects the added controls and proceeds to create a layout.

Creating a Custom Viewer (Developer Task)

The developer implements the actions of the user-specific controls.
The developer also creates a custom viewer that incorporates the user-specific controls.
A utility and a sample program are provided for easy implementation of a custom viewer.
With this custom viewer, the designer can easily check the actions of parts using previewing in the usual way.


Additional Information

Replacing Standard Controls

When you place the NW_DefaultControl.flct control definitions file below the parts root, LayoutEditor replaces the contents of the control settings so that you can create layouts using these user-specific controls.

Using Multiple Control Definition Files

You can simultaneously load and use multiple control definition files.
The files are sorted and loaded in alphabetical order by filename, so change the filenames to control the order in which they are loaded.
