Amount of Working Memory Required for the Texture Cache

Introduction

This page describes the nn::font::TextureCache::InitializeArg::workMemorySize and nn::font::TextureCache::InitializeArg::noPlotWorkMemorySize memory regions that are required for the scalable font texture cache.

nn::font::TextureCache::InitializeArg::workMemorySize

nn::font::TextureCache::InitializeArg::workMemorySize is the size of the memory used when working with glyphs that are actually plotted by a function such as nn::font::TextureCache::RegisterGlyph().

The amount of required memory increases as the number of plotted characters increases and as the font size increases.

nn::font::TextureCache::InitializeArg::noPlotWorkMemorySize

nn::font::TextureCache::InitializeArg::noPlotWorkMemorySize is the size of the memory used when retrieving glyph attributes, such as character width or kerning information, without actually plotting the glyphs.

The required memory size is fixed and does not change, even when the number of plotted characters or font size increases.
