Terminology

What Is a Layout?

A layout, then, is the resulting arrangement of panes (described in the next section) on the XY plane.

Panes

The basic element of Layout layouts is the pane. Panes consist of one or more rectangular polygons. A layout, then, is the resulting arrangement of panes on the XY plane.

Panes are organized in a hierarchical parent-child structure called the pane tree.

NW4F Layout maintains parameters such as position, scale, rotation, and transparency for each pane.
The values for these parameters are also inherited by child panes.
For example, moving a parent pane also moves any associated child panes.
This allows you to move and scale complicated pane objects consisting of several images at the same time.

There are six types of panes.

Picture Panes

Picture panes are for displaying simple image data.
In most cases, each picture pane has a single image. However, you can add or subtract multiple images, combining them in a single picture pane by using the multi-texture feature.

Text Box Panes

Text box panes are for displaying text.
You can display the text in almost any font by using the FontConverter tool to convert and use fonts installed on your PC.
Each character is rendered inside its own rectangular polygon, but these polygons automatically inherit the position and scale of the text box pane in which they reside.

Text box panes support a variety of features for customizing the appearance of text, such as centering or aligning text, italics, and shadows.

Window Panes

Window panes are similar to picture panes but different in that they consist of multiple images: one for the outline of the pane and one used inside the pane.
Changing the size of a window pane leaves the size of the outline images unchanged but scales the size of the inner image accordingly.
Window panes are well-suited when you want to be able to change the size of an image without the pane outline showing any scaling artifacts.

Null Pane

As the name suggests, null panes are panes that are not used to display anything.
Null panes have a variety of uses. For example, you can create a null pane and then create several child panes to the null pane so that you can move the child panes at the same time.

Boundary Panes

Boundary panes are similar to null panes in that they are not used for displaying anything. When selected or tapped, however, boundary panes display the selected region.

Parts Pane

Parts panes are a special type of pane that work slightly differently from all the other types of panes. Parts panes effectively function as sub-layouts. Using parts panes allows you to place a whole layout with all its constituent panes inside an entirely different layout.

For example, the parts pane in the preceding image is itself a layout comprising of a window pane, a text box pane, a null pane, and a boundary pane.

The parts in a parts pane can be configured to act as buttons using the controls feature.

Groups

Groups are used for grouping several panes together.
To create a group, select several panes that you want in the group and then right-click and select Create Group from the shortcut menu.
Groups are used primarily for specifying the panes you want to animate when configuring animations.

Animations

Each of the parameters maintained for panes can be animated.
To apply an animation to one of these parameters, use the Curve Editor window as shown in the lower-left corner of the screenshot.

The black points on the curve in the Curve Editor window are called keys. Keys represent the value that an animation parameter has at a particular time in the animation.
How you define units of time depends on how your game is implemented, but typically each unit of time corresponds to one game frame. In a game that runs at 60 fps, each unit of time is equal to 1/60 of a second, and similarly, in a game that runs at 30 fps, each unit of time is equal to 1/30 of a second.

The preceding image, for example, shows a key at time 0 where the X scale is set to 1.0 and a key at time 2 where the X scale is set to 1.1. This key configuration changes the scale of the N_NormalBtn_00 null pane from 1.0 to 1.1 over the period of time from time 0 to time 2.

In the sample layout, N_NormalBtn_00 is the name for the child pane of a null pane. The hierarchy places panes called W_Base_00 and T_TextShoot_00. Animating the scale of the parent pane also causes child panes to increase in size.

Animation Segment Tags

Animation curves that are applied to a pane can be segmented and named using animation segment tags.

In the preceding image, for example, the string OnOff is specified as an animation tag for the period of time from time 0 to time 2. It registers an animation tag named OnOff that corresponds to the animation that occurs during the corresponding period of time (increasing the scale from 1.0 to 1.1).

After an animation tag is created, you can use the Specify Related Groups feature to select other panes to apply that animation to them. Always make sure to specify the related groups. If you do not specify any related groups, the animation is applied to all panes in the layout.

In the application source code, you can specify an animation tag name using the CreateGroupAnimatorAuto member function in the Layout class to create an instance of the Animator class for managing and playing animations.
