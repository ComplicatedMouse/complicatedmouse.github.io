Fence

Fence

Overview

A fence provides a feature to synchronize the CPU and GPU in command buffer units.

This means that a fence can be set when executing a command buffer in a queue, and that a fence transitions to a signaled state after all commands in the command buffer have been executed.
The fence is not required to be submitted, and the command buffer can be executed without a fence. In this case, synchronization at the command buffer unit cannot be done, but all the commands that are requested to be executed can still be synchronized. For more information, see Queues.

Information

Fence information is defined as nn::gfx::FenceInfo.

Object

The fence object is defined as nn::gfx::Fence.

Initialization and Destruction

Initializing and freeing the fence follows the method for normal objects.

Synchronization by the Fence

To synchronize using a fence, first specify the target fence in the second argument of ExecuteCommand of the nn::gfx::Queue class to set the fence in the queue.

The fence starts in a non-signaled state at first. When in a non-signaled state, IsSignaled returns false.
The fence transitions to a signaled state when the GPU executes all commands set in the command buffer specified in the first argument of ExecuteCommand when the fence is set. When in a signaled state, IsSignaled returns true.

The CPU can also be made to stand by until the fence is in a signaled state, and this is performed by calling Sync. This stalls the CPU thread of the call originator until the fence is in a signaled state. If this is called without flushing the queue, note that the CPU may be in standby during the period when the fence cannot be in a signaled state.
When using standby, specify a timeout time that indicates the approximate maximum time to standby until the fence is in a signaled state. This time is an estimate and not one that guarantees precision to the least significant bit. If the fence is not in a signaled state when the timeout time expires, SyncResult_TimeoutExpired is returned and processing returns to the call originator. If the fence is in a signaled state before that, SyncResult_Success is returned at that time and processing returns to the call originator.
