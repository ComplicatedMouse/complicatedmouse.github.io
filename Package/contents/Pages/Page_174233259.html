Operating Environment Controls

Feature Overview

These controls are used to regulate the operations of the application itself and the SDK features called from the application.

Some operating environment controls are also determined by the application management data that is specified when the application is built. For more information, see Writing the NMETA File.

Using the Operating Environment Controls

To use the functions in the nn::oe namespace you must #include the oe library as follows.

#include <nn/oe.h>

Handling Notification Messages

The program can receive notification messages about changes in system states and program operating environment states.

Handle these messages as needed. For more information, see Notifications For Programs.

Allowing and Prohibiting the Taking of Screenshots

Users can briefly press the Capture Button to take screenshots of the output screen and save them to an album while the application or system features are running. For more information about operations when the Capture Button is briefly pressed, see Using the Capture Button. The program can use the following functions to allow and prohibit the capture of screenshots depending on the scene.

Function Description
nn::oe::EnableScreenShot() Allows the capturing of screenshots.
nn::oe::DisableScreenShot() Prohibits capturing screenshots.
If capturing is prohibited by this function, play video capture (described later) is also prohibited.
Info

To enable screenshot captures, you must permit them in advance, as listed in Writing NMETA Files.

Info

When changing between allowing and prohibiting screenshot capture, wait an extra two or three frames before switching the setting and actually changing the output screen image.

That is, if you are changing the setting to prohibit screenshots, first call nn::oe::DisableScreenShot() and then wait a few frames before generating a screen image that cannot be captured. And if you are changing the setting to enable screenshots, first output a screen image that can be captured and then wait a few frames before calling nn::oe::EnableScreenShot().

Permitting and Blocking Video Captures

While the application is running, the user can long press the Capture Button to save a video file of the gameplay up to that point in an album. For more information about operations when the Capture Button is pressed for a long time, see Using the Capture Button. (For more information about the saved movie file, see System Applications > Captures and Albums in the System Features Guide.)

Depending on the application scenario, you can use the following API functions to permit or block video capture.

Function Description
nn::oe::EnableRecording() Enables video capture. Permits the saving of video files.
nn::oe::DisableRecording() Disables video capture. Blocks the saving of video files.

However, when screenshot captures are blocked, as described previously, video capture is also blocked, and video files cannot be saved regardless of the preceding functions.

Info

To enable video captures, you must enable them in advance, as listed in Writing NMETA Files.

Info

When changing between allowing and prohibiting video capture, wait an extra two or three frames before switching the setting and actually changing the output screen image.

That is, if you are changing the setting to prohibit video capture, first call nn::oe::DisableRecording() and then wait a few frames before generating an image that cannot be video captured. And if you are changing the setting to enable video capture, first output an image that can be video captured and then wait a few frames before calling nn::oe::EnableRecording().

Screenshot Capture and Video Capture Settings

When you save screenshots and videos that have been captured with the Capture Button to an album, you can use the following function to embed information about the rotational orientation. For more information about rotational orientation, see the description for nn::album::ImageOrientation. The default is nn::album::ImageOrientation_None.

Function Description
nn::oe::SetAlbumImageOrientation() Specifies the rotational orientation of the screenshot image or video file.

Analyzing Screenshot Capture and Video Capture Behavior

Info

This feature is available in the SDK for NX Add-On 7.1 and later.

When the user saves screenshots by pressing the Capture Button for a short time, or saves videos by pressing the button for a long time, you can use a feature to notify the program of capture events. The purpose of this feature is to use the screenshots and videos that users collect in a game as a basis for analyzing their behavior. Note that you cannot use this feature to handle pressing the Capture Button.

This feature is disabled by default. You can use the following functions to enable and disable it.

Function Description
nn::oe::EnableAlbumImageTakenNotification() Enables the screenshot and video capture notification feature.
nn::oe::DisableAlbumImageTakenNotification() Disables the screenshot and video capture notification feature.

When you use the function to enable the notification feature, the program is notified with the following message when the Capture Button captures a screenshot or video.

Notification Message Description
nn::oe::MessageAlbumScreenShotTaken Notification message indicating that a screenshot was taken.
nn::oe::MessageAlbumRecordingSaved Notification message indicating that a constant video capture video was saved.

However, the messages sent by this function contain the following kinds of ambiguities, so it cannot be used for features that provide precise feedback to users. Do not use this function for purposes other than behavior analysis.

  • When several screenshots or videos are captured in a short period of time, only one notification may be sent.
  • Sometimes messages are not sent when the capture file cannot be created because there is not enough storage capacity, or if data is corrupted.

Integrating the Copyright Notice Image

A specific copyright notice can be integrated into screenshots and video files saved to an album with the Capture Button. Use the following functions to integrate the copyright notice. After setting and initializing the frame buffer with the nn::oe::InitializeCopyrightFrameBuffer() function, register the actual image with the nn::oe::SetCopyrightImage() function. You can then use the nn::oe::SetCopyrightVisibility() function as needed to change whether the copyright notice is integrated.

Function Description
nn::oe::InitializeCopyrightFrameBuffer() Sets the framebuffer for integrating the copyright notice. This must be called first.
nn::oe::SetCopyrightImage() Transfers the copyright notice to the framebuffer that was set up.
nn::oe::SetCopyrightVisibility() Specifies whether to integrate the copyright notice image.

Do not use the nn::oe::SetCopyrightImage() function to frequently update the copyright notice image. This image does not need to reset for each frame because its content is held in the framebuffer that is passed to the system by the nn::oe::InitializeCopyrightFrameBuffer() function. To change the copyright notice image, use the nn::oe::SetCopyrightVisibility(false) function once to temporarily stop integration. When one Vsync cycle has passed, change the image with nn::oe::SetCopyrightImage(), and then resume integration with nn::oe::SetCopyrightVisibility(true).

Warning

Do not use the copyright notice image integration feature for any other purpose.

The feature for adding a copyright notice adds particular information to image and video files captured in the screenshot. In the future, this information may serve as the basis to restrictions on some editing, so do not use this feature for any other purpose.


Controlling the Volume Balance

Some of the SDK features called from the program, are accompanied by a user interface. The screen and audio become switchable when certain functions are called, such as those for the software keyboard.

For calls to certain SDK features, the assumption is that audio output from the program itself and audio output from the SDK feature will be mixed. For these kinds of SDK features, the following functions can occasionally be used to specify the volume balance when audio sources are mixed. This volume balance value is the value expected by the calling program. The actual volume balance is determined by the called SDK feature. For more information, see the API reference and the manual for that SDK feature.

Function Description
nn::oe::SetExpectedVolumeBalance() Specifies the volume balance between the program itself and the called SDK feature.
nn::oe::GetExpectedVolumeBalance() Gets the volume balance between the program itself and the called SDK feature.

Specifying a Theme Color

As with the audio balance, you can use the following function to specify the theme color for the UI that accompanies an SDK feature called from the program. This specification is the value expected by the calling program. The actual UI color is determined by the called SDK feature. For more information, see the API reference and the manual for that SDK feature.

Function Description
nn::oe::SetExpectedThemeColor() Specifies the theme color for the called SDK feature.

Controlling the Auto-Sleep and the Screen Burn-in Reduction Features

The system detects when the user has remained inactive for a set period of time and activates the auto-sleep and screen burn-in reduction features.

You can use the following functions in your program to delay the activation of these features in some scenes. Examples of where the user is inactive. You may want to use these functions to include scenes where a video clip is playing or the closing credits are rolling.

Function Description
nn::oe::SetUserInactivityDetectionTimeExtended() Specifies whether to enable or disable the feature to delay auto-sleep and screen burn-in reduction.
nn::oe::IsUserInactivityDetectionTimeExtended() Determines whether this feature is currently enabled.

Getting the Startup Storage Device

During development, there may be times when you want to change where the various kinds of save data are stored, depending on which storage device the application is started from. With this objective in mind, the following API can be used to get the type of storage device from which the application was started. At the same time, it can also get storage information about a patch applied to the application.

Function Description
nn::oe::GetLaunchStorageInfoForDebug()

Gets the storage device where the application starts from and the storage device with the patch.


To use this API, you must explicitly #include the following.

#include <nn/oe/oe_DebugApis.h>

If a program built with some version of the SDK is started with newer firmware or development hardware, the program may start from an unknown storage device. In such cases, nn::oe::StorageTypeForDebug_Unknown will be returned. If you are using this API, we recommend considering this return value in your implementation.

Info

The various functions in Include/nn/oe/oe_DebugApis.h are for development purposes only and cannot be used in the production ROM.

Getting the Firmware Version

During development, you may want to leave the version information for the firmware on which the program was run. In this application only, you can use the following function to get the firmware version.

Function Description
nn::oe::GetFirmwareVersionForDebug()

Gets the firmware version.


To use this API, you must explicitly #include the following.

#include <nn/oe/oe_DebugApis.h>

In addition, this function aborts internally if it is called in a nondevelopment environment (like in a retail unit or PROD).

Info

The various functions in Include/nn/oe/oe_DebugApis.h are for development purposes only and cannot be used in the production ROM.

Getting the Cumulative Actual Operation Time of the Program

With NX Add-On 6.4 and later, you can get the cumulative actual operation time of the program since starting the program using the following function. Unlike the system tick that can be obtained with nn::os::GetSystemTick(), this time does not include the time during automatic suspension when the system goes to sleep or when there are changes in the focus state. However, it is reset to 0 when the program ends, and returns the actual operation time the next time it starts.

Function Description
nn::oe::GetProgramTotalActiveTime()

Gets the cumulative actual operation time of the program since starting the program, excluding time in states such as sleep mode.
The return value is returned in the nn::TimeSpan format.

Support for System Transition to Sleep State

The system can enter one of two sleep modes, default and pseudo, as described in Controlling the Sleep Mode in DevMenuCommand > Development Support Features. In default mode, the system enters the real sleep state. In pseudo mode, the system wakes automatically from sleep after approximately one second, which is useful for application development.
