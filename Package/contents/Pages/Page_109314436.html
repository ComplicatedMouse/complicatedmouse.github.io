Quick Start

Overview

The steps to build and run libraries and applications for NintendoSDK are the same as for normal Visual C++ projects.

The NintendoSDK provides a sample application that demonstrates the build and run methods.

Location of Sample Programs

Sample programs are in the NintendoSDK Samples package.

When the NintendoSDK Samples package is installed, the sample programs are placed in NintendoSDK\Samples\Sources\Applications.

Building Applications

The following description of the build procedure for a sample program uses the GfxPrimitiveRenderer sample as an example.

Procedure Operation
1

Open NintendoSDK\Samples\Sources\Applications\GfxPrimitiveRenderer\GfxPrimitiveRenderer-spec.NX.autogen.vs2017.sln.

Info

This procedure assumes that Visual Studio 2017 is being used.

If you are using Visual Studio 2015, open the GfxPrimitiveRenderer-spec.NX.autogen.vs2015.sln file that is in the same folder.
2

From the Visual Studio Solution Platform, select NX64.
3

Select Build > Build Solution.

Running Applications

The following description of the execution procedure for a sample program uses the GfxPrimitiveRenderer sample as an example.

Procedure Operation
1 Build a sample program following the procedure in the previous section.
2

Verify that NX64 is selected in the Visual Studio Solution Platform.
3

In Visual Studio, select Debug > Start Debugging or Debug > Start without Debugging.
4

When Target Manager (described later) starts and the sample is run, the following image is displayed on the SDEV LCD screen.

Note: Target Manager

Target Manager is software for managing the development tools connected to a PC.

You can use it to register an SDEV to a PC and to run built applications on an SDEV.


Target Manager starts automatically when you run an application from Visual Studio on the SDEV.

If you are operating the SDEV without using Visual Studio, manually start Target Manager from the Start menu.

Info

For more information about the features of the Target Manager, see NintendoSDK Documents > Features > Controlling the Development Hardware > Target Manager.
