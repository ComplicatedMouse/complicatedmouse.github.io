Overview

SystemUpdateEdev is a command-line tool for updating the EDEV development hardware's firmware with a process equivalent to the system update feature.

It offers an advantage compared to InitializeEdev, where you can update the firmware version while maintaining the settings.

This tool is added to NX Add-On 4.4.1 and later.

Package

Included in the DevKitVersionUpdater package.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/SystemUpdateEdev.exe

There are several dependent files in addition to this EXE file for operations.

Command Format

Option Format

SystemUpdateEdev [target options] [other options]

target options:
    --target <serial number | EDEV name>               Specify the serial number or name of the target to update.

other options:
    --target-version                                   Specifies the firmware type to use for system updates.
    --display-available-version                        Display all available firmware types.
    --verbose                                          Generate detailed logs.
    --version                                           Display to tool version.

Exit Code

You can use the exit code to check whether the tool succeeded.

Nonzero value: Terminated with error.

How to Use

The following examples demonstrate different ways of using SystemUpdateEdev.

Updating by Specifying the Development Hardware Serial Number

SystemUpdateEdev --target XAL03100000000

Specifying the Development Hardware Name and Updating

SystemUpdateEdev --target (target name)

Important Information

  • You cannot downgrade.
  • You can use this tool on development hardware running the following firmware versions. It does not work with earlier versions of the firmware.
    • EDEV: NX Add-On 4.0.0 and later
    • HDEV: NX Add-On 7.4.0 and later
    • ADEV: NX Add-On 11.1.0 and later
  • When specifying the development hardware serial number, specification is possible with or without the check digit.
  • NX Add-On 12.2.X and earlier versions of SystemUpdateEdev do not support updating DevMenuApp. Individually rewrite after executing the update tool.
  • In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after update is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.
    • For more information, see Using Target Manager 2.
