Introduction

Purpose of This Documentation

This documentation describes the nfp library provided with NintendoSDK and the amiibo access features available in the library.

Note

To use this library, you must set up your development environment correctly for using the NFC features.

Intended Audience

This documentation is intended for people who develop applications using the NintendoSDK nfp library.

Chapter Descriptions

For more information about using the features provided by the nfp library, see the Manual.

For more information about using the various tools provided by the nfp library, see the Tool Reference.

Related Documentation

Also see the API Reference and the Sample Backgrounder.

What can I do with the nfp library?

The nfp library currently supports the following features.

Feature Notes
NFC Tag Detection

The NFC device can detect amiibo products and IC-enabled commuter passes (Suica, PASMO, Kitaca, TOICA, manaca, ICOCA, SUGOCA, nimoca, Hayakaken).

This documentation refers to all detectable IC tags and cards as NFC tags.

Although the NFC device may also be able to detect other NFC tags, the nfp library is not guaranteed to work with such tags.
Reading and Writing NFP Tags The library can read from and write to NFP tags only.
Backing Up NFP Tags

The nfp library automatically backs up NFP tag data to the system (as save data in the amiibo Settings application).

If the data on an NFP tag ever becomes corrupted, the application can explicitly restore that data using the backup.

Only one tag can be used at a time. Simultaneous detection of two or more tags is not supported and may result in problematic application behavior.

Shared Region and Application-Specific Region

An NFP tag includes two data regions: a shared region and an application-specific region.

The shared region stores information about the amiibo itself, such as the character ID and registration information.
Any NFP-compatible application can read from the shared region, but only the amiibo Settings system application can write to the region.

The application-specific region can store up to 216 bytes of data. The application can decide what data to store. However, only one application can store data at a time.
Data in the application-specific region can only be read by an application with the same access ID as the application that created the region.
To overwrite the data in the application-specific region, prompt the user to delete the data using the amiibo Settings application and then create a new region.
