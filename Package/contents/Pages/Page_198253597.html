Overview

NfotWriter is a tool for writing NTF files to tags to create Nintendo Original Format Tags (NOFTs).

Location

This tool is included with NintendoSDK NX Add-On and is found in the following location.

TargetTools/NX-NXFP2-a64/NoftWriter/Release/NoftWriter.nsp

How to Use

Preparations

Write the NTF file in the amiibo list package to anywhere on the SD card. Characters other than ASCII cannot be used for the filename or folder name.

The amiibo data list package is included in the CSV format data list file, amiibo_DataList_yyyymmdd.csv. The "yyyymmdd" part of the filename is the date indicating the package version. By saving this CSV format data list file somewhere on the SD card, the US_English name defined for the character in the NTF file will be displayed as well. There are no problems with NoftWriter behavior even if there is no data list file.

Starting the Tool

Start NoftWriter with the SD card containing the NTF file inserted into the NX.

$ Tools/CommandLineTools/RunOnTarget.exe TargetTools/NX-NXFP2-a64/NoftWriter/Release/NoftWriter.nsp

You can also install the NSP file on the development hardware and use the tool. For more information about installing the NSP file, see Installing and Running an NX Application on the Development Hardware.

After it is started, an image like the following is displayed on the NX screen. This shows all of the NTF files saved in the SD card.

When using a development NTF file created by NTFMaker, it is possible to add the character names and file descriptions to the CSV format data restore file.

NVL-AAAA_00_00.ntf,Mario
:
NVL-AACC_00_00.ntf,PAC-MAN
NVL-AACB_00_00.ntf,Mega Man
NVL-DUM1_00_00.ntf, NTF File for Debug 1
NVL-DUM4_00_00.ntf, NTF File for Debug 4

The first row shows the NTF filename, and the second row describes the string to display. Only ASCII characters can be used.

Write

Select the NTF file to write. Move the focus up and down using the directional buttons and press the A Button to select the item. A screen like the following will be displayed on the NX screen after selecting the NTF file.

This tool can be used to write different NTF files later to NOFT created by Write(except lock bits). A different NTF file cannot be written later to NOFT created using Write. In addition, while it is possible to use Write to overwrite an NOFT created using Write(except lock bits) with a different NTF file, this will fail after the write process even if you use Write(except lock bits).

Tag to Write Write Method Whether Writing Is Possible
Tags that have not yet been written to Write(except lock bits) Writes allowed
Tags that have not yet been written to Write Can be written, but may not be written again afterwards
Tags written with Write(except lock bits) Write(except lock bits) Writes allowed
Tags written with Write(except lock bits) Write Can be written, but may not be written again afterwards
Tags written with Write Write(except lock bits) Write-protected
Tags written with Write Write Write-protected

Select Write(except lock bits) or Write and press the A Button. The waiting for write screen will be displayed. Place a writable tag onto the NFC device.

Info

If multiple controllers are connected, place the tag on the controller used to operate the tool.

The tool will transition to the writing screen when a tag is detected. Do not move the tag away during the write process.

Note

Pay particular attention to not removing the tag when writing to a tag that has not yet had any content written. NoftWriter writes to the important areas of tags to which data is being written to for the first time. If this write fails, the tag may get into a state where it is not any of the following: "tag that has not yet been written to", "tag written with Write(except lock bits)", or "tag written with Write".

Displaying Results

"Finished!!" will be displayed on the NX screen as indicated below if writing to the tag completes normally.

"Write failed..." will be displayed on the NX screen as indicated below if writing to the tag fails.

Note:

If incorrect data is written, the tag could be rendered unusable.
