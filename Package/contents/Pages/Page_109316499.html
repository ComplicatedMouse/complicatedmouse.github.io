Overview

InitializeSdevWin is a GUI tool to initialize the firmware of the SDEV development hardware.

The command-line version of this tool is InitializeSdev. If you are using automatic tests, use those.

Packages

InitializeSdevWin is included in the SystemUpdater for NX package.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/InitializeSdevWin.exe

There are several dependent files in addition to this EXE file for operations.

Important Information

When you initialize SDEV with this tool, all applications and save data installed by Installing and Running an NX Application on the Development Hardware are deleted.

Option Format

There are no command-line options that must be specified in this executable file.

A window is displayed when you run EXE files without arguments.

Display Screen

Main Window

This screen is displayed when you start InitializeSdevWin.

To use it, specify the development hardware to initialize (1), specify the firmware version (2), and click the Start Initialization button (3).
When you click the Start Initialization button, a log window is displayed and initialization starts.

If the development hardware you want to initialize is not displayed on the screen in 1, click the Re-Scan button to search for it again.
If you select Tools > Change network settings from the menu, the network connection is displayed and you can change the NIC settings.

Shortcut Menu

In the list of connected SDEV hardware, right-click a line to bring up the shortcut menu.

  • Start initialization
    • Select this option to start initializing the specified SDEV. The same operation is performed by clicking the Start initialization button at the bottom of the window.
  • Blink LEDs
    • Select this option to make the LED lights on the back of the SDEV console blink. Use this feature when you cannot specify a particular console because the serial numbers are not displayed.
  • Write serial number to SDEV
    • Select this option to write a serial number to a specified SDEV.

Log Window

This screen is displayed when you perform initialization.

The progress is displayed in the progress bar, and the log is displayed in the log window.
When initialization is finished, the result is displayed in Result in the lower left of the screen, and the Exit button in the lower right of the screen becomes enabled.

Descriptions of Other Features

The following features becomes available after you have selected the type of firmware from the combo box.

Initializing Using Firmware Approximating the Production Hardware

To initialize the SDEV using firmware approximating the production hardware, select the item that includes the "+ Prod" firmware version selection combo box.

To initialize the SDEV using firmware approximating the production hardware with functionality for exporting logs from a microSD memory card, select the item that includes the "+ ProdWithLog" suffix.

Note

The Initialize using firmware approximating the production hardware feature is only available with the SDEV MP. Note that an error results if it is executed on other SDEV systems.

Important Information

  • In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after initialization is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.
    • For more information, see Using Target Manager 2.
