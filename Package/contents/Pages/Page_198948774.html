Separately Outputting the g3d Binary

This section describes the process for separately outputting the g3d binary in the effect binary, and binding it with the effect binary in the runtime.

Separately Outputting the g3d Binary

To output the g3d binary separately, specify the g3d binary filename (*.bfres) in the binary conversion command-line argument --output-g3d-primitive-binary.
The g3d binary file will be output as a BFRES file, and can also be used as a separate g3d binary.

EffectConverter.exe --output effectBinary.ptcl --convert-list ConvertList.xml --output-g3d_primitive-binary effectG3dPrimitive.bfres ...

Binding the Separately Output g3d Binary to the Effect Binary

The g3d binary that has been output separately must be bound to the effect binary in the runtime.

// Initialize effect texture binary.
nn::g3d::ResFile* pResFile = nn::gfx::ResTextureFile::ResCast( textureBinary );
pResTextureFile->Setup( pGfxDevice );

// Bind the registered effect resource to the g3d binary.
pVfxSystem->GetResource(resourceId)->BindExternalG3dResFile( pResTextureFile );

For more information, see the VfxBinarySegmentation sample.
