Overview

Do not record the following information in the user save data or use it as the user save data filename unless you have a specific reason to do so.
Use this information according to the following rules only if it needs to be used in the application design plans.

  • Pseudo device ID (PseudoDeviceId)
  • User identifier (nn::account::Uid)

Pseudo Device ID (PseudoDeviceId)

Note the following when using a pseudo device ID (PseudoDeviceId) that can be obtained with the nn::oe::GetPseudoDeviceId() function in the user save data.

Do not use this information unless it is required in your application plan.
If you use this information, verify that there are no problems when user save data is transferred to another device with User Account Migration or Copy Save Data.

When using the User Account Migration or Copy Save Data features, the save data will be copied or transferred to a device with a different PseudoDeviceId from the environment where it was created.
As a result, an unexpected PseudoDeviceId may be detected when the application references the save data.
Applications must be able to operate normally under these conditions.

Check That There Are No Problems With Transfering Save Data to a Device With a Different PseudoDeviceId

Use the following method to check whether the application works normally after transferring save data to a device with a different PseudoDeviceId.

  1. Prepare a console that can start the DevMenu and already has a user account.

  2. Start the application, accumulate save data for the user account, and exit the application.
  3. Start DevMenu and go to the Debug screen.
    1. Click the Reset button for the Reset Pseudo Device ID.
  4. Start the application and verify that the save data created in step 2 works in the application without any problems.


Note

This testing method only works for consoles that were initialized in DEV mode.

User Identifier (nn::account::Uid)

Note the following when using the user identifier (nn::account::Uid) in user save data.
(This information applies regardless of whether the UID is from the user account that owns the user save data or that of a separate user account.)

Do not use this information unless it is required in your application plan.
If you use this information, verify that there are no problems when the user save data owner is changed to another user account with Copy Save Data.

When using Copy User Save Data, the user save data is copied across user accounts with different nn::account::Uid.
For this reason, the copied user save data must work correctly on both of the user accounts.

Info

If the application specifications require the user save data to be highly dependent on nn::account::Uid, contact Nintendo if there are unavoidable problems when changing the owner to a different user account.

Check that There Are No Problems With Copying Save Data Between User Accounts With Different nn::account::Uid

Use the following method to check whether user save data copied to a different user account works correctly in the application.

  1. Prepare a console that can start the DevMenu and that contains two accounts (A and B).

  2. Start the application, accumulate save data for user account A, and exit the application.
  3. Start DevMenu and go to the SaveData screen.
    1. Select the save data for user account A and export it to the microSD card.
    2. From the drop-down list at the top of the screen, select Backup in SD card.
    3. Select the user save data that was exported in step 3-a. Select user account B as the import target.
  4. Delete user account A.
  5. Start the application and verify that the save data for user account B works in the application without any problems.
