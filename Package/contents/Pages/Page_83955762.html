Creating and Adding Players

To create or add a player, select the list region in the Sound Set tab and then select Project > Add Item > Player.

You can create or add a new player by using the shortcut menu that opens when you right-click the player list.

Editing a Player

Players are edited by using the player list.

Elements of a Player List

This section describes the meaning of each of the elements that defines a player list.

Name

Specifies the name of the player.

The characters that can be used in a name are restricted. The following characters can be used.

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, _] for subsequent characters

Sound Limit

Specifies the number of sounds that can be played simultaneously by the player.

If the player receives a request for a sound that exceeds the maximum number of playable sounds specified by this value, the sounds played are selected based on the player priority of each sound. Sounds left out because of this selection stop regardless of whether they are actually voiced.
Note that it is the priority of virtual voices, not physical voices, that matters in this specification.

This value specifies the maximum number of sounds, such as a sequence sound or a wave sound.
Although multiple MIDI channels and voices exist for each sound, priority settings for these separate voices are carried out based on channel priority. Channel priority is applied at the voice level at the point where each sound is about to be played as a physical voice.

Heap Size

Specifies the size of the player heap.

For more information about the player heap, see About the Player Heap.
