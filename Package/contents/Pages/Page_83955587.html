Shape Animation - 3ds Max

Shape Animation

Shape animations are set by applying a morpher modifier and animating a weight value.
Normal animations can appear differently on 3ds Max and in NintendoWare.

Vertex color animations can only be checked on the target platform.

Configure shape animations by applying a morpher modifier.

The number of vertices and faces of the target object must be the same as the base object.

Info

Do not perform any operations that may change the number of vertices or faces of the base object or target object after applying a morpher.

Info

Sometimes shape animations for which the face configuration differs between the base object and target object are not displayed correctly.
For example, if the target object is converted into an editable polygon. Be sure to convert the base object to an editable polygon as well, while making absolutely sure that conditions match.

You can change the position of vertices after applying a morpher.

When there are changes to the vertex position or normal/vertex color of the target object after applying the morpher, the most recent updated target object vertex information is output at export time even if Automatically Reload Target has not been selected for the base object.
In such cases, the display appears differently than on 3ds Max.

Use the plug-in for setting shape animations to set the vertex attributes that will change when playing the shape animation.

Vertex coordinates, normal vectors, tangent vectors, and binormal vectors are updated by default. (Tangent and binormal vectors are only updated when using normal mapping or an anisotropic shader.)

Info

The more vertex attributes to update, the greater the amount of data and the greater the processing load on the target platform.

You cannot export key vertex attribute data that does not change between the base shape and target shape for materials in the same polygon group.
You cannot export a shape animation of a polygon group if none of the vertex attributes change.

If the vertex attributes of an object vary over only a narrow range, the amount of data and the processing load on NintendoWare can be reduced by changing the material IDs for that range of the surface and making it a multi-material.

Vertex attribute data for the target shape is output to the model file.
Output a model file again if the number of target objects, shape, or vertex color changes.

Info

Animation curve data is exported to the shape animation file only if a key is set or a weight value other than 0 is used.

When using multiple vertex colors, you can only animate color sets that exist for all of the base and target shapes.

Base shape settings are used for the type and number of components used with a color set.

Assume, for example, that the color sets indicated with a circle (○) in the table below all exist.

Vertex color channel Base Shape Target Shape A Target Shape B
nw4f_color0 ○ ○ ○
nw4f_color1 ○ × ○
nw4f_color2 ○ ○ ○
nw4f_color3 × × ○

In this example, color0 and color2 will be animated.

color1 is exported as a fixed value and is not animated. color3 is not exported at all because it is not included in the base shape.
