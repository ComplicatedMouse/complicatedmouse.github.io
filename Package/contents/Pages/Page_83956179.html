Overview

Option Setting Types

There are two types of SoundMaker option settings available: tool settings (called "options") and project settings.

Option specifications are made in the Tools > Options dialog box.
Options are used to specify operation settings for the overall tool.
Because this setting is saved in the initial settings file, it is always enabled under SoundMaker as long as it runs on the same computer.

Specify the Project settings using the Project Settings dialog box, opened from Project > Project Settings.
Project settings are used to specify operation settings at the sound project level.
Because this setting is saved in the sound project file, it is enabled even when the same sound project file is opened by using a version of SoundMaker that is running on another computer.

Options Dialog Box

The Options dialog box is used to specify general operation settings for SoundMaker.

Opening the Options Dialog Box

Select Tools > Options.

General

  • In the Project area, you can configure whether to open the last project that was open when you start SoundMaker.

If this option is selected, the project that was opened when the application was last exited is loaded automatically when it starts the next time.
You can also select whether to show the full path for the project name in the title bar.

  • The List area enables you to set full paths for reference files, whether to display or hide line numbers, and whether to display or hide the bar for numerical value cells that that have upper or lower limits.
    If you select the Set Group item comment to Reference item check box, comments about the original reference item appear in the comments field of items registered to a group.
    If you select the Paste item below selected item check box, when you paste items they are inserted below the selected item.
  • The Convert area enables you to make settings for the convert dialog box and parallel conversions.

You can configure whether the conversion dialog box closes when an error occurs by selecting or clearing the Do not close conversion dialog when an error occurs check box.

Conversion operations for sounds without any mutual dependencies can be run in parallel by using the value set for Maximum number of parallel conversions as the upper limit value.
The maximum number of processes can be set to any value in the range from 0 through 99.
When a zero is set, it is interpreted as CPU core count.

  • The Other area is used for other functions, such as setting the type of file selection dialog box and setting the state when opening a sound project.

If the Use select waveform dialog with preview feature check box is selected, when the waveform file is selected, the select file preview feature is added to the lower part of the file select dialog box.

If the Remember whether files are open or closed check box is selected when you open a sound project, the Sound Project tab and Bank tab that were open the last time the project ended both open in a maintained state. Opening a sound project may take a lot of time because so many tabs are open.

External Editor

The editor used to open the file from Open Reference Rile can be specified here.

  • Specify the path to the editor in the Command area.
  • Specify the arguments to pass to the editor in the Arguments area.
    • If $(file) is specified, it will be replaced by the reference file path.
    • For a text sequence file, $(line) can be specified, which will be replaced by the line number.

For example, to specify Notepad as an external editor, specify the following.

  • [Command] C:\Windows\System32\notepad.exe
  • [Arguments] $(file)

For a text sequence file, instead of specifying an external editor, Open with Notepad can also be selected.

If the Command area is empty, the application associated with the reference file in the operating system will be used.

  • In this case, the Arguments area is not used.

Keyboard Settings

Functions can be set to some of the keys on the PC keyboard.

  • In the List area, you can set what operations to perform when the ENTER key is pressed.

If you set both Move to cell after pressing Enter key and Press Enter key to start editing cell, pressing the ENTER key under normal circumstances enables editing of the selected cell. However, if the cell is already editable, pressing the ENTER key applies the edits to the current cell and moves to the next cell.
If the cell happens to be a check box in the list, the focus moves to the next cell without changing the value in the first cell.

  • To set the SPACEBAR or F9 key to trigger preview playback, use the Sound List area.

Select Play to play from the start every time the SPACEBAR is pressed, or select Play/Stop to switch between playing and stopping with each press of the key.

List Items

You can rename or delete items that have been added to the list.

The names of items added in Register List Item Settings are displayed by list type.
You can rename or delete an item in the list that is selected.

In-Game Editing

  • When the If the parameters are edited, make them editable in-game check box is selected, when parameters listed in the sound set tab are changed, they are automatically subject to in-game editing.
  • When the When making a sequence sound editable in-game, make the bank editable as well check box is selected, all of the banks accessed by sequence sounds become subject to in-game editing when the sequence sounds do, which saves you the trouble of making banks subject to in-game editing individually.

Audio

Used to select the audio output device to use for PC emulation output.

Changes to the output device are not immediately applied.
To apply the changes, restart SoundMaker.

MIDI

Used to select the MIDI keyboard and MIDI sequencer SoundMaker to use for signal input.

  • MIDI signals that are input from the device set in the MIDI Keyboard Input Device field are played back using the active bank in the Bank tab.

Only one keyboard can be selected.

  • MIDI signals that are input from the device set in the MIDI Sequencer Input Devices field are played back using the banks in the Preview Bank window.

Multiple sequencers can be selected.

  • MIDI devices configured as MIDI Keyboard Input appear dimmed and are disabled in the MIDI Sequencer Input list.

If you want to enable a device in the MIDI Sequencer Input Devices box, do not select the same device in the MIDI Keyboard Input Device box.

Statistics

Loudness

Select Measure Average Loudness Automatically in Background to automatically measure the average loudness in the background.

Convert1

Used to set attributes for conversion. (The settings are divided into two tabs because there are a large number of them.)

Exporting

Select Sound Archive Output Location to specify the output location of sound archive files.
If this location is left blank, it is output to the same location as the sound project.
If you specify the directory using a relative path, the directory that has the project file becomes the origin.

As mentioned in the Importing Other Projects section, when project files from NintendoWare for Revolution (RSPJ files) and NintendoWare for CTR (CSPJ files) are imported and the setting for the output location is left blank, "output" is attached automatically.

Select Stream File Output Location to specify a relative path from the Sound Archive Output Location to the directory where the stream file is output when it is converted.
If you specify a nonexistent directory, the output location is created automatically at conversion time.

In User-Managed File Output Location, set where to output group files (BFGRP files) when the output method is User Managed.
Specifies the relative path directory from the sound archive output location.

When the check box for Output Labels by Items is selected, in addition to exporting an FSID file that is the collection of all item names (IDs) in the sound project, the item ID headers are also divided into files by type and output.

If the check box for Specify use of hardware in OPUS decoding in the target is selected, the hardware accelerator will be uniformly used for Opus decoding registered in SoundMaker.

For more information about file export, see Convert - Generated Files.

Cache

Intermediate binary files are exported to the folder that is specified in the Location of Cache Output text box.
However, the instrument list FINL file of the bank is exported to the same location as the source file.

Warnings
  • Warn if there are any unused items.
  • Warn when a group item reference is invalid.

Select these check boxes if you want these warnings to be shown in the Conversion Log window at conversion time.

Convert2

Used to set attributes for conversion. (The settings are divided into two tabs because there are a large number of them.)

Resampling (Downsampling, Downmixing)

Specifies the SoX needed for the downsampling and downmixing feature.

  • Use the SoX bundled with SoundMaker

Normally, select this option.

  • Specify the SoX location

If you want to use a SoX other than the bundled SoX, specify the path to sox.exe.
Specify a relative path or an absolute path from the project.

Sequence Sounds

To use the quarter-note resolution when converting the standard MIDI file format (SMF), select SMF quarter note resolution.

Convert Event

Command before Conversion and Command after Conversion are described in the Other Features section of Automatic Processing Before and After Conversion.

Sound Archive Settings 1

Used to set attributes for sound archive files that are created during the conversion process.

To disable export of the string table in your sound archives, clear the Output the string table check box.
This allows you to reduce the size of sound archives when sounds are not called using strings.

The numeric values set in the Sound Archive Player area of the window represent the maximum number of sounds and channels that the atk library prepares when the created sound archive is played.

You can economize on the memory consumed by the atk library by setting the number of unused sounds to 0, and by using only the minimum required number of channels.
However, be careful when changing numeric values for settings such as Number of stream sounds, because it is impossible to play more than that number of stream sounds simultaneously.

Specify a multiple of the stream buffer size (1 to 4) for Stream Buffer Size.
The total size of the stream buffer corresponding to the specified multiple is displayed in the dialog box, along with estimates of the number of seconds of audio data that can be played at 32 KHz for the ADPCM and PCM16 compression formats.

Supplemental Information for the Sound Archive Player Area

The numeric values set in the Sound Archive Player area relate to the numbers of instances held internally by the nn::atk::SoundArchivePlayer that is bound to the relevant sound archive (the BFSAR file).

When these numbers are large, the SoundArchivePlayer can hold that many instances, but the functions SoundArchivePlayer::GetRequiredMemSize and SoundArchivePlayer::GetRequiredStreamBufferSize require more memory.

Consider this example:

    • Sequence sounds
    • Sequence tracks
    • Stream sounds
    • Wave sounds

Also, when there is a large number of stream channels, the GetRequiredStreamBufferSize function requires more memory.

The instances stored in SoundArchivePlayer are used when the system plays stream sounds, sequence sounds, and wave sounds. Only the numbers of sounds determined here can be played.
If you try to play more sounds, the following warning message displays and the lowest-priority sound stops.

Info

atk_SoundInstanceManager.h:144 Warning:Sound (id:0xXXXXXXXX) is stopped for not enough seq sound instance.

Consider the following conditions.

    • Only one stream sound is played at a time.
    • Two channels and one track of stream sounds, and two channels and three tracks of stream sounds, may be played back.

With these conditions, the required minimum values are as follows.

    • Number of stream sounds: 1. Number of stream channels: 6

Setting higher values is not a problem, but it consumes unnecessary memory.


You can compare these values to the Sound Limit of the player to determine the maximum playback number for each sound type (stream sounds, sequence sounds, and wave sounds).

To repeat, these values affect the required amount of memory, so you can control memory size by considering how many of each type of sound can be expected to be played at the same time in the application, and setting the values as low as possible.

Supplemental Information for the Sequence Sounds Area

This section provides details about the rounding direction set for sequence sound ticks.

If the timing of a certain sequence sound tick falls between audio frames (which delimit the minimum intervals of time in which the atk library can control volume and sound production), this setting defines whether to process the sequence commands included in that tick in the former audio frame or the latter audio frame.
The relationship between audio frames and ticks is illustrated as follows.


If rounding-up has been selected, the sequence commands are processed in audio frames with a timing that is slower than indicated by tick, as shown below.
Rounding-up is set by default when a new project file (FSPJ file) is created.
When playback is cued from the sample position as calculated from tick, the end of a sound that should have finished being played might possibly be included in the start of the part that has been cued.


If rounding-down has been selected, the sequence commands are processed in audio frames with a timing that is faster than indicated by tick, as shown below.
The behavior is compatible with the behavior when converted with SoundMaker version 1.46.0.0 or earlier (NintendoSDK 13.0.0 or earlier).
Update the earlier version of the project file, or import the project file using the feature described in Importing External Files with rounding-down selected as the default.
When playback is cued from the sample position as calculated from tick, a sound that should play might possibly be treated as already played.

Sound Archive Settings 2

Configure the settings for creating sound archives.

Specify the following settings.

    • Do Not Output
    • Sound Archive (Main Sound Archive)
    • Add-On Sound Archives

for each sound set.

This configuration is equivalent to right-clicking a sound set in the project tree.
For more information about add-on sound archives, see Guide > Add-On Sound Archive Feature.


For more information about handling add-on sound archive items, see Guide > Add-On Sound Archives.

HTML Sound List

Configures the HTML sound list.

By configuring an HTML sound list, you can automatically export an HTML-formatted sound list when sound data is converted.

A summary is output to the top of the HTML file, and lists of enabled items can be created. You can now list sound labels and comments in sortable tables.

Note: Instrument lists can also be exported as HTML-formatted files, but not while converting.
For more information about instrument lists, see Banks - HTML Instrument List Output.

The Create New Sound List Output Settings dialog box appears when you click the Add button. Use this dialog box to enter information such as the output settings name.

If you set more than one output settings name, the Basic Settings and Output items tabs appear.

Basic Settings

You can configure the following sound list output settings with the check boxes in this tab.

  • Output File Path
    A relative path from the directory where the project file is located.
    This path can also be an absolute path.
  • Output a separate list for each sound set
    Select whether to enable projects with multiple sound sets to export one list for each sound set or one list for all sound sets.
  • Do not output disabled items
    Select whether to output temporarily disabled items to the lists.

Output Items

Select the items to write to the sound list.
Items are written to the sound list in the order that they are listed.
To rearrange items, select one or more items, and then click Move up or Move down.

Item Names

You can configure the default settings for the item name when a new item is created.

Add Prefix

Select Add Prefix to set an arbitrary prefix for use with the default names for a new item.
This prefix cannot be set for sound sets, however.
If the prefix string is left blank, no prefix is added to the item name.

Create From Filename

The original filename is used as the default name for the item if you click the Create From Filename button in the Create New dialog box or drag a file to create the item.

Select the Delete everything after first delimiter check box to delete all characters in the filename from the first delimiter found.

This area also has selections for how to treat uppercase and lowercase letters, and how to treat characters that cannot be used for item names.

Create From Copy and Paste

Use this item to specify the suffix to attach to item names when they are created using the paste operation.
If the suffix string is left blank, only a number is added to the item name.

Comments

Sets the names for the Comments column that are shown in the lists in the Sound Set tab and the Bank tab.

In the default state, only one comment is displayed on the list.
You can enable display of multiple comments from the Customize Displayed Columns option in the shortcut menu that appears when you right-click the list header.

User Parameters

Configures the number of user parameters that can be set for use with each sound, and each bit structure.

Up to four user parameters can be set.

All user parameters can be enabled or disabled by selecting or clearing the check box in each tab.
Disabled user parameters are not included in the sound archive, so the size of the binary sound archive file (BFSAR file) does not increase.

To display the dialog box to configure the bit structure, click the Add button.

In the dialog box opened when setting user parameters from the sound list, the specified Name is displayed as an item and the Selection value is displayed as a drop-down menu.

Also, when the user parameters are defined, they appear as individual items in the list, so you can change the values for each name from the drop-down menu for that cell.

An equal sign (=) cannot be included in a name.

User Commands

Configures user commands to run from the menu or the toolbar.

The Command Name is the string that is displayed as the tool tip for the menu item or toolbar selection, and the Command is the actual string that is executed.
To set the icon for the menu item or toolbar button, specify an image file in Icon File Path.
The file path specified must be either a relative path from the project file (FSPJ file) or an absolute path.

For more information about user commands, see the description in the separate section (Other Features) for User Commands.

Color

Sets comments about item colors.

You can change the strings that display in context menus and in submenus of the main menu when colors are set for list items.
When you set comments, the default strings such as Red and Green are replaced with the comment strings.

Project Comments

You can write comments for individual project files.

Comments can be useful for confirming and sharing project data.

File Events

You can execute your own user-defined commands before and after saving sound project (FSPJ) files, sound set (FSST) files, bank (FBNK) files, and text sequence (FSEQ) files.

In Specified command path before file is saved and Specified command path after file is saved, specify the path of the file containing the commands you want to run immediately before or after saving a file.
When the specified command file is executed, specify the path of a text file as a parameter.
This text file contains a list of absolute paths to the files on which to execute the commands in your command file (one per line). (Note: This text file is automatically generated and temporary.)

You can also temporarily disable the pre- and post-save commands by clearing the appropriate check boxes.

Statistics

Loudness

If Warn if the maximum average loudness value is exceeded is enabled and the Maximum average loudness value is set, a warning icon will be displayed on the list when the average loudness value exceeds this value.

Maximum measurement time specifies the maximum duration to measure the average loudness of sequence sounds.
