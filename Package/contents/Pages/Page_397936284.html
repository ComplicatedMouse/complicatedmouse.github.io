For example, if you created the file ExportTexture_LockAll.jsxinc and configured it as follows, it would look something like the following figure.

var lock_output_folder = true; // The output folder for the FTX file.
var default_output_folder = "test"; // Fixed value for the FTX file output folder.

var lock_output_ftx = true; // Output FTX
var default_output_ftx = true; // Fixed value for Output ftx.

var lock_output_tga = true; // Output TGA
var default_output_tga = false; // Fixed value for Output TGA.

Added a feature to dump the size information for the binary resource file to the 3D binary converter.

When using --dump-binary-info to specify a directory, an XML file named <ResFile_filename>_size_info.xml, that contains the size information, is exported to that directory.
You can use the feature as follows.

// Convert model.fmdb to model.bfres and generate the size information of model.bfres in the result directory.
>3dBinaryConverter.exe --output model.bfres model.fmdb --dump-binary-info="result"

// Note: The binary resource file (BFRES) can be input into the converter only when the --dump-binary-info option is specified.
// The size information for model.bfres is exported to the result directory.

Added support for packing model vertex attributes.

Added support for packing multiple model vertex attributes into a single attribute. For example, depending on the platform, handling two vec2 vertex attributes together as a single vec4 attribute may increase performance.
By performing model vertex attribute packing, you can, for example, convert two vec2 vertex stream data into a single vec4 vertex stream data.

The simplified procedure for performing packing.

  1. Set the component with a layout declaration for the vertex attribute variables that you want to pack from the shader source.

    // Declare packing of the vertex attributes assigned to _u0 and _u1 into vec4.
// Set different components for the same location.
// Both of the following examples are for vec2, so the components are 0 and 2. But, for example, when packing float+vec2+float, set components to 0, 1, 3.
// In the runtime, vertex attributes are only assigned to only vertex attributes variables with component=0.
layout(location = 8, component = 0) in vec2 i_UV0; // @@ id="_u0"
layout(location = 8, component = 2) in vec2 i_UV1; // @@ id="_u1"
  2. In the 3D shader converter, convert the shader settings file (FSC) into a shader definitions file (FSD).
  3. In the 3D Vertex Attribute Packer, enter the shader definition file (FSD) and model intermediate file (FMD), then pack the model vertex attributes. At the same time, delete the assignment information for vertex attributes other than those with component=0.

Runtime Libraries


Changed the characteristics of memory pools used in the models in the binary resource file (BFRES).

Changed the CPU characteristics of the memory pool held by nn::g3d::ResFile from nn::gfx::MemoryPoolProperty_CpuUncached to nn::gfx::MemoryPoolProperty_CpuCached.
Note that this change affects the following points.

  • If the CPU references the vertex buffer and index buffer values, access will be faster compared to previous versions.
  • If the vertex buffer and index buffer in the resources are updated from the CPU or GPU, it may not be possible to read or write the correct values.

Added support for the shader image variables (Image Load Store).

Added support for shader image variables in G3D. Added definitions for the shader annotations, intermediate files, and runtime.

  • Annotations
    Added annotation specifying ID for image variables.
  • Classes
    Added nn::g3d::ResImageVar to the nn::g3d::ResShadingModel class member variable.
    Use the nn::g3d::ResShaderProgram::GetImageLocation() function to get the image variable location.
  • Shader definitions files
    Added <image_var_array> as a child element of <shading_model>.
