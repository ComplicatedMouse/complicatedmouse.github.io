Soft Particles, Process as Fragment

This section describes the Process as Fragment feature which handles Soft Particles processing on a particle fragment basis.

Overview

The soft particles feature, Process as Fragment, is a feature that processes the Soft Particles at the level of the primitive polygon face's pixels.
The alpha gradation will appear cleanly, even for primitives with few vertices.
The following equation determines the alpha value for certain fragments in a particle.

  • Depth Value = the depth value for the particle fragment you are attempting to render
  • Z Buffer = the depth value for pixels already rendered

Fragment Alpha Value (ranging from 0.0 to 1.0) = (Depth Value - Z Buffer) ÷ Setting Value

The following images show the changes to alpha values as a result of the Difference in distance (…) between the Floor surface depth value (█) and the distance to the Particle surface fragment (█), from the viewpoint.
Start-to-fade distance is set to 10.
A … = 0 When the equation is 0 ÷ a setting value of 10 = 0.0, the alpha value is 0.
B … = 5 When the equation is 5 ÷ a setting value of 10 = 0.5, the alpha value is an intermediate value of 0.5.
C … = 20 The equation is 20 ÷ a setting value of 10 = 2.0.
The alpha value will be 1.0 because it must fall within the range from 0.0 to 1.0.
