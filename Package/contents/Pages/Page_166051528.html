Writing to NX Development Cards

Writing NX applications to NX development cards makes it possible to test them without needing to connect to a host PC.

This section describes how to write NX applications to NX development cards and then run them on the NX development hardware (SDEV or EDEV).

Warning

Currently, when written to an NX development card, most of the sample programs will not run normally or will not display anything on the screen even if they do run.
You can use the following sample programs to test this procedure.

  • GfxShaderCompile
  • GfxSimple
  • NvnTutorial01GettingStartedWithNVN
  • HidSimple
Info

NX Development Cards

NX development cards are NX game cards intended for development use on the NX development hardware (SDEV and EDEV).
For more information, see the Hardware > Peripheral Hardware section and the Development Environment section in the NX Overview.


What is a Multi-Application Card?

Starting with NX Add-On 5.2.0, multiple applications can now be written to an NX development card.
This type of NX development card, with multiple applications written to it, is called a multi-application card.


While you will be able to create a multi-application card starting with NX Add-On 5.2.0, you will not be able to use the cards to submit applications as retail products.

Info

What to Do When Applications Written to an NX Development Card Do Not Run

Confirm that your application satisfies the following conditions.

  • The application operates as expected when no program arguments are passed at startup.
    • Currently, no program arguments are specified when an application that has been installed using the following procedure is run.
  • The application operates as expected when the feature for communicating with the host PC is not used.

Creating an Application Image That Can Be Written to an NX Development Card

To write an NX application to an NX development card, you must create a writable image of the application in the form of an NSP file.

To output an NSP file, from the NX application select Project Configuration Properties > General > Application Program Format > Nca, and build the file.

Info

NSP files can also be created by using the command line with the AuthoringTool.
If you plan to create the NSP file by using the command line or by constructing your own build system, Creating Application Imagessee Creating an Independent Build System.

NX development card parameters can be configured in the NMETA file that is used when creating the NSP file.
For more information about writing NMETA files, see this page. For more information about NX development card parameters, see this page.

Note

The values specified in the NMETA file are used in the HOME Menu and in DevMenu. However, the system automatically caches the values for some of these parameters, so sometimes the expected values may not be displayed.
If the values are not updated to the values specified in the NMETA file, see Disabling the Application Control Data Cache.

Writing the Application

Required Items

The following items are required before you can write an NX application to an NX development card.

  • NX development card
  • DevMenuCommand
  • RunOnTarget

DevmenuCommand and RunOnTarget are included in the NintendoSDK NX Add-On package.
See here for the speed settings and actual capacity of the application you want to write.

For more information about getting NX development cards, contact Nintendo support.

Write Procedure

  1. Start the development hardware and connect it to a PC.
  2. Insert the NX development card into the Game Card slot.
    Insert the card with the label facing up and press it in until you hear a click.
  3. Use RunOnTarget and DevmenuCommand to write the NX application to the NX development card. For more information, see Execution Command, which follows.
  4. If the "[SUCCESS]" message is displayed, the write operation is complete (see Execution Log).


Warning

Do not remove the NX development card during the write operation.

Execution Command

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- gamecard write "C:\application.nsp"
  • Specify the full path to the NSP file for writing.
  • For more information, see NX Development Card Features.

Execution Log

The following execution log is displayed.

---- given Parameter ----

---- Card Parameter ----

WriteSize    : 4107264 Byte

---- Write Start ----

4044800 / 4107264
Write rest sectors.
Finished
[SUCCESS]
  • CardSize shows the size for Card specified in the NSP file. WriteSize shows the size to write.

Creating Multi-Application Cards

Required Items

In addition to the application write preparations, an NSP file for uploading is also required.

For more information about creating an NSP file for uploading, see Multi-Application Cards.

Info

A multi-application card can be created even when not using NX Add -On 5.2.0.

Execution Command

For more information about using NX Add-On 5.2.0, see here. 
%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- gamecard write "C:\application0.nsp" "C:\application1.nsp"
  • List the full paths of the NSP files being written.
    • NSP files for applications and patches can be listed.
  • If a multi-application card is inserted, when this command is executed, the applications are listed in the DevMenu and HomeMenu, according to the illustrated sequence.
  • For more information, see DevMenuCommand.


When writing a multi-application card, an NSP file for uploading and NSP files for the included applications are required.

The same items must be specified in the same order for the included applications as the order specified when creating the NSP file for uploading.

When creating the NSP file for uploading, specify the text file specified by the --nsp-list-file option to write a multi-application card.


%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- gamecard write --multi-application-card "C:\mac.nsp" --nsp-list-file "C:\multi-application-card-file-list.txt"

Initializing

You can also use DevMenuCommand to initialize NX development cards.

  1. Start the development hardware and connect it to a PC.
  2. Insert the NX development card into the Game Card slot.
    Insert the card with the label facing up and press it in until you hear a click.
  3. Use RunOnTarget and DevmenuCommand to write the NX application to the NX development card. For more information, see Execution Command, which follows.
  4. If the "[SUCCESS]" message is displayed, the write operation is complete (see Execution Log).

Execution Command

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- gamecard erase

Execution Log

The following execution log is displayed.

Erase gamecard.
[SUCCESS]

Running Applications Written to an NX Development Card

You can use DevMenu or RunOnTarget to run NX applications written on an NX development card.

Running Using DevMenu

When the NX development card is inserted in the development hardware, the application IDs of applications written to the NX development card are displayed in the Application tab of DevMenu.

Press right on the +Control Pad or press the R Button to move the focus to the screen with the list of applications.
Select the application you want to run and then press the A Button to run it.

Running With RunOnTarget

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe gamecard

In the case of a multi-application card, the preceding command runs the application, among those on the NX development card, that is displayed at the top of the Applications List screen.

Important Information

  • After the application IDs of applications written to the NX development card are displayed in the Application tab of DevMenu, they continue to show there, even after the card is removed. This also applies to the production version of the menu.
  • Data written to an NX development card cannot be directly read from the NX development card.
