Presence Group ID

Presence contains features to set and get application-specific information.
Usually, this information can only be shared between friends who are running the same application.

However, there are situations in which you may want to share friend presence across multiple applications, such as when multiple versions of the same application are released or the ROM is separated, depending on the country in which the product is sold.

The presence group ID exists for this purpose.
The system determines whether to share application-specific information, based on whether the presence group IDs of the applications match.

Sharing Parameters

The various parameters of nn::friends::FriendPresence are as described below, depending on whether the presence group matches.

Function Name Effect of presence group ID Description
GetStatus None Can be used to get the presence status, regardless of the presence group ID.
GetLastPlayedApplication None Can be used to get the last played application.
GetLastUpdateTime None Can be used to get the time when the presence was last updated, regardless of the presence group ID.
GetDescription Yes Returns an empty string if the presence group ID does not match.
GetAppValue Yes Returns an empty string if the presence group ID does not match.
IsSamePresenceGroupApplication Yes Returns false if the presence group ID does not match.

Presence Group ID Settings

Specify the presence group ID in the PresenceGroupId item of the NMETA file.
The same value as the application ID is set by default.

To share presence between multiple applications, choose one application to represent the group and set the presence group ID of that application as the PresenceGroupId of the other applications.
