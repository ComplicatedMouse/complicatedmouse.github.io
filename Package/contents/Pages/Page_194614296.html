Voided the need to rebuild the library when enabling the Spy library in a release build.

Previously, when you wanted to use the Spy library in a release build, you had to rebuild the library after replacing the spy_Config.h file.

This release voids the need to rebuild the Spy library. That library is enabled just by replacing the spy_Config.h file.

Note

You must rebuild any code that calls Spy library functionality after replacing the spy_Config.h file.

Deprecating the NN_SPY_ENABLE Macro

The macro that indicates whether the Spy library has been enabled has been changed to NN_BUILD_CONFIG_SPY_ENABLED, in conjunction with NintendoSDK coding conventions.

The existing NN_SPY_ENABLE macro will be deprecated in NintendoSDK 1.1.0.

(Testing) Addition of nn::spy::DebugModule

Note

This functionality is currently in testing. You can use it in your applications, but note that it may be changed or deleted without notice.

Added nn::spy::DebugModule, to record the working state of the Spy library.
