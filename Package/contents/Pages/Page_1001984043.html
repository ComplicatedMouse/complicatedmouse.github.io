Main Window

Figure

 Description

Displays whether ToolClip is enabled or disabled.

You can check whether it is displayed in Maya > Windows > Settings/Preferences > Preferences > Help > ToolClip.

These options are the preset options.

The term preset refers to a feature where a preset file (an FDES file set to edit locked status) is used to set some or all option settings to a particular fixed value.

Related Links Preset Options

Options that relate to exporting intermediate files.

Export Target: Specifies the export target.

Output Folder: Specifies the output folder for intermediate files.

Merge fmd File: Specifies an FMD file path to merge the specified FMD files.

Merge ftx File: Select this option to merge FTX files.

Merge Animation Folder: Specifies a folder containing animation intermediate files to merge the specified animation intermediate files. For more information, see the related links.

Related Links For more information, see Notes When Re-exporting. Related Links Output Options

General intermediate files options.

Magnify: Specifies the magnification ratio for the model's vertex coordinates and translate value.
Texture SRT Mode: Specifies the calculation method for texture SRT.
Remove Namespace: Select this option to remove the namespace part of node names and material names, and then export data.
Comment: Include comments about editing the intermediate file.

Animation options.

Optimization options for the model.

Detailed Description (Click to expand)

  • Compress Bone
    Specify the bone compression mode to use for the entire model.
    To reduce the load due to the matrix computations and the amount of memory used, lower the number of bones.

    Info

    Bones exported to the intermediate file are compressed, but data saved in the DCC tool is not changed.

    The following files support compression: FMD, FSK, and FVB files.

    Modes Description
    None Do not compress bones. Exports the hierarchical structure of the DCC tool as is.
    Cull

    Deletes the bones that are not required to display the model located at the end of the hierarchical structure, and then exports.

    This mode is useful for models that use skinning.

    Bones for which the no-compress flag has been set are not deleted.
    Merge

    In addition to the processing performed when Cull is specified, bones with matrices that can be merged are merged and exported.

    Generally, this reduces the number of bones even more than using Cull, but the following restriction applies: a non-uniform scale (where X, Y, and Z are not the same) must be set for bones that have children.

    If bones that do not follow this restriction or children of such a bone are compressed, the model does not display correctly.

    All frames must observe this restriction when the scale is being animated.

    Bones for which the no-compress flag has been set are not deleted.
    Unite Bones for which the compress disable flag has been set are collected together into one. This feature cannot be used in skinning models. Skinning models are converted to rigid bodies.
    Unite All

    Collects all bones together into one regardless of the compress disable flag.

    The root bone matrix is represented by a unit matrix. Model coordinate system values are used for the vertex coordinates, normal vector, tangent vector, and binormal vector of all shapes.

    This option is used for single objects and for topographical data that does not require a hierarchical structure.

    The following files support compression: FMD, FSK, and FVB files.

  • Unite Child
    If this option is selected, nodes that have shapes located at the end of hierarchical structures are gathered together into one node and exported. Compress Node is useful at times when Unite is not specified.
    All child bones of a bone for which the compress disable flag has been set are compressible and are compressed if skinning is not affected. If the compress disable flag has not been set for any node, this option does nothing.

  • Compress Material
    Select this option to compress materials that have the same attributes and gather them into one material for export.
    Materials for which Compress has been set to Don’t compress, using the plug-in for setting material attributes, are not compressed.

  • Compress Shape
    If there is more than one shape (<shape> element in an FMD file) having the same bones and materials, selecting this option gathers the shapes and exports them together.
    If this option is not selected, shapes are exported individually even if they have the same bones and materials.
    This option is effective when Compress Bone is set to a value other than None or Unite Child, and more than one material associated with a particular bone has the same shape after bones have been grouped together.
    This option is also effective when Compress Material is selected and multiple shapes with the same material exist on a single bone after bones have been grouped together.

  • Divide Submesh
    Specify whether to enable submesh division.
    To perform submesh division, select this option and then enter the value for the --divide-submesh-options intermediate file optimizer option on the right. If this option is left blank, submeshes are not split.

    Info

    Example: --max-bounding-box-width=1.0

  • Polygon Reduction
    Specify whether to enable polygon reduction.
    To perform polygon reduction, select this feature and then enter the value for the --polygon-reduction-options intermediate file optimizer option to the right.
    However, if you do not enter a value after the option flag, polygon reduction is not performed.
    If you enable this option while using LOD export mode, you can overwrite the LOD model created with polygon reduction with a Maya LOD model or add the Maya LOD model in addition to this LOD model at a later time.
    For the --optimize-position-policy option, specify a value other than 1, so vertices are not shared.

    Info

    Example: --target-rate-lod-level1=0.5 --target-rate-lod-level2=0.2

Select this option to specify the tolerated margin for error when animation is baked and optimized.

Detailed Description (Click to expand)

Select this option to specify the tolerated margin for error when animation is baked and optimized.

This setting specifies the maximum allowable difference between the start frame value and the value used for any other frame. If the difference remains less than this maximum, the animation is assumed to be fixed and is exported by using only the value for the start frame.

When animation curve data is created automatically from baked data, a key is created for each frame (in the units specified by Frame Precision), and the key for the below tolerance value is deleted by the maximum differential value when performing Hermite interpolation.

Info

The smaller the allowable difference, the greater the animation precision. However, as the number of keys grows larger, the amount of animation curve data also increases.

  • Node Translate, Node Rotate, Node Scale
    Specify the tolerance as an absolute value for the translation, rotation, and scaling of skeletal animations and for the translation, rotation (direction) of scene animations.
    Node Rotate is specified in units of degrees.
    Node Translate is the tolerance before the value is multiplied by Magnify.
  • Texture Translate, Texture Rotate, Texture Scale
    Specify the tolerance as an absolute value for the translation, rotation, and scaling of texture SRT animations.
    Texture Rotate is specified in units of degrees.
  • Color
    Specify the allowable error for the translation, rotation, and scaling of color animations and scene animations as an absolute value.

Related Links Animation Bake Tolerance Options

Specifies the allowable range of errors when an animation is quantized.


Detailed Description (Click to expand)
  • Node Translate, Node Rotate, Node Scale
    Specify the allowable error for the translation, rotation, and scaling of skeletal animations and the translation and rotation (direction) of scene animations.
    For Node Translate and Node Scale, this setting is specified as a percent of the range of values of animation results.
    For Node Rotate, this setting is specified as an absolute value in units of degrees.
  • Texture Translate, Texture Rotate, Texture Scale
    Specify the allowable error for the translation, rotation, and scaling of texture SRT animations.
    For Texture Translate and Texture Scale, this setting is specified as a percent of the range of values of animation results.
    For Texture Rotate, this setting is specified as an absolute value in units of degrees.


This option is not applied if an animation intermediate file is merged. The value of the intermediate file to merge is used.

Specifies the scripts (MEL) to execute before and after export.

Detailed Description (Click to expand)
  • Pre-Export Script
    Specify the MEL file to execute before exporting. (MEL commands can be entered directly.)
    No script is executed if this option is left blank.
    If the Export Target option is set to Selection, you can use this option to specify the nodes to export by using the following syntax: select root_node.
    You can also look at the value of the NINTENDO_EXPORT_IS_MULTI_EXPORT environment variable and select nodes if they are not currently being exported by the multi-export plug-in.
    If @node is specified for the output filename, the node name selected using this option is used.
    If an environment variable is used with the Output Folder, Merge fmd File, or Merge Animation Folder options, this option applies the value in the environment variable.
    Call the NintendoSetOptimizerExtraOptionFmd function to specify additional intermediate file optimizer options for the FMD file.
    Call the NintendoOverwriteExportOption procedure to overwrite the export options settings and some of the configuration settings that are being exported.
  • Post-Export Script
    Specify the MEL file to execute after exporting. (MEL commands can be entered directly.)
    No script is executed if this option is left blank.
    The specified MEL is executed even if an error occurs during export.
    Use the NINTENDO_EXPORT_SUCCEEDED environment variable to determine whether export was successful. For more information, see Script Options.
    If merging intermediate files is enabled, the specified MEL file is executed.
    Use the NintendoGetExportedTransformPaths procedure to get the transform node group exported as <bone>.

    Use the getenv command inside the MEL file specified as a Pre-Export Script or a Post-Export Script to get information from environment variables. For more information, see Script Options.


Related Links Script Options

To export an intermediate file, specify the appropriate options and click Export or Apply.
