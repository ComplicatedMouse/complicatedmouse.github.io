New Effect Expressions

This section introduces features that broaden the scope of what can be depicted with effects.

Z-Fighting Suppression

Added a feature to suppress z-fighting to reduce the flickering that can occur when a particle is drawn at the same position as another polygon.

This feature suppresses z-fighting by adding a fixed value to the particle's depth information so it gets rendered slightly further in the foreground.
Using this feature removes the need to micro-adjust the positions of emitters in the application to avoid this problem.

The following overview describes the settings ().

Select or clear the check box to enable or disable z-fighting suppression ().
This setting can also be changed programmatically within game applications by using the nn::vfx::ViewParam::Set() function.

Info

Note the following when using this feature.

  • As described previously, do not use this feature to adjust the actual depth relationships between polygon.
  • This feature cannot be used to suppress z-fighting within emitters. To suppress z-fighting within emitters, assign the emitter shape a nonzero thickness and configure particles to be emitted from non-overlapping positions on the emitter faces.

The following images compare the results of enabling or disabling z-fighting suppression.

A ON When z-fighting suppression is enabled, no flickering occurs when polygons overlap at the same position.
B OFF (the default) When z-fighting suppression is disabled, a flickering effect occurs when polygons overlap at the same position.

Effect Emitters

Created a page describing the behavior and characteristics of the various emitter types in the NintendoWare Effect module.

NintendoWare Effect makes use of the following three emitter types.
The processing of a particle's position and velocity depends on the emitter type.

 CPU Emitter CPU emitters are calculated by the CPU.
GPU Emitter

The GPU emitter is calculated by the GPU using the vertex shader.
GPU+SO emitter

The GPU + SO emitters are calculated by the GPU using the compute shader, which is good at computation processing.
