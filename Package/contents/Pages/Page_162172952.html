In-Game Editing Workflow

This section describes the in-game editing workflow that allows you to create effects while directly checking them in the game application.

Recommended Uses for In-Game Editing

Speeding Up the Effect Creation Cycle

The in-game editing workflow allows you to preview effects directly in-game, without having to regenerate the binary data or other data for the game, which makes it possible to significantly reduce the length of the editing cycle when creating effects.

Editing Effects Directly Within the Game

The in-game editing workflow allows you to create and edit new effects directly within the game, thereby making it possible to create and preview effects in the intended game context, rather than out of context in the effect viewer.

Overview

In-game editing provides a way to create and edit effects while checking them in the game application.

A Preview Effects

When a new emitter set is created in EffectMaker, a preview effect is played in the game application.
This preview effect also includes the looping settings configured in the preview controls and as a result it can be used and edited similarly to in EffectViewer.

General Workflow in the Past

In general, the past workflow consisted of creating data in EffectMaker while checking the effects in EffectViewer.
Checking effects in coordination with game elements was performed in the game application, but revising the effects required repeating the following flow many times.

EffectMaker and EffectViewer Create and edit the data in EffectMaker while checking the effects in EffectViewer.
Binary Conversion Effect data can be temporarily saved in EffectMaker and then converted to binary data for in-game use.
This process can sometimes take a little while to complete.
Game

Check whether the effects display correctly in the game.

In-Game Editing Workflow

The workflow using in-game editing allows you to adjust the effects from the time they are created while checking them in the game application.
Edits in EffectMaker are immediately applied to the game application, which allows effects to be completed efficiently in a short amount of time.

EffectMaker and the Game

It is also possible to create and edit effect data in EffectMaker while checking the effects in the game.

Advantages of In-Game Editing

Implementing in-game editing in a game application has the following advantages.

  • Effects can be adjusted to match in-game elements
  • It is easy to rework effects as necessary

Adjusting Effects to Match In-Game Elements

Unlike when working with EffectViewer, creating and editing effect data while also checking it in the game makes it possible to adjust effects to be consistent with the game graphics and game elements related to game playability.

Graphical Elements Playability Elements

  • Apply lighting in the game to effects
  • Editing effects in a state of post-effects
    such as glare and depth of field applied
  • Timing of displaying and deleting effects
  • Effect display priority with playability in mind

Reworking Effects

Edits made in EffectMaker are immediately applied in the effects in the game application. As a result, in most cases, effects can be created, edited, and finished all in one session, which in turn makes it easier to rework the effects later if necessary.
Consider the following example of the previous effect creation workflow.

  • It takes 30 seconds to convert effects for in-game use
  • Each effects goes through 100 edits and iterations
  • There are 100 effect elements

Using this example workflow would incur a cost of 30 seconds × 100 times × 100 effects = ≈80 hours in terms of required work hours.
