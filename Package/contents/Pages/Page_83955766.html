Merged Formats for FSP (FCL, FTS), FTP, and FVM Intermediate Files

The FSP (FCL, FTS), FTP, and FVM formats have been merged into the FMA (material animation file) format.

During the period of migration to the FMA file format, FSP (FCL, FTS), FTP, and FVM formats can still be handled in the same way as before, but you cannot use these formats after the period ends.

We have not yet determined when the migration period will end, but we will provide notice of the version that represents the end of the period when we release the preceding version.

The following information is required to migrate to the FMA format.

Building the Same Workflow as NW4F

Converting FSP (FCL, FTS), FTP, and FVM Files to FMAB Files

Use %NINTENDO_SDK_ROOT%/Tools/Graphics/3dTools/ConvertToFmab.bat to convert FSP (FCL, FTS), FTP, and FVM files into FMAB files.

Input the path to the file to convert or the path to a folder containing the files. If a folder path is input, all files below that folder with extensions representing convertible files will be recursively converted.

The converted files will be output to the same folder as the input files, named as [InputFileName]_[OldExtension].fmab. For example, if the input file is named human.fsp then the converted file will be output as human_fsp.fmab

ConvertToFmab.bat human.fsp human.ftp human.fvm ./ResourceDirectory

You can also drag files and folders to ConvertToFmab.bat to convert the files.

Switching the Maya Plug-In and 3ds Max Plug-In to Output FMA Files Instead of FCL, FTS, and FTP Files

The Maya plug-in and 3ds Max plug-in normally output FCL, FTS, and FTP files, but you can switch the settings in the config file (Export.ini) to output FMA files instead. 

use_fcl_fts_ftp="false"

When you switch to output in the FMA format, the standard practice is to export texture pattern animation, color animation, and texture SRT animation together as a single FMA file.

To write each type of animation as an individual FMA file, to match the previous functionality, make the following specification in the config file. This specification ensures that each type of animation is output separately.

separate_material_anim="true"

In this case, each animation type will be output with the name [output name]_[old filename extension].fmab. For example, if the output name is Human, the color animation is output as Human_fcl.fmab, the texture SRT animation is output as Human_fts.fmab, and the texture pattern animation is output as Human_ftp.fmab.

Switching From FSP (FCL, FTS), FTP, and FVM Files to FMA Files for Newly Created Files in 3DEditor

In 3DEditor, both FSP (FCL, FTS), FTP, and FVM files and FMA files can be loaded, saved, and edited without any special settings.

By default, the Create New Animation menu displayed when right-clicking in File View shows the following items.

  • Shader Parameters
  • Color
  • Texture SRT
  • Texture patterns
  • Bone Visibility
  • Material Visibility

To switch these items to the menu to create FMA files, configure the following team settings.

<Preset>
    <EnableMaterialAnimCreation>true</EnableMaterialAnimCreation>
</Preset>

When these settings are configured, the Create New menu will display the following items.

  • Material Animation
  • Bone visibility

To create individual animations for each animation type when creating an FMA file, as with the previous functionality, configure the following team settings.

<Preset>
    <EnableMaterialAnimCreation>true</EnableMaterialAnimCreation>
    <SeparateMaterialAnimCreationMenu>true</SeparateMaterialAnimCreationMenu>
</Preset>

When these settings are configured, the Create New Animation menu in 3DEditor will display the following items.

  • Material (Shader Parameter)
    • The name [model name]_fsp will be inserted as the default name.
  • Material (Color)
    • The name [model name]_fcl will be inserted as the default name.
  • Material (Texture SRT)
    • The name [model name]_fts will be inserted as the default name.
  • Material (Texture Pattern)
    • The name [model name]_ftp will be inserted as the default name.
  • Material (Material Visibility)
    • The name [model name]_fvm will be inserted as the default name.
  • Bone Visibility

Collecting Multiple FMA Files Into a Single FMA File

You can use the combine function of the intermediate file optimizer to collect multiple FMA files into a single FMA file.

3dIntermediateFileOptimizer.exe Human_ShaderParamAnim.fmab  -o Human.fmab --combine --combine-options "Human_TexturePatternAnim.fmab Human_MaterialVisibilityAnim.fmab"

You can also manage separate intermediate files for each shader parameter animation, texture pattern animation, and material visibility animation, as was done previously, and then collect them into one material animation file right before converting them to binary.

Cautions for Binary Conversion

The 3D binary converter does not support input of FSP (FSL, FTS), FTP, and FVM format files at the same time as FMA format files, so be careful not to mix these formats when transitioning to FMA format.

Use 3DEditor Deprecated

The Use 3DEditor option (use_3deditor, lock_use_3deditor) in the export plug-in option window has been deprecated.

Even without Use 3DEditor, if you have an intermediate file loaded in 3DEditor, it will automatically be reloaded when overwritten after export.

Using the config file's user_custom_ui, you can also add a button in the export plug-in option window for purposes such as starting 3DEditor or loading any intermediate file to 3DEditor.

Starting 3DEditor and file loading to 3DEditor can also be done using 3DEditorBridge.

If you need the same features as Use 3DEditor, you can get them using user_custom_ui and 3DEditorBridge.

The following examples realize the same features as Use 3DEditor.

  1. Specify a MEL procedure to create the same check box as Use 3DEditor with user_custom_ui.

    global proc CreateUse3dEditorUi()
{
    columnLayout;
        checkBox -l "Use 3DEditor" use3dEditorCheckBox;
    setParent ..; // columnLayout
} 
    user_custom_ui="CreateUse3dEditorUi"

  2. Start 3DEditor with 3DEditorBridge in a post-export script, and load the output file to 3DEditor.

    global proc Use3dEditorOnPostExportScript()
{
    int $use3dEditor = `checkBox -ex use3dEditorCheckBox`&&`checkBox -q -v use3dEditorCheckBox`;
    if ($use3dEditor)
    {
        string $bridge = "%NINTENDO_SDK_ROOT%/Tools/Graphics/3dEditor/3dEditorBridge.exe";
        // Specify the team settings and start 3DEditor
        system($bridge + " start-3deditor --team-settings-file %MY_3DEDITOR_TEAM_SETTINGS_FILE_PATH%");
        string $temp_output = `getenv NINTENDO_EXPORT_TEMP_FMD_FILE_PATH`;
        string $output = `getenv NINTENDO_EXPORT_FMD_FILE_PATH`;
        if ($temp_output != "")
        {
            // If use_temp_folder_for_script is true, a temporary file is copied to the output destination and then loaded to 3DEditor.
            sysFile -copy $output $temp_output;
        }
        system($bridge + " load-files " + $output);
    }
}

Intermediate File Optimizer (Previously Optimization Converter)

The following changes were made to how commands are specified for Intermediate File Optimizer.

--(command) --(command)-options="command sub-options"

Text surrounded by single quotation marks in the "command sub-options" part is parsed as a single string including spaces.

In addition, to standardize the command-line interfaces, the position argument specifications of multiple input files no longer can be processed with one iteration.

Restrictions Specific to Material Animations
  • Previously, when FSP (FCL, FTS), FTP, and FVM intermediate files were contained in a single archive, you could convert the files even if they had the same names. You must now assign different names to the different intermediate files. For example, before you could have a single archive containing the files human.fspb, human.ftpb, and human.fvmb. You must now assign them different names like human_fsp.fspb, human_ftp.ftpb, and human_fvm.fvmb if you want to include them together in an archive.
  • You cannot enter existing FSP (FCL, FTS), FTP, and FVM files at the same time as FMA files. Doing so will result in an error.
