Shader Type - Texture Distortion

Texture Distortion

This section describes the Color Buffer Refraction feature for distorting other textures using the RGB values of a texture.

Overview

The Texture Distortion feature is for distorting the color that is output as the result of the combiner calculations using the RGB values of the texture registered in the Texture 0 tab.
This feature could be used, for example, to express billowing smoke that distorts as its rises. You would register a cloud-pattern texture to somewhere other than Texture 0, and register a texture for distortion to the rising particles.
The following overview describes the settings ().

Setting the shader type to Texture Distortion displays the Texture Distortion () and Coefficient Animation ().
The group has the following settings.

Coefficient

Set the amount of distortion along the x-axis and y-axis using the dials () or in the text boxes ().
The texture that has been set for distortion is multiplied by the value set for the coefficient after it has been displaced in the range from -1.0 to 1.0.
These values can also be used to animate the amount of distortion with Coefficient animation.
Related Link For more information, see Coefficient Animation.

For example, if a pixel of the texture is 0.5 gray (█), the calculation looks like this.

  • (0.5 - 1.0) × 2.0 + 1.0 = 0.0

In other words, this pixel will not display distortion no matter what value is set for the coefficient.

B X,Y = 0,1 To determine the amount of distortion, multiply the y direction (V direction) of the distortion texture by the configured value.
In addition, multiply by the Coefficient animation to animate the distortion.
Info

The distortion image registered to texture 0 is output as unorm_8_8, with the Component section as RGGG.
In the RGGG channel format, the red component is used for horizontal distortion, and the first green component is used for vertical distortion.
