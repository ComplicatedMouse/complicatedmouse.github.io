nvn Library

Introduction

NVN is a low-level 3D graphics API that is supported by NintendoSDK NX Add-On. This API is directly connected to the hardware specification and has no abstraction layers to absorb hardware differences. As a result, the CPU overhead for this API is reduced.

NVN provides the ability to manipulate objects with finer granularity than the gfx library, a benefit to NintendoSDK users when they incorporate the APIs into their own proprietary game engines. However, because NVN is specialized for the hardware, for reasons of compatibility, we recommend using the gfx library (which wraps NVN) as the graphics API for NintendoSDK. Also, use the features described in Interoperability With Low-Level Graphics APIs to use NVN only where needed.

PC Operating Environment

This environment is the PC operating environment required for NVN. The SDEV operating environment complies with the NintendoSDK operating environment.

Installing the Graphics Driver

To use NVN on a PC, you must install a dedicated driver.

You can get the installer file by installing the NVIDIA Graphics Driver package from the Nintendo Dev Interface. You can start the appropriate installer for your operating system from the dialog box displayed during package installation. If you did not start the installer from the dialog box, you can install the driver by running the Install.bat file located in NintendoSDK/Installers/NvnDriver. Batch files are prepared for each operating system, so run the batch file for the operating system you use.

Standard and DCH Versions

Info

The DCH version of NVIDIA Graphics Driver is provided in NX Add-On 9.1.0 and later.


There are two versions of the driver for Windows 10: the standard version and the DCH version.

There are no differences in the NVN driver functionality provided by these two versions. Select the version that suits your environment.

In an environment where NVIDIA Graphics Driver is already installed, you cannot install a different type (standard or DCH) of the driver.

Supported GPU and Supported Features

The library supports Maxwell first generation and later GPU architectures with some feature limitations. Note that several NX hardware features will not be available if you are using a GPU architecture earlier than Maxwell second generation. In addition, some GPU model numbers are not supported. For more information, see the NVN programming guide.

Also, note that when using the dedicated NVN graphics debugger for NVN applications that run on PC, several of the graphics debugger features will not be available if you are using a GPU architecture other than Maxwell second generation. For more information, see the documentation included in the NVN Graphics Debugger package.

nvn.dll

Starting with NX Add-On 12.1.0, nvn.dll is required in addition to the NVIDIA Graphics Driver to use NVN in the PC environment. This is located in %NINTENDO_SDK_ROOT%/Tools/Graphics/NvnLoader.

For more information, see the NVN Programming Guide.


Limitations

NVN uses GLSL/SPIR-V as shader languages. When they are used from NVN, testing using the Khronos Conformance Test is not performed. For this reason, there may be differences with the Khronos GLSL/SPIR-V specifications.
