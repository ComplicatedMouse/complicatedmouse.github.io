gfx Library

Overview

The gfx library provides GPU graphics features.

Objective

The set of graphics features is made up of many APIs. Applications must use specific coding syntax to call these features.

This provides the highest level of performance by the target graphics API, but also requires an extensive rewrite of application code when the target graphics API is changed.

The gfx library provides a minimally abstracted graphics API layer to reduce the dependency between the application and the low-level graphics API.

Using the gfx library allows for an application to be ported to a different low level graphics API without rewrites to the application code.

The goal of the gfx library is to reduce software development costs by decreasing the complexity of porting applications to a future low level graphics API, or allowing testing of alternate low level graphics API while using the same application code.

Policy

Low-level graphics APIs have many differences, especially functional differences between low-level graphics APIs of differing hardware generations.

The gfx library is designed to cushion these differences and provide a thin layer of abstraction. The library design is not intended to be all-inclusive, and avoids abstracting elements that have a high abstraction cost or only provides a heavily degraded performance as a result of abstraction.

The gfx library does not handle those features that are compromised as a result of this policy.

In places where these features are required, call the low-level graphics API functions directly, in addition to the gfx library functions.
