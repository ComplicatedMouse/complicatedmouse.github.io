Introduction

Input Director is a tool that sends HID input from the PC to the development hardware. The transferred values are treated as the HID input for the development hardware.

Operating the development hardware through HID input on the PC enables the following activities.

  • Seamless debugging
  • Application development that does not require an actual device


The following HID input is supported.

  • Gamepad
  • Six-Axis Sensor
  • Keyboard
  • Mouse
  • Digitizer (pen tablet)


The following devices can be used.

HID Input Device Support Status Notes
Gamepad Nintendo Switch Pro Controller NintendoSDK 10.4.1 and later Only supports Bluetooth connections.
Gamepad XInput-compatible gamepad All
Gamepad DirectInput-compatible gamepad All Not ensured to work.
Six-Axis Sensor Nintendo Switch Pro Controller NintendoSDK 12.1.0 and later Only supports Bluetooth connections.
Keyboard HID-compliant keyboard All
Mouse HID-compliant mouse All
Digitizer Wacom Cintiq Pro 16 NintendoSDK 10.1.0 and later

How to Use

Starting an Application

Start the following program on the development hardware before starting debugging. 

Tools/Hid/InputDirector/InputDirector.exe

This program resides in memory and does not need to be restarted.

After startup, if the development hardware is connected to Target Manager, a connection to Input Director is also created.

When a connection is established with the development hardware, the name of the development hardware is displayed with a check mark next to Connect, as shown below.

The first time you run this tool you must select Settings and configure the defaults.

Clear the check boxes for Switch when mouse moves to edge of screen and Switch using keyboard (as shown in the figure), and click OK to apply the settings.

Transferring Input From a Gamepad

If a connection to the development hardware has been established, all input from the gamepad is automatically sent to the development hardware.

The sent input is treated as input from either NpadStyleFullKey or DebugPad.

You cannot map buttons for the gamepad that the input was sent from.

Gamepad correspondence can be changed in the Gamepad Settings tab of the Settings for the resident program.

Operating as NpadStyleFullKey

In Gamepad, select the gamepad to configure and then set the role to Switch Pro Controller.

Operating as DebugPad

In Gamepad, select the gamepad to configure and then set the role to Debug Controller.

Transferring Input From a Keyboard, Mouse, or Digitizer

When Input Director is in capture mode, input from a keyboard, mouse, or digitizer is transferred to the development hardware.

The transferred input is handled as input from the Keyboard, Mouse, or Digitizer mechanism respectively.

The capture mode can be switched using the mouse or a keyboard shortcut. Use the Settings to configure this operation.
