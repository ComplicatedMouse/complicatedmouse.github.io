ASTC Textures

Features

The following table provides a summary of the types and characteristics of ASTC texture formats available.

Format BPP Comments
ASTC4x4
 8.00 Same size as BC7, BC3, and BC5, but with a higher quality than BC7.
ASTC5x4 6.40
ASTC5x5 5.12
ASTC6x5 4.27
ASTC6x6 3.56

Smaller than BC4 (which has a BPP of 4.0) and has a higher quality than BC4.
ASTC8x5 3.20
ASTC8x6 2.67
ASTC8x8 2.00
ASTC10x5 2.56
ASTC10x6 2.13
ASTC10x8 1.60
ASTC10x10 1.28
ASTC12x10 1.07
ASTC12x12
 0.89 Has the smallest file size of the ASTC texture formats but also results in a significant decrease in quality.
  • These are all compressed formats.
  • You can select from among the 14 compression formats in the table to achieve the desired balance between compression ratio and the resulting image quality with a high level of control.
  • These compression formats can be used with all texture types from single-channel textures (L and A formats) to 4-channel textures (RGBA formats).
  • The compression algorithm is also designed to reduce noise by automatically determining the optimal compression scheme from characteristics of the source image.
    • RGB values are all the same ⇒ Treat as grayscale
    • Alpha values all equal to 1 ⇒ Treat as image that does not have alpha values
    • Use a 32-bit TGA image to test the preceding formats to find the appropriate format for your end use case.

As shown in the following images, with the ASTC8x8 format, for example, the quality used for the RGB channels is different when alpha values are present and when the alpha channel is white (all 1).

Noise may be slightly more noticeable when alpha values are present.

 When the alpha values are all white (1), the RGB values are compressed at a higher quality.
Info

Bits per pixel (bpp) is a unit that quantifies the number of bits required to represent the color of a single pixel.
A bpp value of 8, for example, means that 8 bits are used to represent the color of each pixel.

Important Considerations

  • The compression process may take a non-negligible amount of time to complete. (This becomes more pronounced the larger the source images are. In some cases, the compression process may take up to several minutes.)
  • The resulting image files cannot be directly displayed on PCs.
    • Check image quality and size on the development hardware.
    • When previewing images in LayoutEditor on PCs, all images are force-converted to the BC3 format to avoid running into performance limitations in LayoutEditor itself.

Using

You can enable the ASTC formats by setting AstcTextureEnabled to true in the project settings. (By default, this setting is set to false and ATSC textures are disabled.) For more information, see Project Settings File Format.
