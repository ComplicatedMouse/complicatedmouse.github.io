You can now specify additional options to pass to 3D Texture Converter.

A feature has been added to the Photoshop FTX File Format plug-in and Texture Export plug-in for specifying additional options to pass to 3D Texture Converter.

With the Texture Export plug-in, specify these options as follows in the config file.

var texture_converter_options="--resize-single-color 4 --comment 'string with space'";

Use single quotation marks when enclosing option values in quotation marks.
You cannot specify options like --args-file, --job-list, --help, and --silent.

You can use environment variables for the values.
var texture_converter_options="%MYNW_TEX_OPTIONS%";
Environment variables set with Pre-Export Script are also applied.

$.setenv("MYNW_TEX_OPTIONS", "-x 150% -y 200%");

Options specified with texture_converter_options take precedence over existing options specified in the Options dialog box.

Maya Plug-In

Added support for Maya 2020.

The new Transform Offset Parent Matrix feature is conditionally supported.

Cases Where Data Cannot Be Exported Correctly

The scale component of the offset parent matrix is not uniform in the xyz-directions, and the Rotate or Rotate Axis transform attribute or Joint Orient is set to a nonzero value.

The shear component of the offset parent matrix is nonzero. (A warning is displayed if it is a nonzero value in the start frame.)
The offset parent matrix of the transform node of the shape is changed after skinning has been set.
Inherits Transform is cleared and the offset parent matrix has been used as the world matrix.
(The usual restrictions apply to Inherits Transform.)
The scale component of the offset parent matrix of the joint node to not set to 1 and Segment Scale Compensate is selected for that node's child nodes.

If the offset parent matrix is not an identity matrix, the amount of animation data may increase. (For example, this can happen if Rotate Y is the only transform attribute for the animation and Rotate X is not set to 0 for the offset parent matrix.)

To export the standard surface shader (the standardSurface node), specify extended material attribute information in the config file.
Example of the Config File Settings

material_attr_info_standardSurface="baseColor=albedo,_a,as_diffuse; opacity=opacity; emissionColor=emission; normalCamera=normal; specularColor=specular; metalness=metal,_m,no_color"

Upcoming End to Support for Maya 2017

Support will end in NintendoSDK 12.2.0.

You can now specify additional options to pass to 3D Texture Converter.

A feature has been added to the Maya plug-in for specifying additional options to pass to 3D Texture Converter when exporting FTX files.

Specify the following in the config file.

texture_converter_options="--format-with-a srgb_bc7 --resize-single-color 4"

Use single quotation marks when enclosing option values in quotation marks.
You cannot specify options like --job-list, --help, and --silent.

You can use environment variables for the values.
texture_converter_options="%MYNW_TEX_OPTIONS%"

You can also overwrite options using Pre-Export Script as follows.

NintendoOverwriteExportOption("texture_converter_options", "-x 50%")


You can get the current value from the NINTENDO_EXPORT_OPTION_texture_converter_options environment variable.

The options specified using texture_converter_options take precedence over the options set in options windows and other config settings.

Added a feature for disabling the merging of rigid skinning shapes and smooth skinning shapes.

Added a feature for disabling the merging of rigid skinning shapes and smooth skinning shapes when compressing shapes for export with the Maya plug-in.

This action is specified in the config file.

disable_merge_different_skinning_type_shapes="true"

The setting is valid when compress_ignoring_vertex_skinning_count is set to true and the Compress Shape option is selected.

You can also overwrite options using Pre-Export Script as follows.

NintendoOverwriteExportOption("disable_merge_different_skinning_type_shapes", "true")

You can use Pre-Export Script or Post-Export Script to get the current mode from the NINTENDO_EXPORT_OPTION_disable_merge_different_skinning_type_shapes environment variable.

As an example, the execution results for a model with the following composition are provided.

  • Two kinds of material
  • A rigid skinning mesh
  • A smooth skinning mesh (with two weights)
  • A smooth skinning mesh (with four weights)

The table shows the results of exporting this model with different config file settings.

  • Compress Bone: Merge
  • Compress Shape: selected
Export.ini Settings Result 
compress_ignoring_vertex_skinning_count="false"
disable_merge_different_skinning_type_shapes="false"

Shapes are not merged if they have different maximum influence counts for skinning. The resulting vertex data is a bare minimum, but it is hard to reduce the number of shapes.


 
compress_ignoring_vertex_skinning_count="true"
disable_merge_different_skinning_type_shapes="false"


Shapes that have different maximum influence counts for skinning are forcibly merged based on the larger count. This merge yields a bare minimum number of shapes, but the vertex data increases.


 
compress_ignoring_vertex_skinning_count="true"
disable_merge_different_skinning_type_shapes="true"

Shapes that have different maximum influence counts for skinning are forcibly merged based on the larger count, but rigid skinning shapes and smooth skinning shapes are not merged together.

The optimal result depends on the application, so we recommend exporting the model with each of these settings and measuring the performance of the models to select the optimal settings.

You can now use NintendoLiveEditing with materials other than the NintendoWare default materials.

Animation Live Editing has been improved and now applies attribute information included in the config file.

Live editing now enables you to use materials other than NintendoWare default materials like lambert and phong.
If you set material_attr_info_* using the Export.ini config file, you can use the Export plug-in to export the intermediate file from any material. If you parse this setting, you can now change the attributes that you get for the material animation information during live editing.
