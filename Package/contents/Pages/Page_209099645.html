Times When Mii Characters Might Be Added and Deleted

This sections describes the times when Mii characters might be added to or deleted from the database.

When an Applet is Called By the Application

If the application has called the Mii applet or the amiibo settings applet, there is a possibility that Mii characters will be added or deleted.

When the Mii Applet Is Called From System Settings

If the HOME Button is pressed while the application is running, and System Settings are used to create or edit a Mii or to register an amiibo owner and nickname, there is a possibility that Mii characters will be added or deleted.

If the Database is Manipulated Using Debugging Functions

If the debugging functions of the nn::mii::Database class are called from the application, there is a possibility that Mii characters will be added or deleted.

Notes About Additions and Deletions

If you are going to employ the following operations in you application, consider the possibility that Mii characters will be added or deleted at those times.

  • Detecting updates to the database using the nn::mii::Database::IsUpdated() function.
  • Determining whether the database is full using the nn::mii::Database::IsFullDatabase() function.
  • Getting Mii characters from the database using the nn::mii::Database::Get() function.
  • Getting the count in the database using the nn::mii::Database::GetCount() function.
  • Searching for Mii characters in the database using the nn::mii::Database::GetIndex() function.
  • Getting the latest update for a Mii in the database using the nn::mii::Database::UpdateLatest() function.
