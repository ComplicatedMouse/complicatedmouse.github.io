EffectViewer

This section describes EffectViewer, which can be used to check the editing performed with EffectMaker.

Overview

EffectViewer is a viewer for checking the editing performed with EffectMaker.
If you start EffectMaker at the same time and leave them connected, you can perform editing tasks in EffectMaker while viewing the playback status of the effects.

The EffectViewer executable file differs depending on the platform you are using. The following information is for the basic viewer.

Location and Name of Executable File Description
EffectViewer/Win32/EffectViewerGfx.exe A 32-bit build of the viewer for communicating with EffectMaker and playing effect data, implemented using nn::vfx.
EffectViewer/Win64/EffectViewerGfx.exe A 64-bit build of the viewer for communicating with EffectMaker and playing effect data, implemented using nn::vfx.

A Toolbar

Frequently used functions in the viewer debugging features are displayed as buttons in the toolbar.
The following table describes the features in the toolbar.



Show Ground

Shows the ground so it is easier to calculate the position of effects when you operate the camera.
You have the following options.

  • Hide ground ( ): Hides the ground.
  • Show ground using grid (): Displays a grid.
  • Show ground using blocks (): Displays translucent blocks.
Specify Size

Sets the size for individual grid points or blocks.

Lets you standardize the particle size when creating effects.
Specify the Number of Divisions Sets the number of divisions when the ground is being shown by using a grid.

Info

The debug information is displayed using a meter bar (█) to depict the processing load of an effect.
You can use it as a load indicator when using the EffectMaker features.
A red meter bar (█ ) is used to indicate the processing cost of calculations by the CPU. A green meter bar (█) is used to indicate the processing cost of rendering by the GPU.
