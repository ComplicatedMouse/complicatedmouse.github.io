List Operations

All items handled by a sound project are displayed in a list in the Sound Set pane. Each line in the list corresponds to a single item.
This section describes common operations on lists.

Index Cell Operations

The first cell in each line contains an icon. This cell is called the index cell.

The index cell indicates the type of item that the line references.
You can manipulate individual cells in a list just like you manipulate cells in a spreadsheet. But the index cell also enables you to operate on an entire line that represents one item.

Double-click an index cell in a bank list to open a tab showing the referenced bank file.

To select a line, click the index cell. To rearrange lines in the list, drag the selected index cell to a new location.

To select a range of lines, click the index cell for an unselected line and drag.

Registering and Unregistering Items in Groups

If the list contains items that can be registered to a group, you can register the items by using the shortcut menu for the list.

Right-click the list for the items that you want to register to a group, and select Register in Group from the shortcut menu.
A special dialog box opens and displays a list of all groups. Select the group that you want, and then click OK.

To remove an item from a group to which it is already registered, clear the check box.

If multiple items are selected and registered to groups, a gray check mark appears next to groups that contain a mix of registered and unregistered items.
After this gray check mark is cleared, the group does not return to its original mixed state unless you cancel the action.

Batch Editing Cells

To edit many cells at the same time, use the Batch Edit dialog box.
Select the multiple cells you want to edit, and open the Batch Edit dialog box in one of the following ways.

  • Press SHIFT+F2.
  • On the Edit menu, select Batch Edit.
  • Select Batch Edit from the shortcut menu.

Select the value to apply and the application method.

  • Specify directly: The specified value is applied directly, unchanged. Use the true and false check boxes to enable and disable.
  • Specify increase/decrease: The specified value is added to the original value (for example, -20 or +150).
  • Multiply by the specified factor: The specified value is applied to the original value as a scaling factor, such as 0.01 or 20.0.

Selections that would result in values above the upper limit or below the lower limit for the cell are not applied.

To batch edit items that use combo boxes, such as compression formats, players, or banks, type the desired value into the edit box and select Specify directly.

For item names, to prevent duplication, only the uppermost cell can be edited even when several are selected.

Item Name Completion

When you edit player, bank, or group items on the list, you can enter the item name directly.

Also, when you enter names directly, you can use the auto-complete feature.

Sorting Columns and Showing and Hiding Columns

Sorting

You can rearrange the display order of columns, such as Name and Data Size, by dragging the column names.

You can perform the same sorting operations by dragging the column names in the List Item Settings dialog box, as described in the following section.

List Item Settings

You can select to show or hide each column in the list. If the list has too many columns so that it is difficult to view them all, you can hide columns that are used infrequently.

To open a shortcut menu that enables you to show or hide columns, right-click a column header in the Sound Set pane. The selections are applied to the list display.

To open the List Item Settings dialog box and configure show/hide for all columns, select List Item Settings at the top of the shortcut menu.

To change the display order, drag the columns up and down in the list or click the Move Up and Move Down buttons.

Columns that are hidden are not targeted when a cell range is selected.
If a cell is selected and then hidden, the selection is canceled.

Registering List Item Settings

You can register the current list settings (column order and sort order) for each type of list.
The information that is saved includes the column show/hide settings and the column order and sort order for the items.

You can open the Register List Item Settings dialog box by right-clicking the column name and selecting Register List Item Settings from the shortcut menu.

You can save the current list settings by entering a name for the settings.

Applying List Item Settings

This operation applies registered list settings to the current list.

Right-click the column name to display the shortcut menu, and then select one of the names in the Apply List Item Settings submenu to apply those settings to the current list.

In addition to the shortcut menu, you can also apply the settings from the main toolbar using List item:.

The list item settings for the list with the focus are displayed in the button.
If no list settings have been added, the button displays "---" and appears dimmed.
The button also appears dimmed (even if you have already added settings) if the focus has shifted from the list.

Modifying List Items

You can rename or delete list items from the List Items tab in the Options dialog box.

The List Items Settings file is saved to %APPDATA%\Nintendo\SoundMaker\version\user.columns.

Optimizing the List Width

To optimize cell width based on the displayed list contents, double-click the right edge of a column header.

Sort List for Display

To sort by the information in a column, click the column header.
Repeated clicking switches between three sorting types: ascending, descending, and unsorted (the original order).

Sorting by the index cell displays the list in its original order.

"Original order" means (1) the order the list had when it was created or (2) the order that resulted when the user dragged the index cells.

The sorting of items changes only the order in which they are displayed in the list. The "original order" retains the order in which items are saved in the project.

You can also press F12 to update the list display.
Use this feature when an item is inserted or created, after sorting the list by a specific header.
This operation only updates the list display.
If you need to update the project file, select Reload Project from the File menu.

Copy and Paste

You can carry out copy and paste operations on cells and lines, by using key combinations or the menu that appears when you right-click the list.

Copying a Cell

To copy a cell, select Copy Cell from the shortcut menu, or press CTRL+SHIFT+C.
The copied cell is outlined by a dashed line.

Copying a Line

If a line has been copied, the Select Items and Paste feature becomes enabled in the shortcut menu.
If the copy source and the paste target are in different lists, or if multiple lines have been copied, the Select Items and Paste option is disabled.

Executing the paste operation opens a dialog box for the selection of items.
Select the items that you want to overwrite, and then click OK.


Temporarily Disabling Items

You can disable an item or line in the list. This makes it inactive without deleting it from the project.
Disabled items are not subject to conversion. The parameters of disabled items are saved to a file and can be easily restored.

To configure a row, right-click to open the shortcut menu and select Disable Item or Enable Item, or open the Edit menu of the main window and select Disable Item or Enable Item.
Text for disabled items appears dimmed, and a disabled mark is displayed for index cells and reference item entries.
For items also shown in the project tree, the display in the project tree also includes a disabled mark and the item name appears dimmed.

You can disable sounds (including stream sound tracks, wave sound sets, and sequence sound sets), banks, players, groups, wave archives, and instrument lists.
Any of these kinds of sounds can be previewed even when they are disabled.

Coloring Items

You set a color for each item or line.

To set a color, right-click the selected row to open the shortcut menu or from the Edit menu of the main window, select Item Color, and then set the color in the submenu.

Color information is also applied to HTML sound lists and HTML instrument lists.

You can color the following lists.

  • Stream Sound List
  • Stream sound track list
  • Wave Sound Set List
  • Wave Sound List
  • Bank list
  • Sequence Sound Set list
  • Sequence Sound List
  • Waveform archive list
  • Player list
  • Group List
  • Instrument List

Items in a group item list are displayed in the colors that were originally set for them.

In the Project Settings dialog box, on the Color tab, you can change the colors used for text in submenus.

Switching Between Showing and Hiding the Bar for Numerical Value Cells

You can show a bar in cells that display numbers with upper and lower limits. This bar represents a ratio of the maximum value.

The default is to hide, but you can set the show/hide settings for the bar by selecting Tools > Options in the main menu, and then opening the Options dialog box.

Update Points

After you edit the list, sound sets that have unsaved update points are denoted by an asterisk (*) to the right of the sound set node in the project tree. Banks that have unsaved update points are denoted by an asterisk (*) to the right of the bank filename on the Bank tab.


Updating the Information for Waveform Files

To update the information for waveform files displayed in the list, select View > Update Waveform File Information.
You can also press CTRL+R for this operation.

The following items of information are updated.

  • Playback time
  • Ticks
  • The sample rate.
  • Bit Rate
  • Number of Quantization Bits
  • Number of channels

Item Warnings

If the link for the reference label of an item is broken, or the item name is a duplicate, a warning icon appears on the list.


Upper Limit on the Number of Items

The upper limit on the number of items (such as stream sounds and banks) that can be handled by the system at runtime is 16,777,216 (2 to the 24th power).
If the number of items exceeds 16,777,216, the converter and the nn::atk library on the hardware become unstable.

There is no upper limit on the number of items in SoundMaker. You can register as many as memory allows.

Splitting the View

The view on the Sound Set tab can be split to display two sets of information. This section describes the split view features.

Terminology

  • View: The editing screen that contains the tab controls in the Sound Set tab and the Bank tab.
  • Top view, bottom view, left view, right view: The views displayed when split.
  • Active view: The view that is the target of operations. The last view that had focus is the active view.
  • Active tab: The tab selected inside the active view. If the view is split, there is one selected tab in each view (top/bottom or right/left).

Splitting the View

Splitting the View

The view can be split in half either horizontally (top/bottom), or vertically (left/right).

To split the view, select Window > Split Horizontally or Split Vertically. Or from the toolbar, click Split Horizontally or Split Vertically.

After a single view is split, the active tab from the original view is also displayed in the newly opened view.

The two view types in split view have the same functionality and operate independently.


Split Horizontally

If the view is not split, the current view appears on the top, and the new view appears on the bottom.
If the view is split vertically, the left-side view appears on the top, and the right-side view appears on the bottom.

Split Vertically

If the view is not split, the current view appears on the left, and the new view appears on the right.
If the view is split horizontally, the top view appears on the left, and the bottom view appears on the right.


Removing a Split View

To remove the split view, close all the tabs for the view that you want to remove.

You can easily do this by selecting Close All Tabs from the shortcut menu on the tabs.


Tab Operations

Tab Operations

If you open multiple tabs in the same view, you can rearrange them by dragging the tabs with the mouse.
Note that you cannot move a tab to another view.

To close one or more tabs, right-click a tab to display a shortcut menu, and then select Close to close the selected tab, Close All Other Tabs to close all tabs in the view except the selected tab, or Close All Tabs to close all tabs in the view.
From the main menu, selecting Window > Close All Tabs closes all tabs in the split view.


You can also click the center button of the mouse to close the selected tab, which is equivalent to selecting Close from the shortcut menu.

Shortcut Keys

See the List of Tab Operation Shortcuts in Common Operations: List Operations.

Operations Targets

If you switch the tab display by using the Project Tree or the Search window, the operation is performed on the active view.

Minimizing the View

You can minimize the split view without closing it.

To minimize the view when the view is split, click the Minimize button that appears to the side of the tab display area. Only the tabs are displayed.
When the view is minimized, the Minimize button switches to the Undo button. Click the Undo button to restore the display to its normal size.


The following methods are other ways to switch between Minimize and Undo.

  • Double-click the splitter that defines the border of the split view.
  • If the view is split horizontally, click the Split Horizontally button.
  • If the view is split vertically, click the Split Vertically button.

The operation minimizes the view that is not the active view.


When a view is minimized, you can restore it to its original size by doing any of the following:

  • Double-click a tab in the minimized view.
  • If the view is split horizontally, click the Split Vertically button.
  • If the view is split vertically, click the Split Horizontally button.



About Files

The characters that can be used in a filename or path name are restricted. The following characters can be used.

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character of the filename.
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9, _] for subsequent characters in the filename.

Directory names must be 248 characters or less, and the path that includes the filename must be 260 characters or less.

Types of Project Files

SoundMaker handles the following three types of project files:

  • Sound project files
  • Sound set files
  • Bank files

Types of Reference Files

SoundMaker can play back and convert the following types of files, and register them as reference files for items.

  • WAV files (waveforms)
  • AIFF files (waveforms)
  • Opus files (waveforms)
  • Standard MIDI files (sequences)
  • Text sequence files (sequences)

SoundMaker can register other file types, but it cannot play or convert them.

Selecting Open Reference File from the shortcut menu in the item list opens the corresponding application, if possible.

  • The application for opening the reference file can also be specified in the options dialog box, in the External Editor tab.

Saving Reference File Paths

To save the file path referenced by an item, use a relative path when the file being referenced is on the same drive as the Sound Project file, and use an absolute path when the file being referenced is on a different drive.

Previewing Sounds Using the Wave File Selection Dialog Box

You can preview the selected wave file from the dialog box used to select wave files.

If Play Automatically When Selecting Files is selected, the wave file is played automatically at the same time you select it from the list.
