NX Addon 0.2.0

When the NX add-on is installed, sample resource loading fails.

For samples where resource conversion occurs during the sample project build, an error occurs when attempting to load incompatible resources.
This problem only reproduces in systems where the NX add-on is installed.

As a workaround, build the sample in Visual Studio before executing it.

Cannot start the Ui2dUserShader sample when NX Add-On has been applied.

If you start the Ui2DUserShader sample after the NX Add-On has been applied, an exception will be thrown and the system will crash.

The cause lies in a failure in resource conversion. To fix this issue, replace lines 23 and 24 in the Convert_Win32.bat and Convert_x64.bat files in the Resources directory with the following.

%SHADER_CONVERTER% -pNx --glsl-version=330 --preprocessor-definition="NEED_330_EXTENSION=1" %PREPROCESSOR_OPTION% -oArchiveShader-User.bnsh -sGlsl -cBinary -aNvn --reflection --max-thread=1 --vertex-shader="..\..\..\Resources\Ui2d\UserShader\shader\ui2d_UserVertexShader.glsl" --pixel-shader="..\..\..\Resources\Ui2d\UserShader\shader\ui2d_UserPixelShader.glsl" --include-directory="..\..\..\Resources\Ui2d\UserShader\shader" --variation="..\..\..\Resources\Ui2d\UserShader\shader\ui2d_ArchiveShaderVariation.xml" --print-progress
%SHADER_CONVERTER% -pNx --glsl-version=330 --preprocessor-definition="NEED_330_EXTENSION=1" %PREPROCESSOR_OPTION% -oArchiveShader-UserG.bnsh -sGlsl -cBinary -aNvn --reflection --max-thread=1 --vertex-shader="..\..\..\Resources\Ui2d\UserShader\shader\ui2d_UserVertexShader.glsl" --geometry-shader="..\..\..\Resources\Ui2d\UserShader\shader\ui2d_UserGeometryShader.glsl" --pixel-shader="..\..\..\Resources\Ui2d\UserShader\shader\ui2d_UserPixelShader.glsl" --include-directory="..\..\..\Resources\Ui2d\UserShader\shader" --variation="..\..\..\Resources\Ui2d\UserShader\shader\ui2d_ArchiveShaderVariation.xml" --print-progress
