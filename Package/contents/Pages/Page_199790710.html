Changes for Developers

This section describes the changes made to NintendoWare audio (nn::atk) in NintendoSDK 1.1.0.

Contents

  • Added a feature for prefetching stream sounds.
  • Added new channel settings that can be used for audio effects.

Added a feature for prefetching stream sounds.

Added a prefetching feature that pre-loads the data at the beginning of stream sounds into memory to eliminate the loading wait time when the stream sound is played.

Before you can use prefetching functionality, you must first enable prefetching for stream sounds by using SoundMaker, which will generate prefetch data as a BFSTP file. For more information about settings, see Stream Sounds in the SoundMaker manual.

The ways you use the prefetching function in programs will vary depending on the ways the stream sounds are played back. For more information, see Stream Sounds in the Programming Guide.

  • Normal Stream Sound Playback
    • Prefetching is automatically applied when MemorySoundArchive is in use.
    • As you will see in Stream Sounds in the Programming Guide, you must load sound archives and any prefetched data before playback when using FsSoundArchiveArchive.
  • Playing Stream Sounds With Paths Specified Using StartInfo
    • To play stream sounds that use nn::atk::SoundStartable::StartInfo::StreamSoundInfo::externalPath, load the BFSTP file directly in the program, and then specify the address to which the BFSTP file was loaded to the nn::atk::SoundStartable::StartInfo::StreamSoundInfo::prefetchData function.

Added new channel settings that can be used for audio effects.

Added new channel settings that can be used for nn::atk::EffectDelay, nn::atk::EffectReverb, and nn::atk::EffectI3dl2Reverb.
The following channel settings are currently available.

  • nn::atk::EffectDelay : 1ch, 2ch, 4ch, 6ch (new)
  • nn::atk::EffectReverb : 1ch (new), 2ch, 4ch
  • nn::atk::EffectI3dl2Reverb : 1ch (new), 2ch, 4ch, 6ch
