LayoutEditor

This section introduces the new features in LayoutEditor.

HDR Settings Can Now Be Configured

HDR color now can be output using LayoutEditor.

Enable or disable this feature using the HDR color check box in the Color Interpolation Settings in the property window.
When the check box is selected, color values are changed to numerical input, with color value 255 = 1.0.

It is also now possible to set values greater than 1.0 in the color settings, values such as 10.0 and 100.0.
This lets you preview glare effects.
You can use animation curves as with normal colors, meaning you can express things such as changing glare effects.

Added the Ability to Use ASTC Formats for Layouts and Fonts

ASTC formats were added to the layouts and fonts available in LayoutEditor.

ASTC formats are compressed formats, and 11 kinds are available, letting you select a fine balance between compression ratio and compression quality.
You can use them with all formats, from one-channel formats (L, A ) to RGBA four-channel formats.
In addition, the characteristics of the source image are determined automatically and compression is optimized to reduce noise.

You can enable the ASTC formats by setting AstcTextureEnabled to true in Project Settings.
(By default, this setting is false and ASTC formats are disabled.)

ASTC4x4
Same size as BC7, BC3, and BC5, but with a higher quality than BC7.
ASTC12x12

Has the smallest file size of the ASTC texture formats.

Quality decreases sharply.

Added Threshold-Based Alpha Blending to the Layout Samples

Added the new Threshold-Based Alpha Blending feature to the sample data in LayoutEditor.

You can use Threshold-Based Alpha Blending as a kind of high-performance alpha test for finer control and smoother borders.
In addition, by applying this setting to a texture whose alpha channel has been converted to a distance field, you can also reduce the blurring around the borders of a texture that has been scaled.
(The alpha channel can be converted to a distance field using the NintendoWare Photoshop plug-in.)

Samples/Sources/Resources/Ui2d/thresholdingAlphaInterpolation/thresholdingAlphaInterpolation.flyt

 Not Using Threshold-Based Alpha Blending (Initial Setup)

The values entered for black color blending and white color blending affect the colors of the texture.

  • The black color blending alpha value affects the opacity of the transparent parts of the texture.
  • The white color blending alpha value affects the opacity of the opaque parts of the texture.
Using Threshold-Based Alpha Blending

The alpha values are stored, and the alpha values set for black color blending and white color blending act as threshold values.

  • The black color blending alpha value affects the opacity of the transparent parts of the texture.
    The specified number is taken as a threshold value, and all texture alpha values below this threshold are treated as transparent.
  • The white color blending alpha value affects the opacity of the opaque parts of the texture.
    The specified number is taken as a threshold value, and all texture alpha values above this threshold are treated as opaque.
