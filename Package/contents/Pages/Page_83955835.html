Features

Use the Import plug-in to import intermediate files to Maya and then create models, animations, and texture image files.

The Import plug-in is useful when you lose a Maya scene file from which intermediate files were exported, or when you want to check the contents of intermediate files in Maya.

The Import plug-in also supports batch-converting FMD files in a specified directory to Maya scene files (with the .ma and .mb filename extensions).

How to Use

Select Nintendo > Import to display the window.

Common Options

Magnify Specifies the multiplier applied to the vertex coordinates and translation values of the model.
Namespace

Specifies the namespace for the node being created.
If you leave this property blank, the node is created using the current namespace.
If you specify @file, the name of the intermediate file is applied to the namespace.
If you do not begin the entry with a colon (:), the specified namespace is created below the current namespace.
For example, if the current namespace is :model and you specify test, the node is created in the :model:test namespace. However, if you specify :test, the node is created in the :test namespace, regardless of the current namespace.
Click Browse to select an existing namespace or the name of the intermediate file specified by the FMD, FSK, or FSN file option.
Create Material

Select this option to create material nodes when FMD files are imported.
(shader / shadingEngine / materialInfo)
If the option is cleared, the default material (lambert1) is assigned to all shapes.
Material Type Specifies the node type for the material to create. (This feature is in NintendoSDK 13.3.0 and later.)
By default, this type cannot be changed from blinn (standard material), but you can select another material by specifying extended material attribute information (material_attr_info_????) in the config file of the Export plug-in.
However, currently you cannot select materials that cannot be created. If a plug-in is required for the material, load it before you run the Import plug-in (for Arnold, load mtoa.mll).
If the Create Material option is cleared, this option is disabled.
Create Texture Node

Select this option to create texture nodes when FMD files are imported.
(file / place2dTexture / uvChooser / layeredTexture / bump2d)
If the option is cleared, only the material color, disable-compression flag, and user data are applied.
If the Create Material option is cleared, this option is disabled.
Create Texture File

Enable this feature to create texture (TGA, PNG, DDS, or EXR) files from FTX files when importing FMD files.
If FTX files exist in the following folders, image files are created in the import destination sourceimages folder.

  1. (FMD file folder)
  2. (fmd file folder)/textures


Disable this option if you cannot locate the FTX files referenced by the FMD file.

If there are no FTX files, but image files (in TGA, PNG, DDS, or EXR format) exist in the following folders, the paths of those image files are set for the names of the images of the file node, regardless of the value set for this option.

  1. (FMD file folder)
  2. (fmd file folder)/textures
  3. sourceimages folder at the import destination


This option is disabled if Create Material or Create Texture Node is cleared.

If the FMD file references an external material (an external FMT file), the FTX files (the image files) in the following folders are used for the textures referenced by that FMT file.

  1. (Folder for FMT files)
  2. (Folder for FMT files)/textures
LDR

Specifies the file type when creating LDR image files from integer-type FTX files.

If Auto is specified, the file is created with the same extension as the value of the src_path_r attribute of the FTX file. (If the src_path_r extension is something other than TGA, PNG, or DDS, a TGA file is created.)
HDR

Specifies the file type when creating HDR image files from real data type FTX files.

If Auto is specified, the file is created with the same extension as the value of the src_path_r attribute of the FTX file. (If the src_path_r extension is something other than DDS, EXR, or PNG, a DDS file is created.)
Import Tab Select to import the intermediate file to the current scene (normal import).
Batch Convert Tab

Select to convert all FMD files in a specified folder to Maya scene files (batch conversion).

Normal Import

FMD File Specifies the path of the FMD file to import.
FSK File Specifies the path of the FSK file to import. If left blank, nothing will be imported.
FSN File Specifies the path of the FSN file to import. If left blank, nothing will be imported.

Click the Import All button to import all of the specified intermediate files.
The animations in the FSK files are applied to the nodes created from the FMD files.
If you specify an FSK file but do not specify an FMD file, the plug-in returns an error.

Click the Import fmd button to import FMD files only.

Click the Import fsk button to import skeletal animations to the nodes in the current scene.

If there are several nodes with the same name in the scene, the nodes to which to apply the animation are prioritized as follows.

  1. Nodes with the same name under the selected node
  2. Nodes with the same name (except for the namespace) under the selected node
  3. Nodes with the same name specified by the Namespace option
    (If that option is empty, nodes with the same name as the current namespace)
  4. Nodes with the same name as the root namespace
  5. Nodes with the same name as another namespace.

Click the Import fsn button to import FSN files only.

When importing FSN files, the animations in those files are applied to any cameras, lights, and fogs in the current scene that have the same names as the animations. If there are no such cameras, lights, or fogs, new ones are created and the animations are applied to them.

Batch Conversion

Input Folder Specifies the input folder for batch-converting files.
Output Folder Specifies the folder to which to export batch-converted files.
File Type Specifies the type of scene file to which to batch-convert the intermediate files.

Click the Convert button to start batch-converting intermediate files.

A save confirmation message is displayed if the current scene is modified. Note that a new scene will be opened after the batch conversion process is complete.

If Create Texture File is selected, the FTX files used by the FMD files are also converted.

The Maya project tree is created with the same folder structure as the input folder and exported to the output folder.

For example, if you specified the following Intermediate folder as your input folder, the following Converted output folder would be exported.

Intermediate/                      ->      Converted/
  + Object/                                  + Object/
      + Enemy1/                                  + Enemy1/
          + body1.fmdb                               + scenes/
          + weapon1.fmdb                                 + body1.ma
          + textures/                                    + weapon1.ma
              + metal1.ftxb                          + sourceimages/
  + Stage/                                               + metal1.tga
      + Stage1/                              + Stage/
          + Room1/                               + Stage1/
              + stage1_1.fmdb                        + Room1/
              + textures/                                + scenes/
                  + block1.ftxb                              + stage1_1.ma
                                                         + sourceimages/
                                                             + block1.tga

Important Information

General Considerations

  • Information that is not contained in the intermediate files is not imported. This includes information about IK, constraints, and expressions.
  • If a bone with a skin shape is the root bone and has child bones, or if a bone with a skin shape influences skinning, the skin shape will be created as a separate object under the world object.
Ex: Intermediate file bone structure      ->     DAG node structure after importing
joint1 (shape)                             joint1          
  + joint2                                   + joint2
      + joint3                                   + joint3
                                           pPlane1 (shape)
(shape name = pPlane1__blinn1)
  • Triangles that use the same vertex two or more times and triangles with a geometry region of zero (all three vertices are on the same line) are deleted when you import the intermediate files.
  • Vertex normals in the FMD file that differ from Maya's automatic calculations are locked.
    Depending on the shape, the normals might be locked even if the FMD file you import was exported using Maya's automatically calculated normals.

Important Information About Materials

  • If normal import is used, the texture image files are created in the sourceimages folder of the current project.
  • The location where the file node is connected is determined using sampler hint information in the FMD file.
    However, the node is not connected to anywhere if the material is a standard material and the hint information is tangent.
    The node is connected to the material's translucence depth attribute if the material is a standard material and the hint information is neither albedo, opacity, emission, normal, tangent, specular, nor reflection.
    If the material is an extended material and hint information is not in the attribute information, the node is connected to the attribute that has extra_default as the attribute information flag, if such an attribute exists.
  • If the material is an extended material and there is no import destination attribute, a custom attribute is added automatically. The default is float3 type (a compound type), but if the attribute information flag is float1, a float type (a single value) attribute is added.
    However, an attribute is not added if the FMD file has neither a sampler element nor an original color element that matches the hint information.
  • If the FMD file references external materials, the FMT files that are in the following folders are imported.
    1. (Folder for FMD files)
    2. (Folder for FMD files)/Materials
    (The Materials folder is supported in NintendoSDK 13.0.0 and later.)
  • If the material name is lambert1, it is imported to the default lambert1 node.
    In Maya 2020 and later, if the material name is standardSurface1, it is imported to the default standardSurface1 node. (However, an error occurs if material_attr_info_standardSurface is not specified in the config file.)

Important Information About Animations

  • If you import only FSK files, or import FSN files, the graph editor closes automatically if it is currently displayed.
    In Maya 2018, importing FSK or FSN files while a key is displayed in the graph editor and then displaying the graph editor after the import sometimes causes Maya to crash. We recommend manually saving the scene after importing.
