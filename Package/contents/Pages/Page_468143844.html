3DEditor

Hints can now be set in the Samplers page only when sampler editing restrictions have been cleared.

You can now set hints for samplers only when the 3DEditor sampler editing restrictions have been cleared.

Use the following procedure to disable editing restrictions.

In the Team Settings presets, set Use sampler names that conform to the DCC tool's naming conventions to Yes.
This causes a button for disabling editing restrictions to be displayed in the Sampler property window. Click the button to disable editing restrictions.

Photoshop Plug-In

You can now export DDS files from PSD files with 32 bits per channel.

In the Texture Exporter for Photoshop plug-in, you can now export DDS file from PSD files with 32 bits per channel.

If the image mode is 32 bits per channel, you can now select Output DDS from the Nintendo Export Texture Settings menu.

By default, the files are exported as DDS files, but you can change this and export them as EXR files by specifying the following line in the config file. 

var hdr_image_file_type = "exr";

Maya Plug-In

You can now use a post-export script with the LOD Export option of the Maya plug-in to get the name of the DAG path to the LOD group root.

Added a feature to the LOD Export option of the Maya plug-in for getting the name of the DAG path to the LOD group root using a post-export script.
You can use this feature to get the name of the DAG path to the LOD group root from the NINTENDO_EXPORT_LOD_GROUP_ROOT_PATH environment variable.

Related Links For more information, see the following pages.

You can now use a post-export script with the Maya plug-in to get the exported transform nodes.

Added a procedure for using a post-export script to get the group of transform (joint) nodes exported as the <bone> elements in the intermediate file.
string[] NintendoGetExportedTransformPaths()
The returned value is a DAG path array.
Returned in ascending order of <bone> indexes.
The array also includes nodes corresponding to deleted <bone> elements due to bone compression.

When LOD is exported, the group of transform (joint) nodes for the base model is returned.

3D Command-Line Tools

Modified the exit code values for 3dBinaryConverter.exe and 3dShaderConverter.exe.

The exit code values for 3D Binary Converter and 3D Shader Converter have been modified.
Previously, the two exit code values were 0 for success, and 1 for failure. Starting in SDK 8.0.0, the exit code for success is still 0, but the exit code for failure is an integer value of 1 or greater.

As of version 8.0.0, the exit code values are private. However, they will be published in a future release.

If you have batch processes or similar that are designed to make decisions based on whether the exit code is 1 or not, you must revise those processes accordingly.

Configuring packed vertex attributes to be included in different vertex buffers now causes a conversion error.

3D Shader Converter now displays a conversion error when creating a shader definition file (FSDA or FSDB) from a shader configuration file (FSCA) if multiple vertex attributes that have been specified for packing in shader annotations have been configured to be included in different vertex buffers.

Consider the following example, where _u0 and _u1 are packed using the following settings.

layout(location = 8, component=0) in vec2   i_UV0;          // @@ id="_u0"   hint="uv0"
layout(location = 8, component=2) in vec2   i_UV1;          // @@ id="_u1"   hint="uv1"

Setting these vertex attributes to be included in different vertex buffers as follows now causes a conversion error.

/* @@
interleave="_u0"
interleave="_u1"
*/

The vertex attributes to be packed must instead be included in the same vertex buffer, as follows.

/* @@
interleave="_u0 _u1"
*/
