Blending

Overview

This page describes the blending feature for skeletal animations.
The blending feature allows multiple skeletal animations to be applied simultaneously to a model.

The blending feature allows you to choose between two blending methods: interpolated blending and additive blending.
For interpolated blending, you can also choose between linear interpolation and spherical linear interpolation.

Blending Methods

Interpolated Blending

Interpolated blending is a blending method that only uses the ratio for specified blending weights for multiple skeletal animations.
Use it to represent animations between each of the skeletal animations.

For example, if you blend a running animation with a jumping animation at ratios of 0.5 each, you can create an animation of jumping while running.
These blending results are applied to models to allow you to use that animation of jumping while running.
You can change the blending weight value to adjust the blending result animation.

Running Animation + Jumping Animation = Blending Result



Info

If the total blending weight value is a value other than 1, the Scale and Translation components are normalized and blended such that the total blending weight becomes 1. The Rotate component is not normalized.

Additive Blending

Additive blending enables you to add the difference from two different skeletal animations to skeletal animations or the blending results for skeletal animations.

Those two different skeletal animations are referred to as the ReferenceAnimation, or Ref, and the SourceAnimation, or Src. The difference between them is referred to as the DiffAnimation, or Diff.
Conceptually, the DiffAnimation is equal to Src minus Ref, and the SourceAnimation is equal to Ref plus Diff.

DiffAnimations can be used to apply a variety of states and effects to other unrelated animations. These unrelated animations are referred to as TargetAnimations, or Targets.

For example, you can create a DiffAnimation for a tired state by finding the difference between a walking-while-tired SourceAnimation and a walking ReferenceAnimation.
When you add the tired DiffAnimation to a running TargetAnimation, you can create a running-while-tired animation.

By applying a variety of DiffAnimations at various times, you can also add noise to an animation, add randomness to a completely repeating cycle, or overlap reactions.


How to Use

Interpolated Blending


  1. Prepare a nn::g3d::ModelObj instance (to which you will apply a skeletal animation) and several nn::g3d::ShapeAnimObj instances, using Skeletal Animation through to step 3.

    // The instance of the model to apply a skeletal animation to.
nn::g3d::ModelObj modelObj;


// Instances of blend target skeletal animations.
nn::g3d::SkeletalAnimObj skeletalAnimObjA;
nn::g3d::SkeletalAnimObj skeletalAnimObjB;

  2. Use nn::g3d::SkeletalAnimBlender::Builder to build nn::g3d::SkeletalAnimBlender.

    nn::mem::StandardAllocator standardAllocator; // The allocator.
nn::g3d::SkeletalAnimBlender skeletalAnimBlender // The instance of the skeletal animation blender.

//----------------------------------------------------------------------------------------------------------------------------------


// The variable for building the skeletal animation blender instance.
nn::g3d::SkeletalAnimBlender::Builder builder;


// Set the model resource to be animated.
// Set the parameters required for blending.
builder.Reserve(modelObj.GetResource());


// Calculate the memory size required to build the instance.
builder.CalculateMemorySize();


// Get the memory size required to build the instance.
size_t workMemorySize = builder.GetWorkMemorySize();


// Reserve the memory required to build the instance.
void* pWorkMemory = standardAllocator.Allocate(workMemorySize, nn::g3d::SkeletalAnimBlender::Alignment_Buffer);

// Build the instance.
bool isSucceeded = builder.Build(&skeletalAnimBlender, pWorkMemory, workMemorySize);
NN_ASSERT(isSuceeded);

  3. Set the skeletal animation instance and the blending ratio, or blend weight, you want to blend for the nn::g3d::SkeletalAnimBlender you built.
    After setting the interpolated blending flag and posture interpolation method flag using nn::g3d::SkeletalAnimBlender::BeginBlend(), run nn::g3d::SkeletalAnimBlender::Blend() to execute the blending process. Then run nn::g3d::SkeletalAnimBlender::EndBlend() to end the blending process.

    // Reset the total blend weight value.
skeletalAnimBlender.ClearResult();

// Declare the start of blending and specify interpolated blending.
// At the same time, specify the attitude interpolation method. Linear interpolation is selected when nothing is specified.
skeletalAnimBlender.BeginBlend(nn::g3d::SkeletalAnimBlender::BlendMode::BlendMode_Interpolate
    || nn::g3d::SkeletalAnimBlender::InterpolateMode::InterpolateMode_Lerp);

// Set the blending weight, and blend the animation instance.
skeletalAnimBlender.Blend(&skeletalAnimObjA, 0.75);
skeletalAnimBlender.Blend(&skeletalAnimObjB, 0.25);

// End the blending.
skeletalAnimBlender.EndBlend();
    Info

    If the total blend weight value does not equal 1, blend results are normalized in either the nn::g3d::SkeletalAnimBlender::EndBlend() or the nn::g3d::SkeletalAnimBlender::ApplyTo() function. However, this normalization process is costly, so we recommend making the total blend weight value 1 in the application.

    When nn::g3d::SkeletalAnimBlender::BeginBlend() is omitted, interpolated blending is selected as the blending method and linear interpolation is selected as the attitude interpolation method.
    As a result, you can get the same results in the preceding code even when omitting BeginBlend() and EndBlend().



  4. Apply the blending results to the skeleton of the model instance.

    // Apply the animation blending result to the model instance's skeleton.
skeletalAnimBlender.ApplyTo(modelObj.GetSkeleton());

// Update the animation instance frame.
skeletalAnimObjA.GetFrameCtrl().UpdateFrame();
skeletalAnimObjB.GetFrameCtrl().UpdateFrame();

Additive Blending


  1. Prepare a nn::g3d::ModelObj instance (to which you will apply a skeletal animation) and two nn::g3d::SkeletalAnimObj instances to use as the SourceAnimation, or Src, and ReferenceAnimation, or Ref, using Skeletal Animation through to step 3.

    // The instance of the model to apply a skeletal animation to.
nn::g3d::ModelObj modelObj;

// Instances of blend target skeletal animations.
nn::g3d::SkeletalAnimObj srcSkeletalAnimObj; // The SourceAnimation instance.
nn::g3d::SkeletalAnimObj refSkeletalAnimObj; // The ReferenceAnimation instance.



  2. Use nn::g3d::SkeletalAnimBlender::Builder to build nn::g3d::SkeletalAnimBlender. The procedure is the same as step 2 when using interpolated blending.

  3. Set the SourceAnimation, ReferenceAnimation, and blend weight you want to use for the nn::g3d::SkeletalAnimBlender instance you built.
    After setting the additive blending flag and attitude interpolation method flag using nn::g3d::SkeletalAnimBlender::BeginBlend(), run nn::g3d::SkeletalAnimBlender::Blend() to execute the blending process. Then run nn::g3d::SkeletalAnimBlender::EndBlend() to end the blending process.

    // Reset the total blend weight value.
skeletalAnimBlender.ClearResult();

// Declare the start of blending and specify additive blending.
// At the same time, specify the attitude interpolation method. Linear interpolation is selected when nothing is specified.
skeletalAnimBlender.BeginBlend(nn::g3d::SkeletalAnimBlender::BlendMode::BlendMode_Add)
    || nn::g3d::SkeletalAnimBlender::InterpolateMode::InterpolateMode_Lerp);

// Calculate a value that multiplies the blend weight by the DiffAnimation calculated from the SourceAnimation and ReferenceAnimation.
skeletalAnimBlender.Blend(&srcSkeletalAnimObj, &refSkeletalAnimObj, 0.75);

// End the blending.
skeletalAnimBlender.EndBlend();

  4. Apply the blending results to the skeleton of the model instance.

    // Apply the animation blending result to the model instance's skeleton.
skeletalAnimBlender.ApplyTo(modelObj.GetSkeleton());

// Update the animation instance frame.
srcSkeletalAnimObj.GetFrameCtrl().UpdateFrame();
refSkeletalAnimObj.GetFrameCtrl().UpdateFrame();
