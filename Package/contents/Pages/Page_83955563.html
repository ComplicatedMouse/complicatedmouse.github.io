TimerEvent Class

Feature Overview

The TimerEvent class is a feature that uses a single bit (two values) to represent a signaled or non-signaled state as a way to synchronize between threads. This feature is the same as the standard event feature, with the addition of a timer function that can be used to specify a delay before putting the event in a signaled state. However, it does not include a timeout.

The TimerEvent class wraps the Timer Event Features using the nn::os::TimerEventType object and contains the following member functions. For more information, see the API reference.

Name Description Remarks
TimerEvent Constructor Executes initialization equivalent to InitializeTimerEvent.
~TimerEvent Destructor Executes destruction equivalent to FinalizeTimerEvent.
StartOneShot Starts the one-shot timer Starts the one-shot timer equivalent to StartOneShotTimerEvent.
StartPeriodic Starts the periodic timer Starts the periodic timer equivalent to StartPeriodicTimerEvent.
Stop Stops the timers Stops the timers equivalent to StopTimerEvent.
Wait Waits for a timer event Waits for a timer event equivalent to WaitTimerEvent.
TryWait Timer event polling Timer event polling equivalent to TryWaitTimerEvent.

Timer event is set as a signaled state equivalent to SignalTimerEvent.

Examples of Using the TimerEvent Class

This example implements an alarm handler using the TimerEvent class.

#include <nn/os.h>

nn::os::TimerEvent g_SimpleAlarm(nn::os::EventClearMode_AutoClear);

void SimplAlarmThread()
{
    for (;;)
    {
        g_SimpleAlarm.Wait();
        CallbackFunction();
    }
}

void MainThread()
{
             :
    // Starts the alarm at the required timing (500 msec)
    g_SimpleAlarm.StartOneShot(nn::TimeSpan::FromMilliSeconds(500));
             :
    // Stops the alarm at the required timing
    g_SimpleAlarm.Stop();
             :
}

The above sample is an alarm feature sample that waits for the signal state of a g_SimpleAlarm instance then calls a callback function. Because SimpleAlarmThread is available as a function for calling the callback function, a single worker thread is prepared, and SimpleAlarmThread is then called from it.

The MainThread function starts and stops timer operation for the g_SimpleAlarm instance. Wait in the SimpleAlarmThread function waits for g_SimpleAlarm to become a signaled state. g_SimpleAlarm changes to a signal state after a specified amount of time (500 milliseconds in the preceding example) passes after calling the StartOneShot function. After g_SimpleAlarm becomes a signaled state, SimpleAlarmThread is released from its Wait, calling CallbackFunction.

StartOneShot can be called any number of times while the timer is operating. Because the time to make g_SimpleAlarm into a signal is updated in this way, this can be used as a watch dog feature. In addition, CallbackFunction can be called as a periodic handler if the StartPeriodic function is used.

To stop the timer operation, call the Stop function.
