When EDEV Initialization Fails

Factors such as the EDEV status can cause initialization to fail.

  • Initialization fails.
  • The EDEV console you want to initialize does not appear (or it no longer appears) on the EDEV connection list in InitializeEdevWin.

If this happens, verify the following.

Using InitializeEdev.exe (or InitializeEdevWin.exe)

We recommend directly calling InitializeEdev.exe or InitializeEdevWin.exe.

The following files also call these functions directly.

  • %NINTENDO_SDK_ROOT\InitializeEdev.bat
  • %NINTENDO_SDK_ROOT\Tools\CommandLineTools\DevKitSetup.bat
  • %NINTENDO_SDK_ROOT\Tools\CommandLineTools\DevKitSetup.ps1

Verifying the Specified Parameters

If you are using the InitializeEdev.exe command-line tool, verify that the specified parameters match. (The serial number works with or without a check digit.)

(When Downgrading the Version) Verifying Whether the Version Has Restrictions

When using InitializeEdev.exe included with an earlier SDK version to initialize an EDEV that was initialized with InitializeEdev.exe included with a new SDK version, in some cases depending on the version, initialization may fail or safe mode must be entered manually.

The following earlier versions have these constraints.

  • Versions 2.2.0-0 and 2.2.0-1 of the firmware
  • All versions earlier than NX Add-On 1.5.0 (with firmware version 2.1.0-0)

Repeating Initialization

Sometimes initializing the EDEV several times is successful. Try reattempting initialization.

Fully Charging the EDEV Console

Try initialization after fully charging the EDEV console.

To charge the device you must attach the AC adapter. You cannot charge simply by connecting the device to a PC with a USB cable.

Reconnecting the USB Cable and Turning On the Power

Try initializing the EDEV console again after reconnecting the USB cable and turning the power on.

If initialization is particularly unsuccessful, reconnect the USB cable with the following procedure.

  1. Remove the USB cable from the EDEV system.
  2. Hold down (for approximately 12 seconds) the POWER Button on the upper left of the EDEV console to turn off the power.
  3. Connect the USB cable to the EDEV console while the power is off.
  4. After the console is connected, it automatically turns on. Try initialization again.

If you are using the device connected to a relay box, try connecting it directly to the PC without using the relay box.

Exiting Target Manager if It Is Running

Exit the Target Manager if it is running, and then try to initialize EDEV.

Sometimes the Target Manager stays in the system tray, so also check there.

Verifying That the EDEV Console Is Recognized by the Device Manager

Verify that the EDEV is recognized as a USB device in Device Manager.
For more information, see the EDEV Drivers section of Initializing the EDEV.

Manually Activating Safe Mode

Sometimes initialization is successful after activating safe mode.

Info

Safe mode is a mode that runs with minimum features.

Use safe mode in the following situations.

  • The console is not operating properly and must be restored.
  • EDEV initialization fails.
  • The power is on but the system menu does not appear.

Safe mode can be activated with user operations.

After safe mode is activated, you may not be able to start in normal mode until the EDEV is initialized, so we recommend using safe mode only when you must restore or initialize.

Procedures for Entering Safe Mode

There are two ways to enter safe mode.

[Method 1] With Joy-Con (R) Attached

We recommend this method.

  1. Hold down the POWER Button (for approximately 12 seconds) and turn off the EDEV console with the USB cable connected. (The system will restart after the button is held down for 20 seconds, so do not go much longer than 12 seconds.)
  2. Attach the right Joy-Con to the EDEV console. (Do not attach the left Joy-Con.)
  3. Press the POWER Button while holding down the volume down button.
  4. To check whether safe mode is running, see Method for Verifying Whether the System Has Entered Safe Mode. If the system did not enter safe mode correctly, restart the process from step 1.

[Method 2] Without Joy-Con Attached

  1. Hold down the POWER Button (for approximately 12 seconds) and turn off the EDEV console with the USB cable connected. (The system will restart after the button is held down for 20 seconds, so do not go much longer than 12 seconds.)
  2. Press the buttons on the upper left of the console in the following order.
    1. Press the POWER Button while holding down the volume down button.
    2. While holding down the POWER Button, release the volume down button and press the volume up button.
    3. While holding down the volume up button, release the POWER Button.
    4. Release the volume up button.
  3. To check whether safe mode is running, see Method for Verifying Whether the System Has Entered Safe Mode. If the system did not enter safe mode correctly, restart the process from step 1.
Info

For cases where the system enters the standard startup sequence and displays the menu despite attempting to enter safe mode, sometimes adjusting the intervals between button operations will fix the issue. (This is because some operations may not be detected if the intervals between button operations is too fast.)

Use "every 0.6 second" as a guide for the intervals. It is equivalent to "4 beat intervals at 100 BPM." If you search for a video with "BPM=100," you should be able to find a video that you can use as a guide.

Verifying Whether the System Has Entered Safe Mode

Depending on the firmware version, there are various methods for verifying whether the system has entered safe mode.

  1. Press the Re-Scan button to display the systems that are in safe mode "(Safe Mode)" in the list of connected systems. (However, only do this when using InitializeEdevWin.exe.)
  2. "NX DevKit Safe Mode" is displayed for four seconds on the EDEV console screen. (However, this happens only when using an EDEV initialized from NX Add-On 3.0.0 and later.)
  3. The EDEV console screen remains dark. (However, this happens only when using an EDEV initialized with NX Add-On 1.X or earlier.)
    1. This screen may momentarily light up white.

When using an EDEV console initialized prior to NX Add-On 1.0, the following "screen may turn red during safe mode operations" situation may occur.


Note

Depending on the SDK version, there are cases where InitializeEdev is not detected even when entering safe mode manually.
The fix for this issue was implemented in the following SDK versions, so we recommend using these versions to run safe mode.

  • NX Add-On 0.12x series versions 0.12.17 and later (firmware versions 1.0.0-21 and later)
  • NX Add-On 1.x series versions 1.4.0 and later (firmware versions 2.0.0-16 and later)
  • NX Add-On 3.0.0 SS1 and later


Info

When using an EDEV console that has not been initialized after NX Add-On 1.0, performing the operations to enter safe mode may cause the screen to turn red. If this state occurs, hold down the POWER Button for 12 seconds, and then press the POWER Button again to start safe mode.


Info

To enter safe mode on an EDEV console with NX Add-On 0.10.0 or earlier (the firmware displayed in the DevMenu will be 0.7.0-2 or earlier), it must be the only EDEV console connected to the PC.

If any other EDEV consoles are connected to the PC, remove them.

Using USB 2.0 When USB 3.0 Connection Fails

EDEV setup may fail depending on the compatibility with the USB 3.0 controller.
Try the following.

  • Update the USB 3.0 controller driver to the latest version.
  • Update the USB 3.0 controller firmware to the latest version (sometimes this is not publicly available).
  • Perform setup by connecting the EDEV to a USB 2.0 port.

Setup has been confirmed to fail when using a PC with the following USB 3.0 controller.

  • Renesas Electronics USB 3.0 Host Controller

Reducing the Number of EDEV Connected to the PC to One

EDEV consoles initialized with NX Add-On 0.10.0 or earlier (the firmware displayed in the DevMenu will be 0.7.0-2 or earlier) may fail if other EDEV consoles are connected to the PC.
If this occurs, remove all but one of the EDEV consoles, restart the remaining console, and then reinitialize it.

(Note that this step is unnecessary with systems initialized with firmware later than NX Add-On 0.10.0.)

Contacting Support

If initialization is unsuccessful after attempting these procedures, contact Nintendo support.

When contacting support, the process will be smoother if you include the following information.

  • Serial number of the EDEV unit
  • SDK version
  • Initialization log
    • For InitializeEdevWin, copy the log that is output to the log window to a text file and send in the text file.
    • For InitializeEdev, execute a log with the --verbose option added and save it to a file.
