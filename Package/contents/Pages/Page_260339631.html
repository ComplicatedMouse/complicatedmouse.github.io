Added a capture texture feature.

Added a feature to capture the results of framebuffer and layout rendering to use them as a texture.

When the feature is enabled in the project settings file, select Pane > Create Pane from the main menu, and then select Capture Pane.
When Capture Pane is selected, Capture Pane is created in the Hierarchy window.
Pane rendering is not normally performed for the Capture Pane hierarchy in the Hierarchy window.
The capture pane is only rendered in the generated texture.

The texture created by the capture pane is displayed in the Texture window, delineated from the normal texture by a █ yellow stripe.
As with a normal texture, this texture can be used after being registered in the pane Properties Window > Texture.

Capture Pane Properties

A

Set whether to load the framebuffer in the █Capture Area in the Layout window.
B

Set whether to generate the texture just once in only the initial frame.
Set to OFF when you want to animate the pane of the Capture Pane hierarchy.
C Maximizes the resolution of the capture pane and generates a texture by applying the scale at the specified ratio.
D

Sets the texture color in the following cases.

  • When the framebuffer capture is not performed
  • When the capture area exceeds the texture size of the framebuffer

In particular, setting the alpha value to 0 also sets initial alpha value to 0 for the generated texture.
As a result, the transparent part at the outside of the pane in the Capture Pane hierarchy can be made transparent in the generated texture.

The capture pane can capture a specific area in the framebuffer as a texture.
The framebuffer of a game application can be captured by setting FramebufferCaptureTexture as the material.
FramebufferCaptureTexture is always displayed in the texture selection list of the material.

Detailed settings like in the capture pane cannot be set with FramebufferCaptureTexture.
It captures the framebuffer texture in the area of the pane using a material.

It is possible to pseudo-verify capture behavior because setting a background image in LayoutEditor causes it to be treated as a framebuffer texture.

This method of use is as follows.

Info

[Restrictions on parent/child relationships]
You cannot create parent/child relationship structures that cause dependencies to occur in the capture texture.
Examples include structures that place picture panes in the children of the capture pane that contain textures created by the capture pane.

[Rotation and scaling restrictions]
If rotation or scaling are configured for the capture texture, you cannot perform capture following the shape of the pane.
