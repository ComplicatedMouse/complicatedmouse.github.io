Downloadable Content Terminology

Downloadable Content Terminology

Category Name Description
Metadata for downloadable content Downloadable content index

A value between 1 and 2000. It is used for identifying downloadable content.
Release Version

A 16-bit value representing the release version of the downloadable content.
For the initial release, this is set to 0. For each subsequent update that is released, the value is incremented by 1.


This provides a way for the system to detect updates to the downloadable content.
Application ID

A 64-bit value that identifies the application that uses the downloadable content.
Required application release version

A 16-bit value that is greater than 0.

This value is the application release version required before you can use the downloadable content.

The application release version specified in the application ID must be the same or later than the required application release version.


For firmware versions in NX Add-On 9.x and earlier, if the application did not meet the version condition, it would simply not start.

In NX Add-On 10.x and later, to decrease the occurrence of applications not starting, applications were allowed to start even if this version condition was not met.

However, any downloadable content that does not meet the required application version will be not available to the application. (The application treats the downloadable content as if it does not exist.)
Tags

Any string describing the content of the downloadable content.

The maximum number of characters that can be specified is 128.

When setting the price for the downloadable content, you must select the downloadable content that is the operation target. Tag information is shown at the same time as reference information.

We recommend setting a string that makes it possible to guess the content of the downloadable content.

Duplicate values must not be specified in the same NMETA file.

You can only use the character codes supported by the Japan, US, and Europe fonts, and the Korean, Simplified Chinese, and Traditional Chinese fonts described in Shared Data > Internal Fonts in the System Features Guide.

Info

Korean, Simplified Chinese, and Traditional Chinese fonts can be configured in NX Add-On 7.2.0 and later.
