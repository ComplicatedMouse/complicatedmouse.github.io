NintendoWare includes platform-specific information.

Although common packages are not supposed to depend on specific platforms, NintendoWare includes some platform-specific implementations, documentation, and samples.
These resources will be removed from the common package in future releases.

Note that the information about these resources is hard to use and understand in some parts of the package.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer. If you try to use it, you cannot enter anything in the text box, making search impossible.

Use a web browser to use the Find Result page.

Audio-Related

Exporting sound waveforms does not work in NintendoWare SoundMaker and SoundProjectWaveExporter.

Attempts to export sound waveforms using the NintendoWare tools SoundMaker and SoundProjectWaveExporter always result in the creation of blank waveform files.

This problem is expected to be fixed by the next release.

In NintendoWare's SoundMaker and WaveConverter, you can specify the PCM8 compression format, which is not supposed to be available.

Currently, 8-bit PCM is not supported by nn::audio, but you can specify the PCM8 compression format in SoundMaker and WaveConverter.
Note that when PCM8 is specified and an attempt is made to playback that format using either the nn::atk or nw::snd library, that attempt always fails.

We are planning to make PCM8 unspecifiable in SoundMaker and WaveConverter in the next release.

When you convert a sound project imported using NintendoWare's SoundMaker, an invalid sound archive is exported.

When, in SoundMaker, an NW4C/NW4F sound project (one for which the cache file remains for converting purposes) is imported and (normally) converted, NW4C/NW4F binary source files become mixed in with the NintendoSDK binary sound archive (BFSAR) files. As a result, they cannot be correctly read during the nn::atk and nw::snd runtimes. This is a problem.

The workaround is to make sure that you reconvert the sound project after importing it. This makes the old cache file unusable, which avoids the problem.

NintendoWare's SoundMaker cannot start when there are no available playback devices.

If SoundMaker attempts to start when there are no available playback devices on the PC, an exception is thrown internally, and the program halts. The workaround is to have a playback device available.

The problem that results in the exception being thrown is expected to be fixed in either the next release or the one after that.

An exception is thrown when you use nn::audio::AudioRenderer when there are no available playback devices.

If you attempt to use nn::audio::AudioRenderer when there are no available playback devices on the PC, an exception is thrown within the library, and the program halts. The workaround is to have a playback device available.

The problem that results in the exception being thrown is expected to be fixed in either the next release or the one after that.

Building the Nw4f/ut samples results in link errors.

Currently, if you build the FsFileStream and SharedPtr samples in Samples\Applications\Nw4f\ut, link errors result.

As a workaround, as was done in the RuntimeTypeInfo sample, replace the NwDemoMain function with the main function.

Attempting to exit a sample with the Close Button (the x Button) does not result in a clean exit.

When you use the Close Button (the x Button) to exit a gfx, g3d, vfx, ui2d, or atk sample, it does not exit cleanly. When this happens, a massive log is output.

When samples are built, there is a warning to the effect that PDB files cannot be found.

The following warning may appear.

warning LNK4099: The PDB file, 'libnn_XXX.pdb' was not found in 'libnn_XXX.lib(XXX.obj)' or 'C:\XXX.pdb'. Will be linked to object without debugging information.
