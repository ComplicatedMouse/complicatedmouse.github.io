Audio-Related

Exporting sound waveforms does not work in NintendoWare SoundMaker and SoundProjectWaveExporter.

Attempts to export sound waveforms using the NintendoWare tools SoundMaker and SoundProjectWaveExporter always result in the creation of blank waveform files.

This problem will be corrected in a future release.

When you convert a sound project imported using NintendoWare's SoundMaker, an invalid sound archive is exported.

When, in SoundMaker, an NW4C/NW4F sound project (one for which the cache file remains for converting purposes) is imported and (normally) converted, NW4C/NW4F binary source files become mixed in with the NintendoSDK binary sound archive (BFSAR) files. As a result, they cannot be correctly read during the nn::atk and nw::snd runtimes. This is a problem.

The workaround is to make sure that you reconvert the sound project after importing it. This makes the old cache file unusable, which avoids the problem.

Sometimes effects are not processed in the order they were added in nn::audio.

The intended specification is for effects to be processed in the order in which they were added to SubMix or FinalMix. However, if you have two or more effects registered to a single SubMix or FinalMix, and you then delete an effect and add another new effect, the effects sometimes run in the wrong order.

You can avoid this problem by deleting all effects from SubMix and FinalMix and then adding the effects.

We plan to fix this problem in a future release.

When I use nn::atk::viewer, a MemoryPool warning is displayed.

nn::atk::viewer does not currently support nn::audio::MemoryPoolType. As a result, if you use nn::atk::viewer to set wave sounds or sequence sounds as in-game editing targets and then play the sounds, the following warning message is displayed on the console.

[nn::audio] A memory pool has not been created. Using a memory pool is recommended.
[nn::audio] Please prepare nn::audio::MemoryPoolType. See the nn::audio documents.

This also happens in the AtkPlayer and AtkViewer samples. For more information about memory pools, see the section about memory pools in the AUDIO library manual.

We plan to fix this problem in a future release.

Finalizing nn::atk while an Opus stream sound is playing causes the process to stop with an assertion.

Finalizing nn::atk while an Opus stream sound is playing causes the process to stop with an assertion.

We plan to fix this problem in a future release.

Playing several nn::atk Opus stream sounds at the same time generates audio noise.

Playing several nn::atk Opus stream sounds at the same time generates audio noise. This bug can also be reproduced by doing the same thing in AtkPlayer.

We plan to fix this problem in a future release.

NintendoWare AtkPlayer can stop running if you change the DelayTimeMax value.

The application can stop running if you change the DelayTimeMax parameter for the Delay effect in the AtkPlayer Effect panel.

We plan to fix this problem in a future release.

Reloading a sound archive in NintendoWare AtkPlayer causes the application to crash.

After editing a stream sound in NintendoWare SoundMaker and converting the sound archive file, reloading the archive in AtkPlayer sometimes causes the application to crash.

We plan to fix this problem in a future release.

Switching the sound in NintendoWare AtkPlayer sometimes causes the application to freeze or crash.

While a sound is playing in the AtkPlayer SoundArchivePlayer panel, switching to another sound sometimes causes the application to freeze or crash. If the application freezes, try waiting a few moments for it to become responsive again. If the application crashes, restart the application. For now, this problem can be avoided by making sure no sounds are playing when you switch to another sound and by not switching between sounds too frequently.

We plan to fix this problem in a future release.

Finalizing the nn::atk library immediately after initialization causes the process to stop with an assertion.

Finalizing nn::atk immediately after initializing nn::atk::SoundSystem or nn::atk::SoundArchivePlayer causes the process to stop with an assertion. For now, this problem can be avoided by making sure to wait for a few moments after initialization before finalizing the library.

We plan to fix this problem in a future release.

The process for freeing nn::atk::SoundHeap objects sometimes takes a long time.

If you try to free a SoundHeap after loading a large amount of data into it, the deallocation process sometimes takes a long time to complete.

We plan to fix this problem in a future release.

In rare cases, the program crashes when playing stream sounds being edited using the in-game editing feature of the nn::atk::viewer library.

In rare cases, the following end error occurs and the program crashes when repeatedly playing stream sounds being edited using the in-game editing feature or when rapidly changing the stream sound parameters using the in-game editing feature.

Abort: 'delimiter != nullptr' in nn::fs::detail::GetMountName()

We plan to fix this problem in a future release.

FS Bugs

When nn::fs::ReadDirectory() is called for the root directory of a drive, sometimes the first entry is not listed.

There is a bug where sometimes the first entry is not listed when nn::fs::ReadDirectory() is called via nn::fs::MountHost()/MountHostRoot() immediately below a drive when the target is a type of removable media.

The PerfSimple sample exits with an assertion in certain PC environments.

This bug will be fixed in a future release.

3D Bugs

Changing the name of a model or animation in 3DEditor causes the names of objects to stop being updated correctly in the connected application.

Changing the name of a model or animation in 3DEditor while 3DEditor is connected to an application causes the names of objects to stop being updated correctly in the connected application.

We plan to fix this problem in a future release.

An invalid access event sometimes occurs when running the G3dDemo Town sample.

Depending on the environment used, an invalid access event sometimes occurs when running the Generic version of the G3dDemo Town sample.

This bug does not occur if you stop using nn::perf or change the number of models rendered. Further details about the cause of this bug are unknown at this time.

gfx Bugs

The nn::gfx::MemoryPool::InvalidateMappedRange function cannot be used in the OpenGL version of the library.

A linking failure occurs if you attempt to use the nn::gfx::MemoryPool::InvalidateMappedRange function of the OpenGL version of the library in a program.

This bug will be fixed in a subsequent release.

The alphabet list is not displayed correctly in functions such as API references.

In API references, under Namespace > Namespace Members, the alphabet lists displayed under All, Functions, and Enumerated Values do not display the correct lists when selected.

Pages at deep hierarchical levels disappear when the page tree is expanded.

Pages at deep hierarchical levels (at least 9 levels below the top page) in the page tree in the frame on the left side of the screen disappear from the tree when expanded. Any child pages beneath the affected page also disappear.

Sometimes there is no response when deleting an environment in Nintendo Dev Interface.

When the separately installed Nintendo TargetManager is running, environment deletion processing may stop running.

You can avoid this problem by closing Nintendo TargetManager.

When a Visual Studio project file is opened, you are prompted to upgrade the compiler and libraries.

When a project file not attached to a solution file, such as a Visual Studio project for an SDK library whose source code is publicly available, is opened in Visual Studio, a dialog box might be displayed prompting you to upgrade the compiler and libraries.

If you upgrade them, you will no longer be able to build in earlier versions of Visual Studio. Click Cancel and close the dialog box.
