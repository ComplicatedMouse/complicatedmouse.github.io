Six Axis Sensor (NX Add-On)

Features Added With NintendoSDK NX Add-On

Overview

With the introduction of the NintendoSDK NX Add-On, the following features have been added to SixAxisSensor.

The following play formats are added.

  • Joy-Con dual-controller grip operations (NpadStyleJoyDual)
  • Joy-Con (L) operations (NpadStyleJoyLeft)
  • Joy-Con (R) operations (NpadStyleJoyRight)
  • Handheld operations (NpadStyleHandheld)

Play Formats Available With NintendoSDK NX Add-On

Style of Operation NpadStyleSet Definition Overview
Dual-Controller Grip Play NpadStyleJoyDual

The play format that models Joy-Con dual-controller grip play.

Six-axis sensor input can be obtained from both the right and left controllers separately.

Call GetSixAxisSensorHandles() with the play format for the handle to get as NpadStyleJoyDual.

The SixAxisSensorHandle for Joy-Con (L) is stored in the output pointer pOutValues[0], and for Joy-Con(R) is stored in pOutValues[1].
Joy-Con (L) Play NpadStyleJoyLeft

The play format that models Joy-Con (L) play.

Six-axis sensor input can be obtained from the Joy-Con (L).

Call GetSixAxisSensorHandles() with the play format for the handle to get as NpadStyleJoyLeft.

SixAxisSensorHandle for Joy-Con (L) is stored in the output pointer pOutValues[0].
Joy-Con (R) Play NpadStyleJoyRight

The play format that models Joy-Con (R) play.

Six-axis sensor input can be obtained from the Joy-Con (R).

Call GetSixAxisSensorHandles() with the play format for the handle to get as NpadStyleJoyRight.

SixAxisSensorHandle for Joy-Con (R) is stored in the output pointer pOutValues[0].

Using the Six-Axis Sensor When a Joy-Con Is Not Installed on the System

If you want to use the six-axis sensor when a Joy-Con is not installed on the system, call the SetNpadHandheldActivationMode() function with the mode set to NpadHandheldActivationMode_None. That call enables the handheld operating style, which enables you to use the NpadStyleHandheld six-axis sensor feature.
