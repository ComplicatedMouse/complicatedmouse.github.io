Parent Material Reference Path

Overview

The parent material reference path is used to search for models containing the material shown in the dialog box for adding and changing parent materials.
If more than one parent material reference path has been set, models are searched in the order that the paths were set.

Specify a folder or file path for the parent material reference path. You can set the absolute path.
Path strings may include environment variables surrounded by percent signs (%). 3DEditor interprets such paths by replacing any environment variables surrounded by percent signs (%) with the value the environment variable represents.
You can use wildcard characters (* or ?) in the path.

You can filter the models displayed in the Material List by name.

Searching for Models That Have Parent Materials

In the dialog box for adding and changing parent materials, only filenames that do not include paths are saved for parent material models specified from team settings.

When opening parent material model with only a filename from a child material model, search for it using the same method as Additional Reference Path.

Replacing a Name Using Wildcard Characters

You can replace a portion of the name with a matching part of the path using wildcard characters.
In the replaced portion, specify \1, \2, \3... from the beginning of the path in the same order as the wildcard characters.

Info
<ParentMaterialPath Name="TownModel(\1)(\2)">%NINTENDO_SDK_ROOT%\Samples\Sources\Applications\G3dDemo\Resources\Town\*\*</ParentMaterialPath>

Add the preceding code to the <FileIo> team settings file. The following names will be displayed in the dialog box for adding parent materials.

TownModel(bg_WhiteTown)(bg_WhiteTown.fmdb)
TownModel(bg_WhiteTown)(bg_WhiteTown_cloth.fmdb)
TownModel(bg_WhiteTown)(sky_blueA.fmdb)
TownModel(Duck)(Duck.fmdb)
TownModel(FemaleA)(FemaleA.fmdb)
TownModel(Fish)(Fish.fmdb)
...
