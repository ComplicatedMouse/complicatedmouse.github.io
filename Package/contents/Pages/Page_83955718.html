Project File Format

Project File Format

Project files are stored in XML format.
This page describes components included in project files.

Sample Output

The following example shows output in project file format.

<?xml version="1.0" encoding="utf-8"?>
<Project Version="3.6.0.0">
    <file_info>
        <create user="user name" host="host name" date_time="2000-01-01T00:00:00+09:00" tool_name="3DEditor" tool_version="3.4.0" />
        <modify user="user name" host="host name" date_time="2000-01-01T00:00:00+09:00" tool_name="3DEditor" tool_version="3.6.0" />
    </file_info>
    <Scene PreviewDefaultAnimationSet="true">
        <Animation Name="Scene.fsnb" />
    </Scene>
    <Item Path="Human.fmdb" PreviewDefaultAnimationSet="false">
        <AnimationSet Name="run" Preview="true">
            <Animation Name="Human_run.fskb" ItemIndex="2" Preview="true" />
        </AnimationSet>
        <Animation Name="Human_walk.fskb" ItemIndex="3" Preview="true" />
        <Material>
            <Name>human_all</Name>
            <OptimizeShader>true</OptimizeShader>
            <OptimizeShaderOnReload>true</OptimizeShaderOnReload>
        </Material>
        <Preview ScaleX="2" ScaleY="2" ScaleZ="2" RotateX="0" RotateY="0" RotateZ="0" TranslateX="0" TranslateY="0" TranslateZ="0" Visible="true">
    </Item>
    <Item Relative="SearchPath" Base="Textures" Path="HumanA.ftxb" />
    <Item Relative="Project" Path="Human_run.fskb" />
    <Item Relative="Project" Path="Human_walk.fskb" />
    <Item Relative="Project" Path="../shaders/shader.fsdb" />
    <Item Relative="SearchPath" BasePath="%NINTENDO_SDK_ROOT%/../Textures" Path="HumanB.ftxb" />
    <comment color="1 0 0" comment="comment" />
</Project>

<Project>

The root component of a project file.

Attributes

Name Description
Version The version number of the project file.

Description

The following components appear in a project file.

  • <file_info>: Saves a record of the creation of the project file and changes made to it.
    This option may be omitted.
  • <Scene>: Stores animation set information for a scene.
    Only one appears.
  • <Item>: Represents files included in the project.
    Zero or more may appear.
  • <comment>: Stores metadata such as editing comments.
    This option may be omitted.

<file_info>

Saves a record of the creation of the project file and changes made to it.
If <file_info> is not present when a project file is loaded, it is not output when the file is saved either.

Description

The following components appear once, at most.

  • <create> : Saves a record of the creation of the project file.
  • <modify>: Saves a record of changes made to the project file.

<create>

Saves a record of the creation of the project file.

Attributes

Name Description
user The name of the user who created the project file.
host The name of the machine used to create the project file.
date_time The date and time the project file was created. Specified in YYYY-MM-DDThh:mm:ssTZD format according to the ISO 8601 standard.
tool_name The name of the tool used to create the project file.
tool_version The version number of the tool used to create the project file.

<modify>

Element that saves a record of changes to the project file.

Attributes

Name Description
user The name of the user who changed the project file.
host The name of the machine used to change the project file.
date_time The date and time the project file was changed. Specified in YYYY-MM-DDThh:mm:ssTZD format according to the ISO 8601 standard.
tool_name The name of the tool used to change the project file.
tool_version The version number of the tool used to create the project file.

<Scene>

Element that stores animation set information for a scene.

Attributes

Name Description
PreviewDefaultAnimationSet Specify true to preview the default animation set in the connected application.

Description

The following elements may appear zero or more times.

  • <AnimationSet>: Specifies non-default animation sets for the scene.
  • <Animation>: Represents an animation included in the default animation set for the scene.

<Item>

Represents files included in the project.

Attributes

Name Description
Relative Indicates the type of base for Path. It can be one of the following: Project, SearchPath, or StartUpFolder. These values represent the project file path, additional reference path, and startup folder path, respectively.
Base The name of the search path. Omit this component if the path is not a relative path from the additional reference path.
BasePath The additional reference path. This attribute can be omitted if Path is not a relative path from an absolute path that starts from an environment variable enclosed by percent signs (%).
Path The file path.
PreviewDefaultAnimationSet Specify true to preview the default animation set in the connected application. Omitted if <Item> does not point to a model file.
RetargetingHostModelName The name of the model to retarget the skeletal animation to. It is omitted if <Item> does not point to a skeletal animation file or no retargeting model is specified.
Info

In version 3.5.1.0 and earlier, you could specify StartUp for Relative to represent the additional reference path.
3DEditor can also load old project files for which this StartUp element is specified.

Description

The following elements appear when <Item> refers to a model file.

  • <AnimationSet>: Represents a non-default animation set to bind to a model.
    May appear zero or more times.
  • <Animation>: Specifies an animation included in the default animation set for a model.
    May appear zero or more times.
  • <Preview>: Represents information about the preview of a model.
    At most one may appear.
  • <Material>: Represents information about the preview of a material used in a model.
    May appear zero or more times.

<AnimationSet>

Specifies an animation set.

Attributes

Name Description
Name Name of the animation set.
Preview Specify true to preview the animation set in the connected application.

Description

The following components may appear zero or more times.

  • <Animation>: An animations included in the animation set.

<Animation>

Indicates the animations included in an animation set.

Attributes

Name Description
Name The filename of the animation.
ItemIndex The index of the animation <Item> in the <Project>.
Preview Specify true to preview animations in the animation set in the connected application.

<Preview>

Information about model previews.

Attributes

Name Description
BindModelName Specifies the model to bind to.
Select a model currently loaded into 3DEditor.
BindBoneName Specifies the bone to bind to from among those included in the model specified for BindModelName.
ScaleX Specifies the scale for the x-axis.
ScaleY Specifies the scale for the y-axis.
ScaleZ Specifies the scale for the z-axis.
RotateX Specifies the rotation around the x-axis.
RotateY Specifies the rotation around the y-axis.
RotateZ Specifies the rotation around the z-axis.
TranslateX Specifies the translation in the x direction.
TranslateY Specifies the translation in the y direction.
TranslateZ Specifies the translation in the z direction.
Visible Specify true to make the model preview visible.
ShowInObjView Specify true to show the model in object view.

<Material>

Information about previews for materials used in a model.

Description

The following components appear once, at most.

  • <Name>: The name of the material.
  • <OptimizeShader>: Specify true to send an optimized version of the shader.
  • <OptimizeShaderOnReload>: Specify true to regenerate the optimized shader according to changes made to the material.

<comment>

The edit data for projects to be used by 3DEditor.

Attributes

Name Description
Color Specifies the color used for editing in a form such as 1 0.5 0. Specify an empty string if there is no specification.
Comment Specify an editing comment represented by a single line of text.
