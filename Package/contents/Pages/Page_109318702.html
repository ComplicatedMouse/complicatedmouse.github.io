Overview

Although the procedure described in Initializing the SDEV completes the process of setting up the development hardware, there are several benefits to taking the additional step of assigning a static IP address for the SDEV.

This document describes those benefits and describes how to set a static IP address.

Info

When the SDEV ships, it is configured to get an IP address automatically.
The processes described in Initializing the SDEV and the Tutorial work fine if this setting is left alone and the IP address is obtained automatically.

Advantages

Setting a static IP address for the SDEV provides the following advantages.

Advantage 1

The Target Manager discovers the SDEV and connects to it as soon as the SDEV is powered on (connected to the AC adapter).
This allows you to run your NX applications sooner after powering on the SDEV.

If the SDEV does not have a static IP address, it can take as long as 60 seconds before the Target Manager can connect to the SDEV.

Advantage 2

The SDEV can be used on networks where a static IP address is set manually.

The device becomes as easy to use as the CAT-DEV (the development hardware for the Wii U) with a static IP address.

Info

When the SDEV is set to get an IP address automatically, the Target Manager may fail to connect to the development hardware when you select to Connect to Target after powering on the SDEV.
If Connect to Target fails, you must repeat the process of deleting the target and adding the target (and at this time, the IP address for the SDEV is changed).

As a workaround, either set a static IP address as described in this document, or wait for around 60 seconds after powering on the SDEV before selecting Connect to Target.

Setting a Static IP Address

Info

In addition to the following method, setting a static IP address using Target Manager is also possible.


You must take the following steps to set a static IP address for the SDEV.

  • Assigning a Static IP Address for the SDEV
  • Adjusting the Host PC's Network Settings
  • Re-Register With Target Manager

Assigning a Static IP Address for the SDEV

Use the CUI application ControlTarget, included in the NintendoSDK NX Add-On package.
The ControlTarget tool is located in Tools\CommandLineTools.

Warning

ControlTarget might fail if Target Manager is running, so exit Target Manager before running the application.

Target Manager might be in the task tray. Is Target Manager running? Also check the task tray when looking at these elements.

The following example shows how to use ControlTarget.

> ControlTarget.exe set-ip --target XAW02131008023 192.168.10.11 --netmask 255.255.255.0

In this example, the following values are set.

Item Content
Name of the SDEV you want to set (the --target option) XAW02131008023
IP Address 192.168.10.11
Subnet mask (the --netmask option) 255.255.255.0
Info

Finding the Name of the SDEV

To find the name for the SDEV you want to set, use the ControlTarget subcommand detect-target.

> ControlTarget.exe detect-target
40-D2-8A-B4-72-B3       10.121.33.212
XAW02131008023  169.254.10.228

The output result of this command is 40-D2-8A-B4-72-B3 and XAW02131008023, and one of these is the name of the SDEV.

The default SDEV name is determined as follows.

SDEV Version Default Name and How to Check For It
SDEV MP

The serial number.


The serial number is printed on the sticker to the right on the bottom of the SDEV.

For the SDEV shown in the picture, the serial number (and the default name) is XAW02131008023.

After the application runs, a log like the following appears. Proceed to the next section (Adjusting the Host PC's Network Settings).

> ControlTarget.exe set-ip --target XAW02131008023 192.168.10.11 --netmask 255.255.255.0
Discovering target...
Network setting has been changed to:
    IP address:      192.168.10.11
    Subnet mask:     255.255.255.0
Change Internet Protocol Version 4 (TCP/IPv4) Properties to "Use the following IP address".

Adjusting the Host PC's Network Settings

You must assign an IP address to the host PC that can communicate with the SDEV.

Adjust the network device's properties as follows.

  1. Open the Properties window for the network device.
    1. To open the Run dialog box, press Windows key + R, and then enter ncpa.cpl.
    2. Open Properties for the network device connected to SDEV.
    3. Select Internet Protocol version 4 (TCP/IPv4) and open Properties.
  2. Set the IP address and the subnet mask.
    1. Change the option from Obtain an IP address automatically to Use the following IP address.
    2. Set the IP address and subnet mask to the following values.

Verifying Complete Settings

The following type of log appears when the settings are complete.

Change Internet Protocol Version 4 (TCP/IPv4) Properties to "Use the following IP address".
Discovering target...
Discovering target...
Done.

For more information, see the description of the set-ip subcommand in ControlTarget.

You can also restore the default setting so that the IP address is obtained automatically. For more information about this procedure, see the description of the set-ip subcommand.

Checking the Settings

You can use the detect-target subcommand from ControlTarget to make sure the settings are set correctly.

> ControlTarget.exe detect-target
XAW02131008023      192.168.10.11

We see here that for the SDEV named XAW0213008023, it indicates that it is set to an IP address, 192.168.10.11.

That IP address is the same one that was used to specify the Lock the SDEV IP address option, so seeing that indicates that the settings are successfully completed.

Re-Registering With Target Manager

To run NX applications on an SDEV with a locked IP address, you must register an SDEV with a changed IP address to Target Manager.

When you set up SDEV normally, an SDEV with a flexible IP address is registered to Target Manager. In this case, you must re-register with the following steps.

  1. Start Target Manager and set SDEV with a flexible IP address to Remove Target.
  2. Select Add Target, and add the SDEV with a changed IP address.

Troubleshooting

The following type of log can appear, depending on when the adjusting of your host PC's network settings completes.

Change Internet Protocol Version 4 (TCP/IPv4) Properties to "Use the following IP address".
Discovering target...
Discovering target...
Warning: Failed to discover specified target: XAW0213008023.
Done.

In this case, you can use the detect-target subcommand introduced in the Checking the Settings section. If the configured IP address appears, settings were made without a problem.
