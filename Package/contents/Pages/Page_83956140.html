3DEditor will not start.

If you cannot start 3DEditor after installing NintendoSDK, check the NintendoSDK operating environment described in NintendoSDK Overview / Operating Environment and install the required environment.

If it will not start even though the correct environment has been installed, check with your system engineers.

I want to check the usage status of optimized shaders.

Can I determine whether an optimized shader is used for each material loaded in 3DEditor?

A: Yes, you can.

If you change the 3DEditor Object View to the material list, while connected to the application, you can check the usage status of optimized shaders for each loaded material.

Info

Yellow icon: The shader has not yet been transferred.
Blue icon: Optimized shader transfer is complete.

I want to switch the LOD used in the preview.

When I load a model that has shape LOD levels configured, can I select the LOD level to use when previewing the model?

A: Yes, you can.

When you load a model that is configured with shape LOD levels, you can select which levels to preview using the LOD Preview option in the preview page of the model properties window.

I want to select or clear the check boxes of multiple files.

Can I toggle the check boxes for multiple files at the same time in the Save Changed File or Specify save location dialog box?

A: Yes, you can.

If you hold down the CTRL key, select multiple files listed in the dialog box, and then select (or clear) a check box, the same change is made for all selected files.

When I load a TGA file, it is loaded as a texture intermediate file (FTX file), but I cannot save it.

A: When a TGA file is opened in 3DEditor, it is automatically converted to a texture intermediate file (FTX file) and loaded.

Info

To simplify the handling of files, only the texture converter's default conversion format is supported.
Note that the loaded TGA file cannot be saved.

Note: The file is marked as not savable in File View.

I want to add and modify textures in the sampler.

Is there a method to add or modify textures in the sampler list on the Samplers page of the material Property Window?

A: Yes, you can add and modify textures by dragging texture files into the list on the Samplers page of the material Property Window.

Procedure

  1. Open the property window of the material you want to add the texture to, and select the Samplers page.
  2. Drag the intermediate texture file or TGA file that you want to add into the list of samplers on the Samplers page.
  • If you drag the texture into an empty row in the samplers list, a new entry is created.
  • If you drag the texture onto an existing sampler, the texture is updated.
Info

You can add or modify the textures using both files that are already loaded into the File View of 3DEditor and files that are not yet loaded into the program.

When Multiple Materials Are Selected

If you drag an intermediate texture file onto a sampler while multiple materials are selected, all of the selected materials are overwritten if the sampler name is the same. If not, new samplers are created.

The "transferring" dialog box stays open when connecting.

In some cases, the "transferring" dialog box does not close when establishing a connection.

The cause of this behavior has not been identified, but sometimes it can be resolved with one of the following operations.

  • Restarting 3DEditor
  • Restarting the application on the connection target
  • Restarting the PC
  • Restarting the development hardware

How do I copy the content of a dialog box?

The content of a dialog box, such as error information displayed while using 3DEditor, may be copied and pasted.

The content of a dialog box displayed while using 3DEditor may be copied and pasted using the following steps.

Copy CTRL+C
Paste CTRL+V or select Paste from the shortcut menu

Optimized shader conversion takes a long time when connected to the runtime.

When a lot of dynamic options are used in shaders, 3DEditor generates shader variations corresponding to the number of option combinations. This can cause shader conversion to take longer when 3DEditor is connected to the runtime. The following describes ways to shorten the time it takes for shader conversion.

Reduce the shader variations generated by 3DEditor.

Generating shader variations can be customized using the <CreateShaderVariationCommand> team setting. Shader variations not required in real time editing are deleted to shorten the conversion time.

Use the shader cache feature.

The <ShaderConverterAdditionalArgs> and <ShaderConverterAdditionalArgsForOptimizedShader> team settings can be used to specify additional options for shader conversion processing, executed internally by 3DEditor. If you specify the --shader-cache-directory option, shader conversion skips shaders with the same key created in the previous conversion, and the conversion time can be shortened from the second conversion instance. The conversion time can be shortened further by configuring the cache folder in a folder on the server, and sharing the cache among multiple users. (Note that due to variables such as access speed to the server, conversion time may also take longer.)

For more information about the options, see 3D Shader Converter.
