Settings Manager

When developing web content, you can use the settings manager to customize the behavior of the web applet and Offline Web Applet.

How to Use

For more information about using the settings manager, see Settings Manager.
The following table shows the web_settings category settings for the web applet and Offline Web Applet.

Item Data Type Default Value Key Value
Host PC path that is mounted as the root path. String that represents the path. Empty string. web_settings.host_root_path
Host that ignores the SSL certificate and allows connection. String that represents the hostname. - web_settings.allowed_ssl_hosts[].host_name
Whether Web Inspector is enabled. Boolean value. false web_settings.use_web_inspector
Whether the memory usage rate display is enabled Boolean value. false web_settings.show_memory_usage_rate

Settings

Changing the Host PC Path Mounted as the Root Path

To develop web content, the web applet or Offline Web Applet and WebAppletCaller are mounted with the "C:/NXWebApplet" path as the root path on the host PC by default.
Use host_root_path in the settings to change the host PC path that is mounted.
For more information about using this feature, see Root Paths for Host PC and SD Card for WebAppletCaller.
The following shows an example where the path for host PC mounting is changed from the default "C:/NXWebApplet" to "D:/Sample".

"web_settings": {
    "host_root_path": {
        "current": "D:/Sample",
        "default": ""
    }
}

If the settings are configured correctly, an entry like the following is added to the log.

[Host] hostRootPath: "D:/Sample"

If there is a problem with the settings, an entry like the following is added to the log.

[Host] Failed to mount hostRootPath: "D:/Sample"

Specification to Ignore SSL Certificate Error and Allow Connection to the Host

Use the allowed_ssl_hosts setting to specify hosts for which SSL certificate errors are ignored and connections are allowed.
The following example specifies www.nintendo.co.jp and www.nintendo.com.

"web_settings": {
    "allowed_ssl_hosts": [
        {
            "host_name": "www.nintendo.co.jp"
        },
        {
            "host_name": "www.nintendo.com"
        }
    ]
}

If the settings are configured correctly, an entry like the following is added to the log in red.

[AllowSslHost] www.nintendo.co.jp by SettingsManager
[AllowSslHost] www.nintendo.com by SettingsManager

For more information, see Allowing Connections Despite SSL Certificate Errors

Enabling Web Inspector

Web Inspector can be enabled with the use_web_inspector configuration item.
The following is an example of enabling Web Inspector.

"web_settings": {
    "use_web_inspector": {
        "current": true,
        "default": false
    }
}

If the settings are configured correctly, an entry like the following is added to the log in red.

[WebInspector] uses WebInspector by SettingsManager

For information about how to use Web Inspector, see Web Inspector.

Displaying Memory Usage Rates

Use the show_memory_usage_rate configuration item to also display the memory usage rate of the displayed webpage on the screen.

The following example shows how to enable the memory usage rate display.

"web_settings": {
    "show_memory_usage_rate": {
        "current": true,
        "default": false
    }
}

If the settings are configured correctly, an entry like the following is added to the log in red.

[MemoryUsageRate] shows MemoryUsageRate by SettingsManager

For more information about memory, see Memory.
