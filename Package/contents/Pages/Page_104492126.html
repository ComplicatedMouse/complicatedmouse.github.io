Operating Environment

Operation is not ensured when the system locale of the operating system is changed to anything other than the default value.

Operation is not ensured when the system locale of the operating system is changed to anything other than the default value. (For example, if the locale is set to anything other than "Japanese [Japan]" in the Japanese version of Windows.)

VSI

The NintendoSDK wizard is not deleted.

A NintendoSDK wizard is added to the Visual Studio project templates when you install the NintendoSDK using NDI.

There is an issue, however, where that wizard is not deleted even when you deleted the NintendoSDK.

To delete the wizard, you must manually delete the following folders.

  • %USERPROFILE%\Documents\Visual Studio 2017\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2017\NintendoSDK\CustomWizard
  • %USERPROFILE%\Documents\Visual Studio 2015\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2015\NintendoSDK\CustomWizard
  • %USERPROFILE%\Documents\Visual Studio 2013\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2013\NintendoSDK\CustomWizard
  • %USERPROFILE%\Documents\Visual Studio 2012\Wizards\NintendoSDK
  • %USERPROFILE%\Documents\Visual Studio 2012\NintendoSDK\CustomWizard

Cannot use the NintendoSDK wizard.

The files that make up the NintendoSDK wizard are placed within the %USERPROFILE% directory for the user account that was used when installing the NintendoSDK.

As a result, when the user who installed the NintendoSDK is different from the user that wants to use the wizard, the NintendoSDK wizard will not appear among the Visual Studio project templates.

To use the wizard, install the NintendoSDK with the user account that will use the wizard.

Warning message LNK4099 appears when the project is being rebuilt.

Sometimes the warning message LNK4099 is displayed when you attempt to rebuild a NintendoSDK application project that links to a library that was created as a NintendoSDK library project.

This warning is generated because the PDB file format is used for storing library debugging information. The warning will not occur if, instead of rebuilding, you first clean the project and then build.

As a workaround, use either of the following methods.

  • When creating the project, select the Create directory for solution option.
  • As the Debug Information Format, select either C7 Compatible (/Z7) or None.

Visual Studio Help

In environments without Internet connectivity, the documents package fails to install.

In environments where you cannot connect to the Internet, the installation of the NintendoSDK Documents package sometimes fails.
This problem occurs because it cannot connect to the expiration server for the digital certificate and check the validity of the signature, so the Help file is not installed.

To avoid this problem, try the following procedure.

  1. Connect to the Internet
    or
    install an intermediate certificate issued by an authority higher than the digital certificate (CN = DigiCert SHA2 Assured ID Code Signing CA).
  2. Reinstall the NintendoSDK Documents package.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer. If you try to use it, you cannot enter anything in the text box, making search impossible.

Use a web browser to use the Find Result page.

BuildSystem

Visual Studio has been updated to a version not supported by NintendoSDK.

Use the procedure described in the following article to use an earlier version of the MSVC compiler. It provides a workaround to problems associated with MSVC compiler differences.

Side-by-Side Minor Version MSVC Toolsets in Visual Studio 2017

Article Summary

  • Install VC ++ 2017 version 15.x v14.1y toolset (the MSVC compiler corresponding to Visual Studio 2017 15.x).
  • Copy "C:\Program Files (x86)\Microsoft Visual Studio\2017\Professional\VC\Auxiliary\Build\14.1y\Microsoft.VCToolsVersion.14.1y.props to the project directory and import it into the VXCPROJ project file.

The operating environment differs from the environment in Operating Environment, so investigate the process thoroughly.

Installing NintendoSDK VsExtension fails when both Build Tools for Visual Studio 2017 and Visual Studio 2017 are installed.

Installing NintendoSDK VsExtension can fail when both Build Tools for Visual Studio 2017 and Visual Studio 2017 are installed.

This failure is caused by a bug in the VSIXInstaller for Visual Studio 2017 (for version 15.4) where the Visual Studio extensions are not installed when Build Tools for Visual Studio 2017 has already been installed. For more information, see the following page: vsixinstaller.exe throws NullReferenceException when Visual Studio Build Tools 2017 are installed.

To avoid this problem, try any of the following procedures.

  • Uninstall Build Tools for Visual Studio 2017.

FS

nn::fs::Commit() and nn::fs::CommitSaveData() fail and return ResultTargetLocked.

The nn::fs::Commit() and nn::fs::CommitSaveData() functions do not work correctly when the save data file is open in an editor or another program on the host PC. Make sure that the save data file is not open from another program when running the application.

Virus scanners and other programs may automatically access the file. In this case, take some measure such as setting the path where the application is located to not be a target of the virus scanner.


When nn::fs::ReadDirectory() is called for the root directory of a drive, occasionally the first entry is not listed.

If the target is a type of removable media, occasionally the first entry is not listed when nn::fs::OpenDirectory() and nn::fs::ReadDirectory() are called via nn::fs::MountHost() or nn::fs::MountHostRoot() directly under the drive.

Work around this issue by placing a dummy entry in the location and so avoiding using the root directory of removable media.

Cannot connect to the network.

nn::nifm::IsNetworkAvailable() does not become true.

Check that the settings are as you intended.

To check the imported content, see nifm Library > Registering Network Connection Settings to export and verify the settings.

The settings for authentication and encryption must be a combination supported by the SDK.

To include targets used automatically by the system, you must enable the settings used for automatic connections.

To connect to wireless access points with the stealth settings, you must enable the option to try making connections to stealth access points.

In a Windows environment, you must register connection settings information that match the network connection used by Windows.

nn::nifm::IsNetworkAvailable() is true, but cannot connect to the external network

Check that you are using a line that can connect to the external network.

As of NintendoSDK 0.11.0, proxy settings configured in the network settings are automatically applied. Also, any proxy settings configured using curl_easy_setopt() are ignored. To disable proxy settings, follow the procedure in nifm Library > Registering Network Connection Settings when configuring your connection settings or specify 0 for CURLOPT_PROXYAUTOCONFIG in curl_easy_setopt().

Nintendo Dev Interface (NDI)

Deleting an environment sometimes fails.

If there are still any programs installed in an environment (such as Nintendo Target Manager) that are still running when you attempt to delete the environment, the process for deleting the environment will fail because the program files cannot be deleted.
You can avoid this issue by making sure to close all running programs before proceeding with the deletion.


The NINTENDO_SDK_ROOT environment variable is not configured.

When you create an environment in NDI, the path to the last created NintendoSDK is set for the NINTENDO_SDK_ROOT environment variable.

If the NINTENDO_SDK_ROOT environment variable is not set for the intended environment, or if you want to set the NINTENDO_SDK_ROOT environment variable to a different environment, select that environment in NDI and then select Manage > Set to Switch default.

System Features

The authentication screen is a temporary image.

Nonproduction-equivalent devices have the temporary image on the authentication screen.

On systems and Joy-Con controllers prior to the MP model, using the Joy-Con stick calibration feature causes bugs to occur.

The stick calibration is only supported on the MP model and later. The EP models are not supported, so make sure to use the MP model or later.


HID

Vibration does not occur in the PC environment.

Symptoms

A bug prevents vibrations from occurring when using vibration functionality with NintendoSDK 0.16.0 or later if the PC you are using had used version 0.15.0 or earlier for the NintendoSDK.

You can confirm this bug by starting a program that uses this functionality, like HidVibrationBasic or HidVibrationPlayer.

Workaround

Delete the following file manually.

C:\Users\<user name>\AppData\Local\Temp\Nintendo_SystemSaveData_SystemSettings

If you then restart the affected program on the PC, vibrations will occur because this file will be automatically re-created.


You can avoid the problem by using Settings Manager to change the settings file.

Specifically, export the settings file, change the hid_settings.vibration_master_volume value to 1.0, and import the settings file.

  1. Export settings file

    Tools\CommandLineTools\Settings\SettingsManager.exe Export "destination absolute path for exported file"

  2. Settings file value that must be edited (search for vibration_master_volume)

    "hid_settings": {
    "vibration_master_volume": {
        "current": 1.0,
        "default": 1.0
    }
},

  3. Import settings file

    Tools\CommandLineTools\Settings\SettingsManager.exe Import "absolute path to settings file"

In the PC environment, vibrations can continue for seconds when an application ends.

Symptoms

When an application in the PC environment uses the vibration feature, sometimes the controller will continue to vibrate for several seconds after the application has ended if vibration values were sent when the application was ended.

Workaround

For applications in the PC environment, stop the vibrations before acting to end the application.

This problem only arises in the PC environment.


Audio Tools

When you convert a sound project imported using NintendoWare SoundMaker, an invalid sound archive is exported.

When an NW4C/NW4F sound project (still with a cache file for conversion purposes) is imported and the normal conversion process is performed, the binary source files for NW4C/NW4F get mixed in with the NintendoSDK binary sound archive files (BFSAR files) and the nn::atk runtime cannot properly load the data.

This problem is due to the use of old cached data, so run Reconvert after you import the sound project.

When a sound waveform is output from NintendoWare SoundMaker or SoundProjectWaveExporter, silence is occasionally added to the beginning of the waveform.

When a sound waveform is output from NintendoWare SoundMaker or SoundProjectWaveExporter, sometimes silence is added to the beginning of the output waveform.

If silence has been added to the start of the waveform output, delete that portion using a waveform editor.

Reloading a sound archive in NintendoWare AtkPlayer causes the application to crash.

The application may stop if you perform the following operations after starting AtkPlayer.

  • You edit a stream sound in SoundMaker, convert the sound archive file, and then reload the sound archive in AtkPlayer.
  • You open a different sound project in SoundMaker and then reload the sound archive in AtkPlayer.

As workarounds, you can restart AtkPlayer after performing these operations, or wait for a while after converting in SoundMaker to reload the archive in AtkPlayer.

Cannot configure the maximum number of voices using nn::atk::SoundSystem::SetVoiceCountMax.

When you use nn::atk::SoundSystem::SetVoiceCountMax and attempt to configure the setting for the maximum number of voices used by the sound library, that setting is ignored and the value set by nn::atk::SoundSystem::SoundSystemParam::voiceCountMax is used.

We do not recommend using nn::atk::SoundSystem::SetVoiceCountMax. Consider using the recommended feature nn::atk::SoundSystem::SoundSystemParam::voiceCountMax. For more information, see the reference for nn::atk::SoundSystem::SoundSystemParam::voiceCountMax.

Sometimes parameters that have been changed before a sound is played do not get applied to the sound at playback time by the NintendoWare AtkPlayer.

If parameters are changed after AtkPlayer has started for a sound that has never been played, in some cases the changes do not get applied to the sound when it is played. The workaround is to play the sound again.
