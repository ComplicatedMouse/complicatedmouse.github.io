Overview

The system features provide the following functionality for users to manipulate user save data.

  • User Account Migration
    • This feature transfers user save data to a different console.
    • Provided to the user as Transferring Users and Save Data in system version 4.0.0 and later.
  • Save Data Transfer
    • This feature transfers individual user save data to a different console.
    • Provided to the user as Transfer Your Save Data in system version 8.0.0 and later.
  • User Save Data Backup
    • This feature copies the user save data to a different location than the system internal storage.
    • Provided to the user as save data cloud backup in system version 6.0.0 and later.
  • Copy User Save Data
    • This feature copies user save data to a different user account.
    • Provided to the user as save data cloud backup in system version 6.0.0 and later.

This section describes the characteristics of each of these features.

User Account Migration

This feature enables you to transfer all user save data for a particular user to a different console.
This feature is called User Account Migration.

User Account Migration transfers a user account (nn::account::Uid and profile information) and all of the user save data for that user to a different console.
The same user account owns the transferred user save data both before and after transfer.
On the other hand, the transfer target device will be used for information other than the user account (such as system settings and temporary storage).

Save Data Transfer

This feature transfers individual user save data to a different console.
This feature is called Save Data Transfer.

User Save Data Backup

This feature enables copying user save data to a location other than the system internal storage.
This feature is called User Save Data Backup.
It is possible for each application to select whether to support this feature.

The User Save Data Backup feature allows the user to create backups of the user save data at any time.
The user can restore previously backed up user save data to the same user account at any time.

While the owner of the user save data (nn::account::Uid) will not change before and after the backup and restore process, current user save data may be rolled back.

Created backups are only accessible from user accounts that are linked to the same Nintendo Account.

Copy User Save Data

This feature enables copying specific user save data to a different user account.
This feature is called Copy User Save Data.
It is possible for each application to select whether to support this feature.

This feature does not differentiate between whether the user account to which the save data is copied is on the same console or a different one.
As a result, all of the information that can be obtained from the system features may change after the user save data is copied.
(There is no effect if the application does not depend on information obtained from the system features.)

User save data can only be copied between user accounts that are linked to the same Nintendo Account.

Recommendations for Supporting Copying User Save Data

Providing support for copying user save data allows the application to provide additional value for users who want to freely manage save data independently of the user account itself.
In addition, you do not have to implement any specific feature in the application because the user save data copy process is handled by the system features.

We recommend supporting this feature as long as the application specifications have no reason for avoiding copying user save data (such as to maintain uniqueness between users).

Note

There is additional information to consider when supporting copying user save data and saving user save data.
For more information, see Notes When Using User Save Data.

Note

Enabling support for copying user save data also enables backup and restore on the copy target device.
In this case, the copy source and copy target are checked independently for the restriction periods when restoring from backup.
As a result, supporting copying save data removes the effectiveness of restrictions on backing up user save data specified by the application.

When supporting copying user save data, verify that there are no restrictions on restoring backup data.
