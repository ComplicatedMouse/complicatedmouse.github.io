Mii Applet

This library applet provides features to create new Mii characters and edit created Mii characters.

This section covers the basic workflow through starting the Mii applet.

Note

To submit a ROM that uses this applet, you must submit a separate request. Contact Nintendo if you want to submit this type of ROM.

Including Header Files

Add the following code to your program.

#include <nn/mii.h>
or
#include <nn/mii/mii_Applet.h>

Starting the Mii Applet

The Mii applet can be started when the mode is Add Mii Mode.

Add Mii Mode

This mode allows you to add one Mii to the database.

Start the Add Mii Mode by calling the following API. This API blocks other processes until the applet is closed.

nn::Result result = AppendMii(int* pOutAppendIndex);

After the applet is closed, use the preceding API to get the results.

Result Description of the result.
nn::ResultSuccess One Mii was added and the applet was closed.
nn::mii::ResultCanceled The applet was terminated without adding a Mii character.

The database index for the added Mii is set in the pOutAppendIndex argument only if the result is nn::ResultSuccess.

After returning to your application from the Mii applet with the result nn::ResultSuccess, re-obtain the Mii from the database. If a result other than nn::ResultSuccess was obtained, obtain the Mii again according to the application specifications.

The following table includes the patterns to use when the applet is called normally, until returning to the calling application, and the obtained results. Note that the Mii may be deleted regardless of the value of the results obtained.

Pattern Obtained Result
Add one Mii and return to the application. nn::ResultSuccess
The database is full, so first delete one Mii, then add one Mii, and return to the application. nn::ResultSuccess
The database is full, so delete one Mii and return to the application without adding a Mii. nn::mii::ResultCanceled
Return to the application without adding or deleting Mii characters. nn::mii::ResultCanceled
