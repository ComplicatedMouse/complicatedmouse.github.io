Variations of the Low-Level Graphics API

Low-Level Graphics API

This section describes how to handle the low-level graphics API variations contained in the gfx library.

These variations are not intended for use out of special circumstances, and have no use in normal cases.

For more information about selecting the low-level graphics API, see Selecting Low-Level Graphics API.

Definition of Low-Level Graphics API

The low-level graphics API is defined in the gfx_Variation-api.*.h header for each of the low-level graphics.

Low-Level Graphics API Types

The type of low-level graphics API is represented by the nn::gfx::ApiType structure.

The nn::gfx::ApiType structure holds a static variable enumerator value, which represents the low-level graphics API.
This allows for the comparison of low-level graphics API.

Version of the Low-Level Graphics API

The version of the low-level graphics API is represented by the nn::gfx::ApiVersion structure.

Thenn::gfx::ApiVersion structure holds a static variable value, which represents the low-level graphics API version number.
This allows for the comparison of low-level graphics API versions.

Variations of the Low-Level Graphics API

The variation of the low-level graphics API is represented by the nn::gfx::ApiVariation structure, which combines the low-level graphics API type and version.

The nn::gfx::ApiVariation structure is the type definition of the nn::gfx::ApiType and nn::gfx::Version structures. The type and version of the low-level graphics API may be found by referencing these structures.

Target Low-Level Graphics API

Information about the target low-level graphics API is represented as the nn::gfx::TargetConfig structure.

The variation, type, and version of the low-level graphics API may be found in the nn::gfx::TargetConfig structure.

Function Variation

A small number of functions are provided as functions in gfx.

Functions that have a different process depending on the low-level graphics API will have variations.

This description uses the nn::gfx::Finalize function as an example.

Target Variation

The target low-level graphics API is called without adding a suffix.

nn::gfx::Finalize();

Specifying a Variation

You may explicitly call the variation of the low-level graphics API by adding a suffix.

The following example specifies GL4.

nn::gfx::FinalizeGl4();

The version can be omitted for brevity. In this case, the target version is selected.

nn::gfx::FinalizeGl();

You cannot specify a variation of a low-level graphics API that has not been added.

Specifying by Parameter

The nn::gfx::ApiVariation structure may be taken as a template parameter.

Use the following code when you want to specify the variation as a parameter instead of a different symbol name.

nn::gfx::Finalize< nn::gfx::ApiVariation< nn::gfx::ApiType< nn::gfx::LowLevelApi_Gl >, nn::gfx::ApiVersion< 4 > >();

Class Variation

Most functions in gfx are provided as a class.

The Object class has variations.

This description uses the nn::gfx::Device class as an example.

Target Variation

The target low-level graphics API class is used without adding a suffix.

nn::gfx::Device

Specifying a Variation

You may explicitly call the variation of the low-level graphics API class by adding a suffix.

The following example specifies GL4.

nn::gfx::DeviceGl4

The version can be omitted for brevity. In this case, the target version is selected.

nn::gfx::DeviceGl

You cannot specify a variation of a low-level graphics API that has not been added.

Specifying by Parameter

There is also a version with a prefix "T" that takes the nn::gfx::ApiVariation structure as a template parameter.

Use the following code when you want to specify the variation as a parameter instead of a different symbol name.

nn::gfx::TDevice< nn::gfx::ApiVariation< nn::gfx::ApiType< nn::gfx::LowLevelApi_Gl >, nn::gfx::ApiVersion< 4 > >

This function is mainly intended to be used in static programming.
A variadic template may be used to make a static operation to all variations.

Getting Variation

A class with variations all have a Target type definition.
Target is a nn::gfx::ApiVariation structure, making it possible to statically get the low-level graphics API variation information.

Example:

const int version = nn::gfx::Device::Target::ApiVersion::value;
