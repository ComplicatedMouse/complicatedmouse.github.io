Custom Shader Settings

The page describes the custom shader settings available for programmable shaders, created for a game application.

Overview

The Custom Shader Setting feature can set shaders created for the game application to emitters.
Using custom shaders expands the graphic expression of effects.
The following overview describes the settings ().

Select a shader from Custom Shader Setting. Compile Shader Code, , and Show Shader Code, , will appear.
The group has the following settings.

Compile Shader Code

Select Compile Shader Code, , to check for errors within the selected shader code. Check updated content by clicking the button after a user edits a shader.
The following table describes the buttons.

Shader 0 Used when Def1 and Def2 are not used within the Render Settings included in the Project Settings file.
Shader 1 Used when Def1 is used within the Render Settings included in the Project Settings file.
Shader 2 Used when Def2 is used within the Render Settings included in the Project Settings file.

Show Shader Code

Display the data for vertex and fragment shaders in a text editor using Show Shader Code, .

Enabling Preprocessing

Use the check box, , to toggle display of the results prior to and after preprocessing when showing shader code.
Enabling this setting will display results after preprocessing.
This feature is not effected by the options settings, and what is selected in the UI is how the code is shown.

Displaying Vertex and Fragment Shaders

Display shader data using the various buttons.
The following are button types.

Vertex shader Fragment shader
Code Displays the vertex shader source code. Displays the fragment shader source code.
Assemble Displays the assembly source code for the vertex shader. Displays the assembly source code for the fragment shader.

Example Configurations

This section describes the sample custom shader that is located in the following directory.
\Samples\Sources\Applications\VfxCustomShader\Resources\Eset\CustomShader.eset

A Color Changes the color The RGB values set for Color () in the Cstm Shader tab are multiplied by the emitter color.
B Position Changes the rendering position Adjusts the position where the effect is rendered by adding the x, y, and z values of the Added Position setting () of the Cstm Shader tab to the emitter position.
C Light Lighting Applies simple lighting.
D Normal Normal color You can check the particle normal as a color in the viewer.
E NormalMap Normal map

Applies normal maps. Registers textures for normal maps to the Texture tab () Texture 1 ().

Related Link: For more information about normal mapping in EffectMaker, see here.
F SimpleBL Light Map Applies a simple light map with the values specified for the IBL Color 0 () and IBL Color 1 () values in the Cstm Shader tab as the colors for the sky and ground, respectively.
