Importing Other Projects

NintendoWare SoundMaker enables you to import project files from other platforms.

Info
Note: After importing the file, select Project > Reconvert.

Importing Projects From Other Platforms

First, select File > Import > Project from Other Platform to open the file selection dialog box. You can then load non-NintendoWare sound projects by selecting File Type > Project from Other Platform (*.cspj;*.rspj).

Importing From NintendoWare for CTR SoundMaker

The files that can be imported are: project files (CSPJ), sound set files (CSST), bank files (CBNK), and text sequence sound files (CSEQ).

The original files are not changed when you import them; new files are saved using a different extension in the same folder.

The application runs the following processes when it imports text sequence sound files.

  • The strings "cseq" and "cinl" within the file are converted en masse to "fseq" and "finl."
  • A recursive import is run for any "cseq" files.

If Sound Archive Output Destination in the Project Setting dialog box is blank, output is automatically attached.

If the instrument interpolation type is No interpolation, it is changed to Linear interpolation during import.

Importing From NintendoWare for Revolution SoundMaker

Project (RSPJ) files, sound-set (RSST) files, bank (RBNK) files, and text-sequence sound (RSEQ) files.

The original files are not changed when you import them; new files are saved using a different extension in the same folder.

The application runs the following processes when it imports text sequence sound files.

  • The strings "rseq" and "rinl" within the file are converted en masse to "fseq" and "finl."
  • A recursive import runs for any "rseq" files.

If Sound Archive Output Destination in the Project Setting dialog box is blank, output is automatically attached.

Importing From Bank Files

When bank lists are archived, use File > Import > Bank File to import only the pertinent project bank files (CBNK, RBNK, and BNK files).

If the instrument interpolation type is No interpolation in an NW4C bank file, it is changed to Linear interpolation during import.

Importing Sound Set Files

Import a sound set file (a RSST or CSST file) by first making sure a project file is open and then selecting File > Import > Sound Set from the menu.

The associated bank files are imported at the same time.

Importing STXT Files

An STXT file is an effect sound-definition file that is used by the Sysdolphin audio library. Sysdolphin is the development environment provided for the Nintendo GameCube.
This feature is used when porting game projects created using Sysdolphin.

You can import STXT files only when the wave sound set is open in the Sound Set tab, and the focus is on the Sound Set tab. Select File > Import > Stxt and specify the STXT file to be imported.

Importing STXT Files

Create a wave sound set by using the filename of the specified STXT file. The effect sound name that is defined in the STXT file is used as the converted wave sound name.

The following information is imported into the wave sound from the playback information listed in the STXT file.

  • Reference waveform filenames and paths
  • Volume (AVOL command)
  • Priority (APRI command)
    Priorities listed in the STXT file are converted to channel priorities.
  • Pitch (ATUNE command)
  • Pan (APAN command)
  • Surround pan (ASPAN command)
  • Effect mix (AMIXA and AMIXB commands)
  • Comments (denoted starting with //)
    What gets imported is only the first comment in the part of the STXT file where sound effects are described (inside the curly brackets).
Info

The parameter values that are configured in the STXT file are automatically scaled to SoundMaker values. However, priority values are not scaled and the STXT file value is used without modification.

At the time of this release, you can import only those parameters that are specified at the start of an effect sound in an STXT file.
