Overview

A comprehensive CUI application to request the various command processes for multiple development hardware units at the same time.

You can set the conditions, and then select the target (development hardware) for executing the processing.

Important Information

  1. Do not perform the following operations when executing TargetShell.
    • Developer configuration changes such as the number of connected devices (such as by connecting or disconnecting development hardware from the PC)
    • Operations in Nintendo Target Manager (connect, disconnect, register, delete, or change the existing target)
  2. Make sure that you connect development hardware that has been registered in Nintendo Target Manager to the PC.
  3. The development hardware registration status in Nintendo Target Manager may change unintentionally under the following circumstances.
    • No development hardware is registered to Nintendo Target Manager.
    • No default target is set.
    • Connecting to Nintendo Target Manager fails when TargetShell closes.

Location

  • Executable file 
Tools\CommandLineTools\TargetShell.exe
  • Plug-Ins 
Under Tools\CommandLineTools\TargetShellPlugins.

How to Use

  • Process for initializing the development hardware
    (In the following example, all development hardware units registered in Nintendo Target Manager are initialized.)
1. Connect the development hardware to a PC.
2. Open the Command Prompt window.
3. cd %NINTENDO_SDK_ROOT%\Tools\CommandLineTools
4. TargetShell.exe initialize


  • Process for updating Joy-Con firmware
    (In the following example, the Joy-Con firmware is updated for all development hardware units registered in Nintendo Target Manager.)
1. Connect the Joy-Con controllers to the development hardware in handheld mode.
2. Connect the development hardware to a PC.
3. Open the Command Prompt window.
4. cd %NINTENDO_SDK_ROOT%\Tools\CommandLineTools
5. TargetShell.exe update-joycon

Common TargetShell Options

The common options can be used by all TargetShell subcommands.

Common Options That Specify the Target

Note: The command is executed for targets that meet all the criteria.

Note: Subsequent subcommands are also applied.

Name Description 
--detect-device

The command is executed for all targets connected to the PC.

If this option is not specified, the command is executed for targets registered in Nintendo Target Manager.

 
--name <arg>

The command is executed for the targets that match the specified name.


You can specify a wildcard (*) for the device name.

Example: --name=XAW02*

You can also specify multiple device names.

Example: --name=XAL01234567890 --name=XAL01234567891

 
--serial-number <arg>

The command will be executed for targets that match the specified serial number.


You can specify a wildcard (*) for the serial number.

Example: --serial-number=XAL*

You can also specify multiple serial numbers.

Example: --serial-number=XAL01234567890 --serial-number=XAL01234567891

 
--ip-address <arg>

The command will be executed for targets that match the specified IP address.

(If you specify an IP address, the command is not executed for development hardware units connected via USB.)


You can specify a wildcard (*) for the IP address.

Example: --ip-address=169.254.*

You can also specify multiple IP addresses.

Example: --ip-address=169.254.11.111 --ip-address=169.254.11.112

 
--hw-type <arg>

The command will be executed for targets that match the specified development hardware type.


You can specify the following types of development hardware.

Not specified: NotSpecified (default)

SDEV: Sdev, SDEV, sdev
SDEV-D: Sdev-D, SDEV-D, sdev-d

EDEV: Edev, EDEV, edev

EDEV-D: Edev-D, EDEV-D, edev-d

HDEV: Hdev, HDEV, hdev
ADEV: Adev, ADEV, adev

Example: --hw-type=Sdev

You can also set multiple development hardware types.

Example: --hw-type=Sdev --hw-type=Edev

 
--connection-path <arg>

The command will be executed for targets that match the specified connection method.


You can specify the following connection methods.

Not specified: NotSpecified (default)

Host bridge: HostBridge, HOSTBRIDGE, hostbridge

USB: Usb, USB, usb

Example: --connection-path=HostBridge

You can also set multiple connection methods.

Example: --connection-path=HostBridge --connection-path=Usb

Other Common Options

Name Description 
-v, --verbose

Specify this option to generate detailed logs for initialization. (For the output destination of the logs, see the --output option.)

A simple log is generated if this option is not specified.

 
--output <arg>

Specifies the output folder for the log.

The log file is not generated if this option is not specified.


Example: --output=C:\templog

 
--loglevel <arg>

Sets the log level.

For the log level, you can set trace, debug, info, warn, error, or fatal.

If no log level has been set, the default is info.

For more information, see TargetShell Log Levels.


Example: --loglevel=warn.

Initializing Development Hardware (initialize Subcommand)

Options

Name Description 
--fwtype <arg>

Specifies the firmware type.

If no firmware is specified, the standard firmware for development hardware (DEV MODE) will be used.

Firmware that approximates retail hardware (PROD MODE): Prod

With the feature to generate logs from the microSD card: ProdWithLog


Example: --fwtype=Prod

Also see Common Options.

Info

If you execute this subcommand, Target Manager and NintendoSdkDaemon restart in the following cases.

  • NintendoSDK NX Add-On 13.0.0 or later
    • When operating an EDEV or HDEV unit
  • NintendoSDK NX Add-On 13.2.0 or later
    • When operating an EDEV, EDEV-D, HDEV, or ADEV unit

Installing, Using the install Subcommand, Applications, Patches, and Downloadable Content to Development Hardware

  • Installs specified applications, patches, or downloadable content to specified development hardware, in parallel.

Options

Name Description 
--type

Configures the installation type.

application: Installs applications.

path: Installs patches.

addoncontent: Installs downloadable content.

Note: This setting is required.

 
--path

Configures the path to install from, including the NSP file name.

Note: This setting is required.

Installs with the forced flag enabled.

Installation overwrites files when the forced flag is enabled and the application has already been installed.

Only valid when --type is set to application. Ignored for other settings.

Installs patches and downloadable content to the system.

sdcard: Installs to an SD card.

auto: Installs to an SD card if one has been inserted in the development hardware.

Install to the system when this option is not set.
