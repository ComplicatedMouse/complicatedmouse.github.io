Binary Intermediate File Converter

This chapter describes procedures and important information about using the binary intermediate file converter.

  • Overview
  • Command-Line Options
  • Usage Examples

Overview

Converts text intermediate files (with .f??a extensions) to binary intermediate files (with .f??b extensions).

Use this tool after you have modified a text intermediate file using a text editor to convert that file to a binary intermediate file.

When you drag that text intermediate file or its folder to the executable file (3dIntermediateFileAsciiToBinary.exe) or the executable's shortcut, a binary intermediate file gets saved in the same folder as the text intermediate file.
Files other than text intermediate files are ignored.

If you want to output binary intermediate files to another location, either copy text intermediate files first and then convert or move the binary intermediate files after conversion.

Command-Line Options

3dIntermediateFileAsciiToBinary.exe [--args-file ""] [--version] [-h] [-s] [-j ""] [--disable-xsd-check]
[--enable-extension ""] [--disable-extension ""] [--error-detail-output ""] [intermediate_file|folder]...

Converts text intermediate files (with .f??a extensions) to binary intermediate files (with .f??b extensions).

--args-file ""

Specifies the file.
You can use the contents in the parameter file to specify arguments.
The parameter file allows you to specify each option over several lines.
Lines starting with a number sign (#) are treated as comments.
Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--version

Displays version information.

-h --help

Displays Help.

-s --silent

Suppresses the display of messages during processing.

-j "" --jobs ""

Specifies the number of jobs. When unspecified, the processor count is assigned.

--disable-xsd-check

Disables format checking of intermediate files using schemas (XSD files).
Only the latest version intermediate files can be processed when this option is enabled.

--enable-extension ""

Specifies the extensions to be processed among text intermediate files. 　
Use a comma (,) as a delimiter when listing more than one filename extension. 　
Example: --enable-extension "fmdb,ftxb"

--disable-extension ""

Specifies the extensions to be processed among text intermediate files. 　
Use a comma (,) as a delimiter when listing more than one filename extension. 　
Example: --disable-extension "fmdb,ftxb"

--error-detail-output

Specifies a folder for saving detailed information when an error occurs. If not specified, this information is written to the default folder.

Intermediate File | Folder

Specifies the file or folder to be processed.
Files other than text intermediate files are ignored.
Files located under the specified folder are recursively processed when a folder is specified.
Files other than intermediate files are ignored.

Supports wildcards (?: Replaces one character; *: Replaces one or more characters).
Example: "a?b*"
Converts files with a first letter "a" and a third letter "b”.

Usage Examples

This section describes how to use the binary intermediate file converter.

Converting the Text Intermediate File

Converts text.fmda to text.fmdb.

3dIntermediateFileAsciiToBinary.exe text.fmda

Converting Folders

You can convert all of the text intermediate files in the text folder and its subfolders to binary intermediate files.

3dIntermediateFileAsciiToBinary.exe text\
