Introduction

Purpose of This Documentation

This documentation describes the ec library features included in NintendoSDK.

Intended Audience

This documentation is intended for people developing applications using NintendoSDK.

Related Documentation

Also see the ec API Reference.

How the ec Library Is Provided

See the following pages.

  • How the Libraries Are Provided and How to Use Them

The ec library is currently only provided for the NX.

Features Provided by the ec Library

Utilities for Accessing eShop Services

The following utilities for accessing eShop services are provided.

  • Feature for getting a list of titles and bundles including titles that are currently available for purchase on Nintendo eShop
  • Feature for getting a list of downloadable content that is currently available on Nintendo eShop
  • Feature for getting a list of consumable service items currently available on Nintendo eShop

Getting Lists of Titles, Bundles Including Titles, Downloadable Content, and Consumable Service Items Currently Available on Nintendo eShop

The ec library provides features for getting lists of products available on Nintendo eShop to make purchasing titles, bundles including titles, downloadable content, and consumable service items from within applications a smoother process.

The account library (account Library) must be initialized in advance before using the features for getting lists of products on Nintendo eShop.

The ec library provides only a means for accessing eShop services.
When actually getting lists of products on Nintendo eShop, you must be ready to refer not only to the SDK reference but also to the reference for the server that provides shop services.

In the E-Commerce Guide, see Features for Getting Information Within Application: Titles and Bundles Including Titles, Downloadable Content, and Consumable Service Items.

Feature to Jump to Nintendo eShop

The following features are provided for jumping to Nintendo eShop.

  • Feature to jump to a screen listing and selling downloadable content currently being sold on Nintendo eShop
  • Feature to jump to a screen listing and selling subscription service items currently being sold on Nintendo eShop
  • Feature to jump to a screen listing and selling consumable service items that are currently being sold on Nintendo eShop.
  • Feature to show the detailed information of applications in the Nintendo eShop, and jump to the sales page for the application.
Note

When jumping to the shop screen, you must have a user account associated with a Nintendo Account. The ec library performs this check before jumping to the shop. If the selected user account exists and has been associated with a Nintendo Account, that user account is used. If no user account satisfies these conditions, or if the user account indicated by the value specified for the argument does not satisfy the conditions, the account management applet is called and prompts you to associate the user account with a Nintendo Account.

Listing and Selling Downloadable Content, Subscription Service Items, and Consumable Service Items

The ec library provides features for jumping to the eShop purchase screens to easily purchase certain downloadable content, subscription service items, and consumable service items from inside the application. This feature uses the ec applet, which is a library applet. You jump to the ec applet by calling a particular ec library function.

To use the jump feature to the shop purchase screen, you must have initialized the account library (account Library).

Displaying Detailed Application Information and Selling the Application

The ec library also provides a feature to display the detailed information for the application itself, and jump to the screen where the application can be purchased. This feature is also provided by using the ec applet, and performs the jump by calling a specific function. For example, from the demo version of the application screen, you can specify the application ID of the retail version application and call nn::ec::ShowShopApplicationInformation to display the detailed information for the retail version of the application.

This feature also requires that the account library is initialized ahead of time.

Feature for providing consumable service items without an independent server

The ec library provides a client library to enable the user save data controlled method, which is a method for providing consumable service items without an independent server.

For more information, see Consumable Service Items Client Libraries.

Feature for Detecting the Purchase of Products in the Shop

The ec library provides a feature for detecting the purchase of products in the shop.
You can detect when subscription service items and consumable service items are purchased.

Other

For an overview of the downloadable content, subscription service items, and consumable service items handled by the ec library, see the E-Commerce Guide. For more information about the aoc library that is used when using downloadable content inside the application, see aoc Library.

The feature to jump to the purchase screen uses the ec applet, which is a library applet. For an overview and specifications of library applets, see the section about library applets in the System Features Guide.
