Overview

WebAppletCaller is a target-oriented tool for calling the web applet and Offline Web Applet to preview web content that is in development.

Include settings like web content and options in the file. Select the file to use from the menu. Also, there is a way to configure all settings from the command line.

Offline HTML displayed in the Offline Web Applet is loaded from a host PC or SD card.

If you install the NSP file on the tool onto the development hardware and place the configuration files and offline HTML on the SD card, you can use it without having to connect to a host PC.

Info

This tool is available in NintendoSDK NX Add-On 10.3.0 (Firmware for NX 10.0.0-4.0) and later.

Location

This tool is included with NintendoSDK NX Add-On and is found in the following location.

TargetTools/NX-NXFP2-a64/WebAppletCaller/Release/WebAppletCaller.nsp

Important Usage Information

Root Paths for the Host PC and SD Card

WebAppletCaller loads configuration files and offline HTML from a host PC or a SD card.

By default, the host PC is mounted with C:/NXWebApplet as the root path. This host PC mount path can be changed using the settings manager. For more information, see Changing the Host PC Path Mounted as the Root Path.

The SD card is mounted with /NXWebApplet as the root path.

Offline HTML Placement and Path Specification

Instead of displaying the offline HTML page attached to the ROM, the WebAppletCaller displays the offline HTML page stored either on a host PC or a SD card.

Create a html-document folder in the root path of the host PC or SD card, and place the offline HTML file in that folder.

In the WebAppletCaller, specify the relative path from the html-document folder as the path for the offline HTML. For example, if you put sample.htdocs/index.html in the html-document folder, you would specify sample.htdocs/index.html as the documentPath regardless of whether the file is stored on a host PC or SD card.

If the same offline HTML file is present both on the host PC and on the SD card, the file on the PC is prioritized.

Note

Whether the path to offline HTML stored on a PC or SD card is case-sensitive depends on the PC environment and SD card format, but offline HTML attached to the application is always case-sensitive (distinguishing between uppercase and lowercase, such that files with names with uppercase and lowercase distinctions can be placed in the same directory).

Even when using WebAppletCaller to handle offline HTML stored on a PC or SD card, it is safest to assume that the application will have offline HTML added, and handle paths to offline HTML in a manner that accounts for case-sensitivity.

Selecting and Using Configuration Files

Use RunOnTarget to start the tool.

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebAppletCaller\Release\WebAppletCaller.nsp

You can also install the NSP file on the development hardware and use the tool. For more information about installing the NSP file, see Installing Applications on Development Hardware.

A list of configuration files is displayed when you start the tool. Selecting the configuration file of the web content you want to display calls the web applet and Offline Web Applet.

If there is an error in the format of the configuration file or the parameter settings, an error dialog box is displayed.

Configuration File

When writing web content to display and configuring options, use the JSON format.

Use .json as the filename extension of the configuration files. If you are placing the configuration files on a SD card, do not use any multibyte characters in the filename.

Configuration files can be placed either on a host PC or a SD card. Create a settings folder in the root path of either the host PC or the SD card and place the files in that folder.

A maximum of 128 configuration files can be placed in this folder.

Display of Settings

Click the icon on the right in the list of configuration files to display the content of a configuration file.

If there is an error in the format of the configuration file, an error dialog box is displayed. If there is an error in the parameter settings, the error in the pertinent parameter is shown in red.

Info

Available in NintendoSDK NX Add-On 14.0.0 and later.

Calling the Web Applet

Enter the requestUrl in the configuration file and call the web applet.

Specify the URL of the webpage you want to display as the requestUrl parameter.

For more information about optional parameters you can set, see List of Web Applet Configurations and Options.

The following example shows how to open http://www.nintendo.com using a user with index number 0.

{
  "requestUrl" : "http://www.nintendo.com",
  "userIndex" : 0
}

The following example shows how to specify the applet to close when it reaches http://www.nintendo.com/countryselector after opening http://www.nintendo.com.

{
  "requestUrl" : "http://www.nintendo.com",
  "callbackUrl" : "http://www.nintendo.com/countryselector"
}

Calling the Offline Web Applet

Enter the documentPath in the configuration file and call the Offline Web Applet.

Specify the offline HTML path you want to display as the documentPath parameter.

For more information about optional parameters you can set, see List of Offline Web Applet Configurations and Options.

The following example shows how to open the index.html file located in the sample.htdocs folder.

{
  "documentPath" : "sample.htdocs/index.html"
}

The following example shows how to disable the feature for switching the Left Stick mode to pointer mode and also hides the footer, and then opens the index.html file located in the sample.htdocs folder.

{
  "documentPath" : "sample.htdocs/index.html",
  "pointerEnabled" : false,
  "footerEnabled" : false
}

Specifying and Using Configuration on the Command Line

Use RunOnTarget to start the tool.

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebAppletCaller\Release\WebAppletCaller.nsp -- --help
usage: WebAppletCaller <command> [<args>]

The WebAppletCaller commands are:
   web <requestUrl>       ShowWebApplet
   offline <documentPath> ShowOfflineApplet
[SUCCESS]

Calling the Web Applet

Specify the web command and call the web applet.

Specify the URL of the webpage you want to display as the requestUrl parameter.

For more information about optional parameters you can specify, see List of Web Applet Configurations and Options.

   [--js-extension-enabled <true|false>]
   [--display-url-kind <FullPath|Domain>]
   [--user-agent-additional-string <userAgentAdditionalString>]
   [--touch-enabled-on-contents <true|false>]
   [--page-cache-enabled <true|false>]
   [--page-fade-enabled <true|false>]
   [--page-scroll-indicator-enabled <true|false>]
   [--media-player-speed-control-enabled <true|false>]

   [--web-audio-volume <volume>]

The following example shows how to specify the applet to close when it reaches http://www.nintendo.com/countryselector after opening http://www.nintendo.com.
