gll Library

Overview

The gll library provides features to batch load OpenGL cores and extension commands.

Objective

OpenGL commands that are platform dependent and cannot be statically linked must be loaded using a platform-specific load function. This operation can be performed manually, but normally all commands are loaded simultaneously through a dedicated load library immediately after generating the OpenGL context.

Many OpenGL load libraries already exist in the public domain and could be used. However, they may not be optimized for platforms supported by NintendoSDK. For example, you may need to perform some extra processes such as manually loading specific commands separately. The purpose of the gll library is to absorb any differences in platforms supported by NintendoSDK and allow all of these platforms to use the library.

The gll library loads OpenGL commands in a C interface format to maximize compatibility with other load libraries. This allows you to migrate applications that are using other load libraries to the gll library with minimal modifications.
