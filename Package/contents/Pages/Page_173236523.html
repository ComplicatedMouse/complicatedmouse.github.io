Z-Fighting Suppression

This page describes the z-fighting suppression feature, which reduces the flickering effect that occurs when a particle is drawn at the same position as another polygon.

Overview

This feature suppresses z-fighting by adding a fixed value to the particle's depth information so it gets rendered slightly further in the foreground.
This makes it possible to suppress z-fighting, which is a flickering effect that occurs when particles are drawn at the same position as another particle.
Using this feature removes the need to micro-adjust the positions of emitters in the application to avoid this problem.

Difference Between Z-Fighting Suppression and Particle Offset

Z-fighting suppression is different from the particle offset feature in that it does not adjust the actual depth relationships between polygons.
As a result, the value that is added to particles’ depth values is fixed to the value set in the runtime.

The Offset Z Value feature provides a similar functionality by allowing you to specify micro-adjustments as offset values.
However, the units of the offset values specified when using the Offset Z Value feature are in the world coordinate system, and as a result, the final offset applied to the depth value varies according to the camera distance.
As a result, when the distance between the camera and the effect becomes large, z-fighting once again becomes a problem due to precision limitations for depth values (even when using offset values that work well at smaller distances).

The z-fighting suppression feature works by adding a fixed offset to the depth value in conjunction with camera distance, preventing potential issues that can be caused by the distance between the effect and the camera.

The following overview describes the settings ().

Z-Fighting Suppression

Select or clear the check box to enable or disable z-fighting suppression ().
This setting can also be changed programmatically within game applications by using the nn::vfx::ViewParam::Set() function.

Info

Note the following when using this feature.

  • As described previously, do not use this feature to adjust the actual depth relationships between polygon.
  • This feature cannot be used to suppress z-fighting within emitters. To suppress z-fighting within emitters, assign the emitter shape a nonzero thickness and configure particles to be emitted from non-overlapping positions on the emitter faces.
