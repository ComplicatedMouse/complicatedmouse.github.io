3D News

This section introduces new features added to the 3D module.

3DEditor Material referencing settings icons There are now icons that represent the various material referencing settings.
Insertion of optional text in the material settings UI It is now possible to display optional text in the position specified in the Shader page of the material Property window.
Check box UI for string-type rendering information You can now use the check box UI for string-type rendering information.
Support for commands after files are closed You can not execute optional commands after files have been closed
Maya Plug-In Support for Maya 2018 The Maya plug-in is now compatible with Maya 2018.
Shortcut menu customization You can now set a user custom menu to display when you right-click on the export item list.
Importing the multi-export settings

You can now export and import the multi-export settings.

The Start/End Frame export items can be moved forward and backward.

It is now possible to move the range of a registered animation forward and backward.
Improved the specification for bone compression

Bone visibility animation now can be retained when Cull bone compression and Merge bone compression are used.

Added a feature to enter optional text in the Shader page of the material properties window.

By coding a text block in the shader source annotation, you now can display optional text in a specified position, as demonstrated below.

For example, text is displayed by adding the following text block to the shader annotation.

// @@ textblock text="Inserted text" order="0" group="diffuse_group"

Support for a check box UI for string-type rendering information.

Previously, only the combo box UI was supported for string-type rendering information. But now, a check box UI, as shown below, can be displayed for string-type rendering information with two values.

Fore example, a check box is displayed when check is specified for item in the shader annotation for the rendering information. When this check box is selected or cleared in 3DEditor, the string true or false is sent to the connected application.

// @@ renderinfo="checkbox_test" type="string" default="false" choice="false, true" optional="false" item="check" label="The string 'true' or 'false' will be sent to the runtime, depending on the state of this check box."

Maya Plug-In

Support for Maya 2018

The Maya plug-in is now compatible with Maya 2018. This version of Maya can be launched using the following batch file.

%NINTENDO_SDK_ROOT%\NintendoSDK\Tools\Graphics\MayaPlugins\StartMaya-2018.bat

Related Links For more information, see the following pages.

Maya Operating Environment

Added a feature for customizing the shortcut menu of the Multi-Export plug-in.

By coding the following in the config file, you can display a user-customized shortcut menu when you right-click the export item list in the Multi-Export plug-in.

multi_export_user_custom_list_menu0="menu name, procedure name, type, comment"


For example, the menu item My Export is displayed when you right-click the export item list.

multi_export_user_custom_list_menu0="My Export,MyNwMultiExportCustomData,multi,Export custom data"

Added a feature to the Multi-Export plug-in for moving the Start/End Frame export items forward and backward.

It is now possible in the Multi-Export plug-in, to move the range of a registered animation forward and backward.

Right-click to open the Move Start / End Frame window, and then enter a number for the frame that you want to move. (Negative values are allowed.)

Improved the specification so bone visibility animation can be retained when Cull bone compression or Merge bone compression is used.


The specification for bone compression has been changed such that if bone visibility animation has been created for a bone set as incompressible, that bone visibility animation will be retained when Cull bone compression or Merge bone compression is used. Previously, it was not retained. For a description of the specification for bone compression, see Bone Compression.

For example, a schematic that depicts the exported result when bone visibility animation is attached to Group, and it is made incompressible.
　　The shape is below Group in the hierarchy, and the bone visibility animation is retained.
