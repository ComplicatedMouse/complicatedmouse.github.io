Introduction

WebCallerCommand is a target-oriented tool for calling the web applet and Offline Web Applet from the command line.
This tool can be used to do the following.

  • Start the web applet.
  • Start the Offline Web Applet.

Usage

Use RunOnTarget to start the tool.

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- --help
usage: WebCallerCommand <command> [<args>]

The WebCallerCommand commands are:
   web <requestUrl>       ShowWebApplet
   offline <documentPath> ShowOfflineApplet
[SUCCESS]

Calling the Web Applet

Use the web command to call the web applet.

$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\WebCallerCommand\Release\WebCallerCommand.nsp -- web --help
usage: WebCallerCommand web <requestUrl>
   [--callback-url <callbackUrl>]
   [--callbackable-url <callbackableUrl>]
   [--boot-as-media-player <true|false>]
   [--media-player-auto-close-enabled <true|false>]
   [--web-audio-enabled <true|false>]
   [--pointer-enabled <true|false>]
   [--left-stick-mode <SpatialNavi|StickPointer>]
   [--user-index <userIndex>]
   [--footer-fixed-kind <Auto|Shown>]
   [--boot-display-kind <Default|White|Black>]
   [--background-kind <Normal|ApplicationCapture|ApplicationCaptureBlur>]
   [--js-extension-enabled <true|false>]
   [--display-url-kind <FullPath|Domain>]
   [--user-agent-additional-string <userAgentAdditionalString>]
   [--touch-enabled-on-contents <true|false>]
   [--page-cache-enabled <true|false>]
   [--page-fade-enabled <true|false>]
   [--page-scroll-indicator-enabled <true|false>]
   [--media-player-speed-control-enabled <true|false>]

   [--web-audio-volume <volume>]

[SUCCESS]

Examples

Offline HTML File Paths

WebCallerCommand can also open offline HTML pages saved on a PC or SD card instead of opening offline HTML pages included in a ROM.

By default, the web applet and the Offline Web Applet are mounted with the C:/NXWebApplet path on the host PC as the root path. This host PC mount path can be changed using the settings manager. For more information, see Changing the Host PC Path Mounted as the Root Path. To place offline HTML files on the PC, create a folder named html-document in the host PC mount path and place the files in this folder.

The SD card is mounted with /NXWebApplet as the root path. To place offline HTML files on the SD card, create a folder named html-document in the SD card mount path and place the files in this folder.

Specify the paths of offline HTML pages to open as relative paths from these folders. For example, if you placed a file named sample.htdocs/index.html in these folders, you would specify sample.htdocs/index.html for documentPath both for a PC and for the SD card. If the same offline HTML page is present both on the PC and on the SD card, the file on the PC is prioritized.

Note

Whether the path to offline HTML stored on a PC or SD card is case-sensitive depends on the PC environment and SD card format, but offline HTML attached to the application is always case-sensitive (distinguishing between uppercase and lowercase, such that files with names with uppercase and lowercase distinctions can be placed in the same directory).

Even when using WebCallerCommand to handle offline HTML stored on a PC or SD card, it is safest to assume that the application will have offline HTML added, and handle paths to offline HTML in a manner that accounts for case-sensitivity.

Examples

Opens the index.html file located in the sample.htdocs folder.

Opens the index.html file located in the sample.htdocs folder, but disables the feature for switching the Left Stick mode to pointer mode and also hides the footer.
