Nintendo SDK 0.4.0

The following known bugs are confirmed to occur as of Nintendo SDK 0.4.0.

  • The PackageManager version update feature does not work correctly.
  • PackageManager generates an error when uninstalling packages and leaves empty directories behind.
  • A dialog box appears when you install NintendoSDK-Documents.
  • Changes to the NINTENDO_SDK_ROOT environment variables are not applied.
  • Visual Studio sometimes behaves strangely right after a new application has been created.

The PackageManager version update feature does not work correctly.

The features to Update and Replace earlier versions do not work correctly in PackageManager.
If you use these features, packages will become invalid, with no consideration of package dependencies or existing versions.
To install the latest versions, first uninstall all packages and then install the new versions.

PackageManager generates an error when uninstalling packages and leaves empty directories behind.

There is a bug when you uninstall packages that causes an error like the one below to display in the process log and empty directories to be left undeleted.

Deleting an empty directory.
Process failed.
The generated exception was System.UnauthorizedAccessException.

The workaround for this bug is to execute PackageManager with administrator privileges.
(As the admin, right-click the executable file and select from the menu.)

A dialog box appears when you install NintendoSDK-Documents.

The following dialog box appears when you install NintendoSDK-Documents from the package manager, and the installation process is suspended.

Click Yes to proceed with the installation.
If you are installing this together with Visual Studio 2012 or 2013, the dialog box might be displayed twice.

Changes to the NINTENDO_SDK_ROOT environment variables are not applied.

Sometimes the changes to NINTENDO_SDK_ROOT are not applied in the package manager, and the old setting for NINTENDO_SDK_ROOT remains in place.
If this bug occurs, you must log out and then log back in.

Visual Studio sometimes behaves strangely right after a new application has been created.

The following problems sometimes occur immediately after you create a new application as instructed in the tutorial.

  • IntelliSense does not operate correctly.
  • When you attempt to start the application, an error message appears, stating that the specified path cannot be found.

If this bug occurs, you can resolve the problem by closing the project file and then opening it again.
