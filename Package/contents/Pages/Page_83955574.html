Startup Path

Overview

Multiple settings can be configured for the startup path in both the User Settings and the Team Settings.

You can configure 3DEditor to open the following files on startup by setting a startup path.

  1. Intermediate files in the folder specified for the startup path, including any subfolders
  2. Intermediate files and the project file in the folder specified by the startup path
  3. Intermediate files and the project file in the folder named Startup located in the same folder as 3dEditor.exe, including subfolders
Info

Files opened on the startup path are not closed when Close All is selected from the File menu of 3DEditor.
If you want to close a file from within 3DEditor, select that file and select Close on the File menu.

Startup Path Setting

The startup path can be set as a path to a directory or a file.
Path strings may include environment variables surrounded by percent signs (%). 3DEditor interprets such paths by replacing any environment variables surrounded by percent signs (%) with the value the environment variable represents.

Just as with standard file loading, you can undo files loaded at startup.

Note the following when setting the startup path.

  • To open a project file at startup, do not include settings for opening any other file.
    This is not just limited to startup. You cannot open any other files at the same time you open a project file.
  • Specify the startup path as an absolute path or as a relative path from the additional reference path.
