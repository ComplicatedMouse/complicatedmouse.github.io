Project - 3ded

Project

Overview

Only one project exists at a time when 3DEditor is running, and it is used to manage 3DEditor tasks.

A project maintains the following information:

  • The paths to files that are open in 3DEditor
  • Animation binding information
  • Animation set configuration and binding information
  • Information for model preview
  • The name of the model to retarget skeletal animation to
  • Editing comments and editing colors to show in the project Property window
Info

3DEditor creates an empty project named Project at startup.

Saving Projects

Projects are saved to project files (F3PJ files).
The filename matches the project name, except for the filename extension.

When the project is saved, the paths to all files opened on 3DEditor become fixed.

Filed Paths of Saved Project Files

  • If a file has been opened via an additional reference path set to an absolute path, the file is saved to a relative path from that additional reference path.
  • Files on the same drive as the project file are saved using relative paths.
    Note: Files on different drives are saved using absolute paths.
Info

You cannot exclude specific files from a project when saving.
Also, files opened from the startup path in the Startup folder are saved using a relative path to that folder, rather than a relative path to the project file.

For more information about the format of project files, see here.

Opening Projects

When you open a project file, all files currently open in 3DEditor are closed and the files listed in the project file are opened.

Note: A search is made to load all textures referenced by models, even if the textures are not listed in the project file.

Info

You cannot open multiple project files at the same time or specify another file to open.
If you are opening a project file, specify the project file by itself.

A message appears if 3DEditor cannot find a listed file when it opens the project file.
Note: Files that cannot be found are excluded from the project file.

If a project file is being shared among multiple working environments, you must appropriately deal with the relative placement of files, the configuration of additional reference paths, and the configuration of environment variables included in the settings of the additional reference paths.

New Project

When you create a new project, you can choose whether to close already open files that are not part of the project.

  • If Close is selected, an empty project is created.
  • If Do not close is selected, files, animations, and animation set binding information are inherited by the new project.
