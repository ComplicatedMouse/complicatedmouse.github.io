Converting

This section describes how to create binary files for use by the atk library from the sound project created with the procedures described to this point.

Making Conversions

Making Conversions

Select Project > Convert.

Conversion generates many kinds of sound data files from your created sound projects that you can use in the NintendoWare audio development environment.
The files created with this operation represent all of the data handled by the NintendoWare audio development environment runtime library in the application.

The Convert dialog box appears during the conversion process to show the log messages for the converter.
During conversion, an animated icon displays in the upper-left part of the dialog box.

If you want to close the Convert Error dialog box even when errors occur during conversion, clear the selection from the Don't close Convert dialog box when errors occur check box.
This setting can also be made in the Options dialog box.

Reconverting Specified Items

Select Project > Reconvert Specified Item to view a dialog box from which you can specify the item to be converted.

If you reconvert a specified item, the usual conversion target (the converted item) is converted. In addition, however, it reconverts all items of the specified type.

Displaying Convert Warnings

Warnings display in the Convert dialog box and in the Convert Log window when any unreferenced items or other possibly inappropriate settings are detected during conversion.

The Convert dialog box displays the error content, and the Move to Target Item button becomes available.
To move to the selected item position where the error occurred, click Move to Target Item.

For more information about the Conversion Log window, see Conversion Log.
The warning feature can be enabled and disabled by using the Project Settings dialog box.

Generated Files

This document describes the types of files that are generated by the conversion process, and the locations where they are exported.

You need the following files to play back a sound edited in SoundMaker from a program on a development hardware.

  • Binary Sound Archive File
  • Binary stream file
  • Sound ID file (only needed when playing a sound specified by a sound ID)

Generated File Output Destination

By default, a file output folder named output is created in the folder where the sound project files are. You can change the location of this folder by defining a Sound Archive Output Location in the Project Settings dialog box.

Binary Sound Archive File

The sound archive file stores the body of the sound data as a sound archive.
The file is output to the folder specified by the Sound Archive Output Location, using the same name as the sound project file, but with the .bfsar extension.

Binary Stream File

The stream file stores the waveform data for stream sounds that are included in the sound archive.
Export to the specified folder using Stream File Output Location.
By default, files are exported to the output\stream folder within the project folder, as stream sounds, using labelName.bfstm.

The location of the binary stream file is stored as a relative path in the binary sound archive file.
The directory path below the output location is the path where the stream file is actually located in ROM, and this information is essential when the sound archive plays stream sounds.

Sound ID Files

The sound ID file is a text file that contains a list of all item names that are included in the sound archive.
The file is output to the folder specified by the Sound Archive Output Location, using the same name as the sound project file, but with the .fsid extension.

To export files with the ID headers divided out by type, select the Output Labels by Items check box in the Project Settings dialog box.
The following types of files are exported:

  • *.BANK.fsid
  • *.GROUP.fsid
  • *.PLAYER.fsid
  • *.SEQSET.fsid
  • *.SOUND.fsid
  • *.WAVEARCHIVE.fsid
  • *.WSDSET.fsid

Intermediate Binary Files

These are the intermediate binary files that are created during conversion, and include files with the .bfstm and .bfwar extensions.
These files are all exported to the cache folder within the sound project folder.

You can choose whether to export the intermediate binary files to the cache folder from Project Settings.

All files within the cache folder are deleted if the project is converted again.

HTML Sound List

An HTML-formatted sound list is automatically generated from the content of the converted project.
You can select settings to be output to the sound list from the Project Settings Dialog Box - HTML Sound List.
The list is exported to the sound project folder by using the same name as the sound project file but with the .html filename extension.

Instrument lists are not exported as HTML-formatted files during conversion. For more information, see Banks - HTML Instrument Lists Output.

MAP Files

Sound archive map files are created during conversion.

An XML-formatted file with the same name as the sound project file and an HTML-formatted file with Map appended to the end of the name of the sound project file are both exported to the Sound Archive Output Location folder.

The file can be structured in the following ways:

  • Group List
  • File List
  • Binary Map

These are the three structures.

From the MAP file, you can determine such information as the ROM size, the header size, the bank capacity, and the waveform (archive) capacity.
The following table shows an example of the relationship between Name and capacity for Binary Map items.

Name Type
SoundArchiveBinaryFile ROM size
InfoBlock Header size
../cache/BANK_BGM.bfbnk Bank size
../cache/WARC_SAMPLE.bfwar Waveform archive size
