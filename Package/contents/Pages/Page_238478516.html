Parts

The Parts feature provides the functionality for constructing a layout from an assemblage of numerous layouts.
The layout is made up of structural elements called parts layouts.
A parts layout can be placed in other layouts.
In addition, you can alter the sizes of the placed parts, and you can overwrite parts to change them in just some panes.

Benefits

Using the Parts feature for buttons and the like used in the game UI makes it easier to create, correct, and adjust data.
In addition, it makes it easier for designers to check for themselves how the layouts appear, and to try out different combinations of parts.

Problems With Conventional Methods

In game screen design, various parts and places can have similar design and animation representations. (For example, a menu with a variety of buttons.)
Problems can arise when they are produced using conventional methods. Consider the following conventional methods and the problems that can occur.

  • Make multiple copies of the same pane.
    • Making revisions is annoying.
    • Some revisions may be missed.
  • The developer makes copies at run time.
    • The ultimate display results cannot be checked by the designers themselves.
    • Much time passes between a revision and checking the result, so fine-tuning is troublesome.
    • The developer is annoyed.

Using Parts

By using the Parts feature, for the portions of the screen that are the same, you only need to create one parts layout and then place that layout in multiple locations.

To adjust the functionality of a shared part, you only need to make the revision to one place: the parts layout. When making revisions you do not need to worry about omissions.
The placement results can be checked on the Viewer, so the designers can check and make repeated adjustments themselves.

After the parts layout has been positioned, the size can be adjusted and the content can be rewritten.

This allows you to change the contents a little at a time and create many variations.

(For example, you can make buttons that exhibit the same animation functionality but have different text.)
