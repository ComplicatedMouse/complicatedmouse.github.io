NumberLineAllocator

NumberLineAllocator Class

Feature Overview

The NumberLineAllocator class is an integer allocator class that can allocate or deallocate any memory from an integral segment.

Memory of any length can be allocated from the integer memory that is indicated using an index in the range from 0 to 16777215 (0xFFFFFF).

The management space for NumberLineAllocator is managed using a dedicated callback function and is separate from the memory handled by NumberLineAllocator.


The NumberLineAllocator class has the following member functions. For more information, see the API reference.

Using NumberLineAllocator

Start by initializing NumberLineAllocator.

The functions for allocating and deallocating managed memory will be needed during NumberLineAllocator initialization, so define those callback functions.

Then call AddRange() to configure the allocatable memory region.

#include <nn/mem/mem_NumberLineAllocator.h>

// Callback function for allocating managed memory.
void* ManagAlloc(size_t size, void* pParam)
{
    NN_UNUSED(pParam);
    return std::malloc(size);
}

// Callback function for deallocating managed memory.
void ManagFree(void* addr, void* pParam)
{
    NN_UNUSED(pParam);
    std::free(addr);
}


extern "C" void nnMain()
{
    nn::mem::NumberLineAllocator allocator;
    // Initialization.
    allocator.Initialize(ManagAlloc, nullptr, ManagFree, nullptr);
    // Specify allocatable memory.
    allocator.AddRange(0, 0x01000000u);
 
    // Any of the following processes.
}


After initialization, you can perform memory operations such as allocation and deallocation.

// As an example, allocate a segment with a length of 3.
int numberLine;
bool result = allocator.Allocate(&numberLine, 3);
if(!result)
{
    // Allocation failed.
}
 
// As an example, allocate a segment with a length of 16 and an alignment of 4.
int alignedNumberLine;
bool result = allocator.Allocate(&alignedNumberLine, 16, 4);
if(!result)
{
    // Allocation failed.
}
 
// Free the memory.
allocator.Free(numberLine);
allocator.Free(alignedNumberLine);


If you are no longer using NumberLineAllocator, call Finalize() to free the allocator.

allocator.Finalize();


For more information about other functions, see the API reference for each function.

Dump Format

When the Dump() function is called, the allocator's internal data is displayed in the following XML format.

<heapinfo>
  <offset>0</offset>
  <factor>1</factor>
  <max_allocatable_size>16777214</max_allocatable_size>
  <free_memory>16777214</free_memory>
  <allocated_memory>2</allocated_memory>
  <freelists>
    <free index='255' first='0000000000000002' last='0000000001000000' size='16777214'/>
  </freelists>
  <allocs>
    <alloc level0='0' level1='0' level2='0' first='0000000000000000' last='0000000000000002' size='2'/>
  </allocs>
</heapinfo>

The following table shows the various elements.

Tags Description
heapinfo

Tag for the largest XML frame exported by Dump().
offset Offset to shift the displayed index. Specify as a parameter when the user calls Dump().
factor

The integer to specify to display with a constant multiplication factor the segment with a length of 1.

For example, when using 256 for factor, subsequent values appear, using 256 for a segment with a length of 1.
max_allocatable_size The maximum allocatable size.
free_memory The total amount of free memory.
freelists Tag that indicates the set of free memory.
free

The amount of the free memory in the allocator.

index indicates which free memory is associated with that class broken into 256, based on the length. For index values from 0 to 254, the index value + 1 is the length. A value of 255 indicates a length of at least 256 for the segmented number line.

first is the starting index for the free memory.

last is the final index, plus one, for the free memory.

size is the length of the free memory.
allocs Tag that indicates the set of allocated memory.
alloc

The allocated memory.

The indices level0, level1, and level2 indicate the position for the data structure that controls the allocated memory within the allocator. The user does not normally need to keep track of this information.

first is the starting index for the allocated memory.

last is the final index, plus one, for the allocated memory.

size is the length of the allocated memory.

offset and factor are used when a memory allocator was created that used NumberLineAllocator as an internal implementation.

When you pass the start address for the internally created memory allocator to offset and the minimum allocation size for an internally created memory allocator for which a length of 1 was assigned to factor, you can export the values used by the internally created memory allocator during a call to Dump().

Considerations When Using NumberLineAllocator

Maximum Allocations

The memory that can be handled with NumberLineAllocator is limited to a maximum size of 16,777,215 (0xFFFFFF).

As a result, a length of 1 can only be allocated 16,777,215 times, a length of 3 can only be allocated 5,592,405 times, and a length of 16,777,215 can only be allocated once.

AllocationMode Enumerated Type

By default, NumberLineAllocator cannot join memory adjacent to a region for which Free() was run, even when that memory region to join is free.

This setting can be changed by specifying the AllocationMode enumerated type in SetAllocationMode().

Management Space

Management space refers to the actual memory region needed to maintain the allocated or deallocated state for an integral segment by NumberLineAllocator.

The management space required for NumberLineAllocator uses the callback function specified during Initialize for allocation and deallocation. It is separate from the memory managed by NumberLineAllocator itself.

Cache for Allocating Management Memory

Management memory is not deallocated when Free() is called. Instead, it is cached for a set period of time. As a result, sometimes it appears that there are more instances of Allocate for the management memory region.

The management memory region maintains a set volume until the allocator is freed. Call Finalize() to deallocate all management memory.
