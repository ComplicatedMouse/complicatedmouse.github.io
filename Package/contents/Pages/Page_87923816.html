Team Settings File Format

Team settings files are stored in XML format.

This section describes the elements included in a team settings file.

<TeamSettings>

The root element of the file.

Attributes

Name Description
Version The version of the team settings file. The current version is 0.0.0.0.

Description

This element includes the following elements.

  • <FileIo>: Settings for operations when reading and writing files. One of these elements at most.
  • <Preset>: The preset settings. One of these elements at most.
  • <DefaultValue>: The default settings. One of these elements at most.
  • <Error>: The error settings. One of these elements at most.
  • <Preview>: The preview settings. One of these elements at most.
  • <Color>: The color settings. One of these elements at most.

<FileIo>

File input/output settings.

Description

This element includes the following elements.

    • <SearchPath>: Settings for the Additional Reference Path of the texture. Zero or more may appear.
    • <AutoSaveTextureSearchPaths>: Settings for the auto-saved Additional Reference Path of the texture. One of these elements at most.
    • <StartUpPath>: Startup path settings. Zero or more may appear.
    • <AttachPath>: Attach path settings. Zero or more may appear.
    • <ParentMaterialPath>: Material reference path settings. Zero or more may appear.
    • <AutoSaveMaterialSearchPaths>: Settings for the auto-saved Additional Reference Path of the material. One of these elements at most.
    • <MaterialTemplatePath>: Material template reference path settings. Zero or more may appear.
    • <DisableParentMaterialFileAccessControl>: Disables file access controls for automatically loaded parent materials. One of these elements at most.
    • <TextureCompositor>: 3dIntermediateFileTextureCompositor.exe settings. One of these elements at most.
    • <PreSaveCommand>: Specifies a program to run before saving the file. One of these elements at most.
    • <PostSaveCommand>: Specifies a program to run after saving the file. One of these elements at most.
    • <PreOpenCommand>: Specifies a program to run before opening the file. One of these elements at most.
    • <PostCloseCommand>: Specifies a program to run after closing a file. One of these elements at most.
    • <PreBinarizeCommand>: Specifies a program to run before binarizing data for transfer to the runtime. One of these elements at most.
    • <LogShaderConvertTime>: Settings for the feature that displays the optimized shader creation time in the log bar. One of these elements at most.
    • <CreateShaderVariationCommand>: Sets the command for creating the FSVA file for generating optimal shaders. One of these elements at most.
    • <ShaderConverterAdditionalArgs>: Specifies additional converter options to pass to the shader converter. One of these elements at most.
    • <ShaderConverterAdditionalArgsForOptimizedShader>: Converter option settings for additional specifications to the shade converter when creating an optimization shader. One of these elements at most.
    • <BinaryConverterAdditionalArgs>: Specifies additional converter options to pass to the binary converter. One of these elements at most.
    • <UserCommand>: Settings for an optional program to run on an intermediate file. Zero or more may appear.
    • <RuntimeUserScript>: Settings for an optional script to send to the application to connect to. Zero or more may appear.
    • <ShaderGraph>: Settings to start ShaderGraphEditor.exe. One of these elements at most.


<SearchPath>

Element that includes the settings of the Additional Reference Path of the texture.

Attributes

Name Description
Name The additional reference path name. This attribute can be omitted for a relative path.
Recursive Specifies whether to include subfolders of the folder specified by the additional reference path in searches. Set to true or false. If omitted, false is assumed.

Description

The absolute or relative path to the folder. This path may include environment variables surrounded by percent signs (%). The relative path specifies the relative path from the reference source file.

Element that includes the settings of the auto-saved Additional Reference Path of the texture.

Description

This element includes the following elements.

  • <AutoSaveTextureSearchPath>: Element that includes the settings of the auto-saved Additional Reference Path of the texture. Zero or more may appear.

<AutoSaveTextureSearchPath>

  • Element that includes the settings of the auto-saved Additional Reference Path of the texture. Zero or more may appear.

Attributes

Name Description

Recursive Specifies whether to include subfolders of the folder specified by the additional reference path in searches. Set to true or false. If omitted, false is assumed.

Description

The absolute or relative path to the folder. This path may include environment variables surrounded by percent signs (%). The relative path specifies the relative path from the reference source file.

<PreSaveCommand>

This element specifies a program to run prior to saving a file.

Targets are intermediate files and project files.
You can pass the paths for save directories as a text file (UTF-8 no BOM) to the specified program. Put each directory on its own line.

Description

The path of the program to run prior to saving a file. This path may include environment variables surrounded by percent signs (%).

<PostSaveCommand>

This element specifies a program to run after saving a file.

Targets are intermediate files and project files.
You can pass the paths for save directories as a text file (UTF-8 no BOM) to the specified program. Put each directory on its own line.

Description

<PreOpenCommand>

This element specifies a program to run before opening a file.

Targets are intermediate files and project files.
The paths to the files are written on separate lines in a text file (UTF-8 no BOM), and a path to this text file is passed as a parameter in the specified program.

Info

You can also execute the program before performing a redo operation on a file open operation, or before performing an undo operation on a file close operation.
The program does not execute when reopening a file that is already open.

Description

Specifies the path of the program to run before opening a file. This path may include environment variables surrounded by percent signs (%).

<PostCloseCommand>

This element contains settings for programs to run after closing a file.

Targets are intermediate files and project files.

Info

You can also execute the program after performing a redo operation on a file open operation, or after performing an undo operation on a file close operation.
The program does not execute when reopening a file that is already open.

Description

Specifies the path of the program to run after saving a file. The path may include environment variables surrounded by percent signs (%).

<CreateShaderVariationCommand>

Sets the command for generating shader variations for optimizing shaders. When omitted, shader variations are generated internally in 3DEditor and an optimized shader is created.

Attributes

Name Description
AllowNoOutput

Specifies whether to allow only some of the shader variations in a shader archive to be generated.

Set to true or false. If omitted, false is assumed.


While normally an error would occur if shader variations were not output, if this attribute is set to true then the creation of shader variations will be treated as having been intentionally skipped. The connecting application will be notified that creation of shader variations has been skipped.

Specifically, when calling the nn::g3d::viewer::CallbackType_ShaderAssignUpdated callback, the following arguments are passed to materials for which creating shader variations was skipped: nn::g3d::viewer::ShaderAssignUpdatedArg::MaterialShaderInfo::pResShaderArchive is NULL, nn::g3d::viewer::ShaderAssignUpdatedArg::MaterialShaderInfo::isOptimized is true, and nn::g3d::viewer::ShaderAssignUpdatedArg::MaterialShaderInfo::isOptimizationSkipped is true.


For materials that match these conditions, it is possible to assign shader archives stored in the application in the callback, in place of shader archives created in 3DEditor.

This feature shortens the shader conversion time when the 3DEditor is optimizing shaders.

Description

Sets the command. 3DEditor passes the path of the directory containing the files to be processed to the first argument of the command, "%1". The command can be written across multiple lines like a patch or update data file.

3DEditor generates shader definitions files (FSDB) and model files (FMDB) that must be processed before calling the commands into the first argument directory. Shader assignments for materials other than the materials applicable to generating optimized shaders, will be output as blank to the model. Have the generate shader variations command output the shader variations to the first argument directory in the form of ShaderArchiveName.fsva. If you want to assign multiple shader archives to the model, output multiple shader variation (FSVA) files.

<BinaryConverterAdditionalArgs>

Settings for converter options that are specified additionally when running the binary converter in 3DEditor.

Description

Specifies converter options. When running the binary converter in 3DEditor, these converter options are also specified. You can also include environment variables surrounded by percent signs (%).


<UserCommand>

Runs an optional program on the selected intermediate file in either file or object view, from the shortcut menu.

Only model intermediate files are presently supported.

Attributes

Name Description
Name

Specifies the name of the command that appears in 3DEditor. Use the forward slash (/) delimiter to display the menu in a hierarchy.
Filter Specifies the extensions files to be passed to the program. When specifying multiple filename extensions, separate them with a comma (,).
When this attribute is omitted, it is applied to all files.
Reload

Set to true to use the program to temporarily reload intermediate files in 3DEditor after processing. If omitted, false is assumed.
Args

Specifies the parameters for the instance of UserCommand to run.

The following macros are available for Args. They are available when selected using 3DEditor.

Macro Description
$(ArgFile) Stores the path to the parameter (text describing the temporary file path).
$(Materials) Stores the selected material names for the target when that target is a model intermediate file. Spaces delimit the material names. For example, Mat1 Mat2 Mat3.
$(Bones) Stores the selected bone names for the target when that target is a model intermediate file. The format is identical to that for $(Materials).
$(Shapes) Stores the selected shape names for the target when that target is a model intermediate file. The format is identical to that for $(Materials).
$(Models) Stores the model names when the target is a model intermediate file.

Treated as Args="$(ArgFile)" when omitted.
Temporary

Set to false to run a user program after passing a file path without exporting the temporary file. If omitted, true is assumed.

The file will be automatically reloaded, regardless of the Reload settings, if that file is edited.
ShortCut Specifies a shortcut key for running UserCommand. Use the following format: ShortCut="Shift+A".
FileTreeShortCut Set to true to make the file selected in file view the target for a shortcut key. Set to false to target a model selected in object view.

If omitted, false is assumed.

Description

Specifies the command to execute. This path may include environment variables surrounded by percent signs (%).

Commands can be batch-added when you use wildcards.

<UserCommand Name="Filter" Filter="fmdb, fmda" Reload="true">%SCRIPT_ROOT%\Filters\*.bat</UserCommand>

<RuntimeUserScript>

Sends an optional string and a target to an application while connected to 3DEditor and that application. The application then runs that string.
The application parses the string as needed. You can specify what to run using a callback from the g3d viewer library. For more information about application implementations, see Running Optional Processing on Certain Models From Within 3DEditor.

Only model intermediate files are presently supported.

Attributes

Name Description
Name

Specifies the name of the script that appears in 3DEditor. Use the forward slash (/) delimiter to display the menu in a hierarchy.
ShortCut Specifies a shortcut key for running RuntimeUserScript. Use the following format: ShortCut="Shift+A".
FileTreeShortCut

Set to true to make the file selected in file view the target for a shortcut key. Set to false to target a model selected in object view.

If omitted, false is assumed.

Description

Specifies the command to execute. This path may include environment variables surrounded by percent signs (%).

<ShaderConverterAdditionalArgs>, <ShaderConverterAdditionalArgsForOptimizedShader>

Settings for the converter options that are additionally configured when 3DEditor calls the ShaderConverter. The options that 3DEditor can specify include those that can be specified using <ShaderConverterAdditionalArgs> or <ShaderConverterAdditionalArgsForOptimizedShader> within <DeviceType>, and those that can be specified using <ShaderConverterAdditionalArgs> or <ShaderConverterAdditionalArgsForOptimizedShader> within <FileIo>. <ShaderConverterAdditionalArgsForOptimizedShader> is used when creating optimization shaders. That said, <ShaderConverterAdditionalArgs>will be used when it is specified within <DeviceType> without specifying <ShaderConverterAdditionalArgsForOptimizedShader>. The same is true for <FileIo>. <ShaderConverterAdditionalArgs> is used in other situations besides creating optimization shaders. If additionally specified options overlap with options specified internally in 3DEditor, the options specified internally by 3DEditor take precedence.

Info

Use -- and = when specifying options that take values. Example: --dump-shader-source=DirectoryPath
This restriction is planned to be deprecated in the future.

Info

When 3DEditor calls the ShaderConverter to internally generate an FSDB file, it does not use <ShaderConverterAdditionalArgsForOptimizedShader>, or <ShaderConverterAdditionalArgs> in the <DeviceType>. Related specifications may change in the future.

Info

The following shader converter options are added from the team settings.

Option Condition for Adding
The value of the <ShaderConverterAdditionalArgsForOptimizedShader> element in <FileIo>.

Added if an optimized shader is created.
The value of the <ShaderConverterAdditionalArgs> element in <FileIo>. Added if an optimized shader is not created. Also added when an optimized shader is created but the <ShaderConverterAdditionalArgsForOptimizedShader> element in <FileIo> has not been configured.
The value of the <ShaderConverterAdditionalArgsForOptimizedShader> element in <DeviceType>.

Added if an optimized shader is created.
The value of the <ShaderConverterAdditionalArgs> element in <DeviceType>. Added if an optimized shader is not created. Also added when an optimized shader is created but the <ShaderConverterAdditionalArgsForOptimizedShader> element in <DeviceType> has not been configured.

In addition, 3DEditor can also add the following converter options.

Option Condition for Adding
--unified-annotation Must be added.
--dump-statistics-information Added if the setting to export compile-time information during shader optimization has been enabled in the <IO> element in the user settings.

<wrap>

The wrap settings when creating a new texture sampler.

Attributes

Name Description
u Specify wrapping in the U direction. Corresponds to the u element of wrap in the intermediate file. Specify repeat, mirror, clamp, or mirror_once.
v Specifies wrapping in the V direction. Corresponds to the v element of wrap in the intermediate file. Specify repeat, mirror, clamp, or mirror_once.
w Specifies wrapping in the W direction. Corresponds to the w element of wrap in the intermediate file. Specify repeat, mirror, clamp, or mirror_once.
ValueBehavior Specifies reference settings for common values for u, v, and w. Corresponds to the SamplerParamBehavior value in the intermediate file. Specify Override, Refer, or Default.
ChildRestriction Specifies restrictions on common child materials for u, v and w. Corresponds to the ChildRestriction element of SamplerParamBehavior in the intermediate file. Specify None, ForceRefer, or ForceOverride.

<filter>

The filter settings when creating a new texture sampler.

Attributes

Name Description
mag Specifies a filter for enlarging the texture. Corresponds to the mag element of filter in the intermediate file. Specify either point or linear.
min Specifies a filter for reducing the texture. Corresponds to the min element of filter in the intermediate file. Specify either point or linear.
mip Specifies a filter between texture mipmaps. This attribute corresponds to mip appearing under the filter element in an intermediate file. Specify none, point, or linear.
max_aniso Specifies the anisotropic filter value. Corresponds to the max_aniso element of filter in the intermediate file. Specify aniso_1, aniso_2, aniso_4, aniso_8, or aniso_16.
ValueBehavior Specifies reference settings for common values for mag, min, mip, and max_aniso. Corresponds to the SamplerParamBehavior value in the intermediate file. Specify Override, Refer, or Default.
ChildRestriction Specifies restrictions on common child materials for mag, min, mip, max_aniso. Corresponds to the ChildRestriction element of SamplerParamBehavior in the intermediate file. Specify None, ForceRefer, or ForceOverride.

<lod>

The LOD settings when creating a new texture sampler.

Attributes

Name Description
min Specifies the minimum value for LOD. Corresponds to the min element of lod in the intermediate file. Specify a value between 0 and 13.
max Specifies the maximum value for LOD. Corresponds to the max element of lod in the intermediate file. Specify a value between 0 and 13.
bias Specifies the LOD bias value. Corresponds to the bias element of lod in the intermediate file. Specify a value between -32 and 31.984375.
ValueBehavior Specifies reference settings for common values for min, max, and bias. Corresponds to the SamplerParamBehavior value in the intermediate file. Specify Override, Refer, or Default.
ChildRestriction Specifies restrictions on common child materials for min, max, and bias. Corresponds to the ChildRestriction element of SamplerParamBehavior in the intermediate file. Specify None, ForceRefer, or ForceOverride.

<UIRange>

This element provides the range of shader parameters.

Attributes

Name Description
Clamp Specifies whether to clamp the values that can be entered as shader parameters to values between ui_min and ui_max. When clamped, ui_default_min and ui_default_max define the default range of the slider bar. When not clamped, ui_min and ui_max define the default range of the slider bar, and ui_default_min and ui_default_max are ignored. Treated as true when omitted.

<Preview>

The Preview.config root element or the child element of <TeamSettings>.

Description

This element includes the following elements.

  • <Fps> : Expected FPS setting. One of these elements at most.
  • <MessageInterval> : Message send interval setting. One of these elements at most.

<Fps>

Assumed FPS setting.

Description

Specifies the assumed FPS on development hardware. When omitted, it is handled as 59.94. The value is affected by the 3DEditor time slider playback speed. In addition, the value specified in Fps is set in the animation dcc_fps newly prepared in 3DEditor.

<MessageInterval>

Message send interval setting.

Description

Specifies the interval at which messages are sent to the runtime.

This setting affects the frequency and amount of messages sent when dragging sliders or other UI elements in Curve Editor. Specify this value in whole milliseconds. When omitted, it is handled as 50.

<Color>

Sets the colors to display in 3DEditor.

Description

This element includes the following elements.

  • <GammaCorrection> : Specifies the gamma correction setting. One of these elements at most.

<GammaCorrection>

Gamma correction setting.

Description

When the shader parameter or animation parameter is displayed in 3DEditor, specifies whether to perform gamma correction.
The gamma value when performing gamma correction is 1/2.2. Specifies true or false. Treated as true when omitted.

If color is used in the linear color space, we recommend specifying true.
