Material Property Window: Shader Page

This page is used to edit the shader definition file associated with a material.

Related Links You can switch between showing and hiding the slider controls on the Shader page using the user settings in the Tools menu.

Info

The parameters displayed on the Shader page depend on the shader definition file (FSDB file) loaded into 3DEditor.

When Data Inheritance Between Shading Models Is On

Enabling data inheritance between shading models makes it so that values that are not displayed in the UI are also inherited.

Example

Sampler assignments with this state are also inherited for parameters with visible="false" in a shader definition file, as follows.

vec4 param; //@@ id="param" default="1 1 0 0" visible="false"

Info

Parameters with editable set to false are not inherited.

Displaying Controls Hidden in the Parent Material

You can switch between whether to forcibly display the shader parameter controls set to not display in the parent material.
If this check box is selected, the child material display shows the material set with Fix using the parent material value.



Editing Custom Labels

The custom labels on the shader page can be edited by right-clicking to display the shortcut menu and selecting the Edit custom label option.
You can also edit group labels.

Select Edit custom label to display the Edit custom label dialog box, where you can edit the labels.

You can also change the color of labels and add references to labels for other elements.

For more information about changing the color of labels and adding references to labels for other elements, see Label Attributes in the Shader Annotation Manual (provided separately).

Displaying the Original Label for Custom Labels

When editing custom labels, after changing a label you can check the original label.
If this check box is selected, when a custom label is changed, the original label is displayed next to it in parentheses.

Editing Shader Parameter Values

Specifies what to set for the shader parameter values. You can switch between referencing the parent material's values, overwriting the values, and using the defaults.
The setting determines which icon is displayed. The icon displays the values ultimately adopted after applying the parent material. For this reason, the value reference set in the material and the icon display might not match, depending on the referenced parent material's settings. For more information about applying a parent material, see Material Referencing.

Value Reference Icon Display Description
Reference Parent Material Values

Reference and use the values set by the parent material. (Its own configured values are disabled.)
Overwrite None Uses its own configured values. The referenced parent material values are not used.
Use Default Values

Uses the default shader parameter values defined in the shader definitions file.
Info

If the value reference setting is Reference the parent material () or Use the default value (), you can change the setting by double-clicking the icon.
(Useful when changing multiple settings to Overwrite.)

Custom Material Referencing Settings Labels

  • When a child material has no custom label but its parent material does have a custom label, the parent material's custom label is applied to the child material's custom label in the UI.
  • When a child material has its own custom label, the parent material's custom label is ignored, and the child material's custom label is used as is.
  • Custom labels for child materials remain editable at all times.

Switching Between Color Control and Numerical Control

When the type is float3 or float4 and UI type switching is set to auto-detect, you can switch between color control and numerical control.

Right-click the label, and from the shortcut menu specify color control or numerical control.

Switching Between Numerical Control and Color Control

To switch to color control, right-click the label (in this case, light direction (dir_light_dir):) and from the shortcut menu, select Display color control.
(Right-click to display the shortcut menu, and then click Display numerical control to switch to numerical control.)


Copying and Pasting Groups and Group Custom Labels, and Restoring Default Values

Right-clicking a group label opens a shortcut menu that allows you to copy and paste groups.
You can also restore the settings to their default values for each group. You can also perform operations on the group custom labels in the same way.

You can paste data as long as group_name is the same, even if the shader and shading model are different.
Related Links For more information about group_name, see the <ui_group> element in the list of elements in the 3D Intermediate File Specification Manual.

Color Labeling of Render Information When Runtime Linking

When runtime linking, the render information labels are always colored in the following ways.

  • [Yellow] Indicates render information received from the shader definitions file.
    The data can be edited even if the runtime link is disconnected.
  • [Blue] Indicates render information parameters that have been edited with 3DEditor and sent to the runtime.
  • [Orange] Indicates render information parameters sent from the runtime.
  • [Gray] Indicates render information not received from anywhere.
    Data in this state cannot be edited.

Error Displayed When Evaluating Group Condition

A warning symbol is displayed when the conditional expression for the group cannot be evaluated.
Point to the warning symbol to display the error information.

Jump Button

Use this button to jump to the animation curve for a shader parameter, texture SRT, or color animation you have created.

This button is activated if Curve Editor includes a corresponding animation curve or parameters. Clicking the jump button automatically opens Curve Editor and displays the corresponding animation curve and parameters.
The jump button appears dimmed if there is no corresponding animation curve or parameters.

Related Links In what amounts to the reverse pattern, you can use the shortcut menu in the Curve tree view in Curve Editor to display the items corresponding to the Shader page in the material property window.
