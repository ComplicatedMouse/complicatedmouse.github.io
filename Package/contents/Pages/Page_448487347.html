Notifications of Access to Shop Services

Overview

This development support feature displays an overlay notification with request information on the screen when the application accesses the shop service server.
It can be used for purposes like the following.

  • Checking whether the application is complying with the guidelines when it uses the feature to access the shop item catalog service server
  • Detecting and investigating instances of unintended access to the shop service server

Conditions for Using the Feature

This development support feature works when the following conditions are satisfied.

  • The startup menu is set to DevMenu.
  • The shop service server is accessed via nn::ec::ShopServiceAccessor.

Notes When Using This Feature

Note the following information when using this development support feature.

  • The overlay notification that accompanies each access request is displayed for five seconds. (It disappears after five seconds.)
  • If a succession of access requests are issued, the notification for the next request is not displayed until the notification for the previous request has disappeared.
  • Queuing delays the display of overlay notifications somewhat, but after a period of time the notification of an access request is not made and is ignored.

Checking the Notification Feature

  1. Start the application with the startup menu set to DevMenu.
  2. Access the shop service server via nn::ec::ShopServiceAccessor in the running application.
  3. When the system receives a request, an overlay notification appears in the upper-left corner of the DevMenu screen.

Shop Item Catalog Service Server (nn::ec::Type_Catalog)

If the request is for access to the shop item catalog service server, the overlay notification includes the following information.

  • The ID of the application that made the access request
  • The target server type name (Catalog)
  • The URL for which access is being requested


Consumable Service Item Service Server (nn::ec::Type_OwnedConsumableServiceItem)

If the request is for access to the consumable service item service server, the overlay notification includes the following information.

  • The ID of the application that made the access request
  • The target server type name (OwnedConsumableServiceItem)
