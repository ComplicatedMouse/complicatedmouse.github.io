Changes to NintendoWare for Cafe Features in Nintendo SDK NintendoWare

Features for Which Effect Data Must Be Updated

Due to changes made to NintendoWare for Cafe features in NintendoSDK NintendoWare, data that uses the following features must be updated.

Node Names Tab Name Item Name Differences when loading changing from NintendoWare for Cafe to NintendoSDK NintendoWare
Emitter Basic Settings Particle Sort
  • When using "ascending order", all will be changed to "no sort" in NintendoSDK NintendoWare.
  • The appearance of "ascending order" and "descending order" is reversed in NintendoSDK NintendoWare.
    You can make the appearances match by making the following changes.
    • Change "ascending order" to "descending order."
    • Change "descending order" to "ascending order."
Emitter Particle Infinite Lifespan In NintendoSDK NintendoWare, simply enabling the infinite lifespan feature will not actually give particles an infinite lifespan.
To fully enable the Infinite Lifespan feature, in the Emission tab, enable One Time.
Emitter Particle Particle Type

The directionality for directional polygons is different in NintendoWare for Cafe and in NintendoSDK NintendoWare.
You can make the appearances match by making the following changes.

  • Rotate particles 180 degrees, using the x-axis.
  • Rotate emitters 180 degrees, using either the x- or z-axis.
Emitter Particle Particle Offset

Specifications for the z-value offset have been changed to use the units for the distance in the world coordinate system. Until now, the offset value was arbitrary and had no unit.

Adjust the offset value when the offset is not what was intended. Parameters cannot be automatically adjusted.
Emitter Combiner Color Buffer Refraction

There is no problem when the rendering results are as intended in the latest version of NintendoWare for Cafe, but upgrades to NintendoWare for Cafe include changes to specifications for how coefficients are applied.

Make sure the visuals are as intended when suddenly updating from an earlier version of NintendoWare for Cafe.
Emitter Combiner Soft particle/fragment processing

The Volume feature was deleted from NintendoSDK NintendoWare.

If Volume is set to a value other than 0 in NintendoWare for Cafe, the display will differ from the display in NintendoSDK NintendoWare.
Emitter Color Color 0 to 1 behavior type Setting colors for consolidated stripes is not applied in NintendoSDK NintendoWare.
To set colors for consolidated stripes in NintendoSDK NintendoWare, you must reconfigure the values for Global multiplication color alpha in the Color tab.
Child Basic Settings Child follow type/Follow parent emitter The particle follow type in the basic settings in NintendoSDK NintendoWare is set to Do not follow emitter regardless of whether the check box for the feature is selected.
If Follow parent emitter is enabled in NintendoWare for Cafe, you must change the particle type to Follow parent emitter in the NintendoSDK NintendoWare basic settings.
Child Basic Settings Inheritance Settings > Emitter Assignment for Particle

In the initial version of NintendoSDK NintendoWare, the specifications had changed to use a format where Emitter Assignment for Particle was enabled by default. As a result of subsequent changes, options were added to Emitter Assignment for Particle, which now allows for settings for behavior equivalent to NintendoWare for Cafe when Emitter Assignment for Particle is disabled.

Emitter Assignment for Particle will be automatically enabled after updating the intermediate files. You can consider disabling it after reviewing the rendering content and the rendering load.

Features for Which Effect Data Does Not Need to Be Updated

The data format used for the following NintendoWare for Cafe features has been changed in NintendoSDK NintendoWare.
This data was changed to values appropriate for loading into NintendoSDK NintendoWare EffectMaker, so fixing the data is not required.

Node Names Tab Name Item Name Differences When Loading From NintendoWare for Cafe to NintendoSDK NintendoWare
Emitter Emission Time/Emission interval The value display has been changed.
Example: NintendoWare for Cafe "10" → NintendoSDK NintendoWare "11"
Emitter Emission Gravity/Strength (Emitter-time animation) The value display has been changed.
Example: NintendoWare for Cafe "0.1" → NintendoSDK NintendoWare "10"
Fields Curl Noise Parameter Weight The value display has been changed.
Example: NintendoWare for Cafe "0.01" → NintendoSDK NintendoWare "0.1"
Fields Curl Noise Noise Movement Speed The value display has been changed.
Example: NintendoWare for Cafe "1.0" → NintendoSDK NintendoWare "10"
Child Basic Settings Enable children When you load data that includes child objects, the Emitter Assignment for Particle setting in Inheritance Settings becomes enabled.
Child Basic Settings Apply parent's field/Apply parent's field

When this check box is selected for data, a field with the same values as the field applied to the parent object is created for the child object in NintendoSDK NintendoWare.

The appearance will be nearly identical to data created in NintendoWare for Cafe.
Child Emission Emission interval The value display has been changed.
Example: NintendoWare for Cafe "10" → NintendoSDK NintendoWare "11"
Child Emission Gravity/Strength (Emitter-time animation) The value display has been changed.
Example: NintendoWare for Cafe "0.1" → NintendoSDK NintendoWare "10"
