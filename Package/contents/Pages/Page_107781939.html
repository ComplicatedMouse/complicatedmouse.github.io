Introduction

About This Page

This tutorial takes you though the actual procedure of adding project configurations for the NX development hardware to an existing Visual Studio project that creates binary files for Windows.

The tutorial is for either of the following kinds of projects.

  • A regular project for Windows that does not use NintendoSDK
  • A project for Windows that uses the NintendoSDK, created using the NintendoSDK common package

Related Items

This tutorial uses NintendoSDK Configuration Manager, which is a Visual Studio extension provided in the NintendoSDK. For more information, see NintendoSDK Configuration Manager.

Notes

The steps and screenshots in this tutorial may differ in part from your actual experience depending on your Visual Studio version and environment settings.

Procedure

Advance Preparations

Use NDI to install the NintendoSDK VSI for NX package and the NintendoSDK VsExtension package in advance. After installation, restart Visual Studio.

Adding NX Development Hardware Platforms

If NintendoSDK VSI for NX is installed for Visual Studio, you can select NX32 and NX64 as project platforms. These are the platforms for building the 32-bit and 64-bit binary for the NX development hardware. Add these platforms to your project.

Open the project in Visual Studio and then open the Configuration Manager. For example, from the menu select Build > Configuration Manager.

Select <New...> from the Active solution platform drop-down list.

From the Type or select the new platform list, select NX32 to create the 32-bit binary or NX64 to create the 64-bit binary, and then click OK. If you plan to reuse the existing build settings, select the appropriate platform in Copy settings from. Select the option Create new project platform(s).

Use NintendoSDK Configuration Manager

Using the NintendoSDK Configuration Manager tool, configure the settings necessary for the use of NintendoSDK.

Starting NintendoSDK Configuration Manager

To start NintendoSDK Configuration Manager, select Tools (T) > NintendoSDK > NintendoSDK Configurations.

Configuring Settings to Use NintendoSDK

Perform the following operations on the project configuration you want to use with NintendoSDK.

  • In the Use NintendoSDK column, select the check box.
  • In the SDK Path column, click the [...] icon and enter the path for the SDK, or paste it from an existing configuration.
  • In the API Set column, set the NX API.
  • In the SDK Build Type column, set the build type you want to use.

For more information about build types, see Build Targets and Build Types.

Applying The Settings

To apply the values to the project settings, click OK.

In addition, the project sheet ImportNintendoSdk.props and a file for saving the settings named <project_name>nnsdk.xml are both created in the same directory as the project file. Do not delete these two files. In addition, if you move the project file, make sure that you move these files together with the project file.

Checking the Build

Select either NX32 or NX64 as the solution platform and determine whether you can build the project.

With NintendoSDK, different compiler tool chains are used in the Windows environment and the development hardware environment, so you may need to revise the source code. If the build does not work even after the source code has been revised, perform the following check.

Confirming the Use of Build Settings for the Development Hardware

Confirm that you are using the correct project-specific preprocessor macro definitions, include directories, library directories and other settings for builds for the NX32 and NX64 platforms.

  • If a Win32 preprocessor macro has been defined, you may not be able to get the correct compilation results.
  • If ..\LibraryProject\Win32\Release has been specified as a library directory, at link time, the linker may search for libraries for Windows instead of libraries for the NX development hardware. Correct the path to something like ..\LibraryProject\NX32\Release.

If your project was using NintendoSDK even before you added platforms using this tutorial, you must pay special attention to which library files get linked.

  • The structure of the library files provided by NintendoSDK for the Windows environment differs from the structure for the NX development hardware environment. Also, some library files provided for the Windows environment are not found in the NX development hardware environment. For more information about the libraries provided in both environments, see How the Libraries Are Provided and How to Use Them.
  • The files of the static libraries for the NX development hardware environment have the .a filename extension. Note that this differs from the .lib extension of the files for the Windows environment.

Confirming That the NintendoSDK Build System Settings Have Not Been Overwritten

The NintendoSDK preprocessor macro definitions, include directories, library directories and other settings are configured by the NintendoSDK property sheets, but the build might not work if these have been overwritten in the project. Use the following procedure to determine whether these settings have been overwritten.

Open the property sheets for the project. For example, right-click the project in the Solution Explorer and select Properties.

Open the drop-down list for a setting and select <Edit...>. Select the include directories.

In the window that opens, look at Inherit from parent or project defaults. If this check box is cleared, settings will be overwritten. Select this check box.
