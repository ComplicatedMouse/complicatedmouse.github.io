Changes to Graphics for Developers

This section describes the changes made to NintendoWare Graphics for developers in NintendoSDK 0.10.0.
NintendoWare Graphics refers to the following modules.

  • nn::g3d
  • nn::ui2d
  • nn::font
  • nn::vfx

For more information about the major changes to NintendoWare Graphics implemented in NintendoSDK 0.8.0, see Changes to NintendoWare Graphics in 0.8.0.

Major Changes in the 3D Runtime

Changed the return values for functions performing animation and texture association.

Changed the return values for functions performing animation and texture association. Until now, the return values indicated only success or failure. The new return values can now get the following states.

  • Complete success
  • Complete failure
  • Partial success
  • Partial failure


The following functions were changed.

  • nn::g3d::BoneVisibilityAnimObj::Bind()
  • nn::g3d::MaterialAnimObj::Bind()
  • nn::g3d::MaterialAnimObj::SubBind()

  • nn::g3d::SkeletalAnimObj::BindFast()
  • nn::g3d::ResBoneVisibilityAnim::PreBind()
  • nn::g3d::ResMaterialAnim::PreBind()

Main Changes in ui2d/font Runtime

Added UTF-8 functionality to nn::font.

Added functionality to support UTF-8 strings while leaving the UTF-16 functionality that had been in use intact.

The parameters and return values for functions that work with text at the character level have new specifications to expand from uint16_t to uint32_t to support characters in UTF-16 surrogate-pair planes.

For more information, see the Features of the Versions of ui2d/font Supporting UTF-8 page.

Changed the nn::font::TextureCache::CompleteTextureCache() parameters.

nn::font::TextureCache::CompleteTextureCache() no longer requires the CommandBuffer parameter. Because rendering commands will no longer be issued within the function, the restrictions on when the function can be called have been removed.

Related to this fix, the capacity of the internal buffer was reduced by almost half.

Added Feature to Pass User Data to the User Shader Update Callback

Added the ability to pass user data to the user shader update callback.
Note that, as a result of this change, there are more user data parameters for nn::ui2d::UpdateUserShaderCallback and nn::ui2d::DrawInfo::SetUpdateUserShaderCallback.

Other API Changes

The following non-compatible API changes have been made. Fix your code accordingly because build errors will occur.

  • Changed the way that TextureCache for scalable fonts is initialized.

    • Until now, the use of a variable-length initialization code complicated things when adding multiple fonts. Adopting a fixed-length array simplified that.

    • For more information, see the FontSimple sample page and the Font Pairs and Multiple Scalable Fonts page.

  • Combined the parameters for nn::ui2d::viewer::Viewer::Initialize() into InitializeArg.
