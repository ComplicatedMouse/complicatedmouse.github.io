Creating Patches

Introduction

This section describes the steps for creating patches from VSI.
For more information about using the created patches, see Building and Running a Patch.

Preparations

Prepare a pre-patch version of the NSP file.
If the patch you are creating is the second or later version of the patch, also have the previous version of the patch file ready.

Creating the Patch

To create the patch, configure the following settings for Project Properties > Authoring > Patch and then build the patch.
The patch file will be output as $(target name).patch.nsp.

  • Set Generate Patches to Yes.
  • For Initial NSP file, specify the pre-patch version of the NSP file.
  • If this version of the patch will be the second or later version, in Previous NSP file, specify the previous .patch.nsp file.
Warning

When the patch is built, the current version of the NSP file will be output as the intermediate file $(OutDir)$(target name).nsp.
For this reason, the patch file will not be created as intended if you specify the same path for Initial NSP file or Previous NSP file as you did for $(OutDir)$(target name).nsp.
Make sure that the file you specify for Initial NSP file or Previous NSP file is located in a different path.

Cache

When generating the patch, the intermediate output will be output to the path specified in Project Properties > Authoring > Patches > Cache Directory.
Having this intermediate output available, reduces the time required to generate the next patch.
The default value for this parameter is %USERPROFILE%\AppData\Local\Nintendo\Cache\Patch.
