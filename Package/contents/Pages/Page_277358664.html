Testing Environment


Perform the final testing before submitting a master ROM in the following environment.

  • Use the Versions of the SDK and Firmware That Are Approved for Master ROM Submission
  • Using the Controller Firmware Required for Testing
  • Using an EDEV or ADEV Set to PROD MODE
  • Using NX Development Cards
  • Using the Appropriate Platform Region

Use the Versions of the SDK and Firmware That Are Approved for Master ROM Submission

The version of the SDK and the firmware required for testing and master ROM submission changes periodically and according to the current release version of the SDK.
Test your applications using the versions of the SDK and firmware listed on the Master ROM Submission Information page in the Nintendo Developer Portal.

Using the Controller Firmware Required for Testing

The version of the controller firmware required for testing changes periodically, according to the current release version of the firmware.
Test your applications using the latest version of the firmware listed on the Master ROM Submission Information page in the Nintendo Developer Portal.

For more information about the controller firmware, see the following pages in the documentation.

  • Controller Firmware Release Notes
  • DevMenu > Controller Management

Depending on the SDK version, applications that use the irsensor library may require a different version of the controller firmware, so a firmware update may take place while the application is running. To verify that the application can run without trouble even if the controller firmware gets updated, test the behavior of your application when the system is connected to a controller with the version of the firmware at the time of the product launch.

For more information about rolling back the controller firmware to the firmware at the time of the product launch, see Rolling Back the Controller Firmware For Debugging Purposes.

Using an EDEV or ADEV Set to PROD MODE

EDEV and ADEV devices can be used for debugging and for using various development features such as connecting to Target Manager. However, you can disable these features and make the development hardware operate in essentially the same way as a retail unit by setting the mode to PROD MODE. As a final test, make sure that your application runs correctly on an EDEV or ADEV set to this PROD MODE.

See the following page for more information about PROD MODE and how to enable it.

  • Initialization for PROD MODE, the Firmware That Approximates Retail Hardware

If a problem, such as the application stops responding, only occurs in PROD MODE, verify whether it is using development features that are not available to retail version software (such as htc Library or pcm Library).

Using NX Development Cards

When developing applications that will be distributed as card applications, always test application functionality using an NX development card.

This testing is not required for applications that are only sold as a downloadable version, but is required for applications that are sold in both downloadable and card versions.

Using the Appropriate Platform Region

When verifying behavior, set the platform region corresponding to the sales region for the application.

For more information about platform region, see the following page.

  • Testing Environment

You do not need to verify behavior for platform regions different from the sales region.

For example, for applications to be sold outside of China, behavior does not need to be verified with the platform region set to China.
