Handling Textures With an Odd Number for the Pixel Size

Although the system can handle textures that have a size that includes an odd number of pixels for elements such as picture panes, there are some considerations.

Images can appear blurry because of filtering under the following conditions if you display a texture that has a size that includes an odd number of pixels dot-by-dot on a picture pane.

  1. The texture size is odd.
  2. The texture size and pane size are the same (dot-by-dot).
  3. The horizontal and vertical direction settings for the center coordinate of the pane are set to center.

Use the following formula to find the upper-left coordinate value under condition 3.
Upper left coordinate = Pane Location - (Pane Size / 2)

Consider an example where the pane location is (100, 100), the pane size is (39, 39), and the upper-left coordinate is (80.5, 80.5).

These settings introduce a fractional decimal component into the coordinate value. The pixels of the texture are offset compared to the pixels in the framebuffer. As multiple pixels in the texture image are sampled, the image appears blurry.

To prevent this situation, you must either use texture sizes that are even, or round everything past the decimal point of a pane position to .5.

Functionality Differences for Texture Formats of Indirect Textures

The functionality of indirect textures varies depending on their texture formats (or, more accurately, the number of format channels they have).

1-channel texture formats (BC4, L8, A8)―The same values for s and t are used as indirect offset values. 2-channel texture formats (BC5, LA8)―For indirect offset values, R and A are used as s and t, respectively. 3-channel texture formats (BC1)―For indirect offset values, R and G are used as s and t, respectively. The B value is used as the value for alpha elimination after applying indirect textures. For more information, see Optimizing Processing of Masks or Alphas in a Pane That Uses Indirect Textures. 4-channel texture formats (BC2, BC3, RGBA8)―For indirect offset values, R and G are used as s and t, respectively. The A value is used as the value for alpha elimination after applying indirect. The B value is not used. For more information, see Optimizing Processing of Masks or Alphas in a Pane That Uses Indirect Textures.
