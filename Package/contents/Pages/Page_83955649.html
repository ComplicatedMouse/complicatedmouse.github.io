Configuring Batch Files to Run Before and After Conversion

You can run specified batch files (.bat) before and after conversion.

You can use batch files, for example, to always copy the binary file created by the conversion to another location, or to change the file name.

This section describes the procedure for executing a batch file before and after conversion.

Configuring Batch Files to Start

Use the Project Settings dialog box to set the batch file that you want to start.

To open the Project Settings dialog box, select Project > Project Settings.

In the Project Settings dialog box, select the Convert Event tab. Enter the path of the batch files to run before conversion in Command before conversion, and enter the path of the batch files to run after conversion in Command after conversion.
You can use either an absolute or a relative path. If you specify a relative path, it is parsed as a relative path from the directory where the project file is located.

You can temporarily disable a command by clearing its check box.
Starting from the next conversion, the batch file that you specified for a command runs if the check box for that command is selected.

Info

Interactive input that requires input during execution is not supported when you use Command before conversion or Command after conversion.

Reference to the Output Filepath List

Specify the %OUTPUT_LIST_FILE_PATH% environment variable after the covert command to include an absolute path to a file with a list of the files converted (the output file path list).

This list will contain the absolute paths of all of the files of the following types exported by the converter (one per line).

  • BFSAR
  • BFSTM
  • BFGRP
  • FSID
  • XML (map file)
  • HTML (map file, HTML sound list file)

This environment variable can only be used after the convert command, and never before.
