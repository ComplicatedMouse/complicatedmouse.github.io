Installing Applications on Development Hardware

You can test the NX application you have developed without connecting to a host PC by installing and running it on the development hardware.

This section describes how to install and run your NX application on the development hardware.

Warning

Currently, most of the sample programs, when installed on the development hardware using the following procedure, will not run normally or will show nothing on the screen even if they do run.
You can use the following sample programs to test this procedure.

  • GfxShaderCompile
  • GfxSimple
  • NvnTutorial01GettingStartedWithNVN
  • HidSimple
Info

If the Installed Application Will Not Run

Confirm that your application satisfies the following conditions.

  • The application operates as expected when no program arguments are passed at startup.
    • Currently, no program arguments are specified when an application that has been installed using the following procedure is run.
  • The application operates as expected when the feature for communicating with the host PC is not used.

Creating an Installable Application Image

To install your NX application on the development hardware, you must create an installable image of the application in the form of an NSP file.

To output an NSP file, from the NX application select Project Configuration Properties > General > Application Program Format > Nca, and build the file.

Info

NSP files can also be created by using the command line with the AuthoringTool.
If you plan to create the NSP file by using the command line or by constructing your own build system, Creating Application Imagessee Creating an Independent Build System.

About the Application ID

An application ID must be specified for these subsequent operations. For these operations, specify the NintendoSdkMeta/Core/ApplicationId parameter in the NMETA file (XML format) that was specified when the NX application was built.
If you want to install several applications at the same time, change out the specified NMETA file for each NSP file you create.

Info

Starting with NX Add-On 3.0, the default root directive for NMETA files has been changed from Meta to NintendoSdkMeta.

Although you can continue to use Meta, note that going forward, samples and documentation will assume use of NintendoSdkMeta for NMETA files.


Installing Applications

Use DevMenuCommand to install the application.

Use the following commands to install the NSP file that was just created.
Note that the path for the specified NSP file must be an absolute path.

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- application install %NspFile%

Use the -s option to specify the location where the application will be installed.
To run the installed application, use either DevMenu or DevMenuCommand.

You can also install NX applications on the SDEV or EDEV console, and run NX applications on the development hardware alone.
For more information, see DevMenu - Managing Applications and the following references.

  • SDEV Operations Manual
    • Installing Applications
  • EDEV Operations Manual
    • Installing Applications

Running the Installed Application

To run the installed application, use either DevMenu or DevMenuCommand.
Operations using the debug gamepad and the touch screen are supported. Currently, some touch screen operations are not supported.

In some situations, such as when an application is written to an NX development card and to the internal system storage, applications with the same ApplicationId may be present in multiple storage regions.
When this happens, the storage region used by the system is determined in the following order of priority.

  1. NX development card
  2. SD Card
  3. Internal storage

Running Using DevMenu

After the development hardware is powered on, DevMenu starts and displays a list of installed applications.

Press right on the +Control Pad to move the focus to the screen with the list of applications.
Select the application you want to run and then press the A Button to run it.

Running Using DevMenuCommand

Use the following commands to check for the IDs of the installed applications.

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- application list

Then, use the following command to run the application with the specified ID (which for this example is 0x0100 f001 0348 8000).

%NINTENDO_SDK_ROOT%\Tools\CommandLineTools\RunOnTarget.exe ^
%NINTENDO_SDK_ROOT%\TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp ^
-- application launch 0x1000f00103488000
