Introduction

About This Page

Complete this tutorial to configure the settings to use NintendoSDK from an existing, regular Visual Studio project that creates binary files for Windows.

Related Items

This tutorial uses NintendoSDK Configuration Manager, which is a Visual Studio extension provided in the NintendoSDK. For more information, see NintendoSDK Configuration Manager.

Notes

The steps and screenshots in this tutorial may differ in part from your actual experience depending on your Visual Studio version and environment settings.

Procedure

Advance Preparations

Use NDI and install the NintendoSDK VsExtension package in advance. After installation, restart Visual Studio.

(Optional) Adding a Configuration for NintendoSDK

Info

Follow these steps to add a new configuration to your project if you want to be able to use the one project to build both the existing binary that does not use NintendoSDK and the new binary that does use NintendoSDK. This step can be omitted if you do not mind overwriting the existing binary build settings. Proceed to the next step, Use NintendoSDK Configuration Manager.

To build a binary that uses NintendoSDK, add a new project configuration.

Open the project in Visual Studio and then open the Configuration Manager. For example, from the menu select Build > Configuration Manager.

Select <New...> from the Active solution configuration drop-down list.

Enter a name in the Name field (in our example we enter NN_Debug), and then click OK. If you plan to reuse the existing build settings, select the appropriate configuration in Copy settings from. Select the option Create new project configuration(s).

Repeat these steps to add the required configurations. For example, if there are already configurations named Debug and Release, simply add corresponding configurations for NintendoSDK (such as NN_Debug and NN_Release).

Using NintendoSDK Configuration Manager

Using the NintendoSDK Configuration Manager tool, configure the settings necessary for the use of NintendoSDK.

Starting NintendoSDK Configuration Manager

To start NintendoSDK Configuration Manager, select Tools (T) > NintendoSDK > NintendoSDK Configurations.

Configuring Settings to Use NintendoSDK

Perform the following operations on the project configuration you want to use with NintendoSDK.

  • In the Use NintendoSDK column, select the check box.
  • In the SDK Path column, click the [...] icon and enter the path for the SDK.
  • In the API Set column, specify the API set you want to use.
  • In the SDK Build Type column, set the build type you want to use.

For more information about build types, see Build Targets and Build Types.

Applying The Settings

To apply the values to the project settings, click OK.

In addition, the project sheet ImportNintendoSdk.props and a file for saving the settings named <project_name>nnsdk.xml are both created in the same directory as the project file. Do not delete these two files. In addition, if you move the project file, make sure that you move these files together with the project file.

Checking the Build

For the solution configuration, select a configuration that uses NintendoSDK and confirm that the project can be built. If the build does not work, perform the following check.

Confirming That the NintendoSDK Build System Settings Have Not Been Overwritten

The NintendoSDK preprocessor macro definitions, include directories, library directories and other settings are configured by the NintendoSDK property sheets, but the build might not work if these have been overwritten in the project. Use the following procedure to determine whether these settings have been overwritten.

Open the property sheets for the project. For example, right-click the project in the Solution Explorer and select Properties.

Open the drop-down list for a setting and select <Edit...>. Select the include directories.

In the window that opens, look at Inherit from parent or project defaults. If this check box is cleared, settings will be overwritten. Select this check box.
