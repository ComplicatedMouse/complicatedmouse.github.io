Important Information About Child Emitter Bug Fixes

This release includes several bug fixes for child emitters.
This section describes the updated features and important information about using them.

Bugs in Particle Follow Type for Child Emitters

Previously, there was a bug with the Perfectly follow emitter SRT feature in the Basic Settings tab for child emitters.
The Perfectly follow emitter SRT and Follow only emitter position features now work correctly.

The following images compare data for which the Perfectly follow emitter SRT feature was enabled before and after this fix.
█ is the parent emitter and █ is the child emitter.

Before Fix After Fix
Exhibits the same behavior as when Do not follow emitter is enabled. Follows the parent emitter, with the Perfectly follow emitter SRT feature working as intended.
Note

While the default while creating data was Perfectly follow emitter SRT, it did not work correctly before the fix.
Effects that used the child emitter to adjust appearance may change in appearance due to this fix.
The following combinations of settings from before the fix resulted in unexpected behavior.

Particle follow type

Perfectly follow emitter SRT
Follow only emitter position
Emitter assignment for particles Check box () cleared

Bugs in Child Emitter Transformations

Previously, there was a bug in the Emitter Transform feature for child emitters.
Values specified for Emitter Transform are now applied in the emitter coordinate system rather than the world coordinate system.

The following images provide a comparison of a child emitter for which the position was set to X = 8 in the Emitter Transform feature before and after this fix.
█ is the parent emitter and █ is the child emitter.

Before Fix After Fix
An offset is applied in the x direction in the world coordinate system. The offset is now correctly applied in the x direction in the emitter coordinate system.
Note

Note that if you configured Emitter Transform settings for a child emitter prior to this fix and adjusted the data to achieve a particular appearance, that data may now look different.
