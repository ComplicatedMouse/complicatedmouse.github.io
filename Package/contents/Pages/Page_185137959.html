Terminology

  • Initial Application
  • Revised Application
  • Previous Patch
  • Partial File Patch
  • On-Card Patch
  • Required Version

Initial Application

The application that was initially released by the user, and the NSP file for that application. To build the patch, you must correctly specify the released application.

Revised Application

An application that contains the content you want to revise with the patch, created using the normal build method, and the NSP file for that application. You create a patch by first creating a revised application and then specifying that application together with the initial application.

Previous Patch

The most recently released among all the patches released by the user, and the NSP file for that patch. When you create a second or subsequent patch, you must correctly specify the previous patch.

Partial File Patch

The name of a feature for incorporating in the patch only those files that need revising, as opposed to incorporating all of the files contained in the initial application. This feature is enabled automatically in the NX system when all of the conditions are satisfied.

On-Card Patch

The name of a feature for incorporating both the initial application and a patch in the game card so the game card can be manufactured with the revisions in place.

Required Version

This version is required to start the application. The application cannot be started by the system if it is earlier than this required version. As a result, the developer does not need to consider forward compatibility for save data when developing patches.

The required version is saved in system memory.
