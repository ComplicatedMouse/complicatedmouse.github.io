Multiple Animations From the Same Group Sometimes Play in the Playback Confirmation Panel

Question
Within animation segment tags, when some panes have show/hide keys and some do not, segment tags with keys may be affected by segment tags without keys, causing them to be hidden.
In the following animation, the Tag_01 and Tag_02 segment tags are attached to shown/hidden animations in the same group.
When you select Tag_00 that is not attached to an animation, the animation becomes hidden.

Answer

If you specify the same group for which animation is specified as a related group in multiple segment tags, multiple animations are played in the same group.
In this state, if you then record the animation frame state in the Playback Confirmation panel, the invalid state described in the question occurs.


You can resolve the invalid state if you restore the animation playback frame, recorded in the Playback Confirmation panel, to its initial state.
In the following animation, the recorded animation is restored to its initial state, and the status of the playback frame is re-recorded.
As a result, the animation is shown when you select Tag_00.
