Functions Tab

The Functions tab displays all of the functions that were sampled during a profile.

To change the measurement units (Percent, Time, Avg, or Samples), use the Unit buttons in the Home tab of the Ribbon Bar.

Note

These notes point out differences in the Functions tab when the Thread-Centric Data Views experimental feature is enabled. See Experimental Features Tab for additional information.

Sorting Functions

Functions can be sorted by Total, Self, Diff Total, Diff Self, Core, Thread, Module, Opcodes, Seen In Calls, Seen Out Calls, or Name by clicking on each respective heading.

The following columns are optionally shown:

  • Total: This column is only shown if a Callstack profile was taken.
  • Diff Total, Diff Self: These columns appear when clicking the Diff toggle button.
  • Thread: This column appears when clicking the Thread toggle button.
  • Module: This column appears when clicking the Module toggle button.
  • Opcodes, Seen In Calls, Seen Out Calls: These columns appear when clicking the Stats toggle button.

Each column heading can be clicked to sort by that column. By default, the functions are sorted by Self, which is the amount of time spent in each function (not including any sub-functions). Click on the Total heading to sort by Total, which is the amount of time spent in each function and every function it calls.

Selecting Functions

Click on a function to select it. Click the function again to unselect it. Up to 1000 functions can be selected (each will be highlighted in a different color). Selected functions will be graphed in the Timeline and highlighted in the Call Tree (the Call Tree is only visible if the profile was taken with Callstack profile data).

To quickly select the top 50 functions, click on the button marked Select Top in the Functions toolbar. To quickly deselect all functions, click on the Deselect All button in the Functions toolbar.

Note

When you click on a function to select it from anywhere in the application, it will take the viewing mode into consideration, and select all of the rows that are appropriate for the mode. These include the  Function  rows as well as any  Thread  or  Core  rows underneath the  Function  in the hierarchy. For example, when the mode is set to Functions then Threads, clicking on the  Function  row will select or deselect the  Function  row, and all of the  Thread  and  Core  rows underneath it in the hierarchy. When the mode is set to Threads then Functions, clicking on a  Function  row will select the  Function  row, and all of the  Core  rows underneath it in the hierarchy. Clicking on a  Thread  row will select all of the  Function  rows beneath it, as well as all of the  Core  rows. (Note that the 1,000 selections limit still applies to the individual rows.)

Selected functions are maintained when the viewing mode changes, although the number of selected rows may increase due to the need to select every row below each selected  Function  row to maintain consistency.

When the Select Top  button is clicked, the top 50 functions, from across all groupings will be selected, according to the sort criteria. When the view mode is  Functions Then Threads, this will have the effect of picking the top 50 functions in the list, based on the function totals, and not differentiating by core or thread. This means that although 50 functions will be selected, more then 50 rows in the list can be selected.

Alternative Selection Methods

Multiple items can be selected or deselected with Shift+Click (select/deselect an item without holding the Shift key, then hold the Shift key and click on a second item). The range between the two clicks will be either selected or deselected.

To quickly select only a single item and deselect everything else, Alt+Click can be used. This action deselects all and then selects the clicked upon item.

Right Click Context Menu

A right-click in the tab will bring up a context menu containing a variety of options.

View From Here

Note

This menu item is only available when Thread-Centric Data Views are enabled.

Selecting this item will change the viewing mode, and bring the selected row into view with a temporary highlight. For example, if you right-click on a Thread row and selectView from Here, the viewing mode will change to Threads then Functions. The Thread row corresponding to the thread that you right-clicked on will be scrolled into view and highlighted.

Copy Function Name to Clipboard

Note

This menu item is labeled Copy Name to Clipboard, and will copy the thread name if a Thread row is highlighted.

Copies the full function name with arguments to the clipboard.

Deselect All Other Functions

Note

This menu item is labeled Deselect All Others.

Forces all functions other than the one clicked on to become deselected. This will not result in an unselected function becoming selected.

Show Assembly

Moves focus to the Assembly Tab, showing the assembly of the clicked function.

Show Details

Opens a new Function Details window, showing the details of the clicked function.

Classification Overrides

For each sampled function, the profiler automatically detects the system (like Graphics, Physics, Animation, etc.) that it belongs to. To override the automatically detected system classification of any function, right-click the function in the Functions tab and choose an Override Classification item.

Override Classification: Shows the current system classification state of the clicked function. Click on any of the shown system names to reclassify the function. After you make a choice, the function will be immediately reclassified in all views. Note: To reset a classification to the default, choose Auto-Detect.

Rerun Classifications: Redetects the classifications of every sampled function. You may want to click this option after specifying several classification overrides. Your overrides helps the auto-detection infer the system classification of related functions.

Clear All Overridden Classifications: Resets all classifications back to auto-detect. Because classification overrides are applied across all profiles, this will clear all user-specified classification overrides across all profiles. An additional confirmation is required before continuing with this action.

Functions Tab Toolbar

The following controls are in the Functions tab toolbar.

Function Grouping Mode Drop Down

Note

This option is only available when Thread-Centric Data Views are enabled.

This drop down list displays and controls the function grouping mode. Note that it will also control the function grouping mode of the on the Call Tree Tab. A description of each mode can be found on the page Experimental Features Tab.

Hierarchy Buttons

Note

These buttons are only available when Thread-Centric Data Views are enabled.

The buttons listed below control the expanded state of the items in the list.

Expand All Button

The  Expand All  button will expand all rows.

Collapse All Button

The  Collapse All  button will collapse all rows.

Expand Selected Only Button

The  Expand Selected Only  button will expand all rows necessary to display all of the selected rows and collapse all other rows.

Core Drop Down Filter and Core Column Heading

Note

This option is not available when Thread-Centric Data Views are enabled.

Click on the Cores drop-down in the toolbar to select between All Cores and each individual core. Only cores that were profiled will be available in the Cores drop-down. The default selection is All Cores and each new profile taken or loaded will reset the Cores drop-down selection to All Cores.

When a particular core is selected, such as Core 1, the Core column will be hidden since all functions shown belong to the same core. This can be helpful when focusing on a particular core and can help optimize horizontal space in the tab.


Thread Column Heading and Thread Drop Down Filter

By default, the Thread column is hidden (to conserve horizontal space). However, if you want to see which thread a function belongs to, click the Thread toggle button in the toolbar. Click the button again to hide the column.

If you want to only view functions belonging to a particular thread, use the Thread drop-down to select a particular thread. When an individual thread is selected, the Thread column will be hidden since all shown functions belong to the same thread. The default selection is All Threads and each new profile taken or loaded will reset the Thread drop-down selection to All Threads.

By default, threads are named with a 32-bit identifier (such as 0x1052D298). If the profiler is aware of a name for the thread, that name will be displayed in the drop-down. However, the Thread column will always only use the 32-bit identifier.

System Classification Drop Down Filter

If you want to only view functions belonging to a particular system (like Graphics, Physics, Animation, etc.), use the System Classification drop-down to select a particular system. The default selection is All Systems and each new profile taken or loaded will reset the System Classification drop-down selection to All Systems.

Deselect All Button

To quickly deselect all selected items, click on the Deselect All button.

Select Top Button

To quickly select the top 50 functions, click on the button marked Select Top in the Functions toolbar.

Diff Toggle Button and Diff Total / Diff Self Column Headings

By default, the Diff Total and Diff Self columns are hidden (to conserve horizontal space), but can be enabled by clicking on the Diff button in the Functions toolbar. Click the button again to hide the columns. The Diff Total and Diff Self columns compare the current profile with the last loaded profile, showing a percent difference for each function. For example, if a particular function's Self percentage is 10% in the last loaded profile and 12% in the current profile, then a value of +2% will be shown in the Diff Self column. Likewise, if a particular function's Self percentage is 10% in the last loaded profile and 8% in the current profile, then a value of -2% will be shown in the Diff Self column.

To see how your game's performance has changed over time, you can first load a profile taken a while ago (i.e. yesterday, a week ago, a month ago), and then take a current profile. Within the Diff Total and Diff Self columns it will show how much each function has increased or decreased as a percentage. However, be aware that if a particular function was optimized to take less time (and its percentage decreased), percentage-wise it will cause all other functions to increase (even if the absolute amount of time they took was the same between both profiles).

To sort all functions by the absolute value of the percentage difference, click on either the Diff Total or Diff Self column headers. The sorting is based on absolute value in order to prioritize functions that had the largest percentage change, whether that change was positive or negative.

Note: The Diff Total column is only available on callstack profiles.

Module Toggle Button and Modules Column Heading

By default, the Module column is hidden (to conserve horizontal space), but can be enabled by clicking on the Module button in the Functions toolbar. Click the button again to hide the column. The Module column contains the name of the module that each function came from. To sort all functions by the module they came from (alphabetical sort), click on the Module column header.

Note: The Module column is only useful if there were relocatable modules used in the application.

Opcodes Column Heading

By default, the Opcodes column is hidden (to conserve horizontal space), but can be enabled by clicking on the Stats toggle button in the Functions toolbar. Click the button again to hide the column.

This column shows how many assembly opcodes are in each function (the size of the function). To sort all functions by the number of assembly opcodes (most to least), click on the Opcodes column header.

Seen In Calls and Seen Out Calls Column Headings

By default, the Seen In Calls and Seen Out Calls columns are hidden (to conserve horizontal space), but can be enabled by clicking on the Stats toggle button in the Functions toolbar. Click the button again to hide the column.

This column shows the number of in or out calls seen during the profile compared with the number of possible in or out calls detected through static and runtime analysis. For example, if a given function was seen to be called from 6 different functions during profiling, yet through analysis it was determined that it could have been called from 30 different functions, then the value "6/30" would appear in the Seen In Calls column for that function. To sort all functions by the number of possible in or out calls (the denominator of the reported value), click on the Seen In Calls or Seen Out Calls column header. If you are interested in viewing the function names for the seen in and out calls, select the function and click on the Call Tree tab to further inspect the call flow. If you are interested in viewing the function names of all possible in or out calls, then right-click on the function name in the Functions tab and select Show Details.

Args Toggle Button

By default, function names don't include their argument list when seen in the Functions tab and Call Tree tab (in order to make the list more readable and speed up GUI drawing time). If you wish to show the function arguments, click the Args toggle button.

Copy Button

The Copy button will copy to the clipboard whatever is currently shown using the current sorting and filtering. You can then paste the content into an email, document, or Excel spreadsheet. Because the copied content uses tab delimiters, it will retain the column organization when pasted into an Excel spreadsheet.

The Copy button takes into account what is currently selected. If any items are selected at the time the Copy button is pressed, then only the selected items will be copied. If nothing is currently selected, everything will be copied.

Selected Only Toggle Button

The Selected Only button forces the list display to only show items which are currently selected. If this mode is enabled and a function is deselected, it will be removed from the displayed list.

Filtering Functions

Functions can be filtered using the Filter text box at the bottom of the Functions tab. This allows you to find a particular function or allow you to look at a class of functions or a certain namespace (such as nn).

Type any string into the text box and a search is performed for the items that contain the string. Only items matching the search text will be displayed. To negate the results (show everything that does not match the filter), click on the Negate ("!") toggle button to the right of the text box. To enable case sensitive searches, click on the Match Case ("Aa") toggle button. If there are no matches for the search, the text box will be highlighted orange.

Regular expressions are also supported (using .NET RegEx, which is Perl 5 compatible). To enable regular expression searches, click on the Regular Expression (".*") toggle button to the right of the text box.

When filtering functions, the standard filter only checks against the function name; no arguments are included in the search. This eliminates results where a match occurs in the function arguments, since showing matches to function arguments is typically not desirable and results in unwanted matches. To include function arguments in the search, use the Regular Expression (".*") toggle button.

All filters in the Functions, Last, Threads, Counters, Instrumented, Gfx Groups, Shaders, Call Tree, and Code Coverage tabs act similarly.

Clearing the Filter

When text has been entered into the Filter text box, the Clear ("X") button appears. Clicking on the button will clear the text from the Filter text box, but does not change the toggle buttons that are selected. While typing in the Filter text box, you can also press the Escape key to quickly clear the text.

Negate "!" Button

The Negate ("!") button controls how the filter text box is interpreted:

  • Negate selected: The search results are negated. All items that do not match the filter text box are shown.
  • Negate unselected: The search results are not modified.

Match Case "Aa" Button

The Match Case ("Aa") button controls how the filter text box is interpreted:

  • Match Case selected: Searches are case-sensitive. For example, a search for "foo" will not match "Foo".
  • Match Case unselected: Searches are case-insensitive. For example, a search for "foo" will match "Foo".

Regular Expression ".*" Button

The Regular Expression button controls how the filter text box is interpreted:

  • Regular Expression selected: The filter text box is treated as a regular expression.
  • Regular Expression unselected: The filter text box is not treated as a regular expression.
Info

Before the Match Case option was added, regular expressions defaulted to case-sensitive searching. They now use the Match Case option.


Saved Filters

Saved filters allow for using the same filters between profiling sessions, with any saved filters being loaded when the profiler is opened.

Filters are automatically saved when either the focus leaves the text entry or the Enter key is pressed. The five most recent entries are saved and can be reselected by using the drop-down. When selecting a previously saved filter, any toggle button options that were selected at the time of the search are also reapplied.

Saved filters are shared between tabs depending on the general contents of the tab.

  • Function filters: Function, Last, Instrumented, Call Tree, and Code Coverage tabs.
  • Thread filters: Threads tab.
  • Counter filers: Counters tab.
  • Graphics filters: Gfx Groups and Shaders tabs.
