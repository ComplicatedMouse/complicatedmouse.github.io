Bank

Select the Bank node in the project tree to show the banks in the Sound Set tab.
The displayed list is called the bank list.

Banks and Bank Files

Banks reference bank files.
The instruments actually voiced by sequence sounds are defined in bank files.
Bank files are edited in the Bank tab, explained later in this section.

Creating and Adding Banks

To create or add a bank, select the list region in the Sound Set tab and then select Project > Add Item > Bank.

You can also create and add banks by using the shortcut menu that appears when you right-click the bank list.

Editing a Bank

Banks are edited in the bank list.
As described in the section on banks and bank files, sounds themselves are defined in bank files that are referenced by banks.
The only editing performed on the bank list is editing a bank name and the bank files that are referenced.

Elements of a Bank List

This section describes the meaning of each of the elements of a bank list.

Name

Specifies the name of the bank.
The characters that can be used in a name are restricted. The following characters can be used:

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9_] for subsequent characters

Waveform Archive

Used to configure which waveform archive a bank is associated with. In other words, the waveform archive element chooses the waveform archive that you want to share.

  • Automatic (Shared)
    Generates a waveform archive automatically.
    The other wave sound sets and bank waveform archives in the same embedding group are also collected all together.
    (Generates a single waveform archive containing all the waveforms used within the embedding group.)
    The waveform data of a waveform archive cannot be loaded individually. The data is loaded all together at the same time.
  • Automatic (Separate)
    Generates a waveform archive automatically.
    However, waveform archives, even though they belong to the group with the other wave sound sets and banks, are not collected together with them.
    Waveform archives that are configured with this setting are not shared with any other banks. Such archives are loaded all together.
  • Specify waveform archives explicitly.
    You can specify whether the waveform files within a waveform archive are loaded individually or all together.
    However, even if a waveform archive is set to load individually, it is loaded all together with the other waveform archives if the output method for the group is set to Embed.

For more information about using waveform archives in combination with output methods, see Waveform Archive Automatic (Shared, Separate) Settings.

File path

Specifies the path name of the bank file.

The characters that can be used in a filename or path name are restricted. The following characters can be used.

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character of the filename.
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9, _] for subsequent characters in the filename.

Data Size

Displays the data size of the waveform data of a bank file after compression conversion, as referenced by the bank.
The content on display is updated when files are converted.

HTML Instrument Lists Output

You can export instrument lists to an HTML-formatted file in the same way as HTML Sound List Output.

Right-click the bank list to open the shortcut menu, and select HTML Instrument List Output to display the Settings dialog box.

Select the items to export, and export the file.

Unlike HTML sound lists, HTML instrument lists are not exported while converting.
HTML instrument lists are exported to the same directory as each bank file for each bank file.
You can view the instrument label names and comments in a list, and sort the instruments using each item as a filter.
