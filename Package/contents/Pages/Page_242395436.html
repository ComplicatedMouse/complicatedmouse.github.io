Overview

SoundProjectUtility.exe is a command-line tool that processes NintendoWare SoundMaker project files in various ways.

This section describes SoundProjectUtility functionality and how to use it.

Using SoundProjectUtility

Running SoundProjectUtility

SoundProjectUtility is used by running it from a command line with the following format.

SoundProjectUtility.exe <subcommand> [Options] <project file path>

The options that can be specified differ for each subcommand.

For example, the list-files subcommand outputs a list of files referenced by the specified sound project.

$ cd %NINTENDO_SDK_ROOT%
$ ./Tools/Audio/SoundMaker/SoundProjectUtility.exe list-files --relative . Samples/Sources/Resources/Atk/Common/Common.fspj
Samples/Sources/Resources/Atk/Common/aif/bass_pull.e2.aiff
Samples/Sources/Resources/Atk/Common/aif/bass_slpthm.e2.aiff
Samples/Sources/Resources/Atk/Common/aif/bassdrum.aiff
Samples/Sources/Resources/Atk/Common/aif/cabasa.aiff
...

Project File Path

Specifies the project file (FSPJ) path.

Subcommands

The following subcommands are available.

Subcommands Description
list-files

Outputs a list of files referenced by the sound project.
update-project

Updates the version of the sound project.

SoundProjectUtility Functions

list-files Subcommand

Outputs a list of files referenced by the sound project.

Option Description
-o, --output <filepath>

Sends the results output to the specified file path rather than standard output.

--relative <basepath>

Outputs relative paths. Specify the base path for relative paths.
--backslash

Use the backslash as a separator for output directories.

The paths of the specified sound project file (FSPJ file), referenced sound set files (FSST files), bank files (FBNK files), text sequence files that they reference, standard MIDI files, and waveform files are output to standard output.

  • When the -output option is used, the file paths are output to the specified file instead of standard output.
  • You can adjust the path format using -relative and -backslash.

update-project Subcommand

Updates the version of the sound project.

Option Description
--backslash

Use the backslash as a separator for output directories.
--backup

Stores a copy of the original when updating.
--dry-run

Does not actually perform processing.

--relative <basepath>

Outputs relative paths. Specify the base path for relative paths.
-s, --silent

Suppresses output.

Updates the content of the specified sound project file (FSPJ file) and the referenced sound set files (FSST file) and bank files (FBNK file) to the latest versions of files supported by SoundMaker.

The paths of the updated files are sent to standard output.

  • You can adjust the path format using -relative and -backslash.
  • Paths are not output if the --silent option is specified.
  • If --dry-run is specified, the paths of files to be updated are output, but they are not actually updated.

Additional Information

  • SoundProjectUtility.exe cannot be used on its own. There must be one in each installation_directory\Tools\Audio\SoundMaker folder.
