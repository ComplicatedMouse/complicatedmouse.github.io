Introduction

The FONTLL Library provides low-layer APIs handing scalable fonts.

Supported Font File Formats

The following font file formats are supported by FONTLL.

  • Scalable fonts for system fonts (BFTTF)
  • TrueType (TTF)
  • OpenType (OTF)

BFTTF

BFTTF is a proprietary format created for system fonts.

Differences From nn::font

The nn::font module provides features similar to nn::fontll.

nn::fontll is a lower level module that provides only the minimum features.
In contrast, nn::font is a top level module that allows advanced features to be used more easily (it uses nn::fontll internally).

When categorizing the processing steps until the font is displayed on the game screen as shown, nn::fontll specifically provides features 1 and 2.

  1. (nn::fontll) Load the font, access the glyph data
  2. (nn::fontll) Extract the glyph bitmap in the main memory
  3. (nn::font) Convert the glyph bitmap to texture in the nn::font::TextureCache class. Enables the GPU to display the font on the screen
  4. (nn::font) Manage the texture in the nn::font::TextureCache class (cache structure)
  5. (nn::font) Use in concert with the nn::font library and nn::ui2d library (interface consistency, helper)

Glyph Member Details

Describes glyph data (ScGlyphMap) in detail.

Origin

The baseline defined for the text.

Character Spacing (i_dx, i_dy)

The interval between the origin of the current character and the origin of the next character.
For characters written horizontally, i_dy is normally zero.
The value of fixed-point expressions with dx, dy can be obtained. (This value does not usually need to be used.)

Glyph Position (lo_x, hi_y)

The upper-left corner of the bounding box where the pixels of the glyph image are actually drawn relative to the origin.

Glyph Size (width, height)

The width and height of the bounding box where the pixels of the glyph image are actually drawn.
