ssl Library

Introduction

Purpose of This Documentation

This documentation describes the SSL library included in NintendoSDK.

Intended Audience

This documentation is intended for developers who need secure socket communication over SSL/TLS or HTTPS.

Associated Documentation

Please use this documentation in conjunction with the nn::ssl namespace API reference.

Features provided by the SSL library

The Nintendo’s SSL library implements the Secure Socket Layer (SSL) protocol and the Transport Layer Security (TLS) protocol for socket based secure network connection along with server/client authentication. The SSL library is dedicated only for the SSL client, thus it accepts only client socket for underlying network connection.

The SSL library provides the following features:

Supported protocols Key exchange algorithms Cryptographic algorithms
  • TLSv1.0
  • TLSv1.1
  • TLSv1.2
  • TLSv1.3
  • ECDHE-PSK
  • DHE-PSK
  • ECDHE
  • RSA



  • AES-CBC (128 and 256 bits)
  • AES-GCM (128 and 256 bits)
  • ChaCha20 Poly1305 (256 bits)
  • 3DES (112 bits)
  • RC4 (128 bits)
  • SHA-1
  • SHA-2 (224, 256, 384 and 512bits)
  • MD5
  • HMAC
