Converting Symbol Names Into Hash Values

__attribute__((annotate("nx-mangle")))

This is the attribute you can use to convert function names or variable names into hash values.

It may be used primarily in the following case.

  • When using a dynamic module (NRO), the symbol names in NRO referenced by main and the symbol names in main referenced by NRO remain in the ROM, so these names are converted into hash values to make it more difficult for them to be discovered.

The following describes how to use it.

  1. Add __attribute__((annotate("nx-mangle"))) to the function or variable you want to covert to a hash value.
    1. This needs to be applied both on the referring side and the referenced side. If it is only applied to one side, a link error occurs because the symbol names are different.
  2. Specify the -fnx-mangle-salt= string in the compile options.
    1. You can specify the salt when converting to a hash value.
