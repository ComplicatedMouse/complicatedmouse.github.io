Project Configuration File Format

Project Configuration File Format

The project configuration file is a file in XML format to record common settings for the game project.
It is created under the parts root folder.

Settings can be found in the project configuration file for the following elements.

Specify project settings as child elements of the ProjectSettings element, unless you have a specific reason to do otherwise.

Is there a way to run my own commands when saving a file? You can write your own commands to be executed before and after a file is saved or deleted. To do this, specify the path of the file in which the commands are scripted.

Linear Space in the Layout

You can specify whether to introduce linear workflow in the game project.
Related Links For more information, see Linear Workflow.


The settings for previewing in the viewer, and in LayoutEditor, can be controlled using the following elements.
The settings for these elements can be changed from the Edit menu in Tool Environment Settings.

<SRGBWriteEnabled>true</SRGBWriteEnabled>
<DegammaTextureConversionEnabled>true</DegammaTextureConversionEnabled>
<DegammaParameterConversionEnabled>false</DegammaParameterConversionEnabled>
Output Float Colors

Set the following tag to true or false to enable or disable float color output.
The default is false.

  <FloatColorWriteEnabled>false</FloatColorWriteEnabled>


ASTC Textures

Specify true or false for <AstcTextureEnabled> to enable or disable the ASTC texture formats.

For more information about ASTC textures, see ASTC Textures.
Preview Resolution Settings

Specify the image dimensions to <PreviewDisplayWidth><PreviewDisplayHeight>
to set the resolution of the internal buffer during Preview.


In addition, you can use ChangeablePreviewDisplaySizes to switch between multiple options to resolve the problem using operations in the viewer.

When the viewer starts, the largest size is used as the resolution for previewing. Note: The values set for <PreviewDisplayWidth><PreviewDisplayHeight> are not used.

If the viewer has already started, sizes larger than the viewer are excluded from the selections.

<ChangeablePreviewDisplaySizes>
  <Size>
    <Width>1920</Width>
    <Height>1080</Height>
  </Size>
  <Size>
    <Width>1280</Width>
    <Height>720</Height>
  </Size>
  <Size>
    <Width>720</Width>
    <Height>480</Height>
  </Size>
</ChangeablePreviewDisplaySizes>
Preview Scaling Settings　　　　

When previewing layout data, use the <ViewMode> tag to specify the scaling.

  • original
    • Displays the size set in LayoutEditor Layout Environment Settings > Screen Size.
      Same behavior as in version 0.9.0 and earlier.
  • fit_screen
    • Displays the range set in LayoutEditor Layout Environment Settings > Screen Size, stretched to fit the entire screen.

Specifying the Frame Rate

Set the frame rate for the playback of animation in LayoutEditor and the viewer.

Specify true for 30 fps. Specify false for 60 fps.

<Is30Fps>true</Is30Fps>

Save Thumbnail Image

When the setting is enabled and you save the layout file, perform batch synchronization, or other operations, a thumbnail image is saved to LayoutFilename.flyt.png in the .NwThumbs folder next to the layout file.

The Update Thumbnail Image in the Specified Folder menu option is also added to the File menu.

No thumbnail is generated if this setting is omitted.

You can also specify the maximum size of the image in the project configuration file.

Saved images can be displayed using the Thumbnail Explorer Extension Features.

  <SaveThumbnailImage>true</SaveThumbnailImage>
  <ThumbnailMaximumWidth>256</ThumbnailMaximumWidth>
  <ThumbnailMaximumHeight>256</ThumbnailMaximumHeight>

Commands Prior to Executing Archiver

LayoutEditor can call optional commands before the archiver runs.

The archive target folder is passed to the first argument of the command.

The subsequent arguments get the values specified in the <PreArchiverCommandAdditionalArguments> platform settings.
(You can use absolute paths, paths that are relative to the project file, or environment variables.)

<PreArchiverCommandPath>%COMMANDS%\PreArchiverCommand.bat</PreArchiverCommandPath>
 
rem %1 is the archive target directory.
Copy to rem backup.
robocopy %1 %~dp0\..\backup /MIR

Exit normally if less than rem 7.
if %errorlevel% leq 7 exit 0
exit 1
Search Path for the PSD Source File for Texture Assets

Specifies the folder to search in for the PSD source file for the texture assets.
You can use this option from the function to open PSD texture asset source files for LayoutEditor, using the Textures window.
(You can use absolute paths, paths that are relative to the project file, or environment variables.)

 <TextureSourceFileSearchRootPath >../TexSrc</TextureSourceFileSearchRootPath>

Specifying a New Pane Name

You can specify a new pane name prefix for each type of pane.

If <UseTextureNameAsPicturePaneName> is set to true, the <Picture> setting is ignored, and the name of the texture is used as the name for new picture panes.

Example code in the project configuration file:

<DefaultPaneNames>
 <Null>N_null_</Null>
 <Picture>P_pict_</Picture>
 <Text>T_text_</Text>
 <Window>W_window_</Window>
 <Bounding>B_bounding_</Bounding>
 <Parts>L_parts_</Parts>
 <Group>G_group_</Group>
 <UseTextureNameAsPicturePaneName>false</UseTextureNameAsPicturePaneName>
</DefaultPaneNames>
Specifying a Text ID in the Text Box Pane

Use this to specify the string that is set when you click the button for setting the text box ID in the Text Box page of the property window.

Replace $(Pane) with the pane name.

<DefaultTextBoxPaneTextID>@$(Pane)</DefaultTextBoxPaneTextID>
Specifying the Default Texture Format

You can set a default texture format.

A total of five types are available: grayscale (with and without an alpha channel), color (with and without an alpha channel), and color all white (with alpha channel).

Color all white (with alpha channel) and WhiteWithAlpha are set as image formats that only use the alpha channel.
                
If the WhiteWithAlpha setting is omitted, GrayScaleWithAlpha is used.


Example code in the project configuration file:

 <DefaultTextureFormat>    
    <GrayScale>BC4L</GrayScale>
    <GrayScaleWithAlpha>BC5</GrayScaleWithAlpha>
    <RGBColor>BC1</RGBColor>
    <RGBColorWithAlpha>BC3</RGBColorWithAlpha>
    <WhiteWithAlpha>BC4A</WhiteWithAlpha>
 </DefaultTextureFormat>

Specifying User Commands

You can execute any command from the tool menu.

To display the menu in a hierarchy, use the forward slash (/) to delimit Name.

You can specify arguments with Args.

The selected layout file path is substituted in the macro $(LayoutFile), and the selected pane is substituted in $(Panes).

Specify true using ConfirmSave to control command execution in the state of having unsaved changes.

Specify combinations of shortcut keys using ShortCut.

Specifies whether to reload a layout file after running a command using Reload.

In the content, specify the command name or the command file path. This path may include environment variables surrounded by percent signs (%).

Commands can be batch-added when you use wildcards.

 <UserCommands>
  <UserCommand Name="script" Args="$(LayoutFile)" ConfirmSave="true" ShortCut="" Reload="true">%SCRIPT_ROOT%\*.bat</UserCommand>
  <UserCommand Name="notepad" Args="$(LayoutFile)" ConfirmSave="true" ShortCut="Ctrl+Shift+N" Reload="false">notepad</UserCommand>
 </UserCommands>
Target Groups for Animation Segment Tags

Sets whether to allow there to be no target groups specified for animation segment tags.
The default value is false, which does not allow no target groups to be specified. The basic approach is considered to be one where target groups are always specified.

 <AllowNoGroupAnimationTag>true</AllowNoGroupAnimationTag>

Specifying the Binary Conversion Method for Animation Keys When Previewing

Specifies the method to use for the binary conversion of animation keys when previewing. (Specifically, it controls whether to specify the --omit-nokey-all option for the binary converter.
When true is specified, animation data is not generated if there are no keys in the frame segment specified by the animation segment tags.

The default is false.

 <PreviewConverterOptionOmitNoKeyAllEnabled>false</PreviewConverterOptionOmitNoKeyAllEnabled>

Specifying Separated Mode Layout Values During Previews

Specifies whether to transfer the values in LayoutEditor as the layout values when transferring separated-mode layouts containing animations to the viewer.

Specify true to transfer the values in LayoutEditor. Specify false to transfer the base values.

The default is false.

<PreserveValuesOnSendingSeparatedModeLayoutToViewer>false</PreserveValuesOnSendingSeparatedModeLayoutToViewer>

Maintain the Scale When Switching Fonts

Specifies that the scale be maintained when the font is changed.

When the font size changes when switching fonts, by default scaling is automatically adjusted so that the text width and wrap point do not change. However, when the default font itself is changed, or when a multi-language font is switched in and the scale should not be changed, set this function to true to change only the font size, preserving the scale.

To replace a font while maintaining scaling at conversion, specify the --keep-font-scale option for the binary converter.

If set to true, the warning message about changing the font size is not displayed in LayoutEditor.

The default is false.

<KeepingSizeScaleEnabled>false</KeepingSizeScaleEnabled>

Capture Texture

Set the following tag to true or false to enable or disable the feature for capturing textures.
The default is false.

<CaptureTextureFunctionEnabled>false</CaptureTextureFunctionEnabled>


Align Panes

Set the following tag to true or false to enable or disable the feature for keeping panes properly aligned.

The default is true.

<AlignmentPaneEnabled>true</AlignmentPaneEnabled>

Textures Window Display Target

Specifies additional display targets for the Textures window.

Label is the name displayed in the Textures window. Base represents the path origin. In Base, specify Project (project configuration file), Layout (layout file) or Absolute (absolute path). AllDirectories specifies whether the textures under child directories are displayed. ShowDirectories specifies whether to display child directories as a hierarchy. ShowDirectories is ignored if AllDirectories is false. Specify a relative path or an absolute path for the description. This path may include environment variables surrounded by percent signs (%).

<TextureDirectories>
  <TextureDirectory Label="Common" Base="Project" AllDirectories="true" ShowDirectories="false">Common</TextureDirectory>
  <TextureDirectory Label="Texture" Base="Layout" AllDirectories="false" ShowDirectories="false">Texture</TextureDirectory>
  <TextureDirectory Label="Project" Base="Project" AllDirectories="true" ShowDirectories="true"></TextureDirectory>
</TextureDirectories>


If the <TextureDirectories> description is omitted, the following specifications are assumed.

<TextureDirectories>
  <TextureDirectory Label="under project" Base="Project" AllDirectories="true" ShowDirectories="true"></TextureDirectory>
</TextureDirectories>

Default Animation Edit Mode Setting

Specifies whether to set the animation editing mode to separated mode for newly created layouts.

The default is false (merged mode).

<IsAnimationSeparatedModeDefault>false</IsAnimationSeparatedModeDefault>

Setting Frame 0 as the Initial State in Separated Mode

Sets frame 0 as the initial state of layouts when the animation editing mode is separated mode.

When this setting is enabled, the initial state represents a state where the segment including frame 0 has been played up to frame 0.

It is disabled by default.

<InitializeSeparatedModeLayoutAtZeroFrame>false</InitializeSeparatedModeLayoutAtZeroFrame>

Setting the Initial State to the Saved State When Editing in Separated Mode

When the animation editing mode is separated mode, the state when the file was saved is used as the initial state.

It is disabled by default.

<SaveStateAsInitialState>false</SaveStateAsInitialState>


Default Animation

Specifies the name of the default animation when the animation editing mode is separated mode.

If a default animation is set in the layout, it is always played back to the final frame in the initial state of the layout.

If not needed, this setting does not need to be configured.

When <InitializeSeparatedModeLayoutAtZeroFrame> is enabled, <InitialStateAnimationName> is ignored.


<InitialStateAnimationName>In</InitialStateAnimationName>

Animation Conflict Warning

Specifies whether to use the Animation Conflict Warning feature in the Edit Animation Segment Tags window.

This option is enabled by default.

<IsWarnAnimTagCollisionEnabled>true</IsWarnAnimTagCollisionEnabled>


Default Capture Effect Degamma Setting

Specifies the default setting when applying a degamma process to an effect when the capture effect is a linear color space.

This option is enabled by default.

<DefaultCaptureEffectDegammaFlag>true</DefaultCaptureEffectDegammaFlag>

Default Setting to Capture First Frame Only for Capture Pane and Capture Effect

Specifies the default setting for capturing only the first frame for the capture pane and the capture effect feature.

This option is enabled by default.

<DefaultCaptureEffectFirstFrameOnlyFlag>true</DefaultCaptureEffectFirstFrameOnlyFlag>


Default Setting for Creating New Related Group When Adding Segments

Specifies the default setting for creating a new related group when adding a segment.
It is disabled by default.

<DefaultCreateNewGroup>false</DefaultCreateNewGroup>

Default Setting for the Auto Update of the Newly Created Group When Adding a Segment in Separated Mode

Specifies the default setting for the auto update of the group that is newly created when adding a segment in separated mode.
This option is enabled by default.

<DefaultAutoUpdateGroup>true</DefaultAutoUpdateGroup>

Default Settings for the Auto Update for Output Animation Type for a New Segment

Specifies the default settings for the auto update for the output animation type for a new segment.

This option is enabled by default.

<DefaultAutoUpdateAnimationKind>true</DefaultAutoUpdateAnimationKind>

Default Frame Buffer Capture Setting for Capture Panes

Specifies the default value set for the framebuffer capture flag when a new capture pane is created.

This option is enabled by default.

<DefaultCapturePaneFrameBufferCaptureFlag>true</DefaultCapturePaneFrameBufferCaptureFlag>

Mode to Ignore the Capture Effect Setting to Capture First Frame Only in LayoutViewer

Specify true to set the mode to ignore the Capture Effect setting to capture the first frame only and instead always update the content in every frame when previewing in LayoutViewer.

This mode is disabled by default.

<IgnoreCaptureEffectFirstFrameOnlyFlag>false</IgnoreCaptureEffectFirstFrameOnlyFlag>

Settings for Adjusting Limit Values on Capture Textures When Previewing in LayoutViewer

By default, the following values are set for the numbers of capture textures that can be previewed in LayoutViewer.


  • CaptureTextureShareInfoCountMax - 64
    • The number of capture textures that can be used in layouts, including parts (capture panes, capture effects).
  • VectorGraphicsTextureShareInfoCountMax - 64
    • The number of vector graphics textures that can be used in layouts, including parts.
  • DynamicTextureShareInfoCountMax - 32
    • The number of parts that can be arranged in a layout (one part is used for the body layout, so the number of parts that can be arranged is this value minus one).
  • DynamicTextureShareInfoPartsStackMax - 8
    • The maximum number of hierarchical parts levels for when using capture textures to overwrite textures within parts.


<CaptureTextureShareInfoCountMax>64</CaptureTextureShareInfoCountMax>
<VectorGraphicsTextureShareInfoCountMax>64</VectorGraphicsTextureShareInfoCountMax>
<DynamicTextureShareInfoCountMax>32</DynamicTextureShareInfoCountMax>
<DynamicTextureShareInfoPartsStackMax>8</DynamicTextureShareInfoPartsStackMax>

Tiling Optimization Mode When Generating Texture Binary

Specify the tiling optimization mode for the texture binary file. The value you specify conforms to the --tile-optimize option of 3DTextureConverter.

<TileOptimize>performance</TileOptimize>

Optimization Threshold When Generating Texture Binary

Specify the optimization threshold for when the texture binary is generated. The value you specify conforms to the --tile-size-threshold option of 3DTextureConverter.

<TileSizeThreshold>75</TileSizeThreshold>


Animation Window Setting

Specify true to show the Animation window when previewing in LayoutViewer.

<IsSimultaneousAnimationPlayEnabled>true</IsSimultaneousAnimationPlayEnabled>

Take Italics Into Account For Capture Effects in Text Panes

Specify true to check the portion of the capture effect that extends beyond the pane size using LayoutViewer.

<IsSpreadPaneSizeOfItalicEnabled>false</IsSpreadPaneSizeOfItalicEnabled>
Info

This feature does not support automatic wrapping based on per-character transformations and pane size. Depending on the usage, the following situations apply.

  • When the per-character transformation feature is being used, text that extends outside the pane size region is not shown in LayoutViewer.
  • Text that extends beyond the pane size region due to line breaks in the text is shown with lower quality in LayoutViewer.

Use GPU for Texture Conversion Configuration

Controls whether to enable GPU encoding for binary conversion at font generation and preview within LayoutEditor. Specify true, false, or auto). Compliant with the Layout Binary Converter's --gpu-encoding option.
Treated as true when omitted.

<IsGpuEncodingEnabled>true</IsGpuEncodingEnabled>

Specify the Types of Panes for Which to Configure Exclude From Align Pane Processing

Specify whether Exclude from Align Pane processing can be set for each type of pane. If it is set to false, the Exclude from Align Pane processing check box becomes inoperable in the property menu for that pane. Treated as true when omitted.

 <EnableAlignmentIgnoreCheckBoxes>
    <Null>false</Null>
    <Picture>true</Picture>
    <Text>true</Text>
    <Window>true</Window>
    <Bounding>false</Bounding>
    <Alignment>false</Alignment>
    <Capture>false</Capture>
    <Scissor>false</Scissor>
    <Parts>true</Parts>
  </EnableAlignmentIgnoreCheckBoxes>

Specify Calculate the area under the child panes when performing Align Pane processing for the null pane and parts pane

Specifies whether Calculate the area under the child panes when performing Align Pane processing can be set for the null pane and parts pane. If it is set to true, the Calculate the area under the child panes when performing Align Pane processing check box is displayed in the property window. You can get the settings configured in the property window with nn::ui2d::Pane::IsAlignNullPaneByDescendants or nn::ui2d::Pane::IsAlignPartsPaneByItself. If omitted, this is treated as false. 

<SpecifyAlignmentMethodForNullPanesAndPartsPanes>true</SpecifyAlignmentMethodForNullPanesAndPartsPanes>

Apply transparency to children setting setting of new panes

Specifies the initial value for Apply transparency to children for new panes. This setting is not applied for null and parts panes. The initial value for null panes is specified in the tool settings window. The initial value for parts panes are set to be enabled. If omitted, this is treated as false.

<DefaultInfluenceChildrenTransparency>true</DefaultInfluenceChildrenTransparency>

Use RenderTargetTextureLifetime, passed when capture effects are used to generate textures, to enable RenderTargetTextureLifetime_OneFrame.

Specify true to pass RenderTargetTextureLifetime_OneFrame to the LifetimeHint argument of the callback function that is called when generating the temporary texture used for capture effects. (When Capture First Frame Only is disabled, this remains as RenderTargetTextureLifetime_Layout)

This setting only affects transfers from LayoutEditor to LayoutViewer. 

<IsCaptureEffectLifetimeHintOneFrameEnabled>true</IsCaptureEffectLifetimeHintOneFrameEnabled>


Enabling specification of normal fetch texture format

When specification of normal fetch texture format is enabled, format the texture in the Texture window, then BC1_N, BC5_N, BC7_N, BC4L_N, L8_N, LA8_N, and ASTC4x4_N become available for selection. Textures that specify these formats are not subject to gamma conversion in sRGB fetch or Layout Converter regardless of the Linear Color Space Mode specified by the tool environment settings. If omitted, this is treated as false.

<NormalFetchTextureFormat>true</NormalFetchTextureFormat>



Specify the maximum size for drop shadow capture effect

Specify a maximum size for the Drop Shadow capture effect. When omitted, it is handled as 100.
Setting a value larger than 100 looks basically no different than 100 in LayoutViewer.

<CaptureEffectDropShadowMaximumSize>100</CaptureEffectDropShadowMaximumSize>

Enable ASTC texture emulation in the PC viewer

Specify whether to enable ASTC texture emulation. It is disabled when this option is omitted. Emulation is not performed when using the PC viewer on the WIN platform, or when using the development hardware viewer.

<AstcTextureEmulationEnabled>false</AstcTextureEmulationEnabled>
