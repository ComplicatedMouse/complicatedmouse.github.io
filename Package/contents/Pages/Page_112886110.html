SD Card

Feature Overview

Provides features for mounting SD cards formatted for FAT or exFAT. Usage is restricted to debugging and development purposes.

Warning
  • These features cannot be used in the production environment.
    • For more information, see Using Debugging APIs in fs Library > Development Overview.
  • Files or directories on an SD card containing multiple-byte characters in their names cannot be handled.

Sample Programs

Check the SD Card Mount Feature for Debugging in the Descriptions of Samples.

Mounting SD Cards

Call nn::fs::MountSdCardForDebug() to mount the SD card.

Journaling is not supported. nn::fs::Commit() does not need to be executed.

Important Information in the Development Hardware Environment

For applications built for the NX platform, the SD card inserted in the development hardware is mounted.

The SD card must be formatted for FAT or exFAT.

Important Information in the Windows Environment

In the Windows environment, the sdcard directory located in the same directory as the application's executable file is mounted instead as an alternate region. If there is no directory, one will be created automatically.

Note that even if an SD card is inserted in the host PC it is not what gets mounted.

Info

We plan to make it possible to specify an alternative directory path in the future.

Path

  • As the directory delimiter, use either the slash (/) or backslash (\).
  • The symbols < > * ? : | and " cannot be used for entry names.
  • Tab, carriage return, and other control characters cannot be used.
  • Letters are not case-sensitive.
  • Multi-byte characters cannot be used.

File Timestamp

Info

In NX Add-On 4.1.0 and later, the timestamps for files in the SD card are saved.

FAT

The creation date timestamp has a precision of one second.

The creation date timestamp has a precision of two seconds. Information for one second or less is discarded.

The last access time timestamp has a precision of one day. The hour, minute, and second information is discarded.

exFAT

The creation date timestamp has a precision of one second.

The creation date timestamp has a precision of one second.

The last access time timestamp has a precision of two seconds. Information for one second or less is discarded.


Info

Because the timestamp precision for files in the SD card is different, in some cases the Last Updated or Last Accessed timestamps appear to be at a time or date earlier than the time when the file was created.

For example, if a file was created on an FAT formatted SD card at 2018-1-1 1:23:45, the following timestamps would remain.

  • Time of creation: 2018-1-1 1:23:45
  • Time of last update: 2018-1-1 1:23:44
  • Time of last access: 2018-1-1 0:00:00
Note

On the NX platform, to restrict write operations to the SD card, the last access time is not updated even if read access is performed.
