Fixed a bug where pan curves differed between surround mode and stereo mode.
After the fix, the pan curve is the same for surround mode and stereo mode.

Specifically, it has been changed so that the sound volume increases when the pan value is -0.5f or lower, or it is 0.5f or higher upon specifying nn::atk::OutputMode_Surround in nn::atk::SoundSystem::SetOutputMode.
Now, even when stereo sound is played with dual pan selected for the pan mode, the pan value meets this condition in internal processing, so the volume increases similarly.

This same big is also reproduced in NW4F, so a NW4F compatibility mode has also been prepared.
Consider using the following function when performing a port using NW4F data.

  • nn::atk::SoundSystem::SoundSystemParam::enableCompatiblePanCurve

For more information about pan curves in NW4F compatibility mode, see NW4F Compatibility Mode Pan Curve in Pan Operations.
