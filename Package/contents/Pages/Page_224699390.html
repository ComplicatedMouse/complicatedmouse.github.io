5. Multi-Export

In this section of the tutorial, we export the model and the animation data with separate settings. Animation data can be exported separately for each specified frame segment.

You can create a continuous animation in Maya and then export that animation to each specified segment of frames.
This feature enables the convenient configuration of output settings (such as those for models and multiple animations) by managing a single scene in Maya.















As shown in the preceding model animation, create settings to export the following four types of model animations.
The four types of model animations can be output immediately after the settings are configured.

Selecting a Model

Select and set the model to export.

Select TopL from the outliner.


Nintendo Multi Export Window

Select Nintendo > Multi Export to open the Nintendo Multi Export window.

Add "Output Items"

Click Add to open the Input Multi Export Item window.
Click OK to add the item.

When the item has been added, it is displayed in the Nintendo Multi Export window.

Set the Export Items (Model and Texture)

Configures the export settings for the model and the texture.

To open Input Multi Export Item, double-click the list added to the window.

Configure "Export_Model" by setting the order specified in the following figure and table. Designates models and textures to output.

Order
Category
Operation
1 Output File Name Enter Export_Model. The filename of the file to be output.
2 Node (Display Layer)

Click Get. The selected TopL is registered.

(It is also possible to directly enter the items to output.)

Check only Model Data and Texture Data. Models and textures will be output.

Check that the Output File Name is Export_Model.

Info

If the Output Folder (indicated by A in the preceding figure) is unspecified, the items are exported to the folder set in the Nintendo Export Options in the Export step of this tutorial. It is also possible to click the folder icon and specify any folder.

If you have a model without skinning, it will be exported with its starting frame at the value set for Start Frame, indicated by B in the figure, but in this example we are using a model with skinning, so this setting will have no effect on the exported model.
