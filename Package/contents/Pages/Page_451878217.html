Runtime Libraries

Describes changes to the runtime libraries.

Changed the method for managing dynamic rendering texture resources.

Previously, dynamic rendering textures such as capture textures or vector graphics textures were managed by registering them with names to ResourceAccessor.
However, because texture names within ResourceAccessor must be unique, some cases were difficult to handle, such as setting root names when copying layouts or not being able to create multiple instances of the same layout data without cloning it.

To resolve these types of situations, the implementation was changed to manage dynamic rendering textures in each pane without using ResourceAccessor.

The following specifications were changed for this purpose.

  • Dynamic rendering textures can no longer be registered in ResourceAccessor.
    • The ResourceAccessor::AcquireDynamicGenerateTexture() or ResourceAccessor::AcquireDynamicGenerateTextureWithResolvePrefix() methods for dynamic rendering textures are not used.
  • Functionality for specifying the root name or prefix for capture textures, such as those within copy constructors or functions, have been deprecated.
