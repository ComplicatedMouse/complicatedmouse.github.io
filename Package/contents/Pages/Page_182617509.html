LightEvent Class

Feature Overview

The LightEvent class is a feature that uses a 1-bit binary value representing the signaled and non-signaled states as a way to synchronize threads.

The LightEvent class wraps the lightweight event feature, which uses an nn::os::LightEventType object, and contains the following member functions. For more information, see the API reference.

Name Description Notes
LightEvent Constructor. Performs the same initialization process as nn::os::InitializeLightEvent().
~LightEvent Destructor. Performs the same destruction process as nn::os::FinalizeLightEvent().
Wait Waits for an event. Waits for an event just like nn::os::WaitLightEvent().
TryWait Event polling. Polls for an event just like nn::os::TryWaitLightEvent().
TimedWait Waits for an event for a limited time. Waits for a limited time just like nn::os::TimedWaitLightEvent().
Signal Signals an event.

Signals an event just like nn::os::SignalLightEvent()
Clear Unsignals an event. Unsignals an event just like nn::os::ClearLightEvent().


Example of Using the LightEvent Class

This example code implements the StickyLightEvent class using the nn::os::LightEvent class.

#include <nn/os.h>

class StickyLightEvent
{
public:
    StickyLightEvent() : m_LightEvent(nn::os::EventClearMode_ManualClear) {}

    void Wait()
    {
        m_LightEvent.Wait();
    }

    void Set()
    {
        m_LightEvent.Signal();
    }

    bool IsSignaled()
    {
        return m_LightEvent.TryWait();
    }

private:
    nn::os::LightEvent   m_LightEvent;
};


The StickyLightEvent class provides a 1-bit event flag that cannot be cleared after it has been set. This state persists until the object instance is freed.

m_LightEvent is initialized by the constructor. Set the clear mode to Manual to prevent automatic clearing.

The Wait function is used to wait until an event is set. Multiple threads can wait simultaneously. If an event is already set, this will return immediately.

The Set function is used to set an event. This event may not be cleared after being set. When multiple threads are waiting, all threads are released when Set is issued.

The IsSignaled function returns the event status. Returns true if the event has already been set, or false otherwise.

These kinds of StickyLightEvent features can be used in the process of initializing objects and modules.
