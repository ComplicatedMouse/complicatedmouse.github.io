Error Features

Use the error command.

  • Writing Error Information to a File
  • Enabling and Disabling Report Uploading
  • Enabling and Disabling Screenshots When an Application Crashes 

$ RunOnTarget.exe DevMenuCommand.nsp -- savedata --help
usage: DevMenuCommand error export <absolute_host_file_path>
       DevMenuCommand error enable-report-upload
       DevMenuCommand error disable-report-upload
       DevMenuCommand error enable-crash-screenshot
       DevMenuCommand error disable-crash-screenshot

Writing Error Information to a File

Use this command to save all error information to a file on the PC.

If there is a fatal error for which you cannot determine the cause and you contact Nintendo to resolve the problem, consider submitting the log obtained from executing the following command.
The contents are encrypted and cannot be read.

$ RunOnTarget.exe DevMenuCommand.nsp -- error export c:\temp\error_report.bin
[SUCCESS]

Enabling and Disabling Report Uploading

Info

Available in NintendoSDK Firmware for NX 6.0.0-2.0 and later.

Use the enable-report-upload subcommand and the disable-report-upload subcommand. See Enabling Crash Reports in Development Environments when using this command to check the crash reporting feature.

$ RunOnTarget.exe DevMenuCommand.nsp -- error enable-report-upload

Enabling and Disabling Screenshots When an Application Crashes

Info

Available in NintendoSDK Firmware for NX 10.2.0-4.0 and 11.0.0-6.0 and later.

Use the enable-crash-screenshot and disable-crash-screenshot subcommands. When checking the functionality for using this command to get crash report screenshots, also see Enabling Crash Reports in Development Environments.

$ RunOnTarget.exe DevMenuCommand.nsp -- error enable-crash-screenshot
