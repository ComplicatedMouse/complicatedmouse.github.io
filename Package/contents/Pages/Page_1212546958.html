Files Tab

The Files tab displays all of the files that were opened, read, or written during the course of the profile.

Sorting Files

Files can be sorted by clicking on each of the column headers. By default, files are sorted by the length of time they were open.

Time Open

The total time the file was known to be open. If a file is opened before profiling begins or after profiling ends, the time is capped to the profile begin or end time, respectively. If the same path is opened twice, both instances count toward the total time open.

When a time range is selected, any file that was open during that time range will have the entire time it was open or closed shown in the Files tab, not just the time it was open within the selected time range.

Opens

The number of times the file was opened during the profile. Files will show open counts even if the file was opened before profiling began. This was done so that there would not be a situation where reads or writes were occurring on a file that was showing it was never opened, as such behavior is not possible.

When a time range is selected, any file that was open during that time range will have an open count shown in the Files tab, even if the actual Open call is not part of the selected time range.

Accesses

The number of times a file was either read from or written to during the profile.

When a time range is selected, any file that is actively being accessed during any of the selected time range will increase the access count, even if the call was started or ended outside of the selected time range.

Transfer Rate

The average transfer rate of all accesses that occurred for this file.

When a time range is selected, this value is recalculated based on the accesses that occurred during the selected time range.

Buffer Size

The average buffer size in bytes used when reading from or writing to this file. As this value is an average, having one very small read followed by numerous large reads of all the same size will result in the average being lower than may be initially expected.

When a time range is selected, this value is recalculated based on the accesses that occurred during the selected time range.

Path

The path used to open the file. Paths are generally standardized so that forward and backward slashes for directory separators are treated as the same file path. However, multiple directory separators in a row are counted as different paths (i.e., "example:/path.bin" is the same as "example:\path.bin", but different from "example://path.bin"). When sorting by this column, all paths are sorted alphabetically.

There are no changes to this column based on time range selections.

Files Tab Toolbar

Deselect All Button

To quickly deselect all selected files, click on the Deselect All button.

Select Top Button

The Select Top button selects the first 50 files in the list, dependent on the current sort and filter.

Copy Button

The Copy button will copy to the clipboard whatever is currently shown using the current sorting and filtering. You can then paste the content into an email, document, or Excel spreadsheet. Because the copied content uses tab delimiters, it will retain the column organization when pasted into an Excel spreadsheet.

Mounts Drop Down

Click on the Mounts drop-down in the toolbar to select between "All Mounts" and individual mount points created. Mounts will appear in this drop-down even if no files were opened or accessed on the mount at the time of the profile. Only files that were on the selected mount point will appear in the list display.

In the drop down, two pieces of information are shown, the mount name and the mount type. The mount name matches the name showing up in the path. For example, in "foo:/bar.bin", the text "foo" will appear in the drop-down. The mount type corresponds to the nn::fs::Mount* API function that was called. In the case of the mount name "foo", if it was mounted via nn::fs::MountRom, you will see "foo Rom" in the drop-down.

Info

You can duplicate this effect with regular expression filtering. For example, "^foo:" in a regular expression filter will find all of the files that have a path that starts with the foo mount name.


Selected Only Toggle Button

The Selected Only button forces the list display to only show files which are currently selected. If this mode is enabled and a file is deselected, it will be removed from the displayed list. If all files are deselected, the toggle automatically disables.

Selecting Files

Click on a file to select it. Click on the file again to unselect it. While a large number of files can be selected at once, it is recommended to keep the selected count low to improve the visibility and performance of the Timeline tab. When a file is selected, the file will draw on the Timeline tab, showing when the file was opened in a light blue and when the file was accessed in a dark blue. If you can't see the file, try zooming all the way out or looking underneath the path shown in on the Timeline.

To quickly select the top 50 files, based on the current sort and filter, click the Select Top button in the toolbar. To quickly deselect all files, click on the Deselect All button in the toolbar.

Note

To view the selected files on the Timeline tab, the Thread-Centric Data Views experimental feature must be enabled.


Right Click Context Menu

A right-click in the tab will bring up a context menu with a single option.

Copy Path

Selecting this item will copy the path of the file clicked on to the clipboard.

Filtering Files

Files can be filtered using the Filter text box at the bottom of the Files tab. This behaves similar to the Filter text box on the Functions tab.
