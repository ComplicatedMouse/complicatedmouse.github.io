General

Operation is not ensured when the system locale of the operating system is changed to anything other than the default value.


Operation is not ensured when the system locale of the operating system is changed to anything other than the default value. (For example, if the locale is set to anything other than "Japanese [Japan]" in the Japanese version of Windows.)

Audio

nn::audio::ContainsAudioOutBuffer() does not return valid results.

Symptoms

nn::audio::ContainsAudioOutBuffer() is intended to return true until the nn::audio::GetReleasedAudioOutBuffer() function releases nn::audio::AudioOutBuffer from
management under nn::audio::AudioOut, but it sometimes returns false.

Workaround


Avoid the issue by managing the state on the application side while nn::audio::AudioOutBuffer is under the control of nn::audio::AudioOut, so that a function is not called that has a precondition that nn::audio::ContainsAudioOutBuffer()
returns false, such as the nn::audio::AppendAudioOutBuffer() function.

Fix Schedule

This problem will be fixed in an upcoming release.

libcurl

HTTPS communications cannot be sent from multiple applications on the same PC.

Symptoms

If you start HTTPS communications from multiple applications on the same PC, sometimes an error occurs.

Workaround

No workaround is currently available.

Fix Schedule

This bug will be fixed in an upcoming release.

socket

The MSG_DONTWAIT flag cannot be used in the nn::socket::Recv() function.

Symptoms

Setting the MSG_DONTWAIT flag using the nn::socket::Recv() function does not put the module in non-blocking mode.

Workaround

You can achieve the same type of non-blocking behavior by using the nn::socket::Fcntl() function to set the O_NONBLOCK flag.

Fix Schedule

This problem will be fixed in an upcoming release.

The nn::socket::Select() and nn::socket::Poll() functions sometimes return EBADF.

Symptoms

The nn::socket::Select() and nn::socket::Poll() functions sometimes return EBADF when called if the socket descriptor is closed.

Workaround

No workaround is currently available.

Fix Schedule

This bug will be fixed in an upcoming release.

PERF

The PerfSimple sample crashes in some PC environments.

The PerfSimple sample exits with an assertion in certain PC environments.

This bug will be fixed in a future release.

ssl

Call the nn::ssl::Initialize() and nn::ssl::Finalize() functions on the main thread.

Symptoms

Calling the nn::ssl::Initialize() and nn::ssl::Finalize() functions on a thread other than the main thread causes the library to stop working.

Workaround

Call the nn::ssl::Initialize() and nn::ssl::Finalize() functions on the main thread.

Fix Schedule

This problem will be fixed in an upcoming release.

Audio Tools

In rare cases, the program crashes when playing stream sounds being edited using the in-game editing feature of the nn::atk::viewer library.

In rare cases, the following end error occurs and the program crashes when repeatedly playing stream sounds being edited using the in-game editing feature or when rapidly changing the stream sound parameters using the in-game editing feature.

Abort: 'delimiter != nullptr' in nn::fs::detail::GetMountName()

This problem will be fixed in an upcoming release.

The pan mode and pan curve settings are not applied to stream sounds.

Symptoms

When stream sounds are played, the pan mode and pan curve settings are fixed to Dual Sqrt (-3dB) regardless of the parameters set in SoundMaker.
This bug does not affect the SoundMaker features that measure loudness and output waveforms.

Workaround

No workaround is currently available.

Fix Schedule

This problem will be fixed in an upcoming release.

3D-G3D

An invalid access event sometimes occurs when running the G3dDemo sample Town scene.

Depending on the environment used, an invalid access event sometimes occurs when running the Generic version of the G3dDemo sample Town scene.

This bug does not occur if you stop using nn::perf or change the number of models rendered. Further details about the cause of this bug are unknown at this time.

Effects: VFX

When the Emitter Assignment for Particles inheritance setting for child emitters in effect data is disabled, specifying Historical Stripe or Historical Stripe2 as the particle type causes the VFX runtime to stop responding.

Symptoms

When the Emitter Assignment for Particles inheritance setting for child emitters in effect data is disabled, specifying Historical Stripe or Historical Stripe2 as the particle type causes an invalid access event to occur in the VFX runtime.

Workaround

You can avoid this bug by enabling Emitter Assignment for Particles in the effect data settings.

Fix Schedule

This bug will be fixed in an upcoming release.

FontConverter

The settings in the preferences file are not applied when running the console version of the font converter.

Symptoms

The settings in the preferences file (preference.xml) are not applied when the console version of the font converter is executed.

Workaround

Use command-line arguments to specify the necessary settings.

Fix Schedule

This bug will be fixed in an upcoming release.

time

In nn::time::ClaimPeriodicBenefitWithUserSystemClock(), the penalty sometimes starts when the network clock is adjusted.

Symptoms

When network time is enabled, and after operating the user clock by switching Synchronize Clock via Internet from OFF to ON in the system settings, if nn::time::ClaimPeriodicBenefitWithUserSystemClock() is used, the penalty period starts when the network clock is adjusted.
The network clock is adjusted automatically by the system, so the start of the penalty period is undefined in this case.

After switching Synchronize Clock via Internet from OFF to ON, and then returning the setting to OFF, penalty periods starts correctly and according to clock operations.

Workaround

None.

Fix Schedule

This bug is fixed in the NintendoSDK 6.x.x series.

tool

With the HlslCrossCompiler, locations may not be assigned to hull shader I/O variables.

Symptoms

Sometimes no location is assigned for input and output variables when converting a hull shader with no hs_control_point_phase for DXBC.

Workaround

None.

Fix Schedule

This bug will be fixed in the next major version update.
