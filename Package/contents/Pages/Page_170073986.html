** When nn::ssl::Connection::VerifyOption::VerifyOption_EVCertFingerprint is enabled on an nn::ssl::Connection, EV Qualified CAs will have their SHA256 fingerprint verified to make sure they are legit.

*** Trusted: If it's `YES`, it means that the nn::ssl library trusts a root CA certificate by default without explicit certificate import. An application can perform SSL/TLS handshake successfully with servers which trust root is one of the root CA certificate found in the above table. If it's `NO`, handshake with a server which trust root is one of the certificate marked as `NO` could fail due to unknown CA. But the system still have these certificates so that an application can read a certificate data if necessary. Importing such certificates may be done for successful SSL/TLS handshake.
