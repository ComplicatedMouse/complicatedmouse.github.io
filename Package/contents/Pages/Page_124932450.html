Output BFFNT/BFFNA Tab

BFFNT/BFFNA

Binary font files are ultimately output in one of these formats.
This page describes the BFFNT/BFFNA tab UI.

A File Output Specify the file to output to.
B Encoding Specifies the text character encoding format that corresponds to the font for output.
C Alternate Characters

Sets alternate characters.

When trying to render characters that are not in the font resource, this character is displayed instead.
D Kerning Specifies whether to output kerning information.

If the input does not have kerning information, a warning is displayed.

(Kerning is a feature to adjust the space between characters so that they appear natural, according to the combination of characters before and after.)
E Linefeed

Specifies the height of one line (0 to 255).

When Use Default is selected, it is treated as though the same value as the font height was specified.
F Default Left Space Width Specifies the default left space width (-128 to 127).
The default left space width is used by characters that do not have character-specific left spaces.
When Use Default is selected, 0 is specified.
G Default Glyph Width

Specifies the default glyph width.

Default Glyph Width is used by characters that do not have character-specific glyph widths.

When Use Default is selected, it is treated as though the maximum glyph width in the font was specified.
H Default Right Space

Specifies the default right space width.

The default right-space width is used by characters that do not character-specific left spaces.

When Use Default is selected, it is treated as though 0 was specified.
I Use Default

Selecting the Use Default check box to the right of each input field enables the default value for the corresponding parameter.

However, if bffnt is the input side, the default values correspond to the values configured in bffnt.
Info

Specifies the character by either directly inputting it or by specifying the character code.
If only one character is input, direct character is specified. If two or more characters are input, character codes are specified.
Specify character codes in octal, decimal, or hexadecimal according to the rules in the C language.
When Use Default is selected, the characters with the smallest character codes among the characters included in the font resource are selected.
