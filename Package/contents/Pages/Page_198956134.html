Changes to Graphics for Developers

This section describes the changes made to NintendoWare Graphics for developers in NintendoSDK 1.1.0.
“NintendoWare Graphics” refers to the nn::g3d, nn::ui2d, nn::font, and nn::vfx modules.

nn::g3d News

Support for Resource Data Created With NX Add-On 0.12.x

Shader archive files (BFSHA) created with NX Add-On 0.12.x are not compatible because the major version of the binary is different, but support has been added to handle them as resource data. This change means that shader archive files created with NX Add-On 1.0.0 are no longer supported. They must be reconverted in the new 3D shader converter. The following resource data is supported by the g3d runtime in this package.

  • Resource data created with NX Add-On 0.12.x
  • Resource data created with NX Add-On 1.1.0

nn::vfx News

Added VfxBinarySegmentation sample.

Added a sample that outputs the textures, g3d primitives, and shaders in the effect binary as individual files when they are output, and combines and uses them in the runtime.

Made changes so that the number of jobs can be specified when using ShaderConverter.

Made changes so that the number of jobs specified with -j is passed to ShaderConverter when using EffectMaker for binary conversion.

Info

Whether the number of jobs passed to ShaderConverter operates correctly is based on the API specified by --api-type.
If the number of jobs is not specified, it will be based on the ShaderConverter specifications.
For more information, see the ShaderConverter documentation.

Workaround for Breaking Change Due to Improvements to Stripe Feature

You need to be aware of the following points when playing data that was created using a version of the stripe feature prior to this release.

  • When stripes are used, the Masking Check preset is selected for Soft particles in the Color tab.
  • When stripes are used, Z-axis rotation is enabled in the Rotate tab.

As a result of these changes, the results of rendering data that was created with prior versions, may change in unintended ways.
Run the following script to detect emitter set intermediate files to which this pertains.

import glob
from xml.etree.ElementTree import *

print("");
print("");
print("-------------------------------------------------------");
files = glob.glob('C:/data/*.eset');
print("find eset using StripeHistory [StripeType == 1 or 2]");
for file in files:
    print("-----------------------");
    print(file);
    tree = parse(file);
    elem = tree.getroot();
    for emitter in elem.findall("./EmitterList/EmitterData"):
        item = emitter.find(".//ReservedShader/PageData/ContentsData");
        if(item == None):
            continue;
        print("  " + item.tag);
        prop = item.get('{http://www.w3.org/2001/XMLSchema-instance}type')
        if(prop == None):
            print("not found xsi:type");
            continue;
        if(prop != "EPStripeHistoryData" and prop != "EPStripeHistoryTailData"):
            continue;
        print("    xsi:type = " + prop);
        stripeTypeNode = item.find(".//StripeType");
        if(stripeTypeNode == None):
            print("-----");
            continue;
        print("    StripeType = " + stripeTypeNode.text);
        if(stripeTypeNode.text != "1" and stripeTypeNode.text != "2"):
            continue;
        print("found file = " + file);

nn::ui2d News

Added a feature for searching for all resource names of a type specified by the resource accessor.

Added the FindResourceByType method to search for all resource names of the type specified by the nn::ui2d::ArcResourceAccessor and nn::ui2d::MultiArcResourceAccessor classes.

FindResourceByType is added to nn::ui2d::ResourceAccessor as a pure virtual function, so overwrite it if nn::ui2d::ResourceAccessor is inherited by the application.
