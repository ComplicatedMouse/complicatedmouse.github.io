<?xml version="1.0" encoding="utf-8"? >
<fragmentOperationPresets xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns="http://www.nintendo.co.jp/NW4F/LayoutEditor">
  <fragmentOperationPreset name="preset1" order="-5" useDefaultBlendSettings="true" useDefaultAlphaTestSettings="true">
    <label>Preset 1</label>
    <description>Default settings</description>
    <alphaCompare comp="Always" ref="0" />
    <blendMode type="Blend" srcFactor="SrcAlpha" dstFactor="InvSrcAlpha" blendOp="Add" />
    <blendModeAlpha type="None" />
  </fragmentOperationPreset>
  <fragmentOperationPreset name="preset2" order="5">
    <label>Preset 2</label>
    <alphaCompare comp="Less" ref="0.5" />
    <blendMode type="Logic" logicOp="InvCopy" />
    <blendModeAlpha type="None" />
  </fragmentOperationPreset>
  <fragmentOperationPreset name="preset3" visible="false">
    <description>Hidden</description>
  </fragmentOperationPreset>
  <fragmentOperationPreset name="_AlphaBlend" order="15" useDefaultAlphaTestSettings="true">
    <label>Translucent</label>
    <description>Overwrites standard settings</description>
    <alphaCompare comp="Always" ref="-1" />
    <blendMode type="Blend" srcFactor="SrcAlpha" dstFactor="InvSrcAlpha" blendOp="Add" />
    <blendModeAlpha type="Blend" srcFactor="V1_0" dstFactor="InvSrcAlpha" blendOp="Add" />
  </fragmentOperationPreset>
</fragmentOperationPresets>

The configured presets are then displayed in the presets combo box in the Fragment Operations page of the property window.

fragmentOperationPreset Attributes

Name Description
name Specifies the preset name.
visible If false, the preset is not displayed. Treated as true when omitted.
order Specifies the preset display priority as an integer. Presets are displayed in descending order, with the smallest values on top. The presets available in LayoutEditor from the beginning are equivalent to order="0". 
useDefaultBlendSettings
 If true, the default blend settings are used. If omitted, this is treated as false. 
useDefaultAlphaTestSettings
 If true, the default alpha test settings are used. If omitted, this is treated as false.

fragmentOperationPreset Elements

Name Description
label

Specifies the preset label. The label is displayed in the presets combo box.
description Writes the preset description. The description is displayed on the presets page when the preset is selected. 
alphaCompare
 Alpha test settings. 
blendMode

Color blend settings.

 
blendModeAlpha
 Alpha blend settings.
Info

The method for writing useDefaultBlendSettings, useDefaultAlphaTestSettings,alphaCompare, blendMode, and blendModeAlpha is the same as the method for the items of the same name for materialCtr in the layout intermediate FLYT file.
When preparing presets, we recommend copying these items from the layout intermediate file.

Overwrites the standard LayoutEditor presets.

You can change or hide the settings of the standard LayoutEditor presets by preparing presets with the following names.

Name Standard Label
_None Opaque
_Mask Transparent
_AlphaBlend Translucent (default)
_Add Additive
_Mul Subtractive
_Sub Multiplicative
_AlphaBlendMaxA Translucent: Max alpha

Using Multiple Preset Files

All FLFP files in the parts root folder (the same folder as the FLPJ files) are loaded.

The setting for loading later files first when there are multiple presets with the same name.
For example, if you divide files into an FLFP file that determines the overall rules, and an FLFP file that determines individual team rules, you can use the individual rules that are loaded later to override the overall rules.

The files are loaded in alphabetical order (case-insensitive), so change the filenames to adjust the order.
