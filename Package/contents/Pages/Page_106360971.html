Overview

NintendoSDK Build System

NintendoSDK assumes that development is taking place in the development hardware environment and the Windows environment together with Visual Studio. Similarly, the build system provided by NintendoSDK is centered around components that are required for development in Visual Studio. These include the following components:

  • Visual Studio property sheets
  • Visual Studio wizards for creating new NintendoSDK application projects and library projects
  • Visual Studio extensions (including a VSI debugger)

The add-on packages provide the following components:

  • Compiler tool chains
  • Various PC tools

NintendoSDK also provides developers with the means of constructing their own build system that is not dependent on Visual Studio. However, this is provided merely in the form of information, and without other support like examples of implementation using specific build tools.

Constructing a Build Environment

NintendoSDK assumes that developers will use one of the following methods to build applications and libraries that use the NintendoSDK.

  1. Create a new project file using the Visual Studio project creation wizard.
  2. Add a new project project configuration to an existing Visual Studio project file, and set it to use NintendoSDK.
    1. Configure using the Visual Studio extensions provided by NintendoSDK.
    2. Import the project sheet provided by NintendoSDK.
    3. The developer configures the necessary build settings.
  3. Construct a build environment from scratch.
    1. Create a Visual Studio project from scratch.
    2. Construct a proprietary build system without using Visual Studio.

If you are using NintendoSDK for the first time, we recommend using one of the following procedures to construct the build environment. The optimal procedure depends on which build methods you select.

If Creating a New Visual Studio Project

Create a NintendoSDK Application project from Visual Studio using the NintendoSDK wizard for creating a new project.

This procedure is suitable if you just want to use NintendoSDK and skip the detailed configuration of settings.

If There Is an Existing Visual Studio Project

In this case, you can begin development using NintendoSDK by configuring the necessary project settings as described in Using NintendoSDK With an Existing Visual C++ Project. Also see the Tutorial, which presents an example of configuring a Visual Studio project.

If you want, you can manually configure the necessary settings, using Creating a Proprietary Build System as a guide, rather than using the Visual Studio extensions and property sheets provided by NintendoSDK.

This procedure is suitable if you already have libraries, middleware, and other Visual Studio projects, and you want to migrate them to NintendoSDK.

If Constructing a Build Environment From Scratch

Refer to Creating a Proprietary Build System.

This procedure is suitable for developers who want to manage all of the essential build settings on their own.

Other Information

You can obtain information about the NintendoSDK build system from the Manual and from the Reference. Read these documents as needed to clarify points when constructing and tweaking your build environment.

Information specific to the compiler is available from Compiler, and information specific to the VSI debugger is available from VSI.
