Overview

In-game editing is one of the features of atk. You can connect to SoundMaker and adjust the content of sound archives while the application is running.

In-Game Editing Overview

Possible Operations Using In-Game Editing

The following examples show the kinds of editing that are possible.

  • Adjusting the sound volume and other parameters
  • Adding and deleting stream sound tracks
  • Substituting waveforms, sequences, and banks
  • Playing sounds in the application

Note: You cannot change sound types in SoundMaker. (Wave sound > Sequence sound)

Advantages of Including the Library

Using in-game editing has the following advantages.

  • The opportunity to conduct a greater amount of trial and error

    Until now, you had to restart the application every time you made an adjustment to parameters and waveforms.
    By using in-game editing, changes to volume and substitutions of waveforms are applied immediately without restarting the application.

  • The ability to make adjustments in the application

    You can adjust parameters while listening to how they sound in real time in the actual game.

In-Game Editing System

This section provides a simple description of the in-game editing system.

To conduct in-game editing, you must include the atk::viewer library in the application so that you can make a host-target connection to SoundMaker.
SoundMaker sends the edited data to the atk::viewer library, which uses that data to substitute the data that the atk library plays.

Editing Procedure

  1. Open the project that you want to edit with SoundMaker.
  2. Connect SoundMaker to the application that includes the atk::viewer library.
    • Select the platform and connection (computer or development hardware) from the SoundMaker toolbar and click the Connect button ().
  3. From the project tree list in SoundMaker, select the sounds and banks you want to edit.
  4. The edited data is automatically converted and sent.
    The sent data is cached.
  5. The edits are applied the next time sounds play.
    If there is a cache, it is prioritized. If not, the original data is used.

In-Game Editing Specifications

Operating Conditions

  • By default, the library is only enabled in the Debug and Develop builds.
  • To ensure that names match, you must select the Output the string table option in the sound project settings.

Types of Items That Can Be Edited

The following types of items can be edited with in-game editing.

  • Stream sounds
  • Wave sounds
  • Sequence sounds
  • Bank

Note: Waveform archives, players, and groups are not included.

Timing for Applying Edits

After the edited data is sent, the edits are applied the next time sounds play.
The exceptions are edits to the volume for sounds and Stream Sounds tracks, which are immediately applied to the playing sounds.

Monitoring for Updates to Edit Target Files

SoundMaker monitors for updates to the files that are referenced by the items that are the targets of in-game editing, and automatically resends them.
For example, if a wave sound is the target of editing, the changes are applied to the executing application just by the external editor saving the relevant waveform file.

Consistency of Sound Archives and Sound Projects

When you try to play a sound in the application, the matching edit data is applied to the sound. This enables you to edit sound projects that are open in SoundMaker and play the sounds in the application, even if you are using an earlier version of the sound archive created from that sound project in the application.
