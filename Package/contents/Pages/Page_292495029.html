Overview

Use the nn::oe::ReportUserIsActive() function at any time to reset the count of continuous idle time.

Using this feature for retail products requires approval because special considerations are required when using it.

In addition, you must comply with additional guidelines when using this function.

This page describes the anticipated use of this feature, important considerations during use, and applying for approval to use this feature for retail products.

Anticipated Use

The function is designed to be used in situations where the application is using a particular play format, such as with a special accessory, and has its own defined way of detecting user operations which the system itself cannot detect. In such cases, the application can use this function to reset the idle-time count when it detects an operation.

Info

Do not use this function in NX Add-On 5.2.0 and later in cases such as in music playback mode where the user would not be directly using the device, but the device continues to operate.

Use the Feature to Temporarily Disable Auto-Sleep (Approval Required).

Implementation Notes

If this feature is called continuously, the Screen Burn-In Reduction and auto-sleep features might be disabled unintentionally by the user. It is important to implement this function with care, calling it only when the user conducts actual, meaningful, operations.

For example, you want to call nn::oe::ReportUserIsActive() periodically so the Screen Burn-In Reduction feature does not run while the user is operating a special accessory device. In that case, refrain from calling this function when the user is not operating the device.

Do not call the function in every frame, just because you expect the user to be handling the device at all times.

Getting Permission To Use in Retail Product

To use this feature in the retail product, you must apply for permission ahead of time. If you want to use the feature, contact Nintendo.

When your apply for permission to use the feature, include the following information so the reasonableness of the request can be determined.

  1. Scenes in the application where nn::oe::ReportUserIsActive() will be used.
    • Example: A scene where a special accessory device will be used for operations in xx mode.
  2. The trigger for the nn::oe::ReportUserIsActive() function call (the criteria for determining that a user operation has occurred) in the scene described in step 1 (or multiple triggers if there are multiple scenes).
    • Example: When the user has pressed a button on the special accessory device.

Guidelines

Guideline items have been prepared for using the feature, in the way that was described when you applied for usage permission.

For that reason, the person responsible for reviewing the guidelines when a master submission occurs will reference any data reported when the submission was made.

For more information about these guidelines, see the following sections in the Nintendo Switch Guidelines.

  • 0130: Use of nn::oe::ReportUserIsActive() is prohibited for situations and triggers other than what was reported in the usage approval application.
