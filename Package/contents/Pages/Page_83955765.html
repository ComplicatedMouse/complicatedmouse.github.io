Waveform archives

Select the Waveform Archive node in the project tree to display the waveform archive list in the Sound Set tab.
The displayed list is called the waveform archive list.

Role of Waveform Archives

Waveform archives group multiple waveform data files that are associated with banks or wave sound sets into a single entity.

Multiple banks and multiple wave sound sets can be associated with a single waveform archive.

Creating New Waveform Archives

To create new waveform archives, double-click the Waveform Archive node in the project tree to display the waveform archive list in the Waveform Archive tab.

Then, right-click the waveform archive list, and select either the Add Item or Insert Item option from the shortcut menu. This action displays the Create New Waveform Archive dialog box. Type the name of your waveform archive, and specify whether to load it individually.

You can also delete waveform archives by right-clicking an archive and clicking Delete on the shortcut menu.
If the waveform archive that you selected for deletion is already being referenced by banks or wave sound sets, a confirmation dialog box appears asking whether you are sure you want to delete it.
Deleting a waveform archive deletes only the links to files referenced by sound sets. It does not delete the actual files being referenced.

Elements of the Waveform Archive List

This section describes the meaning of each of the elements that define a waveform archive list.

Name

Specifies the filename of the waveform archive.

The characters that can be used in a name are restricted. The following characters can be used:

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9_] for subsequent characters

Load Individually

Select this check box to individually load only the required waveform data within a waveform archive. If you select this option, the entire waveform archive might not be loaded.

This option works well when a single waveform archive is shared over multiple banks, as in the following.

Waveform data that is used only in Bank A may be included in the shared waveform archive. When you use Bank B, if you load waveform data for Bank A that Bank B does not use, memory efficiency decreases.
If the Load Individually option is enabled for this shared waveform archive, only the waveform data used by Bank B is loaded when using Bank B. Any waveform data not used by Bank B that is contained within the shared waveform archive is not loaded.

Overhead on Memory Consumption and Load Time

Selecting the Load Individually option can introduce some overhead in terms of memory usage and load time, relative to what is required when all waveform data in the archive is loaded at the same time.
In most cases, the overhead issue can be ignored because unnecessary data sets are not loaded, which reduces memory consumption and shortens data set loading time. However, the overhead increases when there is little or no unnecessary data.
