Changes for Developers

This section describes the changes made to NintendoWare audio (nn::atk) in NintendoSDK 0.16.0.

Contents

  • Spy now supports nn::atk thread visualizations.
  • Revisions to the Documentation

Spy now supports nn::atk thread visualizations.

Added support for visualization of some nn::atk sound and task threads and nn::audio DSP processes to the Spy timeline.

For more information, see the Spy.exe Atk performance panel documentation.
For more information about embedding this functionality in applications, see the SpyAtk sample.

Revisions to the Documentation

The following revisions have been made.

  • Fixed a typo in the functions used in the Using Sample Data Obtained From CircularBufferSink sample code on the Quick Start page of the Programming Guide.
    • Changed nn::atk::SoundSystem::GetChannelCountMax to nn::atk::SoundSystem::GetRendererChannelCountMax.
      The former function is deprecated as of version 0.16.0. Please use the latter function from now on.
