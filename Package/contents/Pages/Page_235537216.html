Getting Launch Parameters

Feature Overview

The application can receive launch parameters for itself. These startup parameters can be used by the application to control its own behavior according to the context of when they are opened.

The content of the application launch parameters are, regardless of circumstances, determined by the parties responsible for application development, planning, or operation. The system does not, on its own, pass a startup parameter that is unknown to the application. In addition, the system and NintendoSDK functions do not affect the content of the launch parameters.

System Features That Can Pass Launch Parameters

Launch parameters for the following features can be passed to the application.

  • News
    • Launch parameters can be specified when starting applications from game news.
    • For more information, see Nintendo Switch News Delivery Guide > BCAT Management Tool: Setting Items and Setting Rules > Starting a Specific Application.

Using the Feature for Getting Launch Parameters

  1. include the oe library header.

     #include <nn/oe.h>

  2. Call the function for getting oe library launch parameters while passing a buffer for storing those parameters.

    char parameterBuffer[4096];
size_t outRealParameterSize;
auto isSuccess = nn::oe::TryPopLaunchParameter(&outRealParameterSize, parameterBuffer, sizeof(parameterBuffer));

    Receive the parameter after allocating a sufficient buffer to receive the expected startup parameter (parameterBuffer[] array in the preceding example).

Getting Launch Parameters

Currently, launch parameters are not passed for anything other than News. Multiple launch parameters may be passed in the future with feature expansions. In this case, it is possible to get each parameter according to the order in which the parameters were pushed.

Notes About Launch Parameters

Implement the application so that it expects to get multiple launch parameters, skips the unknown parameters, and handles the known parameters correctly.

In particular, if older applications are run on newer firmware in the future, the application may pass launch parameters that it does not recognize. To ensure there are no issues, carefully consider the following information before using launch parameters.

  • Launch parameter format definitions
    • Make sure that the older version of the application can determine when a launch parameter, defined in the future, is in an unknown format.
  • Implementation for getting launch parameters
    • You must prepare an adequate buffer size in advance.
    • Ignore unknown launch parameters.
    • To process known parameters correctly, repeatedly get parameters until none are left.

Development Support Features for Getting Launch Parameters

The DevMenu has a feature to pass application launch parameters from a binary file installed on the SD card.

For more information, see Installing and Uninstalling Launch Parameters in the DevMenu Manual.
