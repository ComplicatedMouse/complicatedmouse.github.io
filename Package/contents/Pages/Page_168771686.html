Operation is not assured when the system locale is changed to anything other than the default value.

The operation when the system locale is changed to anything other than the default value is not assured. (For example, if the locale is set to anything other than "Japanese [Japan]" in the Japanese version of Windows.)

audio Bugs

Sometimes effects are not processed in the order they were added in nn::audio.

By design, effects are supposed to be processed in the order in which they were added to SubMix or FinalMix. However, if you have two or more effects registered to a single SubMix or FinalMix, and you then delete an effect and add another new effect, the effects sometimes run in the wrong order.

You can avoid this problem by deleting all effects from SubMix and FinalMix and then adding the effects.

This bug will be fixed in an upcoming release.

When a sound waveform is output from NintendoWare SoundMaker or SoundProjectWaveExporter, silence is occasionally added to the beginning of the waveform.

When a sound waveform is output from NintendoWare SoundMaker or SoundProjectWaveExporter, sometimes silence is added to the beginning of the output waveform.

This bug will be fixed in a future release.

When you convert a sound project imported using NintendoWare's SoundMaker, an invalid sound archive is exported.

When, in SoundMaker, an NW4C/NW4F sound project (one for which the cache file remains for converting purposes) is imported and (normally) converted, NW4C/NW4F binary source files become mixed in with the NintendoSDK binary sound archive (BFSAR) files. As a result, they cannot be correctly read during the nn::atk and nw::snd runtimes. This is a problem.

The workaround is to make sure that you reconvert the sound project after importing it. This makes the old cache file unusable, which avoids the problem.

NintendoWare AtkPlayer can stop running if you change the DelayTimeMax value.

The application can stop running if you change the DelayTimeMax parameter for the Delay effect in the AtkPlayer Effect panel.

This bug will be fixed in an upcoming release.

Reloading a sound archive in NintendoWare AtkPlayer causes the application to crash.

After editing a stream sound in NintendoWare SoundMaker and converting the sound archive file, reloading the archive in AtkPlayer sometimes causes the application to crash.

This bug will be fixed in an upcoming release.

Switching the sound in NintendoWare AtkPlayer sometimes causes the application to freeze or crash.

While a sound is playing in the AtkPlayer SoundArchivePlayer panel, switching to another sound sometimes causes the application to freeze or crash. If the application freezes, try waiting a few moments for it to become responsive again. If the application crashes, restart the application. For now, this problem can be avoided by making sure no sounds are playing when you switch to another sound and by not switching between sounds too frequently.

This bug will be fixed in an upcoming release.

The process for freeing nn::atk::SoundHeap objects sometimes takes a long time.

If you try to free a SoundHeap after loading a large amount of data into it, the deallocation process sometimes takes a long time to complete.

This bug will be fixed in an upcoming release.

When using the in-game editing feature in the nn::atk::viewer library, changing the file path of a sequence sound sometimes causes the program to crash.

Changing the file path of a sequence sound that is being edited using the in-game editing feature sometimes causes the program to exit with an assertion. Currently, it has been confirmed that the program is exiting with the following assertion.

SDK Assertion Failure: 'velocity >= 0 &amp;&amp; velocity <= 127' in nn::atk::detail::driver::SequenceTrack::NoteOn()

This problem will be fixed in an upcoming release.

In rare cases, the program crashes when playing stream sounds being edited using the in-game editing feature of the nn::atk::viewer library.

In rare cases, the following end error occurs and the program crashes when repeatedly playing stream sounds being edited using the in-game editing feature or when rapidly changing the stream sound parameters using the in-game editing feature.

Abort: 'delimiter != nullptr' in nn::fs::detail::GetMountName()

This bug will be fixed in an upcoming release.

In nn::atk, playing stream sounds while using PlayerHeap sometimes causes the nn::atk thread to become deadlocked.

If you play a stream sound while already using PlayerHeap to play another stream sound or a sequence sound, the nn::atk sound thread and task thread sometimes transition to a waiting state where you cannot use the library to perform any other tasks.

This bug will be fixed in an upcoming release.

In the nn::atk library, when looped playback is enabled for an Opus sound, sometimes erroneous noise occurs when the sound loops.

Symptoms

When looped playback is enabled for an Opus stream sound, a small amount of silence is inserted each time the sound is looped, which manifests as audible noise.

Workaround

No workaround is currently available.

Fix Schedule

This bug will be fixed in an upcoming release.

fs Bugs

When nn::fs::ReadDirectory() is called for the root directory of a drive, occasionally the first entry is not listed.

There is a bug where sometimes the first entry is not listed when nn::fs::ReadDirectory() is called via nn::fs::MountHost()/MountHostRoot() immediately below a drive when the target is a type of removable media.

The mounting API functions cause memory leaks.

Symptoms

After calling a mounting function at least one time, when you unmount all resources, approximately 144 bytes of memory allocated from the allocator specified using nn::fs::SetAllocator() are not freed.

This bug causes an assertion failure in a memory leak check in the following sample demos when the demo is closed.

  • Ui2dSimple
  • Ui2dUserShader
Workaround

No workaround is currently available.

Fix Schedule

This bug will be fixed in the next release.

perf Bugs

The PerfSimple sample crashes in some PC environments.

The PerfSimple sample exits with an assertion in certain PC environments.

This bug will be fixed in a future release.

socket Bugs

The MSG_DONTWAIT flag cannot be used in the nn::socket::Recv() function.

Symptoms

Setting the MSG_DONTWAIT flag using the nn::socket::Recv() function does not put the module in non-blocking mode.

Workaround

You can achieve the same type of non-blocking behavior by using the nn::socket::fcntl() function to set the O_NONBLOCK flag.

Fix Schedule

This problem will be fixed in an upcoming release.

The nn::socket::Select() and nn::socket::Poll() functions sometimes return EBADF.

Symptoms

The nn::socket::Select() and nn::socket::Poll() functions sometimes return EBADF when called if the socket descriptor is closed.

Workaround

No workaround is currently available.

Fix Schedule

This problem will be fixed in an upcoming release.

SSL Bugs

Call the nn::ssl::Initialize() and nn::ssl::Finalize() functions on the main thread.

Symptoms

Calling the nn::ssl::Initialize() and nn::ssl::Finalize() functions on a thread other than the main thread causes the library to stop working.

Workaround

Call the nn::ssl::Initialize() and nn::ssl::Finalize() functions on the main thread.

Fix Schedule

This problem will be fixed in an upcoming release.

Do not call the nn::ssl::Initialize() and nn::ssl::Finalize() functions multiple times.

Symptoms

The nn::ssl::Initialize() and nn::ssl::Finalize() functions are not intended to be called multiple times. Calling these functions multiple times may result in unexpected errors.

Workaround

Only call the nn::ssl::Initialize() and nn::ssl::Finalize() functions one time each (when you start using the library and when you stop using the library, respectively).

Fix Schedule

This problem will be fixed in an upcoming release.

HTTPS Bugs

HTTPS communications cannot be sent from multiple applications on the same PC.

Symptoms

If you start HTTPS communications from multiple applications on the same PC, sometimes an error occurs.

Workaround

No workaround is currently available.

Fix Schedule

This problem will be fixed in an upcoming release.

ui2d font Bugs

Alpha Test does not work.

The GL implementation of alpha test does not work.

This problem will be fixed in an upcoming release.

Texture display cannot be performed correctly for LayoutViewer in some graphics driver environments.

In some graphics driver environments, the textures in LayoutViewer are not displayed correctly. (We have seen this bug in the Quadro2000 driver 341.05 environment.)

Right Wrong

In LayoutEditor, when a screenshot is captured using the PC viewer, the entire screen is not captured correctly.

Symptoms

If you select Capture current frame with PC Viewer from the tool menu to capture a screenshot, only a portion of the screen of the resolution specified in Layout Environment Settings > Screen Size is captured.

Workaround

When taking screenshots, set Layout Environment Settings > Screen Size to 1280×720 to make the capture feature behave the same as in previous releases.

Fix Schedule

This problem will be fixed in an upcoming release.

3D Bugs

Changing the name of a model or animation in 3DEditor causes the names of objects to stop being updated correctly in the connected application.

Changing the name of a model or animation in 3DEditor while 3DEditor is connected to an application causes the names of objects to stop being updated correctly in the connected application.

This bug will be fixed in an upcoming release.

An invalid access event sometimes occurs when running the G3dDemo Town sample.

Depending on the environment used, an invalid access event sometimes occurs when running the Generic version of the G3dDemo Town sample.

This bug does not occur if you stop using nn::perf or change the number of models rendered. Additional details about the cause of this bug are unknown at this time.

Mii characters cannot be displayed correctly using the libnn_g3Mii library.

Symptoms

When a Mii character is displayed, a portion of the character model is not drawn in the correct shape.

Workaround

In the libnn_g3dMii source code (which is available to all developers), change line 123 of NintendoSDK\Sources\Libraries\g3d\mii\g3d_MiiConverter.cpp from:

vertex.count = static_cast<uint32_t>( pDrawParam->GetBufferSize(nn::mii::DrawParam::AttributeType_Position) / ( sizeof(float) * 4 ) );

to:

vertex.count = static_cast<uint32_t>( pDrawParam->GetBufferSize(nn::mii::DrawParam::AttributeType_Position) / ( sizeof(float) * 3 ) );

Then, rebuild the library to fix this problem.

Fix Schedule

This bug will be fixed in an upcoming release.

Spy Bugs

If multiple instances of Spy.exe are running, you cannot communicate normally with the application.

If you run multiple instances of Spy.exe and attempt to connect each instance to the application, the second and later instances of Spy.exe cannot communicate normally with the application after being connected.

Continue to use the first connected instance of Spy.exe to communicate with the application.

vfx Bugs

When the Emitter Assignment for Particles inheritance setting for child emitters in effect data is disabled, specifying Historical Stripe or Historical Stripe2 as the particle type causes the VFX runtime to stop responding.

Symptoms

When the Emitter Assignment for Particles inheritance setting for child emitters in effect data is disabled, specifying Historical Stripe or Historical Stripe2 as the particle type causes an invalid access event to occur in the VFX runtime.

Workaround

You can avoid this bug by enabling Emitter Assignment for Particles in the effect data settings.

Fix Schedule

This bug will be fixed in an upcoming release.

Time Bugs

A link to libcurl.lib is required to enable use of the network clock feature.

Symptoms

A link to libcurl.lib is required to use the nn::time::EnsureNetworkClockAvailability() function and the nn::time::AsyncEnsureNetworkAvailabilityTask class.

Workaround

Link libcurl.lib. However, libcurl does not need to be initialized.

Fix Schedule

This bug will be fixed in an upcoming release.

The alphabet list is not displayed correctly in functions such as API references.

In API references, under Namespace > Namespace Members, the alphabet lists displayed under All, Functions, and Enumerated Values do not display the correct lists when selected.
