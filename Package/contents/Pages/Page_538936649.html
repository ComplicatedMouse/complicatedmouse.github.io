Sample Overview

Overview

The Auto Test Assist Tool library includes samples written in Python, C#, and C++.

  • Python samples
    • Samples\Sources\Tools\AutoTestAssistTools\ScriptSamples
  • C# samples
    • Samples\Sources\Tools\AutoTestAssistTools\CSharpSamples
  • C++ samples
    • Samples\Sources\Tools\AutoTestAssistTools\CppSamples

This section provides an overview of each sample and its operating environments. For more information about the implementation and behavior of each sample, see the sample source codes.

Python Samples

Overview

These sample scripts are based on Python and are intended for checking the various features of the Auto Test Assist Tool library.

Operating Environment

Before running the sample, connect Target Manager to the target development hardware and start InputDirector.

Specify the serial number of the operation target device as a common argument in the first argument.


Operation is confirmed for the following environments.

  • Python 3.10.0 (x64)


In the sample that includes the screen capture functionality, you can change the value of the capture_mode argument to switch between capture settings.

Setting Value Overview
CaptureMode.CaptureMode_None No capture
CaptureMode.CaptureMode_720p 720p capture using the capture board
CaptureMode.CaptureMode_1080p 1080p capture using the capture board
CaptureMode.CaptureMode_DevKit Capture using the DevKit (SDEV only)
CaptureMode.CaptureMode_RemoteVideo Capture equivalent to RemoteVideo in Target Manager
CaptureMode.CaptureMode_Screenshot Capture equivalent to TargetManager screenshots

Wrapper Sample

File Overview
HACDllDriver.py

Wrapper class definition file for using the Auto Test Assist Tool library features in Python.

The samples depend on the functionality of HACDllDriver.py to run.

Samples

File Overview
ConfirmDeviceList.py

Sample to display a list of available capture devices.

Use this sample in situations such as when you are confirming the correlation between device names and device IDs when multiple capture devices are connected in the runtime environment.
ExportScript.py

Sample demonstrating how to record HID operations and use the Python script export feature for the recorded operations.

After running this sample, HID input continues to be recorded until Enter is entered on the console, and operations recorded to RecordedInput.py are saved to a script after execution ends.
HidInputterSample.py

Sample showing the execution of the various features of the HID Operation library.

Makes it easier to check what kinds of operations are being performed when run while the HidSimple sample (mples\Sources\Applications\HidSimple) has been started on the target development hardware.
HomeMenuSample.py Sample combining simple Mii editing, get capture, and delete capture operations from the HOME Menu screen.
ImageProcessSample.py

Sample of image processing using the capture library, and of getting pixel arrays from the capture library and image processing with Python-based OpenCV instead of DLLs.

To run, the cv2 and numpy libraries must be installed in Python.
LogReaderSample.py

Sample showing how to retrieve a log, like getting the latest log or searching for a log using regular expressions.
Ui2dSample.py Sample that automatically operates the Ui2dDemo sample (Samples\Sources\Applications\Ui2dDemo) to check the various screen displays through template matching.
Run when Ui2dDemo has been executed.
VirtualControllerSample.py

Sample showing example controller connections with the device type and interface type specified.

Opens the controller support applet from the HOME Menu screen and connects and disconnects the Switch Pro Controller or Joy-Con.

Program Overview
CaptureProcessorSample

This sample demonstrates the basic use of CaptureProcessorNet.

Some image processing is likely to run in the HOME Menu Screen. Open the HOME Menu on the target development hardware and run the sample.

The samples depend on AutoTestAssistToolWrapper to run.
