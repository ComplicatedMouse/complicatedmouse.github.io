os

Cannot get the correct thread identifier for the main thread in the Windows environment

Symptoms

Calling the nn::os::GetThreadId() function for the main thread outside of the main thread in the Windows environment returns an invalid thread identifier.

Workaround

When getting the thread identifier of the main thread in the Windows environment, call nn::os::GetThreadId() from the main thread.

Fix Schedule

This will be fixed in NintendoSDK 12.0.0.
