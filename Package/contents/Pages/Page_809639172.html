In-Process vs. Out-Of-Process

Profiling Modes

There are two profiling modes that are selectable on the Sampled Profile tab:

  1.  In-Process (best mode if your game does not have migrating threads)
  2.  Out-of-Process (best mode if your game has migrating threads)

In-Process Profiling Mode

In this mode, all sampling takes place within the game application, meaning the profiler lives within your application. Profiling with  In-Process requires integrating the profiler library into your game (Full Integration Setup). 

Info

In-Process profiling is the best mode to use if your game does not allow threads to migrate between cores and all threads are running at a lower priority than nn::os::HighestThreadPriority.

Benefits of In-Process

  • This method typically has lower overhead than  Out-of-Process.
  • The debugger can be attached while profiling.
  • Works on both EDEV and SDEV. (EDEV requires using the In-Process Only Scenario.)

Issues with In-Process

  • If any thread can migrate between cores (the thread has an affinity of more than one core), the profile data will not be accurate. In this case, you must use  Out-of-Process profiling for accurate results.
  • If any thread is running at nn::os::HighestThreadPriority then it cannot be sampled.

Out-of-Process Profiling Mode

Out-of-Process records profiles from a process separate from your game process.

Benefits of Out-of-Process

  • Can accurately profile threads that migrate between cores (have more than one core affinity).
  • Can accurately profile threads running at nn::os::HighestThreadPriority.
  • No profiler library integration is required (but you should still perform Full Integration Setup to get the maximum benefit from the profiler, such as recording thread synchronization events).

Issues with Out-of-Process

  • This method typically has higher overhead than  In-Process.
  • The profiler library must still be integrated to use features other than basic Sampled profiling.
  • The profiler and debugger cannot both be attached to the application at the same time.
  • Only works in 6GB mode; so this is currently only supported on the SDEV.
