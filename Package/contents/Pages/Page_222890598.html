nn::g3d News ()

Updated the binary resource file format.

While reserved regions for BFRES and BFSHA files were added in NX Add-On 3.0.0, in some cases, this change affected the CPU load. Therefore, the BFRES and BFSHA file reserved regions were adjusted again in NX Add-On 3.1.0.

This change breaks the compatibility of binary files created using previous versions of 3D Binary Converter and 3D Shader Converter, so resource files must be reconverted.

Added support for the model intermediate file --magnify option to the intermediate file converter.

You can now magnify or shrink models by loading the model intermediate file in the intermediate file converter and specifying the --magnify option.

Use this option when reusing old model intermediate files for new projects.

nn::vfx News ()

Changed the fragment shaders to use the out variable.

Stopped using gl_FragData in the standard fragment shader and replaced it with out variables.

A shader compile error occurs when using gl_FlagData in the custom shader, so change the custom shader to use the out variables.

nn::ui2d News

Added a feature for setting non-rectangular mesh shapes in Picture panes.

Added a feature for specifying non-rectangular mesh shapes in the Shape settings of the Picture pane. It is possible to specify Rounded Rectangle and Circle, and adjust each of these shapes according to items such as the number of mesh divisions.

Using this feature requires allocation of the vertex buffer and index buffer, for displaying the mesh in the GpuBuffer region, so set nn::gfx::GpuAccess_VertexBuffer and nn::gfx::GpuAccess_IndexBuffer, in addition to nn::gfx::GpuAccess_ConstantBuffer, for the GpuAccessFlag in GpuBuffer. With this feature added, it is also necessary to link to libnn_gfxUtil when using libnn_ui2d.

Added a feature to nn::ui2d::GraphicsResource::Setup to get external memory pools.

Added an overload function to nn::ui2d::GraphicsResource::Setup to get external memory pools.
For more information about how to use this feature, see InitializeGraphicsResource() inSamples/Sources/Applications/Ui2dDemo/main.cpp.

nn::font News ()

Added a feature to nn::font::RectDrawer::Initialize to receive external memory pools.

Added an overload function to nn::font::RectDrawer::Initialize to receive external memory pools.
For more information about how to use this feature, see InitRectDrawer() in Samples/Sources/Applications/FontDemo/main.cpp.

Fixed a bug where nn::font::WordWrapping::CalculateWordWrapping() does not calculate the correct oscillating point.

There was a bug where using nn::font::WordWrapping::CalculateWordWrapping() to receive 6 parameters, caused it to fail to calculate the correct oscillating point on the development hardware. This is has been fixed.

There was a bug in the string length calculation processing, which resulted in cases where the passed strings were only partially parsed or parsed beyond the string length.

There is no issue when using an overload function with the same name, that receives the seven parameters, explicitly specifying string length.

Added a feature for drawing using HYPHEN-MINUS glyphs instead of NON-BREAKING-HYPHEN glyphs.

Becasue the NON-BREAKING-HYPHEN glyph does not exist in the internal fonts for some platforms, a feature has been added to render the NON-BREAKING-HYPHEN(U+2011) glyph by using the HYPHEN-MINUS(U+002D) glyph instead.

Enable this feature by setting nn::font::ScalableFont::InitializeArg::isDrawingHyphenMinusInsteadOfNonBreakingHyphen to true.

The default is true, so set this to false if there will be a problem with the differences in displaying NON-BREAKING-HYPHEN.
