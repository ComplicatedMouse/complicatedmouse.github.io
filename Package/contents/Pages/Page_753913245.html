Profiler-Sync

Overview

Syncs to the runtime application using the specified NSS.

Rather than use this for automation, it is generally recommended that you use the Command Line Options to specify an NSS to load automatically.

Profiler-Sync [[-FileName] <String>] [-Target <Identifier>]


[[-FileName] <String>]

The full path to the NSS to load. It is recommended that this path name be enclosed in quotation marks.

This argument can only be omitted if an NSS has already been specified using Profiler-NX-Scenario either manually or by requesting Autofill.


[-Target <Identifier>]

Specifies a target dev kit to connect to if there is more than one valid target. The identifier should be a way to uniquely identify a kit.

    • IP Address
    • Serial Number
    • Target Name


Returns

A Boolean value if the Sync was successful or not.

Examples

Basic Usage

Provided here are some basic usage examples.

$success = Profiler-Sync "C:/Game/bin/Release/game.nss"
$success = Profiler-Sync "C:/Game/bin/Release/game.nss" -Target "123.4.56.78"
