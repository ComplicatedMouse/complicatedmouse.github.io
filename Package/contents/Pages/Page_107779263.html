Creating New Projects

This section describes how to create a Visual C++ project to develop an NX application.

There are two ways to create such a project.

  • Creating Projects
    • Creating Applications
    • Creating a Static Library
    • Creating a New Dynamic Library
  • Adding Support for NX Development to an Existing Visual C++ Project
  • Developing Using Distributed Builds
  • Developing Applications in Coordination With an External Build System

Creating Projects

Info

Check Operating Environment (NX Add-On) to see which versions of Visual Studio are supported by NintendoSDK and NX Add-On.

Creating Applications

Start Visual Studio and select File > New > Project.

Select Templates > Wizards > NintendoSDK > NintendoSDK Application and enter values for the Name, Location, and Solution Name.

Info

Use when the default Environment Settings in Visual Studio are General, Visual C++, or Web Development. If you are using different settings (such as Visual C#), the settings are in Templates > Other Languages > Wizard > NintendoSDK > NintendoSDK Application.

Start the NintendoSDK wizard.

Select the environment installed using the NDI tool from the drop-down list at the top.

Warning

Currently, the absolute path that was selected here is added to the project.
As a result, if you plan to use the created project in an SDK environment installed in a different directory, you must rewrite the path written in this property sheet.
In the future, a method other than using the absolute path will be added to support referencing the root path. For now, you can also work around this problem by selecting the second option (Specify Environment variable or Path) and using an environment variable.
For more information, see Changing the Path to the SDK Used by the Project.


This opens the window for selecting the API set. Select NX to create a project file to which a property sheet has been imported for using the NX API set.
(If you select Generic, you will create a project file that is the same as one created in the NintendoSDK environment that does not include NX Add-On.)

Info

The NintendoSDK adopts the concept of API sets.

The API provided by NintendoSDK itself is called the generic API set.
The name for the API set that becomes available when you install NintendoSDK NX Add-On is called the NX API set.

Click Finish to create the project.
The properties of the Visual Studio project created by the wizard (such as Additional Include Directories, Additional Library Directories, and Additional Dependencies) are configured to reference the resources provided by NintendoSDK.

Creating a Static Library

In Visual Studio, select File > New > Project like you did to create a new application, and then select Templates > Wizards > NintendoSDK > NintendoSDK Library. Enter the appropriate values for Name, Location, and Solution Name.

Info

Use when the default Environment Settings in Visual Studio are General, Visual C++, or Web Development. If you are using different settings (such as Visual C#), the settings are in Templates > Other Languages > Wizard > NintendoSDK > NintendoSDK Library.

Follow the same procedure that you used when creating a new application to select the SDK environment and the API set, and then click Finish to create the project.

Creating a New Dynamic Library

In Visual Studio, select File > New > Project like you did to create a new application, and then select Templates > Wizards > NintendoSDK > NintendoSDK Library. Enter the appropriate values for Name, Location, and Solution Name.

Info

Use when the default Environment Settings in Visual Studio are General, Visual C++, or Web Development. If you are using different settings (such as Visual C#), the settings are in Templates > Other Languages > Wizard > NintendoSDK > NintendoSDK Library.

Similar to when creating a new static library, the wizard is displayed for you to input the settings. When the following library Advanced Settings page is displayed, select DLL for the library type and then click Finish.

Then the project will be created.

Adding Support for NX Development to an Existing Visual C++ Project

If NX Add-On has been added to the environment, an existing Visual C++ project can be configured for the purpose of NX application development. You can also set projects that were created with versions up to NX Add-On 0.3.0.
For information about the procedure, see Adding NX Development Hardware Platforms to an Existing Visual C++ Project.

Developing Using Distributed Builds

You can now also use the free distributed build tool FASTBuild as a distributed build.
You can use it by converting the project mentioned earlier into a BFF file.

Install the FASTBuildUtilizeKit package to use FASTBuild.

Developing Applications in Coordination With an External Build System

You can develop applications in coordination with an external build system called makefile.
For more information, see Using Makefile.
