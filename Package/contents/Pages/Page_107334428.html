Coordinate System

The gfx library has no features for defining and controlling its own coordinate systems, including 3D, clip, and window coordinate systems. It is dependent on low-level graphics APIs. As a result, when using gfx, you must be careful because coordinate system behavior can change depending on which low-level graphics API is in use. In particular, if you use different low-level graphics APIs for window coordinate systems, when you attempt to render textures you may find that some have been inverted.

There are two main types of coordinate systems: OpenGL and DirectX. To see which coordinate system the low-level graphics API is defining, see its documentation.

Many low-level graphics APIs that use the OpenGL coordinate systems have APIs for converting several of the coordinate systems to DirectX definitions. For example, with OpenGL, you can use the glClipControl function to set the origin of the window coordinate system to either the upper left or the lower left, and the depth mode of the clip coordinate system to either the range from -1 to 1, or the range from 0 to 1. Changes to the clip coordinate system's depth mode change the z equation when converting from a normal device coordinate system to a window coordinate system. When the depth mode range is from 0 to 1, floating-point addition is no longer needed for the equation, and the precision could be revised.
