Sequence Sounds

Sequence Sounds

Select the Sequence Sound node in the project tree to show the sequence sounds on the Sound Set tab.
The list that appears is called the Sequence Sound List.

Creating and Adding Sequence Sounds

To create or add a sequence sound, on the Sound Set tab, select the list, and then select Project > Add Item > Sequence Sound.

You can also create or add a sequence sound by right-clicking the Sequence Sound List and using the shortcut menu that appears.

Editing Sequence Sounds

Sequence sounds are edited from the Sequence Sound List.
The columns in the Sequence Sound List show the defined elements that are essential to the sequence sound. They correspond to the elements that are essential for the sequence sound data in the sound archive.

Elements of a Sequence Sound List

This section describes the meanings of the defined elements in the Sequence Sound List.

Play

Clicking this button registers the selected sound in the Preview Player window, and triggers playback of the sequence sound.

Name

Specifies the name of a sequence sound.
Programs use this name to play the sequence sound through the atk library from the sound archive that is ultimately created.

The characters that can be used in a name are restricted. The following characters can be used:

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9_] for subsequent characters

File Type

Select the type of reference file.
Select SMF when using a standard MIDI file or text when using a text sequence file.

File path

Specifies the file to reference when creating a sequence sound.

The characters that can be used in a filename or path name are restricted. The following characters can be used.

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character of the filename.
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9, _] for subsequent characters in the filename.

To open the file that the file path specifies, right-click the Sequence Sound List and select Open reference file.

Starting Playback Position

When you use a text sequence file as a reference file, specify the sequence label that plays in the sequence sound.

If you do not specify a sequence label, the very first sequence in the text sequence file plays.

Bank

Specifies the bank to use when playing a sequence sound.

If the bank the sequence sound item references is a single bank, you can open the bank file being referenced on the Bank tab by right-clicking the Sequence Sound List and selecting Open Reference Bank from the shortcut menu.

Multibank Settings

You can specify multiple banks when working with sequence sounds.
Up to a maximum of four banks can be set for a single sequence sound.
Create a multibank by specifying banks for bank 1 through bank 3.

Player

Specifies the player to use when voicing a sequence sound.

Volume

Specifies the volume (in the range from 0 through 255) to use when the Atk library plays the sequence sound.

If sound data volume is set to 127, playback is at the same volume (1.0 times) as the source data.

However, volume can be set in many ways (data or program).
The result after all setting values are applied is clamped between 0.0 times and 2.0 times the source volume.
For more information, see Sound Parameter Control Operations in the Sound System Manual.

Average Loudness Value

To display the average loudness value, you can right-click an item and select Measure average loudness value from the shortcut menu.

Channel Priority

Specifies the channel priority of the sequence sound as a value from 0 through 127.
The higher the value, the higher the priority.

This value is added to the sound priority that is set for each channel in the sequence file that is referenced. The result is the final sound priority for each voice when it sounds.
To read how to set the sound priority for sequence files, see the Sequence Data Manual.

Player Priority

Specifies the player priority of the sequence sound as a value from 0 through 127.
The higher the value, the higher the priority.

Player Priority indicates which sounds to play (or stop) when the player attempts to play more than its sound limit, or more than the maximum number of sounds and tracks as configured in the system settings.
For more information, see the Sound System Manual.

Actor Players

The values 0 through 3 represent the four types of actor players to play.

Because actor players correspond to a real-world sound source or, specifically, the vocal chords of a particular voice, you can limit the number of sounds that can be played at the same time for each actor player.
Just as with a player, if the specified number of sounds is exceeded, the player priority set for each sound is compared, and the sound with the lowest priority is stopped.
The initial value of the number of restricted sounds for each actor player is unlimited for actor player number 0, and 1 for actor player numbers 1 through 3.

Actor players are mainly used in conjunction with 3D sound actors.
As an example, use actor players when you want to suppress the playback of a sound at the actor level in situations where the same sound might be played simultaneously by multiple 3D sound actors.

Single Playback

Specifies the control mode for single playback.

Use Single playback to control playback so only one instance of a particular sound is played at the same time on the same player. You can specify any of the five following modes as the control method to control what happens when a particular sound is playing and the same sound plays once.

  • Do nothing (the default)
    The playing instance of the sound and the new instance of the sound both play.
  • Last note priority
    The playing sound stops and the new instance of the sound plays.
  • Last note within time priority
    The same controls as those from Last note priority are applied if the time that has elapsed since the start of the first sound falls within the period specified in the Single playback time column. Controls are same as when single playback is not used when outside the time.
  • First note priority
    The playing sound continues to play and the new instance of the sound fails to play.
  • First note within time priority
    The same controls as those from First note priority are applied if the time that has elapsed since the start of the first sound falls within the period specified in the Single playback time column. Controls are same as when single playback is not used when outside the time.

Single Playback Time

If either Last note within time priority or First note within time priority has been specified as the control mode, this option is used to specify, in milliseconds, the duration of time to enable control over single playback.

Fix Priority at Release

If this option is selected, the voice priority of the corresponding sound is not reduced even during release.

3D Vol

Sets whether 3D sound volume control is applied.
Clear this check box to disable 3D sound volume control for sounds.

3D Pan

Sets whether 3D sound pan control is applied.
Clear this check box to disable 3D sound pan control for sounds.

3D SPan

Sets whether 3D sound surround pan control is applied.
Clear this check box to disable 3D sound surround pan control for sounds.

3D Pri

Sets whether 3D sound player priority control is applied.
Clear this check box to disable 3D sound player priority control for sounds.

3D Filter

Sets whether the 3D sound Biquad filter control is applied.
Select this item to enable the 3D sound Biquad filter controls for sounds.

3D Attenuation Curve

Selects the volume attenuation system to apply to 3D sounds.

3D Attenuation Ratio

Selects the volume attenuation ratio to apply to 3D sounds.

3D Doppler Factor

Sets the Doppler effect to perform by the 3D sound.
Sets the pitch variation of the Doppler effect as a value in the range from 0 through 255.
The default value is 0. The larger the value, the greater the change in pitch. A value of 0 represents absolutely no change in pitch.

Maximum Number of Voiced Sounds

Displays the maximum number of voiced sounds for a sequence sound.

The display is updated when you measure the maximum number of voiced sounds for a sequence sound.

User Parameters

Configures the 32-bit numerical value that the user is free to set.

To set the composition of the bits, on the Project menu, click Project Settings. In the Project Settings dialog box, click the User Parameter tab.

Measuring the Maximum Number of Voiced Sounds for a Sequence Sound

Use this feature to measure the maximum number of voiced sounds for sequence sounds and list the values in this item of the Sequence Sound List.

Select Preview > Measure Maximum Number of Voiced Sounds for Sequence Sound.
In the dialog box that appears, set a maximum measurement time, and then click Measure.

The measurement is made starting from the Starting Playback Position of each sound.

Up to 96 voiced sounds can be counted.

Create a Text Sequence File

You can create a converted text sequence file (FSEQ file) from the SMF file.
Use these features to create and edit text sequences.

Right-click the Sequence Sound List to open the shortcut menu, and then select Create a Text Sequence File.

When the dialog box to specify the output folder opens, specify the folder and create a text sequence file.

The output filename is the same as the file selected on the list, but with the extension changed to .fseq.

Writing a Template Text Sequence

The Starting Playback Position label does not exist in the text sequence file (FSEQ file), but you can write one to it.

For example, if the Starting Playback Position label "abcd" is added and you run Write Template Text Sequence, the following text sequence is added to the end of the FSEQ file.

Info

abcd:
fin

is added to the end of the FSEQ file.
