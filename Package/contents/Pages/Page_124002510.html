audio Bugs

Exporting sound waveforms does not work in NintendoWare SoundMaker and SoundProjectWaveExporter.

Attempts to export sound waveforms using the NintendoWare tools SoundMaker and SoundProjectWaveExporter always result in the creation of blank waveform files.

This problem will be corrected in a future release.

When you convert a sound project imported using NintendoWare's SoundMaker, an invalid sound archive is exported.

When, in SoundMaker, an NW4C/NW4F sound project (one for which the cache file remains for converting purposes) is imported and (normally) converted, NW4C/NW4F binary source files become mixed in with the NintendoSDK binary sound archive (BFSAR) files. As a result, they cannot be correctly read during the nn::atk and nw::snd runtimes. This is a problem.

As a workaround to this problem, make sure to reconvert the sound project after importing it. This conversion makes the old cache file unusable.

If some of the parameters in Biquad filter are set, the audio may be cut off.

If the following parameters are set in nn::atk::SoundHandle::SetBiquadFilter() or nn::atk::SoundPlayer::SetBiquadFilter(), audio from some sources may be cut off.

  • When the filter in nn::atk::BiquadFilterType_LowPassFilter is selected and the value is set to 1.0
  • When the filter in nn::atk::BiquadFilterType_HighPassFilter is selected and the value is set to 0.1

Sometimes effects are not processed in the order they were added in nn::audio.

The intended specification is for effects to be processed in the order in which they were added to SubMix or FinalMix. However, if you have two or more effects registered to a single SubMix or FinalMix, and you then delete an effect and add another new effect, the effects sometimes run in the wrong order.

You can avoid this problem by deleting all effects from SubMix and FinalMix and then adding the effects.

This bug will be fixed in an upcoming release.

When I use nn::atk, a warning specific to MemoryPool appears.

The following warning messages appear on the console when using nn::atk, because it does not yet support nn::audio::MemoryPoolType.

[nn::audio] A memory pool has not been created. We recommend using a memory pool.
[nn::audio] Please prepare nn::audio::MemoryPoolType. See the nn::audio documents.

The same is true for AtkPlayer and the various nn::atk samples. For more information about memory pools, see the section about memory pools in the audio library manual.

This bug will be fixed in an upcoming release.

I cannot use multiple instances of AudioRenderer at the same time in a Windows environment.

I cannot use multiple instances of AudioRenderer at the same time in a Windows environment.

This bug will be fixed in an upcoming release.

The application can sometimes halt when monitoring waveforms with the NintendoWare AtkPlayer SoundOutput panel.

The application can sometimes stop working when monitoring waveforms with the AtkPlayer SoundOutput panel. The workaround is to stop monitoring waveforms. For information about stopping waveform monitoring, see the AtkPlayer documentation.

This bug will be fixed in an upcoming release.

Waveform channels are not appropriately assigned in the NintendoWare AtkPlayer SoundOutput panel.

Waveforms that conform to the FC, LFC, RL, and RR output signals do not appear because waveform channels are not appropriately assigned in the AtkPlayer SoundOutput panel. Waveforms that conform to the FL and FR output signals do appear, however.

This bug will be fixed in an upcoming release.

NintendoWare AtkPlayer can stop running if you change the DelayTimeMax value.

The application can stop running if you change the DelayTimeMax parameter for the Delay effect in the AtkPlayer Effect panel.

This bug will be fixed in an upcoming release.

Reloading sound archives does not work in the NintendoWare AtkPlayer.

A bug prevents the operation that reloads sound archives in AtkPlayer from running correctly. Restart AtkPlayer when the sound archive has been converted.

This bug will be fixed in an upcoming release.

fs Bugs

There is a bug where sometimes the first entry is not listed when nn::fs::ReadDirectory() is called via nn::fs::MountHost()/MountHostRoot() immediately below a drive when the target is a type of removable media.

Alpha Test does not work in this package.

There is a bug where the display in horizontal window panes and in window panes that use upper left material sharing functionality is incorrect. This bug results in different appearances in LayoutEditor, the viewer, and in the game applications.

3D Bugs

The G3dDemo Edit demo does not display correctly in the Visual Studio 2013 Release/Develop build.

When you run the G3dDemo Edit demo in the Visual Studio 2013 Win32/x64 Release/Develop build, the aspect ratio is not calculated correctly when rendering and the demo display may be corrupted.

Either set the compiler optimization options in the Visual Studio project file properties to disabled (/Od), or build in something other than Visual Studio 2013 Release build.

This bug will be fixed in an upcoming release.

A dialog box requesting that NDI shut down might be displayed when installing the Thumbnail Explorer Extension package.

If a folder selection dialog box is displayed using the Nintendo Dev Interface when installing the Thumbnail Explorer Extension package, a dialog box requesting that the tool be shut down might be displayed. If you shut the tool down, it becomes impossible to continue installation processing correctly. Click Cancel in the dialog box.

You can work around this problem by restarting the Nintendo Dev Interface and installing the package without displaying the folder selection dialog box.

This bug will be fixed in an upcoming release.

If I cancel a shader attachment in 3DEditor, there will be no updates to editing results even if I subsequently attach it.

If I cancel a shader attachment in 3DEditor once, a bug prevents real-time editing results from being updated, even after attaching the same shader.

This bug will be fixed in an upcoming release.

Editing results are not applied to the correct target if you add visibility keys for a material animation to more than one material in 3DEditor.

There is a bug where editing results are not applied to the correct target if you add visibility keys for a material animation to more than one material in 3DEditor.

This bug will be fixed in an upcoming release.

The application will not work correctly if you include a shader parameter animation and a texture pattern animation at the same time to a material animation.

The application can start failing on an assertion in the runtime or stop working correctly when a shader parameter animation and a texture pattern animation are included in the same material animation.

This bug will be fixed in an upcoming release.

socket Bugs

Cannot use nn::socket::RecvFrom() in asynchronous mode.

In a Windows environment, because the MSG_DONTWAIT flag is undefined, nn::socket::RecvFrom() cannot be used in asynchronous mode.

We plan to fix this problem in the next release.

The nn::socket::Select() function does not work correctly.

An error is returned when reaching the specified timeout time when using nn::socket::Select().

vi Bugs

The number of layers that can be created in the development hardware environment differs from that for the Windows environment.

You can create up to eight layers in the Windows environment, compared to just one for development hardware.

We plan to reduce the number for the Windows environment to one in the next release.

Settings Bugs

Exporting settings while using the Settings Manager in a Windows environment will fail when you have never read or written settings before.

You can use the following workarounds. Use the settings API to get settings values and change those values, or temporarily import a blank settings file as in the following sample.

<?xml version="1.0" encoding="utf-8"?>
<dict>
  <key>head</key>
  <dict>
    <key>version</key><string>1.0</string>
  </dict>
  <key>body</key>
  <dict>
  </dict>
</dict>

perf Bugs

The application stops when particular settings are made for LoadMeterInfo and LoadMeterCenterInfo.

Currently, we know that the application halts when setting the number of CpuMeter buffers maintained by CpuMeter and LoadMeterCenter to an odd number.

The application could unintentionally halt as a result of other settings not yet determined.

We plan to fix this problem in the next release.

The alphabet list is not displayed correctly in functions such as API references.

In API references, under Namespace > Namespace Members, the alphabet lists displayed under All, Functions, and Enumerated Values do not display the correct lists when selected.

Sample demo batch processing fails when () is included in the path for the NintendoSDK.

Batch processing fails when building the sample demo that performs batch processing if () is included in the path to the NintendoSDK. You can avoid this problem by not using () in the NintendoSDK path.

Sometimes there is no response when deleting an environment in Nintendo Dev Interface.

When the separately installed Nintendo TargetManager is running, environment deletion processing may stop running.

You can avoid this problem by closing Nintendo TargetManager.

When a Visual Studio project file is opened, you are prompted to upgrade the compiler and libraries.

When a project file not attached to a solution file, such as a Visual Studio project for an SDK library whose source code is publicly available, is opened in Visual Studio, a dialog box might be displayed prompting you to upgrade the compiler and libraries.

If you upgrade them, you will no longer be able to build in earlier versions of Visual Studio. Click Cancel and close the dialog box.
