Counters Tab

The  Counters  tab displays all of the counters that were sampled during a profile. This includes performance counters and user logged counters.

Note

These notes point out differences in the  Functions  tab when the  Thread-Centric Data Views  experimental feature is enabled. See  Experimental Features Tab  for additional information.

Note

When Thread-Centric Data Views are enabled, the Counters tab displays only performance counters, and the Logs tab displays user logged counters.

Counters Tab Toolbar

The following controls are in the Counters tab toolbar.

Deselect All Button

To quickly deselect all selected items, click on the Deselect All button.

Select Similar Toggle Button

When the Select Similar button is enabled, selecting or deselecting an item causes similarly named items to be selected or deselected.

Select Similar allows for selecting similar counters across cores. For example, a single click on a specific counter on a core will select (or deselect) that counter on all profiled cores.

Stats Toggle Button

The Stats toggle button enables and disables viewing the Min, Max, Average, and Median columns on the Counters tab. When the button is enabled, these four columns will be visible and can also be used for sorting. When the button is disabled, the four columns are hidden and the sorting will revert to the visible columns.

Copy Button

The Copy button will copy to the clipboard whatever is currently shown using the current sorting and filtering. You can then paste the content into an email, document, or Excel spreadsheet. Because the copied content uses tab delimiters, it will retain the column organization when pasted into an Excel spreadsheet.

Core Filtering Drop Down and Core Column

Click on the Cores drop-down in the toolbar to select between All Cores and an individual core. Only cores that were profiled will be available in the Cores drop-down. The default selection is All Cores and each new profile taken or loaded will reset the Cores drop-down selection to All Cores.

When a particular core is selected, such as Core 1, the Core column will be hidden since all functions shown belong to the same core. This can be helpful when focusing on a particular core and can help optimize horizontal space in the tab.

Note

When Thread-Centric Data Views are enabled, the Core column is not hidden when a core is selected.

Group Filtering Drop Down and Group Column

The Groups drop-down in the Counters toolbar allows for selection between All Groups and any of the groups that were present in the profile (such as Performance Counters). Only groups that are in the profile will be listed in the Groups drop-down. The default selection is All Groups and each new profile taken or loaded will reset the Groups drop-down selection to All Groups.

When a particular group is selected, for example Performance Counters, the Group column will be hidden since all groups shown are of the same type.

Specifically for the group Performance Counters, the drop-down will contain additional info about which performance counter group was recorded for the profile. For example, if a performance counter group named Example Group was profiled, the Groups drop-down will contain an entry for Performance Counters (Example Group).

Note

When Thread-Centric Data Views are enabled, the the Group drop-down and column are not displayed.

Selected Only Toggle Button

When the Selected Only button is enabled, only selected items will be visible. If there are no counters selected, then the button will be disabled and all counters will be visible.

Filtering Counters

Counters can be filtered using the Filter text box at the bottom of the Counters tab. This behaves similar to the Filter text box on the Functions tab.
