Connecting the SDEV to a PC

This section details the different ways of connecting the SDEV to a PC, presenting the advantages and disadvantages of each method.

  • Connect the SDEV and the PC directly via an Ethernet cable.
  • Connecting the SDEV Using the Network Connection of the PC
  • Determining the IP Address

Review

As mentioned in Preparing the SDEV, there are two ways to connect the SDEV to a PC.

  • Connect the SDEV and the PC directly via an Ethernet cable.
  • Connect the SDEV using the network connection of the PC.

This section reviews both of these procedures.

Connect the SDEV and the PC directly via an Ethernet cable.

You can choose between two options.
The following table shows the advantages and disadvantages of each method.

Connection Method Advantages Disadvantages

1. Direct Connection
  • Isolates the connection from specific LAN policies and traffic.
  • Does not require a reserved DHCP or static IP address from your IT department.
  • Communication bandwidth is maximized.
  • The host PC is isolated from the LAN and the Internet.
  • Other hosts cannot access the SDEV.

2. Direct Connection With a Second NIC
  • Isolates the connection from LAN policies.
    These LAN policies can include disabling DHCP, prohibiting broadcast traffic, and restricting unicast traffic.
  • Isolates the connection from other LAN traffic.
  • Communication bandwidth is maximized.
  • IT policies and procedures may not allow this type of setup.
  • To maximize bandwidth, you must connect the second NIC to a PCI slot or another high-speed I/O slot.
  • The SDEV is not easily accessible from other hosts.
  • You must ensure that the IP address of the second NIC does not conflict with an existing IP address on the LAN.
  • Setup and configuration are more complex.

Connecting the SDEV Using the Network Connection of the PC

This method also has two options.
The following table shows the advantages and disadvantages of each method.

Connection Method Advantages Disadvantages

3. Unmanaged Switch
  • Isolates the host PC and SDEV connection from most LAN policies.
    These policies include disabling DHCP, prohibiting broadcast traffic, and restricting unicast traffic.
  • Isolates the connection from most LAN traffic.
  • Other hosts can access the SDEV.
  • Simplest to configure.
  • Multiple SDEV units can be controlled.
  • Other hosts can access the SDEV.
  • Communication bandwidth may be affected by other network traffic from the host PC (such as HTTP and email).
  • Requires a reserved DHCP or a static IP address from your IT department.

4. Second NIC and Unmanaged Switch
  • Isolates the connection from LAN policies.
    These LAN policies can include disabling DHCP, prohibiting broadcast traffic, and restricting unicast traffic.
  • Isolates the connection from other LAN traffic.
  • Communication bandwidth is maximized.
  • Multiple SDEV units can be controlled.
  • IT policies and procedures may not allow this type of setup.
  • To maximize bandwidth, you must connect the second NIC to a PCI slot or another high-speed I/O slot.
  • The SDEV is not easily accessible from other hosts.
  • You must ensure that the IP address of the second NIC does not conflict with an existing IP address on the LAN.
  • Setup and configuration are more complex.

Determining the IP Address

When the SDEV ships, it is configured to get an IP address automatically, regardless of which connection method is being used.
The Auto IP feature decides the IP address when it is obtained automatically.
The specific procedure for deciding the IP address is as follows.

  1. An address is requested from the DHCP server on the network.
    • If the DHCP server responds, the IP address is set to the one assigned by the DHCP server.
  2. If the DHCP server does not respond, the IP address is set to an address starting with 169.254.

Generally, no issue arises when using the IP address that has been automatically obtained this way. However, there are many advantages to setting a static IP address for the SDEV.
For more information, see Setting a Static IP Address for SDEV.
