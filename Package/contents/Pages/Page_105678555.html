Overview

3DEditorBridge is a command-line tool that connects 3DEditor with external tools.

It performs processing such as getting information for intermediate files being edited in 3DEditor and loading intermediate files edited externally into 3DEditor.

The executable file is %NINTENDO_SDK_ROOT%\Tools\Graphics\3dEditor\3dEditorBridge.exe.

When you run 3DEditorBridge, it closes when the specified processing ends in 3DEditor, except when errors occur, such as when 3DEditor does not start.

If the operation succeeds, it returns 0. In any other case it returns a value other than 0. Messages from 3DEditor, such as error messages, are output normally.

Options

3dEditorBridge.exe [--args-file] [--version] [-h] 
[start-3deditor [--team-settings-file team settings file path or filename]]  |
[close-3deditor] |
[export-info --output ""] |
[export-temporary-files --path "" intermediate file name ... ] |
[reload-temporary-files --path "" intermediate file name ... ] |
[load-files file path or folder path ... ] |
[save-files [--disable-file-info] file path or filename ... ] |
[save-all [--disable-file-info]]  |
[close-all] |
[connect-runtime --target "Device"|"Pc" [--platform platform name]]  |
[disconnect-runtime] |
[clear-history] |
[optimize-shader [--disable]] 
[set-current-frame --frame ""]

--args-file ""

Specifies the file.
You can use the contents in the parameter file to specify arguments.
The parameter file allows you to specify each option over several lines.
Lines starting with a number sign (#) are treated as comments.
Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--version

Displays version information.

-h --help

Displays Help.

start-3deditor

Start 3DEditor.

--team-settings-file

Specifies the team settings file path or filename.

close-3deditor

Closes the 3DEditor.

export-info

Subcommand that exports the files and preview information loaded in 3DEditor.

-o "" --output ""

Specifies the file to output. 　

export-temporary-files

Subcommand that exports intermediate files for models and other objects being edited in 3DEditor to the specified directory.

--path ""

Specifies the folder that includes intermediate files.

reload-temporary-files

Subcommand that reloads intermediate files into 3DEditor from the specified directory.

load-files

Subcommand that loads the specified project file, intermediate files, or files in the specified folder.

save-files

Subcommand that saves the files in the specified directory or saves the files with the specified filenames. Files with no changes will not be saved.

save-all

Subcommand that overwrites and saves all files with unsaved changes.

close-all

Subcommand that closes all files currently open in 3DEditor.

No dialog box is displayed to confirm saving unsaved files.

connect-runtime

Subcommand that connects to the runtime.

--target

Specifies the target to connect to.

--platform

Specifies the platform to connect to.

disconnect-runtime

Subcommand that disconnects the runtime connection.

Also disables the automatic connection feature if it is enabled. Does not do anything if you are already disconnected from the runtime.

clear-history

Subcommand that deletes the edit history.

optimize-shader

Subcommand that enables and disables shader optimization.

query-connection-state

Indicates the runtime's connection state.

set-current-frame

Sets the current 3DEditor frame.

Function Details

Starting 3DEditor

3dEditorBridge.exe start-3deditor [--team-settings-file team settings file path or filename]
  • Start 3DEditor. If 3DEditor has already started, this function does nothing.
  • You can specify the team settings file using --team-settings-file.

Closing 3DEditor

3dEditorBridge.exe close-3deditor
  • Exits 3DEditor. If 3DEditor is already closed, this function does nothing.

Getting Information in 3DEditor

3dEditorBridge.exe export-info --output "output file"
  • Outputs file information loaded in the current file view as an XML file.
  • You can get the following information.
    • Model path
      • Animation set information, and bind animation full path (for each model)
    • Animation full path
    • Texture full path
    • Shader definition full path
    • Scene animation full path
    • Animation current frame position

Getting the Edit State of a 3DEditor Intermediate File as a Temporary Intermediate File

3dEditorBridge.exe export-temporary-files --path "output destination" [filename]...
  • Outputs models edited in 3DEditor as intermediate files.
  • Cannot output files with the same name but different paths.
  • Outputs even when a modal dialog box is displayed in 3DEditor.

Reloading a Temporary Intermediate File to 3DEditor

3dEditorBridge.exe reload-temporary-files --path "folder to be reloaded" [filename]...
  • Reloads a temporary intermediate file that has been output and applies it in 3DEditor.

Loading Files Into 3DEditor

3dEditorBridge.exe load-files [file path or folder path]...
  • Loads a file into 3DEditor.
  • If you specify a folder, intermediate files in that folder will be loaded.

Overwriting and Saving Files Loaded Into 3DEditor

3dEditorBridge.exe save-files [--disable-file-info] [file path pr filename]...
  • Overwrites and saves files loaded into 3DEditor.
  • Files with no changes will not be saved.
  • If multiple files with the same name are not loaded, you can specify the filename.
  • When the --disable-file-info option has been specified, the <file_info> element of the intermediate file is saved without being updated.
Note

The --disable-file-info option is included in NintendoSDK 1.0.0 and later versions.


Saving All Files Loaded Into 3DEditor With Unsaved Changes

3dEditorBridge.exe save-all [--disable-file-info]
  • Saves all files with unsaved changes.
  • When the --disable-file-info option has been specified, the <file_info> element of the intermediate file is saved without being updated.
Note

The --disable-file-info option is included in NintendoSDK 1.0.0 and later versions.

Closing All Files Loaded Into 3DEditor

3dEditorBridge.exe close-all
  • Closes all files loaded into 3DEditor.

Connecting to the Runtime

3dEditorBridge.exe connect-runtime --target "Device"|"Pc" [--platform platform name]
  • Connects to the target specified with the --target option. Specify Device to connect to the development hardware runtime. Specify Pc to connect to the PC runtime.
  • Connects to the platform specified with the --platform option. Connects to the current platform if unspecified.
  • If you are already connected to a different target, it will try connecting to the new target after disconnecting.
  • Does not do anything if you are already connected to the specified runtime.

Disconnecting the Runtime Connection

3dEditorBridge.exe disconnect-runtime
  • Disconnects the runtime connection.
  • Does not do anything if you are already disconnected from the runtime.
  • If automatic connection is enabled in 3DEditor, this option is disabled.

Deleting the Edit History

3dEditorBridge.exe clear-history
  • Deletes the edit history.
  • The memory used to maintain the history is also released.

Enabling and Disabling Shader Optimization

3dEditorBridge.exe optimize-shader [--disable]
  • Enables and disables shader optimization.
  • When the --disable option is not specified, shader optimization is enabled.
  • When the --disable option is specified, shader optimization is disabled.


Displaying the Runtime's Connection State

3dEditorBridge.exe query-connection-state
  • Indicates the runtime's connection state.
  • "Disconnected" is displayed when the runtime is disconnected.
  • "Connected" is displayed when the runtime is connected.
  • "Communicating" is displayed when the runtime is connected and communications are currently ongoing.

Setting the Current 3DEditor Frame

3dEditorBridge.exe set-current-frame --frame "Current frame"
  • Sets the current 3DEditor frame.
