Updating Controller Firmware

  • Overview
  • Necessary Tools
  • Procedure

Overview

This page describes how to update the controller firmware.
For the controller firmware revision history, see Controller Firmware Release Notes.

Note

When using the SDEV/EDEV to update the Joy-Con firmware, connect the Joy-Con to US Writer.
If you leave the Joy-Con connected to the US Writer while updating the firmware, the update may fail.



Note

Starting with NX Add-On 3.1.0, the firmware can be updated using the controller support applet in the system features.

However, the controller support applet cannot be used to update the firmware on prototype devices that are in their factory-default states (such as the black Joy-Con included with the SDEV MP).
If you are using any of these devices, you must first update the firmware using the procedure described in this document.

This restriction does not apply to production versions of the Joy-Con and the Switch Pro Controller.


Necessary Tools

  • Controller Firmware Updater

    • UpdateControllerFirmware.bat, directly under the NintendoSDK folder.

    • A tool to update the NX controller firmware.

Procedure

  1. Start the controller firmware updater and
    Run UpdateControllerFirmware.bat, located directly under the NintendoSDK folder.


    Info
    $ UpdateControllerFirmware.bat --auto

    If you run UpdateControllerFirmware.bat with the --auto option, all controllers connected to the system using the rail or wirelessly after starting the tool are automatically updated.



  2. Connect the controllers to the SDEV/EDEV.
    Press the SYNC Button on each controller and connect the controller via Bluetooth or by connecting to the rail.
    The connected controllers appear in the List of Connected Controllers.

    Note
    To enable firmware updates by connecting the controller to the rail, you must update the Joy-Con firmware to version 3.46.x.x or later.
    If the firmware version is earlier than 3.46.x.x, update the firmware by connecting the controller via Bluetooth.
    If a controller is low on battery power, first recharge the battery by attaching the controller to the rail or another power source like the classic-grip accessory.

  3. Press the L Button and ZL Button or the R Button and the ZR Button on the controller at the same time, and then release them.
    The firmware update will start as soon as you release the buttons.


  4. The firmware update will run.
    While the firmware is updating, do not connect a controller that is connected via Bluetooth to the rail, and do not detach a controller that is connected to the rail, from the rail.

    While the NFC-supported controller is being updated, the controllers will both restart one time.
    The controllers normally reconnect to the system automatically after restarting. However, in rare cases this process fails (such as when the communication environment is poor).
    If the reconnection process fails, press any button on the controllers to manually reconnect the controllers.
    After the controllers are reconnected, the update resumes.


  5. After the firmware update is complete, the message "Firmware update completed" is displayed on the screen.
    Controllers that have completed the update are disconnected from the SDEV/EDEV.

    Note

    If "Verify error occurred" is displayed, the update may have failed due to signal strength or another issue.

    If the update process fails, try to update the firmware again.

  6. You can complete this procedure as many times as is necessary to update the firmware. To update another controller, repeat the procedure starting from step 2.


  7. After the firmware update is complete, restart the SDEV/EDEV.
    (In a future SDK release, it will not be necessary to restart.)
