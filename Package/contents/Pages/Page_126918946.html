Resource

Overview

nn::mii::Resource is the class for managing the graphics resources such as shapes and textures that are needed for rendering a Mii character.

ResourceInfo

Resource settings, including quality and gamma, are defined by nn::mii::ResourceInfo.

The information that must be configured is listed below. For more information, see the API reference.

  • The quality of the texture resource.
  • The quality of the shape resource.
  • The format in which the resources are loaded.
  • The type of gamma.

Loading the Resource Data

Before you initialize nn::mii::Resource, you need to load the resource data into memory.

On the NX development hardware, the resource data is obtained from the system resources.

The Windows version of an application cannot get this data from the system resources, therefore the data must be saved to resource files in the application.

Loading Resource Data on the NX Development Hardware

Use the following procedure to load the Face Library's resource data.

  1. Call the nn::mii::InitializeResourceLoader() function to initialize the Mii resource loader.
  2. Load the texture resource using the nn::mii::LoadResourceTexture() function.
    The buffer to which the resource is being loaded must be a memory space of the size returned by the nn::mii::GetResourceTextureSize() function, and it must be aligned to nn::mii::ResourceMemoryAlignment. Get the size by passing to nn::mii::GetResourceTextureSize() the same nn::mii::ResourceInfo that will be passed to nn::mii::LoadResourceTexture().
  3. Load the shape resource in the same way, using the appropriate functions to calculate the buffer size and to load the resource into the buffer.
  4. When you have finished loading these resources, call the nn::mii::FinalizeResourceLoader() function to finalize the loader.

In the MiiSystemResource sample program, these resources are obtained from the system resources.

Loading Resource Data in the Windows Version

Use the nn::fs library to load the resource files that will be used by the application. For more information about the various resource files, see Mii Resources.

In the MiiSimple sample program, the resources are saved in the application and loaded using the nn::fs library.

Initialization and Finalization

The following cautions are for initialization and finalization. You can also view the API reference for each function.

Initialization

Use the nn::mii::Resource::Initialize() function to initialize the Resource object. Resource must be initialized for other classes like CharModel to use it.

Shape resource data and texture data are necessary for initialization and must be loaded into memory beforehand. This data must be retained until the Resource object is finalized.

Use the functions nn::mii::Resource::CalculateMemorySize() and nn::mii::Resource::CalculateMemoryAlignment() to determine the size and alignment of memory required for initialization. The resource data and ResourceInfo passed to these functions must be the same as those used for initialization.

The memory and resource data passed when Resource was initialized, must be restrained until the object is finalized.

Finalization

To finalize the Resource object, call the nn::mii::Resource::Finalize() function. When finalization has completed, the memory and resources used for initialization can be destroyed.
