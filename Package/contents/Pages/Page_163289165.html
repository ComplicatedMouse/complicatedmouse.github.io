Creating a New Project

  1. In VSI, select File > New > Project, and then select Templates > Visual C++ > General > Makefile Project.
  2. On the Debug Configuration Settings screen, enter the required information.
    • For more information about building executable binaries for NX, see Creating an Independent Build System (NX Development Hardware Environment).
    • In the output field (for debugging), specify the desired path for the final NSP file.
    • The specified value can also be changed after the project is created from NMake in the project properties.
  3. To use different settings for the release build and the debugging build, clear the Same as debug configuration check box.
  4. In Build > Configuration Manager > Active solution platform, select New.
  5. In the Type or select the new platform box, select NX32 or NX64.
  6. In Copy settings from, select x86, select the Create new project platform(s) check box, and then click OK.
    • x64 is not supported, so do not select it.
  7. In Project Properties > VC++ Directory > Library Directories, specify the path to the directory with the nnSdk.nso and nnrtld.nso files that will be linked at build time.
    • You do not need to set this path to run the program, but if it is not set you will not be able to search the debug information and perform debugging.
  8. To run the file specified in NMake > General > Output in debugging mode, change the platform to the one created in steps 4 through 6, and then start debugging as for a normal project.
