Character width of multiple scalable fonts.

Question

What happens to character width when you specify a character that does not exist within glyph data for any multiple scalable font?

Answer

The user can configure a replacement character to render when you specify a character that does not exist within glyph data for any font. That replacement character and its width will be used.
By default, the replacement character is a question mark (?). The first font that has glyph data for a question mark will be used.
