Overview

SystemUpdateSdev is a command-line tool to update the firmware on development SDEV hardware with the same procedures used for the system update functionality.

It offers the following advantage. You can improve the firmware version while maintaining the settings as compared to those from InitializeSdev.

This tool is added to versions 4.3.0 and later of the NX Add-On.

Package

Included in the DevKitVersionUpdater package.

Placement

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/SystemUpdateSdev.exe

There are several dependent files in addition to this EXE file for operations.

Command Format

Option Format

SystemUpdateSdev [target options] [other options]

target options:
    --target <IP address | serial number | SDEV name>  Specifies the IP address and serial number or name of the target for system update.

other options:
    --target-version                                   Specifies the firmware type to use for system updates.
    --display-available-version                        Display all available firmware types.
    --verbose                                          Generate detailed logs.
    --version                                           Display to tool version.

Exit Code

You can use the exit code to check whether the tool succeeded.

Nonzero value: Terminated with error.

How to Use

The following examples demonstrate different ways of using SystemUpdateSdev.

Specifying the Development Hardware IP Address and Updating

SystemUpdateSdev --target 192.168.10.11

Updating by Specifying the Development Hardware Serial Number

SystemUpdateSdev --target XAW03121000000

Specifying the Development Hardware Name and Updating

SystemUpdateSdev --target (target name)

Important Information

  • You cannot downgrade.
  • You can run this tool on any SDEV with a firmware from NX Add-On 4.0.0 or later. Note: It cannot be run on earlier firmware installations.
  • When specifying the development hardware serial number, specification is possible with or without the check digit.
  • NX Add-On 12.2.X and earlier versions of SystemUpdateSdev do not support updating DevMenuApp. Individually rewrite after executing the update tool.
  • In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after update is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.
    • For more information, see Using Target Manager 2.
