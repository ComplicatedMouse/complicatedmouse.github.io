Percussion List

Percussion List

The Bank tab is divided into upper and lower sections. The Instrument List appears in the upper section. Tabs for the Sample Map and Percussion List appear in the lower section.
The Percussion List appears in the lower section when the Percussion List tab is selected.

Role of the Percussion List

The Percussion List is a special editor for editing instruments.
You can edit instruments that are selected on the Instrument List.

The Percussion List is an editor for multi-region instruments, similar to the Sample Map. However, unlike the Sample Map, the Percussion List displays a vertical keyboard.
The Percussion List differs from the Sample Map in that it is used to create instruments that require many small key maps in a very limited range, such as percussion instruments.
Velocity regions cannot be handled in the Percussion List.

Percussion List Operations

This section describes the operations possible with the Percussion List.

Creating Key Regions

To create key regions, right-click the list and select Split Key Region from the shortcut menu.

The Percussion List corresponds to a keyboard, with notes listed in order starting from low notes to high notes.
The created key region extends downward in the list from the row where the division operation was performed, to the higher note where the next key region starts.

Creating Velocity Regions

Velocity regions cannot be created in the Percussion List.

Opening a Folder or Reference File

To open a folder or reference file, right-click the file to open the shortcut menu, and select either Open Folder or Open Referenced File.

Editing the Percussion List

This section describes the meaning of each of the elements that defines a Percussion List.

Compression Format

Specifies the compression format to use when you convert reference waveform files.

  • ADPCM
    Converts the waveform file to the proprietary ADPCM format.
  • PCM16
    Converts the file to 16-bit PCM data.

File path

Specifies the waveform file that the region references.

The characters that can be used in a filename or path name are restricted. The following characters can be used.

  • Single-byte alphabetic characters and the underscore [a-z, A-Z, _] for the first character of the filename.
  • Single-byte alphanumeric characters and the underscore [a-z, A-Z, 0-9, _] for subsequent characters in the filename.

Playback time

Displays the playback time for sounds (the length of the waveform data expressed as time).
The units are displayed in mm:ss. ms and the value in milliseconds are rounded to the second digit.

For looped waveform data, this shows the time to the end of the loop, not the length of the waveform itself.
In addition, an L is displayed at the front of the cell.

Ticks

Displays the length of the required note commands.
Ticks are units for specifying the length of notes and rests within text sequences.

The number of ticks per minute (tpm) and ticks per second (tps) depend on both the tempo and the quarter note resolution. Assuming that the default tempo value of 120 quarter notes per minute (qpm) is set, and that the quarter note resolution of the text sequence is set to the default value of 48 (which differs from the value of the SMF Quarter Note Resolution setting under Project Settings), the tpm and tps rates can be calculated as follows.

  • 1 minute = 120 * 48 = 5,760 ticks
  • 1 second = 120 * 48 / 60 = 96 ticks

The tempo and quarter note resolution values correspond to these rates.

Knowing the number of ticks is useful when writing text sequences.

Note that changing the pitch of an instrument causes the actual playback duration to increase or decrease, and the number of actual ticks taken for playback to change accordingly.

Sample Rate

Displays the sample rate of the waveform file.

Bit Rate

Displays the bit rate of the waveform file.

Number of Quantization Bits

Displays the quantifying bit number of the waveform file.

Number of Channels

If there are one or two waveform channels, either Mono or Stereo appears. If there are three or more channels, just the number of channels appears.

Data Size

Shows the size of the waveform data after compression and conversion.
If the instrument references multiple waveform files (for example, a multi-region instrument), the total size of the reference files is shown.
The content on display is updated when files are converted.

Original Key

Specifies the original key for the region.

Env A

Envelopes can be set either per instrument or per region.
For a description of envelopes, see Instrument Parameters.

For multi-region instruments, specify the envelope's attack time in the range of 0 to 127 for the regions.
The same is true for each phase of the following envelope specifications.

Env H

Specifies the envelope hold time as a value in the range from 0 through 127.

Env D

Specifies the envelope decay time as a value in the range from 0 through 127.

Env S

Specifies the envelope sustain level as a value in the range from 0 through 127.

Env R

Specifies the envelope release time as a value in the range from 0 through 127.

Volume

Specifies the region volume as a value in the range from 0 through 255.

If sound data volume is set to 127, playback is at the same volume (1.0 times) as the source data.

However, volume can be set in many ways (data or program).
The result after all setting values are applied is clamped between 0.0 times and 2.0 times the source volume.
For more information, see Sound Parameter Control Operations in the Sound System Manual.

Pitch (Semitones)

Specifies the pitch of the region, in semitones.

Pitch (Cents)

Specifies the pitch of the region, in cents (1 cent = 1/100th of a semitone).

Pan

Specifies the pan for the region.

Percussion Mode

Specifies the percussion mode for the region.
When selected, it indicates that percussion mode is enabled.

In regions where percussion mode has been specified, the note-off command is ignored, and the wave file is played to the end regardless of the note length.
The pitch calculation that corresponds to the note height is performed in the usual manner.

Be careful when applying percussion mode to regions for which a looping wave file has been set.

If percussion mode has been enabled for a loop wave, the following playback operations result.

  • Notes continue playing because note off is not used.
  • Release status is not entered because note off is not used.
  • For this reason, the fix priority at release OFF setting is not actually accessed.

This invites the following results.

  • Because a note only stops when it loses priority, if the note is played at a high priority level, the number of notes that can be used decreases.

Key Group

Specifies the key group for a region as a value in the range from 0 to 15.

Operations are not performed for the key group when 0 is specified. Zero is set as the key group default.

In regions where the same nonzero key group has been set, each time a new note plays, playback of whatever is currently being played for the same key group stops.
This feature enables emulation of such effects as closed high hat and open high hat.

Key group operations are only enabled within the same track of a sequence.
If the same instrument is used from multiple sequences, or if the same instrument is used concurrently from multiple tracks within a sequence, the key group feature cannot be used.

Interpolation Type

Specifies the interpolation type at the instrument level.

Configure the voice interpolation method. Select 4-point interpolation or Linear interpolation. The default interpolation value is 4.

Currently, even if this interpolation type setting is changed, the change is not applied to the interpolation method.
