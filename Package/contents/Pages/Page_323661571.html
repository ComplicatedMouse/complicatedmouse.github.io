Features for Retail Interactive Demos

Overview

Retail interactive demo applications run on retail interactive displays. Retail interactive displays run in a special mode called retail interactive demo (RID) mode, which has its own unique features.
For more information about running a devkit in RID mode, see Setting Up a Testing Environment for Retail Interactive Demos.

Info

RID mode features are available in NX Add-On 5.3.0 and later.

How to Use

Checking for Retail Interactive Demo Mode

You can call the nn::oe::IsInStoreModeEnabled function to determine whether RID mode is enabled.
You can even use this function on systems that are not running in RID mode (for example, for conditional processing based on whether RID mode is enabled).

  1. Include the oe library’s header file.

    #include <nn/oe.h>

  2. Call the oe library function for determining whether RID mode is enabled.

    nn::oe::IsInStoreDemoModeEnabled();

Exiting Retail Interactive Demo Applications

Call the nn::oe::ExitApplicationAndGoBackToInStoreDemoMenu function to exit the demo application and return to the RID menu.
Note that this function aborts if the system is not running in RID mode.

  1. Include the oe library’s header file.

    #include <nn/oe.h>

  2. Call the oe library function for exiting the retail interactive demo application.

    nn::oe::ExitApplicationAndGoBackToInStoreDemoMenu();
