Overview

AdpcmEncoder is a PC tool to convert PCM data to a proprietary Nintendo ADPCM format. It is located in Tools/Audio/AdpcmTools/.

This tool is dependent on the following DLLs. These are contained in NintendoSDK.

  • Tools/Audio/AdpcmTools/AdpcmCore.dll
  • Tools/Audio/AdpcmTools/AudioFile.dll

While the ADPCM format is equivalent to the format of past Nintendo platforms, it is not identical. AdpcmEncoder must be used to create data that may be read using the nn::audio library.

How to Use

$ AdpcmEncoder.exe [-o|--output <output-filepath>] [--loop-begin <sample-index>] [--loop-end <sample-index>] [-h|--help] [-v|--verbose] <input-filepath>

Argument List

input-filepath

Specifies the input filename. The following file formats can be specified as input.

  • 16-bit WAV
  • 16-bit AIFF

Sound sources with more than two channels can also be converted. The output file will create a file of one channel for each input channel.

Options

-o, --output Specifies the output filename. The filename extension must be .adpcm.
--loop-begin Specifies the loop start position. This position is represented as a sample position with a lead value of 0.
This option must be specified at the same time as --loop-end.
--loop-end

Specifies the loop end position. This position is represented as a sample position with a lead value of 0.

This option must be specified at the same time as --loop-begin.
-h, --help Displays Help.
-v, --verbose Exports a detailed log.

Example

Encode input.wav into output.adpcm

$ AdpcmEncoder.exe -o output.adpcm input.wav

Specify a loop position and encode

$ AdpcmEncoder.exe --output output.adpcm --loop-begin 1000 --loop-end 2000
