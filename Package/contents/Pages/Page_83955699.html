Text Intermediate File Converter

  • Overview

Overview

Converts binary intermediate files (with .f??b extensions) into text intermediate files (with .f??a extensions).

This converter is used when converting to text intermediate files to check data strings in binary intermediate files using a text editor.

When you drag a binary intermediate file or its folder to the executable file (3dIntermediateFileBinaryToAscii.exe) or the executable's shortcut, a text intermediate file gets saved in the same folder as that binary intermediate file.
Files other than binary intermediate files are ignored.

If you want to output text intermediate files to another location, either copy the binary intermediate files first and then convert or move the text intermediate files after conversion.

Command-Line Options

3dIntermediateFileBinaryToAscii.exe [--args-file ""] [--version] [-h] [-s] [-j ""] [--disable-xsd-check]
[--enable-extension ""] [--disable-extension ""] [--error-detail-output ""] [intermediate_file|folder]...

Converts binary intermediate files (with .f??b extensions) into text intermediate files (with .f??a extensions).

--args-file ""

Specifies the file.
You can use the contents in the parameter file to specify arguments.
The parameter file allows you to specify each option over several lines.
Lines starting with a number sign (#) are treated as comments.
Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--version

Displays version information.

-h --help

Displays Help.

-s --silent

Suppresses the display of messages during processing.

-j "" --jobs ""

Specifies the number of jobs. When unspecified, the processor count is assigned.

--disable-xsd-check

Disables format checking of intermediate files using schemas (XSD files).
Only the latest version intermediate files can be processed when this option is enabled.

--enable-extension ""

Specifies the extensions to be processed among text intermediate files. 　
Use a comma (,) as a delimiter when listing more than one filename extension. 　
Example: --enable-extension "fmda,ftxa"

--disable-extension ""

Specifies the extensions to be processed among text intermediate files. 　
Use a comma (,) as a delimiter when listing more than one filename extension. 　
Example: --disable-extension "fmda,ftxa"

--error-detail-output

Specifies a folder for saving detailed information when an error occurs. If not specified, this information is written to the default folder.

Intermediate File | Folder

Specifies the file or folder to be processed.
Files other than binary intermediate files are ignored.
Files located under the specified folder are recursively processed when a folder is specified.
Files other than intermediate files are ignored.

Supports wildcards (?: Replaces one character; *: Replaces one or more characters).
Example: "a?b*"
Converts files with a first letter "a" and a third letter "b”.

Usage Examples

This section describes how to use the binary intermediate file converter.

Converting the Binary Intermediate File

Converts binary.fmdb to binary.fmda.

3dIntermediateFileBinaryToAscii.exe binary.fmdb

Converting Folders

You can convert all of the binary intermediate files in the binary folder and its subfolders to text intermediate files.

3dIntermediateFileBinaryToAscii.exe binary\
