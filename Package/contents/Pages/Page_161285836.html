pcm Library

Introduction

Purpose of This Documentation

This documentation describes the pcm library included in NintendoSDK.

Intended Audience

This documentation is intended for people developing applications using NintendoSDK.

Related Documentation

For more information, see nn::pcm Library API Reference.

Features Provided by the pcm Library

The pcm library provides the following features.

  • Measuring power consumption on the board.

For more information about each feature, see the manuals.

Simple Replacement Feature

For basic purposes such as wanting to know the power consumption when running an application and the estimated battery life, there is a simpler method even if the application does not directly use this library.

By using PowerConsumptionMeter in DevMenuOverlay, you can display the GPU usage time, power consumption, (expected) battery life in handheld mode, and memory bandwidth consumption of the production device running the application without making changes in the application. Consider using this feature.

pcm Library Restrictions

These features can be used only in the development environment. They cannot be used in the production environment.

Care must be taken because control will not return to the caller if the library's functions are called in the production environment (PROD MODE).

To ensure that the API is not called from the application, we recommend that you do not link nnPcm.nso in your Release build.

For more information about PROD MODE debugging, see Testing Applications.

If you attempt to submit a master ROM that includes calls to the pcm library, it will be detected as an error in the check for prohibited APIs.
