Data Type Compatibility

This table shows the compatibility of the type specified in the Play Report library and the type specified in the analysis tool.


Type Play Report Library Type Types in Analysis Tool Notes
Types in BigQuery Types in Redshift
Signed integer int64_t
  • Integer
  • SMALLINT
  • INTEGER
  • BIGINT
 Cannot handle unsigned 64-bit integers.
Single-precision floating-point number float
  • Float
  • REAL
  • DECIMAL


Double-precision floating-point number double
  • Float
  • DOUBLE PRECISION
  • REAL
  • DECIMAL


Boolean value bool
  • Boolean
  • BOOLEAN
String const char*
  • String
  • CHAR
  • VARCHAR
Date const char*
  • Date
  • DATE

Specifies a string that represents the date. Time zones are parsed as UTC.

Example: '2018-01-01'
Time const char*
  • Timestamp
  • TIMESTAMP

Specifies a string that represents the time. Time zones are parsed as UTC.

Example: '2018-01-01 00:00:00'
64-bit length ID nn::prepo::Any64BitId
  • Integer
  • BIGINT
 Forcibly converted to a signed 64-bit integer and recorded.
ID match/mismatch comparisons are possible when performing analysis, but size comparisons are not.
Binary const void*
  • String
  • VARCHAR
 Converted to a string encoded in Base64 [RFC 4648, url-unsafe, has padding (=)] and recorded.
Structure nn::prepo::Struct
  • Record
Cannot be used with Redshift.
