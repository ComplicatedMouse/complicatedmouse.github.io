Descriptor Pool

Handling the Descriptor Pool in the vfx Runtime

This section describes how to handle the descriptor pool in the vfx runtime.

Texture objects and sampler objects can be used after registering them to the descriptor pool and obtaining a descriptor slot. For objects created in the vfx runtime, you must inform the application about the created object using a callback function, register that object to the descriptor pool in the application, and get a descriptor slot.

Info

Descriptor slots can be managed using the gfx_DescriptorPoolAllocator and gfx_TransientDescriptorAllocator classes in the gfxutil module.


Textures

In the vfx runtime, the following textures must be registered to the descriptor pool.

  • Textures generated in the nn::vfx::System class
  • Textures Contained in the Effect Binary

Textures Generated in the nn::vfx::System Class

When you create nn::vfx::System, you must call nn::vfx::System::RegisterTextureViewToDescriptorPool to register the texture generated in nn::vfx::System to the descriptor pool. Currently, one curl noise texture (nn::vfx::SystemParameters_StaticTextureDescriptorSlotCount) is generated when nn::vfx::System is generated. This texture is used for particle behavior using curl noise.

// Create the effect system.
m_pVfxSys = new nn::vfx::System( config );
 
// Register the texture generated by the effect system to the descriptor pool.
m_pVfxSys->RegisterTextureViewToDescriptorPool( pRegisterTextureSlotCallback, NULL );

// Delete the texture generated by the effect system from the descriptor pool.
m_pVfxSys->UnregisterTextureViewFromDescriptorPool( pUnregisterTextureSlotCallback, NULL );

Textures Contained in the Effect Binary

The texture included in the effect binary must also be registered to the descriptor pool. This registers the texture to the descriptor pool through the nn::vfx::Resource class after registering to the binary.

// Adds the effect binary to the effect system.
int resourceId = 0;
m_pVfxSys->EntryResource( &vfxHeap, binary, resourceId );
 
// Register the texture included in the resource to the descriptor pool.
m_pVfxSys->GetResource( newResId )->RegisterTextureViewToDescriptorPool( pRegisterTextureSlotCallback, NULL );

// Delete the texture included in the resource from the descriptor pool.
m_pVfxSys->GetResource( newResId )->UnregisterTextureViewFromDescriptorPool( pUnregisterTextureSlotCallback, NULL );

Callback Functions

Registering textures to the descriptor pool and deleting textures from descriptor pool are handled using user-defined callback functions.

// Callback function that registers the texture to the descriptor pool.
bool AllocateSlotForTexture( nn::gfx::DescriptorSlot* dstSlot, const nn::gfx::TextureView& textureView, void* pUserData )
{
    // pUserData takes objects to be registered to the texture descriptor pool.
    TextrueDescriptorManager* manager = reinterpret_cast< TextrueDescriptorManager* >( pUserData );

    // The textureView must be registered to the descriptor pool, and it must then return the dstSlot from the registration destination.
    int slot = manager->GetEmptyTextureDescriptorSlot();
    nn::gfx::DescriptorPool::BeginUpdate();
    nn::gfx::DescriptorPool::SetTextureView( slot, &textureView );
    nn::gfx::DescriptorPool::EndUpdate();
    nn::gfx::DescriptorPool::GetDescriptorSlot( dstSlot, slot );
    return true;
}

// Register the texture generated internally by the system immediately after the effects system generation to the descriptor pool.
m_pVfxSys->RegisterTextureViewToDescriptorPool( AllocateSlotForTexture );


Sampler

Currently, the specification of the sampler used in vfx runtime is that it generates available sampler patterns when nn::vfx::System is generated, and reuses those patterns. A total of 32 patterns (nn::vfx::SystemParameters_StaticSamplerDescriptorSlotCount) of texture filter, texture wrap U, and texture wrap V combinations are generated.

// Callback function that registers the texture to the descriptor pool.
bool AllocateSlotForSampler( nn::gfx::DescriptorSlot* dstSlot, const nn::gfx::Sampler& sampler, void* pUserData )
{
    SamplerDescriptorManager* manager = reinterpret_cast< SamplerDescriptorManager* >( pUserData );
    
    // The sampler must be registered to the descriptor pool, and it must then return the dstSlot from the registration destination.
    int slot = manager->GetEmptyTextureDescriptorSlot();
    nn::gfx::DescriptorPool::BeginUpdate();
    nn::gfx::DescriptorPool::SetSampler( slot, &sampler);
    nn::gfx::DescriptorPool::EndUpdate();
    nn::gfx::DescriptorPool::GetDescriptorSlot( dstSlot, slot );
    return true;
}
 
// Register the sampler generated by the system to the descriptor pool.
m_pVfxSys->RegisterSamplerToDescriptorPool( pRegisterSamplerSlotCallback );

// Delete the sampler generated by the system from the descriptor pool.
m_pVfxSys->UnregisterSamplerFromDescriptorPool( pUnregisterSamplerSlotCallback );
