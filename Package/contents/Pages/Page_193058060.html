Guide for Migrating From NW4C Effect to NintendoSDK

This page describes the transition from NintendoWare for CTR, or NW4C, effects to NintendoSDK NintendoWare effects.

Compatibility

You cannot migrate the old NW4C effects module, gfx.

Files can be migrated for NW4C Effect, eft,
NW4C effects and NintendoSDK effects, however, are not completely compatible.

Handling Settings for Which No Compatibility Exists

The following table describes incompatible settings between the NW4C effects and the NintendoSDK effects, and how to handle them.

Description Resolution
Color for consolidated stripes not updated. Configure color using Global multiplication color alpha.
Related Link: For more information about Global multiplication color alpham see here.
Color is animated.

Change the color, from among either color 0 or color 1, to a fixed behavior type in NW4C effects, for whichever color does not have an animation.
The specifications for NW4C effects indicated that only either color 0 or color 1 will have an updated emitter animation.
Those animations will be updated with the NintendoSDK effects, however, when settings remain within the data.
Particle types are reversed, vertically, for directional polygons. Rotate either the x- or z-axis by 180 degrees, using emitter transformation, because particle directionality differs.

Migrating Effect-Related Files

This section describes how to migrate effect files and files that use effect data.

Migrating ESET Files

This section describes how to migrate effect files.
Use EffectMaker to convert ESET files to files for NintendoSDK.
Instances of xxx.tga in the ESET files will be replaced by xxx.ftxb, and instances of yyy.cmdl in the ESET files will be replaced by yyy.fmdb.

EffectMaker.exe  --update-nw4c-to-nx ConvertList.xml

Related Links: For the ConvertList.xml format, see Binary Converter.

Migrating CMDL Files

This section describes how to migrate model files that use things like primitive particles.
Use the CMDL converter to convert CMDL files to intermediate FMDB files for the NintendoSDK.

Related Links: For more information about the converted CMDL, see Guide to Migrating from NW4C Graphics to the NintendoSDK .

Migrating TGA Files

This section describes how to migrate texture files.
Use the 3D texture converter to convert TGA files to NintendoSDK texture intermediate files (FTXB and FTXA files).

3dTextureConverter.exe image.tga -o image.ftxb

Related Link For more information about the 3D texture converter, see 3D Texture Converter.
