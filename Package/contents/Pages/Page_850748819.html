Remote Input Troubleshooting

Causes and Solutions

  1. Remote input is recognized as a debug controller, and the game is not implemented for operating with a debug controller.
    1. Proceed to Check Remote Input Settings.
  2. Remote input is recognized as a debug controller, and the game is implemented for operating with a debug controller, but the controller support screen or Home Menu is displayed.
    1. The controller support screen and Home Menu cannot be operated with a debug controller.
    2. Proceed to Check Remote Input Settings.
  3. The development hardware is in TV mode, and remote input is set to Joy-Con dual-controller grip.
    1. When the device is in TV mode, set remote input to Pro Controller.
      1. Proceed to Check Remote Input Settings.
    2. If set to Pro Controller, you may encounter the 1P or 2P issue in issue 4.
      1. Proceed to Unlink a Controller That Is Already Connected.
  4. The development hardware recognizes the Joy-Con controllers or Pro Controller as 1P, and remote input as 2P.
    1. From the controller support screen, reconnect the controller for remote input as 1P.
      1. Proceed to Unlink a Controller That Is Already Connected.
  5. The remote input setting is not a style accepted by the application.
    1. Proceed to Check Remote Input Settings.
  6. The remote input window was just opened. (It takes a while to be able to use this feature.)
    1. Proceed to Check InputDirector Connection.
  7. The startup order of Target Manager and InputDirector startup is reversed.
    1. Proceed to Check InputDirector Connection.
  8. InputDirector and development hardware firmware versions are incompatible.
    1. Proceed to Start the Appropriate InputDirector.

Check Remote Input Settings

Right-clicking in the remote input window displays a menu where you can set the controller that the remote input is recognized as. Confirm whether this is the intended controller.

Note: Joy-Con dual-controller grip refers to the handheld state with the Joy-Con controllers attached to the system, not the dual state where the controllers are held while not attached.

Check InputDirector Connection

Check whether InputDirector can communicate with the development hardware correctly.
Right-click the InputDirector icon in the task tray and see whether the intended development hardware is selected in the Connect menu.

Option Is Not Selected

Select this check box.

Connect Is Disabled

The Connect menu is disabled immediately after starting InputDirector. Try after starting InputDirector and then waiting for a while.

Connect Is Still Disabled Even After Waiting

If InputDirector was started before Target Manager, connecting may not work well.
In that case, close and then restart InputDirector.

Also, the connection does not work immediately after InputDirector is started, so wait a while after starting it.

Unlink a Controller That Is Already Connected

If remote input is 2P, you must disconnect the controller recognized by the development hardware.

Start the Controller Support Applet

By using the controller support applet, you may be able to unlink the already connected 1P and set remote input as 1P.

Remove Controller Pairing From DevMenu

You can delete pairings by clicking Delete All Controller Pairing Settings on the DevMenu Controller page.

However, the remote input controller also disappears, so also switch the controller type for remote input. After switching this type, the controller will be recognized again.

Start the Appropriate InputDirector

When the remote input window or remote video window is opened, Target Manager automatically starts $NINTENDO_SDK_ROOT\Tools\Hid\InputDirector\InputDirector.exe.
When this happens, if the SDK version indicated by the NINTENDO_SDK_ROOT environment variable differs from the firmware version of the connection target, remote input may not work. Switch NINTENDO_SDK_ROOT to the appropriate SDK beforehand.
