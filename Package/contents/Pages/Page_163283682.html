2. Previewing the LOD Model in 3DEditor and Looking for Problems

Checking the LOD Models by Previewing in 3DEditor

Use 3DEditor and 3dSampleViewer to preview the reduced LOD model and look for problems after the default parameters have been applied.

The process to preview the LOD model is as follows.

Start 3DEditor and 3dSampleViewer

Start 3DEditor (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dEditor\3dEditor.exe) and 3dSampleViewer (%NINTENDO_SDK_ROOT%\Samples\BuiltApplications\3dSampleViewer\3dSampleViewer.exe).

For more information, see the Starting 3DEditor and Starting 3dSampleViewer sections in the Preparation page of the 3DEditor Tutorial.

Connect 3DEditor and 3dSampleViewer

Start 3DEditor and 3dSampleViewer.

For more information about connecting to these tools, see the Linking With Viewer section in the Linking With Viewer page of the 3DEditor Tutorial.

Load the Reduced Model Intermediate File Into 3DEditor

Load the model intermediate file, bg_WhiteTown.fmdb, that was reduced in step 1 in 3DEditor.

You can also load the file by dragging it to the 3DEditor GUI.

After the file is loaded, you can verify that the loaded model is displayed in 3dSampleViewer.

Opening the Preview Tab in the Model Property Window

In File View, double-click bg_WhiteTown.fmdb to open the Property window for the file.

The Preview tab will be opened by default when you open the properties window.

Specifying the LOD Level to Preview

Change the LOD preview option in the Preview tab to 1 or 2 to preview the specified level of the LOD model in 3dSampleViewer.

Black box: Appears darker than expected (UV coordinate values distorted, or a problem with the normals). Also, the shapes of the objects on the pillar in the lower blue box appear to have changed significantly. Next, we will resolve this issue by adding parameters.

This model includes multiple UV atlases (known as UV shells in Maya), so UV coordinates are considered even when polygon reduction is executed using the default settings.
