Switching the mode changes the result played back in the viewer.

When you switch from merged mode () to separated mode (), the result played back in the viewer is different.

To understand this situation, consider the case where segments have been configured in the following way in merged mode () in the Edit Animation Segment Tags window.
This table represents the presence or absence of set keys for each segment using the symbols ○ and ×.

Name Start End Scale Color Transparency
In 0 30 × Yes ×
Wait 40 60 ○ No ×
Out 70 90 × × ○

When segments are switched and played in the game application, the values taken by segments without any keys (segments with an × in the table), differ depending on the mode.
Because of this difference, the result played back in the viewer looks different when the mode is switched.

You can also check the behavior of the game application in LayoutViewer.
 The remainder of this section describes the differences by comparing the behavior in merged mode () and separated mode () as seen in LayoutViewer.

Merged Mode

For segments without any keys (the × segments), the values of keys before and after that segment in the timeline are applied, as shown it the following table.

Name Start End Scale Color Transparency
In 0 30 The value of the first key in the Wait segment: 1.0 Animate red → yellow. The value of the first key in the Out segment: 1.0
Wait 40 60

Animate 1.0 → 5.0.

 The value of the last key in the In segment: yellow The value of the first key in the Out segment: 1.0
Out 70 90 The value of the last key in the Wait segment: 5.0 The value of the last key in the In segment: yellow Animate 1.0 → 0.0.

The following animation depicts the behavior in LayoutViewer when the mode is merged mode () and the segment switches to In, at frame 10, during playback of Wait.
The value of the first key in Wait (1.0), is applied as the scale value for In.

Info

The behavior of the animation in merged mode () can be changed from the behavior shown above by using an option in the Tool Environment Settings.
The check box for the Do not export animation if there are no keys in the frame segment of the segment tag option (--omit-nokey-all) is selected by default ().
When this option is selected, animations for segments without keys are not exported.
If you clear this check box (), keys will be generated automatically at the boundaries between segments in the curve, and the animations will be exported.

Related Link For more information about the Do not export animation if there are no keys in the frame segment of the segment tag option, see here.


Separated Mode

In separated mode (), all segments start at frame 0.
Because there is no contextual relationship between segments in the timeline, the segments without keys (the × segments) are unmodified, unlike in merge mode ().
The values applied to the × segments (without keys) are the values during playback, as shown in the following table.

Name Start End Scale Color Transparency
In 0 30 Value during playback. Animate red → yellow. Value during playback.
Wait 0 20 Animate 1.0 → 5.0. Value during playback. Value during playback.
Out 0 20 Value during playback. Value during playback. Animate 1.0 → 0.0.

The following animation depicts the behavior in LayoutViewer when the mode is separated mode (), and the segment switches to In, at frame 10, during playback of Wait.
The value of the key at frame 10 (2.5), which is midway through the 20 frames of the Wait segment, is applied as the scale value for In.
