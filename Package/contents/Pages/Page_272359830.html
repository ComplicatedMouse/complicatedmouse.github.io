Low-Level Graphics Debugger

Overview

As its name implies, the Low-Level Graphics Debugger is a dedicated low-level graphics debugger for NVN. Like the NVN Graphics Debugger, the Low-Level Graphics Debugger allows you to examine resources and the state of the graphics pipeline—at a lower level—in order to debug your application.

Low-Level Graphics Debugger is intended to be used for debugging applications. ROMs that use this library cannot be uploaded to the Nintendo Developer Portal (NDP) as master ROMs.

Packages

The Low-Level Graphics Debugger is provided as a separate package from the NintendoSDK NX Add-On. It may also be released later than other NintendoSDK packages.

The following table lists the packages.

Package Name Description
Low-Level Graphics Debugger A dedicated low-level graphics debugger for NVN.

Location

The Low-Level Graphics Debugger can be found at the following path directly under the NintendoSDK installation directory.

NintendoSDK/
LowLevelGraphicsDebugger/

In addition, a library file named llgd-target.nso, for the Low-Level Graphics Debugger, is located in the Libraries directory of the NintendoSDK. To use the Low-Level Graphics Debugger, you need to link llgd-target.nso as an additional NSO file to the target application.
A header file named llgd.h for the Low-Level Graphics Debugger is located in the Include directory of the NintendoSDK. Include this llgd.h header file to use the API of the Low-Level Graphics Debugger.

Manual

For more information about the package, see the documentation in the package.

Differences From the NVN Graphics Debugger

The Low-Level Graphics Debugger provides many of the same features as the NVN Graphics Debugger; the two programs are also intended to be used in similar ways to achieve similar goals. This section provides a brief overview of the differences between the two tools to help you decide which is best suited to your needs. For more information about the different feature sets, see their respective documentation.

The NVN Graphics Debugger captures NVN API calls, but the Low-Level Graphics Debugger captures commands sent to the GPU. Consequently, this tool captures the actual state of the hardware and tends to make it impossible to perfectly reproduce the arguments that your application passes to the NVN API. It also tends to result in fewer available features than the NVN Graphics Debugger.

The Low-Level Graphics Debugger can only be used with applications running on NX development hardware. Unlike the NVN Graphics Debugger, it cannot capture data from applications running on a PC.

The Low-Level Graphics Debugger is designed to be lightweight. It uses less than 10 MB of memory while it is capturing data, and it has practically no CPU overhead while it is idle. On the other hand, it cannot analyze data in real time like the NVN Graphics Debugger. A data capture can also be triggered using the LLGD API from the target application instead of via the tool’s UI.
