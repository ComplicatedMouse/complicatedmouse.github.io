Introduction

This documentation describes the Target Manager 2 included in NintendoSDK.
Target Manager 2 is a GUI application for managing development hardware connected to the PC.
Target Manager 2 has been created to replace Target Manager. In the future, only Target Manager 2 will be usable.

To use Target Manager 2, configuration is required on both the host PC and development machine.
For more information, see Using Target Manager 2.


Package

The NintendoSDK Target Manager 2 and NintendoSDK Target Manager 2 NX Add-On packages are downloaded and installed automatically when NintendoSDK 10.4 or later is installed from NDI.
Installing these packages installs Target Manager 2 and the Nintendo SDK Daemon.
To install these manually after downloading the packages, run the following in the SDK Installers/HtcTools directory.

  • Install.bat
  • InstallHtcToolsPluginNx.bat

It does not matter which is done first, but all must be installed.

Nintendo SDK Daemon is the daemon program needed when Target Manager 2 communicates with the target.

For more information about how the configuration is different from Target Manager, see Using Target Manager 2.


Location

Target Manager 2 and the Nintendo SDK Daemon are installed at the following paths.

C:\Program Files\Nintendo\NintendoTargetManager2\NintendoTargetManager2.exe
C:\Program Files\Nintendo\NintendoSdkDaemon\NintendoSdkDaemon.exe

After it is installed, Target Manager 2 can be started from the Start menu.



Supported Platforms

Target Manager 2 supports Windows and NX.

Functionality

Target List

Displays information for each added target.
You can take a variety of actions using the buttons on the toolbar or using the shortcut menu that appears when you right-click a target.
Right-click the header to open a shortcut menu for changing the items shown in the target list.

Adding Targets

Displays a list of targets available through network search, and targets that are connected to the PC via USB.
Select the target to add, and then double-click or click the Add target button to add it to the target list.


Connecting to and Disconnecting From Targets

This operation is for connecting to or disconnecting from the selected target.
If the connection or disconnection succeeds, the status changes to Connected or Disconnected, respectively.


Working With Executable Files

Select the executable file to load, specify any necessary arguments or target environment variable files (TDF files), and click the Launch button to run the file.
In addition to loading the executable file, you can also perform the following operations for the executable file.

  • Terminate the running program
  • Install and manage the application
  • Create a dump from a running application

You do not need any tools other than Target Manager 2 to create a dump file, but to analyze the dump file you need VSI and the tm library. For more information, see VSI > Using the Dump Feature.

Screenshots

While the target is selected, select Take a Screenshot or Take an application screenshot to take a screenshot of the screen of the currently selected target

Selecting Take a Screenshot of the Application Screen takes a screenshot of the application screen, regardless of whether it is running in the background.
The following table shows the behavior when taking a screenshot while the application is in the foreground or background.


 Foreground Background
Take a screenshot Application screen Screen displayed on target
Take an application screenshot Application screen Application screen

You can change the naming convention, output folder, and format of screenshot files from Tools > EDEV Settings > Screenshots.

Displaying Logs

Displays a window that shows only logs for the selected target.
For more information about using the displayed logs, see Target Log.

Remote Video and Operations

For more information, see Remote Video and Remote Input.

Controller Status

Info

Available in Nintendo Target Manager 2 version 13.0.0.0 and later.

Displays a window that shows the status of the controllers for the selected target.

Power Operations

This setting is for controlling power behavior for the target.
The available operations depend on the target. For more information, see Functions That Operate Differently Depending on the Development Hardware.

Specifying the Working Directory

By specifying the working directory for each target, you can use a relative path in the root directory to specify by MountHost.

  1. Select the target, and then select Set working directory.
  2. In Working Directory, specify the directory.
  3. You can reference the specified directory using "." from the MountHost function.
    • For example, for the structure C:/Directory/SubDirectory, by specifying the working directory as C:/Directory, you can specify MountHost("host", "./SubDirectory").

Target Configuration

Various settings can be configured for the target.

The items that can be configured differ depending on the target. For more information, see Functions That Operate Differently Depending on the Development Hardware.


Setting a Default Target

The selected target is set as the default target.
If a default target is set, the target used is the one specified when RunOnTarget and ControlTarget are executed without the -t or --target option.
This is also the default target that is used by VSI.

Connect/Disconnect Battery

You can set the remaining battery level.
The remaining battery level is set to 49% when connected, and 1% when it is disconnected.

This feature is only available for the SDEV.

LED Flashing

This option makes the LED lights on the back of the SDEV unit blink.
It can be used to identify the hardware that is registered in Target Manager 2.

This feature is only available for the SDEV.


Target Log

Displays the logs generated by applications, targets, and Target Manager 2.
Logs are generated for all targets connected to Target Manager 2.

You can search the logs displayed in the target log by entering the text to search for into Search for String.
You can also filter the logs by specifying filter conditions. For more information about the grammar for filter conditions, click the question mark (?) icon next to the text box or see Log Filtering.

HTCS Entry

Displays a list of HTCS ports currently opened by Target Manager 2.

Attributes

Details
Peer Name Supports nn::htcs::HtcsPeerName and nn::tm::HtcsPeerName.
Port Name Supports nn::htcs::HtcsPortName and nn::tm::HtcsPortName.
IP Address The IP address used for TCP/IP.
Port Number The port number used for TCP/IP.
Target Name The name set on the corresponding equipment. If no name is set, the serial number is displayed.

Displaying Settings Manager

You can use the functionality for configuring the system settings, such as changing or resetting the system settings of the target, that are held by the settings manager.
In addition, you can search for the setting by name.

The following system settings operations are available.

System Settings Operation Details
Load

Loads the system settings of the selected target to the screen.

For more information about the primary settings that are loaded, see Settings Manager > Settings Manager > Settings.
Write

Writes the settings that were changed on the screen to the target.

You can select individual items on the screen to write to the target.
Reset Resets all of the system settings of the selected target to the default values.
Open File

Loads a saved system settings file.

For more information about file formats, see Settings Manager> Settings Manager > Configuration File.
Save File

Saves the loaded system settings as a file.
You can select the settings to include in the file individually from the screen.

For more information about file formats, see Settings Manager> Settings Manager > Configuration File.


Tools

You can configure various settings from the Tools menu.

HTCS Port Mapping

Open the settings file to specify the HTCS port map in Editor.
For the settings file format, see Fixed Port Mapping in the HTCS Manual.

Settings

Target Log

  • Maximum Number of Log Lines
    • The log output will be maintained until it reaches this number of rows.
    • If the number of rows in the log exceeds this value, the excess number of rows are deleted in order from the oldest.
    • Limits the amount of memory used by Target Manager 2.
    • You can specify a value in the range from 10000 to 1000000.
  • Display Log Metadata With Mouse Pointer
    • Specify whether to show the log metadata pop-up window.
  • Select Font
    • Specify the log font.
Trigger Action
Info

This feature is available in version 14.0.0.0 and later.

The feature executes any program when logs that satisfy specified criteria are output by the target.

When using this feature, add new items from the Add button.
You can disable items by clearing the check boxes for enabling each item.

For more information about the settings, see Log Trigger Action, or see the Help in the dialog box displayed when adding or editing.

Screenshots

  • Naming Conventions
    • Specify the naming conventions when saving. For the legend, see the bottom of the same screen.
  • Output Folder
    • Specify the save destination for screenshots.
  • Format
    • For the output format, specify BMP, JPG, PNG, or TIF.

Remote Video

Recording Settings
  • Naming Conventions
    • Specify the naming conventions when saving. For the legend, see the bottom of the same screen.
  • Output Folder
    • Specify the save location for screenshots.
  • Action When Saving
    • Allows you to choose to do nothing, open the destination folder, or open the saved video.
Display Settings
Info

This feature is available in version 14.0.0.0 and later.

  • Displays a notification when remote video has stopped.
    • It displays a notification in the remote video window when remote video communication from the target stops.
    • Notifications are not displayed when running in SDEV alternate mode.
Decoding Settings
Info

This feature is available in version 13.2.0.0 and later.

  • Use hardware acceleration if available
    • Use hardware acceleration for decoding remote videos. Not used if not supported.
      If the remote video is not displayed correctly, disable the setting.

HTCS

  • Current Control Port
    • Displays the port number that is currently used as the htcs Library control port.
  • Specify Control Port
    • Specifies the port number that is currently used as the htcs Library control port.
    • You must restart Nintendo SDK Daemon to enable the settings.

The nn::tm::GetHtcsControlPort() function in tm Library can be used to get the port that is currently open as the control port.

Hardware Management

Info

This feature is available in version 14.1.0.0 and later.

  • Enable Hardware Logging
    • If this check box is selected, the initial and final connection date and time and serial number of the target connected to will be saved to the log.
  • Log File Path
    • Specifies a path other than the default path to save hardware management logs to.

Changing Layout

The following windows can be merged with or split from Target Manager 2 windows freely.

  • Target List
  • HTCS Entry
  • Target Log
  • Remote Video
  • Remote Input
  • Capturing Screenshots (version 12.1.0.0 and later)

When merging windows, drag the title bar of the window and overlap it with the window you want to merge with.
When splitting windows, drag the title bar of the merged window outside the window.

Top Most Settings

Info

This feature is available in version 13.0.0.0 and later.

You can use either of the following methods to display windows that can be merged or split on top.

  • Click the Top most button on the window title bar.
  • Right-click the title bar or tab of the window, and select Top most.

Determining whether a window is on top most

If a window is split by itself, its top most display settings are reflected as is.

When multiple windows are merged, if even one of the visible windows is set to display on top, the whole window will be displayed on top.

Visible windows indicate windows that satisfy the following conditions.

  • The window is shown
  • Auto-Hide is disabled
  • If tabbed, the tap that is selected (displayed)
Info

In other words, windows in the following states will not be displayed on top.

  • The window is closed
  • Auto-Hide is enabled
  • The window is tabbed and hidden

Reporting Bugs

If you encounter a bug while using Target Manager 2, submit a bug report and include the dump files and logs.

In version 14.1.0.0 or later, you can output a compressed file from the Help > Bug Report menu in Target Manager 2 that contains the dump file and log from the date the issue occurred. Be sure to include this.
If the issue is that Target Manager 2 itself does not start, the same compressed file can be generated using Start Menu > Target Manager 2 Bug Report.

For earlier versions, please include the following files and logs generated under the %TEMP%\Nintendo directory.

  • TargetManager2\TargetManager2-(date and time).dmp
  • NintendoSdkDaemon\NintendoSdkDaemon-(date and time).dmp
  • NintendoSdkDaemon\NintendoSdkDaemon-(date and time).log
