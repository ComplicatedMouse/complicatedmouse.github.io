Target Manager

Overview

Two types of data are collected in Target Manager: data on usage statistics, and crash reports.

In Target Manager, you can set whether either of these types are sent in Tools > Options > Data Collection.

Use the following commands to change this setting with CLI.

NintendoTargetManager.exe -nolaunch -set ReportAnalytics=yes
NintendoTargetManager.exe -nolaunch -set ReportAnalytics=no
NintendoTargetManager.exe -nolaunch -set ReportAnalytics=ask
NintendoTargetManager.exe -nolaunch -set ReportCrashes=yes
NintendoTargetManager.exe -nolaunch -set ReportCrashes=no
NintendoTargetManager.exe -nolaunch -set ReportCrashes=ask

Usage Statistical Data

Usage data is collected for Target Manager and Visual Studio. The collected Information is used to improve the SDK.

Processing to collect this data occurs in the background and does not affect performance. Personal information and source code are also not subject to being collected. All application information is anonymous.

For more information about the specific information in the collected data, see Target Manager > Tools > Options > Data Collection > Usage Statistical Data (Sample).

Incident Reporting

Crash reports are collected when Target Manager crashes.

These reports used to analyze and fix the corresponding Target Manager issue, and are not used for any other purpose.
