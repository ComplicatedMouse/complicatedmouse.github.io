Overview

InitializeEdev is a command-line tool for initializing the firmware of the EDEV development hardware.

Packages

InitializeEdev is included in the SystemUpdater for NX package.

Location

$NINTENDO_SDK_ROOT/Tools/CommandLineTools/InitializeEdev.exe

There are several dependent files in addition to this EXE file for operations.

Command Definitions

Option Format

InitializeEdev [target options] [other options]

target options:
    - target <serial number> Specifies the serial number of the target to be initialized.

other options:
    --target-version                                   Specify the type of firmware to use for initialization.
    --display-available-version                        Output all available firmware types.
    --verbose                                          Output detailed logs.
    --version                                           Display to tool version.

When specifying the serial number, initialization works with or without the check digit.

Exit Code

You can use the exit code to check whether the tool succeeded.

Nonzero value: Terminated with error.

How to Use

The following example shows how to use InitializeEdev.

Initializing by Specifying an EDEV Serial Number

InitializeEdev --target XAW03121000000


Examples of other options are omitted because they are not currently used.

Descriptions of Other Features

Initialization With Firmware That Approximates the Production Hardware (PROD MODE)

You can also initialize the development hardware using PROD MODE, which is firmware that approximates the production hardware.

InitializeEdev.exe --target XAL0310000XXXX --target-version "0.X.Y-Z + Prod"

As shown in the preceding example, to initialize the SDEV using firmware approximating the production hardware, add "+ Prod" after the --target-version option.

In addition:

InitializeEdev.exe --target XAL0310000XXXX --target-version "0.X.Y-Z + ProdWithLog"

To initialize the SDEV using firmware approximating the production hardware with functionality for exporting logs from a microSD memory card, add "+ ProdWithLog" after the --target-version option.

Note

The feature for initializing with firmware that approximates the production hardware is only available with the EDEV MP. Note that an error results if it is executed on other EDEV systems.

Important Information

You can use this tool to initialize the firmware on the following models of development hardware.

EDEV MP ー

ADEV MP NX Add-On 13.2.0 (FW 13.0.0-3.0)

Downgrading the Version and Initializing

If you have already initialized EDEV using the current version of InitializeEdev, you cannot downgrade and initialize EDEV using InitializeEdev (and InitializeEdevWin) prior to NX Add-On 0.10.

Target Communication Protocols

In NintendoSDK 10.4.0 and later, the target communication protocol (htc-gen1/htc-gen2) after initialization is automatically switched according to the htc-gen1/htc-gen2 setting on the host PC.
