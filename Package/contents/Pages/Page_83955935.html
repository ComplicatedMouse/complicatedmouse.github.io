Overview

  • This section lists the file formats used in the NintendoWare audio development environment.
    • There are two main types of sound files: those used by SoundMaker, and those used by the Atk library.
    • Temporary sound files are also created during conversion. These files ultimately end up in the binary sound archive (BCFAR) file, but in most cases you do not need to work with them directly.

File Formats Used in SoundMaker

fspj

SoundMaker project files (SoundMaker Project) .

The FSPJ file is an XML file that contains information about all sound set files (FSST files) in the project, in addition to any information entered in the Project > Project Settings of SoundMaker.

This file is created when you start a new project in SoundMaker.
You can specify the directory in which to create the file at that time.

You can open this FSPJ file by starting SoundMaker and selecting File > Open Project, or by dragging it to the Project Tree.

fsst

SoundMaker sound set files (SoundSet).

This file is an XML file containing information about the sounds, wave and sequence sound sets, sound banks, waveform archives, groups, and sound player settings in a sound set.

A file is created each time you create a new sound set in the SoundMaker project tree.
You can specify the directory in which to create the file at that time.

You can see what a sound set contains by looking at the node for that sound set under its project node in the SoundMaker Project Tree.

fbnk

SoundMaker bank files (Bank).

This file is an XML file with information about the instruments in that sound bank.

A file is created each time you create a new sound bank in the SoundMaker bank list.
You can specify the directory in which to create the file at that time.

You can see what a sound bank contains by looking at the bank list that is displayed when you select the node for that bank under its sound set node in the SoundMaker Project Tree, or by looking at the Bank tab that is displayed when you select any of the child nodes of the bank.

fseq

SoundMaker text sequence files (Sequence).

FSEQ files contain text sequences in a proprietary NintendoWare format.

You can create these text sequence files using any text editor.
You can also create text sequence files by converting the SMF files in the SoundMaker sequence sound list.
You can specify the directory in which to create the file at that time.

The character codes that can be used in text sequence files are ANSI and Shift_JIS.

For more information about text sequence files, see Text Sequences in the Sequence Data Manual.

finl

SoundMaker instrument list files (Inst List).

This file defines the list of IDs for the instrument labels in a sound bank.

The file is created in the same directory as the corresponding bank file (FBNK file) when that bank file is converted. An instrument list file is created for each sound bank file.

You can use the instrument labels stored in the FINL file by including it at the beginning of a text sequence (FSEQ) file.
For an example, see Sequence Data Manual - Preprocessor Directives - Using #include.

Temporary Intermediate Files Created During Conversion

These files are ultimately stored in the final sound archive (BFSAR) file.
In most cases, you never need to deal with these files directly.

bfwav

NintendoWare audio binary waveform files.

These files contain data from waveform files (such as WAV or AIFF) that have been converted to binary.

They are created when you convert a project (FSPJ) file using SoundMaker or SoundProjectConverter.
Normally, they are created in the cache folder at the same level in the directory as the base project file.

bfbnk

NintendoWare audio binary bank files.

These files are simply sound bank (FBNK) files that have been converted to binary.

They are created when you convert a project (FSPJ) file using SoundMaker or SoundProjectConverter.
Normally, they are exported into the cache folder at the same level in the directory as the base project file. One BFBNK file is created for each sound bank in a sound set (FSST) file.

bfseq

NintendoWare audio binary sequence files.

These files are simply sequence (FSEQ) files that have been converted to binary.

They are created when you convert a project (FSPJ) file using SoundMaker or SoundProjectConverter.
Normally, they are exported into the cache folder at the same level in the directory as the base project file. One BFSEQ file is created for each sequence (FSEQ or MID) file.

(If you are using SMF-type sequence sound files, they are first converted to text sequence (FSEQ) files and then into binary sequence files. The intermediate files are exported to cache/name_of_text_sequence_file.fseq. )

bfwsd

NintendoWare audio binary wave sound files.

These files are binary files with information about the wave sounds in the sound set (FSST) files specified by the project (FSPJ) file.

They are created when you convert a project file using SoundMaker or SoundProjectConverter.
Normally, they are exported into the cache folder at the same level in the directory as the base project file. One binary sound file is created for each wave sound set in each sound set file.

bfwar

NintendoWare audio binary waveform archive files.

These files are binary files with information about the waveform archives in the sound set (FSST) files specified by the project (FSPJ) file.

They are created when you convert a project file using SoundMaker or SoundProjectConverter.
Normally, they are exported into the cache folder at the same level in the directory as the base project file. One binary waveform archive file is created for each waveform archive in each sound set file.

If you specified Automatic (shared/individual) in the waveform archive settings for bank/wave sound sets, one binary waveform archive file is created for each bank/wave sound set.

If, however, you specified Automatic (shared) and those bank/wave sound sets belong to a group, only one binary waveform archive file might be created for each group.
For more information about the relationship between groups and waveform archive settings for bank/wave sound sets, see Groups.

Files Used in the Atk Library

fsid

Sound ID files used by the Atk library.

These files are C/C++ files that contain the names of all items in a sound archive.

They are created when you convert a project (FSPJ) file using SoundMaker or SoundProjectConverter.
Normally, they are exported to the output folder at the same level in the directory as the base project file. One sound ID file is created for each project file.

By including the file in your application, you can specify the IDs defined in the file to load sound data and play sounds.

(By using label string data,
you can use the Atk library without including sound ID files. For more information, see Programming Guide.)

bfsar

Binary sound archive file used by the Atk library.

This file is a binary file with information about the various items in the sound set (FSST) files specified by the project (FSPJ) file.

They are created when you convert a project file using SoundMaker or SoundProjectConverter.
Normally, it is exported to the output folder at the same level in the directory as the base project file. One binary sound archive file is created for each project file.

You can get information about the sounds, banks and other items in the sound sets by using the appropriate SoundArchive subclass.
You can load the sound data from sound archives using the SoundDataManager class.

bfstm

Binary stream file used by the Atk library.

These files are binary files with information about the stream sounds in the sound set (FSST) files specified by the project (FSPJ) file.

They are created when you convert a project file using SoundMaker or SoundProjectConverter.
Normally, they are exported into the stream folder under the output folder at the same level in the directory as the base project file. One binary stream file is created for each stream sound registered in each sound set file.

The sound data and parameters in these files are used when playing stream sounds with the sound library.

bfgrp

Binary group file used by the Atk library.

These files are binary files with information about user-managed groups in the sound set (FSST) files specified by the project (FSPJ) file.

They are created when you convert a project file using SoundMaker or SoundProjectConverter.
Normally, they are exported into the userManagementFiles folder under the output folder at the same level in the directory as the base project file. One binary group file is created for each user-managed group registered in each sound set file.

Loading the files in your application enables you to access all of the sounds in those groups.
