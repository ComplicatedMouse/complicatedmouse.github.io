Control Definition Files

Control Definition Files

Introduction

Control definition files are files that contain data for allowing control settings defined uniquely by the user to be performed in layout editors. This document explains how to create control definition files.

Control Definition File Format

Control definition files are described as XML. The schema can be found in LayoutEditor/XMLSchemata/ver_1_5/flct.xsd.

Placement of Control Definition Files

Control definition files can be loaded if they are located in a subfolder below parts root folders.
All files are loaded and added, even if there is more than one. Use the .flct filename extension.

Generally, controls defined using the control definition files and default controls are both enabled.
However, in some situations it is preferable to use only the controls defined with applications and not the default controls. In such cases, place a control definition file named NW_DefaultControl.flct in the parts root folder.
If this file is present, LayoutEditor disables the default controls.

Description of Control Definition Files

Example Notation

A sample notation of a control definition file can be found in $NINTENDO_SDK_ROOT/SampleData/Layout/sample_NW_DefaultControl.flct. The contents are as follows.

<?xml version="1.0" encoding="utf-8"?>
<nw4f_layout xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:xsd="http://www.w3.org/2001/XMLSchema"
    version="1.5.0" xmlns="http://www.nintendo.co.jp/NW4F/LayoutEditor">
<head>
    <create user="" host="" date="2009-09-10T18:38:14.84375+09:00" source="" />
    <title />
    <comment />
    <generator name="NW4C LayoutEditor Core" version="2007_07_00_000" />
</head>
<body>
    <lct>
        <controlForm name="TestButton" uiName="Test Button">
            <description>A button for custom button testing. </description>
            <parameterAnimation name="On" isRequired="true">
                <description>[Required] Animation for switching the state from Off to On.</description>
            </parameterAnimation>
            <parameterAnimation name="Off" isRequired="true">
                <description>[Required] Animation for switching the state from On to Off.</description>
            </parameterAnimation>
            <parameterPane name="Hit" isRequired="true">
                <description>[Required] A pane that indicates the vicinity of a button.</description>
            </parameterPane>
            <parameterPane name="Cursor" isRequired="false">
                <description>[Optional] A pane for the selection cursor.</description>
            </parameterPane>
            <constraintAnimationEqual>
                <animation name="On" position="End" />
                <animation name="Off" position="End" />
            </constraintAnimationEqual>
        </controlForm>
    </lct>
</body>
</nw4f_layout>

Description of Components

nw4f_layout/body/lct/controlForm

This component indicates the control definition. Lining up these elements allows multiple controls to be defined.

This element has two attributes: name and uiName.
name is the name used when a program handles this control. Only half-width alphanumeric characters and hyphens can be used. You can get this string by using the GetControlName method of ControlSrc. This attribute is required.
uiName is the name that appears in the user interface of LayoutEditor. It can be a name that is written in the native language of the designers for easy understanding. This attribute is not required. If left out, name is used instead.

Deriving From Another Control

You can use baseControlForm to define a control derived from another control. You can also define the hideBaseControlForm attribute to hid the derivation source. Use the following sample code.


<controlForm name="NormalButton2" uiName="inherited control example" hideBaseControlForm="false">
        <description> Example of inherited control. To hide the inheritance source, add the hideBaseControlForm attribute. You can overwrite or append the attribute. Multiple step inheritance and multiple inheritance are not available. </description>
        <baseControlForm>NormalButton</baseControlForm>
        <parameterPane name="Hit" isRequired="true">
            <description> [Required] Overwritten at the inheritance destination. </description>
        </parameterPane>
        <parameterPane name="Hit2" isRequired="false">
            <description> [optional] Appended the attribute at the inheritance destination. Appended to the end of the attribute list at the inheritance source. </description>
        </parameterPane>
        <parameterAnimation name="OnOff" isRequired="true">
            <description> [Required] Overwritten at the inheritance destination. </description>
        </parameterAnimation>
        <parameterAnimation name="OnOff2" isRequired="false">
            <description> [Required] Appended at the inheritance destination. </description>
        </parameterAnimation>
</controlForm>
<controlForm name="ButtonBaseA" uiName="Hidden attribute test (derivation source)">
        <description> This is a derivation source control. </description>
</controlForm>
<controlForm name="ButtonBaseA2" uiName="Hidden attribute test (derivation destination)" hideBaseControlForm="true">
        <baseControlForm>ButtonBaseA</baseControlForm>
        <description> Example of hide attribute (hideBaseControlForm). Because the control specified by the derivation source (ButtonBaseA) is removed at the final step of initialization processing, it is not displayed in the LayoutEditor options. </description>
</controlForm>


nw4f_layout/body/lct/controlForm/description

This component describes the control.
This description is not used in runtime programs. It appears on the part setting screen of LayoutEditor.

nw4f_layout/body/lct/controlForm/parameterAnimation

This component describes control function animations. Lining up these elements allows for multiple specifications of function animations. You can get the animation tag name specified in LayoutEditor by using the GetFunctionalAnimName method of ControlSrc.
The order these elements are arranged in becomes the index specified in GetFunctionalAnimName.

This element has two attributes: name and isRequired.
name is the name that appears in LayoutEditor. It is not used in runtime programs. (It is for specifying in the index.) This attribute is required.
isRequired indicates whether it is necessary to always set this function pane. If this attribute is true, a warning message appears in the Parts Settings window if it is not set. If it is omitted, it is assumed to have been set to false (not required).

nw4f_layout/body/lct/controlForm/parameterPane

This component describes the control's function pane. Lining up these elements allows for multiple specifications of function panes. You can get the name of the pane specified in LayoutEditor by using the GetFunctionalPaneName method of ControlSrc.
Note that the order of these elements becomes the index specified in GetFunctionalPaneName.

This element has two attributes: name and isRequired.
name is the name that appears in LayoutEditor. It is not used in runtime programs. (It is for specifying in the index.) This attribute is required.
isRequired indicates whether this function animation must always be set. If this attribute is true, a warning message appears in the Parts Settings window if it is not set. If it is omitted, it is assumed to have been set to false (not required).

nw4f_layout/body/lct/controlForm/ConstraintAnimationEqual

This component represents a constraint on functional animations. It defines the constraint that all listed animation child elements have the same animation effect.
Set the attributes of the animation elements as follows. Set the name attribute to the name of the functional animation, and set the position attribute to the start and end frames (Start and End).
Although the system currently ignores constraints, support for this feature is planned. For example, LayoutEditor will display a warning when a constraint is violated.
