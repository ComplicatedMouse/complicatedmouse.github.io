WGL Initialization

Overview

WGL initialization is a feature to batch load WGL extension commands.

WGL is a Windows platform function, and WGL initialization is only available in a Windows version.

WGL initialization only loads WGL extension features. Window creation and OpenGL context creation for performing WGL initialization are not handled. You must perform these operations separately as necessary.

When WGL initialization is performed, each extension command is loaded depending on the current context. Even if a feature could not be loaded, it is not handled as a failure and the remaining features continue to be loaded.

When WGL initialization is successful, you can call the WGL commands that could be loaded, determine whether a specific WGL command could be loaded, and determine whether a specific extension is supported.

If you want to re-initialize with a different context, re-initialize the WGL after setting the other context as the current context.

How to Use

WGL initialization is performed by nngllInitializeWgl.

Before calling this function, you must configure the GL context to be used in loading.

After loading is successful, call WGL functions in the C interface format to use the WGL commands.

Usage Examples

For example, you can use this feature as follows.

// The GL context is configured.
nngllResult result = nngllInitializeWgl(); // Loads the WGL functions.
NN_ASSERT_EQUAL( result, nngllResult_Succeeded ); // Checks whether loading was successful.
wglSetSwapIntervalEXT( -1 ); // Loading was successful, so WGL functions can be called.

Including Header Files

nn/gll.h contains the types, enumeration, and functions needed to use the WGL. When initializing the WGL using the gll library, instead of wglext.h, include nn/gll.h in the locations where the WGL is used. Including nn/gll.h when wglext.h is included causes a compilation error.
