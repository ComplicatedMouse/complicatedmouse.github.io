Animation Segment Tag Animation Conflicts and Workarounds

Animation Segment Tag Animation Conflicts

Automatic Creation of Segment Tag Key Frames

LayoutConverter automatically creates key frames when you use animation segment tags.

If a particular segment has no key frames but the previous segment does, and as a result its animation curve is distorted, key frames are inserted automatically in that segment to reproduce the curve.

Conflicts by Automatically Created Animations

When these automatically created animations (animations that the designer did not intend) are played, the state of the pane can be changed accidentally. When animation playback is then switched among a number of animation segment tags, the result may not be as intended.

The playback of multiple animations can create conflict issues.

Countermeasures

Use the LayoutConverter Option --omit-nokey-all

If you specify the --omit-nokey-all option when you use LayoutConverter, it suppresses the automatic generation of key frames. (Reference: Layout Binary Converter)

When this option is specified, the only segment tags that get animated are those that the designer explicitly set key frames for in LayoutEditor, making the behavior easier to control.

Although you could also use the options --omit-samekey and --omit-samekey-all to control the automatic generation of key frames, the --omit-nokey-all option works best and we recommend using it.

Use Animation Segment Tag Separated Mode

By using the separated mode for animation segment tags, you can create animations that are not affected by the previous and next segments.

Animations (FLAN files) created in separated mode can be handled the same way as when the --omit-nokey-all option has been specified for the animations.

For more information, see Animation Segment Tags.

Notes

Change in the Output Result

Regardless of whether you use the --omit-nokey-all option or separated mode, the playback result will change because key frames will no longer be generated automatically. Be careful when switching practices during the course of animation production.

As a countermeasure, explicitly insert key frames in segments where no key frames exist.

Specifying a Target Group For Use

To narrow down the panes being animated, work on the assumption that a target group for the animation segment tags has been specified.
By default, you cannot omit the target group and so this issue is not a concern. However, pay attention to this issue if you are migrating old data and so have specified --allow-nogroup-animtag (to allow the omission of a target group) and --cvtr-all-anim-content (to output all animation ) in LayoutConverter.
