Icons in Application Images

Icons Required to Create an Application Image

The following two types of icons are required before you can create an application image.

  • Large icons for use on smart devices, such as the Nintendo Switch Parental Controls smart device application
  • Small icons for the NX system

Of the two, only the large icons must be input to AuthoringTool.

The small icons are automatically generated and incorporated in the application image by AuthoringTool.
Only if there is an issue with the auto-generated results can you use the procedure described later to manually enter the small icons.

Specifications for Icons Entered in the AuthoringTool

AuthoringTool can only handle icons that conform to the following specifications.

Type Format Image Size The size of the file.
Large icons 24-bit BMP 1024 × 1024 -
Small icons (normally not specified) Baseline JPEG 256 × 256 100 KB or Less

For more information about icon formats and content, see the Application Icon Creation Manual.

Specifying Icons

Specify an icon by defining the Icon element, inside the Application element, in the NMETA file.

For more information about the overall structure of the NMETA file, see here.


Element Content Described Notes
Language

Specify one of the following language names.

  • AmericanEnglish
  • BritishEnglish
  • Japanese
  • French
  • German
  • LatinAmericanSpanish
  • Spanish
  • Italian
  • Dutch
  • CanadianFrench
  • Portuguese
  • Russian
 Required.
IconPath

Specifies the path to a large icon.

If you specify a relative path, it is treated as the relative path from the NMETA file.

 Required.
NxIconPath

Specifies the path to a small icon.

If you specify a relative path, it is treated as the relative path from the NMETA file.

In principle, do not specify it.


Only use this specification if there is a problem with the quality of the automatically generated icon.


The following example shows how to code this specification.


<Application>
  <Icon>
    <Language>Japanese</Language>
    <IconPath>Icon_Japanese.bmp</IconPath>
  </Icon>
  <Icon>
    <Language>AmericanEnglish</Language>
    <IconPath>Icon_AmericanEnglish.bmp</IconPath>
    <NxIconPath>Icon_AmericanEnglish_small.jpg</NxIconPath>
  </Icon>
</Application>


Large Icons

The large icon specified by <IconPath> is converted into a JPEG file with the following specifications, and stored in the application image.

  • The Exif Maker tag is set to Nintendo co., ltd and the Software tag is set to Nintendo AuthoringTool.
  • The JPEG quality is 98.
  • Baseline JPEG format.


Automatically Generated Small Icons

The small icon is generated automatically from the large icon and stored in the application image.

The automatically generated icon has the following properties.

  • The small icon is created from the large icon image using the Graphics class in the .NET Framework (with the interpolation mode set to HighQualityBicubic).
  • The Exif Maker tag is set to Nintendo co., ltd and the Software tag is set to Nintendo AuthoringTool.
  • The JPEG quality is set as high as is possible while keeping the file size less than 100 KB.
  • Baseline JPEG format.


Manually Entering a Small Icon

The small icon specified by <NxIconPath> is checked to make sure that the data conforms to the specifications. After which, the Exif for the JPEG file is edited as follows, and the icon is stored in the application image.

  • The thumbnail image is deleted.
  • The Maker tag is set to Nintendo co., ltd.
