Introduction

This section provides information about how to use network features with the SDEV and EDEV.

SDEV/EDEV Network Settings

The SDEV/EDEV can be connected to a network via a wireless or wired connection. You can use the Settings Manager or the system settings in the system application to configure the access point settings to connect to.

If you use the Settings Manager, a convenient option is to use the ImportNetworkSettings-spec.*.bat file included in the nifm library sample (NifmNetworkConnectionSimple). For more information, see nifm Library.

Note that using a wired connection requires the Switch dock and a USB Ethernet adapter. The Ethernet port on the top of the SDEV is exclusively for connecting to a PC. This port cannot be used to connect directly to a network. For more information about configuring settings, see Notes for Development Hardware.


Using Network-Service Accounts

To use network features including an independent server and NEX, a network service account must be registered to the SDEV/EDEV. Network service accounts are registered in the Accounts menu of DevMenu or in the system settings in the system application.

For the registration procedure using the Accounts menu of DevMenu, see Using Network Service Accounts. As described in the reference material, note that in the development environment, the value for Require Nintendo Account for use of Network Service must be changed to Disabled in the Accounts menu of DevMenu.

When configuring settings in the system application, you must link either a Nintendo Account or account for China to create a network service account. For more information about linking each of these accounts, see the following documentation.

  • Creating and Managing Nintendo Accounts in the Development Environment
  • Creating an Account For China in the Development Environment
Info

For more information about network service accounts, Nintendo Accounts, and accounts for China, see the Nintendo Switch Account Guide.


Preparing Network Feature Development

The following preparations are required if the application uses network features such as an independent server and NEX.

  1. Issue a game code for the target product in Nintendo Developer Portal > My Products.
  2. Make a request in OMAS to use network features for the product.
  3. Configure the application ID for the project.

Requesting Use in OMAS

Online title MAnagement System (OMAS) is a website for applying for using network services when developing an application that supports network services. OMAS can be accessed from the Nintendo Developer Portal link.

The Nintendo server settings required to use network features are configured based on the OMAS application.

Applications may use network features from the time the content in the application to OMAS is approved by Nintendo and the application information is registered in the Nintendo server.

Application ID Settings

When an application uses network features, the application ID set in the Nintendo server, and the application ID set in the application metadata, must be the same. The application ID is written in the NMETA file and set in the application. For more information, see Application Development.

Network Feature Development

When developing network features, see the relevant manuals on the Nintendo Developer Portal, including the Independent Server Configuration Manual, the E-Commerce Guide, and the NEX Guide.
