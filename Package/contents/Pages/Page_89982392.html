Shader Converter (EXE)

Overview

Objective

GFX ShaderConverter (EXE) is a tool that runs GFX ShaderConverter (DLL).
It provides the file input and output features that GFX ShaderConverter (DLL) does not.

It can also be used directly as an executable file without the need to write code to call the DLL.

Policy

This thin layer provides only file I/O functionality to avoid any differences in functionality compared to calling the API from the DLL.

How the Tool Is Provided

GFX ShaderConverter is provided as an EXE file and a corresponding PDB file.

Currently, only a 32-bit version is provided, but a 64-bit version is planned for a future release.
Depending on the low-level hardware, a 32-bit runtime might require the 32-bit version and a 64-bit runtime might require the 64-bit version.

Using

Run the EXE file along with the following command-line options.

When running, processing differs greatly based on whether you are compiling or merging existing shader files.
If the --merge-shader-file option is specified, the process for merging existing shader files is run.
Note that the available options differ based on which processing is run.

Command-Line Options

There is no input file; all options are specified at a command prompt.

In the following table, the Compile column indicates whether an option is available when compiling, and the Merge column indicates whether an option is available when merging.

The following options are required for conversions.

Examples

The following example generates a binary named sample.bnsh from a vertex shader named sample.vs, a pixel shader named sample.ps, and a variation file named sample.xml. 

ShaderConverter32.exe -o="sample.bnsh" -sGlsl -cBinary_Source -aGl --glsl-version=430 --vertex-shader="sample.vs" --pixel-shader="sample.ps" --variation="sample.xml" --print-progress
