Overview

Capabilities of the Font (nn::font) Module

Use the font module (nn::font) to display characters on the screen.

Characters are displayed by mapping textures to polygons, so a lot of characters can be displayed quickly.
In addition, you can apply the following types of effects when displaying characters.

  • Colored characters and color gradients
  • Italics
  • Characters with shadows
  • Control of character spacing and line spacing
  • Kerning (adjusting the spacing between characters for a better appearance)


The font module can display three types of font formats.

  • Bitmap fonts (fonts with a fixed-size display created using a dedicated tool)
  • Scalable fonts (fonts in OTF/TTF formats able to change the display size)
  • System fonts (scalable fonts in a special format)

Description of Terms

Scalable Fonts

The font format that the font module can handle.
Scalable fonts can handle files in the general file formats of TTF (TTC) and OTF.
When displaying scalable fonts, you can scale the font to an optional size.

They are handled as FCPX files by tools and the converter, and ultimately converted to BFCPX files for handling in the programs.

Requires dedicated pre-processing to prepare the necessary texture for display at the time of running the program. As a result, adjustments to how you create the program are necessary.
Scalable fonts have the advantage of allowing support with little memory even when you cannot predict the character content to display in advance.

Bitmap Font

The font format that the font module can handle.
Bitmap fonts are created using the dedicated FontConverter tool, described later.
A major point of difference with scalable fonts is that bitmap fonts are displayed at a fixed size that is specified when they are created.

They are handled as FFNT files by tools and the converter, and ultimately converted to BFFNT files for handling in the programs.

System Fonts

Fonts that can be read and used from the game console.

These fonts are specially formatted scalable fonts.

Use pl Library to read fonts.

The filename extension is .bffnt. Users cannot create this format.

FontConverter

A tool to create bitmap fonts. The tool comes in both GUI and CUI versions.

A specially formatted bitmap image or scalable font (TTF/OTF) can be selected as the input format.

For more information, see here.

Font Runtime (nn::font)

The runtime library to read the font and display it on the screen (a partial program the developer embeds and uses in the game program).

Related Modules

nn::fontll

nn::fontll is a module to handle scalable fonts.

Performs only the basic processing of reading and interpreting the font, accessing the font information of width and height, and drawing the character images in the main memory.

nn::font uses nn::fontll as a lower level module. nn::fontll can also be used on its own.

For more information, see here.

nn::gfx::util::DebugFont

DebugFont is a character drawing module specialized for debugging use.

It can do fewer things than nn::font, but is much easier to use.

For more information, see here.

nn::ui2d

The ui2d module is designed to enable efficient development mainly of the parts drawn in two dimensions, such as the status display and menu images.

ui2d uses nn::font when displaying strings.

For more information, see here.
