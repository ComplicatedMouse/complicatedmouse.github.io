pthread_setschedparam()


int pthread_setschedparam (pthread_t thread, int policy, const struct sched_param *param)


Description
The pthread_setschedparam() function sets the scheduling policy and parameters of the passed-in thread.

NX Specific
  • Only the scheduling policies 'SCHED_FIFO' and 'SCHED_OTHER' are supported on NX
  • May set errno to 'ERANGE' (see 'Errno' section below)
  • Until SDK 12: NX supports priorities from -16(max) to 15(min) such that 0 is default and with a range of 32
  • Since SDK 13: supported priorities are in the range 0 < pthread_get_priority_min() < pthread_get_priority_max(), default priority = (pthread_get_priority_min() + pthread_get_priority_max()) / 2


Note

Scheduling modes *not* supported on NX:
- SCHED_RR
- SCHED_BATCH
- SCHED_IDLE
- SCHED_DEADLINE
- SCHED_RESET_ON_FORK




Errno

ENOTSUP The passed-in policy is an invalid value

ESRCH No thread with the ID thread could be found

EPERM The caller does not have appropriate privileges to set the specified scheduling policy and parameters

EINVAL The policy is not a recognized policy, or param does not make sense for the policy


Errno (NX specific)

ERANGE The passed-in param->sched_priority is not within the priority range supported by NX
More info
http://linux.die.net/man/3/pthread_setschedparam

pthread_once()


int pthread_once (pthread_once_t *once_control, void (*init_routine)(void))

Description

The first call to pthread_once() by any thread in a process shall call the init_routine with no arguments.

Subsequent calls of pthread_once() with the same once_control shall not call the init_routine


NX Specific
  • The 'pthread_once_t' parameter must be statically initialized to PTHREAD_ONCE_INIT
  • 'pthread_once_t' is defined as an 'int'
Note

The 'pthread_once_t' parameter must be statically initialized to PTHREAD_ONCE_INIT

(because if it equals 'PTHREAD_ONCE_INIT', then it implies that the init_routine function hasn't been called yet)

pthread_create()


int pthread_create (pthread_t *thread, const pthread_attr_t *attr, void *(*start_routine) (void *), void *arg)

Description

The pthread_create() function starts a new thread in the calling process by invoking the start_routine() parameter


If attr._a_sched is set to 'PTHREAD_EXPLICIT_SCHED', then the created thread's priority will be set to attr._a_prio.

If attr._a_sched is set to 'PTHREAD_INHERIT_SCHED', then the created thread's priority will be set to the parent thread's priority.


NX Specific

  • If the attr parameter is set to __ATTRP_C11_THREAD (which is "(void*)(uintptr_t)-1"), then the attr parameter is ignored

    and the stack is set to the default size of 81920.

  • This function allocates 90632 bytes using malloc().

  • An additional 1072 bytes is allocated for every thread created, and 2.5KB is allocated after nninitStartup() but before nnMain().
    If you create a custom nninitStartup() that doesn't call nn::os::SetMemoryAllocatorForThreadLocal(), an additional allocate is disabled.

    Note

    If TLS is enabled, an additional 1072 bytes is allocated for every thread created.

    TLS also allocates 2.5KB after nninitStartup() but before nnMain().

    TLS can be disabled by creating a custom nninitStartup() that doesn't call nn::os::SetMemoryAllocatorForThreadLocal().


    If there isn't enough memory, the program will abort



Errno

EAGAIN Failed to create the thread. An abort or assert will likely trigger to explain the issue

EINVAL Invalid settings in attr

EPERM No permission to set the scheduling policy and parameters specified in attr


More info

http://linux.die.net/man/3/pthread_create

pthread_getname_np() and pthread_setname_np()


int pthread_getname_np (pthread_t *thread, char *name, size_t len)

int pthread_setname_np (pthread_t *thread, const char *name)

Description

The pthread_getname_np() retrieves a thread's name

The pthread_setname_np() function can be used to set a unique name for a thread.


NX Specific (until SDK 12)

This implementation supports a maximum of 32 characters for the name (including the null-terminator)


NX Specific (since SDK 13)

The pthread_getname_np() function requires a name buffer of at least 16 bytes or the number of characters in the specified thread's current name (including the null-terminator), whichever is smaller, to succeed.

If the buffer is large enough to contain the specified thread's current name (including the null-terminator), the whole name will be copied.

If the buffer's size is at least 16 bytes, but not large enough to contain the specified thread's current name, the resulting name will be truncated to len - 1 bytes.

For the specified thread's default name and for names set via pthread_setname_np() or nn::os::SetThreadName() a buffer of nn::os::ThreadNameLengthMax bytes will always be sufficient to contain the thread's whole name.

For thread names set via nn::os::SetThreadNamePointer() no such guarantees can be made and the resulting name might be truncated if the buffer does not have a sufficient size to hold the whole name.

It is recommended to use a buffer with a size of nn::os::ThreadNameLengthMax bytes.


The pthread_setname_np() function supports strings of up to nn::os::ThreadNameLengthMax bytes (including the null-terminator).

Passing NULL to the 'name' parameter will set the specified thread's name to its default name.



Errno

ERANGE The name and len parameters for pthread_getname_np() are too small to hold the thread's name

ERANGE The name parameter for pthread_setname_np() is too long


Errno (NX Specific, since SDK 13)

pthread_getname_np() may fail if:

EINVAL the 'name' parameter is NULL.

ERANGE the name buffer's size specified by the 'len' parameter is smaller than 16 and not large enough to contain the currents thread's name (including the null-terminator).


pthread_setname_np() may fail if:

ERANGE the 'name' parameter is not NULL and the string pointed to by it is longer than 32 bytes (including the null-terminator).

More info

http://linux.die.net/man/3/pthread_getname_np

http://linux.die.net/man/3/pthread_setname_np

pthread_getaffinity_np() and pthread_setaffinity_np()


int pthread_getaffinity_np(pthread_t thread, size_t cpusetsize, cpu_set_t *cpuset)

int pthread_setaffinity_np(pthread_t thread, size_t cpusetsize, const cpu_set_t *cpuset)
Description

The pthread_getaffinity_np() function returns the CPU affinity mask of the thread thread in the buffer pointed to by cpuset.

The pthread_setaffinity_np() function sets the CPU affinity mask of the thread thread to the CPU set pointed to by cpuset.


The various "CPU_*" macros in "sched.h" that "pthread.h" includes (such as CPU_SET and CPU_ISSET) are also supported.


NX Specific (until SDK 12)
  • The NX implementations use a single 'nn::Bit64' (uint64_t) for the affinity mask rather than a 'cpu_set_t' array. As a result, the
    mask is stored in either cpuset.__bits[0] (when built as a 64-bit application),
    or the low bits of the 'nn::Bit64' stored in cpuset.__bits[0] and the high bits in cpuset.__bits[1] (32-bit application).
    This means that only __bits[0] and/or __bits[1] are ever used and that only sizeof(nn::Bit64)*8 cores are supported in a single mask.
  • In pthread_setaffinity_np(), the argument 'cpusetsize' can be used to pass in an "ideal core" value.
    The internal NX implementation accepts an "ideal core" value. Since the standard pthread function declaration doesn't directly support that;
    'cpusetsize' can optionally be used to take advantage of this extension that NX provides. Supported values of 'cpusetsize' below:
Value of cpusetsize Meaning
Value of one of the cores in the 'cpuset' mask User can specify a preferred core if the 'cpuset' mask contains more than one core.
IdealCoreDontCare ("-1") Make the thread run on any of the cores in the affinity mask as determined by the system
IdealCoreUseDefaultValue ("-2") Make the thread run on the same core as the process that created the thread
IdealCoreNoUpdate ("-3") Update affinity mask, but don't move thread to another core unless internal scheduler requested a core change
sizeof(cpu_set_t) The argument is ignored, and "IdealCoreUseDefaultValue (-2)" is used as the "ideal core" value.
Value not contained in 'cpuset' and not one of the other values above Assert will trigger
NX Specific (since SDK 13)
The NX implementations use a single 'nn::Bit64' (uint64_t) for the affinity mask. Therefore only the first 64 CPUs of a cpu_set_t are supported.
The 'cpusetsize' parameter is only used to specify the byte size of the CPU set pointed to by the 'cpuset' parameter.

The function pthread_getaffinity_np() requires a cpu_set_t that can contain at least 64 CPUs, i.e., a size of at least 8 bytes. All CPUs beyond 64 will be cleared.

The function pthread_setaffinity_np() allows arbitrary 'cpusetsize' parameter values, as long as they could also be acquired via the use of one of  CPU_SETSIZE, sizeof(cpu_set_t), or CPU_ALLOC_SIZE().  Using different sizes will trigger an assert.
At least one CPU contained in the mask returned by nn::os::GetThreadAvailableCoreMask() must be specified, all other CPUs (within the first 64) will be ignored. CPUs higher than the first 64 are not allowed to be set and will result in EINVAL being returned.
Calling pthread_setaffinity_np() successfully will set the specified thread's  "ideal core" value to nn::os::IdealCoreDontCare.
To set a thread's "ideal core" to any other value, please use  nn::os:: SetThreadCoreMask().



Errno

The pthread_getaffinity_np() function may fail if:

EINVAL The 'thread' parameter is invalid or was not properly initialized.
ESRCH Could not find the thread represented by 'thread', or the thread was not properly initialized.

The pthread_setaffinity_np() function may fail if:

EINVAL The 'thread' parameter is invalid or was not properly initialized.

ESRCH Could not find the thread represented by 'thread', or the thread was not properly initialized.

EFAULT A supplied memory address was invalid.
EINVAL (pthread_setaffinity_np()) The affinity bit mask mask contains no processors that are currently physically on the system.
EINVAL (pthread_setaffinity_np()) cpuset specified a CPU that was outside the set supported by the kernel.


Errors (NX specific, until SDK 12)

The pthread_getaffinity_np() function shall fail if:

EOVERFLOW The 'cpuset' parameter is the wrong size to contain an nn::Bit64

The pthread_setaffinity_np() function shall fail if:

EOVERFLOW The 'cpuset' parameter is the wrong size to contain an nn::Bit64

ASSERT An assert will trigger if the 'cpuset' parameter is invalid, meaning it's empty or if an ideal core was passed in

through 'cpusetsize' but that core isn't contained in the mask.

ASSERT An assert will trigger if 'cpusetsize' was used to pass in an ideal core value instead of just "sizeof(cpu_set_t)" and that

core value isn't contained in the 'cpuset' mask

Errors (NX specific, since SDK 13)

The pthread_getaffinity_np() function shall fail if:

EINVAL The 'size' parameter is smaller than sizeof(nn::Bit64).

ESRCH The 'thread' parameter does not refer to a valid thread.

ASSERT An assert will trigger if the 'size' parameter is not a value that could also be acquired via the use of one of CPU_SETSIZE, sizeof(cpu_set_t), or CPU_ALLOC_SIZE().

The pthread_setaffinity_np() function shall fail if:

EINVAL The CPU set specified by 'cpuset' does not contain any CPUs that are also contained in the mask returned by nn::os::GetThreadAvailableCoreMask().

EINVAL The CPU set specified by 'cpuset' contains CPUs that do not fit into nn::Bit64 (i.e. `CPU_ISSET_S(cpu, size, set) != 0` for any cpu >= 64).

ESRCH The 'thread' parameter does not refer to a valid thread.

ASSERT An assert will trigger if the 'size' parameter is not a value that could also be acquired via the use of one of CPU_SETSIZE, sizeof(cpu_set_t), or CPU_ALLOC_SIZE().
