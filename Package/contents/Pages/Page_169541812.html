Changes for Developers

This section describes the changes made to NintendoWare audio (nn::atk) in NintendoSDK 0.15.0.

Contents

  • Added an API for directly specifying the volume for each channel.
  • Added the I3DL2 Reverb effect.
  • Added an API for getting the rendering results.
  • Changed the initial value of the flag for enabling or disabling effects.
  • Revisions to the Documentation

Added an API for directly specifying the volume for each channel.

It is already possible to specify the volume for each channel indirectly using the panning parameter, but now a feature has been added for specifying the volume for each channel directly.

For more information, see the AtkMixMode sample demo, the section about directly specifying the volume for each channel in the Mix page of the programming guide, and the API references for the following functions.

  • nn::atk::SoundHandle::SetMixMode
  • nn::atk::SoundHandle::SetMixVolume

Added the I3DL2 Reverb effect.

Added the I3DL2 Reverb effect. In addition, added I3DL2 Reverb to the effects that are applied in the AtkEffect sample demo.
For more information, see the API reference for the nn::atk::EffectI3dl2Reverb function.

Added an API for getting the rendering results.

Added a feature for getting the rendering results of the audio renderer.

For more information about getting and using the sample data, see the section about getting the rendering results on the Quick Start page of the programming guide, and the API references for the following functions.

  • nn::atk::SoundSystem::ReadCircularBufferSink
  • nn::atk::SoundSystem::GetCircularBufferSinkBufferSize
  • nn::atk::SoundSystem::GetRendererSampleCount
  • nn::atk::SoundSystem::GetRendererChannelCountMax

Changed the initial value of the flag for enabling or disabling effects.

Previously, the flag for enabling or disabling effects was enabled by default, but if you temporarily deleted an effect and then added it back in later, you had to explicitly enable the flag for the effect that was added back in. This was different from what you had to do the first time the effect was added, resulting in a confusing user experience. As a result, this flag is now disabled by default and must always be explicitly enabled.

Note that if you had previously not been using the SetEnabled() function to explicitly set this flag to true for each effect, some of your effects may no longer be enabled.

Revisions to the Documentation

The following revisions have been made.

  • Corrected the table named Chart of Sound Parameter Types and Processing Methods on the Sound Parameter Control Operations page of the Sound System Manual.
  • Corrected the information about the default output mode for the nn::atk::SoundSystem::SetOutputMode function in the API reference.
