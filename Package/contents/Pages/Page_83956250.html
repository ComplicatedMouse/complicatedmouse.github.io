Nodes

In this manual, the term node refers to the basic unit for the hierarchical structure of a 3D model.

In 3ds Max, nodes correspond to such objects as standard primitives, dummies, and bones.

3ds Max nodes are exported as bones in model intermediate files.

Node Names

Although you can assign the same name to multiple nodes with 3ds Max, make sure that you assign a unique name to each node because there is no overlap of node names in the intermediate file.

If multiple nodes with the same name exist, the Export plug-in exports the first node that it fetches with the existing name, and then alters the name of each subsequent node by adding the underscore symbol and a number. (For example, name_01, name_02, and so on.)
The user cannot check or change the order in which numbers and nodes are exported by name.

Info

This order of export may change depending on the version of the DCC tool, or the environment.

Root Nodes

The intermediate file contains only one root bone for each skeletal model.

If there are multiple root nodes on 3ds Max, the plug-in will group them together and add them for export to the intermediate file as a single bone with the name nw4f_root.

Node Scale and Inheritance

The plug-in supports scaling only in the general local coordinate system.

The following 3ds Max features are not supported.
Note that if your model includes these features, the appearance differs slightly between 3ds Max and NintendoWare.

  • There is no support for non-local scaling (that is, world and view).
  • There is no support to turn off inheritance of Translate, Rotate and Scale from the parent.

Nodes Not Exported

Cameras, lights and the targets they point at are not exported. Nodes having any of these as parents are also not exported.
The hierarchical structure of nodes not displayed by 3ds Max is exported but not the mesh.
Nodes and helpers that cannot be converted to meshes such as shapes, are output as meshless nodes.

Info

Nodes whose compression enable flag has been set using the plug-in for not compressing nodes will always be exported, regardless of these conditions.

Base Point

When the data are exported, the base point is treated as the origin in local coordinates.

Note that the position of the 3ds Max base point as it appears on 3ds Max does not always match its internal position.
If you have rotated or moved the base point, confirm the position by selecting Utilities > Reset Xform.
If you do not reset Xform this way, the internal position of the base point is taken to be its position when Reset Pivot was performed.
