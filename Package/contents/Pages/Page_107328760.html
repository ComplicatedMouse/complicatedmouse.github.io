Feature Overview

DevMenuCommand is a target-oriented tool that can be used to run the features of DevMenu from the command line. The tool can use the following features.

Basic Features for Development

The following basic features are essential for application development.

Command Name Description Page Main Provided Features
application Running and Controlling Applications
  • List installed applications, and start, exit, delete, and install applications.
  • Configure the application startup environment.
patch Managing Patches
  • Install, list, and delete patches.
addoncontent Managing Add-On Content
  • Install, list, and delete add-on content (downloadable content).
controller Managing Controllers
  • Check and delete pairing settings.
  • Test wireless connections and disconnect wireless connections.
savedata Managing Save Data
  • List, delete, back up, and restore save data.
account Managing User Accounts
  • Create, delete, and list user accounts.
  • Configure the account system.

The following features can be used when microSD cards and NX development cards are being used.

Command Name Description Page Main Provided Features
sdcard Managing microSD Cards
  • Initialize and format microSD cards and get their status.
gamecard NX Development Card Features
  • Write an NSP file to an NX development card.
  • Get the status of an NX development card.

How to Use

Start the tool using the devmenu subcommand of ControlTarget, or by using RunOnTarget.

$ Tools\CommandLineTools\ControlTarget.exe devmenu -- --help
usage: DevMenuCommand <command> [<args>]

The DevMenu commands are:
   firmware        Show firmware information
   application     Install, uninstall, list, or start applications.
   savedata        List or delete savedata
[SUCCESS]
$ Tools\CommandLineTools\RunOnTarget.exe TargetTools\NX-NXFP2-a64\DevMenuCommand\Release\DevMenuCommand.nsp -- --help

usage: DevMenuCommand <command> [<args>]

The DevMenu commands are:
   firmware        Show firmware information
   application     Install, uninstall, list, or start applications.
   savedata        List or delete savedata
   ...
[SUCCESS]
Info

To run, DevMenuCommand borrows some of the memory used by the system features, so minimize the use of these features (particularly the HOME Menu, but also DevMenu) while the tool is running. Using these features while DevMenuCommand is running may cause a fatal error due to insufficient memory. If this occurs, reset and restart the system.

Executing Multiple Commands

Several processes can be executed in order with a single call to DevMenuCommand. This is quicker than making a succession of calls to DevMenuCommand.nsp using RunOnTarget.exe.
To specify several processes, you delimit the commands using special tokens.

Token Description
Semicolon (;) Sequentially executes the commands.
&&

Executes the next command if the previous process succeeded (SUCCESS).
|| Executes the next command if the previous process failed (FAILURE).
THEN Alternative to the semicolon (;) token. Must be preceded by a space.
AND Alternative to the ampersands (&&) token. Must be preceded by a space.
OR Alternative to the pipes (||) token. Must be preceded by a space.

The semicolon (;), ampersands (&&), and pipes (||) tokens are parsed by shells like bash, so use a backslash or another escape character with these tokens, or use the alternatives THEN, AND, and OR instead.
See the following example.

$ RunOnTarget.exe DevMenuCommand.nsp -- application list AND application uninstall 0x01004b9000490000 OR application uninstall 0x01004b9000490001
1 application(s) found.
id                          version
0x01004b9000490000            0.0.0
Not found.
[SUCCESS]

Feel free to also use the batch command.

If you intend to specify option values that include a semicolon (;) or a double ampersand (&&) as follows, prefix the command name with the --ignore-no-space-command-delimiter option. This prevents options that include a semicolon (;) or a double ampersand (&&) from being accidentally split.

$ RunOnTarget.exe DevMenuCommand.nsp -- --ignore-no-space-command-delimiter application install "C:\delimited;option.nsp"

When using this command with the batch command, described later, it works when specifying the path for the text file for the batch process, but it does not work for any options specified in that text file.

Executing a Batch of Commands

Use the batch command to execute multiple DevMenuCommand commands at the same time.

To do this, first prepare a text file with a list of strings after the DevMenuCommand command, as shown in the following example. For this example, the file is named batch.txt.
You can make it a comment line by adding a number sign (#) or two slash marks (//) to the beginning.

// Installing Applications
application install "C:\application1.nsp"
application install "C:\application2.nsp"
application install "C:\application3.nsp"
application install "C:\application4.nsp"

Specify the preceding text file in a batch command.

$ RunOnTarget.exe DevMenuCommand.nsp -- batch C:\batch.txt
Preparing to install C:\application1.nsp...
Preparing to install C:\application2.nsp...
Preparing to install C:\application3.nsp...
Preparing to install C:\application4.nsp... 
[SUCCESS]

Specify the --ignore-error option to proceed with processing even if there is an error.

Info

If you want to specify options that include a semicolon (;) or a double ampersand (&&) in the text file, specify the --ignore-no-space-command-delimiter option in one of the following ways.
You can also combine the text file with the --ignore-no-space-command-delimiter option of DevMenuCommand itself.

  • Specify lines in the text file just like you would on the command line to prevent the options on those lines from being split.
  • Specify --ignore-no-space-command-delimiter after the text file path in the batch command, as shown below, to prevent the splitting of all options in the file. 
    batch C:\batch.txt --ignore-no-space-command-delimiter
