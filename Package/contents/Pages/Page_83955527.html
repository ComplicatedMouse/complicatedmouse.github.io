Texture Pattern Animation

Texture pattern animations are set by animating the file node Image Number (the Frame Extension attribute).

Select a file node, open the Attribute Editor, and then select Use Image Sequence (the Use Frame Extension attribute).

Delete the expression when applying a key because, currently, the default expression is automatically connected to the Image Number (the Frame Extension attribute).

Next, set the animation for the Image Number (Frame Extension attribute). Driven keys, expressions, and animation layers can also be used.
However, when expressions or animation layers are being used, and when keys are not being set directly (for example when they are being set via the utility node), you must use the Texture Pattern Animation Setting plug-in to configure the list of textures to output.

Set the Frame Offset attribute to 0.
When the following cases apply, the animation curve data is created automatically from baked data and exported.

  • There is at least one key whose linear type is not Stepped.
  • Driven keys, expressions, and animation layers are being used.
  • Keys are not set directly to image numbers (Frame Extension attribute).

A number that corresponds to the frame extension is applied to the names of the files used for texture pattern animation. The number appears immediately before the filename extension, as in name.number.tga, such as name.1.tga, name.2.tga, name.3.tga, and so on.
Filenames where numbers start with a zero, such as name.01.tga and name.001.tga, are also supported. (Numbers up to four digits long can start with zeros.) For example, if two files have the names name.1.tga and name.01.tga, the texture without a 0 in the filename is exported.

Info

The only textures that are exported by default are those set by using the Image Name attribute of the file node, and those that correspond to the keys that were applied to the Frame Extension attribute.

For instance, if the Image Name is name.1.tga and only two keys have been applied where the Frame Extension is 1 for the start frame and 4 for the end frame, only name.1.tga and name.4.tga are output.
Even if the animation curve tangents are linear, name.2.tga and name.3.tga are not output.
If the texture for the Frame Extension of the animation curve is not exported, the texture for the closest Frame Extension is used as the pattern animation data.

Info

Use the plug-in for setting texture patterns to configure a texture to be forcibly exported. Textures configured using this plug-in are exported even if an animation has not been set for the image number (extended frame attribute).

For example, if you set the forced output Frame Extension range to 1-4 (from 1 through 4), there will only be two-key hits with Frame Extension equal to 1 for the start frame and 4 for the end frame. The Tangents of the animation curves can be output as Linear.
When you use multiple FMA files (FTP files) with a single model, make sure that you use the Texture Pattern Animation Setting plug-in to configure the settings so that all necessary textures are exported.

Info

If the Compress Material export option is enabled, even materials for which texture pattern color animations are not set sometimes are animated as a result of united materials. If this happens, use the plug-in for setting material attributes to change settings so that materials for which color animation is set are not compressed.
