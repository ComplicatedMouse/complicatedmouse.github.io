Getting Network Information

Use the network command to get network information.
The network must be in an available state.

$ RunOnTarget.exe DevMenuCommand.nsp -- network --help
usage: DevMenuCommand network confirm-internet-connection [--disconnect] [--exclusive] [--try <count>]
       DevMenuCommand network get-current-primary-ip-addr
       DevMenuCommand network get-global-ip-addr
       DevMenuCommand network get-nat-type
       DevMenuCommand network get-wlan-mac-addr
       DevMenuCommand network enable-wireless-communication-for-test
       DevMenuCommand network disable-wireless-communication-for-test
       DevMenuCommand network measure-band-width-for-download [--timeout <time_in_msec>]
       DevMenuCommand network measure-band-width-for-upload [--timeout <time_in_msec>]

Internet Connectivity Test

Internet connectivity of the development hardware is tested.

$ RunOnTarget.exe DevMenuCommand.nsp -- network confirm-internet-connection [--disconnect] [--exclusive] [--try <count>]
Internet Connection: Confirmed
[SUCCESS]
Option Description
--disconnect

Disconnects from the network beforehand, and then connects to perform the connectivity test.
--exclusive Cancels all network use requests from system processes before performing the connectivity test.
--try <count> Tries up to the maximum count until the connectivity test is successful.

Checking the IP Address

The IP address assigned to the development hardware is displayed.

$ RunOnTarget.exe DevMenuCommand.nsp -- network get-current-primary-ip-addr
Waiting for network availability...
[RESULT] xxx.xxx.xxx.xxx
[SUCCESS]

Verifying the Global IP Address

The global IP address is displayed. This same information can be verified from Connection Test in the menu.

$ RunOnTarget.exe DevMenuCommand.nsp -- network get-global-ip-addr
Waiting for network availability...
...
[RESULT] xxx.xxx.xxx.xxx
[SUCCESS]

Checking the NAT Type

This command displays the NAT type. This same information can be verified from Connection Test in the menu.

$ RunOnTarget.exe DevMenuCommand.nsp -- network get-nat-type
Waiting for network availability...
...
[RESULT] NAT type X
[SUCCESS]

Verifying the MAC Address of the Wireless LAN Interface

The MAC address of the development hardware’s wireless LAN interface is displayed.

$ RunOnTarget.exe DevMenuCommand.nsp -- network get-wlan-mac-addr
[RESULT] xx:xx:xx:xx:xx:xx
[SUCCESS]

Enabling Wireless LAN

The temporarily disabled wireless LAN is enabled again. You must first enable nifm test mode using either the DevMenu or the debug enable-nifm-testing-mode command.

$ RunOnTarget.exe DevMenuCommand.nsp -- network enable-wireless-communication-for-test
[SUCCESS]

Disabling Wireless LAN

Wireless LAN is temporarily disabled. Unlike airplane mode, this disabled state is not permanent. You must first enable nifm test mode using either the DevMenu or the debug enable-nifm-testing-mode command.

$ RunOnTarget.exe DevMenuCommand.nsp -- network disable-wireless-communication-for-test
[SUCCESS]

Measuring the Download Bandwidth

This measures the download bandwidth from the development hardware.

$ RunOnTarget.exe DevMenuCommand.nsp -- network measure-band-width-for-download [--timeout <time_in_msec>]
[RESULT] Download speed : xxxxxxx.xxxxxx (bytes/sec)
[SUCCESS]
Option Description
--timeout <time_in_msec>

Use this option to set the timeout time, in milliseconds.

If the --timeout option is not specified, the timeout time is set to 10 seconds by default.

Measuring the Upload Bandwidth

This option measures the upload bandwidth from the development hardware.

$ RunOnTarget.exe DevMenuCommand.nsp -- network measure-band-width-for-upload [--timeout <time_in_msec>]
[RESULT] Upload speed : xxxxxxx.xxxxxx (bytes/sec)
[SUCCESS]
Option Description
--timeout <time_in_msec>

Use this option to set the timeout time, in milliseconds.

If the --timeout option is not specified, the timeout time is set to 10 seconds by default.
