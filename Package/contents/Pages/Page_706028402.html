Connect to a Dev Kit

The first step in taking a profile is connecting to your dev kit. This is done by clicking the  Sync button. This will pop open the Select a Profiling Scenario window that helps guide you through the steps. On the left side of the window are tabs to select one of the following scenarios:

Attach: Connect to an already running game

The most common situation is to simply Attach the profiler to your game that is already running (please ensure that your game is not connected to the debugger - if using Visual Studio, launch your game with Start without Debugging). When attaching to an already running game, you'll need to specify the NSS file in the Attach tab. If you performed Full Integration Setup, you can just click the Autofill button and it will auto-detect and fill in the correct file. Otherwise, you'll need to locate your game's NSS file by clicking on the  button.

Attach on Launch: Wait for the game to launch and then attach to it

The profiler can wait for your game to launch and then immediately attach to it. However, make sure that your game is not connected to the debugger. If using Visual Studio, launch your game with Start without Debugging. As with Attach, you'll need to specify an NSS file.

Launch: The profiler starts the game and then attaches to it

In this scenario, the profiler launches your game through Target Manager and then attaches to it. For this to work properly, you'll need your NINTENDO_SDK_ROOT environment variable to be set and pointing to your SDK root.

This scenario is the best way to profile the startup of your game.

In-Process Only: Attach to your game with in-process only profiling

If you performed Full Integration Setup, then you can attach to your already running game. This mode restricts you to  In-Process profiling, which is only appropriate if your game doesn't have threads that can migrate between cores. The benefit is that the profiler overhead is lower than  Out-of-Process profiling. This mode also allows the debugger to be attached to your game.

Please see In-Process vs. Out-Of-Process for clarification on the difference between these profiling modes.
