Legend

Item Format Description
<index> Any value 0 through 7. Specifies the index for a user account displayed in the Accounts area of DevMenu.

<nsa_id>

 A 16-digit hexadecimal value.
Example: 0000000000000000		 Specifies the network service account ID.
<request_id>

A 16-digit hexadecimal value.

Example: 0000000000000000

 The ID for managing a friend request.
You can check these IDs using the get-sent-friend-request-list subcommand and the get-received-friend-request-list subcommand.
<friend_code> A 12-digit decimal value.
Separated by hyphens every four digits.
Example: 0000-0000-0000

Specifies the friend code.

A user can check their own friend code using the get-user-setting subcommand.

Sending Friend Invitation Information During Application Startup

Normally, only after a friend invitation has been received and an application is started or resumed from the User Page system feature, the application can use the following functions to detect that a notification is present and get any parameters attached by the sender of the friend invitation.

  • nn::friends::GetFriendInvitationNotificationEvent()
  • nn::friends::TryPopFriendInvitationNotificationInfo()

This DevMenuCommand feature is a debugging feature for simulating that a pseudo-notification of a friend invitation is present without having to actually start the application to test that parameters are received correctly when the application is then started or resumed. (This debug feature is available in NX Add-On 9.2.0 and later.)

Advance Reservation of Friend Invitation Information During Application Startup

$ RunOnTarget.exe --suppress-auto-kill DevMenuCommand.nsp -- friends set-friend-invitation --user 0 --app 0x01004b9000490000 --file d:/parameter.dat
  • Re-creates an application startup state where the application has been started after a friend invitation has been received.
    • By default, if this command has not been executed, this feature is disabled.
    • After this command has been executed, a pseudo-notification of a friend invitation is generated each time the application is started.
  • The following options can be specified.
    • Use --user to specify the user account that is the target of the friend invitation, with its index value in the user account list.
    • The --file <path> option sets the specified file on the host PC to be passed as an application parameter. Binary files of up to nn::friends::FriendInvitationApplicationParameterSizeMax bytes in size can be passed.
      • When the --file <path> option is not specified, the system responds as if a 0-byte parameter was set.
    • The --app <id> option can be used to specify a 16-digit hexadecimal ApplicationId, which causes the notification to be generated only when the corresponding application is started.
      • When the --app <id> option is not specified, the notification is generated when any application is started.

Disabling Advance Reservation of Friend Invitation Information During Application Startup

$ RunOnTarget.exe --suppress-auto-kill DevMenuCommand.nsp -- friends clear-friend-invitation
  • Disables the advance reservation setting configured using the set-friend-invitation subcommand.

Instant Notification of Friend Invitation Information While Application Is Running

$ RunOnTarget.exe --suppress-auto-kill DevMenuCommand.nsp -- friends push-friend-invitation --user 0 --file d:/parameter.dat
  • This subcommand instantly generates a friend invitation notification while an application is running.
    • Use --user to specify the user account that is the target of the friend invitation, with its index value in the user account list.
    • The --file <path> option sets the specified file on the host PC to be passed as an application parameter. Binary files of up to nn::friends::FriendInvitationApplicationParameterSizeMax bytes in size can be passed.
      • When the --file <path> option is not specified, the system responds as if a 0-byte parameter was set.
    • If no application is currently running, this subcommand does nothing.

Reissuing Friend Codes

$ RunOnTarget.exe DevMenuCommand.nsp -- friends issue-friend-code 1

Issue friend code ...
Issue friend code ... failed. e = 0011fa79, code = 121-2301
The issue of the friend code is disallowed. (next issuable time = 1476936852, posix time)
[FAILURE]
Info

After a friend code is issued, it cannot be reissued for a period of time.
The time when the next friend code can be issued is displayed in the log.

In the previous example, this time is 1476936852 = 2016-10-20 04:14:12 (UTC).
