Library

Overview

The PC version of OpenGL provides only header files. The library does not need a static link, and the OpenGL library entry point uses WGL, GLEW, or similar, to get function pointers from the graphics driver at runtime. The specific usage varies depending on the PC GPU vendor. For more information, see the OpenGL website.

The NX version of OpenGL provides a specialized library (EGL) in addition to the header files.

Currently, the NX version of OpenGL exports the functions that exist in the unextended OpenGL ES 3.2 implementation. However, we do not recommend statically linking these functions for use because static linking may not be possible in the future. Instead, dynamically link all required OpenGL functions.

To link to functions dynamically, first use eglGetProcAddress to get a function pointer, and then use the obtained function pointer to call the function. The task of loading these function pointers can be left to the GLEW library (described later), the GLL library, or another OpenGL Loading Library. The GLL library is used to load these pointers when the device is initialized in the OpenGL version of the GFX library.

Note

In NX Add-On 8.1.0 and later, the NX version of OpenGL does not export OpenGL functions, including those existing since OpenGL 1.0. Accordingly, in NX Add-On 8.1.0 and later, you are required to dynamically link the functions. Use a loading library or eglGetProcAddress to get the functions dynamically. The GLL library loads all OpenGL functions, including those existing since OpenGL 1.0.

Header File

The NintendoSDK NX Add-On provides headers such as gl.h.

These header files are placed in the following directory, along with other required header files.

$NINTENDO_SDK_ROOT/Include/EGL
$NINTENDO_SDK_ROOT/Include/GL
$NINTENDO_SDK_ROOT/Include/GLES2
$NINTENDO_SDK_ROOT/Include/GLES3
$NINTENDO_SDK_ROOT/Include/KHR

OpenGL Extension Wrangler Library

The NintendoSDK NX Add-On provides the OpenGL Extension Wrangler Library (GLEW), which is used for getting function pointers from the graphics driver with OpenGL.

The header files and libraries required to use this functionality are placed in the following directory.

$NINTENDO_SDK_ROOT/Externals/glew

libglew.a must be linked to use GLEW in the NX version.

Note

GLEW does not automatically get the functions that have existed since OpenGL 1.x (such as glPolygonMode and glLogicOp). To use these functions on the NX, use eglGetProcAddress to get the function pointer.

gll Library

The NintendoSDK NX Add-On provides the gll Library for getting function pointers from the graphics driver in OpenGL. For more information about using this library, see gll Library.

Unlike the GLEW library, the GLL library gets all functions dynamically in the NX environment.
