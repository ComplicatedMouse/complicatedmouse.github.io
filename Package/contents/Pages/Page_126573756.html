Resources

Overview

Resources provide features to interpret binary images created from tools in advance.

Resources in gfx are not primitive features, but provide a variety of features by using gfx primitive information and objects so that, for example, the shader resources hold shader information internally.

In particular, the top level class that expresses binary images is referred to as the resource file class.

Structure

Binary images used in gfx resources must begin with nn::util::BinaryFileHeader.
As a result, you can load only sizeof(nn::util::BinaryFileHeader) and check the file's alignment and size as nn::util::BinaryFileHeader.

The gfx resource classes consist of accessors and data.
Accessors inherit the nn::util::AccessorBase that takes the corresponding data as template parameters.

Name

The classes handling the resources in gfx start with the "Res" prefix.
Resource file classes also end with the "File" suffix.

Common Features

This section describes features that are shared among all gfx resource file classes.

Signature

Resource file classes have the static variable Signature that represents the first 8 bytes of binary images.

Version

Resource file classes have the static variables MajorVersion, MinorVersion, and MicroVersion that represent the version.

MajorVersion cannot use a different resource file, and MinorVersion cannot use a large resource file.

Determining the Validity of the Binary

By passing IsValid to the pointer to the start of the binary image, you can check the version of the target binary and otherwise determine whether it can be used as a resource.

Casting to Resource File Classes

To get the pointer to the resource file class, pass the pointer to the beginning of the binary image to ResCast.

When casting to resource file classes, relocation is executed to resolve the pointer member internally from the offset value to the actual pointer.

Getting the Binary Header

Use GetBinaryFileHeader to get the pointer to nn::util::BinaryFileHeader.
You can use this to get the file alignment and size, for purposes such as manuallycontrollling relocation.
