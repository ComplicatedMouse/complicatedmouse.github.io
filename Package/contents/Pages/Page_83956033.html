Plug-In for Selecting Normal Vectors With Negative Z Value

This section describes the use and features of the plug-in for selecting normal vectors with a negative z value.

  • How to Use
  • Notes
  • Related Links

How to Use

The z component of a vector is not displayed correctly if it is negative when the pixel shader calculates a z component using a texture format such as snorm_bc5 that only includes xy components for normal mapping.
This plug-in is used to detect pixels where the z component of the normal is negative.

Select the layer to be processed and then execute File > Automate > Nintendo Select Negative Z Normal.

Pixels with a negative z component for the normal are selected if they exist.
If there are none, the message "All pixels have positive Z" is displayed.

Notes

These notes relate to the use of the plug-in for selecting normal vectors with a negative z value.

Image Mode

This option cannot be executed on files with an image mode other than RGB color or that do not use 8 bits per channel.

Layers

Selecting a hidden layer and then running this plug-in causes an error.
Selecting a completely transparent layer with no valid pixel values and then running this plug-in also causes an error.
