Introduction

About NintendoSDK Configuration Manager

NintendoSDK Configuration Manager is a Visual Studio extension for configuring the required settings to use NintendoSDK from an existing Visual C++ project.

To use NintendoSDK from a Visual C++ project, you must import certain property sheets provided by NintendoSDK and define the properties for the NintendoSDK features that will be used. NintendoSDK Configuration Manager provides a set of features that simplify the process of configuring these settings on Visual Studio. To be specific, it provides the following features.

  • Switching to use NintendoSDK in the project
  • Changing the root path of the NintendoSDK to use
  • Changing the NintendoSDK API set and build type to use

Intended Audience

All developers using Visual Studio as their build system. We recommend using NintendoSDK Configuration Manager particularly to developers who are managing the following kinds of Visual Studio projects.

  • Regular projects for Windows that do not use the NintendoSDK
  • Projects that were created using the NintendoSDK wizard for creating a new project

Using NintendoSDK Configuration Manager is not required for project configuration. Developers are free to configure their projects by defining the necessary properties and importing the necessary property sheets.

Installation

Use NDI to install the NintendoSDK VsExtension package. NintendoSDK Configuration Manager will be added to each installed version of Visual Studio 2015, 2017, and 2019. If Visual Studio was running when you conducted the installation, restart the program.

To confirm that the extension was properly installed, from the Visual Studio menu, select Tools > Extensions and Updates. If you see the NintendoSDK Visual Studio Extension menu item, the extension was properly installed.

Info

Depending on the environment, installation of the NintendoSDK VsExtension package can fail. For methods for resolving a number of issues, see the Troubleshooting page.

Uninstalling

Use NDI to uninstall the NintendoSDK VsExtension package.

How to Use

To start NintendoSDK Configuration Manager, from the Visual Studio menu, select Tools > NintendoSDK > NintendoSDK Configurations. A window like the following will open.

This window shows the NintendoSDK-related settings items for each of the project configurations of each of the projects in the opened Visual Studio solution. The following table describes these settings.

Item Description
Use NintendoSDK Select if the project will use NintendoSDK. When this check box is selected, the NintendoSDK property sheets will be imported and the settings for include paths and library paths will be added to the project.
SDK Path The root path of the NintendoSDK to use.
API Set The NintendoSDK API set to use. The values available for this setting vary depending on the structure of the NintendoSDK Add-On packages specified in the SDK path.
SDK Build Type The NintendoSDK build type to use. For more information, see Build Targets and Build Types.

Click OK to apply the settings to each project.

Info
  • To properly apply these settings, close the opened solution immediately after you have clicked OK in NintendoSDK Configuration Manager, and then reopen the solution. Any changes made to any files will be saved automatically.
  • When the settings are applied, NintendoSDK Configuration Manager creates files in the same directory as the project file. For more information, see Settings Configured in the Project by NintendoSDK Configuration Manager later in this document.

Troubleshooting

The following message appears: "The current project was configured using an earlier NintendoSDK format."

Sometimes a message like the following will be displayed immediately after you open NintendoSDK Configuration Manager.

It appears when the project was created with a version of the SDK earlier than NintendoSDK 0.15.0.

For NintendoSDK Configuration Manager to save settings like the SDK path in a manner different from the opened project, the project format must be updated to a format that can be edited by NintendoSDK Configuration Manager. To update the project to the new format, click OK.

The following message appears: "The NintendoSDK configuration could not be applied to the project."

Sometimes a message like following will be displayed when you click OK in the NintendoSDK Configuration Manager window.

It appears when a project does not correctly reflect the changes to settings configured by NintendoSDK Configuration Manager.

One reason for this situation is if a developer overwrites the definitions for property values added by NintendoSDK Configuration Manager to the project file. Check whether the definitions of the properties that are shown in this dialog box have had been overwritten.

For more information about the properties defined by the NintendoSDK Configuration Manager, see Settings Configured in the Project by NintendoSDK Configuration Manager.

Settings Configured in the Project by NintendoSDK Configuration Manager

When you click OK in NintendoSDK Configuration Manager, the following actions take place in the opened projects.

Old, Imported Property Sheets Are Removed

If any of the following past NintendoSDK property sheets were imported to the project file, they are treated as no longer imported.

  • NintendoSdkSpec_*.props
  • NintendoSdkBuildType_*.props
  • NintendoSdkVcProjectSettings.props

(The asterisk (*) represents a string that differs depending on the project file.)

Property sheets that are no longer imported can be deleted from the project.

Properties Are Defined

If the Use NintendoSDK check box is selected, the following properties are defined in the project file.

  • NintendoSdkRoot
  • NintendoSdkSpec
  • NintendoSdkBuildType

If the Use NintendoSDK check box is cleared, these properties become no longer defined. For more information about the specifications of these properties, see NintendoSDK Property Sheets.

Property Sheets Are Imported

If the Use NintendoSDK check box is selected, the ImportNintendoSdk.props property sheet is imported to the project file.

The ImportNintendoSdk.props property sheet has features for configuring the various settings to use NintendoSDK. It is created by NintendoSDK Configuration Manager in the same directory as the project file. Do not delete the created ImportNintendoSdk.props and make sure it is always in the same directory as the project file.

If the Use NintendoSDK check box is cleared, this property sheet is no longer imported.

A NintendoSDK Configuration Manager Settings File Is Created

A NintendoSDK Configuration Manager settings file named <Project_File_Name>.nnsdk.xml is created in the same directory as the project file. This file is for saving the settings for project configurations with the Use NintendoSDK check box cleared.
