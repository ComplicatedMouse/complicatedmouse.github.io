Overview

AuthoringEditor is a GUI tool for checking and editing NMETA files and NSP files.

The created NMETA files are passed to the AuthoringTool as input to create NSP or NSPD files.

Location

This tool is packaged with NintendoSDK and is available at the following location.

Tools/AuthoringEditor/AuthoringEditor.exe

How to Use

The startup screen is displayed when AuthoringEditor.exe is run.

Select the operation to perform, and then create or edit an NMETA file, or edit an NSP file.

Restrictions

NSP Files Built With an SDK Whose Major Version Differs From That of the SDK Which Contains AuthoringEditor

Info

With NX Add-On 9.5.0 or NX Add-On 10.2.0 and later, you can edit NSP files created using earlier versions of Nintendo SDK.

  • NSP files created by a SDK with a later major version than the SDK containing the AuthoringEditor cannot be edited.
  • Editing is not supported for NSP files created using NX Add-On 0.12.x series.
  • Patches must be created using the same NX Add-On version as the revised version of the application.
    For more information about patches, see Patches > Building and Running a Patch > Creating the Patch.

Capabilities

Creating and Editing NMETA Files

  • Entering and selecting items from the GUI tool is possible for each of the settings described in Writing the NMETA File.
  • A red X mark indicates locations where there are errors in the input content.
  • The NMETA file will not be exported if there is even a single error.

Checking NSP Files

  • You can open created NSP files in AuthoringEditor.
  • You can check the NSP file's ROM identification hash, the SDK and the middleware that was used, and the title information set in the NMETA file.
    • For more information about the ROM identification hash, see Identifying the Build Output.
    • If the file path is not the same as when the application was built, the ROM identification hash will be a different value. For more information, see Identifying the Build Output.

Editing NSP Files

  • Some of settings described in Writing the NMETA File can be changed, and the file with the changes can be saved as a separate NSP file.

Comparing NSP and NMETA Files

  • By passing the file paths to two NSP files or to two NMETA files, you can use the tool to check whether the files differ in composition.

Simulation of the Size of the User Account Save Data Area

  • Use the size simulator to calculate the number of required save data bytes from the number of directories, and number of files.
  • The number of save data bytes is calculated based on the Save Data Characteristics section in the Save Data page.
  • The size simulator calculation results include a margin of several bytes that is equivalent to two blocks.
