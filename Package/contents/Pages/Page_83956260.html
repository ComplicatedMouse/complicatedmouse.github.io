Animation Overview

Automatic Creation of Animation Curve Data

3ds Max key frames cannot be used because the method used by 3ds Max to calculate animation curves differs from the method used by NintendoWare.
When using NintendoWare, you must first bake animation curves before exporting intermediate files, and then convert animation data for use under NintendoWare.

For the rotation value, the animation curve is automatically generated after adjusting the baked data so that it is continuous and has a small absolute value.

All values in the output range are output for baked animations.
All animations with parameter curves outside the acceptable range are also baked, but are not included in the repeat type of the animation curve.

Loop Animation

When Loop is selected in the Export options, specify an end frame having the same conditions as the start frame.
For example, if animation is being looped in 3ds Max and frames 1 and 10 have the same conditions, specify 1 for the Start Frame and 10 for the End Frame.

Whether Loop is selected or cleared, the FrameSize in the intermediate file is specified by the EndFrame minus the StartFrame.

Values Exported to the Model File

If attributes are animated, the value specified for the start frame in the Export option is output to the model file.
However, the Translate, Rotate, and Scale values of nodes that affect skinning are exported in the skinning-configured state.
Note that these values are not the same as the values for the current frame in 3ds Max.
