Question

Animation plays back incorrectly in the viewer.

Answer

Part 1

The viewer (via the viewer library) manages animations in the 'layoutname_segmentname' format.

If any parts layouts have a name with this same format (layoutname_segmentname), the parts cannot be distinguished from the animation.

Example of Problem:

  • A body layout named Layout1 and a parts layout named Layout1_button1 have been created, and a segment named button1 has been set to Layout1.

Make sure that you name segments in a way that does not cause name collisions with parts layouts when the segment name is appended to a layout name.

Part 2

Pane names that satisfy certain conditions are sometimes erroneously associated with animations.


Conditions (this bug occurs when all of the following are satisfied):

  • The last two characters of the pane name are any of the following strings: "LT", "RT", "LB", "RB", "L ", "R ", "T ", "B ", or "C ".
  • When the last two characters of the pane name are removed, the resulting string matches the name of another pane.


Example of Problem:

  • A pane named Pane1 and a pane named Pane1LT have been created, and both are animated. In this case, the animation that is supposed to be bound to Pane1LT would also be played for Pane1.


To avoid this problem, simply revise your pane names so that they do not end in any of the specified strings. You can also work around this problem by naming your panes in a way that avoids any possibility of name collisions.

(This limitation is the result of a bug in the process that handles Window pane parts in LayoutConverter.)
