Introduction

Info

In NX Add-On 5.1.0, AlbumDownloader has been placed under the album subcommand of DevMenuCommand.
In addition, the packages that have been discontinued in NX Add-On 6.0.0 are not included, so migrate to DevMenuCommand.

AlbumDownloader is a target-oriented tool for operations on all of the files saved in albums on the development hardware. It can copy the files to a PC and also delete the files.

This tool can be used to do the following.

  • Copy all files saved in an album on the development hardware to a PC.
  • Delete all files saved in an album on the development hardware.

With this tool, you can use a PC to check the files that were captured by pressing the Capture Button, and the files that were saved by the application using the album Library.

Info

The files saved to albums on the development hardware cannot be individually copied or deleted.

Files cannot be copied from the PC to an album on the system.

Info

AlbumDownloader borrows some of the memory used by these features to run. While AlbumDownloader is running, minimize use of these features as much as possible (particularly the HOME Menu and DevMenu). Using these features while AlbumDownloader is running may result in fatal errors due to memory shortages. If this happens, reset and restart the system.

How to Use

Use RunOnTarget to start the tool.

$ Tools\CommandLineTools\RunOnTarget.exe --suppress-auto-kill TargetTools\NX-NXFP2-a64\AlbumDownloader\Release\AlbumDownloader.nsp -- <action> [<arguments>]


Displaying Help

To display the Help for AlbumDownloader, run the tool without any parameters or with --help specified. You can use the Help to determine the AlbumDownloader commands.

Command Examples

$ RunOnTarget.exe AlbumDownloader.nsp
$ RunOnTarget.exe AlbumDownloader.nsp -- --help

Copying All Album Files to a PC

Specify --download as the action to take all files saved to the album on the development hardware, and copy them to the specified directory on the PC.

You must specify both the album storage, that is the copy source, and the PC directory, that is the copy destination. The files will not be copied if the destination directory is not empty. Use the --force option to copy the files, even if the destination directory is not empty.

Parameter

Parameter Required/Optional Value Description
--storage Required

One of the following.

  • BuiltIn
  • SdCard

Specifies the album storage that is the copy source.

  • BuiltIn … System internal storage.
  • SdCard … SD card.
--directory Required

The absolute path for the directory on the PC.

  • Example: C:\Album\

Specifies the absolute path of the copy destination directory on the PC.

Cannot specify the root directory for the drive (for example, C:\).
--force Optional

Copies the files even if the copy destination directory is not empty.

If there are overlapping filenames, the content is overwritten.

Command Examples

$ RunOnTarget.exe --suppress-auto-kill AlbumDownloader.nsp -- --download --storage BuiltIn --directory C:/Album
$ RunOnTarget.exe --suppress-auto-kill AlbumDownloader.nsp -- --download --storage BuiltIn --directory C:/Album --force


Deleting All Album Files

Specify --clean as the action to delete all files saved to the album on the development hardware.

You must specify the storage with the target album.

Parameter

Parameter Required/Optional Value Description
--storage Required

One of the following.

  • BuiltIn
  • SdCard

Specifies the storage with the album whose files will be deleted.

  • BuiltIn … System internal storage.
  • SdCard … SD card.

Command Examples

$ RunOnTarget.exe --suppress-auto-kill AlbumDownloader.nsp -- --clean --storage SdCard
