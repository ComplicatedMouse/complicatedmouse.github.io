Network Identifiers

Network Identifiers

The network identifier (nn::ldn::NetworkId) is comprised of a nn::ldn::IntentId that indicates the network objective, and a nn::ldn::SessionId that is created for every session.

IntentId

nn::ldn::IntentId is a 128-bit identifier that indicates the network objective. It is comprised of a local communication identifier (LocalCommunicationId) and a scene identifier (SceneId). When you are scanning for networks in the vicinity, you can filter the scan results using nn::ldn::IntentId to exclude networks created by other applications and networks created by the same application but in a different scene.

The local communication identifier is used for identifying application titles and series. As a general rule, the application ID is used for the local communication identifier. However, you must use a shared local communication identifier for communication between applications with different application IDs, so in these cases it is all right to use one or the other application ID as the local communication identifier for both. All local communication identifiers used in an application must be registered in the application control data. If not registered, nn::ldn::CreateNetwork and nn::ldn::Connect() fail on a precondition violation. Eight local communication identifiers can be registered in the application control data. As long as seven or fewer are registered, if the application ID itself is not registered as a local communication identifier, it is automatically registered as one.

The scene identifier is used for distinguishing between different scenes in applications that include features such as online matches and data exchanges. It can be set freely by the developer.

As shown in the following example, create nn::ldn::IntentId using the nn::ldn::MakeIntentId() function.

const nn::Bit64 localCommunicationId = 0x01004B9000490000; // Application ID
const nn::Bit16 sceneId = 0;
nn::ldn::IntentId intentId = nn::ldn::MakeIntentId(localCommunicationId, sceneId);


SessionId

nn::ldn::SessionId is a 128-bit identifier that is randomly generated every time an access point creates a network, and that remains unchanged until that network is destroyed.
