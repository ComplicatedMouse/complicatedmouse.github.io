Restarting the Program

The application program can restart automatically when calling nn::oe::RestartProgram(). Restarting the program is expected in the following situations.

  • You want to use the user account selection and language/audio selections that are made at initial program startup in place of the resource destruction and reinitialize processing that occurs when the user account or language/audio selections are changed in the game.
  • You want a quick method for implementing the reset processing for each sub-program in a multi-program application.

However, a program is restarted by first closing the program and then restarting the same program, so it takes the same amount of time as starting the program from scratch. You must avoid unnatural user experiences before and after switching between programs. For more information, see Switching Between Programs.
