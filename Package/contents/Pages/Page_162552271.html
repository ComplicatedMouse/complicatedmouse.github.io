Troubleshooting

Spy.exe Cannot Connect to Application

  • The Spy.exe connection settings may not match those of the application you want to connect to.
    • In the menu, from the connection target settings, specify the platform on which the application is running.

Pressing the Connect Button Does Not Enable the Connection

  • The application can connect to only one instance of Spy.exe. If multiple instances are running and one instance is connected, the Connect button will be disabled for the rest.
    • Disconnect the connected instance of Spy.exe.

I cannot connect the release build to Spy.exe.

Because Spy is a development feature, it is disabled in release builds to prevent it from accidentally being included in retail products.

Use the following procedure to use Spy in release builds.

  1. Revise Include\nn\spy\spy_Config.h as follows.

    #if defined(NN_SDK_BUILD_LIBRARY) \
    || defined(NN_SDK_BUILD_DEBUG) \
    || defined(NN_SDK_BUILD_DEVELOP) \
    || defined(NN_SDK_BUILD_RELEASE) /* enable in release builds */
#define NN_ENABLE_SPY
#endif
    Info

    For safety reasons, you cannot enable Spy with just a macro definition in a user project.

  2. Define the NN_ENABLE_HTC macro using a user project.
  3. Rebuild the user project.
