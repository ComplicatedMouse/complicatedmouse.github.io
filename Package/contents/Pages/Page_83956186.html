Basic Features of the Export Plug-In

To export only specific nodes using the export plug-in, select the nodes to export in Maya.
Even nodes below the selected node in the hierarchy are exported.

Info

When you export the entire scene, it is not necessary to select nodes.

Select Nintendo > Export with Setting. The Export Options window appears.
To export an intermediate file, specify the appropriate options and click Export or Apply.
Unlike the Export button, Apply does not close the options window after export.

Maya internally records the option settings used at the time of export as its current options.
You can also save the option settings to a scene and the import/export option settings using 3D DCC Plug-In Export Settings File (FDES file) . (For more information, see Managing Option Settings.)

To export the file using the current options without opening the options window, select Nintendo > Export.

To save option settings within Maya, without exporting intermediate files, select Edit > Save Settings in the upper-left menu of the options window. (Settings are also saved in the scene if Save / Load Scene Settings is selected.)
To restore option settings to their default values, select Edit > Reset Settings.

Edit Locking

If you select Lock from the shortcut menu that appears when you right-click any of the options listed in the options window, the ability to edit that option is locked (edit lock) and a lock icon is displayed.
The edit lock can be released by right-clicking the lock icon and selecting Unlock from the shortcut menu that appears.
For some options, check boxes and text boxes can be separately locked against editing.

The edit locked status is saved in scene and FDES files.
To lock or unlock all options at the same time, in the options window, on the Nintendo Settings menu, select Lock All Settings or Unlock All Settings.

Option Settings

The options set in the export plug-in are as follows.

Preset Options These options are the preset options.
Output Options Options that relate to exporting intermediate files.
General Options General intermediate files options.
Animation Options These options relate to animation.
Output File Selection Select this option to specify the type of intermediate file to export.
Optimization Options You can optimize data by using the intermediate file optimizer when you export intermediate files from Maya. Use this setting to configure options for this optimization process.
Animation Bake Tolerance Options Select this option to specify the tolerated margin for error when animation is baked and optimized.
Animation Quantization Tolerance Options Specifies the allowable range of errors when an animation is optimized.
Script Options Specifies the scripts (MEL) to execute before and after export.

Preset Options

These options are the preset options.

The term preset refers to a feature where a preset file (an FDES file set to edit locked status) is used to set some or all option settings to a particular fixed value.

This option can be used for the following purposes.

  • To prevent option settings determined by the design lead from being carelessly changed by another designer.
  • To prepare presets for characters and presets for geography, and to easily change between the option settings.
Info

The preset file is usually located in the Tools\Graphics\MayaPlugins\Config\Presets folder.
If you specified the folder containing the config file for the NINTENDO_MAYA_CONFIG environment variable, the preset file is in the %NINTENDO_MAYA_CONFIG%\Presets folder.

You can also directly specify a folder for the preset file by specifying a path for the NINTENDO_MAYA_PRESETS environment variable. (The preset file is placed here rather than in the directory set for NINTENDO_MAYA_CONFIG.)
Select Open Presets Folder in the Nintendo Settings menu of the options window to open the Presets folder in Windows Explorer. (This selection simultaneously closes the options window.)

Presets are evaluated when export is executed. If you are using the same presets for more than one scene, you can change the export options for all scenes by changing the contents of the preset file.

Options under Preset Options cannot be locked.

  • Preset
    Specify the name of the preset to use.
    The names of the FDES files located in the Presets folder are listed as the names of the presets in a menu.
    If you are not using presets, select None. If you are using presets, select a preset name.
    When the preset is switched, the options that are in an edit lock state in the preset file are overwritten with the preset value.
    Options that are not in an edit lock state in the preset file are not changed.
    If a preset has been selected, you cannot release edit locking of an option that has been locked in the preset file. In addition, you cannot lock an option that is not edit locked.
    If you click Save Preset without a preset being used, you can export the current setting to the Presets folder as a preset. To create a new preset, select New and enter a preset name. To overwrite an existing preset, select Overwrite and select a preset name.
    When you click Save, a list of preset files appears and the output preset is selected for use.
    If you click Delete Preset while using a preset, the preset file for the preset being used is deleted and the preset is put in unused status.
    Perform the following steps to edit an existing preset value.
    1. Select the preset you want to edit in the Preset options.
    2. Change the Preset option to None.
    3. Change the various options or the edit locked state.
    4. Click the Save Preset button to overwrite the preset settings.
  • Show Preset UI
    When using presets, select this setting to display the options that are locked in the preset file.
    Only a UI for options that can be edited is displayed if this option is cleared.

Output Options

Options that relate to exporting intermediate files.

  • Export Target
    Specify the export target.
    Select All to export all active nodes in the scene.
    Select Selection to export only the selected nodes and the nodes below them in the hierarchy.
  • LOD Export
    Enable this option to export an intermediate file with a merged LOD model. For more information, see LOD Export.

  • Output File Name
    Specify the name of the intermediate file. (Omit the filename extension.)

    Info

    Non-ASCII characters and symbols other than the hyphen (-), underscore (_), and period (.) cannot be used.
    If left blank, the current scene name is used for the intermediate filename.

    When @node is entered, the selected node's name is used in the intermediate filename. If multiple selections are made, the top node's name in the hierarchy is used.
    Click Scene on the right to set the current scene name.
    Click Node to set the name of the selected node, or the top node in the hierarchy if more than one node is selected.
    Click Node without selecting a node to set the name of the top node in alphabetical order.

  • Output Folder
    Specify the output folder for intermediate files.
    Texture files are exported to the textures folder located inside the folder specified using Output Folder.
    If the folder specified for Output Folder does not exist, that folder is created and exported.
    If a full path is not specified for the output folder, a relative path from the current project folder is used.
    Environment variables can be used for Output Folder. The environmental variable is written as %NAME% or $NAME.
  • Merge fmd File
    Select this option and specify an FMD file path to merge the specified FMD files.
    If the FMD file path is left blank, the export file to overwrite with is merged. In this case, no error occurs even if there are no output files. (You can specify how to display warning messages in config files).
    If the specified FMD file path is not a full file path, it is treated as a relative path from the current project folder.
    You can use environment variables in the Merge fmd File path.
    The ability to enable or disable this option, and the FMD file path, can each be separately locked against editing (in NintendoSDK 10.0.0 and later.)
    The intermediate file optimizer executes the process for merging FMD files. For more information about this merge process, see the Help for the intermediate file optimizer.
  • Merge ftx File
    Select this option to merge FTX files.
    If you are also merging FMD files, files are merged using the same filename located in the textures folder inside the file folder specified under Merge FMD File.
    If an FMD file is not being merged, files are merged using the same filename located in the textures folder inside the folder specified by Output Folder.
    Also, if an FMD file is not being exported, files are merged using the same filename located in the textures folder inside the folder specified by Output Folder.
    Note that if the FMD file and the FTX file are not merged at the same time, the mipmap filter of the FMD file may not correspond to the existence of mipmaps in the FTX file.
    If the FMD file is not merged, the mipmap settings of the FTX file to merge affect the mipmap filter of the FMD file.
    The 3DTextureConverter executes the process of merging FTX files. For more information about the merge process, see the Help for the 3DTextureConverter.
  • Merge Animation Folder
    Select this option and specify a folder containing animation intermediate files to merge the specified animation intermediate files.
    If the folder specification is left blank, animation intermediate files located in the folder specified by Output Folder are merged. In this case, no error occurs even if there are no animation intermediate files in the folder specified by Output Folder. (You can specify how to display warning messages in config files).
    Environment variables can be used for Merge Animation Folder.
    In Merge Animation Name, specify the name (excluding the filename extension) of the animation intermediate file to merge.
    If Merge Animation Name is left blank, the file with the name specified by Output File Name is merged.
    The intermediate file optimizer executes the process for merging animation intermediate files. For more information about this merge process, see Help for the intermediate file optimizer.

General Options

General intermediate files options.

  • Magnify
    Specify the magnification ratio for the model's vertex coordinates and translate value.
  • Texture SRT Mode
    Specify the calculation method for texture SRT.
    The method of calculating the texture SRT affects FMD, FMA, and FTS files.
    Selecting Maya results in the selection of the Maya calculation method. Under Maya, the scale center is the lower left, the rotation center is the center, and the positive rotation direction is clockwise.
    When 3ds Max is selected, the 3ds Max calculation method is used. Under 3ds Max, the center for both scaling and rotation is the center and the positive rotation direction is clockwise.
    Selecting Softimage specifies the calculation method used by Softimage. Under Softimage, the center for both scaling and rotation is the lower left, and the positive rotation direction is counterclockwise.
    Even if the 3ds Max or Softimage method is specified, it is not applied on the Maya screen. (The texture display shown in Maya may differ from the display shown on the target platform.)

  • Remove Namespace
    Select this option to remove the namespace part of node names and material names, and then export data.
    If this option is cleared, the namespace part is exported. Each colon (:) is converted to an underscore (_).

    Info

    Example
    If the node name under Maya is model:final:head and this option is selected, data is exported using the name head; however, if this option is not selected, data is exported using the name model_final_head.

    If duplicate names occur as a result of removing the namespace part, names are changed according to the same rule used when more than one node (or material) has the same name.

  • Comment
    Include comments about editing the intermediate file.

Model Options

Options for the model.

  • Reset Root Transform
    Specifies the translation, rotation, and scale reset for the root bone of the FMD file. (This feature is in NintendoSDK 11.1.0 and later.)
    Use the feature when you want to output models in Maya placed in locations other than the origin in the same manner as if they were placed at the origin (without rotation or scaling).
    For output, when Translate is enabled, translation is reset to 0; when Rotate is enabled, rotation is reset to 0; when Scale is enabled, scaling is reset to 1.
    If there are multiple root nodes in Maya, and nw4f_root is added, the various bones corresponding to the root node in Maya are not reset.
    Shapes for which skinning and skinning influence are configured are exported in the bind pose without being reset.
    The root bone prior to bone compression is reset. After bone compression, translate and rotate take nonzero values for the root bone (and the scale takes a value other than 1).

Animation Options

These options relate to animation.

  • Start / End Frame
    Specify the range (start frame and end frame) of the animation.
    Select All to export the full range of animation on Maya (Animation Start / End).
    Select Playback to export the playback range of animation in Maya (Playback Start / End).
    Select Range to specify the start frame and end frame by numeric value.
    If attributes are animated, the value specified for the position of the start frame is exported to the FMD file.

    Info

    Make sure that you specify the same mode for the end frame of a loop animation as for the start frame.

    Info

    Animation playback is not assured for the frame before the start frame and the frame after the end frame.

  • Loop
    Select this option to export the data as looped playback data.
    If this option is cleared, the animation range is exported as one-time playback data
  • Bake All Animation
    If this option is selected, animation curve data is created automatically from baked data and exported for all animations.
    If this option is cleared, the animation curve in Maya is exported ‘as is’ whenever possible. Parts that cannot be exported as is are automatically created and exported from baked data.

  • Frame Precision
    Specify the frame precision when animation curve data is automatically created from baked data.
    If 1.0 is set, data is baked at the integer frame level.
    If 0.5 is set, data is baked at the 0.5 fractional frame level.
    If 0.2 is set, data is baked at the 0.2 fractional frame level.
    If 0.1 is set, data is baked at the 0.1 fractional frame level.

    Info

    Increasing precision can more accurately reproduce animations that vary discontinuously at a fractional frame timing, but the amount of time required for output increases.

  • Motion Mirroring
    Specify ON to use motion mirroring, and OFF to not use it.
    CPU load when playing animations will be higher when set to ON than OFF.
    This option is applied to the FMD file. The option is invalid when not exporting an FMD file.

Output File Selection

Select this option to specify the type of intermediate file to export.

Clicking the Search Model Animation button turns on only the intermediate files associated with animations that are set for the model being exported.
If an animation curve is then connected, it is treated as if an animation is set, even if its value does not change.

If the Export Target is Selection, select the desired nodes and then click Search Model Animation.

Info

When exporting all material animations to a single FMA file, the only animations that will be included in that file are those that are enabled.

Info

Because scene animations are not searched for, make sure that you manually select on or off.

Optimization Options

You can optimize data by using the Intermediate File Optimizer when you export intermediate files from Maya. Use this setting to configure options for this optimization process.

  • Compress Bone
    Specify the bone compression mode to use for the entire model.
    To reduce the load due to the matrix computations and the amount of memory used, lower the number of bones.

    Info

    Bones exported to the intermediate file are compressed, but data saved in the DCC tool is not changed.

    The following files support compression: FMD, FSK, and FVB files.

    Mode Description
    None Do not compress bones. Exports the hierarchical structure of the DCC tool as is.
    Cull

    Deletes the bones that are not required to display the model located at the end of the hierarchical structure, and then exports.

    This mode is useful for models that use skinning.

    Bones for which the no-compress flag has been set are not deleted.
    Merge

    In addition to the processing performed when Cull is specified, bones with matrices that can be merged are merged and exported.

    Generally, this reduces the number of bones even more than using Cull, but the following restriction applies: a non-uniform scale (where X, Y, and Z are not the same) must be set for bones that have children.

    If bones that do not follow this restriction or children of such a bone are compressed, the model does not display correctly.

    All frames must observe this restriction when the scale is being animated.

    Bones for which the no-compress flag has been set are not deleted.
    Unite Bones for which the compress disable flag has been set are collected together into one. This feature cannot be used in skinning models. Skinning models are converted to rigid bodies.
    Unite All

    Collects all bones together into one regardless of the compress disable flag.

    The root bone matrix is represented by a unit matrix. Model coordinate system values are used for the vertex coordinates, normal vector, tangent vector, and binormal vector of all shapes.

    This option is used for single objects and for topographical data that does not require a hierarchical structure.

    The following files support compression: FMD, FSK, and FVB files.

  • Unite Child
    If this option is selected, nodes that have shapes located at the end of hierarchical structure are gathered together into one node and exported. Compress Node is useful at times when Unite is not specified.
    All child bones of a bone for which the compress disable flag has been set are compressible and are compressed if skinning is not affected. If the compress disable flag has not been set for any node, this option does nothing.

  • Compress Material
    Select this option to compress materials that have the same attributes and gather them into one material for export.
    Materials for which Compress has been set to Don’t compress, using the plug-in for setting material attributes, are not compressed.

  • Compress Shape
    If there is more than one shape (<shape> element in an FMD file) having the same bones and materials, selecting this option gathers the shapes and exports them together.
    If this option is not selected, shapes are exported individually even if they have the same bones and materials.
    This option is effective when Compress Bone is set to a value other than None or Unite Child, and more than one material associated with a particular bone has the same shape after bones have been grouped together.
    This option is also effective when Compress Material is selected and multiple shapes with the same material exist on a single bone after bones have been grouped together.

  • Divide Submesh
    Specify whether to enable submesh division.
    To perform submesh division, select this option and then enter the value for the --divide-submesh-options intermediate file optimizer option on the right. If this option is left blank, submeshes are not split.

    Info

    Example: --max-bounding-box-width=1.0

  • Polygon Reduction
    Specify whether to enable polygon reduction.
    To perform polygon reduction, select this feature and then enter the value for the --polygon-reduction-options intermediate file optimizer option to the right.
    However, if you do not enter a value after the option flag, polygon reduction is not performed.
    If you enable this option while using LOD export mode, you can overwrite the LOD model created with polygon reduction with a Maya LOD model or add the Maya LOD model in addition to this LOD model at a later time.
    For the --optimize-position-policy option, specify a value other than 1, so vertices are not shared.

    Info

    Example: --target-rate-lod-level1=0.5 --target-rate-lod-level2=0.2

Animation Bake Tolerance Options

Select this option to specify the tolerated margin for error when animation is baked and optimized.

This setting specifies the maximum allowable difference between the start frame value and the value used for any other frame. If the difference remains less than this maximum, the animation is assumed to be fixed and is exported by using only the value for the start frame.

When animation curve data is created automatically from baked data, a key is created for each frame (in the units specified by Frame Precision), and the key for the below tolerance value is deleted by the maximum differential value when performing Hermite interpolation.

Info

The smaller the allowable difference, the greater the animation precision. However, as the number of keys grows larger, the amount of animation curve data also increases.

  • Node Translate, Node Rotate, Node Scale
    Specify the tolerance as an absolute value for the translation, rotation, and scaling of skeletal animations and for the translation, rotation (direction) of scene animations.
    Node Rotate is specified in units of degrees.
    Node Translate is the tolerance before the value is multiplied by Magnify.
  • Texture Translate, Texture Rotate, Texture Scale
    Specify the tolerance as an absolute value for the translation, rotation, and scaling of texture SRT animations.
    Texture Rotate is specified in units of degrees.
  • Color
    Specify the allowable error for the translation, rotation, and scaling of color animations and scene animations as an absolute value.

Animation Quantization Tolerance Options

Specifies the allowable range of errors when an animation is optimized.

  • Node Translate, Node Rotate, Node Scale
    Specify the allowable error for the translation, rotation, and scaling of skeletal animations and the translation and rotation (direction) of scene animations.
    For Node Translate and Node Scale, this setting is specified as a percent of the range of values of animation results.
    For Node Rotate, this setting is specified as an absolute value in units of degrees.
  • Texture Translate, Texture Rotate, Texture Scale
    Specify the allowable error for the translation, rotation, and scaling of texture SRT animations.
    For Texture Translate and Texture Scale, this setting is specified as a percent of the range of values of animation results.
    For Texture Rotate, this setting is specified as an absolute value in units of degrees.
Info

This option is not applied if an animation intermediate file is merged. The value of the intermediate file to merge is used.

Script Options

Specifies the scripts (MEL) to execute before and after export.

  • Pre-Export Script
    Specify the MEL file to execute before exporting. (MEL commands can be entered directly.)
    No script is executed if this option is left blank.
    If the Export Target option is set to Selection, you can use this option to specify the nodes to export by using the following syntax: select root_node.
    You can also look at the value of the NINTENDO_EXPORT_IS_MULTI_EXPORT environment variable and select nodes if they are not currently being exported by the multi-export plug-in.
    If @node is specified for the output filename, the node name selected using this option is used.
    If an environment variable is used with the Output Folder, Merge fmd File, or Merge Animation Folder options, this option applies the value in the environment variable.
    Call the NintendoSetOptimizerExtraOptionFmd function to specify additional intermediate file optimizer options for the FMD file.
    Call the NintendoOverwriteExportOption procedure to overwrite the export options settings and some of the configuration settings that are being exported.
  • Post-Export Script
    Specify the MEL file to execute after exporting. (MEL commands can be entered directly.)
    No script is executed if this option is left blank.
    The specified MEL is executed even if an error occurs during export. You can determine whether export is successful with the NINTENDO_EXPORT_SUCCEEDED environment variable, described later.
    If merging intermediate files is enabled, the specified MEL file is executed.
    Use the NintendoGetExportedTransformPaths procedure to get the transform node group exported as <bone>.

    You can use the getenv command to get information from the following environment variables within the MEL file specified as a Pre-Export or Post-Export Script.

To temporarily disable a script, comment it out by adding two slashes (//) to the beginning of each line of script.

//parent -w pCube1

The source command is used to execute MEL files.

source "D:/data/post_export1.mel"

Python scripts can be executed using the python command of MEL.

python "import maya.cmds; maya.cmds.move(1, 0, 0, r=1)"

The execfile function is used to execute Python files.

python "execfile(\"D:/data/post_export1.py\")"

MEL files can be executed as Windows batch files using the system command.

// When not displaying a command prompt.
system "D:/data/post_export1.bat"

// When displaying a command prompt.
system "start D:/data/post_export1.bat"

// When echoing a text string in the batch file to the script editor.
print `system "D:/data/post_export1.bat"`

You can also use the preceding environment variables in Windows batch files.

This example uses the full path of each FTX file in a REM batch file.

for %%I in (%NINTENDO_EXPORT_FTX_FILE_NAMES%) do (
    echo %NINTENDO_EXPORT_FTX_FOLDER%%%I
)

Managing Option Settings

Use the item in the option window's Nintendo Settings menu to save option settings by scene or input/output option settings using an FDES file.

Saving Option Settings for a Scene

If Save/Load Scene Settings is selected (this is the default), the option settings used the last time an intermediate file was output are saved for the scene data when a scene is saved. (Internally, these option settings are saved to the script node named nw4fExpDialog_Setting1.)
When the scene is opened, the option settings that were saved in the scene data are loaded as the current settings.

The current option settings remain if option settings were not saved with the scene data.

Even when Edit > Save Settings is selected in the upper-left menu of the options window , the option settings are updated in the same manner as when intermediate files are output.

If you clear Save / Load Scene Settings, no option settings are saved as scene data.
Option settings are not loaded when scene data is opened.

Selecting Delete Scene Settings deletes the script node used to save option settings.

Selecting Revert to Original Scene Settings restores the options settings of the current scene to what were saved when the scene was first opened.

Input/Output Option Settings Using FDES Files

Select Save Settings to fdes File to output option settings to an FDES file. Option settings are exported when an output filename is specified in the file selection window.
If the extension of the output file is something other than .fdes, .fdes is added automatically.

Select Load Settings from fdes File to enter option settings from an FDES file.
Option settings are entered when an input filename is specified in the file selection window. Option settings are saved in the scene if Save / Load Scene Settings is selected on the Nintendo Settings menu.

Config File

You can customize the operations of the export plug-in by editing the Export.ini config file located in the Tools\Graphics\MayaPlugins\Config folder.

If there is no Export.ini file when the export is started, the Export_Sample.ini file in the same folder is copied to Export.ini and loaded.

If you want to use a config file in another folder, specify the path of that folder for the NINTENDO_MAYA_CONFIG environment variable before using the plug-in.

The lines in the config file describe keywords that correspond to customizable items. The values for those keywords and descriptions of the settings they configure are as follows.

Keyword = "Value"

Lines that begin with the pound symbol (#) are comments and are ignored.

The following list describes the various keywords.
Defaults are used if there are no keywords.

  • display_profile
    Specify true to display the processing time for each process and the intermediate file optimizer commands and messages in the script editor at export time. Specify false to not display this information. The default is false.
  • make_anim_curve_parallel
    Specify true to use parallel evaluation during the animation curve creation process at export, and specify false to not use parallel evaluation. The default is true.
    Set to false to keep CPU load and memory usage low during export.
  • project_root
    Specify the root directory of the application's project file. (This project is not a Maya project.)
    If specified, the relative path from the root folder is exported to src_path in <create> in the intermediate file. If you specify an empty string, the absolute path is exported. You can also include environment variables enclosed in percent signs (%).
  • max_vertex_skinning_count
    Specify an integer value in the range from 1 to 8 for the maximum number of nodes that skinning weights can be distributed to.
    An error results if the weight is distributed over more nodes than the specified value.
    However, if 8 has been specified, weights from highest to lowest are distributed up to eight nodes without an error, and occurs even if distributing weights to more than eight nodes.
    The default value is 8.
  • skinning_weight_precision
    Specify the precision of the skinning weight. When set to 0.01, snaps to a multiple of 0.01 and is exported. When set to float, exported with the precision of a single-precision floating-point number. The default value is 0.01.
  • adjust_smooth_skinning
    Specify true to determine the smooth skinning of shapes in the intermediate file in terms of units of nodes in Maya, or false to determine in units of shapes (groups of polygons with the same material). The default is true.
    If you notice missing dots caused by a calculation error between smooth skinning and rigid skinning, specify true.
  • compress_ignoring_vertex_skinning_count
    Used when the shape compression option is enabled. Specify true to compress shapes that have different numbers of skinning nodes that affect a single vertex, or specify false to not compress the shapes. The default is false.
    For more information, see Help for the intermediate file optimizer.
  • disable_merge_different_skinning_type_shapes
    If you have set compress_ignoring_vertex_skinning_count to true and selected the Compress Shape option, set this keyword to true to disable the merging of rigid skinning and smooth skinning shapes, or set it to false to enable this merging. (This feature is available in NintendoSDK 12.1.0 and later.) The default is false.
  • optimize_triangulation
    Specifies the optimization mode when converting polygons with 4 or more corners into triangles. (This feature is available in NintendoSDK 11.3.0 and later.)
    When subdividing the polygons for the start frame (or in the bind pose if you are using skinning), specify angle if you want to make the smallest angle of the triangle as large as possible. Specify dcc if you want to divide it according to the same pattern as Maya. The default is angle.
    If you specify angle or use a deformer, there are restrictions on the shape of the polygons that can be output correctly.

  • optimize_primitive
    Specify the primitive optimization mode.

    Mode Description
    none Do not optimize primitives.
    default Use default primitive optimization.
    force Optimize primitives over time.
    brute_force Optimize primitives over an even longer time.
    forsyth Optimize primitives using the Forsyth algorithm.

    Forsyth is the default primitive optimization algorithm.

  • quantization_analysis
    Specify true to enable quantization analysis, or false otherwise. The default is true.
  • delete_near_vertex
    Specify the deletion of near vertices. Specify the value for the intermediate file optimizer option --delete-near-vertex-options. If left blank, near vertices will not be deleted. It is blank by default.
  • export_material_array
    Specify true if you are exporting the <material_array> element to the FMD file, and false otherwise. (This feature is available in NintendoSDK 10.4.0 and later.) The default is true.
    Specify false if you are referencing an existing material intermediate (FMT) file.
  • material_attr_info_node type
    Specify the extended material attribute information. (This feature is available in NintendoSDK 10.4.0 and later.)
  • disable_original_image
    Specify true to omit the pre-encoded source image data from the FTX file. Specify false to include the data. The default is false.
  • texture_hint
    Specify the gamma linear conversion for each texture hint.
    To convert gamma from sRGB to linear, specify 1 for each RGBA component, in that order. To not convert a particular color component, specify 0 for that component.
    For example, specifying 1110 converts the R, G, and B components, but not the A component.
    The default is 1110 when the hint is albedo, emission, specular, or reflection; otherwise it is 0000 (no component is converted).
  • use_srgb_fetch
    If the gamma linear conversion specification for a texture is 1110 or 1111 and it is not a grayscale image, specify true to use srgb_* format, or false to use unorm_* format. The default is true.
  • normal_texture_format
    Specify the texture format to use for normal maps. You can specify formats supported by the 3DTextureConverter (other than bc6 and float_*. The default is snorm_bc5.
    You can also specify your component selection as "format, component selection". If you do not specify a component selection the component selection supported by the format is applied.
  • enable_weighted_compress
    Specify true to enable perceptual weighting for errors in RGB when compressing textures in bc1, bc2, bc3, etc1, and etc2. Specify false to disable perceptual weighting. The default is false.
  • gpu_encoding
    If the texture is a format supported by GPU encoding, specify true to encode with the GPU and false to encode with the CPU. Specify auto to encode with the GPU if possible and encode with the CPU otherwise. The default is true.
  • texture_converter_options
    Use this to specify additional options to pass to the 3D Texture Converter tool when exporting FTX files. (This feature is available in NintendoSDK 12.1.0 and later.)
    Use single quotation marks when enclosing option values in quotation marks. You cannot specify options like --job-list, --help, and --silent. You can use environment variables enclosed in percent signs (%).
    Specify the options in the following kind of way: --format-with-a srgb_bc7 --comment 'string with space'
    The options specified using texture_converter_options take precedence over the options set in options windows and other config settings.
  • filter_mip
    Specify point or linear for the mipmap filter when texture mipmapping is enabled. The default is linear.
  • sampler_merge_priority
    Specify true to prioritize the Maya sampler information when merging FMD files. Specify false to prioritize the sampler information in the merge file. The default is false.
    For more information, see the intermediate file optimizer documentation.
  • texture_srt_merge_priority
    When merging FMD files where there is a <shader_param> of type="texsrt", specify true to prioritize the Maya side texture SRT when merging, and false to prioritize the merge file texture SRT. (This feature is available in NintendoSDK 11.3.0 and later.) The default is false.
    For more information, see the intermediate file optimizer documentation.
  • material_assign_merge_nonpriority
    When merging FMD files, set true to prioritize the shape material assignments of the merged file, and false to prioritize the material assignments in Maya. The default is false.

  • use_fcl_fts_ftp

    This keyword is ignored when use_fcl_fts_ftp is true.
  • bone_visibility_merge_priority

  • export_bone_visibility_all
