Added a feature for displaying a warning message when no material animations have been bound to the output model.

In the Maya plug-in, added a feature for displaying a warning message when FMA file output is specified and no material animations have been bound to the output model.

Specify the following option in the config file to enable this warning message. 

warning_no_material_anim="true"


When the following option is used, a warning message is displayed regardless of the specified animation types.
(FMA files are combined into a single file. A warning message is displayed when none of the material animations are present.)

separate_material_anim="false"


When the following option is used, a warning message is displayed unless at least one of the specified animation types is present.
(FMA files are separated into different files based on animation type (such as color, texture SRT, or texture pattern). A warning message is displayed if one of those files does not have a material animation.)


separate_material_anim="true"

3D Command-Line Tools

The shader variation generator now supports inputs from multiple shader definition FSD files.

You can now specify a directory using the --shader-variation option for the 3D shader variation generator.
When a directory is specified, all of the FSDB shader definition files in that directory are taken as input.
When multiple shader definition files are specified in this way, multiple FSVA shader variation files corresponding to the shader definition files are generated based on the assignment information contained in the input FMD and FMA intermediate model files.

Related Links For more information, see the following pages.

Shader Variation Generator Describes the behavior of the shader variation generator.

In Vertex Attribute Packer, if some vertex attributes are missing in the source data to be packed, some data is no longer filled with zeros in the packed data.

Previously, when packing vertex attributes in 3D Vertex Attribute Packer, if some of the vertex attributes were missing in the model to be packed, the missing stream data was filled with zeros in the packed data. Starting this version, this missing data is no longer filled with zeros.

For example, when packing vec2 and vec2, if the latter vec2 did not exist, previously the former vec2 and a zero-filled instance of the latter vec2 filled with zeros would be packed as vec4 vertex attributes. Starting in this version, the former vec2 is simply exported as a packed vertex attribute as is.

Changed the merging behavior for motion_mirroring_enable value in the Intermediate File Optimizer.

Changed the merging behavior for motion_mirroring_enable in <skeleton_info> when merging FMDA and FMDB intermediate model files using the Intermediate File Optimizer.
Previously, the motion_mirroring_enable value in the merge file specified with the --merge-file option was used. Starting in this version, the value specified in the parameter file is used.

For example, when using 3dIntermediateFileOptimizer.exe --merge src.fmda --merge-options" --merge -file dest.fmda "-o out.fmda, the src.fmda values are used by motion_mirroring_enable for out.fmda.
