Fonts in Special Bordered Format

This section describes the Fonts in Special Bordered Format feature, which adds borders to the outlines of characters for bitmap fonts (in Packed4 format).

Overview

The Fonts in Special Bordered Format feature adds borders to the outlines of characters for bitmap fonts (in Packed4 format).
You can specify whether to use borders for individual font files.
This takes up less memory than preparing separate font data.

The various settings for this feature are described below, together with comparative images.

Normal The glyph and border are drawn for each character.
If the characters overlap, the border for the first character may get covered by the border of the second character.

Remove border The character border is not drawn.

Render in two cycles The borders for the characters in the string are drawn first, after which the glyphs are drawn.
Therefore, it differs from Normal, and prevents the overlapping of glyphs and borders.

Info

Render in two cycles has a heavier processing load than Normal, so use Normal if the overlapping of glyphs and borders is not an issue.
