System Classifications

Every function is automatically classified as to what system it belongs to (Physics, Graphics, Audio, etc). This is primarily accomplished with string matching using commonly accepted terms for each system (for example, "broadphase" for Physics or "matrix" for Math). It is fully expected that some functions will be classified incorrectly, but since it is mostly accurate, it is still immensely helpful in aiding understanding.

Selected functions will be color coded to their system classification when selected. This is reflected in the Functions tab, Call Tree tab, and Timeline tab. The key for what each color means is displayed above each core in the Timeline tab.

Functions can be filtered by their system classification by using the System Classification drop down box on the Functions tab toolbar.

Overriding Classifications

Function classifications can be overridden within the Functions tab. Right click on a function and select a new classification in the Override Classification submenu.

Disable Classification Coloring

In the Options window under the General section, there is an option for an alternate function coloring scheme not based on classification. This might be helpful if the classifications colors are distracting or incorrect.
