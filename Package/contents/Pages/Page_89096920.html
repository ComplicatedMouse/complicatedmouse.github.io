mem Library

Overview

The mem library provides memory allocation features for the allocation of memory space.

Purpose of This Documentation

This documentation describes the memory allocator library included in NintendoSDK.

Intended Audience

This documentation is intended for people developing applications using NintendoSDK.

Related Documentation

Use this document in conjunction with the nn::lmem:: namespace API reference, and the nn::mem:: namespace API reference.

nn::lmem:: and nn::mem:: Namespaces

The mem library uses the nn::lmem:: and nn::mem:: namespaces.

The nn::lmem:: namespace provides memory management features that are focused on providing compatibility with the memory management features of existing platforms.

The nn::mem:: namespace provides higher efficiency memory allocation and deallocation features that were not provided with existing platforms.

We recommend using the features in the nn::mem:: namespace, with some exceptions.

Features Provided by the mem Library

The mem library provides memory allocators that allocate memory from a preallocated memory space called a memory pool.

Memory space allocated from the memory pool is called a memory block. You can use the memory space efficiently by using a memory allocator to manage the memory blocks.

The following four types of memory allocators are provided.

  • nn::mem:: namespace
    • StandardAllocator
  • nn::lmem:: namespace
    • Expanded heap (ExpHeap)
    • Frame heap (FrameHeap)
    • Unit heap (UnitHeap)

Other integer allocators are available besides those for memory allocators.

The following integer allocator type is an example.

  • nn::mem:: namespace
    • NumberLineAllocator


For more information about each feature, see the manual.

About malloc() and Other Functions That Get Memory

Functions that get memory, such as malloc(), use the nn::mem internal namespace functionality.

For more information, see init Library and About malloc() and Other Functions That Get Memory.
