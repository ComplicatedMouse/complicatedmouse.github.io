Game Crash Report

When the profiler is Attached to an application to allow for Out-of-Process profiling, it will intercept crashes that occur within that application. The Game Crash Report window will appear when this happens, giving information similar to what is also output to Target Manager's log window. This includes:

  • Crash type
  • Current thread details
  • Register values
  • The callstack

Click the Copy button to copy the report to the clipboard.

Once a crash is detected, the profiler GUI enters an error state and profiling is disabled. To re-enable profiling, please Unsync, restart the application, and Sync again.

Debugging

While the profiler is Attached to the application, it is not possible for the debugger to also attach. After closing the Game Crash Report, Unsync the profiler. In the debugger, it is then possible to Attach to the crashed process to start debugging.

For example, in Visual Studio 2015, this would be done by with the following steps:

  1. From the menu, select 'Debug' → 'Attach to Process'.
  2. For 'Transport' select 'NX Devices'.
  3. For 'Qualifier' select the name of the dev kit.
  4. Select the process from the list of 'Available Processes'.
  5. Click 'Attach'.
