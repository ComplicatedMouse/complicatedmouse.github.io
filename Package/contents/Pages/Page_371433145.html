NX Microphone Input Features

Overview

This page describes the NX-specific issues when using the microphone input features provided by NX.
The NX microphone input features are provided through the nn::audio::AudioIn API. For more information about the API specifications, see the API reference.
For specific examples, see the AudioEchoback sample in the SDK.
In addition, when using these features in an actual application, also see the Important Information About Using the Microphone Feature page.

Behavior When the Microphone Jack Is Not Inserted Into the Console

nn::audio::AudioIn can be initialized and finalized when a microphone is attached to the NX system.
When the microphone is detached from the system, nn::audio::AudioIn continues to supply data for silence.

Supported Sample Rate, Number of Channels, and Format

The following sample rate, number of channels, and sample format are supported.
While it is possible to get two-channel data, the data for each channel is the same.

Sample Rate 48000 Hz
Number of Channels 2 (Stereo)
Sample format 16-bit signed integer PCM format

Limit on the Number of Simultaneous USB Microphones

A maximum of two USB microphones can be used at the same time.
