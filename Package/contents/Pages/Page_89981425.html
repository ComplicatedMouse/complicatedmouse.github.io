Nintendo SDK 0.6.0

The following known bugs are confirmed to occur as of Nintendo SDK 0.6.0.

  • A dialog box appears when you install NintendoSDK-Documents.
  • When the platform selection window is redisplayed in the project file creation window, the selected information is overwritten.
  • The sampling interval grows longer when Xpad initialization is run without a USB GamePad connected.
  • AudioRenderer produces strange output with audio devices that use a sampling rate other than 48 kHz.
  • NintendoWare includes platform-specific information.
  • Setting the input buffer size to 0 in nn::util::StreamingDecompress produces an "ERROR (-5) : Unexpected Error" message.

A dialog box appears when you install NintendoSDK-Documents.

The following dialog box appears when you install NintendoSDK-Documents from the package manager, and the installation process is suspended.

Click Yes to proceed with the installation.
If you are installing this together with Visual Studio 2012 or 2013, the dialog box might be displayed twice.

When the platform selection window is redisplayed in the project file creation window, the selected information is overwritten.

When creating a project file in Visual Studio, if the Next button is used to navigate away from the platform selection window that appears when some packages other than the shared packages are installed, and then the platform selection window is displayed again, the selection always returns to the first option.

This behavior will be fixed in a future release.

The sampling interval grows longer when Xpad initialization is run without a USB GamePad connected.

If Xpad initialization is run without a corresponding USB GamePad being connected, a timeout of up to 4 ms for each device occurs every time a sample is taken.

For instance, if initialization is run on four Xpad devices without any USB GamePad controllers connected, the sampling interval for all Xpad devices would be up to 16 ms.

However, if only one device is initialized, sampling takes place at the original 5 ms interval. This allows for emulation using keyboard input as one Xpad without issue.

Note that the timeout occurs on an internal sampling thread and not on the thread that calls the HID API.

AudioRenderer produces strange output with audio devices that use a sampling rate other than 48 kHz.

Sounds played back using the AudioRenderer feature of the nn::audio library do not have the correct pitch if the default playback device in Windows is not set to 48 kHz.
Set the sampling rate of the default playback device to 48 kHz.

This problem will be resolved in a future release.

NintendoWare includes platform-specific information.

Although common packages are not supposed to depend on specific platforms, NintendoWare includes some platform-specific implementations, documentation, and samples.
These resources will be removed from the common package in future releases.

Note that the information about these resources is hard to use and understand in some parts of the package.

Setting the input buffer size to 0 in nn::util::StreamingDecompress produces an "ERROR (-5) : Unexpected Error" message.

If the input buffer is set to an invalid size, the function is designed to output an error message to the log, but this version incorrectly displays an ERROR (-5) : Unexpected Error message.

If this message appears, confirm that the input buffer size is set correctly.
