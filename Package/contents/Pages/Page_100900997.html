NintendoWare includes platform-specific information.

Although common packages are not supposed to depend on specific platforms, NintendoWare includes some platform-specific implementations, documentation, and samples.
These resources will be removed from the common package in future releases.

Note that the information about these resources is hard to use and understand in some parts of the package.

If you try to uninstall the Documents package after uninstalling Visual Studio but with the Microsoft Help Viewer still installed, the uninstall will fail.

If you try to uninstall the Nintendo SDK Documents package after uninstalling Visual Studio (any version) but with the Microsoft Help Viewer still installed, the uninstall will fail.

Uninstall Microsoft Help Viewer first, and then uninstall the package.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer.

The Find Result page for the NintendoSDK API Reference Manual does not work correctly in the Microsoft Help Viewer. If you try to use it, you cannot enter anything in the text box, making search impossible.

Use a web browser to use the Find Result page.

Audio-Related

An error may occur when attempting to reload a project with the Search tab open in NintendoWare SoundMaker.

A SoundMaker error may occur when attempting to reload a project while the Item search tab or Instrument search tab is open in NintendoWare SoundMaker.

If the Remember whether tabs are open or closed option is selected, an error occurs every time a project where this bug has occurred is opened. For this reason, clear this option.

You can avoid this bug by closing all search tabs before reloading.

Exporting sound waveforms does not work in NintendoWare SoundMaker and SoundProjectWaveExporter.

Attempts to export sound waveforms using the NintendoWare tools SoundMaker and SoundProjectWaveExporter always result in the creation of blank waveform files.

This problem is expected to be fixed by the next release.

When you convert a sound project imported using NintendoWare's SoundMaker, an invalid sound archive is exported.

When, in SoundMaker, an NW4C/NW4F sound project (one for which the cache file remains for converting purposes) is imported and (normally) converted, NW4C/NW4F binary source files become mixed in with the NintendoSDK binary sound archive (BFSAR) files. As a result, they cannot be correctly read during the nn::atk and nw::snd runtimes. This is a problem.

As a workaround to this problem, make sure to reconvert the sound project after importing it. This conversion makes the old cache file unusable.

Some of the APIs for nn::atk do not work.

APIs related to LowPassFilter do not work.

APIs related to LowPassFilter, such as nn::atk::SoundHandle::SetLowPassFilterFrequency, can currently be set, but they are not applied.

The LowPassFilter value set in SoundMaker is not applied either.
BiquadFilter can still be used, as before.

APIs related to SurroundPan do not work.

APIs related to surround pan, such as nn::atk::SoundHandle::SetSurroundPan, can currently be set, but they are not applied.

The surround pan value set in SoundMaker is not applied either.

If some of the parameters in Biquad filter are set, the audio may be cut off.

If the following parameters are set in SoundHandle/Player::SetBiquadFilter, audio from some sources may be cut off.

  • Apply the BiquadFilterType_LowPassFilter filter and substitute a value of 1.0
  • Apply the BiquadFilterType_HighPassFilter filter and substitute a value of 0.1

An exception is thrown when you start the NintendoWare AtkPlayer when there are no available playback devices.

If AtkPlayer is started when there are no available playback devices on the PC, an exception is thrown, and the program halts. The workaround is to have a playback device available.

The problem that results in the exception being thrown is expected to be fixed in either the next release or the one after that.

The constant buffer operations for ui2d cannot be performed from multiple threads simultaneously.

As of this package, the ui2d constant buffer has been gathered in one library, but mutual exclusion is not in place. As a result, the constant buffer cannot be updated from multiple threads simultaneously.

For more information, see Changes to NintendoWare Graphics 0.8.0.

The correct operation flag configuration value is not returned as the return value for nn::htcs::HTCS_F_GETFL in nn::htcs::Fcntl().

In nn::htcs::Fcntl(), using command == nn::htcs::HTCS_F_GETFL to get the configuration value normally returns 0x04 if nn::htcs::HTCS_O_NONBLOCK is set, but currently it returns 1.

Jumping to change log details does not work.

In the change log list, when you try to click each history to jump to the details, it may jump to the wrong location.

This problem does not occur with Internet Explorer.
