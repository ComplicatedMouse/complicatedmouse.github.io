Exhibition Features

You can use the exhibition command to change the settings of various functions for exhibition and confirm the settings.
For more information, see DevMenu > Exhibition Features.

$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition --help
usage: DevMenuCommand exhibition get-exhibition-mode-enabled
       DevMenuCommand exhibition set-exhibition-mode-enabled <true|false>
       DevMenuCommand exhibition get-playable-time
       DevMenuCommand exhibition set-playable-time <seconds>
       DevMenuCommand exhibition get-idle-detection-time
       DevMenuCommand exhibition set-idle-detection-time <seconds>
       DevMenuCommand exhibition show-asset-data-settings
       DevMenuCommand exhibition delete-all-asset-data
       DevMenuCommand exhibition set-waiting-for-play-image <absolute_png_file_path>
       DevMenuCommand exhibition delete-waiting-for-play-image
       DevMenuCommand exhibition set-play-exited-image <absolute_png_file_path>
       DevMenuCommand exhibition delete-play-exited-image
[SUCCESS]

Enabling and Disabling Exhibition Features

Exhibition features are made available by enabling Exhibition mode. This mode is disabled by default.
To update the settings after changing them, restart the development hardware.

# Get the enabled/disabled status of Exhibition mode.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition get-exhibition-mode-enabled
enabled
[SUCCESS]

# Enable Exhibition mode.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition set-exhibition-mode-enabled true
Please restart your target to apply this change.
[SUCCESS]

# Disable Exhibition mode.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition set-exhibition-mode-enabled false
Please restart your target to apply this change.
[SUCCESS]
Info

JIT debugging functionality is disabled when Exhibition mode is enabled. The reason is to shut down an application when the application aborts while it is running in Exhibition mode.
Conversely, JIT debugging functionality is enabled when Exhibition mode is disabled.

Play Timer Setting

By setting a playable time (in seconds), the system can be made to automatically terminate an application after a set period of time has passed after starting the application.

# Get the playable time.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition get-playable-time
Playable time: 600
[SUCCESS]

# Set the playable time.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition set-playable-time 600
Playable time: 600
[SUCCESS]

Controller Idle Timer Settings

By setting a time (in seconds) to allow a state in which the controller is not being operated and no one is playing the application, the system can be made to automatically terminate an application after input from the controller has not been detected for a set period of time.

# Allow idle state time.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition get-idle-detection-time
Playable time: 600
[SUCCESS]

# Allow idle state time.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition set-idle-detection-time 600
Playable time: 600
[SUCCESS]

Waiting for Demo Image, Demo Exited Image Settings

You can set the images to be displayed before and after an application demo.
Only PNG format (1920 × 1080, RGBA32) image files are supported.

# Display the image settings state.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition show-asset-data-setting
Waiting for play image : Existing
Play exited image : Not found
[SUCCESS]

# Set the waiting for demo image.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition set-waiting-for-play-image D:\waiting.png
[SUCCESS]

# Delete the waiting for demo image.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition delete-waiting-for-play-image
[SUCCESS]

# Set the demo exited image.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition set-play-exited-image D:\exited.png
[SUCCESS]

# Delete the demo exited image.
$ RunOnTarget.exe DevMenuCommand.nsp -- exhibition delete-play-exited-image
[SUCCESS]
