Changing the SDK Path Used in the Project

Introduction

About This Page

This page describes how to change the SDK directory path (the SDK root path) that will be referenced by the NintendoSDK application project or the library project.

Important Information

This section describes how to change the path for a project that was created in the typical fashion. However, some projects may have unique ways of configuring the SDK root path. If you cannot correctly change the SDK root path of a project using the procedure described on this page, check whether the path was set in some unique way for your project.

Using NintendoSDK Configuration Manager

You can use NintendoSDK Configuration Manager to change the SDK root path referenced by the project. NintendoSDK Configuration Manager is a Visual Studio extension provided by NintendoSDK. For more information about installing and using this utility, see NintendoSDK Configuration Manager.

If for any reason you do not use NintendoSDK Configuration Manager, you can also act directly to change the SDK root path. For more information, see the following section, Directly Changing the Project's SDK Root Path Settings > Directly Changing the Project's SDK Root Path Settings. However, if the settings for your project were configured using NintendoSDK Configuration Manager, make sure that you use the same utility to change the SDK root path.

Directly Changing the Project's SDK Root Path Settings

If a project was configured to use NintendoSDK, some method was used to store the root path to the SDK to use. This section how to change the SDK root path for two typical kinds of projects.

  • Project Created Using the Wizard
  • Sample Moved to a Different Location

Whatever method you use, you must first close the project file if it is open in Visual Studio.

Project Created Using the Wizard

If you created a project using the wizard, the SDK root path configured following the wizard's instructions is saved in the project file.

To change the configured SDK root path, complete the following procedure.

  1. Use a text editor to open the project file that was created with the wizard.

  2. Change the NintendoSdkRoot tag's value to the path that you want to specify. You can also specify a path based on an environment variable.

    Example: Setting "C:\foo\bar" as the SDK Root Path 
    <NintendoSdkRoot>C:\foo\bar</NintendoSdkRoot>
    Example: Specifying the Path Set by Environment Variable XXX_ROOT as the SDK Root Path 
    <NintendoSdkRoot>$(XXX_ROOT)</NintendoSdkRoot>

For more information using the wizard, see Creating Projects

If there is no NintendoSdkRoot tag in the project file....

Projects created using wizards prior to NintendoSDK 0.15.0 may not contain a NintendoSdkRoot tag in the project file. In this case, set the SDK root path using the method described in Sample Moved to a Different Location.

Sample Moved to a Different Location

NintendoSDK samples use as their SDK root path the NintendoSDK directory in which the sample is located.

If a sample program has been copied to a different directory for testing purposes, you must reconfigure the SDK root path. To change the SDK root path, create an XML file like the following, name it Directory.Build.props, and place it in the same directory as the project.

<?xml version="1.0" encoding="utf-8"? >
<Project xmlns="http://schemas.microsoft.com/developer/msbuild/2003">
  <PropertyGroup>
    <NintendoSdkRoot>C:\NintendoSDK\</NintendoSdkRoot> <!-- Set the root path for NintendoSDK. -->
  </PropertyGroup>
</Project>
