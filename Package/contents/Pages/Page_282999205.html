Text Box Panes

This section describes text box panes, which can use registered fonts as layout assets and can handle strings.

Overview

Text box panes can use registered fonts as layout assets and can place strings in the layout.
The following are the main settings that can be configured in text box panes.

  • Font file registration
  • String input
  • Font size, character spacing, and the degree of italicization
  • Text color settings

The following table shows the text box pane settings.

Base Pane Sets information including the pane name, position, shape, and center position.
Text Box Panes Describes the text box pane UI to set the font to use in the pane, the string to display on the screen, and so on.
Color Interpolation Settings Sets the input values that affect the string's color and alpha.
Fragment Operations Sets the fragment calculation method from presets, and changes preset values to configure custom fragments.
Extended User Information User-defined extended information for the pane.

Control Strings

The ^ symbol (the caret) can be used inside the string in the text box pane as a control string.
You can expand the behavior of the text box pane on the user end by using control strings within game applications.

The layout converter interprets the two characters after the caret as a hexadecimal number (0x00-0x1F). It then performs a conversion to replace these characters with a control code.
By using these control codes, you can display an image or change the color or size of a character in the middle of a string.

If the specification is not valid, the caret character (^) is deleted in the runtime, and the two characters that follow it are displayed as normal characters.
You can disable the conversion process by specifying a converter option.

To use this functionality, some support is required on the program end.
