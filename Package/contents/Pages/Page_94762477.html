Shader Type - Color Buffer Refraction

Color Buffer Refraction

This section describes the Color Buffer Refraction feature for refracting the background using the RGB values of the texture.

Overview

Color Buffer Refraction is a feature for refracting the background using the RGB values of the texture registered in the Texture 0 tab.
As an example, you can use this feature to express a waterfall among rocks in the background using a refraction texture attached to free-falling particles.
The following overview describes the settings ().

Setting the shader type to Color Buffer Refraction displays the Color Buffer Refraction () and Coefficient Animation ().
The group has the following settings.

Coefficient

Set the amount of refraction along the x-axis and y-axis using the dials () or in the text boxes ().
These values can also be used to animate the amount of distortion with Coefficient animation.
Related Links For more information, see Coefficient Animation.

Distance-From-Camera-Based Distortion

This feature increases refraction as particles move away from the camera.
Related Links For more information, see Distance-from-Camera-Based Distortion.

Reflect Alpha Value

You can switch whether to control the amount of refraction with Texture 1 or 2 alpha elements when Color Buffer Refraction is applied by selecting or clearing the check box.
Related Links For more information, see Reflect Alpha Value.

Info

In the combiner, texture 1 and texture 2 are combined based on the result of refracting the color buffer value.

A X, Y = 0.01, 0

To determine the amount of refraction, multiply the x-direction (u-direction) of the refraction texture by the configured value.
In addition, multiply by the Coefficient animation to animate the refraction.
B X, Y = 0, 0.01 To determine the amount of refraction, multiply the y-direction (v-direction) of the refraction texture by the configured value.
In addition, multiply by the Coefficient animation to animate the refraction.
Info

The refraction image registered to texture 0 is output as unorm_8_8, with the Component section as RGGG.
In the RGGG channel format, the red component is used for horizontal refraction, and the first green component is used for vertical refraction.
