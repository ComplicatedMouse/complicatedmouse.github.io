Using Scalable Fonts in the LayoutEditor

Using Scalable Fonts

There are three ways to use scalable fonts when creating layout data.
Related LinkFor more information about scalable fonts, see here.

Method Using Composite Fonts (Recommended)

Use composite fonts to load scalable fonts directly from LayoutEditor or the program.

Composite font is a feature that allows multiple fonts to be combined into one font, but it can also be used to handle a single scalable font without combining it with another font.

For information about how to use composite fonts, see Composite Fonts.

Method of Using the Project Settings File to Load Fonts Into the Editor (Previously Used Method)

This section describes the process for loading the scalable font into the editor using the project settings file.

Writing the Scalable Font Settings to the Project Configuration File

The ScalableFontDescSet element is written like the following example.

<ScalableFontDescSet>
    <ScalableFontDesc>
        <FontName>ScFontA</FontName>
        <FilePath>.\commonRes\font\ttfont.ttf</FilePath>
        <FontSize>64</FontSize>
        <BoldWeight>0</BoldWeight>
        <BorderWidth>0</BorderWidth>
    </ScalableFontDesc>
    <ScalableFontDesc>
        <FontName>ScFontB</FontName>
        <FilePath>.\commonRes\font\nintendo_NTLG-DB_002.bfttf</FilePath>
        <FontSize>64</FontSize>
        <BoldWeight>0</BoldWeight>
        <BorderWidth>0</BorderWidth>
    </ScalableFontDesc>
</ScalableFontDescSet>

Make sure that you do not make duplicate entries for FontName.
Use a relative path from the project file for FilePath. For the format, you can specify TTF, OTF, or BFTTF.
BoldWeight specifies a bold weight in the range from 0.0 to 1.0. (0.0 is the normal width.)
BorderWidth specifies the width of the border. (0 turns off border rendering.)

Related LinkFor more information about the project settings file, seeProject File Format.

<TextureCacheSizeForScalableFont>
      <Width>1024</Width>
      <Height>2048</Height>
</TextureCacheSizeForScalableFont>

When using large fonts, if the texture cache is too small a “?” will be displayed in place of the correct characters.
If this happens, increase the size of the texture cache.

Handling in Programs

For the font name, substitute the scalable font to actually display when the program runs.

Current Restrictions

There are currently limits on the items that you can check using the viewer.

  • Outline fonts and bold font weights are not shown in the viewer

Using Bitmap Fonts Created From Scalable Fonts (Conventional Method)

This section describes how to use bitmap fonts for layouts but use scalable fonts in the actual game.

  • 1. Create the layout using a pseudo-font made into a bitmap font (BFFNT file) with FontConverter.
  • 2. Substitute with the scalable font that will actually display when the program runs.

The pseudo font and the actual font display nearly the same way, although the difference is not zero.

Make sure that you append kerning information when you create the pseudo font.
Kerning is a feature to adjust the spacing between the characters of a font so they appear more naturally together no matter how they are combined.
If kerning is not enabled, the font displays with discrepancies.

Related Link For more information about creating bitmap fonts using the GUI version of FontConverter, seeCreate Bitmap Fonts.
