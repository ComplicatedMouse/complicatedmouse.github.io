Log Viewer
Note

The Log Viewer tool has been removed as of SDK 11.0.0. To view logs, use Target Manager instead of the Log Viewer tool.

The Log Viewer tool can be used to view logs received from the development hardware.

How to Start

To run the tool, double-click LogViewer.exe from the following folder in the NintendoSDK package.

NintendoSDK\Tools\Log\LogViewer

Screen Layout

The configuration of windows and tabs and the filtering expression set for each tab are all saved when Log Viewer ends.

① Log View Field

Displays the log received from the development hardware.

Rest the mouse pointer on the log to view the associated metadata.

If a filtering expression has been entered, only the filtered log is shown.

② Filtering Expression Field

Use this field to enter a filtering expression and set criteria for showing a log. You can enter a different filtering expression for each tab.

③ Tab

Tabs let you switch among log view fields and filtering expressions.

To change the name of a tab, right-click the tab and select Rename.

Tabs are a convenient way to classify logs according to development hardware and thread names.

④ New Tab

Opens a new tab.

⑤ Menu

Use the menu to save logs and open subwindows.

Log Viewer will continue to run as long as the main window is open, even after closing any subwindows.

Subwindows can be distinguished from the main window by their lack of a menu bar.

Filtering

You can filter the log that is shown by configuring a filtering expression.

Log output that is filtered out is nevertheless received in the background. As a result, by changing the filtering expression you can display past log output that had been previously filtered out.

Operator

The following operators can be used for the filtering expressions.

Operator Description
( ) Parentheses
! Logical NOT.
>, <, <=, >= Comparison operators.
==, != Equality and inequality operators.
=~, !~ Regular expression match operators.
&&, || AND, OR.
Info

Use regular expression matching if you want to extract the log with partial string matches.

As the equality operator checks the entire string for a match, the log cannot be extracted if a different string is included.

Operands

The following operands can be used for the filtering expressions.

Operands Description
Key A key for data in the log.
Numeric Literal Only decimal notation is supported.
String Literal Enclosed in double quotation marks. Example: "This string"
Regular Expression Literal Enclosed in forward slashes. Example: /A regular expression pattern/

Data That Can Be Used for Filtering

The following kinds of data in the log can be used as filter expression operands.

Key Type Description
PeerName String The name of the development hardware system that output the log.
ProcessId Numbers The ID of the process that output the log.
ThreadId Numbers The ID of the thread that output the log.
Severity Numbers The severity of the log (assigning method not provided).
Verbosity Numbers The verbosity of the log (assigning method not provided).
LineNumber Numbers The number of the line that output the log.
FileName String The name of the file that output the log.
FunctionName String The name of the function that output the log.
ModuleName String The name of the module that output the log (assigning method not provided).
ThreadName String The name of the thread that output the log.
TextLog String The main body of the log.

Saving Logs

Saving the Entire Log

Use the menu in the main window to save the log. The entire log is saved without any filtering.

Saving the Filtered Log

Right-click and use the shortcut menu to save the filtered log for a tab in which a filter expression has been entered. Only the part of the log that meets the filtering criteria is saved.

Log Save Formats

Plain Text

The strings output by the log API are saved in plain text format. Metadata is not included.

JSON

The strings output by the log API are saved in JSON format, saving both the strings and the associated metadata.

Console Control Using Escape Codes

The following features are supported. Moving the cursor is not supported, nor are other formatting specifications.

  • Bold, italic, underline, and strikethrough text
  • 3/4-bit, 8-bit, and 24-bit colors for the foreground and the background.
  • Reversing foreground and the background colors.

Important Information

All features of Log Viewer will be merged with Target Manager in the future. For this reason, Log Viewer is not being updated.

Log Viewer has the following problems. As a workaround, use Target Manager.

When a log file larger than a certain size is opened, the second half of the log is not displayed.

When Log Viewer opens an NX Binary Log file (.nxbinlog) that is larger than several dozen megabytes, the latter part of the log is omitted and not displayed.

There are no plans to fix this issue. Use Target Manager, selecting File > Open log file.

When you attempt to select a range within the log, you get a broader selection of the log.

Use the Log view in Target Manager.

The indentation is broken for any log metadata displayed in tooltips when you use Windows 7.

Use the Log view in Target Manager.
