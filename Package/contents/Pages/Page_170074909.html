TouchScreen (NX Addon)

Installing the NX Add-On enables the TouchScreen mechanism to be used in the environments described below.

Note that to match operations in these development hardware environments, the structure in the Windows environment is also different.

Structure in the Windows Environment

The coordinates of the 15 pixels around the periphery of the screen are clamped to the interior.

Structure in the SDEV Environment

Note

Precision, as it relates to latency, may change in the future. Use this information for reference purposes only.

The input state is determined by the state of the built-in touch screen.

Up to 10 taps can be detected simultaneously.

The 15 pixels of the periphery return coordinates clamped to the interior.

When the touch point moves, there is a latency in the coordinates due to the characteristics of the controller IC.

When the touch moves at a speed of 200 mm/sec, the latency is approximately 12 ms. The slower the movement, the larger the latency tends to be.

The input from the touch screen is ignored when the system is docked.

Structure in the EDEV Environment

The same as in the SDEV environment.
