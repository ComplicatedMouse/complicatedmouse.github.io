Special Functions and Threads

Functions

The following is a list of special functions that the profiler GUI will display. A function is special if it is constructed inside the GUI rather than being read out of a module.

UNRESOLVED

If a function is marked UNRESOLVED, the profiler is not able to map the function's address to a module (one of the NSS or NRS files). In Callstack profiles, UNRESOLVED functions are typically due to either callstack frames not being enabled or corrupted frame data. In Leaf Only profiles, these should generally not occur.

In the GUI, this function is given a fixed address of 0x00000000.

<Module>_internal (unresolved)

Functions in this form, for example "nnSdk.nss_internal (unresolved)", are functions that are attributable to a specific module, but for which the profiler was unable to find a function symbol. This is common for SDK-provided modules, such as nnSdk.nss, as these modules usually remove their internal function symbols.

System Idle Thread

If a function is marked System Idle Thread, it means that when the profiler took a sample, the thread running at the time was the system-level idle thread.

In the GUI, this function is given a fixed address of 0x00000001.

SYSTEM THREAD

If a function is marked SYSTEM THREAD, it means that some system-level thread that is not the System Idle Thread was running at the time that sampling occurred. This is similar to the "SYSTEM: <Function>" special function listed below, but it was not possible for the profiler to map the system call back to a specific function or thread.

In the GUI, this function is given a fixed address of 0x00000004.

SYSTEM: <Function>

Any function starting with the string "SYSTEM:" is a function that is in the process of making a system call. If taking a Callstack profile, the Call Tree tab can be used to look at the callstack to determine what kind of system calls are being made.

Other Process

If a function is marked Other Process, it means that some other process was running on the core and when the profiler took a sample, that other process was the one running. For standard applications, this function will generally not occur.

In the GUI, this function is given a fixed address of 0x00000003.

BAD FRAME POINTER

If a function is marked BAD FRAME POINTER, it means that while walking the callstack frames, a frame pointer existed that jumped outside of the current stack. These are typically due to either callstack frames not being enabled or corrupted frame data. This function type should only occur in Callstack profiles. All functions under this node in the Call Tree, except the leaf function, are possibly invalid.

In the GUI, this function is given a fixed address of 0x00000005.

SENTINEL SAVE

If a function is marked SENTINEL SAVE, it is due to running out of space in the current buffer while walking the callstack frame. This function type should only occur in callstack profiles, and should only occur occasionally. The functions under the node in the Call Tree are still valid.

In the GUI, this function is given a fixed address of 0x00000006.

UNKNOWN STACK DETAILS

If a function is marked UNKNOWN STACK DETAILS, it means that some unknown problem has occurred while reading the callstack on the PC. In general, this function should not occur.

In the GUI, this function is given a fixed address of 0x00000002.

Threads

The following is a list of the special threads that the profiler GUI will display. A thread is considered special if it is constructed inside the GUI rather than being one of the active threads in the profiled application.

System Idle Thread

The System Idle Thread is a system-level thread that runs when no other thread on the core can be scheduled.

In the GUI, this thread is given a fixed thread id of 0x00000000.

SYSTEM THREAD

Any thread marked SYSTEM THREAD is a system-level thread that was not the System Idle Thread.

In the GUI, this thread is given a fixed thread id of 0x00000001.

Other Process

If any other process has a thread running on a core, it is possible that the thread may be sampled. The profiler is not capable of determining what the other process is, so any and all other processes are included as a single grouping per core in Other Process. For standard applications, this thread will generally not occur.

In the GUI, this thread is given a fixed thread id of 0x00000002.
