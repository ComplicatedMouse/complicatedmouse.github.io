Using Derivative Parts

This section describes the procedure for using the Derivative Parts feature to create different button parts from an existing selection button part.
For example, as shown in the following image, the Menu_Bar.flyt part created in the Creating Part Animations tutorial is used to derive a button that can only be used on a particular menu screen.

The mark New !! (indicating a new feature), is set to the upper right corner of the selection button.

Preparation

To prepare for creating the layout data, start LayoutEditor in advance.

Related Links For more information about starting LayoutEditor, see Workflow.

  • Hierarchy

Creating a Derivative Part

Use the following procedure to create the layout data for a derivative part derived from the Menu_Bar.flyt parts layout.

1
In the Parts Palette window, select the Menu_Bar.flyt parts layout.
2 Right-click to open the shortcut menu, and select Create Derivative Part.
3.

In the layout window, confirm that a new set of layout data named Untitled.flyt has been created.
4 Confirm in the Hierarchy window that the Menu_Bar.flyt pane is displayed.

Icons display in red in the Hierarchy window to indicate that editing is prohibited, as shown in the following image.
The content of Menu_Bar.flyt, the selection button part that is the source for the derivative part, cannot be changed.

Saving Layout Data Under a Different Name

Save the layout data that will be used for the derivative part in Untitled.flyt.
From the main menu, select File > Save as, and in the dialog box that appears, set Menu_DerivingParts.flyt as the filename.
Click OK in the Save as dialog box to save the edited content.

In the following, we will edit the derivative part named Menu_DerivingParts.flyt to create a different selection button part derived from the Menu_Bar.flyt parts layout.

Creating a Picture Pane　

From the Texture window that displays the textures, create a Picture pane ().

In the Texture window, double-click Derivation_Base.tga (image).
In the Hierarchy window, select the created Picture pane () and change the pane name to Derivation_Base.
In the property window, specify the following settings.

Base Pane ()

 Position X,Y = 240,16 This pane becomes the frame for the string New!! that will be placed on the upper right side of the select button.

Color Interpolation Settings ()

 White Color Blending R,G,B,A = 0,0,0,200

Related Links For an overview of picture panes, see here.

Creating a Text Box Pane

Create the string New!! in a Text Box Pane ().

From the main menu, select Pane > Create Pane > Text Box Pane.
In the Create Text Box dialog box that opens, configure the settings to the following.

Font File Specification Samples/Sources/Resources/Ui2d/Common/Fonts/sample.bffnt
String New!!

Click the OK button in the Create Text Box dialog box.
In the Hierarchy window, select the created Text Box pane () and change the pane name to T_Derivation.
In the property window, specify the following settings.

Base Pane ()

 Position X,Y = 245,18

This pane will become the text that is placed in front of the Derivation_Base picture pane () that was placed earlier.

Adjust the position so it is placed in the center of the Derivation_Base picture pane.

Text Box Pane ()

 Italic Tilt (Angle) -10 Adjust the incline angle of the font so it matches the shape of the Derivation_Base picture pane.

Color Interpolation Settings ()

 White Color Blending R,G,B,A = 255,200,0,255

Incorporating the Derivative Part Intro the Hierarchy

Place the created Derivation_Base () and T_Derivation () below N_Menu () in the Hierarchy window.
This allows these parts to inherit the animation of the derivative base part, N_Menu ().
Checking in the viewer confirms that the created New !! mark is now linked to the selection button animation, as shown in the following image.

Info

With this method, a new pane can be incorporated into the original layout data, placed under the derivative base pane in the Hierarchy window.
When the derivative base pane, located higher in the hierarchy, is animated, the incorporated pane inherits that animation.
By using this aspect of the Derivative Parts feature, you can set an animation that you want to be shared to the derivative base part. This is so it also will be used by any derivative parts.

Removing a Derivative Part

If you want to remove the association of a derivative part with its derivative base, use the Parts Palette window.
Note that this operation cannot be undone. We recommend making a backup copy before using this procedure.

To describe the procedure, we use the example of removing the association with the derivative base Menu_Bar.flyt and converting the data to normal layout data.

1

From the main menu, select File > Close to close Menu_DerivingParts.flyt.

If multiple layouts are loaded in LayoutEditor, use the same procedure to close them all.
2 Select the Menu_DerivingParts.flyt derivative part in the Parts Palette window.
Right-click and select Modify Relationship with Base, from the context menu, to open the dialog.
3.

Confirm that the check box next to Menu_Bar.flyt, in the list of derivative files, is selected () in this dialog.
4

From the New name selection item, select End derivative relationship and make it a normal layout ().
5 Click OK to confirm the setting and close the dialog.
6 To check the setting, select the Menu_DerivingParts.flyt derivative part in the Parts Palette window.
Right-click and select Open in the context menu to open Menu_DerivingParts.flyt.
7

When you check the Hierarchy window, you see that the red icon that indicates a derived state () has reverted to a locked icon ().
Change it to an editable icon by clicking the lock mark.
