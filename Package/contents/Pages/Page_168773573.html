Buffer Texture View

Overview

The buffer texture view provides features for interpreting and referencing certain buffers in a certain format.
The buffer texture view does not itself manage buffer memory. That resembles referencing with C++.

The buffer texture view can be set for variables used by shaders to access buffers in a certain format. Unlike Texture View, the buffer texture view is set on its own, without combining it with a sampler.

Information

Use nn::gfx::BufferTextureViewInfo to define the buffer texture view information.

Format

Use the SetImageFormat function to specify the format to use when interpreting the source buffer that is referenced.

Buffer Pointer

Use the SetBufferPtr function to specify the source buffer that is referenced.
The source buffer must be initialized in a way that includes nn::gfx::GpuAccess_Texture or nn::gfx::GpuAccess_Image as a method for GPU access.

Offset

Use the SetOffset function to specify the byte offset from the start of the source buffer that is referenced.

Size

Use the SetSize function to specify the byte size of the source buffer that is referenced.

Object

Use nn::gfx::BufferTextureView to define the buffer texture view object.

Initialization and Destruction

Buffer texture views are initialized and finalized according to the rules for objects.

Setting the Buffer Texture View

Setting to the Descriptor Pool

The buffer texture view is set to the descriptor pool as either a buffer texture for reading (by using the nn::gfx::DescriptorPool class SetBufferTextureView member function) or as an image for image load/store (by using the SetBufferImage member function).
Get the descriptor slot for the set buffer texture view using the GetDescriptorSlot function of the nn::gfx::DescriptorPool class.

Setting to the Command Buffer

To add a command to set the buffer texture view, call the SetTexture function of the nn::gfx::CommandBuffer class. To add a command to set an image for image load/store, use the SetImage function of the nn::gfx::CommandBuffer class. At this time, set the shader slot number using the value dynamically obtained from the shader object. (See Uniform Blocks and Sampler Initialization Overwrites in the GLSL Shader).
