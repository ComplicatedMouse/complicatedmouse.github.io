Optimizing CPU Processing

This section describes how to optimize CPU processing.
The CPU mainly performs two types of processing.

  1. Emitter processing
  2. Particle processing (when using CPU emitters)

Emitters

In the day-to-day creation of data you probably are careful to limit the number of particles. However, the number of emitters can also become a bottleneck.
Particularly if you do not implement processes such as culling in your project, the number of emitters that play at the same time can become enormous and cause problems.

Processing for Emitters

The following processing can take place for emitters. (It is common to both CPU and GPU emitters.)

  • Emitter time animations (when used)
  • Emitter matrix updating
  • Particle emission
  • Packing of emitter data sent to shaders
  • Rendering (configuring states, configuring textures, entries, and so on)

The number of emitters cannot compare to the number of particles, but one emitter is more costly to the CPU than one particle.
When there are hundreds of emitters playing at the same time, the CPU cost can become too high to ignore.
