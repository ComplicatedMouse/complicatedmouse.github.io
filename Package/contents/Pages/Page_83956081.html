User Settings File

Overview

The user settings file is 3DEditor.user.config.

This file can be edited by the user to customize 3DEditor operations.

The following settings can be made in the user settings file.

  • Startup Path
  • Reloading Settings When Files Are Updated
  • Editing History Settings
  • External Program Setting
  • I/O Settings
  • UI Control Settings

Note: For information about the user settings file format, see here.

Reloading Settings When Files Are Updated

You can set whether 3DEditor reloads updated files in user settings when another application has updated files loaded into 3DEditor.

  • The settings can be made separately for each file type.
  • Any type of intermediate file can be targeted for reloading.
  • You can set whether to display a confirmation dialog box when reloading files.
    Note that a confirmation dialog box appears regardless of this setting when reloading unsaved files.
    You can select whether to reload files in the confirmation dialog box.
  • When reloading model or texture pattern animations, texture files are also reloaded if the texture files being referenced have not yet been loaded into 3DEditor.

Editing History Settings

Edit history settings allow you to set whether to delete the edit history when closing or reloading a file and the number of items to be maintained in the edit history by 3DEditor.

External Program Setting

Sets whether to output debugging logs for the text editor that is run from the 3DEditor, or output debugging logs from the application that it is connected to.

I/O Settings

You can specify whether to automatically revise materials when a model file is loaded. You can also specify the export directory for model information when compiling shaders and whether to automatically reload open files afterwards.

UI Control Settings

Specifies whether to display the slider control on the Shader page in the material Property window.
