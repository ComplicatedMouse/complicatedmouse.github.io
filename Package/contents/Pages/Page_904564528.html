Unintended lines appear between the text box panes and capture effects.

Question Depending on the settings, unintended lines can appear between the text and the effect when a capture effect is used in a text box pane.

In the following use case, shadow (■ ) and a border capture effect (■ ) are used on an "L" character in a text box pane.
For easy checking, the color of the text, shadow, and border are all set to white.
You can see the unintended lines between the text box pane and the capture effect (shown in the background color ■).

Property Window Settings in Each Pane

Shadow Settings

Text Box Pane ()

  • Text Box Pane ()
    • String: "L"
    • Shadow:
      • Enabled


Capture Pane ()
  • Place a text box pane lower in the tree,
    and capture text with shadow.

Outline Settings

Picture Pane ()

  • Texture ()
    • Register the captured texture.
  • Capture Effect ()
    • Enable outlines
      • On

Answer Improve the situation by using the following settings when you use the capture feature.

  • Set Fragment Operations for the capture target pane.
  • Set Clear Color of the capture feature to be the color of the capture target pane.
  • Set Knockout (Do not render effects in areas that overlap the original image) of the capture feature.

The specific workaround is as follows.

Property Window Workaround for Each Pane

  • Fragment Operations ()

      • Off
    • Calculate the blend alpha.


Capture Pane ()
  • Capture Pane ()
    • Clear Color
      • Set the same color as the text.

  • Capture Effect ()
