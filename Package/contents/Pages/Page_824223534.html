Response Simulation Feature

Overview

This development support feature simulates a response from the shop service server. You can configure it to return a desired response.
It can be used for purposes like the following.

  • Register expected responses from the shop item catalog service server to verify application operation in advance.
  • Register errors that occur when accessing the shop service server to verify how the application handles them.

Conditions for Using the Feature

This development support feature works when the following conditions are satisfied.

  • The shop service server is accessed via nn::ec::ShopServiceAccessor.
  • Must be in DEV MODE. It does not work in PROD MODE.

Notes When Using This Feature

Note the following information when using this development support feature.

  • The JSON compatibility check feature does not add dummy elements to the registered responses.

Using the Response Simulation Feature

  1. Register the responses to simulate using either DevMenuCommand or DevMenu.
    Register responses using JSON files. For information about the notation method, see Shop Features > Registering a Response for EC Response Simulation.
  2. Access the shop service server via nn::ec::ShopServiceAccessor in the running application.
  3. nn::ec::ShopServiceAccessor returns a registered response to verify that the application is working properly.

Configuration Using DevMenuCommand

See Shop Features > Configuring the EC Response Simulation Feature.

Registering Responses

As shown in the following screen, click the Register by File button, select the JSON file on the microSD card, and register the response. Click the Unregister button to unregister a response.


Put the JSON file to use for registration in the following directory.
