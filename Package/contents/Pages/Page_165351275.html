stdlib.h int clearenv(void);

Description

Sets the value of the external variable  environ  to NULL but doesn't free it.

The first time putenv() is called it will malloc() memory for environ.

Since clearenv() doesn't free it, the user must call "free(environ);" before the program exits.


More Info

http://linux.die.net/man/3/clearenv

string.h errno_t memset_s(void *dest, rsize_t destsz, int ch, rsize_t count);

Description

In order to use memset_s(), programmer needs to include the following headers and define the corresponding macro in the order shown below:


#include <features.h>

#if defined(__STDC_LIB_EXT1__) || defined(__NN_STDC_LIB_EXT1_MEMSET_S__)
#define __STDC_WANT_LIB_EXT1__ 1
#include <string.h>
#endif


…
        <use memset_s()>
…


Rynda currently does not define __STDC_LIB_EXT1__ and __STDC_WANT_LIB_EXT1__ as we do not fully support the *_s functions.


RSIZE_MAX, errno_t and rsize_t are currently defined in string.h, which deviate from what the standard states. These macros may be moved to the specific headers when more *_s functions are supported.


External info

http://en.cppreference.com/w/c/string/byte/memset

unistd.h

int dup(int);
int dup2(int, int);

Description

dup() and dup2() are limited to only duplicating file descriptors that represent consoles, files or directories (no sockets).

EBUSY and EINTER error codes are not supported.

dup() and dup2() can use up to 515 file descriptors (open() always fail when the number of sum of opened handle is more than 515).


More Info


https://linux.die.net/man/2/dup
