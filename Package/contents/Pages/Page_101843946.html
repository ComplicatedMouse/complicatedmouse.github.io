3D Texture Preset Converter

This section describes how to use the 3D texture preset converter (%NINTENDO_SDK_ROOT%\Tools\Graphics\3dTools\3dTexturePresetConverter.bat), together with notes about the tool.

Overview

The 3D texture preset converter is a command-line tool that uses the options defined in an XML-format preset (NTCP) file to convert image (TGA, PNG, DDS, EXR) files to texture intermediate (FTXB) files.
The 3D texture preset converter is a utility that internally calls the 3D texture converter.

The preset file describes the output folder, global options, preset definitions, preset overwrite, and preset assignments.
You can also import a preset file into another preset file and use the definitions of the imported preset file.
In addition, user-specified commands can be set to run before and after the conversion process.

Command-Line Options

3dTexturePresetConverter.bat [image file|input folder]... [--args-file ""] [--version] [-h] [-s]
[--continue-on-error] [--preset-file ""]... [--output-job-list ""]

Image File|Input Folder

Specify the path to an image (TGA, PNG, DDS, EXR) file or to a folder containing image files.
If a folder is specified, all image files in that folder and its subfolders are converted.
You can specify multiple image files and folders by separating them with spaces.

As a special method, you can specify and reconvert texture intermediate files instead of image files.
However, if a folder containing texture intermediate files is specified as the input folder, those files are not reconverted.
This limitation prevents unnecessary conversion when the specified folder contains both an image file folder and a texture intermediate file folder.
To reconvert all of the texture intermediate files in the specified folder and its subfolders, use a batch script such as the following.

type nul > args.txt
for /r "%~1" %%I in (*.ftxb) do echo %%I >> args.txt
call 3dTexturePresetConverter.bat --args-file args.txt --preset-file preset1.ntcp

--args-file ""

You can use the contents in the parameter file to specify arguments.
The parameter file allows you to specify each option over several lines.
Lines starting with a number sign (#) are treated as comments.
Newline characters cannot be included in escape sequences used in option specifications that appear between quotation marks (").

--version

Displays version information.

-h --help

Displays Help.

-s --silent

Suppresses the display of messages during processing.
It also controls the standard output for the commands to execute before and after conversion.

--continue-on-error

Processing continues even if errors occur in a portion of image conversions. (Available in NintendoSDK 9.1.0 and later)
However, conversion is suspended in the following cases.

  • The preset file is invalid.
  • The pre_convert_command fails.
  • Texture intermediate files are exported multiple times to the same path.

--preset-file ""

Specifies the preset file (NTCP file) with the presets to apply. You can specify this option multiple times.
When this option is specified multiple times, the combined definitions of all the preset files are applied.
If multiple definitions have the same name, the one in the preset file that was specified last takes precedence.

--output-job-list ""

Specifies a text file (job list file) for saving 3D texture converter arguments when converting.
Used in situations such as when investigating the relationships between image files and texture intermediate files.
If this option is specified, the usual conversion is not performed, a texture intermediate file is not saved, and <pre_convert_command> and <post_convert_command> are not executed.
The job list file is in a format that can be specified in the --job-list option of a 3D texture converter.
The job list file uses an ANSI encoding (Shift-JIS in Japanese environments).

Usage Examples

Converts one image file.

3dTexturePresetConverter.bat inputAlb.tga --preset-file preset1.ntcp

All image files in the specified folder and its subfolders are converted.

3dTexturePresetConverter.bat D:\data\sourceimages --preset-file preset1.ntcp

Saves the arguments for the 3D Texture Converter to a text file, if converting without actually doing the conversion.

3dTexturePresetConverter.bat D:\data\sourceimages --preset-file preset1.ntcp --output-job-list jobs.txt

Preset File Specification

Overview

The preset file is an XML file that defines a combination of 3D texture converter options to apply when converting the image files.
The extension is .ntcp (Nintendo Texture Converter Preset, or NTCP).

Constituent Elements

The value in the square brackets indicates the number of times that the element appears.

  • <texture_convert_preset>[1]
    The root element.

    Attribute Name Description
    version

    The version of the preset file.


  • <import> [0 and greater] (Can be specified multiple times.)
    References an external preset file.
    If <output_folder>, <preset> with the same name, <preset_override> with the same name, or <preset_assign> with the same file_name elements exist after the <import> element, the element that is described last gets first priority.
    If <global_option> elements are present after the <import> element, the <global_option> elements are combined and the resulting options are applied. If there are redundant options, the values specified later take precedence.


    Attribute Name Description
    path

    Specifies an absolute path to the preset file to reference, or the relative path from the imported folder containing the preset file.
    This path may include environment variables surrounded by percent signs (%).

  • <output_folder> [0 to 1]
    Specifies the path of the output folder.
    If this element not specified, the data is saved to the same folder that contains the input files.

    Attribute Name Description
    path

    Specifies the absolute path of the output folder, or a relative path from the folder with the input files.
    This path may include environment variables surrounded by percent signs (%).
    $1, $2, ... is replaced by strings that match the regular expression specified by search_input.
    search_input

    When using the string found from searching for the input file folder path for the output folder path, specify the search string as a regular expression. (This feature is available in NintendoSDK 10.4.0 and later.)
    The $1, $2, ... in the path attributes are replaced by strings that match the regular expression, surrounded by parentheses.
    (They are not replaced if the search_input attribute is omitted.)
    Use a slash (/) as the character for separating folders.
    This path may include environment variables surrounded by percent signs (%).
    The input file folder to search is the absolute path with a slash (/) at the end.

    If there is no match to the regular expression, the conversion of this input file is skipped.
    (The conversion process is also skipped if any of $1, $2, ... is not replaced.)

    The following example is for when you want to export image files from a different folder hierarchy into a folder with the same hierarchy depth.

    D:/data/ProjectA/sourceimages/texA1.png
D:/data/ProjectA/sourceimages/sub/texA2.png
D:/data/ProjectB/sourceimages/texB3.png

    ↓

    D:/data/ProjectA/output/textures/texA1.ftxb
D:/data/ProjectA/output/textures/texA2.ftxb
D:/data/ProjectB/output/textures/texB3.ftxb

    Because you cannot handle this with a common relative path, specify the path as follows.

    <output_folder path="$1/output/textures" search_input="(.*?) /sourceimages/" />

  • <global_option> [0 to 1]
    Defines global options.
    The global options are a group of options that are passed to the 3D texture converter by all of the presets.
    This element does not need to be described if there are not any options that you want to apply in all the presets.
    If the same option is described in both <global_option> and a <preset> element, the one in <preset> takes precedence.
    If an option with no parameters is specified in the <global_option> element, it can be negated by specifying the none attribute in a <preset> element.
    For example, if disable_file_info="none" is specified, --disable-file-info is not specified by that preset.
    If the <global_option> element is present in multiple preset files, the contents of the elements are combined and the resulting options are applied.
    If there are redundant options, the ones specified in the later preset file take precedence.

    Attribute Name Description
    3D texture converter options Describes using the same format used for the 3D texture converter options in the <preset> element.
    alternative_format_??? Describes in a format similar to the <preset> element.
  • <presets> [0 to 1]
    The parent element for the <preset> elements.

  • <preset> [0 and greater]
    Defines presets for the 3D texture converter settings.
    The preset definitions are a group of options (such as format and hint information) that are passed to the 3D texture converter.
    <preset> elements are described under the <presets> element.
    If multiple preset definitions have the same name, the one defined last takes precedence.

    Attribute Name Description
    name Specifies the name of the preset definition.
    base_name

    Specifies the name of the preset definition from the inheritance source when inheriting other presets.
    The options specified for the inheritance source are all inherited.
    The options can be added to or changed for the inherited preset.
    If an option with no parameters is specified in the inheritance source, it can be negated by specifying none for the attribute value in the inherited preset.
    It is possible to inherit additional presets that were inherited from another preset.
    This attribute does not need to be described if inheritance is unnecessary.
    3D texture converter options

    Specify 3D texture converter options in the following format: delete the two hyphens (--) found at the start of long-name options, and replace the single hyphen (-) with an underscore (_).
    For example, the 3D texture converter option --comp-sel="rgba" would be defined as comp_sel="rgba" in a <preset> element.

    For options that do not have any parameters, use the null character for the attribute, as in disable_file_info="".

    The 3D texture converter's default values are used for options that are not specified.

    If the path specified for output is not an absolute path, it is treated as a relative path from the folder specified in the <output_folder> element.

    If an absolute path is not specified for options such as merge and replace_r/g/b/a, the paths are handled as relative paths from the folder containing the input file.

    alternative_format_???

    Specifies an alternative format and selected components if you are exporting the image file in a changed format but some or all of the channel replacement files specified by the replace_r/g/b/a attributes do not exist.
    If this attribute is not described, an error occurs if the replacement file or any part of it does not exist.
    The string after alternative_format_ represents the channels that have replacement files.
    For example, alternative_format_rg specifies the alternative format when replacement files exist only for the R and G components (see Example of Combining Multiple Input File Channels Into One Output File).
    The string after alternative_format_ must be described in rgba order.
    Use alternative_format_none to specify a replacement format even when there are no replacement files.

    This attribute takes the form “alternative format, selected components, hint information, linear conversion, which channels to replace.”

    (Hint information and linear conversion can be specified in NintendoSDK 8.0.0 and later.)
    (Which channels to replace can be specified in NintendoSDK 9.0.0 and later.)

    When omitting the component selection, the value specified by the comp_sel attribute is used.

    When omitting hint information, the value specified by the hint attribute is used.
    For linear conversion, specify the value passed to the --linear option for the 3D texture converter. When omitting linear conversion, the value specified by the linear attribute is used.

    When changing which channels to replace, specify the channels being replaced in RGBA order, specifying r / g / b / a as the targets of the replacement files. Specify the hyphen (-) for channels that are not being replaced.

    Info

    If you specify rb--, the G channel is replaced by the image file specified by replace_b.
    If you specify bg--, the R channel will be replaced by the image file specified by replace_b.
    If you specify rga-, the B channel will be replaced by the image file specified by replace_a.


    If only the images files specified by replace_r and replace_b exist, specify the attribute in the following way to use the BC5 format.
    alternative_format_rb="unorm_bc5,r0g1,,,rb--"

    When combining channels and converting the image to ASTC format, you might achieve higher quality by using which channels to replace to make the format Luminance + Alpha (for example, rrrg / rrrb / gggb).


    Use the normal format, comp_sel, hint, and linear attributes to specify values when all replacement files are present. Do not write alternative_format_??? when replacement files all exist (an error will occur).


  • <preset_overrides> [0 to 1]
    The parent element for the <preset_override> elements.

  • <preset_override> [0 and greater]
    Defines the preset overwrite.
    Specify this element when partially overwriting the <preset> element options.
    Additional options that were not previously specified with the <preset> element can be specified.
    These options are included as <preset_override> elements under the <preset_overrides> element.
    An error occurs if the <preset> with the name specified in <preset_override> does not exist.
    If an option with no parameters is specified in the <preset> element, it can be negated by specifying none for the attribute value.
    To negate the resize_w、resize_h setting in <preset>, specify resize_w="100%" resize_h="100%".
    If multiple <preset_override> elements have the same name, only the last element is applied.
    When inheriting preset definitions, the <preset_override> for the inheritance source is also applied in the inherited presets.

    Attribute Name Description
    name Specifies the name of the preset definition to overwrite.
    3D texture converter options Uses the same format as the 3D texture converter options in the <preset> element.
    alternative_format_??? Describes in a format similar to the <preset> element.
  • <preset_assigns> [0 to 1]
    The parent element for the <preset_assign> elements.

  • <preset_assign> [0 and greater]
    Defines preset assignments.
    Preset assignments are the rules that describe which presets to apply to image files with particular names.
    They are included as <preset_assign> elements under the <preset_assigns> element.

    For example, to apply the preset definition named Albedo1 to files with names ending in Alb (such as imageAlb.tga), you would use code like the following.
    file_name=".*Alb" preset_name="Albedo1"

    If the input file matches the file_name attribute of multiple <preset_assign> elements, the last one defined takes precedence.
    An error occurs if the input file does not match the file_name attribute of any of the <preset_assign> elements.

    Attribute NameDescription
    file_nameSpecifies the filename (not including the filename extension) for sorting through the presets. You can use regular expressions.
    extension

    Specifies the filename extension (not including the dot) of the image files to which the preset is applied. (Available in NintendoSDK 10.0.0 and later.)
    You can specify multiple extensions by separating them by commas.
    If the extension attribute is not specified or if the null character has been specified, the preset is applied to image files of all extensions.
    Note that if you code a <preset_assign> element with specified extensions followed by a <preset_assign> element that has the same file_name attribute but no specified extensions, only this latter element is valid.

    preset_name

    Specifies the name of the preset definition to apply.

    If the preset_name attribute is the null character, that image file is not converted.


    If multiple preset definition names are specified and separated by commas, all of the presets are applied. (For more information, see Example of Conversion to Multiple Texture Intermediate Files From a Single Input File.)
    When applying multiple presets, specify the output option in the preset definitions to prevent files in the same path from being exported multiple times.

    channel_packing

    When converting multiple input files into a single output file, set true to convert once for merging channels, and false to display an error.
    If this attribute is omitted, the default value is false.
    Even if true is specified, an error occurs if multiple input files with different presets are included in a single output file.
    When true is specified, the file specified by the replace_? attribute of the <preset> element is the original file (src_path attribute of the <create> element) for the texture intermediate file. (If multiple replace_? attributes are specified, the order of priority is replace_r > replace_g > replace_b > replace_a).

    minimum_number_only

    If the same preset is being applied and multiple input files all have the same filename prefix, specify true to convert only the file whose filename ends with the smallest number, or false to convert all of the files. (This feature is in Nintendo SDK 9.1.0 and later.)
    Use this attribute when converting images to 3D textures or array textures and the number for the first file is not fixed.
    You cannot specify true for both the minimum_number_only attribute and the channel_packing attribute.

  • <pre_convert_command> [0 to 1]
    <post_convert_command> [0 to 1]
    These elements contain the settings for commands executed before and after conversion.
    The path to the text file that describes the output file paths in each row (output list file) (UTF-8 BOM) is passed as the argument to the specified command.
    If commands are defined in multiple preset files, only the command that was defined last is executed.

    Attribute Name Description
    path

    Specifies the file path to the command to run before or after conversion.

    This path may include environment variables surrounded by percent signs (%).
    arguments_before

    Specifies any arguments to be inserted before the path to the output list file in the command argument.
    Multiple arguments can be specified when separated by spaces.

    This attribute does not need to be described if there are no arguments to insert.
    arguments_after Specifies any arguments to be added after the path to the output list file in the command argument.

    Multiple arguments can be specified when separated by spaces.

    This attribute does not need to be described if there are no arguments to add.


Environment Variables Usable in Preset Files

In addition to user-defined environment variables, the following environment variables also can be used for attribute values in the <output_folder>, <preset>, <preset_override> and <global_option> elements.

Environment Variables Description
%INPUT_FILE_PATH% The absolute path for the input file.
%INPUT_FILE_NAME% The input filename, excluding a filename extension.
%INPUT_FILE_EXT% The extension for the input file.
%INPUT_FILE_FOLDER% The absolute path to the folder containing the input file.
%INPUT_FILE_FOLDER_NAME% The name of the folder containing the input file.

%INPUT_FILE_DEPTH%

The number of files, if the name of the input file ends with a serialized number.

For example, if the input file is named image1.tga but the same folder also contains files named image2.tga and image3.tga, %INPUT_FILE_DEPTH% would be set to 3.

To convert image files to 2D array textures, specify dimension="2d_array" depth="%INPUT_FILE_DEPTH%" in the <preset> element.
If the value of the discrete_number attribute of the <preset> element is true, the value of this environment variable is taken as the number of files that have the same prefix as the input filename and that end in numbers greater than or equal to the input file. (The numbers at the end are counted even if they are not in numerical sequence.) (This feature is in Nintendo SDK 9.1.0 and later.)
%OUTPUT_FILE_PATH% The absolute path for the output file.
%OUTPUT_FILE_NAME% The output filename, excluding a filename extension.
%OUTPUT_FILE_EXT% The filename extension for the output file.
%OUTPUT_FILE_FOLDER% The absolute path for the folder storing the output file.
%OUTPUT_FILE_FOLDER_NAME% The name of the folder storing the output file.
%MATCH_1%

If you are using regular expressions enclosed in parentheses in the file_name attribute of the <preset_assign> element, this is the string that matches the first of those regular expressions.

For example, to export FTX files with names obtained by deleting the suffixes from the input filenames, you could specify file_name="(.*)Alb" in the <preset_assign> element and specify output="%MATCH_1%.ftxb" in the <preset> element.

%MATCH_2%

If you are using regular expressions enclosed in parentheses in the file_name attribute of the <preset_assign> element, this is the string that matches the second of those regular expressions.

Same for %MATCH_3% and so on.

These environment variables related to the output file can only be used in the <preset> element.

Sample Preset Files

Examples of Using Each Element

<texture_convert_preset version="1.0.0">
    <import path="%COMMON_PRESETS%/common.ntcp" /> <!-- COMMON_PRESETS is a user-defined environment variable. -->
    <output_folder path="../textures" />
    <pre_convert_command path="%SCRIPTS%/PreConvert.bat" /> <!-- SCRIPTS is a user-defined environment variable. -->
    <post_convert_command path="%SCRIPTS%/PostConvert.bat" />
    <global_option weighted_compress="false" mip_gen_filter="linear" />
    <presets>
        <preset name="Albedo1" format="srgb_bc1" comp_sel="rgba" hint="albedo" />
        <preset name="Normal1" format="unorm_bc5" comp_sel="rg01" hint="normal" />
        <preset name="Mixed1" format="srgb_bc3" comp_sel="rgba" hint="mixed" replace_a="%INPUT_FILE_NAME%.A.%INPUT_FILE_EXT%" />
        <preset name="EmissionHdr1" format="ufloat_bc6" hint="emission" />
        <preset name="Test1" format="unorm_bc4" hint="test1" output="%MATCH_1%.ftxb" /> <!-- Export with the name obtained from deleting the suffix. -->
    </presets>
    <preset_overrides>
        <preset_override name="Normal1" resize_w="50%" />
    </preset_overrides>
    <preset_assigns>
        <preset_assign file_name=".*Alb" preset_name="Albedo1" />
        <preset_assign file_name=".*Nrm" preset_name="Normal1" />
        <preset_assign file_name=".*Mix" preset_name="Mixed1" />
        <preset_assign file_name=".*\.[RGBA]" preset_name="" /> <!-- Suppress conversion of files for switching the channels -->
        <preset_assign file_name=".*Ems" extension="exr" preset_name="EmissionHdr1" /> <!-- Apply preset only to EXR files. -->
        <preset_assign file_name="(.*)Test1" preset_name="Test1" /> <!--Use regular expression surrounded by parentheses. -->
    </preset_assigns>

</texture_convert_preset>

Example of Conversion to Multiple Texture Intermediate Files From a Single Input File

<texture_convert_preset version="1.0.0">
    <output_folder path="../textures" />
    <presets>
        <preset name="Albedo_L" format="srgb_bc7" hint="albedo" />
        <preset name="Albedo_M" base_name="Albedo_L" resize_w="50%" output="%INPUT_FILE_NAME%_M.ftxb" />
        <preset name="Albedo_S" base_name="Albedo_L" resize_w="25%" output="%INPUT_FILE_NAME%_S.ftxb" />
        <!-- Albedo_M and Albedo_S inherit the format and hint of Albedo_L. -->
    </presets>
    <preset_assigns>
        <!-- Changes the resolution and creates multiple texture intermediate files. -->
        <preset_assign file_name=".*Alb" preset_name="Albedo_L,Albedo_M,Albedo_S" />
    </preset_assigns>
</texture_convert_preset>

Example of Combining Multiple Input File Channels Into One Output File

<texture_convert_preset version="1.0.0">
    <output_folder path="../textures" />
    <presets>
        <preset name="Packed1" format="unorm_bc7" comp_sel="rgba" hint="packed" output="%MATCH_1%.ftxb"
            alternative_format_rgb="unorm_bc1,rgb1"
            alternative_format_rg="unorm_bc5,rg01"
            alternative_format_rb="unorm_bc5,r0g1,,,rb--"
            alternative_format_r="unorm_bc4,rrr1"
            replace_r="%MATCH_1%.R.%INPUT_FILE_EXT%"
            replace_g="%MATCH_1%.G.%INPUT_FILE_EXT%"
            replace_b="%MATCH_1%.B.%INPUT_FILE_EXT%"
            replace_a="%MATCH_1%.A.%INPUT_FILE_EXT%"
        />
    </presets>
    <preset_assigns>
        <preset_assign file_name="(.*)\.(R|G|B|A)" preset_name="Packed1" channel_packing="true" />
    </preset_assigns>
</texture_convert_preset>

The RGBA channels are all replaced, so a image1.ftxb (unorm_bc7 format) file with the same content is saved even if image1.R.tga, image1.G.tga, image1.B.tga, or image1.A.tga is specified for the argument in 3dTexturePresetConverter.bat.
If both image1.R.tga and image1.G.tga are specified, an error occurs unless the channel_packing attribute of the <preset_assign> element is true
If image1.A.tga does not exist, the output uses the unorm_bc1 format specified by the alternative_format_rgb attribute.
If image1.R.tga does not exist, the alternative_format_gba attribute is not written and an error occurs.
If image1.G.tga and image1.A.tga do not exist, the alternative_format_rb attribute is written with rb-- specified for which channels to replace. So the channel that will be replaced by image1.B.tga is changed to the G channel. Then, by specifying r0g1 for the selected components, the G channel is applied to the B channel.
