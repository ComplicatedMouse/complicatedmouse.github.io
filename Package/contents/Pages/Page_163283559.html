1. Performing Reduction Using the Default Parameter

Try polygon reduction using the orthodox parameter (called the default parameter or just default), which can be applied to any type of model. If you get a good result with the default parameter, that is great, but in most cases the purpose of using the default is to detect problems.


Note: When there are multiple UV atlases, UV coordinates are always considered, even when reduction is executed using the default parameter.

Default Parameter
"--target-rate-lod-level1 0.5 --target-rate-lod-level2 0.25 --property-flag Position --optimize-position-policy OptimalOnSpace"

Use the following batch file.

%NINTENDO_SDK_ROOT%\Samples\Sources\Tools\3dTools\3dIntermediateFileOptimizer\PolygonReductionTutorial \ExecuteDefaultParameter.bat

This parameter has the following meaning.

  • Create LOD models with reduction rates of 50 percent and 25 percent for LOD levels 1 and 2, respectively.
    --target-rate-lod-level1 0.5 --target-rate-lod-level2 0.25
  • Run a reduction considering only the vertex positions.
    --property-flag Position.
  • Run a reduction that minimizes the change in volume from the original model using processing.
    --optimize-position-policy OptimalOnSpace.
