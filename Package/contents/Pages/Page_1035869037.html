GPU Crash Hash

Introduction

This page describes the GPU crash hash feature that was added in NX Add-On 11.0.0.

For more information about dump features when the GPU crashes, see Dump Feature During GPU Crash.

In addition, for stopping applications when the GPU crashes, see Automatically Starting the System Thread in nnMain() and Library Initialization.

About GPU Crash Hash

When a GPU crash occurs, the GPU error content is recorded and output as a dump. This error content contains the type of GPU error, detailed information for each type of GPU error, and related GPU register or memory content.

The GPU crash hash is a 32-bit hash value obtained by extracting the aggregation information from this GPU crash dump. This hash value is normally sent to the server as part of the crash report, and can be used for aggregation.

The items extracted from the GPU crash dump are those attached to the GPU Error Guide in Manual.

Different values are used for each GPU Exception Type.

Basically, the items displayed in "Graphics Exception Summary" when analyzing NxgcdFormatter are used, but this also includes register values and addresses, which are inappropriate for aggregation, so not all of the values are used.

Items Used in the GPU Crash Hash

The dump that occurs when the GPU crashes comprises multiple blocks. The GPU crash hash is generated from the values of the following items in these blocks.

Blocks Members
gcd_gr_isr_data_block intr
exception
chid
offset
class_num
class_error
gcd_fifo_isr_data_block intr
chid
engine_id
engine_status
gcd_fifo_mmu_fault_data_block faultInfo
gcd_fifo_pbdma_block intr0
intr1

For these values, the blocks and members that are set differ depending on GPU Exception Type, and are zero-filled if not set. The GPU crash hash always generates a hash value from all of these values, regardless of the GPU Exception Type.
