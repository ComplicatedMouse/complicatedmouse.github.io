Introduction

Purpose of This Documentation

This document describes the zlib library included in NintendoSDK and how to use it.

About zlib

zlib is an open-source library for data compression and decompression using the deflate algorithm. It is distributed at http://www.zlib.net/.

Intended Audience

This document is intended for developers of applications and middleware using NintendoSDK.

zlib in NintendoSDK

NintendoSDK contains a version of zlib that has been modified for NintendoSDK.
The following zlib versions are included.

  • NX Add-On 5.1.0 and earlier: 1.2.8
  • NX Add-On 5.2.0 and later: 1.2.11

.

Modifications

  • The version number has been changed (as per the terms of the license).
  • When Z_PREFIX is enabled, the prefix is changed from z to nnutilZlib.
    • However, Z_PREFIX is not defined at build time, so definitions with prefixes will result in linker errors.
  • fcntl.h and sys/types.h are not included.

API functions that begin with gz are not compiled. (gzip file access functions cannot be used.)

Provision by the util Library and the zlib API

The version of zlib in the NintendoSDK can be used in the following two ways.

Provision by the util Library

The Compress and Decompress functions of the nn::util library use zlib internally. You can easily use zlib by using these APIs.

To use these APIs, include the following header files.

  • nn/util/util_Compression.h
  • nn/util/util_Decompression.h
  • nn/util/util_StreamingDecompression.h

Provision by the zlib API

Developers who understand the zlib API can use it directly to use the library.

If you intend to directly use the zlib APIs, make sure that you include zlib.h.

Features Provided by Directly Using the zlib API

When you directly use the zlib API, you use the zlib API interface to make use of the features of the zlib library.

However, note the following restrictions and considerations.

Restrictions

  • The gzip file access functions cannot be used.

Notes

  • The version of zlib in NintendoSDK is compiled in a way that does not make use of the Windows API.
  • A linker error will occur if the FAR macro is defined as anything but empty before zlib.h is included.
    • For example, this error occurs if the FAR macro was defined as far by nn_Windows.h.
    • For this reason, do not include zlib.h and nn_Windows.h from the same file.
