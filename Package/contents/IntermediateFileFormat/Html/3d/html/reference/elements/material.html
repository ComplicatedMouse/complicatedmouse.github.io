<material>

Description

Element that defines materials.

Parent Elements

  • <shader_graph>
  • <material_array>: A collection of <material> elements.
  • <nw4f_3dif>: Root element for the 3D intermediate file format.

Attributes

Name Types and Values Description DCC
index int
0 – 65,534		 Represents an index of the array.
Serving as a hint to improve the readability of the file, this value is ignored when loaded by a tool.
Configurable
name string
[0-9A-Za-z\-\._]*		 The name of the material.
It is paired with the mat_name attribute of the reference source <original_material>, <shader_param_mat_anim>, <original_material_anim>, <tex_pattern_mat_anim>, or <mat_vis_mat_anim>.
Configurable
version enum
4.2.0		 This attribute can be omitted.
 -

Description

The elements appear in the following order. Numeric values inside square brackets ([ ]) indicate the number of times the element may appear.

  • <process_log_array> [0 or 1]: A collection of <process_log> elements.
  • <material_info> [1]: Element that represents information related to <material>.
  • <material_reference_array> [0 or 1]: Array element of <material_reference>.
  • <texture_search_path_array> [0 or 1]: Array element of <search_path> used for texture search.
  • <material_search_path_array> [0 or 1]: Array element of <search_path> used for material search.
  • <sampler_array> [0 or 1]: A collection of <sampler> elements.
  • <shader_assign> [0 or 1]: Specifies the shader to assign to the material.
  • <user_data_array> [0 or 1]: A collection of <user_data> elements.
  • <stream_array> [0 or 1]: A collection of <stream> elements.
  • <comment> [0 or 1]: Element that stores metadata such as editing comments.
  • <tool_data> [0 or 1]: Element that stores data for NintendoWare tools.
  • <user_tool_data> [0 or 1]: Element that stores data for original user tools.
