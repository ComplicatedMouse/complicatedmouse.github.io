<camera_anim>

Description

Element used to animate user-defined cameras.

Parent Elements

  • <camera_anim_array>: A collection of <camera_anim> elements.

Attributes

Name Types and Values Description DCC
index int
0 – 65,534		 Represents an index of the array.
Serving as a hint to improve the readability of the file, this value is ignored when loaded by a tool.
Configurable
camera_name string
[0-9A-Za-z\-\._]+		 The name of the user-defined camera to animate.
Configurable
frame_count int
0 – 16,777,215		 Frame count of the animation.
Configurable
loop bool
true or false		 Specify true if loop playback is used on the animation.
 Configurable
rotate_mode enum
aim
euler_zxy		 Specifies the rotation mode of a user-defined camera.
aim Rotates the camera using its aim point (aim) and twist (twist).
euler_zxy Rotates the camera by rotating ZXY axes (rotate).

 Configurable
projection_mode enum
ortho
persp		 Specifies the projection mode of a user-defined camera.
ortho Makes an orthogonal projection using aspect, near, far, and ortho_height.
persp Makes a perspective projection using aspect, near, far, and persp_fovy.

 Configurable

Description

target always includes a <camera_anim_target> element giving values for position_x, position_y, position_z, aspect, near, and far.
If rotate_mode is aim, target includes a <camera_anim_target> element that gives values for aim_x, aim_y, aim_z, and twist.
If rotate_mode is euler, target includes a <camera_anim_target> element that gives value for rotate_x, rotate_y, and rotate_z.
If projection_mode is orthor, target includes a <camera_anim_target> element that gives values for ortho_height.
If projection_mode is persp, target includes a <camera_anim_target> element that gives values for persp_fovy.
<camera_anim_target> is sorted in the order that targets are defined.

The elements appear in the following order. Numeric values inside square brackets ([ ]) indicate the number of times the element may appear.

  • <camera_anim_target> [0 – 15]: Element that animates each parameter of a user-defined camera.
  • <user_data_array> [0 or 1]: A collection of <user_data> elements.
  • <comment> [0 or 1]: Element that stores metadata such as editing comments.
  • <tool_data> [0 or 1]: Element that stores data for NintendoWare tools.
  • <user_tool_data> [0 or 1]: Element that stores data for original user tools.
