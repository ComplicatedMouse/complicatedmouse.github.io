<mesh>

Description

This element represents vertex indices used to define a polygon.

Parent Elements

  • <mesh_array>: A collection of <mesh> elements.

Attributes

Name Types and Values Description DCC
index int
0 – 254		 Represents an index of the array.
Serving as a hint to improve the readability of the file, this value is ignored when loaded by a tool.
Configurable
mode enum
triangles
triangle_strip		 Specifies the vertex index primitive recorded in stream.
When exporting from a DCC tool, always specify triangles.
triangles Indicates that vertex indices are specified using triangles.
triangle_strip Indicates that vertex indices are specified using triangle strips.

 Fixed
primitive_topology enum
triangle_list
triangle_strip
triangle_list_adjacency
patch_list		 Specifies the PrimitiveTopology to use with the runtime. Corresponds to the nn::gfx::PrimitiveTopology type.
If mode is triangles, you can specify triangle_list, triangle_list_adjacency, and patch_list.
If mode is triangle_strip, you can specify triangle_strip.
When exporting from a digital content creation tool, always specify triangle_list.
triangle_list Indicates that vertex indices are specified using triangles.
triangle_strip Indicates that vertex indices are specified using triangle strips.
triangle_list_adjacency Indicates that vertex indices are specified using triangles with adjacent data.
3dBinaryConverter overwrites the index buffer as appropriate.
patch_list A list of patches.

 Fixed
quantize_type enum
none
uint_16
uint_32		 Specifies the vertex index type.
When exporting from a DCC tool, always specify none.
Specify uint_16 or uint_32 when optimizing polygons using the intermediate file optimizer.
none Quantization precision is not specified.
Values are handled as u32 types if binarization is performed.
uint_16 Specifies quantization using 16-bit integers.
uint_32 Specifies quantization using 32-bit integers.

 Fixed
count int
1 – MAX		 Specifies the number of vertex indices. Configurable
stream_index int
0 – MAX		 The index of the <stream> used to store actual vertex index data.
Represents a polygon rotating counterclockwise as seen from the view point.
Configurable

Description

The elements appear in the following order. Numeric values inside square brackets ([ ]) indicate the number of times the element may appear.

  • <submesh_array> [1]: A collection of <submesh> elements.
