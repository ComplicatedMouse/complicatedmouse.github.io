Changes in NintendoSDK NX Addon 14.1.0 (based on Generic 14.1.0)

NintendoSDK

Specification change that breaks your code

  • *14-1-0S01: (EC) Deleted the ec_Result.http.h header file.
  • *14-1-0S02: (FS) Delete operations on the root of the drive in the host file system are now precondition violations.

Added Features

  • *14-1-0S03: (TOOL) Added a setting to the NMETA file for the target age for applications for consoles for China.
  • 14-1-0S04: (AOC) Changed the default setting for the policy for changes to available content to Stable.
  • 14-1-0S05: (AOC) In light of the easing of restrictions on user accounts, a setting was added to the NMETA file for setting the policy for changes to available content.
  • 14-1-0S06: (HID) Added a feature for connecting the Nintendo Switch Pro Controller via a USB connection in the PC environment.
  • 14-1-0S07: (INIT) Added a function that can be used inside nninitStartup().
  • 14-1-0S08: (NVN) New nvn Features
  • 14-1-0S09: (TOOL) Added a feature for creating an NSP file for secure startup from an NSP file that merges a compacted application and a patch.
  • 14-1-0S10: (TOOL) Added a feature to display library applets started from the application in remote video.
  • 14-1-0S11: (TOOL) Added a feature to Target Manager 2 for adjusting the remote video volume.
  • 14-1-0S12: (TOOL) Added a hardware management log feature to Target Manager 2.
  • 14-1-0S13: (TOOL) Added a link to the manual page in the SDK environment checker.
  • 14-1-0S14: (TOOL) Added DumpViewer as a tool for listing NXDMP files.
  • 14-1-0S15: (TOOL) Added the application ID to the target log.

Improvements

  • *14-1-0S16: (BuildSystem) Changed the supported version of MSVC v142 to v14.29-16.11.
  • *14-1-0S17: (Compiler) Notice of removal of the ld and gold linkers.
  • *14-1-0S18: (TOOL) Changed the behavior when installing Target Manager to forcibly close any running instance of Target Manager ahead of time.
  • 14-1-0S19: (Compiler) Updated to Clang for NX 1.11.6.
  • 14-1-0S20: (CPU Profiler) Nintendo NX CPU Profiler Version 1.7
  • 14-1-0S21: (DevMenu) Added support to Install via HTTP > Automatic for deleting all history.
  • 14-1-0S22: (DevMenuCommand) Added support for launch parameters when using DevMenuCommand to start NSP and NSPD files.
  • 14-1-0S23: (FS) Improved the error message in FsAccessLogChecker version 41.9 when analysis results are entered.
  • 14-1-0S24: (FS) When nn::fs::CreateFile() fails, the file that was being created is now deleted in the host file system.
  • 14-1-0S25: (Sample) Added a way to exit the HidGesture sample.
  • 14-1-0S26: (Sample) Added support for C++14 features to the HidNpadIntegrate sample.
  • 14-1-0S27: (SSL) OpenSSL has been updated to version 1.1.1l.
  • 14-1-0S28: (TOOL) Added a feature to Target Manager 2 for exporting a ZIP file that includes the collection of dump files for bug reports and the log files.
  • 14-1-0S29: (TOOL) Added a feature to Target Manager 2 for restoring the target name to the default value in the SDEV settings and the USB development hardware settings.
  • 14-1-0S30: (TOOL) Added support to Target Manager 2 for displaying an icon for unsupported targets.
  • 14-1-0S31: (TOOL) Changed the default addr2line in the crash report retrieval tool to llvm-addr2line.exe.
  • 14-1-0S32: (TOOL) In AuthoringEditor, the list of middleware no longer shows NintendoSdk_BuildTarget-64Bit or NintendoSdk_BuildTarget-32Bit.
  • 14-1-0S33: (TOOL) The analyze-patch command of AuthoringTool now shows whether application compaction has been performed.
  • 14-1-0S34: (TOOL) USB targets are now found if newly recognized while searching with the detect-target subcommand of ControlTarget in the htc-gen2 environment.

Bug Fixes

  • 14-1-0S35: (DevMenu) Fixed a bug in DevMenu where a fatal error occurred when content in an application was deleted.
  • 14-1-0S36: (DevMenu) Fixed a bug where a fatal error occurred in the DevMenu environment when an application was started and ended from the host PC.
  • 14-1-0S37: (FS) Fixed a bug in FsAccessLogChecker version 41.10 where an incorrect warning message was added to the nn::fs::IsMounted() access log prior to mounting.
  • 14-1-0S38: (FS) Fixed a bug where an access log was not generated when nn::fs::ExtendSaveData() was called during mounting.
  • 14-1-0S39: (IRSENSOR) Corrected the omission of the irsensor library's pointing mode functions from the API Reference.
  • 14-1-0S40: (Sample) Fixed a bug where Visual Studio 2019 could not build the Release x64 configuration of the TmRemoteVideo sample.
  • 14-1-0S41: (TOOL) Corrected typos in the error messages of RunOnTarget and ControlTarget.
  • 14-1-0S42: (TOOL) Fixed a bug where the wrong symbols were displayed when the crash report retrieval tool resolved symbols.
  • 14-1-0S43: (TOOL) Fixed a display bug that occurred when Target Manager 2 was used in a high DPI environment.
  • 14-1-0S44: (TOOL) Fixed the error that occurred when the remote video API was called right after nn::tm::CloseTargetVideoStream().

NintendoWare

Added Features

  • 14-1-0W01: (3DTool) Added the bound-model-path and shader-path options to the assign-original-curve subcommand of the assign utility.
  • 14-1-0W02: (3DTool) Added the --continue-on-error option to the Shader Variation Generator.
  • 14-1-0W03: (3DTool) Added the --submesh-to-shape and --shape-to-submesh options to the Intermediate File Optimizer.
  • 14-1-0W04: (3DTool) The Photoshop plug-in now supports Photoshop 2022 (version 23).
  • 14-1-0W05: (Layout-UI2D) Added a feature to LayoutViewer for capturing screenshots of all frames of a specific animation tag.
  • 14-1-0W06: (Layout-UI2D) Added a feature to LayoutViewer for saving screenshots as PNG files.

Improvements

  • 14-1-0W07: (3DTool) Improved the quantization analysis feature of the Intermediate File Optimizer to maintain the quantization type for vertex attributes that have a hint that is not the default hint.
  • 14-1-0W08: (LayoutTool) Added a feature to LayoutEditor for specifying the output folder when performing batch synchronization using a command-line argument.
  • 14-1-0W09: (LayoutTool) Added a LayoutEditor command-line option for specifying the animation segment tag to evaluate when capturing screenshots.

Bug Fixes

  • 14-1-0W10: (3D-G3D) Fixed a bug where the Shader Variation Generator did not work correctly even if a folder was specified in the --shader-definition option.
  • 14-1-0W11: (3DTool) Fixed a bug in 3DEditor where the parent material's reference setting was not applied correctly if set to ForceRefer or Override.
  • 14-1-0W12: (3DTool) Fixed a bug in the assign utility where unnecessary elements were added if the apply-parent-material subcommand was executed multiple times on the same model intermediate file.
  • 14-1-0W13: (3DTool) Fixed a bug in the Intermediate File Optimizer where merging of LOD models failed for model intermediate files that reference external materials.
  • 14-1-0W14: (3DTool) Fixed a bug in the shape compression feature of the Intermediate File Optimizer where the quantize_type for <mesh> was not reset to none.
  • 14-1-0W15: (3DTool) Fixed a bug where 3DEditor forcibly closed when copying a group containing hidden RenderInfo.
  • 14-1-0W16: (3DTool) Fixed a bug where the 3D Texture Preset Converter did not work correctly if the path specified for the project_root attribute ended with a backslash (\).
  • 14-1-0W17: (Layout-UI2D) Fixed a bug where the interior was filled with black if the rounded corner radius was smaller than the border size.

Changes in NintendoSDK NX Addon 13.3.2 (based on Generic 13.3.2)

NintendoSDK

Added Features

  • *13-3-2S01: (TOOL) Added a setting to the NMETA file for the target age for applications for consoles for China.
  • 13-3-2S02: (AOC) In light of the easing of restrictions on user accounts, a setting was added to the NMETA file for setting the policy for changes to available content.
  • 13-3-2S03: (TOOL) Added a feature for creating an NSP file for secure startup from an NSP file that merges a compacted application and a patch.
  • 13-3-2S04: (WEB) Added a feature to WebAppletCaller for displaying the settings.

Improvements

  • 13-3-2S05: (TOOL) Added support in AuthoringEditor for displaying the compression ratio of NSP files for which the ROM compression feature was enabled.
  • 13-3-2S06: (TOOL) Changed the default addr2line in the crash report retrieval tool to llvm-addr2line.exe.
  • 13-3-2S07: (TOOL) In AuthoringEditor, the list of middleware no longer shows NintendoSdk_BuildTarget-64Bit or NintendoSdk_BuildTarget-32Bit.
  • 13-3-2S08: (TOOL) The mergensp command in AuthoringTool now supports all compacted applications created in any SDK version.

Bug Fixes

  • 13-3-2S09: (GFX) Fixed a bug where eglSetBlobCacheFuncsANDROID() did not work correctly.
  • 13-3-2S10: (TOOL) Fixed a bug where DirectXShaderCompiler could not compile files larger than 64 KiB.
  • 13-3-2S11: (TOOL) Fixed a bug where the wrong symbols were displayed when the crash report retrieval tool resolved symbols.
